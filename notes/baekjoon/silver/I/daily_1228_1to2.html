    문제 6467 — Prime Cuts

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Prime Cuts (BOJ 6467)는
    1부터 N까지의 “소수 리스트(문제에서는 1도 포함)”를 만든 뒤, 그 리스트의 가운데 부분만 규칙에 맞춰 잘라 출력하는 문제다.

    • 리스트는 1과 N 이하의 모든 소수를 오름차순으로 담는다. (이 코드도 1을 따로 넣는다)
    • 리스트 길이를 size라고 할 때,
      • size가 짝수면 가운데에서 2*C개
      • size가 홀수면 가운데에서 2*C - 1개
      를 출력한다.
    • 만약 잘라낼 개수(slice)가 리스트 전체 크기보다 크면, 리스트 전체를 출력한다.
    • 출력 형식은 N C: 뒤에 선택된 수들을 공백을 두고 나열하고, 각 케이스마다 빈 줄을 한 줄 추가한다.

    전략 개요

    • 에라토스테네스의 체로 N 이하 소수를 구한다.
    • 문제 규칙에 맞게 리스트 맨 앞에 1을 추가한다.
    • 리스트 길이 size로부터 출력해야 할 개수 slice를 계산한다.
    • slice > size이면 전체 출력.
    • 그 외에는 가운데에서 slice개가 되도록 시작 인덱스 start = (size - slice) / 2 를 구해 연속으로 출력한다.

    코드가 하는 일

    • 1) 입력 처리 (EOF까지 반복) 
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
          int N = Integer.parseInt(st.nextToken());
          int C = Integer.parseInt(st.nextToken());
      • 여러 테스트케이스가 줄 단위로 주어지므로 EOF까지 반복한다.
    • 2) 소수 판별용 배열 준비 + 1 포함 리스트 시작 
      boolean[] chk = new boolean[N + 1];
      Arrays.fill(chk, true);
      
      List<Integer> list = new ArrayList<>();
      
      chk[0] = false;
      chk[1] = false;
      
      list.add(1);
      • chk[i]는 i가 소수일 가능성(true/false)을 의미한다.
      • 0과 1은 소수가 아니므로 false 처리한다.
      • 문제 규칙에 따라 소수 리스트에 1을 포함해야 하므로 list.add(1)로 시작한다.
    • 3) 에라토스테네스의 체로 N 이하 소수 찾기 
      for (int i = 2; i * i <= N; i++) {
          if (chk[i]) {
              for (int j = i * i; j <= N; j += i) {
                  chk[j] = false;
              }
          }
      }
      • 소수 i의 배수들을 모두 지우며(false) 소수만 남긴다.
      • i*i부터 시작하는 것은 중복 마킹을 줄이기 위한 표준 최적화다.
    • 4) 소수들을 리스트에 추가 
      for (int i = 2; i < chk.length; i++) {
          if (chk[i]) list.add(i);
      }
      • 2부터 N까지 true로 남아있는 값들을 리스트에 담는다.
      • 이 시점에 리스트는 [1, (N 이하 소수들...)] 오름차순이 된다.
    • 5) 출력 개수(slice) 계산 및 출력 범위 결정 
      int size = list.size();
      int slice = size % 2 == 0 ? 2 * C : 2 * C - 1;
      
      sb.append(N).append(" ").append(C).append(": ");
      • 리스트가 짝수 길이면 2*C, 홀수 길이면 2*C - 1개를 출력한다.
      • 출력 헤더는 N C: 형태로 맞춘다.
    • 6) slice가 size보다 크면 전체 출력 
      if (slice > size) {
          for (int i : list) {
              sb.append(i).append(" ");
          }
      
          sb.append('\n').append('\n');
          continue;
      }
      • 원하는 개수가 전체보다 많으면, 문제 조건대로 리스트 전체를 출력한다.
      • 이 케이스는 출력 후 다음 테스트케이스로 넘어간다.
    • 7) 가운데 구간 시작점 계산 후 slice개 출력 
      int start = (size - slice) / 2;
      
      for (int i = 0; i < slice; i++) {
          sb.append(list.get(start + i)).append(" ");
      }
      
      sb.append('\n').append('\n');
      • 가운데에서 slice개를 뽑기 위해 시작 인덱스를 (size - slice) / 2로 잡는다.
      • 그 위치부터 연속으로 slice개를 출력하면 가운데 구간이 된다.
      • 각 테스트케이스 출력 뒤 빈 줄을 한 줄 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 에라토스테네스의 체: O(N log log N)
      • 리스트 구성 및 출력: O(N) (N ≤ 1000이라 충분히 빠름)
    • 공간 복잡도
      • 체 배열 chk와 소수 리스트를 저장하므로 O(N)

    문제 6474 — Palindromes

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Palindromes (BOJ 6474)는
    입력으로 주어진 문자열이 일반 회문(palindrome)인지, 거울 문자열(mirrored string)인지, 혹은 둘 다 만족하는 거울 회문(mirrored palindrome)인지 판별해 정해진 문장을 출력하는 문제다.

    • palindrome: 문자열을 뒤집어도 동일한 문자열
    • mirrored string: 문자열을 뒤집은 뒤 각 문자를 “거울 대응 문자”로 치환했을 때 원래 문자열과 같음
    • mirrored palindrome: 위 두 조건을 모두 만족
    • 입력은 EOF까지 여러 줄로 주어진다.
    • 출력은 문자열 + " -- " + 판정 문구, 그리고 빈 줄 1줄이 추가된다.

    전략 개요

    • 문제에서 제공한 “거울 대응” 규칙을 Map<Character, Character>으로 만든다.
    • 각 문자열에 대해
      • 앞/뒤 문자가 같은지 비교해서 palindrome 여부 판단
      • 뒤쪽 문자에 대한 거울 대응 문자를 찾아서 앞쪽 문자와 같은지 비교해 mirrored 여부 판단
    • 두 boolean 값(palindrome, mirrored) 조합으로 출력 문구를 결정한다.

    코드가 하는 일

    • 1) 거울 문자 매핑(Map) 구성 
      Map<Character, Character> map = new HashMap<>();
      map.put('E', '3');
      map.put('3', 'E');
      map.put('J', 'L');
      map.put('L', 'J');
      map.put('S', '2');
      map.put('2', 'S');
      map.put('Z', '5');
      map.put('5', 'Z');
      map.put('A', 'A');
      map.put('H', 'H');
      map.put('I', 'I');
      map.put('M', 'M');
      map.put('O', 'O');
      map.put('T', 'T');
      map.put('U', 'U');
      map.put('V', 'V');
      map.put('W', 'W');
      map.put('X', 'X');
      map.put('Y', 'Y');
      map.put('1', '1');
      map.put('8', '8');
      • 문제 표에 나온 “문자 ↔ 거울 문자” 관계를 그대로 저장한다.
      • 서로 바뀌는 쌍(E↔3, J↔L 등)은 양방향으로 모두 넣는다.
      • 자기 자신으로 대응되는 문자(A, H, I, ...)도 넣는다.
    • 2) 입력 문자열을 EOF까지 처리 
      String line;
      while ((line = br.readLine()) != null) {
          boolean palindrome = true;
          boolean mirrored = true;
      • 한 줄이 하나의 문자열이고, EOF까지 반복한다.
      • 각 문자열마다 두 조건을 각각 검사한다.
    • 3) palindrome 판별: 양끝에서 가운데로 비교 
      for (int i = 0; i < line.length(); i++) {
          if (i > line.length() - 1 - i) break;
          if (line.charAt(i) != line.charAt(line.length() - 1 - i)) {
              palindrome = false;
              break;
          }
      }
      • i와 len-1-i를 비교하며 회문인지 확인한다.
      • 한 번이라도 다르면 회문이 아니므로 false로 바꾸고 종료한다.
      • 가운데를 지나면 더 볼 필요가 없어 중단한다.
    • 4) mirrored 판별: 뒤쪽 문자의 거울값이 앞쪽 문자와 같은지 확인 
      for (int i = 0; i < line.length(); i++) {
          int backIdx = line.length() - 1 - i;
          Character c = map.get(line.charAt(backIdx));
      
          if (c == null || line.charAt(i) != c) {
              mirrored = false;
              break;
          }
      }
      • 거울 문자열 조건은 “뒤에서 i번째 문자를 거울 변환한 값”이 “앞에서 i번째 문자”와 같아야 한다.
      • 따라서 backIdx 위치 문자를 맵에서 변환한 뒤, line.charAt(i)와 비교한다.
      • 대응 문자가 없거나(null) 값이 다르면 mirrored가 아니다.
    • 5) 조건 조합에 따라 문구 출력 
      sb.append(line).append(" -- ");
      
      if (palindrome && mirrored) {
          sb.append("is a mirrored palindrome.").append('\n').append('\n');
          continue;
      } else if (palindrome) {
          sb.append("is a palindrome.").append('\n').append('\n');
      } else if (mirrored) {
          sb.append("is a mirrored string.").append('\n').append('\n');
      } else {
          sb.append("is not a palindrome.").append('\n').append('\n');
      }
      • 문제에서 요구한 출력 형식을 그대로 따른다.
      • 각 케이스마다 마지막에 빈 줄을 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 L이라 하면, palindrome 검사 O(L), mirrored 검사 O(L) → 전체 O(L).
      • L ≤ 20이라 매우 작다.
    • 공간 복잡도
      • 거울 매핑 테이블은 고정 크기 → O(1).
