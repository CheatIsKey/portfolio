    문제 1080 — 행렬

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    행렬 (BOJ 1080)은
    0과 1로 이루어진 행렬 A를 행렬 B로 바꾸기 위해, 3×3 부분 행렬을 뒤집는 연산을 최소 몇 번 해야 하는지 구하는 문제다.

    • 연산 1회는 선택한 3×3 영역의 모든 원소를 0 ↔ 1로 뒤집는다.
    • N 또는 M이 3보다 작으면 3×3 연산 자체가 불가능하다.
    • 가능한 경우에는 좌상단부터 비교하면서 “필요할 때만” 뒤집는 그리디가 정답이 된다.

    전략 개요

    • (i, j)를 3×3 뒤집기의 좌상단으로 둘 수 있는 범위(0..N-3, 0..M-3)를 순회한다.
    • 좌상단부터 보면서 A[i][j] != B[i][j]이면, 그 칸을 맞추기 위해서는 반드시 (i, j)에서 3×3 뒤집기를 해야 한다.
    • 따라서 차이가 나는 순간에만 flip(A, i, j)를 수행하고 횟수를 증가시킨다.
    • 전체 순회가 끝난 뒤 A와 B가 완전히 같으면 횟수를 출력하고, 아니면 -1을 출력한다.

    코드가 하는 일

    • 1) 입력으로 행렬 A, B 구성 
      int[][] A = new int[N][M];
      for (int i = 0; i < N; i++) {
          String s = br.readLine();
          for (int j = 0; j < M; j++) {
              A[i][j] = s.charAt(j) - '0';
          }
      }
      
      int[][] B = new int[N][M];
      for (int i = 0; i < N; i++) {
          String s = br.readLine();
          for (int j = 0; j < M; j++) {
              B[i][j] = s.charAt(j) - '0';
          }
      }
      • 문자열로 들어오는 0/1을 정수로 변환해 2차원 배열에 저장한다.
    • 2) 3×3 연산 자체가 불가능한 경우 처리 
      if (N < 3 || M < 3) {
          System.out.println(same(A, B) ? 0 : -1);
          return;
      }
      • N이나 M이 3보다 작으면 뒤집기 연산을 할 수 없다.
      • 이때는 처음부터 같으면 0, 다르면 -1이 된다.
    • 3) 좌상단부터 그리디로 뒤집기 수행 
      int cnt = 0;
      for (int i = 0; i <= N - 3; i++) {
          for (int j = 0; j <= M - 3; j++) {
              if (A[i][j] != B[i][j]) {
                  flip(A, i, j);
                  cnt++;
              }
          }
      }
      • (i, j) 위치를 “이제부터는 다시 건드릴 수 없는 좌상단”으로 생각한다.
      • 해당 칸이 다르면, 그 칸을 뒤집는 방법은 (i, j)에서 flip하는 것뿐이므로 즉시 수행한다.
      • 이 과정을 통해 좌상단부터 순서대로 맞춰 나간다.
    • 4) 최종 동일 여부 판단 후 출력 
      System.out.println(same(A, B) ? cnt : -1);
      • 모든 뒤집기가 끝난 뒤 A와 B가 같으면 cnt가 최솟값이다.
      • 끝까지 맞출 수 없으면 불가능이므로 -1.
    • 5) flip(): 3×3 부분 행렬 뒤집기 
      private static void flip(int[][] a, int i, int j) {
          for (int k = i; k < i + 3; k++) {
              for (int l = j; l < j + 3; l++) {
                  a[k][l] = 1 - a[k][l];
              }
          }
      }
      • 0 → 1, 1 → 0을 1 - 값으로 처리한다.
      • 좌상단 (i, j) 기준 3×3 영역을 정확히 뒤집는다.
    • 6) same(): 두 행렬이 완전히 같은지 검사 
      private static boolean same(int[][] a, int[][] b) {
          for (int i = 0; i < a.length; i++) {
              for (int j = 0; j < a[0].length; j++) {
                  if (a[i][j] != b[i][j]) return false;
              }
          }
          return true;
      }
      • 모든 칸이 같아야 true를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 좌상단 순회는 O(N*M).
      • 각 flip은 3×3(9칸)만 뒤집으므로 상수 시간.
      • 전체는 O(N*M)이며, N, M ≤ 50이라 충분히 빠르다.
    • 공간 복잡도
      • 행렬 A, B 저장에 O(N*M).
      • 추가로 큰 자료구조는 사용하지 않는다.

    문제 6474 — Palindromes

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Palindromes (BOJ 6474)는
    입력으로 주어진 문자열이 일반 회문(palindrome)인지, 거울 문자열(mirrored string)인지, 혹은 둘 다 만족하는 거울 회문(mirrored palindrome)인지 판별해 정해진 문장을 출력하는 문제다.

    • palindrome: 문자열을 뒤집어도 동일한 문자열
    • mirrored string: 문자열을 뒤집은 뒤 각 문자를 “거울 대응 문자”로 치환했을 때 원래 문자열과 같음
    • mirrored palindrome: 위 두 조건을 모두 만족
    • 입력은 EOF까지 여러 줄로 주어진다.
    • 출력은 문자열 + " -- " + 판정 문구, 그리고 빈 줄 1줄이 추가된다.

    전략 개요

    • 문제에서 제공한 “거울 대응” 규칙을 Map<Character, Character>으로 만든다.
    • 각 문자열에 대해
      • 앞/뒤 문자가 같은지 비교해서 palindrome 여부 판단
      • 뒤쪽 문자에 대한 거울 대응 문자를 찾아서 앞쪽 문자와 같은지 비교해 mirrored 여부 판단
    • 두 boolean 값(palindrome, mirrored) 조합으로 출력 문구를 결정한다.

    코드가 하는 일

    • 1) 거울 문자 매핑(Map) 구성 
      Map<Character, Character> map = new HashMap<>();
      map.put('E', '3');
      map.put('3', 'E');
      map.put('J', 'L');
      map.put('L', 'J');
      map.put('S', '2');
      map.put('2', 'S');
      map.put('Z', '5');
      map.put('5', 'Z');
      map.put('A', 'A');
      map.put('H', 'H');
      map.put('I', 'I');
      map.put('M', 'M');
      map.put('O', 'O');
      map.put('T', 'T');
      map.put('U', 'U');
      map.put('V', 'V');
      map.put('W', 'W');
      map.put('X', 'X');
      map.put('Y', 'Y');
      map.put('1', '1');
      map.put('8', '8');
      • 문제 표에 나온 “문자 ↔ 거울 문자” 관계를 그대로 저장한다.
      • 서로 바뀌는 쌍(E↔3, J↔L 등)은 양방향으로 모두 넣는다.
      • 자기 자신으로 대응되는 문자(A, H, I, ...)도 넣는다.
    • 2) 입력 문자열을 EOF까지 처리 
      String line;
      while ((line = br.readLine()) != null) {
          boolean palindrome = true;
          boolean mirrored = true;
      • 한 줄이 하나의 문자열이고, EOF까지 반복한다.
      • 각 문자열마다 두 조건을 각각 검사한다.
    • 3) palindrome 판별: 양끝에서 가운데로 비교 
      for (int i = 0; i < line.length(); i++) {
          if (i > line.length() - 1 - i) break;
          if (line.charAt(i) != line.charAt(line.length() - 1 - i)) {
              palindrome = false;
              break;
          }
      }
      • i와 len-1-i를 비교하며 회문인지 확인한다.
      • 한 번이라도 다르면 회문이 아니므로 false로 바꾸고 종료한다.
      • 가운데를 지나면 더 볼 필요가 없어 중단한다.
    • 4) mirrored 판별: 뒤쪽 문자의 거울값이 앞쪽 문자와 같은지 확인 
      for (int i = 0; i < line.length(); i++) {
          int backIdx = line.length() - 1 - i;
          Character c = map.get(line.charAt(backIdx));
      
          if (c == null || line.charAt(i) != c) {
              mirrored = false;
              break;
          }
      }
      • 거울 문자열 조건은 “뒤에서 i번째 문자를 거울 변환한 값”이 “앞에서 i번째 문자”와 같아야 한다.
      • 따라서 backIdx 위치 문자를 맵에서 변환한 뒤, line.charAt(i)와 비교한다.
      • 대응 문자가 없거나(null) 값이 다르면 mirrored가 아니다.
    • 5) 조건 조합에 따라 문구 출력 
      sb.append(line).append(" -- ");
      
      if (palindrome && mirrored) {
          sb.append("is a mirrored palindrome.").append('\n').append('\n');
          continue;
      } else if (palindrome) {
          sb.append("is a palindrome.").append('\n').append('\n');
      } else if (mirrored) {
          sb.append("is a mirrored string.").append('\n').append('\n');
      } else {
          sb.append("is not a palindrome.").append('\n').append('\n');
      }
      • 문제에서 요구한 출력 형식을 그대로 따른다.
      • 각 케이스마다 마지막에 빈 줄을 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 L이라 하면, palindrome 검사 O(L), mirrored 검사 O(L) → 전체 O(L).
      • L ≤ 20이라 매우 작다.
    • 공간 복잡도
      • 거울 매핑 테이블은 고정 크기 → O(1).
