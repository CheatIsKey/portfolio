    문제 1080 — 행렬

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    행렬 (BOJ 1080)은
    0과 1로 이루어진 행렬 A를 행렬 B로 바꾸기 위해, 3×3 부분 행렬을 뒤집는 연산을 최소 몇 번 해야 하는지 구하는 문제다.

    • 연산 1회는 선택한 3×3 영역의 모든 원소를 0 ↔ 1로 뒤집는다.
    • N 또는 M이 3보다 작으면 3×3 연산 자체가 불가능하다.
    • 가능한 경우에는 좌상단부터 비교하면서 “필요할 때만” 뒤집는 그리디가 정답이 된다.

    전략 개요

    • (i, j)를 3×3 뒤집기의 좌상단으로 둘 수 있는 범위(0..N-3, 0..M-3)를 순회한다.
    • 좌상단부터 보면서 A[i][j] != B[i][j]이면, 그 칸을 맞추기 위해서는 반드시 (i, j)에서 3×3 뒤집기를 해야 한다.
    • 따라서 차이가 나는 순간에만 flip(A, i, j)를 수행하고 횟수를 증가시킨다.
    • 전체 순회가 끝난 뒤 A와 B가 완전히 같으면 횟수를 출력하고, 아니면 -1을 출력한다.

    코드가 하는 일

    • 1) 입력으로 행렬 A, B 구성 
      int[][] A = new int[N][M];
      for (int i = 0; i < N; i++) {
          String s = br.readLine();
          for (int j = 0; j < M; j++) {
              A[i][j] = s.charAt(j) - '0';
          }
      }
      
      int[][] B = new int[N][M];
      for (int i = 0; i < N; i++) {
          String s = br.readLine();
          for (int j = 0; j < M; j++) {
              B[i][j] = s.charAt(j) - '0';
          }
      }
      • 문자열로 들어오는 0/1을 정수로 변환해 2차원 배열에 저장한다.
    • 2) 3×3 연산 자체가 불가능한 경우 처리 
      if (N < 3 || M < 3) {
          System.out.println(same(A, B) ? 0 : -1);
          return;
      }
      • N이나 M이 3보다 작으면 뒤집기 연산을 할 수 없다.
      • 이때는 처음부터 같으면 0, 다르면 -1이 된다.
    • 3) 좌상단부터 그리디로 뒤집기 수행 
      int cnt = 0;
      for (int i = 0; i <= N - 3; i++) {
          for (int j = 0; j <= M - 3; j++) {
              if (A[i][j] != B[i][j]) {
                  flip(A, i, j);
                  cnt++;
              }
          }
      }
      • (i, j) 위치를 “이제부터는 다시 건드릴 수 없는 좌상단”으로 생각한다.
      • 해당 칸이 다르면, 그 칸을 뒤집는 방법은 (i, j)에서 flip하는 것뿐이므로 즉시 수행한다.
      • 이 과정을 통해 좌상단부터 순서대로 맞춰 나간다.
    • 4) 최종 동일 여부 판단 후 출력 
      System.out.println(same(A, B) ? cnt : -1);
      • 모든 뒤집기가 끝난 뒤 A와 B가 같으면 cnt가 최솟값이다.
      • 끝까지 맞출 수 없으면 불가능이므로 -1.
    • 5) flip(): 3×3 부분 행렬 뒤집기 
      private static void flip(int[][] a, int i, int j) {
          for (int k = i; k < i + 3; k++) {
              for (int l = j; l < j + 3; l++) {
                  a[k][l] = 1 - a[k][l];
              }
          }
      }
      • 0 → 1, 1 → 0을 1 - 값으로 처리한다.
      • 좌상단 (i, j) 기준 3×3 영역을 정확히 뒤집는다.
    • 6) same(): 두 행렬이 완전히 같은지 검사 
      private static boolean same(int[][] a, int[][] b) {
          for (int i = 0; i < a.length; i++) {
              for (int j = 0; j < a[0].length; j++) {
                  if (a[i][j] != b[i][j]) return false;
              }
          }
          return true;
      }
      • 모든 칸이 같아야 true를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 좌상단 순회는 O(N*M).
      • 각 flip은 3×3(9칸)만 뒤집으므로 상수 시간.
      • 전체는 O(N*M)이며, N, M ≤ 50이라 충분히 빠르다.
    • 공간 복잡도
      • 행렬 A, B 저장에 O(N*M).
      • 추가로 큰 자료구조는 사용하지 않는다.

    문제 1105 — 팔

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    팔 (BOJ 1105)는
    L부터 R 사이(포함)의 자연수 중에서, 숫자 8이 가장 적게 들어있는 수에 포함된 8의 개수를 구하는 문제다.

    • 핵심 포인트는 “어떤 수를 골라도 반드시 포함될 수밖에 없는 8의 개수”를 찾는 것이다.
    • L과 R의 자릿수가 다르면, 자릿수가 더 작은 수(또는 8을 피할 수 있는 수)를 만들 수 있으므로 정답은 0이 된다.
    • 자릿수가 같다면, 왼쪽(가장 큰 자리)부터 비교했을 때 두 수가 같은 접두사(prefix)를 공유하는 동안은 그 자릿값이 모든 수에서 고정된다.
    • 따라서 공통 접두사 부분에서 숫자가 8인 자리의 개수만큼은 어떤 수를 골라도 피할 수 없다.

    전략 개요

    • L, R의 자릿수를 구한다.
    • 자릿수가 다르면 바로 0 출력.
    • 자릿수가 같으면 각 자리를 배열로 분해한다(가장 높은 자리부터).
    • 앞자리부터 순서대로 보면서
      • 자리가 서로 다르면 그 순간부터는 해당 자리 이후 숫자를 바꿔서 8을 피할 수 있으므로 중단
      • 자리가 같고 그 값이 8이면 정답 카운트 증가
    • 누적된 값이 “최소로 포함될 수밖에 없는 8의 개수”가 된다.

    코드가 하는 일

    • 1) L과 R 입력 및 자릿수 계산 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int L = Integer.parseInt(st.nextToken());
      int R = Integer.parseInt(st.nextToken());
      
      int A = L;
      int B = R;
      int lenA = 0;
      int lenB = 0;
      
      while (A != 0) {
          A /= 10;
          lenA++;
      }
      
      while (B != 0) {
          B /= 10;
          lenB++;
      }
      • 원본 L, R을 복사해 자릿수를 센다.
    • 2) 자릿수가 다르면 정답은 0 
      if (lenA != lenB) {
          System.out.println(0);
          return;
      }
      • 자리수가 다르면 공통 접두사가 강제되지 않으므로 8을 0개로 만들 수 있다.
    • 3) 각 자리를 배열로 분해 (가장 높은 자리부터) 
      int[] LArr = new int[lenA];
      int[] RArr = new int[lenA];
      
      for (int i = lenA - 1; i >= 0; i--) {
          LArr[i] = L % 10;
          RArr[i] = R % 10;
      
          L /= 10;
          R /= 10;
      }
      • 뒤에서부터(1의 자리부터) 뽑아서 배열의 뒤쪽 인덱스에 채운다.
      • 결과적으로 LArr[0], RArr[0]이 가장 높은 자리가 된다.
    • 4) 공통 접두사 구간에서만 8 카운트 
      int min = 0;
      for (int i = 0; i < lenA; i++) {
          int a = LArr[i];
          int b = RArr[i];
      
          if (a != b) {
              break;
          } else if (a == 8){
              min++;
          }
      }
      • 왼쪽 자리부터 비교한다.
      • 어느 자리에서든 L과 R의 숫자가 달라지면 그 뒤는 선택의 폭이 생겨 8을 피할 수 있으므로 중단한다.
      • 달라지기 전까지 동일한 자리 값이 8이면 그 자리는 모든 후보에 강제로 포함되므로 min++.
    • 5) 결과 출력 
      System.out.println(min);
      • 공통 접두사에서 강제되는 8의 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 자릿수는 최대 10자리 수준(2,000,000,000 이하)이므로 전부 상수 시간이다.
      • 자리 비교도 최대 자릿수만큼만 수행 → O(d) (d=자릿수, 사실상 O(1))
    • 공간 복잡도
      • 자릿수 배열 2개만 사용 → O(d) (사실상 O(1))
