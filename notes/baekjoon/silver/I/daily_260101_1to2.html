    문제 1124 — 언더프라임

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    언더프라임 (BOJ 1124)는
    정수 X를 소인수분해했을 때 나오는 소수 인수(중복 포함)의 개수가 소수이면 X를 언더프라임이라 정의하고, A부터 B 사이(포함)에서 언더프라임의 개수를 세는 문제다.

    • 예: 12 = 2 × 2 × 3 → 소수 인수 개수는 3개, 3은 소수이므로 12는 언더프라임.
    • 1은 소수가 아니며, 소인수 개수도 0 또는 1이면(예: 소수 자체) 언더프라임이 될 수 없다.
    • 따라서 각 수마다
      • 소인수(중복 포함) 개수 cnt를 구하고
      • cnt가 소수인지 판별
      을 하면 된다.

    전략 개요

    • x = A..B까지 하나씩 보면서:
      • x를 2부터 sqrt(x)까지 나눠 소인수분해하며 (중복 포함) 인수 개수 cnt를 센다.
      • 마지막에 n > 1이면 남은 소인수가 1개 더 있으므로 cnt++.
      • cnt가 2 이상일 때만(소수 인수 개수가 0/1이면 소수가 될 수 없음) 소수 판정을 한다.
      • cnt가 소수이면 정답 카운트 증가.

    코드가 하는 일

    • 1) 입력 처리 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int A = Integer.parseInt(st.nextToken());
      int B = Integer.parseInt(st.nextToken());
      • 구간 [A, B]를 입력받는다.
    • 2) A부터 B까지 순회하며 소인수 개수 계산 
      int total = 0;
      
      for (int x = A; x <= B; x++) {
          int n = x;
          int cnt = 0;
          boolean ok = true;
      
          for (int i = 2; i * i <= n; i++) {
              while (n % i == 0) {
                  cnt++;
                  n /= i;
              }
          }
      
          if (n > 1) cnt++;
          if (cnt < 2) continue;
      • n을 나누어가며 소인수(중복 포함) 개수를 cnt에 누적한다.
      • 루프가 끝난 뒤 n이 1보다 크면 마지막 소인수가 남은 것이므로 cnt++.
      • cnt가 0 또는 1이면 소수가 아니므로(2 이상부터 소수 가능) 바로 다음 수로 넘어간다.
    • 3) cnt가 소수인지 판별 
      for (int i = 2; i * i <= cnt; i++) {
          if (cnt % i == 0) {
              ok = false;
              break;
          }
      }
      
      if (ok) total++;
      • cnt를 2부터 sqrt(cnt)까지 나눠보며 약수가 있으면 소수가 아니다.
      • 소수라면 언더프라임이므로 total을 증가시킨다.
    • 4) 결과 출력 
      System.out.println(total);
      • 구간 내 언더프라임 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 x에 대해 소인수분해를 최대 sqrt(x)까지 시도하므로 대략 O((B-A+1) * sqrt(B)).
      • B ≤ 100,000이라 가능한 수준이지만, 더 빠르게 하려면 소수 체/최소 소인수(SPF)를 쓰는 방식도 가능하다.
    • 공간 복잡도
      • 추가 배열 없이 변수만 사용 → O(1).

    문제 1141 — 접두사

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    접두사 (BOJ 1141)는
    주어진 단어들 중에서, 어떤 단어도 다른 단어의 접두어(prefix)가 되지 않도록 골라 만든 접두사X 집합의 최대 크기를 구하는 문제다.

    • 어떤 단어가 다른 단어의 접두어가 되면 그 두 단어를 동시에 포함할 수 없다.
    • 중복 단어가 입력될 수 있지만, 집합의 원소는 “단어”이므로 같은 단어는 하나로 취급할 수 있다.
    • 정답을 크게 하려면, “접두어가 되는 단어”는 제외하고 “더 긴 단어”를 남기는 것이 유리하다.

    전략 개요

    • 중복 단어는 Set으로 제거한다.
    • 단어들을 사전순으로 정렬한다.
    • 사전순 정렬을 하면, 어떤 단어 s가 다른 단어의 접두어라면 접두어 관계가 발생하는 후보는 인접한 위치에서 확인할 수 있다.
    • 따라서 정렬된 리스트에서
      • 현재 단어가 다음 단어의 접두어면 현재 단어는 포함시키지 않는다.
      • 접두어가 아니라면 현재 단어는 안전하므로 포함(카운트 증가)한다.
    • 마지막 단어는 항상 “다음 단어가 없으므로” 접두어로 판단될 일이 없어 포함된다.

    코드가 하는 일

    • 1) 입력 후 중복 제거 
      int N = Integer.parseInt(br.readLine());
      
      Set<String> set = new HashSet<>();
      for (int i = 0; i < N; i++) {
          set.add(br.readLine());
      }
      • 같은 단어가 여러 번 들어올 수 있으므로 Set으로 중복을 제거한다.
    • 2) 정렬을 위해 List로 변환 
      List<String> list = new ArrayList<>(set);
      Collections.sort(list);
      • 사전순 정렬을 하면 접두어 관계를 인접 비교로 처리할 수 있다.
    • 3) 인접 원소끼리 접두어 여부 확인하며 카운트 
      int answer = 0;
      for (int i = 0; i < list.size() - 1; i++) {
          if (!list.get(i + 1).startsWith(list.get(i))) {
              answer++;
          }
      }
      • list[i]가 list[i+1]의 접두어이면 list[i]는 포함하면 안 된다.
      • 접두어가 아니라면 안전하므로 answer++.
    • 4) 마지막 단어 처리 
      if (!list.isEmpty()) answer++;
      • 마지막 단어는 “다음 단어”가 없어서 접두어 판정을 당하지 않으므로 항상 포함된다.
      • 단어가 하나도 없을 수는 없지만(문제상 N≥1), 방어적으로 빈 리스트를 체크한다.
    • 5) 정답 출력 
      System.out.println(answer);
      • 최대 크기의 접두사X 부분집합 크기를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 중복 제거: O(N)
      • 정렬: 중복 제거 후 단어 수를 K라 하면 O(K log K)
      • 인접 비교: O(K)
      • 전체: O(K log K)
    • 공간 복잡도
      • Set/List에 단어 저장 → O(K)
