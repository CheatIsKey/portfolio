    문제 1124 — 언더프라임

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    언더프라임 (BOJ 1124)는
    정수 X를 소인수분해했을 때 나오는 소수 인수(중복 포함)의 개수가 소수이면 X를 언더프라임이라 정의하고, A부터 B 사이(포함)에서 언더프라임의 개수를 세는 문제다.

    • 예: 12 = 2 × 2 × 3 → 소수 인수 개수는 3개, 3은 소수이므로 12는 언더프라임.
    • 1은 소수가 아니며, 소인수 개수도 0 또는 1이면(예: 소수 자체) 언더프라임이 될 수 없다.
    • 따라서 각 수마다
      • 소인수(중복 포함) 개수 cnt를 구하고
      • cnt가 소수인지 판별
      을 하면 된다.

    전략 개요

    • x = A..B까지 하나씩 보면서:
      • x를 2부터 sqrt(x)까지 나눠 소인수분해하며 (중복 포함) 인수 개수 cnt를 센다.
      • 마지막에 n > 1이면 남은 소인수가 1개 더 있으므로 cnt++.
      • cnt가 2 이상일 때만(소수 인수 개수가 0/1이면 소수가 될 수 없음) 소수 판정을 한다.
      • cnt가 소수이면 정답 카운트 증가.

    코드가 하는 일

    • 1) 입력 처리 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int A = Integer.parseInt(st.nextToken());
      int B = Integer.parseInt(st.nextToken());
      • 구간 [A, B]를 입력받는다.
    • 2) A부터 B까지 순회하며 소인수 개수 계산 
      int total = 0;
      
      for (int x = A; x <= B; x++) {
          int n = x;
          int cnt = 0;
          boolean ok = true;
      
          for (int i = 2; i * i <= n; i++) {
              while (n % i == 0) {
                  cnt++;
                  n /= i;
              }
          }
      
          if (n > 1) cnt++;
          if (cnt < 2) continue;
      • n을 나누어가며 소인수(중복 포함) 개수를 cnt에 누적한다.
      • 루프가 끝난 뒤 n이 1보다 크면 마지막 소인수가 남은 것이므로 cnt++.
      • cnt가 0 또는 1이면 소수가 아니므로(2 이상부터 소수 가능) 바로 다음 수로 넘어간다.
    • 3) cnt가 소수인지 판별 
      for (int i = 2; i * i <= cnt; i++) {
          if (cnt % i == 0) {
              ok = false;
              break;
          }
      }
      
      if (ok) total++;
      • cnt를 2부터 sqrt(cnt)까지 나눠보며 약수가 있으면 소수가 아니다.
      • 소수라면 언더프라임이므로 total을 증가시킨다.
    • 4) 결과 출력 
      System.out.println(total);
      • 구간 내 언더프라임 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 x에 대해 소인수분해를 최대 sqrt(x)까지 시도하므로 대략 O((B-A+1) * sqrt(B)).
      • B ≤ 100,000이라 가능한 수준이지만, 더 빠르게 하려면 소수 체/최소 소인수(SPF)를 쓰는 방식도 가능하다.
    • 공간 복잡도
      • 추가 배열 없이 변수만 사용 → O(1).

    문제 1141 — 접두사

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Palindromes (BOJ 6474)는
    입력으로 주어진 문자열이 일반 회문(palindrome)인지, 거울 문자열(mirrored string)인지, 혹은 둘 다 만족하는 거울 회문(mirrored palindrome)인지 판별해 정해진 문장을 출력하는 문제다.

    • palindrome: 문자열을 뒤집어도 동일한 문자열
    • mirrored string: 문자열을 뒤집은 뒤 각 문자를 “거울 대응 문자”로 치환했을 때 원래 문자열과 같음
    • mirrored palindrome: 위 두 조건을 모두 만족
    • 입력은 EOF까지 여러 줄로 주어진다.
    • 출력은 문자열 + " -- " + 판정 문구, 그리고 빈 줄 1줄이 추가된다.

    전략 개요

    • 문제에서 제공한 “거울 대응” 규칙을 Map<Character, Character>으로 만든다.
    • 각 문자열에 대해
      • 앞/뒤 문자가 같은지 비교해서 palindrome 여부 판단
      • 뒤쪽 문자에 대한 거울 대응 문자를 찾아서 앞쪽 문자와 같은지 비교해 mirrored 여부 판단
    • 두 boolean 값(palindrome, mirrored) 조합으로 출력 문구를 결정한다.

    코드가 하는 일

    • 1) 거울 문자 매핑(Map) 구성 
      Map<Character, Character> map = new HashMap<>();
      map.put('E', '3');
      map.put('3', 'E');
      map.put('J', 'L');
      map.put('L', 'J');
      map.put('S', '2');
      map.put('2', 'S');
      map.put('Z', '5');
      map.put('5', 'Z');
      map.put('A', 'A');
      map.put('H', 'H');
      map.put('I', 'I');
      map.put('M', 'M');
      map.put('O', 'O');
      map.put('T', 'T');
      map.put('U', 'U');
      map.put('V', 'V');
      map.put('W', 'W');
      map.put('X', 'X');
      map.put('Y', 'Y');
      map.put('1', '1');
      map.put('8', '8');
      • 문제 표에 나온 “문자 ↔ 거울 문자” 관계를 그대로 저장한다.
      • 서로 바뀌는 쌍(E↔3, J↔L 등)은 양방향으로 모두 넣는다.
      • 자기 자신으로 대응되는 문자(A, H, I, ...)도 넣는다.
    • 2) 입력 문자열을 EOF까지 처리 
      String line;
      while ((line = br.readLine()) != null) {
          boolean palindrome = true;
          boolean mirrored = true;
      • 한 줄이 하나의 문자열이고, EOF까지 반복한다.
      • 각 문자열마다 두 조건을 각각 검사한다.
    • 3) palindrome 판별: 양끝에서 가운데로 비교 
      for (int i = 0; i < line.length(); i++) {
          if (i > line.length() - 1 - i) break;
          if (line.charAt(i) != line.charAt(line.length() - 1 - i)) {
              palindrome = false;
              break;
          }
      }
      • i와 len-1-i를 비교하며 회문인지 확인한다.
      • 한 번이라도 다르면 회문이 아니므로 false로 바꾸고 종료한다.
      • 가운데를 지나면 더 볼 필요가 없어 중단한다.
    • 4) mirrored 판별: 뒤쪽 문자의 거울값이 앞쪽 문자와 같은지 확인 
      for (int i = 0; i < line.length(); i++) {
          int backIdx = line.length() - 1 - i;
          Character c = map.get(line.charAt(backIdx));
      
          if (c == null || line.charAt(i) != c) {
              mirrored = false;
              break;
          }
      }
      • 거울 문자열 조건은 “뒤에서 i번째 문자를 거울 변환한 값”이 “앞에서 i번째 문자”와 같아야 한다.
      • 따라서 backIdx 위치 문자를 맵에서 변환한 뒤, line.charAt(i)와 비교한다.
      • 대응 문자가 없거나(null) 값이 다르면 mirrored가 아니다.
    • 5) 조건 조합에 따라 문구 출력 
      sb.append(line).append(" -- ");
      
      if (palindrome && mirrored) {
          sb.append("is a mirrored palindrome.").append('\n').append('\n');
          continue;
      } else if (palindrome) {
          sb.append("is a palindrome.").append('\n').append('\n');
      } else if (mirrored) {
          sb.append("is a mirrored string.").append('\n').append('\n');
      } else {
          sb.append("is not a palindrome.").append('\n').append('\n');
      }
      • 문제에서 요구한 출력 형식을 그대로 따른다.
      • 각 케이스마다 마지막에 빈 줄을 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 L이라 하면, palindrome 검사 O(L), mirrored 검사 O(L) → 전체 O(L).
      • L ≤ 20이라 매우 작다.
    • 공간 복잡도
      • 거울 매핑 테이블은 고정 크기 → O(1).
