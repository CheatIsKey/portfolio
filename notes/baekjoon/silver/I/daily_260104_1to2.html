    문제 1195 — 킥다운

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    킥다운 (백준 1195)는
    두 개의 톱니(기어)를 문자열로 표현했을 때, 한 기어를 좌우로 이동(겹치게)시키며 서로 맞물릴 수 있는지 확인하고, 가능한 배치들 중 전체 길이(겹친 결과의 최소 길이)를 구하는 문제다.

    • 각 기어는 문자열로 주어지며, 문자 '1' 또는 '2'로 구성된다.
    • 두 기어가 겹치는 구간에서 '2'와 '2'가 겹치면 충돌이 발생해 불가능한 배치다.
    • 충돌이 없는 배치 중, 두 문자열을 합쳐서 덮는 구간의 길이를 최소화해야 한다.

    전략 개요

    • 한 기어(gearTwo)를 다른 기어(gearOne) 위에서 좌우로 슬라이딩하며 모든 위치를 검사한다.
    • 각 이동량 move에 대해
      • 겹치는 구간에서 '2'가 동시에 등장하는지(충돌) 확인한다.
      • 충돌이 없으면 해당 배치에서의 전체 길이를 계산해 최솟값을 갱신한다.
    • 이동 범위는 “완전히 왼쪽으로 벗어난 상태”부터 “완전히 오른쪽으로 벗어난 상태”까지 포함하면 된다.

    코드가 하는 일

    • 1) 입력 및 기본 정보 준비 
      String gearOne = br.readLine();
      String gearTwo = br.readLine();
      
      int gearOneLength = gearOne.length();
      int gearTwoLength = gearTwo.length();
      int min = Integer.MAX_VALUE;
      • 두 기어 문자열과 길이를 받아 둔다.
      • 정답(최소 전체 길이)을 min으로 관리한다.
    • 2) gearTwo를 좌우로 이동시키며 모든 배치 검사 
      for (int move = -gearTwoLength; move <= gearOneLength; move++) {
          if (chk(gearOne, gearTwo, move)) {
              int len = Math.max(gearOneLength, gearTwoLength + move) - Math.min(0, move);
              min = Math.min(min, len);
          }
      }
      • move는 gearTwo의 인덱스 0이 gearOne의 어디에 위치하는지를 의미한다.
      • 예) move = 0이면 gearTwo[0]가 gearOne[0]와 정렬된다.
      • move < 0이면 gearTwo가 왼쪽으로 튀어나온 상태, move > 0이면 오른쪽으로 밀려 들어간 상태가 된다.
      • 가능한 배치인지(chk)를 확인하고, 가능하면 전체 덮는 길이 len을 계산해 최솟값 갱신한다.
    • 3) 전체 길이(len) 계산 방식 
      int len = Math.max(gearOneLength, gearTwoLength + move) - Math.min(0, move);
      • 두 기어가 차지하는 “가장 왼쪽 위치”는 min(0, move)
      • “가장 오른쪽 끝”은
        • gearOne의 끝: gearOneLength
        • gearTwo의 끝: move + gearTwoLength
        이 둘 중 큰 값이므로 max(gearOneLength, move + gearTwoLength)
      • 따라서 전체 길이 = (오른쪽 끝) - (왼쪽 시작)으로 계산된다.
    • 4) 충돌 검사 함수 chk() 
      private static boolean chk(String gearOne, String gearTwo, int move) {
          int gearOneLength = gearOne.length();
          int gearTwoLength = gearTwo.length();
      
          for (int i = 0; i < gearTwoLength; i++) {
              int pos = i + move;
      
              if (0 <= pos && pos < gearOneLength) {
                  if (gearOne.charAt(pos) == '2' && gearTwo.charAt(i) == '2') return false;
              }
          }
      
          return true;
      }
      • gearTwo의 각 문자 i가 gearOne에서 겹치는 위치 pos = i + move를 계산한다.
      • pos가 gearOne 범위 내일 때만 “겹침”이므로 그때만 비교한다.
      • 겹치는 칸에서 둘 다 '2'이면 충돌 → 불가능 → false
      • 끝까지 충돌이 없으면 가능한 배치 → true
    • 5) 최솟값 출력 
      System.out.println(min);
      • 가능한 모든 배치 중 최소 전체 길이를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 이동 횟수는 대략 (gearOneLength + gearTwoLength) 수준이다.
      • 각 이동마다 gearTwo 전체를 훑으며 충돌을 검사하므로 O(L1 + L2) * O(L2) 형태가 된다.
      • 즉, O((L1 + L2) * L2) (L1=gearOne 길이, L2=gearTwo 길이)
      • 문제 크기에서 충분히 빠르게 동작한다.
    • 공간 복잡도
      • 추가 배열 없이 입력 문자열과 상수 변수만 사용 → O(1)

    문제 1262 — 알파벳 다이아몬드

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    알파벳 다이아몬드 (백준 1262)는
    크기 N으로 만들어지는 다이아몬드 패턴이 무한히 반복되는 평면에서, 직사각형 범위 (R1, C1) ~ (R2, C2)만 잘라 출력하는 문제다.

    • 한 타일(한 주기)의 크기는 (2N-1) × (2N-1)이다.
    • 각 칸의 문자는 중심에서의 맨해튼 거리로 결정된다.
    • 거리 dist가 N 이상이면 다이아몬드 밖이므로 '.'를 출력한다.
    • 다이아몬드 안이면 'a' + dist 형태로 출력하되, 알파벳은 26으로 순환한다. (즉 dist % 26)
    • 전체 평면은 주기적으로 반복되므로, 실제 좌표를 한 주기 내부 좌표로 줄이면 된다. (모듈러)

    전략 개요

    • 출력해야 할 범위는 R1..R2, C1..C2로 최대 1001×1001 정도라서(문제 조건) 칸별로 바로 계산해도 된다.
    • 좌표 (r, c)가 주어지면,
      • 주기 크기 size = 2N - 1
      • 주기 내부 좌표 x = r % size, y = c % size
      • 중심 좌표는 (N-1, N-1)
      • 맨해튼 거리 dist = |(N-1)-x| + |(N-1)-y|
    • dist >= N이면 '.', 아니면 (char)('a' + (dist % 26))를 출력한다.

    코드가 하는 일

    • 1) 입력 파싱 및 출력 범위 순회 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int N = Integer.parseInt(st.nextToken());
      int R1 = Integer.parseInt(st.nextToken());
      int C1 = Integer.parseInt(st.nextToken());
      int R2 = Integer.parseInt(st.nextToken());
      int C2 = Integer.parseInt(st.nextToken());
      
      for (int r = R1; r <= R2; r++) {
          for (int c = C1; c <= C2; c++) {
              sb.append(getChar(N, r, c));
          }
          sb.append('\n');
      }
      • 요구한 직사각형 영역만 이중 루프로 순회한다.
      • 각 칸의 문자는 getChar()로 즉시 계산해서 붙인다.
    • 2) (r, c) 한 칸의 문자를 결정하는 getChar() 
      private static char getChar(int N, int r, int c) {
          int size = 2 * N - 1;
          int x = r % size;
          int y = c % size;
      
          int dist = Math.abs((N - 1) - x) + Math.abs((N - 1) - y);
      
          if (dist >= N) return '.';
          return (char) ('a' + (dist % 26));
      }
      • size = 2N-1로 한 주기의 크기를 만든다.
      • 무한 반복이므로 (r, c)를 한 주기 내부 좌표 (x, y)로 축소한다.
      • 중심 (N-1, N-1)에서의 맨해튼 거리 dist를 계산한다.
      • dist >= N이면 다이아몬드 밖이므로 '.' 출력. 그렇지 않으면 거리만큼 알파벳이 증가하며, 26으로 순환한다.
    • 3) 출력 
      System.out.println(sb);
      • 한 번에 모아 출력해 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 출력하는 칸 수를 H = R2-R1+1, W = C2-C1+1이라 하면
      • 각 칸을 O(1)에 계산하므로 전체는 O(H * W).
    • 공간 복잡도
      • 추가 배열 없이 상수 변수 + 출력 버퍼만 사용 → O(1) (출력 문자열 제외).
