    문제 1149 — RGB거리

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    RGB거리 (백준 1149)는
    N개의 집을 R/G/B 중 하나로 칠할 때, 인접한 집끼리는 같은 색을 사용할 수 없다는 조건 하에서 전체 비용의 최솟값을 구하는 문제다.

    • 각 집 i(0-index)의 색 비용: RGB[i][0]=R, RGB[i][1]=G, RGB[i][2]=B
    • 제약: i번 집과 i-1번 집의 색은 달라야 한다.
    • 목표: 모든 집을 칠하는 총 비용 최소화

    전략 개요

    • 전형적인 DP 문제로, “i번째 집을 특정 색으로 칠했을 때의 최소 누적 비용”을 저장한다.
    • DP 정의:
      • cost[i][c] = i번째 집을 색 c로 칠했을 때 0~i까지 최소 비용
    • 점화식:
      • cost[i][0] = RGB[i][0] + min(cost[i-1][1], cost[i-1][2])
      • cost[i][1] = RGB[i][1] + min(cost[i-1][0], cost[i-1][2])
      • cost[i][2] = RGB[i][2] + min(cost[i-1][0], cost[i-1][1])
    • 마지막 답: min(cost[N-1][0], cost[N-1][1], cost[N-1][2])

    코드가 하는 일

    • 1) 입력을 RGB 비용 배열에 저장 
      int[][] RGB = new int[N][3];
      
      for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          int R = Integer.parseInt(st.nextToken());
          int G = Integer.parseInt(st.nextToken());
          int B = Integer.parseInt(st.nextToken());
      
          RGB[i][0] = R;
          RGB[i][1] = G;
          RGB[i][2] = B;
      }
      • 각 집의 색별 비용을 RGB[i][0..2]에 담는다.
    • 2) DP 테이블 초기값 설정 (0번째 집) 
      int[][] cost = new int[N][3];
      cost[0][0] = RGB[0][0];
      cost[0][1] = RGB[0][1];
      cost[0][2] = RGB[0][2];
      • 첫 집은 이전 집이 없으므로, 그대로 비용이 누적 비용이 된다.
    • 3) 1번째 집부터 점화식으로 최소 누적 비용 갱신 
      for (int i = 1; i < N; i++) {
          cost[i][0] = RGB[i][0] + Math.min(cost[i-1][1], cost[i-1][2]);
          cost[i][1] = RGB[i][1] + Math.min(cost[i-1][0], cost[i-1][2]);
          cost[i][2] = RGB[i][2] + Math.min(cost[i-1][0], cost[i-1][1]);
      }
      • 현재 색이 R이면 이전 집은 G/B만 가능 → 그 중 최소를 더한다.
      • 현재 색이 G이면 이전 집은 R/B만 가능 → 그 중 최소를 더한다.
      • 현재 색이 B이면 이전 집은 R/G만 가능 → 그 중 최소를 더한다.
    • 4) 마지막 집까지 칠했을 때 최소값 출력 
      System.out.println(Math.min(cost[N-1][0], Math.min(cost[N-1][1], cost[N-1][2])));
      • 마지막 집의 색이 무엇이든 상관 없으므로 3개 중 최소를 정답으로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 집 N개를 한 번 순회하며 각 집에서 상수 연산(3색) → O(N).
    • 공간 복잡도
      • RGB[N][3], cost[N][3] 사용 → O(N).
      • (참고) 직전 행만 필요하므로 O(1)로도 최적화 가능하지만, 현재 코드는 표 형태로 명확하게 구현한 버전이다.

    문제 1189 — 컴백홈

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    컴백홈 (백준 1189)은
    R×C 격자에서 시작점 (R-1, 0)에서 도착점 (0, C-1)까지 정확히 K칸을 이동해서 도달하는 경로의 개수를 구하는 문제다.

    • 이동은 상/하/좌/우 4방향
    • T는 장애물(이동 불가)
    • 같은 칸을 두 번 방문할 수 없음
    • 정확히 K칸 이동(시작 칸 포함 거리 카운트 방식에 주의)

    전략 개요

    • 방문 체크가 필요한 “경로 개수 세기” 유형이므로 DFS(백트래킹)로 모든 가능한 경로를 탐색한다.
    • DFS 상태:
      • 현재 좌표 (r, c)
      • 현재까지 이동한 칸 수(거리) dist
    • 가지치기(Pruning):
      • 도착점에 도달했을 때 dist == K면 정답 카운트 증가
      • dist == K인데 아직 도착점이 아니면 더 진행해도 실패 → 즉시 종료
    • 각 이동에서 다음 칸이
      • 격자 범위 안
      • 장애물(T)이 아님
      • 미방문
      조건을 만족할 때만 진행한다.

    코드가 하는 일

    • 1) 입력 처리 + 시작점 방문 처리 
      R = Integer.parseInt(st.nextToken());
      C = Integer.parseInt(st.nextToken());
      K = Integer.parseInt(st.nextToken());
      
      map = new char[R][C];
      visited = new boolean[R][C];
      visited[R-1][0] = true;
      
      for (int r = 0; r < R; r++) {
          map[r] = br.readLine().toCharArray();
      }
      • 격자와 방문 배열을 준비한다.
      • 시작점 (R-1, 0)은 출발 즉시 방문 처리한다. (중복 방문 방지)
    • 2) DFS 시작 (dist=1부터) 
      dfs(R-1, 0, 1);
      • 현재 칸을 포함해 거리를 세므로 시작 시 dist=1로 시작한다.
    • 3) 도착점에 도달했을 때 dist==K인지 확인 
      if (r == 0 && c == C - 1) {
          if (dist == K) total++;
          return;
      }
      • 도착점이라도 거리가 K가 아니면 정답이 아니다.
      • 도착점에서 더 이동하는 경우는 허용되지 않으므로 바로 종료한다.
    • 4) dist가 K에 도달했는데 아직 도착점이 아니면 가지치기 
      if (dist == K) return;
      • 정확히 K칸이어야 하므로, 여기서 더 가면 dist가 K를 초과해 무조건 실패다.
    • 5) 4방향 탐색 + 조건 체크 + 백트래킹 
      for (int i = 0; i < 4; i++) {
          int dr = r + ROW[i];
          int dc = c + COL[i];
      
          if (dr < 0 || R <= dr || dc < 0 || C <= dc) continue;
          if (map[dr][dc] == 'T') continue;
          if (visited[dr][dc]) continue;
      
          visited[dr][dc] = true;
          dfs(dr, dc, dist + 1);
          visited[dr][dc] = false;
      }
      • 범위/장애물/방문 여부를 검사하고 통과하면 방문 처리 후 재귀 호출한다.
      • 재귀가 끝나면 방문을 되돌려(백트래킹) 다른 경로를 탐색할 수 있게 한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최악의 경우(장애물 거의 없음) 경로를 모두 탐색하므로 지수적으로 증가한다.
      • 상한은 대략 분기 최대 4, 깊이 K 수준 → O(4^K) (실제로는 방문 제한 때문에 더 줄어듦)
    • 공간 복잡도
      • 방문 배열 visited[R][C] → O(R*C)
      • 재귀 호출 스택 깊이 최대 K → O(K)
