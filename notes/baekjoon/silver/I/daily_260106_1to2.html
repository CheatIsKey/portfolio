    문제 1276 — PLATFORME

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    PLATFORME (백준 1276)는
    여러 개의 플랫폼(수평 구간)이 주어졌을 때, 각 플랫폼의 양 끝을 지지하기 위한 기둥(pillar)의 총 길이를 구하는 문제다.

    • 플랫폼은 높이 Y, 구간 [X1, X2]로 주어진다.
    • 기둥은 플랫폼 끝에서 0.5만큼 안쪽 위치에 세운다.
      • 왼쪽 끝 지점: X1 + 0.5
      • 오른쪽 끝 지점: X2 - 0.5
    • 각 기둥은 바닥(0) 또는 그 지점 아래에 존재하는 다른 플랫폼의 윗면 위에 서야 한다.
    • 따라서 기둥 길이는 현재 플랫폼 높이 - (아래에서 받쳐주는 높이)가 된다.

    전략 개요

    • 플랫폼을 높이(Y) 오름차순으로 정렬한다.
    • 현재 플랫폼의 두 지지점(왼쪽/오른쪽)에 대해,
      • 그 지점 아래에 있는 플랫폼들 중, 해당 x좌표를 포함하는 플랫폼의 최대 높이를 찾는다.
      • 그 높이가 “바닥 역할”이므로 현재 높이 - 최대 높이만큼이 기둥 길이이다.
    • 정렬 순서대로 진행하면 “이미 처리한 플랫폼”만 보아도 아래 지지대를 결정할 수 있다.
    • x좌표의 0.5를 다루기 위해, 모든 x를 2배 스케일로 정수화한다.
      • X1 + 0.5 → 2*X1 + 1
      • X2 - 0.5 → 2*X2 - 1
      • 플랫폼 구간은 열린 구간으로 체크: 2*X1 < col < 2*X2

    코드가 하는 일

    • 1) 입력 플랫폼을 배열로 받고, 높이 기준 정렬 
      int N = Integer.parseInt(br.readLine());
      Board[] boards = new Board[N];
      
      for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int r = Integer.parseInt(st.nextToken());
          int c1 = Integer.parseInt(st.nextToken());
          int c2 = Integer.parseInt(st.nextToken());
      
          boards[i] = new Board(r, c1, c2);
      }
      
      Arrays.sort(boards, (a, b) -> {
          return Integer.compare(a.r, b.r);
      });
      • r는 높이(Y), c1/c2는 X1/X2이다.
      • 낮은 플랫폼부터 처리하면 “아래 플랫폼”은 항상 이미 처리된 것들이다.
    • 2) 각 플랫폼의 두 지지점(왼쪽/오른쪽)에서 필요한 기둥 길이를 더함 
      long sum = 0;
      List<Board> list = new ArrayList<>();
      
      for (Board board : boards) {
          int left = 2 * board.c1 + 1;
          int right = 2 * board.c2 - 1;
      
          sum += board.r - height(list, left);
          sum += board.r - height(list, right);
      
          list.add(board);
      }
      • 왼쪽 지지점은 2*X1+1, 오른쪽 지지점은 2*X2-1로 정수화한다.
      • height(list, col)는 그 x지점 아래에서 받쳐줄 수 있는 플랫폼의 최대 높이(없으면 0)를 반환한다.
      • 현재 높이에서 그 값을 빼면 필요한 기둥 길이가 된다.
      • 현재 플랫폼을 처리한 뒤 list에 추가해서 이후 플랫폼의 받침 후보가 되게 한다.
    • 3) height(): 특정 지점(col) 아래의 최대 받침 높이 찾기 
      private static int height(List<Board> list, int col) {
          int floor = 0;
      
          for (Board board : list) {
              int left = 2 * board.c1;
              int right = 2 * board.c2;
      
              if (left < col && col < right) {
                  if (board.r > floor) floor = board.r;
              }
          }
      
          return floor;
      }
      • 각 플랫폼 구간을 2배 스케일로 (2*X1, 2*X2)로 보고,
      • 지지점 col이 그 열린 구간 안에 있으면 그 플랫폼이 해당 지점을 “덮는” 것이다.
      • 덮는 플랫폼들 중 가장 높은 r이 실제로 기둥이 닿는 받침 높이가 된다.
      • 아무 것도 없으면 바닥(0)이다.
    • 4) 누적 합 출력 
      System.out.println(sum);
      • 모든 플랫폼의 양 끝 기둥 길이를 합친 결과를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 각 플랫폼마다 지지점 2개를 검사하고, height()가 현재까지의 플랫폼을 선형 탐색 → 최악 O(N)
      • 총합: O(N log N + N^2)
      • 문제 제한이 N ≤ 100이라서 N^2도 충분히 빠르다.
    • 공간 복잡도
      • 플랫폼 저장 배열/리스트: O(N)
      • 추가로 큰 자료구조는 사용하지 않는다.

    문제 1283 — 단축키 지정

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    단축키 지정 (백준 1283)은
    메뉴 이름이 여러 줄로 주어질 때, 각 줄마다 단축키(알파벳 1개)를 하나 지정해서 해당 문자를 [ ]로 감싸 출력하는 문제다.

    • 단축키로 지정된 알파벳은 다른 줄에서 다시 사용할 수 없다. (대소문자 구분 없이 동일)
    • 단축키 선택 규칙은 다음 우선순위를 따른다.
      • 1순위: 각 단어의 첫 글자(문장 시작 또는 공백 다음 글자) 중에서 가능한 것
      • 2순위: 문장 전체를 왼쪽부터 보면서 가능한 것
      • 끝까지 없으면 그 줄은 그대로 출력

    전략 개요

    • 알파벳 사용 여부를 boolean[26]으로 관리한다. (대소문자 통일해서 체크)
    • 각 입력 줄에 대해:
      • 먼저 “단어의 첫 글자” 위치를 순회하며 아직 안 쓴 알파벳이 있으면 그 위치를 선택한다.
      • 없으면 전체 문자열을 왼쪽부터 순회하며 아직 안 쓴 알파벳이 있으면 그 위치를 선택한다.
      • 선택된 위치가 있으면 해당 문자를 [문자]로 감싸 출력하고, 사용 체크를 한다.

    코드가 하는 일

    • 1) 알파벳 사용 체크 배열 준비 
      boolean[] charChk = new boolean[26];
      int N = Integer.parseInt(br.readLine());
      • charChk[i]는 알파벳 ('a' + i)가 이미 단축키로 쓰였는지 여부다.
      • 대소문자 구분이 없으므로 항상 toLowerCase로 통일해 인덱싱한다.
    • 2) 1순위: 각 단어 첫 글자에서 단축키 후보 찾기 
      String line = br.readLine();
      int idx = -1;
      
      for (int j = 0; j < line.length(); j++) {
          if (j == 0 || line.charAt(j - 1) == ' ') {
              char c = line.charAt(j);
      
              if (isAlpha(c) && !charChk[Character.toLowerCase(c) - 'a']) {
                  idx = j;
                  break;
              }
          }
      }
      • j == 0 또는 line.charAt(j-1) == ' '이면 “단어의 첫 글자” 위치다.
      • 그 문자가 알파벳이고 아직 사용하지 않았다면 그 위치 idx를 확정한다.
    • 3) 2순위: 전체 문자열에서 단축키 후보 찾기 
      if (idx == -1) {
          for (int j = 0; j < line.length(); j++) {
              char c = line.charAt(j);
      
              if (isAlpha(c) && !charChk[Character.toLowerCase(c) - 'a']) {
                  idx = j;
                  break;
              }
          }
      }
      • 1순위에서 못 찾았으면, 문자열 전체를 왼쪽부터 순회하며 가능한 첫 알파벳을 찾는다.
    • 4) 출력 문자열 만들기 (선택/미선택 분기) 
      if (idx == -1) {
          sb.append(line);
      } else {
          char c = line.charAt(idx);
          charChk[Character.toLowerCase(c) - 'a'] = true;
          sb.append(line, 0, idx).append('[').append(c).append(']').append(line.substring(idx + 1));
      }
      
      sb.append('\n');
      • 끝까지 후보가 없으면 원문 그대로 출력한다.
      • 후보가 있으면:
        • 사용 체크를 하고
        • idx 위치의 문자를 [ ]로 감싸서 출력한다.
      • sb.append(line, 0, idx)는 Java의 부분 문자열 append로 깔끔하게 앞부분을 붙인다.
    • 5) 알파벳 판별 함수 
      private static boolean isAlpha(char c) {
          return ('a' <= c && c <= 'z') || ('A' <= c && c <= 'Z');
      }
      • 입력에는 공백도 포함되므로 알파벳만 단축키 후보로 인정하기 위해 체크한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 줄 길이를 L이라 하면
        • 1순위 탐색 O(L) + (필요 시) 2순위 탐색 O(L)
      • 한 줄당 최악 O(L) 수준(상수배 2)이고, 전체는 입력 크기에 비례한다.
    • 공간 복잡도
      • 알파벳 체크 배열 boolean[26]만 추가로 사용 → O(1).
