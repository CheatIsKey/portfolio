    문제 1303 — 전쟁 - 전투

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    전쟁 - 전투 (백준 1303)는
    전장 맵에서 병사들이 상하좌우로 연결된 그룹을 이룰 때, 각 팀(W, B)의 전투력을 계산해 출력하는 문제다.

    • 전투력 규칙: 같은 팀이 연결된 한 그룹의 크기를 k라 하면 그 그룹 전투력은 k².
    • 전체 전투력은 팀별로 모든 그룹의 k²를 합한 값이다.
    • 즉, 맵을 돌면서 방문하지 않은 칸에서 시작해 같은 팀으로 연결된 영역 크기를 구하면 된다.

    전략 개요

    • 각 칸을 한 번씩만 처리하기 위해 visited 배열을 사용한다.
    • 모든 칸을 순회하면서 아직 방문하지 않은 칸을 발견하면:
      • 그 칸의 팀 문자(W 또는 B)를 기준으로 BFS를 수행한다.
      • BFS로 같은 팀으로 연결된 칸 개수 cnt를 구한다.
      • 해당 팀 전투력에 cnt * cnt를 더한다.

    코드가 하는 일

    • 1) 입력 및 맵/방문 배열 준비 
      int N = Integer.parseInt(st.nextToken());
      int M = Integer.parseInt(st.nextToken());
      
      war = new char[M][N];
      visited = new boolean[M][N];
      
      for (int r = 0; r < M; r++) {
          war[r] = br.readLine().toCharArray();
      }
      • 문제 입력은 N(가로), M(세로) 순서다.
      • 코드는 war[M][N]로 (행=M, 열=N) 형태로 저장한다.
    • 2) 전체 맵 순회하며 그룹 단위로 BFS 수행 
      int white = 0;
      int blue = 0;
      
      for (int r = 0; r < M; r++) {
          for (int c = 0; c < N; c++) {
              if (!visited[r][c]) {
                  char team = war[r][c];
                  int cnt = bfs(r, c, team);
      
                  if (team == 'W') white += cnt * cnt;
                  else blue += cnt * cnt;
              }
          }
      }
      • 방문하지 않은 칸에서 시작해 연결된 같은 팀의 크기 cnt를 구한다.
      • 팀에 따라 white 또는 blue에 cnt²를 누적한다.
    • 3) bfs(): 같은 팀으로 연결된 영역 크기 계산 
      private static int bfs(int row, int col, char team) {
          ArrayDeque<int[]> queue = new ArrayDeque<>();
          visited[row][col] = true;
          queue.add(new int[]{row, col});
      
          int cnt = 1;
      
          while (!queue.isEmpty()) {
              int[] cur = queue.poll();
              int r = cur[0];
              int c = cur[1];
      
              for (int i = 0; i < 4; i++) {
                  int nr = r + ROW[i];
                  int nc = c + COL[i];
      
                  if (nr < 0 || war.length <= nr || nc < 0 || war[0].length <= nc) continue;
                  if (visited[nr][nc]) continue;
                  if (war[nr][nc] != team) continue;
      
                  visited[nr][nc] = true;
                  queue.add(new int[]{nr, nc});
                  cnt++;
              }
          }
      
          return cnt;
      }
      • 시작 칸을 방문 처리하고 큐에 넣은 뒤, BFS로 상하좌우를 확장한다.
      • 확장 조건:
        • 범위 안
        • 미방문
        • 같은 팀 문자
      • 조건을 만족하면 방문 처리 후 큐에 넣고 카운트를 증가시킨다.
      • BFS가 끝나면 해당 그룹의 크기 cnt를 반환한다.
    • 4) 최종 출력 
      System.out.println(white + " " + blue);
      • 요구 출력 형식대로 W팀 전투력과 B팀 전투력을 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 칸은 BFS 과정에서 최대 한 번만 큐에 들어가므로 전체가 O(M * N).
    • 공간 복잡도
      • war, visited 배열이 O(M * N).
      • BFS 큐는 최악의 경우 한 컴포넌트 크기만큼 커질 수 있어 O(M * N).

    문제 1309 — 동물원

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    동물원 (백준 1309)는
    세로로 N칸(행), 가로로 2칸(열)인 우리에 사자를 배치할 때, 가로/세로로 인접한 칸에 사자를 동시에 둘 수 없다는 조건을 만족하는 배치 경우의 수를 구하는 문제다.

    • 각 행(가로 2칸)에서 사자를 둘 수 있는 상태는 3가지다:
      • 0: 해당 행에 사자를 두지 않음
      • 1: 왼쪽 칸에만 둠
      • 2: 오른쪽 칸에만 둠
    • 한 행에서 양쪽 칸을 동시에 채우는 경우는 가로 인접이므로 불가능하다.
    • 정답은 매우 커질 수 있어 9901로 나눈 나머지를 출력한다.

    전략 개요

    • DP로 “i번째 행까지 배치했을 때의 경우의 수”를 누적한다.
    • field[i][state]를 다음과 같이 정의한다:
      • field[i][0]: i번째 행에 사자를 두지 않는 경우의 수
      • field[i][1]: i번째 행 왼쪽에만 사자를 둔 경우의 수
      • field[i][2]: i번째 행 오른쪽에만 사자를 둔 경우의 수
    • i행의 상태는 (i-1)행의 상태와 인접 제한에 따라 전이된다.

    코드가 하는 일

    • 1) DP 테이블 준비 및 초기값 설정 
      int[][] field = new int[N + 1][3];
      
      field[1][0] = 1;
      field[1][1] = 1;
      field[1][2] = 1;
      • 1행은 가능한 상태가 3가지(비움/왼쪽/오른쪽)이고 각각 1가지씩 존재한다.
    • 2) 점화식으로 i행 상태 갱신 
      for (int i = 2; i <= N; i++) {
          field[i][0] = (field[i-1][0] + field[i-1][1] + field[i-1][2]) % 9901;
          field[i][1] = (field[i-1][0] + field[i-1][2]) % 9901;
          field[i][2] = (field[i-1][0] + field[i-1][1]) % 9901;
      }
      • i행이 비어있음(0): (i-1)행이 어떤 상태든 가능 → 모두 합
      • i행 왼쪽(1): (i-1)행 왼쪽(1)은 세로 인접이라 불가 → 0, 2만 가능
      • i행 오른쪽(2): (i-1)행 오른쪽(2)은 세로 인접이라 불가 → 0, 1만 가능
      • 각 단계마다 9901로 나머지 처리한다.
    • 3) N행에서 가능한 모든 상태 합이 정답 
      int result = (field[N][0] + field[N][1] + field[N][2]) % 9901;
      System.out.println(result);
      • N행이 어떤 상태로 끝나든 조건만 만족하면 valid이므로 3상태를 합친다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 행을 2부터 N까지 한 번씩 갱신하고, 각 행은 상수(3칸) 연산 → O(N).
    • 공간 복잡도
      • DP 테이블 (N+1) x 3 사용 → O(N).
      • 원하면 이전 행만 유지해 O(1)로도 줄일 수 있다.
