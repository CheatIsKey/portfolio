    문제 1389 — 케빈 베이컨의 6단계 법칙

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    케빈 베이컨의 6단계 법칙 (백준 1389)은
    사람들을 정점, 친구 관계를 간선으로 보는 무방향 그래프에서, 각 사람 기준으로 다른 모든 사람까지의 최단 거리 합(케빈 베이컨 수의 합)을 구한 뒤 그 합이 가장 작은 사람의 번호를 출력하는 문제다.

    • 어떤 사람 i에서 시작해서, 모든 사람 j까지의 최단 거리 dist[j]를 구한다.
    • 그 최단 거리들을 전부 더한 값이 i의 점수(케빈 베이컨 수의 합)다.
    • 점수가 가장 작은 사람을 찾으면 된다.

    전략 개요

    • 친구 관계를 그래프로 만든다. (양방향 연결)
    • 모든 사람 i(1부터 N까지)에 대해 BFS를 한 번씩 수행한다.
    • BFS로 i에서 시작하는 최단 거리 배열 dist를 만든다.
    • dist[1..N]을 전부 더해 score[i]에 저장한다.
    • score가 가장 작은 사람의 번호를 출력한다.

    코드가 하는 일

    • 1) 그래프(친구 관계) 만들기 
      Set<Integer>[] set = new HashSet[N+1];
for (int i = 1; i <= N; i++) set[i] = new HashSet<>();

set[A].add(B);
set[B].add(A);
      • set[x]에는 x의 친구들이 들어간다.
      • 친구 관계는 양방향이므로 A->B, B->A 둘 다 추가한다.
    • 2) 각 사람을 시작점으로 BFS 실행 
      for (int i = 1; i <= N; i++) {
    int[] dist = new int[N+1];
    Arrays.fill(dist, -1);

    deque.clear();
    dist[i] = 0;
    deque.add(i);

    while (!deque.isEmpty()) {
        int cur = deque.poll();

        for (int next : set[cur]) {
            if (dist[next] == -1) {
                dist[next] = dist[cur] + 1;
                deque.add(next);
            }
        }
    }
}
      • dist는 시작점 i에서 각 사람까지의 최단 거리를 저장한다.
      • -1은 아직 방문하지 않음을 의미한다.
      • BFS는 가까운 순서(거리 0, 1, 2, ...)로 퍼져나가므로 처음 도착한 거리가 최단 거리다.
    • 3) dist[next] = dist[cur] + 1; 를 왜 이렇게 하는가 
      dist[next] = dist[cur] + 1;
      • cur에서 next로 가는 건 간선 1개를 더 이용하는 것이다.
      • 그래서 i -> cur까지의 거리(dist[cur])에 1을 더한 값이 i -> next의 거리가 된다.
      • 그런데 BFS는 거리 순서대로 탐색하므로, dist[next]를 처음 채우는 순간이 곧 최단 거리 확정이다.
      예시(층별 탐색 느낌)

i(0)  --- 친구들(1) --- 그 친구들의 친구들(2) --- ...

현재 cur가 거리 1이면,
cur의 이웃 next는 거리 2가 된다.
그래서 dist[next] = dist[cur] + 1;
    • 4) 거리 합을 점수로 저장 
      int sum = 0;
for (int j = 1; j <= N; j++) {
    sum += dist[j];
}
score[i] = sum;
      • i에서 모든 사람까지의 최단 거리 합이 i의 케빈 베이컨 수(합)이다.
    • 5) 가장 작은 점수의 사람 찾기 
      int minValue = Integer.MAX_VALUE;
int minIdx = 0;

for (int i = 1; i <= N; i++) {
    if (minValue > score[i]) {
        minValue = score[i];
        minIdx = i;
    }
}
      • 점수가 더 작은 사람을 만나면 갱신한다.
      • >만 사용하므로 점수가 같을 때는 기존 번호가 유지되어 더 작은 번호가 자연스럽게 선택된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • BFS 1번은 O(N + M).
      • 모든 정점에서 BFS를 수행하므로 O(N(N + M)).
    • 공간 복잡도
      • 그래프 저장 O(N + M), 거리 배열 O(N) → O(N + M).

    문제 1393 — 음하철도 구구팔

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    음하철도 구구팔 (백준 1393)은
    정류장 좌표 (xs, ys)가 주어지고, 열차가 현재 (xe, ye)에서 시작해 매 초 (dx, dy) 방향으로 직선 이동할 때, 정류장에 가장 가까워지는 순간 열차에서 뛰어내릴 좌표 (x, y)를 구하는 문제다. (정답 좌표는 항상 정수로 주어진다.)

    • 열차가 지나가는 길은 (xe, ye)에서 시작하는 한 방향의 직선 경로다.
    • 그 경로 위에서 정류장 (xs, ys)까지의 거리가 최소가 되는 정수 좌표를 찾아 출력한다.
    • 핵심은 (dx, dy)를 그대로 쓰면 중간의 정수 좌표를 건너뛸 수 있으니, 방향 벡터를 최소 단위로 줄여서 검사하는 것이다.

    전략 개요

    • g = gcd(|dx|, |dy|)를 구한다.
    • 이동을 (dx, dy)가 아니라 (dx/g, dy/g)로 바꾼다.
      • 이렇게 하면 열차 경로 위의 “가능한 정수 좌표”를 한 칸도 빠짐없이 순서대로 보게 된다.
    • 현재 위치를 (curX, curY) = (xe, ye)로 두고, 다음 위치 (curX + stepX, curY + stepY)로 갔을 때 정류장까지의 거리가 더 가까워지면 계속 전진한다.
    • 더 이상 가까워지지 않는 순간의 (curX, curY)가 정답이다.

    코드가 하는 일

    • 1) 방향 벡터를 최소 단위로 줄이기 (gcd 사용) 
      int g = gcd(Math.abs(dx), Math.abs(dy));
int stepX = dx / g;
int stepY = dy / g;
      • (dx, dy)로 이동하면 정수 좌표를 여러 칸씩 “점프”할 수 있다.
      • gcd로 나눈 (stepX, stepY)는 같은 직선 위를 가면서도, 가능한 정수 좌표를 빠짐없이 따라간다.
    • 2) 거리 비교는 제곱거리로 처리 
      long dist2(int x1, int y1, int x2, int y2) {
    long dx = x1 - x2;
    long dy = y1 - y2;
    return dx * dx + dy * dy;
}
      • 실제 거리(루트)는 비교할 때 필요 없다.
      • 제곱거리는 값의 대소가 동일하므로 sqrt 없이 더 빠르고 안전하게 비교한다.
    • 3) 더 가까워지는 동안만 앞으로 이동 
      int curX = xe, curY = ye;

while (dist2(curX, curY, xs, ys) > dist2(curX + stepX, curY + stepY, xs, ys)) {
    curX += stepX;
    curY += stepY;
}
      • 정류장까지의 거리는 “직선 위를 따라가면” 어느 지점까지는 줄어들다가, 최소점을 지나면 다시 늘어난다.
      • 그래서 다음 칸이 더 가까운 동안만 이동하면, 멈춘 지점이 최단 거리의 정수 좌표가 된다.
      그림으로 보면 이런 느낌

정류장 S
   *
   |
   |   (열차가 가는 방향으로 한 칸씩 이동)
---*---*---*---*--->
  A   B   C   D

A->B->C로 갈수록 S에 가까워지다가,
C가 가장 가까우면 그 다음(D)부터는 멀어지므로 C에서 멈춘다.
    • 4) 정답 출력 
      System.out.println(curX + " " + curY);
      • 가장 가까운 위치에서 뛰어내릴 좌표를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 한 칸씩 전진하는 횟수를 K라 하면 거리 비교를 K번 수행 → O(K).
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).
