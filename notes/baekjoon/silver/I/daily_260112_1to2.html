    문제 1325 — 효율적인 해킹

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    효율적인 해킹 (백준 1325)는
    신뢰 관계(단방향 그래프)가 주어졌을 때, 어떤 컴퓨터를 해킹했을 때 가장 많은 컴퓨터를 해킹할 수 있는지를 구하는 문제다.

    • 입력 A B는 “A가 B를 신뢰한다”는 뜻이다.
    • B가 해킹되면, B를 신뢰하는 A도 연쇄적으로 해킹할 수 있다.
    • 따라서 B에서 시작하면 도달 가능한 컴퓨터 수(정점 수)를 세어야 한다.
    • 도달 가능한 수가 최대가 되는 시작점(컴퓨터 번호)을 모두 출력한다.

    전략 개요

    • 문제의 전파 방향을 그대로 쓰면 “B → A”로 이동해야 한다.
    • 따라서 입력 A B를 받을 때 인접 리스트를 computers[B].add(A)로 역방향 그래프로 만든다.
    • 각 노드 i를 시작점으로 BFS(또는 DFS)를 돌려, 방문 가능한 노드 수를 센다.
    • 그 중 최대 방문 수를 갱신하고, 최대값과 같은 노드만 모아 출력한다.

    코드가 하는 일

    • 1) 입력을 역방향 인접 리스트로 저장 
      List<Integer>[] computers = new ArrayList[N + 1];
      for (int i = 1; i <= N; i++) computers[i] = new ArrayList<>();
      
      for (int i = 0; i < M; i++) {
          st = new StringTokenizer(br.readLine());
      
          int computer1 = Integer.parseInt(st.nextToken());
          int computer2 = Integer.parseInt(st.nextToken());
      
          computers[computer2].add(computer1);
      }
      • A B는 “A가 B를 신뢰”이므로, 해킹 전파는 B → A로 이동한다.
      • 그래서 computers[B]에 A를 넣어 “B에서 갈 수 있는 노드들”을 만든다.
    • 2) 모든 시작점 i에 대해 도달 가능한 정점 수 계산 
      int[] reach = new int[N + 1];
      int max = 0;
      
      for (int i = 1; i <= N; i++) {
          reach[i] = cnt(i, computers, N);
          if (reach[i] > max) max = reach[i];
      }
      • cnt(i)는 i에서 시작했을 때 BFS로 방문 가능한 노드 수를 반환한다.
      • 각 i에 대해 결과를 저장하고 최대값을 갱신한다.
    • 3) 최대 도달 수를 가진 노드만 출력 
      for (int i = 1; i <= N; i++) {
          if (reach[i] == max) sb.append(i).append(" ");
      }
      
      System.out.print(sb.toString());
      • 최대값과 동일한 노드 번호를 공백으로 이어 출력한다.
    • 4) cnt(): BFS로 방문 가능한 노드 수 세기 
      static int cnt(int computer, List<Integer>[] computers, int size) {
          boolean[] visited = new boolean[size + 1];
          ArrayDeque<Integer> deque = new ArrayDeque<>();
      
          visited[computer] = true;
          deque.add(computer);
      
          int cnt = 0;
          while (!deque.isEmpty()) {
              int cur = deque.poll();
              cnt++;
      
              for (int i : computers[cur]) {
                  if (!visited[i]) {
                      visited[i] = true;
                      deque.add(i);
                  }
              }
          }
      
          return cnt;
      }
      • 시작 노드를 큐에 넣고, 방문 체크 배열로 중복 방문을 막는다.
      • 큐에서 하나씩 꺼내며 인접 노드를 확장하고, 방문한 노드 수를 누적한다.
      • 최종적으로 방문한 정점 개수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 노드(최대 N개)마다 BFS를 수행한다.
      • BFS 1회는 최악의 경우 간선/정점을 모두 훑으므로 O(N + M).
      • 총합은 O(N * (N + M))이다.
    • 공간 복잡도
      • 인접 리스트: O(N + M)
      • 각 BFS의 방문 배열/큐: O(N)
      • 전체적으로 O(N + M) 수준이다.

    문제 1342 — 행운의 문자열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    행운의 문자열 (백준 1342)는
    주어진 문자열의 문자를 재배열했을 때, 인접한 두 문자가 같은 경우가 한 번도 없는 문자열(행운의 문자열)이 총 몇 개인지 세는 문제다.

    • 문자열 길이는 최대 10이지만, 같은 문자가 여러 번 등장할 수 있다.
    • 단순 순열 생성은 중복이 폭발하므로, 문자 개수(빈도) 기반으로 조합을 세는 게 핵심이다.
    • 인접 문자 중복 금지 조건 때문에 직전에 쓴 문자와 같은 문자는 다음에 사용할 수 없다.

    전략 개요

    • 문자열을 직접 섞지 말고, 알파벳 26개의 빈도 배열로 관리한다.
    • DFS(백트래킹)로 한 글자씩 선택한다.
      • 아직 남은 문자가 있고(s[i] > 0)
      • 직전에 고른 문자(prev)와 다르면 선택 가능
    • 한 글자를 선택하면 빈도를 1 줄이고 다음 깊이로 진행, 복귀 시 다시 1 늘린다.
    • 남은 글자 수(left)가 0이면 유효한 문자열 1개를 만든 것이므로 1을 반환한다.

    코드가 하는 일

    • 1) 입력 문자열을 빈도 배열로 변환 
      String line = br.readLine();
      int[] s = new int[26];
      
      for (char c : line.toCharArray()) {
          s[c - 'a']++;
      }
      • s[i]는 i번째 알파벳(0=a, 1=b, …)의 남은 개수다.
      • 이 구조를 쓰면 같은 문자 때문에 생기는 중복 순열을 자동으로 제거할 수 있다.
    • 2) DFS로 가능한 경우의 수 세기 
      int cnt = dfs(s, -1, line.length());
      System.out.print(cnt);
      • prev를 -1로 두면 “이전에 선택한 문자가 없음”을 의미한다.
      • 총 길이만큼 문자를 채워야 하므로 left = line.length()로 시작한다.
    • 3) dfs(): 남은 문자로 만들 수 있는 행운의 문자열 개수 반환 
      private static int dfs(int[] s, int prev, int left) {
          if (left == 0) return 1;
      
          int cnt = 0;
      
          for (int i = 0; i < 26; i++) {
              if (s[i] == 0) continue;
              if (i == prev) continue;
      
              s[i]--;
              cnt += dfs(s, i, left - 1);
              s[i]++;
          }
      
          return cnt;
      }
      • left == 0이면 모든 자리를 채웠고, 조건을 지키며 완성한 문자열 1개이므로 1 반환.
      • 각 문자 i에 대해
        • 남은 개수가 0이면 선택 불가
        • 직전 문자와 같은 i면 인접 중복이므로 선택 불가
        • 그 외에는 선택 → 재귀 → 복구(백트래킹)
      • 이렇게 누적한 값이 “현재 상태에서 만들 수 있는 행운의 문자열 개수”가 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최악의 경우(모든 문자가 서로 다름) 길이를 NN! 수준으로 증가한다.
      • 하지만 N ≤ 10이고, 같은 문자가 많을수록 경우의 수가 크게 줄어든다.
      • 따라서 백트래킹으로 충분히 통과 가능한 범위다.
    • 공간 복잡도
      • 빈도 배열 s[26]는 상수 크기 → O(1)
      • 재귀 깊이는 최대 문자열 길이 N → O(N)
