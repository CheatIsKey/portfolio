    문제 1421 — 나무꾼 이다솜

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    나무꾼 이다솜 (백준 1421)은
    길이가 서로 다른 나무 N개가 주어졌을 때, 어떤 길이 L로 나무를 잘라 길이 L짜리 조각들을 팔아 얻는 이익의 최댓값을 구하는 문제다.

    • 길이 L짜리 나무 조각 1개를 팔면 수익은 W * L이다.
    • 나무를 자를 때마다 비용 C가 든다.
    • 어떤 L을 선택하느냐에 따라 만들 수 있는 조각 수와 자르는 횟수(비용)가 달라진다.
    • 모든 가능한 L(1부터 최댓값까지)를 시도해 이익의 최댓값을 찾으면 된다.

    전략 개요

    • 자르는 조각의 길이를 L이라고 두고, L = 1부터 가장 긴 나무 길이까지 전부 시도한다.
    • 각 나무 길이 wood에 대해
      • 만들 수 있는 조각 수: slices = wood / L
      • 나머지: rest = wood % L
      • 자르는 횟수(비용 계산용):
        • rest == 0이면 정확히 딱 떨어지므로 절단 횟수는 slices - 1
        • rest != 0이면 남는 부분이 생기므로 절단 횟수는 slices
      • 이 나무의 이익: slices * W * L - cost * C
    • 이익이 0 이하인 나무는 아예 팔지 않는 게 낫기 때문에 합산하지 않는다.
    • 모든 나무의 이익을 합친 값의 최댓값을 출력한다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int N = Integer.parseInt(st.nextToken());
int C = Integer.parseInt(st.nextToken());
int W = Integer.parseInt(st.nextToken());

int[] woods = new int[N];
for (int i = 0; i < N; i++) {
    woods[i] = Integer.parseInt(br.readLine());
}

Arrays.sort(woods);
      • 가장 긴 나무 길이를 쉽게 알기 위해 정렬한다. (마지막 값이 최댓값)
    • 2) 가능한 모든 길이 L(코드에서는 maxLen) 시도 
      int maxLen = 1;
long max = 0;

while (maxLen <= woods[N-1]) {
    long totalSum = 0;
    ...
    max = Math.max(totalSum, max);
    maxLen++;
}
      • L은 최소 1부터 최대 maxWood까지가 후보가 된다.
      • 각 L마다 전체 이익 totalSum을 계산해 최댓값을 갱신한다.
    • 3) 한 나무에서 만들 수 있는 조각 수와 절단 비용 계산 
      int slices = wood / maxLen;

if (slices == 0) continue;

int rest = wood % maxLen;

int cost = 0;

if (rest == 0) {
    cost = slices - 1;
} else {
    cost = slices;
}
      • slices는 길이 L짜리 조각이 몇 개 나오는지다.
      • slices == 0이면 아예 조각을 못 만들므로 넘어간다.
      • rest가 0이면 딱 떨어져서 마지막 조각을 만들 때는 추가 절단이 필요 없다.
      • 그래서
        • rest == 0이면 절단 횟수 = slices - 1
        • rest != 0이면 절단 횟수 = slices
      예)
wood = 10, L = 5 → slices = 2, rest = 0
10을 5,5로 만들려면 한 번만 자르면 된다 → cost = 2 - 1 = 1

wood = 11, L = 5 → slices = 2, rest = 1
5,5를 만들고 남은 1이 생기므로 두 번 자르는 경우가 된다 → cost = 2
    • 4) 이익 계산 후, 손해면 제외 
      long price = (long) slices * W * maxLen - (long) cost * C;

if (price > 0) totalSum += price;
      • 판매 수익: slices * (W * L)
      • 절단 비용: cost * C
      • 이익이 0 이하이면 팔지 않는 게 더 낫기 때문에 합산하지 않는다.
    • 5) 정답 출력 
      System.out.println(max);
      • 모든 L에 대해 계산한 전체 이익의 최댓값을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 가능한 길이 L의 개수는 최대 maxWood(가장 긴 나무 길이)다.
      • 각 L마다 모든 나무 N개를 확인 → O(N * maxWood).
    • 공간 복잡도
      • 나무 배열 외 추가 자료구조 없음 → O(N).

    문제 1446 — 지름길

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    지름길 (백준 1446)은
    0부터 D까지 이어진 고속도로가 있고, 중간중간 이용할 수 있는 지름길(시작, 끝, 거리)이 주어질 때 0에서 D까지 가는 최소 거리를 구하는 문제다.

    • 기본 도로는 1씩 이동하면 거리 1이 늘어난다. (즉, i → i+1 비용 1)
    • 지름길은 start → end로 바로 이동하며 비용이 dist다.
    • 단, 지름길이 이득이 되려면 end - start > dist여야 한다.
    • 또한 목적지 D를 넘어가는 지름길은 사용할 수 없다.

    전략 개요

    • dp[x]를 0에서 x까지 가는 최소 거리로 둔다.
    • 기본값은 지름길을 하나도 안 쓰는 경우이므로 dp[i] = i로 초기화한다.
    • 좌표 i를 0부터 D까지 순서대로 보면서
      • 일반 도로 이동: dp[i+1] = min(dp[i+1], dp[i] + 1)
      • 지름길이 있으면: dp[end] = min(dp[end], dp[i] + dist)
    • 최종 답은 dp[D]다.

    코드가 하는 일

    • 1) dp 초기화 (지름길 없이 가는 거리) 
      List<int[]>[] optimalRoute = new ArrayList[D + 1];
int[] dp = new int[D + 1];

for (int i = 0; i <= D; i++) dp[i] = i;
      • dp[i] = i는 0에서 i까지 그냥 직진(1칸씩)만 했을 때 거리다.
    • 2) 지름길 입력 저장 (쓸모없는 지름길은 제거) 
      int start = Integer.parseInt(st.nextToken());
int end = Integer.parseInt(st.nextToken());
int dist = Integer.parseInt(st.nextToken());

if (end - start <= dist || D < end) continue;

if (optimalRoute[start] == null) optimalRoute[start] = new ArrayList<>();
optimalRoute[start].add(new int[]{end, dist});
      • end - start <= dist면 지름길이 오히려 이득이 아니므로 버린다.
      • end > D면 목적지 밖으로 나가서 사용할 수 없으므로 버린다.
      • 유효한 지름길만 start 인덱스에 모아 둔다.
    • 3) 0부터 D까지 순회하며 dp 갱신 
      for (int i = 0; i < D; i++) {
    dp[i + 1] = Math.min(dp[i + 1], dp[i] + 1);

    if (optimalRoute[i] != null) {
        for (int[] route : optimalRoute[i]) {
            int end = route[0];
            int dist = route[1];

            dp[end] = Math.min(dp[end], dp[i] + dist);
        }
    }
}
      • 기본 도로로 한 칸 가는 경우를 먼저 반영한다.
      • 현재 위치 i에서 시작하는 지름길이 있다면 그 지름길을 타는 경우도 반영한다.
      • 좌표를 작은 것부터 순서대로 보므로, dp[i]는 이미 그 시점의 최솟값에 가깝게 갱신된 상태다.
      핵심 갱신식

1) 일반 도로:
dp[i+1] = min(dp[i+1], dp[i] + 1)

2) 지름길:
dp[end] = min(dp[end], dp[i] + dist)
    • 4) 정답 출력 
      System.out.println(dp[D]);
      • 0에서 D까지의 최소 거리를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 좌표를 0부터 D까지 한 번 순회 → O(D).
      • 각 지름길은 한 번씩만 처리 → 총 O(N).
      • 합치면 O(D + N).
    • 공간 복잡도
      • dp 배열 O(D), 지름길 저장 O(N) → O(D + N).
