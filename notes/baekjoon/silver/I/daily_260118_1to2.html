    문제 1495 — 기타리스트

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    기타리스트 (백준 1495)는
    시작 볼륨 S에서 출발해, i번째 곡을 연주하기 전마다 볼륨을 +V[i] 또는 -V[i]로 바꾸되 볼륨이 항상 0 이상 M 이하가 되도록 해야 한다. 모든 곡을 순서대로 연주했을 때, 마지막 곡에서 가능한 볼륨 중 최댓값을 구하는 문제다. 가능한 볼륨이 하나도 없으면 -1을 출력한다.

    • 각 곡마다 볼륨을 올리거나 내리는 두 가지 선택지가 있다.
    • 중간에 한 번이라도 가능한 볼륨이 0개가 되면 이후는 진행 불가능이다.
    • “i번째 곡 직전까지 가능한 볼륨들”을 상태로 관리하는 DP가 핵심이다.

    전략 개요

    • dp[x]를 “현재 곡까지 진행했을 때 볼륨 x가 가능한가?”로 둔다.
    • 초기에는 시작 볼륨만 가능하므로 dp[S] = true.
    • 각 곡 i에 대해 다음 상태 next를 새로 만들고,
      • 현재 가능한 볼륨 j에 대해
      • j + V[i]가 범위 안이면 next[j + V[i]] = true
      • j - V[i]가 범위 안이면 next[j - V[i]] = true
    • 한 곡 처리가 끝나면 dp = next로 갱신한다.
    • 마지막에 dp에서 true인 가장 큰 인덱스를 찾으면 정답이다.

    코드가 하는 일

    • 1) 입력 처리 및 시작 상태 설정 
      int N = Integer.parseInt(st.nextToken());
int S = Integer.parseInt(st.nextToken());
int M = Integer.parseInt(st.nextToken());

int[] volumes = new int[N];

st = new StringTokenizer(br.readLine());
for (int i = 0; i < N; i++) volumes[i] = Integer.parseInt(st.nextToken());

boolean[] dp = new boolean[M + 1];
dp[S] = true;
      • volumes[i]는 i번째 곡 전에 바꿀 수 있는 볼륨 차이 V[i]다.
      • dp[S] = true로 시작 볼륨만 가능하다고 표시한다.
    • 2) 곡마다 가능한 볼륨 집합을 갱신 
      for (int i = 0; i < N; i++) {
    int volume = volumes[i];
    boolean[] next = new boolean[M + 1];

    for (int j = 0; j <= M; j++) {
        if (dp[j]) {
            if (j + volume <= M) {
                next[j + volume] = true;
            }
            if (0 <= j - volume) {
                next[j - volume] = true;
            }
        }
    }

    dp = next;
}
      • next는 “다음 곡으로 넘어갔을 때 가능한 볼륨들”을 표시하는 배열이다.
      • 현재 가능한 볼륨 j에서
        • 볼륨을 올린 j + volume
        • 볼륨을 내린 j - volume
        두 경우를 범위(0..M) 안에서만 체크한다.
      • 한 곡이 끝나면 dp = next로 갱신해서 다음 곡으로 진행한다.
      이 DP가 의미하는 것

dp[j] = true  → 현재 곡까지 왔을 때 볼륨 j로 연주하는 것이 가능하다
next[...]      → 다음 곡 직전에 가능한 볼륨들

곡을 하나 처리할 때마다 가능한 볼륨 후보가 갱신된다.
    • 3) 마지막 곡에서 가능한 볼륨 중 최댓값 찾기 
      for (int i = M; i >= 0; i--) {
    if (dp[i]) {
        System.out.println(i);
        return;
    }
}

System.out.println(-1);
      • M부터 0까지 내려오며 true인 첫 볼륨이 최댓값이다.
      • 끝까지 없으면 마지막 곡을 연주할 수 없는 경우이므로 -1을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 곡이 N개이고, 각 곡마다 볼륨 0..M을 확인 → O(NM).
      • (N ≤ 50, M ≤ 1000 이라 충분히 빠르다.)
    • 공간 복잡도
      • 불리언 배열 2개 크기 M+1 사용 → O(M).

    문제 1497 — 기타콘서트

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    기타콘서트 (백준 1497)는
    기타 N개가 각각 연주할 수 있는 곡 정보(Y/N)로 주어질 때, 기타를 몇 개 골라서 연주할 수 있는 곡 수를 최대로 만들고, 그 최대 곡 수를 만들 수 있는 방법들 중에서 필요한 기타 개수를 최소로 출력하는 문제다. 만약 어떤 곡도 연주할 수 없다면 -1을 출력한다.

    • 목표 1: 연주 가능한 곡 수 최대로 만들기
    • 목표 2: (목표 1을 만족하는 경우 중) 선택한 기타 개수 최소로 만들기
    • N ≤ 10이라 모든 기타를 선택/비선택하는 완전탐색(부분집합)이 가능하다.

    전략 개요

    • 각 기타를 “선택한다 / 선택하지 않는다”로 나누는 DFS(부분집합 탐색)를 한다.
    • 탐색 중에 현재까지 연주 가능한 곡들을 songs[]로 관리한다.
    • DFS 파라미터
      • idx : 현재 보고 있는 기타 번호
      • cnt : 지금까지 선택한 기타 개수
      • songCnt : 지금까지 확보된(연주 가능한) 곡 수
    • 끝까지 갔을 때(idx == N)
      • songCnt가 더 크면 최대값 갱신 + 기타 개수 저장
      • songCnt가 같으면 기타 개수 더 작은 것으로 갱신
    • 선택 분기에서는 곡 상태가 바뀌므로, 바뀐 인덱스만 기록했다가 DFS가 끝나면 원복(backtracking)한다.

    코드가 하는 일

    • 1) 입력을 연주 가능 여부 배열로 저장 
      N = Integer.parseInt(st.nextToken());
M = Integer.parseInt(st.nextToken());

availableList = new boolean[N][M];

for (int i = 0; i < N; i++) {
    st = new StringTokenizer(br.readLine());
    st.nextToken();
    String available = st.nextToken();

    for (int j = 0; j < M; j++) {
        if (available.charAt(j) == 'Y') {
            availableList[i][j] = true;
        }
    }
}
      • availableList[i][j] = i번째 기타로 j번째 곡을 연주할 수 있으면 true.
      • 기타 이름은 필요 없어서 읽기만 하고 버린다.
    • 2) DFS 시작 및 결과 출력 조건 
      songs = new boolean[M];
dfs(0, 0, 0);

if (maxSongs == 0) System.out.println(-1);
else System.out.println(minGuitars);
      • songs는 현재 선택한 기타들로 연주 가능한 곡들을 표시한다.
      • DFS를 끝낸 뒤 최대 곡 수가 0이면 아무 곡도 못 치는 경우이므로 -1을 출력한다.
    • 3) DFS 기본 구조: 선택/비선택 분기 
      private static void dfs(int idx, int cnt, int songCnt) {
    if (idx == N) {
        ...
        return;
    }

    dfs(idx + 1, cnt, songCnt);

    ...
    dfs(idx + 1, cnt + 1, songCnt + changedCnt);

    ...
}
      • 현재 기타(idx)를 선택하지 않는 경우를 먼저 탐색한다.
      • 그 다음 선택하는 경우를 탐색한다.
    • 4) 리프(종료)에서 최대 곡 수 / 최소 기타 수 갱신 
      if (idx == N) {
    if (songCnt > maxSongs) {
        maxSongs = songCnt;
        minGuitars = cnt;
    } else if (songCnt == maxSongs && songCnt > 0 && cnt < minGuitars) {
        minGuitars = cnt;
    }
    return;
}
      • songCnt가 더 크면 무조건 더 좋은 해다. (목표 1 우선)
      • songCnt가 같으면 기타 개수 cnt가 작은 해를 선택한다. (목표 2)
      • songCnt > 0 조건은 “연주 가능한 곡이 하나도 없는 경우”를 최소 기타 후보로 잡지 않도록 하기 위한 장치다.
    • 5) 기타를 선택했을 때 새로 추가되는 곡만 반영하고, 변경 내역 기록 
      int[] changed = new int[M];
int changedCnt = 0;

for (int i = 0; i < M; i++) {
    if (availableList[idx][i] && !songs[i]) {
        songs[i] = true;
        changed[changedCnt++] = i;
    }
}
      • 이 기타로 연주 가능하고, 아직 확보되지 않은 곡만 새로 true로 만든다.
      • 나중에 원복하기 위해 바뀐 곡의 인덱스만 changed에 기록한다.
      • changedCnt는 “이번 기타 선택으로 새로 확보된 곡 수”이며, 그대로 songCnt 증가분이다.
    • 6) 선택 분기 DFS 후, 변경된 곡 상태 원복(backtracking) 
      dfs(idx + 1, cnt + 1, songCnt + changedCnt);

for (int i = 0; i < changedCnt; i++) {
    songs[changed[i]] = false;
}
      • 선택 분기 탐색이 끝나면, 다음 분기에 영향을 주지 않도록 곡 상태를 원래대로 되돌린다.
      • 전체를 다시 초기화하지 않고 “변경된 부분만” 되돌려서 효율적으로 백트래킹한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 기타 선택/비선택으로 부분집합 개수는 2^N.
      • 각 선택 분기에서 곡 M개를 확인하므로 최악의 경우 O(2^N · M).
      • (N ≤ 10, M ≤ 50이라 충분히 가능하다.)
    • 공간 복잡도
      • 연주 가능 배열 availableList는 O(NM), 곡 상태 songs는 O(M).
      • 추가로 재귀 호출 깊이는 최대 N → 전체는 O(NM) 수준이다.
