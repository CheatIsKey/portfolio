    문제 2839 — 설탕 배달

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    설탕 배달 (백준 2839)은
    정확히 N 킬로그램의 설탕을 배달해야 할 때, 3kg 봉지와 5kg 봉지를 사용하여 가져가야 할 봉지의 최소 개수를 구하는 문제다.

    • 봉지는 3kg, 5kg 두 종류만 있다.
    • 정확하게 N킬로그램을 만들 수 없으면 -1을 출력한다.
    • 최소 개수를 구하기 위해서는 가장 무거운 5kg 봉지를 최대한 많이 사용하는 것이 유리하다.

    전략 개요

    • 전형적인 그리디(Greedy) 문제로, 5kg 봉지를 최대한 많이 쓰는 상황부터 가정한다.
    • 우선 N을 5로 나눈 몫만큼 5kg 봉지를 챙긴다.
    • 남은 무게(remain)가 3으로 나누어 떨어지는지 확인한다.
      • 나누어 떨어지면: (5kg 봉지 수 + 3kg 봉지 수)가 정답이다.
      • 나누어 떨어지지 않으면: 5kg 봉지를 하나 줄이고, 남은 무게에 5kg를 더한 뒤 다시 확인한다.
    • 5kg 봉지 개수가 0개 미만이 될 때까지 해결되지 않으면 만들 수 없는 무게다.

    코드가 하는 일

    • 1) 5kg 봉지 최대 개수와 초기 나머지 계산 
      int fiveCnt = N / 5;
int remain = N % 5;
      • 일단 가능한 최대의 5kg 봉지 개수(fiveCnt)를 구한다.
      • 5kg로 채우고 남은 무게를 remain에 저장한다.
    • 2) 조건을 만족할 때까지 5kg 봉지를 줄여가며 탐색 
      while (fiveCnt >= 0) {
    if (remain % 3 == 0) {
        int threeCnt = remain / 3;
        System.out.println(fiveCnt + threeCnt);
        return;
    } else {
        fiveCnt--;
        remain += 5;
    }
}
      • 현재 남은 무게 remain이 3으로 나누어 떨어지면, 최소 봉지 개수를 찾은 것이다.
      • 그렇지 않다면 fiveCnt를 1 감소시키고, remain에 5를 더해(즉, 5kg 봉지 하나를 풀어서 나머지에 보탬) 다시 검사한다.
      예) N = 18일 때
1. fiveCnt = 3, remain = 3
   → remain(3) % 3 == 0 이므로 정답은 3 + 1 = 4

예) N = 4일 때
1. fiveCnt = 0, remain = 4 (X)
2. fiveCnt = -1, remain = 9 (loop 종료)
   → -1 출력
    • 3) 불가능한 경우 처리 
      System.out.println(-1);
      • 반복문이 fiveCnt < 0이 되어 종료될 때까지 리턴되지 않았다면, 정확한 무게를 만들 수 없는 경우다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최대 반복 횟수는 N/5번에 비례한다.
      • N ≤ 5000이므로 매우 빠르게 동작하며, O(N)으로 볼 수 있다.
    • 공간 복잡도
      • 별도 배열 없이 정수 변수만 사용하므로 O(1).

    문제 9012 — 괄호

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    괄호 (백준 9012)는
    주어진 괄호 문자열이 짝이 올바르게 맞는 VPS(Valid Parenthesis String)인지 판단하여 "YES" 또는 "NO"를 출력하는 문제다.

    • 여는 괄호 (와 닫는 괄호 )는 1:1로 매칭되어야 한다.
    • 닫는 괄호가 나올 때, 대응되는 여는 괄호가 이미 스택에 있어야 한다.
    • 모든 처리가 끝난 후 스택에 남은 괄호가 없어야 한다.

    전략 개요

    • 자료구조 스택(Stack)을 사용하여 괄호의 짝을 검사한다.
    • 문자열을 순회하면서:
      • (를 만나면 스택에 넣는다(push).
      • )를 만나면 스택에서 뺀다(pop). 이때 스택이 비어있다면 잘못된 문자열이다.
    • 문자열 순회가 끝난 뒤 스택이 비어있어야 올바른 괄호 문자열이다. 남은 게 있다면 (가 너무 많은 경우다.

    코드가 하는 일

    • 1) 여는 괄호와 닫는 괄호 처리 
      if (line.charAt(i) == ')') {
    if (stack.isEmpty()) {
        ok = false;
        break;
    }
    Character pop = stack.pop();
    if (pop != '(') { // (스택에 '('만 넣으므로 사실상 불필요하지만 방어적 코드)
        ok = false;
        break;
    }
} else {
    stack.push(line.charAt(i));
}
      • )가 나왔을 때 스택이 비어있으면 짝이 안 맞으므로 실패(NO) 처리한다.
      • (가 나오면 스택에 쌓아서 나중에 짝을 맞출 수 있게 대기시킨다.
    • 2) 최종 검증 및 초기화 
      if (!stack.isEmpty()) {
    ok = false;
}

if (ok) sb.append("YES").append('\n');
else sb.append("NO").append('\n');

stack.clear();
      • 반복문 종료 후 스택에 잔여물이 있으면 여는 괄호가 더 많은 것이므로 실패 처리한다.
      • 결과를 버퍼(StringBuilder)에 기록하고, 다음 테스트 케이스를 위해 stack.clear()로 비워준다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열의 길이를 N이라 할 때, 각 문자를 한 번씩 순회하므로 O(N).
      • 테스트 케이스가 T개이므로 전체는 O(T × N)이다.
    • 공간 복잡도
      • 스택에 최대 N개의 괄호가 쌓일 수 있으므로 O(N).
