    문제 1455 — 뒤집기 II

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    뒤집기 II (백준 1455)는
    N×M 동전판에서 0(앞면), 1(뒷면)이 주어질 때, 어떤 칸 (a, b)를 선택하면 (1 ≤ i ≤ a, 1 ≤ j ≤ b) 범위의 a×b 직사각형이 한 번에 뒤집히는 규칙으로, 모든 동전을 0으로 만들기 위한 최소 뒤집기 횟수를 구하는 문제다.

    • 한 번 뒤집을 때 뒤집히는 동전 개수(a×b)는 상관없고, 뒤집기 횟수는 1회로 센다.
    • 목표는 모든 칸을 0으로 만드는 최소 횟수다.
    • 핵심은 오른쪽 아래에서부터 보면서 1이 나오면 그 지점까지를 뒤집는 그리디다.

    전략 개요

    • 아래 오른쪽(가장 끝 칸)부터 위 왼쪽 방향으로 역순 탐색한다.
    • 현재 칸 (r, c)가 1이라면,
      • 이 칸을 0으로 만들 수 있는 유일한 선택은 (r, c)를 포함하는 뒤집기인데,
      • 역순 탐색에서는 지금 시점에서 (r, c)까지(0..r, 0..c)를 뒤집는 것이 최선이다.
    • 그래서 board[r][c] == 1이면
      • 뒤집기 횟수 1 증가
      • (0..r, 0..c) 범위를 전부 뒤집는다
    • 이 과정을 끝까지 하면 모든 칸이 0이 되며, 그 횟수가 최소가 된다.

    코드가 하는 일

    • 1) 입력을 2차원 보드로 변환 
      int[][] board = new int[N][M];

for (int i = 0; i < N; i++) {
    String line = br.readLine();

    for (int j = 0; j < M; j++) {
        board[i][j] = line.charAt(j) - '0';
    }
}
      • 각 줄의 문자 '0', '1'을 정수로 바꿔 board에 저장한다.
    • 2) 오른쪽 아래에서 시작해 역순으로 탐색 
      int cnt = 0;

for (int r = N - 1; r >= 0; r--) {
    for (int c = M - 1; c >= 0; c--) {
        if (board[r][c] == 1) {
            cnt++;
            reverse(board, r, c);
        }
    }
}
      • 끝 칸부터 보면, 이미 지나간(더 오른쪽/더 아래) 칸은 다시 건드릴 일이 없게 된다.
      • 현재 칸이 1이면 반드시 뒤집어서 0으로 만들어야 하므로 즉시 뒤집기를 수행한다.
      • 이때 뒤집는 범위는 규칙에 따라 (0..r, 0..c)가 된다.
      왜 역순 탐색이 유리한가?

(r,c)를 0으로 만들려면 (r,c)를 포함하는 뒤집기를 해야 한다.
뒤집기 규칙은 항상 (0..a, 0..b) 형태이므로,
(r,c)가 1인 순간에는 (a=r, b=c)로 뒤집는 선택이 가장 직접적이다.

그리고 오른쪽/아래쪽을 먼저 확정해두면,
이후 뒤집기는 왼쪽/위쪽만 영향을 주므로 이미 확정된 칸이 다시 바뀌지 않는다.
    • 3) (0..r, 0..c) 직사각형을 실제로 뒤집는 함수 
      private static void reverse(int[][] board, int r, int c) {
    for (int i = 0; i <= r; i++) {
        for (int j = 0; j <= c; j++) {
            if (board[i][j] == 0) {
                board[i][j] = 1;
            } else board[i][j] = 0;
        }
    }
}
      • 문제 규칙 그대로, 좌상단(0,0)부터 (r,c)까지 모든 칸을 뒤집는다.
      • 0이면 1로, 1이면 0으로 토글한다.
    • 4) 뒤집은 횟수 출력 
      System.out.println(cnt);
      • 그리디로 수행한 뒤집기 횟수 cnt가 최소 횟수다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최악의 경우 모든 칸에서 뒤집기를 수행할 수 있고, 한 번 뒤집을 때 최대 N×M을 토글한다.
      • 따라서 최악 시간은 O((N×M)×(N×M)) = O(N^2 M^2).
      • 단, N, M ≤ 100 범위에서는 충분히 통과 가능한 구현이다.
    • 공간 복잡도
      • 보드 배열만 사용 → O(NM).

    문제 1474 — 밑 줄

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    밑 줄 (백준 1474)는
    단어 N개를 주어진 순서대로 이어 붙이되, 단어 사이에만 _를 넣어서 전체 길이를 정확히 M으로 만들고, 그 중 사전 순으로 가장 앞서는 문자열을 출력하는 문제다.

    • _는 단어 사이에만 추가 가능하다. (처음/끝에 올 수 없음)
    • 모든 간격의 _ 개수는 최대한 같아야 한다.
    • 완전히 같게 불가능하면, 간격들 사이의 _ 개수 차이는 1이어야 한다.
    • 알파벳 순서에서 'A'..'Z' < '_' < 'a'..'z' 이다.

    전략 개요

    • 우선 단어 길이 합을 totalLen이라 하면, 넣어야 할 전체 밑줄 수는 underCnt = M - totalLen이다.
    • 단어 사이 간격 개수는 placeCnt = N - 1이다.
    • 각 간격에 최소로 깔아야 하는 밑줄 개수는
      • underAvgCnt = underCnt / placeCnt
      • restUnderCnt = underCnt % placeCnt (추가로 1개 더 줘야 하는 간격 수)
    • 모든 간격에 underAvgCnt를 먼저 배치한 다음, restUnderCnt개만큼 간격을 골라 1개씩 더 준다.
    • 사전 순을 가장 앞서게 하려면, “어느 간격에 밑줄을 1개 더 줄지”를 결정해야 한다.
    • 문자 순서가 대문자 < '_' < 소문자이므로,
      • 다음 단어가 소문자로 시작하면, 그 앞 간격에 밑줄을 더 주는 것이 문자열을 더 작게 만든다.
      • 소문자 후보가 다 처리되고 남으면, 다음 단어가 대문자로 시작하는 간격은 뒤쪽부터 채워 사전 순 손해를 최소화한다.

    코드가 하는 일

    • 1) 단어 입력과 전체 길이 합 계산 
      List<String> list = new ArrayList<>();

int totalLen = 0;

for (int i = 0; i < N; i++) {
    String word = br.readLine();
    list.add(word);
    totalLen += word.length();
}
      • totalLen은 단어만 이어 붙였을 때의 길이다.
      • 여기에 밑줄을 추가해 총 길이 M을 맞춘다.
    • 2) 밑줄 배치 기본값(평균)과 남는 개수 계산 
      int placeCnt = list.size() - 1;
int underCnt = M - totalLen;
int underAvgCnt = underCnt / placeCnt;
int restUnderCnt = underCnt % placeCnt;

int[] gap = new int[placeCnt];
Arrays.fill(gap, underAvgCnt);
      • 간격은 총 N-1개다.
      • 모든 간격에 최소로 들어갈 밑줄 수는 underAvgCnt.
      • 나머지 restUnderCnt개는 어떤 간격에 1개씩 추가해서 분배해야 한다.
      • gap[i]는 i번째 단어와 i+1번째 단어 사이의 밑줄 개수다.
    • 3) 사전 순을 앞당기기: 다음 단어가 소문자로 시작하는 간격을 앞에서부터 채움 
      for (int i = 1; i < list.size() && restUnderCnt > 0; i++) {
    char c = list.get(i).charAt(0);
    if ('a' <= c && c <= 'z') {
        gap[i - 1]++;
        restUnderCnt--;
    }
}
      • 문자 순서에서 '_'는 소문자보다 작다.
      • 따라서 다음 단어가 소문자로 시작하면, 그 앞에 밑줄을 더 추가해 두는 것이 문자열을 더 작게 만든다.
      • 사전 순 비교는 앞에서부터 결정되므로, 이런 간격은 앞쪽부터 먼저 1개씩 더 준다.
      왜 소문자 앞 간격을 먼저?

... _ a...
에서 밑줄이 하나 더 늘어나면
... __ a... 가 되어,
'_'가 더 빨리 등장하고 '_' < 'a' 이므로
전체 문자열이 더 작아진다.
    • 4) 남은 밑줄 처리: 다음 단어가 대문자인 간격은 뒤에서부터 채움 
      for (int i = list.size() - 1; i >= 1 && restUnderCnt > 0; i--) {
    char c = list.get(i).charAt(0);
    if ('A' <= c && c <= 'Z') {
        gap[i - 1]++;
        restUnderCnt--;
    }
}
      • 대문자는 '_'보다 작다. ('A'..'Z' < '_')
      • 대문자 앞에 밑줄을 더 넣으면 오히려 '_'가 빨리 등장해 사전 순에서 손해가 될 수 있다.
      • 그래서 어쩔 수 없이 대문자 앞 간격에 추가해야 한다면, 사전 순 영향이 가장 작은 뒤쪽부터 채운다.
    • 5) gap 배열대로 문자열 구성 
      for (int i = 0; i < list.size(); i++) {
    sb.append(list.get(i));
    if (i < placeCnt) {
        for (int j = 0; j < gap[i]; j++) sb.append('_');
    }
}
      • 단어를 순서대로 붙이고, 각 간격에 gap[i]만큼 밑줄을 붙인다.
      • 마지막 단어 뒤에는 밑줄을 붙이면 안 되므로 i < placeCnt일 때만 붙인다.
    • 6) 결과 출력 
      System.out.println(sb.toString());
      • 조건을 만족하면서 사전 순으로 가장 앞서는 문자열을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 단어 입력/분배 로직은 O(N).
      • 밑줄을 실제로 붙이는 과정은 최종 문자열 길이 M만큼 수행 → O(M).
      • 전체는 O(M)으로 볼 수 있다. (N ≤ 10, M ≤ 200)
    • 공간 복잡도
      • 간격 배열 gap와 결과 문자열 저장 → O(M).
