    문제 10828 — 스택

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    스택 (백준 10828)은
    정수를 저장하는 스택(Stack) 자료구조를 직접 구현하거나 라이브러리를 사용하여, 주어지는 5가지 명령(push, pop, size, empty, top)을 처리하는 기초적인 자료구조 문제다.

    • LIFO(Last In First Out): 가장 나중에 들어간 데이터가 가장 먼저 나온다.
    • 명령의 수 N이 최대 10,000개이므로, 잦은 입출력에 의한 시간 초과를 막기 위해 BufferedReader와 StringBuilder를 사용하는 것이 좋다.
    • pop과 top 명령은 스택이 비어있을 때 -1을 출력해야 하는 예외 처리가 필요하다.

    전략 개요

    • 자바의 java.util.Stack 클래스를 활용하여 스택 기능을 수행한다.
    • 명령어가 "push 123"처럼 공백을 포함할 수 있으므로 StringTokenizer로 명령어와 숫자를 분리한다.
    • 출력량이 많을 수 있으므로 System.out.println을 매번 호출하는 대신, StringBuilder에 결과를 모아 한 번에 출력한다.

    코드가 하는 일

    • 1) 명령어 파싱 및 push 처리 
      st = new StringTokenizer(br.readLine());
String s = st.nextToken();

if (s.equals("push")) {
    stack.push(Integer.parseInt(st.nextToken()));
    continue;
}
      • 한 줄을 읽어 명령어(s)를 확인한다.
      • push인 경우, 뒤이어 나오는 토큰(숫자)을 파싱하여 스택에 넣는다.
    • 2) pop과 top 처리 (예외 처리 포함) 
      if (s.equals("pop")) {
    if (stack.empty()) {
        sb.append(-1).append("\n");
    } else {
        sb.append(stack.pop()).append("\n");
    }
    continue;
}
      • pop은 스택에서 값을 꺼내 출력해야 한다.
      • 만약 스택이 비어있다면 -1을 기록하고, 아니면 pop()한 값을 기록한다.
      • top(코드의 peek() 사용) 또한 로직은 동일하되, 값을 제거하지 않고 확인만 한다.
    • 3) empty와 size 처리 
      if (s.equals("empty")) {
    if (stack.isEmpty()) {
        sb.append(1).append("\n");
    } else  {
        sb.append(0).append("\n");
    }
    continue;
}
      • empty 명령어는 비어있으면 1, 아니면 0을 출력한다. (자바의 boolean 결과를 문제 요구사항인 정수로 변환)
      • size는 stack.size()를 그대로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 명령의 개수 N만큼 반복하며, 스택의 삽입/삭제/조회는 모두 상수 시간이다.
      • 따라서 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • 스택에 최대 N개의 정수가 쌓일 수 있으므로 O(N)이다.

    문제 11399 — ATM

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    ATM (백준 11399)은
    ATM 앞에 N명의 사람들이 줄을 서 있을 때, 각 사람이 돈을 인출하는 데 걸리는 시간을 바탕으로 모든 사람이 돈을 인출하는 데 필요한 시간의 총합이 최소가 되도록 줄을 세우는 문제다.

    • 앞 사람이 늦게 끝날수록 뒷사람들은 그만큼 더 기다려야 한다.
    • 즉, 앞쪽에 서는 사람의 인출 시간이 전체 대기 시간의 총합에 더 많이 중첩되어 더해진다.
    • 시간의 합을 최소화하려면 인출 시간이 짧은 사람이 먼저 일을 처리해야 한다.

    전략 개요

    • 그리디(Greedy) 알고리즘: 각 단계에서 가장 최적인 선택(시간이 가장 적게 걸리는 사람을 먼저 배치)을 하는 것이 전체 최적해를 보장한다.
    • 주어진 시간 배열 arr를 오름차순 정렬한다.
    • 정렬된 순서대로 누적 합(Prefix Sum)을 구하면, 각 사람이 기다려야 하는 총시간이 나오며 이를 모두 더하면 정답이 된다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      Arrays.sort(arr);
      • 인출 시간이 짧은 순서대로 처리하기 위해 오름차순으로 정렬한다.
      • 이 과정이 문제 해결의 핵심(그리디)이다.
    • 2) 누적 시간 계산 (DP 활용) 
      int[] dp = new int[N];
dp[0] = arr[0];
int totalSum = dp[0];

for (int i = 1; i < N; i++) {
    dp[i] = dp[i - 1] + arr[i];
    totalSum += dp[i];
}
      • dp[i]는 i번째 사람이 돈을 인출하고 나오기까지 걸린 전체 시간이다.
      • dp[i-1](이전 사람까지의 누적 시간)에 arr[i](자기 자신의 인출 시간)를 더해 구한다.
      • 각 사람의 종료 시간(dp[i])을 totalSum에 계속 더해 최종 답을 구한다.
      예) [1, 2, 3] 순서라면
i=0: 1분 소요 (누적 1)
i=1: 1(대기) + 2(본인) = 3분 소요 (누적 1+3=4)
i=2: 3(대기) + 3(본인) = 6분 소요 (누적 1+3+6=10)

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 정렬에 O(N log N)이 소요된다.
      • 이후 반복문은 O(N)이므로 전체 복잡도는 O(N log N)이다. (N ≤ 1,000)
    • 공간 복잡도
      • 입력 배열 arr와 누적 합 배열 dp를 사용하므로 O(N)이다.
