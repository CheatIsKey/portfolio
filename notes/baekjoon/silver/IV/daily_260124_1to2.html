    문제 11047 — 동전 0

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    동전 0 (백준 11047)은
    동전 N종류가 주어졌을 때, 그 가치의 합을 K로 만들기 위해 필요한 동전 개수의 최솟값을 구하는 문제다.

    • 동전의 가치는 오름차순으로 주어지며, 큰 단위의 동전은 항상 작은 단위 동전의 배수라는 조건이 있다.
    • 이 조건 덕분에 단순히 큰 동전부터 최대한 많이 사용하는 그리디(Greedy) 방식이 최적해를 보장한다.
    • 목표 금액 K를 0으로 만들 때까지 가장 큰 동전부터 차감해 나간다.

    전략 개요

    • 그리디 알고리즘: 가치가 큰 동전부터 사용하여 금액을 채우는 것이 개수를 줄이는 가장 빠른 방법이다.
    • 입력받은 동전들을 배열에 저장한 뒤, 가장 큰 가치의 동전(마지막 인덱스)부터 역순으로 탐색한다.
    • 현재 동전 가치가 남은 금액 K보다 작거나 같다면:
      • 해당 동전으로 낼 수 있는 최대 개수(K / coin)를 카운트에 더한다.
      • 남은 금액을 K % coin으로 갱신한다.

    코드가 하는 일

    • 1) 입력 및 동전 정보 저장 
      int[] wallet = new int[N];

for (int i = 0; i < N; i++) {
    wallet[i] = Integer.parseInt(br.readLine());
}
      • 동전의 개수 N과 목표 금액 K를 입력받고, N개의 동전 가치를 배열 wallet에 저장한다.
      • 입력이 오름차순으로 주어지므로 별도의 정렬은 필요 없다.
    • 2) 큰 동전부터 그리디 탐색 
      for (int i = N - 1; i >= 0; i--) {
    if (wallet[i] <= K) {
        cnt += K / wallet[i];
        K %= wallet[i];
    }

    if (K == 0) break;
}
      • 배열의 끝(가장 큰 동전)부터 0번 인덱스(가장 작은 동전)까지 역순으로 순회한다.
      • 현재 동전 wallet[i]로 K를 구성할 수 있다면(wallet[i] <= K):
        • 몫(K / wallet[i])만큼 개수를 더하고,
        • 나머지(K % wallet[i])로 K를 업데이트한다.
      • K가 0이 되면 더 이상 계산할 필요가 없으므로 반복을 종료한다.
      예) K=4200, 동전=[..., 1000, 5000]
1. 5000원: 4200보다 크므로 패스
2. 1000원: 4200 / 1000 = 4개 사용, 남은 K = 200
... (이후 200원은 더 작은 동전으로 처리)
    • 3) 결과 출력 
      System.out.println(cnt);
      • 최종적으로 계산된 최소 동전 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 동전의 종류가 N개일 때, 배열을 한 번 순회하므로 O(N)이다.
    • 공간 복잡도
      • 동전의 가치를 저장할 배열이 필요하므로 O(N)이다.

    문제 1920 — 수 찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수 찾기 (백준 1920)는
    N개의 정수가 주어졌을 때, 이후에 주어지는 M개의 정수들이 그 안에 존재하는지 여부를 빠르게 알아내어 1(존재) 또는 0(존재 안 함)을 출력하는 문제다.

    • N과 M이 최대 100,000이므로, 단순한 순차 탐색(Linear Search)으로는 O(N×M)이 되어 시간 초과가 발생한다.
    • 따라서 탐색 시간을 획기적으로 줄일 수 있는 이분 탐색(Binary Search)이나 해시 등을 사용해야 한다.
    • 이 코드는 이분 탐색을 사용하여 구현되었다.

    전략 개요

    • 정렬(Sorting): 이분 탐색을 수행하기 위한 전제 조건은 데이터가 정렬되어 있어야 한다는 점이다. 먼저 numbers 배열을 오름차순으로 정렬한다.
    • 이분 탐색(Binary Search):
      • 찾고자 하는 값과 중간값(mid)을 비교한다.
      • 값이 중간값보다 작으면 왼쪽 절반을, 크다면 오른쪽 절반을 탐색 범위를 좁혀가며 찾는다.
      • 이 방식은 한 번 비교할 때마다 탐색 범위가 반으로 줄어들어 O(log N)의 속도를 보장한다.

    코드가 하는 일

    • 1) 데이터 입력 및 정렬 
      int[] numbers = new int[N];
// ... 입력 루프 ...
Arrays.sort(numbers);
      • N개의 수를 입력받은 뒤 Arrays.sort()를 사용하여 오름차순 정렬한다.
      • 정렬은 이분 탐색을 위해 반드시 필요하다.
    • 2) 각 쿼리에 대해 이분 탐색 수행 
      for (int i = 0; i < M; i++) {
    int num = Integer.parseInt(st.nextToken());

    if (binarySearch(numbers, num)) {
        sb.append(1).append('\n');
    } else  {
        sb.append(0).append('\n');
    }
}
      • 찾아야 할 수 M개를 하나씩 읽어 binarySearch 함수로 존재 여부를 확인한다.
      • 결과에 따라 1 또는 0을 StringBuilder에 저장한다.
    • 3) 이분 탐색 구현 
      while (left <= right) {
    int mid = (left + right) / 2;

    if (numbers[mid] == num) {
        return true;
    } else if (numbers[mid] > num) {
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}
return false;
      • left와 right 인덱스를 이용해 탐색 범위를 조절한다.
      • 중간값 numbers[mid]가 찾는 수보다 크면 왼쪽 부분(right = mid - 1)을, 작으면 오른쪽 부분(left = mid + 1)을 탐색한다.
      • 찾으면 true, 반복문이 끝날 때까지 못 찾으면 false를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 정렬에 O(N log N)이 소요된다.
      • M개의 수에 대해 각각 이분 탐색(O(log N))을 수행하므로 탐색 부분은 O(M log N)이다.
      • 전체 복잡도는 O((N + M) log N)으로, 시간 제한 내에 충분히 통과한다.
    • 공간 복잡도
      • 데이터 저장을 위한 배열 numbers 크기만큼 필요하므로 O(N)이다.
