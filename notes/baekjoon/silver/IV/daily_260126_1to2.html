    문제 1065 — 한수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    한수 (백준 1065)는
    어떤 양의 정수 X의 각 자릿수가 등차수열을 이룰 때, 이를 '한수'라고 한다.
    자연수 N이 주어졌을 때, 1부터 N까지의 숫자 중 한수의 개수를 구하는 문제다.

    • 등차수열: 연속된 두 수의 차이가 일정한 수열이다.
    • 1~99 범위: 1자리 수나 2자리 수는 항상 등차수열을 이룬다. (비교할 차이가 하나뿐이거나 없기 때문)
    • 100 이상: 100의 자리, 10의 자리, 1의 자리 숫자를 비교하여 (백 - 십) == (십 - 일) 인지 확인해야 한다.

    전략 개요

    • 두 자리 수 이하 처리: N이 99 이하인 경우, 1부터 N까지 모든 수가 한수이므로 N을 그대로 출력한다.
    • 세 자리 수 이상 처리:
      • 이미 1~99는 모두 한수이므로 cnt를 99로 초기화하고 시작한다.
      • 100부터 N까지 반복하며 각 숫자의 자릿수를 분리한다.
      • 각 자리의 차이가 일정한지(등차수열 조건) 확인하여 카운트를 증가시킨다.
      • 입력 N은 최대 1,000이지만, 1,000은 한수가 아니므로 3자리 수 로직으로(또는 루프 내 로직으로) 자연스럽게 걸러진다.

    코드가 하는 일

    • 1) 99 이하인 경우 예외 처리 
      if (N <= 99) {
    sb.append(N);
}
      • 1부터 99까지는 모두 한수이므로 계산 없이 N을 답으로 확정한다.
    • 2) 100 이상인 경우: 자릿수 분리 및 검사 
      else {
    int cnt = 99; // 1~99는 이미 확보

    for (int i = 100; i <= N; i++) {
        int x = i;
        int third = x / 100;    // 백의 자리
        x %= 100;
        int second = x / 10;    // 십의 자리
        x %= 10;
        int first = x;          // 일의 자리

        if ((third - second) == (second - first)) {
            cnt++;
        }
    }
    sb.append(cnt);
}
      • third(백), second(십), first(일) 자리를 수학적 연산(/, %)으로 분리한다.
      • 등차수열 조건인 백 - 십 == 십 - 일(공차 일정)을 만족하면 카운트를 증가시킨다.
      • 1,000의 경우 코드는 third=10, second=0, first=0으로 처리되어 10 != 0이므로 카운트되지 않는다. (문제 범위상 올바른 동작)

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최대 N은 1,000이므로 루프는 최대 900회 정도 돈다. 이는 상수 시간이므로 O(1)이나 다름없다. (엄밀히는 O(N))
    • 공간 복잡도
      • 별도의 배열 없이 몇 개의 변수만 사용하므로 O(1)이다.

    문제 10773 — 제로

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    제로 (백준 10773)는
    장부에 숫자를 적다가 0을 외치면 가장 최근에 쓴 수를 지우는 규칙을 적용했을 때, 최종적으로 남은 수들의 합을 구하는 문제다.

    • 0이 아닌 정수: 장부에 해당 수를 적는다.
    • 0: 가장 최근에 쓴 수를 지운다. (문제 조건상 지울 수가 있음이 보장된다.)
    • 자료구조 스택(Stack)의 LIFO(Last In First Out) 성질을 그대로 이용하는 문제다.

    전략 개요

    • 배열을 이용한 스택 구현: 자바의 Stack 클래스를 사용할 수도 있지만, 주어진 코드처럼 배열(Array)과 인덱스(cursor/cnt)를 이용하면 더 가볍고 빠르게 구현할 수 있다.
    • 포인터 조작:
      • 숫자가 들어오면 현재 포인터 위치에 저장하고 포인터를 1 증가시킨다. (push)
      • 0이 들어오면 포인터를 1 감소시켜, 논리적으로 가장 최근 값을 삭제(무시)한다. (pop)
    • 최종 합산: 마지막에 유효한 포인터 개수만큼만 배열을 순회하여 합을 구한다.

    코드가 하는 일

    • 1) 배열 기반 스택 시뮬레이션 
      int[] array = new int[K];
int cnt = 0;

for (int i = 0; i < K; i++) {
    int num = Integer.parseInt(br.readLine());

    if (num == 0) {
        cnt--;
    } else {
        array[cnt++] = num;
    }
}
      • K가 최대 10만이므로 배열 크기를 K로 잡으면 오버플로우 없이 모든 수를 담을 수 있다.
      • num == 0이면 cnt--를 통해 이전 값을 덮어쓸 준비를 한다(삭제 효과).
      • 그 외에는 array[cnt]에 값을 저장하고 cnt를 증가시킨다.
    • 2) 유효 범위 합산 (Stream 활용) 
      System.out.println(IntStream.range(0, cnt)
        .map(i -> array[i])
        .sum()
);
      • 배열에는 지워진 값들의 잔재가 남아있을 수 있으므로, 0부터 cnt 직전까지만 순회해야 한다.
      • IntStream.range(0, cnt)를 사용하여 유효한 인덱스만 접근해 합을 구한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 입력받은 수 K개만큼 순회하며 저장/삭제 연산을 수행하므로 O(K)이다.
      • 마지막 합산 또한 유효한 원소 수(≤ K)만큼 수행하므로 전체적으로 O(K)다.
    • 공간 복잡도
      • 크기 K인 정수 배열을 사용하므로 O(K)이다.
