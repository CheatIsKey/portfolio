    문제 10816 — 숫자 카드 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자 카드 2 (백준 10816)는
    상근이가 가지고 있는 N개의 숫자 카드 중에서, 주어진 M개의 정수가 적힌 카드가 각각 몇 개씩 존재하는지를 구하는 문제다.

    • 숫자 카드의 값은 -10,000,000부터 10,000,000까지 존재할 수 있다.
    • N과 M이 최대 500,000이므로, $O(N \times M)$ 방식의 이중 반복문으로는 시간 초과가 발생한다.
    • 특정 숫자의 개수를 빠르게($O(1)$) 조회할 수 있는 방법이 필요하다.

    전략 개요

    • 카운팅 배열 (Counting Array): 입력되는 수의 범위가 제한적일 때(-1,000만 ~ +1,000만), 배열의 인덱스를 이용해 각 숫자의 등장 횟수를 저장하는 방식이다.
    • 오프셋(Offset) 적용: 배열의 인덱스는 음수가 될 수 없으므로, 등장 가능한 가장 작은 수인 -10,000,000을 인덱스 0에 매핑하기 위해 모든 수에 +10,000,000을 더해 저장한다.
    • 이 방식을 사용하면 해시맵(HashMap)이나 이분 탐색(Binary Search)보다 오버헤드가 적어 더 빠르게 동작한다.

    코드가 하는 일

    • 1) 배열 크기 설정 및 선언 
      final int SIZE = 10_000_000;
int[] cards = new int[SIZE * 2 + 1];
      • 수의 범위는 -10,000,000 ~ +10,000,000이므로 총 20,000,001개의 정수를 표현해야 한다.
      • 따라서 배열 크기를 SIZE * 2 + 1로 설정하여 모든 범위를 커버한다.
    • 2) 카드 개수 카운팅 (Input) 
      for (int i = 0; i < N; i++) {
    int num = Integer.parseInt(st.nextToken());
    cards[num + SIZE] += 1;
}
      • 상근이가 가진 카드 숫자를 읽어서 해당 인덱스의 값을 1 증가시킨다.
      • 이때 음수 인덱스를 방지하기 위해 num + SIZE 위치에 저장한다. (예: -10,000,000 → 인덱스 0)
    • 3) 쿼리 처리 및 출력 (Output) 
      for (int i = 0; i < M; i++) {
    int num = Integer.parseInt(st.nextToken());
    sb.append(cards[num + SIZE]).append(' ');
}
      • 찾아야 할 숫자 num이 주어지면, 전처리해 둔 cards 배열에서 num + SIZE 인덱스의 값을 바로 꺼낸다.
      • 이 값은 해당 숫자가 등장한 횟수이므로 즉시 출력 버퍼에 담는다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 카드 입력(N)과 쿼리 처리(M)를 각각 한 번씩 순회하므로 O(N + M)이다.
      • 배열 접근은 O(1)이므로 매우 빠르다.
    • 공간 복잡도
      • 약 2,000만 개의 int 배열을 사용하므로 약 80MB 정도의 메모리를 사용한다. (문제 제한 256MB 내 충분) → O(Range).

    문제 1764 — 듣보잡

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    듣보잡 (백준 1764)은
    "듣도 못한 사람"의 명단과 "보도 못한 사람"의 명단이 주어질 때, 두 명단에 모두 포함된 사람(교집합)을 구하고, 그 수와 명단을 사전순으로 출력하는 문제다.

    • 입력으로 주어지는 N과 M은 최대 500,000이다.
    • 단순 이중 반복문($O(N \times M)$)으로 비교하면 시간 초과가 발생한다.
    • 빠른 검색을 위해 해시(Hash)나 이분 탐색을 사용해야 한다.

    전략 개요

    • 자료구조 선택 (HashSet): 문자열 검색 속도가 매우 빠른 HashSet을 사용한다. 첫 번째 명단을 Set에 저장해두면, contains 연산은 평균적으로 O(1)에 수행된다.
    • 교집합 추출: 두 번째 명단을 입력받으면서 즉시 Set에 존재하는지 확인하고, 존재하면 결과 리스트에 추가한다.
    • 정렬 및 출력: 문제 조건에 따라 결과 리스트를 사전순으로 정렬한 뒤 출력한다.

    코드가 하는 일

    • 1) 듣도 못한 사람 저장 (HashSet) 
      Set<String> listen = new HashSet<>();

for (int i = 0; i < N; i++) {
    listen.add(br.readLine());
}
      • 첫 번째 그룹(듣도 못한 사람)의 이름을 모두 HashSet에 저장한다.
      • 중복이 없다는 조건이 있지만, Set은 중복을 자연스럽게 처리하므로 안전하다.
    • 2) 보도 못한 사람 확인 및 교집합 추적 
      List<String> result = new ArrayList<>();

for (int i = 0; i < M; i++) {
    String s = br.readLine();

    if (listen.contains(s)) {
        result.add(s);
    }
}
      • 두 번째 그룹(보도 못한 사람)의 이름을 입력받으며, listen 셋에 이미 존재하는지 확인한다.
      • 존재한다면 "듣보잡"에 해당하므로 result 리스트에 추가한다.
    • 3) 정렬 및 출력 
      Collections.sort(result);

sb.append(result.size()).append('\n');
for (String s : result) {
    sb.append(s).append('\n');
}
      • 찾아낸 교집합 명단을 Collections.sort()를 이용해 사전순으로 정렬한다.
      • 총 인원수와 명단을 순서대로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N개의 이름을 Set에 넣는 데 O(N).
      • M개의 이름을 조회하는 데 O(M).
      • 교집합의 크기를 K라 할 때, 정렬에 O(K log K)가 소요된다.
      • 전체 복잡도는 O(N + M + K log K)로, 시간 제한 내에 충분하다.
    • 공간 복잡도
      • N개의 문자열을 저장할 HashSet과 결과 리스트를 위한 공간이 필요하므로 O(N)이다.
