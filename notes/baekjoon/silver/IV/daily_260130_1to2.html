    문제 2164 — 카드2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    카드2 (백준 2164)는
    1부터 N까지의 카드가 순서대로 쌓여있을 때, "맨 앞 버리기"와 "그 다음 장을 맨 뒤로 보내기"를 반복하여 마지막에 남는 한 장의 카드를 구하는 문제다.

    • 전형적인 큐(Queue) 자료구조 문제다. (FIFO: First In First Out)
    • 맨 위(front)에서 빼고, 다시 맨 뒤(rear)로 넣는 과정을 반복한다.
    • 카드가 한 장 남을 때까지(큐의 크기가 1이 될 때까지) 수행한다.

    전략 개요

    • 배열을 이용한 큐 구현: LinkedList나 Queue 인터페이스를 사용할 수도 있지만, 배열(Array)과 포인터(front, rear)를 직접 관리하면 오버헤드가 적어 수행 속도가 더 빠르다.
    • 배열 크기 산정: 카드를 뒤로 옮길 때마다 배열의 뒤쪽 공간이 필요하다. N장의 카드가 모두 처리되는 과정에서 인덱스는 최대 약 2 * N까지 증가할 수 있으므로 배열 크기를 N * 2로 설정한다.
    • 포인터 이동:
      • front: 현재 큐의 맨 앞(삭제 대상)을 가리킨다.
      • rear: 현재 큐의 맨 뒤(삽입 위치)를 가리킨다.

    코드가 하는 일

    • 1) 배열 초기화 및 카드 배치 
      int[] array = new int[N * 2];

for (int i = 1; i <= N; i++) {
    array[i - 1] = i;
}

int front = 0;
int rear = N;
      • 배열 크기를 넉넉하게 2N으로 잡는다.
      • 0번 인덱스부터 N-1번 인덱스까지 1~N의 값을 채운다.
      • front는 시작점(0), rear는 데이터가 채워진 바로 다음 위치(N)를 가리킨다.
    • 2) 시뮬레이션 (버리고 옮기기) 
      while (rear - front > 1) {
    front++;
    array[rear++] = array[front++];
}
      • rear - front는 현재 큐에 들어있는 데이터의 개수다. 이 값이 1보다 클 때(2장 이상) 계속 반복한다.
      • front++: 맨 앞의 카드를 버리는 동작이다.
      • array[rear++] = array[front++]:
        • 버린 후, 그 다음 맨 앞의 카드(array[front])를 맨 뒤(array[rear])로 복사한다.
        • 그리고 rear(뒤쪽 확장)와 front(앞쪽 처리)를 모두 1씩 증가시킨다.
    • 3) 결과 출력 
      System.out.println(array[front]);
      • 반복문이 종료되면 큐에는 1장의 카드만 남게 된다.
      • 그때 front가 가리키는 값이 마지막 남은 카드다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N장의 카드를 처리하는 동안, 각 카드는 최대 1번 버려지고 1번 뒤로 이동한다.
      • 반복 횟수는 N에 비례하므로 O(N)이다.
    • 공간 복잡도
      • 2 * N 크기의 배열을 사용하므로 O(N)이다.

    문제 10845 — 큐

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    큐 (백준 10845)는
    정수를 저장하는 큐(Queue) 자료구조를 직접 구현하여, 주어지는 6가지 명령(push, pop, size, empty, front, back)을 처리하는 문제다.

    • FIFO(First In First Out): 가장 먼저 들어온 데이터가 가장 먼저 나가는 구조다.
    • 명령의 수 N이 최대 10,000이므로, 배열을 이용하여 큐를 간단하게 구현할 수 있다.
    • 큐가 비어있는지 확인하는 로직과 인덱스 관리가 핵심이다.

    전략 개요

    • 배열 기반 선형 큐 구현: 최대 명령 수 N만큼의 크기를 가진 배열을 선언하여 큐로 사용한다.
    • Two Pointers:
      • front: 큐의 맨 앞(삭제될 위치)을 가리키는 인덱스.
      • rear: 큐의 맨 뒤(데이터가 추가될 위치)를 가리키는 인덱스.
      • front == rear인 상태는 큐가 비어있음을 의미한다.
    • 명령 처리: 문자열 파싱을 통해 명령어를 구분하고 switch-case 문으로 각 기능을 수행한다.

    코드가 하는 일

    • 1) 큐 초기화 
      int N = Integer.parseInt(br.readLine());
int front = 0;
int rear = 0;

int[] array = new int[10_000];
      • 명령의 개수 N(최대 10,000)만큼 들어올 수 있으므로, 배열 크기를 10,000으로 설정한다.
      • front와 rear를 0으로 초기화한다.
    • 2) push 명령 (데이터 삽입) 
      case "push":
    array[rear++] = Integer.parseInt(st.nextToken());
    break;
      • array[rear]에 값을 저장하고 rear를 1 증가시킨다.
    • 3) pop 명령 (데이터 삭제) 
      case "pop":
    if (front == rear) sb.append(-1).append('\n');
    else {
        sb.append(array[front]).append('\n');
        front++;
    }
    break;
      • 큐가 비어있으면(front == rear) -1을 출력한다.
      • 아니라면 front 위치의 값을 버퍼에 담고, front를 1 증가시켜 삭제 처리한다.
    • 4) 기타 명령 (size, empty, front, back) 
      case "size":
    sb.append(rear - front).append('\n'); // 데이터 개수
    break;
case "empty":
    sb.append(front == rear ? 1 : 0).append('\n'); // 비어있음 여부
    break;
case "front":
    sb.append(front == rear ? -1 : array[front]).append('\n'); // 맨 앞 값 확인
    break;
case "back":
    sb.append(front == rear ? -1 : array[rear - 1]).append('\n'); // 맨 뒤 값 확인
    break;
      • size: rear - front로 현재 들어있는 개수를 계산한다.
      • back: rear는 '다음' 빈칸을 가리키므로, 실제 마지막 데이터는 rear - 1에 있다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 총 N개의 명령을 순차적으로 처리하며, 각 명령은 O(1)에 수행된다.
      • 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • 크기 10,000인 정수 배열을 사용하므로 O(N)이다.
