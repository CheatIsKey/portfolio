    문제 11866 — 요세푸스 문제 0

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    요세푸스 문제 0 (백준 11866)은
    1번부터 N번까지 원형으로 둘러앉은 사람들 중에서 매 K번째 사람을 제거하는 과정을 반복하여,
    사람들이 제거되는 순서(요세푸스 순열)를 구하는 문제다.

    • 원이기 때문에 마지막 사람 다음은 다시 첫 번째 사람으로 이어진다.
    • 한 사람이 제거되면 남은 사람들끼리 원을 유지하며 과정을 반복한다.
    • 출력 형식을 <3, 6, 2, ...> 형태로 맞춰야 한다.

    전략 개요

    • 큐(Queue)를 이용한 시뮬레이션: 원형 구조를 가장 직관적으로 구현하기 위해 큐를 사용한다.
    • 회전(Rotate): K번째 사람을 찾기 위해, 앞의 K-1명은 큐에서 뺀 뒤 다시 뒤로 넣는다.
    • 제거(Remove): K번째 사람은 큐에서 완전히 빼내어 결과 문자열에 추가한다. 이 과정을 큐가 빌 때까지 반복한다.

    코드가 하는 일

    • 1) 큐 초기화 
      Queue<Integer> queue = new ArrayDeque<>();
for (int i = 1; i <= N; i++) {
    queue.add(i);
}
      • 1부터 N까지의 번호를 순서대로 큐에 넣는다.
    • 2) K번째 사람 제거 로직 
      int i = 1;
while (!queue.isEmpty()) {
    if (i == K) {
        // K번째 사람인 경우: 제거 및 출력
        if (queue.size() == 1) {
            sb.append(queue.poll()).append(">");
            break;
        } else {
            sb.append(queue.poll()).append(", ");
        }
        i = 1; // 카운트 초기화
    } else {
        // K번째가 아님: 뒤로 보냄 (회전)
        queue.add(queue.poll());
        i++;
    }
}
      • i 변수로 현재 몇 번째 사람인지 센다.
      • i == K: 해당 사람을 poll()로 완전히 제거하고, 출력 버퍼에 추가한다. 제거 후 카운트 i를 다시 1로 초기화한다.
      • i != K: 해당 사람을 poll()로 꺼내어 바로 add()로 다시 넣는다. (순서 미루기)

    시간 · 공간 복잡도

    • 시간 복잡도
      • 총 N명을 제거해야 하며, 한 명을 제거하기 위해 최대 K번의 큐 연산(Poll/Add)이 일어난다.
      • 따라서 전체 연산 횟수는 대략 N × K회이므로 O(N × K)이다. (N, K ≤ 1,000 이므로 충분하다.)
    • 공간 복잡도
      • 큐에 최대 N개의 정수를 저장하므로 O(N)이다.

    문제 4949 — 균형잡힌 세상

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    균형잡힌 세상 (백준 4949)은
    주어진 문자열에 포함된 소괄호 ()와 대괄호 []가 올바르게 짝을 이루고 있는지(균형 잡혀 있는지) 판단하는 문제다.

    • 각 문자열은 온점(.)으로 끝난다.
    • 입력의 종료 조건은 점 하나(.)만 있는 줄이다.
    • 여는 괄호와 닫는 괄호의 종류가 정확히 일치해야 하며, 1:1로 매칭되어야 한다.

    전략 개요

    • 스택(Stack) 활용: 괄호의 짝을 검사하는 가장 표준적인 방법이다. 가장 최근에 열린 괄호가 가장 먼저 닫혀야 하기 때문이다 (LIFO).
    • 자바의 ArrayDeque를 스택처럼 사용하여(push, poll, peek) 구현한다.
    • 검증 로직:
      • 여는 괄호 (, [는 스택에 넣는다.
      • 닫는 괄호 ), ]가 나오면 스택의 맨 위(top)와 짝이 맞는지 확인하고 제거한다.
      • 중간에 짝이 안 맞거나, 처리가 끝난 후 스택에 괄호가 남아있으면 불균형 문자열이다.

    코드가 하는 일

    • 1) 입력 처리 및 종료 조건 
      while (true) {
    String line = br.readLine();
    if (line.equals(".")) break;
    // ...
}
      • 무한 루프를 돌며 한 줄씩 입력을 받는다.
      • 입력 줄이 정확히 점 하나(.)라면 프로그램을 종료한다.
    • 2) 괄호 검사 및 스택 연산 
      for (char c : line.toCharArray()) {
    if (c == '(' || c == '[') {
        deque.push(c);
    } else if (c == ')') {
        if (deque.isEmpty() || deque.peek() != '(') {
            ok = false;
            break;
        }
        deque.poll();
    } else if (c == ']') {
        if (deque.isEmpty() || deque.peek() != '[') {
            ok = false;
            break;
        }
        deque.poll();
    }
}
      • 문자열을 한 글자씩 순회한다.
      • 여는 괄호는 무조건 push한다.
      • 닫는 괄호가 나왔을 때 스택이 비어있거나(짝 없음), 스택의 top과 종류가 다르다면(괄호 엇갈림) ok = false 처리하고 반복을 멈춘다.
      • 짝이 맞다면 poll()로 스택에서 여는 괄호를 제거한다.
    • 3) 최종 판별 및 출력 
      if (!deque.isEmpty()) ok = false;

if (ok) sb.append("yes").append('\n');
else sb.append("no").append('\n');
      • 모든 문자를 다 돌았는데 스택에 괄호가 남아있다면, 닫히지 않은 괄호가 있는 것이므로 실패 처리한다.
      • 최종 ok 플래그에 따라 "yes" 혹은 "no"를 버퍼에 기록한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 문자열의 길이를 N이라 할 때, 한 번 순회하며 스택 연산을 수행하므로 줄마다 O(N)이다.
    • 공간 복잡도
      • 스택에 최대 N개의 괄호가 쌓일 수 있으므로 O(N)이다.
