    문제 10866 — 덱

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    덱 (백준 10866)은
    앞뒤 양쪽에서 데이터를 넣고 뺄 수 있는 덱(Deque, Double-ended Queue) 자료구조를 구현하고, 주어진 8가지 명령을 처리하는 문제다.

    • 덱(Deque): 스택과 큐의 연산을 모두 지원하는 자료구조다.
    • 명령어에 따라 push_front, push_back, pop_front, pop_back 등을 수행한다.
    • 비어있는 상태에서의 예외 처리(-1 출력)와 현재 크기(size), 비어있는지 여부(empty) 등을 관리해야 한다.

    전략 개요

    • 배열을 이용한 덱 구현: LinkedList를 사용할 수도 있지만, 배열과 인덱스 포인터를 직접 관리하면 더 빠르고 가볍게 구현할 수 있다.
    • 중앙 시작 전략:
      • 앞(front)과 뒤(back) 양쪽으로 데이터가 늘어날 수 있다.
      • 따라서 배열의 중간 지점(10000)을 시작점으로 잡고, front는 왼쪽으로, rear는 오른쪽으로 확장해 나간다.
      • 명령의 수 N이 최대 10,000이므로, 앞뒤로 10,000씩 여유를 둔 크기(약 20,000)의 배열이면 충분하다.

    코드가 하는 일

    • 1) 배열 및 포인터 초기화 (중앙 배치) 
      int[] numbers = new int[20001];

int front = 10000;
int rear = 10000;
      • 배열 크기를 20001로 설정하여 양쪽 확장에 대비한다.
      • front와 rear를 배열의 정중앙인 10000에서 시작하도록 초기화한다.
      • 데이터의 범위는 [front, rear) 구간이 된다. (front는 포함, rear는 제외)
    • 2) 삽입 연산 (push_front / push_back) 
      if (s.equals("push_front")) {
    numbers[--front] = num;
} else if (s.equals("push_back")) {
    numbers[rear++] = num;
}
      • push_front: front를 먼저 1 감소시킨 후 해당 위치에 값을 저장한다 (왼쪽 확장).
      • push_back: 현재 rear 위치에 값을 저장한 후 rear를 1 증가시킨다 (오른쪽 확장).
    • 3) 삭제 및 조회 연산 (pop / front / back) 
      // pop_front
if (front == rear) sb.append(-1);
else {
    sb.append(numbers[front]);
    front++;
}

// pop_back
if (front == rear) sb.append(-1);
else {
    sb.append(numbers[rear - 1]);
    rear--;
}
      • front == rear는 덱이 비어있음을 의미한다.
      • pop_front: 맨 앞(front)의 값을 출력하고 front를 1 증가시켜 범위를 줄인다.
      • pop_back: 맨 뒤의 값은 rear - 1 위치에 있다. 이를 출력하고 rear를 1 감소시킨다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 모든 명령(삽입, 삭제, 조회)은 인덱스 접근 및 산술 연산뿐이므로 O(1)이다.
      • 전체 명령 N개를 처리하므로 총 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • 고정된 크기(20,001)의 정수 배열을 사용하므로 O(1) (상수 공간) 또는 입력 N에 비례하는 크기라 볼 수 있다.

    문제 1026 — 보물

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    보물 (백준 1026)은
    길이가 같은 두 정수 배열 A와 B가 주어졌을 때, A의 순서를 바꾸어 각 원소의 곱의 합 $S = \sum A[i] \times B[i]$가 최소가 되도록 만드는 문제다.

    • 곱의 합을 최소로 만들기 위해서는 큰 수는 작은 수와, 작은 수는 큰 수와 곱해야 한다.
    • 문제에서 "B에 있는 수는 재배열하면 안 된다"고 명시했지만, 이는 문제 해결 로직(최솟값 계산)에서 B의 값들을 크기순으로 매칭하는 것을 금지하는 것이 아니라, 단순히 A만 움직이라는 설정이다.
    • 결국 값의 관점에서는 A는 오름차순, B는 내림차순으로 정렬해 짝지으면 된다.

    전략 개요

    • 정렬 (Sorting): 배열 A를 오름차순으로 정렬한다.
    • B 배열 처리: B 배열의 원소들도 크기 순서대로 짝을 지어야 하므로, B를 복사한 배열 C를 만들어 오름차순으로 정렬한다.
    • 그리디 매칭:
      • A는 앞에서부터(작은 값), C는 뒤에서부터(큰 값) 접근하여 곱한다.
      • 이렇게 하면 (최소 × 최대) 조합이 만들어져 전체 합 $S$가 최소가 된다.

    코드가 하는 일

    • 1) A 배열 정렬 
      Arrays.sort(A);
      • 배열 A를 오름차순으로 정렬하여 작은 값이 앞에 오도록 한다.
    • 2) B 배열 복사 및 정렬 
      int[] C = B.clone();
Arrays.sort(C);
      • 원본 배열 B를 보존해야 한다는 문제의 뉘앙스를 고려해 복제본 C를 만든다.
      • C 역시 오름차순으로 정렬하여 값의 크기 순서를 파악한다.
    • 3) 최소 합 계산 (교차 곱셈) 
      long answer = 0;

for (int i = 0; i < N; i++) {
    answer += (long) A[i] * C[N - i - 1];
}
      • A[i]는 i번째로 작은 수, C[N - i - 1]은 i번째로 큰 수다.
      • 서로 반대 순서의 크기를 가진 원소끼리 곱하여 answer에 더한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 A와 C를 정렬하는 데 각각 O(N log N)이 소요된다.
      • 반복문은 O(N)이므로 전체 시간 복잡도는 O(N log N)이다. (N ≤ 50이므로 매우 빠르다.)
    • 공간 복잡도
      • A, B, C 배열을 위한 공간 O(N)이 필요하다.
