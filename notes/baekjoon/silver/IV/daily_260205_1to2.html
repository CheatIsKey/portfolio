    문제 1620 — 나는야 포켓몬 마스터 이다솜

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    나는야 포켓몬 마스터 이다솜 (백준 1620)은
    도감에 수록된 N마리의 포켓몬 이름과 번호를 저장한 뒤, 문제가 주어지면 이름을 번호로, 혹은 번호를 이름으로 맞추는 문제다.

    • 입력되는 포켓몬 순서대로 1번부터 N번까지 번호가 부여된다.
    • 문제가 숫자로 들어오면 해당 번호의 포켓몬 이름을 출력한다.
    • 문제가 문자로 들어오면 해당 포켓몬의 번호를 출력한다.
    • N, M이 최대 100,000이므로 빠른 검색(O(1) 혹은 O(logN))이 필요하다.

    전략 개요

    • 해시맵(HashMap) 2개 사용: 양방향 검색을 빠르게 수행하기 위해 맵을 두 개 만든다.
      • 하나는 번호(Integer) → 이름(String) 저장용
      • 다른 하나는 이름(String) → 번호(Integer) 저장용
    • 입력 타입 판별: 들어온 문제가 숫자인지 문자인지 판단하기 위해 첫 글자의 아스키 코드 범위를 확인한다.

    코드가 하는 일

    • 1) 양방향 맵핑 데이터 저장 
      HashMap<Integer, String> findNameByIdx = new HashMap<>();
HashMap<String, Integer> findIdxByName = new HashMap<>();

for (int i = 1; i <= N; i++) {
    String input = br.readLine();
    findNameByIdx.put(i, input);
    findIdxByName.put(input, i);
}
      • 포켓몬 이름을 입력받을 때마다, (번호, 이름) 쌍과 (이름, 번호) 쌍을 각각의 맵에 모두 저장한다.
      • 이로써 어떤 형태의 질문이 와도 get() 한 번으로 답을 찾을 수 있다.
    • 2) 문제 유형 판별 및 정답 출력 
      if ('0' <= s.charAt(0) && s.charAt(0) <= '9') {
    String name = findNameByIdx.get(Integer.parseInt(s));
    sb.append(name).append('\n');
} else {
    int idx = findIdxByName.get(s);
    sb.append(idx).append('\n');
}
      • 입력 문자열 s의 첫 글자(charAt(0))가 숫자 범위('0'~'9')인지 확인한다.
      • 숫자라면: Integer.parseInt로 변환 후 findNameByIdx 맵에서 이름을 찾는다.
      • 문자라면: findIdxByName 맵에서 번호를 찾는다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N개의 데이터를 맵에 넣는 과정: O(N).
      • M개의 문제를 푸는 과정: 해시맵 접근은 평균적으로 O(1)이므로 O(M).
      • 전체 시간 복잡도는 O(N + M)으로, 시간 제한 내에 충분히 통과한다.
    • 공간 복잡도
      • 두 개의 해시맵에 각각 N개의 데이터를 저장하므로 O(N)이다.

    문제 18258 — 큐 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    큐 2 (백준 18258)는
    기본적인 큐(Queue) 자료구조를 구현하는 문제로, 큐 1 (10845번)과 기능은 같으나 입력 명령의 수 N이 최대 2,000,000(200만)이라는 점이 다르다.

    • N이 매우 크기 때문에 시간 복잡도에 민감하다.
    • 자바의 Scanner나 System.out.println을 빈번하게 사용하면 시간 초과가 발생할 수 있어, BufferedReader와 StringBuilder가 필수적이다.
    • 모든 연산(삽입, 삭제, 조회)은 반드시 O(1)에 처리되어야 한다.

    전략 개요

    • 배열 기반 큐 구현: LinkedList 객체를 생성하고 삭제하는 오버헤드조차 줄이기 위해, 충분한 크기의 int 배열과 인덱스 포인터를 사용한다.
    • 포인터 정의:
      • front: 큐의 맨 앞(삭제될 데이터)을 가리키는 인덱스.
      • rear: 큐의 맨 뒤(다음 데이터가 들어갈 자리)를 가리키는 인덱스.
    • 배열 크기: 명령이 최대 200만 개이므로, 모두 push일 경우를 대비해 배열 크기를 N으로 잡으면 인덱스 초과 없이 안전하게 처리할 수 있다.

    코드가 하는 일

    • 1) 배열 및 변수 초기화 
      int N = Integer.parseInt(br.readLine());
int[] queue = new int[N]; // 최대 N개 저장 가능

int front = 0;
int rear = 0;
      • queue 배열을 N 크기로 생성한다. (선형 큐로 사용)
      • front와 rear를 0으로 초기화한다.
    • 2) Push 연산 
      if (s.equals("push")) {
    queue[rear++] = Integer.parseInt(st.nextToken());
}
      • queue[rear]에 값을 넣고 rear를 1 증가시킨다.
    • 3) Pop 연산 
      else if (s.equals("pop")) {
    if (front == rear) {
        sb.append(-1).append('\n');
    } else {
        sb.append(queue[front++]).append('\n');
    }
}
      • front == rear이면 큐가 비어있는 상태이므로 -1을 출력 버퍼에 담는다.
      • 그렇지 않으면 queue[front] 값을 출력하고 front를 1 증가시켜 논리적으로 삭제한다.
    • 4) 기타 연산 (size, empty, front, back) 
      // size: 데이터 개수
sb.append(rear - front).append('\n');

// empty: 비었는지 확인
sb.append(front == rear ? 1 : 0).append('\n');

// front: 맨 앞 데이터 확인 (삭제 X)
sb.append(front == rear ? -1 : queue[front]).append('\n');

// back: 맨 뒤 데이터 확인
// rear는 '빈 공간'을 가리키므로 실제 마지막 데이터는 rear - 1에 있음
sb.append(front == rear ? -1 : queue[rear - 1]).append('\n');
      • 포인터 간의 차이로 크기를 계산하고, 인덱스를 이용해 값을 조회한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 명령은 단순 배열 접근과 산술 연산이므로 O(1)이다.
      • N개의 명령을 처리하므로 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • 최대 N개의 정수를 저장할 배열이 필요하므로 O(N)이다.
