    문제 14425 — 문자열 집합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 집합 (백준 14425)은
    N개의 문자열로 이루어진 집합 S가 주어졌을 때, 이후 입력되는 M개의 문자열 중 집합 S에 포함된 문자열의 개수를 구하는 문제다.

    • 단순한 이중 반복문($O(N \times M)$)으로 하나씩 비교하면 시간 초과가 발생할 가능성이 높다.
    • 효율적인 검색을 위해 해시(HashSet)를 사용하거나, 이 코드처럼 정렬 후 이분 탐색(Binary Search)을 사용해야 한다.

    전략 개요

    • 정렬 (Sorting): 이분 탐색을 수행하기 위한 전제 조건이다. 집합 S의 문자열들을 사전순으로 정렬한다.
    • 이분 탐색 (Binary Search):
      • 검사할 문자열이 정렬된 배열의 중앙값보다 사전순으로 앞서면 왼쪽을, 뒤면 오른쪽을 탐색한다.
      • 탐색 범위를 절반씩 줄여나가므로 검색 속도가 매우 빠르다. ($O(\log N)$)
    • 문자열 비교: compareTo 메서드를 사용하여 두 문자열의 사전순 우위를 판단한다.

    코드가 하는 일

    • 1) 집합 S 입력 및 정렬 
      String[] str1 = new String[N];

for (int i = 0; i < N; i++) {
    str1[i] = br.readLine();
}

Arrays.sort(str1);
      • N개의 문자열을 배열에 저장한 뒤, Arrays.sort()를 이용해 사전순으로 정렬한다.
      • 이 과정이 끝나면 str1 배열은 이분 탐색이 가능한 상태가 된다.
    • 2) 검사 및 카운팅 
      for (int i = 0; i < M; i++) {
    String word = br.readLine();
    boolean exists = binarySearch(str1, word, 0, N - 1);

    if (exists) {
        cnt++;
    }
}
      • M개의 문자열을 하나씩 입력받아 binarySearch 함수로 존재 여부를 확인한다.
      • 존재하면(true) 카운트(cnt)를 증가시킨다.
    • 3) 이분 탐색 구현 (재귀) 
      int comparison = word.compareTo(str1[mid]);

if (comparison == 0) return true;
else if (comparison < 0) return binarySearch(str1, word, left, mid - 1);
else return binarySearch(str1, word, mid + 1, right);
      • word.compareTo(target)의 결과가:
        • 0이면: 두 문자열이 같음 (탐색 성공).
        • 음수(< 0)면: word가 사전순으로 앞서므로 왼쪽 절반 탐색.
        • 양수(> 0)면: word가 사전순으로 뒤이므로 오른쪽 절반 탐색.
      • left > right가 될 때까지 못 찾으면 false를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열의 최대 길이를 L이라고 할 때, 문자열 비교는 $O(L)$이 걸린다.
      • 정렬: $O(N \log N \times L)$
      • 탐색: M개의 문자열에 대해 이분 탐색을 수행하므로 $O(M \log N \times L)$
      • 전체적으로 $O((N + M) \log N \times L)$이다.
    • 공간 복잡도
      • N개의 문자열을 저장할 배열 공간 $O(N \times L)$이 필요하다.
      • 재귀 호출 스택 깊이는 $O(\log N)$이다.

    문제 2217 — 로프

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    로프 (백준 2217)는
    N개의 로프가 있을 때, 이 로프들을 병렬로 연결하여 들어올릴 수 있는 물체의 최대 중량을 구하는 문제다.

    • 여러 개의 로프를 사용할 때, 물체의 무게는 각 로프에 고르게 분산(w/k)된다.
    • 즉, 어떤 로프 그룹이 버틸 수 있는 최대 중량은 (그룹 내에서 가장 약한 로프의 한계) × (로프의 개수)가 된다.
    • 가장 약한 로프보다 더 무거운 하중이 걸리면 끊어지기 때문이다.
    • 모든 로프를 다 쓸 필요는 없으며, 최적의 조합(부분 집합)을 찾아야 한다.

    전략 개요

    • 정렬(Sorting): 로프의 중량을 오름차순으로 정렬한다. 이렇게 하면 특정 로프를 선택했을 때, 그보다 튼튼한 로프가 뒤에 몇 개 있는지 바로 알 수 있다.
    • 그리디(Greedy) 탐색:
      • 특정 로프를 '가장 약한 로프'로 기준 잡았을 때, 최대 중량을 낼 수 있는 방법은 자신보다 튼튼한 모든 로프를 포함시키는 것이다.
      • 배열을 순회하며 현재 로프 중량 × (자신을 포함한 남은 로프 개수)를 계산하고, 그 중 최댓값을 찾는다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] ropes = new int[N];
for (int i = 0; i < N; i++) {
    ropes[i] = Integer.parseInt(br.readLine());
}

Arrays.sort(ropes);
      • 모든 로프의 한계 중량을 배열에 담고 오름차순으로 정렬한다.
      • 예: [10, 15]라면, 10이 0번 인덱스, 15가 1번 인덱스에 위치한다.
    • 2) 역순 순회 및 최대 중량 계산 
      int max = 0;

for (int i = N - 1; i >= 0; i--) {
    int count = N - i;
    int weight = ropes[i] * count;

    max = Math.max(max, weight);
}
      • 가장 튼튼한 로프(뒤쪽)부터 약한 로프(앞쪽) 순으로 확인한다. (오름차순이므로 i부터 N-1까지는 모두 ropes[i]보다 크거나 같다.)
      • count = N - i: 현재 로프(ropes[i]) 이상으로 버틸 수 있는 로프의 개수다.
      • weight = ropes[i] * count: 현재 로프가 버틸 수 있는 무게 × 개수로 총 중량을 계산한다.
      • 기존의 max값과 비교하여 더 큰 값으로 갱신한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N개의 로프를 정렬하는 데 O(N log N)이 소요된다.
      • 이후 한 번 순회하며 최댓값을 찾는 데 O(N)이 걸리므로, 전체 시간 복잡도는 O(N log N)이다.
    • 공간 복잡도
      • N개의 정수를 저장할 배열이 필요하므로 O(N)이다.
