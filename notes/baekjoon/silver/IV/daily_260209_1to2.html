    문제 1269 — 대칭 차집합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    대칭 차집합 (백준 1269)은
    두 집합 A와 B가 있을 때, (A - B)와 (B - A)의 합집합인 '대칭 차집합'의 원소 개수를 구하는 문제다.

    • 직관적으로는 A에는 속하지만 B에는 없는 것, 그리고 B에는 속하지만 A에는 없는 것들의 총개수다.
    • 수학적으로 집합의 원소 개수 성질을 이용하면, (A의 크기 + B의 크기) - 2 × (교집합의 크기)와 같다.
    • N과 M이 최대 200,000이므로, 교집합을 구하기 위해 $O(N \times M)$ 방식을 쓰면 시간 초과가 난다. 해시(HashSet)를 이용해 검색 시간을 단축해야 한다.

    전략 개요

    • 해시셋(HashSet) 활용: 집합 A의 원소들을 HashSet에 저장하여, 특정 숫자가 A에 존재하는지 확인하는 연산을 평균 O(1)에 수행한다.
    • 교집합 개수(Intersection) 파악: 집합 B의 원소를 입력받으면서, 해당 원소가 A에도 있는지(contains) 즉시 확인하여 교집합의 개수를 센다.
    • 수식 계산: 굳이 차집합을 직접 만들 필요 없이, 전체 개수에서 중복된 교집합 부분을 두 번 빼는 수식을 사용한다.

    코드가 하는 일

    • 1) 집합 A 입력 및 저장 
      Set<Integer> A = new HashSet<>();
for (int i = 0; i < N; i++) {
    A.add(Integer.parseInt(st.nextToken()));
}
      • 첫 번째 집합 A의 모든 원소를 HashSet에 저장한다. 빠른 검색을 위한 준비 단계다.
    • 2) 집합 B 입력 및 교집합 카운팅 
      int intersectionCnt = 0;

for (int i = 0; i < M; i++) {
    int num = Integer.parseInt(st.nextToken());
    B.add(num);
    if (A.contains(num)) {
        intersectionCnt++;
    }
}
      • 집합 B의 원소를 하나씩 입력받으면서, A.contains(num)이 참이면 두 집합 모두에 존재하는 값이므로 intersectionCnt를 증가시킨다.
      • (코드상 B.add(num)을 수행하지만, 결과 계산 공식에는 B 집합 객체 자체보다 M(개수)과 교집합 수만 있으면 된다.)
    • 3) 대칭 차집합 크기 계산 
      System.out.println(N + M - 2 * intersectionCnt);
      • N + M은 두 집합의 모든 원소를 합친 개수(중복 포함)다.
      • 여기서 교집합에 해당하는 원소들은 A에서도 한 번, B에서도 한 번 세어졌으므로 총 2번 포함되어 있다.
      • 대칭 차집합은 교집합을 아예 제외해야 하므로, 2 * intersectionCnt를 빼주면 정답이 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 집합 A에 데이터를 넣는 데 O(N).
      • 집합 B의 데이터를 입력받으며 A를 조회하는 데 O(M).
      • 전체 시간 복잡도는 O(N + M)으로 매우 효율적이다.
    • 공간 복잡도
      • 두 집합 A, B를 모두 메모리에 올리므로 O(N + M)의 공간이 필요하다.

    문제 2003 — 수들의 합 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수들의 합 2 (백준 2003)는
    N개의 수로 이루어진 수열에서 연속된 부분 수열의 합이 정확히 M이 되는 경우의 수를 구하는 문제다.

    • 연속된 수들의 합(A[i] + ... + A[j])만 인정된다. (중간을 건너뛸 수 없다.)
    • 모든 수는 자연수(양수)이므로, 구간을 늘리면 합이 커지고 구간을 줄이면 합이 작아지는 단조 증가 성질을 가진다.
    • 이 성질 덕분에 투 포인터(Two Pointers) 알고리즘을 사용하여 효율적으로 해결할 수 있다.

    전략 개요

    • 투 포인터 (start, end):
      • start: 부분 수열의 시작 인덱스 (구간에서 뺄 위치)
      • end: 부분 수열의 끝 인덱스 (구간에 더할 위치)
      • sum: 현재 [start, end) 구간의 합
    • 포인터 이동 규칙:
      • Sum < M: 값이 부족하므로 end를 오른쪽으로 이동시켜 값을 더한다.
      • Sum ≥ M: 값이 같거나 크다면, start를 오른쪽으로 이동시켜 값을 뺀다. (값이 같다면 카운트 증가 후 이동)

    코드가 하는 일

    • 1) 초기화 
      int totalCnt = 0;
int start = 0, end = 0, sum = 0;
      • 두 포인터 모두 0에서 시작하며, 현재 합(sum)도 0이다.
    • 2) 투 포인터 루프 (While) 
      while (true) {
    if (sum >= M) {
        if (sum == M) totalCnt++;
        sum -= numbers[start++];
    } else if (end == N) {
        break;
    } else {
        sum += numbers[end++];
    }
}
      • 조건 1: sum >= M
        • 현재 합이 M 이상이면 구간을 줄여야 한다.
        • 만약 sum == M이라면 정답이므로 totalCnt를 증가시킨다.
        • start가 가리키는 값을 sum에서 빼고 start를 한 칸 전진시킨다.
        • 이 로직은 end가 끝까지 갔더라도 sum이 M 이상인 경우를 처리하기 위해 가장 먼저 체크해야 한다.
      • 조건 2: end == N
        • sum이 M보다 작은데(위 조건 불충족), 더 이상 더할 숫자(end)가 없으면 탐색을 종료한다.
      • 조건 3: sum < M
        • 아직 합이 부족하므로 end가 가리키는 값을 sum에 더하고 end를 한 칸 전진시킨다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • start와 end 포인터는 각각 0부터 N까지 최대 한 번씩만 증가한다.
      • 따라서 반복문은 최대 2N번 실행되므로 O(N)이다. (이중 반복문 $O(N^2)$보다 훨씬 빠르다.)
    • 공간 복잡도
      • N개의 정수를 저장할 배열이 필요하므로 O(N)이다.
