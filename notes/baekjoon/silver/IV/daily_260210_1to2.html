    문제 17219 — 비밀번호 찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비밀번호 찾기 (백준 17219)는
    사이트 주소와 비밀번호가 쌍으로 주어질 때, 특정 사이트 주소를 입력하면 그에 해당하는 비밀번호를 빠르게 찾아내는 문제다.

    • 저장된 사이트 수 N과 찾으려는 사이트 수 M이 최대 100,000이다.
    • 매번 배열을 순회하며 찾는 방식($O(N \times M)$)은 시간 초과가 발생하므로, 해시(Hash) 자료구조를 사용하여 검색 시간을 $O(1)$로 단축해야 한다.

    전략 개요

    • 해시맵 (HashMap) 활용:
      • Key: 사이트 주소 (String)
      • Value: 비밀번호 (String)
      • 맵에 데이터를 저장해두면 get() 메서드를 통해 키값으로 바로 밸류를 조회할 수 있다.
    • 입출력 최적화: 입력 데이터가 많기 때문에 BufferedReader와 StringTokenizer, StringBuilder를 사용하여 I/O 속도를 높인다.

    코드가 하는 일

    • 1) 데이터 저장 (Mapping) 
      HashMap<String, String> map = new HashMap<>();

for (int i = 0; i < N; i++) {
    st = new StringTokenizer(br.readLine());
    String key = st.nextToken();   // 사이트 주소
    String value = st.nextToken(); // 비밀번호
    map.put(key, value);
}
      • N개의 줄에 걸쳐 주어지는 사이트 주소와 비밀번호를 읽어와 map.put()으로 저장한다.
    • 2) 비밀번호 검색 및 출력 
      for (int i = 0; i < M; i++) {
    String url = br.readLine();
    sb.append(map.get(url)).append('\n');
}
      • 찾고자 하는 M개의 사이트 주소를 입력받는다.
      • map.get(url)을 사용하여 해당 주소(Key)에 매핑된 비밀번호(Value)를 즉시 가져온다.
      • 결과를 StringBuilder에 모아 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N개의 데이터를 저장하는 데 $O(N)$, M개의 데이터를 조회하는 데 $O(M)$이 소요된다. (해시맵의 평균 접근 시간은 $O(1)$)
      • 전체 시간 복잡도는 $O(N + M)$이다.
    • 공간 복잡도
      • N개의 사이트 주소와 비밀번호 문자열을 저장해야 하므로 $O(N)$이다. (문자열 길이에 비례하지만, 최대 길이가 20으로 작으므로 상수로 취급 가능)

    문제 28278 — 스택 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    스택 2 (백준 28278)는
    정수를 저장하는 스택(Stack) 자료구조를 구현하고, 주어지는 5가지 명령(넣기, 빼기, 크기, 비어있는지 확인, 맨 위 값 확인)을 처리하는 문제다.

    • LIFO(Last In First Out): 가장 나중에 들어간 데이터가 가장 먼저 나오는 구조다.
    • 명령의 수 N이 최대 1,000,000(100만)이므로, System.out.println을 남발하면 시간 초과가 날 수 있다. StringBuilder를 사용해야 한다.
    • 자바의 Stack 클래스는 동기화 처리로 인해 느리므로, 배열이나 ArrayDeque 등을 사용하는 것이 좋다.

    전략 개요

    • 배열 기반 스택 구현: 스택의 최대 크기는 명령의 수 N을 넘지 않으므로, 크기 N인 int 배열을 생성하여 스택으로 사용한다.
    • 포인터(Cursor) 활용:
      • cur 변수를 사용하여 현재 스택의 맨 위 원소(Top)의 인덱스를 가리킨다.
      • 초기값은 -1(비어있음)로 설정한다.
      • 데이터가 추가되면 cur를 증가시키고, 삭제되면 감소시킨다.
    • 빠른 입출력: BufferedReader로 입력을 받고, 결과는 StringBuilder에 모아서 한 번에 출력한다.

    코드가 하는 일

    • 1) 스택 및 포인터 초기화 
      int[] stack = new int[N];
int cur = -1;
      • 명령의 개수 N만큼 배열을 할당한다. (모든 명령이 push여도 수용 가능)
      • cur를 -1로 두어 스택이 비어있음을 표시한다.
    • 2) 명령 1: 데이터 삽입 (Push) 
      case 1:
    int x = Integer.parseInt(st.nextToken());
    stack[++cur] = x;
    break;
      • cur를 먼저 1 증가시킨 뒤(++cur), 해당 위치에 값 x를 저장한다.
    • 3) 명령 2: 데이터 삭제 및 출력 (Pop) 
      case 2:
    if (cur == -1) {
        sb.append(-1).append('\n');
    } else {
        sb.append(stack[cur--]).append('\n');
    }
    break;
      • 스택이 비어있으면(cur == -1) -1을 출력한다.
      • 그렇지 않으면 현재 stack[cur] 값을 출력 버퍼에 담고, cur를 1 감소시켜(cur--) 논리적으로 삭제한다.
    • 4) 기타 명령 (Size, Empty, Top) 
      case 3: // size
    sb.append(cur + 1).append('\n');
    break;
case 4: // empty
    if (cur == -1) sb.append(1)...
    else sb.append(0)...
    break;
case 5: // top
    if (cur == -1) sb.append(-1)...
    else sb.append(stack[cur])...
    break;
      • size: 인덱스는 0부터 시작하므로 개수는 cur + 1이다.
      • top: 삭제하지 않고 stack[cur] 값만 확인한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 명령(1~5)은 배열 인덱스 접근과 산술 연산만 수행하므로 O(1)이다.
      • 총 N개의 명령을 처리하므로 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • 최대 N개의 정수를 저장할 배열을 사용하므로 O(N)이다.
