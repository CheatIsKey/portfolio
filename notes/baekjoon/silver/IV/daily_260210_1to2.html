    문제 10610 — 30

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    30 (백준 10610)은
    주어진 숫자 N의 각 자릿수를 섞어서 30의 배수가 되는 가장 큰 수를 만드는 문제다.

    • 30의 배수 조건: 30은 10과 3의 공배수이므로 다음 두 조건을 만족해야 한다.
      • 10의 배수: 끝자리가 반드시 0이어야 한다. 즉, 입력된 숫자에 0이 적어도 하나 포함되어 있어야 한다.
      • 3의 배수: 각 자릿수의 합이 3으로 나누어떨어져야 한다.
    • 가장 큰 수: 위 조건을 만족한다면, 숫자를 내림차순(큰 수부터)으로 정렬했을 때 가장 큰 값이 된다.

    전략 개요

    • 문자열 처리: N은 최대 105개의 숫자로 구성되므로 int나 long 범위를 벗어난다. 따라서 문자열로 입력받아 각 자릿수를 배열로 관리해야 한다.
    • 정렬 및 조건 검사:
      • 배열을 오름차순으로 정렬한다.
      • 가장 작은 값(numbers[0])이 0이 아니면 10의 배수가 될 수 없으므로 -1을 출력한다.
      • 모든 자릿수의 합을 구해 3으로 나누어떨어지지 않으면 3의 배수가 아니므로 -1을 출력한다.
    • 출력: 모든 조건을 만족하면, 오름차순 정렬된 배열을 뒤에서부터(내림차순) 출력하여 가장 큰 수를 만든다.

    코드가 하는 일

    • 1) 입력 변환 및 정렬 
      int[] numbers = new int[length];
for (int i = 0; i < length; i++) {
    numbers[i] = N.charAt(i) - '0';
}
Arrays.sort(numbers);
      • 문자열 N의 각 자릿수를 정수 배열 numbers에 담고 오름차순 정렬한다.
    • 2) 10의 배수 여부 확인 (0 존재 여부) 
      if (numbers[0] != 0) {
    System.out.println(-1);
    return;
}
      • 오름차순 정렬했으므로 0이 있다면 반드시 numbers[0]에 위치한다.
      • 만약 첫 번째 원소가 0이 아니라면, 입력값에 0이 없다는 뜻이므로 30의 배수가 불가능하다.
    • 3) 3의 배수 여부 확인 (자릿수 합) 
      int sum = 0;
for (int number : numbers) {
    sum += number;
}

if (sum % 3 != 0) {
    System.out.println(-1);
    return;
}
      • 모든 자릿수의 합이 3의 배수인지 확인한다. 아니면 -1을 출력하고 종료한다.
    • 4) 가장 큰 수 생성 (역순 출력) 
      for (int i = numbers.length - 1; i >= 0; i--) {
    sb.append(numbers[i]);
}
      • 오름차순 정렬된 배열을 뒤에서부터 읽어 StringBuilder에 담으면, 자연스럽게 가장 큰 수(내림차순 배치)가 완성된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 입력받은 수의 길이(자릿수)를 N이라 할 때, 정렬에 O(N log N)이 소요된다.
      • 나머지 순회 로직은 O(N)이므로 전체 복잡도는 정렬에 지배된다.
    • 공간 복잡도
      • 길이 N인 정수 배열과 문자열을 저장하므로 O(N)이다.

    문제 11656 — 접미사 배열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    접미사 배열 (백준 11656)은
    주어진 문자열 S의 모든 접미사(Suffix)를 구하고, 이를 사전순(알파벳순)으로 정렬하여 출력하는 문제다.

    • 접미사: 문자열의 특정 인덱스부터 끝까지 이어지는 부분 문자열이다.
    • 예를 들어 baekjoon의 접미사는 baekjoon, aekjoon, ekjoon, ... , n으로 총 길이만큼 존재한다.
    • 이들을 알파벳 순서대로 정렬한 뒤 줄바꿈하여 출력하면 된다.

    전략 개요

    • 접미사 생성: 문자열의 길이가 N일 때, 0번째부터 N-1번째 인덱스까지 시작하는 모든 부분 문자열(substring)을 생성하여 배열에 저장한다.
    • 정렬 (Sorting): 생성된 문자열 배열을 오름차순(사전순)으로 정렬한다.
    • 출력: 정렬된 결과를 순서대로 출력한다.

    코드가 하는 일

    • 1) 접미사 생성 및 저장 
      String[] str = new String[S.length()];

for (int i = 0; i < S.length(); i++) {
    str[i] = S.substring(i);
}
      • 문자열 S의 길이만큼 String 배열을 생성한다.
      • substring(i) 메서드는 인덱스 i부터 끝까지의 문자열을 반환하므로, 이를 반복하여 모든 접미사를 배열에 담는다.
    • 2) 정렬 및 출력 
      Arrays.sort(str);

for (String s : str) {
    sb.append(s).append('\n');
}
      • Arrays.sort()를 사용하여 문자열 배열을 사전순으로 정렬한다.
      • 정렬된 접미사들을 StringBuilder에 담아 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열의 길이를 N이라고 할 때, 접미사를 생성하는 데 $O(N^2)$(문자열 복사)이 소요된다.
      • 정렬(Sorting)은 비교 연산에 문자열 길이만큼 시간이 걸리므로 $O(N^2 \log N)$이다.
      • N ≤ 1,000이므로 충분히 빠르다.
    • 공간 복잡도
      • N개의 접미사를 저장해야 하며, 전체 문자 개수는 약 $N^2/2$개이므로 $O(N^2)$이다.
