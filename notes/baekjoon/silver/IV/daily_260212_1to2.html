    문제 10825 — 국영수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    국영수 (백준 10825)는
    학생 N명의 이름과 국어, 영어, 수학 점수가 주어졌을 때,
    4가지 우선순위 조건에 맞춰 정렬하는 문제다.

    • 조건 1: 국어 점수가 감소하는 순서로 (내림차순).
    • 조건 2: 국어 점수가 같으면, 영어 점수가 증가하는 순서로 (오름차순).
    • 조건 3: 국어, 영어 점수가 같으면, 수학 점수가 감소하는 순서로 (내림차순).
    • 조건 4: 모든 점수가 같으면, 이름이 사전 순으로 증가하는 순서로 (오름차순).

    전략 개요

    • 객체(Class) 활용: 이름과 세 과목의 점수를 하나의 단위로 묶기 위해 Student 클래스를 정의한다.
    • 커스텀 정렬(Comparator): 자바의 sort 메서드와 람다식을 사용하여 복합 정렬 조건을 구현한다.
    • 비교 로직:
      • 오름차순: Integer.compare(a, b) (작은 값이 앞)
      • 내림차순: Integer.compare(b, a) (큰 값이 앞)
      • 문자열: a.compareTo(b) (사전순)

    코드가 하는 일

    • 1) 데이터 입력 및 객체 생성 
      List<Student> students = new ArrayList<>();
for (int i = 0; i < N; i++) {
    // ... 입력 파싱 ...
    students.add(new Student(name, score1, score2, score3));
}
      • N명의 학생 정보를 입력받아 Student 객체로 생성하고 리스트에 저장한다.
    • 2) 다중 조건 정렬 (Core Logic) 
      students.sort((a, b) -> {
    if (a.score1 != b.score1) return Integer.compare(b.score1, a.score1); // 국어 내림차순
    if (a.score2 != b.score2) return Integer.compare(a.score2, b.score2); // 영어 오름차순
    if (a.score3 != b.score3) return Integer.compare(b.score3, a.score3); // 수학 내림차순
    return a.name.compareTo(b.name); // 이름 오름차순
});
      • 람다식을 사용하여 정렬 기준을 정의한다.
      • 우선순위가 높은 조건(국어 점수)부터 비교하고, 값이 다르면 즉시 정렬 순서를 결정한다.
      • 값이 같으면 다음 우선순위 조건(영어 → 수학 → 이름)으로 넘어간다.
    • 3) 결과 출력 
      for (Student student : students) {
    sb.append(student.name).append('\n');
}
      • 정렬된 리스트를 순회하며 학생의 이름만 출력 버퍼에 담는다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N명의 데이터를 정렬하는 데 O(N log N)이 소요된다. (Java의 Collections.sort는 TimSort 사용)
      • N이 최대 100,000이므로 1초 내에 충분히 수행 가능하다.
    • 공간 복잡도
      • N개의 Student 객체를 저장할 리스트가 필요하므로 O(N)이다.

    문제 2960 — 에라토스테네스의 체

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    에라토스테네스의 체 (백준 2960)는
    소수를 구하는 알고리즘인 '에라토스테네스의 체'를 수행하는 과정에서, K번째로 지워지는 수가 무엇인지 구하는 구현 문제다.

    • 일반적인 소수 구하기와 달리, 소수 P 자체도 지우는 순서에 포함된다.
    • 이미 지워진 수(예: 6은 2의 배수로서 지워짐)는 3의 배수를 지울 때 다시 지우지 않는다.
    • N과 K가 최대 1,000으로 작기 때문에 단순 시뮬레이션으로 해결 가능하다.

    전략 개요

    • 체크 배열 (Boolean Array): 수가 지워졌는지 여부를 저장할 chk 배열을 사용한다.
    • 이중 반복문:
      • 바깥 반복문은 아직 지워지지 않은 가장 작은 수(소수 P)를 찾는다.
      • 안쪽 반복문은 P의 배수들을 순서대로 지워나간다.
    • 카운팅: 실제로 지우는 작업(chk[j] = true)이 일어날 때마다 카운트를 증가시키고, K번째가 되면 해당 수를 출력하고 종료한다.

    코드가 하는 일

    • 1) 배열 및 변수 초기화 
      boolean[] chk = new boolean[N + 1];
int cnt = 0;
      • 1부터 N까지의 상태를 저장하기 위해 크기가 N + 1인 배열을 생성한다. (기본값 false: 안 지워짐)
      • 지운 횟수를 셀 cnt를 0으로 초기화한다.
    • 2) 소수 찾기 및 배수 지우기 
      for (int i = 2; i <= N; i++) {
    if (!chk[i]) { // 아직 지워지지 않은 수(소수) 발견
        for (int j = i; j <= N; j += i) { // i의 배수들 순회
      • chk[i]가 false라면 i는 소수다.
      • j는 i부터 시작하여 i씩 증가하므로, i, 2i, 3i... 즉 배수들을 가리킨다.
    • 3) 중복 처리 방지 및 정답 출력 
                  if (chk[j]) continue; // 이미 지워진 수는 패스

            chk[j] = true; // 지움 처리
            cnt++;

            if (cnt == K) { // K번째 지우는 수라면
                System.out.println(j);
                return;
            }
        }
    }
}
      • 이미 지워진 수(chk[j] == true)는 카운트하지 않고 건너뛴다. (예: 6은 2의 배수에서 이미 지워졌으므로 3의 배수에서 또 지우지 않음)
      • 새롭게 지울 때만 cnt를 증가시키고, K번째가 되면 즉시 프로그램을 종료한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 에라토스테네스의 체 알고리즘의 시간 복잡도는 $O(N \log (\log N))$이다.
      • N이 1,000 이하로 매우 작기 때문에 순식간에 계산된다.
    • 공간 복잡도
      • 지움 여부를 저장할 배열이 필요하므로 $O(N)$이다.
