    문제 1244 — 스위치 켜고 끄기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    스위치 켜고 끄기 (백준 1244)는
    일렬로 배치된 스위치들의 상태를 학생들의 성별과 받은 번호에 따라 조작한 후,
    최종 스위치 상태를 한 줄에 20개씩 출력하는 문제다.

    • 남학생은 받은 번호의 배수에 해당하는 모든 스위치를 토글한다.
    • 여학생은 받은 번호를 중심으로 좌우 대칭인 최대 구간의 모든 스위치를 토글한다.
    • 스위치는 최대 100개, 학생은 최대 100명이며, 출력 시 한 줄에 20개씩 끊어서 출력해야 한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션, 배열 조작
    • 선택 이유: 문제에서 주어진 규칙을 그대로 구현하면 되는 시뮬레이션 문제이므로, boolean 배열로 스위치 상태를 관리하며 각 학생의 조작을 순차적으로 적용한다.
    • 세부 전략:
      • 남학생: 받은 번호의 배수 인덱스를 반복문으로 찾아 토글
      • 여학생: 중심에서 시작해 좌우로 범위를 확장하며 대칭 여부 확인 후, 유효한 최대 구간의 스위치를 토글
      • 출력: 카운터를 사용해 20개마다 줄바꿈 처리

    코드가 하는 일

    • 1) 초기 스위치 상태 입력 받기 
      boolean[] chk = new boolean[N];
      for (int i = 0; i < N; i++) {
          int num = Integer.parseInt(st.nextToken());
          chk[i] = (num == 1);
      }
      • 스위치 개수만큼 boolean 배열을 생성하고, 1이면 true(켜짐), 0이면 false(꺼짐)로 저장한다.
    • 2) 남학생의 스위치 조작 
      if (gender == 1) {
          for (int j = turn; j <= N; j += turn) {
              chk[j - 1] = !chk[j - 1];
          }
      }
      • 받은 번호(turn)의 배수에 해당하는 모든 스위치(j)를 찾아 상태를 반전시킨다.
      • 배열 인덱스는 0부터 시작하므로 j-1 위치의 스위치를 토글한다.
    • 3) 여학생의 대칭 구간 찾기 
      int center = turn - 1;
      int range = 0;
      
      while (center - range >= 0 && center + range < N) {
          if (chk[center - range] == chk[center + range]) {
              range++;
          } else {
              break;
          }
      }
      • 받은 번호를 중심(center)으로 하여, 좌우로 확장하며 대칭 여부를 확인한다.
      • 대칭이 유지되는 동안 range를 증가시켜 최대 범위를 찾는다.
      • 배열 범위를 벗어나거나 대칭이 깨지면 종료한다.
    • 4) 여학생의 스위치 토글 
      for (int j = center - range + 1; j <= center + range - 1; j++) {
          chk[j] = !chk[j];
      }
      • while 루프에서 range가 1 증가한 상태로 종료되므로, 실제 대칭 구간은 center - (range-1) 부터 center + (range-1)까지다.
      • 해당 구간의 모든 스위치를 토글한다.
    • 5) 결과 출력 (한 줄에 20개씩) 
      int cnt = 1;
      for (boolean isTurnOn : chk) {
          if (cnt > 20) {
              sb.append('\n');
              cnt -= 20;
          }
          if (isTurnOn) {
              sb.append(1).append(" ");
          } else {
              sb.append(0).append(" ");
          }
          cnt++;
      }
      • 카운터(cnt)를 사용해 20개를 출력할 때마다 줄바꿈을 추가한다.
      • 각 스위치 상태를 1 또는 0으로 변환하여 공백과 함께 StringBuilder에 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N × S): 학생 수(S)만큼 반복하며, 각 학생마다 최악의 경우 스위치 전체(N)를 확인하거나 토글할 수 있다.
      • 남학생의 경우 배수 개수만큼, 여학생의 경우 최대 N개의 스위치를 확인하므로 전체적으로 O(N × S)이다.
    • 공간 복잡도
      • O(N): 스위치 상태를 저장하는 boolean 배열과 출력용 StringBuilder가 필요하다.

    문제 1940 — 주몽

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    주몽 (백준 1940)는
    N개의 재료 중에서 두 개를 선택하여 그 고유 번호의 합이 정확히 M이 되는 조합의 개수를 찾는 문제로,
    서로 다른 두 재료의 번호 합이 목표값 M과 일치하는 쌍의 개수를 구하는 문제다.

    • 재료는 최대 15,000개, 목표값 M은 최대 10,000,000이다.
    • 각 재료의 고유 번호는 100,000 이하의 자연수이다.
    • 갑옷 하나를 만들려면 정확히 두 개의 서로 다른 재료가 필요하다.

    전략 개요

    • 핵심 알고리즘: 해시셋(HashSet)을 이용한 Two Sum 패턴
    • 선택 이유: 배열을 순회하면서 각 재료마다 "M - 현재 재료 번호"가 이미 등장했는지를 O(1) 시간에 확인할 수 있어, 전체 시간 복잡도를 O(N)으로 유지할 수 있다. 이중 반복문을 사용하면 O(N²)이 되어 비효율적이다.
    • 세부 전략:
      • 재료를 하나씩 읽으면서, 현재 재료(key)와 짝을 이룰 수 있는 값(value = M - key)이 이미 HashSet에 존재하는지 확인한다.
      • 존재한다면 갑옷을 만들 수 있는 쌍을 찾은 것이므로 카운트를 증가시킨다.
      • 현재 재료를 HashSet에 추가하여, 이후에 등장할 재료들이 이 재료와 짝을 이룰 수 있는지 확인할 수 있도록 한다.

    코드가 하는 일

    • 1) 입력 받기 및 초기화 
      int N = Integer.parseInt(br.readLine());
      int M = Integer.parseInt(br.readLine());
      Set<Integer> set = new HashSet<>();
      int cnt = 0;
      • 재료의 개수 N과 목표값 M을 입력받고, 이미 등장한 재료를 저장할 HashSet과 갑옷 개수를 세는 변수를 준비한다.
    • 2) 재료를 순회하며 짝 찾기 
      for (int i = 0; i < N; i++) {
          int key = Integer.parseInt(st.nextToken());
          int value = M - key;
          
          if (set.contains(value)) {
              cnt++;
          }
          
          set.add(key);
      }
      • 각 재료(key)를 읽을 때마다, 이 재료와 합쳐서 M이 되는 보완 재료(value = M - key)를 계산한다.
      • 만약 보완 재료가 이미 set에 존재한다면, 이전에 등장한 재료와 현재 재료로 갑옷을 만들 수 있으므로 cnt를 1 증가시킨다.
      • 현재 재료를 set에 추가하여, 이후에 읽을 재료들이 현재 재료와 짝을 이룰 수 있는지 확인할 수 있게 한다.
    • 3) 결과 출력 
      System.out.println(cnt);
      • 찾은 갑옷 쌍의 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 재료를 한 번씩 순회하며, 각 재료마다 HashSet의 contains와 add 연산을 수행한다. 이 두 연산은 평균적으로 O(1) 시간이 걸리므로 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • O(N): 최악의 경우 모든 재료를 HashSet에 저장해야 하므로 O(N)의 추가 공간이 필요하다.
