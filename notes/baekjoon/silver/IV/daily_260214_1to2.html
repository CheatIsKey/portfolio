    백준 알고리즘 Silver IV - Daily 문제 묶음

    작성일 2026-02-14 · 분류: 코딩 테스트(백준 알고리즘), Silver IV

    문제 18110 — solved.ac

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    solved.ac (백준 18110)는
    문제의 난이도에 대한 여러 의견이 주어질 때, 극단적인 값을 제외한 평균을 구하는 문제로,
    상위 15%와 하위 15%를 제외한 절사평균(Trimmed Mean)을 계산하는 문제다.

    • 난이도 의견은 최대 300,000개가 주어지며, 각 의견은 1 이상 30 이하의 정수다.
    • 의견이 0개인 경우도 있으며, 이때는 0을 출력한다.
    • 제외할 인원 수는 전체의 15%를 반올림한 값이며, 최종 평균도 반올림하여 정수로 출력한다.

    전략 개요

    • 핵심 알고리즘: 정렬 후 절사평균 계산
    • 선택 이유: 상위/하위 극단값을 제외하려면 먼저 데이터를 정렬해야 한다. 정렬 후 앞뒤에서 일정 개수를 제외하고 평균을 구하면, 극단적인 의견의 영향을 줄인 신뢰도 높은 난이도를 계산할 수 있다.
    • 세부 전략:
      • 의견이 0개인 경우를 먼저 처리하여 예외 상황 방지
      • N × 0.15를 반올림하여 제외할 개수(outlier) 계산
      • 배열을 오름차순 정렬하여 하위 15%와 상위 15% 식별
      • outlier개를 앞뒤로 제외한 중간 구간의 합을 구하고 평균 계산
      • 최종 평균을 반올림하여 정수로 출력

    코드가 하는 일

    • 1) 의견이 없는 경우 처리 
      if (N == 0) {
          System.out.println(0);
          return;
      }
      • 난이도 의견이 0개인 경우, 평균을 계산할 수 없으므로 0을 출력하고 종료한다.
    • 2) 제외할 의견 개수 계산 
      int outlier = (int) Math.round(N * 0.15);
      • 전체 의견 수의 15%를 계산하고 반올림하여, 상하위에서 각각 제외할 의견의 개수를 구한다.
      • 예를 들어 N=10이면 10 × 0.15 = 1.5 → 반올림하여 2개씩 제외한다.
    • 3) 의견 입력 및 정렬 
      int[] numbers = new int[N];
      
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(br.readLine());
      }
      
      Arrays.sort(numbers);
      • 모든 난이도 의견을 배열에 저장한 후 오름차순으로 정렬한다.
      • 정렬 후 배열의 앞부분은 하위 15%, 뒷부분은 상위 15%가 된다.
    • 4) 중간 구간의 합 계산 
      double sum = 0;
      
      for (int i = outlier; i < N - outlier; i++) {
          sum += numbers[i];
      }
      • 인덱스 outlier부터 N - outlier - 1까지 순회하며, 하위 15%와 상위 15%를 제외한 중간 70%의 의견만 합산한다.
      • 예를 들어 outlier=2, N=10이면 인덱스 2~7(6개 항목)의 합을 구한다.
    • 5) 절사평균 계산 및 출력 
      if (sum == 0) {
          System.out.println(sum);
          return;
      }
      
      int n = N - outlier * 2;
      long avg = Math.round(sum / n);
      
      System.out.println(avg);
      • 합이 0인 경우를 예외 처리한 후, 남은 의견 개수(n)로 나누어 평균을 계산한다.
      • Math.round()를 사용하여 평균을 반올림하고, 정수로 변환하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): Arrays.sort()를 사용한 정렬이 O(N log N)이며, 배열 순회는 O(N)이므로 전체 시간 복잡도는 O(N log N)이다.
    • 공간 복잡도
      • O(N): N개의 난이도 의견을 저장하는 배열이 필요하므로 O(N)의 공간이 필요하다.

    문제 1049 — 기타줄

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    기타줄 (백준 1049)는
    N개의 기타줄을 구매할 때, 여러 브랜드가 제공하는 6개 패키지 가격과 낱개 가격 중에서
    최소 비용으로 N개 이상의 기타줄을 구매하는 방법을 찾는 문제다.

    • 필요한 기타줄은 최대 100개, 브랜드는 최대 50개가 주어진다.
    • 각 브랜드마다 6개 패키지 가격과 낱개 가격이 주어진다.
    • 패키지만 구매, 낱개만 구매, 패키지와 낱개 혼합 구매 등 모든 경우를 고려해야 한다.
    • N개 이상을 구매해야 하므로, 패키지만 구매할 때는 남는 기타줄이 생길 수 있다.

    전략 개요

    • 핵심 알고리즘: 그리디 알고리즘, 경우의 수 비교
    • 선택 이유: 모든 브랜드 중 가장 저렴한 패키지 가격과 가장 저렴한 낱개 가격을 찾은 후, 가능한 모든 구매 방법(패키지만, 낱개만, 혼합)을 계산하여 최솟값을 선택하면 된다. 서로 다른 브랜드의 패키지와 낱개를 조합할 수 있으므로, 각각 최솟값을 찾는 것이 최적이다.
    • 세부 전략:
      • 모든 브랜드의 패키지 가격과 낱개 가격을 정렬하여 각각의 최솟값 확보
      • 세 가지 구매 방법 계산:
        • 패키지만 구매: ⌈N/6⌉ × 최소 패키지 가격
        • 낱개만 구매: N × 최소 낱개 가격
        • 혼합 구매: ⌊N/6⌋ × 최소 패키지 가격 + (N%6) × 최소 낱개 가격
      • 세 가지 방법 중 최솟값 선택

    코드가 하는 일

    • 1) 입력 받기 및 배열 저장 
      int[] minPackage = new int[M];
      int[] minSingle = new int[M];
      
      for (int i = 0; i < M; i++) {
          st = new StringTokenizer(br.readLine());
          minPackage[i] = Integer.parseInt(st.nextToken());
          minSingle[i] = Integer.parseInt(st.nextToken());
      }
      • M개 브랜드의 패키지 가격과 낱개 가격을 각각의 배열에 저장한다.
    • 2) 최소 가격 찾기 
      Arrays.sort(minPackage);
      Arrays.sort(minSingle);
      • 두 배열을 각각 오름차순 정렬하면, 인덱스 0에 각각의 최솟값이 위치한다.
      • 서로 다른 브랜드의 패키지와 낱개를 조합할 수 있으므로, 각각 독립적으로 최솟값을 찾는다.
    • 3) 패키지만 구매하는 경우 
      int onlyPackage = (N % 6 == 0) ? ((N / 6) * minPackage[0]) : ((N / 6 + 1) * minPackage[0]);
      • N이 6으로 나누어떨어지면 정확히 N/6개의 패키지를 구매한다.
      • 그렇지 않으면 (N/6 + 1)개의 패키지를 구매하여 N개 이상을 확보한다 (일부 남음).
      • 예: N=10이면 6개 패키지 2개를 구매하여 12개를 확보 (2개 남음).
    • 4) 낱개만 구매하는 경우 
      int onlySingle = N * minSingle[0];
      • 정확히 N개를 최소 낱개 가격으로 구매한다.
    • 5) 패키지와 낱개를 혼합 구매하는 경우 
      int packageAndSingle = ((N / 6) * minPackage[0]) + ((N % 6) * minSingle[0]);
      • N/6개의 패키지로 6의 배수만큼 구매하고, 나머지 N%6개는 낱개로 구매한다.
      • 예: N=10이면 6개 패키지 1개(6개) + 낱개 4개 = 총 10개 구매.
    • 6) 최소 비용 선택 및 출력 
      int amount = Math.min(onlyPackage, Math.min(onlySingle, packageAndSingle));
      
      System.out.println(amount);
      • 세 가지 구매 방법 중 가장 저렴한 방법을 선택하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M log M): 브랜드 수 M개의 패키지 가격과 낱개 가격을 각각 정렬하는 과정이 O(M log M)이며, 나머지 연산은 O(1)이므로 전체 시간 복잡도는 O(M log M)이다.
    • 공간 복잡도
      • O(M): M개의 패키지 가격과 낱개 가격을 저장하는 두 배열이 필요하므로 O(M)의 공간이 필요하다.
