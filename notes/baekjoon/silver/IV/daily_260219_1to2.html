    문제 9613 — GCD 합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    GCD 합 (백준 9613)는
    n개의 양의 정수가 주어질 때, 가능한 모든 쌍의 최대공약수(GCD)를 구하고 그 합을 구하는 문제로,
    조합을 이용해 모든 쌍을 생성하고 유클리드 호제법으로 GCD를 계산하는 문제다.

    • 테스트 케이스는 최대 100개, 각 테스트 케이스마다 수의 개수는 최대 100개가 주어진다.
    • 각 수는 1,000,000 이하의 양의 정수다.
    • n개의 수에서 2개를 선택하는 모든 조합(nC2)에 대해 GCD를 계산해야 한다.
    • GCD의 합이 int 범위를 초과할 수 있으므로 long 타입을 사용해야 한다.

    전략 개요

    • 핵심 알고리즘: 조합(Combination), 유클리드 호제법
    • 선택 이유: n개 중에서 2개를 선택하는 모든 쌍을 만들려면 이중 반복문으로 조합을 생성하면 되고, 각 쌍의 GCD는 유클리드 호제법을 사용하여 효율적으로 계산할 수 있다. 중복과 순서를 제거하기 위해 r < c 조건으로 탐색한다.
    • 세부 전략:
      • 이중 반복문으로 서로 다른 두 수의 모든 조합 생성 (중복 없이)
      • 각 쌍에 대해 유클리드 호제법으로 GCD 계산
      • 모든 GCD를 long 타입 변수에 누적하여 오버플로우 방지

    코드가 하는 일

    • 1) 테스트 케이스 입력 및 반복 
      int t = Integer.parseInt(br.readLine());
      
      for (int i = 0; i < t; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          int n = Integer.parseInt(st.nextToken());
          int[] numbers = new int[n];
          long total = 0;
      • 테스트 케이스의 개수 t를 입력받고, 각 테스트 케이스마다 수의 개수 n과 누적 합계 total을 초기화한다.
      • total을 long으로 선언하여 GCD 합이 int 범위를 초과하는 경우를 대비한다.
    • 2) n개의 수 입력받기 
      for (int j = 0; j < n; j++) {
          numbers[j] = Integer.parseInt(st.nextToken());
      }
      • n개의 양의 정수를 배열에 저장한다.
    • 3) 모든 쌍의 GCD 계산 및 합산 
      for (int r = 0; r < n - 1; r++) {
          for (int c = r + 1; c < n; c++) {
              total += gcd(numbers[r], numbers[c]);
          }
      }
      • 이중 반복문으로 n개 중 2개를 선택하는 모든 조합을 생성한다.
      • r은 0부터 n-2까지, c는 r+1부터 n-1까지 순회하여 중복과 자기 자신과의 쌍을 제거한다.
      • 예: n=4이면 (0,1), (0,2), (0,3), (1,2), (1,3), (2,3) 총 6개 쌍 생성
      • 각 쌍에 대해 GCD를 계산하여 total에 누적한다.
    • 4) 유클리드 호제법으로 GCD 계산 
      private static long gcd(int num1, int num2) {
          if (num2 == 0) return num1;
          return gcd(num2, num1 % num2);
      }
      • 재귀적으로 두 수의 최대공약수를 구한다.
      • num2가 0이 될 때까지 num1을 num2로, num2를 (num1 % num2)로 계속 갱신한다.
      • 예: gcd(24, 18) → gcd(18, 6) → gcd(6, 0) → 6
    • 5) 결과 출력 
      sb.append(total).append("\n");
      
      System.out.println(sb);
      • 각 테스트 케이스의 GCD 합을 StringBuilder에 추가하고, 모든 테스트 케이스 처리 후 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(t × n² × log M): t개의 테스트 케이스마다 n개 중 2개를 선택하는 조합의 개수는 O(n²)이고, 각 쌍의 GCD 계산은 O(log M) 시간이 걸린다 (M은 수의 최댓값). 따라서 전체 시간 복잡도는 O(t × n² × log M)이다.
    • 공간 복잡도
      • O(n): 각 테스트 케이스마다 최대 n개의 수를 저장하는 배열이 필요하므로 O(n)의 공간이 필요하다.

    문제 9372 — 상근이의 여행

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    상근이의 여행 (백준 9372)는
    N개의 국가가 M개의 비행기 노선으로 연결되어 있을 때, 모든 국가를 여행하기 위해 최소한으로 이용해야 하는 비행기 종류의 수를 구하는 문제로,
    연결된 그래프에서 신장 트리(Spanning Tree)를 만드는 최소 간선 개수를 구하는 문제다.

    • 국가는 최대 1,000개, 비행기 노선은 최대 10,000개가 주어진다.
    • 모든 국가는 연결되어 있으며(하나의 연결 요소), 같은 국가 쌍을 여러 번 왕복 가능하다.
    • 연결된 그래프에서 N개의 노드를 모두 방문하려면 정확히 N-1개의 간선이 필요하다.
    • 그래프 이론에 따르면, 답은 항상 N-1이지만, BFS/DFS로 실제 트리를 구성하며 확인할 수도 있다.

    전략 개요

    • 핵심 알고리즘: BFS를 이용한 신장 트리 구성
    • 선택 이유: 연결된 그래프에서 모든 노드를 방문하는 최소 간선 수는 항상 N-1개다. BFS로 그래프를 탐색하면서 처음 방문하는 노드로 이동할 때 사용한 간선을 카운트하면, 결과적으로 N-1개의 간선을 사용한 신장 트리를 구성하게 된다.
    • 세부 전략:
      • 인접 리스트로 양방향 그래프 구성
      • 1번 국가부터 BFS 시작하여 방문하지 않은 국가 탐색
      • 새로운 국가를 방문할 때마다 카운트 증가 (사용한 비행기 종류)
      • 모든 국가를 방문하면 총 N-1개의 간선을 사용하게 됨

    코드가 하는 일

    • 1) 인접 리스트 생성 및 그래프 구성 
      List<Integer>[] list = new ArrayList[N + 1];
      for (int j = 1; j <= N; j++) {
          list[j] = new ArrayList<>();
      }
      
      for (int j = 0; j < M; j++) {
          st = new StringTokenizer(br.readLine());
          int plane1 = Integer.parseInt(st.nextToken());
          int plane2 = Integer.parseInt(st.nextToken());
          
          list[plane1].add(plane2);
          list[plane2].add(plane1);
      }
      • 각 국가마다 인접 리스트를 생성한다 (1번부터 N번까지).
      • M개의 비행기 노선을 입력받아 양방향 그래프로 구성한다.
      • 비행기는 양방향으로 운항하므로, 두 국가를 서로의 인접 리스트에 추가한다.
    • 2) BFS 초기화 
      Queue<Integer> queue = new LinkedList<>();
      int[] visited = new int[N + 1];
      int cnt = 0;
      
      queue.add(1);
      visited[1] = 1;
      • BFS를 위한 큐와 방문 여부를 체크할 배열을 준비한다.
      • 사용한 비행기 종류를 카운트할 변수 cnt를 0으로 초기화한다.
      • 1번 국가를 시작점으로 설정하고 방문 표시한다.
    • 3) BFS로 신장 트리 구성 
      while (!queue.isEmpty()) {
          int polled = queue.poll();
          
          for (int next : list[polled]) {
              if (visited[next] == 0) {
                  visited[next] = 1;
                  queue.add(next);
                  cnt++;
              }
          }
      }
      • 큐에서 현재 국가를 꺼내어, 인접한 모든 국가를 확인한다.
      • 아직 방문하지 않은 국가를 발견하면, 방문 처리하고 큐에 추가한다.
      • 새로운 국가로 이동할 때 비행기를 사용하므로 cnt를 1 증가시킨다.
      • 이 과정은 트리의 간선을 하나씩 추가하는 것과 같으며, 최종적으로 N-1개의 간선을 사용하게 된다.
    • 4) 결과 출력 
      sb.append(cnt).append("\n");
      
      System.out.println(sb);
      • 각 테스트 케이스마다 사용한 비행기 종류의 수를 StringBuilder에 추가한다.
      • 모든 테스트 케이스 처리 후 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T × (N + M)): T개의 테스트 케이스마다 BFS를 수행한다. BFS는 모든 노드(N)와 간선(M)을 한 번씩 방문하므로 O(N + M)이다. 따라서 전체 시간 복잡도는 O(T × (N + M))이다.
    • 공간 복잡도
      • O(N + M): 인접 리스트에 N개의 노드와 M개의 간선 정보를 저장하고, 방문 배열과 큐에 최대 N개의 원소가 들어갈 수 있으므로 O(N + M)의 공간이 필요하다.
