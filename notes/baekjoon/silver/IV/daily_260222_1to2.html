    문제 28279 — 덱 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    덱 2 (백준 28279)는
    정수를 저장하는 덱(Deque)을 직접 구현하여 8가지 명령어를 처리하는
    자료구조 시뮬레이션 문제다.

    • 명령어는 앞/뒤 삽입, 앞/뒤 삭제, 크기 조회, 비어있는지 확인, 앞/뒤 원소 조회로 총 8가지다.
    • 덱이 비어있는 상태에서 삭제·조회를 시도하면 -1을 출력한다.
    • N은 최대 1,000,000이므로 ArrayDeque나 직접 구현한 배열 덱을 사용해야 빠르게 처리할 수 있다.

    전략 개요

    • 핵심 알고리즘: 배열 기반 덱 직접 구현 (Array Deque)
    • 선택 이유: Java의 ArrayDeque를 사용해도 되지만, 배열과 front/rear 포인터로 직접 구현하면 객체 생성 오버헤드 없이 O(1) 삽입·삭제가 가능하다. N이 최대 100만이므로 빠른 입출력과 함께 이 방식이 안정적이다.
    • 세부 전략:
      • 배열 크기를 N * 2 + 1로 잡고, front와 rear를 모두 중앙값 N으로 초기화해 앞뒤로 충분히 공간을 확보한다.
      • 앞에 삽입할 때는 front를 먼저 감소시킨 뒤 저장하고, 뒤에 삽입할 때는 현재 rear 위치에 저장 후 증가시킨다.
      • front == rear이면 덱이 비어있는 상태다.
      • 덱의 크기는 rear - front로 O(1)에 계산한다.

    코드가 하는 일

    • 1) 배열 덱 초기화 
      int[] deque = new int[N * 2 + 1];
      int front = N;
      int rear = N;
      • 배열 크기를 N * 2 + 1로 설정해 앞뒤 방향 모두 최대 N번의 삽입을 수용할 수 있도록 한다.
      • front와 rear를 배열 중앙인 N으로 초기화한다. 두 포인터가 같은 위치를 가리키므로 초기 상태는 빈 덱이다.
    • 2) 앞/뒤 삽입 (명령어 1, 2) 
      case 1:
          num = Integer.parseInt(st.nextToken());
          deque[--front] = num;
          break;
      case 2:
          num = Integer.parseInt(st.nextToken());
          deque[rear++] = num;
          break;
      • 명령어 1(앞 삽입): front를 먼저 1 감소시킨 뒤 해당 위치에 값을 저장한다. 앞쪽 공간을 향해 포인터가 이동한다.
      • 명령어 2(뒤 삽입): 현재 rear 위치에 값을 저장한 뒤 rear를 1 증가시킨다. 뒤쪽 공간을 향해 포인터가 이동한다.
    • 3) 앞/뒤 삭제 (명령어 3, 4) 
      case 3:
          if (front == rear) {
              sb.append(-1).append("\n");
          } else {
              sb.append(deque[front++]).append("\n");
          }
          break;
      case 4:
          if (front == rear) {
              sb.append(-1).append("\n");
          } else {
              sb.append(deque[--rear]).append("\n");
          }
          break;
      • 명령어 3(앞 삭제): 덱이 비어있으면 -1 출력. 그렇지 않으면 deque[front]를 출력하고 front를 1 증가시킨다.
      • 명령어 4(뒤 삭제): 덱이 비어있으면 -1 출력. 그렇지 않으면 rear를 먼저 1 감소시킨 뒤 deque[rear]를 출력한다.
    • 4) 크기·비어있는지·앞/뒤 조회 (명령어 5, 6, 7, 8) 
      case 5:
          sb.append(rear - front).append("\n");
          break;
      case 6:
          if (front == rear) { sb.append(1).append("\n"); }
          else { sb.append(0).append("\n"); }
          break;
      case 7:
          if (front == rear) { sb.append(-1).append("\n"); }
          else { sb.append(deque[front]).append("\n"); }
          break;
      case 8:
          if (front == rear) { sb.append(-1).append("\n"); }
          else { sb.append(deque[rear - 1]).append("\n"); }
      • 명령어 5(크기): rear - front가 현재 덱에 저장된 원소 수다.
      • 명령어 6(비어있는지): front == rear이면 1(비어있음), 아니면 0을 출력한다.
      • 명령어 7(앞 원소 조회): front가 가리키는 위치의 값을 출력한다. 포인터를 이동시키지 않으므로 삭제 없이 조회만 한다.
      • 명령어 8(뒤 원소 조회): rear - 1 위치의 값을 출력한다. 마찬가지로 포인터를 이동시키지 않는다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 명령어를 순서대로 처리하며, 각 명령어는 배열 인덱스 접근과 포인터 이동만으로 모두 O(1)에 처리된다. 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • O(N): 덱을 저장하는 배열의 크기가 N * 2 + 1이므로 입력 크기에 비례하는 공간을 사용한다.

    문제 2578 — 빙고

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    빙고 (백준 2578)는
    5×5 빙고판에서 사회자가 번호를 하나씩 부를 때, 빙고가 3줄 이상 완성되는 순간 몇 번째 숫자가 불렸는지 출력하는 문제다.
    가로·세로·대각선 총 12방향을 매번 검사해 빙고 개수를 센다.

    • 빙고판은 5×5 크기이며, 1부터 25까지의 수가 한 번씩 등장한다.
    • 빙고는 가로 5줄, 세로 5줄, 대각선 2줄로 총 12가지 방향이 존재한다.
    • 번호가 불릴 때마다 해당 위치를 체크하고, 빙고가 3개 이상이 되는 즉시 현재까지 불린 번호의 순서를 출력한다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 번호-좌표 매핑 + 빙고 카운팅 시뮬레이션
    • 선택 이유: 사회자가 번호를 부를 때마다 빙고판에서 해당 번호의 위치를 빠르게 찾아야 한다. HashMap에 번호를 키, 좌표를 값으로 저장하면 O(1)에 위치를 조회할 수 있다. 빙고판이 5×5로 고정되어 있으므로 매 호출마다 12방향 전체를 검사해도 O(1)에 수렴한다.
    • 세부 전략:
      • 빙고판을 읽으며 각 숫자의 (행, 열) 좌표를 HashMap에 저장한다.
      • 사회자가 부르는 번호를 순서대로 읽으며, HashMap으로 좌표를 찾아 visited 배열을 체크한다.
      • 체크 후 countBingo()를 호출해 현재 완성된 빙고 수를 계산하고, 3 이상이면 즉시 답을 출력하고 종료한다.

    코드가 하는 일

    • 1) 빙고판 읽기 및 HashMap 구성 
      Map map = new HashMap<>();
      
      for (int r = 0; r < 5; r++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          for (int c = 0; c < 5; c++) {
              int num = Integer.parseInt(st.nextToken());
              map.put(num, new int[]{r, c});
          }
      }
      • 5×5 빙고판을 읽으며 각 숫자를 키로, 해당 숫자가 위치한 (행, 열) 배열을 값으로 HashMap에 저장한다.
      • 이후 사회자가 번호를 부를 때 map.get(num)으로 O(1)에 좌표를 찾을 수 있다.
    • 2) 번호 호출 및 체크 
      for (int r = 0; r < 5; r++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          for (int c = 0; c < 5; c++) {
              int num = Integer.parseInt(st.nextToken());
              int[] ok = map.get(num);
              answer++;
      
              visited[ok[0]][ok[1]] = true;
              if (countBingo() >= 3) {
                  System.out.println(answer);
                  return;
              }
          }
      }
      • 사회자가 부르는 번호를 순서대로 읽으며 answer(호출 순서)를 1씩 증가시킨다.
      • HashMap으로 해당 번호의 좌표를 찾아 visited 배열의 해당 위치를 true로 표시한다.
      • 표시 후 즉시 countBingo()를 호출해 빙고 수를 확인하고, 3 이상이면 현재 answer를 출력하고 종료한다.
    • 3) 빙고 수 계산 — 가로·세로 
      for (int r = 0; r < 5; r++) {
          int temp = 0;
          for (int c = 0; c < 5; c++) {
              if (visited[r][c]) temp++;
          }
          if (temp == 5) count++;
      }
      
      for (int r = 0; r < 5; r++) {
          int temp = 0;
          for (int c = 0; c < 5; c++) {
              if (visited[c][r]) temp++;
          }
          if (temp == 5) count++;
      }
      • 첫 번째 이중 반복문은 가로 방향 5줄을 검사한다. 한 행의 5칸이 모두 true이면 빙고 1개로 카운트한다.
      • 두 번째 이중 반복문은 세로 방향 5줄을 검사한다. 행과 열 인덱스를 바꿔 visited[c][r]로 접근해 열 단위로 검사한다.
    • 4) 빙고 수 계산 — 대각선 
      int temp = 0;
      for (int idx = 0; idx < 5; idx++) {
          if (visited[idx][idx]) temp++;
      }
      if (temp == 5) count++;
      
      temp = 0;
      for (int idx = 0; idx < 5; idx++) {
          if (visited[idx][4 - idx]) temp++;
      }
      if (temp == 5) count++;
      • 첫 번째 반복문은 좌상→우하 방향 대각선을 검사한다. visited[idx][idx]로 (0,0), (1,1), (2,2), (3,3), (4,4)를 순서대로 확인한다.
      • 두 번째 반복문은 우상→좌하 방향 대각선을 검사한다. visited[idx][4-idx]로 (0,4), (1,3), (2,2), (3,1), (4,0)을 순서대로 확인한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 빙고판이 5×5로 고정되어 있으므로 모든 연산(번호 호출, 좌표 조회, 빙고 카운팅)이 상수 시간 내에 완료된다. 번호는 최대 25개 호출되며, 매 호출마다 빙고 카운팅 비용은 5×5 = 25칸 고정이므로 전체 O(25 × 25) = O(1)로 볼 수 있다.
    • 공간 복잡도
      • O(1): visited 배열(5×5)과 HashMap(최대 25개 항목)이 고정 크기를 사용하므로 상수 공간이다.
