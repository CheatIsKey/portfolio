    문제 26069 — 붙임성 좋은 총총이

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    붙임성 좋은 총총이 (백준 26069)는
    총총이와 춤을 춘 사람은 총총이 댄스를 배우고, 그 사람과 춤을 춘 또 다른 사람도 댄스를 배우는 방식으로
    총총이 댄스를 추게 된 사람이 최종적으로 몇 명인지 구하는 문제다.
    댄스는 총총이와 직·간접적으로 연결된 모든 사람에게 전파된다.

    • 총총이("ChongChong")는 처음부터 댄스를 출 수 있으며, 총총이 댄스를 아는 사람과 만나면 상대방도 댄스를 배운다.
    • 만남은 순서대로 주어지며, 한 번 댄스를 배운 사람은 이후 다른 사람과의 만남에서 댄스를 전파할 수 있다.
    • 최종적으로 댄스를 아는 사람의 수(총총이 포함)를 출력한다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 댄스 전파 시뮬레이션
    • 선택 이유: 댄스를 아는 사람 목록을 관리하고, 각 만남에서 둘 중 한 명이 댄스를 아는지 O(1)로 조회해야 한다. HashMap은 문자열 키에 대한 삽입·조회가 평균 O(1)이므로 최적이다.
    • 세부 전략:
      • 총총이("ChongChong")를 true로 먼저 HashMap에 등록한다.
      • 각 만남에서 두 사람 중 한 명이라도 HashMap에 true로 등록되어 있으면, 나머지 한 명도 true로 등록한다.
      • getOrDefault를 사용해 등록되지 않은 사람은 false로 처리하여 NullPointerException을 방지한다.
      • 모든 만남을 처리한 뒤 HashMap의 값 중 true인 항목의 수를 세어 출력한다.

    코드가 하는 일

    • 1) 초기화 — 총총이 등록 
      Map map = new HashMap<>();
      map.put("ChongChong", true);
      • 댄스를 아는 사람을 관리하는 HashMap을 생성하고, 총총이를 true로 등록한다.
      • 이후 모든 전파의 시작점은 이 초기 등록에서 비롯된다.
    • 2) 만남 처리 — 댄스 전파 
      for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          String people1 = st.nextToken();
          String people2 = st.nextToken();
      
          if (map.getOrDefault(people1, false)) {
              map.put(people2, true);
          } else if (map.getOrDefault(people2, false)) {
              map.put(people1, true);
          }
      }
      • 만남이 주어질 때마다 두 사람의 이름을 읽는다.
      • map.getOrDefault(people1, false)는 people1이 댄스를 아는 경우 true를, 등록되지 않은 경우 기본값 false를 반환한다. 이를 통해 등록되지 않은 사람에 대한 예외 처리를 간결하게 해결한다.
      • people1이 댄스를 알면 people2를 true로 등록하고, people2가 댄스를 알면 people1을 true로 등록한다.
      • 둘 다 댄스를 모르는 경우에는 아무런 변화가 없다.
      • 주의: 두 조건을 if-else if로 처리하므로, 두 사람 모두 댄스를 아는 경우 이미 둘 다 등록된 상태이므로 어느 분기를 타든 결과가 동일하다.
    • 3) 댄스를 아는 사람 수 집계 및 출력 
      int cnt = 0;
      
      for (Boolean value : map.values()) {
          if (value) {
              cnt++;
          }
      }
      
      System.out.println(cnt);
      • HashMap의 모든 값을 순회하며 true인 항목의 수를 센다.
      • 이 구현에서는 HashMap에 삽입될 때 항상 true로만 등록되므로, 실질적으로 map.size()와 동일한 결과를 낸다. 다만 명시적으로 true 여부를 확인해 안전성을 높였다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N번의 만남을 순서대로 처리하며, 각 만남에서 HashMap 조회·삽입이 평균 O(1)이다. 마지막 집계 단계도 등록된 사람 수에 비례하지만 최대 2N+1명이므로 O(N)이다. 전체 시간 복잡도는 O(N)이다.
    • 공간 복잡도
      • O(N): HashMap에 저장되는 사람의 수는 최대 2N+1명(총총이 포함)이므로 입력 크기에 비례하는 공간을 사용한다.

    문제 1015 — 수열 정렬

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수열 정렬 (백준 1015)는
    수열 A를 정렬했을 때 얻어지는 수열 B에 대해, B[i] = A[P[i]]를 만족하는 순열 P를 구하는 문제다.
    P가 여러 개일 경우 사전순으로 가장 앞서는 P를 출력해야 한다.

    • B는 A를 오름차순 정렬한 수열이므로, P[i]는 B의 i번째 원소가 A의 몇 번째 인덱스에서 왔는지를 나타낸다.
    • A에 같은 값이 여러 개 있을 경우, 사전순으로 가장 앞선 P를 만들려면 같은 값 중 원래 인덱스가 작은 것을 먼저 배치해야 한다.
    • N은 최대 50으로 매우 작으므로 정렬 기반 접근으로 충분히 해결된다.

    전략 개요

    • 핵심 알고리즘: 인덱스 보존 정렬 (Index-tagged Sort)
    • 선택 이유: 정렬 후 각 원소가 원래 어느 인덱스에 있었는지 추적해야 하므로, 원소에 원래 인덱스를 함께 저장한 뒤 정렬하는 방식이 가장 직관적이다. 값이 같을 때 인덱스 오름차순으로 정렬하면 사전순 최솟값 조건도 자동으로 만족된다.
    • 세부 전략:
      • 각 원소를 (원래 인덱스, 값)의 쌍으로 저장한 리스트를 만든다.
      • 값 오름차순으로 정렬하되, 값이 같으면 원래 인덱스 오름차순으로 정렬한다.
      • 정렬 결과에서 i번째 원소의 원래 인덱스가 P[i]의 값이 된다. 즉, 정렬 후 i번째에 위치한 원소의 원래 인덱스 위치에 i를 기록한다.

    코드가 하는 일

    • 1) 인덱스-값 쌍 리스트 구성 
      List list = new ArrayList<>();
      
      for (int i = 0; i < N; i++) {
          int num = Integer.parseInt(st.nextToken());
          list.add(new int[]{i, num});
      }
      • 각 원소를 {원래 인덱스, 값} 형태의 배열로 리스트에 저장한다.
      • 정렬 후에도 원래 인덱스 정보를 잃지 않기 위한 핵심 전처리 단계다.
    • 2) 값 오름차순, 동률 시 인덱스 오름차순 정렬 
      list.sort((a, b) -> {
          if (a[1] == b[1]) {
              return Integer.compare(a[0], b[0]);
          }
          return Integer.compare(a[1], b[1]);
      });
      • 기본 정렬 기준은 값(a[1]) 오름차순이다.
      • 값이 같은 경우 원래 인덱스(a[0]) 오름차순으로 정렬한다. 이렇게 하면 중복 값이 있을 때 원래 앞에 있던 원소가 먼저 배치되어, 사전순으로 가장 앞서는 P가 자동으로 만들어진다.
      • 예시: A = [3, 1, 3]이면 정렬 후 [(1,1), (0,3), (2,3)]이 된다. 값 3이 인덱스 0과 2에 있으므로 인덱스 오름차순에 따라 인덱스 0이 먼저 온다.
    • 3) 순열 P 배열 구성 
      int[] numbers = new int[N];
      
      for (int i = 0; i < N; i++) {
          int idx = list.get(i)[0];
          numbers[idx] = i;
      }
      • 정렬된 리스트의 i번째 원소의 원래 인덱스 idx를 꺼낸다.
      • numbers[idx] = i로 기록한다. 즉, 원래 인덱스 idx에 있던 원소가 정렬 후 i번째 위치로 이동했음을 의미하며, 이것이 P[idx]의 값이 된다.
      • 예시: A = [3, 1, 3]에서 정렬 후 [(1,1), (0,3), (2,3)]이므로 numbers[1]=0, numbers[0]=1, numbers[2]=2. 결과 P = [1, 0, 2].
    • 4) 결과 출력 
      for (int number : numbers) {
          sb.append(number).append(" ");
      }
      System.out.println(sb);
      • 완성된 순열 P를 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 리스트 정렬 비용이다. 이후 순열 구성과 출력은 O(N)이므로 전체 O(N log N)이다. N이 최대 50으로 매우 작아 실질적으로 거의 즉시 수행된다.
    • 공간 복잡도
      • O(N): 인덱스-값 쌍을 저장하는 리스트와 결과를 저장하는 배열이 각각 N 크기를 사용한다.
