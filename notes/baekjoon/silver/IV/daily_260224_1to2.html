    문제 2847 — 게임을 만든 동준이

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    게임을 만든 동준이 (백준 2847)는
    N개의 레벨 점수가 주어질 때, 각 레벨의 점수가 이전 레벨보다 반드시 커지도록 만들기 위해
    점수를 감소시켜야 하는 총 횟수의 최솟값을 구하는 문제다.
    단, 점수는 1 이상이어야 하며 점수를 늘리는 것은 불가능하고 줄이는 것만 가능하다.

    • 레벨은 1번부터 N번까지 있으며, 레벨 번호가 높을수록 점수가 더 커야 한다.
    • 점수를 늘릴 수 없으므로, 조건을 위반하는 경우 앞 레벨의 점수를 줄여야 한다.
    • 감소 횟수를 최소화하려면 뒤에서부터 역순으로 탐색하며, 조건 위반 시 앞 레벨 점수를 최소한으로 줄이는 것이 최적이다.

    전략 개요

    • 핵심 알고리즘: 그리디 (뒤에서부터 역순 탐색)
    • 선택 이유: 점수를 늘릴 수 없으므로, 조건을 위반하는 앞 레벨의 점수를 최소한으로 줄여야 한다. 뒤에서부터 탐색하면 이미 확정된 뒤 레벨 점수를 기준으로 앞 레벨을 조정할 수 있어, 연쇄적인 영향을 자연스럽게 처리할 수 있다.
    • 세부 전략:
      • 배열의 뒤에서 두 번째 인덱스(N-2)부터 0까지 역순으로 순회한다.
      • levels[i-1] >= levels[i]이면 조건 위반이므로, levels[i-1]을 levels[i] - 1로 줄인다. 이때 줄인 양만큼 total에 누적한다.
      • 줄인 값을 실제로 levels[i-1]에 반영해, 다음 이터레이션에서 연쇄 조건 위반을 올바르게 처리한다.

    코드가 하는 일

    • 1) 입력 및 배열 구성 
      int N = Integer.parseInt(br.readLine());
      int[] levels = new int[N];
      
      for (int i = 0; i < N; i++) {
          levels[i] = Integer.parseInt(br.readLine());
      }
      • 레벨 수 N을 읽은 뒤, 각 레벨의 점수를 순서대로 levels 배열에 저장한다.
      • 입력 순서가 레벨 순서이므로 levels[0]이 1레벨, levels[N-1]이 N레벨의 점수다.
    • 2) 뒤에서부터 역순으로 조건 위반 교정 
      int total = 0;
      for (int i = N - 1; i > 0; i--) {
          if (levels[i - 1] >= levels[i]) {
              int decrease = levels[i - 1] - (levels[i] - 1);
              total += decrease;
              levels[i - 1] = levels[i] - 1;
          }
      }
      • i를 N-1부터 1까지 역순으로 순회하며, 인접한 두 레벨 levels[i-1]과 levels[i]를 비교한다.
      • levels[i-1] >= levels[i]이면 앞 레벨 점수가 뒤 레벨 점수 이상이므로 조건을 위반한 것이다.
      • 이 경우 levels[i-1]을 levels[i] - 1로 줄인다. 줄인 양 decrease = levels[i-1] - (levels[i] - 1)를 total에 더한다.
      • 줄인 값을 실제로 levels[i-1]에 반영해, 그 다음 이터레이션(i-1번째)에서 levels[i-2]와 비교할 때 갱신된 값을 기준으로 삼도록 한다.
      • 예시: levels = [5, 5, 5]이면 i=2에서 levels[1]을 4로 줄이고(total=1), i=1에서 levels[0]을 3으로 줄인다(total=3). 최종 total = 3.
    • 3) 결과 출력 
      System.out.println(total);
      • 누적된 총 감소 횟수 total을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 역순으로 순회하며 각 인덱스에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 레벨 점수를 저장하는 levels 배열이 N 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 2776 — 암기왕

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    암기왕 (백준 2776)는
    수첩1에 적힌 수들을 기억하고, 수첩2에 적힌 각 수가 수첩1에 존재하면 1, 존재하지 않으면 0을 출력하는 문제다.
    N과 M이 최대 1,000,000이므로 선형 탐색 대신 이진 탐색을 사용해야 한다.

    • 테스트 케이스 T가 주어지며, 각 케이스마다 수첩1의 N개의 수와 수첩2의 M개의 수를 처리한다.
    • 수첩1의 수는 중복이 있을 수 있지만, 존재 여부만 확인하면 되므로 중복 제거 없이 정렬 후 이진 탐색으로 충분하다.
    • N, M이 최대 100만이므로 O(N log N) 정렬 + O(M log N) 탐색으로 처리해야 시간 초과를 피할 수 있다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 이진 탐색 (Binary Search)
    • 선택 이유: 수첩1을 정렬해두면 수첩2의 각 수에 대해 O(log N)에 존재 여부를 확인할 수 있다. N, M이 최대 100만이므로 선형 탐색 O(N × M)은 시간 초과가 발생하지만, 이진 탐색은 O(M log N)으로 충분히 빠르다.
    • 세부 전략:
      • 수첩1의 수를 배열에 저장하고 오름차순 정렬한다.
      • 수첩2의 각 수에 대해 직접 구현한 이진 탐색을 수행해 존재하면 1, 없으면 0을 반환한다.
      • 중간값 계산 시 start + (end - start) / 2를 사용해 정수 오버플로우를 방지한다.

    코드가 하는 일

    • 1) 수첩1 입력 및 정렬 
      int N = Integer.parseInt(br.readLine());
      int[] numbers = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(numbers);
      • 수첩1의 N개의 수를 배열에 저장한 뒤 Arrays.sort()로 오름차순 정렬한다.
      • 정렬된 배열이어야 이진 탐색을 적용할 수 있다.
    • 2) 수첩2의 각 수에 대해 이진 탐색 수행 
      int M = Integer.parseInt(br.readLine());
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < M; i++) {
          int number = Integer.parseInt(st.nextToken());
          sb.append(binarySearch(numbers, number, 0, N - 1)).append('\n');
      }
      • 수첩2의 M개의 수를 순서대로 읽으며, 각 수에 대해 binarySearch()를 호출한다.
      • 반환값(0 또는 1)을 StringBuilder에 누적해 마지막에 한 번에 출력함으로써 I/O 비용을 줄인다.
    • 3) 이진 탐색 구현 
      private static int binarySearch(int[] numbers, int target, int start, int end) {
          while (start <= end) {
              int mid = start + (end - start) / 2;
      
              if (numbers[mid] == target) {
                  return 1;
              } else if (numbers[mid] < target) {
                  start = mid + 1;
              } else {
                  end = mid - 1;
              }
          }
      
          return 0;
      }
      • start <= end인 동안 반복하며, 중간 인덱스 mid = start + (end - start) / 2를 계산한다. (start + end) / 2 대신 이 방식을 사용하면 두 값의 합이 int 범위를 초과하는 오버플로우를 방지할 수 있다.
      • numbers[mid] == target이면 존재하므로 즉시 1을 반환한다.
      • numbers[mid] < target이면 탐색 범위를 오른쪽(start = mid + 1)으로 좁히고, 크면 왼쪽(end = mid - 1)으로 좁힌다.
      • 반복이 종료될 때까지 찾지 못하면 0을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N + M log N): 수첩1 정렬에 O(N log N), 수첩2의 M개의 수에 대한 이진 탐색에 O(M log N)이 소요된다. N, M이 최대 100만이므로 약 2,000만 번의 연산으로 충분히 처리된다.
    • 공간 복잡도
      • O(N): 수첩1의 N개의 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
