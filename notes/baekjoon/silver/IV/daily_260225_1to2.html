    문제 1205 — 등수 구하기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    등수 구하기 (백준 1205)는
    현재 상위 P개의 점수가 저장된 순위표에 새로운 점수를 추가할 때,
    새 점수의 등수를 구하는 문제다.
    순위표에 진입하지 못하는 경우에는 -1을 출력해야 한다.

    • 순위표는 내림차순으로 정렬되어 있으며, 현재 N개의 점수가 저장되어 있다. (N ≤ P)
    • 순위표가 꽉 찬 상태(N == P)에서 새 점수가 마지막 점수 이하이면 순위표에 진입하지 못하므로 -1을 출력한다.
    • 등수는 새 점수보다 높은(큰) 점수의 수 + 1로 계산한다. 동점자가 있으면 같은 등수를 부여한다.

    전략 개요

    • 핵심 알고리즘: 선형 탐색 + 예외 처리
    • 선택 이유: 순위표가 이미 내림차순으로 정렬되어 있으므로, 앞에서부터 순회하며 새 점수보다 큰 점수의 수를 세면 O(N)에 등수를 구할 수 있다. P가 최대 50,000이므로 선형 탐색으로 충분하다.
    • 세부 전략:
      • N이 0이면 순위표가 비어있으므로 새 점수의 등수는 무조건 1이다.
      • N == P이고 새 점수가 마지막 점수 이하이면 순위표에 진입 불가이므로 -1을 출력한다.
      • 그 외의 경우, 정렬된 배열을 앞에서부터 순회하며 새 점수보다 큰 점수의 수를 세고, 1을 더해 등수를 구한다.
      • 배열이 내림차순이므로 새 점수 이상인 점수를 만나는 순간 반복을 종료한다.

    코드가 하는 일

    • 1) 입력 및 N == 0 예외 처리 
      int N = Integer.parseInt(st.nextToken());
      int newScore = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      if (N == 0) {
          System.out.println(1);
          return;
      }
      • 현재 순위표에 저장된 점수의 수 N, 새 점수 newScore, 순위표 최대 크기 P를 입력받는다.
      • N이 0이면 순위표가 비어있으므로 새 점수는 무조건 1등이다. 즉시 1을 출력하고 종료한다.
    • 2) 순위표 진입 불가 판별 
      if (N == P && newScore <= scores[N - 1]) {
          System.out.println(-1);
          return;
      }
      • 순위표가 꽉 찬 상태(N == P)에서 새 점수가 마지막 점수(scores[N-1]) 이하이면 순위표에 진입할 수 없다.
      • 동점인 경우(newScore == scores[N-1])도 -1 처리함에 주의한다. 순위표가 꽉 찬 상태에서 동점이면 기존 점수를 밀어내고 들어오지 못하기 때문이다.
      • N < P이면 순위표에 빈 자리가 있으므로 이 조건에 해당하지 않는다.
    • 3) 등수 계산 
      int rank = 1;
      for (int i = 0; i < N; i++) {
          if (scores[i] > newScore) {
              rank++;
          } else {
              break;
          }
      }
      
      System.out.println(rank);
      • rank를 1로 초기화한 뒤, 내림차순 정렬된 배열을 앞에서부터 순회하며 새 점수보다 큰 점수를 만날 때마다 rank를 1 증가시킨다.
      • 새 점수 이상인 점수를 만나는 순간 반복을 종료한다. 동점자는 같은 등수를 부여하므로 scores[i] == newScore이면 rank를 증가시키지 않고 즉시 break한다.
      • 예시: scores = [100, 90, 80], newScore = 85이면 100 > 85(rank=2), 90 > 85(rank=3), 80 ≤ 85(break). 결과 등수는 3.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 순위표 배열을 최대 한 번 순회하며 등수를 계산한다. 각 단계의 비교 연산은 O(1)이므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 현재 순위표의 N개 점수를 저장하는 scores 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 2491 — 수열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    암기왕 (백준 2776)는
    수첩1에 적힌 수들을 기억하고, 수첩2에 적힌 각 수가 수첩1에 존재하면 1, 존재하지 않으면 0을 출력하는 문제다.
    N과 M이 최대 1,000,000이므로 선형 탐색 대신 이진 탐색을 사용해야 한다.

    • 테스트 케이스 T가 주어지며, 각 케이스마다 수첩1의 N개의 수와 수첩2의 M개의 수를 처리한다.
    • 수첩1의 수는 중복이 있을 수 있지만, 존재 여부만 확인하면 되므로 중복 제거 없이 정렬 후 이진 탐색으로 충분하다.
    • N, M이 최대 100만이므로 O(N log N) 정렬 + O(M log N) 탐색으로 처리해야 시간 초과를 피할 수 있다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 이진 탐색 (Binary Search)
    • 선택 이유: 수첩1을 정렬해두면 수첩2의 각 수에 대해 O(log N)에 존재 여부를 확인할 수 있다. N, M이 최대 100만이므로 선형 탐색 O(N × M)은 시간 초과가 발생하지만, 이진 탐색은 O(M log N)으로 충분히 빠르다.
    • 세부 전략:
      • 수첩1의 수를 배열에 저장하고 오름차순 정렬한다.
      • 수첩2의 각 수에 대해 직접 구현한 이진 탐색을 수행해 존재하면 1, 없으면 0을 반환한다.
      • 중간값 계산 시 start + (end - start) / 2를 사용해 정수 오버플로우를 방지한다.

    코드가 하는 일

    • 1) 수첩1 입력 및 정렬 
      int N = Integer.parseInt(br.readLine());
      int[] numbers = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(numbers);
      • 수첩1의 N개의 수를 배열에 저장한 뒤 Arrays.sort()로 오름차순 정렬한다.
      • 정렬된 배열이어야 이진 탐색을 적용할 수 있다.
    • 2) 수첩2의 각 수에 대해 이진 탐색 수행 
      int M = Integer.parseInt(br.readLine());
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < M; i++) {
          int number = Integer.parseInt(st.nextToken());
          sb.append(binarySearch(numbers, number, 0, N - 1)).append('\n');
      }
      • 수첩2의 M개의 수를 순서대로 읽으며, 각 수에 대해 binarySearch()를 호출한다.
      • 반환값(0 또는 1)을 StringBuilder에 누적해 마지막에 한 번에 출력함으로써 I/O 비용을 줄인다.
    • 3) 이진 탐색 구현 
      private static int binarySearch(int[] numbers, int target, int start, int end) {
          while (start <= end) {
              int mid = start + (end - start) / 2;
      
              if (numbers[mid] == target) {
                  return 1;
              } else if (numbers[mid] < target) {
                  start = mid + 1;
              } else {
                  end = mid - 1;
              }
          }
      
          return 0;
      }
      • start <= end인 동안 반복하며, 중간 인덱스 mid = start + (end - start) / 2를 계산한다. (start + end) / 2 대신 이 방식을 사용하면 두 값의 합이 int 범위를 초과하는 오버플로우를 방지할 수 있다.
      • numbers[mid] == target이면 존재하므로 즉시 1을 반환한다.
      • numbers[mid] < target이면 탐색 범위를 오른쪽(start = mid + 1)으로 좁히고, 크면 왼쪽(end = mid - 1)으로 좁힌다.
      • 반복이 종료될 때까지 찾지 못하면 0을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N + M log N): 수첩1 정렬에 O(N log N), 수첩2의 M개의 수에 대한 이진 탐색에 O(M log N)이 소요된다. N, M이 최대 100만이므로 약 2,000만 번의 연산으로 충분히 처리된다.
    • 공간 복잡도
      • O(N): 수첩1의 N개의 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
