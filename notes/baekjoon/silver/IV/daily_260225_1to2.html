    문제 1205 — 등수 구하기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    등수 구하기 (백준 1205)는
    현재 상위 P개의 점수가 저장된 순위표에 새로운 점수를 추가할 때,
    새 점수의 등수를 구하는 문제다.
    순위표에 진입하지 못하는 경우에는 -1을 출력해야 한다.

    • 순위표는 내림차순으로 정렬되어 있으며, 현재 N개의 점수가 저장되어 있다. (N ≤ P)
    • 순위표가 꽉 찬 상태(N == P)에서 새 점수가 마지막 점수 이하이면 순위표에 진입하지 못하므로 -1을 출력한다.
    • 등수는 새 점수보다 높은(큰) 점수의 수 + 1로 계산한다. 동점자가 있으면 같은 등수를 부여한다.

    전략 개요

    • 핵심 알고리즘: 선형 탐색 + 예외 처리
    • 선택 이유: 순위표가 이미 내림차순으로 정렬되어 있으므로, 앞에서부터 순회하며 새 점수보다 큰 점수의 수를 세면 O(N)에 등수를 구할 수 있다. P가 최대 50,000이므로 선형 탐색으로 충분하다.
    • 세부 전략:
      • N이 0이면 순위표가 비어있으므로 새 점수의 등수는 무조건 1이다.
      • N == P이고 새 점수가 마지막 점수 이하이면 순위표에 진입 불가이므로 -1을 출력한다.
      • 그 외의 경우, 정렬된 배열을 앞에서부터 순회하며 새 점수보다 큰 점수의 수를 세고, 1을 더해 등수를 구한다.
      • 배열이 내림차순이므로 새 점수 이상인 점수를 만나는 순간 반복을 종료한다.

    코드가 하는 일

    • 1) 입력 및 N == 0 예외 처리 
      int N = Integer.parseInt(st.nextToken());
      int newScore = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      if (N == 0) {
          System.out.println(1);
          return;
      }
      • 현재 순위표에 저장된 점수의 수 N, 새 점수 newScore, 순위표 최대 크기 P를 입력받는다.
      • N이 0이면 순위표가 비어있으므로 새 점수는 무조건 1등이다. 즉시 1을 출력하고 종료한다.
    • 2) 순위표 진입 불가 판별 
      if (N == P && newScore <= scores[N - 1]) {
          System.out.println(-1);
          return;
      }
      • 순위표가 꽉 찬 상태(N == P)에서 새 점수가 마지막 점수(scores[N-1]) 이하이면 순위표에 진입할 수 없다.
      • 동점인 경우(newScore == scores[N-1])도 -1 처리함에 주의한다. 순위표가 꽉 찬 상태에서 동점이면 기존 점수를 밀어내고 들어오지 못하기 때문이다.
      • N < P이면 순위표에 빈 자리가 있으므로 이 조건에 해당하지 않는다.
    • 3) 등수 계산 
      int rank = 1;
      for (int i = 0; i < N; i++) {
          if (scores[i] > newScore) {
              rank++;
          } else {
              break;
          }
      }
      
      System.out.println(rank);
      • rank를 1로 초기화한 뒤, 내림차순 정렬된 배열을 앞에서부터 순회하며 새 점수보다 큰 점수를 만날 때마다 rank를 1 증가시킨다.
      • 새 점수 이상인 점수를 만나는 순간 반복을 종료한다. 동점자는 같은 등수를 부여하므로 scores[i] == newScore이면 rank를 증가시키지 않고 즉시 break한다.
      • 예시: scores = [100, 90, 80], newScore = 85이면 100 > 85(rank=2), 90 > 85(rank=3), 80 ≤ 85(break). 결과 등수는 3.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 순위표 배열을 최대 한 번 순회하며 등수를 계산한다. 각 단계의 비교 연산은 O(1)이므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 현재 순위표의 N개 점수를 저장하는 scores 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 2491 — 수열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수열 (백준 2491)는
    N개의 수로 이루어진 수열에서 단조 증가(비감소) 또는 단조 감소(비증가)하는 가장 긴 연속 부분 수열의 길이를 구하는 문제다.
    같은 수가 연속으로 나오는 경우 증가와 감소 모두에 해당한다.

    • 연속 부분 수열이므로 수열에서 연속된 구간만 고려한다.
    • 같은 수가 연속될 경우(prev == number) 증가 조건(prev <= number)과 감소 조건(prev >= number) 모두를 만족하므로 양쪽 카운트가 동시에 늘어난다.
    • N이 최대 100,000이므로 O(N) 동적 프로그래밍으로 해결해야 한다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (DP)
    • 선택 이유: 각 위치에서의 최장 비감소·비증가 부분 수열의 길이는 바로 이전 위치의 값에만 의존한다. 따라서 increase[i]와 decrease[i] 두 DP 배열을 한 번의 순회로 동시에 채울 수 있어 O(N)에 해결된다.
    • 세부 전략:
      • increase[i]: i번째 수에서 끝나는 가장 긴 비감소(오름차순) 연속 부분 수열의 길이.
      • decrease[i]: i번째 수에서 끝나는 가장 긴 비증가(내림차순) 연속 부분 수열의 길이.
      • 두 배열 모두 1로 초기화하고, 조건을 만족하면 이전 값에 1을 더하고 그렇지 않으면 1로 리셋한다.
      • 매 단계에서 두 배열의 현재 값 중 최댓값을 answer에 갱신한다.

    코드가 하는 일

    • 1) DP 배열 초기화 
      int[] increase = new int[N];
      int[] decrease = new int[N];
      
      Arrays.fill(increase, 1);
      Arrays.fill(decrease, 1);
      • 두 DP 배열을 모두 1로 초기화한다. 어떤 위치에서든 자기 자신만으로 길이 1의 부분 수열을 이루기 때문이다.
    • 2) 비감소 수열 길이 갱신 
      if (prev <= number) {
          increase[i] = increase[i - 1] + 1;
      } else {
          increase[i] = 1;
      }
      • 현재 수(number)가 이전 수(prev) 이상이면 비감소 조건을 만족하므로 increase[i] = increase[i-1] + 1로 연장한다.
      • 조건을 만족하지 않으면 연속성이 끊기므로 increase[i] = 1로 리셋한다.
    • 3) 비증가 수열 길이 갱신 
      if (prev >= number) {
          decrease[i] = decrease[i - 1] + 1;
      } else {
          decrease[i] = 1;
      }
      • 현재 수가 이전 수 이하이면 비증가 조건을 만족하므로 decrease[i] = decrease[i-1] + 1로 연장한다.
      • 조건을 만족하지 않으면 decrease[i] = 1로 리셋한다.
      • prev == number인 경우 prev <= number와 prev >= number를 동시에 만족하므로 increase[i]와 decrease[i] 모두 증가한다.
    • 4) 최댓값 갱신 및 출력 
      prev = number;
      answer = Math.max(answer, Math.max(increase[i], decrease[i]));
      • 매 단계마다 increase[i]와 decrease[i] 중 큰 값을 answer와 비교해 최댓값을 유지한다.
      • prev를 현재 수로 갱신해 다음 단계 비교에 사용한다.
      • 예시: 수열 = [1, 2, 2, 1]이면 increase = [1, 2, 3, 1], decrease = [1, 1, 2, 3]. 최댓값은 3.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 비감소·비증가 길이를 저장하는 increase와 decrease 두 배열이 각각 N 크기를 사용한다. 이전 값만 참조하므로 변수 두 개로 최적화할 수 있지만, 가독성을 위해 배열로 구현했다.
