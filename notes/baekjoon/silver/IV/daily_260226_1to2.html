    문제 2331 — 반복수열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    반복수열 (백준 2331)는
    수열 A, D(A), D(D(A)), ...를 계속 만들어 나갈 때, 반복되는 부분에 포함되지 않는 앞부분의 원소 개수를 구하는 문제다.
    D(n)은 n의 각 자리 수를 P제곱한 값의 합으로 정의된다.

    • 수열은 언젠가 반드시 이전에 등장한 수를 다시 만나게 되며, 그 순간 수열이 순환(사이클)에 진입한다.
    • 반복에 포함되지 않는 원소의 수란, 수열에서 처음으로 재등장하는 수의 인덱스(0-based)를 의미한다.
    • A는 최대 9999, P는 최대 5이므로 D(n)의 값이 빠르게 수렴해 사이클이 짧게 형성된다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 방문 추적 + 재귀 시뮬레이션
    • 선택 이유: 수열을 순서대로 생성하면서 각 수의 등장 인덱스를 HashMap에 저장한다. 새로 생성된 수가 HashMap에 이미 존재하면 사이클이 시작되는 지점을 찾은 것이므로, 그 인덱스를 즉시 반환한다. HashMap의 O(1) 조회 덕분에 사이클 감지가 효율적이다.
    • 세부 전략:
      • 시작값 A를 인덱스 0으로 HashMap에 등록한다.
      • 재귀적으로 D(number)를 계산하고, 결과값이 HashMap에 존재하면 그 인덱스를 반환한다.
      • 존재하지 않으면 현재 인덱스로 등록한 뒤 다음 단계를 재귀 호출한다.
      • 반환된 인덱스가 곧 반복에 포함되지 않는 원소의 수다.

    코드가 하는 일

    • 1) 초기화 및 시작값 등록 
      int A = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      map.put(A, 0);
      int answer = recursion(A, P, 1);
      • 시작값 A와 제곱수 P를 입력받는다.
      • A를 인덱스 0으로 HashMap에 등록한다. 수열의 첫 번째 원소가 이미 존재한다고 표시해 이후 사이클 감지의 기준점으로 삼는다.
      • recursion(A, P, 1)을 호출해 A에서 시작하는 다음 수(인덱스 1)부터 탐색을 시작한다.
    • 2) D(n) 계산 — 각 자리 수의 P제곱 합 
      int result = 0;
      
      while (number > 0) {
          result += (int) Math.pow(number % 10, P);
          number /= 10;
      }
      • number % 10으로 1의 자리 수를 추출해 P제곱한 뒤 result에 누적한다.
      • number /= 10으로 다음 자리로 이동하며, number가 0이 될 때까지 반복한다.
      • 예시: number = 12, P = 2이면 2² + 1² = 5. result = 5.
    • 3) 사이클 감지 및 재귀 진행 
      if (map.containsKey(result)) {
          return map.get(result);
      }
      
      map.put(result, idx);
      return recursion(result, P, idx + 1);
      • 계산된 result가 HashMap에 이미 존재하면 사이클이 시작되는 지점을 찾은 것이다. 해당 수의 처음 등장 인덱스를 반환한다. 이 값이 곧 반복에 포함되지 않는 원소의 수다.
      • 존재하지 않으면 result를 현재 인덱스 idx로 HashMap에 등록하고, result를 다음 수로 삼아 재귀 호출한다.
      • 예시: 수열이 [A, b, c, d, b, ...]처럼 b에서 사이클이 시작되면, b의 인덱스인 1이 반환된다. 반복에 포함되지 않는 원소는 A 하나뿐이므로 정답은 1.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K × log A): K는 사이클이 형성되기까지 생성되는 수열의 길이, log A는 각 수에서 D(n)을 계산할 때 자릿수만큼 반복하는 비용이다. A ≤ 9999이므로 자릿수는 최대 4이며, D(n)의 값도 빠르게 수렴해 K가 매우 작다.
    • 공간 복잡도
      • O(K): 사이클이 형성되기까지 등장한 수와 인덱스를 저장하는 HashMap의 크기가 K에 비례한다. 재귀 호출 스택도 K 깊이만큼 사용된다.

    문제 1969 — DNA

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    DNA (백준 1969)는
    N개의 DNA 문자열이 주어질 때, 각 위치에서 해밍 거리의 합이 최소가 되는 DNA 문자열(Hamming Distance 최솟값)과 그 거리를 구하는 문제다.
    DNA 문자열은 A, C, G, T로만 구성되며 길이는 모두 M으로 동일하다.

    • 해밍 거리는 두 문자열의 같은 위치에서 다른 문자의 수를 의미한다.
    • 각 위치마다 독립적으로 가장 많이 등장한 문자를 선택하면 해당 위치의 해밍 거리 합이 최소가 된다.
    • 같은 빈도의 문자가 여러 개일 경우 알파벳 순서로 가장 앞선 문자를 선택한다.

    전략 개요

    • 핵심 알고리즘: 열(Column)별 빈도 카운팅 + 그리디
    • 선택 이유: 각 위치는 서로 독립적이므로, 위치마다 가장 많이 등장한 문자를 선택하는 그리디 방식이 최적이다. 위치별로 A~Z 26개 문자의 빈도를 세어 최댓값을 찾으면 O(N × M)에 해결된다.
    • 세부 전략:
      • M개의 각 위치에 대해 N개의 DNA 문자열에서 해당 위치의 문자 빈도를 count 배열에 센다.
      • count 배열을 앞에서부터 순회해 최대 빈도를 찾는다. 알파벳 순서대로 순회하므로 동률이면 자동으로 앞선 문자가 선택된다.
      • 해당 위치의 해밍 거리 기여분은 N - maxCnt이며, 이를 누적해 전체 최소 해밍 거리를 구한다.

    코드가 하는 일

    • 1) DNA 문자열 입력 및 2차원 배열 저장 
      char[][] chars = new char[N][M];
      
      for (int i = 0; i < N; i++) {
          String str = br.readLine();
          for (int j = 0; j < M; j++) {
              chars[i][j] = str.charAt(j);
          }
      }
      • N개의 DNA 문자열을 2차원 char 배열에 저장한다. chars[i][j]는 i번째 DNA 문자열의 j번째 문자를 의미한다.
      • 이후 열(Column) 단위로 접근하기 위해 2차원 배열로 저장하는 것이 편리하다.
    • 2) 위치별 문자 빈도 카운팅 
      for (int i = 0; i < M; i++) {
          int[] count = new int[26];
      
          for (int j = 0; j < N; j++) {
              count[chars[j][i] - 'A']++;
          }
      }
      • 외부 반복문의 i는 현재 처리 중인 위치(열)를 나타낸다.
      • 해당 위치에서 N개의 DNA 문자열의 문자를 모두 읽어 count 배열에 빈도를 기록한다. chars[j][i] - 'A'로 문자를 0~25 인덱스로 변환한다.
    • 3) 최빈 문자 선택 및 해밍 거리 누적 
      int maxCnt = -1;
      int idx = 0;
      
      for (int j = 0; j < count.length; j++) {
          if (maxCnt < count[j]) {
              maxCnt = count[j];
              idx = j;
          }
      }
      
      DNA[i] = (char) (idx + 'A');
      total += (N - maxCnt);
      • count 배열을 A(0)부터 Z(25) 순서로 순회하며 최대 빈도 maxCnt와 해당 인덱스 idx를 찾는다.
      • maxCnt < count[j]로 엄격한 부등호를 사용하므로, 동률인 경우 먼저 발견된(알파벳 순서로 앞선) 문자가 유지된다. 이것이 동률 시 사전순 최솟값 조건을 자동으로 만족하는 핵심이다.
      • DNA[i] = (char)(idx + 'A')로 인덱스를 다시 문자로 변환해 결과 DNA 배열에 저장한다.
      • 해당 위치의 해밍 거리 기여분은 선택된 문자가 아닌 나머지 문자의 수, 즉 N - maxCnt이다. 이를 total에 누적한다.
    • 4) 결과 출력 
      sb.append(DNA).append('\n').append(total);
      System.out.println(sb);
      • 완성된 DNA 문자 배열과 최소 해밍 거리 합 total을 줄 바꿈으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N × M): M개의 위치 각각에 대해 N개의 문자를 순회하며 빈도를 세고, 26개의 알파벳 중 최댓값을 찾는 작업은 O(26) = O(1)이다. 전체 시간 복잡도는 O(N × M)이다.
    • 공간 복잡도
      • O(N × M): N개의 DNA 문자열을 저장하는 2차원 배열이 주요 메모리 사용처이며, 그 외 count 배열(크기 26)과 결과 배열 DNA(크기 M)는 상수 혹은 O(M) 크기를 사용한다.
