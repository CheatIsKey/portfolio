    문제 2331 — 반복수열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    반복수열 (백준 2331)는
    수열 A, D(A), D(D(A)), ...를 계속 만들어 나갈 때, 반복되는 부분에 포함되지 않는 앞부분의 원소 개수를 구하는 문제다.
    D(n)은 n의 각 자리 수를 P제곱한 값의 합으로 정의된다.

    • 수열은 언젠가 반드시 이전에 등장한 수를 다시 만나게 되며, 그 순간 수열이 순환(사이클)에 진입한다.
    • 반복에 포함되지 않는 원소의 수란, 수열에서 처음으로 재등장하는 수의 인덱스(0-based)를 의미한다.
    • A는 최대 9999, P는 최대 5이므로 D(n)의 값이 빠르게 수렴해 사이클이 짧게 형성된다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 방문 추적 + 재귀 시뮬레이션
    • 선택 이유: 수열을 순서대로 생성하면서 각 수의 등장 인덱스를 HashMap에 저장한다. 새로 생성된 수가 HashMap에 이미 존재하면 사이클이 시작되는 지점을 찾은 것이므로, 그 인덱스를 즉시 반환한다. HashMap의 O(1) 조회 덕분에 사이클 감지가 효율적이다.
    • 세부 전략:
      • 시작값 A를 인덱스 0으로 HashMap에 등록한다.
      • 재귀적으로 D(number)를 계산하고, 결과값이 HashMap에 존재하면 그 인덱스를 반환한다.
      • 존재하지 않으면 현재 인덱스로 등록한 뒤 다음 단계를 재귀 호출한다.
      • 반환된 인덱스가 곧 반복에 포함되지 않는 원소의 수다.

    코드가 하는 일

    • 1) 초기화 및 시작값 등록 
      int A = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      map.put(A, 0);
      int answer = recursion(A, P, 1);
      • 시작값 A와 제곱수 P를 입력받는다.
      • A를 인덱스 0으로 HashMap에 등록한다. 수열의 첫 번째 원소가 이미 존재한다고 표시해 이후 사이클 감지의 기준점으로 삼는다.
      • recursion(A, P, 1)을 호출해 A에서 시작하는 다음 수(인덱스 1)부터 탐색을 시작한다.
    • 2) D(n) 계산 — 각 자리 수의 P제곱 합 
      int result = 0;
      
      while (number > 0) {
          result += (int) Math.pow(number % 10, P);
          number /= 10;
      }
      • number % 10으로 1의 자리 수를 추출해 P제곱한 뒤 result에 누적한다.
      • number /= 10으로 다음 자리로 이동하며, number가 0이 될 때까지 반복한다.
      • 예시: number = 12, P = 2이면 2² + 1² = 5. result = 5.
    • 3) 사이클 감지 및 재귀 진행 
      if (map.containsKey(result)) {
          return map.get(result);
      }
      
      map.put(result, idx);
      return recursion(result, P, idx + 1);
      • 계산된 result가 HashMap에 이미 존재하면 사이클이 시작되는 지점을 찾은 것이다. 해당 수의 처음 등장 인덱스를 반환한다. 이 값이 곧 반복에 포함되지 않는 원소의 수다.
      • 존재하지 않으면 result를 현재 인덱스 idx로 HashMap에 등록하고, result를 다음 수로 삼아 재귀 호출한다.
      • 예시: 수열이 [A, b, c, d, b, ...]처럼 b에서 사이클이 시작되면, b의 인덱스인 1이 반환된다. 반복에 포함되지 않는 원소는 A 하나뿐이므로 정답은 1.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K × log A): K는 사이클이 형성되기까지 생성되는 수열의 길이, log A는 각 수에서 D(n)을 계산할 때 자릿수만큼 반복하는 비용이다. A ≤ 9999이므로 자릿수는 최대 4이며, D(n)의 값도 빠르게 수렴해 K가 매우 작다.
    • 공간 복잡도
      • O(K): 사이클이 형성되기까지 등장한 수와 인덱스를 저장하는 HashMap의 크기가 K에 비례한다. 재귀 호출 스택도 K 깊이만큼 사용된다.

    문제 1969 — DNA

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    암기왕 (백준 2776)는
    수첩1에 적힌 수들을 기억하고, 수첩2에 적힌 각 수가 수첩1에 존재하면 1, 존재하지 않으면 0을 출력하는 문제다.
    N과 M이 최대 1,000,000이므로 선형 탐색 대신 이진 탐색을 사용해야 한다.

    • 테스트 케이스 T가 주어지며, 각 케이스마다 수첩1의 N개의 수와 수첩2의 M개의 수를 처리한다.
    • 수첩1의 수는 중복이 있을 수 있지만, 존재 여부만 확인하면 되므로 중복 제거 없이 정렬 후 이진 탐색으로 충분하다.
    • N, M이 최대 100만이므로 O(N log N) 정렬 + O(M log N) 탐색으로 처리해야 시간 초과를 피할 수 있다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 이진 탐색 (Binary Search)
    • 선택 이유: 수첩1을 정렬해두면 수첩2의 각 수에 대해 O(log N)에 존재 여부를 확인할 수 있다. N, M이 최대 100만이므로 선형 탐색 O(N × M)은 시간 초과가 발생하지만, 이진 탐색은 O(M log N)으로 충분히 빠르다.
    • 세부 전략:
      • 수첩1의 수를 배열에 저장하고 오름차순 정렬한다.
      • 수첩2의 각 수에 대해 직접 구현한 이진 탐색을 수행해 존재하면 1, 없으면 0을 반환한다.
      • 중간값 계산 시 start + (end - start) / 2를 사용해 정수 오버플로우를 방지한다.

    코드가 하는 일

    • 1) 수첩1 입력 및 정렬 
      int N = Integer.parseInt(br.readLine());
      int[] numbers = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(numbers);
      • 수첩1의 N개의 수를 배열에 저장한 뒤 Arrays.sort()로 오름차순 정렬한다.
      • 정렬된 배열이어야 이진 탐색을 적용할 수 있다.
    • 2) 수첩2의 각 수에 대해 이진 탐색 수행 
      int M = Integer.parseInt(br.readLine());
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < M; i++) {
          int number = Integer.parseInt(st.nextToken());
          sb.append(binarySearch(numbers, number, 0, N - 1)).append('\n');
      }
      • 수첩2의 M개의 수를 순서대로 읽으며, 각 수에 대해 binarySearch()를 호출한다.
      • 반환값(0 또는 1)을 StringBuilder에 누적해 마지막에 한 번에 출력함으로써 I/O 비용을 줄인다.
    • 3) 이진 탐색 구현 
      private static int binarySearch(int[] numbers, int target, int start, int end) {
          while (start <= end) {
              int mid = start + (end - start) / 2;
      
              if (numbers[mid] == target) {
                  return 1;
              } else if (numbers[mid] < target) {
                  start = mid + 1;
              } else {
                  end = mid - 1;
              }
          }
      
          return 0;
      }
      • start <= end인 동안 반복하며, 중간 인덱스 mid = start + (end - start) / 2를 계산한다. (start + end) / 2 대신 이 방식을 사용하면 두 값의 합이 int 범위를 초과하는 오버플로우를 방지할 수 있다.
      • numbers[mid] == target이면 존재하므로 즉시 1을 반환한다.
      • numbers[mid] < target이면 탐색 범위를 오른쪽(start = mid + 1)으로 좁히고, 크면 왼쪽(end = mid - 1)으로 좁힌다.
      • 반복이 종료될 때까지 찾지 못하면 0을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N + M log N): 수첩1 정렬에 O(N log N), 수첩2의 M개의 수에 대한 이진 탐색에 O(M log N)이 소요된다. N, M이 최대 100만이므로 약 2,000만 번의 연산으로 충분히 처리된다.
    • 공간 복잡도
      • O(N): 수첩1의 N개의 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
