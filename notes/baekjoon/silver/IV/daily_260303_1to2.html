    문제 1822 — 차집합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    차집합 (백준 1822)는
    두 집합 A, B가 주어질 때 A에는 속하지만 B에는 속하지 않는 원소들의 집합(A - B)을 오름차순으로 출력하는 문제다.
    각 집합에 중복 원소는 없으며, 원소의 값은 최대 2,000,000,000이므로 long 타입을 사용해야 한다.

    • N과 M은 최대 500,000이므로 집합 연산이 효율적인 자료구조를 사용해야 한다.
    • 결과는 오름차순으로 출력해야 하므로, 정렬 순서를 유지하는 TreeSet이 적합하다.
    • 차집합의 원소가 없는 경우 0만 출력하고 빈 줄은 출력하지 않아도 된다.

    전략 개요

    • 핵심 알고리즘: TreeSet을 이용한 집합 차 연산
    • 선택 이유: TreeSet은 내부적으로 Red-Black Tree로 구현되어 삽입·삭제·조회가 O(log N)이며, 원소를 항상 오름차순으로 유지한다. Java의 removeAll() 메서드로 차집합 연산을 한 줄에 처리할 수 있어 구현이 간결하다.
    • 세부 전략:
      • A와 B를 각각 TreeSet에 저장해 중복을 자동으로 제거하고 오름차순 정렬 상태를 유지한다.
      • A.removeAll(B)로 A에서 B에 속한 원소를 모두 제거해 차집합을 구한다.
      • 남은 A의 크기를 먼저 출력하고, TreeSet의 순회 순서가 오름차순이므로 그대로 출력하면 된다.

    코드가 하는 일

    • 1) 두 집합 입력 및 TreeSet 구성 
      Set A = new TreeSet<>();
      Set B = new TreeSet<>();
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          A.add(Long.parseLong(st.nextToken()));
      }
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < M; i++) {
          B.add(Long.parseLong(st.nextToken()));
      }
      • 원소 값이 최대 20억이므로 Long 타입으로 읽어 저장한다.
      • TreeSet은 삽입 시 자동으로 오름차순 정렬을 유지하고 중복을 허용하지 않으므로, 별도의 정렬·중복 제거 과정 없이 집합 조건을 만족한다.
    • 2) 차집합 연산 
      A.removeAll(B);
      • Java Collection의 removeAll() 메서드는 A에서 B에 포함된 원소를 모두 제거한다. 이것이 곧 차집합 A - B 연산이다.
      • 내부적으로 B의 각 원소에 대해 A에서 조회·삭제를 수행하므로 O(M log N)의 시간이 소요된다.
    • 3) 결과 출력 
      sb.append(A.size()).append("\n");
      
      for (Long number : A) {
          sb.append(number).append(" ");
      }
      
      System.out.println(sb);
      • 차집합의 원소 수 A.size()를 먼저 출력한다.
      • TreeSet은 오름차순 순서를 보장하므로, 향상된 for문으로 순회하면 별도 정렬 없이 오름차순으로 출력된다.
      • 결과를 StringBuilder에 누적해 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O((N + M) log N): A에 N개의 원소를 삽입하는 데 O(N log N), B에 M개의 원소를 삽입하는 데 O(M log M), removeAll()로 차집합을 구하는 데 O(M log N)이 소요된다. 전체 시간 복잡도는 O((N + M) log N)이다.
    • 공간 복잡도
      • O(N + M): 두 TreeSet이 각각 최대 N개, M개의 원소를 저장하므로 전체 O(N + M)의 공간을 사용한다.

    문제 2670 — 연속부분최대곱

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    연속부분최대곱 (백준 2670)는
    N개의 실수로 이루어진 수열에서 하나 이상의 연속된 수의 곱이 최대가 되는 값을 소수점 셋째 자리까지 출력하는 문제다.
    입력값이 0 이상 1 이하의 실수일 수 있으므로 곱이 커질수록 유리하지 않은 경우도 존재한다.

    • 연속 부분 수열이므로 반드시 인접한 수들의 곱만 고려한다.
    • 입력값이 0 이상 1 이하의 실수를 포함하므로, 이전 곱에 현재 수를 곱하면 오히려 값이 줄어드는 경우가 있다. 이 경우 현재 수 단독으로 새 부분 수열을 시작하는 것이 유리하다.
    • N이 최대 10,000이므로 O(N) DP로 해결해야 한다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (DP) — 카데인(Kadane) 알고리즘의 곱 버전
    • 선택 이유: 각 위치에서의 최대 연속 곱은 이전 위치의 최대 연속 곱에만 의존한다. 이전 곱을 연장하는 것과 현재 수 단독으로 시작하는 것 중 큰 값을 선택하면 O(N)에 해결된다.
    • 세부 전략:
      • dp[i]는 i번째 수에서 끝나는 연속 부분 수열의 최대 곱을 의미한다.
      • 점화식: dp[i] = Math.max(dp[i-1] * input[i], input[i]). 이전 최대 곱을 연장하는 것보다 현재 수 단독이 더 크면 새로운 부분 수열을 시작한다.
      • 매 단계마다 dp[i]와 전역 최댓값 max를 비교해 갱신한다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      double[] input = new double[N];
      for (int i = 0; i < N; i++) {
          input[i] = Double.parseDouble(br.readLine());
      }
      
      double[] dp = new double[N];
      dp[0] = input[0];
      double max = dp[0];
      • 입력값이 실수이므로 double 배열에 저장한다.
      • dp[0]을 첫 번째 수로 초기화한다. 부분 수열은 최소 1개의 원소를 포함해야 하므로, 첫 번째 수 단독이 초기 최댓값이 된다.
      • max도 dp[0]으로 초기화해 이후 비교의 기준점으로 삼는다.
    • 2) DP 점화식 적용 및 최댓값 갱신 
      for (int i = 1; i < N; i++) {
          dp[i] = Math.max(dp[i - 1] * input[i], input[i]);
          max = Math.max(max, dp[i]);
      }
      • dp[i-1] * input[i]는 이전까지의 최대 연속 곱에 현재 수를 곱해 부분 수열을 연장한 값이다.
      • input[i]는 현재 수 단독으로 새 부분 수열을 시작한 값이다.
      • 둘 중 큰 값을 dp[i]에 저장한다. 입력값이 0 이상 1 이하인 경우 곱이 줄어들 수 있으므로, 이 경우 현재 수 단독으로 시작하는 것이 유리하다.
      • 예시: input = [0.5, 0.9, 0.3]이면 dp = [0.5, 0.45, 0.3]. 최댓값은 0.500.
      • 예시: input = [0.9, 1.5, 2.0]이면 dp = [0.9, 1.35, 2.7]. 최댓값은 2.700.
    • 3) 결과 출력 
      System.out.printf("%.3f", max);
      • printf의 %.3f 포맷으로 소수점 셋째 자리까지 반올림해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 입력값을 저장하는 input 배열과 DP 결과를 저장하는 dp 배열이 각각 N 크기를 사용한다. dp[i]는 dp[i-1]만 참조하므로 변수 하나로 최적화해 O(1)로 줄일 수 있지만, 가독성을 위해 배열로 구현했다.
