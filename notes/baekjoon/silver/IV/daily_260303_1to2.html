    문제 5568 — 카드 놓기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    카드 놓기 (백준 5568)는
    n장의 카드 중 k장을 골라 일렬로 나열할 때, 만들 수 있는 서로 다른 수의 개수를 구하는 문제다.
    카드를 이어 붙인 결과가 같더라도 서로 다른 카드 조합이면 같은 수로 취급한다.

    • 카드의 순서가 다르면 다른 수이므로, 순열(Permutation)을 구해야 한다.
    • 카드에 적힌 수가 예를 들어 1과 12처럼 이어 붙이면 동일한 결과("112")를 만들 수 있으므로, 단순 수치 계산이 아닌 문자열 이어붙이기로 처리해야 한다.
    • 중복된 결과는 HashSet으로 자동 제거하고, 최종적으로 HashSet의 크기가 답이 된다.

    전략 개요

    • 핵심 알고리즘: DFS 기반 순열 생성 + HashSet 중복 제거
    • 선택 이유: n장 중 k장을 순서 있게 선택하는 모든 경우의 수를 탐색해야 하므로 DFS 백트래킹이 적합하다. n이 최대 10, k가 최대 4이므로 최대 10 × 9 × 8 × 7 = 5,040가지 경우밖에 없어 완전 탐색으로 충분하다. 이어붙인 문자열이 같은 경우를 자동으로 걸러내기 위해 HashSet을 사용한다.
    • 세부 전략:
      • visited 배열로 이미 선택된 카드를 표시하고, 선택하지 않은 카드만 순서대로 추가한다.
      • 현재까지 선택한 카드 수(depth)가 k에 도달하면 이어붙인 문자열을 HashSet에 저장한다.
      • 재귀 호출 전후로 visited[i]를 true·false로 토글해 백트래킹을 구현한다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      n = Integer.parseInt(br.readLine());
      k = Integer.parseInt(br.readLine());
      
      visited = new boolean[n];
      card = new int[n];
      
      for (int i = 0; i < n; i++) {
          card[i] = Integer.parseInt(br.readLine());
      }
      • 카드의 총 수 n과 선택할 카드의 수 k를 입력받는다.
      • 각 카드의 수를 card 배열에 저장하고, 방문 여부를 추적할 visited 배열을 false로 초기화한다.
    • 2) DFS로 순열 생성 
      private static void dfs(int depth, String number) {
          if (depth == k) {
              set.add(number);
              return;
          }
      
          for (int i = 0; i < n; i++) {
              if (!visited[i]) {
                  visited[i] = true;
                  dfs(depth + 1, number + card[i]);
                  visited[i] = false;
              }
          }
      }
      • depth가 k에 도달하면 현재까지 이어붙인 문자열 number를 HashSet에 추가하고 재귀를 종료한다.
      • 방문하지 않은 카드에 대해 visited[i] = true로 표시한 뒤, number + card[i]로 현재 카드를 문자열에 이어붙여 재귀 호출한다.
      • 재귀 호출이 끝나면 visited[i] = false로 복원(백트래킹)해 다른 카드 조합을 탐색할 수 있도록 한다.
      • 문자열 이어붙이기를 사용하는 이유: 카드 1과 12를 이어붙이면 "112"가 되고, 카드 11과 2를 이어붙여도 "112"가 된다. 정수 덧셈이 아닌 문자열 연결로 처리해야 이 중복을 HashSet에서 올바르게 감지할 수 있다.
    • 3) 결과 출력 
      System.out.println(set.size());
      • HashSet에 저장된 서로 다른 문자열의 수가 곧 만들 수 있는 서로 다른 수의 개수이므로, set.size()를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n! / (n-k)!): n장 중 k장을 순서 있게 선택하는 순열의 수만큼 DFS가 실행된다. n ≤ 10, k ≤ 4이므로 최대 5,040번의 탐색으로 매우 빠르게 동작한다.
    • 공간 복잡도
      • O(n! / (n-k)!): HashSet에 저장되는 문자열의 수가 최대 순열의 수에 비례한다. 재귀 호출 스택은 최대 k 깊이이므로 O(k) = O(1)에 수렴한다.

    문제 1388 — 바닥 장식

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    바닥 장식 (백준 1388)는
    N × M 크기의 바닥에 가로('-') 또는 세로('|') 나무판자가 깔려 있을 때,
    사용된 나무판자의 총 개수를 구하는 문제다.
    같은 방향으로 연속된 칸은 하나의 나무판자로 이어진 것이다.

    • '-'는 가로 방향 판자이며, 같은 행에서 연속으로 이어진 '-'는 하나의 판자다.
    • '|'는 세로 방향 판자이며, 같은 열에서 연속으로 이어진 '|'는 하나의 판자다.
    • 판자의 끝을 감지하는 시점에 카운트를 증가시키면 O(N × M)에 해결할 수 있다.

    전략 개요

    • 핵심 알고리즘: 판자의 끝 위치 카운팅 (선형 탐색)
    • 선택 이유: 연속된 칸을 하나의 판자로 묶어야 하므로, 판자가 끝나는 시점(연속이 끊기거나 경계에 도달하는 순간)에만 카운트를 1 증가시키면 된다. 전체 배열을 한 번 순회하는 O(N × M)으로 해결된다.
    • 세부 전략:
      • 현재 칸이 '-'이면 가로 판자다. 오른쪽 칸이 없거나(마지막 열) 오른쪽이 '|'이면 가로 판자가 끝나는 지점이므로 카운트를 증가시킨다.
      • 현재 칸이 '|'이면 세로 판자다. 아래쪽 칸이 없거나(마지막 행) 아래쪽이 '-'이면 세로 판자가 끝나는 지점이므로 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 바닥 입력 및 2차원 배열 저장 
      char[][] floor = new char[N][M];
      
      for (int r = 0; r < N; r++) {
          floor[r] = br.readLine().toCharArray();
      }
      • 각 행을 문자열로 읽어 toCharArray()로 변환해 2차원 char 배열에 저장한다.
    • 2) 가로 판자('-') 끝 감지 
      if (floor[r][c] == '-') {
          if (c == M - 1 || floor[r][c + 1] == '|') {
              cnt++;
          }
      }
      • 현재 칸이 '-'일 때, 오른쪽 다음 칸을 확인한다.
      • c == M - 1이면 현재 칸이 마지막 열이므로 더 이상 이어질 수 없어 판자가 끝난다.
      • floor[r][c + 1] == '|'이면 오른쪽이 다른 방향의 판자이므로 현재 가로 판자가 여기서 끝난다.
      • 두 조건 모두 가로 판자의 마지막 칸임을 의미하므로 cnt를 1 증가시킨다.
    • 3) 세로 판자('|') 끝 감지 
      } else {
          if (r == N - 1 || floor[r + 1][c] == '-') {
              cnt++;
          }
      }
      • 현재 칸이 '|'일 때, 아래쪽 다음 칸을 확인한다.
      • r == N - 1이면 현재 칸이 마지막 행이므로 판자가 끝난다.
      • floor[r + 1][c] == '-'이면 아래쪽이 다른 방향의 판자이므로 현재 세로 판자가 여기서 끝난다.
      • 예시: 바닥이 ["--", "||"]이면 가로 판자 1개('-' 2칸), 세로 판자 2개('|' 각 1칸) → cnt = 3.
    • 4) 결과 출력 
      System.out.println(cnt);
      • 감지된 판자의 총 개수 cnt를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N × M): 전체 배열을 한 번 순회하며 각 칸에서 O(1) 연산만 수행하므로 전체 O(N × M)이다.
    • 공간 복잡도
      • O(N × M): 바닥 정보를 저장하는 2차원 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
