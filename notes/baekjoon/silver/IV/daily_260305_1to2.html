    문제 1059 — 좋은 구간

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    좋은 구간 (백준 1059)는
    집합 S와 정수 n이 주어질 때, n을 포함하면서 양 끝점이 S에 속하지 않는 구간 [i, j]의 개수를 구하는 문제다.
    i < j이어야 하며, i와 j 모두 S의 원소가 아니어야 한다.

    • n 자체가 S에 속하면 n을 포함하는 좋은 구간이 존재하지 않으므로 0을 출력한다.
    • n이 S에 속하지 않으면, n 바로 왼쪽 S의 원소(start)와 바로 오른쪽 S의 원소(end)를 찾는다.
    • 유효한 i의 범위는 (start, n], 유효한 j의 범위는 [n, end)이며, i = j = n인 경우(길이 0의 구간)를 제외한 모든 쌍이 정답이다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 범위 탐색 + 경우의 수 계산
    • 선택 이유: n을 기준으로 S에서 바로 인접한 왼쪽·오른쪽 값만 찾으면, 유효한 i와 j의 범위가 결정된다. 이후 이중 반복문으로 유효한 쌍의 수를 세면 O((n - start) × (end - n))에 해결된다. S의 크기가 최대 50으로 작으므로 선형 탐색으로 충분하다.
    • 세부 전략:
      • S를 오름차순 정렬한 뒤, n보다 작은 S의 원소 중 가장 큰 값을 start, n보다 큰 S의 원소 중 가장 작은 값을 end로 설정한다.
      • n이 S의 최솟값보다 작으면 start = 0으로 초기화한다.
      • 유효한 i: start + 1 이상 n 이하, 유효한 j: n 이상 end - 1 이하. 단 i = j인 경우(= n)는 구간이 아니므로 제외한다.

    코드가 하는 일

    • 1) 집합 입력 및 정렬 
      int[] set = new int[L];
      for (int i = 0; i < L; i++) {
          set[i] = Integer.parseInt(st.nextToken());
      }
      Arrays.sort(set);
      • L개의 집합 원소를 배열에 저장하고 오름차순 정렬한다. 이후 n을 기준으로 인접한 원소를 순서대로 찾기 위함이다.
    • 2) n이 S에 속하는 경우 즉시 종료 
      for (int i = 0; i < set.length; i++) {
          if (set[i] == n) {
              System.out.println(0);
              return;
          }
          ...
      }
      • n이 S의 원소이면 구간의 끝점이 될 수 없으므로 좋은 구간이 존재하지 않는다. 0을 출력하고 즉시 종료한다.
    • 3) start와 end 탐색 
      if (n < set[0]) {
          start = 0;
          end = set[0];
      } else {
          for (int i = 0; i < set.length; i++) {
              if (set[i] < n) {
                  start = set[i];
                  continue;
              }
              if (n < set[i]) {
                  end = set[i];
                  break;
              }
          }
      }
      • n이 S의 최솟값보다 작으면 왼쪽 경계가 없으므로 start = 0으로 설정하고, end = set[0]으로 설정한다.
      • 그 외의 경우 S를 순회하며 n보다 작은 원소 중 마지막으로 만난 값을 start에, n보다 큰 첫 번째 원소를 end에 저장한다.
      • 예시: S = [3, 7, 10], n = 5이면 start = 3, end = 7. 유효 i: 4~5, 유효 j: 5~6.
    • 4) 유효한 구간 쌍 카운팅 
      for (int i = start + 1; i <= n; i++) {
          for (int j = n; j < end; j++) {
              if (i == j) continue;
              cnt++;
          }
      }
      • i는 start + 1부터 n까지, j는 n부터 end - 1까지 순회한다. 이 범위가 S의 원소를 포함하지 않으면서 n을 구간 내에 포함하는 끝점의 유효 범위다.
      • i == j인 경우는 i = j = n으로, 길이가 0인 구간(점)이므로 건너뛴다.
      • 이 이중 반복문은 수식으로 표현하면 (n - start) × (end - n) - 1과 동일하다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L log L + (n - start) × (end - n)): 정렬에 O(L log L), start·end 탐색에 O(L), 카운팅 이중 반복문에 O((n - start) × (end - n))이 소요된다. n과 S 원소의 값이 최대 10,000이므로 카운팅 비용도 충분히 작다.
    • 공간 복잡도
      • O(L): 집합 원소를 저장하는 배열이 L 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 1758 — 알바생 강호

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    알바생 강호 (백준 1758)는
    N명의 손님이 각자 줄 의향이 있는 팁 금액을 가지고 있을 때, 강호가 받을 수 있는
    팁의 합을 최대화하는 문제다.
    k번째로 응대한 손님은 자신의 팁에서 (k - 1)을 뺀 금액을 지불하며, 0 이하가 되면 팁을 주지 않는다.

    • 손님을 응대하는 순서에 따라 팁이 달라지므로, 순서를 어떻게 정하느냐가 핵심이다.
    • 팁이 많은 손님을 먼저 응대할수록 차감되는 금액이 작아지므로, 팁 내림차순으로 응대하는 것이 최적이다.
    • 팁이 0 이하가 되는 손님은 실제로 팁을 주지 않으므로 누적에서 제외한다.

    전략 개요

    • 핵심 알고리즘: 그리디 (내림차순 정렬)
    • 선택 이유: k번째 손님의 실제 팁은 people[k] - (k - 1)이다. 총 팁을 최대화하려면 팁이 큰 손님일수록 순번(k)이 작아야 차감이 줄어든다. 따라서 팁 내림차순으로 정렬해 앞 순번부터 배치하는 그리디 전략이 최적임을 보장한다.
    • 세부 전략:
      • 배열을 오름차순 정렬한 뒤 뒤에서부터 순회해 내림차순 효과를 낸다.
      • 각 손님의 실제 팁 people[i] - (sequence - 1)을 계산하고, 0 이하이면 팁을 받지 못하므로 건너뛴다.
      • 0보다 큰 팁만 total_tip에 누적한다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] people = new int[N];
      for (int i = 0; i < N; i++) {
          people[i] = Integer.parseInt(br.readLine());
      }
      Arrays.sort(people);
      • N명의 팁 의향 금액을 배열에 저장하고 오름차순 정렬한다.
      • 이후 배열을 뒤에서부터 순회하면 내림차순으로 처리하는 것과 동일한 효과를 낸다.
    • 2) 내림차순 순회 및 실제 팁 계산 
      long total_tip = 0;
      int sequence = 1;
      
      for (int i = N - 1; i >= 0; i--) {
          int tip = people[i] - (sequence++ - 1);
          if (tip <= 0) {
              continue;
          }
          total_tip += tip;
      }
      • 배열을 뒤에서부터 순회해 팁이 큰 손님부터 순번 1, 2, 3, ...을 부여한다.
      • sequence는 현재 응대 순번을 나타내며, sequence++ - 1은 후위 증가 연산자를 이용해 현재 순번에서의 차감값 (k - 1)을 계산하고 다음 반복을 위해 sequence를 1 증가시킨다.
      • 실제 팁 tip = people[i] - (sequence - 1)이 0 이하이면 팁을 받지 못하므로 건너뛴다. 정렬된 배열을 내림차순으로 순회하므로, 한 번 tip이 0 이하가 되면 이후 모든 손님도 0 이하가 된다.
      • 예시: people = [5, 8, 10]을 내림차순으로 처리하면 10-0=10, 8-1=7, 5-2=3. total_tip = 20.
    • 3) 결과 출력 
      System.out.println(total_tip);
      • 누적된 최대 팁 합 total_tip을 출력한다. 팁의 최대 합이 int 범위를 초과할 수 있으므로 long 타입으로 선언했다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 선형 순회는 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N명의 팁 금액을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
