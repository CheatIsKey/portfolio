    문제 10994 — 별 찍기 - 19

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    별 찍기 - 19 (백준 10994)는
    N이 주어질 때 정사각형 테두리가 안쪽으로 2칸씩 줄어들며 중첩된 N겹의 별 패턴을 출력하는 문제다.
    전체 크기는 4 × (N - 1) + 1이며, 가장 바깥 테두리에서 시작해 안쪽으로 2칸씩 좁혀가며 별을 찍는다.

    • N = 1이면 크기 1 × 1의 별 하나만 출력된다.
    • N = 2이면 크기 5 × 5의 테두리와 그 안쪽 1 × 1 별이 출력되는 형태로, 각 겹은 이전 겹보다 상하좌우로 2칸씩 안쪽에 위치한다.
    • 재귀 호출마다 시작 좌표를 (row + 2, col + 2)로 이동하고 크기를 4 줄여 안쪽 정사각형을 그린다.

    전략 개요

    • 핵심 알고리즘: 재귀를 이용한 중첩 정사각형 테두리 그리기
    • 선택 이유: 각 겹의 정사각형 테두리는 이전 겹보다 2칸 안쪽에서 시작하고 크기가 4 줄어드는 동일한 구조를 반복하므로, 재귀 호출이 자연스럽다. 2차원 배열에 미리 공백을 채운 뒤 별을 덮어씌우는 방식으로 구현한다.
    • 세부 전략:
      • 전체 배열을 공백으로 초기화한 뒤, star(field, 0, 0, length)를 호출해 가장 바깥 테두리부터 그린다.
      • 현재 시작 좌표 (row, col)과 크기 length를 받아 네 변의 테두리에 별을 찍는다.
      • length > 1이면 star(field, row + 2, col + 2, length - 4)를 재귀 호출해 안쪽 테두리를 그린다.
      • length ≤ 0이면 더 이상 그릴 테두리가 없으므로 재귀를 종료한다.

    코드가 하는 일

    • 1) 배열 크기 계산 및 공백 초기화 
      int length = 4 * (N - 1) + 1;
      
      char[][] field = new char[length][length];
      
      for (int i = 0; i < length; i++) {
          Arrays.fill(field[i], ' ');
      }
      • 전체 정사각형의 한 변의 길이는 4 × (N - 1) + 1이다. N = 1이면 1, N = 2이면 5, N = 3이면 9로 4씩 증가한다.
      • 모든 칸을 공백으로 초기화한 뒤, 재귀 함수에서 별이 찍힐 위치만 덮어씌운다.
    • 2) 정사각형 테두리 그리기 
      int right = col + length - 1;
      int down = row + length - 1;
      
      for (int i = 0; i < length; i++) {
          field[row][col + i] = '*';
          field[down][col + i] = '*';
          field[row + i][col] = '*';
          field[row + i][right] = '*';
      }
      • right는 현재 정사각형의 오른쪽 끝 열, down은 아래쪽 끝 행이다.
      • 하나의 반복문으로 위쪽 변, 아래쪽 변, 왼쪽 변, 오른쪽 변 네 변을 동시에 그린다.
      • 네 꼭짓점은 네 변이 겹치는 위치이므로 자연스럽게 중복 처리되어 별이 찍힌다.
    • 3) 재귀 호출로 안쪽 테두리 그리기 
      if (length > 1) {
          star(field, row + 2, col + 2, length - 4);
      }
      • 현재 테두리 안쪽 2칸으로 이동한 위치 (row + 2, col + 2)에서 크기 length - 4의 정사각형을 재귀적으로 그린다.
      • length가 1이면 단일 별 하나이므로 재귀를 호출하지 않는다. length가 0 이하이면 함수 진입 시 즉시 반환한다.
      • 예시: N = 3이면 length = 9 → 5 → 1 순서로 재귀 호출되며 3겹의 정사각형이 그려진다.
    • 4) 결과 출력 
      for (char[] row : field) {
          sb.append(row).append('\n');
      }
      System.out.println(sb);
      • 완성된 2차원 배열의 각 행을 StringBuilder에 누적해 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 전체 배열의 크기가 (4N - 3)²이며, 초기화와 출력 모두 배열 크기에 비례한다. 재귀 호출은 N번이며 각 호출에서 현재 length만큼 별을 찍으므로 총 별 찍기 비용의 합도 O(N²)이다.
    • 공간 복잡도
      • O(N²): 결과를 저장하는 2차원 배열의 크기가 (4N - 3) × (4N - 3)이므로 O(N²)의 공간을 사용한다. 재귀 호출 스택은 최대 N 깊이로 O(N)이다.

    문제 11508 — 2+1 세일

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    2+1 세일 (백준 11508)는
    N개의 물건을 살 때 가격이 높은 순서대로 3개씩 묶어 그 중 가장 싼 것을 무료로 받는 2+1 세일을 적용해
    최소 비용을 구하는 문제다.
    3의 배수 묶음마다 가장 저렴한 1개가 무료가 되므로, 어떤 순서로 묶느냐가 핵심이다.

    • 3개씩 묶을 때 무료로 받는 것은 그 묶음에서 가장 싼 물건이다.
    • 총 비용을 최소화하려면 무료로 받는 물건의 가격 합이 최대가 되어야 한다.
    • 가격 내림차순으로 정렬한 뒤 3번째 물건(인덱스 2, 5, 8, ...)을 무료로 받으면 가장 비싼 물건들을 최대한 무료로 받을 수 있어 최적이다.

    전략 개요

    • 핵심 알고리즘: 그리디 (내림차순 정렬 후 3번째마다 제외)
    • 선택 이유: 무료로 받는 물건의 가격 합을 최대화해야 총 비용이 최소가 된다. 내림차순 정렬 후 3번째 물건(묶음 내 가장 싼 것)을 순서대로 무료로 받으면, 가능한 한 비싼 물건을 무료로 받을 수 있어 최적임이 보장된다.
    • 세부 전략:
      • 배열을 오름차순 정렬한 뒤 뒤에서부터 순회해 내림차순 효과를 낸다.
      • cnt를 0부터 세어 2가 되면(3번째 물건) 해당 물건을 무료로 받으므로 total에 더하지 않고 cnt를 0으로 리셋한다.
      • 그 외의 경우 total에 가격을 누적하고 cnt를 1 증가시킨다.

    코드가 하는 일

    • 1) N이 3 미만인 경우 예외 처리 
      if (prices.length < 3) {
          for (int price : prices) {
              total += price;
          }
          System.out.println(total);
          return;
      }
      • 물건이 3개 미만이면 2+1 세일을 적용할 수 없으므로 모든 물건의 가격을 그대로 합산해 출력하고 종료한다.
    • 2) 입력 및 정렬 
      int[] prices = new int[N];
      for (int i = 0; i < N; i++) {
          prices[i] = Integer.parseInt(br.readLine());
      }
      Arrays.sort(prices);
      • N개의 물건 가격을 배열에 저장하고 오름차순 정렬한다.
      • 이후 배열을 뒤에서부터 순회해 가격이 높은 순서대로 처리한다.
    • 3) 내림차순 순회하며 3번째 물건 무료 적용 
      int cnt = 0;
      
      for (int idx = N - 1; idx >= 0; idx--) {
          if (cnt == 2) {
              cnt = 0;
              continue;
          }
          total += prices[idx];
          cnt++;
      }
      • 배열 뒤에서부터 순회하므로 가격이 높은 물건부터 처리된다.
      • cnt가 2에 도달하면 현재 물건이 묶음 내 3번째(가장 싼 것)이므로 total에 더하지 않고 cnt를 0으로 리셋한다.
      • 그 외에는 total에 가격을 누적하고 cnt를 1 증가시킨다.
      • 예시: prices = [1, 2, 3, 4, 5, 6]을 내림차순으로 처리하면 6(cnt=1), 5(cnt=2), 4(무료, cnt=0), 3(cnt=1), 2(cnt=2), 1(무료, cnt=0). total = 6 + 5 + 3 + 2 = 16.
    • 4) 결과 출력 
      System.out.println(total);
      • 누적된 최소 비용 total을 출력한다. 가격 합이 int 범위를 초과할 수 있으므로 long 타입으로 선언했다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 선형 순회는 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N개의 가격을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
