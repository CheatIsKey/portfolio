    문제 19939 — 박 터뜨리기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    박 터뜨리기 (백준 19939)는
    N개의 사탕을 K개의 그룹에 나눠 담을 때, 각 그룹의 사탕 수가 모두 다르고 양의 정수이면서
    가장 많은 그룹과 가장 적은 그룹의 사탕 수 차이를 최소화하는 문제다.
    불가능한 경우 -1을 출력한다.

    • K개의 그룹에 서로 다른 양의 정수를 배분할 때 필요한 최솟값은 1 + 2 + 3 + ... + K = K × (K + 1) / 2이다.
    • N이 이 최솟값보다 작으면 K개의 서로 다른 양의 정수로 N을 만들 수 없으므로 -1을 출력한다.
    • 나머지 R = N - K × (K + 1) / 2를 K개 그룹에 균등하게 추가 배분할 수 있으면 최솟값과 최댓값의 차이는 K - 1이고, 균등하게 나눠지지 않으면 K가 된다.

    전략 개요

    • 핵심 알고리즘: 수학적 그리디
    • 선택 이유: 차이를 최소화하려면 기본 배분(1, 2, ..., K) 후 남은 사탕 R을 모든 그룹에 동일하게 추가하는 것이 최적이다. R을 각 그룹에 동일하게 더하면 그룹 간 차이는 기본 배분의 차이인 K - 1 그대로 유지된다. R % K ≠ 0이면 일부 그룹에 1씩 더 추가해야 하므로 차이가 K로 1 늘어난다.
    • 세부 전략:
      • N < K × (K + 1) / 2이면 즉시 -1을 출력한다.
      • R = N - K × (K + 1) / 2를 계산한다.
      • R % K == 0이면 모든 그룹에 R/K씩 균등 추가가 가능하므로 답은 K - 1이다.
      • R % K ≠ 0이면 일부 그룹에 1씩 더 추가해야 하므로 답은 K이다.

    코드가 하는 일

    • 1) 불가능 조건 검사 
      if (N < (K * (K + 1) / 2)) {
          System.out.println(-1);
          return;
      }
      • K개의 서로 다른 양의 정수를 만들기 위한 최소 합은 1 + 2 + ... + K = K × (K + 1) / 2이다.
      • N이 이 값보다 작으면 조건을 만족하는 배분이 불가능하므로 -1을 출력하고 종료한다.
    • 2) 나머지 R 계산 
      int R = N - ((K * (K + 1) / 2));
      • 기본 배분(1, 2, ..., K) 후 남은 사탕의 수 R을 계산한다.
      • R은 K개 그룹에 추가로 배분해야 하는 사탕의 수이며, 0 이상임이 보장된다.
    • 3) R의 균등 분배 여부에 따른 답 결정 
      if (R % K == 0) {
          System.out.println(K - 1);
          return;
      }
      
      System.out.println(K);
      • R % K == 0이면 R을 K개 그룹에 R/K씩 균등하게 추가할 수 있다. 기본 배분이 (1, 2, ..., K)이므로 R/K씩 더하면 (1 + R/K, 2 + R/K, ..., K + R/K)가 된다. 모든 그룹의 차이가 동일하게 유지되어 최솟값과 최댓값의 차이는 K - 1이다.
      • R % K ≠ 0이면 R을 균등하게 나눌 수 없어 일부 그룹에 1개씩 더 배분해야 한다. 이로 인해 그룹 간 차이가 1 늘어나 K가 된다.
      • 예시: N = 11, K = 3이면 기본 합 = 6, R = 5. R % K = 5 % 3 = 2 ≠ 0이므로 답은 3.
      • 예시: N = 12, K = 3이면 기본 합 = 6, R = 6. R % K = 6 % 3 = 0이므로 답은 2.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 입력을 읽은 뒤 상수 번의 산술 연산과 조건 분기만 수행하므로 입력 크기와 무관하게 O(1)에 동작한다.
    • 공간 복잡도
      • O(1): N, K, R을 저장하는 상수 개의 변수만 사용하므로 O(1)의 공간만 필요하다.

    문제 9507 — Generations of Tribbles

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Generations of Tribbles (백준 9507)는
    Tribble의 번식 규칙에 따라 n세대에 존재하는 Tribble의 수를 구하는 문제다.
    점화식 dp[n] = dp[n-1] + dp[n-2] + dp[n-3] + dp[n-4]를 따르는 4항 점화식 DP 문제다.

    • 초기값은 dp[0] = 1, dp[1] = 1, dp[2] = 2, dp[3] = 4로 고정되어 있다.
    • n은 최대 67이며, dp[67]은 long 범위 내에서 처리 가능하다.
    • 테스트 케이스마다 쿼리에 응답해야 하므로, 미리 dp 배열 전체를 채워두는 메모이제이션(사전 계산) 방식이 효율적이다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (4항 점화식 DP, 사전 계산)
    • 선택 이유: n이 최대 67로 고정되어 있으므로, 입력을 받기 전에 dp[0]~dp[67]을 모두 계산해 두면 각 테스트 케이스에 O(1)로 응답할 수 있다. 여러 테스트 케이스가 주어지는 구조에서 중복 계산을 방지하는 가장 효율적인 접근이다.
    • 세부 전략:
      • dp[0] = 1, dp[1] = 1, dp[2] = 2, dp[3] = 4로 초기값을 설정한다.
      • dp[4]부터 dp[67]까지 점화식 dp[i] = dp[i-1] + dp[i-2] + dp[i-3] + dp[i-4]로 순서대로 채운다.
      • 테스트 케이스마다 n을 입력받아 dp[n]을 즉시 출력한다.

    코드가 하는 일

    • 1) 초기값 설정 
      long[] dp = new long[68];
      
      dp[0] = 1;
      dp[1] = 1;
      dp[2] = 2;
      dp[3] = 4;
      • dp 배열의 크기를 68로 선언해 인덱스 0~67을 모두 사용한다.
      • 값이 지수적으로 증가하므로 long 타입으로 선언한다. dp[67]은 약 2.7 × 10¹⁸로 long 최댓값(약 9.2 × 10¹⁸) 이내다.
      • 문제에서 정의된 초기값 4개를 직접 설정한다.
    • 2) 사전 계산 — dp 배열 전체 채우기 
      for (int i = 4; i < 68; i++) {
          dp[i] = dp[i - 1] + dp[i - 2] + dp[i - 3] + dp[i - 4];
      }
      • i = 4부터 67까지 순서대로 점화식을 적용해 dp 배열을 채운다.
      • 각 항은 바로 이전 4개의 값의 합이므로, 배열을 앞에서부터 채우면 항상 필요한 이전 값들이 이미 계산된 상태임이 보장된다.
      • 예시: dp[4] = 4 + 2 + 1 + 1 = 8, dp[5] = 8 + 4 + 2 + 1 = 15, dp[6] = 15 + 8 + 4 + 2 = 29.
    • 3) 테스트 케이스별 쿼리 응답 
      int t = Integer.parseInt(br.readLine());
      
      for (int tc = 0; tc < t; tc++) {
          int n = Integer.parseInt(br.readLine());
          sb.append(dp[n]).append('\n');
      }
      
      System.out.println(sb);
      • 테스트 케이스 수 t를 읽은 뒤, 각 n에 대해 미리 계산된 dp[n]을 O(1)에 조회해 StringBuilder에 누적한다.
      • 모든 테스트 케이스를 처리한 뒤 한 번에 출력해 I/O 비용을 최소화한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(68 + T): dp 배열 사전 계산이 O(68) = O(1), T개의 쿼리 응답이 각 O(1)이므로 전체 O(T)에 수렴한다. n의 범위가 상수(0~67)로 고정되어 있으므로 사전 계산 비용은 입력 크기와 무관한 상수 시간이다.
    • 공간 복잡도
      • O(1): dp 배열의 크기가 68로 고정되어 있으므로 입력 크기와 무관한 상수 공간이다.
