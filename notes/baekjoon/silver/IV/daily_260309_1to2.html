    문제 2870 — 수학숙제

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수학숙제 (백준 2870)는
    N개의 문자열에서 숫자로만 이루어진 연속된 부분을 모두 추출해 오름차순으로 출력하는 문제다.
    숫자 앞에 0이 붙은 경우도 그대로 하나의 수로 취급하며, 수의 크기가 매우 클 수 있다.

    • 문자열에서 숫자가 아닌 문자를 기준으로 분리하면 숫자로만 이루어진 부분 문자열을 추출할 수 있다.
    • 추출된 수는 선행 0을 포함한 채로 비교해야 하며, 값 자체로 정렬해야 하므로 문자열 정렬이 아닌 수치 정렬이 필요하다.
    • 수의 자릿수가 매우 커질 수 있으므로 int나 long이 아닌 BigInteger를 사용해야 한다.

    전략 개요

    • 핵심 알고리즘: 정규식 기반 문자열 분리 + BigInteger 정렬
    • 선택 이유: 숫자가 아닌 모든 문자를 구분자로 사용해 문자열을 분리하면 숫자 부분 문자열을 한 번에 추출할 수 있다. 수의 크기가 long 범위를 초과할 수 있으므로 BigInteger로 변환해 저장하면 크기 비교와 선행 0 처리가 자동으로 해결된다.
    • 세부 전략:
      • 각 줄을 정규식 [^0-9]로 분리해 숫자 부분만 추출한다.
      • 빈 문자열은 건너뛰고, 나머지는 new BigInteger(num)으로 변환해 리스트에 추가한다. BigInteger 생성자가 선행 0을 자동으로 제거하므로 별도 처리가 필요 없다.
      • Collections.sort()로 수치 오름차순 정렬 후 출력한다.

    코드가 하는 일

    • 1) 정규식으로 숫자 부분 추출 
      String s = br.readLine();
      String[] nums = s.split("[^0-9]");
      • [^0-9]는 숫자가 아닌 모든 문자를 의미하는 정규식이다. 이를 구분자로 사용해 문자열을 분리하면 숫자로만 이루어진 부분 문자열 배열을 얻는다.
      • 예시: "abc12de34fg"를 분리하면 ["abc", "12", "de", "34", "fg"] → 숫자가 아닌 부분은 빈 문자열 또는 문자열이 되고, "12"와 "34"만 유효한 숫자 부분이다.
    • 2) 유효한 숫자 부분만 BigInteger로 변환해 리스트에 추가 
      for (String num : nums) {
          if (!num.isEmpty()) {
              list.add(new BigInteger(num));
          }
      }
      • 분리 결과에서 빈 문자열은 건너뛴다. 연속된 구분자나 문자열 앞뒤의 비숫자 문자로 인해 빈 문자열이 생성될 수 있기 때문이다.
      • new BigInteger(num)으로 변환하면 "007"과 같은 선행 0이 포함된 수도 올바르게 7로 처리되며, 자릿수 제한 없이 큰 수를 다룰 수 있다.
    • 3) 정렬 및 출력 
      Collections.sort(list);
      
      for (BigInteger num : list) {
          sb.append(num).append('\n');
      }
      
      System.out.println(sb);
      • BigInteger는 Comparable을 구현하므로 Collections.sort()가 수치 오름차순으로 정렬한다. 단순 문자열 정렬과 달리 "9"와 "10"을 올바르게 9 < 10으로 처리한다.
      • 정렬된 결과를 StringBuilder에 누적해 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M log M × D): M은 추출된 숫자 부분 문자열의 총 개수, D는 숫자의 최대 자릿수다. Collections.sort()는 O(M log M)번의 BigInteger 비교를 수행하며, 각 비교는 자릿수 D에 비례한다. 문자열 분리와 변환은 전체 입력 길이에 비례하는 O(L)이다.
    • 공간 복잡도
      • O(M × D): 추출된 M개의 BigInteger를 저장하는 리스트가 주요 메모리 사용처이며, 각 BigInteger는 자릿수 D에 비례하는 공간을 사용한다.

    문제 17266 — 어두운 굴다리

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    어두운 굴다리 (백준 17266)는
    길이 N의 굴다리에 M개의 가로등이 설치되어 있을 때, 굴다리 전체를 밝히기 위한 가로등의 최소 높이를 구하는 문제다.
    높이 H인 가로등은 좌우로 H만큼 빛을 비추며, 모든 구간이 빛의 범위 안에 들어야 한다.

    • 가로등의 높이가 높을수록 더 넓은 범위를 밝힐 수 있으므로, 높이에 대한 단조성이 성립한다. 즉 이진 탐색을 적용할 수 있다.
    • 굴다리의 시작(0)과 끝(N)도 반드시 빛이 닿아야 한다.
    • 인접한 두 가로등 사이의 간격이 2 × H를 초과하면 그 사이가 어두워지므로 조건을 위반한다.

    전략 개요

    • 핵심 알고리즘: 이진 탐색 (Binary Search on Answer)
    • 선택 이유: 높이 H가 커질수록 굴다리를 밝힐 가능성이 높아지는 단조성이 성립하므로, 가능한 최솟값을 이진 탐색으로 O(log N)에 찾을 수 있다. 각 높이에 대한 검증은 O(M)이므로 전체 O(M log N)으로 해결된다.
    • 세부 전략:
      • 탐색 범위를 1부터 N으로 설정하고, 중간값 mid를 높이로 가정해 굴다리 전체가 밝혀지는지 검증한다.
      • 검증 조건 세 가지: ① 첫 번째 가로등이 굴다리 시작(0)에 빛이 닿는지, ② 인접 가로등 간격이 2 × H 이내인지, ③ 마지막 가로등이 굴다리 끝(N)에 빛이 닿는지.
      • 조건을 만족하면 더 작은 높이를 탐색하고, 만족하지 않으면 더 큰 높이를 탐색한다.

    코드가 하는 일

    • 1) 이진 탐색 범위 설정 및 탐색 
      int left = 1;
      int right = N;
      int result = N;
      
      while (left <= right) {
          int mid = left + (right - left) / 2;
      
          if (isAllBright(mid)) {
              result = mid;
              right = mid - 1;
          } else {
              left = mid + 1;
          }
      }
      • 탐색 범위를 1(최소 높이)에서 N(최대 높이)으로 설정한다. 높이가 N이면 가로등 하나가 굴다리 전체를 커버할 수 있으므로 항상 가능한 상한값이다.
      • isAllBright(mid)가 참이면 더 낮은 높이도 가능할 수 있으므로 result = mid로 갱신하고 탐색 범위를 왼쪽으로 좁힌다.
      • 거짓이면 높이가 부족하므로 탐색 범위를 오른쪽으로 좁힌다.
    • 2) 굴다리 시작 조건 검사 
      if (light[0] - height > 0) {
          return false;
      }
      • 첫 번째 가로등의 위치에서 왼쪽으로 height만큼 빛이 닿아야 굴다리 시작점(0)이 밝혀진다.
      • light[0] - height > 0이면 빛이 0까지 닿지 않으므로 조건 위반이다.
    • 3) 인접 가로등 간격 조건 검사 
      for (int i = 1; i < M; i++) {
          if (light[i] - light[i - 1] > 2 * height) {
              return false;
          }
      }
      • 인접한 두 가로등은 각각 height만큼 서로를 향해 빛을 비추므로, 두 가로등 사이를 완전히 밝히려면 간격이 2 × height 이하여야 한다.
      • 간격이 이를 초과하는 구간이 하나라도 있으면 어두운 영역이 생기므로 즉시 false를 반환한다.
    • 4) 굴다리 끝 조건 검사 
      if (light[M - 1] + height < N) {
          return false;
      }
      
      return true;
      • 마지막 가로등의 위치에서 오른쪽으로 height만큼 빛이 닿아야 굴다리 끝점(N)이 밝혀진다.
      • light[M-1] + height < N이면 빛이 N까지 닿지 않으므로 조건 위반이다.
      • 세 조건을 모두 통과하면 true를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M log N): 이진 탐색은 O(log N)번 반복되며, 각 반복에서 isAllBright()가 M개의 가로등을 선형 탐색하므로 O(M)이 소요된다. 전체 시간 복잡도는 O(M log N)이다.
    • 공간 복잡도
      • O(M): 가로등 위치를 저장하는 light 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
