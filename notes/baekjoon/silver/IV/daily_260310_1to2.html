    문제 2870 — 수학숙제

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수학숙제 (백준 2870)는
    N개의 문자열에서 숫자로만 이루어진 연속된 부분을 모두 추출해 오름차순으로 출력하는 문제다.
    숫자 앞에 0이 붙은 경우도 그대로 하나의 수로 취급하며, 수의 크기가 매우 클 수 있다.

    • 문자열에서 숫자가 아닌 문자를 기준으로 분리하면 숫자로만 이루어진 부분 문자열을 추출할 수 있다.
    • 추출된 수는 선행 0을 포함한 채로 비교해야 하며, 값 자체로 정렬해야 하므로 문자열 정렬이 아닌 수치 정렬이 필요하다.
    • 수의 자릿수가 매우 커질 수 있으므로 int나 long이 아닌 BigInteger를 사용해야 한다.

    전략 개요

    • 핵심 알고리즘: 정규식 기반 문자열 분리 + BigInteger 정렬
    • 선택 이유: 숫자가 아닌 모든 문자를 구분자로 사용해 문자열을 분리하면 숫자 부분 문자열을 한 번에 추출할 수 있다. 수의 크기가 long 범위를 초과할 수 있으므로 BigInteger로 변환해 저장하면 크기 비교와 선행 0 처리가 자동으로 해결된다.
    • 세부 전략:
      • 각 줄을 정규식 [^0-9]로 분리해 숫자 부분만 추출한다.
      • 빈 문자열은 건너뛰고, 나머지는 new BigInteger(num)으로 변환해 리스트에 추가한다. BigInteger 생성자가 선행 0을 자동으로 제거하므로 별도 처리가 필요 없다.
      • Collections.sort()로 수치 오름차순 정렬 후 출력한다.

    코드가 하는 일

    • 1) 정규식으로 숫자 부분 추출 
      String s = br.readLine();
      String[] nums = s.split("[^0-9]");
      • [^0-9]는 숫자가 아닌 모든 문자를 의미하는 정규식이다. 이를 구분자로 사용해 문자열을 분리하면 숫자로만 이루어진 부분 문자열 배열을 얻는다.
      • 예시: "abc12de34fg"를 분리하면 ["abc", "12", "de", "34", "fg"] → 숫자가 아닌 부분은 빈 문자열 또는 문자열이 되고, "12"와 "34"만 유효한 숫자 부분이다.
    • 2) 유효한 숫자 부분만 BigInteger로 변환해 리스트에 추가 
      for (String num : nums) {
          if (!num.isEmpty()) {
              list.add(new BigInteger(num));
          }
      }
      • 분리 결과에서 빈 문자열은 건너뛴다. 연속된 구분자나 문자열 앞뒤의 비숫자 문자로 인해 빈 문자열이 생성될 수 있기 때문이다.
      • new BigInteger(num)으로 변환하면 "007"과 같은 선행 0이 포함된 수도 올바르게 7로 처리되며, 자릿수 제한 없이 큰 수를 다룰 수 있다.
    • 3) 정렬 및 출력 
      Collections.sort(list);
      
      for (BigInteger num : list) {
          sb.append(num).append('\n');
      }
      
      System.out.println(sb);
      • BigInteger는 Comparable을 구현하므로 Collections.sort()가 수치 오름차순으로 정렬한다. 단순 문자열 정렬과 달리 "9"와 "10"을 올바르게 9 < 10으로 처리한다.
      • 정렬된 결과를 StringBuilder에 누적해 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M log M × D): M은 추출된 숫자 부분 문자열의 총 개수, D는 숫자의 최대 자릿수다. Collections.sort()는 O(M log M)번의 BigInteger 비교를 수행하며, 각 비교는 자릿수 D에 비례한다. 문자열 분리와 변환은 전체 입력 길이에 비례하는 O(L)이다.
    • 공간 복잡도
      • O(M × D): 추출된 M개의 BigInteger를 저장하는 리스트가 주요 메모리 사용처이며, 각 BigInteger는 자릿수 D에 비례하는 공간을 사용한다.

    문제 17266 — 어두운 굴다리

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Generations of Tribbles (백준 9507)는
    Tribble의 번식 규칙에 따라 n세대에 존재하는 Tribble의 수를 구하는 문제다.
    점화식 dp[n] = dp[n-1] + dp[n-2] + dp[n-3] + dp[n-4]를 따르는 4항 점화식 DP 문제다.

    • 초기값은 dp[0] = 1, dp[1] = 1, dp[2] = 2, dp[3] = 4로 고정되어 있다.
    • n은 최대 67이며, dp[67]은 long 범위 내에서 처리 가능하다.
    • 테스트 케이스마다 쿼리에 응답해야 하므로, 미리 dp 배열 전체를 채워두는 메모이제이션(사전 계산) 방식이 효율적이다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (4항 점화식 DP, 사전 계산)
    • 선택 이유: n이 최대 67로 고정되어 있으므로, 입력을 받기 전에 dp[0]~dp[67]을 모두 계산해 두면 각 테스트 케이스에 O(1)로 응답할 수 있다. 여러 테스트 케이스가 주어지는 구조에서 중복 계산을 방지하는 가장 효율적인 접근이다.
    • 세부 전략:
      • dp[0] = 1, dp[1] = 1, dp[2] = 2, dp[3] = 4로 초기값을 설정한다.
      • dp[4]부터 dp[67]까지 점화식 dp[i] = dp[i-1] + dp[i-2] + dp[i-3] + dp[i-4]로 순서대로 채운다.
      • 테스트 케이스마다 n을 입력받아 dp[n]을 즉시 출력한다.

    코드가 하는 일

    • 1) 초기값 설정 
      long[] dp = new long[68];
      
      dp[0] = 1;
      dp[1] = 1;
      dp[2] = 2;
      dp[3] = 4;
      • dp 배열의 크기를 68로 선언해 인덱스 0~67을 모두 사용한다.
      • 값이 지수적으로 증가하므로 long 타입으로 선언한다. dp[67]은 약 2.7 × 10¹⁸로 long 최댓값(약 9.2 × 10¹⁸) 이내다.
      • 문제에서 정의된 초기값 4개를 직접 설정한다.
    • 2) 사전 계산 — dp 배열 전체 채우기 
      for (int i = 4; i < 68; i++) {
          dp[i] = dp[i - 1] + dp[i - 2] + dp[i - 3] + dp[i - 4];
      }
      • i = 4부터 67까지 순서대로 점화식을 적용해 dp 배열을 채운다.
      • 각 항은 바로 이전 4개의 값의 합이므로, 배열을 앞에서부터 채우면 항상 필요한 이전 값들이 이미 계산된 상태임이 보장된다.
      • 예시: dp[4] = 4 + 2 + 1 + 1 = 8, dp[5] = 8 + 4 + 2 + 1 = 15, dp[6] = 15 + 8 + 4 + 2 = 29.
    • 3) 테스트 케이스별 쿼리 응답 
      int t = Integer.parseInt(br.readLine());
      
      for (int tc = 0; tc < t; tc++) {
          int n = Integer.parseInt(br.readLine());
          sb.append(dp[n]).append('\n');
      }
      
      System.out.println(sb);
      • 테스트 케이스 수 t를 읽은 뒤, 각 n에 대해 미리 계산된 dp[n]을 O(1)에 조회해 StringBuilder에 누적한다.
      • 모든 테스트 케이스를 처리한 뒤 한 번에 출력해 I/O 비용을 최소화한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(68 + T): dp 배열 사전 계산이 O(68) = O(1), T개의 쿼리 응답이 각 O(1)이므로 전체 O(T)에 수렴한다. n의 범위가 상수(0~67)로 고정되어 있으므로 사전 계산 비용은 입력 크기와 무관한 상수 시간이다.
    • 공간 복잡도
      • O(1): dp 배열의 크기가 68로 고정되어 있으므로 입력 크기와 무관한 상수 공간이다.
