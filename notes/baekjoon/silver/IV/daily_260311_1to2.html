    문제 10211 — Maximum Subarray

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Maximum Subarray (백준 10211)는
    N개의 정수로 이루어진 배열에서 연속된 부분 배열의 합이 최대인 값을 구하는 문제다.
    부분 배열은 반드시 하나 이상의 원소를 포함해야 한다.

    • 배열 원소에 음수가 포함될 수 있으므로, 이전 합을 연장하는 것이 오히려 불리한 경우가 존재한다.
    • 이 경우 현재 원소 단독으로 새 부분 배열을 시작하는 것이 유리하다.
    • T개의 테스트 케이스가 주어지며, 각각 독립적으로 처리한다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (카데인 알고리즘, Kadane's Algorithm)
    • 선택 이유: 각 위치에서의 최대 부분 배열 합은 바로 이전 위치의 값에만 의존하므로, 한 번의 선형 순회로 O(N)에 해결된다. 브루트포스 O(N²)보다 훨씬 효율적이며, N이 최대 1,000이므로 충분히 빠르다.
    • 세부 전략:
      • dp[i]는 i번째 원소에서 끝나는 연속 부분 배열의 최대 합을 의미한다.
      • 점화식: dp[i] = Math.max(numbers[i], dp[i-1] + numbers[i]). 이전 최대 합을 연장하는 것보다 현재 원소 단독이 더 크면 새 부분 배열을 시작한다.
      • 매 단계마다 dp[i]와 전역 최댓값 max를 비교해 갱신한다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      int N = Integer.parseInt(br.readLine());
      int[] numbers = new int[N];
      int[] dp = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      dp[0] = numbers[0];
      int max = dp[0];
      • N개의 원소를 numbers 배열에 저장한다.
      • dp[0]을 첫 번째 원소로 초기화한다. 부분 배열은 최소 1개의 원소를 포함해야 하므로 첫 번째 원소 단독이 초기 기준점이 된다.
      • max도 dp[0]으로 초기화해 이후 최댓값 갱신의 기준으로 삼는다.
    • 2) DP 점화식 적용 및 최댓값 갱신 
      for (int i = 1; i < N; i++) {
          dp[i] = Math.max(numbers[i], dp[i - 1] + numbers[i]);
      
          if (dp[i] > max) {
              max = dp[i];
          }
      }
      • dp[i-1] + numbers[i]는 이전까지의 최대 부분 배열 합에 현재 원소를 더해 연장한 값이다.
      • numbers[i]는 현재 원소 단독으로 새 부분 배열을 시작한 값이다.
      • 둘 중 큰 값을 dp[i]에 저장한다. dp[i-1]이 음수이면 연장하는 것이 오히려 손해이므로 현재 원소 단독으로 시작하는 것이 유리하다.
      • 예시: numbers = [-2, 1, -3, 4, -1, 2, 1, -5, 4]이면 dp = [-2, 1, -2, 4, 3, 5, 6, 1, 5]. 최댓값은 6.
    • 3) 결과 출력 
      sb.append(max).append('\n');
      • 각 테스트 케이스의 최댓값을 StringBuilder에 누적해 마지막에 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T × N): T개의 테스트 케이스 각각에 대해 배열을 한 번 순회하며 O(N) 연산을 수행한다. 각 단계의 비교 및 갱신은 O(1)이므로 전체 O(T × N)이다.
    • 공간 복잡도
      • O(N): 입력값을 저장하는 numbers 배열과 DP 결과를 저장하는 dp 배열이 각각 N 크기를 사용한다. dp[i]는 dp[i-1]만 참조하므로 변수 하나로 최적화해 O(1)로 줄일 수 있지만, 가독성을 위해 배열로 구현했다.

    문제 13699 — 점화식

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    점화식 (백준 13699)는
    주어진 점화식 f(n) = Σ f(j) × f(n-1-j) (0 ≤ j < n), f(0) = 1에 따라
    f(n)의 값을 구하는 문제다.
    이 점화식은 카탈란 수(Catalan Number)와 동일한 구조를 갖는다.

    • f(0) = 1로 초기값이 고정되어 있으며, f(1)부터 f(n)까지 순서대로 계산한다.
    • f(n)을 계산하려면 f(0)~f(n-1)이 모두 필요하므로 바텀업 방식의 DP가 적합하다.
    • n은 최대 35이며, f(35)는 long 범위 내에서 처리 가능하다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (바텀업 DP)
    • 선택 이유: f(n)은 이전에 계산된 f(0)~f(n-1)의 곱의 합으로 정의된다. 작은 값부터 순서대로 계산해 배열에 저장하면 중복 계산 없이 O(N²)에 해결된다. n이 최대 35로 매우 작아 이중 반복문으로 충분히 빠르다.
    • 세부 전략:
      • dp[0] = 1로 초기화한다.
      • i = 1부터 n까지 순서대로, j = 0부터 i-1까지 순회하며 dp[j] × dp[i-1-j]를 누적해 dp[i]를 계산한다.
      • 최종적으로 dp[n]을 출력한다.

    코드가 하는 일

    • 1) dp 배열 초기화 
      long[] dp = new long[n + 1];
      dp[0] = 1;
      • dp 배열의 크기를 n + 1로 선언해 인덱스 0~n을 모두 사용한다.
      • f(n)의 값이 지수적으로 증가하므로 long 타입으로 선언한다.
      • 문제에서 정의된 초기값 f(0) = 1을 설정한다. 나머지 원소는 Java 기본값인 0으로 자동 초기화된다.
    • 2) 점화식 적용 — 바텀업 DP 
      for (int i = 1; i <= n; i++) {
          for (int j = 0; j < i; j++) {
              dp[i] += dp[j] * dp[i - 1 - j];
          }
      }
      • 외부 반복문의 i는 현재 계산 중인 f(i)의 인덱스다. i = 1부터 n까지 순서대로 계산한다.
      • 내부 반복문의 j는 0부터 i-1까지 순회하며 dp[j] × dp[i-1-j]를 dp[i]에 누적한다.
      • 이때 j + (i-1-j) = i-1이므로, 항상 인덱스 합이 i-1인 두 항의 곱을 더하는 구조다.
      • 예시: dp[1] = dp[0] × dp[0] = 1. dp[2] = dp[0] × dp[1] + dp[1] × dp[0] = 2. dp[3] = dp[0] × dp[2] + dp[1] × dp[1] + dp[2] × dp[0] = 5.
    • 3) 결과 출력 
      System.out.println(dp[n]);
      • 계산이 완료된 dp[n]을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 외부 반복문이 N번, 내부 반복문이 최대 N번 실행되므로 전체 O(N²)이다. n이 최대 35이므로 최악의 경우에도 35² = 1,225번의 연산으로 매우 빠르게 동작한다.
    • 공간 복잡도
      • O(N): f(0)~f(n)을 저장하는 dp 배열이 N + 1 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
