    문제 16173 — 점프왕 쩰리 (Small)

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    점프왕 쩰리 (Small) (백준 16173)는
    N × N 격자의 (0, 0)에서 출발해 각 칸에 적힌 수만큼 오른쪽 또는 아래로 이동할 때,
    목표 칸(-1)에 도달할 수 있는지 여부를 출력하는 문제다.
    도달 가능하면 "HaruHaru", 불가능하면 "Hing"을 출력한다.

    • 각 칸의 값이 이동 칸 수를 의미하며, 오른쪽과 아래 방향으로만 이동할 수 있다.
    • 값이 0인 칸에서는 이동할 수 없으므로 막힌 칸이 된다.
    • 목표 칸은 -1로 표시되며, 이 칸에 도달하는 순간 즉시 "HaruHaru"를 출력하고 종료한다.

    전략 개요

    • 핵심 알고리즘: BFS (너비 우선 탐색)
    • 선택 이유: 도달 가능 여부만 판단하면 되므로 BFS가 적합하다. 방문 배열로 중복 방문을 방지하고, 목표 칸에 도달하면 즉시 종료한다. N이 최대 3으로 매우 작아 DFS로도 해결 가능하지만, BFS가 구조적으로 명확하다.
    • 세부 전략:
      • (0, 0)을 큐에 넣고 시작한다.
      • 현재 칸의 값 moveCnt만큼 오른쪽(c + moveCnt)과 아래쪽(r + moveCnt)으로 이동한 칸을 큐에 추가한다.
      • 큐에서 꺼낸 칸이 -1이면 "HaruHaru"를 출력하고 종료한다.
      • 큐가 빌 때까지 도달하지 못하면 "Hing"을 출력한다.

    코드가 하는 일

    • 1) 입력 및 BFS 초기화 
      Deque queue = new ArrayDeque<>();
      boolean[][] visited = new boolean[N][N];
      
      queue.add(new int[]{0, 0});
      visited[0][0] = true;
      • BFS에 사용할 큐와 방문 배열을 초기화한다.
      • 시작 위치 (0, 0)을 큐에 넣고 방문 표시를 한다.
    • 2) 목표 칸 도달 확인 
      int[] polled = queue.poll();
      int r = polled[0];
      int c = polled[1];
      
      if (area[r][c] == -1) {
          System.out.println("HaruHaru");
          return;
      }
      • 큐에서 꺼낸 칸의 값이 -1이면 목표 칸에 도달한 것이므로 즉시 "HaruHaru"를 출력하고 종료한다.
    • 3) 다음 이동 칸 탐색 및 큐 추가 
      int moveCnt = area[r][c];
      if (moveCnt == 0) continue;
      
      if (r + moveCnt < N && !visited[r + moveCnt][c]) {
          visited[r + moveCnt][c] = true;
          queue.add(new int[]{r + moveCnt, c});
      }
      
      if (c + moveCnt < N && !visited[r][c + moveCnt]) {
          visited[r][c + moveCnt] = true;
          queue.add(new int[]{r, c + moveCnt});
      }
      • 현재 칸의 값 moveCnt가 0이면 이동할 수 없으므로 건너뛴다.
      • 아래쪽 (r + moveCnt, c)와 오른쪽 (r, c + moveCnt) 방향으로의 이동을 시도한다.
      • 격자 범위를 벗어나지 않고 아직 방문하지 않은 칸만 큐에 추가하고 방문 표시한다.
      • 방문 배열로 동일한 칸을 중복으로 처리하는 것을 방지한다.
    • 4) 도달 불가 처리 
      System.out.println("Hing");
      • 큐가 빌 때까지 목표 칸에 도달하지 못하면 "Hing"을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 격자의 각 칸은 방문 배열에 의해 최대 한 번씩만 큐에 추가되므로, 전체 탐색 비용은 격자 크기에 비례하는 O(N²)이다. N이 최대 3이므로 사실상 O(1)에 수렴한다.
    • 공간 복잡도
      • O(N²): 격자를 저장하는 2차원 배열과 방문 여부를 저장하는 visited 배열이 각각 N × N 크기를 사용하며, 큐도 최대 N² 크기로 증가할 수 있다.

    문제 2422 — 한윤정이 이탈리아에 가서 아이스크림을 사먹는데

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    한윤정이 이탈리아에 가서 아이스크림을 사먹는데 (백준 2422)는
    N가지 아이스크림 중 3가지를 고를 때, 궁합이 맞지 않는 조합을 포함하지 않는 경우의 수를 구하는 문제다.
    M개의 궁합이 맞지 않는 쌍이 주어지며, 선택한 3가지 아이스크림 중 어느 두 쌍도 금지 조합에 포함되면 안 된다.

    • N은 최대 200, M은 최대 10,000이므로 3중 반복문으로 모든 조합을 탐색해도 최대 200³ / 6 ≈ 1,333,333번으로 충분히 빠르다.
    • 아이스크림은 순서 없이 3가지를 고르는 조합이므로 i < j < k 순서로만 탐색해 중복을 방지한다.
    • 금지 쌍을 2차원 배열로 관리하면 두 아이스크림이 궁합이 맞지 않는지를 O(1)에 확인할 수 있다.

    전략 개요

    • 핵심 알고리즘: 브루트포스 조합 탐색 + 2차원 배열 금지 쌍 관리
    • 선택 이유: N이 최대 200으로 작아 3중 반복문으로 모든 조합을 탐색해도 시간 내에 처리된다. 금지 쌍을 boolean[N+1][N+1] 배열에 저장하면 두 아이스크림 사이의 금지 여부를 O(1)에 조회할 수 있어 효율적이다.
    • 세부 전략:
      • 금지 쌍 (ice, cream)을 입력받아 iceCream[ice][cream]과 iceCream[cream][ice] 모두 true로 설정한다.
      • i < j < k 조건으로 3중 반복문을 순회해 모든 3가지 조합을 탐색한다.
      • (i, j), (i, k), (j, k) 세 쌍 중 하나라도 금지 쌍이면 해당 조합을 건너뛰고, 모두 허용이면 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 금지 쌍 입력 및 배열 구성 
      boolean[][] iceCream = new boolean[N + 1][N + 1];
      
      for (int i = 0; i < M; i++) {
          int ice = Integer.parseInt(st.nextToken());
          int cream = Integer.parseInt(st.nextToken());
      
          iceCream[ice][cream] = true;
          iceCream[cream][ice] = true;
      }
      • 금지 쌍을 대칭 행렬 형태로 저장한다. iceCream[a][b] = true이면 a와 b는 함께 선택할 수 없다.
      • 양방향으로 저장하므로 이후 탐색에서 순서에 관계없이 O(1)로 조회할 수 있다.
    • 2) 3중 반복문으로 모든 조합 탐색 
      for (int i = 1; i <= N; i++) {
          for (int j = i + 1; j <= N; j++) {
              if (iceCream[i][j]) continue;
      
              for (int k = j + 1; k <= N; k++) {
                  if (!iceCream[i][k] && !iceCream[j][k]) {
                      cnt++;
                  }
              }
          }
      }
      • i < j < k 조건으로 순회해 중복 없이 모든 3가지 조합을 탐색한다.
      • iceCream[i][j]가 true이면 (i, j) 쌍이 이미 금지이므로, k를 탐색할 필요 없이 즉시 다음 j로 건너뛴다. 이 조기 종료(pruning)로 불필요한 내부 반복을 줄인다.
      • 내부 반복문에서 (i, k)와 (j, k) 쌍이 모두 허용인 경우에만 카운트를 증가시킨다.
      • 예시: i=1, j=2, k=3일 때 iceCream[1][3]과 iceCream[2][3]이 모두 false이면 {1, 2, 3} 조합이 유효하다.
    • 3) 결과 출력 
      System.out.println(cnt);
      • 금지 쌍을 포함하지 않는 유효한 3가지 조합의 총 수 cnt를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N³): 3중 반복문으로 모든 조합을 탐색하므로 최악의 경우 O(N³ / 6)이다. N이 최대 200이므로 최악의 경우에도 약 133만 번의 연산으로 충분히 빠르다. (i, j) 쌍이 금지인 경우 내부 반복을 건너뛰므로 실제 수행 횟수는 더 적다.
    • 공간 복잡도
      • O(N²): 금지 쌍을 저장하는 boolean[N+1][N+1] 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
