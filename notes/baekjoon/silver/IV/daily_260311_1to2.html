    문제 16173 — 점프왕 쩰리 (Small)

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    점프왕 쩰리 (Small) (백준 16173)는
    N × N 격자의 (0, 0)에서 출발해 각 칸에 적힌 수만큼 오른쪽 또는 아래로 이동할 때,
    목표 칸(-1)에 도달할 수 있는지 여부를 출력하는 문제다.
    도달 가능하면 "HaruHaru", 불가능하면 "Hing"을 출력한다.

    • 각 칸의 값이 이동 칸 수를 의미하며, 오른쪽과 아래 방향으로만 이동할 수 있다.
    • 값이 0인 칸에서는 이동할 수 없으므로 막힌 칸이 된다.
    • 목표 칸은 -1로 표시되며, 이 칸에 도달하는 순간 즉시 "HaruHaru"를 출력하고 종료한다.

    전략 개요

    • 핵심 알고리즘: BFS (너비 우선 탐색)
    • 선택 이유: 도달 가능 여부만 판단하면 되므로 BFS가 적합하다. 방문 배열로 중복 방문을 방지하고, 목표 칸에 도달하면 즉시 종료한다. N이 최대 3으로 매우 작아 DFS로도 해결 가능하지만, BFS가 구조적으로 명확하다.
    • 세부 전략:
      • (0, 0)을 큐에 넣고 시작한다.
      • 현재 칸의 값 moveCnt만큼 오른쪽(c + moveCnt)과 아래쪽(r + moveCnt)으로 이동한 칸을 큐에 추가한다.
      • 큐에서 꺼낸 칸이 -1이면 "HaruHaru"를 출력하고 종료한다.
      • 큐가 빌 때까지 도달하지 못하면 "Hing"을 출력한다.

    코드가 하는 일

    • 1) 입력 및 BFS 초기화 
      Deque queue = new ArrayDeque<>();
      boolean[][] visited = new boolean[N][N];
      
      queue.add(new int[]{0, 0});
      visited[0][0] = true;
      • BFS에 사용할 큐와 방문 배열을 초기화한다.
      • 시작 위치 (0, 0)을 큐에 넣고 방문 표시를 한다.
    • 2) 목표 칸 도달 확인 
      int[] polled = queue.poll();
      int r = polled[0];
      int c = polled[1];
      
      if (area[r][c] == -1) {
          System.out.println("HaruHaru");
          return;
      }
      • 큐에서 꺼낸 칸의 값이 -1이면 목표 칸에 도달한 것이므로 즉시 "HaruHaru"를 출력하고 종료한다.
    • 3) 다음 이동 칸 탐색 및 큐 추가 
      int moveCnt = area[r][c];
      if (moveCnt == 0) continue;
      
      if (r + moveCnt < N && !visited[r + moveCnt][c]) {
          visited[r + moveCnt][c] = true;
          queue.add(new int[]{r + moveCnt, c});
      }
      
      if (c + moveCnt < N && !visited[r][c + moveCnt]) {
          visited[r][c + moveCnt] = true;
          queue.add(new int[]{r, c + moveCnt});
      }
      • 현재 칸의 값 moveCnt가 0이면 이동할 수 없으므로 건너뛴다.
      • 아래쪽 (r + moveCnt, c)와 오른쪽 (r, c + moveCnt) 방향으로의 이동을 시도한다.
      • 격자 범위를 벗어나지 않고 아직 방문하지 않은 칸만 큐에 추가하고 방문 표시한다.
      • 방문 배열로 동일한 칸을 중복으로 처리하는 것을 방지한다.
    • 4) 도달 불가 처리 
      System.out.println("Hing");
      • 큐가 빌 때까지 목표 칸에 도달하지 못하면 "Hing"을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 격자의 각 칸은 방문 배열에 의해 최대 한 번씩만 큐에 추가되므로, 전체 탐색 비용은 격자 크기에 비례하는 O(N²)이다. N이 최대 3이므로 사실상 O(1)에 수렴한다.
    • 공간 복잡도
      • O(N²): 격자를 저장하는 2차원 배열과 방문 여부를 저장하는 visited 배열이 각각 N × N 크기를 사용하며, 큐도 최대 N² 크기로 증가할 수 있다.

    문제 2422 — 한윤정이 이탈리아에 가서 아이스크림을 사먹는데

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    점화식 (백준 13699)는
    주어진 점화식 f(n) = Σ f(j) × f(n-1-j) (0 ≤ j < n), f(0) = 1에 따라
    f(n)의 값을 구하는 문제다.
    이 점화식은 카탈란 수(Catalan Number)와 동일한 구조를 갖는다.

    • f(0) = 1로 초기값이 고정되어 있으며, f(1)부터 f(n)까지 순서대로 계산한다.
    • f(n)을 계산하려면 f(0)~f(n-1)이 모두 필요하므로 바텀업 방식의 DP가 적합하다.
    • n은 최대 35이며, f(35)는 long 범위 내에서 처리 가능하다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (바텀업 DP)
    • 선택 이유: f(n)은 이전에 계산된 f(0)~f(n-1)의 곱의 합으로 정의된다. 작은 값부터 순서대로 계산해 배열에 저장하면 중복 계산 없이 O(N²)에 해결된다. n이 최대 35로 매우 작아 이중 반복문으로 충분히 빠르다.
    • 세부 전략:
      • dp[0] = 1로 초기화한다.
      • i = 1부터 n까지 순서대로, j = 0부터 i-1까지 순회하며 dp[j] × dp[i-1-j]를 누적해 dp[i]를 계산한다.
      • 최종적으로 dp[n]을 출력한다.

    코드가 하는 일

    • 1) dp 배열 초기화 
      long[] dp = new long[n + 1];
      dp[0] = 1;
      • dp 배열의 크기를 n + 1로 선언해 인덱스 0~n을 모두 사용한다.
      • f(n)의 값이 지수적으로 증가하므로 long 타입으로 선언한다.
      • 문제에서 정의된 초기값 f(0) = 1을 설정한다. 나머지 원소는 Java 기본값인 0으로 자동 초기화된다.
    • 2) 점화식 적용 — 바텀업 DP 
      for (int i = 1; i <= n; i++) {
          for (int j = 0; j < i; j++) {
              dp[i] += dp[j] * dp[i - 1 - j];
          }
      }
      • 외부 반복문의 i는 현재 계산 중인 f(i)의 인덱스다. i = 1부터 n까지 순서대로 계산한다.
      • 내부 반복문의 j는 0부터 i-1까지 순회하며 dp[j] × dp[i-1-j]를 dp[i]에 누적한다.
      • 이때 j + (i-1-j) = i-1이므로, 항상 인덱스 합이 i-1인 두 항의 곱을 더하는 구조다.
      • 예시: dp[1] = dp[0] × dp[0] = 1. dp[2] = dp[0] × dp[1] + dp[1] × dp[0] = 2. dp[3] = dp[0] × dp[2] + dp[1] × dp[1] + dp[2] × dp[0] = 5.
    • 3) 결과 출력 
      System.out.println(dp[n]);
      • 계산이 완료된 dp[n]을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 외부 반복문이 N번, 내부 반복문이 최대 N번 실행되므로 전체 O(N²)이다. n이 최대 35이므로 최악의 경우에도 35² = 1,225번의 연산으로 매우 빠르게 동작한다.
    • 공간 복잡도
      • O(N): f(0)~f(n)을 저장하는 dp 배열이 N + 1 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
