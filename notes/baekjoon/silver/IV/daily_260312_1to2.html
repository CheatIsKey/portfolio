    문제 15624 — 피보나치 수 7

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    피보나치 수 7 (백준 15624)는
    N번째 피보나치 수를 1,000,000,007로 나눈 나머지를 출력하는 문제다.
    N이 최대 1,000,000으로 매우 크기 때문에 모듈러 연산을 적용한 바텀업 DP로 해결해야 한다.

    • 피보나치 수는 F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)로 정의된다.
    • N이 최대 100만이므로 값이 매우 커져 long 범위를 초과할 수 있다. 매 단계에서 1,000,000,007로 나머지를 취해야 한다.
    • N = 0과 N = 1은 별도로 처리한다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (바텀업 DP) + 모듈러 연산
    • 선택 이유: F(n)은 F(n-1)과 F(n-2)에만 의존하므로 작은 값부터 순서대로 계산하는 바텀업 DP가 적합하다. 재귀로 구현하면 N이 최대 100만일 때 스택 오버플로우가 발생할 수 있으므로 반복문 기반 DP를 사용한다.
    • 세부 전략:
      • F(0) = 0, F(1) = 1로 초기값을 설정한다.
      • i = 2부터 N까지 fibo[i] = (fibo[i-1] + fibo[i-2]) % 1,000,000,007을 순서대로 계산한다.
      • 매 단계에서 모듈러 연산을 적용해 값이 long 범위를 초과하지 않도록 한다.

    코드가 하는 일

    • 1) 예외 처리 — N = 0, N = 1 
      if (N == 0) {
          System.out.println(0);
          return;
      }
      
      if (N == 1) {
          System.out.println(1);
          return;
      }
      • N = 0이면 F(0) = 0을, N = 1이면 F(1) = 1을 즉시 출력하고 종료한다.
      • 이 두 경우를 먼저 처리하면 이후 배열 인덱스 접근 시 범위 오류를 방지할 수 있다.
    • 2) 초기값 설정 
      long[] fibo = new long[N + 1];
      fibo[0] = 0;
      fibo[1] = 1;
      • 크기 N + 1의 long 배열을 선언해 인덱스 0~N을 모두 사용한다.
      • 피보나치 수는 지수적으로 증가하지만, 매 단계 모듈러 연산으로 값을 1,000,000,007 미만으로 유지하므로 long으로 충분하다.
    • 3) 바텀업 DP로 피보나치 수 계산 
      for (int i = 2; i <= N; i++) {
          fibo[i] = (fibo[i - 1] + fibo[i - 2]) % 1000000007;
      }
      • i = 2부터 N까지 순서대로 점화식을 적용한다.
      • 매 단계에서 % 1,000,000,007을 적용해 값이 모듈러 범위 내로 유지된다. fibo[i-1] + fibo[i-2]의 최대값은 약 2 × 10⁹으로 long 범위 이내이므로 덧셈 후 나머지를 취하는 방식이 안전하다.
    • 4) 결과 출력 
      System.out.println(fibo[N]);
      • 계산이 완료된 fibo[N]을 출력한다. 이 값은 이미 1,000,000,007로 나눈 나머지가 적용된 상태다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다. N이 최대 100만이지만 단순 덧셈과 나머지 연산만으로 처리되어 충분히 빠르다.
    • 공간 복잡도
      • O(N): F(0)~F(N)을 저장하는 fibo 배열이 N + 1 크기를 사용한다. fibo[i]는 fibo[i-1]과 fibo[i-2]만 참조하므로 변수 두 개로 최적화해 O(1)로 줄일 수 있지만, 가독성을 위해 배열로 구현했다.

    문제 2417 — 정수 제곱근

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 제곱근 (백준 2417)은
    정수 N이 주어질 때 p² ≥ N을 만족하는 가장 작은 양의 정수 p를 구하는 문제다.
    N이 최대 2⁶³ - 1로 long 범위를 초과할 수 있으므로 BigInteger를 사용해야 한다.

    • p² ≥ N을 만족하는 최솟값을 구하는 것이므로, p에 대한 단조성이 성립해 이진 탐색을 적용할 수 있다.
    • N의 최댓값이 2⁶³ - 1이면 p의 최댓값은 √(2⁶³) ≈ 3,037,000,499이므로, 탐색 범위의 상한을 3,037,000,500으로 설정한다.
    • mid² 계산 시 long 오버플로우가 발생할 수 있으므로 BigInteger로 변환해 곱셈을 수행한다.

    전략 개요

    • 핵심 알고리즘: 이진 탐색 (Binary Search on Answer)
    • 선택 이유: p가 커질수록 p² ≥ N을 만족할 가능성이 높아지는 단조성이 성립하므로, 조건을 만족하는 최솟값을 이진 탐색으로 O(log(√N))에 찾을 수 있다. 직접 제곱근을 계산하면 부동소수점 오차가 발생할 수 있으므로 이진 탐색이 더 안전하다.
    • 세부 전략:
      • 탐색 범위를 0부터 3,037,000,500으로 설정한다.
      • 중간값 mid를 BigInteger로 변환해 mid² ≥ N이면 조건을 만족하므로 result = mid로 갱신하고 더 작은 값을 탐색한다.
      • mid² < N이면 p가 더 커야 하므로 탐색 범위를 오른쪽으로 좁힌다.

    코드가 하는 일

    • 1) 입력 및 탐색 범위 초기화 
      BigInteger N = new BigInteger(br.readLine());
      
      long start = 0;
      long end = 3037000500L;
      long result = 0;
      • N이 최대 2⁶³ - 1이므로 long 범위를 초과할 수 있어 BigInteger로 입력받는다.
      • 탐색 범위의 상한을 3,037,000,500으로 설정한다. 이 값의 제곱이 약 9.22 × 10¹⁸으로 long의 최댓값(2⁶³ - 1 ≈ 9.22 × 10¹⁸)을 커버할 수 있다.
      • result는 조건을 만족하는 가장 작은 p를 저장할 변수다.
    • 2) 이진 탐색으로 최솟값 탐색 
      while (start <= end) {
          long mid = start + (end - start) / 2;
          BigInteger midBI = BigInteger.valueOf(mid);
      
          if (midBI.multiply(midBI).compareTo(N) >= 0) {
              result = mid;
              end = mid - 1;
          } else {
              start = mid + 1;
          }
      }
      • mid를 BigInteger로 변환한 뒤 midBI.multiply(midBI)로 mid²를 계산한다. long으로 직접 곱하면 최대 약 9.2 × 10¹⁸을 초과해 오버플로우가 발생할 수 있으므로 반드시 BigInteger로 처리해야 한다.
      • compareTo(N) >= 0이면 mid² ≥ N이므로 조건을 만족한다. result = mid로 갱신하고 더 작은 값을 찾기 위해 end = mid - 1로 범위를 좁힌다.
      • mid² < N이면 p가 더 커야 하므로 start = mid + 1로 범위를 오른쪽으로 좁힌다.
      • 예시: N = 10이면 p = 4 (4² = 16 ≥ 10). N = 9이면 p = 3 (3² = 9 ≥ 9).
    • 3) 결과 출력 
      System.out.println(result);
      • 탐색이 완료된 뒤 p² ≥ N을 만족하는 가장 작은 정수 result를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(log(√N)): 탐색 범위가 0부터 약 3 × 10⁹이므로 이진 탐색은 최대 log₂(3 × 10⁹) ≈ 32번 반복된다. 각 반복에서 BigInteger 곱셈과 비교가 수행되지만 상수 시간에 가깝다.
    • 공간 복잡도
      • O(1): BigInteger 변수 몇 개와 상수 개의 long 변수만 사용하므로 입력 크기와 무관한 상수 공간이다.
