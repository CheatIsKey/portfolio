    문제 15624 — 피보나치 수 7

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    피보나치 수 7 (백준 15624)는
    N번째 피보나치 수를 1,000,000,007로 나눈 나머지를 출력하는 문제다.
    N이 최대 1,000,000으로 매우 크기 때문에 모듈러 연산을 적용한 바텀업 DP로 해결해야 한다.

    • 피보나치 수는 F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)로 정의된다.
    • N이 최대 100만이므로 값이 매우 커져 long 범위를 초과할 수 있다. 매 단계에서 1,000,000,007로 나머지를 취해야 한다.
    • N = 0과 N = 1은 별도로 처리한다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (바텀업 DP) + 모듈러 연산
    • 선택 이유: F(n)은 F(n-1)과 F(n-2)에만 의존하므로 작은 값부터 순서대로 계산하는 바텀업 DP가 적합하다. 재귀로 구현하면 N이 최대 100만일 때 스택 오버플로우가 발생할 수 있으므로 반복문 기반 DP를 사용한다.
    • 세부 전략:
      • F(0) = 0, F(1) = 1로 초기값을 설정한다.
      • i = 2부터 N까지 fibo[i] = (fibo[i-1] + fibo[i-2]) % 1,000,000,007을 순서대로 계산한다.
      • 매 단계에서 모듈러 연산을 적용해 값이 long 범위를 초과하지 않도록 한다.

    코드가 하는 일

    • 1) 예외 처리 — N = 0, N = 1 
      if (N == 0) {
          System.out.println(0);
          return;
      }
      
      if (N == 1) {
          System.out.println(1);
          return;
      }
      • N = 0이면 F(0) = 0을, N = 1이면 F(1) = 1을 즉시 출력하고 종료한다.
      • 이 두 경우를 먼저 처리하면 이후 배열 인덱스 접근 시 범위 오류를 방지할 수 있다.
    • 2) 초기값 설정 
      long[] fibo = new long[N + 1];
      fibo[0] = 0;
      fibo[1] = 1;
      • 크기 N + 1의 long 배열을 선언해 인덱스 0~N을 모두 사용한다.
      • 피보나치 수는 지수적으로 증가하지만, 매 단계 모듈러 연산으로 값을 1,000,000,007 미만으로 유지하므로 long으로 충분하다.
    • 3) 바텀업 DP로 피보나치 수 계산 
      for (int i = 2; i <= N; i++) {
          fibo[i] = (fibo[i - 1] + fibo[i - 2]) % 1000000007;
      }
      • i = 2부터 N까지 순서대로 점화식을 적용한다.
      • 매 단계에서 % 1,000,000,007을 적용해 값이 모듈러 범위 내로 유지된다. fibo[i-1] + fibo[i-2]의 최대값은 약 2 × 10⁹으로 long 범위 이내이므로 덧셈 후 나머지를 취하는 방식이 안전하다.
    • 4) 결과 출력 
      System.out.println(fibo[N]);
      • 계산이 완료된 fibo[N]을 출력한다. 이 값은 이미 1,000,000,007로 나눈 나머지가 적용된 상태다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다. N이 최대 100만이지만 단순 덧셈과 나머지 연산만으로 처리되어 충분히 빠르다.
    • 공간 복잡도
      • O(N): F(0)~F(N)을 저장하는 fibo 배열이 N + 1 크기를 사용한다. fibo[i]는 fibo[i-1]과 fibo[i-2]만 참조하므로 변수 두 개로 최적화해 O(1)로 줄일 수 있지만, 가독성을 위해 배열로 구현했다.

    문제 2417 — 정수 제곱근

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    한윤정이 이탈리아에 가서 아이스크림을 사먹는데 (백준 2422)는
    N가지 아이스크림 중 3가지를 고를 때, 궁합이 맞지 않는 조합을 포함하지 않는 경우의 수를 구하는 문제다.
    M개의 궁합이 맞지 않는 쌍이 주어지며, 선택한 3가지 아이스크림 중 어느 두 쌍도 금지 조합에 포함되면 안 된다.

    • N은 최대 200, M은 최대 10,000이므로 3중 반복문으로 모든 조합을 탐색해도 최대 200³ / 6 ≈ 1,333,333번으로 충분히 빠르다.
    • 아이스크림은 순서 없이 3가지를 고르는 조합이므로 i < j < k 순서로만 탐색해 중복을 방지한다.
    • 금지 쌍을 2차원 배열로 관리하면 두 아이스크림이 궁합이 맞지 않는지를 O(1)에 확인할 수 있다.

    전략 개요

    • 핵심 알고리즘: 브루트포스 조합 탐색 + 2차원 배열 금지 쌍 관리
    • 선택 이유: N이 최대 200으로 작아 3중 반복문으로 모든 조합을 탐색해도 시간 내에 처리된다. 금지 쌍을 boolean[N+1][N+1] 배열에 저장하면 두 아이스크림 사이의 금지 여부를 O(1)에 조회할 수 있어 효율적이다.
    • 세부 전략:
      • 금지 쌍 (ice, cream)을 입력받아 iceCream[ice][cream]과 iceCream[cream][ice] 모두 true로 설정한다.
      • i < j < k 조건으로 3중 반복문을 순회해 모든 3가지 조합을 탐색한다.
      • (i, j), (i, k), (j, k) 세 쌍 중 하나라도 금지 쌍이면 해당 조합을 건너뛰고, 모두 허용이면 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 금지 쌍 입력 및 배열 구성 
      boolean[][] iceCream = new boolean[N + 1][N + 1];
      
      for (int i = 0; i < M; i++) {
          int ice = Integer.parseInt(st.nextToken());
          int cream = Integer.parseInt(st.nextToken());
      
          iceCream[ice][cream] = true;
          iceCream[cream][ice] = true;
      }
      • 금지 쌍을 대칭 행렬 형태로 저장한다. iceCream[a][b] = true이면 a와 b는 함께 선택할 수 없다.
      • 양방향으로 저장하므로 이후 탐색에서 순서에 관계없이 O(1)로 조회할 수 있다.
    • 2) 3중 반복문으로 모든 조합 탐색 
      for (int i = 1; i <= N; i++) {
          for (int j = i + 1; j <= N; j++) {
              if (iceCream[i][j]) continue;
      
              for (int k = j + 1; k <= N; k++) {
                  if (!iceCream[i][k] && !iceCream[j][k]) {
                      cnt++;
                  }
              }
          }
      }
      • i < j < k 조건으로 순회해 중복 없이 모든 3가지 조합을 탐색한다.
      • iceCream[i][j]가 true이면 (i, j) 쌍이 이미 금지이므로, k를 탐색할 필요 없이 즉시 다음 j로 건너뛴다. 이 조기 종료(pruning)로 불필요한 내부 반복을 줄인다.
      • 내부 반복문에서 (i, k)와 (j, k) 쌍이 모두 허용인 경우에만 카운트를 증가시킨다.
      • 예시: i=1, j=2, k=3일 때 iceCream[1][3]과 iceCream[2][3]이 모두 false이면 {1, 2, 3} 조합이 유효하다.
    • 3) 결과 출력 
      System.out.println(cnt);
      • 금지 쌍을 포함하지 않는 유효한 3가지 조합의 총 수 cnt를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N³): 3중 반복문으로 모든 조합을 탐색하므로 최악의 경우 O(N³ / 6)이다. N이 최대 200이므로 최악의 경우에도 약 133만 번의 연산으로 충분히 빠르다. (i, j) 쌍이 금지인 경우 내부 반복을 건너뛰므로 실제 수행 횟수는 더 적다.
    • 공간 복잡도
      • O(N²): 금지 쌍을 저장하는 boolean[N+1][N+1] 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
