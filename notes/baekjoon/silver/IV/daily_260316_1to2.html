    문제 14469 — 소가 길을 건너간 이유 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    소가 길을 건너간 이유 3 (백준 14469)는
    N마리의 소가 각각 도착 시간과 검문 소요 시간을 가질 때, 모든 소의 검문이 완료되는 최소 시간을 구하는 문제다.
    검문소는 한 번에 한 마리만 처리할 수 있으며, 소들은 도착 순서대로 검문을 받는다.

    • 검문소는 한 번에 한 마리만 처리하므로, 이전 소의 검문이 끝나야 다음 소의 검문을 시작할 수 있다.
    • 소가 도착하기 전에 검문소가 비어있으면 소가 도착하는 시점부터 검문이 시작된다.
    • 소들을 도착 시간 오름차순으로 정렬하면, 현재 시간을 기준으로 각 소의 시작 시점을 순서대로 계산할 수 있다.

    전략 개요

    • 핵심 알고리즘: 그리디 (도착 시간 오름차순 정렬 + 시뮬레이션)
    • 선택 이유: 검문소는 선착순으로 처리되므로, 도착 시간 순으로 정렬한 뒤 현재 검문 완료 시간(currentTime)을 기준으로 각 소의 검문 시작 시점을 결정하면 O(N)에 시뮬레이션할 수 있다.
    • 세부 전략:
      • 소들을 도착 시간 오름차순으로 정렬한다.
      • 각 소에 대해 현재 검문 완료 시간 currentTime과 소의 도착 시간을 비교한다.
      • 소가 도착했을 때 검문소가 비어있으면(cow.arrive > currentTime) 소의 도착 시간부터 검문을 시작한다.
      • 검문소가 아직 사용 중이면 이전 검문이 끝나는 즉시 다음 소의 검문을 시작한다.

    코드가 하는 일

    • 1) 입력 및 Cow 객체 리스트 구성 
      List list = new ArrayList<>();
      
      for (int i = 0; i < N; i++) {
          int arrive = Integer.parseInt(st.nextToken());
          int validate = Integer.parseInt(st.nextToken());
          list.add(new Cow(arrive, validate));
      }
      • 각 소의 도착 시간(arrive)과 검문 소요 시간(validate)을 Cow 객체로 묶어 리스트에 저장한다.
    • 2) 도착 시간 오름차순 정렬 
      list.sort((a, b) -> {
          return Integer.compare(a.arrive, b.arrive);
      });
      • 검문소는 먼저 도착한 소를 먼저 처리하므로, 도착 시간 오름차순으로 정렬한다.
      • 정렬 후에는 앞에서부터 순서대로 처리하면 된다.
    • 3) 시뮬레이션 — 검문 완료 시간 계산 
      int currentTime = 0;
      
      for (Cow cow : list) {
          if (cow.arrive > currentTime) {
              currentTime = cow.arrive;
              currentTime += cow.validate;
          } else {
              currentTime += cow.validate;
          }
      }
      • currentTime은 검문소가 다음 소를 받을 수 있는 가장 빠른 시각(현재까지의 검문 완료 시각)을 나타낸다.
      • cow.arrive > currentTime이면 소가 도착했을 때 검문소가 비어있는 경우다. 검문소가 소의 도착을 기다려야 하므로 currentTime을 cow.arrive로 갱신한 뒤 cow.validate를 더한다.
      • cow.arrive <= currentTime이면 소가 도착했을 때 검문소가 아직 사용 중인 경우다. 이전 검문이 끝나는 즉시 다음 검문이 시작되므로 currentTime에 cow.validate만 더한다.
      • 예시: 소들이 [(1, 5), (2, 3), (10, 2)] 순으로 도착하면 currentTime: 0 → 6 → 9 → 12. 최종 답은 12.
    • 4) 결과 출력 
      System.out.println(currentTime);
      • 모든 소의 검문이 완료되는 시각 currentTime을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 도착 시간 기준 정렬 비용이다. 이후 선형 시뮬레이션은 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N마리의 소를 저장하는 Cow 객체 리스트가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 1755 — 숫자놀이

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자놀이 (백준 1755)는
    M 이상 N 이하의 정수를 각 자릿수를 영어로 읽은 문자열 기준으로 사전순 정렬한 뒤 10개씩 줄 바꿔 출력하는 문제다.
    각 자리 숫자를 영어 단어로 변환한 문자열을 기준으로 정렬하며, 출력은 원래 정수로 한다.

    • 예를 들어 1은 "one", 11은 "one one", 21은 "two one"으로 변환해 사전순 비교한다.
    • M과 N은 1 이상 99 이하이므로 최대 99개의 수를 처리한다.
    • 10개 출력마다 줄 바꿈이 필요하며, 각 숫자는 공백으로 구분한다.

    전략 개요

    • 핵심 알고리즘: 커스텀 정렬 (사전순 문자열 비교)
    • 선택 이유: 각 수를 영어 문자열로 변환한 뒤, 해당 문자열을 기준으로 정렬하면 된다. 원래 정수와 변환된 문자열을 함께 저장하는 Node 클래스를 만들어 Comparable을 구현하면 Collections.sort()로 깔끔하게 처리할 수 있다.
    • 세부 전략:
      • M부터 N까지 각 수의 자릿수를 순서대로 영어 단어로 변환해 공백으로 이어붙인 문자열을 만든다.
      • 변환된 문자열과 원래 정수를 Node 객체로 묶어 리스트에 저장한다.
      • Node의 compareTo가 영어 문자열 기준 사전순 비교를 수행하도록 구현한다.
      • 정렬 후 원래 정수를 10개씩 줄 바꿔 출력한다.

    코드가 하는 일

    • 1) 영어 단어 변환 및 Node 리스트 구성 
      String[] english = {"zero", "one", "two", "three", "four",
                          "five", "six", "seven", "eight", "nine"};
      
      for (int i = M; i <= N; i++) {
          String s = String.valueOf(i);
          sb.setLength(0);
      
          for (int j = 0; j < s.length(); j++) {
              sb.append(english[s.charAt(j) - '0']).append(" ");
          }
      
          list.add(new Node(sb.toString().trim(), i));
      }
      • 0~9에 대응하는 영어 단어 배열 english를 준비한다.
      • 각 수 i를 문자열로 변환한 뒤, 자릿수 문자 s.charAt(j) - '0'을 인덱스로 사용해 영어 단어를 꺼내 공백으로 이어붙인다.
      • sb.toString().trim()으로 끝의 불필요한 공백을 제거하고, 원래 정수 i와 함께 Node 객체로 리스트에 추가한다.
      • 예시: i = 23이면 "two three", i = 9이면 "nine", i = 10이면 "one zero".
    • 2) 사전순 정렬 — Comparable 구현 
      @Override
      public int compareTo(Node o) {
          return this.eng.compareTo(o.eng);
      }
      • Node 클래스가 Comparable을 구현해 영어 문자열 기준 사전순 비교를 수행한다.
      • Java의 String.compareTo()는 유니코드 기준 사전순 비교를 수행하므로, 알파벳 순서에 따른 정렬이 자동으로 이루어진다.
      • Collections.sort(list) 호출만으로 사전순 정렬이 완료된다.
    • 3) 10개씩 줄 바꿔 출력 
      for (int i = 0; i < list.size(); i++) {
          System.out.print(list.get(i).num + " ");
          if ((i + 1) % 10 == 0) System.out.println();
      }
      • 정렬된 리스트를 순회하며 원래 정수 num을 공백으로 구분해 출력한다.
      • (i + 1) % 10 == 0일 때 줄 바꿈을 출력해 10개 단위로 행을 나눈다.
      • 마지막 행의 수가 10개 미만이면 줄 바꿈 없이 자연스럽게 종료된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K log K): K = N - M + 1은 처리할 수의 개수로 최대 99다. 문자열 변환은 O(K), 정렬은 O(K log K)이며, K가 최대 99이므로 사실상 O(1)에 수렴한다.
    • 공간 복잡도
      • O(K): K개의 Node 객체를 저장하는 리스트가 주요 메모리 사용처이며, 각 객체는 영어 문자열과 정수를 저장한다.
