    문제 14495 — 피보나치 비스무리한 수열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    피보나치 비스무리한 수열 (백준 14495)는
    점화식 f(n) = f(n-1) + f(n-3), f(1) = f(2) = f(3) = 1을 따르는 수열에서
    n번째 항의 값을 구하는 문제다.
    피보나치 수열과 유사하지만 두 칸이 아닌 세 칸 앞의 항을 더하는 점이 다르다.

    • n이 3 이하이면 초기값 1을 바로 반환한다.
    • n이 최대 100,000으로 크고 값이 빠르게 증가하므로 long 타입을 사용한다.
    • 각 항이 이전에 계산된 값에만 의존하므로 바텀업 DP가 적합하다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (바텀업 DP)
    • 선택 이유: f(n)은 f(n-1)과 f(n-3)에만 의존하므로 작은 값부터 순서대로 계산해 배열에 저장하면 중복 계산 없이 O(N)에 해결된다. 재귀로 구현하면 n이 최대 100,000일 때 스택 오버플로우가 발생할 수 있으므로 반복문 기반 DP를 사용한다.
    • 세부 전략:
      • n이 3 이하이면 즉시 1을 출력하고 종료한다.
      • dp[1] = dp[2] = dp[3] = 1로 초기값을 설정한다.
      • i = 4부터 n까지 dp[i] = dp[i-1] + dp[i-3]을 순서대로 계산한다.

    코드가 하는 일

    • 1) n ≤ 3 예외 처리 
      if (n <= 3) {
          System.out.println(1);
          return;
      }
      • n이 1, 2, 3이면 초기값 정의에 의해 f(n) = 1이므로 즉시 출력하고 종료한다.
      • 이 처리를 먼저 하지 않으면 이후 배열 접근 시 인덱스 오류가 발생할 수 있다.
    • 2) 초기값 설정 
      long[] dp = new long[n + 1];
      
      dp[1] = 1;
      dp[2] = 1;
      dp[3] = 1;
      • 크기 n + 1의 long 배열을 선언해 인덱스 1~n을 사용한다.
      • 수열이 빠르게 증가하므로 long 타입으로 선언한다.
      • 문제에서 정의된 초기값 f(1) = f(2) = f(3) = 1을 설정한다.
    • 3) 바텀업 DP로 수열 계산 
      for (int i = 4; i <= n; i++) {
          dp[i] = dp[i - 1] + dp[i - 3];
      }
      • i = 4부터 n까지 순서대로 점화식 dp[i] = dp[i-1] + dp[i-3]을 적용한다.
      • 배열을 앞에서부터 채우므로 dp[i-1]과 dp[i-3]은 항상 이미 계산된 상태임이 보장된다.
      • 예시: dp[4] = dp[3] + dp[1] = 2, dp[5] = dp[4] + dp[2] = 3, dp[6] = dp[5] + dp[3] = 4, dp[7] = dp[6] + dp[4] = 6.
    • 4) 결과 출력 
      System.out.println(dp[n]);
      • 계산이 완료된 dp[n]을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다. n이 최대 100,000이므로 충분히 빠르게 동작한다.
    • 공간 복잡도
      • O(N): f(1)~f(n)을 저장하는 dp 배열이 N + 1 크기를 사용한다. dp[i]는 dp[i-1]과 dp[i-3]만 참조하므로 변수 세 개로 최적화해 O(1)로 줄일 수 있지만, 가독성을 위해 배열로 구현했다.

    문제 1235 — 학생 번호

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    학생 번호 (백준 1235)는
    N명의 학생 번호가 주어질 때, 모든 학생을 구별할 수 있는 번호 뒷부분의 최소 자릿수를 구하는 문제다.
    번호의 뒷자리부터 1자리씩 늘려가며 모든 학생의 부분 문자열이 서로 다른 순간의 자릿수를 출력한다.

    • 모든 학생 번호의 길이는 동일하다.
    • 뒷자리 k자리만으로 N명을 모두 구별하려면, k자리 부분 문자열이 N개 모두 달라야 한다.
    • HashSet에 뒷자리 부분 문자열을 추가한 뒤 크기가 N과 같으면 중복이 없다는 의미이므로 그 k가 답이다.

    전략 개요

    • 핵심 알고리즘: 브루트포스 + HashSet 중복 검사
    • 선택 이유: 뒷자리를 1자리씩 늘려가며 각 자릿수에 대해 N개의 부분 문자열을 HashSet에 추가하면 중복 여부를 O(1)에 확인할 수 있다. 학생 번호의 최대 길이가 20이므로 최대 20번의 반복으로 충분하다.
    • 세부 전략:
      • 뒷자리 길이 i를 1부터 번호 길이까지 순서대로 증가시킨다.
      • 각 i에 대해 모든 학생 번호의 뒷 i자리 부분 문자열을 HashSet에 추가한다.
      • HashSet의 크기가 N과 같으면 모든 부분 문자열이 서로 다른 것이므로 즉시 i를 출력하고 종료한다.
      • 크기가 N보다 작으면 중복이 있는 것이므로 HashSet을 초기화하고 i를 1 증가시켜 다시 시도한다.

    코드가 하는 일

    • 1) 학생 번호 입력 
      String[] numbers = new String[N];
      
      for (int i = 0; i < N; i++) {
          numbers[i] = br.readLine();
      }
      • N개의 학생 번호를 문자열 배열에 저장한다. 모든 번호의 길이는 동일하므로 numbers[0].length()로 전체 자릿수를 참조한다.
    • 2) 뒷자리 길이를 1씩 늘려가며 중복 검사 
      for (int i = 1; i <= numbers[0].length(); i++) {
          for (int j = 0; j < N; j++) {
              set.add(numbers[j].substring(numbers[0].length() - i));
          }
      
          if (set.size() == N) {
              System.out.println(i);
              return;
          }
      
          set.clear();
      }
      • 외부 반복문의 i는 현재 확인 중인 뒷자리 자릿수다.
      • numbers[j].substring(numbers[0].length() - i)는 j번째 학생 번호의 뒷 i자리 부분 문자열을 추출한다. 예를 들어 번호가 "12345"이고 i = 2이면 "45"를 추출한다.
      • N개의 부분 문자열을 모두 HashSet에 추가한 뒤 크기를 확인한다. HashSet은 중복을 허용하지 않으므로, 크기가 N이면 모든 부분 문자열이 서로 다른 것이다.
      • 크기가 N보다 작으면 중복이 존재하므로 set.clear()로 초기화하고 다음 자릿수를 시도한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L × N): L은 학생 번호의 자릿수(최대 20), N은 학생 수(최대 1,000)다. 외부 반복문이 최대 L번, 내부 반복문이 N번 실행되며 각 반복에서 HashSet 삽입이 O(1)이므로 전체 O(L × N)이다.
    • 공간 복잡도
      • O(N): 각 자릿수 검사마다 HashSet에 최대 N개의 부분 문자열을 저장하며, 검사 후 초기화된다. 학생 번호를 저장하는 배열도 O(N × L)의 공간을 사용한다.
