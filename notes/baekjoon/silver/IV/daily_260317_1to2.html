    문제 20125 — 쿠키의 신체 측정

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    쿠키의 신체 측정 (백준 20125)는
    N × N 격자에 그려진 쿠키 캐릭터에서 심장의 위치와 왼팔·오른팔·허리·왼다리·오른다리의 길이를 구하는 문제다.
    심장은 가장 위쪽에 있는 별표('*')의 바로 아래 칸에 위치하며, 나머지 신체 부위는 심장을 기준으로 측정한다.

    • 심장의 위치를 먼저 찾은 뒤, 심장 행의 좌우로 팔, 심장 열의 아래로 허리, 허리 끝에서 좌우 대각선으로 다리를 측정한다.
    • 각 신체 부위는 연속된 '*'의 개수로 길이를 나타내며, '*'가 아닌 칸을 만나면 측정을 종료한다.
    • 다리는 허리 바로 아래 칸부터 시작하며, 왼다리는 col - 1 열, 오른다리는 col + 1 열에서 측정한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (선형 탐색)
    • 선택 이유: 신체 부위 구조가 규칙적으로 정의되어 있으므로, 심장 위치를 기준으로 각 방향을 선형 탐색하면 모든 길이를 O(N)에 측정할 수 있다. N이 최대 1,000이므로 완전 탐색으로 충분하다.
    • 세부 전략:
      • 격자를 위에서부터 순회해 첫 번째 '*'를 찾고, 그 바로 아래 칸을 심장 위치로 설정한다.
      • 심장 행(row)에서 col 기준 왼쪽으로 연속된 '*' 수를 세어 왼팔 길이를 구한다.
      • 심장 행(row)에서 col 기준 오른쪽으로 연속된 '*' 수를 세어 오른팔 길이를 구한다.
      • 심장 열(col)에서 row 아래로 연속된 '*' 수를 세어 허리 길이를 구한다.
      • 허리가 끝난 다음 행(row + waist + 1)부터 col - 1 열 아래로 왼다리, col + 1 열 아래로 오른다리 길이를 측정한다.

    코드가 하는 일

    • 1) 심장 위치 탐색 
      for (int r = 0; r < N; r++) {
          for (int c = 0; c < N; c++) {
              if (board[r][c] == '*') {
                  heartPos = new int[]{r + 1, c};
                  flag = true;
                  break;
              }
          }
          if (flag) break;
      }
      • 격자를 위에서부터 순회해 첫 번째 '*'를 찾는다. 이 '*'는 머리이므로, 심장은 그 바로 아래 행인 r + 1에 위치한다.
      • 찾은 즉시 flag를 true로 설정해 이중 반복문을 모두 탈출한다.
    • 2) 팔 길이 측정 
      for (int i = col - 1; i >= 0; i--) {
          if (board[row][i] == '*') { leftArm++; continue; }
          break;
      }
      
      for (int i = col + 1; i < N; i++) {
          if (board[row][i] == '*') { rightArm++; continue; }
          break;
      }
      • 심장 행(row)에서 col 왼쪽으로 연속된 '*' 수를 세어 왼팔 길이를 구한다.
      • col 오른쪽으로 연속된 '*' 수를 세어 오른팔 길이를 구한다.
      • '*'가 아닌 칸을 만나는 즉시 break해 측정을 종료한다.
    • 3) 허리 길이 측정 
      for (int i = row + 1; i < N; i++) {
          if (board[i][col] == '*') { waist++; continue; }
          break;
      }
      • 심장 열(col)에서 row 아래로 연속된 '*' 수를 세어 허리 길이를 구한다.
      • 허리가 끝나는 행 인덱스는 row + waist이며, 다리 측정의 시작점 계산에 사용된다.
    • 4) 다리 길이 측정 및 출력 
      for (int i = row + waist + 1; i < N; i++) {
          if (board[i][col - 1] == '*') { leftLeg++; continue; }
          break;
      }
      
      for (int i = row + waist + 1; i < N; i++) {
          if (board[i][col + 1] == '*') { rightLeg++; continue; }
          break;
      }
      
      System.out.println((row + 1) + " " + (col + 1));
      System.out.println(leftArm + " " + rightArm + " " + waist + " " + leftLeg + " " + rightLeg);
      • 허리가 끝난 다음 행(row + waist + 1)부터 왼다리(col - 1 열)와 오른다리(col + 1 열)의 연속된 '*' 수를 각각 측정한다.
      • 심장 위치는 0-based 인덱스이므로 출력 시 각각 1을 더해 1-based로 변환한다.
      • 순서대로 왼팔·오른팔·허리·왼다리·오른다리 길이를 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 심장 위치 탐색에 최악의 경우 O(N²), 각 신체 부위 측정은 최대 O(N)이므로 전체 시간 복잡도는 O(N²)이다. N이 최대 1,000이므로 최악의 경우 100만 번의 연산으로 충분하다.
    • 공간 복잡도
      • O(N²): 격자를 저장하는 2차원 배열이 N × N 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 1064 — 평행사변형

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    학생 번호 (백준 1235)는
    N명의 학생 번호가 주어질 때, 모든 학생을 구별할 수 있는 번호 뒷부분의 최소 자릿수를 구하는 문제다.
    번호의 뒷자리부터 1자리씩 늘려가며 모든 학생의 부분 문자열이 서로 다른 순간의 자릿수를 출력한다.

    • 모든 학생 번호의 길이는 동일하다.
    • 뒷자리 k자리만으로 N명을 모두 구별하려면, k자리 부분 문자열이 N개 모두 달라야 한다.
    • HashSet에 뒷자리 부분 문자열을 추가한 뒤 크기가 N과 같으면 중복이 없다는 의미이므로 그 k가 답이다.

    전략 개요

    • 핵심 알고리즘: 브루트포스 + HashSet 중복 검사
    • 선택 이유: 뒷자리를 1자리씩 늘려가며 각 자릿수에 대해 N개의 부분 문자열을 HashSet에 추가하면 중복 여부를 O(1)에 확인할 수 있다. 학생 번호의 최대 길이가 20이므로 최대 20번의 반복으로 충분하다.
    • 세부 전략:
      • 뒷자리 길이 i를 1부터 번호 길이까지 순서대로 증가시킨다.
      • 각 i에 대해 모든 학생 번호의 뒷 i자리 부분 문자열을 HashSet에 추가한다.
      • HashSet의 크기가 N과 같으면 모든 부분 문자열이 서로 다른 것이므로 즉시 i를 출력하고 종료한다.
      • 크기가 N보다 작으면 중복이 있는 것이므로 HashSet을 초기화하고 i를 1 증가시켜 다시 시도한다.

    코드가 하는 일

    • 1) 학생 번호 입력 
      String[] numbers = new String[N];
      
      for (int i = 0; i < N; i++) {
          numbers[i] = br.readLine();
      }
      • N개의 학생 번호를 문자열 배열에 저장한다. 모든 번호의 길이는 동일하므로 numbers[0].length()로 전체 자릿수를 참조한다.
    • 2) 뒷자리 길이를 1씩 늘려가며 중복 검사 
      for (int i = 1; i <= numbers[0].length(); i++) {
          for (int j = 0; j < N; j++) {
              set.add(numbers[j].substring(numbers[0].length() - i));
          }
      
          if (set.size() == N) {
              System.out.println(i);
              return;
          }
      
          set.clear();
      }
      • 외부 반복문의 i는 현재 확인 중인 뒷자리 자릿수다.
      • numbers[j].substring(numbers[0].length() - i)는 j번째 학생 번호의 뒷 i자리 부분 문자열을 추출한다. 예를 들어 번호가 "12345"이고 i = 2이면 "45"를 추출한다.
      • N개의 부분 문자열을 모두 HashSet에 추가한 뒤 크기를 확인한다. HashSet은 중복을 허용하지 않으므로, 크기가 N이면 모든 부분 문자열이 서로 다른 것이다.
      • 크기가 N보다 작으면 중복이 존재하므로 set.clear()로 초기화하고 다음 자릿수를 시도한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L × N): L은 학생 번호의 자릿수(최대 20), N은 학생 수(최대 1,000)다. 외부 반복문이 최대 L번, 내부 반복문이 N번 실행되며 각 반복에서 HashSet 삽입이 O(1)이므로 전체 O(L × N)이다.
    • 공간 복잡도
      • O(N): 각 자릿수 검사마다 HashSet에 최대 N개의 부분 문자열을 저장하며, 검사 후 초기화된다. 학생 번호를 저장하는 배열도 O(N × L)의 공간을 사용한다.
