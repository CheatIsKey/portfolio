    문제 20125 — 쿠키의 신체 측정

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    쿠키의 신체 측정 (백준 20125)는
    N × N 격자에 그려진 쿠키 캐릭터에서 심장의 위치와 왼팔·오른팔·허리·왼다리·오른다리의 길이를 구하는 문제다.
    심장은 가장 위쪽에 있는 별표('*')의 바로 아래 칸에 위치하며, 나머지 신체 부위는 심장을 기준으로 측정한다.

    • 심장의 위치를 먼저 찾은 뒤, 심장 행의 좌우로 팔, 심장 열의 아래로 허리, 허리 끝에서 좌우 대각선으로 다리를 측정한다.
    • 각 신체 부위는 연속된 '*'의 개수로 길이를 나타내며, '*'가 아닌 칸을 만나면 측정을 종료한다.
    • 다리는 허리 바로 아래 칸부터 시작하며, 왼다리는 col - 1 열, 오른다리는 col + 1 열에서 측정한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (선형 탐색)
    • 선택 이유: 신체 부위 구조가 규칙적으로 정의되어 있으므로, 심장 위치를 기준으로 각 방향을 선형 탐색하면 모든 길이를 O(N)에 측정할 수 있다. N이 최대 1,000이므로 완전 탐색으로 충분하다.
    • 세부 전략:
      • 격자를 위에서부터 순회해 첫 번째 '*'를 찾고, 그 바로 아래 칸을 심장 위치로 설정한다.
      • 심장 행(row)에서 col 기준 왼쪽으로 연속된 '*' 수를 세어 왼팔 길이를 구한다.
      • 심장 행(row)에서 col 기준 오른쪽으로 연속된 '*' 수를 세어 오른팔 길이를 구한다.
      • 심장 열(col)에서 row 아래로 연속된 '*' 수를 세어 허리 길이를 구한다.
      • 허리가 끝난 다음 행(row + waist + 1)부터 col - 1 열 아래로 왼다리, col + 1 열 아래로 오른다리 길이를 측정한다.

    코드가 하는 일

    • 1) 심장 위치 탐색 
      for (int r = 0; r < N; r++) {
          for (int c = 0; c < N; c++) {
              if (board[r][c] == '*') {
                  heartPos = new int[]{r + 1, c};
                  flag = true;
                  break;
              }
          }
          if (flag) break;
      }
      • 격자를 위에서부터 순회해 첫 번째 '*'를 찾는다. 이 '*'는 머리이므로, 심장은 그 바로 아래 행인 r + 1에 위치한다.
      • 찾은 즉시 flag를 true로 설정해 이중 반복문을 모두 탈출한다.
    • 2) 팔 길이 측정 
      for (int i = col - 1; i >= 0; i--) {
          if (board[row][i] == '*') { leftArm++; continue; }
          break;
      }
      
      for (int i = col + 1; i < N; i++) {
          if (board[row][i] == '*') { rightArm++; continue; }
          break;
      }
      • 심장 행(row)에서 col 왼쪽으로 연속된 '*' 수를 세어 왼팔 길이를 구한다.
      • col 오른쪽으로 연속된 '*' 수를 세어 오른팔 길이를 구한다.
      • '*'가 아닌 칸을 만나는 즉시 break해 측정을 종료한다.
    • 3) 허리 길이 측정 
      for (int i = row + 1; i < N; i++) {
          if (board[i][col] == '*') { waist++; continue; }
          break;
      }
      • 심장 열(col)에서 row 아래로 연속된 '*' 수를 세어 허리 길이를 구한다.
      • 허리가 끝나는 행 인덱스는 row + waist이며, 다리 측정의 시작점 계산에 사용된다.
    • 4) 다리 길이 측정 및 출력 
      for (int i = row + waist + 1; i < N; i++) {
          if (board[i][col - 1] == '*') { leftLeg++; continue; }
          break;
      }
      
      for (int i = row + waist + 1; i < N; i++) {
          if (board[i][col + 1] == '*') { rightLeg++; continue; }
          break;
      }
      
      System.out.println((row + 1) + " " + (col + 1));
      System.out.println(leftArm + " " + rightArm + " " + waist + " " + leftLeg + " " + rightLeg);
      • 허리가 끝난 다음 행(row + waist + 1)부터 왼다리(col - 1 열)와 오른다리(col + 1 열)의 연속된 '*' 수를 각각 측정한다.
      • 심장 위치는 0-based 인덱스이므로 출력 시 각각 1을 더해 1-based로 변환한다.
      • 순서대로 왼팔·오른팔·허리·왼다리·오른다리 길이를 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 심장 위치 탐색에 최악의 경우 O(N²), 각 신체 부위 측정은 최대 O(N)이므로 전체 시간 복잡도는 O(N²)이다. N이 최대 1,000이므로 최악의 경우 100만 번의 연산으로 충분하다.
    • 공간 복잡도
      • O(N²): 격자를 저장하는 2차원 배열이 N × N 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 1064 — 평행사변형

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    평행사변형 (백준 1064)는
    세 점 A, B, C가 주어질 때, 이 세 점 중 두 점을 한 변으로 삼아 만들 수 있는
    세 가지 평행사변형의 둘레 중 최댓값과 최솟값의 차이를 구하는 문제다.
    세 점이 일직선 위에 있으면 평행사변형을 만들 수 없으므로 -1.0을 출력한다.

    • 세 점 A, B, C로 만들 수 있는 평행사변형은 어떤 두 변을 서로 이웃하는 변으로 선택하느냐에 따라 3가지다.
    • AB와 AC를 이웃 변으로 하면 둘레 = 2 × (|AB| + |AC|), AB와 BC이면 2 × (|AB| + |BC|), AC와 BC이면 2 × (|AC| + |BC|)이다.
    • 세 점이 일직선이면 평행사변형이 존재하지 않으므로 기울기 비교로 공선성을 검사한다.

    전략 개요

    • 핵심 알고리즘: 기하 (두 점 사이의 거리 계산 + 경우의 수 열거)
    • 선택 이유: 세 점으로 만들 수 있는 평행사변형은 정확히 3가지이므로, 각 경우의 둘레를 직접 계산해 최댓값과 최솟값의 차를 구하는 완전 탐색이 가장 직관적이다.
    • 세부 전략:
      • 세 점이 일직선인지 벡터 외적(기울기 비교)으로 먼저 검사한다.
      • 세 변의 길이 lineA(|AB|), lineB(|AC|), lineC(|BC|)를 계산한다.
      • 3가지 평행사변형의 둘레를 각각 계산해 최댓값과 최솟값의 차를 출력한다.

    코드가 하는 일

    • 1) 공선성 검사 — 세 점이 일직선인지 판별 
      if ((yB - yA) * (xC - xB) == (yC - yB) * (xB - xA)) {
          System.out.println(-1.0);
          return;
      }
      • 벡터 AB의 기울기와 벡터 BC의 기울기가 같으면 세 점이 일직선 위에 있는 것이다.
      • 부동소수점 오차를 피하기 위해 기울기를 분수 형태로 비교한다. (yB-yA)/(xB-xA) == (yC-yB)/(xC-xB)를 교차곱으로 변환하면 (yB-yA)×(xC-xB) == (yC-yB)×(xB-xA)가 된다.
      • 공선이면 평행사변형을 만들 수 없으므로 -1.0을 출력하고 종료한다.
    • 2) 세 변의 길이 계산 
      double lineA = Math.sqrt(Math.pow(xB - xA, 2) + Math.pow(yB - yA, 2));
      double lineB = Math.sqrt(Math.pow(xC - xA, 2) + Math.pow(yC - yA, 2));
      double lineC = Math.sqrt(Math.pow(xC - xB, 2) + Math.pow(yC - yB, 2));
      • lineA는 |AB|, lineB는 |AC|, lineC는 |BC|를 의미한다.
      • 유클리드 거리 공식 √((x₂-x₁)² + (y₂-y₁)²)으로 각 변의 길이를 계산한다.
    • 3) 3가지 평행사변형 둘레 계산 및 차이 출력 
      double lengthA = 2 * (lineA + lineB);
      double lengthB = 2 * (lineA + lineC);
      double lengthC = 2 * (lineB + lineC);
      
      double maxLength = Math.max(lengthA, Math.max(lengthB, lengthC));
      double minLength = Math.min(lengthA, Math.min(lengthB, lengthC));
      
      System.out.println(maxLength - minLength);
      • lengthA는 AB와 AC를 이웃 변으로 하는 평행사변형의 둘레 = 2 × (|AB| + |AC|)이다.
      • lengthB는 AB와 BC를 이웃 변으로 하는 평행사변형의 둘레 = 2 × (|AB| + |BC|)이다.
      • lengthC는 AC와 BC를 이웃 변으로 하는 평행사변형의 둘레 = 2 × (|AC| + |BC|)이다.
      • 세 둘레 중 최댓값과 최솟값의 차를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 입력을 읽은 뒤 상수 번의 산술 연산만 수행한다. 입력 크기와 무관하게 항상 일정한 수의 연산으로 처리된다.
    • 공간 복잡도
      • O(1): 좌표 6개와 길이·둘레 변수 몇 개만 사용하므로 입력 크기와 무관한 상수 공간이다.
