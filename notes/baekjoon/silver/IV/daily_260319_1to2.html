    문제 1358 — 하키

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    하키 (백준 1358)는
    하키 경기장이 직사각형 영역과 양쪽 끝에 붙은 두 반원으로 구성될 때,
    P명의 선수 중 경기장 안에 있는 선수의 수를 구하는 문제다.
    경계선 위에 있는 선수도 경기장 안에 있는 것으로 간주한다.

    • 경기장은 좌측 상단 좌표 (X, Y), 너비 W, 높이 H인 직사각형과, 왼쪽과 오른쪽 끝에 붙은 반원으로 구성된다.
    • 반원의 반지름은 H / 2이며, 왼쪽 반원의 중심은 (X, Y + H/2), 오른쪽 반원의 중심은 (X + W, Y + H/2)이다.
    • 각 선수의 좌표가 직사각형 내부 또는 두 반원 중 하나의 내부에 있으면 경기장 안에 있는 것이다.

    전략 개요

    • 핵심 알고리즘: 기하 (직사각형 내부 판별 + 원 내부 판별)
    • 선택 이유: 경기장이 직사각형과 두 반원의 합집합으로 구성되므로, 각 선수에 대해 세 영역 중 하나라도 포함되면 카운트하면 된다. 각 판별이 O(1)이므로 P명에 대해 O(P)에 해결된다.
    • 세부 전략:
      • 직사각형 내부 판별: X ≤ x ≤ X+W이고 Y ≤ y ≤ Y+H이면 내부다.
      • 왼쪽 반원 판별: 중심 (X, Y + radius)에서 선수까지의 거리가 radius 이하이면 내부다.
      • 오른쪽 반원 판별: 중심 (X+W, Y + radius)에서 선수까지의 거리가 radius 이하이면 내부다.
      • 부동소수점 오차를 피하기 위해 거리 비교 시 제곱 형태로 비교한다.

    코드가 하는 일

    • 1) 입력 및 반지름 계산 
      int W = Integer.parseInt(st.nextToken());
      int H = Integer.parseInt(st.nextToken());
      int X = Integer.parseInt(st.nextToken());
      int Y = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      int radius = H / 2;
      • 경기장의 너비 W, 높이 H, 좌측 상단 좌표 (X, Y), 선수 수 P를 입력받는다.
      • 반원의 반지름은 직사각형 높이의 절반인 H / 2이다.
      • 두 반원의 중심 y좌표는 Y + radius이며, 왼쪽 반원의 중심 x는 X, 오른쪽은 X + W이다.
    • 2) 직사각형 내부 판별 
      if ((X <= x && x <= X + W) && (Y <= y && y <= Y + H)) {
          answer++;
          continue;
      }
      • 선수의 x좌표가 X 이상 X+W 이하이고 y좌표가 Y 이상 Y+H 이하이면 직사각형 내부에 있다.
      • 경계선 위도 포함이므로 부등호에 등호가 포함된다.
      • 조건을 만족하면 즉시 카운트를 증가시키고 다음 선수로 넘어간다.
    • 3) 왼쪽·오른쪽 반원 내부 판별 
      if (Math.pow(X - x, 2) + Math.pow(Y + radius - y, 2) <= Math.pow(radius, 2)) {
          answer++;
          continue;
      }
      
      if (Math.pow(X + W - x, 2) + Math.pow(Y + radius - y, 2) <= Math.pow(radius, 2)) {
          answer++;
      }
      • 왼쪽 반원의 중심 (X, Y + radius)에서 선수 (x, y)까지의 거리의 제곱이 radius² 이하이면 내부다.
      • 오른쪽 반원의 중심 (X + W, Y + radius)에 대해 동일하게 판별한다.
      • 제곱 형태로 비교하면 sqrt()를 사용하지 않아 부동소수점 오차를 줄일 수 있다.
      • 직사각형 내부에서 이미 카운트된 선수는 continue로 건너뛰므로 중복 카운트가 발생하지 않는다.
    • 4) 결과 출력 
      System.out.println(answer);
      • 경기장 안에 있는 선수의 총 수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(P): P명의 선수 각각에 대해 세 가지 판별을 수행하며, 각 판별은 O(1)이다. 전체 시간 복잡도는 O(P)이다.
    • 공간 복잡도
      • O(1): 선수 좌표를 배열에 저장하지 않고 입력받는 즉시 판별하므로 입력 크기와 무관한 상수 공간만 사용한다.

    문제 1246 — 온라인 판매

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    온라인 판매 (백준 1246)는
    N개의 물건을 M명의 고객에게 판매할 때, 총 수익을 최대화하는 판매 가격과 그 수익을 구하는 문제다.
    모든 고객에게 동일한 가격으로 판매하며, 가격 이상을 지불할 의사가 있는 고객에게만 판매된다.

    • 판매 가격을 price로 설정하면, price 이상을 지불할 의사가 있는 고객 수만큼 판매된다.
    • 실제 판매 수량은 재고 N과 구매 가능 고객 수 중 작은 값이다.
    • 가능한 판매 가격은 고객들의 지불 의사 금액 중 하나로 설정하는 것이 최적이므로, 고객 배열을 정렬 후 각 값을 가격으로 시도하면 된다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 그리디 (가격 후보 순회)
    • 선택 이유: 최적 판매 가격은 반드시 어떤 고객의 지불 의사 금액과 같다. 고객 배열을 오름차순 정렬하면, i번째 고객의 금액을 가격으로 설정할 때 구매 가능한 고객 수는 정확히 M - i명이다. 모든 후보 가격을 O(M)에 순회해 최대 수익을 찾을 수 있다.
    • 세부 전략:
      • 고객 배열을 오름차순 정렬한다.
      • i번째 고객의 금액을 판매 가격으로 설정하면, i번째 이상의 고객 M - i명이 구매 가능하다.
      • 실제 판매 수량은 재고 N과 구매 가능 고객 수 M - i 중 작은 값이다.
      • 수익 = 실제 판매 수량 × 가격을 계산해 현재 최대 수익보다 크면 갱신한다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] customers = new int[M];
      for (int i = 0; i < M; i++) {
          customers[i] = Integer.parseInt(br.readLine());
      }
      Arrays.sort(customers);
      • M명의 고객 지불 의사 금액을 배열에 저장하고 오름차순 정렬한다.
      • 정렬 후 인덱스 i에서의 고객 수는 i번째 가격 이상을 낼 의향이 있는 고객 수인 M - i와 바로 대응된다.
    • 2) 가격 후보별 수익 계산 및 최댓값 갱신 
      for (int i = 0; i < M; i++) {
          int price = customers[i];
          int count = M - i;
      
          int available = Math.min(N, count);
          int totalProfit = available * price;
      
          if (totalProfit > profit) {
              profit = totalProfit;
              money = price;
          }
      }
      • i번째 고객의 금액을 판매 가격 price로 설정한다.
      • 배열이 오름차순이므로 인덱스 i 이상의 고객 M - i명이 이 가격에 구매할 의향이 있다.
      • 실제 판매 수량 available = Math.min(N, count)은 재고와 구매 가능 고객 수 중 작은 값이다.
      • 총 수익 totalProfit = available × price가 현재 최대 수익보다 클 때만 갱신한다. >를 사용하므로 동률인 경우 더 낮은 가격(먼저 등장한 값)이 유지된다. 동일한 수익이면 더 낮은 가격을 출력해야 하기 때문이다.
      • 예시: customers = [1, 3, 5], N = 2이면 price=1일 때 2×1=2, price=3일 때 2×3=6, price=5일 때 1×5=5. 최대 수익은 6, 가격은 3.
    • 3) 결과 출력 
      System.out.println(money + " " + profit);
      • 최대 수익을 내는 판매 가격 money와 그 수익 profit을 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M log M): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 선형 순회는 O(M)이므로 전체 O(M log M)이다.
    • 공간 복잡도
      • O(M): 고객 지불 의사 금액을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
