    문제 15970 — 화살표 그리기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    화살표 그리기 (백준 15970)는
    N개의 점이 각각 색깔과 위치를 가질 때, 각 점에서 같은 색깔의 가장 가까운 점으로 향하는 화살표의 길이 합을 구하는 문제다.
    각 점은 반드시 같은 색의 다른 점 중 가장 가까운 점을 향해 화살표를 그려야 한다.

    • 같은 색의 점들을 위치 기준으로 오름차순 정렬하면, 각 점에서 가장 가까운 같은 색 점은 바로 왼쪽 또는 바로 오른쪽 이웃이다.
    • 맨 왼쪽 점은 오른쪽 이웃만, 맨 오른쪽 점은 왼쪽 이웃만 선택할 수 있다.
    • 중간 점은 왼쪽과 오른쪽 이웃 중 더 가까운 쪽을 선택한다.

    전략 개요

    • 핵심 알고리즘: 색깔별 그룹화 + 정렬 + 그리디
    • 선택 이유: 같은 색의 점들을 위치 기준으로 정렬하면, 각 점의 가장 가까운 같은 색 점은 반드시 정렬된 배열에서 인접한 원소다. 이를 이용하면 O(N log N) 정렬 후 O(N) 선형 탐색으로 전체 합을 구할 수 있다.
    • 세부 전략:
      • 색깔을 인덱스로 하는 리스트 배열에 각 점의 위치를 저장한다.
      • 각 색깔 그룹을 오름차순 정렬한다.
      • 각 그룹에서 맨 왼쪽 점은 오른쪽 이웃과의 거리, 맨 오른쪽 점은 왼쪽 이웃과의 거리, 중간 점은 두 이웃 중 더 가까운 거리를 누적한다.

    코드가 하는 일

    • 1) 색깔별 위치 그룹화 
      List[] colors = new ArrayList[N + 1];
      for (int i = 0; i <= N; i++) {
          colors[i] = new ArrayList<>();
      }
      
      for (int i = 0; i < N; i++) {
          int x = Integer.parseInt(st.nextToken());
          int y = Integer.parseInt(st.nextToken());
          colors[y].add(x);
      }
      • 색깔 번호를 인덱스로 하는 리스트 배열 colors를 생성한다.
      • 각 점의 위치 x를 해당 색깔 y의 리스트에 추가한다.
    • 2) 각 색깔 그룹 정렬 
      for (int i = 0; i <= N; i++) {
          if (colors[i].size() < 2) continue;
          colors[i].sort(Comparator.comparing(Integer::intValue));
      }
      • 같은 색의 점이 2개 미만이면 화살표를 그릴 수 없으므로 건너뛴다.
      • 2개 이상인 그룹만 오름차순 정렬한다. 정렬 후 인접한 원소가 각 점의 가장 가까운 같은 색 점 후보가 된다.
    • 3) 각 점에서 가장 가까운 같은 색 점까지의 거리 누적 
      for (List color : colors) {
          if (color.size() < 2) continue;
      
          for (int i = 0; i < color.size(); i++) {
              if (i == 0) {
                  answer += color.get(i + 1) - color.get(i);
              } else if (i == color.size() - 1) {
                  answer += Math.abs(color.get(i - 1) - color.get(i));
              } else {
                  answer += Math.min(
                      Math.abs(color.get(i - 1) - color.get(i)),
                      color.get(i + 1) - color.get(i)
                  );
              }
          }
      }
      • 맨 왼쪽 점(i == 0): 오른쪽 이웃만 존재하므로 color.get(i+1) - color.get(i)를 더한다.
      • 맨 오른쪽 점(i == color.size() - 1): 왼쪽 이웃만 존재하므로 color.get(i) - color.get(i-1)을 더한다.
      • 중간 점: 왼쪽 이웃과의 거리와 오른쪽 이웃과의 거리 중 더 작은 값을 더한다. 정렬된 배열이므로 오른쪽 이웃과의 거리는 항상 양수이지만, 일관성을 위해 Math.abs()를 사용했다.
      • 예시: 같은 색의 점 위치가 [1, 5, 8]이면 1→(5-1=4), 5→min(4, 3)=3, 8→(8-5=3). 합계 = 10.
    • 4) 결과 출력 
      System.out.println(answer);
      • 모든 점에서 그린 화살표 길이의 합 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 각 색깔 그룹을 정렬하는 비용의 합이 전체 점의 수 N에 대한 정렬 비용 O(N log N)과 같다. 이후 각 점을 한 번씩 순회하는 비용은 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): 색깔별 리스트 배열에 N개의 점 위치를 저장하므로 전체 O(N)의 공간을 사용한다.

    문제 15828 — Router

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Router (백준 15828)는
    버퍼 크기가 N인 라우터를 시뮬레이션할 때, 패킷 수신·처리 명령을 순서대로 처리한 후 버퍼에 남아있는 패킷을 출력하는 문제다.
    양수는 패킷 수신, 0은 패킷 처리(앞에서 제거), -1은 시뮬레이션 종료를 의미한다.

    • 버퍼는 FIFO(선입선출) 구조이므로 큐를 사용한다.
    • 버퍼가 가득 찬 상태에서 새 패킷이 들어오면 해당 패킷은 버려진다.
    • 버퍼가 비어있는 상태에서 처리 명령(0)이 들어오면 아무 일도 일어나지 않는다.
    • 시뮬레이션 종료 후 버퍼가 비어있으면 "empty"를 출력한다.

    전략 개요

    • 핵심 알고리즘: 큐(Queue) 시뮬레이션
    • 선택 이유: 라우터 버퍼는 먼저 들어온 패킷이 먼저 처리되는 FIFO 구조이므로 ArrayDeque가 가장 적합하다. 삽입(addLast)과 삭제(pollFirst)가 모두 O(1)에 처리된다.
    • 세부 전략:
      • 입력이 양수이면 버퍼 크기가 N 미만일 때만 큐의 뒤에 추가한다. 가득 찬 경우 패킷을 버린다.
      • 입력이 0이면 큐가 비어있지 않을 때 큐의 앞에서 패킷을 제거한다.
      • 입력이 -1이면 반복을 종료하고 큐의 현재 상태를 출력한다.

    코드가 하는 일

    • 1) 버퍼 초기화 
      int N = Integer.parseInt(br.readLine());
      Deque queue = new ArrayDeque<>(N);
      • 버퍼 크기 N을 읽고, 최대 N개의 패킷을 저장할 수 있는 ArrayDeque를 생성한다.
      • 초기 용량을 N으로 지정해 불필요한 재할당을 방지한다.
    • 2) 패킷 수신 및 처리 시뮬레이션 
      while (true) {
          int input = Integer.parseInt(br.readLine());
          if (input == -1) break;
      
          if (input == 0) {
              if (!queue.isEmpty()) {
                  queue.pollFirst();
              }
          } else {
              if (queue.size() < N) {
                  queue.addLast(input);
              }
          }
      }
      • 입력이 -1이면 즉시 반복을 종료한다.
      • 입력이 0이면 패킷 처리 명령이다. 큐가 비어있지 않을 때만 pollFirst()로 가장 먼저 들어온 패킷을 제거한다. 비어있으면 아무 동작도 하지 않는다.
      • 양수이면 새 패킷이 들어온 것이다. queue.size() < N이면 버퍼에 여유가 있으므로 addLast()로 큐의 뒤에 추가한다. 버퍼가 가득 찬 경우(queue.size() == N)에는 패킷을 버린다.
    • 3) 결과 출력 
      if (queue.isEmpty()) {
          System.out.println("empty");
          return;
      }
      
      for (int num : queue) {
          sb.append(num).append(" ");
      }
      System.out.println(sb);
      • 시뮬레이션 종료 후 큐가 비어있으면 "empty"를 출력한다.
      • 큐에 패킷이 남아있으면 앞에서부터 순서대로 공백으로 구분해 출력한다.
      • 결과를 StringBuilder에 누적해 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(Q): Q는 입력 명령의 수다. 각 명령에 대해 addLast()와 pollFirst()가 모두 O(1)이므로 전체 O(Q)이다.
    • 공간 복잡도
      • O(N): 버퍼에 저장되는 패킷의 수가 최대 N이므로 큐가 O(N)의 공간을 사용한다.
