    문제 13015 — 별 찍기 - 23

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    별 찍기 - 23 (백준 13015)는
    N이 주어질 때 두 개의 마름모(X자 형태)가 가운데 꼭짓점에서 맞닿은 패턴을 출력하는 문제다.
    전체 행은 2N - 1행, 전체 열은 4N - 3열이며, 각 행에 최대 4개의 별이 찍힌다.

    • 첫 행과 마지막 행에는 왼쪽 N개, 오른쪽 N개의 별이 연속으로 찍힌다.
    • 중간 행에서는 행 인덱스를 기준으로 양쪽 마름모의 경계에 해당하는 4개의 위치에만 별이 찍힌다.
    • 각 행 끝의 불필요한 공백은 제거하고 출력한다.

    전략 개요

    • 핵심 알고리즘: 수학적 패턴 분석 + 행별 별 위치 직접 계산
    • 선택 이유: 각 행에서 별이 찍히는 위치가 행 인덱스 i에 대한 수식으로 정확히 결정되므로, 전체 배열을 공백으로 초기화한 뒤 해당 위치만 별로 덮어씌우는 방식이 간결하고 명확하다.
    • 세부 전략:
      • 첫 행(i=0)과 마지막 행(i=2N-2)은 0~N-1 구간과 3N-3~4N-4 구간에 별을 연속으로 채운다.
      • 중간 행에서는 r = min(i, 2N-2-i)를 계산해 현재 행이 꼭짓점에서 얼마나 떨어져 있는지를 구한다.
      • 왼쪽 마름모의 두 꼭짓점 r과 r + N - 1, 오른쪽 마름모의 두 꼭짓점 3N-3-r과 4N-4-r에 별을 찍는다.
      • 각 행 출력 시 끝 공백을 제거해 불필요한 출력을 방지한다.

    코드가 하는 일

    • 1) 배열 크기 계산 및 공백 초기화 
      int totalRows = 2 * N - 1;
      int totalCols = 4 * N - 3;
      
      char[] line = new char[totalCols];
      Arrays.fill(line, ' ');
      • 전체 행 수는 2N - 1, 열 수는 4N - 3이다. N = 3이면 5행 9열 패턴이 된다.
      • 각 행마다 char 배열을 공백으로 초기화한 뒤 별이 찍힐 위치만 덮어씌운다.
    • 2) 첫 행·마지막 행 처리 
      if (i == 0 || i == 2 * N - 2) {
          for (int j = 0; j < N; j++)
              line[j] = '*';
          for (int j = 3 * N - 3; j <= 4 * N - 4; j++)
              line[j] = '*';
      }
      • 첫 행과 마지막 행은 두 마름모의 가장 넓은 부분으로, 왼쪽 N개(0~N-1)와 오른쪽 N개(3N-3~4N-4)에 별을 연속으로 채운다.
      • 예시: N = 3이면 첫 행은 열 0~2와 6~8에 별. "*** ***" 형태.
    • 3) 중간 행 처리 — r 값 기반 4개 위치 계산 
      } else {
          int r = Math.min(i, 2 * N - 2 - i);
          line[r] = '*';
          line[r + N - 1] = '*';
          line[3 * N - 3 - r] = '*';
          line[4 * N - 4 - r] = '*';
      }
      • r = min(i, 2N-2-i)는 현재 행이 꼭짓점(가운데 행)에서 얼마나 가까운지를 나타낸다. 위쪽 절반에서는 i가 커질수록 r이 커지고, 아래쪽 절반에서는 반대로 작아진다.
      • 왼쪽 마름모의 왼쪽 경계: line[r], 오른쪽 경계: line[r + N - 1].
      • 오른쪽 마름모의 왼쪽 경계: line[3N-3-r], 오른쪽 경계: line[4N-4-r].
      • 예시: N = 3, i = 1이면 r = 1. 별 위치: 1, 3, 5, 7. " * * * *" (1-based).
    • 4) 끝 공백 제거 및 출력 
      int last = totalCols - 1;
      while (last >= 0 && line[last] == ' ') last--;
      sb.append(new String(line, 0, last + 1)).append('\n');
      • 배열 뒤에서부터 공백이 아닌 마지막 문자를 찾아 그 위치까지만 문자열로 변환해 출력한다.
      • 불필요한 후행 공백을 제거해 출력 형식을 맞춘다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N²): 전체 행 수는 2N-1이며, 각 행 처리에서 배열 초기화와 끝 공백 제거가 O(N)이므로 전체 O(N²)이다.
    • 공간 복잡도
      • O(N): 각 행을 처리하는 char 배열이 최대 4N-3 크기를 사용하며, 결과를 누적하는 StringBuilder는 전체 출력 크기에 비례하는 O(N²) 공간을 사용한다.

    문제 20044 — Project Teams

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Project Teams (백준 20044)는
    2N명의 학생을 N개의 2인 팀으로 나눌 때, 각 팀의 두 멤버 능력치 합 중 최솟값을 최대화하는 문제다.
    팀 구성은 자유롭게 할 수 있으며, 가장 약한 팀의 합이 최대가 되도록 팀을 구성해야 한다.

    • 팀 합의 최솟값을 최대화하는 것이 목표이므로, 가장 약한 팀이 최대한 강해지도록 팀을 구성해야 한다.
    • 능력치를 오름차순 정렬한 뒤 가장 작은 값과 가장 큰 값을 짝짓는 방식이 최적임을 그리디하게 증명할 수 있다.
    • 이 방식으로 구성한 N개의 팀 합 중 최솟값이 답이 된다.

    전략 개요

    • 핵심 알고리즘: 그리디 (정렬 후 양 끝 짝짓기)
    • 선택 이유: 가장 작은 값은 어떤 팀에 배치되든 그 팀의 합을 제한한다. 가장 작은 값에 가장 큰 값을 짝짓는 것이 최솟값을 최대화하는 데 유리하다. 만약 가장 작은 값에 큰 값 대신 중간 값을 짝지으면, 남는 큰 값끼리 팀을 이루어 자원이 낭비되고 전체 최솟값이 작아진다.
    • 세부 전략:
      • 학생 능력치를 오름차순 정렬한다.
      • 인덱스 i(0부터 n-1)와 인덱스 size - 1 - i를 짝지어 N개의 팀을 구성한다. 즉, 1번째와 2N번째, 2번째와 (2N-1)번째, ...를 짝짓는다.
      • N개의 팀 합 중 최솟값을 출력한다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int size = 2 * n;
      int[] students = new int[size];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < size; i++) {
          students[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(students);
      • 2N명의 능력치를 배열에 저장하고 오름차순 정렬한다.
      • 정렬 후 인덱스 0이 가장 작은 값, 인덱스 size-1이 가장 큰 값이 된다.
    • 2) 양 끝 짝짓기로 팀 합의 최솟값 계산 
      int answer = Integer.MAX_VALUE;
      
      for (int i = 0; i < n; i++) {
          answer = Math.min(answer, students[i] + students[size - 1 - i]);
      }
      • i = 0부터 n-1까지 순회하며 students[i]와 students[size-1-i]를 짝지어 팀 합을 계산한다.
      • i = 0이면 가장 작은 값과 가장 큰 값의 합, i = 1이면 두 번째로 작은 값과 두 번째로 큰 값의 합, ...의 방식으로 N개의 팀 합이 계산된다.
      • N개의 팀 합 중 최솟값을 answer에 유지한다.
      • 예시: students = [1, 2, 3, 4], n = 2이면 팀1: 1+4=5, 팀2: 2+3=5. answer = 5. students = [1, 2, 3, 6], n = 2이면 팀1: 1+6=7, 팀2: 2+3=5. answer = 5.
    • 3) 결과 출력 
      System.out.println(answer);
      • 팀 합의 최솟값의 최댓값 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 N번의 선형 순회는 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): 2N명의 능력치를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
