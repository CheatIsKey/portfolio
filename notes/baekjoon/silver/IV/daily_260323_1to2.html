    문제 1331 — 나이트 투어

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    나이트 투어 (백준 1331)는
    6 × 6 체스판에서 나이트가 36칸을 모두 정확히 한 번씩 방문하고
    마지막 위치에서 시작 위치로 나이트 이동이 가능한지 검증하는 문제다.
    모든 조건을 만족하면 "Valid", 하나라도 위반하면 "Invalid"를 출력한다.

    • 나이트의 이동은 (±1, ±2) 또는 (±2, ±1)의 8가지 방향이다.
    • 연속된 두 위치 사이의 행 차이와 열 차이가 나이트 이동 규칙을 만족해야 한다.
    • 이미 방문한 칸을 다시 방문하면 Invalid이며, 마지막 칸에서 시작 칸으로 나이트 이동이 불가능해도 Invalid다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (이동 규칙 검증 + 방문 체크)
    • 선택 이유: 36개의 위치를 순서대로 읽으며 연속된 두 위치 간의 나이트 이동 유효성과 중복 방문 여부를 검사하면 된다. 입력이 고정 크기(36개)이므로 O(1)에 수렴하는 간단한 시뮬레이션으로 해결된다.
    • 세부 전략:
      • 첫 번째 위치를 시작점으로 방문 표시한다.
      • 나머지 35개의 위치를 순서대로 읽으며 이전 위치에서 현재 위치로의 이동이 나이트 규칙을 만족하는지, 이미 방문한 칸인지 검사한다.
      • 모든 이동이 유효하면 마지막 위치에서 시작 위치로 나이트 이동이 가능한지 검사한다.

    코드가 하는 일

    • 1) 시작 위치 입력 및 초기화 
      int startPosCol = line.charAt(0) - 'A';
      int startPosRow = line.charAt(1) - '0' - 1;
      
      board[startPosRow][startPosCol] = true;
      • 첫 번째 위치를 읽어 열은 'A'를 빼서 0~5 인덱스로, 행은 '0'을 빼고 1을 빼서 0~5 인덱스로 변환한다.
      • 시작 위치를 방문 표시하고, 이후 마지막 위치에서의 복귀 검사에 사용하기 위해 저장해둔다.
    • 2) 연속된 위치 간 나이트 이동 검증 
      for (int i = 0; i < 35; i++) {
          currPosCol = c - 'A';
          currPosRow = r - '0' - 1;
      
          if ((Math.abs(prevPosRow - currPosRow) == 2 && Math.abs(prevPosCol - currPosCol) == 1
                  || Math.abs(prevPosRow - currPosRow) == 1 && Math.abs(prevPosCol - currPosCol) == 2)
                  && !board[currPosRow][currPosCol]) {
      
              board[currPosRow][currPosCol] = true;
              prevPosCol = currPosCol;
              prevPosRow = currPosRow;
              continue;
          }
      
          System.out.println("Invalid");
          return;
      }
      • 나머지 35개의 위치를 순서대로 읽으며 이전 위치(prev)와 현재 위치(curr)의 행·열 차이가 나이트 이동 규칙을 만족하는지 확인한다.
      • 행 차이가 2이고 열 차이가 1, 또는 행 차이가 1이고 열 차이가 2인 경우만 유효한 나이트 이동이다.
      • !board[currPosRow][currPosCol]로 이미 방문한 칸인지 동시에 검사한다. 두 조건 중 하나라도 위반이면 즉시 "Invalid"를 출력하고 종료한다.
      • 유효한 이동이면 방문 표시를 하고 prev를 현재 위치로 갱신한다.
    • 3) 마지막 위치에서 시작 위치로의 복귀 검사 
      for (int k = 0; k < ROW.length; k++) {
          int row = prevPosRow + ROW[k];
          int col = prevPosCol + COL[k];
      
          if (row < 0 || row >= 6 || col < 0 || col >= 6) continue;
          if (startPosRow == row && startPosCol == col) {
              System.out.println("Valid");
              return;
          }
      }
      
      System.out.println("Invalid");
      • 마지막 위치에서 나이트가 이동할 수 있는 8방향을 모두 시도해, 그 중 시작 위치와 일치하는 경우가 있으면 "Valid"를 출력하고 종료한다.
      • 8방향 모두 시작 위치로 이동할 수 없으면 "Invalid"를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 체스판 크기(6 × 6 = 36칸)가 고정되어 있으므로 모든 연산이 상수 횟수만큼 수행된다. 35번의 이동 검증과 최대 8번의 복귀 검사 모두 상수 시간이다.
    • 공간 복잡도
      • O(1): 방문 여부를 저장하는 boolean[6][6] 배열이 고정 크기를 사용하므로 입력 크기와 무관한 상수 공간이다.

    문제 13900 — 순서쌍의 곱의 합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    순서쌍의 곱의 합 (백준 13900)는
    N개의 수가 주어질 때, i < j를 만족하는 모든 순서쌍 (i, j)에 대해 A[i] × A[j]의 합을 구하는 문제다.
    N이 최대 100,000이므로 브루트포스 O(N²) 대신 수학적 관계를 이용한 O(N) 풀이가 필요하다.

    • 모든 순서쌍을 직접 구하면 최대 약 50억 번의 연산이 필요해 시간 초과가 발생한다.
    • i번째 원소 A[i]에 대해, A[i]보다 뒤에 있는 모든 원소의 합을 곱하면 A[i]가 관여하는 모든 쌍의 곱의 합을 O(1)에 구할 수 있다.
    • 수의 크기가 최대 100,000이고 N이 최대 100,000이므로 곱의 합이 long 범위를 초과하지 않는지 확인해야 한다.

    전략 개요

    • 핵심 알고리즘: 수학적 최적화 (누적 합 활용)
    • 선택 이유: A[i] × (A[i+1] + A[i+2] + ... + A[N-1])을 모든 i에 대해 계산하면 모든 순서쌍의 곱의 합과 동일하다. 전체 합에서 A[i]를 제거해 나가며 남은 합을 곱하면 O(N)에 해결된다.
    • 세부 전략:
      • 먼저 모든 원소의 합 totalSum을 구한다.
      • 앞에서부터 순회하며 현재 원소 A[i]를 totalSum에서 빼면, totalSum은 A[i+1] ~ A[N-1]의 합이 된다.
      • 이 값에 A[i]를 곱해 answer에 누적한다. 이것이 A[i]가 왼쪽에 오는 모든 쌍의 곱의 합이다.

    코드가 하는 일

    • 1) 입력 및 전체 합 계산 
      long totalSum = 0;
      int[] numbers = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          int num = Integer.parseInt(st.nextToken());
          numbers[i] = num;
          totalSum += num;
      }
      • N개의 수를 배열에 저장하면서 전체 합 totalSum을 동시에 계산한다.
      • totalSum은 이후 각 원소를 제거해 나가며 오른쪽 부분 합을 유지하는 데 사용된다.
    • 2) 순서쌍 곱의 합 계산 
      for (int i = 0; i < N; i++) {
          totalSum -= numbers[i];
          answer += totalSum * numbers[i];
      }
      • i번째 원소를 totalSum에서 먼저 빼면, totalSum은 A[i+1] + A[i+2] + ... + A[N-1]이 된다.
      • totalSum × numbers[i]는 A[i]가 왼쪽 원소가 되는 모든 쌍 (A[i], A[j]) (j > i)의 곱의 합과 동일하다.
      • 예시: numbers = [1, 2, 3]이면 i=0에서 totalSum=5, answer += 1×5=5. i=1에서 totalSum=3, answer += 2×3=6. i=2에서 totalSum=0, answer += 3×0=0. 최종 answer = 11. (1×2 + 1×3 + 2×3 = 2 + 3 + 6 = 11과 일치)
    • 3) 결과 출력 
      System.out.println(answer);
      • 누적된 모든 순서쌍의 곱의 합 answer를 출력한다. 값이 int 범위를 초과할 수 있으므로 long 타입으로 선언했다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 입력을 읽으며 전체 합을 계산하는 O(N)과, 순서쌍 곱의 합을 계산하는 O(N)으로 전체 O(N)이다. 브루트포스 O(N²) 대비 획기적으로 개선된 시간 복잡도다.
    • 공간 복잡도
      • O(N): N개의 원소를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
