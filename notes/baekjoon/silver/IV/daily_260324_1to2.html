    문제 10546 — 배부른 마라토너

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    배부른 마라토너 (백준 10546)는
    N명이 마라톤에 참가했고 N-1명이 완주했을 때, 완주하지 못한 단 한 명의 이름을 찾는 문제다.
    동명이인이 존재할 수 있으므로 이름의 등장 횟수를 기반으로 판별해야 한다.

    • 참가자 N명과 완주자 N-1명의 이름이 순서 없이 주어지며, 총 2N-1개의 이름을 읽게 된다.
    • 완주한 사람의 이름은 참가자 목록에 짝수 번(참가 1회 + 완주 1회 = 2회) 등장하고, 완주하지 못한 사람은 홀수 번(참가 1회만 = 1회) 등장한다.
    • 동명이인이 있는 경우에도 완주한 쌍은 짝수 번 등장하므로 홀수 번 등장하는 이름이 정답이다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 빈도 카운팅
    • 선택 이유: 참가자 명단과 완주자 명단을 합쳐 2N-1개의 이름을 한 번에 읽으며 등장 횟수를 세면, 별도의 자료구조 없이 O(N)에 완주하지 못한 사람을 찾을 수 있다. HashMap의 평균 O(1) 삽입·조회 덕분에 전체 O(N)으로 해결된다.
    • 세부 전략:
      • 참가자 N명과 완주자 N-1명의 이름을 합쳐 총 2N-1개를 순서대로 읽으며 HashMap에 등장 횟수를 누적한다.
      • 모든 이름을 읽은 뒤, 등장 횟수가 홀수인 이름이 완주하지 못한 사람이다.

    코드가 하는 일

    • 1) 2N-1개의 이름 입력 및 빈도 누적 
      Map map = new HashMap<>();
      
      for (int i = 0; i < 2 * N - 1; i++) {
          String runner = br.readLine();
          map.put(runner, map.getOrDefault(runner, 0) + 1);
      }
      • 참가자 N명과 완주자 N-1명의 이름을 순서대로 읽어 총 2N-1개를 처리한다.
      • getOrDefault(runner, 0) + 1로 이미 존재하는 이름이면 카운트를 증가시키고, 처음 등장하는 이름이면 1로 초기화한다.
      • 완주한 사람은 참가 1회 + 완주 1회로 총 2회(짝수) 등장하고, 완주하지 못한 사람은 참가 1회만으로 1회(홀수) 등장한다.
    • 2) 홀수 번 등장한 이름 탐색 및 출력 
      for (Map.Entry entry : map.entrySet()) {
          String runner = entry.getKey();
          int count = entry.getValue();
      
          if (count % 2 != 0) {
              System.out.println(runner);
              return;
          }
      }
      • HashMap의 모든 항목을 순회하며 등장 횟수가 홀수인 이름을 찾는다.
      • 완주하지 못한 사람은 정확히 1명이므로, 홀수 횟수인 이름을 찾는 즉시 출력하고 종료한다.
      • 동명이인이 여러 명 참가한 경우에도, 완주한 쌍은 2회씩 등장해 짝수가 되고 완주하지 못한 1명은 홀수 횟수로 남아 정확히 판별된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 2N-1개의 이름을 읽으며 HashMap에 삽입하는 비용이 평균 O(1)이므로 전체 O(N)이다. 이후 HashMap 순회도 등록된 고유 이름 수에 비례하며 최대 O(N)이다.
    • 공간 복잡도
      • O(N): HashMap에 저장되는 고유 이름의 수가 최대 N개이므로 O(N)의 공간을 사용한다.

    문제 1337 — 올바른 배열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    올바른 배열 (백준 1337)는
    N개의 정수가 주어질 때, 배열이 올바른 배열이 되기 위해 추가해야 하는 최소 원소의 수를 구하는 문제다.
    올바른 배열이란 길이가 5 이상이고, 어떤 5개의 연속 원소를 골라도 그 최댓값과 최솟값의 차이가 4 이하인 배열이다.

    • 추가할 원소의 수를 최소화하려면, 기존 원소 중 크기 범위 4 이내에 속하는 원소를 최대한 많이 포함해야 한다.
    • 배열을 정렬하면, 크기 범위 4 이내의 원소들은 연속된 구간에 모여 있다.
    • 슬라이딩 윈도우로 numbers[j] <= numbers[i] + 4를 만족하는 최대 원소 수 maxCount를 구하면, 답은 5 - maxCount이다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 슬라이딩 윈도우
    • 선택 이유: 배열을 정렬하면 범위 4 이내의 원소들이 연속된 구간에 모인다. 각 인덱스 i를 왼쪽 끝으로 고정하고 numbers[j] <= numbers[i] + 4를 만족하는 가장 오른쪽 j를 찾으면 해당 구간의 원소 수를 구할 수 있다. 이 중 최댓값을 구하면 추가 원소 없이 활용 가능한 기존 원소의 최대 수다.
    • 세부 전략:
      • 배열을 오름차순 정렬한다.
      • 각 i에 대해 numbers[i]를 범위의 최솟값으로 설정하고, numbers[i] + 4 이하인 원소의 수 count를 센다.
      • 모든 i에 대한 count 중 최댓값 maxCount를 구한다.
      • 답은 5 - maxCount이다. maxCount가 이미 5 이상이면 0이 되어야 하지만, 문제 조건상 N < 5인 경우만 최댓값이 5 미만이므로 5 - maxCount가 곧 최솟값이다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] numbers = new int[N];
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(br.readLine());
      }
      Arrays.sort(numbers);
      • N개의 정수를 배열에 저장하고 오름차순 정렬한다.
      • 정렬 후 범위 4 이내의 원소들이 연속된 구간에 모이므로, 윈도우 탐색이 가능해진다.
    • 2) 슬라이딩 윈도우로 최대 원소 수 탐색 
      int maxCount = 0;
      
      for (int i = 0; i < N; i++) {
          int count = 0;
      
          for (int j = i; j < N; j++) {
              if (numbers[j] <= numbers[i] + 4) {
                  count++;
              } else {
                  break;
              }
          }
      
          maxCount = Math.max(maxCount, count);
      }
      • 외부 반복문의 i는 윈도우의 왼쪽 끝(최솟값)을 나타낸다.
      • 내부 반복문에서 numbers[j] <= numbers[i] + 4를 만족하는 동안 count를 증가시킨다. 배열이 정렬되어 있으므로 조건을 위반하는 순간 이후 원소들도 모두 위반하므로 즉시 break한다.
      • 각 i에 대한 count와 현재 maxCount를 비교해 최댓값을 유지한다.
      • 예시: numbers = [1, 2, 3, 4, 10]이면 i=0에서 count=4 (1~4 모두 범위 내), i=4에서 count=1. maxCount = 4. 답: 5-4=1.
    • 3) 결과 출력 
      System.out.println(5 - maxCount);
      • 5개를 채우기 위해 기존 원소 maxCount개를 활용하고, 나머지 5 - maxCount개를 추가하면 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 이중 반복문은 각 원소가 내부 반복에서 최대 한 번씩만 처리되는 슬라이딩 윈도우 구조이므로 O(N)이다. 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N개의 정수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
