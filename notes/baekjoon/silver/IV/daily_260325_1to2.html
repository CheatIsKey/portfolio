    문제 13413 — 오셀로 재배치

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    오셀로 재배치 (백준 13413)는
    N개의 오셀로 돌을 초기 상태에서 목표 상태로 바꿀 때, 뒤집기 연산의 최소 횟수를 구하는 문제다.
    한 번의 연산으로 흰색('W')과 검은색('B')이 다른 위치의 두 돌을 서로 교환할 수 있다.

    • 초기 상태와 목표 상태가 다른 위치만 처리하면 된다. 이미 맞는 위치는 건드릴 필요가 없다.
    • 불일치 위치는 두 종류다. before가 'W'이고 after가 'B'인 경우(wrongWhite)와, before가 'B'이고 after가 'W'인 경우(wrongBlack).
    • 두 종류의 불일치 쌍끼리는 한 번의 교환으로 동시에 해결된다. 남는 쪽은 같은 종류끼리 교환해도 해결할 수 없으므로 추가 연산이 필요하지 않고, 실질적으로 최솟값은 두 카운트 중 큰 값이다.

    전략 개요

    • 핵심 알고리즘: 그리디 (불일치 유형별 카운팅)
    • 선택 이유: 불일치 위치를 두 유형으로 나누면, 서로 다른 유형의 불일치 쌍은 한 번의 교환으로 해결된다. min(wrongWhite, wrongBlack)개의 쌍이 교환으로 해결되고, 나머지 |wrongWhite - wrongBlack|개는 두 번의 연산(뒤집기)이 필요하다. 총 연산 수는 min + 2 × |diff| / 2 = min + |diff| = max(wrongWhite, wrongBlack)가 된다.
    • 세부 전략:
      • 각 위치에서 before와 after가 같으면 건너뛴다.
      • 다른 경우 before가 'W'이면 wrongWhite, 'B'이면 wrongBlack을 증가시킨다.
      • 두 값 중 큰 값이 최소 연산 횟수다.

    코드가 하는 일

    • 1) 초기·목표 상태 입력 
      int N = Integer.parseInt(br.readLine());
      
      char[] before = br.readLine().toCharArray();
      char[] after = br.readLine().toCharArray();
      • 초기 상태와 목표 상태를 각각 char 배열로 읽는다.
    • 2) 불일치 유형별 카운팅 
      int wrongWhite = 0;
      int wrongBlack = 0;
      
      for (int i = 0; i < N; i++) {
          if (before[i] == after[i]) continue;
          if (before[i] == 'W') wrongWhite++;
          else wrongBlack++;
      }
      • 초기 상태와 목표 상태가 같은 위치는 건너뛴다.
      • 다른 위치에서 before가 'W'이면 'W'→'B'로 바꿔야 하는 위치이므로 wrongWhite를 증가시킨다.
      • before가 'B'이면 'B'→'W'로 바꿔야 하는 위치이므로 wrongBlack을 증가시킨다.
      • 'W'→'B' 불일치와 'B'→'W' 불일치가 각각 한 개씩 있으면, 두 위치의 돌을 교환하는 한 번의 연산으로 동시에 해결된다.
      • 예시: wrongWhite=3, wrongBlack=2이면 2쌍은 교환으로 해결(2번), 남은 wrongWhite 1개는 추가로 1번 필요. 총 3번 = max(3, 2).
    • 3) 결과 출력 
      sb.append(Math.max(wrongWhite, wrongBlack)).append('\n');
      • 두 불일치 카운트 중 큰 값이 최소 연산 횟수다.
      • 결과를 StringBuilder에 누적해 마지막에 한 번에 출력함으로써 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T × N): T개의 테스트 케이스 각각에 대해 N개의 위치를 한 번 순회하며 O(1) 연산만 수행하므로 전체 O(T × N)이다.
    • 공간 복잡도
      • O(N): 초기 상태와 목표 상태를 저장하는 두 개의 char 배열이 각각 N 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 3048 — 개미

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    개미 (백준 3048)는
    왼쪽 그룹과 오른쪽 그룹의 개미들이 서로 반대 방향으로 이동할 때, T초 후의 개미 순서를 출력하는 문제다.
    두 개미가 만나면 서로 방향을 바꿔 지나치며, 이는 두 개미가 교환되는 것과 동일하다.

    • 왼쪽 그룹은 오른쪽(+1) 방향, 오른쪽 그룹은 왼쪽(-1) 방향으로 이동한다.
    • 매 초마다 방향이 +1인 개미와 바로 옆 -1인 개미가 만나면 위치를 교환한다.
    • 같은 초에 이미 교환된 개미는 다시 교환되지 않도록 moved 배열로 처리한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (매 초마다 인접 교환)
    • 선택 이유: T가 최대 100, 개미 수가 최대 50이므로 매 초마다 인접한 반대 방향 개미 쌍을 직접 교환하는 시뮬레이션이 O(T × N)으로 충분히 빠르다.
    • 세부 전략:
      • 왼쪽 그룹은 입력 순서를 역순으로 배열에 배치해 실제 위치 순서와 맞춘다. 왼쪽 그룹 문자열의 첫 글자가 오른쪽에서 가장 가까운 개미이므로 배열에서 인덱스 N1-1에 위치한다.
      • 매 초마다 왼쪽에서 오른쪽으로 순회하며 방향이 +1인 개미와 바로 오른쪽의 방향이 -1인 개미를 교환한다.
      • 같은 초에 이미 교환된 개미는 moved 배열로 표시해 중복 교환을 방지한다.

    코드가 하는 일

    • 1) 개미 배열 초기화 
      String s = br.readLine();
      for (int i = 0; i < N1; i++) {
          ants[N1 - 1 - i] = new Ant(s.charAt(i), 1);
      }
      
      s = br.readLine();
      for (int i = 0; i < N2; i++) {
          ants[N1 + i] = new Ant(s.charAt(i), -1);
      }
      • 왼쪽 그룹은 입력 문자열의 첫 글자가 오른쪽 끝(접경 지점에 가장 가까운 위치)에 해당하므로 역순으로 배치한다. ants[N1-1-i] = s.charAt(i)로 처리한다.
      • 오른쪽 그룹은 입력 순서 그대로 N1 인덱스부터 이어 배치한다.
      • 각 개미는 이름(info)과 이동 방향(direction)을 가진 Ant 객체로 표현된다.
    • 2) T초 동안 시뮬레이션 
      for (int t = 0; t < T; t++) {
          boolean[] moved = new boolean[ants.length];
      
          for (int i = 0; i < ants.length - 1; i++) {
              if (ants[i].direction == 1 && ants[i + 1].direction == -1) {
                  if (!moved[i] && !moved[i + 1]) {
                      Ant ant = ants[i];
                      ants[i] = ants[i + 1];
                      ants[i + 1] = ant;
      
                      moved[i] = true;
                      moved[i + 1] = true;
                  }
              }
          }
      }
      • 매 초마다 moved 배열을 초기화해 이번 초의 교환 여부를 추적한다.
      • 왼쪽에서 오른쪽으로 순회하며 ants[i].direction == 1이고 ants[i+1].direction == -1이면 두 개미가 만나는 상황이다.
      • 두 개미 모두 이번 초에 아직 교환되지 않은 경우에만 위치를 교환하고 moved를 true로 표시한다. 이미 교환된 개미가 연쇄적으로 또 교환되는 것을 방지한다.
      • 예시: 개미 배열이 [A(+), B(-), C(+), D(-)]이면 1초 후 [B(-), A(+), D(-), C(+)]가 된다.
    • 3) 결과 출력 
      for (Ant ant : ants) sb.append(ant.info);
      System.out.println(sb);
      • T초 후 배열의 각 개미 이름을 순서대로 이어붙여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T × N): T번의 시뮬레이션 각각에서 N = N1 + N2개의 개미를 한 번 순회하며 O(1) 연산만 수행한다. T ≤ 100, N ≤ 50이므로 최악의 경우에도 5,000번의 연산으로 매우 빠르게 동작한다.
    • 공간 복잡도
      • O(N): N개의 개미를 저장하는 배열과 매 초마다 생성되는 moved 배열이 각각 N 크기를 사용한다.
