    문제 15489 — 파스칼 삼각형

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    파스칼 삼각형 (백준 15489)는
    파스칼 삼각형에서 R행 C열을 꼭짓점으로 하는 크기 W의 삼각형 영역에 속한 모든 수의 합을 구하는 문제다.
    삼각형 영역은 꼭짓점에서 아래로 W행에 걸쳐 왼쪽 변을 따라 펼쳐지는 정삼각형 모양이다.

    • 파스칼 삼각형의 각 원소는 바로 위 행의 왼쪽과 오른쪽 원소의 합으로 정의된다.
    • R, C, W가 모두 최대 30이므로 전체 파스칼 삼각형을 미리 계산해두는 사전 계산 방식이 적합하다.
    • 꼭짓점 (R, C)에서 시작해 r번째 행(0-based)의 원소는 C열부터 C+r열까지 r+1개이다.

    전략 개요

    • 핵심 알고리즘: 동적 프로그래밍 (파스칼 삼각형 사전 계산) + 부분합
    • 선택 이유: R + W ≤ 60이지만 R, C, W가 최대 30이므로 31 × 31 크기의 배열로 모든 경우를 커버할 수 있다. 파스칼 삼각형을 미리 계산해두면 이후 영역 합 계산이 O(W²)의 단순 합산으로 해결된다.
    • 세부 전략:
      • 1-based 인덱스로 파스칼 삼각형 전체를 미리 계산한다. 양 끝(c == 1 또는 c == r)은 1, 나머지는 triangle[r-1][c-1] + triangle[r-1][c]로 채운다.
      • 꼭짓점 (R, C)에서 시작해 r = 0부터 W-1까지 순회하며 각 행의 원소를 합산한다. r번째 행은 triangle[R+r][C]부터 triangle[R+r][C+r]까지 r+1개다.

    코드가 하는 일

    • 1) 파스칼 삼각형 사전 계산 
      int[][] triangle = new int[31][31];
      
      for (int r = 1; r < 31; r++) {
          for (int c = 1; c < r + 1; c++) {
              if (c == 1 || c == r) {
                  triangle[r][c] = 1;
              } else {
                  triangle[r][c] = triangle[r - 1][c - 1] + triangle[r - 1][c];
              }
          }
      }
      • 1-based 인덱스로 31 × 31 배열에 파스칼 삼각형을 계산한다.
      • 각 행의 첫 번째(c == 1)와 마지막(c == r) 원소는 1이고, 나머지는 바로 위 행의 인접 두 원소의 합이다.
      • R + W - 1이 최대 30 + 30 - 1 = 59가 될 수 있으나, 문제 조건상 R, C, W ≤ 30이고 모두 양의 정수이므로 최대 필요한 행은 R + W - 1 ≤ 59다. 그러나 실제 제약상 30행까지만 필요하다.
    • 2) 삼각형 영역 합산 
      int sum = 0;
      
      for (int r = 0; r < W; r++) {
          for (int c = 0; c < r + 1; c++) {
              sum += triangle[R + r][C + c];
          }
      }
      • r은 꼭짓점에서의 상대적 행 오프셋으로 0부터 W-1까지 순회한다.
      • r번째 행의 원소는 꼭짓점 열 C에서 C+r까지 r+1개이므로, c는 0부터 r까지 순회한다.
      • 실제 파스칼 삼각형의 위치는 triangle[R+r][C+c]로 접근한다.
      • 예시: R=2, C=2, W=3이면 합산 영역은 triangle[2][2], triangle[3][2~3], triangle[4][2~4] 총 6개 원소의 합이다.
    • 3) 결과 출력 
      System.out.println(sum);
      • 삼각형 영역의 모든 원소 합 sum을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 파스칼 삼각형 사전 계산이 O(31²) = O(961), 영역 합산이 O(W²) ≤ O(900)으로 모두 상수 크기의 연산이다. 입력 크기와 무관하게 항상 일정한 수의 연산으로 처리된다.
    • 공간 복잡도
      • O(1): 파스칼 삼각형을 저장하는 31 × 31 고정 크기 배열만 사용하므로 입력 크기와 무관한 상수 공간이다.

    문제 2980 — 도로와 신호등

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    도로와 신호등 (백준 2980)는
    길이 L의 도로에 N개의 신호등이 설치되어 있을 때, 출발점에서 도착점까지 걸리는 최소 시간을 구하는 문제다.
    신호등은 빨간불과 초록불을 반복하며, 빨간불일 때는 해당 위치에서 대기해야 한다.

    • 속도는 1로 고정되므로 이동 시간은 이동 거리와 동일하다.
    • 신호등에 도착했을 때 빨간불이면 초록불로 바뀔 때까지 대기해야 한다.
    • 신호등의 주기는 red + green이며, 도착 시각을 주기로 나눈 나머지로 현재 신호 상태를 판별한다.

    전략 개요

    • 핵심 알고리즘: 그리디 시뮬레이션
    • 선택 이유: 신호등을 순서대로 처리하며 현재 시각과 각 신호등의 주기 정보만으로 대기 시간을 O(1)에 계산할 수 있다. 신호등의 수가 최대 1,000이므로 O(N) 선형 순회로 충분하다.
    • 세부 전략:
      • 이전 위치에서 현재 신호등 위치까지의 이동 시간을 totalTime에 더한다.
      • totalTime % cycle로 신호등 도착 시 현재 신호 위상을 계산한다.
      • 위상이 red 미만이면 빨간불 구간이므로 red - color만큼 대기 시간을 추가한다.
      • 마지막 신호등 이후 도착점까지의 이동 시간을 더해 최종 시간을 구한다.

    코드가 하는 일

    • 1) 초기화 및 신호등 순서 처리 
      int totalTime = 0;
      int pos = 0;
      
      for (int i = 0; i < N; i++) {
          int traffic = Integer.parseInt(st.nextToken());
          int red = Integer.parseInt(st.nextToken());
          int green = Integer.parseInt(st.nextToken());
      
          totalTime += (traffic - pos);
          pos = traffic;
          ...
      }
      • totalTime은 현재까지 경과한 총 시간, pos는 현재 위치를 나타낸다.
      • 현재 신호등 위치 traffic까지의 이동 거리 traffic - pos를 totalTime에 더하고, pos를 갱신한다.
    • 2) 신호등 대기 시간 계산 
      int cycle = red + green;
      int color = totalTime % cycle;
      
      if (color < red) {
          totalTime += (red - color);
      }
      • cycle = red + green은 신호등의 한 주기 길이다.
      • color = totalTime % cycle은 신호등 도착 시각을 주기로 나눈 나머지로, 현재 신호 위상을 나타낸다.
      • color < red이면 빨간불 구간에 도착한 것이므로 초록불로 바뀌기까지 남은 시간 red - color를 대기한다.
      • color >= red이면 초록불 구간이므로 즉시 통과한다.
      • 예시: cycle=5(red=3, green=2), totalTime=7이면 color=7%5=2. 2 < 3이므로 3-2=1초 대기.
    • 3) 마지막 구간 이동 및 출력 
      totalTime += (L - pos);
      System.out.println(totalTime);
      • 마지막 신호등 이후 도착점 L까지의 이동 거리 L - pos를 더한다.
      • 최종 총 시간 totalTime을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 신호등을 순서대로 처리하며 각 신호등에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(1): 신호등 정보를 배열에 저장하지 않고 입력받는 즉시 처리하므로 입력 크기와 무관한 상수 공간만 사용한다.
