    문제 2840 — 행운의 바퀴

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    행운의 바퀴 (백준 2840)는
    N칸짜리 원형 바퀴를 K번 돌릴 때의 기록을 바탕으로 바퀴에 적힌 문자 배열을 복원하는 문제다.
    복원이 불가능한 경우 "!"를 출력하며, 가능한 경우 현재 위치에서 역방향으로 출력한다.

    • 각 칸에는 서로 다른 알파벳이 적혀 있어야 하므로, 같은 문자가 서로 다른 위치에 배치되면 모순이다.
    • 이미 다른 문자가 배치된 위치에 새로운 문자가 배치되려 해도 모순이다.
    • 출력 순서는 현재 포인터 위치에서 시작해 역방향(반시계 방향)으로 N칸을 순서대로 나열한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 + 모순 검사
    • 선택 이유: K번의 회전 기록을 순서대로 처리하며 각 위치에 문자를 배치하고, 두 가지 모순 조건을 O(1)에 검사하면 된다. N, K가 최대 1,000이므로 O(K) 선형 시뮬레이션으로 충분하다.
    • 세부 전략:
      • board 배열을 '?'로 초기화하고, used 배열로 이미 사용된 문자를 추적한다.
      • 각 회전 후 포인터 위치를 (pos + S) % N으로 갱신한다.
      • 모순이 없으면 해당 위치에 문자를 배치하고 사용 표시를 한다.
      • 모든 기록 처리 후 포인터 위치에서 역방향으로 N칸을 순서대로 출력한다.

    코드가 하는 일

    • 1) 초기화 
      char[] board = new char[N];
      boolean[] used = new boolean[32];
      int pos = 0;
      
      Arrays.fill(board, '?');
      • N칸의 바퀴를 나타내는 board 배열을 '?'로 초기화한다. '?'는 아직 문자가 배치되지 않은 칸을 의미한다.
      • used 배열은 각 알파벳이 이미 어떤 칸에 배치되었는지를 추적한다. 같은 문자가 두 위치에 배치되는 것을 방지한다.
      • pos는 현재 포인터 위치이며 0으로 초기화한다.
    • 2) 회전 시뮬레이션 및 모순 검사 
      pos = (pos + S) % N;
      
      if (board[pos] != '?' && board[pos] != word) {
          System.out.println("!");
          return;
      }
      
      if (board[pos] == '?' && used[word - 'A']) {
          System.out.println("!");
          return;
      }
      
      board[pos] = word;
      used[word - 'A'] = true;
      • (pos + S) % N으로 S칸 회전 후 포인터 위치를 갱신한다.
      • 첫 번째 모순 조건: 해당 위치에 이미 다른 문자가 배치된 경우. board[pos] != '?'이고 board[pos] != word이면 "!"를 출력하고 종료한다.
      • 두 번째 모순 조건: 해당 위치가 비어있지만 배치하려는 문자가 다른 위치에 이미 사용된 경우. board[pos] == '?'이고 used[word - 'A']가 true이면 "!"를 출력하고 종료한다.
      • 모순이 없으면 해당 위치에 문자를 배치하고 사용 표시를 한다.
    • 3) 결과 출력 
      for (int i = 0; i < N; i++) {
          int idx = (pos - i + N) % N;
          sb.append(board[idx]);
      }
      
      System.out.println(sb);
      • 현재 포인터 위치 pos부터 역방향으로 N칸을 순서대로 순회한다.
      • (pos - i + N) % N으로 음수 인덱스를 방지하며 역방향 순환 접근을 구현한다.
      • 배치되지 않은 칸은 '?'로 그대로 출력된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K + N): K번의 회전 처리가 각 O(1)이므로 O(K), 결과 출력 시 N칸 순회가 O(N)이다. 전체 O(K + N)이다.
    • 공간 복잡도
      • O(N): 바퀴 상태를 저장하는 board 배열이 N 크기를 사용하며, used 배열은 알파벳 수에 비례하는 O(1) 상수 공간이다.

    문제 20186 — 수 고르기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    도로와 신호등 (백준 2980)는
    길이 L의 도로에 N개의 신호등이 설치되어 있을 때, 출발점에서 도착점까지 걸리는 최소 시간을 구하는 문제다.
    신호등은 빨간불과 초록불을 반복하며, 빨간불일 때는 해당 위치에서 대기해야 한다.

    • 속도는 1로 고정되므로 이동 시간은 이동 거리와 동일하다.
    • 신호등에 도착했을 때 빨간불이면 초록불로 바뀔 때까지 대기해야 한다.
    • 신호등의 주기는 red + green이며, 도착 시각을 주기로 나눈 나머지로 현재 신호 상태를 판별한다.

    전략 개요

    • 핵심 알고리즘: 그리디 시뮬레이션
    • 선택 이유: 신호등을 순서대로 처리하며 현재 시각과 각 신호등의 주기 정보만으로 대기 시간을 O(1)에 계산할 수 있다. 신호등의 수가 최대 1,000이므로 O(N) 선형 순회로 충분하다.
    • 세부 전략:
      • 이전 위치에서 현재 신호등 위치까지의 이동 시간을 totalTime에 더한다.
      • totalTime % cycle로 신호등 도착 시 현재 신호 위상을 계산한다.
      • 위상이 red 미만이면 빨간불 구간이므로 red - color만큼 대기 시간을 추가한다.
      • 마지막 신호등 이후 도착점까지의 이동 시간을 더해 최종 시간을 구한다.

    코드가 하는 일

    • 1) 초기화 및 신호등 순서 처리 
      int totalTime = 0;
      int pos = 0;
      
      for (int i = 0; i < N; i++) {
          int traffic = Integer.parseInt(st.nextToken());
          int red = Integer.parseInt(st.nextToken());
          int green = Integer.parseInt(st.nextToken());
      
          totalTime += (traffic - pos);
          pos = traffic;
          ...
      }
      • totalTime은 현재까지 경과한 총 시간, pos는 현재 위치를 나타낸다.
      • 현재 신호등 위치 traffic까지의 이동 거리 traffic - pos를 totalTime에 더하고, pos를 갱신한다.
    • 2) 신호등 대기 시간 계산 
      int cycle = red + green;
      int color = totalTime % cycle;
      
      if (color < red) {
          totalTime += (red - color);
      }
      • cycle = red + green은 신호등의 한 주기 길이다.
      • color = totalTime % cycle은 신호등 도착 시각을 주기로 나눈 나머지로, 현재 신호 위상을 나타낸다.
      • color < red이면 빨간불 구간에 도착한 것이므로 초록불로 바뀌기까지 남은 시간 red - color를 대기한다.
      • color >= red이면 초록불 구간이므로 즉시 통과한다.
      • 예시: cycle=5(red=3, green=2), totalTime=7이면 color=7%5=2. 2 < 3이므로 3-2=1초 대기.
    • 3) 마지막 구간 이동 및 출력 
      totalTime += (L - pos);
      System.out.println(totalTime);
      • 마지막 신호등 이후 도착점 L까지의 이동 거리 L - pos를 더한다.
      • 최종 총 시간 totalTime을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 신호등을 순서대로 처리하며 각 신호등에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(1): 신호등 정보를 배열에 저장하지 않고 입력받는 즉시 처리하므로 입력 크기와 무관한 상수 공간만 사용한다.
