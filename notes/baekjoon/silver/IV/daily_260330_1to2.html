    문제 11502 — 세 개의 소수 문제

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    세 개의 소수 문제 (백준 11502)는
    정수 K가 주어질 때, 세 소수의 합으로 K를 표현하는 가장 작은 조합을 구하는 문제다.
    세 소수는 p ≤ q ≤ r을 만족해야 하며, K가 6 미만이면 표현이 불가능하므로 0을 출력한다.

    • K는 최대 1,000이며, 에라토스테네스의 체로 1,000 이하의 소수를 미리 구해두면 효율적이다.
    • 세 소수 p ≤ q ≤ r의 조건에서 p를 가장 작은 값부터 탐색하면 사전순으로 가장 앞선 조합을 찾을 수 있다.
    • 가장 작은 소수 3개의 합은 2 + 2 + 2 = 6이므로, K < 6이면 항상 불가능하다.

    전략 개요

    • 핵심 알고리즘: 에라토스테네스의 체 + 3중 브루트포스
    • 선택 이유: K가 최대 1,000이므로 소수 범위도 최대 1,000이다. p ≤ q ≤ r 조건으로 중복을 줄이면 최악의 경우에도 약 1,000³ / 6 이하의 탐색이 필요하지만, 소수만 탐색하므로 실제 반복 횟수는 훨씬 적어 충분히 빠르다.
    • 세부 전략:
      • 에라토스테네스의 체로 1,000 이하의 소수를 미리 구한다.
      • p를 2부터, q를 p부터, r을 q부터 순서대로 탐색하며 p + q + r == K를 만족하는 첫 번째 조합을 출력한다.
      • 조합을 찾는 즉시 flag를 통해 3중 반복문을 모두 탈출한다.

    코드가 하는 일

    • 1) 에라토스테네스의 체로 소수 사전 계산 
      boolean[] isPrime = new boolean[1001];
      Arrays.fill(isPrime, true);
      
      isPrime[0] = false;
      isPrime[1] = false;
      
      for (int i = 2; i * i < isPrime.length; i++) {
          if (isPrime[i]) {
              for (int j = i * i; j < isPrime.length; j += i) {
                  isPrime[j] = false;
              }
          }
      }
      • 1,001 크기의 배열을 true로 초기화한 뒤, 0과 1은 소수가 아니므로 false로 설정한다.
      • 2부터 √1,000까지 순회하며 각 소수의 배수를 false로 표시한다. 테스트 케이스마다 반복하지 않고 한 번만 계산하므로 효율적이다.
    • 2) K < 6 예외 처리 
      if (K <= 5) {
          sb.append(0).append('\n');
          continue;
      }
      • 세 소수의 합의 최솟값은 2 + 2 + 2 = 6이므로, K가 5 이하이면 조합이 존재하지 않는다. 즉시 0을 출력하고 다음 테스트 케이스로 넘어간다.
    • 3) 3중 브루트포스로 첫 번째 조합 탐색 
      for (int i = 2; i < K; i++) {
          if (isPrime[i]) {
              for (int j = i; j < K; j++) {
                  if (isPrime[j]) {
                      for (int k = j; k < K; k++) {
                          if (isPrime[k]) {
                              if (i + j + k == K) {
                                  sb.append(i).append(" ").append(j).append(" ").append(k).append('\n');
                                  flag = true;
                                  break;
                              }
                          }
                      }
                  }
                  if (flag) break;
              }
          }
          if (flag) break;
      }
      • p(i)를 2부터, q(j)를 p 이상, r(k)를 q 이상으로 탐색해 p ≤ q ≤ r 조건을 만족하는 조합만 검사한다.
      • 소수인 경우에만 내부로 진입해 탐색 횟수를 줄인다.
      • p + q + r == K를 만족하는 첫 번째 조합을 찾으면 결과를 출력하고 flag = true로 설정해 3중 반복문을 모두 탈출한다. 가장 작은 p를 먼저 탐색하므로 자동으로 사전순 최솟값이 출력된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K² / ln(K)²): 소수 사전 계산은 O(K log log K), 각 테스트 케이스의 3중 탐색은 소수만 탐색하므로 실제 탐색 횟수가 K 이하 소수의 수의 제곱에 비례한다. K ≤ 1,000이고 소수 밀도가 낮아 실제 수행 횟수는 매우 적다.
    • 공간 복잡도
      • O(K): 소수 판별 배열이 K + 1 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 4396 — 지뢰 찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수 고르기 (백준 20186)는
    N개의 수 중 K개를 골라 순서를 정할 때, 선택한 수의 합에서 각 수의 순번(1-based) - 1을 뺀 값의 합을 최대화하는 문제다.
    즉, i번째로 선택한 수의 기여값은 A[i] - (i - 1)이며, 이 합이 최대가 되도록 K개를 선택하고 순서를 정해야 한다.

    • i번째 수의 기여값은 A[i] - (i-1)이므로, 순번이 커질수록 감점이 커진다. 따라서 큰 수를 앞 순번에 배치하는 것이 유리하다.
    • 감점의 합은 0 + 1 + 2 + ... + (K-1) = K(K-1)/2로 고정되므로, 선택한 수의 합만 최대화하면 된다.
    • 수의 합을 최대화하려면 가장 큰 K개를 선택하면 된다.

    전략 개요

    • 핵심 알고리즘: 그리디 (내림차순 정렬 후 상위 K개 선택)
    • 선택 이유: 감점의 합 0 + 1 + ... + (K-1)은 선택하는 수에 관계없이 항상 고정된다. 따라서 총 점수를 최대화하려면 선택하는 수의 합을 최대화해야 하며, 이는 가장 큰 K개를 선택하는 그리디 전략으로 달성된다.
    • 세부 전략:
      • 배열을 오름차순 정렬한 뒤 뒤에서부터 K개를 선택해 내림차순 효과를 낸다.
      • 선택한 K개의 합에서 0 + 1 + 2 + ... + (K-1)을 빼면 최댓값이 된다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] numbers = new int[N];
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(numbers);
      • N개의 수를 배열에 저장하고 오름차순 정렬한다.
      • 정렬 후 배열 뒤에서부터 순회하면 가장 큰 수부터 선택할 수 있다.
    • 2) 상위 K개 합산 및 감점 계산 
      int answer = 0;
      int minus = 0;
      
      for (int i = 0; i < K; i++) {
          answer += numbers[N - 1 - i];
          minus += i;
      }
      • i = 0부터 K-1까지 순회하며 numbers[N-1-i]로 가장 큰 수부터 K개를 answer에 누적한다.
      • minus에는 0 + 1 + 2 + ... + (K-1)의 감점 합을 누적한다. i번째(0-based) 선택 수의 감점은 i이다.
      • 감점의 합은 K(K-1)/2로 고정되므로, 어떤 K개를 선택하든 감점은 동일하다. 따라서 선택하는 수의 합을 최대화하는 것이 핵심이다.
      • 예시: numbers = [1, 3, 5, 7], K = 3이면 상위 3개 선택: 7+5+3=15, minus=0+1+2=3. 답: 15-3=12.
    • 3) 결과 출력 
      System.out.println(answer - minus);
      • 선택한 K개의 합에서 감점의 합을 빼 최종 최댓값을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 K번의 선형 순회는 O(K)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N개의 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
