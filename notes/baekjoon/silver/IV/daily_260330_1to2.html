    문제 2840 — 행운의 바퀴

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    행운의 바퀴 (백준 2840)는
    N칸짜리 원형 바퀴를 K번 돌릴 때의 기록을 바탕으로 바퀴에 적힌 문자 배열을 복원하는 문제다.
    복원이 불가능한 경우 "!"를 출력하며, 가능한 경우 현재 위치에서 역방향으로 출력한다.

    • 각 칸에는 서로 다른 알파벳이 적혀 있어야 하므로, 같은 문자가 서로 다른 위치에 배치되면 모순이다.
    • 이미 다른 문자가 배치된 위치에 새로운 문자가 배치되려 해도 모순이다.
    • 출력 순서는 현재 포인터 위치에서 시작해 역방향(반시계 방향)으로 N칸을 순서대로 나열한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 + 모순 검사
    • 선택 이유: K번의 회전 기록을 순서대로 처리하며 각 위치에 문자를 배치하고, 두 가지 모순 조건을 O(1)에 검사하면 된다. N, K가 최대 1,000이므로 O(K) 선형 시뮬레이션으로 충분하다.
    • 세부 전략:
      • board 배열을 '?'로 초기화하고, used 배열로 이미 사용된 문자를 추적한다.
      • 각 회전 후 포인터 위치를 (pos + S) % N으로 갱신한다.
      • 모순이 없으면 해당 위치에 문자를 배치하고 사용 표시를 한다.
      • 모든 기록 처리 후 포인터 위치에서 역방향으로 N칸을 순서대로 출력한다.

    코드가 하는 일

    • 1) 초기화 
      char[] board = new char[N];
      boolean[] used = new boolean[32];
      int pos = 0;
      
      Arrays.fill(board, '?');
      • N칸의 바퀴를 나타내는 board 배열을 '?'로 초기화한다. '?'는 아직 문자가 배치되지 않은 칸을 의미한다.
      • used 배열은 각 알파벳이 이미 어떤 칸에 배치되었는지를 추적한다. 같은 문자가 두 위치에 배치되는 것을 방지한다.
      • pos는 현재 포인터 위치이며 0으로 초기화한다.
    • 2) 회전 시뮬레이션 및 모순 검사 
      pos = (pos + S) % N;
      
      if (board[pos] != '?' && board[pos] != word) {
          System.out.println("!");
          return;
      }
      
      if (board[pos] == '?' && used[word - 'A']) {
          System.out.println("!");
          return;
      }
      
      board[pos] = word;
      used[word - 'A'] = true;
      • (pos + S) % N으로 S칸 회전 후 포인터 위치를 갱신한다.
      • 첫 번째 모순 조건: 해당 위치에 이미 다른 문자가 배치된 경우. board[pos] != '?'이고 board[pos] != word이면 "!"를 출력하고 종료한다.
      • 두 번째 모순 조건: 해당 위치가 비어있지만 배치하려는 문자가 다른 위치에 이미 사용된 경우. board[pos] == '?'이고 used[word - 'A']가 true이면 "!"를 출력하고 종료한다.
      • 모순이 없으면 해당 위치에 문자를 배치하고 사용 표시를 한다.
    • 3) 결과 출력 
      for (int i = 0; i < N; i++) {
          int idx = (pos - i + N) % N;
          sb.append(board[idx]);
      }
      
      System.out.println(sb);
      • 현재 포인터 위치 pos부터 역방향으로 N칸을 순서대로 순회한다.
      • (pos - i + N) % N으로 음수 인덱스를 방지하며 역방향 순환 접근을 구현한다.
      • 배치되지 않은 칸은 '?'로 그대로 출력된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K + N): K번의 회전 처리가 각 O(1)이므로 O(K), 결과 출력 시 N칸 순회가 O(N)이다. 전체 O(K + N)이다.
    • 공간 복잡도
      • O(N): 바퀴 상태를 저장하는 board 배열이 N 크기를 사용하며, used 배열은 알파벳 수에 비례하는 O(1) 상수 공간이다.

    문제 20186 — 수 고르기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수 고르기 (백준 20186)는
    N개의 수 중 K개를 골라 순서를 정할 때, 선택한 수의 합에서 각 수의 순번(1-based) - 1을 뺀 값의 합을 최대화하는 문제다.
    즉, i번째로 선택한 수의 기여값은 A[i] - (i - 1)이며, 이 합이 최대가 되도록 K개를 선택하고 순서를 정해야 한다.

    • i번째 수의 기여값은 A[i] - (i-1)이므로, 순번이 커질수록 감점이 커진다. 따라서 큰 수를 앞 순번에 배치하는 것이 유리하다.
    • 감점의 합은 0 + 1 + 2 + ... + (K-1) = K(K-1)/2로 고정되므로, 선택한 수의 합만 최대화하면 된다.
    • 수의 합을 최대화하려면 가장 큰 K개를 선택하면 된다.

    전략 개요

    • 핵심 알고리즘: 그리디 (내림차순 정렬 후 상위 K개 선택)
    • 선택 이유: 감점의 합 0 + 1 + ... + (K-1)은 선택하는 수에 관계없이 항상 고정된다. 따라서 총 점수를 최대화하려면 선택하는 수의 합을 최대화해야 하며, 이는 가장 큰 K개를 선택하는 그리디 전략으로 달성된다.
    • 세부 전략:
      • 배열을 오름차순 정렬한 뒤 뒤에서부터 K개를 선택해 내림차순 효과를 낸다.
      • 선택한 K개의 합에서 0 + 1 + 2 + ... + (K-1)을 빼면 최댓값이 된다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] numbers = new int[N];
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(numbers);
      • N개의 수를 배열에 저장하고 오름차순 정렬한다.
      • 정렬 후 배열 뒤에서부터 순회하면 가장 큰 수부터 선택할 수 있다.
    • 2) 상위 K개 합산 및 감점 계산 
      int answer = 0;
      int minus = 0;
      
      for (int i = 0; i < K; i++) {
          answer += numbers[N - 1 - i];
          minus += i;
      }
      • i = 0부터 K-1까지 순회하며 numbers[N-1-i]로 가장 큰 수부터 K개를 answer에 누적한다.
      • minus에는 0 + 1 + 2 + ... + (K-1)의 감점 합을 누적한다. i번째(0-based) 선택 수의 감점은 i이다.
      • 감점의 합은 K(K-1)/2로 고정되므로, 어떤 K개를 선택하든 감점은 동일하다. 따라서 선택하는 수의 합을 최대화하는 것이 핵심이다.
      • 예시: numbers = [1, 3, 5, 7], K = 3이면 상위 3개 선택: 7+5+3=15, minus=0+1+2=3. 답: 15-3=12.
    • 3) 결과 출력 
      System.out.println(answer - minus);
      • 선택한 K개의 합에서 감점의 합을 빼 최종 최댓값을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 전체 시간을 지배하는 것은 Arrays.sort()의 정렬 비용이다. 이후 K번의 선형 순회는 O(K)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): N개의 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
