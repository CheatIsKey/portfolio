    문제 11502 — 세 개의 소수 문제

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    세 개의 소수 문제 (백준 11502)는
    정수 K가 주어질 때, 세 소수의 합으로 K를 표현하는 가장 작은 조합을 구하는 문제다.
    세 소수는 p ≤ q ≤ r을 만족해야 하며, K가 6 미만이면 표현이 불가능하므로 0을 출력한다.

    • K는 최대 1,000이며, 에라토스테네스의 체로 1,000 이하의 소수를 미리 구해두면 효율적이다.
    • 세 소수 p ≤ q ≤ r의 조건에서 p를 가장 작은 값부터 탐색하면 사전순으로 가장 앞선 조합을 찾을 수 있다.
    • 가장 작은 소수 3개의 합은 2 + 2 + 2 = 6이므로, K < 6이면 항상 불가능하다.

    전략 개요

    • 핵심 알고리즘: 에라토스테네스의 체 + 3중 브루트포스
    • 선택 이유: K가 최대 1,000이므로 소수 범위도 최대 1,000이다. p ≤ q ≤ r 조건으로 중복을 줄이면 최악의 경우에도 약 1,000³ / 6 이하의 탐색이 필요하지만, 소수만 탐색하므로 실제 반복 횟수는 훨씬 적어 충분히 빠르다.
    • 세부 전략:
      • 에라토스테네스의 체로 1,000 이하의 소수를 미리 구한다.
      • p를 2부터, q를 p부터, r을 q부터 순서대로 탐색하며 p + q + r == K를 만족하는 첫 번째 조합을 출력한다.
      • 조합을 찾는 즉시 flag를 통해 3중 반복문을 모두 탈출한다.

    코드가 하는 일

    • 1) 에라토스테네스의 체로 소수 사전 계산 
      boolean[] isPrime = new boolean[1001];
      Arrays.fill(isPrime, true);
      
      isPrime[0] = false;
      isPrime[1] = false;
      
      for (int i = 2; i * i < isPrime.length; i++) {
          if (isPrime[i]) {
              for (int j = i * i; j < isPrime.length; j += i) {
                  isPrime[j] = false;
              }
          }
      }
      • 1,001 크기의 배열을 true로 초기화한 뒤, 0과 1은 소수가 아니므로 false로 설정한다.
      • 2부터 √1,000까지 순회하며 각 소수의 배수를 false로 표시한다. 테스트 케이스마다 반복하지 않고 한 번만 계산하므로 효율적이다.
    • 2) K < 6 예외 처리 
      if (K <= 5) {
          sb.append(0).append('\n');
          continue;
      }
      • 세 소수의 합의 최솟값은 2 + 2 + 2 = 6이므로, K가 5 이하이면 조합이 존재하지 않는다. 즉시 0을 출력하고 다음 테스트 케이스로 넘어간다.
    • 3) 3중 브루트포스로 첫 번째 조합 탐색 
      for (int i = 2; i < K; i++) {
          if (isPrime[i]) {
              for (int j = i; j < K; j++) {
                  if (isPrime[j]) {
                      for (int k = j; k < K; k++) {
                          if (isPrime[k]) {
                              if (i + j + k == K) {
                                  sb.append(i).append(" ").append(j).append(" ").append(k).append('\n');
                                  flag = true;
                                  break;
                              }
                          }
                      }
                  }
                  if (flag) break;
              }
          }
          if (flag) break;
      }
      • p(i)를 2부터, q(j)를 p 이상, r(k)를 q 이상으로 탐색해 p ≤ q ≤ r 조건을 만족하는 조합만 검사한다.
      • 소수인 경우에만 내부로 진입해 탐색 횟수를 줄인다.
      • p + q + r == K를 만족하는 첫 번째 조합을 찾으면 결과를 출력하고 flag = true로 설정해 3중 반복문을 모두 탈출한다. 가장 작은 p를 먼저 탐색하므로 자동으로 사전순 최솟값이 출력된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(K² / ln(K)²): 소수 사전 계산은 O(K log log K), 각 테스트 케이스의 3중 탐색은 소수만 탐색하므로 실제 탐색 횟수가 K 이하 소수의 수의 제곱에 비례한다. K ≤ 1,000이고 소수 밀도가 낮아 실제 수행 횟수는 매우 적다.
    • 공간 복잡도
      • O(K): 소수 판별 배열이 K + 1 크기를 사용하며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 4396 — 지뢰 찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    지뢰 찾기 (백준 4396)는
    n × n 크기의 지뢰 찾기 보드와 현재 열린 칸 정보가 주어질 때, 게임 결과판을 출력하는 문제다.
    열린 칸('x')에 지뢰가 있으면 게임 오버로, 모든 지뢰 위치를 '*'로 공개해야 한다.

    • 열린 칸('x')은 주변 8칸의 지뢰 수를 숫자로 표시하고, 열리지 않은 칸은 '.'으로 표시한다.
    • 열린 칸 중 지뢰('*')가 있으면 게임 오버이며, 이 경우 보드 전체에서 지뢰 위치를 '*'로 덮어씌워 출력한다.
    • 게임 오버가 아닌 경우에는 열린 칸의 주변 지뢰 수와 열리지 않은 칸('.')만 출력한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (8방향 탐색 + 게임 오버 판별)
    • 선택 이유: 각 칸에 대한 처리가 독립적이며, 열린 칸의 주변 지뢰 수 계산은 8방향 탐색으로 O(1)에 처리된다. n이 최대 8이므로 O(n²) 시뮬레이션으로 충분히 빠르다.
    • 세부 전략:
      • 보드를 순회하며 열린 칸('x')에 대해 8방향 지뢰 수를 세어 숫자로 기록하고, 열리지 않은 칸은 '.'으로 기록한다.
      • 열린 칸에 지뢰가 있으면 flag = true로 게임 오버를 표시한다.
      • 게임 오버이면 보드 전체를 다시 순회해 지뢰 위치를 '*'로 덮어씌운다.

    코드가 하는 일

    • 1) 입력 및 배열 구성 
      for (int i = 0; i < n; i++) {
          board[i] = br.readLine().toCharArray();
      }
      
      for (int i = 0; i < n; i++) {
          playing[i] = br.readLine().toCharArray();
      }
      • board는 실제 지뢰 배치('*' 또는 '.')를 저장하고, playing은 현재 열린 칸('x') 여부를 저장한다.
      • answer는 최종 출력 결과를 저장하는 배열이다.
    • 2) 열린 칸 처리 및 게임 오버 판별 
      if (playing[r][c] == 'x') {
          if (board[r][c] == '*') flag = true;
      
          int bomb = 0;
          for (int k = 0; k < 8; k++) {
              int row = r + ROW[k];
              int col = c + COL[k];
      
              if (row < 0 || row >= n || col < 0 || col >= n) continue;
              if (board[row][col] == '*') bomb++;
          }
          answer[r][c] = (char) (bomb + '0');
          continue;
      }
      answer[r][c] = '.';
      • 열린 칸에 지뢰가 있으면 flag = true로 게임 오버를 기록한다. 지뢰 수 계산과 게임 오버 판별을 동시에 처리해 한 번의 순회로 해결한다.
      • 8방향 탐색으로 주변 지뢰 수 bomb를 센 뒤 (char)(bomb + '0')으로 숫자 문자로 변환해 answer에 저장한다.
      • 열리지 않은 칸은 '.'으로 기록한다.
    • 3) 게임 오버 시 지뢰 위치 공개 
      if (flag) {
          for (int r = 0; r < n; r++) {
              for (int c = 0; c < n; c++) {
                  if (board[r][c] == '*') {
                      answer[r][c] = '*';
                  }
              }
          }
      }
      • 게임 오버인 경우 보드 전체를 다시 순회해 지뢰 위치를 '*'로 덮어씌운다.
      • 이미 숫자가 기록된 열린 칸이라도 지뢰가 있으면 '*'로 덮어쓴다. 지뢰를 밟은 칸도 '*'로 표시되어야 하기 때문이다.
    • 4) 결과 출력 
      for (char[] chars : answer) {
          sb.append(chars).append('\n');
      }
      System.out.println(sb);
      • 완성된 answer 배열의 각 행을 순서대로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n²): 보드 전체를 최대 두 번 순회하며(처리 + 게임 오버 시 지뢰 공개), 각 칸에서 8방향 탐색은 O(1)이므로 전체 O(n²)이다. n이 최대 8이므로 사실상 O(1)에 수렴한다.
    • 공간 복잡도
      • O(n²): 지뢰 배치, 열린 칸, 결과를 저장하는 세 개의 n × n 배열이 주요 메모리 사용처이며, n이 최대 8이므로 사실상 O(1)에 수렴한다.
