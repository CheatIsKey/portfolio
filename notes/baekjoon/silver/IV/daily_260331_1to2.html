    문제 28215 — 대피소

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    대피소 (백준 28215)는
    N개의 집 중 K개를 대피소로 선정할 때, 각 집에서 가장 가까운 대피소까지의 거리 중 최댓값을 최소화하는 문제다.
    거리는 맨해튼 거리(|x1-x2| + |y1-y2|)로 계산한다.

    • N개 중 K개를 선택하는 모든 조합을 탐색해 최적 대피소 배치를 찾는다.
    • 각 조합에 대해 모든 집의 "가장 가까운 대피소까지의 거리" 중 최댓값을 구하고, 이를 최소화하는 조합이 정답이다.
    • N이 최대 10, K가 최대 N이므로 최악의 경우 C(10, 5) = 252가지 조합으로 브루트포스가 충분하다.

    전략 개요

    • 핵심 알고리즘: 백트래킹 조합 탐색 (완전 탐색)
    • 선택 이유: N이 최대 10으로 매우 작아 N개 중 K개를 선택하는 모든 조합을 탐색해도 최대 C(10, 5) = 252가지에 불과하다. 각 조합 검증에 O(N × K) = O(100)이 소요되므로 전체 탐색이 충분히 빠르다.
    • 세부 전략:
      • 재귀 함수 combine으로 N개 중 K개를 선택하는 모든 조합을 생성한다.
      • K개가 선택되면 각 집에 대해 K개의 대피소 중 가장 가까운 거리를 구하고, 그 중 최댓값을 계산한다.
      • 모든 조합에서의 최댓값 중 최솟값을 answer에 유지한다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      for (int i = 0; i < N; i++) {
          int x = Integer.parseInt(st.nextToken());
          int y = Integer.parseInt(st.nextToken());
          houses.add(new House(x, y));
      }
      
      int[] shelter = new int[K];
      answer = Integer.MAX_VALUE;
      combine(0, 0, shelter);
      • N개의 집 좌표를 House 객체 리스트에 저장한다.
      • shelter는 현재 선택된 대피소의 인덱스를 저장하는 배열이다.
      • answer를 충분히 큰 값으로 초기화하고 조합 탐색을 시작한다.
    • 2) 백트래킹으로 조합 생성 
      for (int i = start; i < N; i++) {
          shelter[count] = i;
          combine(i + 1, count + 1, shelter);
      }
      • start부터 N-1까지 순서대로 대피소 후보를 선택하며 재귀 호출한다.
      • i + 1을 다음 시작점으로 넘겨 중복 없이 오름차순 조합만 생성한다.
    • 3) 조합 완성 시 최솟값·최댓값 계산 
      if (count == K) {
          int max = Integer.MIN_VALUE;
      
          for (int i = 0; i < N; i++) {
              int min = Integer.MAX_VALUE;
      
              for (int j = 0; j < K; j++) {
                  int shelterIdx = shelter[j];
                  int dist = dist(houses.get(i), houses.get(shelterIdx));
                  min = Math.min(min, dist);
              }
      
              max = Math.max(max, min);
          }
      
          answer = Math.min(answer, max);
          return;
      }
      • K개의 대피소가 선택되면 각 집 i에 대해 K개의 대피소 중 가장 가까운 거리 min을 구한다.
      • N개의 집 중 min의 최댓값 max가 현재 조합의 "최악의 집"이 대피소까지 가야 하는 거리다.
      • 모든 조합에서 max의 최솟값을 answer에 유지한다.
      • 예시: 집이 [(0,0), (5,5), (10,0)]이고 K=1이면 세 가지 조합 각각의 max를 비교해 최솟값을 구한다.
    • 4) 맨해튼 거리 계산 
      private static int dist(House houseOne, House houseTwo) {
          return Math.abs(houseOne.x - houseTwo.x) + Math.abs(houseOne.y - houseTwo.y);
      }
      • 두 집 사이의 맨해튼 거리를 |x1-x2| + |y1-y2|로 계산한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(C(N, K) × N × K): C(N, K)가지의 조합 각각에 대해 N개의 집과 K개의 대피소를 비교하는 O(N × K) 검증이 수행된다. N ≤ 10이므로 최악의 경우에도 C(10, 5) × 10 × 5 = 12,600번의 연산으로 매우 빠르다.
    • 공간 복잡도
      • O(N + K): N개의 집을 저장하는 리스트가 O(N), 현재 선택된 대피소 인덱스를 저장하는 배열이 O(K), 재귀 호출 스택이 최대 K 깊이로 O(K)이다.

    문제 28324 — 스케이트 연습

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    지뢰 찾기 (백준 4396)는
    n × n 크기의 지뢰 찾기 보드와 현재 열린 칸 정보가 주어질 때, 게임 결과판을 출력하는 문제다.
    열린 칸('x')에 지뢰가 있으면 게임 오버로, 모든 지뢰 위치를 '*'로 공개해야 한다.

    • 열린 칸('x')은 주변 8칸의 지뢰 수를 숫자로 표시하고, 열리지 않은 칸은 '.'으로 표시한다.
    • 열린 칸 중 지뢰('*')가 있으면 게임 오버이며, 이 경우 보드 전체에서 지뢰 위치를 '*'로 덮어씌워 출력한다.
    • 게임 오버가 아닌 경우에는 열린 칸의 주변 지뢰 수와 열리지 않은 칸('.')만 출력한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (8방향 탐색 + 게임 오버 판별)
    • 선택 이유: 각 칸에 대한 처리가 독립적이며, 열린 칸의 주변 지뢰 수 계산은 8방향 탐색으로 O(1)에 처리된다. n이 최대 8이므로 O(n²) 시뮬레이션으로 충분히 빠르다.
    • 세부 전략:
      • 보드를 순회하며 열린 칸('x')에 대해 8방향 지뢰 수를 세어 숫자로 기록하고, 열리지 않은 칸은 '.'으로 기록한다.
      • 열린 칸에 지뢰가 있으면 flag = true로 게임 오버를 표시한다.
      • 게임 오버이면 보드 전체를 다시 순회해 지뢰 위치를 '*'로 덮어씌운다.

    코드가 하는 일

    • 1) 입력 및 배열 구성 
      for (int i = 0; i < n; i++) {
          board[i] = br.readLine().toCharArray();
      }
      
      for (int i = 0; i < n; i++) {
          playing[i] = br.readLine().toCharArray();
      }
      • board는 실제 지뢰 배치('*' 또는 '.')를 저장하고, playing은 현재 열린 칸('x') 여부를 저장한다.
      • answer는 최종 출력 결과를 저장하는 배열이다.
    • 2) 열린 칸 처리 및 게임 오버 판별 
      if (playing[r][c] == 'x') {
          if (board[r][c] == '*') flag = true;
      
          int bomb = 0;
          for (int k = 0; k < 8; k++) {
              int row = r + ROW[k];
              int col = c + COL[k];
      
              if (row < 0 || row >= n || col < 0 || col >= n) continue;
              if (board[row][col] == '*') bomb++;
          }
          answer[r][c] = (char) (bomb + '0');
          continue;
      }
      answer[r][c] = '.';
      • 열린 칸에 지뢰가 있으면 flag = true로 게임 오버를 기록한다. 지뢰 수 계산과 게임 오버 판별을 동시에 처리해 한 번의 순회로 해결한다.
      • 8방향 탐색으로 주변 지뢰 수 bomb를 센 뒤 (char)(bomb + '0')으로 숫자 문자로 변환해 answer에 저장한다.
      • 열리지 않은 칸은 '.'으로 기록한다.
    • 3) 게임 오버 시 지뢰 위치 공개 
      if (flag) {
          for (int r = 0; r < n; r++) {
              for (int c = 0; c < n; c++) {
                  if (board[r][c] == '*') {
                      answer[r][c] = '*';
                  }
              }
          }
      }
      • 게임 오버인 경우 보드 전체를 다시 순회해 지뢰 위치를 '*'로 덮어씌운다.
      • 이미 숫자가 기록된 열린 칸이라도 지뢰가 있으면 '*'로 덮어쓴다. 지뢰를 밟은 칸도 '*'로 표시되어야 하기 때문이다.
    • 4) 결과 출력 
      for (char[] chars : answer) {
          sb.append(chars).append('\n');
      }
      System.out.println(sb);
      • 완성된 answer 배열의 각 행을 순서대로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n²): 보드 전체를 최대 두 번 순회하며(처리 + 게임 오버 시 지뢰 공개), 각 칸에서 8방향 탐색은 O(1)이므로 전체 O(n²)이다. n이 최대 8이므로 사실상 O(1)에 수렴한다.
    • 공간 복잡도
      • O(n²): 지뢰 배치, 열린 칸, 결과를 저장하는 세 개의 n × n 배열이 주요 메모리 사용처이며, n이 최대 8이므로 사실상 O(1)에 수렴한다.
