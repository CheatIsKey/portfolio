    문제 28215 — 대피소

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    대피소 (백준 28215)는
    N개의 집 중 K개를 대피소로 선정할 때, 각 집에서 가장 가까운 대피소까지의 거리 중 최댓값을 최소화하는 문제다.
    거리는 맨해튼 거리(|x1-x2| + |y1-y2|)로 계산한다.

    • N개 중 K개를 선택하는 모든 조합을 탐색해 최적 대피소 배치를 찾는다.
    • 각 조합에 대해 모든 집의 "가장 가까운 대피소까지의 거리" 중 최댓값을 구하고, 이를 최소화하는 조합이 정답이다.
    • N이 최대 10, K가 최대 N이므로 최악의 경우 C(10, 5) = 252가지 조합으로 브루트포스가 충분하다.

    전략 개요

    • 핵심 알고리즘: 백트래킹 조합 탐색 (완전 탐색)
    • 선택 이유: N이 최대 10으로 매우 작아 N개 중 K개를 선택하는 모든 조합을 탐색해도 최대 C(10, 5) = 252가지에 불과하다. 각 조합 검증에 O(N × K) = O(100)이 소요되므로 전체 탐색이 충분히 빠르다.
    • 세부 전략:
      • 재귀 함수 combine으로 N개 중 K개를 선택하는 모든 조합을 생성한다.
      • K개가 선택되면 각 집에 대해 K개의 대피소 중 가장 가까운 거리를 구하고, 그 중 최댓값을 계산한다.
      • 모든 조합에서의 최댓값 중 최솟값을 answer에 유지한다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      for (int i = 0; i < N; i++) {
          int x = Integer.parseInt(st.nextToken());
          int y = Integer.parseInt(st.nextToken());
          houses.add(new House(x, y));
      }
      
      int[] shelter = new int[K];
      answer = Integer.MAX_VALUE;
      combine(0, 0, shelter);
      • N개의 집 좌표를 House 객체 리스트에 저장한다.
      • shelter는 현재 선택된 대피소의 인덱스를 저장하는 배열이다.
      • answer를 충분히 큰 값으로 초기화하고 조합 탐색을 시작한다.
    • 2) 백트래킹으로 조합 생성 
      for (int i = start; i < N; i++) {
          shelter[count] = i;
          combine(i + 1, count + 1, shelter);
      }
      • start부터 N-1까지 순서대로 대피소 후보를 선택하며 재귀 호출한다.
      • i + 1을 다음 시작점으로 넘겨 중복 없이 오름차순 조합만 생성한다.
    • 3) 조합 완성 시 최솟값·최댓값 계산 
      if (count == K) {
          int max = Integer.MIN_VALUE;
      
          for (int i = 0; i < N; i++) {
              int min = Integer.MAX_VALUE;
      
              for (int j = 0; j < K; j++) {
                  int shelterIdx = shelter[j];
                  int dist = dist(houses.get(i), houses.get(shelterIdx));
                  min = Math.min(min, dist);
              }
      
              max = Math.max(max, min);
          }
      
          answer = Math.min(answer, max);
          return;
      }
      • K개의 대피소가 선택되면 각 집 i에 대해 K개의 대피소 중 가장 가까운 거리 min을 구한다.
      • N개의 집 중 min의 최댓값 max가 현재 조합의 "최악의 집"이 대피소까지 가야 하는 거리다.
      • 모든 조합에서 max의 최솟값을 answer에 유지한다.
      • 예시: 집이 [(0,0), (5,5), (10,0)]이고 K=1이면 세 가지 조합 각각의 max를 비교해 최솟값을 구한다.
    • 4) 맨해튼 거리 계산 
      private static int dist(House houseOne, House houseTwo) {
          return Math.abs(houseOne.x - houseTwo.x) + Math.abs(houseOne.y - houseTwo.y);
      }
      • 두 집 사이의 맨해튼 거리를 |x1-x2| + |y1-y2|로 계산한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(C(N, K) × N × K): C(N, K)가지의 조합 각각에 대해 N개의 집과 K개의 대피소를 비교하는 O(N × K) 검증이 수행된다. N ≤ 10이므로 최악의 경우에도 C(10, 5) × 10 × 5 = 12,600번의 연산으로 매우 빠르다.
    • 공간 복잡도
      • O(N + K): N개의 집을 저장하는 리스트가 O(N), 현재 선택된 대피소 인덱스를 저장하는 배열이 O(K), 재귀 호출 스택이 최대 K 깊이로 O(K)이다.

    문제 28324 — 스케이트 연습

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    스케이트 연습 (백준 28324)는
    N개의 구간마다 최대 속도 제한이 있을 때, 각 구간에서의 속도 합을 최대화하는 문제다.
    인접한 구간 사이에서 속도는 최대 1씩만 증가할 수 있으며, 각 구간의 속도는 제한을 초과할 수 없다.

    • 속도는 최대 1씩 증가할 수 있으므로, 뒤쪽 구간의 낮은 속도 제한이 앞쪽 구간의 속도에 영향을 준다.
    • 앞에서 뒤로 탐색하면 미래의 속도 제한을 알 수 없어 최적 속도를 결정하기 어렵다.
    • 뒤에서 앞으로 탐색하면 각 구간의 속도를 이전 구간(오른쪽) 속도 + 1과 현재 제한 중 최솟값으로 결정할 수 있어 최적임이 보장된다.

    전략 개요

    • 핵심 알고리즘: 그리디 (역방향 순회)
    • 선택 이유: 속도를 최대화하려면 각 구간에서 가능한 한 빠르게 달려야 한다. 뒤에서 앞으로 순회하면, 오른쪽 구간에서 달렸던 속도에 1을 더한 값과 현재 제한 중 작은 값이 현재 구간의 최적 속도가 된다. 이 결정이 항상 지역적으로도 전역적으로도 최적임이 보장된다.
    • 세부 전략:
      • 마지막 구간부터 역순으로 순회한다.
      • 각 구간에서 speed = min(speed + 1, restrict[i])로 현재 속도를 결정한다.
      • 결정된 속도를 answer에 누적한다.

    코드가 하는 일

    • 1) 입력 
      int[] restrict = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          restrict[i] = Integer.parseInt(st.nextToken());
      }
      • N개의 구간별 최대 속도 제한을 배열에 저장한다.
    • 2) 역방향 순회로 최적 속도 계산 
      long speed = 0;
      long answer = 0;
      
      for (int i = N - 1; i >= 0; i--) {
          speed = Math.min(speed + 1, restrict[i]);
          answer += speed;
      }
      • speed는 직전(오른쪽)에 처리한 구간에서의 속도를 나타낸다. 0으로 초기화하면 가장 마지막 구간은 제한 속도 이하에서 자유롭게 시작한다.
      • speed + 1은 오른쪽 구간 속도에서 1 증가한 값으로, 현재 구간에서 낼 수 있는 최대 속도다. 여기에 현재 제한 restrict[i]와 비교해 더 작은 값을 선택한다.
      • 뒤에서 앞으로 순회하므로 미래 제한이 현재 속도 결정에 자연스럽게 반영된다.
      • 예시: restrict = [5, 3, 10, 2, 8]을 역방향 처리하면 speed: 2→3→3→4→5. answer = 2+3+3+4+5=17.
    • 3) 결과 출력 
      System.out.println(answer);
      • 모든 구간의 속도 합 answer를 출력한다. 속도와 구간 수가 크면 합이 int 범위를 초과할 수 있으므로 long 타입으로 선언했다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 배열을 한 번 역순으로 순회하며 각 위치에서 O(1) 연산만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 속도 제한을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
