    문제 20551 — Sort 마스터 배지훈의 후계자

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Sort 마스터 배지훈의 후계자 (백준 20551)는
    N개의 정수로 이루어진 배열을 정렬한 뒤, M개의 쿼리마다 특정 값이 배열에 존재하면 그 첫 번째 인덱스를, 없으면 -1을 출력하는 문제다.
    쿼리마다 선형 탐색을 하면 시간 초과가 발생하므로, 정렬 후 이진 탐색으로 빠르게 처리해야 한다.

    • N과 M이 최대 100,000이므로 쿼리마다 O(N) 탐색을 하면 최대 10억 번의 연산이 발생해 시간 초과가 난다.
    • 배열을 미리 정렬하면 이진 탐색으로 각 쿼리를 O(log N)에 처리할 수 있다.
    • 중복 원소가 있을 수 있으므로, 목표값을 찾았더라도 더 왼쪽에 같은 값이 있을 수 있어 왼쪽 경계(leftmost) 이진 탐색을 사용해야 한다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 좌측 경계 이진 탐색 (Leftmost Binary Search)
    • 선택 이유: 배열을 한 번만 정렬해두면 이후 M개의 쿼리를 각각 O(log N)에 처리할 수 있어 전체 O((N + M) log N)으로 해결된다. 중복 원소가 있을 때 항상 가장 왼쪽 인덱스를 반환해야 하므로, 목표값을 찾아도 탐색 범위를 왼쪽으로 계속 좁히는 leftmost 방식을 사용한다.
    • 세부 전략:
      • N개의 입력을 배열에 저장한 뒤 Arrays.sort()로 오름차순 정렬한다.
      • 각 쿼리마다 binarySearch()를 호출해 목표값의 첫 번째 인덱스를 구한다.
      • 이진 탐색 중 numbers[mid] == target이면 result = mid로 기록하고, end = mid - 1로 좁혀 더 왼쪽에 같은 값이 있는지 계속 탐색한다.
      • 끝까지 찾지 못하면 result의 초깃값인 -1이 반환된다.
      • 결과는 StringBuilder에 누적하여 한 번에 출력해 I/O 비용을 줄인다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      int[] numbers = new int[N];
      
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(br.readLine());
      }
      
      Arrays.sort(numbers);
      • N개의 정수를 배열에 저장하고 오름차순으로 정렬한다. 이후 이진 탐색이 가능한 상태로 만든다.
    • 2) M개의 쿼리 처리 
      for (int i = 0; i < M; i++) {
          int target = Integer.parseInt(br.readLine());
          sb.append(binarySearch(numbers, target)).append("\n");
      }
      • 각 쿼리마다 목표값 target을 읽고 binarySearch()를 호출한 뒤, 결과를 StringBuilder에 누적한다.
      • System.out.println을 M번 호출하는 대신 StringBuilder로 모아 마지막에 한 번만 출력하여 I/O 성능을 높인다.
    • 3) 좌측 경계 이진 탐색 
      private static int binarySearch(int[] numbers, int target) {
          int start = 0;
          int end = numbers.length - 1;
          int result = -1;
      
          while (start <= end) {
              int mid = (start + end) / 2;
      
              if (numbers[mid] == target) {
                  result = mid;
                  end = mid - 1;
              } else if (numbers[mid] > target) {
                  end = mid - 1;
              } else {
                  start = mid + 1;
              }
          }
          return result;
      }
      • result = -1로 초기화하여, 탐색 중 목표값을 한 번도 못 찾으면 -1이 반환되도록 한다.
      • numbers[mid] == target일 때 result = mid로 후보를 기록하고, end = mid - 1로 탐색 범위를 왼쪽으로 좁혀 더 앞선 인덱스에 같은 값이 있는지 계속 확인한다.
      • 루프가 끝나면 result에는 목표값이 처음 등장하는 인덱스(없으면 -1)가 저장된다.
    • 4) 결과 출력 
      System.out.println(sb);
      • StringBuilder에 누적된 M개의 결과를 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O((N + M) log N): 배열 정렬에 O(N log N), M개의 쿼리 각각에 O(log N)의 이진 탐색이 수행되므로 전체 O(N log N + M log N) = O((N + M) log N)이다.
    • 공간 복잡도
      • O(N): 입력 배열 저장에 O(N)이 사용되며, 그 외 변수와 StringBuilder는 O(M) 수준으로 전체 O(N + M)이나 N과 M이 같은 차수이므로 O(N)으로 표기한다.

    문제 14929 — 귀찮아 (SIB)

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    귀찮아 (SIB) (백준 14929)는
    N개의 정수가 주어질 때, i < j인 모든 쌍 (i, j)에 대해 a[i] × a[j]의 합을 구하는 문제다.
    이중 반복문으로 모든 쌍을 직접 탐색하면 O(N²)으로 시간 초과가 발생하므로, 누적합을 활용해 O(N)으로 줄여야 한다.

    • N이 최대 100,000이므로 이중 반복문을 사용하면 최대 50억 번의 연산이 필요해 시간 초과가 난다.
    • i번째 원소 a[i]에 대해, a[i]와 곱해야 할 대상은 i+1부터 N-1까지의 원소들이다. 이는 전체 합에서 a[0]~a[i]를 뺀 값, 즉 a[i] 이후의 합계와 같다.
    • 전체 합 totalSum을 유지하면서 a[i]를 순서대로 빼나가면, 매 순간 totalSum이 현재 원소 이후 원소들의 합이 되어 단일 순회로 해결할 수 있다.

    전략 개요

    • 핵심 알고리즘: 누적합 활용 그리디 (단일 순회)
    • 선택 이유: a[i] × (a[i+1] + a[i+2] + ... + a[N-1])의 형태로 변환하면, 전체 합에서 왼쪽 원소를 순서대로 제거해나가는 방식으로 O(N)에 답을 구할 수 있다. 이중 반복문 대신 합산 한 번으로 동일한 결과를 얻는 핵심 수학적 관찰이다.
    • 세부 전략:
      • 입력을 받는 동시에 전체 합 totalSum을 계산한다.
      • 배열을 순회하면서, 현재 원소 x = a[i]를 totalSum에서 먼저 뺀다.
      • 그러면 totalSum은 a[i+1]~a[N-1]의 합이 되므로, answer += x * totalSum이 i < j인 모든 쌍의 기여분을 한 번에 누적한다.
      • 정수 곱의 합이 int 범위를 초과할 수 있으므로 long 타입으로 선언한다.

    코드가 하는 일

    • 1) 입력 및 전체 합 계산 
      int[] numbers = new int[n];
      long totalSum = 0;
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < n; i++) {
          int x = Integer.parseInt(st.nextToken());
          numbers[i] = x;
          totalSum += x;
      }
      • 입력을 받는 동시에 모든 원소의 합 totalSum을 구한다. 원소값이 크면 합이 int 범위를 초과할 수 있으므로 long으로 선언한다.
    • 2) 단일 순회로 모든 쌍의 곱의 합 계산 
      long answer = 0;
      for (int i = 0; i < n; i++) {
          int x = numbers[i];
          totalSum -= x;
          answer += x * totalSum;
      }
      • totalSum -= x: 현재 원소 x를 합에서 제거하면, totalSum은 a[i+1]~a[N-1]의 합이 된다.
      • answer += x * totalSum: a[i]와 i 이후 모든 원소의 곱을 한 번에 누적한다. 이는 이중 반복문의 안쪽 루프 전체를 단 한 번의 곱셈으로 대체하는 핵심 아이디어다.
      • 예시: [1, 2, 3]이면 i=0일 때 totalSum=5, answer+=1×5=5 / i=1일 때 totalSum=3, answer+=2×3=6 / i=2일 때 totalSum=0, answer+=3×0=0. 최종 answer=11. 실제 1×2+1×3+2×3=2+3+6=11로 일치한다.
    • 3) 결과 출력 
      System.out.println(answer);
      • 모든 i < j 쌍에 대한 a[i] × a[j]의 합 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 입력 시 전체 합을 계산하는 순회 1회, 답을 누적하는 순회 1회로 총 두 번의 O(N) 순회만 수행한다.
    • 공간 복잡도
      • O(N): 입력 배열 저장에 O(N)이 사용되며, 그 외 totalSum, answer 등은 O(1)의 상수 공간을 사용한다.
