    문제 13706 — 제곱근

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    제곱근 (백준 13706)는
    완전 제곱수 N이 주어질 때 √N을 구하는 문제다.
    N의 자릿수가 최대 1,000자리로 매우 크므로 BigInteger와 이진 탐색을 사용해야 한다.

    • N이 완전 제곱수임이 보장되므로 반드시 mid² == N인 mid가 존재한다.
    • N의 자릿수가 최대 1,000자리이므로 long은 물론 일반적인 부동소수점 연산도 사용할 수 없다.
    • 탐색 범위를 1부터 N으로 설정하고 mid²와 N을 비교하는 이진 탐색으로 O(log N)에 해결된다.

    전략 개요

    • 핵심 알고리즘: BigInteger 이진 탐색 (Binary Search on Answer)
    • 선택 이유: √N이 커질수록 mid²도 커지는 단조성이 성립하므로 이진 탐색을 적용할 수 있다. N이 최대 1,000자리의 큰 수이므로 Math.sqrt()의 부동소수점 오차를 피하기 위해 BigInteger 정수 연산으로 정확하게 비교한다.
    • 세부 전략:
      • 탐색 범위를 1부터 N으로 설정한다.
      • 중간값 mid = (low + high) / 2를 계산하고 mid²와 N을 비교한다.
      • mid² == N이면 즉시 mid를 출력하고 종료한다.
      • mid² < N이면 답이 더 크므로 탐색 범위를 오른쪽으로, mid² > N이면 왼쪽으로 좁힌다.

    코드가 하는 일

    • 1) 입력 및 탐색 범위 초기화 
      BigInteger N = new BigInteger(br.readLine());
      
      BigInteger low = BigInteger.ONE;
      BigInteger high = N;
      • N이 최대 1,000자리이므로 BigInteger로 입력받는다.
      • 탐색 범위를 1부터 N으로 설정한다. √N ≤ N이 항상 성립하므로 N이 상한값으로 충분하다.
    • 2) 이진 탐색으로 √N 탐색 
      while (low.compareTo(high) <= 0) {
          BigInteger mid = low.add(high).divide(BigInteger.valueOf(2));
          int compared = mid.multiply(mid).compareTo(N);
      
          if (compared == 0) {
              System.out.println(mid);
              return;
          } else if (compared < 0) {
              low = mid.add(BigInteger.ONE);
          } else if (compared > 0) {
              high = mid.subtract(BigInteger.ONE);
          }
      }
      • mid = (low + high) / 2를 BigInteger 연산으로 계산한다. (low + high) 계산 시 오버플로우가 없으므로 별도의 오버플로우 방지 처리가 필요 없다.
      • mid.multiply(mid).compareTo(N)으로 mid²와 N을 비교한다. compareTo가 0이면 같고, 음수이면 mid² < N, 양수이면 mid² > N이다.
      • N이 완전 제곱수임이 보장되므로 반드시 compared == 0인 경우가 존재하며, 그 즉시 mid를 출력하고 종료한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(D² × log N): D는 N의 자릿수(최대 1,000)다. 이진 탐색은 log₂N ≈ 3,321번 반복되며, 각 반복에서 BigInteger 곱셈이 O(D²)이므로 전체 O(D² × log N)이다.
    • 공간 복잡도
      • O(D): BigInteger 변수들이 각각 최대 D자리의 수를 저장하므로 O(D)의 공간을 사용한다.

    문제 1544 — 사이클 단어

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    사이클 단어 (백준 1544)는
    N개의 문자열이 주어질 때, 회전(rotation)이 동일한 문자열끼리는 같은 것으로 간주하여
    서로 다른 사이클 단어의 개수를 구하는 문제다.

    • 문자열 A를 회전하면 B가 되는지 판별하는 고전적 방법으로, A + A가 B를 포함하는지 확인한다.
    • 단, 두 문자열의 길이가 같아야 회전 관계일 수 있으므로 길이 동치 조건이 선행된다.
    • 이미 집합에 있는 문자열과 회전 관계라면 새 문자열은 같은 그룹으로 보아 집합에 추가하지 않는다.

    전략 개요

    • 핵심 알고리즘: HashSet + 문자열 회전 판별 (str + str 포함 검사)
    • 선택 이유: 회전 동치류(equivalence class)의 대표 원소만 집합에 유지하면, 마지막에 집합의 크기가 곧 서로 다른 그룹의 수가 된다. 회전 판별은 str + str에 s가 포함되는지로 O(L) 안에 확인할 수 있어 직관적이고 효율적이다.
    • 세부 전략:
      • 새 문자열 s를 읽을 때마다 집합 내 원소와 길이가 같은 것만 골라 회전 관계를 확인한다.
      • 회전 관계인 원소가 하나라도 있으면 s는 이미 집합에 대표가 있으므로 추가하지 않는다.
      • 그렇지 않으면 s를 새 동치류의 대표로 집합에 추가한다.
      • 최종적으로 set.size()가 정답이다.

    코드가 하는 일

    • 1) 입력 및 자료구조 초기화 
      int N = Integer.parseInt(br.readLine());
      Set<String> set = new HashSet<>();
      • 문자열의 개수 N을 읽고, 동치류 대표 문자열을 보관할 HashSet을 초기화한다.
    • 2) 각 문자열에 대해 회전 동치 여부 확인 
      for (int i = 0; i < N; i++) {
          String s = br.readLine();
          boolean flag = false;
      
          for (String str : set) {
              if (str.length() == s.length() && (str + str).contains(s)) {
                  flag = true;
                  break;
              }
          }
      
          if (!flag) {
              set.add(s);
          }
      }
      • s를 읽은 뒤 집합의 모든 원소 str을 순회한다.
      • str.length() == s.length(): 길이가 다른 경우 회전 관계일 수 없으므로 먼저 걸러낸다.
      • (str + str).contains(s): 길이 L인 문자열을 두 번 이어붙이면 가능한 모든 회전이 부분 문자열로 등장한다. Java의 contains는 내부적으로 indexOf를 사용하므로 O(L²) 이하의 비용으로 판별한다.
      • flag가 세워지면 이미 같은 그룹의 대표가 있다는 뜻이므로 추가하지 않고 넘어간다.
      • flag가 세워지지 않으면 s가 새로운 동치류이므로 집합에 대표로 추가한다.
    • 3) 결과 출력 
      System.out.println(set.size());
      • 집합에는 각 동치류의 대표 원소 하나씩만 남아 있으므로, set.size()가 곧 서로 다른 사이클 단어의 수다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N² · L): N개의 문자열 각각에 대해 집합 내 최대 N개의 원소와 비교하며, 회전 판별(str + str 생성 및 contains)에 O(L)이 소요된다. 따라서 전체 O(N² · L)이다. (L은 문자열의 최대 길이)
    • 공간 복잡도
      • O(N · L): 최악의 경우 N개의 문자열이 모두 서로 다른 동치류여서 집합에 N개가 저장되며, 각 문자열의 길이가 최대 L이므로 O(N · L)의 공간을 사용한다.
