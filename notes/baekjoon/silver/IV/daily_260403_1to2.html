    문제 11582 — 치킨 TOP N

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    치킨 TOP N (백준 11582)는
    N개의 점수를 K개의 그룹으로 나누어 각 그룹 내에서 독립적으로 정렬한 뒤,
    전체 배열을 순서대로 출력하는 문제다.

    • N개의 점수를 입력받은 뒤, 두 번째 입력값 K(isNum)으로 그룹 크기 group = N / K를 결정한다.
    • 배열을 앞에서부터 group 크기씩 K개의 구간으로 나누고, 각 구간을 오름차순으로 정렬한다.
    • 전체 구간 정렬이 아닌 부분 구간 정렬이므로, 그룹 경계를 넘는 원소 간의 순서는 바뀌지 않는다.

    전략 개요

    • 핵심 알고리즘: 구간 분할 + 부분 정렬 (Arrays.sort(arr, from, to))
    • 선택 이유: 전체 배열을 한꺼번에 정렬하지 않고, N/K 크기의 구간마다 독립적으로 정렬해야 하기 때문에 Java의 Arrays.sort(array, fromIndex, toIndex)를 활용해 범위를 지정한 부분 정렬을 수행한다.
    • 세부 전략:
      • 그룹 크기 group = N / isNum을 계산한다.
      • 인덱스 i를 group씩 증가시키며 scores[i, i + group) 범위를 정렬한다.
      • 정렬이 끝난 배열을 순서대로 출력한다.

    코드가 하는 일

    • 1) 입력 및 그룹 크기 결정 
      int N = Integer.parseInt(br.readLine());
      int[] scores = new int[N];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          scores[i] = Integer.parseInt(st.nextToken());
      }
      
      int isNum = Integer.parseInt(br.readLine());
      int group = N / isNum;
      • N개의 점수를 배열에 저장하고, 이후 입력받은 K(isNum)로 각 그룹의 크기 group = N / isNum을 계산한다.
      • 예시: N=6, isNum=2이면 group=3으로, 배열을 3개씩 2개의 그룹으로 나눈다.
    • 2) 그룹별 부분 정렬 
      for (int i = 0; i < N; i += group) {
          Arrays.sort(scores, i, group + i);
      }
      • i를 0부터 group씩 증가시키며 scores[i]부터 scores[i + group - 1]까지의 구간을 오름차순으로 정렬한다.
      • Arrays.sort(arr, fromIndex, toIndex)는 fromIndex 이상 toIndex 미만의 범위를 정렬하므로, group + i를 toIndex로 전달한다.
      • 각 그룹은 독립적으로 정렬되므로 그룹 경계를 넘는 원소 간의 상대적 위치는 변하지 않는다.
    • 3) 결과 출력 
      for (int score : scores) {
          sb.append(score).append(" ");
      }
      
      System.out.println(sb);
      • 그룹별 정렬이 완료된 배열을 순서대로 StringBuilder에 누적한 뒤 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log(N/K)): 크기 N/K인 그룹을 K번 정렬하므로, 각 정렬에 O((N/K) log(N/K))가 소요되고 이를 K번 반복하면 전체 O(N log(N/K))이다.
    • 공간 복잡도
      • O(N): 입력 점수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 2891 — 카약과 강풍

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    카약과 강풍 (백준 2891)는
    N팀이 카약을 타고 있을 때, 강풍으로 카약이 손상된 팀과 여분 카약을 가진 팀 정보를 바탕으로
    여분 카약을 인접한 팀에게 빌려주어 최종적으로 카약이 없는 팀의 수를 구하는 문제다.

    • 각 팀은 카약 1개로 시작하며, 손상 시 -1, 여분 보유 시 +1로 카약 수를 관리한다.
    • 여분 카약(count == 2)을 가진 팀은 인접한 팀(idx-1 또는 idx+1) 중 카약이 없는 팀(count == 0)에게만 빌려줄 수 있다.
    • 왼쪽 인접 팀을 우선으로 빌려주어야 중복 대여 없이 탐욕적으로 최적 배분이 가능하다.

    전략 개요

    • 핵심 알고리즘: 그리디 (카약 수 배열 시뮬레이션)
    • 선택 이유: 각 팀의 카약 보유 수를 배열로 관리하면, 손상·여분 정보를 단순 증감으로 처리할 수 있다. 이후 왼쪽부터 순서대로 여분 카약을 인접 팀에 배분하면, 한 번의 순회만으로 최적 대여 결과를 얻을 수 있어 그리디가 적합하다.
    • 세부 전략:
      • 모든 팀의 카약 수를 1로 초기화한 뒤, 손상 팀은 -1, 여분 팀은 +1을 적용한다.
      • 왼쪽부터 순회하며 count[idx] == 2인 팀이 있으면 왼쪽 인접 팀을 먼저 확인하고, 없으면 오른쪽 인접 팀에게 카약을 빌려준다.
      • 순회가 끝난 뒤 count[i] == 0인 팀의 수를 세어 출력한다.

    코드가 하는 일

    • 1) 초기화 및 손상·여분 반영 
      int[] count = new int[N + 1];
      Arrays.fill(count, 1);
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < S; i++) {
          int idx = Integer.parseInt(st.nextToken());
          count[idx]--;
      }
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < R; i++) {
          int idx = Integer.parseInt(st.nextToken());
          count[idx]++;
      }
      • 배열을 1로 초기화하여 모든 팀이 카약 1개를 가진 상태로 시작한다. 인덱스를 1-based로 맞추기 위해 크기를 N+1로 선언한다.
      • 카약이 손상된 팀은 count[idx]--로 0이 되고, 여분 카약을 가진 팀은 count[idx]++로 2가 된다.
    • 2) 인접 팀에게 여분 카약 배분 
      for (int idx = 1; idx < count.length; idx++) {
          if (count[idx] == 2) {
              int prev = idx - 1;
              int next = idx + 1;
              if (prev > 0 && count[prev] == 0) {
                  count[prev]++;
                  count[idx]--;
                  continue;
              }
              if (next <= N && count[next] == 0) {
                  count[next]++;
                  count[idx]--;
              }
          }
      }
      • count[idx] == 2인 팀은 여분 카약이 있으므로 인접 팀에게 빌려줄 수 있다.
      • 왼쪽 팀(prev)을 먼저 확인한다. 왼쪽에서 오른쪽으로 순회하기 때문에 왼쪽 팀을 우선 처리해야 이미 해결된 팀을 다시 건드리지 않는다.
      • 왼쪽 팀이 조건을 만족하지 않을 때만 오른쪽 팀(next)을 확인한다. 빌려주는 팀과 받는 팀 모두 카약 수가 1씩 조정된다.
    • 3) 카약 없는 팀 집계 및 출력 
      int answer = 0;
      for (int i = 1; i < count.length; i++) {
          if (count[i] == 0) {
              answer++;
          }
      }
      
      System.out.println(answer);
      • 배분이 끝난 뒤 count[i] == 0인 팀의 수를 세어 출력한다. 이 값이 끝내 카약을 구하지 못한 팀의 수다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 손상·여분 반영에 각각 O(S), O(R), 배분 및 집계에 O(N)이 소요되며, S와 R 모두 N 이하이므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 팀별 카약 수를 저장하는 배열 count가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
