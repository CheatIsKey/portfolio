    문제 9324 — 진짜 메시지

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    진짜 메시지 (백준 9324)는
    주어진 문자열이 특정 인코딩 규칙을 따르는지 검사하여 진짜(OK) 또는 가짜(FAKE) 메시지를 판별하는 문제다.
    어떤 문자가 3번 등장할 때마다 그 직후에 해당 문자가 한 번 더 반복되어야 유효한 메시지다.

    • 각 문자의 등장 횟수를 추적하다가 3의 배수가 되는 순간, 바로 다음 문자가 동일한 문자인지 확인한다.
    • 다음 문자가 없거나 다른 문자라면 규칙 위반으로 FAKE를 출력한다.
    • 규칙을 만족하는 반복 문자를 확인한 뒤에는 해당 문자의 카운트를 0으로 초기화하여 이후 등장을 다시 추적한다.

    전략 개요

    • 핵심 알고리즘: 문자 빈도 배열을 활용한 단일 순회 시뮬레이션
    • 선택 이유: 알파벳 26자의 등장 횟수만 추적하면 되므로 크기 26의 정수 배열로 충분하다. 문자열을 왼쪽에서 오른쪽으로 한 번만 순회하면서 3의 배수 시점마다 즉시 규칙을 검사하면 O(L)에 판별할 수 있다.
    • 세부 전략:
      • 문자를 하나씩 읽으며 해당 문자의 카운트를 증가시킨다.
      • 카운트가 3이 되면 바로 다음 인덱스의 문자가 동일한지 확인한다. 조건 불만족 시 즉시 FAKE를 기록하고 순회를 중단한다.
      • 조건을 만족하면 인덱스를 한 칸 추가로 전진(i++)하여 반복 문자를 건너뛰고, 카운트를 0으로 초기화한다.
      • 순회를 끝까지 마치면 OK를 기록한다.

    코드가 하는 일

    • 1) 테스트케이스별 초기화 
      int[] counts = new int[26];
      boolean flag = true;
      
      String word = br.readLine();
      • 테스트케이스마다 알파벳 26자의 등장 횟수를 저장할 배열 counts를 새로 초기화한다. flag는 메시지가 유효한지 추적하며, 위반이 발생하면 false로 전환된다.
    • 2) 문자열 순회 및 규칙 검사 
      for (int i = 0; i < word.length(); i++) {
          char c = word.charAt(i);
          counts[c - 'A']++;
      
          if (counts[c - 'A'] == 3) {
              if ((i + 1) >= word.length() || word.charAt(i + 1) != c) {
                  sb.append("FAKE").append('\n');
                  flag = false;
                  break;
              }
      
              i++;
              counts[c - 'A'] = 0;
          }
      }
      • counts[c - 'A']++: 현재 문자의 등장 횟수를 누적한다. 대문자 A를 기준으로 인덱스를 계산한다.
      • counts[c - 'A'] == 3: 해당 문자가 세 번째로 등장한 시점이다. 이 순간 바로 다음 문자를 확인해야 한다.
      • (i + 1) >= word.length() || word.charAt(i + 1) != c: 다음 문자가 존재하지 않거나 동일한 문자가 아니면 규칙 위반이므로 FAKE를 기록하고 break로 순회를 종료한다.
      • 규칙을 만족하면 i++로 반복 문자를 건너뛰고, counts[c - 'A'] = 0으로 해당 문자의 카운트를 초기화하여 이후 등장을 다시 추적한다.
    • 3) 결과 누적 및 출력 
      if (flag) sb.append("OK").append('\n');
      
      System.out.println(sb);
      • 순회를 마친 뒤 flag가 여전히 true이면 OK를 기록한다. 모든 테스트케이스의 결과를 StringBuilder에 누적한 뒤 한 번에 출력하여 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · L): T개의 테스트케이스마다 길이 L인 문자열을 한 번 순회하므로 전체 O(T · L)이다. (L은 문자열의 최대 길이)
    • 공간 복잡도
      • O(1): 알파벳 빈도를 저장하는 크기 26의 배열만 사용하며, 이는 입력 크기와 무관한 상수 공간이다.

    문제 16439 — 치킨치킨치킨

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    치킨치킨치킨 (백준 16439)는
    N명의 사람과 M종류의 치킨이 있을 때, 치킨 3가지를 선택하여 모든 사람의 만족도 합을 최대화하는 문제다.
    각 사람의 만족도는 선택된 3가지 치킨 중 자신이 가장 좋아하는 치킨의 점수로 결정된다.

    • M종류 중 3가지를 고르는 조합의 수는 최대 C(30, 3) = 4060으로 완전 탐색이 가능하다.
    • 각 치킨 조합에 대해 N명의 만족도 합을 계산하며, 한 사람의 만족도는 선택된 3가지 중 최댓값이다.
    • 모든 조합 중 만족도 합이 가장 큰 값을 출력한다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (3중 반복문으로 조합 열거)
    • 선택 이유: M이 최대 30이므로 3가지를 고르는 조합의 수는 C(30, 3) = 4060이고, 각 조합마다 N명(최대 30명)의 만족도를 계산하면 최대 약 12만 번의 연산으로 충분히 시간 내에 해결된다. 별도의 최적화 없이 단순 완전 탐색으로 정답을 구할 수 있다.
    • 세부 전략:
      • 3중 반복문으로 i < j < k 조건을 만족하는 치킨 인덱스 조합 (i, j, k)를 모두 열거한다.
      • 각 조합에 대해 N명의 만족도를 순회하며, 각 사람의 점수 중 최댓값을 합산한다.
      • 모든 조합의 합 중 최댓값을 갱신하여 출력한다.

    코드가 하는 일

    • 1) 입력 
      int[][] scores = new int[N][M];
      
      for (int i = 0; i < N; i++) {
          st = new StringTokenizer(br.readLine());
          for (int j = 0; j < M; j++) {
              scores[i][j] = Integer.parseInt(st.nextToken());
          }
      }
      • N×M 크기의 2차원 배열에 각 사람(행)의 치킨 종류별(열) 만족도 점수를 저장한다.
    • 2) 3가지 치킨 조합 완전 탐색 
      for (int i = 0; i < M; i++) {
          for (int j = i + 1; j < M; j++) {
              for (int k = j + 1; k < M; k++) {
                  int sum = 0;
      
                  for (int idx = 0; idx < N; idx++) {
                      int scoreOne = scores[idx][i];
                      int scoreTwo = scores[idx][j];
                      int scoreThree = scores[idx][k];
      
                      sum += Math.max(scoreOne, Math.max(scoreTwo, scoreThree));
                  }
      
                  max = Math.max(sum, max);
              }
          }
      }
      • i < j < k 조건의 3중 반복문으로 중복 없이 모든 치킨 조합을 열거한다.
      • 각 조합에 대해 N명을 순회하며, Math.max(scoreOne, Math.max(scoreTwo, scoreThree))로 해당 사람의 만족도(선택된 3가지 중 최댓값)를 구해 sum에 누적한다.
      • 조합별 합산이 끝나면 max와 비교하여 최댓값을 갱신한다.
    • 3) 결과 출력 
      System.out.println(max);
      • 모든 조합 중 만족도 합의 최댓값 max를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M³ · N): 치킨 조합을 열거하는 3중 반복문이 O(M³), 각 조합마다 N명의 만족도를 계산하므로 전체 O(M³ · N)이다. M ≤ 30, N ≤ 30이므로 최대 약 81,000번의 연산으로 충분하다.
    • 공간 복잡도
      • O(N · M): 만족도 점수를 저장하는 2차원 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
