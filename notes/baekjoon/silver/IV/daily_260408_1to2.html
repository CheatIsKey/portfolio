    문제 13567 — 로봇

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    로봇 (백준 13567)는
    M×M 크기의 격자 위에서 로봇에게 MOVE와 TURN 명령을 내릴 때, 모든 명령을 수행한 뒤 로봇의 최종 위치를 구하는 문제다.
    격자 범위를 벗어나는 순간 즉시 -1을 출력하고 종료해야 한다.

    • 로봇은 (0, 0)에서 출발하며, 초기 방향은 +x 방향(동쪽)이다.
    • MOVE 명령은 현재 방향으로 주어진 칸 수만큼 이동하고, TURN 명령은 오른쪽(0) 또는 왼쪽(1)으로 90도 회전한다.
    • 이동 후 좌표가 0 이상 M 이하를 벗어나면 -1을 출력하고 프로그램을 종료한다.

    전략 개요

    • 핵심 알고리즘: 방향 벡터 시뮬레이션
    • 선택 이유: 방향을 정수 인덱스(0~3)로 관리하고 dx, dy 배열로 방향 벡터를 정의하면, TURN 명령을 나머지 연산으로 간단히 처리할 수 있다. MOVE 명령은 현재 방향 벡터에 이동 거리를 곱해 좌표를 갱신하면 되므로 명령 수 n에 비례한 O(n) 시뮬레이션으로 해결된다.
    • 세부 전략:
      • 방향을 0(동), 1(북), 2(서), 3(남) 순서로 정의하고 dx, dy 배열로 각 방향의 이동량을 관리한다.
      • MOVE 명령 시 현재 방향의 벡터에 이동 거리를 곱해 좌표를 갱신하고, 범위를 벗어나면 즉시 -1 출력 후 종료한다.
      • TURN 0(오른쪽)은 (direction + 1) % 4, TURN 1(왼쪽)은 (direction + 3) % 4로 방향을 갱신한다.
      • 모든 명령을 처리한 뒤 최종 좌표를 출력한다.

    코드가 하는 일

    • 1) 초기 상태 설정 
      int[] curPos = {0, 0};
      int[] dx = {1, 0, -1, 0};
      int[] dy = {0, 1,  0, -1};
      int direction = 0;
      • 로봇의 초기 위치를 (0, 0)으로 설정한다. 방향 벡터 배열 dx, dy는 인덱스 0~3 순서로 동·북·서·남 방향을 나타낸다. 초기 방향은 0(동쪽, +x 방향)이다.
    • 2) MOVE 명령 처리 
      if (commend.equals("MOVE")) {
          curPos[0] += dx[direction] * number;
          curPos[1] += dy[direction] * number;
      
          if (curPos[0] < 0 || curPos[0] > M || curPos[1] < 0 || curPos[1] > M) {
              System.out.println(-1);
              return;
          }
      }
      • 현재 방향의 단위 벡터 (dx[direction], dy[direction])에 이동 거리 number를 곱해 좌표를 갱신한다.
      • 이동 후 좌표가 [0, M] 범위를 벗어나면 즉시 -1을 출력하고 return으로 프로그램을 종료한다.
    • 3) TURN 명령 처리 
      } else if (commend.equals("TURN")) {
          switch (number) {
              case 0 -> direction = (direction + 1) % 4;
              case 1 -> direction = ((direction + 4) - 1) % 4;
          }
      }
      • TURN 0(오른쪽 90도)은 (direction + 1) % 4로 방향 인덱스를 한 칸 증가시킨다.
      • TURN 1(왼쪽 90도)은 음수 인덱스를 방지하기 위해 (direction + 3) % 4로 방향 인덱스를 한 칸 감소시킨다.
    • 4) 최종 위치 출력 
      System.out.println(curPos[0] + " " + curPos[1]);
      • 모든 명령을 정상적으로 수행한 뒤 로봇의 최종 x, y 좌표를 공백으로 구분하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n): n개의 명령을 순서대로 한 번씩 처리하며, 각 명령에서 O(1) 연산만 수행하므로 전체 O(n)이다.
    • 공간 복잡도
      • O(1): 현재 위치, 방향 벡터 배열, 방향 인덱스 등 크기가 고정된 상수 공간만 사용한다.

    문제 13116 — 30번

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    최대 GCD (백준 9417)는
    여러 개의 정수가 주어질 때, 임의의 두 수를 골라 계산한 GCD 중 최댓값을 구하는 문제다.
    모든 쌍에 대해 GCD를 계산하는 완전 탐색으로 해결할 수 있다.

    • 한 줄에 주어지는 정수의 개수가 많지 않으므로, 모든 쌍을 열거하는 이중 반복문이 시간 내에 동작한다.
    • 두 수의 GCD는 유클리드 호제법으로 O(log min(a, b))에 계산한다.
    • 테스트케이스마다 독립적으로 처리하며, 결과를 StringBuilder에 누적하여 한 번에 출력한다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (모든 쌍 열거) + 유클리드 호제법 GCD
    • 선택 이유: 한 줄의 정수 개수가 제한적이므로 모든 쌍을 열거해도 충분히 빠르다. 각 쌍의 GCD는 재귀 유클리드 호제법으로 O(log min(a, b))에 계산하며, 최댓값을 갱신해 나가면 정답을 구할 수 있다.
    • 세부 전략:
      • 한 줄의 토큰을 모두 읽어 리스트에 저장한다.
      • 이중 반복문으로 i < j인 모든 쌍 (i, j)를 열거하며 GCD를 계산한다.
      • 계산된 GCD가 현재 최댓값보다 크면 갱신한다.
      • 테스트케이스가 끝나면 최댓값을 StringBuilder에 누적한다.

    코드가 하는 일

    • 1) 입력 파싱 
      StringTokenizer st = new StringTokenizer(br.readLine());
      
      List<Integer> list = new ArrayList<>();
      
      while (st.hasMoreTokens()) {
          int num = Integer.parseInt(st.nextToken());
          list.add(num);
      }
      • 한 줄에 공백으로 구분된 정수들을 StringTokenizer로 읽어 리스트에 저장한다. 정수의 개수가 고정되지 않으므로 hasMoreTokens()로 끝까지 읽는다.
    • 2) 모든 쌍에 대해 최대 GCD 탐색 
      int answer = 0;
      
      for (int i = 0; i < list.size() - 1; i++) {
          for (int j = i + 1; j < list.size(); j++) {
              int numOne = list.get(i);
              int numTwo = list.get(j);
      
              answer = Math.max(gcd(numOne, numTwo), answer);
          }
      }
      • i < j 조건의 이중 반복문으로 중복 없이 모든 쌍을 열거한다.
      • 각 쌍의 GCD를 계산하여 현재 최댓값 answer와 비교 후 갱신한다.
    • 3) 유클리드 호제법 GCD 
      private static int gcd(int numOne, int numTwo) {
          if (numTwo == 0) return numOne;
          return gcd(numTwo, numOne % numTwo);
      }
      • numTwo가 0이 될 때까지 재귀적으로 나머지 연산을 수행하는 유클리드 호제법이다. O(log min(numOne, numTwo))의 시간 복잡도로 GCD를 계산한다.
    • 4) 결과 누적 및 출력 
      sb.append(answer).append("\n");
      
      System.out.println(sb);
      • 테스트케이스별 최댓값을 StringBuilder에 누적한 뒤 한 번에 출력하여 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N · K² · log V): N개의 테스트케이스마다 K개의 정수에서 모든 쌍을 열거하는 O(K²)와 각 쌍의 GCD 계산 O(log V)가 수행된다. (K는 한 줄의 정수 개수, V는 정수의 최댓값)
    • 공간 복잡도
      • O(K): 테스트케이스마다 한 줄의 정수를 저장하는 리스트가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
