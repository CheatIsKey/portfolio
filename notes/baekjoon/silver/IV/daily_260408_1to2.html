    문제 20436 — ZOAC 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    ZOAC 3 (백준 20436)는
    왼손과 오른손의 초기 위치가 주어질 때, 문자열을 타이핑하는 데 필요한 총 이동 거리와 키 입력 횟수의 합을 구하는 문제다.
    자음은 왼손, 모음은 오른손으로만 입력하며, 각 키까지의 이동 거리는 키보드 배열 위에서의 맨해튼 거리로 계산한다.

    • 키보드는 3행으로 구성되며, 각 행의 열 위치를 기준으로 두 키 사이의 맨해튼 거리를 계산한다.
    • 자음 14개(q w e r t a s d f g z x c v)는 왼손, 나머지 모음 12개(y u i o p h j k l b n m)는 오른손으로 입력한다.
    • 한 키를 입력할 때마다 이동 거리에 1(키 누르는 비용)을 추가로 더한다.

    전략 개요

    • 핵심 알고리즘: 시뮬레이션 (맨해튼 거리 계산)
    • 선택 이유: 키보드 배열을 2차원 배열로 정의하고 각 키의 (행, 열) 좌표를 추출하면, 현재 손의 위치에서 목표 키까지의 이동 거리를 맨해튼 거리로 즉시 계산할 수 있다. 자음·모음 분류 배열로 어느 손을 사용할지 결정한 뒤 손의 위치를 갱신하며 순회하면 O(L)에 해결된다.
    • 세부 전략:
      • 왼손·오른손의 초기 위치를 extractPos()로 (행, 열) 좌표로 변환한다.
      • 문자열의 각 문자에 대해 자음 배열 CONSONANT에 속하면 왼손, 아니면 오른손을 사용한다.
      • 해당 손의 현재 위치에서 목표 키까지 맨해튼 거리를 구하고, 거리 + 1(키 누르기)을 answer에 누적한다.
      • 입력 후 해당 손의 위치를 목표 키 위치로 갱신한다.

    코드가 하는 일

    • 1) 키보드 배열 및 자음·모음 정의 
      static final char[][] KEYBOARD = {{'q', 'w', 'e', 'r', 't', 'y', 'u', 'i', 'o', 'p'},
              {'a', 's', 'd', 'f', 'g', 'h', 'j', 'k', 'l'},
              {'z', 'x', 'c', 'v', 'b', 'n', 'm'}};
      static final char[] CONSONANT = {'q', 'w', 'e', 'r', 't', 'a', 's', 'd', 'f', 'g', 'z', 'x', 'c', 'v'};
      static final char[] VOWEL = {'y', 'u', 'i', 'o', 'p', 'h', 'j', 'k', 'l', 'b', 'n', 'm'};
      • 실제 키보드 배열을 2차원 배열로 정의하여 각 키의 (행, 열) 좌표를 extractPos()로 탐색할 수 있게 한다.
      • CONSONANT에 속하면 왼손, 그렇지 않으면 오른손으로 입력하는 기준으로 사용한다.
    • 2) 초기 손 위치 설정 
      char left = st.nextToken().charAt(0);
      char right = st.nextToken().charAt(0);
      
      int[] curLeft = extractPos(left);
      int[] curRight = extractPos(right);
      • 왼손과 오른손의 초기 위치를 입력받아 extractPos()로 (행, 열) 좌표 배열로 변환한다.
    • 3) 문자열 순회 및 비용 계산 
      for (char c : word.toCharArray()) {
          int[] targetPos = extractPos(c);
          boolean isLeft = false;
      
          for (char isConsonant : CONSONANT) {
              if (c == isConsonant) {
                  isLeft = true;
                  break;
              }
          }
      
          if (isLeft) {
              int dist = Math.abs(targetPos[0] - curLeft[0]) + Math.abs(targetPos[1] - curLeft[1]);
              answer += dist + 1;
              curLeft = targetPos;
          } else {
              int dist = Math.abs(targetPos[0] - curRight[0]) + Math.abs(targetPos[1] - curRight[1]);
              answer += dist + 1;
              curRight = targetPos;
          }
      }
      • 각 문자에 대해 extractPos()로 목표 위치를 구하고, CONSONANT 배열 순회로 자음 여부를 판별해 사용할 손을 결정한다.
      • 맨해튼 거리 |row 차| + |col 차|에 키 누르기 비용 1을 더해 answer에 누적한다.
      • 입력 후 해당 손의 현재 위치(curLeft 또는 curRight)를 목표 위치로 갱신한다.
    • 4) 키 좌표 추출 메서드 
      private static int[] extractPos(char key) {
          for (int r = 0; r < KEYBOARD.length; r++) {
              for (int c = 0; c < KEYBOARD[r].length; c++) {
                  if (KEYBOARD[r][c] == key) {
                      return new int[]{r, c};
                  }
              }
          }
          return null;
      }
      • KEYBOARD 배열을 전체 탐색하여 목표 문자의 (행, 열) 좌표를 반환한다. 키보드 크기가 최대 26자로 고정이므로 O(1)에 수렴한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L): 문자열의 각 문자에 대해 자음 판별(최대 14번), 좌표 추출(최대 26번), 거리 계산(O(1))을 수행하므로 전체 O(L)이다. (L은 문자열의 길이)
    • 공간 복잡도
      • O(1): 키보드 배열과 자음·모음 배열은 크기가 고정된 상수 공간이며, 그 외 변수들도 O(1)의 상수 공간을 사용한다.

    문제 9417 — 최대 GCD

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    최대 GCD (백준 9417)는
    여러 개의 정수가 주어질 때, 임의의 두 수를 골라 계산한 GCD 중 최댓값을 구하는 문제다.
    모든 쌍에 대해 GCD를 계산하는 완전 탐색으로 해결할 수 있다.

    • 한 줄에 주어지는 정수의 개수가 많지 않으므로, 모든 쌍을 열거하는 이중 반복문이 시간 내에 동작한다.
    • 두 수의 GCD는 유클리드 호제법으로 O(log min(a, b))에 계산한다.
    • 테스트케이스마다 독립적으로 처리하며, 결과를 StringBuilder에 누적하여 한 번에 출력한다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (모든 쌍 열거) + 유클리드 호제법 GCD
    • 선택 이유: 한 줄의 정수 개수가 제한적이므로 모든 쌍을 열거해도 충분히 빠르다. 각 쌍의 GCD는 재귀 유클리드 호제법으로 O(log min(a, b))에 계산하며, 최댓값을 갱신해 나가면 정답을 구할 수 있다.
    • 세부 전략:
      • 한 줄의 토큰을 모두 읽어 리스트에 저장한다.
      • 이중 반복문으로 i < j인 모든 쌍 (i, j)를 열거하며 GCD를 계산한다.
      • 계산된 GCD가 현재 최댓값보다 크면 갱신한다.
      • 테스트케이스가 끝나면 최댓값을 StringBuilder에 누적한다.

    코드가 하는 일

    • 1) 입력 파싱 
      StringTokenizer st = new StringTokenizer(br.readLine());
      
      List<Integer> list = new ArrayList<>();
      
      while (st.hasMoreTokens()) {
          int num = Integer.parseInt(st.nextToken());
          list.add(num);
      }
      • 한 줄에 공백으로 구분된 정수들을 StringTokenizer로 읽어 리스트에 저장한다. 정수의 개수가 고정되지 않으므로 hasMoreTokens()로 끝까지 읽는다.
    • 2) 모든 쌍에 대해 최대 GCD 탐색 
      int answer = 0;
      
      for (int i = 0; i < list.size() - 1; i++) {
          for (int j = i + 1; j < list.size(); j++) {
              int numOne = list.get(i);
              int numTwo = list.get(j);
      
              answer = Math.max(gcd(numOne, numTwo), answer);
          }
      }
      • i < j 조건의 이중 반복문으로 중복 없이 모든 쌍을 열거한다.
      • 각 쌍의 GCD를 계산하여 현재 최댓값 answer와 비교 후 갱신한다.
    • 3) 유클리드 호제법 GCD 
      private static int gcd(int numOne, int numTwo) {
          if (numTwo == 0) return numOne;
          return gcd(numTwo, numOne % numTwo);
      }
      • numTwo가 0이 될 때까지 재귀적으로 나머지 연산을 수행하는 유클리드 호제법이다. O(log min(numOne, numTwo))의 시간 복잡도로 GCD를 계산한다.
    • 4) 결과 누적 및 출력 
      sb.append(answer).append("\n");
      
      System.out.println(sb);
      • 테스트케이스별 최댓값을 StringBuilder에 누적한 뒤 한 번에 출력하여 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N · K² · log V): N개의 테스트케이스마다 K개의 정수에서 모든 쌍을 열거하는 O(K²)와 각 쌍의 GCD 계산 O(log V)가 수행된다. (K는 한 줄의 정수 개수, V는 정수의 최댓값)
    • 공간 복잡도
      • O(K): 테스트케이스마다 한 줄의 정수를 저장하는 리스트가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
