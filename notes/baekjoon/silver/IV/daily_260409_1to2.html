    문제 13567 — 로봇

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    로봇 (백준 13567)는
    M×M 크기의 격자 위에서 로봇에게 MOVE와 TURN 명령을 내릴 때, 모든 명령을 수행한 뒤 로봇의 최종 위치를 구하는 문제다.
    격자 범위를 벗어나는 순간 즉시 -1을 출력하고 종료해야 한다.

    • 로봇은 (0, 0)에서 출발하며, 초기 방향은 +x 방향(동쪽)이다.
    • MOVE 명령은 현재 방향으로 주어진 칸 수만큼 이동하고, TURN 명령은 오른쪽(0) 또는 왼쪽(1)으로 90도 회전한다.
    • 이동 후 좌표가 0 이상 M 이하를 벗어나면 -1을 출력하고 프로그램을 종료한다.

    전략 개요

    • 핵심 알고리즘: 방향 벡터 시뮬레이션
    • 선택 이유: 방향을 정수 인덱스(0~3)로 관리하고 dx, dy 배열로 방향 벡터를 정의하면, TURN 명령을 나머지 연산으로 간단히 처리할 수 있다. MOVE 명령은 현재 방향 벡터에 이동 거리를 곱해 좌표를 갱신하면 되므로 명령 수 n에 비례한 O(n) 시뮬레이션으로 해결된다.
    • 세부 전략:
      • 방향을 0(동), 1(북), 2(서), 3(남) 순서로 정의하고 dx, dy 배열로 각 방향의 이동량을 관리한다.
      • MOVE 명령 시 현재 방향의 벡터에 이동 거리를 곱해 좌표를 갱신하고, 범위를 벗어나면 즉시 -1 출력 후 종료한다.
      • TURN 0(오른쪽)은 (direction + 1) % 4, TURN 1(왼쪽)은 (direction + 3) % 4로 방향을 갱신한다.
      • 모든 명령을 처리한 뒤 최종 좌표를 출력한다.

    코드가 하는 일

    • 1) 초기 상태 설정 
      int[] curPos = {0, 0};
      int[] dx = {1, 0, -1, 0};
      int[] dy = {0, 1,  0, -1};
      int direction = 0;
      • 로봇의 초기 위치를 (0, 0)으로 설정한다. 방향 벡터 배열 dx, dy는 인덱스 0~3 순서로 동·북·서·남 방향을 나타낸다. 초기 방향은 0(동쪽, +x 방향)이다.
    • 2) MOVE 명령 처리 
      if (commend.equals("MOVE")) {
          curPos[0] += dx[direction] * number;
          curPos[1] += dy[direction] * number;
      
          if (curPos[0] < 0 || curPos[0] > M || curPos[1] < 0 || curPos[1] > M) {
              System.out.println(-1);
              return;
          }
      }
      • 현재 방향의 단위 벡터 (dx[direction], dy[direction])에 이동 거리 number를 곱해 좌표를 갱신한다.
      • 이동 후 좌표가 [0, M] 범위를 벗어나면 즉시 -1을 출력하고 return으로 프로그램을 종료한다.
    • 3) TURN 명령 처리 
      } else if (commend.equals("TURN")) {
          switch (number) {
              case 0 -> direction = (direction + 1) % 4;
              case 1 -> direction = ((direction + 4) - 1) % 4;
          }
      }
      • TURN 0(오른쪽 90도)은 (direction + 1) % 4로 방향 인덱스를 한 칸 증가시킨다.
      • TURN 1(왼쪽 90도)은 음수 인덱스를 방지하기 위해 (direction + 3) % 4로 방향 인덱스를 한 칸 감소시킨다.
    • 4) 최종 위치 출력 
      System.out.println(curPos[0] + " " + curPos[1]);
      • 모든 명령을 정상적으로 수행한 뒤 로봇의 최종 x, y 좌표를 공백으로 구분하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n): n개의 명령을 순서대로 한 번씩 처리하며, 각 명령에서 O(1) 연산만 수행하므로 전체 O(n)이다.
    • 공간 복잡도
      • O(1): 현재 위치, 방향 벡터 배열, 방향 인덱스 등 크기가 고정된 상수 공간만 사용한다.

    문제 13116 — 30번

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    30번 (백준 13116)는
    완전 이진 트리에서 두 노드 A, B가 주어질 때, 두 노드의 최소 공통 조상(LCA)을 구하고 그 노드 번호에 10을 곱한 값을 출력하는 문제다.
    트리를 배열로 명시적으로 구성하지 않고, 부모 노드 번호가 항상 자식 노드 번호의 절반이라는 완전 이진 트리의 성질만으로 LCA를 구할 수 있다.

    • 완전 이진 트리에서 노드 n의 부모는 항상 n / 2이므로, 두 노드를 번갈아 부모로 올리다 보면 반드시 공통 조상에서 만난다.
    • 두 노드 중 더 큰 번호를 가진 노드를 먼저 부모로 올리면, 항상 같은 깊이로 수렴하며 결국 동일한 노드에서 만난다.
    • LCA 노드 번호에 10을 곱한 값이 문제의 정답이다.

    전략 개요

    • 핵심 알고리즘: 완전 이진 트리의 부모 노드 성질을 이용한 LCA 탐색
    • 선택 이유: 완전 이진 트리에서 부모 노드 번호는 항상 현재 노드 번호를 2로 나눈 값이다. 따라서 별도의 트리 구성 없이 두 노드를 번갈아 부모로 올리는 단순 반복만으로 O(log N)에 LCA를 구할 수 있다.
    • 세부 전략:
      • A와 B가 같아질 때까지 반복한다.
      • 매 반복에서 A와 B 중 더 큰 값을 2로 나눠 부모 노드로 이동시킨다.
      • A == B가 되는 순간이 LCA이며, 이 값에 10을 곱해 출력한다.

    코드가 하는 일

    • 1) 입력 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int A = Integer.parseInt(st.nextToken());
      int B = Integer.parseInt(st.nextToken());
      • 테스트케이스마다 두 노드 번호 A, B를 입력받는다.
    • 2) LCA 탐색 
      while (A != B) {
          if (A > B) {
              A /= 2;
          } else {
              B /= 2;
          }
      }
      • A와 B가 같아질 때까지 더 큰 노드를 2로 나눠 부모로 이동시킨다. 완전 이진 트리에서 노드 번호가 클수록 더 깊은 레벨에 있으므로, 큰 쪽을 먼저 올리면 두 노드의 깊이를 맞춰가며 공통 조상으로 수렴한다.
      • 예시: A=12, B=5일 때 → A=6, B=5 → A=3, B=5 → A=3, B=2 → A=3, B=1 → ... 순으로 진행되며 결국 A == B인 LCA에 도달한다.
    • 3) 결과 누적 및 출력 
      sb.append(A * 10).append('\n');
      
      System.out.println(sb);
      • LCA 노드 번호 A에 10을 곱한 값을 StringBuilder에 누적한 뒤 모든 테스트케이스가 끝나면 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · log N): T개의 테스트케이스마다 두 노드를 루트 방향으로 올리는 반복이 최대 트리 높이인 O(log N)번 수행된다. (N은 노드 번호의 최댓값)
    • 공간 복잡도
      • O(1): 트리를 별도로 구성하지 않고 두 정수 변수 A, B만으로 LCA를 추적하므로 상수 공간만 사용한다.
