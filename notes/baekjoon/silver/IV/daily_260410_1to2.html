    문제 15736 — 청기 백기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    청기 백기 (백준 15736)는
    N개의 깃발이 모두 내려진 상태에서 시작하여 i번째 명령마다 i의 배수 번호 깃발을 토글할 때,
    N번의 명령이 끝난 뒤 올라가 있는 깃발의 수를 구하는 문제다.
    약수의 개수가 홀수인 번호, 즉 완전제곱수에 해당하는 깃발만 최종적으로 올라가 있다.

    • k번 깃발은 k의 약수 개수만큼 토글된다. 약수는 보통 쌍으로 존재하므로 대부분의 깃발은 짝수 번 토글되어 원래 상태(내려짐)로 돌아온다.
    • k가 완전제곱수이면 √k가 약수 쌍의 중복 약수가 되어 약수의 개수가 홀수가 되고, 홀수 번 토글된 깃발은 올라간 채로 남는다.
    • 따라서 1부터 N까지의 완전제곱수의 개수, 즉 ⌊√N⌋이 정답이다.

    전략 개요

    • 핵심 알고리즘: 수학적 관찰 (완전제곱수의 개수 = ⌊√N⌋)
    • 선택 이유: 시뮬레이션으로 N개의 깃발을 직접 토글하면 O(N log N)이 필요하지만, 올라가 있는 깃발이 완전제곱수 번호뿐이라는 수학적 성질을 이용하면 Math.sqrt(N) 한 번으로 O(1)에 답을 구할 수 있다.
    • 세부 전략:
      • N을 입력받는다.
      • (int) Math.sqrt(N)으로 ⌊√N⌋을 계산한다.
      • 이 값이 1부터 N까지의 완전제곱수의 개수이자 최종적으로 올라가 있는 깃발의 수다.

    코드가 하는 일

    • 1) 입력 
      int N = Integer.parseInt(br.readLine());
      • 깃발의 수이자 명령의 횟수 N을 입력받는다.
    • 2) 완전제곱수 개수 계산 및 출력 
      System.out.println((int) Math.sqrt(N));
      • Math.sqrt(N)으로 √N을 구하고 (int)로 내림 처리하여 ⌊√N⌋을 얻는다. 이 값이 1부터 N까지 존재하는 완전제곱수의 개수이며, 최종적으로 올라가 있는 깃발의 수와 같다.
      • 예시: N=16이면 √16=4이므로 완전제곱수는 1, 4, 9, 16으로 4개. N=10이면 √10≈3.16이므로 완전제곱수는 1, 4, 9로 3개.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 입력을 받고 Math.sqrt() 한 번을 호출하는 것이 전부이므로 상수 시간이다.
    • 공간 복잡도
      • O(1): 정수 변수 N 하나만 사용하므로 상수 공간만 필요하다.

    문제 2799 — 블라인드

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    30번 (백준 13116)는
    완전 이진 트리에서 두 노드 A, B가 주어질 때, 두 노드의 최소 공통 조상(LCA)을 구하고 그 노드 번호에 10을 곱한 값을 출력하는 문제다.
    트리를 배열로 명시적으로 구성하지 않고, 부모 노드 번호가 항상 자식 노드 번호의 절반이라는 완전 이진 트리의 성질만으로 LCA를 구할 수 있다.

    • 완전 이진 트리에서 노드 n의 부모는 항상 n / 2이므로, 두 노드를 번갈아 부모로 올리다 보면 반드시 공통 조상에서 만난다.
    • 두 노드 중 더 큰 번호를 가진 노드를 먼저 부모로 올리면, 항상 같은 깊이로 수렴하며 결국 동일한 노드에서 만난다.
    • LCA 노드 번호에 10을 곱한 값이 문제의 정답이다.

    전략 개요

    • 핵심 알고리즘: 완전 이진 트리의 부모 노드 성질을 이용한 LCA 탐색
    • 선택 이유: 완전 이진 트리에서 부모 노드 번호는 항상 현재 노드 번호를 2로 나눈 값이다. 따라서 별도의 트리 구성 없이 두 노드를 번갈아 부모로 올리는 단순 반복만으로 O(log N)에 LCA를 구할 수 있다.
    • 세부 전략:
      • A와 B가 같아질 때까지 반복한다.
      • 매 반복에서 A와 B 중 더 큰 값을 2로 나눠 부모 노드로 이동시킨다.
      • A == B가 되는 순간이 LCA이며, 이 값에 10을 곱해 출력한다.

    코드가 하는 일

    • 1) 입력 
      StringTokenizer st = new StringTokenizer(br.readLine());
      int A = Integer.parseInt(st.nextToken());
      int B = Integer.parseInt(st.nextToken());
      • 테스트케이스마다 두 노드 번호 A, B를 입력받는다.
    • 2) LCA 탐색 
      while (A != B) {
          if (A > B) {
              A /= 2;
          } else {
              B /= 2;
          }
      }
      • A와 B가 같아질 때까지 더 큰 노드를 2로 나눠 부모로 이동시킨다. 완전 이진 트리에서 노드 번호가 클수록 더 깊은 레벨에 있으므로, 큰 쪽을 먼저 올리면 두 노드의 깊이를 맞춰가며 공통 조상으로 수렴한다.
      • 예시: A=12, B=5일 때 → A=6, B=5 → A=3, B=5 → A=3, B=2 → A=3, B=1 → ... 순으로 진행되며 결국 A == B인 LCA에 도달한다.
    • 3) 결과 누적 및 출력 
      sb.append(A * 10).append('\n');
      
      System.out.println(sb);
      • LCA 노드 번호 A에 10을 곱한 값을 StringBuilder에 누적한 뒤 모든 테스트케이스가 끝나면 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · log N): T개의 테스트케이스마다 두 노드를 루트 방향으로 올리는 반복이 최대 트리 높이인 O(log N)번 수행된다. (N은 노드 번호의 최댓값)
    • 공간 복잡도
      • O(1): 트리를 별도로 구성하지 않고 두 정수 변수 A, B만으로 LCA를 추적하므로 상수 공간만 사용한다.
