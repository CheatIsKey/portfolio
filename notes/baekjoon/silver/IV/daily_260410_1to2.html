    문제 15736 — 청기 백기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    청기 백기 (백준 15736)는
    N개의 깃발이 모두 내려진 상태에서 시작하여 i번째 명령마다 i의 배수 번호 깃발을 토글할 때,
    N번의 명령이 끝난 뒤 올라가 있는 깃발의 수를 구하는 문제다.
    약수의 개수가 홀수인 번호, 즉 완전제곱수에 해당하는 깃발만 최종적으로 올라가 있다.

    • k번 깃발은 k의 약수 개수만큼 토글된다. 약수는 보통 쌍으로 존재하므로 대부분의 깃발은 짝수 번 토글되어 원래 상태(내려짐)로 돌아온다.
    • k가 완전제곱수이면 √k가 약수 쌍의 중복 약수가 되어 약수의 개수가 홀수가 되고, 홀수 번 토글된 깃발은 올라간 채로 남는다.
    • 따라서 1부터 N까지의 완전제곱수의 개수, 즉 ⌊√N⌋이 정답이다.

    전략 개요

    • 핵심 알고리즘: 수학적 관찰 (완전제곱수의 개수 = ⌊√N⌋)
    • 선택 이유: 시뮬레이션으로 N개의 깃발을 직접 토글하면 O(N log N)이 필요하지만, 올라가 있는 깃발이 완전제곱수 번호뿐이라는 수학적 성질을 이용하면 Math.sqrt(N) 한 번으로 O(1)에 답을 구할 수 있다.
    • 세부 전략:
      • N을 입력받는다.
      • (int) Math.sqrt(N)으로 ⌊√N⌋을 계산한다.
      • 이 값이 1부터 N까지의 완전제곱수의 개수이자 최종적으로 올라가 있는 깃발의 수다.

    코드가 하는 일

    • 1) 입력 
      int N = Integer.parseInt(br.readLine());
      • 깃발의 수이자 명령의 횟수 N을 입력받는다.
    • 2) 완전제곱수 개수 계산 및 출력 
      System.out.println((int) Math.sqrt(N));
      • Math.sqrt(N)으로 √N을 구하고 (int)로 내림 처리하여 ⌊√N⌋을 얻는다. 이 값이 1부터 N까지 존재하는 완전제곱수의 개수이며, 최종적으로 올라가 있는 깃발의 수와 같다.
      • 예시: N=16이면 √16=4이므로 완전제곱수는 1, 4, 9, 16으로 4개. N=10이면 √10≈3.16이므로 완전제곱수는 1, 4, 9로 3개.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 입력을 받고 Math.sqrt() 한 번을 호출하는 것이 전부이므로 상수 시간이다.
    • 공간 복잡도
      • O(1): 정수 변수 N 하나만 사용하므로 상수 공간만 필요하다.

    문제 2799 — 블라인드

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    블라인드 (백준 2799)는
    M×N 규모의 아파트 단지를 5×5 문자 블록으로 표현한 격자가 주어질 때,
    각 호실에서 내려진 블라인드 수(0~4개)별 호실 수를 구하는 문제다.

    • 전체 격자는 (5M+1)행 × (5N+1)열이며, 각 호실의 내부는 5행 5열 블록 안에 표현된다.
    • 각 호실의 블라인드는 블록 내부 첫 번째 열(j번째 열)의 위쪽부터 연속된 '*'의 수로 나타난다. 연속이 끊기면 그 아래는 세지 않는다.
    • 블록 경계(0, 5, 10, ... 번째 행/열)는 구분선이므로, 호실의 시작점은 (1, 1), (1, 6), (6, 1), ... 과 같이 1-based로 5 간격이다.

    전략 개요

    • 핵심 알고리즘: 2차원 배열 시뮬레이션 (5 간격 블록 탐색)
    • 선택 이유: 격자를 그대로 2차원 배열에 저장한 뒤, 5 간격으로 각 호실의 시작점을 찾아 블라인드 수를 세면 별도의 자료구조 없이 O(M·N) 순회로 해결된다.
    • 세부 전략:
      • 전체 격자를 char[][]에 저장한다.
      • 행과 열 모두 1부터 시작해 5 간격으로 순회하며 각 호실의 시작점 (i, j)를 찾는다.
      • 해당 열(j)에서 위쪽부터 최대 4칸(k = 0~3)을 확인하며 연속된 '*'의 수를 센다. '*'가 아닌 순간 즉시 중단한다.
      • 센 수를 인덱스로 하여 answer 배열을 증가시킨다.

    코드가 하는 일

    • 1) 격자 크기 계산 및 입력 
      int row = 5 * M + 1;
      int col = 5 * N + 1;
      
      char[][] apt = new char[row][col];
      
      for (int i = 0; i < row; i++) {
          st = new StringTokenizer(br.readLine());
          String s = st.nextToken();
      
          for (int j = 0; j < col; j++) {
              apt[i][j] = s.charAt(j);
          }
      }
      • M×N 아파트 단지는 구분선을 포함해 (5M+1)행 × (5N+1)열의 격자로 표현된다. 각 행의 문자열을 읽어 char[][]에 저장한다.
    • 2) 호실별 블라인드 수 집계 
      for (int i = 1; i < row; i += 5) {
          for (int j = 1; j < col; j += 5) {
              int cnt = 0;
      
              for (int k = 0; k < 4; k++) {
                  if (apt[i + k][j] == '*') cnt++;
                  else break;
              }
      
              answer[cnt]++;
          }
      }
      • i와 j를 각각 1부터 5 간격으로 순회하여 각 호실 블록의 시작점을 찾는다. 구분선(0번째 행/열)을 건너뛰고 내부 첫 칸에서 시작하는 방식이다.
      • 해당 열 j에서 k = 0~3까지 최대 4칸을 위에서 아래로 확인하며 연속된 '*'를 센다. '*'가 아니면 즉시 break하여 불필요한 탐색을 막는다.
      • 센 수 cnt를 인덱스로 하여 answer[cnt]를 증가시킨다.
    • 3) 결과 출력 
      for (int i : answer) {
          sb.append(i).append(" ");
      }
      
      System.out.println(sb);
      • 블라인드 수 0개부터 4개까지의 호실 수를 순서대로 공백으로 구분하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M · N): 입력 저장에 O(M · N), 호실 탐색에 각 호실당 최대 4번의 상수 연산이 수행되므로 전체 O(M · N)이다.
    • 공간 복잡도
      • O(M · N): 전체 격자를 저장하는 char[][]가 주요 메모리 사용처이며, 그 외 answer 배열은 크기 5의 고정 배열로 O(1)이다.
