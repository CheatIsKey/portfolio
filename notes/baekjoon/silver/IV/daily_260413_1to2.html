    문제 14468 — 소가 길을 건너간 이유 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    소가 길을 건너간 이유 2 (백준 14468)는
    26마리의 소가 각각 두 번씩 등장하는 52자리 문자열이 주어질 때, 서로 교차하며 길을 건너는 소 쌍의 수를 구하는 문제다.
    소 A의 첫 번째와 두 번째 등장 사이에 소 B가 홀수 번 등장하면, A와 B는 교차하는 관계다.

    • 소 A의 구간 (start, end) 사이에 소 B가 정확히 한 번만 등장하면 A와 B는 교차한다. 두 번 등장하면 교차하지 않는다.
    • 따라서 구간 내에서 각 소의 등장 횟수를 토글 방식으로 추적하면, 최종적으로 true인 소가 A와 교차하는 소다.
    • 이중 카운팅을 방지하기 위해 j > i인 소에 대해서만 교차 여부를 집계한다.

    전략 개요

    • 핵심 알고리즘: 구간 내 토글 플래그를 이용한 교차 쌍 카운팅
    • 선택 이유: 소의 종류가 26가지로 고정이므로, 각 소의 구간 내에서 다른 소의 등장을 토글 방식으로 추적하면 O(26 × 52) = O(1)에 수렴하는 상수 시간 안에 모든 교차 쌍을 셀 수 있다.
    • 세부 전략:
      • 52자리 문자열을 순회하며 각 소의 첫 번째·두 번째 등장 위치를 cows[26][2]에 저장한다.
      • 소 i의 구간 (start+1, end) 사이를 순회하며 등장하는 소마다 flags를 토글한다.
      • 구간 순회가 끝난 뒤 flags[j]가 true이고 j > i이면 교차 쌍으로 집계한다.

    코드가 하는 일

    • 1) 입력 및 각 소의 등장 위치 기록 
      char[] logs = br.readLine().toCharArray();
      int[][] cows = new int[26][2];
      
      for (int i = 0; i < 26; i++) {
          Arrays.fill(cows[i], -1);
      }
      
      for (int i = 0; i < 52; i++) {
          int idx = logs[i] - 'A';
      
          if (cows[idx][0] == -1) {
              cows[idx][0] = i;
          } else {
              cows[idx][1] = i;
          }
      }
      • 52자리 문자열을 char[]로 변환한다. cows[i][0]은 소 i의 첫 번째 등장 위치, cows[i][1]은 두 번째 등장 위치를 저장한다. 초깃값 -1로 채워 아직 등장하지 않은 소를 구분한다.
    • 2) 각 소의 구간 내 교차 소 탐색 
      for (int i = 0; i < 26; i++) {
          int start = cows[i][0];
          int end = cows[i][1];
          boolean[] flags = new boolean[26];
      
          for (int s = start + 1; s < end; s++) {
              flags[logs[s] - 'A'] = !flags[logs[s] - 'A'];
          }
      
          for (int j = i + 1; j < 26; j++) {
              if (flags[j]) count++;
          }
      }
      • 소 i의 구간 (start, end) 사이를 순회하며 등장하는 소마다 flags를 토글한다. 짝수 번 등장하면 false(교차 아님), 홀수 번 등장하면 true(교차)가 된다.
      • 구간 순회 후 j > i인 소에 대해서만 flags[j]가 true인지 확인하여 이중 카운팅을 방지한다.
      • 등장하지 않은 소(start == -1)는 flags가 모두 false로 유지되어 집계에 영향을 주지 않는다.
    • 3) 결과 출력 
      System.out.println(count);
      • 교차하는 소 쌍의 총 수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 소의 종류(26)와 문자열 길이(52)가 모두 고정 상수이므로, 모든 반복문의 순회 횟수가 상수로 결정된다. 전체 연산은 O(26 × 52) = O(1)에 수렴한다.
    • 공간 복잡도
      • O(1): logs(52), cows(26×2), flags(26) 모두 고정 크기 배열이므로 입력 크기와 무관한 상수 공간만 사용한다.

    문제 1895 — 필터

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    필터 (백준 1895)는
    R×C 크기의 픽셀 배열에 중앙값 필터를 적용할 때, 3×3 윈도우의 중앙값이 임계값 T 이상인 픽셀의 수를 구하는 문제다.
    경계를 제외한 내부 픽셀 각각에 대해 3×3 이웃 픽셀 9개의 중앙값을 계산하여 T와 비교한다.

    • 중앙값 필터는 3×3 윈도우 내 9개 픽셀 값을 정렬했을 때 5번째(인덱스 4) 값, 즉 중앙값을 해당 위치의 결과값으로 사용한다.
    • 3×3 윈도우를 적용하려면 상하좌우로 1칸의 여백이 필요하므로, 순회 범위는 행 1~R-2, 열 1~C-2로 제한된다.
    • 9방향 오프셋을 상수 배열 ROW, COL로 정의하면 윈도우 수집 로직을 간결하게 표현할 수 있다.

    전략 개요

    • 핵심 알고리즘: 슬라이딩 윈도우 + 정렬을 통한 중앙값 계산
    • 선택 이유: 윈도우 크기가 3×3으로 고정이므로 매 위치에서 9개의 값을 정렬하는 비용이 O(9 log 9) = O(1)이다. 전체 내부 픽셀 수는 (R-2)×(C-2)이므로 전체 시간 복잡도는 O(R·C)로 충분히 빠르다.
    • 세부 전략:
      • 9방향 오프셋 배열 ROW, COL로 3×3 윈도우의 각 셀을 순서대로 수집한다.
      • 수집한 9개 값을 정렬하여 인덱스 4의 중앙값을 구한다.
      • 중앙값이 T 이상이면 count를 증가시킨다.

    코드가 하는 일

    • 1) 방향 오프셋 정의 및 입력 
      static final int[] ROW = {-1, -1, -1, 0, 0, 0, 1, 1, 1};
      static final int[] COL = {-1, 0, 1, -1, 0, 1, -1, 0, 1};
      • 3×3 윈도우의 9개 셀을 나타내는 행·열 오프셋을 상수 배열로 정의한다. 왼쪽 위부터 오른쪽 아래 순서로 나열되어 있으며, ROW[k]와 COL[k]를 현재 위치에 더하면 k번째 이웃 셀의 좌표가 된다.
    • 2) 내부 픽셀 순회 및 중앙값 계산 
      for (int r = 1; r < R - 1; r++) {
          for (int c = 1; c < C - 1; c++) {
      
              for (int k = 0; k < 9; k++) {
                  numbers[k] = pixel[r + ROW[k]][c + COL[k]];
              }
      
              Arrays.sort(numbers);
      
              if (numbers[4] >= T) {
                  count++;
              }
          }
      }
      • r을 1부터 R-2, c를 1부터 C-2까지 순회하여 3×3 윈도우를 온전히 적용할 수 있는 내부 픽셀만 처리한다.
      • 오프셋 배열을 이용해 현재 위치 (r, c)를 중심으로 한 3×3 윈도우의 픽셀 9개를 numbers에 수집한다.
      • Arrays.sort(numbers)로 9개 값을 정렬한 뒤 인덱스 4(중앙값)를 T와 비교한다. 중앙값이 T 이상이면 count를 증가시킨다.
    • 3) 결과 출력 
      System.out.println(count);
      • 중앙값이 T 이상인 픽셀의 총 수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(R · C): 내부 픽셀 (R-2)×(C-2)개를 순회하며 각 위치에서 9개 값 수집 및 정렬이 O(9 log 9) = O(1)이므로 전체 O(R · C)이다.
    • 공간 복잡도
      • O(R · C): 입력 픽셀 전체를 저장하는 2차원 배열이 주요 메모리 사용처이며, 윈도우 값을 담는 numbers 배열은 크기 9의 고정 배열로 O(1)이다.
