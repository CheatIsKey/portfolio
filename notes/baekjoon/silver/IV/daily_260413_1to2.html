    문제 14468 — 소가 길을 건너간 이유 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    소가 길을 건너간 이유 2 (백준 14468)는
    26마리의 소가 각각 두 번씩 등장하는 52자리 문자열이 주어질 때, 서로 교차하며 길을 건너는 소 쌍의 수를 구하는 문제다.
    소 A의 첫 번째와 두 번째 등장 사이에 소 B가 홀수 번 등장하면, A와 B는 교차하는 관계다.

    • 소 A의 구간 (start, end) 사이에 소 B가 정확히 한 번만 등장하면 A와 B는 교차한다. 두 번 등장하면 교차하지 않는다.
    • 따라서 구간 내에서 각 소의 등장 횟수를 토글 방식으로 추적하면, 최종적으로 true인 소가 A와 교차하는 소다.
    • 이중 카운팅을 방지하기 위해 j > i인 소에 대해서만 교차 여부를 집계한다.

    전략 개요

    • 핵심 알고리즘: 구간 내 토글 플래그를 이용한 교차 쌍 카운팅
    • 선택 이유: 소의 종류가 26가지로 고정이므로, 각 소의 구간 내에서 다른 소의 등장을 토글 방식으로 추적하면 O(26 × 52) = O(1)에 수렴하는 상수 시간 안에 모든 교차 쌍을 셀 수 있다.
    • 세부 전략:
      • 52자리 문자열을 순회하며 각 소의 첫 번째·두 번째 등장 위치를 cows[26][2]에 저장한다.
      • 소 i의 구간 (start+1, end) 사이를 순회하며 등장하는 소마다 flags를 토글한다.
      • 구간 순회가 끝난 뒤 flags[j]가 true이고 j > i이면 교차 쌍으로 집계한다.

    코드가 하는 일

    • 1) 입력 및 각 소의 등장 위치 기록 
      char[] logs = br.readLine().toCharArray();
      int[][] cows = new int[26][2];
      
      for (int i = 0; i < 26; i++) {
          Arrays.fill(cows[i], -1);
      }
      
      for (int i = 0; i < 52; i++) {
          int idx = logs[i] - 'A';
      
          if (cows[idx][0] == -1) {
              cows[idx][0] = i;
          } else {
              cows[idx][1] = i;
          }
      }
      • 52자리 문자열을 char[]로 변환한다. cows[i][0]은 소 i의 첫 번째 등장 위치, cows[i][1]은 두 번째 등장 위치를 저장한다. 초깃값 -1로 채워 아직 등장하지 않은 소를 구분한다.
    • 2) 각 소의 구간 내 교차 소 탐색 
      for (int i = 0; i < 26; i++) {
          int start = cows[i][0];
          int end = cows[i][1];
          boolean[] flags = new boolean[26];
      
          for (int s = start + 1; s < end; s++) {
              flags[logs[s] - 'A'] = !flags[logs[s] - 'A'];
          }
      
          for (int j = i + 1; j < 26; j++) {
              if (flags[j]) count++;
          }
      }
      • 소 i의 구간 (start, end) 사이를 순회하며 등장하는 소마다 flags를 토글한다. 짝수 번 등장하면 false(교차 아님), 홀수 번 등장하면 true(교차)가 된다.
      • 구간 순회 후 j > i인 소에 대해서만 flags[j]가 true인지 확인하여 이중 카운팅을 방지한다.
      • 등장하지 않은 소(start == -1)는 flags가 모두 false로 유지되어 집계에 영향을 주지 않는다.
    • 3) 결과 출력 
      System.out.println(count);
      • 교차하는 소 쌍의 총 수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(1): 소의 종류(26)와 문자열 길이(52)가 모두 고정 상수이므로, 모든 반복문의 순회 횟수가 상수로 결정된다. 전체 연산은 O(26 × 52) = O(1)에 수렴한다.
    • 공간 복잡도
      • O(1): logs(52), cows(26×2), flags(26) 모두 고정 크기 배열이므로 입력 크기와 무관한 상수 공간만 사용한다.

    문제 1895 — 필터

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    블라인드 (백준 2799)는
    M×N 규모의 아파트 단지를 5×5 문자 블록으로 표현한 격자가 주어질 때,
    각 호실에서 내려진 블라인드 수(0~4개)별 호실 수를 구하는 문제다.

    • 전체 격자는 (5M+1)행 × (5N+1)열이며, 각 호실의 내부는 5행 5열 블록 안에 표현된다.
    • 각 호실의 블라인드는 블록 내부 첫 번째 열(j번째 열)의 위쪽부터 연속된 '*'의 수로 나타난다. 연속이 끊기면 그 아래는 세지 않는다.
    • 블록 경계(0, 5, 10, ... 번째 행/열)는 구분선이므로, 호실의 시작점은 (1, 1), (1, 6), (6, 1), ... 과 같이 1-based로 5 간격이다.

    전략 개요

    • 핵심 알고리즘: 2차원 배열 시뮬레이션 (5 간격 블록 탐색)
    • 선택 이유: 격자를 그대로 2차원 배열에 저장한 뒤, 5 간격으로 각 호실의 시작점을 찾아 블라인드 수를 세면 별도의 자료구조 없이 O(M·N) 순회로 해결된다.
    • 세부 전략:
      • 전체 격자를 char[][]에 저장한다.
      • 행과 열 모두 1부터 시작해 5 간격으로 순회하며 각 호실의 시작점 (i, j)를 찾는다.
      • 해당 열(j)에서 위쪽부터 최대 4칸(k = 0~3)을 확인하며 연속된 '*'의 수를 센다. '*'가 아닌 순간 즉시 중단한다.
      • 센 수를 인덱스로 하여 answer 배열을 증가시킨다.

    코드가 하는 일

    • 1) 격자 크기 계산 및 입력 
      int row = 5 * M + 1;
      int col = 5 * N + 1;
      
      char[][] apt = new char[row][col];
      
      for (int i = 0; i < row; i++) {
          st = new StringTokenizer(br.readLine());
          String s = st.nextToken();
      
          for (int j = 0; j < col; j++) {
              apt[i][j] = s.charAt(j);
          }
      }
      • M×N 아파트 단지는 구분선을 포함해 (5M+1)행 × (5N+1)열의 격자로 표현된다. 각 행의 문자열을 읽어 char[][]에 저장한다.
    • 2) 호실별 블라인드 수 집계 
      for (int i = 1; i < row; i += 5) {
          for (int j = 1; j < col; j += 5) {
              int cnt = 0;
      
              for (int k = 0; k < 4; k++) {
                  if (apt[i + k][j] == '*') cnt++;
                  else break;
              }
      
              answer[cnt]++;
          }
      }
      • i와 j를 각각 1부터 5 간격으로 순회하여 각 호실 블록의 시작점을 찾는다. 구분선(0번째 행/열)을 건너뛰고 내부 첫 칸에서 시작하는 방식이다.
      • 해당 열 j에서 k = 0~3까지 최대 4칸을 위에서 아래로 확인하며 연속된 '*'를 센다. '*'가 아니면 즉시 break하여 불필요한 탐색을 막는다.
      • 센 수 cnt를 인덱스로 하여 answer[cnt]를 증가시킨다.
    • 3) 결과 출력 
      for (int i : answer) {
          sb.append(i).append(" ");
      }
      
      System.out.println(sb);
      • 블라인드 수 0개부터 4개까지의 호실 수를 순서대로 공백으로 구분하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M · N): 입력 저장에 O(M · N), 호실 탐색에 각 호실당 최대 4번의 상수 연산이 수행되므로 전체 O(M · N)이다.
    • 공간 복잡도
      • O(M · N): 전체 격자를 저장하는 char[][]가 주요 메모리 사용처이며, 그 외 answer 배열은 크기 5의 고정 배열로 O(1)이다.
