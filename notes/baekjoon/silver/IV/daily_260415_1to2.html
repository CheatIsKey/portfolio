    문제 16499 — 동일한 단어 그룹화하기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    동일한 단어 그룹화하기 (백준 16499)는
    N개의 단어가 주어질 때, 같은 문자로 이루어진 단어끼리는 동일한 그룹으로 간주하여 서로 다른 그룹의 수를 구하는 문제다.
    각 단어를 정렬한 결과가 같으면 동일한 그룹으로 판별한다.

    • 두 단어가 같은 문자 집합으로 구성되어 있는지 확인하려면, 각 문자를 정렬한 결과를 비교하면 된다.
    • 정렬된 문자열을 HashSet에 넣으면 중복은 자동으로 제거되고, 최종 집합의 크기가 서로 다른 그룹의 수가 된다.
    • 단어의 길이가 다르면 정렬 결과도 달라지므로 별도의 길이 비교 없이 정렬 결과만으로 그룹을 구분할 수 있다.

    전략 개요

    • 핵심 알고리즘: 정렬 + HashSet을 이용한 중복 제거
    • 선택 이유: 각 단어를 정렬하면 같은 문자 구성을 가진 단어는 반드시 동일한 문자열로 수렴한다. 이를 HashSet에 저장하면 O(1) 삽입으로 중복을 자동 제거할 수 있어, 전체 O(N · L log L)에 서로 다른 그룹 수를 구할 수 있다.
    • 세부 전략:
      • 단어를 읽어 char[]로 변환한 뒤 정렬한다.
      • 정렬된 char[]를 String으로 변환하여 HashSet에 추가한다.
      • 모든 단어를 처리한 뒤 set.size()가 서로 다른 그룹의 수다.

    코드가 하는 일

    • 1) 입력 및 자료구조 초기화 
      int N = Integer.parseInt(br.readLine());
      Set<String> set = new HashSet<>();
      • 단어의 수 N을 읽고, 정렬된 단어의 대표 문자열을 저장할 HashSet을 초기화한다.
    • 2) 단어 정렬 및 집합 삽입 
      for (int i = 0; i < N; i++) {
          char[] arr = br.readLine().toCharArray();
          Arrays.sort(arr);
          String word = new String(arr);
          set.add(word);
      }
      • toCharArray()로 단어를 문자 배열로 변환한 뒤 Arrays.sort()로 오름차순 정렬한다. 같은 문자 구성의 단어는 정렬 후 반드시 동일한 문자열이 된다.
      • 정렬된 배열을 new String(arr)로 문자열로 변환하여 HashSet에 추가한다. 이미 같은 문자열이 존재하면 HashSet이 자동으로 중복을 무시한다.
    • 3) 결과 출력 
      System.out.println(set.size());
      • HashSet에는 각 그룹의 대표 문자열 하나씩만 남아 있으므로, set.size()가 곧 서로 다른 단어 그룹의 수다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N · L log L): N개의 단어 각각에 대해 길이 L인 문자 배열을 정렬하는 데 O(L log L)이 소요되고, HashSet 삽입은 O(L)이므로 전체 O(N · L log L)이다. (L은 단어의 최대 길이)
    • 공간 복잡도
      • O(N · L): 최악의 경우 N개의 단어가 모두 서로 다른 그룹이어서 HashSet에 N개의 문자열이 저장되며, 각 문자열의 길이가 최대 L이므로 O(N · L)의 공간을 사용한다.

    문제 1590 — 캠프가는 영식

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    배수 스위치 (백준 12927)는
    N개의 전구가 켜짐(Y) 또는 꺼짐(N) 상태로 주어질 때, 최소 횟수의 스위치 조작으로 모든 전구를 꺼진 상태로 만드는 횟수를 구하는 문제다.
    i번 스위치를 누르면 i의 배수 번호 전구가 모두 토글되며, 불가능한 경우 -1을 출력한다.

    • 스위치는 1번부터 N번까지 존재하며, i번 스위치를 누르면 i, 2i, 3i, ... 번 전구가 토글된다.
    • 그리디하게 1번부터 순서대로 처리하면, i번 전구가 켜져 있을 때 i번 스위치를 반드시 한 번 눌러야 한다. i번 전구는 1번~(i-1)번 스위치로는 제어할 수 없기 때문이다.
    • 모든 스위치를 순서대로 처리한 뒤에도 켜진 전구가 남아 있으면 불가능하므로 -1을 출력한다.

    전략 개요

    • 핵심 알고리즘: 그리디 (1번부터 순서대로 스위치 조작)
    • 선택 이유: i번 전구를 끄는 수단은 i번 스위치뿐이다(1~i-1번 스위치는 i를 건드리지 않고, i+1번 이후 스위치는 i의 약수가 아니므로 해당 없음). 따라서 i번 전구가 켜져 있으면 반드시 i번 스위치를 눌러야 하는 유일한 선택이며, 이 결정이 항상 최적임이 보장된다.
    • 세부 전략:
      • 1번부터 N번까지 순회하며, 현재 전구가 켜져 있으면 해당 번호의 스위치를 누른다.
      • 스위치를 누를 때마다 해당 번호의 배수 전구를 모두 토글하고 count를 증가시킨다.
      • 순회가 끝난 뒤 켜진 전구가 남아 있으면 -1, 없으면 count를 출력한다.

    코드가 하는 일

    • 1) 입력 및 전구 배열 초기화 
      String N = br.readLine();
      
      char[] bulbs = new char[N.length() + 1];
      
      for (int i = 1; i < bulbs.length; i++) {
          bulbs[i] = N.charAt(i - 1);
      }
      • 전구 상태 문자열을 읽어 1-based 인덱스로 사용하기 위해 크기 N+1의 char[]에 저장한다. bulbs[0]은 사용하지 않고 bulbs[1]부터 전구 상태를 저장한다.
    • 2) 그리디 스위치 조작 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              count++;
      
              for (int j = i; j < bulbs.length; j += i) {
                  bulbs[j] = (bulbs[j] == 'N') ? 'Y' : 'N';
              }
          }
      }
      • bulbs[i] == 'Y': i번 전구가 켜져 있으므로 반드시 i번 스위치를 눌러야 한다. count를 증가시킨다.
      • 내부 반복문에서 i의 배수 인덱스(i, 2i, 3i, ...)를 순회하며 전구를 토글한다. 이 과정에서 i번 전구 자체도 꺼지고, 이후 배수 전구들의 상태도 갱신된다.
    • 3) 불가능 여부 확인 및 출력 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              System.out.println(-1);
              return;
          }
      }
      
      System.out.println(count);
      • 모든 스위치를 처리한 뒤에도 켜진 전구가 남아 있으면 어떤 조작으로도 끌 수 없는 상태이므로 -1을 출력하고 종료한다. 모든 전구가 꺼졌다면 최소 조작 횟수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 각 스위치 i를 눌렀을 때 N/i개의 배수 전구를 토글하므로, 전체 토글 횟수는 N/1 + N/2 + ... + N/N = O(N log N)이다.
    • 공간 복잡도
      • O(N): 전구 상태를 저장하는 char[] 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
