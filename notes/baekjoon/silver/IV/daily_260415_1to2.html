    문제 16499 — 트리 만들기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    트리 만들기 (백준 14244)는
    n개의 노드로 이루어진 트리를 구성할 때, 단말 노드(리프)의 수가 정확히 m개가 되도록 간선을 출력하는 문제다.
    m의 값에 따라 체인 혹은 스타 구조를 조합하여 리프 수를 정확히 맞춘다.

    • m=1일 때는 모든 노드를 일렬로 연결한 체인(대나무)으로 리프가 끝 노드 하나만 남는다.
    • m=2일 때는 체인을 구성하면 양 끝 노드 두 개가 리프가 된다.
    • m≥3일 때는 루트(노드 0)에 m개의 자식을 붙여 m개의 리프를 만든 뒤, 마지막으로 붙인 자식 노드부터 나머지 노드를 체인으로 이어 해당 자식을 내부 노드로 전환하고 체인 끝 노드를 새 리프로 만들어 리프 수를 m으로 유지한다.

    전략 개요

    • 핵심 알고리즘: 그리디 (스타 + 체인 혼합 구조 생성)
    • 선택 이유: 리프 수를 정확히 m개로 맞추려면, 루트에 자식을 붙이는 것(리프 증가)과 기존 리프에 자식을 붙이는 것(리프 유지 또는 이동)을 적절히 조합해야 한다. lastNumber가 가리키는 노드에 새 노드를 연결하고, 리프 수가 m에 도달한 시점부터 체인으로 전환하면 O(n)에 조건을 만족하는 트리를 구성할 수 있다.
    • 세부 전략:
      • lastNumber는 다음 노드를 연결할 부모 노드 번호를, countLeaf는 현재까지 생성된 리프 수를 추적한다.
      • 리프 수가 m 미만인 동안은 루트(0)에 계속 자식을 붙여 리프를 늘린다. 단, m=2이면 즉시 체인으로 전환한다.
      • 리프 수가 m에 도달하면 lastNumber = i로 갱신하여 이후 노드를 체인으로 연결한다. 체인의 끝 노드가 유일한 새 리프가 되어 총 리프 수가 m으로 유지된다.

    코드가 하는 일

    • 1) 초기화 
      int lastNumber = 0;
      int countLeaf = 0;
      • lastNumber는 다음 노드를 연결할 부모 노드로, 초기값은 루트 노드 0이다. countLeaf는 현재까지 생성된 리프 수를 추적한다.
    • 2) 노드 연결 및 부모 갱신 
      for (int i = 1; i < n; i++) {
          System.out.println(lastNumber + " " + i);
          countLeaf++;
      
          if (countLeaf < m) {
              if (m == 2) lastNumber = i;
          } else {
              lastNumber = i;
          }
      }
      • 노드 i를 lastNumber에 연결하는 간선을 출력하고 countLeaf를 증가시킨다.
      • countLeaf < m: 아직 리프가 부족하다. m=2인 경우에만 lastNumber = i로 갱신하여 체인을 형성하고, 나머지 m≥3은 lastNumber를 0으로 유지해 루트에 계속 자식을 붙인다.
      • countLeaf >= m: 리프가 충분히 만들어졌다. lastNumber = i로 갱신하여 이후 노드들을 체인으로 연결한다. 체인으로 연결된 노드는 자식이 생기는 순간 리프에서 내부 노드로 전환되고, 가장 마지막 노드만 리프로 남아 총 리프 수가 m으로 유지된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(n): 노드 1부터 n-1까지 한 번씩 순회하며 각 반복에서 O(1) 연산만 수행하므로 전체 O(n)이다.
    • 공간 복잡도
      • O(1): lastNumber와 countLeaf 두 개의 정수 변수만 사용하므로 입력 크기와 무관한 상수 공간만 필요하다.

    문제 12927 — 배수 스위치

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    배수 스위치 (백준 12927)는
    N개의 전구가 켜짐(Y) 또는 꺼짐(N) 상태로 주어질 때, 최소 횟수의 스위치 조작으로 모든 전구를 꺼진 상태로 만드는 횟수를 구하는 문제다.
    i번 스위치를 누르면 i의 배수 번호 전구가 모두 토글되며, 불가능한 경우 -1을 출력한다.

    • 스위치는 1번부터 N번까지 존재하며, i번 스위치를 누르면 i, 2i, 3i, ... 번 전구가 토글된다.
    • 그리디하게 1번부터 순서대로 처리하면, i번 전구가 켜져 있을 때 i번 스위치를 반드시 한 번 눌러야 한다. i번 전구는 1번~(i-1)번 스위치로는 제어할 수 없기 때문이다.
    • 모든 스위치를 순서대로 처리한 뒤에도 켜진 전구가 남아 있으면 불가능하므로 -1을 출력한다.

    전략 개요

    • 핵심 알고리즘: 그리디 (1번부터 순서대로 스위치 조작)
    • 선택 이유: i번 전구를 끄는 수단은 i번 스위치뿐이다(1~i-1번 스위치는 i를 건드리지 않고, i+1번 이후 스위치는 i의 약수가 아니므로 해당 없음). 따라서 i번 전구가 켜져 있으면 반드시 i번 스위치를 눌러야 하는 유일한 선택이며, 이 결정이 항상 최적임이 보장된다.
    • 세부 전략:
      • 1번부터 N번까지 순회하며, 현재 전구가 켜져 있으면 해당 번호의 스위치를 누른다.
      • 스위치를 누를 때마다 해당 번호의 배수 전구를 모두 토글하고 count를 증가시킨다.
      • 순회가 끝난 뒤 켜진 전구가 남아 있으면 -1, 없으면 count를 출력한다.

    코드가 하는 일

    • 1) 입력 및 전구 배열 초기화 
      String N = br.readLine();
      
      char[] bulbs = new char[N.length() + 1];
      
      for (int i = 1; i < bulbs.length; i++) {
          bulbs[i] = N.charAt(i - 1);
      }
      • 전구 상태 문자열을 읽어 1-based 인덱스로 사용하기 위해 크기 N+1의 char[]에 저장한다. bulbs[0]은 사용하지 않고 bulbs[1]부터 전구 상태를 저장한다.
    • 2) 그리디 스위치 조작 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              count++;
      
              for (int j = i; j < bulbs.length; j += i) {
                  bulbs[j] = (bulbs[j] == 'N') ? 'Y' : 'N';
              }
          }
      }
      • bulbs[i] == 'Y': i번 전구가 켜져 있으므로 반드시 i번 스위치를 눌러야 한다. count를 증가시킨다.
      • 내부 반복문에서 i의 배수 인덱스(i, 2i, 3i, ...)를 순회하며 전구를 토글한다. 이 과정에서 i번 전구 자체도 꺼지고, 이후 배수 전구들의 상태도 갱신된다.
    • 3) 불가능 여부 확인 및 출력 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              System.out.println(-1);
              return;
          }
      }
      
      System.out.println(count);
      • 모든 스위치를 처리한 뒤에도 켜진 전구가 남아 있으면 어떤 조작으로도 끌 수 없는 상태이므로 -1을 출력하고 종료한다. 모든 전구가 꺼졌다면 최소 조작 횟수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 각 스위치 i를 눌렀을 때 N/i개의 배수 전구를 토글하므로, 전체 토글 횟수는 N/1 + N/2 + ... + N/N = O(N log N)이다.
    • 공간 복잡도
      • O(N): 전구 상태를 저장하는 char[] 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
