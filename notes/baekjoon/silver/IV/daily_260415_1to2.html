    문제 16499 — 동일한 단어 그룹화하기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    동일한 단어 그룹화하기 (백준 16499)는
    N개의 단어가 주어질 때, 같은 문자로 이루어진 단어끼리는 동일한 그룹으로 간주하여 서로 다른 그룹의 수를 구하는 문제다.
    각 단어를 정렬한 결과가 같으면 동일한 그룹으로 판별한다.

    • 두 단어가 같은 문자 집합으로 구성되어 있는지 확인하려면, 각 문자를 정렬한 결과를 비교하면 된다.
    • 정렬된 문자열을 HashSet에 넣으면 중복은 자동으로 제거되고, 최종 집합의 크기가 서로 다른 그룹의 수가 된다.
    • 단어의 길이가 다르면 정렬 결과도 달라지므로 별도의 길이 비교 없이 정렬 결과만으로 그룹을 구분할 수 있다.

    전략 개요

    • 핵심 알고리즘: 정렬 + HashSet을 이용한 중복 제거
    • 선택 이유: 각 단어를 정렬하면 같은 문자 구성을 가진 단어는 반드시 동일한 문자열로 수렴한다. 이를 HashSet에 저장하면 O(1) 삽입으로 중복을 자동 제거할 수 있어, 전체 O(N · L log L)에 서로 다른 그룹 수를 구할 수 있다.
    • 세부 전략:
      • 단어를 읽어 char[]로 변환한 뒤 정렬한다.
      • 정렬된 char[]를 String으로 변환하여 HashSet에 추가한다.
      • 모든 단어를 처리한 뒤 set.size()가 서로 다른 그룹의 수다.

    코드가 하는 일

    • 1) 입력 및 자료구조 초기화 
      int N = Integer.parseInt(br.readLine());
      Set<String> set = new HashSet<>();
      • 단어의 수 N을 읽고, 정렬된 단어의 대표 문자열을 저장할 HashSet을 초기화한다.
    • 2) 단어 정렬 및 집합 삽입 
      for (int i = 0; i < N; i++) {
          char[] arr = br.readLine().toCharArray();
          Arrays.sort(arr);
          String word = new String(arr);
          set.add(word);
      }
      • toCharArray()로 단어를 문자 배열로 변환한 뒤 Arrays.sort()로 오름차순 정렬한다. 같은 문자 구성의 단어는 정렬 후 반드시 동일한 문자열이 된다.
      • 정렬된 배열을 new String(arr)로 문자열로 변환하여 HashSet에 추가한다. 이미 같은 문자열이 존재하면 HashSet이 자동으로 중복을 무시한다.
    • 3) 결과 출력 
      System.out.println(set.size());
      • HashSet에는 각 그룹의 대표 문자열 하나씩만 남아 있으므로, set.size()가 곧 서로 다른 단어 그룹의 수다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N · L log L): N개의 단어 각각에 대해 길이 L인 문자 배열을 정렬하는 데 O(L log L)이 소요되고, HashSet 삽입은 O(L)이므로 전체 O(N · L log L)이다. (L은 단어의 최대 길이)
    • 공간 복잡도
      • O(N · L): 최악의 경우 N개의 단어가 모두 서로 다른 그룹이어서 HashSet에 N개의 문자열이 저장되며, 각 문자열의 길이가 최대 L이므로 O(N · L)의 공간을 사용한다.

    문제 1590 — 캠프가는 영식

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    캠프가는 영식 (백준 1590)는
    N개의 버스 노선이 각각 출발 시각, 배차 간격, 운행 횟수를 가질 때, 영식이가 T 이후로 탈 수 있는 가장 빠른 버스의 대기 시간을 구하는 문제다.
    T 이상인 버스 시각 중 최솟값을 노선별로 이진 탐색으로 구한 뒤 전체 최솟값을 출력한다.

    • 각 버스 노선의 k번째 버스 출발 시각은 S + I × k (0 ≤ k ≤ C-1)이다. k가 최대 10억에 달할 수 있어 선형 탐색은 시간 초과가 발생하므로 이진 탐색이 필요하다.
    • 노선마다 T 이상인 첫 번째 버스 시각을 이진 탐색으로 구하고, 전체 노선 중 최솟값이 정답이다.
    • 모든 노선에서 T 이후 탈 수 있는 버스가 없으면 -1을 출력한다.

    전략 개요

    • 핵심 알고리즘: 노선별 이진 탐색 (T 이상인 최소 출발 시각 탐색)
    • 선택 이유: 각 버스 노선의 출발 시각은 S, S+I, S+2I, ... 로 등차수열을 이루므로 단조 증가한다. 단조 증가 수열에서 T 이상인 첫 번째 값을 찾는 것은 이진 탐색의 전형적인 적용 사례로, 운행 횟수 C가 최대 10억이더라도 O(log C)에 탐색할 수 있다.
    • 세부 전략:
      • 탐색 범위를 운행 인덱스 0~C-1로 설정하고, 중간 인덱스 mid에 해당하는 시각 S + I × mid를 T와 비교한다.
      • 시각이 T 이상이면 후보로 기록하고 더 앞선 인덱스를 탐색한다.
      • 시각이 T 미만이면 더 뒤의 인덱스를 탐색한다.
      • 노선별로 구한 후보 중 전체 최솟값에서 T를 빼면 대기 시간이 된다.

    코드가 하는 일

    • 1) 입력 및 초기화 
      int N = Integer.parseInt(st.nextToken());
      long T = Long.parseLong(st.nextToken());
      
      long answer = Long.MAX_VALUE;
      • 버스 노선 수 N과 영식이의 도착 시각 T를 입력받는다. 출발 시각과 배차 간격의 곱이 int 범위를 초과할 수 있으므로 long으로 처리한다. answer는 T 이후 탈 수 있는 버스 시각의 전체 최솟값을 추적하며 초깃값을 Long.MAX_VALUE로 설정한다.
    • 2) 노선별 이진 탐색 
      long start = 0;
      long end = C - 1;
      long busTime = -1;
      
      while (start <= end) {
          long mid = (start + end) / 2;
          long time = S + (I * mid);
      
          if (time >= T) {
              busTime = time;
              end = mid - 1;
          } else {
              start = mid + 1;
          }
      }
      
      if (busTime != -1) {
          answer = Math.min(answer, busTime);
      }
      • 탐색 범위를 운행 인덱스 0~C-1로 설정한다. 중간 인덱스 mid에 해당하는 출발 시각 time = S + I × mid를 T와 비교한다.
      • time >= T이면 후보로 기록하고 end = mid - 1로 더 앞선 인덱스를 탐색하여 T 이상인 최소 시각을 찾는다.
      • time < T이면 start = mid + 1로 더 뒤의 인덱스를 탐색한다.
      • 탐색이 끝나면 busTime이 해당 노선에서 T 이후 탈 수 있는 가장 빠른 버스 시각이며, 전체 answer와 비교해 최솟값을 갱신한다.
    • 3) 결과 출력 
      if (answer != Long.MAX_VALUE) {
          System.out.println(answer - T);
      } else {
          System.out.println("-1");
      }
      • answer가 갱신되었으면 대기 시간 answer - T를 출력하고, 갱신되지 않았으면 탈 수 있는 버스가 없으므로 -1을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log C): N개의 노선마다 운행 횟수 C에 대해 이진 탐색을 수행하므로 전체 O(N log C)이다. (C는 운행 횟수의 최댓값)
    • 공간 복잡도
      • O(1): 노선 정보를 별도 배열에 저장하지 않고 입력 즉시 처리하므로, answer를 포함한 상수 개수의 변수만 사용한다.
