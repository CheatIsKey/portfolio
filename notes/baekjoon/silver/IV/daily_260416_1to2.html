    문제 1359 — 복권

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    복권 (백준 1359)는
    1부터 N까지의 수 중 M개를 뽑는 복권에서 내가 고른 M개의 번호 중 K개 이상이 일치할 확률을 구하는 문제다.
    조합론을 이용해 당첨 경우의 수를 계산하고 전체 경우의 수로 나누어 확률을 구한다.

    • 복권 당첨 번호 M개와 내가 고른 M개 중 정확히 i개가 일치하는 경우의 수는 C(M, i) × C(N-M, M-i)이다.
    • C(M, i)는 당첨 번호 M개 중 i개를 고르는 경우, C(N-M, M-i)는 나머지 N-M개 중 틀린 번호 M-i개를 고르는 경우다.
    • K개 이상 일치하는 경우의 합을 전체 경우의 수 C(N, M)으로 나누면 확률이 된다.

    전략 개요

    • 핵심 알고리즘: 조합(nCr) 계산을 통한 확률 산출
    • 선택 이유: 정확히 i개 일치하는 경우의 수를 조합으로 표현하면 반복적인 시뮬레이션 없이 수식 하나로 계산할 수 있다. K부터 M까지 각 i에 대해 경우의 수를 합산한 뒤 전체 경우의 수로 나누면 O(M²) 이내에 확률을 구할 수 있다.
    • 세부 전략:
      • i를 K부터 M까지 순회하며 C(M, i) × C(N-M, M-i)를 numerator에 누적한다.
      • 전체 경우의 수 C(N, M)을 denominator로 계산한다.
      • numerator / denominator를 출력한다.
      • nCr에서 r이 음수이거나 n보다 크면 0을 반환하여 유효하지 않은 조합을 처리한다.

    코드가 하는 일

    • 1) K개 이상 일치하는 경우의 수 합산 
      double numerator = 0;
      for (int i = K; i <= M; i++) {
          numerator += nCr(M, i) * nCr(N - M, M - i);
      }
      • i를 K부터 M까지 순회하며 정확히 i개 일치하는 경우의 수 C(M, i) × C(N-M, M-i)를 누적한다. C(N-M, M-i)에서 M-i가 N-M을 초과하거나 음수이면 nCr이 0을 반환하므로 자연스럽게 걸러진다.
    • 2) 전체 경우의 수 계산 및 확률 출력 
      double denominator = nCr(N, M);
      System.out.println(numerator / denominator);
      • N개 중 M개를 고르는 전체 경우의 수 C(N, M)을 구하고, 분자를 분모로 나눠 확률을 출력한다.
    • 3) 조합 계산 메서드 
      public static long nCr(int n, int r) {
          if (r < 0 || r > n) return 0;
          long res = 1;
          for (int i = 1; i <= r; i++) {
              res = res * (n - i + 1) / i;
          }
          return res;
      }
      • r이 음수이거나 n보다 크면 0을 반환하여 유효하지 않은 조합을 처리한다.
      • res = res * (n - i + 1) / i를 반복해 C(n, r)을 계산한다. 매 단계에서 나눗셈을 수행하므로 중간 값이 과도하게 커지지 않으며 정수 범위를 유지할 수 있다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M²): K부터 M까지 최대 M번 순회하며 각 반복에서 nCr을 두 번 호출하고, 각 호출은 O(M)이므로 전체 O(M²)이다.
    • 공간 복잡도
      • O(1): numerator, denominator, res 등 고정된 수의 변수만 사용하므로 상수 공간만 필요하다.

    문제 12782 — 비트 우정지수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    배수 스위치 (백준 12927)는
    N개의 전구가 켜짐(Y) 또는 꺼짐(N) 상태로 주어질 때, 최소 횟수의 스위치 조작으로 모든 전구를 꺼진 상태로 만드는 횟수를 구하는 문제다.
    i번 스위치를 누르면 i의 배수 번호 전구가 모두 토글되며, 불가능한 경우 -1을 출력한다.

    • 스위치는 1번부터 N번까지 존재하며, i번 스위치를 누르면 i, 2i, 3i, ... 번 전구가 토글된다.
    • 그리디하게 1번부터 순서대로 처리하면, i번 전구가 켜져 있을 때 i번 스위치를 반드시 한 번 눌러야 한다. i번 전구는 1번~(i-1)번 스위치로는 제어할 수 없기 때문이다.
    • 모든 스위치를 순서대로 처리한 뒤에도 켜진 전구가 남아 있으면 불가능하므로 -1을 출력한다.

    전략 개요

    • 핵심 알고리즘: 그리디 (1번부터 순서대로 스위치 조작)
    • 선택 이유: i번 전구를 끄는 수단은 i번 스위치뿐이다(1~i-1번 스위치는 i를 건드리지 않고, i+1번 이후 스위치는 i의 약수가 아니므로 해당 없음). 따라서 i번 전구가 켜져 있으면 반드시 i번 스위치를 눌러야 하는 유일한 선택이며, 이 결정이 항상 최적임이 보장된다.
    • 세부 전략:
      • 1번부터 N번까지 순회하며, 현재 전구가 켜져 있으면 해당 번호의 스위치를 누른다.
      • 스위치를 누를 때마다 해당 번호의 배수 전구를 모두 토글하고 count를 증가시킨다.
      • 순회가 끝난 뒤 켜진 전구가 남아 있으면 -1, 없으면 count를 출력한다.

    코드가 하는 일

    • 1) 입력 및 전구 배열 초기화 
      String N = br.readLine();
      
      char[] bulbs = new char[N.length() + 1];
      
      for (int i = 1; i < bulbs.length; i++) {
          bulbs[i] = N.charAt(i - 1);
      }
      • 전구 상태 문자열을 읽어 1-based 인덱스로 사용하기 위해 크기 N+1의 char[]에 저장한다. bulbs[0]은 사용하지 않고 bulbs[1]부터 전구 상태를 저장한다.
    • 2) 그리디 스위치 조작 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              count++;
      
              for (int j = i; j < bulbs.length; j += i) {
                  bulbs[j] = (bulbs[j] == 'N') ? 'Y' : 'N';
              }
          }
      }
      • bulbs[i] == 'Y': i번 전구가 켜져 있으므로 반드시 i번 스위치를 눌러야 한다. count를 증가시킨다.
      • 내부 반복문에서 i의 배수 인덱스(i, 2i, 3i, ...)를 순회하며 전구를 토글한다. 이 과정에서 i번 전구 자체도 꺼지고, 이후 배수 전구들의 상태도 갱신된다.
    • 3) 불가능 여부 확인 및 출력 
      for (int i = 1; i < bulbs.length; i++) {
          if (bulbs[i] == 'Y') {
              System.out.println(-1);
              return;
          }
      }
      
      System.out.println(count);
      • 모든 스위치를 처리한 뒤에도 켜진 전구가 남아 있으면 어떤 조작으로도 끌 수 없는 상태이므로 -1을 출력하고 종료한다. 모든 전구가 꺼졌다면 최소 조작 횟수 count를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 각 스위치 i를 눌렀을 때 N/i개의 배수 전구를 토글하므로, 전체 토글 횟수는 N/1 + N/2 + ... + N/N = O(N log N)이다.
    • 공간 복잡도
      • O(N): 전구 상태를 저장하는 char[] 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
