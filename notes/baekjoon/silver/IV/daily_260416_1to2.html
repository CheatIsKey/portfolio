    문제 1359 — 복권

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    복권 (백준 1359)는
    1부터 N까지의 수 중 M개를 뽑는 복권에서 내가 고른 M개의 번호 중 K개 이상이 일치할 확률을 구하는 문제다.
    조합론을 이용해 당첨 경우의 수를 계산하고 전체 경우의 수로 나누어 확률을 구한다.

    • 복권 당첨 번호 M개와 내가 고른 M개 중 정확히 i개가 일치하는 경우의 수는 C(M, i) × C(N-M, M-i)이다.
    • C(M, i)는 당첨 번호 M개 중 i개를 고르는 경우, C(N-M, M-i)는 나머지 N-M개 중 틀린 번호 M-i개를 고르는 경우다.
    • K개 이상 일치하는 경우의 합을 전체 경우의 수 C(N, M)으로 나누면 확률이 된다.

    전략 개요

    • 핵심 알고리즘: 조합(nCr) 계산을 통한 확률 산출
    • 선택 이유: 정확히 i개 일치하는 경우의 수를 조합으로 표현하면 반복적인 시뮬레이션 없이 수식 하나로 계산할 수 있다. K부터 M까지 각 i에 대해 경우의 수를 합산한 뒤 전체 경우의 수로 나누면 O(M²) 이내에 확률을 구할 수 있다.
    • 세부 전략:
      • i를 K부터 M까지 순회하며 C(M, i) × C(N-M, M-i)를 numerator에 누적한다.
      • 전체 경우의 수 C(N, M)을 denominator로 계산한다.
      • numerator / denominator를 출력한다.
      • nCr에서 r이 음수이거나 n보다 크면 0을 반환하여 유효하지 않은 조합을 처리한다.

    코드가 하는 일

    • 1) K개 이상 일치하는 경우의 수 합산 
      double numerator = 0;
      for (int i = K; i <= M; i++) {
          numerator += nCr(M, i) * nCr(N - M, M - i);
      }
      • i를 K부터 M까지 순회하며 정확히 i개 일치하는 경우의 수 C(M, i) × C(N-M, M-i)를 누적한다. C(N-M, M-i)에서 M-i가 N-M을 초과하거나 음수이면 nCr이 0을 반환하므로 자연스럽게 걸러진다.
    • 2) 전체 경우의 수 계산 및 확률 출력 
      double denominator = nCr(N, M);
      System.out.println(numerator / denominator);
      • N개 중 M개를 고르는 전체 경우의 수 C(N, M)을 구하고, 분자를 분모로 나눠 확률을 출력한다.
    • 3) 조합 계산 메서드 
      public static long nCr(int n, int r) {
          if (r < 0 || r > n) return 0;
          long res = 1;
          for (int i = 1; i <= r; i++) {
              res = res * (n - i + 1) / i;
          }
          return res;
      }
      • r이 음수이거나 n보다 크면 0을 반환하여 유효하지 않은 조합을 처리한다.
      • res = res * (n - i + 1) / i를 반복해 C(n, r)을 계산한다. 매 단계에서 나눗셈을 수행하므로 중간 값이 과도하게 커지지 않으며 정수 범위를 유지할 수 있다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M²): K부터 M까지 최대 M번 순회하며 각 반복에서 nCr을 두 번 호출하고, 각 호출은 O(M)이므로 전체 O(M²)이다.
    • 공간 복잡도
      • O(1): numerator, denominator, res 등 고정된 수의 변수만 사용하므로 상수 공간만 필요하다.

    문제 12782 — 비트 우정지수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비트 우정지수 (백준 12782)는
    같은 길이의 두 이진 문자열 N과 M이 주어질 때, 최소 횟수의 연산으로 N을 M으로 변환하는 연산 횟수를 구하는 문제다.
    연산은 비트 교환(swap)과 비트 반전(flip) 두 가지이며, 교환이 반전보다 비용 효율이 높다.

    • N[i] ≠ M[i]인 위치는 두 종류로 나뉜다. N[i]=0, M[i]=1인 위치(diffZero)와 N[i]=1, M[i]=0인 위치(diffOne)다.
    • 두 종류의 불일치 위치가 각각 하나씩 있으면 교환 1회로 동시에 해결할 수 있다(비용 1, 2개 해결).
    • 짝을 지을 수 없는 나머지 불일치 위치는 반전 1회로 각각 처리해야 한다(비용 1, 1개 해결).

    전략 개요

    • 핵심 알고리즘: 그리디 (교환 우선, 나머지 반전)
    • 선택 이유: 교환 1회로 2개의 불일치를 동시에 해결할 수 있으므로 반전보다 효율적이다. 따라서 짝을 지을 수 있는 만큼 교환을 먼저 수행하고, 남은 불일치를 반전으로 처리하면 항상 최소 연산 횟수가 보장된다.
    • 세부 전략:
      • 불일치 위치를 diffZero(N=0, M=1)와 diffOne(N=1, M=0)으로 분류한다.
      • 두 값의 최솟값만큼 교환을 수행하여 쌍을 이루는 불일치를 모두 해결한다.
      • 남은 diffZero 또는 diffOne은 각각 반전 1회씩으로 처리한다.

    코드가 하는 일

    • 1) 불일치 위치 분류 
      for (int i = 0; i < N.length; i++) {
          if (N[i] != M[i] && N[i] == '0') {
              diffZero++;
          } else if (N[i] != M[i] && N[i] == '1') {
              diffOne++;
          }
      }
      • 두 문자열을 순회하며 N[i] ≠ M[i]인 위치를 두 종류로 분류한다. N[i]='0'인 불일치는 diffZero, N[i]='1'인 불일치는 diffOne에 누적한다.
    • 2) 교환 우선 처리 
      int move = Math.min(diffZero, diffOne);
      count += move;
      
      diffZero = diffZero - move;
      diffOne = diffOne - move;
      • diffZero와 diffOne의 최솟값만큼 교환을 수행한다. 교환 1회가 두 종류의 불일치를 동시에 1개씩 해결하므로, move번의 교환으로 2 × move개의 불일치가 사라진다.
      • 교환 후 남은 불일치 수를 각각 갱신한다. 둘 중 하나는 반드시 0이 된다.
    • 3) 나머지 반전 처리 및 출력 
      if (diffZero != 0) {
          count += diffZero;
      } else if (diffOne != 0) {
          count += diffOne;
      }
      
      sb.append(count).append("\n");
      • 짝을 짓지 못한 나머지 불일치는 반전 1회씩으로 처리하므로 남은 수만큼 count에 더한다. diffZero와 diffOne 중 하나는 반드시 0이므로 둘 다 더해지는 경우는 없다.
      • 결과를 StringBuilder에 누적하고 모든 테스트케이스가 끝난 뒤 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · L): T개의 테스트케이스마다 길이 L인 문자열을 한 번 순회하며 O(1) 연산만 수행하므로 전체 O(T · L)이다. (L은 문자열의 길이)
    • 공간 복잡도
      • O(L): 두 문자열을 char[]로 저장하는 공간이 주요 메모리 사용처이며, 그 외 카운터 변수들은 O(1)의 상수 공간을 사용한다.
