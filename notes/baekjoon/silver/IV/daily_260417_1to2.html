    문제 1730 — 판화

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    판화 (백준 1730)는
    N×N 격자 위에서 명령어에 따라 도장을 이동시킬 때, 이동 경로에 따라 각 칸에 수직선(|), 수평선(-), 교차점(+), 빈칸(.) 중 하나를 출력하는 문제다.
    각 칸을 수직 방향으로 지나갔는지, 수평 방향으로 지나갔는지를 독립적으로 추적하여 최종 상태를 결정한다.

    • 이동 시 현재 칸과 다음 칸 모두에 방향 정보를 기록해야 선분이 올바르게 표현된다.
    • 수직(U, D) 이동이면 vertical 배열, 수평(L, R) 이동이면 horizontal 배열에 표시한다.
    • 격자 범위를 벗어나는 이동 명령은 무시하고 현재 위치를 유지한다.

    전략 개요

    • 핵심 알고리즘: 방향별 방문 배열을 이용한 시뮬레이션
    • 선택 이유: 각 칸의 최종 출력 문자는 수직·수평 방향 통과 여부의 조합으로 결정된다. 두 개의 boolean 배열로 방향을 독립적으로 추적하면, 명령어를 한 번 순회하는 O(L)과 격자를 한 번 출력하는 O(N²)만으로 전체를 해결할 수 있다.
    • 세부 전략:
      • 명령어를 순서대로 처리하며 다음 위치를 계산하고, 범위를 벗어나면 건너뛴다.
      • 이동 방향에 따라 현재 칸과 다음 칸 모두 vertical 또는 horizontal 배열에 true로 표시한다.
      • 모든 명령어를 처리한 뒤 N×N 격자를 순회하며 두 배열의 조합으로 출력 문자를 결정한다.

    코드가 하는 일

    • 1) 초기화 및 명령어 시뮬레이션 
      boolean[][] horizontal = new boolean[N][N];
      boolean[][] vertical = new boolean[N][N];
      
      int row = 0;
      int col = 0;
      
      for (int i = 0; i < command.length; i++) {
          int nr = row;
          int nc = col;
      
          if (command[i] == 'U') nr--;
          else if (command[i] == 'D') nr++;
          else if (command[i] == 'L') nc--;
          else if (command[i] == 'R') nc++;
      
          if (nr < 0 || nr >= N || nc < 0 || nc >= N) continue;
      
          if (command[i] == 'U' || command[i] == 'D') {
              vertical[row][col] = true;
              vertical[nr][nc] = true;
          } else {
              horizontal[row][col] = true;
              horizontal[nr][nc] = true;
          }
      
          row = nr;
          col = nc;
      }
      • 시작 위치 (0, 0)에서 명령어를 순서대로 처리한다. 다음 위치 (nr, nc)가 격자 범위를 벗어나면 continue로 건너뛰고 현재 위치를 유지한다.
      • U·D 이동이면 현재 칸과 다음 칸의 vertical을, L·R 이동이면 horizontal을 true로 표시한다. 양 끝 칸 모두에 표시해야 선분이 완전하게 기록된다.
      • 이동이 유효하면 현재 위치를 (nr, nc)로 갱신한다.
    • 2) 격자 출력 
      for (int i = 0; i < N; i++) {
          for (int j = 0; j < N; j++) {
              if (vertical[i][j] && horizontal[i][j]) sb.append('+');
              else if (vertical[i][j]) sb.append('|');
              else if (horizontal[i][j]) sb.append('-');
              else sb.append('.');
          }
          sb.append('\n');
      }
      • 수직·수평 모두 true이면 교차점 '+', 수직만 true이면 '|', 수평만 true이면 '-', 둘 다 false이면 빈칸 '.'을 출력한다.
      • 결과를 StringBuilder에 누적하여 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L + N²): 명령어 길이 L만큼 시뮬레이션을 수행하고, 이후 N×N 격자를 한 번 순회하며 출력하므로 전체 O(L + N²)이다.
    • 공간 복잡도
      • O(N²): 수직·수평 방향 방문 여부를 저장하는 두 개의 N×N boolean 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 12782 — 비트 우정지수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비트 우정지수 (백준 12782)는
    같은 길이의 두 이진 문자열 N과 M이 주어질 때, 최소 횟수의 연산으로 N을 M으로 변환하는 연산 횟수를 구하는 문제다.
    연산은 비트 교환(swap)과 비트 반전(flip) 두 가지이며, 교환이 반전보다 비용 효율이 높다.

    • N[i] ≠ M[i]인 위치는 두 종류로 나뉜다. N[i]=0, M[i]=1인 위치(diffZero)와 N[i]=1, M[i]=0인 위치(diffOne)다.
    • 두 종류의 불일치 위치가 각각 하나씩 있으면 교환 1회로 동시에 해결할 수 있다(비용 1, 2개 해결).
    • 짝을 지을 수 없는 나머지 불일치 위치는 반전 1회로 각각 처리해야 한다(비용 1, 1개 해결).

    전략 개요

    • 핵심 알고리즘: 그리디 (교환 우선, 나머지 반전)
    • 선택 이유: 교환 1회로 2개의 불일치를 동시에 해결할 수 있으므로 반전보다 효율적이다. 따라서 짝을 지을 수 있는 만큼 교환을 먼저 수행하고, 남은 불일치를 반전으로 처리하면 항상 최소 연산 횟수가 보장된다.
    • 세부 전략:
      • 불일치 위치를 diffZero(N=0, M=1)와 diffOne(N=1, M=0)으로 분류한다.
      • 두 값의 최솟값만큼 교환을 수행하여 쌍을 이루는 불일치를 모두 해결한다.
      • 남은 diffZero 또는 diffOne은 각각 반전 1회씩으로 처리한다.

    코드가 하는 일

    • 1) 불일치 위치 분류 
      for (int i = 0; i < N.length; i++) {
          if (N[i] != M[i] && N[i] == '0') {
              diffZero++;
          } else if (N[i] != M[i] && N[i] == '1') {
              diffOne++;
          }
      }
      • 두 문자열을 순회하며 N[i] ≠ M[i]인 위치를 두 종류로 분류한다. N[i]='0'인 불일치는 diffZero, N[i]='1'인 불일치는 diffOne에 누적한다.
    • 2) 교환 우선 처리 
      int move = Math.min(diffZero, diffOne);
      count += move;
      
      diffZero = diffZero - move;
      diffOne = diffOne - move;
      • diffZero와 diffOne의 최솟값만큼 교환을 수행한다. 교환 1회가 두 종류의 불일치를 동시에 1개씩 해결하므로, move번의 교환으로 2 × move개의 불일치가 사라진다.
      • 교환 후 남은 불일치 수를 각각 갱신한다. 둘 중 하나는 반드시 0이 된다.
    • 3) 나머지 반전 처리 및 출력 
      if (diffZero != 0) {
          count += diffZero;
      } else if (diffOne != 0) {
          count += diffOne;
      }
      
      sb.append(count).append("\n");
      • 짝을 짓지 못한 나머지 불일치는 반전 1회씩으로 처리하므로 남은 수만큼 count에 더한다. diffZero와 diffOne 중 하나는 반드시 0이므로 둘 다 더해지는 경우는 없다.
      • 결과를 StringBuilder에 누적하고 모든 테스트케이스가 끝난 뒤 한 번에 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · L): T개의 테스트케이스마다 길이 L인 문자열을 한 번 순회하며 O(1) 연산만 수행하므로 전체 O(T · L)이다. (L은 문자열의 길이)
    • 공간 복잡도
      • O(L): 두 문자열을 char[]로 저장하는 공간이 주요 메모리 사용처이며, 그 외 카운터 변수들은 O(1)의 상수 공간을 사용한다.
