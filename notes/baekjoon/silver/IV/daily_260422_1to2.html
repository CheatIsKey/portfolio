    문제 12788 — 제 2회 IUPC는 잘 개최될 수 있을까?

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    제 2회 IUPC는 잘 개최될 수 있을까? (백준 12788)는
    N명의 참가자가 각자 데려올 수 있는 사람 수가 주어질 때, 팀 수 × 팀원 수를 충족하기 위해 섭외해야 할 최소 참가자 수를 구하는 문제다.
    가장 많은 사람을 데려올 수 있는 참가자부터 선택하는 그리디 전략으로 최소 인원을 결정한다.

    • 필요한 총 인원 need = team × people를 목표로, 가장 많은 인원을 채울 수 있는 참가자부터 내림차순으로 선택한다.
    • 누적 합이 need 이상이 되는 순간 선택을 멈추며, 그때까지 선택한 참가자 수가 최솟값이다.
    • 모든 참가자를 선택해도 need를 충족하지 못하면 STRESS를 출력한다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 그리디 (내림차순 누적 선택)
    • 선택 이유: 최소 참가자 수로 최대 인원을 채우려면 한 명당 기여도가 큰 순서대로 선택하는 것이 항상 최적이다. 배열을 오름차순 정렬한 뒤 뒤에서부터 순회하면 내림차순으로 참가자를 선택할 수 있으며, 목표 인원을 달성하는 순간 즉시 종료한다.
    • 세부 전략:
      • 총 필요 인원 need = team × people을 계산한다.
      • 참가자 배열을 오름차순 정렬한 뒤 뒤에서부터 순회하며 need에서 순차적으로 차감한다.
      • need가 0 이하가 되면 즉시 중단하고 선택한 참가자 수를 출력한다.
      • 순회가 끝난 후에도 need가 양수이면 STRESS를 출력한다.

    코드가 하는 일

    • 1) 입력 및 필요 인원 계산 
      int team = Integer.parseInt(st.nextToken());
      int people = Integer.parseInt(st.nextToken());
      
      int need = team * people;
      
      int[] CTP = new int[N];
      
      st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          CTP[i] = Integer.parseInt(st.nextToken());
      }
      
      Arrays.sort(CTP);
      • 팀 수 team과 팀원 수 people을 곱해 충족해야 할 총 인원 need를 계산한다. 참가자 배열을 오름차순 정렬하여 이후 뒤에서부터 순회하면 내림차순 선택이 가능하게 준비한다.
    • 2) 내림차순 그리디 선택 
      int answer = 0;
      
      for (int i = N - 1; i >= 0; i--) {
          need -= CTP[i];
          answer++;
      
          if (need <= 0) break;
      }
      • 가장 많은 인원을 데려올 수 있는 참가자부터 선택하며 need를 차감한다. 매 선택마다 answer를 증가시켜 선택한 참가자 수를 추적한다.
      • need가 0 이하가 되는 순간 목표를 달성했으므로 즉시 break하여 불필요한 탐색을 막는다.
    • 3) 결과 출력 
      if (need > 0) {
          System.out.println("STRESS");
          return;
      }
      
      System.out.println(answer);
      • 순회가 끝난 후에도 need가 양수이면 모든 참가자를 선택해도 인원이 부족한 것이므로 STRESS를 출력하고 종료한다. 그렇지 않으면 최소 참가자 수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 배열 정렬에 O(N log N)이 소요되며, 이후 순회는 최대 O(N)이므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): 참가자 인원 수를 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 14594 — 동방 프로젝트 (Small)

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    동방 프로젝트 (Small) (백준 14594)는
    N개의 방이 일렬로 연결되어 있을 때, M개의 구간 쿼리마다 해당 구간의 문을 부수어 최종적으로 남아 있는 방의 그룹 수를 구하는 문제다.
    인접한 두 방 사이의 문이 부서지면 두 방이 하나의 그룹으로 합쳐지므로, 부서진 문의 수만큼 초기 그룹 수 N에서 차감하면 된다.

    • 초기 상태에서 N개의 방은 모두 독립된 그룹이므로 그룹 수는 N이다.
    • 인접한 두 방 사이의 문이 처음으로 부서질 때마다 두 그룹이 하나로 합쳐지므로 그룹 수가 1 감소한다.
    • 같은 문이 여러 쿼리에서 중복으로 부서지더라도 이미 부서진 문은 그룹 수에 영향을 주지 않으므로, 중복 처리를 방지하는 isBroken 배열이 필요하다.

    전략 개요

    • 핵심 알고리즘: 구간 시뮬레이션 + 부서진 문 중복 방지
    • 선택 이유: 문이 N-1개로 고정이므로 크기 N-1의 boolean 배열로 각 문의 상태를 추적할 수 있다. 쿼리마다 구간 내 문을 순회하며 처음 부서지는 문만 카운트하면 전체 O(M × N) 시뮬레이션으로 해결된다. (Small 버전이므로 N, M이 작아 선형 순회가 허용된다.)
    • 세부 전략:
      • 그룹 수 answer를 N으로 초기화한다.
      • 각 쿼리의 구간 [left, right]에 대해 j를 left부터 right-1까지 순회하며 j번 문(방 j와 방 j+1 사이)을 확인한다.
      • 아직 부서지지 않은 문이면 answer를 1 감소시키고 isBroken을 true로 표시한다.
      • 모든 쿼리를 처리한 뒤 answer를 출력한다.

    코드가 하는 일

    • 1) 초기화 
      boolean[] isBroken = new boolean[N - 1];
      int answer = N;
      • N개의 방 사이에 존재하는 N-1개의 문 상태를 추적하는 배열을 초기화한다. 초기 그룹 수는 방의 수 N과 같다.
    • 2) 쿼리별 구간 처리 
      for (int i = 0; i < M; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int left = Integer.parseInt(st.nextToken());
          int right = Integer.parseInt(st.nextToken());
      
          for (int j = left; j < right; j++) {
              if (!isBroken[j - 1]) {
                  answer--;
                  isBroken[j - 1] = true;
              }
          }
      }
      • j를 left부터 right - 1까지 순회하며 방 j와 방 j+1 사이의 문(인덱스 j-1)을 확인한다.
      • isBroken[j - 1]이 false이면 처음 부서지는 문이므로 answer를 1 감소시키고 isBroken[j - 1]을 true로 표시한다.
      • 이미 부서진 문(isBroken[j - 1] == true)은 그냥 건너뛰어 중복 차감을 방지한다.
    • 3) 결과 출력 
      System.out.println(answer);
      • 모든 쿼리를 처리한 뒤 최종 그룹 수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(M · N): M개의 쿼리마다 최대 N-1개의 문을 순회하므로 전체 O(M · N)이다. Small 버전에서는 N과 M이 충분히 작아 시간 내에 해결된다.
    • 공간 복잡도
      • O(N): 문의 상태를 저장하는 크기 N-1의 boolean 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
