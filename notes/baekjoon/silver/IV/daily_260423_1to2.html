    문제 11558 — The Game of Death

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    The Game of Death (백준 11558)는
    N명이 원형으로 앉아 각자 다음 사람을 지목할 때, 1번 사람부터 시작하여 N번 사람에게 지목이 도달하기까지의 이동 횟수를 구하는 문제다.
    N번에 도달하기 전에 이미 방문한 사람을 다시 방문하면 N번에 영원히 도달할 수 없으므로 0을 출력한다.

    • 1번 사람이 지목한 사람을 따라가며 N번에 도달하면 이동 횟수를 출력한다.
    • 이동 도중 이미 방문한 사람을 다시 방문하면 사이클이 형성된 것이므로 N번에 영원히 도달하지 못한다. 이 경우 0을 출력한다.
    • 방문 여부를 boolean 배열로 추적하여 사이클을 O(N)에 감지한다.

    전략 개요

    • 핵심 알고리즘: 링크드 리스트 순회 + 방문 배열을 이용한 사이클 감지
    • 선택 이유: 각 사람이 다음 지목자를 하나씩만 가지므로 구조가 링크드 리스트와 같다. 현재 노드에서 다음 노드로 이동하며 N에 도달하거나 사이클이 발생하는 두 가지 종료 조건만 처리하면 되므로, 방문 배열 하나로 O(N)에 해결된다.
    • 세부 전략:
      • 1번부터 시작하여 지목 체인을 따라가며 이동 횟수 move를 증가시킨다.
      • 다음 사람 x가 이미 방문된 경우 사이클이 형성된 것이므로 0을 기록하고 종료한다.
      • x가 N번인 경우 목표에 도달한 것이므로 현재 move를 기록하고 종료한다.
      • 그 외의 경우 x를 현재 위치로 갱신하고 방문 표시를 한 뒤 계속 순회한다.

    코드가 하는 일

    • 1) 지목 배열 입력 
      int[] toMember = new int[N + 1];
      for (int i = 1; i <= N; i++) {
          toMember[i] = Integer.parseInt(br.readLine());
      }
      • 1번부터 N번까지 각 사람이 지목하는 다음 사람의 번호를 1-based 배열에 저장한다.
    • 2) 지목 체인 순회 및 종료 조건 처리 
      boolean[] visited = new boolean[N + 1];
      int cur = 1;
      int move = 0;
      
      while (true) {
          int x = toMember[cur];
          move++;
      
          if (visited[x]) {
              sb.append(0).append('\n');
              break;
          }
      
          if (x == N) {
              sb.append(move).append('\n');
              break;
          }
      
          cur = x;
          visited[cur] = true;
      }
      • 현재 위치 cur에서 다음 사람 x = toMember[cur]로 이동하며 move를 증가시킨다.
      • visited[x]가 true이면 이미 방문한 사람을 다시 방문한 것으로, 사이클이 형성되어 N번에 도달 불가능하다. 0을 기록하고 즉시 종료한다.
      • x == N이면 목표에 도달한 것이므로 현재 이동 횟수 move를 기록하고 종료한다.
      • 두 조건 모두 해당하지 않으면 cur = x로 이동하고 visited[cur] = true로 방문 표시를 한다.
    • 3) 결과 누적 및 출력 
      System.out.println(sb);
      • 모든 테스트케이스의 결과를 StringBuilder에 누적한 뒤 한 번에 출력하여 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · N): T개의 테스트케이스마다 최대 N번의 이동으로 순회가 종료되므로 전체 O(T · N)이다.
    • 공간 복잡도
      • O(N): 지목 배열 toMember와 방문 배열 visited가 각각 크기 N+1이며, 테스트케이스마다 초기화되므로 동시에 사용되는 최대 공간은 O(N)이다.

    문제 11332 — 시간초과

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    시간초과 (백준 11332)는
    알고리즘의 시간 복잡도 S, 입력 크기 N, 테스트케이스 수 T, 제한 시간 L이 주어질 때,
    총 연산 횟수가 허용 범위를 초과하는지 판별하여 TLE 여부를 출력하는 문제다.

    • 1초당 최대 연산 수를 1억(10⁸)으로 가정하므로, 허용 최대 연산 수는 maxOps = L × 10⁸이다.
    • 복잡도별 총 연산 수는 f(N) × T이며, 이 값이 maxOps를 초과하면 TLE다.
    • N이 충분히 크면 f(N) 계산 자체가 오버플로우를 일으킬 수 있으므로, 복잡도별로 N의 임계값을 먼저 확인하는 조기 종료 조건이 필요하다.

    전략 개요

    • 핵심 알고리즘: 복잡도별 분기 처리 + 오버플로우 방지 임계값 조기 종료
    • 선택 이유: 시간 복잡도 종류가 O(N), O(N²), O(N³), O(2^N), O(N!) 다섯 가지로 고정이므로 분기 처리로 충분하다. 각 복잡도에서 long 범위를 초과하는 N의 임계값을 미리 확인하면 안전하게 연산할 수 있다.
    • 세부 전략:
      • 허용 최대 연산 수 maxOps = L × 10⁸을 계산한다.
      • 복잡도별로 오버플로우 임계값을 먼저 확인하고, 안전한 범위에서만 f(N) × T > maxOps를 검사한다.
      • O(N!)은 팩토리얼을 직접 계산하며, N ≥ 13이면 13! = 6,227,020,800으로 long 안에서도 maxOps를 초과하므로 즉시 TLE로 처리한다.

    코드가 하는 일

    • 1) 허용 최대 연산 수 계산 
      static final long LIMIT = 100_000_000L;
      
      long maxOps = L * LIMIT;
      • 1초당 최대 연산 수 10⁸에 제한 시간 L을 곱해 허용 최대 연산 수 maxOps를 구한다.
    • 2) 복잡도별 TLE 판별 
      if (S.equals("O(N)")) {
          if (N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N^2)")) {
          if (N >= 31623) isTLE = true;
          else if (N * N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N^3)")) {
          if (N >= 1001) isTLE = true;
          else if (N * N * N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(2^N)")) {
          if (N >= 30) isTLE = true;
          else if ((1L << N) * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N!)")) {
          if (N >= 13) isTLE = true;
          else {
              long res = 1;
              for (int i = 1; i <= N; i++) res *= i;
              if (res * T > maxOps) isTLE = true;
          }
      }
      • O(N²): √(Long.MAX_VALUE) ≈ 30억이지만 maxOps가 최대 10¹³이므로, N² 자체가 이미 10¹⁰을 넘는 N ≥ 31623이면 TLE로 처리한다.
      • O(N³): N ≥ 1001이면 N³ ≥ 10⁹으로 T를 곱하면 오버플로우 위험이 있어 즉시 TLE로 처리한다.
      • O(2^N): N ≥ 30이면 2³⁰ ≈ 10⁹으로 T를 곱하면 long 범위를 초과할 수 있어 즉시 TLE로 처리한다.
      • O(N!): N ≥ 13이면 13! = 6,227,020,800으로 어떤 T에 대해서도 maxOps를 초과하므로 즉시 TLE로 처리한다. N < 13이면 팩토리얼을 직접 계산하여 비교한다.
    • 3) 결과 출력 
      System.out.println(isTLE ? "TLE!" : "May Pass.");
      • isTLE가 true이면 "TLE!", false이면 "May Pass."를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(C): C개의 쿼리마다 복잡도 판별에 O(1) 또는 O(N!) 계산에 최대 O(12)의 상수 반복이 수행되므로 전체 O(C)이다.
    • 공간 복잡도
      • O(1): 쿼리별로 고정된 수의 변수만 사용하므로 상수 공간만 필요하다.
