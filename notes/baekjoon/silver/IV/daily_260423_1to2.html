    문제 11558 — The Game of Death

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    The Game of Death (백준 11558)는
    N명이 원형으로 앉아 각자 다음 사람을 지목할 때, 1번 사람부터 시작하여 N번 사람에게 지목이 도달하기까지의 이동 횟수를 구하는 문제다.
    N번에 도달하기 전에 이미 방문한 사람을 다시 방문하면 N번에 영원히 도달할 수 없으므로 0을 출력한다.

    • 1번 사람이 지목한 사람을 따라가며 N번에 도달하면 이동 횟수를 출력한다.
    • 이동 도중 이미 방문한 사람을 다시 방문하면 사이클이 형성된 것이므로 N번에 영원히 도달하지 못한다. 이 경우 0을 출력한다.
    • 방문 여부를 boolean 배열로 추적하여 사이클을 O(N)에 감지한다.

    전략 개요

    • 핵심 알고리즘: 링크드 리스트 순회 + 방문 배열을 이용한 사이클 감지
    • 선택 이유: 각 사람이 다음 지목자를 하나씩만 가지므로 구조가 링크드 리스트와 같다. 현재 노드에서 다음 노드로 이동하며 N에 도달하거나 사이클이 발생하는 두 가지 종료 조건만 처리하면 되므로, 방문 배열 하나로 O(N)에 해결된다.
    • 세부 전략:
      • 1번부터 시작하여 지목 체인을 따라가며 이동 횟수 move를 증가시킨다.
      • 다음 사람 x가 이미 방문된 경우 사이클이 형성된 것이므로 0을 기록하고 종료한다.
      • x가 N번인 경우 목표에 도달한 것이므로 현재 move를 기록하고 종료한다.
      • 그 외의 경우 x를 현재 위치로 갱신하고 방문 표시를 한 뒤 계속 순회한다.

    코드가 하는 일

    • 1) 지목 배열 입력 
      int[] toMember = new int[N + 1];
      for (int i = 1; i <= N; i++) {
          toMember[i] = Integer.parseInt(br.readLine());
      }
      • 1번부터 N번까지 각 사람이 지목하는 다음 사람의 번호를 1-based 배열에 저장한다.
    • 2) 지목 체인 순회 및 종료 조건 처리 
      boolean[] visited = new boolean[N + 1];
      int cur = 1;
      int move = 0;
      
      while (true) {
          int x = toMember[cur];
          move++;
      
          if (visited[x]) {
              sb.append(0).append('\n');
              break;
          }
      
          if (x == N) {
              sb.append(move).append('\n');
              break;
          }
      
          cur = x;
          visited[cur] = true;
      }
      • 현재 위치 cur에서 다음 사람 x = toMember[cur]로 이동하며 move를 증가시킨다.
      • visited[x]가 true이면 이미 방문한 사람을 다시 방문한 것으로, 사이클이 형성되어 N번에 도달 불가능하다. 0을 기록하고 즉시 종료한다.
      • x == N이면 목표에 도달한 것이므로 현재 이동 횟수 move를 기록하고 종료한다.
      • 두 조건 모두 해당하지 않으면 cur = x로 이동하고 visited[cur] = true로 방문 표시를 한다.
    • 3) 결과 누적 및 출력 
      System.out.println(sb);
      • 모든 테스트케이스의 결과를 StringBuilder에 누적한 뒤 한 번에 출력하여 I/O 비용을 줄인다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(T · N): T개의 테스트케이스마다 최대 N번의 이동으로 순회가 종료되므로 전체 O(T · N)이다.
    • 공간 복잡도
      • O(N): 지목 배열 toMember와 방문 배열 visited가 각각 크기 N+1이며, 테스트케이스마다 초기화되므로 동시에 사용되는 최대 공간은 O(N)이다.

    문제 11332 — 시간초과

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    사탕 (백준 1812)는
    N명의 학생이 일렬로 앉아 있을 때 인접한 두 학생의 사탕 합이 주어지면, 각 학생이 원래 가지고 있던 사탕 수를 복원하는 문제다.
    첫 번째 학생의 사탕 수를 수학적으로 유도한 뒤, 인접합 관계를 역추적하여 모든 학생의 값을 결정한다.

    • 입력 candies[i]는 i번째 학생과 i+1번째 학생의 사탕 합, 즉 a[i] + a[i+1]을 나타낸다.
    • 전체 합을 2로 나누면 a[0] + a[1] + ... + a[N-1]의 합이 되고, 여기서 홀수 인덱스의 인접합을 빼면 a[0]만 남는다.
    • a[0]을 구한 뒤 a[i+1] = candies[i] - a[i] 점화식으로 나머지 값을 순서대로 복원할 수 있다.

    전략 개요

    • 핵심 알고리즘: 수학적 초기값 유도 + 점화식 역추적
    • 선택 이유: a[i] + a[i+1] = candies[i] 관계에서 짝수 인덱스 합과 홀수 인덱스 합의 차이를 이용하면 a[0]을 O(N)에 구할 수 있다. a[0]이 결정되면 이후 모든 값은 a[i+1] = candies[i] - a[i]로 O(N)에 순서대로 복원된다.
    • 세부 전략:
      • 전체 인접합의 합을 2로 나눈 값에서 홀수 인덱스(candies[1], candies[3], ...)의 합을 빼면 a[0]이 남는다.
      • a[0]부터 시작해 sub = candies[i] - student로 다음 학생의 사탕 수를 순서대로 계산한다.
      • 결과를 StringBuilder에 누적하여 한 번에 출력한다.

    코드가 하는 일

    • 1) 입력 및 전체 합 계산 
      int[] candies = new int[N];
      int totalCandies = 0;
      
      for (int i = 0; i < N; i++) {
          int candy = Integer.parseInt(br.readLine());
          candies[i] = candy;
          totalCandies += candy;
      }
      • N개의 인접합 candies[i] = a[i] + a[i+1]을 배열에 저장하고 전체 합을 구한다.
    • 2) 첫 번째 학생의 사탕 수 유도 
      totalCandies /= 2;
      
      for (int i = 1; i < candies.length; i += 2) {
          totalCandies -= candies[i];
      }
      
      int student = totalCandies;
      • totalCandies / 2는 a[0] + a[1] + ... + a[N-1]의 합이다. 여기서 홀수 인덱스의 인접합 candies[1], candies[3], ...을 빼면 다음과 같이 전개된다. candies[1] = a[1] + a[2], candies[3] = a[3] + a[4], ...을 빼고 나면 a[0]만 남아 첫 번째 학생의 사탕 수가 결정된다.
    • 3) 점화식으로 나머지 학생 복원 
      sb.append(student).append("\n");
      
      for (int i = 0; i < candies.length - 1; i++) {
          int sub = candies[i] - student;
          sb.append(sub).append('\n');
          student = sub;
      }
      • a[0]을 먼저 출력한 뒤, a[i+1] = candies[i] - a[i] 점화식으로 다음 학생의 사탕 수를 순서대로 계산한다.
      • student를 현재 학생의 값으로 갱신하며 N-1번 반복하면 모든 학생의 값이 복원된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 입력 저장, 홀수 인덱스 합산, 점화식 복원 모두 각각 O(N)의 단일 순회로 구성되므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 인접합을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
