    문제 14584 — 암호 해독

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    암호 해독 (백준 14584)는
    카이사르 암호로 인코딩된 문자열이 주어질 때, 0~25 중 올바른 시프트 값을 찾아 복호화된 문자열을 출력하는 문제다.
    복호화된 문자열에 사전 단어 중 하나라도 포함되면 해당 시프트 값이 정답이다.

    • 카이사르 암호는 알파벳을 일정한 수만큼 순환 이동하는 방식이므로, 가능한 시프트 값은 0부터 25까지 총 26가지다.
    • 각 시프트 값으로 복호화한 문자열이 사전 단어를 포함하는지 확인하며, 처음으로 조건을 만족하는 복호화 결과를 출력한다.
    • 사전 단어를 HashSet에 저장하면 각 단어의 포함 여부 확인을 빠르게 수행할 수 있다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (26가지 시프트 값 순회) + 문자열 포함 검사
    • 선택 이유: 가능한 시프트 값이 26가지로 고정이므로 모든 경우를 직접 시도하는 완전 탐색이 적합하다. 각 시프트마다 문자열 전체를 복호화하고, 사전 단어 포함 여부를 contains()로 확인하면 O(26 × L × W)로 해결된다.
    • 세부 전략:
      • 시프트 값 i를 0부터 25까지 순회하며 암호문의 각 문자를 (c - 'a' + i) % 26 + 'a'로 복호화한다.
      • 복호화된 문자열이 사전 단어를 하나라도 포함하면 즉시 출력하고 종료한다.
      • 조건을 만족하지 않으면 StringBuilder를 초기화하고 다음 시프트 값을 시도한다.

    코드가 하는 일

    • 1) 입력 및 사전 초기화 
      char[] encoded = br.readLine().toCharArray();
      
      Set<String> set = new HashSet<>();
      
      int N = Integer.parseInt(br.readLine());
      for (int i = 0; i < N; i++) {
          set.add(br.readLine());
      }
      • 암호문을 char[]로 저장하고, N개의 사전 단어를 HashSet에 저장한다. HashSet을 사용하면 이후 contains() 호출 시 O(1)에 단어 존재 여부를 확인할 수 있다.
    • 2) 26가지 시프트 값 순회 및 복호화 
      for (int i = 0; i < 26; i++) {
          for (char c : encoded) {
              sb.append((char) ((c - 'a' + i) % 26 + 'a'));
          }
      
          String decoded = sb.toString();
      
          for (String word : set) {
              if (decoded.contains(word)) {
                  System.out.println(decoded);
                  return;
              }
          }
      
          sb.setLength(0);
      }
      • 시프트 값 i로 각 문자를 (c - 'a' + i) % 26 + 'a'로 변환한다. % 26으로 알파벳 범위를 순환시키고, 다시 'a'를 더해 소문자 범위로 매핑한다.
      • 복호화된 문자열 decoded에 대해 사전 단어를 순회하며 contains()로 포함 여부를 확인한다. 하나라도 포함되면 즉시 출력하고 return으로 종료한다.
      • 조건 불만족 시 sb.setLength(0)으로 버퍼를 초기화하고 다음 시프트 값을 시도한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(26 · (L + N · W)): 26가지 시프트 값마다 길이 L인 문자열을 복호화하고, N개의 사전 단어 각각에 대해 contains()를 수행한다. contains()는 내부적으로 O(L · W)이므로 전체 O(26 · L · N · W)이나, L, N, W가 모두 작으므로 실질적으로 빠르게 동작한다.
    • 공간 복잡도
      • O(N · W + L): N개의 사전 단어를 저장하는 HashSet과 복호화 문자열을 담는 StringBuilder가 주요 메모리 사용처이다. (W는 사전 단어의 평균 길이)

    문제 1812 — 사탕

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    사탕 (백준 1812)는
    N명의 학생이 일렬로 앉아 있을 때 인접한 두 학생의 사탕 합이 주어지면, 각 학생이 원래 가지고 있던 사탕 수를 복원하는 문제다.
    첫 번째 학생의 사탕 수를 수학적으로 유도한 뒤, 인접합 관계를 역추적하여 모든 학생의 값을 결정한다.

    • 입력 candies[i]는 i번째 학생과 i+1번째 학생의 사탕 합, 즉 a[i] + a[i+1]을 나타낸다.
    • 전체 합을 2로 나누면 a[0] + a[1] + ... + a[N-1]의 합이 되고, 여기서 홀수 인덱스의 인접합을 빼면 a[0]만 남는다.
    • a[0]을 구한 뒤 a[i+1] = candies[i] - a[i] 점화식으로 나머지 값을 순서대로 복원할 수 있다.

    전략 개요

    • 핵심 알고리즘: 수학적 초기값 유도 + 점화식 역추적
    • 선택 이유: a[i] + a[i+1] = candies[i] 관계에서 짝수 인덱스 합과 홀수 인덱스 합의 차이를 이용하면 a[0]을 O(N)에 구할 수 있다. a[0]이 결정되면 이후 모든 값은 a[i+1] = candies[i] - a[i]로 O(N)에 순서대로 복원된다.
    • 세부 전략:
      • 전체 인접합의 합을 2로 나눈 값에서 홀수 인덱스(candies[1], candies[3], ...)의 합을 빼면 a[0]이 남는다.
      • a[0]부터 시작해 sub = candies[i] - student로 다음 학생의 사탕 수를 순서대로 계산한다.
      • 결과를 StringBuilder에 누적하여 한 번에 출력한다.

    코드가 하는 일

    • 1) 입력 및 전체 합 계산 
      int[] candies = new int[N];
      int totalCandies = 0;
      
      for (int i = 0; i < N; i++) {
          int candy = Integer.parseInt(br.readLine());
          candies[i] = candy;
          totalCandies += candy;
      }
      • N개의 인접합 candies[i] = a[i] + a[i+1]을 배열에 저장하고 전체 합을 구한다.
    • 2) 첫 번째 학생의 사탕 수 유도 
      totalCandies /= 2;
      
      for (int i = 1; i < candies.length; i += 2) {
          totalCandies -= candies[i];
      }
      
      int student = totalCandies;
      • totalCandies / 2는 a[0] + a[1] + ... + a[N-1]의 합이다. 여기서 홀수 인덱스의 인접합 candies[1], candies[3], ...을 빼면 다음과 같이 전개된다. candies[1] = a[1] + a[2], candies[3] = a[3] + a[4], ...을 빼고 나면 a[0]만 남아 첫 번째 학생의 사탕 수가 결정된다.
    • 3) 점화식으로 나머지 학생 복원 
      sb.append(student).append("\n");
      
      for (int i = 0; i < candies.length - 1; i++) {
          int sub = candies[i] - student;
          sb.append(sub).append('\n');
          student = sub;
      }
      • a[0]을 먼저 출력한 뒤, a[i+1] = candies[i] - a[i] 점화식으로 다음 학생의 사탕 수를 순서대로 계산한다.
      • student를 현재 학생의 값으로 갱신하며 N-1번 반복하면 모든 학생의 값이 복원된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 입력 저장, 홀수 인덱스 합산, 점화식 복원 모두 각각 O(N)의 단일 순회로 구성되므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 인접합을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.
