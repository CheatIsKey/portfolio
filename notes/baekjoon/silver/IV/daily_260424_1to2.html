    문제 2358 — 평행선

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    평행선 (백준 2358)는
    N개의 점이 주어질 때, 축에 평행한 직선(수평선 또는 수직선) 중 2개 이상의 점을 지나는 직선의 수를 구하는 문제다.
    같은 x좌표를 공유하는 점이 2개 이상이면 수직선, 같은 y좌표를 공유하는 점이 2개 이상이면 수평선이 존재한다.

    • 축에 평행한 직선은 수직선(x = k)과 수평선(y = k) 두 종류뿐이다.
    • 같은 x좌표를 가진 점이 2개 이상이면 해당 x값에 대한 수직선이 존재하고, 같은 y좌표를 가진 점이 2개 이상이면 해당 y값에 대한 수평선이 존재한다.
    • x좌표별, y좌표별 등장 횟수를 HashMap으로 집계한 뒤, 횟수가 2 이상인 항목의 수를 세면 된다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 좌표별 빈도 집계
    • 선택 이유: x좌표와 y좌표 각각에 대해 등장 횟수를 HashMap으로 집계하면, 2개 이상 등장한 좌표값의 수를 O(N)에 구할 수 있다. 좌표 범위가 넓어 배열 인덱싱이 어렵기 때문에 HashMap이 적합하다.
    • 세부 전략:
      • 각 점의 x좌표와 y좌표를 읽으며 posX와 posY HashMap에 등장 횟수를 누적한다.
      • 모든 점을 처리한 뒤 posX의 값 중 2 이상인 항목 수와 posY의 값 중 2 이상인 항목 수를 합산한다.
      • 합산 결과가 곧 축에 평행한 직선의 총 수다.

    코드가 하는 일

    • 1) 좌표별 등장 횟수 집계 
      Map<Integer, Integer> posX = new HashMap<>();
      Map<Integer, Integer> posY = new HashMap<>();
      
      for (int i = 0; i < n; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int x = Integer.parseInt(st.nextToken());
          int y = Integer.parseInt(st.nextToken());
      
          posX.put(x, posX.getOrDefault(x, 0) + 1);
          posY.put(y, posY.getOrDefault(y, 0) + 1);
      }
      • 각 점의 x좌표와 y좌표를 읽으며 posX에는 x좌표별, posY에는 y좌표별 등장 횟수를 누적한다. getOrDefault로 키가 없는 경우 0을 기본값으로 사용한다.
    • 2) 직선 수 집계 
      int answer = 0;
      
      for (int value : posX.values()) {
          if (value > 1) {
              answer++;
          }
      }
      
      for (int value : posY.values()) {
          if (value > 1) {
              answer++;
          }
      }
      • posX에서 값이 2 이상인 항목은 해당 x값에 수직선이 존재함을 의미하며, posY에서 값이 2 이상인 항목은 해당 y값에 수평선이 존재함을 의미한다. 두 경우를 합산하면 전체 축 평행 직선의 수가 된다.
    • 3) 결과 출력 
      System.out.println(answer);
      • 축에 평행하고 2개 이상의 점을 지나는 직선의 총 수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 점을 순회하며 HashMap에 삽입하는 데 O(N), 이후 두 HashMap의 값을 순회하는 데 O(N)이 소요되므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): x좌표 HashMap과 y좌표 HashMap이 각각 최대 N개의 항목을 저장하므로 전체 O(N)의 공간을 사용한다.

    문제 22351 — 수학은 체육과목 입니다 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    시간초과 (백준 11332)는
    알고리즘의 시간 복잡도 S, 입력 크기 N, 테스트케이스 수 T, 제한 시간 L이 주어질 때,
    총 연산 횟수가 허용 범위를 초과하는지 판별하여 TLE 여부를 출력하는 문제다.

    • 1초당 최대 연산 수를 1억(10⁸)으로 가정하므로, 허용 최대 연산 수는 maxOps = L × 10⁸이다.
    • 복잡도별 총 연산 수는 f(N) × T이며, 이 값이 maxOps를 초과하면 TLE다.
    • N이 충분히 크면 f(N) 계산 자체가 오버플로우를 일으킬 수 있으므로, 복잡도별로 N의 임계값을 먼저 확인하는 조기 종료 조건이 필요하다.

    전략 개요

    • 핵심 알고리즘: 복잡도별 분기 처리 + 오버플로우 방지 임계값 조기 종료
    • 선택 이유: 시간 복잡도 종류가 O(N), O(N²), O(N³), O(2^N), O(N!) 다섯 가지로 고정이므로 분기 처리로 충분하다. 각 복잡도에서 long 범위를 초과하는 N의 임계값을 미리 확인하면 안전하게 연산할 수 있다.
    • 세부 전략:
      • 허용 최대 연산 수 maxOps = L × 10⁸을 계산한다.
      • 복잡도별로 오버플로우 임계값을 먼저 확인하고, 안전한 범위에서만 f(N) × T > maxOps를 검사한다.
      • O(N!)은 팩토리얼을 직접 계산하며, N ≥ 13이면 13! = 6,227,020,800으로 long 안에서도 maxOps를 초과하므로 즉시 TLE로 처리한다.

    코드가 하는 일

    • 1) 허용 최대 연산 수 계산 
      static final long LIMIT = 100_000_000L;
      
      long maxOps = L * LIMIT;
      • 1초당 최대 연산 수 10⁸에 제한 시간 L을 곱해 허용 최대 연산 수 maxOps를 구한다.
    • 2) 복잡도별 TLE 판별 
      if (S.equals("O(N)")) {
          if (N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N^2)")) {
          if (N >= 31623) isTLE = true;
          else if (N * N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N^3)")) {
          if (N >= 1001) isTLE = true;
          else if (N * N * N * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(2^N)")) {
          if (N >= 30) isTLE = true;
          else if ((1L << N) * T > maxOps) isTLE = true;
      }
      else if (S.equals("O(N!)")) {
          if (N >= 13) isTLE = true;
          else {
              long res = 1;
              for (int i = 1; i <= N; i++) res *= i;
              if (res * T > maxOps) isTLE = true;
          }
      }
      • O(N²): √(Long.MAX_VALUE) ≈ 30억이지만 maxOps가 최대 10¹³이므로, N² 자체가 이미 10¹⁰을 넘는 N ≥ 31623이면 TLE로 처리한다.
      • O(N³): N ≥ 1001이면 N³ ≥ 10⁹으로 T를 곱하면 오버플로우 위험이 있어 즉시 TLE로 처리한다.
      • O(2^N): N ≥ 30이면 2³⁰ ≈ 10⁹으로 T를 곱하면 long 범위를 초과할 수 있어 즉시 TLE로 처리한다.
      • O(N!): N ≥ 13이면 13! = 6,227,020,800으로 어떤 T에 대해서도 maxOps를 초과하므로 즉시 TLE로 처리한다. N < 13이면 팩토리얼을 직접 계산하여 비교한다.
    • 3) 결과 출력 
      System.out.println(isTLE ? "TLE!" : "May Pass.");
      • isTLE가 true이면 "TLE!", false이면 "May Pass."를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(C): C개의 쿼리마다 복잡도 판별에 O(1) 또는 O(N!) 계산에 최대 O(12)의 상수 반복이 수행되므로 전체 O(C)이다.
    • 공간 복잡도
      • O(1): 쿼리별로 고정된 수의 변수만 사용하므로 상수 공간만 필요하다.
