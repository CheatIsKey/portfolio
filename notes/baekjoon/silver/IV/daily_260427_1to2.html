    문제 2358 — 평행선

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    평행선 (백준 2358)는
    N개의 점이 주어질 때, 축에 평행한 직선(수평선 또는 수직선) 중 2개 이상의 점을 지나는 직선의 수를 구하는 문제다.
    같은 x좌표를 공유하는 점이 2개 이상이면 수직선, 같은 y좌표를 공유하는 점이 2개 이상이면 수평선이 존재한다.

    • 축에 평행한 직선은 수직선(x = k)과 수평선(y = k) 두 종류뿐이다.
    • 같은 x좌표를 가진 점이 2개 이상이면 해당 x값에 대한 수직선이 존재하고, 같은 y좌표를 가진 점이 2개 이상이면 해당 y값에 대한 수평선이 존재한다.
    • x좌표별, y좌표별 등장 횟수를 HashMap으로 집계한 뒤, 횟수가 2 이상인 항목의 수를 세면 된다.

    전략 개요

    • 핵심 알고리즘: HashMap을 이용한 좌표별 빈도 집계
    • 선택 이유: x좌표와 y좌표 각각에 대해 등장 횟수를 HashMap으로 집계하면, 2개 이상 등장한 좌표값의 수를 O(N)에 구할 수 있다. 좌표 범위가 넓어 배열 인덱싱이 어렵기 때문에 HashMap이 적합하다.
    • 세부 전략:
      • 각 점의 x좌표와 y좌표를 읽으며 posX와 posY HashMap에 등장 횟수를 누적한다.
      • 모든 점을 처리한 뒤 posX의 값 중 2 이상인 항목 수와 posY의 값 중 2 이상인 항목 수를 합산한다.
      • 합산 결과가 곧 축에 평행한 직선의 총 수다.

    코드가 하는 일

    • 1) 좌표별 등장 횟수 집계 
      Map<Integer, Integer> posX = new HashMap<>();
      Map<Integer, Integer> posY = new HashMap<>();
      
      for (int i = 0; i < n; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int x = Integer.parseInt(st.nextToken());
          int y = Integer.parseInt(st.nextToken());
      
          posX.put(x, posX.getOrDefault(x, 0) + 1);
          posY.put(y, posY.getOrDefault(y, 0) + 1);
      }
      • 각 점의 x좌표와 y좌표를 읽으며 posX에는 x좌표별, posY에는 y좌표별 등장 횟수를 누적한다. getOrDefault로 키가 없는 경우 0을 기본값으로 사용한다.
    • 2) 직선 수 집계 
      int answer = 0;
      
      for (int value : posX.values()) {
          if (value > 1) {
              answer++;
          }
      }
      
      for (int value : posY.values()) {
          if (value > 1) {
              answer++;
          }
      }
      • posX에서 값이 2 이상인 항목은 해당 x값에 수직선이 존재함을 의미하며, posY에서 값이 2 이상인 항목은 해당 y값에 수평선이 존재함을 의미한다. 두 경우를 합산하면 전체 축 평행 직선의 수가 된다.
    • 3) 결과 출력 
      System.out.println(answer);
      • 축에 평행하고 2개 이상의 점을 지나는 직선의 총 수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N개의 점을 순회하며 HashMap에 삽입하는 데 O(N), 이후 두 HashMap의 값을 순회하는 데 O(N)이 소요되므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): x좌표 HashMap과 y좌표 HashMap이 각각 최대 N개의 항목을 저장하므로 전체 O(N)의 공간을 사용한다.

    문제 22351 — 수학은 체육과목 입니다 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수학은 체육과목 입니다 3 (백준 22351)는
    연속된 자연수를 이어 붙인 문자열이 주어질 때, 시작 수와 끝 수를 구하는 문제다.
    시작 수를 1부터 999까지 완전 탐색하며, 각 시작 수에서 연속된 수를 이어 붙여 주어진 문자열과 일치하는지 확인한다.

    • 시작 수 i에서 i, i+1, i+2, ...를 순서대로 이어 붙이며 주어진 문자열 S의 접두사인지 확인한다.
    • 이어 붙인 문자열이 S의 접두사가 아닌 순간 해당 시작 수로는 불가능하므로 즉시 중단하고 다음 시작 수를 시도한다.
    • 이어 붙인 문자열이 S와 완전히 일치하면 현재 시작 수 i와 마지막으로 추가한 수 x가 정답이다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (시작 수 순회 + 접두사 검사로 조기 종료)
    • 선택 이유: 시작 수의 범위가 1~999로 제한되어 있으므로 완전 탐색이 가능하다. 각 시작 수에 대해 이어 붙인 문자열이 S의 접두사인지 매 단계 확인하면, 조건을 만족하지 않는 순간 즉시 중단하여 불필요한 탐색을 줄인다.
    • 세부 전략:
      • 시작 수 i를 1부터 999까지 순회한다.
      • i부터 시작해 x를 1씩 증가시키며 sb에 이어 붙인다.
      • 이어 붙인 문자열이 S의 접두사가 아니면 즉시 break하고 다음 i를 시도한다.
      • 이어 붙인 문자열이 S와 완전히 일치하면 i와 x를 출력하고 종료한다.

    코드가 하는 일

    • 1) 시작 수 완전 탐색 
      for (int i = 1; i <= 999; i++) {
          int x = i;
      
          while (sb.length() < S.length()) {
              sb.append(x);
      
              if (!S.startsWith(sb.toString())) break;
      
              if (sb.toString().equals(S)) {
                  System.out.println(i + " " + x);
                  return;
              }
      
              x++;
          }
      
          sb.setLength(0);
      }
      • 시작 수 i를 1부터 999까지 순회하며, 각 i에 대해 x = i부터 숫자를 하나씩 sb에 이어 붙인다.
      • S.startsWith(sb.toString()): 현재까지 이어 붙인 문자열이 S의 접두사가 아니면 이 시작 수로는 정답이 될 수 없으므로 즉시 break한다.
      • sb.toString().equals(S): 이어 붙인 문자열이 S와 완전히 일치하면 시작 수 i와 마지막으로 추가한 수 x를 출력하고 종료한다.
      • 다음 시작 수를 시도하기 전 sb.setLength(0)으로 버퍼를 초기화한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(999 · L): 시작 수 999가지에 대해 각각 최대 L(문자열 길이)만큼 이어 붙이며 접두사 검사를 수행하므로 전체 O(999 · L)이다. 시작 수가 상수로 고정되므로 사실상 O(L)에 수렴한다.
    • 공간 복잡도
      • O(L): 이어 붙인 문자열을 저장하는 StringBuilder가 최대 S의 길이 L만큼 커지므로 O(L)의 공간을 사용한다.
