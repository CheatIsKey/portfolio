    문제 17419 — 비트가 넘쳐흘러

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비트가 넘쳐흘러 (백준 17419)는
    N비트 이진수 문자열이 주어질 때, 모든 비트를 0으로 만들기 위해 필요한 최소 연산 횟수를 구하는 문제다.
    연산은 가장 낮은 1비트와 그보다 낮은 모든 비트를 0으로 만드는 것이므로, 결국 1의 개수만큼 연산이 필요하다.

    • 연산 한 번은 현재 수에서 가장 낮은 위치의 1비트와 그 아래 비트를 모두 0으로 만든다.
    • 이 연산은 정확히 1비트 하나를 제거하는 효과이므로, 초기 이진수에서 1의 개수가 곧 필요한 최소 연산 횟수다.
    • 따라서 문자열에서 '1'의 개수만 세면 정답을 O(N)에 구할 수 있다.

    전략 개요

    • 핵심 알고리즘: 수학적 관찰 (1비트 개수 세기)
    • 선택 이유: 연산 한 번이 정확히 1비트 하나를 소거하는 효과임을 관찰하면, 시뮬레이션 없이 문자열에서 '1'의 개수를 세는 것만으로 O(N)에 정답을 구할 수 있다.
    • 세부 전략:
      • 이진수 문자열을 순회하며 '1'인 문자의 수를 누적한다.
      • 누적된 값이 곧 최소 연산 횟수다.

    코드가 하는 일

    • 1) 입력 및 1비트 개수 집계 
      int N = Integer.parseInt(br.readLine());
      char[] arr = br.readLine().toCharArray();
      
      int answer = 0;
      
      for (char c : arr) {
          if (c == '1') answer++;
      }
      • N비트 이진수 문자열을 char[]로 변환한 뒤 순회하며 '1'인 문자의 수를 answer에 누적한다. 연산 한 번이 1비트를 정확히 하나 소거하므로, 1의 개수가 곧 모든 비트를 0으로 만드는 데 필요한 최소 연산 횟수다.
    • 2) 결과 출력 
      System.out.println(answer);
      • 최소 연산 횟수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N비트 문자열을 한 번 순회하며 각 문자에 대해 O(1) 비교만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 입력 문자열을 저장하는 char[]가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 22351 — 수학은 체육과목 입니다 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수학은 체육과목 입니다 3 (백준 22351)는
    연속된 자연수를 이어 붙인 문자열이 주어질 때, 시작 수와 끝 수를 구하는 문제다.
    시작 수를 1부터 999까지 완전 탐색하며, 각 시작 수에서 연속된 수를 이어 붙여 주어진 문자열과 일치하는지 확인한다.

    • 시작 수 i에서 i, i+1, i+2, ...를 순서대로 이어 붙이며 주어진 문자열 S의 접두사인지 확인한다.
    • 이어 붙인 문자열이 S의 접두사가 아닌 순간 해당 시작 수로는 불가능하므로 즉시 중단하고 다음 시작 수를 시도한다.
    • 이어 붙인 문자열이 S와 완전히 일치하면 현재 시작 수 i와 마지막으로 추가한 수 x가 정답이다.

    전략 개요

    • 핵심 알고리즘: 완전 탐색 (시작 수 순회 + 접두사 검사로 조기 종료)
    • 선택 이유: 시작 수의 범위가 1~999로 제한되어 있으므로 완전 탐색이 가능하다. 각 시작 수에 대해 이어 붙인 문자열이 S의 접두사인지 매 단계 확인하면, 조건을 만족하지 않는 순간 즉시 중단하여 불필요한 탐색을 줄인다.
    • 세부 전략:
      • 시작 수 i를 1부터 999까지 순회한다.
      • i부터 시작해 x를 1씩 증가시키며 sb에 이어 붙인다.
      • 이어 붙인 문자열이 S의 접두사가 아니면 즉시 break하고 다음 i를 시도한다.
      • 이어 붙인 문자열이 S와 완전히 일치하면 i와 x를 출력하고 종료한다.

    코드가 하는 일

    • 1) 시작 수 완전 탐색 
      for (int i = 1; i <= 999; i++) {
          int x = i;
      
          while (sb.length() < S.length()) {
              sb.append(x);
      
              if (!S.startsWith(sb.toString())) break;
      
              if (sb.toString().equals(S)) {
                  System.out.println(i + " " + x);
                  return;
              }
      
              x++;
          }
      
          sb.setLength(0);
      }
      • 시작 수 i를 1부터 999까지 순회하며, 각 i에 대해 x = i부터 숫자를 하나씩 sb에 이어 붙인다.
      • S.startsWith(sb.toString()): 현재까지 이어 붙인 문자열이 S의 접두사가 아니면 이 시작 수로는 정답이 될 수 없으므로 즉시 break한다.
      • sb.toString().equals(S): 이어 붙인 문자열이 S와 완전히 일치하면 시작 수 i와 마지막으로 추가한 수 x를 출력하고 종료한다.
      • 다음 시작 수를 시도하기 전 sb.setLength(0)으로 버퍼를 초기화한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(999 · L): 시작 수 999가지에 대해 각각 최대 L(문자열 길이)만큼 이어 붙이며 접두사 검사를 수행하므로 전체 O(999 · L)이다. 시작 수가 상수로 고정되므로 사실상 O(L)에 수렴한다.
    • 공간 복잡도
      • O(L): 이어 붙인 문자열을 저장하는 StringBuilder가 최대 S의 길이 L만큼 커지므로 O(L)의 공간을 사용한다.
