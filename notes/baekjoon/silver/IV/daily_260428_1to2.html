    문제 17419 — 비트가 넘쳐흘러

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비트가 넘쳐흘러 (백준 17419)는
    N비트 이진수 문자열이 주어질 때, 모든 비트를 0으로 만들기 위해 필요한 최소 연산 횟수를 구하는 문제다.
    연산은 가장 낮은 1비트와 그보다 낮은 모든 비트를 0으로 만드는 것이므로, 결국 1의 개수만큼 연산이 필요하다.

    • 연산 한 번은 현재 수에서 가장 낮은 위치의 1비트와 그 아래 비트를 모두 0으로 만든다.
    • 이 연산은 정확히 1비트 하나를 제거하는 효과이므로, 초기 이진수에서 1의 개수가 곧 필요한 최소 연산 횟수다.
    • 따라서 문자열에서 '1'의 개수만 세면 정답을 O(N)에 구할 수 있다.

    전략 개요

    • 핵심 알고리즘: 수학적 관찰 (1비트 개수 세기)
    • 선택 이유: 연산 한 번이 정확히 1비트 하나를 소거하는 효과임을 관찰하면, 시뮬레이션 없이 문자열에서 '1'의 개수를 세는 것만으로 O(N)에 정답을 구할 수 있다.
    • 세부 전략:
      • 이진수 문자열을 순회하며 '1'인 문자의 수를 누적한다.
      • 누적된 값이 곧 최소 연산 횟수다.

    코드가 하는 일

    • 1) 입력 및 1비트 개수 집계 
      int N = Integer.parseInt(br.readLine());
      char[] arr = br.readLine().toCharArray();
      
      int answer = 0;
      
      for (char c : arr) {
          if (c == '1') answer++;
      }
      • N비트 이진수 문자열을 char[]로 변환한 뒤 순회하며 '1'인 문자의 수를 answer에 누적한다. 연산 한 번이 1비트를 정확히 하나 소거하므로, 1의 개수가 곧 모든 비트를 0으로 만드는 데 필요한 최소 연산 횟수다.
    • 2) 결과 출력 
      System.out.println(answer);
      • 최소 연산 횟수 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): N비트 문자열을 한 번 순회하며 각 문자에 대해 O(1) 비교만 수행하므로 전체 O(N)이다.
    • 공간 복잡도
      • O(N): 입력 문자열을 저장하는 char[]가 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 5637 — 가장 긴 단어

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    가장 긴 단어 (백준 5637)는
    여러 줄의 텍스트에서 특수문자를 구분자로 단어를 추출하여, "E-N-D"가 등장하기 전까지 가장 긴 단어를 소문자로 출력하는 문제다.
    공백 및 다양한 특수문자를 구분자로 처리하여 순수한 단어만 추출해야 한다.

    • 입력 종료 신호는 "E-N-D"이며, 이 단어를 만나는 즉시 탐색을 중단한다.
    • 공백 외에도 ~!@#$%^&*()_+`={}|[]\:";'<>?,./ 등 다양한 특수문자가 구분자로 사용된다.
    • 가장 긴 단어가 여러 개일 경우 먼저 등장한 단어를 출력하며, 최종 출력은 소문자로 변환한다.

    전략 개요

    • 핵심 알고리즘: StringTokenizer 다중 구분자 파싱 + 단일 순회 최댓값 추적
    • 선택 이유: Java의 StringTokenizer는 두 번째 인자로 구분자 문자열을 받아 여러 특수문자를 한 번에 처리할 수 있다. 입력을 한 줄씩 읽으며 토큰을 추출하고, "E-N-D" 감지와 최댓값 갱신을 동시에 수행하면 O(L)에 해결된다.
    • 세부 전략:
      • EOF까지 한 줄씩 읽으며 특수문자를 구분자로 토큰을 추출한다.
      • 토큰이 "E-N-D"이면 이중 루프를 즉시 탈출한다.
      • 그 외 토큰은 현재 최장 단어와 길이를 비교하여 더 길면 갱신한다.
      • 최종적으로 최장 단어를 소문자로 변환하여 출력한다.

    코드가 하는 일

    • 1) 다중 구분자로 토큰 파싱 
      outer:
      while (true) {
          String line = br.readLine();
          if (line == null) break;
      
          StringTokenizer st = new StringTokenizer(line, " ~!@#$%^&*()_+`={}|[]\\:\";'<>?,./");
      
          while (st.hasMoreTokens()) {
              String word = st.nextToken();
              ...
          }
      }
      • EOF를 만날 때까지 한 줄씩 읽으며 공백과 특수문자 전체를 구분자로 지정한 StringTokenizer로 토큰을 추출한다. outer 레이블을 사용하여 "E-N-D" 감지 시 이중 루프를 한 번에 탈출할 수 있다.
    • 2) 종료 신호 감지 및 최장 단어 갱신 
      if (word.equals("E-N-D")) {
          break outer;
      }
      
      if (word.length() > maxLength) {
          maxLength = word.length();
          maxWord = word;
      }
      • 토큰이 "E-N-D"이면 break outer로 즉시 전체 순회를 종료한다.
      • 그 외 토큰은 현재 최장 길이 maxLength보다 길 때만 maxWord와 maxLength를 갱신한다. 길이가 같으면 갱신하지 않으므로 먼저 등장한 단어가 유지된다.
    • 3) 결과 출력 
      System.out.println(maxWord.toLowerCase());
      • 가장 긴 단어를 toLowerCase()로 소문자로 변환하여 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(L): 입력 전체 길이 L에 비례하여 각 문자를 한 번씩 처리하며, 각 토큰에 대해 O(W)(W는 단어 길이)의 비교만 수행하므로 전체 O(L)이다.
    • 공간 복잡도
      • O(W): 현재 최장 단어 문자열 하나만 저장하면 되므로 단어의 최대 길이 W에 비례하는 공간만 사용한다.
