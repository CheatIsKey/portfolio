    문제 22993 — 서든어택 3

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    서든어택 3 (백준 22993)는
    플레이어 1의 공격력과 나머지 N-1명의 공격력이 주어질 때, 플레이어 1이 나머지를 순서대로 모두 처치할 수 있는지 판별하는 문제다.
    처치한 적의 공격력이 누적되어 플레이어 1의 총 공격력이 증가하므로, 약한 적부터 순서대로 처치하는 것이 최적 전략이다.

    • 플레이어 1은 자신의 총 공격력보다 공격력이 낮은 적만 처치할 수 있다.
    • 적을 처치하면 그 적의 공격력이 플레이어 1의 총 공격력에 합산된다.
    • 가장 약한 적부터 처치해야 연쇄적으로 더 강한 적을 상대할 수 있으므로, 나머지 공격력을 오름차순으로 정렬한 뒤 순서대로 처치 가능 여부를 확인한다.

    전략 개요

    • 핵심 알고리즘: 정렬 + 그리디 (약한 적부터 순차 처치)
    • 선택 이유: 현재 총 공격력으로 처치할 수 있는 적 중 가장 약한 적을 먼저 처치하면 공격력이 최대한 빠르게 누적되어 이후 더 강한 적을 상대할 수 있다. 정렬 후 순서대로 처치 가능 여부를 확인하면 O(N log N)에 해결된다.
    • 세부 전략:
      • 나머지 N-1명의 공격력을 오름차순으로 정렬한다.
      • 순서대로 순회하며 현재 총 공격력이 적의 공격력 이하이면 즉시 "No"를 출력하고 종료한다.
      • 처치 가능하면 해당 공격력을 총 공격력에 누적한다.
      • 모든 적을 처치하면 "Yes"를 출력한다.

    코드가 하는 일

    • 1) 입력 및 정렬 
      long[] attack = new long[N - 1];
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      long playerOne = Long.parseLong(st.nextToken());
      
      for (int i = 0; i < N - 1; i++) {
          attack[i] = Long.parseLong(st.nextToken());
      }
      
      Arrays.sort(attack);
      • 플레이어 1의 초기 공격력 playerOne을 읽고, 나머지 N-1명의 공격력을 배열에 저장한 뒤 오름차순 정렬한다. 공격력 합이 int 범위를 초과할 수 있으므로 long으로 선언한다.
    • 2) 순차 처치 가능 여부 확인 
      long totalAttack = playerOne;
      
      for (int i = 0; i < N - 1; i++) {
          if (totalAttack <= attack[i]) {
              System.out.println("No");
              return;
          }
      
          totalAttack += attack[i];
      }
      • 총 공격력 totalAttack을 playerOne으로 초기화한다.
      • totalAttack <= attack[i]: 현재 총 공격력이 적의 공격력 이하이면 처치 불가능하므로 즉시 "No"를 출력하고 종료한다. 등호 조건에서도 처치할 수 없음에 유의한다.
      • 처치 가능하면 totalAttack += attack[i]로 처치한 적의 공격력을 누적하여 다음 적을 상대할 준비를 한다.
    • 3) 결과 출력 
      System.out.println("Yes");
      • 모든 적을 처치할 수 있으면 "Yes"를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N log N): 배열 정렬에 O(N log N), 이후 순회에 O(N)이 소요되므로 전체 O(N log N)이다.
    • 공간 복잡도
      • O(N): 나머지 N-1명의 공격력을 저장하는 배열이 주요 메모리 사용처이며, 그 외 변수들은 O(1)의 상수 공간을 사용한다.

    문제 23827 — 수열 (Easy)

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    수열 (Easy) (백준 23827)는
    N개의 수열이 주어질 때, i < j인 모든 쌍 (i, j)에 대해 a[i] × a[j]의 합을 MOD(10⁹+7)로 나눈 나머지를 구하는 문제다.
    이중 반복문으로 모든 쌍을 직접 탐색하면 O(N²)으로 시간 초과가 발생하므로, 누적합을 활용해 O(N)으로 줄여야 한다.

    • N이 최대 100,000이므로 이중 반복문을 사용하면 최대 50억 번의 연산이 필요해 시간 초과가 난다.
    • i번째 원소 a[i]에 대해 곱해야 할 대상은 a[i+1]~a[N-1]의 합이다. 전체 합에서 a[0]~a[i]를 순서대로 빼나가면 매 위치에서 이후 원소들의 합을 O(1)에 구할 수 있다.
    • 각 항이 MOD 범위를 초과할 수 있으므로 곱셈과 덧셈마다 MOD 연산을 적용해야 한다.

    전략 개요

    • 핵심 알고리즘: 누적합 활용 그리디 (단일 순회)
    • 선택 이유: a[i] × (a[i+1] + a[i+2] + ... + a[N-1])의 형태로 변환하면, 전체 합에서 왼쪽 원소를 순서대로 제거해나가는 방식으로 O(N)에 답을 구할 수 있다. 귀찮아 (SIB)와 동일한 핵심 아이디어이며, 이 문제에서는 추가로 MOD 연산을 적용해야 한다.
    • 세부 전략:
      • 입력을 받는 동시에 전체 합 totalSum을 계산한다.
      • 배열을 순회하면서 현재 원소 numbers[i]를 totalSum에서 먼저 뺀다.
      • totalSum이 a[i+1]~a[N-1]의 합이 되므로, (numbers[i] × (totalSum % MOD)) % MOD를 answer에 누적한다.
      • 매 단계마다 MOD 연산을 적용하여 오버플로우를 방지한다.

    코드가 하는 일

    • 1) 입력 및 전체 합 계산 
      int[] numbers = new int[N];
      long totalSum = 0;
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      for (int i = 0; i < N; i++) {
          numbers[i] = Integer.parseInt(st.nextToken());
          totalSum += numbers[i];
      }
      • 입력을 받는 동시에 모든 원소의 합 totalSum을 구한다. 합이 int 범위를 초과할 수 있으므로 long으로 선언한다.
    • 2) 단일 순회로 모든 쌍의 곱의 합 계산 
      long answer = 0;
      
      for (int i = 0; i < N - 1; i++) {
          totalSum -= numbers[i];
          long term = (numbers[i] * (totalSum % MOD)) % MOD;
          answer = (answer + term) % MOD;
      }
      • totalSum -= numbers[i]: 현재 원소를 합에서 제거하면 totalSum은 a[i+1]~a[N-1]의 합이 된다.
      • totalSum % MOD: 이후 원소들의 합이 MOD 범위를 초과할 수 있으므로 먼저 MOD 연산을 적용한다.
      • numbers[i] * (totalSum % MOD)): a[i]와 이후 원소 합의 곱을 구한 뒤 다시 MOD로 나눠 term을 계산한다. 이 한 번의 곱셈이 이중 반복문의 안쪽 루프 전체를 대체한다.
      • answer = (answer + term) % MOD: 누적합에 매 단계 MOD 연산을 적용하여 오버플로우를 방지한다.
    • 3) 결과 출력 
      System.out.println(answer);
      • 모든 i < j 쌍에 대한 a[i] × a[j]의 합을 MOD로 나눈 나머지 answer를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(N): 입력 시 전체 합을 계산하는 순회 1회, 답을 누적하는 순회 1회로 총 두 번의 O(N) 순회만 수행한다.
    • 공간 복잡도
      • O(N): 입력 배열 저장에 O(N)이 사용되며, 그 외 totalSum, answer 등은 O(1)의 상수 공간을 사용한다.
