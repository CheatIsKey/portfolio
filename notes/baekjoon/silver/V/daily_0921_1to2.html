    문제 1010 — 다리 놓기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 서쪽 N개, 동쪽 M개의 사이트를 교차 없이 연결하려면 서쪽의 순서가 동쪽에서도 유지되어야 한다. 이는 결국 M개 중 N개를 고르는 문제이며 정답은 C(M, N)이다.

    변수 매핑 — 입력에서 west는 고르는 개수 r, east는 전체 개수 n을 의미한다. 즉 이 코드는 C(n=east, r=west)를 구한다.

    왜 BigInteger인가 — 팩토리얼로 직접 계산하면 중간 값이 매우 커져 long을 넘는다(예: 30! ≈ 2.65×1032). 따라서 임의 정밀 정수형 BigInteger로 계산한다. BigInteger.ONE은 값 1을 의미한다.

    계산 방법(팩토리얼 회피) — 등가식 C(n, r) = Π (n - r + i) / i (i=1..r)을 그대로 구현한다. 먼저 대칭성 C(n, r) = C(n, n-r)을 이용해 r ← min(r, n-r)로 줄인다. 그런 다음 반복문에서 매 회차에 분자 하나와 분모 하나를 처리한다.

    • west = Math.min(west, east - west) → r = min(r, n - r)로 축소한다.
    • for (int i = 1; i <= west; i++) → i = 1..r을 순회한다.
    • east - west + i → 분자 (n - r + i)에 해당한다.
    • result = result.multiply(...).divide(...) → 누적값에 × (분자) ÷ (분모)를 순서대로 반영한다.

    입출력 — 여러 테스트 케이스를 처리하므로 각 결과를 StringBuilder에 append로 모아 두었다가 마지막에 한 번에 출력한다. 이는 println을 반복 호출하는 것보다 I/O 비용이 작다.

    예시

    입력
      3
      2 2
      1 5
      13 29
      
      출력
      1
      5
      67863915

    요약

    • 문제는 조합 C(M, N) 계산이다.
    • 등가식 ∏ (n - r + i) / i로 직접 곱하며 나눠 오버플로우를 피한다.
    • BigInteger로 정확도를 보장하고, StringBuilder로 출력 효율을 높인다.

    문제 1094 — 막대기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 길이 64cm의 막대기에서 시작하여 원하는 길이 want만큼 정확히 맞추려면, 막대를 계속 반으로 잘라 나가면서 필요 없는 조각은 버리고 필요한 조각은 남기는 방식으로 시뮬레이션한다. 최종적으로 남은 막대의 개수가 정답이다.

    자료구조 선택 — 현재 가지고 있는 막대들을 담기 위해 List<Integer> have를 사용한다. 매 단계마다 가장 짧은 막대를 꺼내기 위해 stream().min()을 활용했다.

    절차

    • 초기에 64cm 막대 하나만 리스트에 넣는다.
    • 리스트의 합 sum이 want와 같으면 종료한다.
    • 가장 짧은 막대 min을 꺼내 절반 half로 자른 뒤, 일단 한 조각만 리스트에 넣는다.
    • 이 상태에서 다시 합을 구했을 때 sum < want라면, 남은 절반 조각도 추가해 합을 원래대로 보존한다.
    • 이 과정을 반복하여 원하는 길이를 정확히 맞추면, 리스트 크기(have.size())가 정답이 된다.

    remove에서 (Integer) 캐스팅을 한 이유 — List.remove에는 remove(int index)와 remove(Object o)가 오버로드되어 있다. 최소값 min은 int 타입이므로, 그냥 remove(min)을 쓰면 인덱스로 해석된다. 따라서 (Integer) min으로 캐스팅해 값 기반 제거를 명확히 했다.

    입출력 — 입력값은 하나(want)이고, 반복 시뮬레이션 후 남은 막대의 개수를 출력한다.

    예시

    입력
      23
      
      출력
      4

    추가 학습 — 구현을 마친 뒤, 같은 문제를 더 간단히 해결하는 방법도 배웠다. 원하는 길이 want를 2진수로 바꾸었을 때 1의 개수가 정답이라는 점이다. 예를 들어 23(=10111₂)은 1이 4개이므로 막대기 4개가 필요하다. 이 풀이가 훨씬 간결하지만, 직접 시뮬레이션을 구현하면서 문제의 구조를 눈으로 확인할 수 있었다는 점에서 의미가 있었다.
