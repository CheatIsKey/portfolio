    문제 1181 — 단어 정렬

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 단어들을 정렬할 때 조건은 두 가지다. 1) 길이가 짧은 순서, 2) 길이가 같으면 사전순이다. 또한 중복된 단어는 제거해야 한다.

    중복 제거 — 입력받은 단어들을 HashSet에 담아 자동으로 중복을 제거한다. 그 후 ArrayList로 변환하여 정렬을 진행한다.

    정렬 방식 — 두 가지 방법이 있다. 첫째, Comparator.comparingInt(String::length).thenComparing(Comparator.naturalOrder())를 사용하여 길이 → 사전순 정렬 기준을 체인으로 지정한다. 둘째, 람다를 활용해 (a, b) -> { if (a.length() != b.length()) return a.length() - b.length(); else return a.compareTo(b); }와 같이 직접 비교 로직을 작성할 수도 있다.

    입출력 — 중복 제거 후 정렬된 리스트를 순서대로 출력하면 된다.

    정리

    • 중복 제거 → Set, 정렬 → List 변환 후 Comparator 적용
    • 정렬 기준은 "길이 우선, 길이가 같으면 사전순"
    • 람다식과 체인 메서드 두 가지 방식 모두 동일한 결과를 낸다

    문제 1193 — 분수찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 분수들은 대각선 단위로 지그재그하게 나열된다. n번째 항을 찾으려면 먼저 해당 분수가 속한 대각선(line)을 찾고, 그 안에서 몇 번째 위치(pos)인지 구해야 한다. 홀수 번째 대각선은 위에서 아래로, 짝수 번째 대각선은 아래에서 위로 진행하는 규칙이 있다.

    라인 찾기 — 각 라인의 길이는 1, 2, 3 … 순으로 증가한다. 따라서 누적합 count를 갱신하면서 count < n을 만족하는 동안 line을 늘리면, 마지막에 n이 속한 line을 찾을 수 있다.

    위치 계산 — 그 라인의 시작 인덱스는 count - line이다. 따라서 현재 라인에서의 위치는 pos = n - (count - line)으로 계산한다.

    홀짝 판별 — line 번호가 짝수면 아래→위 방향, 홀수면 위→아래 방향이다. 짝수일 때는 분자 = pos, 분모 = line - pos + 1이고, 홀수일 때는 분자 = line - pos + 1, 분모 = pos가 된다.

    출력 — 분자와 분모를 문자열 형태로 numerator + "/" + denominator로 합쳐 출력한다.

    정리

    • 라인 수(line)는 누적합으로 찾는다.
    • 라인 내부 위치(pos)는 n - (count - line)으로 계산한다.
    • 홀짝(line % 2)에 따라 분자/분모를 다르게 계산한다.
