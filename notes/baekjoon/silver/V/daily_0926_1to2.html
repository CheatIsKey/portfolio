    문제 1343 — 폴리오미노

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 문자열에서 연속된 X 구간을 모두 타일로 덮어야 한다. 사용할 수 있는 타일은 AAAA(길이 4)와 BB(길이 2)뿐이므로, 구간의 길이가 홀수라면 덮을 수 없어 -1을 출력한다. 짝수라면 먼저 AAAA로 가능한 만큼 채운 뒤, 남은 2칸을 BB로 채우는 것이 사전순 조건(AAAA < BB)에도 맞는다.

    문자열 분리 — 입력 문자열은 .를 경계로 쪼개 각 X 덩어리를 따로 처리한다. split("\\.", -1)을 사용해 연속된 점이나 마지막에 오는 점 뒤의 빈 조각도 보존한다.

    알고리즘

    • split으로 얻은 배열을 순서대로 순회한다.
    • 각 조각 seg에 대해
      • 길이가 홀수라면 덮을 수 없으므로 즉시 -1 출력 후 종료.
      • 길이가 4 이상이면 AAAA를 반복해서 붙인다.
      • 마지막에 2가 남으면 BB를 붙인다.
    • 조각을 변환한 뒤, 배열의 마지막 조각이 아니라면 원래 있던 .를 다시 붙인다.
    • 모든 조각을 변환해 StringBuilder에 합친 결과를 출력한다.

    시간·공간 복잡도 — 문자열 길이를 N이라 하면, 각 조각을 한 번씩만 순회하므로 O(N). 결과를 담는 StringBuilder를 제외하면 추가 공간은 O(N).

    정리

    • 연속된 X 구간 길이가 홀수 → 불가능(-1).
    • 짝수 구간은 AAAA로 우선 채우고, 남으면 BB.
    • .은 입력과 동일하게 보존.

    추가 학습

    이 문제는 문자열을 쪼개고 복원하는 방식으로 풀 수 있지만, 더 간단하게 정규식 치환으로도 해결할 수 있다. XXXX를 AAAA로, XX를 BB로 순서대로 치환한 뒤, 남은 X가 있으면 -1을 출력하는 방식이다.

    
s = s.replaceAll("XXXX", "AAAA");
s = s.replaceAll("XX", "BB");
if (s.contains("X")) System.out.println(-1);
else System.out.println(s);

    이 방법은 .을 전혀 건드리지 않으므로 점 복원을 신경 쓸 필요가 없다는 장점이 있다.

    문제 1380 — 귀걸이

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 각 시나리오에서 여학생이 n명일 때, 귀걸이 기록은 총 2n−1줄이 주어진다. 정상이라면 한 학생 번호는 압수와 반납으로 두 번 등장해야 하지만, 한 학생만 귀걸이를 못 돌려받아 번호가 한 번만 등장한다. 따라서 등장 횟수가 1인 번호를 찾아 해당 학생의 이름을 출력하면 된다.

    입력 파싱 — 시나리오는 여러 번 주어지며, n = 0이면 종료한다. 먼저 학생 이름을 1 ~ n 인덱스에 그대로 저장한다(공백 포함). 이어지는 2n−1줄은 "번호 A/B" 형식이므로 StringTokenizer로 번호와 라벨을 분리해 읽되, A/B 라벨은 로직에 쓰지 않으므로 무시한다.

    알고리즘

    • name[1..n]에 학생 이름을 저장한다.
    • cnt[1..n] 배열을 0으로 초기화한다.
    • 2n−1줄을 읽으면서, 각 줄의 번호 idx에 대해 cnt[idx]++ 한다. (라벨 A/B는 사용하지 않음)
    • cnt[i] == 1인 i가 귀걸이를 돌려받지 못한 학생 번호다.
    • 출력은 시나리오번호 + 공백 + name[i] 형식으로 한다. 시나리오 번호는 1부터 시작해 시나리오마다 1씩 증가한다.

    시간·공간 복잡도 — 각 시나리오에서 이름 입력 O(n), 기록 처리 O(n), 단 한 번의 선형 탐색 O(n)으로 전체 O(n). 추가 공간은 이름 배열과 카운트 배열로 O(n)이다.

    정리

    • 번호는 이름 리스트의 순서(1~n)와 1:1 대응한다.
    • 각 기록 줄은 번호와 A/B 두 토큰이며, 로직에는 번호만 필요하다.
    • 2n−1줄을 모두 카운팅한 뒤, cnt == 1인 번호의 이름을 출력한다.
    • 여러 시나리오를 처리하며, n = 0을 만나면 종료한다.
