    문제 1384 — 메시지

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 각 아이는 자신의 이름이 맨 위에 적힌 종이를 들고 있으며, 그 밑에는 다른 아이들이 P(positive) 또는 N(nasty) 메시지를 차례대로 쓴다. i번째 줄의 j번째 메시지가 N이라면, 그 메시지를 쓴 사람의 인덱스는 (i - j + n) % n으로 계산할 수 있다. 즉, "누가 누구에 대해 나쁘게 썼는지"를 역추적하여 출력한다.

    입력 파싱 — 첫 줄은 그룹 인원수 n이며, 이후 n줄에는 "이름 + (n−1)개의 메시지"가 주어진다. 이름만 따로 names 배열에 저장하고, 전체 줄은 row 배열에 보관하여 메시지 판별에 활용한다. 여러 그룹이 들어올 수 있으며, 마지막에 0이 입력되면 종료한다.

    알고리즘

    • row[i][0]는 i번째 아이의 이름, row[i][1..n−1]은 메시지이다.
    • 모든 i, j를 순회하며 row[i][j] == "N"일 때
      • 메시지 작성자 = (i - j + n) % n
      • "작성자 was nasty about 대상"을 출력 버퍼에 추가한다.
    • 그룹 내에서 N이 한 번도 없으면 "Nobody was nasty"를 출력한다.

    시간·공간 복잡도 — 각 그룹에서 최대 n × (n−1)번 메시지를 확인하므로 O(n²). n은 최대 20이므로 연산량은 매우 작다. 이름 배열과 메시지 배열을 저장하는 데 O(n²) 공간이 필요하다.

    정리

    • 인덱스 계산은 (i - j + n) % n 공식으로 처리한다.
    • N 메시지를 발견하면 "작성자 → 대상" 관계를 출력한다.
    • 없으면 "Nobody was nasty"를 출력한다.
    • 여러 그룹을 처리하며, n = 0에서 종료한다.

    문제 1402 — 아무래도이문제는A번난이도인것같다

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 문제는 “정수 A를 곱셈 형태로 분해한 뒤, 그 인수들의 합으로 바꿀 수 있다”라는 규칙을 정의하고, 이 과정을 반복해서 B로 바꿀 수 있는지를 묻는다. 처음 보면 실제 변환 과정을 탐색해야 할 것 같지만, 사실 수학적으로 어떤 수든 임의의 다른 수로 변환 가능하다는 것이 성립한다.

    이유 — 임의의 정수 A에 대해 항상 A = 1 * A라는 분해가 가능하다. 이 경우 합은 1 + A = A + 1이므로, 언제든 현재 수에서 +1로 이동할 수 있다. 따라서 A에서 시작해 원하는 만큼 수를 증가시켜 결국 어떤 B에도 도달할 수 있다. (또한 다른 분해를 사용하면 줄어드는 경우도 존재한다.) 즉, 정답은 항상 “yes”.

    입력 파싱 — 첫 줄에 테스트 케이스 개수 t가 주어지고, 이후 t개의 줄마다 A B가 주어진다. 하지만 실제 연산은 필요 없고, 각 줄마다 단순히 yes를 출력하면 된다.

    알고리즘

    • 첫 줄에서 t를 읽는다.
    • 반복문으로 t번 입력(A B)을 받되, 실제로는 사용하지 않는다.
    • 각 케이스마다 yes를 출력한다.

    시간·공간 복잡도 — 입력 줄 수가 t일 때, 단순 반복문으로 O(t)만에 처리 가능하다. 별도의 연산이 없으므로 시간·공간 복잡도는 사실상 O(1)에 가깝다.

    정리

    • 규칙상 어떤 수든 결국 다른 어떤 수로 변환 가능하다.
    • 따라서 출력은 항상 yes.
    • 구현은 입력만 읽고, 각 줄마다 yes를 출력하면 끝이다.
