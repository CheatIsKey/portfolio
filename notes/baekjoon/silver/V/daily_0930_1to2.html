    문제 1439 — 뒤집기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 문자열을 이루는 0과 1을 연속된 묶음 단위로 나누어, 0 묶음의 개수와 1 묶음의 개수를 각각 센다. 모두 0으로 통일하려면 1 묶음을 뒤집으면 되고, 모두 1로 통일하려면 0 묶음을 뒤집으면 된다. 따라서 두 묶음 개수 중 더 작은 값을 답으로 출력한다.

    입력 파싱 — 입력은 문자열 그대로 처리해야 한다. 정수로 변환할 경우 앞자리 0이 사라질 수 있으므로 문자열을 유지한 채 인덱스로 접근한다.

    알고리즘

    • 첫 문자가 0인지 1인지에 따라 초기 묶음 개수를 1 증가시킨다.
    • 두 번째 문자부터 순회하며, 직전 문자와 다르면 새로운 묶음이 시작된 것으로 판단한다.
    • 해당 문자가 0이면 count0++, 1이면 count1++ 한다.
    • 마지막에 min(count0, count1)을 출력한다.

    시간·공간 복잡도 — 문자열 전체를 한 번 순회하므로 O(N), 추가 메모리는 묶음 개수 카운트를 위한 정수 변수 2개로 O(1).

    정리

    • 문자열을 처음부터 끝까지 순회하면서 연속 구간을 카운트한다.
    • 0 묶음 수와 1 묶음 수 중 작은 값을 답으로 사용한다.
    • 간단한 구현으로도 교차 패턴(예: 0101, 1010)까지 정확히 처리할 수 있다.

    문제 1475 — 방 번호

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 각 숫자(0~9)의 등장 횟수를 세고, 6과 9는 서로 뒤집어 쓸 수 있으므로 count[6] + count[9]를 합친 뒤 정수 올림으로 반 나누어 필요한 세트 수를 계산한다. 마지막으로 모든 숫자(0~9) 중 가장 많이 필요한 개수가 정답이 된다.

    입력 파싱 — 입력은 문자열로 받아 각 문자를 순회하면서 count[c - '0']++로 개수를 누적한다. 정수 변환은 불필요하다.

    알고리즘

    • 배열 count[10]에 0~9 등장 횟수를 센다.
    • 6과 9는 공유하므로, shared = (count[6] + count[9] + 1) / 2로 올림 나눗셈 처리한다.
    • count[6] = shared, count[9] = shared로 치환해 동일 기준으로 맞춘다.
    • 0~9 전체에서 max(count[i])를 구해 출력한다.

    예시

    • 9999 → count[9]=4, shared=(0+4+1)/2=2 → 정답 2
    • 669 → count[6]=2, count[9]=1, shared=(2+1+1)/2=2 → 정답 2
    • 122 → 각 숫자 최댓값 2 → 정답 2

    시간·공간 복잡도 — 문자열을 한 번 순회하므로 O(N), 추가 메모리는 길이 10의 카운트 배열과 상수 변수로 O(1).

    정리

    • 카운트 배열로 각 숫자 빈도를 집계한다.
    • 6과 9는 합쳐서 올림 반 나누기((a+b+1)/2)로 필요한 세트 수를 구한다.
    • 최종 답은 모든 숫자 중 최댓값.
