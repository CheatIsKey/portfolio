    백준 알고리즘 Silver V - Daily 문제 묶음

    작성일 2025-10-02 · 분류: 코딩 테스트(백준 알고리즘), Silver V

    문제 1476 — 날짜 계산

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 주어진 세 값 E, S, M은 각각 주기 15, 28, 19를 갖는 달력의 연도 표시다. 우리는 가장 작은 양의 정수 year에 대해 year ≡ E (mod 15), year ≡ S (mod 28), year ≡ M (mod 19)을 동시에 만족하는 값을 찾으면 된다. 세 주기는 서로 서로소이므로 해는 15×28×19=7,980을 주기로 유일하게 결정되며, 1부터 증가시키며 조건을 만족할 때 멈추면 정답을 얻는다.

    입력 파싱 — 공백으로 구분된 세 정수 E, S, M을 순서대로 읽는다(범위: 1≤E≤15, 1≤S≤28, 1≤M≤19). 정수 그대로 보관해 비교 연산에 사용한다.

    알고리즘

    • year=1로 시작한다.
    • 무한 루프에서 다음을 검사한다:
      • (year - E) % 15 == 0
      • (year - S) % 28 == 0
      • (year - M) % 19 == 0
      세 조건이 모두 참이면 year가 정답이므로 출력하고 종료한다. 그렇지 않으면 year++ 후 반복한다.
    • 참고: Java에서 음수 나머지(a % m에서 a<0)가 나올 수 있으나, 나머지가 0인 경우는 정확히 배수일 때뿐이므로 == 0 판정은 안전하다.

    시간·공간 복잡도 — 최악의 경우 1부터 7,980까지 확인하므로 반복 횟수는 상수(≤7,980)이며 O(1)로 볼 수 있다. 추가 메모리는 상수 개의 정수 변수만 사용하므로 O(1).

    정리

    • 문제는 세 모듈로 동시 합동을 찾는 형태이며, 단순 선형 탐색으로도 충분하다.
    • 조건식은 (year - x) % 주기 == 0 형태로 직접 표현해 명확히 판정한다.
    • 답은 반드시 1 이상 7,980 이하에서 등장하므로 시간 제한에 넉넉하다.

    문제 1531 — 투명

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 100×100 평면(문제 조건상 좌표는 1~100, 구간은 양끝 포함)에서 각 종이(직사각형)로 덮인 칸의 겹침 횟수를 누적 카운트한다. 모든 칸에 대해 누적값이 기준 M을 초과(> M)하는 칸의 개수를 세면 정답이다.

    입력 파싱 — 첫 줄에서 종이 수 N과 기준 값 M을 읽는다. 이후 N줄에 걸쳐 각 직사각형의 좌하/좌상(혹은 좌측 하단)과 우상/우하(혹은 우측 상단) 좌표 (x1, y1), (x2, y2)를 읽는다. 좌표와 구간은 모두 포함 범위이므로 내부 루프에서 <=로 누적해야 한다.

    알고리즘

    • 크기 101×101의 배열 board를 0으로 초기화한다(인덱스 0을 여유로 둬도 무방).
    • 각 직사각형에 대해
      • x = x1..x2, y = y1..y2 (모두 포함) 범위를 이중 루프로 돌며 board[x][y]++ 한다.
    • 모든 직사각형 반영 후, 전체 좌표를 순회하며 board[x][y] > M 인 칸을 카운트한다.
    • 카운트 값이 정답.

    시간·공간 복잡도 — 최악의 경우 각 직사각형이 100×100 전역을 덮을 수 있으므로 누적 단계는 O(N·100·100), 마지막 전체 스캔은 O(100·100)로 상수(10,000) 배수이며, 전체적으로 O(N + 1) 수준의 상수계수 연산이다. 공간은 101×101 정수 배열로 O(1).

    정리

    • 좌표/구간은 양끝 포함이 핵심 — 내부 루프는 <=로 처리한다.
    • 정답 판정은 반드시 > M(초과)이어야 하며, >= M로 착각하지 않는다.
    • 배열 크기를 101로 두어 1~100을 안전하게 커버(0행/열은 사용 안 해도 무방).
    • 대안으로 2D 누적합(차분 배열)로도 해결 가능하지만, 입력 크기에서는 단순 누적이 가장 직관적이고 충분히 빠르다.
