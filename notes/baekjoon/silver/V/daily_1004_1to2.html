    문제 1676 — 팩토리얼 0의 개수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 팩토리얼 값의 끝자리 0의 개수는 10이 몇 번 곱해졌는가와 같다. 즉, 인수 분해 시 2와 5가 짝을 이룰 때마다 10이 하나씩 만들어진다. 팩토리얼은 2의 배수가 훨씬 많으므로, 5의 배수 개수만 세면 된다. 실제 팩토리얼 계산(BigInteger 등)을 할 필요 없이 5의 거듭제곱 단위로 나누어 합산하는 것이 정석적이고 효율적인 방법이다.

    입력 파싱 — 한 줄에 정수 N이 주어진다 (0 ≤ N ≤ 500). 문자열을 정수로 변환해 그대로 사용한다.

    잘못된 접근 — 위 코드처럼 BigInteger로 N! 전체를 구한 뒤 10으로 나누어 떨어지는 횟수를 세는 방식은 문제의 의도와 다르며, 큰 수일수록 연산 비용이 급격히 증가한다. 이 문제는 단순한 수학적 규칙을 이용해야 효율적으로 해결된다.

    정석 알고리즘

    • 1부터 N까지의 수 중에서 5의 배수가 몇 개 있는지 세면 된다.
      • N / 5 → 5의 배수 개수 (예: 5, 10, 15, ...)
      • N / 25 → 25, 50, 75, ...처럼 5가 두 번 포함된 경우 추가 카운트
      • N / 125 → 125, 250, ...처럼 5가 세 번 포함된 경우 추가
    • 따라서 정답은 N/5 + N/25 + N/125 + ... (몫이 0이 될 때까지 반복)이다.

    예시

    • N = 10 → 10/5 = 2 → 0의 개수 2 (10!, 실제로 3628800)
    • N = 25 → 25/5 + 25/25 = 5 + 1 = 6 → 0의 개수 6
    • N = 100 → 100/5 + 100/25 = 20 + 4 = 24 → 0의 개수 24

    시간·공간 복잡도 — 반복은 log₅(N)회만 수행되므로 O(log N), 추가 메모리는 상수 크기로 O(1)이다.

    정리

    • 팩토리얼의 끝 0 개수는 5의 배수 개수를 누적합으로 구한다.
    • BigInteger 연산은 불필요하며, 수학적 규칙으로 해결 가능하다.
    • 공식: count = N/5 + N/25 + N/125 + ... (정수 나눗셈, 몫이 0일 때 종료)

    문제 1769 — 3의 배수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 임의의 수가 3의 배수인지 판별하는 방법은 각 자리 숫자의 합이 3의 배수인지 확인하는 것이다. 이 문제는 그 원리를 반복 적용하며, 주어진 수가 한 자리 수가 될 때까지 자릿수 합을 계속 구한 뒤, 변환 횟수와 최종 결과가 3의 배수인지 여부를 출력하는 문제다.

    입력 파싱 — 매우 큰 수(최대 1,000,000자리)가 입력될 수 있으므로, 정수형으로 변환하지 않고 String으로 읽는다. 각 자리 숫자 접근 시 charAt(i) - '0'을 이용해 정수값으로 변환한다.

    알고리즘

    • count = 0으로 초기화한다.
    • X의 길이가 1보다 큰 동안(즉, 아직 한 자리 수가 아닐 때):
      • 문자열의 각 자리 숫자를 더해 sum을 계산한다.
      • X = Integer.toString(sum)으로 자릿수 합을 새로운 문자열로 변환한다.
      • count++ (변환 횟수 증가).
    • 루프 종료 시 X는 한 자리 수 문자열이 된다.
    • count를 출력하고, 마지막 한 자리(int last = X.charAt(0) - '0')가 3의 배수인지 판별:
      • 3의 배수면 YES
      • 아니면 NO

    예시

    • X = "1234567"
      → 1+2+3+4+5+6+7 = 28 → (1회)
      → 2+8 = 10 → (2회)
      → 1+0 = 1 → (3회)
      최종값 1은 3의 배수가 아니므로
      출력: 3 / NO
    • X = "9" → 한 자리이므로 변환 횟수 0, 9는 3의 배수 → 0 / YES

    시간·공간 복잡도 — 각 단계에서 문자열 길이만큼 덧셈을 수행하므로 O(L) (L은 자릿수). 변환 시 길이가 급격히 줄어들어 전체는 O(L)에 수렴한다. 추가 메모리는 상수 크기의 변수 몇 개뿐이므로 O(1).

    정리

    • 매 단계에서 자릿수 합을 반복하여 1자리 수가 될 때까지 계산한다.
    • 반복 횟수(count)와 최종 숫자의 3의 배수 여부를 출력한다.
    • 큰 수이므로 문자열 기반 연산을 해야 하며, 정수형 변환은 금지한다.
    • 3의 배수 판정 규칙을 정확히 적용하면 깔끔하게 해결된다.
