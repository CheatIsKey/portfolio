    문제 1789 — 수들의 합

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 1부터 시작하여 서로 다른 자연수를 더해 나갈 때, 그 합이 주어진 S를 넘지 않도록 만드는 가장 큰 n을 찾는 문제다. 즉, 1 + 2 + 3 + ... + n ≤ S를 만족하는 최대 n을 구하면 된다. 이때 등차수열 합 공식 n(n+1)/2 ≤ S를 만족하는 최대 정수를 구하는 것과 같다.

    입력 파싱 — 단 하나의 정수 S(1 ≤ S ≤ 4,294,967,295)가 주어진다. int 범위를 초과할 수 있으므로 반드시 long형으로 받아야 한다.

    알고리즘

    • sum = 0, n = 0으로 초기화한다.
    • 다음 수 n + 1을 더했을 때 sum + (n + 1) ≤ S인 동안 반복한다.
      • 조건이 참이면 n++하고 sum += n을 수행한다.
      • 조건이 거짓이면 합이 S를 초과하므로 루프를 종료한다.
    • 마지막으로 n을 출력한다.

    예시

    • S = 200 → 1+2+...+19=190, 다음 수 20을 더하면 210>200 → n=19
    • S = 55 → 1+2+...+10=55 → n=10
    • S = 1 → 1+2=3>1 → n=1

    시간·공간 복잡도 — 1 + 2 + ... + n ≈ n²/2이므로 반복 횟수는 O(√S) 수준으로 매우 작다. 공간 복잡도는 상수 크기 변수만 사용하므로 O(1).

    정리

    • 등차수열 합 공식 n(n+1)/2 ≤ S를 만족하는 최대 n을 구하는 문제.
    • 단순 반복문으로도 충분히 빠르게 계산 가능 (S ≤ 4×10⁹).
    • 변수는 반드시 long형으로 선언해야 오버플로를 방지할 수 있다.
    • 수학적으로 풀면 n = ⌊(-1 + √(1 + 8S)) / 2⌋이지만, 반복문 방식이 구현상 더 안전하다.

    문제 1813 — 논리학 교수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 배열 안에 여러 수가 주어졌을 때, x라는 정수가 “자신과 같은 값을 가진 원소가 x개 있다”를 만족하면 그 수 x를 논리적으로 일관된 값이라 한다. 즉, count[x] == x를 만족하는 모든 x 중 가장 큰 값을 찾으면 된다. 만약 그런 수가 존재하지 않으면 -1을 출력한다.

    입력 파싱 — 첫 줄에 정수 N이 주어지고, 이어서 N개의 정수가 공백 또는 개행 단위로 주어진다. 입력 형식이 여러 줄로 끊길 수 있으므로, BufferedReader와 StringTokenizer를 함께 사용해 모든 정수를 정확히 읽는다.

    알고리즘

    • 입력받은 모든 정수를 배열 arr에 저장한다.
    • 해시맵 count를 사용해 각 값의 등장 횟수를 센다.
    • 문제 조건상 값은 0 이상 50 이하이므로, i=0..50 범위를 순회하며 다음을 검사한다:
      • count.getOrDefault(i, 0) == i인 경우, 조건을 만족하므로 answer = i로 갱신한다.
    • 모든 확인 후 answer를 출력한다. 조건을 만족한 값이 없다면 초기값 -1이 그대로 출력된다.

    예시

    • 입력: 5
      0 0 1 2 2
      → count[0]=2, count[1]=1, count[2]=2
      → 조건 만족: i=1 (1개 존재), i=2 (2개 존재)
      → 가장 큰 i=2 출력
    • 입력: 4
      3 3 3 3
      → count[3]=4, 조건 4==3 불일치 → 정답 -1

    시간·공간 복잡도 — 해시맵 카운팅은 O(N), 이후 0~50 범위 탐색은 O(1) 수준이다. 공간도 해시맵 크기(최대 51)로 O(1).

    정리

    • count[x] == x를 만족하는 수를 찾는 단순 빈도 비교 문제.
    • 조건을 만족하는 여러 값이 있을 경우, 가장 큰 값을 출력한다.
    • 없으면 -1.
    • 문제의 핵심은 “값과 등장 횟수의 일치”를 확인하는 것이다.
