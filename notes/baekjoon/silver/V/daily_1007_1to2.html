    문제 1817 — 짐 챙기는 숌

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 여러 개의 책(무게 주어짐)을 일정한 순서대로 상자에 넣을 때, 한 상자의 최대 허용 무게 M을 초과하지 않도록 최소 상자 개수를 구하는 문제다. 즉, 앞에서부터 차례로 책을 담되, 현재 상자에 넣을 수 없을 때 새로운 상자를 추가한다.

    입력 파싱 — 첫 줄에 책의 개수 N과 상자의 최대 허용 무게 M이 주어진다. N이 0이면 책이 없으므로 상자도 필요 없으며, 0을 바로 출력한다. 그 외에는 두 번째 줄에서 책의 무게 N개를 순서대로 읽는다.

    알고리즘

    • box = 1 (최소 1개 상자는 필요)로 시작하고, sum = 0으로 초기화한다.
    • 책 무게 w를 순서대로 읽으며 다음을 수행한다:
      • 현재 상자 무게 합 sum + w가 M을 초과하면 → 새 상자를 추가: box++, sum = w.
      • 초과하지 않으면 → 현재 상자에 그대로 추가: sum += w.
    • 모든 책을 처리한 뒤, 누적된 box 개수를 출력한다.

    예시

    • N = 5, M = 10
      책 무게: 6 3 4 1 5
      [6 3](합9) → 다음 4 추가 시 13>10 → 새 상자
      [4 1 5](합10)
      결과: 2
    • N = 0 → 책이 없으므로 상자 0

    시간·공간 복잡도 — 모든 책을 한 번씩만 처리하므로 O(N), 필요한 추가 메모리는 상수 크기의 변수 몇 개뿐이므로 O(1).

    정리

    • 현재 상자의 누적 무게가 최대 허용 무게 M을 넘으면 새 상자를 시작한다.
    • 초과하지 않는 한 계속 같은 상자에 담는다.
    • N == 0이면 예외 처리로 0을 바로 출력한다.
    • 단순한 누적합 기반의 그리디 알고리즘으로 해결 가능하다.

    문제 1996 — 지뢰 찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 보드에는 빈칸('.')과 숫자('1'..'9')가 주어진다. 숫자는 그 칸이 ‘지뢰’이면서 해당 값(무게/세기)을 의미한다. 출력 규칙은 다음과 같다: 지뢰가 있는 칸은 '*'로 표기하고, 빈칸은 8방향 이웃에 있는 지뢰 숫자의 합을 쓴다. 이 합이 10 이상이면 'M'로 표기한다.

    입력 파싱 — 한 줄에 보드 크기 N, 이어서 N줄의 보드를 읽는다. 각 줄은 길이 N의 문자열로, toCharArray()로 바로 2차원 문자 배열에 담아 문자 비교/연산을 수행한다.

    알고리즘

    • 8방향 탐색을 위해 방향 배열을 준비한다: dx = {-1,-1,-1,0,0,1,1,1}, dy = {-1,0,1,-1,1,-1,0,1}.
    • 모든 칸 (i,j)을 순회한다.
      • 지뢰 칸(map[i][j] != '.')이면 결과에 '*'를 추가하고 다음 칸으로.
      • 빈칸이면 8방향 이웃 (nx, ny)를 확인한다:
        • 보드 밖은 건너뛴다.
        • 이웃이 지뢰면(map[nx][ny] != '.') 그 문자를 정수로 변환해(map[nx][ny] - '0') sum에 누적한다.
      • 누적합 sum이 ≥ 10이면 'M', 아니면 sum의 한 자리 숫자를 그대로 추가한다.
    • 행마다 개행을 붙여 완성 문자열을 출력한다(StringBuilder 사용).

    주의/엣지 케이스

    • 지뢰 칸은 값이 얼마든지 '*'로만 출력한다.
    • 빈칸 주변 합이 0일 수 있으며, 그 경우 문자 '0'이 출력된다.
    • 경계 체크(nx, ny 범위) 누락 시 예외/오동작 발생 가능 — 반드시 보드 범위 내에서만 합산.

    시간·공간 복잡도 — 각 칸에서 최대 8칸을 확인하므로 전체 O(N²). 입력 보드와 출력 버퍼 외 추가 메모리는 상수여서 O(1).

    정리

    • 지뢰 칸: '*', 빈칸: 8방향 지뢰 숫자 합(합 ≥10이면 'M').
    • 8방향 탐색 + 경계 체크 + 문자→정수 변환(c - '0')이 핵심.
    • 간단한 구현으로 문제 요구 형식을 정확히 만족한다.
