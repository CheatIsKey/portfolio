    작성일 2025-10-08 · 분류: 코딩 테스트(백준 알고리즘), Silver V

    문제 2018 — 수들의 합 5

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 1부터 시작하는 연속된 자연수의 합으로 N을 표현하는 경우의 수를 구하는 문제다. 예를 들어 N = 15일 때 1+2+3+4+5, 4+5+6, 7+8, 15의 네 가지가 존재한다. 이는 투 포인터(two-pointer) 혹은 슬라이딩 윈도우로 해결할 수 있다.

    입력 파싱 — 한 줄에 정수 N(1 ≤ N ≤ 10,000)이 주어진다. 연속된 수를 더해야 하므로 입력값은 정수 하나면 충분하다.

    알고리즘

    • 포인터 정의 — start, end를 1로 시작하고, 현재 구간의 합을 sum으로 관리한다.
    • 초기 상태 — sum = 1, count = 1 (N 자체가 하나의 구간으로 성립하므로 초기값 1).
    • 루프 조건 — end != N일 동안 다음을 반복한다:
      • sum == N → 정답 하나 발견, count++; 다음 수(end+1)를 포함해 확장 (end++, sum += end).
      • sum < N → 아직 부족하므로 오른쪽 확장 (end++, sum += end).
      • sum > N → 초과하므로 왼쪽을 줄임 (sum -= start, start++).
    • 루프 종료 후 count 출력.

    예시

    • N = 15
      1+2+3+4+5 = 15 → count=2
      4+5+6 = 15 → count=3
      7+8 = 15 → count=4
      15 → count=5 (초기값 포함)
      정답 4
    • N = 5 → (5), (2+3), (1+2+3−초과X) → 총 2

    시간·공간 복잡도 — 각 포인터가 1~N을 한 번씩만 이동하므로 O(N), 추가 공간은 상수 개의 변수만 사용해 O(1).

    정리

    • 연속된 자연수의 합으로 N을 만드는 모든 경우의 수를 구한다.
    • 슬라이딩 윈도우(투 포인터)로 효율적으로 처리 가능하다.
    • 포인터는 오름차순으로 이동하며, sum == N일 때마다 count를 증가시킨다.
    • 시간 복잡도 O(N)으로 충분히 빠르다.

    문제 2057 — 팩토리얼 분해

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 어떤 수 N이 여러 개의 서로 다른 팩토리얼(k!)의 합으로 표현 가능한지를 판별하는 문제다. 즉, N = a₁! + a₂! + ... + aₖ! (단, a₁ < a₂ < ... < aₖ) 형태로 나타낼 수 있으면 YES, 그렇지 않으면 NO를 출력한다.

    입력 파싱 — 입력은 단 하나의 정수 N이며, 범위는 64비트(long) 범위 내의 수다. 0은 예외적으로 어떤 팩토리얼의 합으로도 표현할 수 없으므로 곧바로 NO를 출력한다.

    알고리즘

    • 1️⃣ 팩토리얼 미리 계산 — 0!부터 20!까지를 배열에 저장한다. (20! ≈ 2.43×10¹⁸로 long 범위 내에서 표현 가능한 최대 팩토리얼이다.)
    • 2️⃣ 큰 팩토리얼부터 차감 —
      • i = 20부터 0까지 역순으로 순회하며,
      • fact[i] ≤ N이면 N -= fact[i]로 차감한다.
      • 이 과정을 반복해 N이 0이 되면 정확히 팩토리얼들의 합으로 표현 가능.
    • 3️⃣ 종료 조건 — 루프 종료 후 N == 0이면 YES, 아니라면 NO.

    예시

    • N = 9
      3! = 6 → 남은 3
      2! = 2 → 남은 1
      1! = 1 → 남은 0 → YES
    • N = 10
      3! = 6 → 남은 4
      2! = 2 → 남은 2
      1! = 1 → 남은 1
      0! = 1 → 남은 0 → YES
    • N = 7
      3! = 6 → 남은 1
      1! = 1 → 남은 0 → YES
    • N = 8
      3! = 6 → 남은 2
      2! = 2 → 남은 0 → YES
    • N = 11
      3! = 6 → 남은 5
      2! = 2 → 남은 3
      1! = 1 → 남은 2 → NO

    시간·공간 복잡도 — 팩토리얼을 최대 21개만 미리 계산 후 한 번씩 확인하므로 시간 복잡도는 O(21) = O(1), 공간 복잡도도 O(1)이다.

    정리

    • 팩토리얼 합 표현 여부를 큰 팩토리얼부터 탐욕적으로 차감해 판정한다.
    • 20!까지만 고려하면 long 범위 내에서 충분하다.
    • N == 0은 예외 처리로 NO를 출력한다.
    • 팩토리얼 합 분해 문제의 기본 패턴: “큰 항부터 차감하며 0 도달 여부 확인”.
