    문제 2740 — 행렬 곱셈

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    두 행렬 A(N1×M1)와 B(N2×M2)의 곱 C는 M1 == N2일 때에만 정의된다. 결과 행렬 C의 크기는 N1×M2이며, 각 원소는 “A의 i번째 행”과 “B의 j번째 열”의 내적(곱의 합)으로 계산한다: C[i][j] = Σk=0..M1-1 A[i][k] × B[k][j].

    제약 정리 —
    입력으로 먼저 N1 M1과 A의 원소들이 주어지고, 다음 줄에 N2 M2와 B의 원소들이 주어진다. 유효한 행렬 곱을 위해서는 M1 == N2가 성립해야 하며, 출력은 크기 N1×M2의 정수 행렬이다.

    코드 핵심 (변수 매핑) —
    N1, M1: A의 행/열 크기  |  N2, M2: B의 행/열 크기  |  i: 결과 행 인덱스(=A의 행)  |  j: 결과 열 인덱스(=B의 열)  |  k: 곱하고 더하는 중간 인덱스(=A의 열, B의 행를 연결)

    • int[][] A = new int[N1][M1];, int[][] B = new int[N2][M2]; → 입력 저장
    • int[][] result = new int[N1][M2]; → 결과 행렬 메모리 할당
    • result[i][j] += A[i][k] * B[k][j]; → 내적 누적합(핵심 연산)

    동작 과정 —
    세 겹의 반복문으로 결과의 각 칸을 채운다. 바깥쪽 두 반복 (i, j)가 결과의 위치를 정하고, 안쪽 반복 k가 해당 위치의 값을 A의 i행과 B의 j열 원소들을 곱해 모두 더해 완성한다.

    // i: A의 행(=결과 행), j: B의 열(=결과 열), k: 중간 인덱스
      for (int i = 0; i < N1; i++) {
        for (int j = 0; j < M2; j++) {
          for (int k = 0; k < M1; k++) { // (M1 == N2)
            result[i][j] += A[i][k] * B[k][j];
          }
        }
      }

    엣지 케이스

    • M1 != N2라면 행렬 곱이 정의되지 않음(문제 조건상 입력은 유효하게 주어짐).
    • 입력 중 0이 포함되어도 결과 원소는 다른 항들의 합으로 0이 아닐 수 있음.
    • 출력 형식: 각 행을 공백으로 구분해 출력하고 행마다 개행(맨 끝 공백은 허용됨).

    시간·공간 복잡도 —
    중첩 루프 3개로 계산하므로 시간복잡도는 O(N1 × M2 × M1).
    추가 공간은 결과 행렬 O(N1 × M2)와 입력 저장 공간.

    정리

    • 결과 크기: (N1×M1) × (N2×M2) → N1×M2 (단, M1 == N2)
    • 핵심 연산: C[i][j] = Σ A[i][k] × B[k][j] (행×열의 내적)
    • 빠른 입출력: BufferedReader, StringTokenizer 사용
    • 정확한 포맷으로 결과 행렬을 행 단위로 출력

    문제 2751 — 수 정렬하기 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    입력된 N개의 정수를 오름차순으로 정렬하는 문제다. 기본형 배열(int[])에 저장한 뒤 Arrays.sort()를 사용하면 자바 내장 퀵소트(dual-pivot quicksort)가 실행되어 매우 빠르게 정렬된다. 정렬 후에는 StringBuilder로 결과를 한 번에 모아 출력하여 입출력 속도를 최적화한다.

    제약 정리 —
    1 ≤ N ≤ 1,000,000이며, 각 수의 범위는 -1,000,000 ≤ x ≤ 1,000,000이다. 입력이 많기 때문에 단순한 System.out.println() 반복 출력은 시간 초과를 유발한다. 따라서 입력은 BufferedReader, 출력은 StringBuilder를 통해 묶어서 처리한다.

    코드 핵심 (변수 매핑) —
    arr: 입력된 정수들을 저장하는 배열
    Arrays.sort(arr): 정렬 수행 (O(N log N))
    sb: 정렬된 결과를 한 문자열로 모으는 버퍼
    System.out.println(sb): 최종 출력 (1회 I/O)

    • int[] arr = new int[N]; → 박싱 없는 기본형 배열로 빠른 메모리 접근
    • Arrays.sort(arr); → 효율적인 기본형 정렬
    • sb.append(num).append('\n'); → 반복 출력 대신 문자열 누적
    • System.out.println(sb.toString()); → 결과를 한 번에 출력

    동작 과정 —
    1. BufferedReader로 N과 N개의 정수를 입력받는다.
    2. 모든 정수를 int[] arr에 저장한다.
    3. Arrays.sort()로 배열을 정렬한다.
    4. 정렬된 값을 StringBuilder에 차례로 추가하고 개행한다.
    5. 마지막에 한 번만 출력하여 입출력 부담을 줄인다.

    부가 설명 (List 방식) —
    List<Integer>를 이용해 list.sort(Integer::compareTo)로도 정렬 가능하지만, Integer는 객체형(박싱 타입)이라 내부에서 언박싱 연산이 반복되어 속도가 느리다. 또한 System.out.println()을 반복 호출하면 I/O 비용이 커져 시간 초과가 발생할 수 있다. 따라서 대용량 입력에서는 int[] + Arrays.sort() + StringBuilder 조합이 필수적이다.

    엣지 케이스

    • N = 1일 때 단일 수만 출력 (정렬 불필요)
    • 모든 값이 같아도 정상 정렬됨
    • 음수, 0, 양수가 섞여 있어도 Arrays.sort()가 정수 기준으로 자동 처리

    시간·공간 복잡도 —
    정렬: O(N log N), 출력: O(N)
    추가 공간: O(N) (배열 + 출력 버퍼)

    정리

    • 입력은 BufferedReader, 출력은 StringBuilder로 처리
    • 기본형 배열 + Arrays.sort()로 빠른 정렬
    • 모든 결과를 한 번에 출력하여 시간 초과 방지
    • 시간 복잡도 O(N log N), 공간 복잡도 O(N)
