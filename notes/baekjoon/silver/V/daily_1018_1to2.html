    문제 2993 — 세 부분

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    주어진 문자열 s를 두 개의 컷 포인트 i, j(1 ≤ i < j ≤ n-1)로 세 부분으로 자른 뒤, 각 부분을 개별적으로 뒤집어 이어 붙인 모든 후보 중 사전순으로 가장 앞선 문자열을 고른다. 즉, 모든 (i, j) 쌍을 완전탐색 → 각 조각을 뒤집어 연결 → 후보 목록을 정렬해 최솟값을 출력한다.

    제약 정리 —
    입력은 공백 없는 단일 문자열 s(알파벳 소문자).
    출력은 위 규칙으로 얻은 사전순 최소 문자열 한 줄.

    코드 핵심 (변수 매핑) —
    s: 원본 문자열  |  i, j: 자를 위치(왼쪽 경계는 포함, 오른쪽 경계는 제외)
    slice1, slice2, slice3: 세 구간 [0, i), [i, j), [j, n)의 역순 문자열
    list: 모든 후보 문자열을 담는 리스트(정렬하여 최솟값 선택)

    • i는 1부터 n-2, j는 i+1부터 n-1까지 이중 루프
    • 각 구간을 substring으로 잘라 StringBuilder.reverse()로 뒤집기
    • 뒤집은 세 조각을 이어 붙여 후보를 list에 추가
    • list를 사전순 정렬 → 첫 요소가 정답

    동작 과정 —
    세 부분으로 나눌 수 있는 모든 경우의 수는 O(n²)개다. 각 경우에 대해 세 조각을 뒤집고 연결하면 하나의 후보가 되고, 모든 후보를 정렬하면 사전순 최소값을 간단히 얻을 수 있다.

    // 1) 입력
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      String s = br.readLine();
      
      // 2) 모든 (i, j) 쌍을 완전탐색하여 후보 생성
      List<String> list = new ArrayList<>();
      for (int i = 1; i < s.length() - 1; i++) {
          for (int j = i + 1; j < s.length(); j++) {
              String slice1 = new StringBuilder(s.substring(0, i)).reverse().toString();
              String slice2 = new StringBuilder(s.substring(i, j)).reverse().toString();
              String slice3 = new StringBuilder(s.substring(j)).reverse().toString();
              list.add(slice1 + slice2 + slice3);
          }
      }
      
      // 3) 사전순 정렬 후, 첫 번째(최소) 출력
      list.sort(String::compareTo);
      System.out.println(list.get(0));

    엣지 케이스

    • n = 3인 최소 길이: 자르는 위치는 (i, j) = (1, 2) 한 가지뿐이며 그대로 처리된다.
    • 동일 후보 다수: 서로 다른 (i, j)로 같은 결과가 나와도 정렬 후 최솟값 선택에 영향 없음.

    시간·공간 복잡도 —
    후보 개수 M = O(n²). 각 후보 생성(세 구간 뒤집기/연결)은 O(n) → 생성 총합 O(n³).
    후보 정렬은 O(M log M)회 비교 × 비교 비용 O(n) → O(n³ log n).
    공간은 후보 M개 × 길이 n 문자열 저장으로 O(M·n) = O(n³) (구현 방식에 따라 상수는 달라질 수 있음).

    정리

    • 두 컷 포인트 (i, j)를 완전탐색해 세 조각을 역순으로 만든다.
    • 모든 후보를 모아 사전순 정렬, 첫 번째를 출력한다.
    • 구현은 간단하지만, 시간/공간은 O(n³) 수준(정렬 포함 시)임에 유의.

    문제 3049 — 다각형의 대각선

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    N개의 점으로 이루어진 다각형(볼록 n각형)에서, 서로 다른 네 꼭짓점을 연결해 만들 수 있는 모든 대각선 교차점의 개수를 구하는 문제다. 즉, N개의 점 중 4개를 선택해 만들 수 있는 조합의 수 C(N, 4)가 곧 정답이다.

    제약 정리 —
    입력은 정수 N (3 ≤ N ≤ 1000).
    출력은 C(N, 4)의 결과값이며, 이는 다각형 내부의 대각선 교차점 개수와 같다.
    단, N < 4인 경우 교차점이 존재하지 않으므로 0을 출력한다.

    코드 핵심 (변수 매핑) —
    N: 꼭짓점의 개수  |  n: BigInteger로 변환된 N (오버플로우 방지)
    result: C(N, 4) = N × (N-1) × (N-2) × (N-3) / 24 계산 결과

    • N < 4일 때는 0 출력 (교차점 불가)
    • 그 외에는 조합 공식 C(N, 4)로 계산
    • long 범위를 초과할 수 있으므로 BigInteger를 사용해 오버플로우 방지

    동작 과정 —
    1. 입력으로 받은 N을 확인하고, N이 4 미만이면 곧바로 0을 출력한다.
    2. 그렇지 않다면, 조합 수식 C(N, 4)를 BigInteger 연산으로 계산한다.
    3. n × (n-1) × (n-2) × (n-3) / 24를 계산한 결과를 그대로 출력한다.

    // 1) 입력
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      long N = Long.parseLong(br.readLine());
      if (N < 4) {
          System.out.println(0);
          return;
      }
      
      // 2) BigInteger로 변환 후 조합 계산
      BigInteger n = BigInteger.valueOf(N);
      BigInteger result = n.multiply(BigInteger.valueOf(N - 1))
              .multiply(BigInteger.valueOf(N - 2))
              .multiply(BigInteger.valueOf(N - 3))
              .divide(BigInteger.valueOf(24));
      
      // 3) 결과 출력
      System.out.println(result);

    엣지 케이스

    • N < 4: 교차점이 존재하지 않으므로 결과는 0.
    • N = 4: 4개의 점으로 만들 수 있는 교차점은 정확히 1개 → C(4,4) = 1.
    • 큰 N (예: 1000): C(1000, 4)는 약 41,417,124,750 → long 오버플로우 발생 → BigInteger 필수.

    시간·공간 복잡도 —
    계산은 상수 시간(O(1)), BigInteger 연산도 입력 크기와 무관하게 빠름.
    공간 복잡도도 O(1) (고정된 변수 몇 개만 사용).

    정리

    • 볼록 N각형에서 대각선 교차점 수 = C(N, 4)
    • 단순히 팩토리얼로 계산하면 long 범위를 초과하므로 BigInteger 사용
    • N < 4이면 0, 그 이상은 조합 공식으로 직접 계산

    구현 시 실수 포인트 —
    처음에는 long을 사용해 (N * (N-1) * (N-2) * (N-3)) / 24로 구현했으나, N = 1000일 때 long 범위를 초과해 오버플로우가 발생했다. 이를 방지하기 위해 BigInteger를 사용하도록 수정하였다.
