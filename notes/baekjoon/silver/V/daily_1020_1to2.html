    문제 3135 — 라디오

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    현재 채널 A에서 목표 채널 B로 가는 최소 버튼 수는 두 선택지의 최솟값이다.
    ① +/-만 눌러 직접 이동: 비용 |A - B|
    ② 즐겨찾기 채널 하나를 한 번 누른 뒤(+1) 거기서 +/-로 보정: 각 즐겨찾기 p에 대해 비용 |p - B| + 1
    따라서 정답은 min( |A-B|, min_p (|p-B|+1) )로 바로 계산된다.

    제약 정리 —
    입력: 첫 줄 A B, 둘째 줄 즐겨찾기 개수 N, 이어서 N줄에 즐겨찾기 채널.
    출력: 목표 채널 B로 이동하는 데 필요한 최소 버튼 수(정수).

    코드 핵심 (변수 매핑) —
    A, B: 시작/목표 채널  |  N: 즐겨찾기 개수
    opt: 현재까지의 최소 비용(초기값 |A-B|)
    reserved: 각 즐겨찾기 채널 값
    갱신 규칙: opt = min(opt, |reserved - B| + 1)

    • 직접 이동 비용: |A - B|
    • 즐겨찾기 경유 비용: |p - B| + 1 (즐겨찾기 버튼 1회 + 보정)
    • 최종 정답: 모든 후보 중 최소값을 출력

    동작 과정 —
    1) opt = |A-B|로 초기화 → 2) N개의 즐겨찾기를 순회하며 각 p에 대해 |p-B|+1 계산 →
    3) opt와 비교해 더 작으면 갱신 → 4) 순회 종료 후 opt 출력.

    엣지 케이스

    • A == B: 초기값 0이므로 바로 정답.
    • 즐겨찾기에 B가 있음: 비용 1이 후보에 포함되어 최소 가능.
    • N = 0: 즐겨찾기 경유가 없으므로 정답은 |A-B|.
    • 중복 즐겨찾기: 최소값 계산에 영향 없음(동일 후보 반복).

    시간·공간 복잡도 —
    즐겨찾기를 한 번씩만 확인하므로 시간 O(N).
    추가 배열 없이 상수 변수만 사용하므로 공간 O(1).

    정리

    • 직접 이동(|A-B|) vs. 즐겨찾기 경유(|p-B|+1) 중 작은 값을 선택.
    • 선형 스캔으로 최소값만 유지하면 되어 구현이 간단하고 효율적.
    • 정답 공식: min( |A-B|, min_p (|p-B|+1) ).

    문제 3211 — kino

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    각 친구가 “자신 포함 몇 명까지 모일 수 있는지”를 나타내는 값을 가진다. 정렬 후, 가능한 최대 모임 인원을 cnt로 유지하면서, 현재 cnt명일 때 조건 friends[cnt-1] + 1 > cnt를 만족하면 더 큰 모임이 가능하다고 본다.
    즉, “현재까지의 모임 크기를 수용할 수 있는 친구 수가 충분한가?”를 반복 점검하며, 가능한 최대 인원 cnt를 갱신한다.

    제약 정리 —
    입력: 첫 줄 N(친구 수), 이후 N개의 줄에 각 친구의 허용 인원 수(자기 포함).
    출력: 조건을 만족하는 최대 모임 인원(정수).

    코드 핵심 (변수 매핑) —
    N: 친구 수
    friends[]: 각 친구의 허용 인원 수 배열
    cnt: 현재 가능한 모임 인원 후보 (초기값 1)
    갱신 규칙: friends[cnt-1] + 1 > cnt이면 cnt = friends[cnt-1] + 1로 확장

    • Arrays.sort()로 오름차순 정렬 — 작은 허용 인원부터 점검
    • cnt는 현재 모임 규모를 의미하며, 인덱스는 cnt-1
    • 조건 불만족 시 더 이상 확장 불가 → 반복 종료

    동작 과정 —
    1) 입력받은 허용 인원 배열을 정렬 →
    2) cnt=1에서 시작 →
    3) 현재 friends[cnt-1]+1 > cnt이면 더 큰 모임 가능 → cnt를 갱신 →
    4) 불가능해지는 시점에서 반복 종료 →
    5) 최종 cnt를 출력.

    엣지 케이스

    • 모든 친구의 허용 인원이 작을 경우: 초기에 조건 불만족 → cnt=1 출력.
    • 허용 인원이 큰 친구가 많을 경우: cnt가 반복적으로 확장되어 큰 결과 가능.
    • N=1: 단일 친구의 friends[0]+1이 1보다 크면 cnt=friends[0]+1.
    • 정렬 덕분에 항상 가장 작은 조건부터 점검되므로 안정적.

    시간·공간 복잡도 —
    정렬이 O(N log N), 이후 선형 반복 O(N) → 전체 O(N log N).
    배열 외 추가 메모리 없음 → 공간 O(N).

    정리

    • 허용 인원 배열을 정렬 후, 가능한 모임 크기를 점진적으로 확장.
    • 조건 friends[cnt-1] + 1 > cnt을 만족할 때마다 cnt를 갱신.
    • 결과적으로 모임의 최대 가능 인원 cnt를 출력.
