    백준 알고리즘 Silver V - Daily 문제 묶음

    작성일 2025-10-21 · 분류: 코딩 테스트(백준 알고리즘), Silver V

    문제 3227 — MO

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    길이 P의 1차원 보드(1-indexed)에서 흰색과 검은색이 번갈아 말을 둔다.
    매 턴 현재 색 c가 위치 pos에 말을 놓은 뒤, 왼쪽과 오른쪽 각각에 대해 연속된 상대색 구간이 자신의 말로 양끝이 감싸져 있으면 그 구간의 상대 말을 모두 제거한다(뒤집지 않음).
    즉, 한쪽 방향에서 ... c (상대)* c ... 형태가 되면 가운데 상대 말들이 사라진다. 두 방향은 서로 독립적으로 처리된다.

    제약 정리 —
    입력: 첫 줄 P N, 이어서 N개의 수(여러 줄에 걸쳐 있을 수 있음)로 각 턴의 착수 위치 pos.
    턴 순서: 0번째(첫 턴)는 흰색, 1번째는 검은색이며 짝수 턴은 흰색, 홀수 턴은 검은색.
    출력: 모든 턴을 처리한 뒤 보드 위에 남아 있는 흰/검 말의 개수 "white black".

    코드 핵심 (변수 매핑) —
    board[1..P]: 각 칸의 상태 (0=EMPTY, 1=WHITE, 2=BLACK)
    whiteCnt, blackCnt: 현재 보드에 남은 말의 개수 집계
    color/opp: 이번 턴의 내 색 / 상대 색 (턴 번호로 결정)
    착수 후 양방향 처리 규칙은 다음과 같다.

    • 왼쪽: i = pos-1에서 시작해 연속된 상대 말을 건너뛴 뒤, 첫 비상대 칸 i가 내 색이면 (i+1 .. pos-1) 구간의 상대 말을 제거한다.
    • 오른쪽: j = pos+1에서 동일하게 진행하며, 내 말로 닫히면 (pos+1 .. j-1) 구간의 상대 말을 제거한다.
    • 착수: board[pos] = color로 말을 놓고 해당 색 개수를 +1 한다.
    • 제거: 상대 말을 EMPTY로 바꾸고 상대 색 개수를 감소시킨다.
    • 왼쪽과 오른쪽은 독립적으로 모두 처리할 수 있다.

    동작 과정 —
    1) 각 턴에서 착수 위치 pos를 읽고 현재 색을 결정한다(짝수=흰, 홀수=검).
    2) 말을 놓고(board[pos]=color) 개수를 갱신한다.
    3) 왼쪽 방향을 스캔하여 상대 연속 구간이 내 말로 닫혀 있으면 그 사이 말을 제거한다.
    4) 오른쪽 방향에서도 동일한 방식으로 처리한다.
    5) 모든 턴이 끝나면 최종적으로 whiteCnt blackCnt를 출력한다.

    엣지 케이스

    • 보드 끝까지 가도 내 말로 닫히지 않으면 제거되지 않는다(중간에 빈칸이 있거나 경계에 닿으면 실패).
    • 양쪽 방향이 모두 조건을 만족하면 두 구간에서 각각 제거가 일어난다.
    • 착수 위치 주변이 즉시 내 말로 둘러싸인 경우(상대가 0개)에는 제거가 발생하지 않는다.
    • 입력의 착수 위치는 항상 유효하고 비어 있다고 가정한다.

    시간·공간 복잡도 —
    공간 복잡도는 보드 배열 O(P)와 상수 변수만 사용하므로 O(P).
    시간 복잡도는 각 턴마다 좌우로 선형 스캔을 수행하므로 최악의 경우 O(P), 전체는 O(N·P).
    단, 실제 입력 분포에서는 각 말이 최대 한 번만 제거되므로 평균적으로는 더 작을 수 있다.

    정리

    • 착수 후 왼쪽·오른쪽 각각에서 “상대 연속 구간이 내 말로 닫히면 해당 구간 제거”.
    • 제거는 뒤집기가 아니라 보드에서 비우기이므로 개수도 즉시 차감된다.
    • 최종 출력은 남은 흰/검 개수 whiteCnt blackCnt이다.

    문제 3230 — 금메달, 은메달, 동메달은 누가?

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    1라운드에서는 각 선수의 최종 순위 위치가 주어진다. 이 정보를 이용해 “순위 → 선수 번호” 형태의 순위를 복원한다.
    그다음 상위 M명을 결선 진출자로 뽑고, 결선 입력은 각 결선 선수가 최종 순위표에서 설 위치를 의미한다.
    이때 결선 진출자를 뒤에서부터 하나씩 꺼내 주어진 위치에 삽입하면, 인덱스 밀림 없이 정확한 최종 순위를 재구성할 수 있다.
    최종 순위표의 1, 2, 3번째 선수가 각각 금·은·동메달을 차지한다.

    제약 정리 —
    입력: 첫 줄에 N M (전체 선수 수, 결선 진출자 수)이 주어진다.
    이후 길이 N의 수열이 주어지며, 각 값은 선수 i가 1라운드 순위표에서 설 위치 x(1-based)를 의미한다.
    다음으로 길이 M의 수열이 주어지며, 이는 결선 진출자가 최종 순위표에서 설 위치를 나타낸다.
    출력: 최종 순위 1, 2, 3위 선수 번호를 각각 한 줄에 출력한다.

    코드 핵심 (변수 매핑) —
    round1: 1라운드 최종 순위표(1등부터 순서대로 선수 번호를 저장한 리스트)
    top: 결선 진출자 배열(1라운드 상위 M명)
    finalOrder: 최종 순위표를 복원하기 위한 리스트
    주요 처리 단계는 다음과 같다.

    • 1라운드 입력 시: 선수 i를 순위표의 x-1 위치에 add하여 “순위 → 선수” 구조를 만든다.
    • 결선 대상 추출: round1의 앞쪽 M명을 top 배열에 저장한다.
    • 결선 복원: 결선 입력을 순회하며, top의 뒤쪽부터 결선 선수를 하나씩 꺼내 주어진 위치 x-1에 삽입한다.
    • 메달 결정: 최종 순위 리스트의 0, 1, 2번째 선수를 각각 금, 은, 동메달로 출력한다.

    동작 과정 —
    1) N, M을 입력받는다.
    2) N개의 1라운드 순위를 읽어 각 위치에 선수 번호를 삽입한다.
    3) round1의 상위 M명을 top 배열에 복사한다.
    4) M개의 결선 입력을 순회하며, top의 뒤쪽 선수부터 주어진 위치에 삽입한다.
    5) 완성된 finalOrder 리스트의 앞 세 명을 출력한다.

    엣지 케이스

    • M = 3인 최소 결선 규모에서도 동일한 로직으로 처리된다.
    • 결선 입력의 x가 1이거나 현재 길이+1인 경우에도 정상적으로 앞이나 뒤에 삽입된다.
    • 문제에서 모든 입력이 순열 형태로 주어진다고 보장하므로 인덱스 오류는 발생하지 않는다.

    시간·공간 복잡도 —
    리스트의 add(index, value) 연산은 내부적으로 요소를 한 칸씩 밀어내므로 평균적으로 선형 시간이 걸린다.
    1라운드 복원은 O(N²), 결선 복원은 O(M²)이 소요된다.
    따라서 전체 시간 복잡도는 O(N² + M²), 공간 복잡도는 O(N)이다.

    정리

    • 1라운드 입력으로 “순위 → 선수” 구조를 복원한다.
    • 결선 입력은 뒤에서부터 순회하며 주어진 위치에 결선 선수를 삽입한다.
    • 최종 순위의 상위 세 선수가 금·은·동메달 수상자가 된다.
