    문제 3787 — Count on Canton

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    양의 유리수들을 대각선(1, 2, 3, …번째 대각선)에 따라 지그재그로 나열했을 때, 주어진 순번 N이 어느 대각선 k에 속하는지를 먼저 찾는다. 대각선 k의 마지막 인덱스는 삼각수 T_k = k(k+1)/2이고, T_{k-1} <= N <= T_k를 만족하는 k를 구하면 해당 대각선 내에서의 상대적 위치 offset = N - T_{k-1}가 정해진다. 대각선의 방향은 k의 짝·홀에 따라 바뀌며,

    • k가 짝수면 위에서 아래로: (num, den) = (offset, k - offset + 1)
    • k가 홀수면 아래서 위로: (num, den) = (k - offset + 1, offset)

    이를 그대로 구현하여 TERM N IS num/den 형태로 출력한다.

    제약 정리 —
    입력: 하나 이상의 정수 N (줄 구분 없이 공백으로 여러 개 주어질 수 있음, EOF까지 처리).
    출력: 각 N에 대해 대응하는 기약 분수 num/den를 TERM N IS num/den 형식으로 출력.
    수의 범위는 문제 원문을 따르며, 구현은 long으로 안전하게 누적한다.

    코드 핵심 (변수/구조 매핑) —

    • long sum: 현재 대각선 k까지의 누적 길이, 즉 T_k
    • long k: N이 위치한 대각선 번호
    • long prevSum = sum - k: 이전 대각선까지의 누적, 즉 T_{k-1}
    • long offset = N - prevSum: 대각선 k 내에서의 1-based 위치
    • 방향 판정: (k & 1) == 0이면 짝수 대각선, 아니면 홀수 대각선
    • 분수 결정:
      • 짝수 대각선: num = offset, den = k - offset + 1
      • 홀수 대각선: num = k - offset + 1, den = offset

    동작 과정 —
    1) 입력 스트림을 EOF까지 읽으며 공백 기준으로 N을 파싱한다.
    2) sum과 k를 0에서 시작해 sum < N인 동안 k를 1씩 늘리고 sum += k로 갱신하여, T_k >= N이 되는 최소 k를 찾는다.
    3) prevSum = sum - k, offset = N - prevSum를 계산한다.
    4) k의 짝·홀에 따라 (num, den)를 위 규칙대로 결정한다.
    5) 포맷에 맞춰 TERM N IS num/den을 출력한다.

    정확성 근거 —
    대각선별 원소 수는 1, 2, 3, …으로 증가하므로 누적 길이는 삼각수 T_k이며, 임의의 N은 유일하게 T_{k-1} < N <= T_k를 만족하는 하나의 k에 속한다. 대각선 내 상대 위치 offset 역시 유일하며, 열거 방향이 대각선마다 교차하므로 짝·홀에 따른 분수 규칙으로 정확히 매핑된다.

    엣지 케이스

    • N = 1: k = 1, offset = 1로 1/1.
    • 대량 입력: 라인 단위가 아닌 토큰 단위 파싱 및 EOF 처리로 연속 입력 지원.
    • 큰 N: 누적 합과 대각선 번호를 long으로 관리해 오버플로를 회피.

    시간·공간 복잡도 —
    k(k+1)/2 ≈ N이므로 k ≈ Θ(√N). 단순 누적으로 k를 찾으므로 각 쿼리당 시간 복잡도는 O(√N), 공간 복잡도는 O(1). (필요시 이분 탐색이나 근사식을 이용해 k 도출을 O(1)/O(log N)로 개선 가능하지만, 현 구현으로도 충분하다.)

    정리

    • N이 속한 대각선 k를 삼각수 누적으로 결정.
    • offset = N - T_{k-1}를 구해 대각선 내부 위치를 확정.
    • k의 짝·홀에 따라 분수 (num, den)를 지그재그 규칙으로 계산.
    • 형식: TERM N IS num/den.

    문제 3845 — 잔디깎기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    각 학생의 학번 배열이 주어졌을 때, 어떤 양의 정수 M에 대해 학번 % M 값들이 모두 서로 달라지도록 만드는 가장 작은 M을 찾는 문제다.
    M을 1부터 증가시키며, 모든 학번의 나머지를 구해 정렬한 뒤 인접 중복이 있는지 검사한다. 중복이 없을 때의 M이 정답이다.

    제약 정리 —
    입력: 테스트케이스 수 N. 각 테스트케이스마다 학생 수 G, 이어서 G개의 학번.
    출력: 각 테스트케이스마다 조건을 만족하는 가장 작은 M.
    정확한 수치 범위(N, G, 학번의 최댓값)는 원문을 따름.

    코드 핵심 (변수/구조 매핑) —

    • int N: 테스트케이스 수.
    • int G: 학생 수.
    • int[] students: 학번 목록.
    • M: 1부터 증가시키며 검사하는 모듈러 값.
    • int[] r: students[j] % M 나머지들을 담는 배열.
    • 중복 판정: Arrays.sort(r) 후 인접한 두 값이 같은지 확인.

    동작 과정 —
    1) M = 1로 시작한다.
    2) 모든 학번에 대해 r[j] = students[j] % M 계산.
    3) r을 정렬하고 인접 중복이 있으면 실패(충돌) → M 증가 후 반복.
    4) 중복이 없으면 현재 M을 출력하고 다음 테스트케이스로 진행.

    정확성 근거 —
    주어진 M에 대해 모든 나머지가 서로 다르면, 서로 다른 두 학번 a ≠ b에 대해 a % M ≠ b % M이므로 충돌 없이 G개의 상이한 클래스로 분류된다.
    가장 작은 M을 1부터 선형 탐색하므로, 처음으로 중복이 사라지는 순간의 M이 문제에서 요구하는 최소값과 일치한다.

    엣지 케이스

    • G = 1: 어떤 M ≥ 1에서도 항상 유일하므로 정답은 1.
    • 아주 큰 학번: 나머지 연산은 int 범위 내에서 안전(학번 입력 범위는 원문 참고).

    시간·공간 복잡도 —
    한 M에 대한 검사는 나머지 계산 O(G) + 정렬 O(G log G) + 스캔 O(G)로 O(G log G).
    정답 M*까지 반복하므로 전체는 대략 O(M* · G log G). (최악의 M*은 입력에 의존)
    공간은 나머지 배열 O(G).

    정리

    • M을 1부터 증가시키며 학번 % M의 충돌 여부를 검사.
    • 나머지 배열을 정렬해 인접 중복이 없으면 그 M이 최소 정답.
    • 간단하고 구현이 직관적이며, 공간은 O(G)로 작다.
