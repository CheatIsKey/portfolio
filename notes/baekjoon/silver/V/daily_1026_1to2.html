    문제 3787 — Count on Canton

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    양의 유리수들을 대각선(1, 2, 3, …번째 대각선)에 따라 지그재그로 나열했을 때, 주어진 순번 N이 어느 대각선 k에 속하는지를 먼저 찾는다. 대각선 k의 마지막 인덱스는 삼각수 T_k = k(k+1)/2이고, T_{k-1} <= N <= T_k를 만족하는 k를 구하면 해당 대각선 내에서의 상대적 위치 offset = N - T_{k-1}가 정해진다. 대각선의 방향은 k의 짝·홀에 따라 바뀌며,

    • k가 짝수면 위에서 아래로: (num, den) = (offset, k - offset + 1)
    • k가 홀수면 아래서 위로: (num, den) = (k - offset + 1, offset)

    이를 그대로 구현하여 TERM N IS num/den 형태로 출력한다.

    제약 정리 —
    입력: 하나 이상의 정수 N (줄 구분 없이 공백으로 여러 개 주어질 수 있음, EOF까지 처리).
    출력: 각 N에 대해 대응하는 기약 분수 num/den를 TERM N IS num/den 형식으로 출력.
    수의 범위는 문제 원문을 따르며, 구현은 long으로 안전하게 누적한다.

    코드 핵심 (변수/구조 매핑) —

    • long sum: 현재 대각선 k까지의 누적 길이, 즉 T_k
    • long k: N이 위치한 대각선 번호
    • long prevSum = sum - k: 이전 대각선까지의 누적, 즉 T_{k-1}
    • long offset = N - prevSum: 대각선 k 내에서의 1-based 위치
    • 방향 판정: (k & 1) == 0이면 짝수 대각선, 아니면 홀수 대각선
    • 분수 결정:
      • 짝수 대각선: num = offset, den = k - offset + 1
      • 홀수 대각선: num = k - offset + 1, den = offset

    동작 과정 —
    1) 입력 스트림을 EOF까지 읽으며 공백 기준으로 N을 파싱한다.
    2) sum과 k를 0에서 시작해 sum < N인 동안 k를 1씩 늘리고 sum += k로 갱신하여, T_k >= N이 되는 최소 k를 찾는다.
    3) prevSum = sum - k, offset = N - prevSum를 계산한다.
    4) k의 짝·홀에 따라 (num, den)를 위 규칙대로 결정한다.
    5) 포맷에 맞춰 TERM N IS num/den을 출력한다.

    정확성 근거 —
    대각선별 원소 수는 1, 2, 3, …으로 증가하므로 누적 길이는 삼각수 T_k이며, 임의의 N은 유일하게 T_{k-1} < N <= T_k를 만족하는 하나의 k에 속한다. 대각선 내 상대 위치 offset 역시 유일하며, 열거 방향이 대각선마다 교차하므로 짝·홀에 따른 분수 규칙으로 정확히 매핑된다.

    엣지 케이스

    • N = 1: k = 1, offset = 1로 1/1.
    • 대량 입력: 라인 단위가 아닌 토큰 단위 파싱 및 EOF 처리로 연속 입력 지원.
    • 큰 N: 누적 합과 대각선 번호를 long으로 관리해 오버플로를 회피.

    시간·공간 복잡도 —
    k(k+1)/2 ≈ N이므로 k ≈ Θ(√N). 단순 누적으로 k를 찾으므로 각 쿼리당 시간 복잡도는 O(√N), 공간 복잡도는 O(1). (필요시 이분 탐색이나 근사식을 이용해 k 도출을 O(1)/O(log N)로 개선 가능하지만, 현 구현으로도 충분하다.)

    정리

    • N이 속한 대각선 k를 삼각수 누적으로 결정.
    • offset = N - T_{k-1}를 구해 대각선 내부 위치를 확정.
    • k의 짝·홀에 따라 분수 (num, den)를 지그재그 규칙으로 계산.
    • 형식: TERM N IS num/den.

    문제 3845 — 잔디깎기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    잔디밭을 가로(X)·세로(Y) 방향으로 각각 완전히 덮을 수 있는지 확인하는 문제다. 각 잔디깎기 경로는 중심 좌표를 기준으로 폭 w의 띠를 만들며, 이는 구간 [pos - w/2, pos + w/2]로 표현된다.
    가로 방향은 [0, 75], 세로 방향은 [0, 100] 범위를 각각 덮어야 한다. 좌표들을 정렬한 뒤, 왼쪽부터 겹치거나 맞닿도록 이어지며 덮이는지를 그리디로 확인한다. 모든 구간을 순회한 뒤 최종 덮임 끝점이 경계에 도달하면 해당 축은 “덮기 성공”이다. 두 축 모두 성공하면 YES, 아니면 NO.

    제약 정리 —
    입력: 정수 nx, ny, 실수 w; 이어서 길이 nx의 X좌표들, 길이 ny의 Y좌표들.
    종료 조건: nx = 0, ny = 0, w = 0일 때 입력 종료.
    출력: 각 케이스마다 YES 또는 NO.

    코드 핵심 (변수/구조 매핑) —

    • double EPS = 1e-7: 부동소수 오차 허용치
    • double half = w / 2.0: 각 띠의 반폭
    • double[] x, y: 가로/세로 방향 중심 좌표
    • Arrays.sort(x/y): 좌표 정렬
    • double covered: 현재까지 연속적으로 덮인 오른쪽 끝점
    • 가로 판정 범위: [0.0, 75.0] / 세로 판정 범위: [0.0, 100.0]
    • 각 좌표 pos에 대해 실제 기여 구간
      • left = max(0.0, pos - half)
      • right = min(BOUND, pos + half) (BOUND는 75 또는 100)

    동작 과정 —
    1) 한 줄에 들어온 nx ny w를 읽고 종료 조건이면 종료한다.
    2) X좌표 nx개, Y좌표 ny개를 각각 읽어 정렬한다.
    3) 가로 검증: covered = 0에서 시작하여, 각 구간의 left가 covered + EPS보다 크면 틈이 생긴 것이므로 실패. 아니면 covered = max(covered, right)로 확장. 최종 covered >= 75 - EPS면 성공.
    4) 세로 검증: 같은 로직으로 BOUND = 100을 사용해 판정.
    5) 두 검증이 모두 성공이면 YES, 하나라도 실패면 NO를 출력 버퍼에 추가한다.

    정확성 근거 —
    정렬 후 왼쪽부터 겹치도록 이어 붙이는 그리디는 구간 덮개(covering)에서 표준적이고 최적이다. 각 구간이 만들 수 있는 최좌측 덮임을 항상 선택해 covered를 확장하므로, 임의의 해가 존재한다면 이 절차로도 경계까지 도달한다. 경계 클리핑([0, BOUND]로 제한)과 EPS 허용은 실수 오차로 인한 위·아래 편차를 안정적으로 처리한다.

    엣지 케이스

    • w가 매우 작아 틈이 생기는 경우: left > covered + EPS에서 즉시 실패.
    • 양 끝 처리: 최초 구간의 left가 0을 덮지 못하거나, 최종 covered가 경계(75, 100)에 도달하지 못하면 실패.
    • 좌표가 없는 축(nx=0 또는 ny=0): 어떤 구간도 없으므로 덮기 실패.
    • 겹치는 구간 다수: covered를 넓히는 데 불리하지 않으며, 그리디가 자동으로 최댓값을 취함.

    시간·공간 복잡도 —
    정렬이 지배적이므로 각 케이스에 대해 O(nx log nx + ny log ny), 선형 스캔은 O(nx + ny). 추가 공간은 정렬 및 입력 저장에 O(nx + ny).

    정리

    • 각 좌표는 폭 w의 덮임 구간을 만든다([pos - w/2, pos + w/2]).
    • 가로는 [0, 75], 세로는 [0, 100]을 대상으로 정렬+그리디로 연속 덮임 여부를 판정.
    • 두 축 모두 경계까지 도달하면 YES, 아니면 NO.
