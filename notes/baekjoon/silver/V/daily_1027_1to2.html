    문제 3982 — Soccer Bets

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    토너먼트의 경기 순서가 섞여 있어도 우승 팀은 한 번도 패하지 않는다는 성질은 변하지 않는다. 총 16경기(16강~결승) 중 결승 우승팀만 패배가 0회이고, 나머지 팀은 최소 1패 이상을 기록한다(준결승 패자도 3위전 존재 여부와 관계없이 최소 1패). 따라서 모든 경기에서 패한 팀을 집합으로 모으고, 등장한 모든 팀 집합에서 이를 뺀 팀이 바로 우승팀이 된다.

    제약 정리 —
    입력: 테스트 케이스 수 c (1 ≤ c ≤ 100). 각 케이스는 임의 순서의 16경기 결과 t1 t2 g1 g2로 구성된다.
    팀명 t1, t2는 정확히 3글자 대문자이며, 득점 g1, g2는 0 ≤ g ≤ 10, g1 ≠ g2로 항상 승패가 존재한다.
    출력: 각 케이스마다 단 한 팀(무패 팀, 즉 우승팀)의 이름을 한 줄로 출력한다.

    코드 핵심 (변수/구조 매핑) —

    • Set<String> teams: 등장한 모든 팀 저장.
    • Set<String> lose: 한 번이라도 패한 팀 저장.
    • 각 경기 처리:
      • teams.add(t1); teams.add(t2); → 전체 팀 집합 갱신
      • 득점 비교 후 패자만 lose.add(loser);
    • 정답 도출: teams 순회 중 !lose.contains(team)인 유일한 팀 출력.
    • I/O 구성: BufferedReader로 입력, StringTokenizer로 파싱, StringBuilder로 결과 누적 출력.

    동작 과정 —
    1) 테스트 케이스 수 t를 읽는다.
    2) 각 케이스에서 16경기를 입력받고 팀 집합 및 패배 집합을 갱신한다.
    3) 입력이 끝나면 lose에 없는 팀(무패 팀)을 찾아 StringBuilder에 추가한다.
    4) 모든 케이스 처리가 끝나면 한 번에 결과를 출력한다.

    정확성 근거 —
    단일 엘리미네이션 구조에서 결승 우승팀만 패배가 0회이고, 나머지 팀은 모두 최소 1패를 갖는다. 따라서 전체 팀 − 패배 팀 집합에는 정확히 하나의 팀만 남으며, 그 팀이 곧 우승팀이다. 경기 순서가 섞여 있어도 이 불변식은 유지된다.

    엣지 케이스

    • 무승부는 존재하지 않음: g1 ≠ g2 보장.
    • 집합 순서 무관: Set은 순서가 없지만 무패 팀은 유일하므로 순회 순서와 무관하게 결과 동일.
    • 토너먼트 완결 보장: 항상 16경기가 주어져 우승팀이 존재하고 유일함.

    시간·공간 복잡도 —
    케이스당 경기 수는 고정(16)이므로 시간 복잡도는 O(1) 수준.
    일반화하면 경기 수 M에 대해 O(M), 공간은 팀/패배 집합 저장에 O(T) (T ≤ 16).

    정리

    • 모든 경기에서 패자를 기록해 집합으로 저장한다.
    • 전체 팀 집합에서 패배 집합을 제외하면 남는 유일한 팀이 우승팀이다.
    • 무승부 없음, 토너먼트 완결 보장으로 항상 정확한 결과 도출.

    문제 4028 — Chain of Fools

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    부러진 톱니(prong)와 휘어진 링크(link)가 동시에 위치 0에 오면 체인이 이탈한다. 한 번 회전할 때마다 두 위치는 각각 1씩 전진하므로, 시점 i에서 위치는 (p + i) % s, (l + i) % c가 된다. 두 값이 모두 0이 되는 가장 작은 i > 0을 찾으면 첫 실패 시점을 알 수 있다.

    제약 정리 —
    입력: 각 테스트는 s c p l. 1 < s < 100, s < c < 200, 초기에는 두 결함이 동시에 0이 아님. 0 0 0 0 입력으로 종료.
    출력: 처음 실패까지 필요한 회전 수를 r m/s 형태로 출력하거나, 결코 일어나지 않으면 Never를 출력한다. 분모는 항상 s이며 기약분수가 아닐 수 있다. m이 0이어도 r 0/s로 출력한다.

    코드 핵심 (변수/구조 매핑) —

    • s, c, p, l: 톱니 수, 링크 수, 부러진 톱니 위치, 휘어진 링크 위치.
    • limit = s * c: 주기 상한. 이 범위 내에서 만나지 않으면 영원히 만나지 않는다.
    • 탐색 변수 i: 1부터 limit까지 증가.
    • 만남 조건: (p + i) % s == 0 그리고 (l + i) % c == 0.
    • 형식 변환: r = i / s, m = i % s → r m/s.

    동작 과정 —
    1) 한 줄을 읽어 s, c, p, l을 파싱한다.
    2) 0 0 0 0이면 종료한다.
    3) i = 1..s×c에 대해 만남 조건을 검사한다.
    4) 조건을 만족하면 r, m을 계산해 r m/s로 기록하고, 만족하지 못한 채 끝나면 Never로 기록한다.

    정확성 근거 —
    상대 위치의 상태공간 크기는 s × c이며, 시간에 따라 순환한다. 따라서 최소 만남 시점은 존재하면 i ≤ lcm(s, c)에 있고, 상한을 s × c로 잡아도 충분하다. 조건이 한 번도 성립하지 않으면 이후에도 성립하지 않는다.

    엣지 케이스

    • 처음부터 동시 0은 없음(문제 보장).
    • m = 0인 경우도 형식대로 r 0/s를 출력한다.
    • 끝까지 조건 미성립이면 Never를 출력한다.

    시간·공간 복잡도 —
    한 케이스당 최대 O(s×c) 검사(s < 100, c < 200 → < 20,000회). 추가 메모리는 O(1).

    정리

    • 모듈러 이동을 이용해 두 결함이 동시에 0이 되는 최소 i를 선형 탐색한다.
    • 발견 시 i를 r m/s로 변환해 출력하고, 없으면 Never를 출력한다.
    • 주기 상한 s×c 내에서 항상 결론에 도달한다.
