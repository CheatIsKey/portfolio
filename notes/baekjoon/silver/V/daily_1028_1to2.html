    문제 4070 — Mahershalalhashbaz, Nebuchadnezzar, and Billy Bob Benjamin Go to the Regionals

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    각 팀의 이름 길이가 팀 평균 길이로부터 2 이내여야 한다는 규칙을 만족해야 한다. 이름 길이를 모두 정수 배열로 변환해 정렬하면, 유사한 길이끼리 묶는 것이 유리하다. 정렬된 상태에서 앞에서부터 k명씩 그룹을 만들어 평균을 계산하고, 모든 구성원이 평균과의 거리 ≤ 2이면 조건을 충족한다. 이 과정을 반복해 모든 그룹이 조건을 만족하면 전체 구성도 가능하다고 판단한다.

    제약 정리 —
    입력: 여러 테스트 케이스. 각 케이스는 n, k (n ≤ 1000, k ≤ 8, n은 k의 배수)와 이름 n개. 이름은 모두 대문자, 공백 없이 최대 80자. 출력: 각 케이스마다 Case x: yes 또는 Case x: no를 출력하며, 케이스 간 빈 줄을 추가한다.

    코드 핵심 (변수/구조 매핑) —

    • int[] lengths: 각 이름의 길이 저장.
    • Arrays.sort(lengths): 길이 오름차순 정렬.
    • 그룹 단위 처리:
      • 연속한 k명의 길이 합으로 평균 계산.
      • 각 구성원이 평균과의 차이가 2 이하인지 확인.
      • 하나라도 초과하면 조건 불만족으로 종료.
    • 출력 구성: StringBuilder로 결과 누적 후 한 번에 출력.

    동작 과정 —
    1) 각 테스트 케이스의 n, k를 입력받는다.
    2) n개의 이름을 읽어 길이를 저장한다.
    3) 길이 배열을 정렬한다.
    4) 앞에서부터 k개씩 묶어 평균을 계산하고, 모든 구성원이 |길이−평균| ≤ 2인지 검사한다.
    5) 한 그룹이라도 조건을 어기면 “no”, 전부 통과하면 “yes”.
    6) 케이스 번호를 붙여 순서대로 출력한다.

    정확성 근거 —
    이름 길이가 가까운 값일수록 조건을 만족하기 쉽기 때문에, 정렬 후 연속 구간으로 묶는 방식이 자연스럽다. 각 그룹 내에서 평균을 직접 계산하고 절대차를 비교함으로써 문제의 조건을 충족하는지를 정확히 판정한다. 모든 팀이 독립적으로 검사되며, 한 팀이라도 조건을 어기면 전체가 불가능하므로 즉시 종료한다.

    엣지 케이스

    • 모든 이름 길이가 동일한 경우: 항상 yes.
    • 한 그룹에서 평균 차이가 정확히 2인 경우: 경계값 포함으로 유효.
    • 입력 종료 조건 “0 0” 처리 후 종료.
    • n이 k의 배수가 아니면 입력 조건상 발생하지 않음.

    시간·공간 복잡도 —
    이름 정렬에 O(n log n), 그룹 검사에 O(n). 전체 복잡도는 O(n log n), 공간 복잡도는 O(n).

    정리

    • 이름 길이를 정렬 후 연속한 k명씩 그룹으로 구성한다.
    • 각 그룹의 평균과 거리 ≤ 2 조건을 검사한다.
    • 모든 그룹이 조건을 만족하면 yes, 아니면 no.
    • 케이스별로 결과를 포맷에 맞게 출력한다.

    문제 4108 — 지뢰찾기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    격자에서 각 칸이 지뢰('*')인지 여부에 따라 출력 문자를 결정한다. 지뢰 칸이면 그대로 '*'를 출력하고, 빈 칸이면 8방향 이웃 칸들 중 지뢰의 개수를 세어 숫자 문자('0'~'8')로 출력한다. 8방향은 상·하·좌·우와 대각선을 모두 포함한다.

    제약 정리 —
    입력: 여러 보드가 주어지며 각 보드는 R C (행, 열)로 시작하고 이어서 R개의 행 문자열이 주어진다. 종료 조건: 0 0을 만나면 입력이 끝난다. 각 행은 길이 C의 문자열로, 문자 '*'(지뢰) 또는 '.'(빈 칸)만 포함된다. 출력: 각 보드에 대해 같은 크기의 결과 보드를 출력한다. 지뢰는 '*', 빈 칸은 주변 지뢰 수('0'~'8').

    코드 핵심 (변수/구조 매핑) —

    • char[][] board: 입력 보드 저장.
    • int[] dx, dy: 8방향 이동 벡터(상·하·좌·우·대각선).
    • StringBuilder sb: 누적 출력 버퍼.
    • 두 중첩 루프(x, y)로 각 칸을 순회:
      • 지뢰면 '*'를 바로 추가.
      • 빈 칸이면 8방향을 검사해 지뢰 개수를 세고 ('0' + cnt)로 변환해 추가.

    동작 과정 —
    1) R, C를 읽는다. R = C = 0이면 종료.
    2) R줄을 읽어 보드를 만든다.
    3) 각 칸을 순회하며: 지뢰 → '*', 빈 칸 → 주변 8칸 범위 내의 지뢰 수를 세어 숫자 출력.
    4) 각 행 처리 후 개행을 추가한다.
    5) 모든 보드 처리가 끝나면 누적 결과를 한 번에 출력한다.

    정확성 근거 —
    주변 8칸만을 검사하고, 보드 경계는 인덱스 범위 체크(0 ≤ nx < R, 0 ≤ ny < C)로 안전하게 처리한다. 지뢰 칸은 변환 없이 그대로 유지하므로 원 보드의 지뢰 배치가 정확히 반영된다.

    엣지 케이스

    • 모서리·테두리 칸: 범위 체크로 보드 밖은 무시.
    • 지뢰만 있는 행/열: 지뢰는 그대로 '*'.
    • 지뢰가 하나도 없는 보드: 전부 '0'.
    • 연속 여러 보드 입력: 각 보드를 순서대로 바로 이어서 출력(문제 규격상 보드 간 공백 줄 요구 없음).

    시간·공간 복잡도 —
    각 칸마다 최대 8칸을 확인하므로 시간 복잡도는 O(R·C). 보드 저장에 O(R·C) 공간이 필요하다.

    정리

    • 지뢰는 그대로 '*', 빈 칸은 주변 8칸의 지뢰 수를 세어 숫자로 출력.
    • 경계는 인덱스 범위로 안전 처리.
    • 모든 보드를 처리해 결과를 누적 출력.
