    문제 4158 — CD

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    두 사람이 가진 CD 번호 목록이 각각 오름차순으로 주어지므로, 두 포인터로 한 번씩만 훑으며 공통 원소의 개수를 센다.
    각 포인터가 가리키는 값이 같으면 공통 개수(cnt)를 증가시키고 둘 다 한 칸 전진, 값이 다르면 더 작은 쪽 포인터만 전진한다.
    입력은 여러 테스트 케이스이며, 마지막에 0 0이 나오면 종료한다.

    제약 정리 —
    입력: 여러 테스트 케이스. 각 케이스는 N, M (각각 최대 1,000,000).
    다음 줄부터 N개의 상근이 CD 번호(오름차순), 이어서 M개의 선영이 CD 번호(오름차순).
    CD 번호는 1 이상 1,000,000,000 이하의 정수. 한 사람 내에서는 중복 없음. 종료 조건은 마지막 줄 0 0.
    출력: 각 케이스마다 두 사람이 동시에 가지고 있는 CD 개수.

    코드 핵심 (변수/구조 매핑) —

    • BufferedReader br: 대용량 입력 처리.
    • StringBuilder sb: 각 케이스 결과 누적 후 일괄 출력.
    • int N, M: 각 사람의 CD 개수.
    • int[] A, B: 상근이/선영이의 CD 번호 배열(오름차순 가정).
    • int idx1, idx2: 두 포인터(각 배열의 현재 위치).
    • int cnt: 공통 CD 개수 카운트.

    동작 과정 —
    1) N M을 읽어 N==0 && M==0이면 루프 종료.
    2) N개의 줄을 읽어 배열 A를 채운 뒤, M개의 줄로 배열 B를 채운다(입력 자체가 오름차순).
    3) 두 포인터 초기화(idx1=0, idx2=0), cnt=0으로 시작.
    4) while (idx1 < N && idx2 < M):

    • A[idx1] == B[idx2] → cnt++, 두 포인터 모두 증가.
    • A[idx1] < B[idx2] → idx1++.
    • 그 외(A[idx1] > B[idx2]) → idx2++.
    5) 루프 종료 후 sb.append(cnt).append('\n')로 결과 누적.
    6) 모든 케이스 처리 뒤 System.out.print(sb)로 한 번에 출력.

    정확성 근거 —
    두 배열이 정렬되어 있으므로, 두 포인터 방식은 각 원소를 최대 한 번씩만 비교하며 공통 원소를 빠짐없이 세어낼 수 있다.
    한 사람 내 중복이 없으므로 같은 값을 만났을 때 둘 다 한 칸 전진해도 누락/중복 카운트가 발생하지 않는다.
    정렬 가정과 비교 규칙에 의해 모든 공통 항목을 정확히 1회씩 집계한다.

    엣지 케이스

    • N==0 또는 M==0: 바로 0 출력.
    • 두 배열이 전혀 겹치지 않는 경우: cnt=0.
    • 모든 원소가 일치하는 경우: cnt = min(N, M) (문제 조건상 각자 중복 없음).
    • 입력 끝 0 0을 읽으면 즉시 종료.
    • 대용량 입력: BufferedReader, StringBuilder 사용으로 I/O 오버헤드 완화.

    시간·공간 복잡도 —
    두 포인터 비교는 각 케이스당 O(N+M). 입력 저장을 위해 배열 A, B가 필요하므로 공간은 O(N+M).
    (입력 자체가 정렬되어 들어오므로 추가 정렬 비용 없음.)

    정리

    • 정렬된 두 목록을 두 포인터로 선형 스캔.
    • 같으면 카운트 후 둘 다 전진, 다르면 작은 쪽만 전진.
    • 대용량에서도 O(N+M)로 효율적이며, 각 케이스 결과를 누적 출력.

    문제 4333 — Major Scales

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    격자에서 각 칸이 지뢰('*')인지 여부에 따라 출력 문자를 결정한다. 지뢰 칸이면 그대로 '*'를 출력하고, 빈 칸이면 8방향 이웃 칸들 중 지뢰의 개수를 세어 숫자 문자('0'~'8')로 출력한다. 8방향은 상·하·좌·우와 대각선을 모두 포함한다.

    제약 정리 —
    입력: 여러 보드가 주어지며 각 보드는 R C (행, 열)로 시작하고 이어서 R개의 행 문자열이 주어진다. 종료 조건: 0 0을 만나면 입력이 끝난다. 각 행은 길이 C의 문자열로, 문자 '*'(지뢰) 또는 '.'(빈 칸)만 포함된다. 출력: 각 보드에 대해 같은 크기의 결과 보드를 출력한다. 지뢰는 '*', 빈 칸은 주변 지뢰 수('0'~'8').

    코드 핵심 (변수/구조 매핑) —

    • char[][] board: 입력 보드 저장.
    • int[] dx, dy: 8방향 이동 벡터(상·하·좌·우·대각선).
    • StringBuilder sb: 누적 출력 버퍼.
    • 두 중첩 루프(x, y)로 각 칸을 순회:
      • 지뢰면 '*'를 바로 추가.
      • 빈 칸이면 8방향을 검사해 지뢰 개수를 세고 ('0' + cnt)로 변환해 추가.

    동작 과정 —
    1) R, C를 읽는다. R = C = 0이면 종료.
    2) R줄을 읽어 보드를 만든다.
    3) 각 칸을 순회하며: 지뢰 → '*', 빈 칸 → 주변 8칸 범위 내의 지뢰 수를 세어 숫자 출력.
    4) 각 행 처리 후 개행을 추가한다.
    5) 모든 보드 처리가 끝나면 누적 결과를 한 번에 출력한다.

    정확성 근거 —
    주변 8칸만을 검사하고, 보드 경계는 인덱스 범위 체크(0 ≤ nx < R, 0 ≤ ny < C)로 안전하게 처리한다. 지뢰 칸은 변환 없이 그대로 유지하므로 원 보드의 지뢰 배치가 정확히 반영된다.

    엣지 케이스

    • 모서리·테두리 칸: 범위 체크로 보드 밖은 무시.
    • 지뢰만 있는 행/열: 지뢰는 그대로 '*'.
    • 지뢰가 하나도 없는 보드: 전부 '0'.
    • 연속 여러 보드 입력: 각 보드를 순서대로 바로 이어서 출력(문제 규격상 보드 간 공백 줄 요구 없음).

    시간·공간 복잡도 —
    각 칸마다 최대 8칸을 확인하므로 시간 복잡도는 O(R·C). 보드 저장에 O(R·C) 공간이 필요하다.

    정리

    • 지뢰는 그대로 '*', 빈 칸은 주변 8칸의 지뢰 수를 세어 숫자로 출력.
    • 경계는 인덱스 범위로 안전 처리.
    • 모든 보드를 처리해 결과를 누적 출력.
