    문제 4359 — Forests

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    스탠이 생각한 수의 가능 범위를 [low, high]로 유지한다(초기값 [1, 10]).
    올리의 말에 대한 스탠의 응답에 따라 범위를 점차 좁힌다:
    "too high" → high = min(high, 추측값-1)
    "too low" → low = max(low, 추측값+1)
    "right on"이 나오면, 마지막 추측값이 현재 가능 범위 [low, high] 안에 있으면 스탠은 정직(Stan may be honest), 아니면 거짓(Stan is dishonest)으로 판정하고, 다음 게임을 위해 범위를 초기화한다.

    제약 정리 —
    입력: 여러 개의 게임이 연속되어 주어진다. 각 게임은 (숫자, 응답)이 번갈아 등장하며, 숫자는 1 이상 10 이하의 정수.
    응답은 "too high", "too low", "right on" 중 하나. "right on"이 등장하면 그 게임은 종료.
    전체 입력 종료는 단독으로 등장하는 0.
    출력: 각 게임마다 스탠이 거짓말을 했는지 여부를 한 줄로 출력.

    코드 핵심 (변수/구조 매핑) —

    • BufferedReader br: 입력 처리.
    • StringBuilder sb: 각 게임 결과 누적 후 일괄 출력.
    • int low, high: 스탠이 생각한 수의 가능한 최소/최대값(초기 1, 10).
    • int num: 올리가 방금 말한 숫자.
    • String answer: 스탠의 응답(too high/too low/right on).

    동작 과정 —
    1) 전체 루프에서 숫자 한 줄을 읽는다. num == 0이면 프로그램 종료.
    2) 이어서 응답 한 줄을 읽고, 응답에 따라 범위를 갱신한다.
      • "too high": high = Math.min(high, num - 1)
      • "too low": low = Math.max(low, num + 1)
    3) "right on"이면 현재 범위 [low, high]에 num이 포함되는지 확인:
      • 포함되면 Stan may be honest
      • 아니면 Stan is dishonest
    4) 판정 후 다음 게임을 위해 low=1, high=10으로 초기화하고 계속.

    정확성 근거 —
    각 "too high"/"too low" 응답은 실제 정답이 될 수 있는 범위를 단조롭게 좁힌다. "right on" 시점에 마지막 추측값이 이 누적 제약을 모두 만족([low, high] 내 포함)하면 스탠의 모든 응답과 모순이 없고, 벗어나면 적어도 한 번의 응답과 모순이 발생하므로 거짓말로 판정할 수 있다.

    엣지 케이스

    • 첫 추측이 곧바로 "right on": 초기 범위 [1,10] 안이면 정직.
    • 중간에 범위가 비게 되는 상황: 출력은 "right on"이 나올 때만 하며, 그때 마지막 추측이 범위 밖이므로 거짓 판정.
    • 최댓값/최솟값 추측(예: 1, 10) 처리: 업데이트 식에 의해 자연스럽게 반영(num-1, num+1).
    • 게임 종료 후 즉시 범위 초기화: 다음 게임에 전 게임의 제약이 섞이지 않도록 보장.

    시간·공간 복잡도 —
    각 입력 라인을 한 번씩만 처리하므로 전체 시간은 O(L)(L: 읽은 줄 수)이고, 범위 변수만 쓰므로 추가 공간은 O(1).

    정리

    • 가능한 정답 구간 [low, high]을 유지하며 응답에 따라 축소.
    • "right on" 시 마지막 추측이 구간 내면 정직, 아니면 거짓.
    • 게임마다 구간을 초기화하여 독립적으로 판정.

    문제 4351 — Hay Points

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    주어진 Hay Point 사전을 기반으로, 각 직무 설명문에서 등장하는 단어의 점수 합을 계산한다.
    사전에 존재하지 않는 단어는 0점으로 간주하며, 한 직무 설명이 "." 한 줄로 끝날 때까지 점수를 누적한다.
    즉, 사전(Map)을 이용한 단어-가치 매핑과 입력 스트림 처리가 핵심이다.

    제약 정리 —
    입력:
    첫 줄에 m (≤1000), n (≤100).
    이후 m개의 줄에 단어(최대 16자, 소문자)와 해당 가치(0~1,000,000).
    그다음 n개의 직무 설명이 주어지며, 각 설명은 여러 줄로 구성되고 "." 한 줄이 등장하면 종료.
    출력: 각 직무 설명마다 계산된 총 Hay Point 점수를 정수로 출력.

    코드 핵심 (변수/구조 매핑) —

    • BufferedReader br: 빠른 입력 처리용.
    • StringTokenizer st: 한 줄의 단어를 분리하여 토큰 단위로 처리.
    • Map<String, Integer> map: 단어 → 가치 매핑.
    • int m, n: 사전 단어 수 / 직무 설명 수.
    • long score: 현재 직무 설명의 총점.

    동작 과정 —
    1) 첫 줄에서 m, n을 입력받는다.
    2) m개의 줄을 읽으며 map.put(word, value)로 사전을 구성한다.
    3) n개의 직무 설명에 대해 다음을 반복한다:
      • score = 0으로 초기화.
      • "."이 나올 때까지 각 줄을 읽어 단어 단위로 분리.
      • 각 단어마다 map.getOrDefault(word, 0)을 더해 점수를 누적.
    4) "."이 나오면 해당 설명의 총점을 출력하고 다음 설명으로 이동.

    정확성 근거 —
    모든 단어를 사전과 대조하므로, 사전에 없는 단어는 자동으로 0으로 처리된다.
    각 직무 설명은 입력상 "."으로 명확히 구분되며, 모든 단어는 공백으로만 분리되어 있어 추가 파싱 오류가 없다.
    HashMap 접근은 평균 O(1)이므로 빠르고 정확한 누적 합이 가능하다.

    엣지 케이스

    • 사전에 없는 단어만 있는 경우: 총점 0.
    • 모든 단어가 사전에 존재하는 경우: 해당 단어 값의 총합 출력.
    • 빈 줄 또는 공백만 있는 경우: 무시.
    • "." 바로 등장 시: 점수 0으로 출력.
    • 값이 매우 큰 경우: long으로 누적하여 오버플로 방지.

    시간·공간 복잡도 —
    사전 구축: O(m).
    각 직무 설명 처리: 모든 단어 합쳐 O(total_words).
    따라서 전체 시간 복잡도는 O(m + total_words), 공간 복잡도는 O(m).

    정리

    • 사전(Map)에 단어-가치를 저장.
    • 직무 설명을 한 줄씩 읽어 단어별 점수를 누적.
    • "."을 만나면 합계를 출력 후 초기화.
    • HashMap으로 빠른 검색, BufferedReader로 효율적 입력 처리.
