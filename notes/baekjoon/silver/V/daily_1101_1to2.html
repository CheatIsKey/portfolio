    문제 4383 — Forests

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Jolly jumper는 길이 n의 수열에서 인접한 두 수의 차의 절댓값들이 {1, 2, ..., n-1}를 정확히 한 번씩 모두 나타나는 수열을 말합니다(길이 1이면 항상 Jolly). 제시한 코드는 각 테스트 케이스마다 연속한 차이의 절댓값 d = |a[i] - a[i+1]|를 계산해 d가 1 ≤ d ≤ n-1 범위에 있고, 중복 없이 한 번씩만 등장하는지를 불리언 배열로 체크합니다. 모든 차이가 조건을 만족하면 Jolly, 아니면 Not jolly를 출력합니다.

    • 입력 파싱: 각 줄을 읽어 첫 토큰을 n으로, 이어서 n개의 정수를 수열로 취함(EOF까지 반복)
    • 단건 예외: n == 1이면 무조건 Jolly
    • 차이 검증:
      • d = |a[i] - a[i+1]| 계산
      • d가 1..n-1 범위를 벗어나거나(d == 0 포함) 이미 본 적이 있으면 실패
      • 그 외에는 diff[d] = true로 표시
    • 판정: 모든 인접쌍을 통과하면 Jolly, 중간에 한 번이라도 어기면 Not jolly

    예시

    • 4 1 4 2 3 → 인접 차이: 3, 2, 1 → {1,2,3} 모두 등장 → Jolly
    • 5 1 4 2 -1 6 → 인접 차이: 3, 2, 3, 7 → 7은 범위 1..4 밖 → Not jolly

    시간 복잡도

    각 케이스에서 인접 차이는 n-1개만 확인하므로 O(n).

    공간 복잡도

    입력 수열 O(n)과 차이 체크용 불리언 배열 O(n) → 전체 O(n).

    주의

    • 입력 형식: 이 문제는 “각 줄”이 하나의 케이스입니다. 제시한 코드는 readLine()으로 한 줄을 읽고 StringTokenizer로 파싱하므로 문제의 입력 형식과 정확히 일치합니다.
    • 중복/범위 체크: d == 0(같은 값이 연속)이나 d ≥ n이면 즉시 실패 처리하는 것이 핵심입니다.
    • 정수 범위: 입력 정수 범위가 매우 크다면 Math.abs(a-b)에서 이론상 오버플로우가 날 수도 있습니다(Integer.MIN_VALUE의 절댓값 등). 안전성을 더 높이고 싶다면 long으로 변환해 차이를 구해도 됩니다.
    대안 아이디어
    • BitSet 활용 — 불리언 배열 대신 공간/연산을 조금 더 효율적으로 표현 
      
      import java.io.*;
      import java.util.*;
      import java.util.BitSet;
      
      public class Main {
          public static void main(String[] args) throws Exception {
              BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
              StringBuilder sb = new StringBuilder();
              String line;
              while ((line = br.readLine()) != null) {
                  StringTokenizer st = new StringTokenizer(line);
                  int n = Integer.parseInt(st.nextToken());
                  int[] a = new int[n];
                  for (int i = 0; i < n; i++) a[i] = Integer.parseInt(st.nextToken());
                  if (n == 1) { sb.append("Jolly\n"); continue; }
      
                  BitSet bs = new BitSet(n);
                  boolean ok = true;
                  for (int i = 0; i < n - 1; i++) {
                      long d = Math.abs((long)a[i] - a[i + 1]); // 오버플로 방지
                      if (d < 1 || d >= n || bs.get((int)d)) { ok = false; break; }
                      bs.set((int)d);
                  }
                  sb.append(ok ? "Jolly\n" : "Not jolly\n");
              }
              System.out.print(sb);
          }
      }

      로직은 동일하며, BitSet을 사용해 비트 단위로 방문 여부를 기록합니다.

    • 정렬 기반 확인 — 이해 목적(성능은 기존이 더 좋음) 
      
      import java.io.*;
      import java.util.*;
      
      public class Main {
          public static void main(String[] args) throws Exception {
              BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
              StringBuilder sb = new StringBuilder();
              String line;
              while ((line = br.readLine()) != null) {
                  StringTokenizer st = new StringTokenizer(line);
                  int n = Integer.parseInt(st.nextToken());
                  int[] a = new int[n];
                  for (int i = 0; i < n; i++) a[i] = Integer.parseInt(st.nextToken());
                  if (n == 1) { sb.append("Jolly\n"); continue; }
      
                  int[] diff = new int[n - 1];
                  for (int i = 0; i < n - 1; i++) diff[i] = Math.abs(a[i] - a[i + 1]);
                  Arrays.sort(diff); // O(n log n)
      
                  boolean ok = true;
                  for (int i = 0; i < n - 1; i++) {
                      if (diff[i] != i + 1) { ok = false; break; }
                  }
                  sb.append(ok ? "Jolly\n" : "Not jolly\n");
              }
              System.out.print(sb);
          }
      }

      차이들을 정렬해 [1,2,...,n-1]과 동일한지 비교합니다(O(n log n)).

    포인트 요약

    • 차이 d는 1..n-1 범위여야 하고, 중복 없이 한 번씩 등장해야 함
    • 불리언(또는 BitSet)으로 등장 여부를 O(1)로 체크 → 전체 O(n)
    • 문제 입력은 “각 줄 = 한 케이스” 형태 → 현재 파싱 방식이 정석

    문제 4402 — Soundex

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Soundex는 철자가 비슷해 발음이 유사한 단어를 같은 코드로 묶는 방식입니다. 각 알파벳을 아래 규칙으로 숫자로 치환하고, 같은 숫자가 연속되면 한 번만 출력하며, A, E, I, O, U, H, W, Y는 출력하지 않습니다.

    • 1: B F P V
    • 2: C G J K Q S X Z
    • 3: D T
    • 4: L
    • 5: M N
    • 6: R

    제시한 코드는 각 줄(단어)을 읽어 문자별로 매핑 숫자 curr를 구합니다. curr == 0(모음 등 비표기 문자)이면 출력하지 않고 연속 중복 방지를 초기화(prev = 0)합니다. curr != 0이고 curr != prev일 때만 출력하여 같은 코드가 연속될 경우 한 번만 나오도록 합니다.

    • 입력: 대문자 단어가 한 줄에 하나씩 주어짐(EOF까지 반복)
    • 치환/출력:
      • curr = map.getOrDefault(c, 0)로 숫자 획득(없으면 0)
      • curr == 0이면 출력 안 함 + prev = 0로 리셋
      • curr != 0이고 curr != prev이면 출력하고 prev = curr

    예시

    • KHAWN → K(2) H(0) A(0) W(0) N(5) → 연속 중복/비표기 처리 → 25
    • PFISTER → P(1) F(1) I(0) S(2) T(3) E(0) R(6) → 11 → 모음으로 리셋 후 2,3,6 → 1236
    • BOBBY → B(1) O(0) B(1) B(1) Y(0) → 1 / 리셋 / 1 / 연속중복무시 / 리셋 → 11

    시간 복잡도

    각 단어 길이를 L이라 하면 문자당 O(1) 처리 → O(L) (한 줄당).

    공간 복잡도

    고정 크기 매핑 테이블과 출력 버퍼만 사용 → O(1) (단, 출력 문자열은 결과 길이에 비례).

    주의

    • 리셋 규칙: 모음/비표기자(AEIOUHWY)를 만나면 prev = 0으로 리셋해야 이후 같은 숫자가 다시 나올 수 있습니다.
    • 연속 중복: 같은 코드가 연속되면 한 번만 출력합니다. (예: PF는 둘 다 1 → 1 한 번)
    • 입력 공백: 빈 줄이 섞일 수 있으므로 isBlank() 체크로 건너뜁니다.
    대안 아이디어
    • 배열 매핑으로 성능/간결성 개선 — 해시맵 대신 int[26] 사용 
      
      import java.io.*;
      import java.util.*;
      
      public class Main {
          public static void main(String[] args) throws Exception {
              BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
              StringBuilder sb = new StringBuilder();
              int[] code = new int[26];
              for (char c : "BFPV".toCharArray()) code[c - 'A'] = 1;
              for (char c : "CGJKQSXZ".toCharArray()) code[c - 'A'] = 2;
              for (char c : "DT".toCharArray()) code[c - 'A'] = 3;
              code['L' - 'A'] = 4;
              for (char c : "MN".toCharArray()) code[c - 'A'] = 5;
              code['R' - 'A'] = 6;
      
              String line;
              while ((line = br.readLine()) != null) {
                  if (line.isBlank()) continue;
                  int prev = 0;
                  for (int i = 0; i < line.length(); i++) {
                      int curr = code[line.charAt(i) - 'A'];
                      if (curr == 0) { prev = 0; continue; }
                      if (curr != prev) { sb.append(curr); prev = curr; }
                  }
                  sb.append('\n');
              }
              System.out.print(sb);
          }
      }

      로직은 동일하고 상수 접근으로 약간 더 빠릅니다.

    포인트 요약

    • 문자 → 코드 치환 후, 연속 동일 코드 1회 출력
    • AEIOUHWY는 출력하지 않으며, 연속성 리셋 역할
    • 한 단어 처리 비용 O(L), 추가 공간 O(1)
