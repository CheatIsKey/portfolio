    문제 4452 — Lightning Lessons

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Lightning Lessons는 번개(수열)의 각 사이클에서 연속 차분을 취해 길이를 1씩 줄여 나가며, 모든 원소가 0이 되는 순간 수렴(converge)으로 간주하고 그 전까지 거친 사이클 수만큼 z를 출력한 뒤 ap!를 붙인다. 반대로 길이가 1이 될 때까지 0이 아니면 발산(diverge)으로 간주하고, 마지막 단일 원소의 부호로 *fizzle*(양수) 또는 *bunny*(음수)를 출력한다.

    • 사이클 규칙:
      • 현재 수열이 a[0], a[1], ..., a[m-1]라면 다음 수열은 b[i] = a[i+1] - a[i] (길이 m-1)
      • 이 과정을 한 번 수행할 때마다 사이클 수 P를 1 증가
      • 어느 시점에든 모든 원소가 0이면 수렴 종료 → "z".repeat(P) + "ap!"
      • 길이가 1이 되었는데 0이 아니면 발산 종료 → 양수면 *fizzle*, 음수면 *bunny*
    • 초기 즉시 종료:
      • 처음부터 모든 원소가 0이면 P = 0이므로 바로 ap!
      • 처음 길이가 1이라면 그 값의 부호/0 여부로 즉시 결정

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 테스트 케이스 수 N을 읽고 각 케이스마다 길이 M과 초기 수열을 읽는다
    • calculate(int[] init) 로직:
      • allZero(init) 면 즉시 "ap!" 반환
      • 아닌 경우, 길이가 1 초과인 동안 다음을 반복:
        • 연속 차분 수열 ints[i] = init[i+1] - init[i]를 만들고 사이클 카운트 cnt++
        • ints가 올제로 되면 "z".repeat(cnt) + "ap!" 반환(올제로 만든 그 사이클까지 센다)
        • ints.length == 1이면 부호로 *fizzle* 또는 *bunny* 결정 후 반환
        • 그 외에는 init = ints로 갱신하고 다음 사이클
      • 루프가 끝난 뒤(초기 길이 1 등) 값이 0이면 ap!, 양수면 *fizzle*, 음수면 *bunny*
    • allZero(int[])는 배열의 모든 원소가 0인지 선형 확인
    • 출력은 케이스별 결과를 줄바꿈으로 이어서 한번에 출력

    예시

    • 2 1 1 →
      • 초기: [1, 1], 차분: [0] (P=1)
      • 길이 1이자 0 → 수렴. 출력: z + ap! = zap!
    • 5 1 3 6 10 15 →
      • 차분 테이블이 두 사이클 후 전부 0 → P=3 → zzzap!
    • 5 1 2 4 8 16 →
      • 길이 1이 될 때까지 0이 아님 → 마지막 값 양수 → *fizzle*
    • 2 1 0 →
      • 차분: [-1] (P=1), 마지막 값 음수 → *bunny*

    시간 복잡도

    각 사이클마다 길이가 1 줄고 차분 계산에 O(현재 길이)가 든다. 최악(M ≤ 20)에서 O(M + (M-1) + ... + 1) = O(M^2). 입력 제약이 작아 충분히 빠르다.

    공간 복잡도

    차분 배열 하나만 추가로 만들며 길이가 사이클마다 감소하므로 O(M). 그 외 보조 변수는 상수.

    주의

    • 차분 방향: 반드시 다음 - 현재 (a[i+1] - a[i]) 순서로 계산해야 한다.
    • 사이클 카운트: 올제로 만들어낸 사이클까지 포함해 P를 센다(올제로 된 상태 자체는 카운트하지 않음).
    • 초기 특수 케이스:
      • 처음부터 전부 0이면 바로 ap!
      • 처음 길이 1이면 값이 0인지/부호를 즉시 판정
    • 정수 범위: 입력 절댓값 ≤ 1000, M ≤ 20이라 차분 누적도 int 범위를 안전하게 벗어나지 않는다.
    • 출력 정확도: 수렴 시 "z".repeat(P) + "ap!", 발산 시 마지막 단일 원소의 부호로 문자열을 정확히 선택.
    대안 아이디어
    • 수학적 관찰 — 수열이 차수가 d인 다항식에서 유도된 값이면 (d+1)차 차분은 0이 된다. 다만 본 문제는 일반 정수 수열이므로 직접 차분을 구현하는 현재 방식이 가장 단순·안전하다.
    • 제자리(in-place) 차분 — 같은 배열을 앞에서부터 덮어쓰며 길이만 논리적으로 줄일 수도 있다. 구현은 복잡해지지만 추가 메모리 할당을 줄일 수 있다(본 제약에서는 이득이 미미).

    포인트 요약

    • 각 사이클에서 a[i+1] - a[i]로 차분, 길이는 1씩 감소
    • 올제로 되면 P개의 z 뒤에 ap! 출력
    • 길이 1인데 0이 아니면 부호로 *fizzle*/*bunny* 결정
    • 초기 전부 0이면 즉시 ap!
    • 시간 O(M^2), 공간 O(M) — 제약 내 충분

    문제 4465 — Tales from DeCrypt

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Tales from DeCrypt는 7-bit 인쇄 가능 ASCII(0x20=' ' ~ 0x7e='~', 총 95개)에 대해 선형 합동 난수생성기(LCG)로부터 나온 난수로 회전 암호를 적용한다. 암호화는 각 인쇄 가능한 문자 c를 ((c-32) + shift) mod 95 + 32로 바꾸며, 여기서 shift = ceil(95 - r*95), r은 LCG가 내놓는 [0,1) 난수이다. 복호화는 같은 r 순서를 재현해 동일한 shift를 빼는 것으로 수행된다.

    • 난수 생성기 LCG (문제 정의 그대로): 
      val = (s mod m) / double(m)   // r ∈ [0, 1)
      s   = (a * s + 1) mod m
      return val
      • 입력 첫 줄에 a m s가 공백 구분으로 주어진다(2 ≤ 값 ≤ 65536).
      • 각 문자마다 위 순서로 val을 구한 뒤 s를 갱신한다(암복호 동일).
    • 문자 처리 규칙:
      • 인쇄 가능 범위(32~126)면 회전 적용, 아니면(개행 등) 그대로 통과.
      • 암호화: c' = ((c-32) + ceil(95 - r*95)) mod 95 + 32
      • 복호화: c = ((c'-32) - ceil(95 - r*95)) mod 95 + 32
      • r∈[0,1)이므로 shift = ceil(95(1-r))는 1..95 범위.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 첫 줄에서 a, m, seed를 읽는다.
      • 이후 파일 끝까지 줄 단위로 읽어 각 문자 처리.
    • 복호화 로직(인쇄 가능 문자일 때):
      • val = (seed % m) / (double)m로 r 계산
      • seed = (a * seed + 1) % m로 갱신(문제의 r()와 동일 순서)
      • shift = ceil(95 - val*95)
      • x = (code - 32) - shift 후 x %= 95 (음수면 +95)
      • code = x + 32를 문자로 변환해 출력
    • 인쇄 불가/범위 밖 문자는 그대로 출력(개행 포함).
    • 성능 향상을 위해 StringBuilder로 누적 출력.

    예시

    • 예제 입력의 첫 줄 12343 65536 11로 LCG를 초기화하고, 이후 각 라인의 인쇄 가능한 문자만 동일한 난수 시퀀스로 역이동 → 예제 출력의 a.../b.../c.../d...가 재현된다.

    시간 복잡도

    각 문자를 상수 시간에 처리하므로 O(L) (L은 전체 문자 수).

    공간 복잡도

    난수 상태(seed)와 보조 변수 몇 개, 출력 버퍼만 사용하므로 O(1) (+ 출력 길이).

    주의

    • LCG 호출 순서: 반드시 val을 먼저 계산하고 seed를 갱신해야 한다 (문제의 r()와 동일). 순서를 바꾸면 난수열이 어긋난다.
    • shift 범위: ceil(95 - r*95)는 1~95. 모듈러 감산 후 음수 보정을 잊지 말 것.
    • 문자 범위 체크: 32 ≤ code ≤ 126일 때만 회전. 그 외(개행 포함)는 원문 유지.
    • 정수 오버플로우: (long)a * seed로 곱한 뒤 % m 하는 방식이 안전하다(코드에서도 캐스팅 사용).
    • 줄 처리: 입력에서 줄 경계를 그대로 보존해야 하므로 라인 사이에 개행을 출력한다.
    대안 아이디어
    • 사전 생성 — 같은 LCG 시드를 사용하므로, 전체 텍스트 길이가 미리 알려진다면 미리 shift 배열을 만들어 한 번에 적용할 수 있다(복잡도/메모리는 동일, 캐시 효율만 약간 개선될 수 있음).

    포인트 요약

    • LCG로 r 생성 → shift = ceil(95 - r*95)
    • 인쇄 가능 문자에만 적용:
      • 암호화: +shift, 복호화: -shift (mod 95)
    • LCG 순서(값 계산 → 상태 갱신) 일치가 핵심
    • 시간 O(L), 공간 O(1) (+ 출력 버퍼)
