    문제 4542 — Blue Jeans

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Blue Jeans는 길이 60의 DNA 문자열들(문자 집합 {A, C, G, T})이 주어질 때, 모든 문자열에 공통으로 등장하는 최장 공통 부분문자열(연속)을 찾는 문제다. 최장 길이가 3 미만이면 no significant commonalities를 출력하고, 같은 최장 길이의 후보가 여러 개면 사전식으로 가장 앞선 문자열을 출력한다.

    • 핵심 정의:
      • 부분문자열: 인접한 연속 구간(예: "TAGACC"의 "GAC").
      • 모든 문자열에 등장: 각 문자열에서 indexOf(s) != -1가 성립.
    • 전략 개요:
      • 후보 수를 줄이기 위해 가장 짧은 문자열을 기준(base)으로 선택.
      • 길이 j를 큰 것부터(len → 3) 내려오며 검사, 첫 성공 시 바로 정답.
      • 같은 길이의 후보들 중에는 사전식 최소를 선택.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 데이터셋 수 n을 읽고, 각 데이터셋마다 문자열 수 m과 길이 60의 DNA 문자열 m개를 읽는다.
    • 기준 문자열 선택:
      • 가장 짧은 문자열의 인덱스 minIdx를 찾아 base = arr.get(minIdx)로 설정.
    • 길이 내림차순 탐색:
      • for j = base.length() down to 3:
        • HashSet<String>에 base의 길이 j 모든 부분문자열을 중복 없이 수집.
        • 각 후보 s에 대해 다른 모든 문자열에 indexOf(s)가 존재하는지 확인.
        • 모두 포함되면 TreeSet<String>에 넣어 사전식 정렬.
      • TreeSet이 비어 있지 않으면 그 길이에서 사전식 첫 원소를 정답으로 채택하고 루프 종료.
    • 출력:
      • 정답이 있으면 그 문자열, 없으면 no significant commonalities 출력.

    예시

    • 예제 1: 공통 부분문자열의 최장 길이가 3 미만 → no significant commonalities.
    • 예제 2: 길이 6의 다수 후보 중 사전식으로 가장 앞선 AGATAC 선택.
    • 예제 3: 세 문자열 모두에 등장하는 최장 공통 부분문자열은 CATCATCAT.

    시간 복잡도

    기준 문자열 길이를 L(≤60)이라 하면, 각 길이 j에서 후보 수는 최대 L-j+1 = O(L), 각 후보를 m개 문자열에 대해 indexOf(O(L))로 검사하므로 총합 O(m · L^3) 내외. 제약이 작아 충분히 빠르다.

    공간 복잡도

    길이 j의 부분문자열 집합(최대 O(L)개)과 정렬용 TreeSet 등으로 O(L)~O(L^2) 정도. L=60이므로 메모리 부담은 매우 작다.

    주의

    • 연속성: LCS(최장 공통 부분수열)이 아니라 최장 공통 부분문자열(연속)임에 주의.
    • 기준 문자열: 가장 짧은 문자열을 기준으로 해야 후보 수가 최소화된다.
    • 중복 제거: 같은 길이에서 동일 부분문자열이 여러 번 나오지 않도록 HashSet 사용.
    • 사전식 우선: TreeSet으로 정렬 후 first()를 선택.
    • 조기 종료: 길이를 내림차순으로 돌며 처음 성공한 길이에서 즉시 종료해야 시간 절약.
    • 입력 공백: 각 라인은 trim() 후 사용(불필요한 공백 제거).
    대안 아이디어
    • 이진 탐색 + 해시(롤링): 길이를 이진 탐색하며 각 길이에 대해 롤링 해시로 교집합 검증. 이론상 O(m · L^2 log L) 수준으로 줄일 수 있으나, L=60에서는 단순 구현이 더 간단하고 충분히 빠름.
    • 접미사 배열/트라이: 모든 문자열의 공통 부분문자열을 구조적으로 탐색. 구현 부담이 크고 본 문제 제약에선 과도.

    포인트 요약

    • 가장 짧은 문자열을 기준으로 길이 내림차순 탐색
    • 각 길이에서 후보 생성(중복 제거) → 모든 문자열 포함 여부 검사
    • 성공 시 해당 길이의 사전식 최소를 정답
    • 최장 길이 < 3 이면 no significant commonalities
    • 시간 O(m · L^3), 공간 O(L)~O(L^2) — 제약 내 충분

    문제 4579 — Judges' Time Calculation

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Judges' Time Calculation은 대회 시작 시각(SH:SM)과 지속 시간(DH:DM)이 주어질 때, 제출 시각의 시(hour) 별로 적용할 수 있는 경과 분(elapsed minutes) 계산 공식을 표 형태로 만드는 문제다. 표의 각 행은 H:XX | XX ± K 꼴이며, 해당 시각대(예: 3:XX)에 제출된 분(XX, 0~59)을 대입하면 경과 분이 된다.

    • 행의 범위:
      • 가장 이른 제출은 시작 시각 그대로(경과 0분), 가장 늦은 제출은 정확히 지속 시간 경과 시각까지 포함.
      • 따라서 시작 시각이 속한 시간대부터, 끝나는 시각이 속한 시간대까지 연속된 모든 시에 대해 한 줄씩 만든다.
    • 행별 공식:
      • 시작 시각 SH:SM에서 j번째 시간대(j = 0,1,2,...)의 분 오프셋은 offset = 60*j - SM.
      • 따라서 그 시간대의 공식은 XX(offset=0), XX + offset(offset>0), XX - (-offset)(offset<0) 중 하나가 된다.
    • 12시간 표기:
      • 시간은 1~12로 순환: hour = ((SH - 1 + j) % 12) + 1.
      • 한 자리 시각(예: 5:XX)은 왼쪽에 공백 1칸을 넣는다. 헤더의 "time" 앞에도 공백 1칸을 둔다.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 테스트 케이스 수 N을 읽고, 각 케이스마다 SH, SM, DH, DM을 읽는다.
    • 행 개수 계산:
      • total = 60*DH + DM (총 지속 시간, 분)
      • rows = (SM + total) / 60 + 1 — 시작 분 SM에서 총 지속 분을 더해 몇 번 시간 경계를 넘는지를 구하고, 시작 시간대(0번째)를 포함해 +1.
    • 표 출력:
      • 상단/헤더/구분선은 예시 형식대로 출력.
      • 각 j = 0..rows-1에 대해:
        • hour = ((SH - 1 + j) % 12) + 1
        • offset = 60*j - SM
        • 한 자리 시각이면 선행 공백 1칸을 출력한 뒤 <hour>:XX | 를 출력
        • offset 부호에 따라 XX / XX + k / XX - k를 출력
    • 모든 케이스에 대해 위 과정을 반복해 표를 이어서 출력.

    예시

    • 12 30 5 0:
      • total=300, rows=(30+300)/60+1=6 → 12,1,2,3,4,5 시간대
      • offset: -30, +30, +90, +150, +210, +270 → 예제와 동일
    • 7 0 2 59:
      • total=179, rows=(0+179)/60+1=3 → 7,8,9 시간대
      • offset: 0, +60, +120 → 예제와 동일
    • 9 59 4 1:
      • total=241, rows=(59+241)/60+1=6 → 9,10,11,12,1,2 시간대
      • offset: -59, +1, +61, +121, +181, +241 → 예제와 동일

    시간 복잡도

    각 케이스마다 행 수는 최대 약 11(지속 ≤ 10:59). 매 행은 상수 시간에 출력하므로 O(rows), 전체는 O(N).

    공간 복잡도

    누적 출력을 위한 StringBuilder 정도만 사용하므로 O(1).

    주의

    • 헤더/시각 정렬: 한 자리 시각 앞과 헤더의 time 앞에 공백 1칸이 있어야 한다.
    • 12시간 순환: 12 → 1로 순환하는 12시간제 표기를 반드시 지켜야 한다.
    • 행 개수: rows = (SM + total)/60 + 1 공식으로 끝나는 시각이 정확히 시 경계일 때도 마지막 시간대를 포함한다.
    • 출력 공백: 채점이 엄격할 수 있으므로 헤더 등 후행 공백이 없도록 주의(예: " time | elapsed" 뒤 공백 없이 개행).
    대안 아이디어
    • 직접 경과 분 계산식 유도 — 각 시간대 시작 시각(분)을 절대 분으로 두고 XX를 더해 경과 = (해당 절대 분) - (시작 절대 분) 형태로 매번 재계산해도 되지만, 현재처럼 offset = 60*j - SM을 한 번 구해 표현하는 편이 단순·명확하다.

    포인트 요약

    • rows = (SM + 60·DH + DM)/60 + 1로 행 수 결정
    • hour = ((SH - 1 + j) % 12) + 1, offset = 60j - SM
    • offset 부호에 따라 XX / XX + k / XX - k를 선택
    • 한 자리 시간과 헤더의 time 앞에는 공백 1칸
    • 시간 O(N), 공간 O(1) — 구현 단순, 포맷 정확도 중요
