    문제 4418 — Request for Proposal

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Request for Proposal (RFP) 문제는 각 테스트 케이스마다 요구사항 수 n과 제안 수 p가 주어진다. 각 제안은 “가격(price)”과 “만족한 요구사항 수(success)”를 가지고, 준수도(compliance)는 success / n으로 계산된다. 목표는 준수도가 가장 높은 제안을 우선으로, 동률이면 가격이 더 낮은 제안을, 그것마저 같으면 입력에서 먼저 등장한 제안을 선택하는 것이다.

    • 입력 파싱:
      • n p를 읽고, 0 0이면 종료
      • 요구사항 n줄은 이름만 주어지고 계산에는 사용되지 않으므로 그대로 읽어서 버린다
      • 각 제안은 제안명 1줄 → price와 success → 이어서 success줄의 “만족한 요구사항 이름”을 읽어 처리한다
    • 준수도 계산: comp = success / n
    • 선정 기준(우선순위 순서):
      1. 준수도 내림차순(큰 값 우선)
      2. 가격 오름차순(작은 값 우선)
      3. 입력 순서 오름차순(먼저 입력된 제안 우선)
    • 출력 형식: 각 케이스마다 
      RFP #k
      <선정된 제안명>
      케이스 사이에는 빈 줄 1개를 출력

    핵심: 코드가 하는 일

    • while 루프로 여러 RFP를 처리하며, readLine()이 공백 줄이면 건너뛴다
    • 요구사항 n줄은 단순히 읽고 버린다
    • 각 제안에 대해:
      • 제안명과 price, success를 읽는다
      • success줄의 “만족한 요구사항”은 읽기만 하고 저장하지 않는다
      • comp = success / n 계산
      • 현재 bestComp, bestPrice와 비교해 필요하면 갱신한다 (1e-12 epsilon으로 부동소수 비교 안정화)
    • 케이스 간 빈 줄은 caseNum > 1일 때만 추가해 출력 포맷을 정확히 맞춘다

    예시

    • 예제 입력의 RFP #1에서 Cadillac은 success=4, n=6 → comp ≈ 0.6667. 다른 제안보다 준수도가 가장 높아 선택된다.
    • RFP #2에서는 Starbucks가 유일한 제안이라 자동으로 선정된다.

    시간 복잡도

    요구사항 읽기 O(n), 제안별로 헤더 1줄 + 만족 요구사항 success줄 읽기 → 전체 O(n + Σ(success)). 준수도 비교와 갱신은 상수 시간이다.

    공간 복잡도

    누적 출력을 위한 StringBuilder와 몇 개의 변수만 사용하므로 O(1) (입력을 저장하지 않음).

    주의

    • 부동소수 비교: comp와 price 비교 시 1e-12 epsilon을 둬서 오차로 인한 잘못된 우선순위 판단을 막는다.
    • 입력 공백/빈 줄: 맨 앞에서 공백 줄을 건너뛰는 루프가 있어 변형 입력에도 안정적으로 동작한다.
    • 요구사항 이름: 선정에 직접 쓰이지 않으므로 저장하지 않고 읽기만 하는 편이 효율적이다.
    • 동률 처리: “같은 준수도 & 더 낮은 가격”에서만 갱신하므로, comp와 price가 모두 같을 때는 먼저 입력된 제안이 유지된다(문제 조건과 동일).
    • 출력 포맷: 케이스 사이에 반드시 빈 줄 1개를 추가해야 한다(마지막 개행 포함).
    대안 아이디어
    • 객체 정렬로 일괄 선정 — 모든 제안을 리스트에 담고 정렬 기준을 명시 
      
      import java.io.*;
      import java.util.*;
      
      class Proposal {
          String name;
          double price, comp;
          int idx; // 입력 순서
          Proposal(String name, double price, int success, int n, int idx) {
              this.name = name;
              this.price = price;
              this.comp  = n == 0 ? 0.0 : (double) success / n;
              this.idx   = idx;
          }
      }
      
      public class Main {
          public static void main(String[] args) throws Exception {
              BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
              StringBuilder sb = new StringBuilder();
              final double EPS = 1e-12;
              int tc = 1; String line;
      
              while (true) {
                  do {
                      line = br.readLine();
                      if (line == null) return;
                  } while (line.trim().isEmpty());
      
                  StringTokenizer st = new StringTokenizer(line);
                  int n = Integer.parseInt(st.nextToken());
                  int p = Integer.parseInt(st.nextToken());
                  if (n == 0 && p == 0) break;
      
                  for (int i = 0; i < n; i++) br.readLine();
      
                  List<Proposal> list = new ArrayList<>();
                  for (int i = 0; i < p; i++) {
                      String name = br.readLine();
                      st = new StringTokenizer(br.readLine());
                      double price = Double.parseDouble(st.nextToken());
                      int success = Integer.parseInt(st.nextToken());
                      for (int j = 0; j < success; j++) br.readLine();
                      list.add(new Proposal(name, price, success, n, i));
                  }
      
                  list.sort((a, b) -> {
                      if (Math.abs(b.comp - a.comp) > EPS) return Double.compare(b.comp, a.comp); // 준수도 내림차순
                      if (Math.abs(a.price - b.price) > EPS) return Double.compare(a.price, b.price); // 가격 오름차순
                      return Integer.compare(a.idx, b.idx); // 입력 순서
                  });
      
                  if (tc > 1) sb.append('\n');
                  sb.append("RFP #").append(tc++).append('\n').append(list.get(0).name).append('\n');
              }
              System.out.print(sb);
          }
      }

      정렬 기준을 명시해 가독성을 높이고, 문제의 우선순위를 명확히 표현한다.

    포인트 요약

    • comp = success / n을 기준으로 최대화 → 동률이면 가격 최소화 → 그래도 같으면 입력 순서
    • 요구사항 이름은 선정 로직에 필요하지 않음 → 읽기만 수행
    • 부동소수 오차 방지를 위해 EPS를 사용해 비교의 안정성을 확보
    • 출력 포맷: 케이스 사이에는 빈 줄 1개

    문제 4446 — ROT13

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    ROT13 (BOJ 4446) 문제는 주어진 문자열의 알파벳을 특정 규칙에 따라 치환하는 문제다. 단순한 ROT13과 달리, 모음(a, i, y, e, o, u)과 자음(b, k, x, z, n, h, d, c, w, g, p, v, j, q, t, s, r, l, m, f)을 서로 다른 고리(cyclic group)로 나누어 각각 회전시킨다.

    • 모음 집합: aiyeou → 각 문자를 +3칸 회전
    • 자음 집합: bkxznhdcwgpvjqtsrlmf → 각 문자를 +10칸 회전
    • 대소문자는 그대로 유지한다. (치환 후 원래의 대소문자 형태로 복원)
    • 모음·자음이 아닌 문자는 변경하지 않는다.

    즉, 입력 문자의 종류에 따라 다음과 같이 치환된다.

    • 모음 예시: a → e, i → o, y → u, e → a, o → i, u → y
    • 자음 예시: b → k, k → d, z → r, n → w, ... (10칸씩 회전)

    코드 동작

    • V와 C 문자열을 통해 모음/자음 매핑 집합을 정의한다.
    • change(char c) 함수는 다음 단계를 수행한다:
      • 입력 문자가 대문자인지 여부를 저장(upper)
      • 소문자로 변환해 인덱스를 찾는다.
      • 모음이면 (i + 3) % V.length()로 회전
      • 자음이면 (i + 10) % C.length()로 회전
      • 회전 결과를 대문자/소문자 형태로 복원 후 반환
      • 해당되지 않는 문자는 그대로 반환
    • main()에서는 입력을 한 줄씩 읽어 각 문자를 변환하고 출력 버퍼에 추가한다.

    예시

    • Hello → H(자음→R), e(모음→a), l(자음→m) → Rammi
    • Python → P(자음→V), y(모음→u), t(자음→s) → Vuswot

    시간 복잡도

    입력 문자열의 길이를 L이라 하면, 각 문자를 상수 시간(O(1))에 처리하므로 전체 시간 복잡도는 O(L).

    공간 복잡도

    고정 문자열 V, C와 StringBuilder 버퍼만 사용하므로 O(1).

    주의

    • 대소문자 유지: 치환 후에도 원래 문자의 대소문자를 그대로 유지해야 한다.
    • 문자 범위: 알파벳 이외의 문자는 그대로 출력한다.
    • 인덱스 계산: 음수 회전이 없으므로 단순히 + 연산으로 처리해도 된다.
    대안 아이디어
    • 배열 기반 매핑 — 문자열 탐색(indexOf()) 대신 배열 매핑으로 속도를 개선할 수 있다. 
      
      import java.io.*;
      import java.util.*;
      
      public class Main {
          static final String V = "aiyeou";
          static final String C = "bkxznhdcwgpvjqtsrlmf";
          static final int[] map = new int[26];
      
          static char convert(char c) {
              boolean upper = Character.isUpperCase(c);
              char lower = Character.toLowerCase(c);
      
              int i = V.indexOf(lower);
              if (i >= 0) return (char)(upper ? Character.toUpperCase(V.charAt((i + 3) % V.length())) : V.charAt((i + 3) % V.length()));
      
              i = C.indexOf(lower);
              if (i >= 0) return (char)(upper ? Character.toUpperCase(C.charAt((i + 10) % C.length())) : C.charAt((i + 10) % C.length()));
      
              return c;
          }
      
          public static void main(String[] args) throws Exception {
              BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
              StringBuilder sb = new StringBuilder();
              String line;
              while ((line = br.readLine()) != null) {
                  for (char c : line.toCharArray()) sb.append(convert(c));
                  sb.append('\n');
              }
              System.out.print(sb);
          }
      }

      문자 집합을 배열화하면 indexOf 탐색을 없애서 실행 속도를 약간 높일 수 있다.

    포인트 요약

    • 모음은 +3칸, 자음은 +10칸 회전
    • 대소문자는 그대로 유지
    • 모음·자음이 아닌 문자는 그대로 출력
    • 한 문자 처리 시간 O(1), 전체 O(L)
