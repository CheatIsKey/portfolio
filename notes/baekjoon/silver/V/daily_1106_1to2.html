    문제 4647 — Safecracker

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Safecracker는 목표 정수 target과 (중복 없는) 대문자 집합이 주어졌을 때, 다섯 글자 v, w, x, y, z를 골라 아래 식을 만족시키는 조합 vwxyz를 찾는 문제다. 각 글자는 알파벳 순서를 값으로 매핑한다(A=1, B=2, …, Z=26). 

    v - w² + x³ - y⁴ + z⁵ = target

    가능한 해가 여러 개면 사전식으로 가장 뒤(내림차순 기준 가장 큰) 조합을 출력하고, 없으면 no solution을 출력한다. 입력은 여러 줄로 주어지며, 마지막 줄 0 END가 종료 신호다.

    • 핵심 정의:
      • 각 문자는 한 번만 사용(입력 글자들이 모두 서로 다름).
      • 목표: 식을 만족하는 다섯 글자의 사전식 최댓값 조합.
      • 사전식 비교는 평범한 문자열 비교와 같다(예: ZYYXX > ZYYXW).
    • 전략 개요:
      • 입력 문자들을 내림차순으로 정렬.
      • 백트래킹(DFS)으로 v→w→x→y→z 순서로 한 글자씩 선택.
      • 선택 시 매 단계 식의 부호와 거듭제곱을 즉시 누적해 목표와 일치하는지 검사.
      • 내림차순 순회이므로 첫 해를 찾는 즉시 그것이 사전식 최댓값이 된다 → 즉시 종료.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 각 줄에서 target과 대문자 문자열 alpha를 읽는다.
      • target==0 && alpha.equals("END")면 종료.
    • 문자 준비(내림차순):
      • array = alpha.toCharArray() → Arrays.sort()로 오름차순 정렬 후 앞뒤를 뒤집어 내림차순으로 만든다.
      • 내림차순으로 DFS를 돌리면 가장 큰(사전식 최댓값) 해를 가장 먼저 만난다.
    • 거듭제곱 전처리:
      • 각 문자 값 val ∈ [1..26]에 대해 val¹, val², val³, val⁴, val⁵를 미리 계산해 pows[5][n]에 저장.
      • 계산 시 매번 거듭제곱을 하지 않고 상수 시간 조회.
    • DFS(백트래킹):
      • dfs(i, sum, sb)에서 i는 선택한 글자 개수(0~5).
      • SIGN = {+1,-1,+1,-1,+1}로 단계별 부호를, pows[i][j]로 대응 거듭제곱을 적용해 next = sum + SIGN[i] * pows[i][j]를 누적.
      • 깊이 5에서 sum == target이면 정답 문자열을 기록하고 found=true로 즉시 종료.
      • used[j]로 중복 선택 방지, StringBuilder로 경로 누적/복원.
      • long을 사용해 거듭제곱 누적 시 오버플로를 피한다.
    • 출력:
      • 해를 찾았으면 그 조합, 아니면 no solution을 출력.

    예시

    • 입력: 1 ABCDEFGHIJKL → 가능한 해가 여럿이며 LKEBA가 사전식 최댓값.
    • 입력: 11700519 ZAYEXIWOVU → YOXUZ.
    • 입력: 3072997 SOUGHT → GHOST.
    • 입력: 1234567 THEQUICKFROG → no solution.

    시간 복잡도

    글자 수를 n(5 ≤ n ≤ 12)라 하면, 탐색 공간은 P(n,5) = n·(n-1)·(n-2)·(n-3)·(n-4).
    최악의 경우 P(12,5)=95,040으로 매우 작고, 각 단계 연산은 전처리 덕분에 상수 시간에 가깝다.

    공간 복잡도

    used[n], pows[5][n], 재귀 깊이 최대 5 및 경로용 StringBuilder로 총합 O(n).

    주의

    • 사전식 최댓값 보장: 문자를 내림차순으로 순회하고 첫 해에서 즉시 종료해야 한다.
    • 부호·지수 매핑: 각 깊이에서 {+1,-1,+1,-1,+1}와 {1,2,3,4,5}의 대응을 혼동하지 말 것.
    • 오버플로: z⁵까지 포함되므로 누적형은 long을 사용(코드와 동일).
    • 중복 금지: 입력 문자는 모두 서로 다르지만, 선택 시 used[]로 재사용을 막아야 한다.
    • 조기 종료 플래그: found로 DFS 가지치기(이미 정답을 찾았으면 더 이상 탐색하지 않음).
    대안 아이디어
    • 휴리스틱 가지치기: 남은 단계에서 만들 수 있는 최대/최소 기여(내림차순/오름차순 선택과 부호 반영)를 상한·하한으로 계산해 target 도달 불가능한 가지를 미리 컷. 본 제약에선 필수는 아님.
    • 비트마스크 DFS: used를 비트마스크로 바꾸고 반복문을 줄여 미세 최적화.

    포인트 요약

    • 문자 집합을 내림차순 정렬 → DFS로 첫 해 = 사전식 최댓값.
    • 전처리로 각 문자의 1~5제곱을 저장, 단계별 부호를 적용해 누적.
    • 해를 찾는 즉시 조기 종료, 없으면 no solution.
    • 시간 O(P(n,5)), 공간 O(n) — 제약 내 충분히 빠름.

    문제 4649 — Tanning Salon

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Tanning Salon은 침대 수 N과 고객 입퇴장 로그(대문자 시퀀스)가 주어질 때, 침대가 없어 돌아간(walked away) 고객 수를 구하는 시뮬레이션 문제다. 각 문자는 정확히 두 번 등장하며, 첫 등장은 도착, 두 번째 등장은 퇴장을 의미한다. 침대가 없을 때 도착한 고객은 즉시 돌아가며, 그런 고객의 두 번째 등장은 단순히 로그 상의 퇴장 표시일 뿐이므로 무시한다.

    • 핵심 정의:
      • N개의 침대(최대 20), 로그 문자열의 길이는 문자쌍 수의 2배.
      • 침대가 비어 있으면 도착 고객을 즉시 배정, 없으면 walk away로 카운트.
      • 이미 돌아간 고객의 두 번째 문자는 아무 효과 없음(문제 조건: 돌아간 고객은 현재 이용 중인 고객보다 먼저 퇴장 처리됨).
    • 전략 개요:
      • 현재 사용 중인 고객 집합 use와 이미 돌아간 고객 집합 walk를 관리.
      • 로그를 왼쪽부터 순차 스캔하며 문자 c에 대해
        1. walk에 있으면 무시;
        2. use에 있으면(퇴장) use에서 제거하고 free++;
        3. 둘 다 아니면(도착) free>0이면 use에 넣고 free--, 아니면 walk에 넣고 walkAway++.
      • 모든 문자를 처리한 뒤 결과 문구를 정확한 형식으로 출력.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 각 줄에서 N과 log를 읽는다. N==0이면 종료.
    • 상태 구조:
      • Set<Character> use: 현재 침대를 쓰는 고객들.
      • Set<Character> walk: 도착했지만 침대를 못 받아 돌아간 고객들.
      • int free = N: 남은 침대 수, int walkAway: 돌아간 인원 수.
    • 로그 처리:
      • 문자 c를 순회하며, walk.contains(c)면 패스(이미 돌아간 고객의 퇴장 토큰).
      • use.contains(c)면 퇴장 처리(use.remove(c), free++).
      • 그 외(도착): free>0이면 use.add(c), free--, 아니면 walk.add(c), walkAway++.
    • 출력:
      • walkAway==0이면 All customers tanned successfully.
      • 아니면 {walkAway} customer(s) walked away.

    예시

    • N=2, log=ABBAJJKZKZ → All customers tanned successfully.
    • N=3, log=GACCBDDBAGEE → 1 customer(s) walked away.
    • N=3, log=GACCBGDDBAEE → All customers tanned successfully.
    • N=1, log=ABCBCA → 2 customer(s) walked away.

    시간 복잡도

    로그 길이를 L이라 하면 각 문자에 대해 HashSet 연산(평균 O(1))을 한 번 수행하므로 전체 O(L). L은 최대 알파벳 종류 수(≤26)의 두 배 범위 내에서 현실적으로 매우 작다.

    공간 복잡도

    use와 walk에 각 문자가 한 번씩만 들어가므로 O(Σ)(Σ≤26), 상수급.

    주의

    • 중복 처리: 이미 walk에 있는 고객의 두 번째 등장은 반드시 무시해야 한다.
    • 퇴장 처리 순서: 문제 보장(돌아간 고객의 퇴장은 현재 이용 중인 고객보다 먼저) 덕분에 walk 체크를 use보다 먼저 수행하면 모순이 없다.
    • 상태 일관성: use에 있는 고객만 퇴장 시 free++ 해야 한다.
    • 출력 형식: 문구와 구두점을 정확히 맞출 것(채점은 문자열 일치).
    대안 아이디어
    • 고정 배열 사용: 문자 집합이 대문자 26개로 한정되므로 boolean[26] 두 개(use/walk)로 교체 가능 → 상수 인자 감소.
    • 카운터 한정: N이 작으므로 free==0인 구간만 빠르게 스킵하는 최적화도 가능하지만 이득은 미미.

    포인트 요약

    • use(이용 중) / walk(돌아감) 집합과 free 카운터로 선형 시뮬레이션.
    • 도착 시: 여유 있으면 배정, 없으면 walkAway++ 및 walk 표시.
    • 퇴장 시: use에 있을 때만 제거 및 free++.
    • 형식에 맞춰 결과 문구 출력. 시간 O(L), 공간 상수급.
