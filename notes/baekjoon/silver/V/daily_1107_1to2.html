    백준 알고리즘 Silver V - Daily 문제 묶음

    작성일 2025-11-07 · 분류: 코딩 테스트(백준 알고리즘), Silver V

    문제 4673 — 셀프 넘버

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    셀프 넘버(BOJ 4673)는 양의 정수 n에 대해 생성 함수 d(n)를 n과 각 자리수의 합으로 정의하고, 어떤 수가 어느 수의 생성자도 갖지 않으면 그 수를 셀프 넘버라고 부른다. 이 문제는 10,000 이하의 모든 셀프 넘버를 오름차순으로 한 줄에 하나씩 출력하는 과제다. 입력은 없다.

    • 예: d(75) = 75 + 7 + 5 = 87, 101의 생성자는 91, 100 등.

    전략 개요

    • 표시 배열을 사용한다: generated[x] = true면 어떤 수의 d 체인을 통해 x가 만들어졌음을 의미.
    • i = 1..10000에 대해 n = d(i)를 계산하고, n ≤ 10000이면 generated[n] = true.
    • 최종적으로 generated[i] == false인 모든 i를 출력하면 그것이 곧 셀프 넘버.

    핵심: 코드가 하는 일

    • 보조 함수 d(int i):
      • sum = i로 시작해, i의 각 자리수를 sum에 누적(i % 10, i /= 10 반복).
      • 결과 sum을 반환.
    • 생성된 수 표시:
      • for i = 1..10000: n = d(i)를 얻고, n ≤ 10000이면 generated[n] = true.
    • 셀프 넘버 출력:
      • for i = 1..10000: !generated[i]인 모든 i를 한 줄씩 println.
    • I/O 및 자료구조:
      • boolean[10001]로 1~10000 인덱스를 직관적으로 매핑(0 미사용).
      • BufferedReader는 입력이 없으므로 실사용은 없으며, 출력은 System.out.println.

    예시

    • 초기 구간 출력(일부): 1, 3, 5, 7, 9, 20, 31, 42, 53, 64, 75, 86, 97, ...
    • 마지막 부근(일부): ... 9903, 9914, 9925, 9927, 9938, 9949, 9960, 9971, 9982, 9993

    시간 복잡도

    d(i)는 자릿수 길이(최대 5자리)에 비례하므로 각 i에 대해 O(1)로 취급 가능. 전체는 i = 1..10000 한 번 순회하므로 O(10000).

    공간 복잡도

    표시 배열 boolean[10001]만 사용하므로 O(10000) (= 상수 배수의 선형 공간).

    주의

    • 경계 처리: n = d(i)가 10000을 초과하면 표시 배열에 기록하지 않는다(if (n ≤ 10000)).
    • 인덱스: 배열 크기는 10001로 선언해 1..10000를 인덱스로 직접 사용(0은 미사용).
    • I/O: 출력 줄 수가 많다. println 반복 제출도 통과하지만, 성능 미세 개선을 원하면 StringBuilder에 누적 후 한 번에 출력하는 편이 낫다. (현재 제시 코드의 sb는 사용되지 않으므로 제거하거나 실제로 활용하도록 정리 권장.)
    • 입력 없음: BufferedReader는 필수 아님. 깔끔함을 위해 제거 가능.
    대안 아이디어
    • 직접 체인 전개 불필요: 어떤 수의 모든 연쇄 d(d(...))를 전개할 필요 없이, 한 번의 i = 1..10000 순회로 충분.
    • 수학적 판정식: 셀프 넘버를 닫힌형으로 판정하는 간단 식은 알려져 있지 않으므로 표시 배열 방식이 가장 명확하고 구현이 간단.

    포인트 요약

    • d(n)로 생기는 수를 표시하고, 끝에 비표시 인덱스만 출력하면 정답.
    • 경계 10000 이하만 기록(if (n ≤ 10000)).
    • 시간 O(10000), 공간 O(10000) — 문제 제약에 충분.
    • 입력 없음, 출력 형식은 숫자만 줄바꿈.

    문제 4675 — Word Amalgamation

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Word Amalgamation(BOJ 4675)는 주어진 사전(dictionary)과 뒤섞인 단어들(scrambled words)에 대해, 각 뒤섞인 단어의 철자를 재배열해서 만들 수 있는 사전 내 단어들을 모두 찾아 사전순으로 출력하는 문제다.

    • 입력 구성:
      • 1부: 사전 단어들(최소 1개, 최대 100개), 한 줄에 하나.
      • 2부: XXXXXX 한 줄 → 사전 종료 신호.
      • 3부: 해독해야 할 뒤섞인 단어들, 한 줄에 하나.
      • 4부: 다시 XXXXXX 한 줄 → 전체 입력 종료.
    • 단어 조건:
      • 모두 소문자 알파벳, 길이 1~6.
      • 사전 단어는 중복 없음.

    전략 개요

    • 아이디어 핵심: 애너그램은 정렬된 결과가 동일하다는 성질을 사용.
    • 각 쿼리 단어 q에 대해:
      • q의 문자들을 정렬한 결과를 기준 패턴으로 사용.
      • 모든 사전 단어 d에 대해:
        • 길이가 다르면 패스.
        • 길이가 같으면 문자들을 정렬하여 비교.
        • 정렬 결과가 같으면 q의 유효한 원 단어로 채택.
      • 찾은 단어들을 알파벳 순으로 정렬해 출력.
      • 하나도 없으면 "NOT A VALID WORD" 출력.
      • 각 쿼리 처리 후 항상 "******" 출력.

    핵심: 코드가 하는 일

    • 입력 처리:
      • dict: 사전 단어 저장 리스트.
      • queries: 뒤섞인 단어 저장 리스트.
      • next 플래그:
        • 0일 때는 사전 구간.
        • XXXXXX를 만나면 next = 1로 전환.
        • next == 1 상태에서 다시 XXXXXX를 만나면 입력 종료.
    • 쿼리 처리 루프(for (String q : queries)):
      • 현재 쿼리 q의 문자 배열을 정렬하여 기준 패턴 arr1 생성.
      • 사전 단어 전체를 순회하며:
        • 길이 다르면 continue.
        • 같으면 문자 배열을 정렬해 arr2 생성.
        • Arrays.equals(arr1, arr2)면 애너그램이므로 결과 리스트에 추가.
      • 결과 리스트가 비었으면:
        • "NOT A VALID WORD"를 sb에 추가.
      • 비어있지 않으면:
        • Collections.sort(list)로 알파벳 순 정렬.
        • 각 단어를 한 줄씩 sb에 추가.
      • 각 쿼리 처리 끝에 "******" 한 줄 추가.
    • 출력:
      • 모든 결과를 StringBuilder에 모았다가 마지막에 한 번에 System.out.print.

    예시

    • q = "resco":
      • 정렬 → "ceors".
      • 사전에서 "score" 정렬 결과와 일치 → 출력.
      • 결과: 
        score
      ******
    • q = "sett":
      • 일치하는 사전 단어 없음.
      • 결과: 
        NOT A VALID WORD
      ******

    시간 복잡도

    사전 크기를 D (≤ 100), 쿼리 개수를 Q, 최대 길이를 L (≤ 6)라 하면:

    • 각 쿼리마다: 모든 사전 단어에 대해 길이 ≤ 6인 문자열 정렬 및 비교 → O(D · L log L) ≒ 상수 수준.
    • 전체: O(Q · D) 규모로, 문제 제한 내에서 충분히 빠르다.

    공간 복잡도

    • 사전 리스트, 쿼리 리스트, 임시 리스트 정도만 사용 → O(D + Q).
    • 각 비교 시 사용하는 문자 배열은 최대 길이 6으로, 추가 공간은 매우 작다.

    주의

    • 대소문자 구분: 입력 단어는 모두 소문자, 구분자 XXXXXX는 대문자이므로 정확히 비교해야 한다.
    • 출력 형식:
      • 각 쿼리 결과 블록 뒤에 반드시 ****** 출력.
      • 여러 정답 단어가 있으면 알파벳 순으로 정렬해서 한 줄씩 출력.
      • 정답이 없으면 NOT A VALID WORD 한 줄만 출력 후 ******.
    • 정렬 비교 방식: 매번 문자열을 정렬해 비교하므로 구현은 단순하지만, 코드에서처럼 q.length() != d.length() 체크로 불필요한 정렬을 먼저 줄여준다.
    대안 아이디어
    • 미리 매핑 구성:
      • 사전 단어마다 정렬된 문자열을 key로 하는 Map<String, List<String>> 생성.
      • 쿼리도 정렬해 key로 lookup하면 O(1) 수준에 가까운 검색 가능.
      • 입력 크기가 작아서 필수는 아니지만, 구조적으로는 더 깔끔한 방법.

    포인트 요약

    • 애너그램 판정: 정렬된 문자열이 같으면 같은 단어 조합.
    • 각 쿼리에 대해 사전 전체를 확인해 가능한 원 단어를 모은 뒤 정렬 출력.
    • 없으면 NOT A VALID WORD, 이후 항상 ****** 출력.
    • 구현은 단순 정렬+비교로 충분, 시간·공간 모두 여유롭다.
