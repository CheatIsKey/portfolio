    문제 4659 — 비밀번호 발음하기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비밀번호 발음하기(BOJ 4659)는 주어진 소문자 문자열이 아래의 규칙을 모두 만족하면 acceptable, 아니면 not acceptable로 판정하는 문제다. 입력은 여러 줄이며 마지막 줄 end가 종료 신호다.

    • 규칙
      • 모음(a,e,i,o,u)을 적어도 하나 포함.
      • 모음 3개 연속 또는 자음 3개 연속 금지.
      • 같은 글자 연속 2번 금지. 단, ee, oo는 예외로 허용.

    전략 개요

    • 각 줄(패스워드)에 대해 왼쪽→오른쪽으로 한 글자씩 스캔한다.
    • 상태를 유지한다:
      • hasVowel: 지금까지 모음을 하나라도 봤는지.
      • vow, cons: 현재 구간의 연속 모음/자음 개수.
      • prev: 직전 문자(연속 중복 체크).
    • 매 글자 처리:
      • 연속 중복이면 ee/oo만 허용, 그 외는 즉시 실패.
      • 모음이면 hasVowel=true, vow++, cons=0.
      • 자음이면 cons++, vow=0.
      • vow ≥ 3 또는 cons ≥ 3가 되면 즉시 실패.
    • 스캔 종료 후 ok && hasVowel이면 acceptable, 아니면 not acceptable.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 한 줄씩 읽어 end면 종료, 아니면 판정 수행.
    • 보조 함수:
      • isVowel(c): a,e,i,o,u 판정.
    • 상태 변수:
      • hasVowel, ok, vow, cons, prev.
    • 연속 규칙 처리:
      • 중복 문자: prev == curr이면 curr이 e 또는 o일 때만 통과.
      • 연속 3개: 모음/자음 카운트를 각각 관리하고 반대 분류를 만나면 0으로 리셋.
    • 출력:
      • 형식: <password> is acceptable. 또는 <password> is not acceptable.

    예시

    • 입력: a → <a> is acceptable. (모음 포함, 규칙 위반 없음)
    • 입력: tv → <tv> is not acceptable. (모음 없음)
    • 입력: wiinq → <wiinq> is not acceptable. (ii 연속 중복이지만 ee/oo가 아님)
    • 입력: eep → <eep> is acceptable. (ee 허용, 모음 존재, 연속 3개 없음)
    • 입력: houctuh → <houctuh> is acceptable. (모음 존재, 연속 3개 없음, 불허 중복 없음)

    시간 복잡도

    각 패스워드 길이를 L (1 ≤ L ≤ 20)이라 하면 한 번의 선형 스캔으로 O(L). 입력 전체에 대해 Σ O(L).

    공간 복잡도

    고정 크기 보조 상태(카운터와 직전 문자)만 사용하므로 O(1).

    주의

    • ee, oo만 예외로 허용한다(다른 이중문자 불가).
    • 모음·자음 연속 카운트는 서로 독립이며, 반대 분류를 만나면 각각 0으로 리셋해야 한다.
    • 하나라도 모음을 포함해야 한다(hasVowel 체크 필수).
    • 위반을 발견하면 즉시 종료해 불필요한 연산을 줄인다.
    대안 아이디어
    • 슬라이딩 윈도: 길이 3 윈도를 굴리며 모음/자음 개수만 세어도 연속 3개 판정은 가능(다만 이중문자 규칙은 별도 처리 필요).
    • 정규식 보조: (?<!e)e(?!e) 같은 복잡한 패턴은 과하고 가독성이 떨어짐 — 본 문제는 단순 선형 스캔이 가장 명료.

    포인트 요약

    • 모음 존재(hasVowel) + 연속 3개 금지 + 이중문자 예외(ee/oo)만 허용.
    • 한 번의 선형 스캔으로 상태(vow/cons/prev) 관리.
    • 위반 발견 시 조기 종료, 최종 판정 후 형식 그대로 출력.
    • 시간 O(L), 공간 O(1) — 제약 내 충분히 빠름.

    문제 4670 — Booklet Printing

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Booklet Printing(BOJ 4670)는 총 페이지 수 n이 주어졌을 때, 접지 소책자(Booklet)로 인쇄하기 위한 시트별 앞/뒤 인쇄 순서를 생성하는 문제다. 한 시트에는 앞면 2페이지, 뒷면 2페이지로 총 4페이지가 배치된다. 소책자는 모든 시트를 포개어 반으로 접으면 정상적인 1,2,3,… 순서로 보이게 된다.

    • 조건
      • 각 테스트케이스는 정수 n (1 ≤ n ≤ 100).
      • 시트를 가득 채우지 못하는 칸은 Blank로 표시.
      • 어떤 면(앞/뒤)이 둘 다 Blank라면 그 줄은 출력하지 않음.
      • 시트 번호 오름차순, front 먼저, 이어서 back을 출력.

    전략 개요

    • 필요 시트 수: paper = ceil(n / 4).
    • 시트별 배치 규칙(시트 i = 1..paper):
      • 왼쪽 큰 번호부터 내려가는 포인터 left = paper*4, 오른쪽 작은 번호부터 올라가는 포인터 right = 1를 사용.
      • 앞면(front): (left, right).
      • 뒷면(back): (right+1, left-1).
      • 다음 시트로 넘어갈 때 left -= 2, right += 2.
    • 페이지 번호가 범위를 벗어나면(<1 또는 >n) 그 칸은 Blank.

    핵심: 코드가 하는 일

    • 입력 파싱:
      • 한 줄씩 n을 읽고, n==0이면 종료.
      • 테스트케이스 사이에 한 줄 공백을 출력(문제 예시 형식 일치).
    • 보조 함수:
      • label(page, n): 1 ≤ page ≤ n이면 숫자 문자열, 아니면 "Blank" 반환.
    • 시트 생성 루프:
      • 시트 i마다 앞면/뒷면의 좌/우 페이지를 label로 문자열화.
      • 앞·뒤 한 면이 Blank, Blank라면 해당 줄은 건너뜀.
      • 출력 라벨은 정확히 Sheet i, front: / Sheet i, back : (공백 포함) 형식 준수.

    예시

    • n = 1
      • Sheet 1, front: Blank, 1 (back은 Blank, Blank → 생략)
    • n = 4
      • Sheet 1, front: 4, 1
      • Sheet 1, back : 2, 3
    • n = 14
      • Sheet 1, front: Blank, 1
      • Sheet 1, back : 2, Blank
      • Sheet 2, front: 14, 3 … (문제 예시와 동일 패턴)

    시간 복잡도

    시트 수는 paper = ⌈n/4⌉ (≤ 25), 각 시트에서 상수 작업이므로 전체 O(paper) = O(n).

    공간 복잡도

    고정 크기 변수(left, right 등)만 사용하므로 O(1).

    주의

    • 출력 포맷의 공백·쉼표·콜론 위치(특히 back :의 공백)까지 예시와 동일해야 한다.
    • 면 전체가 Blank, Blank이면 해당 줄은 출력하지 않는다.
    • left는 큰 수에서 2씩 감소, right는 1에서 2씩 증가 — 업데이트 순서를 혼동하지 말 것.
    • 테스트케이스 사이에 빈 줄 한 줄 출력(예시 형식 일치).
    대안 아이디어
    • 배열 시뮬레이션: 크기 paper×2×2의 시트-면-칸 배열에 직접 채운 뒤 범위 밖은 Blank로 채워 출력.
    • 큐/스택 모델: 좌우 포인터 대신 [1..n]를 큐로, [4·paper..1]를 스택으로 두고 앞면/뒷면 순서대로 꺼내 배치(본 문제 규모에선 포인터 방식이 가장 단순).

    포인트 요약

    • 필요 시트 수 paper=⌈n/4⌉, 포인터 left=4·paper, right=1로 시작.
    • 앞면: (left, right), 뒷면: (right+1, left-1).
    • 범위 밖은 Blank, Blank, Blank면 해당 줄 생략.
    • 시간 O(n), 공간 O(1) — 간단하고 빠름.
