    문제 4187 — Scrolling Sign

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Scrolling Sign (BOJ 4187)는 길이 k인 전광판에 길이 k인 단어 w개를 주어진 순서대로 모두 한 번 이상 나타나게 만들기 위해, 오른쪽에서 흘려 넣어야 하는 문자 수의 최솟값을 구하는 문제다.

    • 전광판은 항상 k칸이며, 매 단계마다 모든 문자가 왼쪽으로 한 칸 밀리고, 오른쪽에 새 문자가 하나 들어온다.
    • 중간에 다른 단어나 의미 없는 문자열이 나와도 상관 없고, 각 주어진 단어가 전광판에 완전히 표시되는 순간이 존재하면 된다.
    • 핵심은 앞 단어의 뒷부분과 다음 단어의 앞부분이 겹치는 만큼 문자를 재사용할 수 있다는 점이다.

    전략 개요

    • 첫 번째 단어를 만들기 위해서는 반드시 k글자를 넣어야 한다.
    • 이후 각 인접한 단어 쌍 (word[j], word[j+1])에 대해:
      • word[j]의 접미사와 word[j+1]의 접두사가 얼마나 긴 길이까지 일치하는지(최대 겹침)를 구한다.
      • 겹침 길이를 overlap이라 하면, 새로 넣어야 할 문자는 k - overlap개.
    • 모든 쌍의 overlap 합을 totalOverlap이라 할 때, 전체 필요한 문자 수 = k * w - totalOverlap.

    핵심: 코드가 하는 일

    • 입력 처리:
      • 첫 줄에서 테스트 케이스 개수 n을 읽는다.
      • 각 테스트 케이스마다 k, w를 읽고, 길이 k인 단어를 w개 입력받아 배열 word[]에 저장한다.
    • 인접 단어 간 최대 겹침 계산:
      • totalOverlap = 0으로 초기화.
      • 각 j = 0..w-2에 대해 s1 = word[j], s2 = word[j+1].
      • l = 1..k에 대해:
        • s1의 뒤에서 l글자(s1[k-l..k-1])와 s2의 앞 l글자(s2[0..l-1])가 모두 같은지 확인.
        • 모두 일치하면 overlap = l로 갱신(최대 겹침 유지).
      • 얻은 overlap을 totalOverlap에 누적.
    • 정답 계산 및 출력:
      • 각 테스트 케이스마다 (k * w) - totalOverlap을 StringBuilder에 추가하고, 마지막에 한 번에 출력.
    • I/O 구현:
      • BufferedReader, StringTokenizer 사용으로 빠른 입력 처리.
      • 여러 테스트 케이스를 순서대로 처리하며, 줄마다 하나의 정답을 출력.

    예시

    • 예제 1: k = 3, 단어: CAT, TED
      • 총 길이 합: k * w = 6.
      • CAT의 접미사와 TED의 접두사 최대 겹침: "T" (길이 1).
      • totalOverlap = 1 → 정답 6 - 1 = 5.
    • 예제 2: k = 3, 단어: CAT, ATE, TEA
      • CAT & ATE: 최대 겹침 "AT" (2) → k - 2 = 1글자 추가.
      • ATE & TEA: 최대 겹침 "TE" (2) → k - 2 = 1글자 추가.
      • totalOverlap = 2 + 2 = 4, k * w = 9 → 정답 9 - 4 = 5.

    시간 복잡도

    각 인접 단어 쌍마다 길이 k까지의 겹침을 검사하고, 각 길이에 대해 최대 k문자를 비교하므로 한 쌍당 O(k^2). 쌍은 w - 1개이므로 전체는 O(w * k^2). (k, w ≤ 100이므로 충분히 빠르다.)

    공간 복잡도

    길이 k인 단어 w개를 저장하는 배열만 사용하므로 O(w * k).

    주의

    • 최대 겹침을 사용해야 한다. 작은 겹침을 찾았더라도 더 큰 길이가 일치하면 그걸로 갱신해야 한다.
    • 인덱스 계산:
      • s1.charAt(k - l + m): 접미사의 시작 인덱스에서 m만큼 이동.
      • s2.charAt(m): 접두사의 m-번째 문자.
      • l 범위를 1..k로 유지해 인덱스 범위 이탈을 방지.
    • 여러 테스트 케이스에서 totalOverlap와 단어 배열을 매번 새로 초기화해야 한다.
    • 입력 형태에 맞게 첫 줄의 테스트 케이스 개수만 믿고 반복문을 도는 구현이 더 직관적이다.
    대안 아이디어
    • KMP 등의 문자열 알고리즘으로 접미사-접두사 최대 겹침을 더 효율적으로 구할 수 있지만, 이 문제의 제약에선 단순 이중 루프 구현으로 충분하다.
    • 실제 전광판 상태를 문자 단위로 시뮬레이션하는 방식은 불필요하게 복잡하므로, 단어 간 최대 겹침만 계산하는 현재 방식이 가장 간단하고 명확하다.

    포인트 요약

    • 인접한 단어 쌍마다 접미사-접두사 최대 겹침 길이를 구한다.
    • 각 쌍에서 새로 넣어야 하는 문자는 k - overlap.
    • 전체 정답은 k * w - Σ(overlap) 형태로 계산된다.
    • 구현은 단순 반복문과 문자 비교만으로 충분하며, 제약 내에서 안전하다.

    문제 4676 — Haiku Review

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Haiku Review (BOJ 4676)는 한 줄에 주어진 하이쿠(슬래시 '/'로 3줄 구분)가 규칙적인 음절 수 5 / 7 / 5를 만족하는지 판정하는 문제다.

    • 각 입력 줄에는 정확히 두 개의 '/'가 있어, 이를 기준으로 3행으로 나뉜다.
    • 모든 입력은 소문자 알파벳, 공백, 슬래시만 포함하며, 마지막 줄 "e/o/i"를 만나면 종료한다.
    • 모든 행은 최소 한 단어를 포함하고, 단어는 공백 또는 슬래시로 구분된다.

    전략 개요

    • 한 줄(하나의 행)에 대해, 연속된 모음들의 묶음 하나를 1 음절로 센다.
    • 모음은 a, e, i, o, u, y 여섯 글자.
    • 즉, 자음 → 모음으로 바뀌는 지점마다 음절 수를 1 증가시키면 된다.
    • 각 행의 음절 수를 구한 뒤:
      • 1행이 5가 아니면 '1'
      • 그렇지 않고 2행이 7이 아니면 '2'
      • 그렇지 않고 3행이 5가 아니면 '3'
      • 모두 맞으면 'Y'
      를 출력한다 (처음으로 규칙을 어기는 행 번호만 출력).

    핵심: 코드가 하는 일

    • 모음 판별 함수 isVowel(char c):
      • a, e, i, o, u, y 중 하나면 true, 아니면 false.
    • 입력 처리:
      • while ((line = br.readLine()) != null)로 줄 단위 입력.
      • 줄이 "e/o/i"와 정확히 같으면 입력 종료.
    • 행 분리:
      • String[] split = line.split("/");로 3개의 문자열(각 행)을 얻는다.
    • 각 행의 음절 수 세기:
      • 각 인덱스 idx = 0..2에 대해:
        • s = split[idx], cnt = 0, prev = false로 시작.
        • 문자 c를 순회하며:
          • cur = isVowel(c)로 현재 문자가 모음인지 확인.
          • cur == true이면서 prev == false이면 새로운 모음 구간 시작 → cnt++.
          • prev = cur 갱신.
        • 완료된 cnt를 n[idx]에 저장.
    • 조건 검사 및 출력 문자 결정:
      • 순서대로 검사:
        • n[0] != 5 → '1'
        • else n[1] != 7 → '2'
        • else n[2] != 5 → '3'
        • else → 'Y'
      • 각 결과 뒤에 개행을 붙여 StringBuilder에 누적 후 마지막에 한 번에 출력.

    예시

    • happy purple frog/eating bugs in the marshes/get indigestion
      • 각 행의 모음 구간 수: 5 / 7 / 5 → Y.
    • computer programs/the bugs try to eat my code/i will not let them
      • 실제 규칙(연속 모음)을 적용하면 2행이 7이 아님 → 2.
    • a e i o u/this is seven syllables/a e i o u y
      • 3행의 모음 구간 수가 5가 아님 → 3.

    시간 복잡도

    각 줄에서 문자열 길이를 L라 하면, 모음 구간을 세기 위해 한 번 순회하므로 O(L). 입력 전체에 대해 각 줄을 한 번씩만 처리하므로 선형 시간.

    공간 복잡도

    한 줄 문자열과 3행의 음절 수(작은 배열)만 사용하므로 O(1) 추가 공간.

    주의

    • 연속된 모음은 하나의 음절로 친다: 예) "ai", "ou"는 각각 1음절.
    • 자음 → 모음으로 바뀌는 지점에서만 카운트 증가시키는 방식이므로 이 규칙이 자연스럽게 구현된다.
    • split("/") 결과는 항상 길이 3임이 보장되므로 추가 검증은 필요 없다.
    • 종료 조건 "e/o/i"는 출력하지 않고 곧바로 루프에서 탈출해야 한다.
    대안 아이디어
    • 정규식을 사용해 [aeiouy]+ 패턴으로 모음 구간을 직접 세는 방법도 가능하지만, 현재 구현처럼 한 글자씩 순회하는 방식이 가장 직관적이고 빠르다.

    포인트 요약

    • 각 행을 슬래시로 분리해 3부분으로 나눈다.
    • 모음 집합 {a,e,i,o,u,y}에 대해, 연속된 모음 묶음을 1음절로 센다.
    • 음절 수가 5/7/5인지 순서대로 검사해, 처음 어긋나는 행 번호 또는 Y를 출력한다.
    • 구현은 한 줄 순회만으로 끝나는 단순 문자열 처리 문제.
