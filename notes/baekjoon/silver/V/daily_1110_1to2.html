    문제 4679 — The Snail

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    The Snail (BOJ 4679)는 달팽이가 하루 동안 낮에 올라가고, 밤에 미끄러지는 과정을 반복할 때, 피로도로 인해 점점 줄어드는 상승 거리까지 고려하여 언제 우물을 탈출하는지, 혹은 언제 바닥 아래로 미끄러져 실패하는지를 시뮬레이션하는 문제다.

    • 우물 높이: H
    • 첫째 날 낮에 올라갈 수 있는 거리: U
    • 매일 밤 미끄러지는 거리: D
    • 피로도 계수: F (%)
      • 둘째 날부터 매일 낮에 올라갈 수 있는 거리가 U × (F / 100)씩 줄어든다.
      • 단, 절대 음수로 내려가지 않으며, 0보다 작아지면 그날은 더 이상 오르지 않는다.
    • 하루는 항상 낮 → 밤 순서로 진행된다.
    • 낮에 올라간 직후 높이가 H를 초과하면 즉시 성공, 밤에 미끄러진 후 높이가 0 미만이 되면 실패.

    전략 개요

    • 이 문제는 복잡한 수식 없이, 조건에 맞게 하루 단위로 그대로 시뮬레이션하면 된다.
    • 각 테스트 케이스에 대해 다음 상태를 관리한다:
      • day: 현재가 몇 번째 날인지
      • height: 현재 달팽이의 높이 (실수형)
      • currentClimb: 해당 날 낮에 오를 수 있는 거리
      • fatigue: 매일 감소하는 오를 수 있는 거리 (U * (F / 100.0))
    • 각 날에 대해 다음 순서로 처리한다:
      1. 날짜 증가: day++
      2. 낮: 상승
        • currentClimb > 0 이면 height += currentClimb
        • 이때 height > H면 "success on day day" 출력 후 종료.
      3. 밤: 하강
        • height -= D
        • 이후 height < 0이면 "failure on day day" 출력 후 종료.
      4. 피로도 반영
        • currentClimb -= fatigue
        • 만약 currentClimb < 0이면 currentClimb = 0으로 보정.
    • 성공 또는 실패가 나올 때까지 위 과정을 반복한다.
    • H = 0이 들어오면 입력 종료.

    핵심: 코드가 하는 일

    • 입력 처리:
      • 각 줄에서 H, U, D, F를 읽는다.
      • H == 0이면 전체 입력을 종료한다.
    • 초기 설정:
      • day = 0, height = 0.0
      • currentClimb = U
      • fatigue = U * (F / 100.0) (매일 줄어드는 양)
    • 시뮬레이션 루프:
      • 무한 루프에서 하루 단위로:
        1. day 1 증가.
        2. 낮:
          • currentClimb > 0이면 그만큼 height 증가.
          • 바로 height > H이면 성공 처리 후 루프 종료.
        3. 밤:
          • height -= D
          • height < 0이면 실패 처리 후 루프 종료.
        4. 피로 적용:
          • currentClimb -= fatigue
          • 0보다 작아지면 currentClimb = 0으로 고정.
    • 출력:
      • 각 테스트 케이스마다 "success on day X" 또는 "failure on day X" 문자열을 StringBuilder에 쌓아두었다가 마지막에 한 번에 출력.
    • I/O 구현:
      • BufferedReader, StringTokenizer로 빠른 입력 처리.
      • EOF까지(또는 H == 0 등장까지) 모든 테스트 케이스를 순차적으로 처리.

    시간 복잡도

    각 테스트 케이스에 대해 하루씩 시뮬레이션하므로 시간 복잡도는 O(Days). 문제 조건상 달팽이는 언젠가 우물 밖으로 나가거나(성공) 바닥 아래로 떨어지기(실패) 때문에 무한 루프가 발생하지 않으며, 테스트 케이스 수와 날짜 수도 충분히 작아 실제로는 매우 빠르게 종료된다.

    공간 복잡도

    상수 개의 변수(day, height, currentClimb, fatigue 등)만 사용하므로 O(1).

    주의

    • 실수 처리:
      • fatigue 계산 시 (F / 100.0) 처럼 실수 연산을 사용해야 한다. 정수 나눗셈을 하면 오답.
    • 검사 순서:
      • 낮에 오른 직후 height > H면 그 날 밤 미끄러지기 전에 바로 성공 처리해야 한다.
      • 성공 판정을 밤 이후로 미루면 오답이 된다.
    • 음수 상승 방지:
      • currentClimb는 0 미만으로 내려가면 안 되므로, 매일 피로도를 적용한 뒤 0으로 보정하는 것이 중요하다.
    • 종료 조건:
      • H == 0 입력 시 즉시 전체 반복 종료 (문제 명세 그대로 구현).

    포인트 요약

    • 하루 진행 순서: 낮(상승) → 성공 여부 확인 → 밤(하강) → 실패 여부 확인 → 피로도 적용.
    • 피로도: 매일 U * (F / 100.0)만큼 currentClimb 감소, 최소 0까지.
    • 낮에 높이가 H를 초과하면 즉시 성공, 밤 후에 0 미만이면 실패.
    • 문제는 그대로 구현하는 시뮬레이션이며, 현재 코드 구조는 조건과 진행 순서를 정확히 잘 반영하고 있다.

    문제 4689 — Basically Speaking

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Basically Speaking (BOJ 4689)는 주어진 수를 임의의 진법(from)에서 다른 진법(to)으로 변환하여, 7자리 디스플레이에 맞게 오른쪽 정렬해 출력하는 문제다.

    • 지원 숫자/문자: 0-9, A-F
    • 지원 진법: 2 ~ 16
    • 각 줄 입력:
      • 변환할 수 s
      • 현재 진법 from
      • 목표 진법 to
      • 사이에 공백 여러 개 존재 가능
    • 출력:
      • 변환 결과를 대문자로 출력
      • 7자리에 맞춰 오른쪽 정렬
      • 결과 자릿수가 7자리를 초과하면 대신 ERROR를 7칸에 맞춰 오른쪽 정렬
    • 입력은 EOF까지 반복 처리.

    전략 개요

    • 핵심 아이디어는 매우 단순하다:
      1. s를 from진법으로 해석해 10진수 정수 값으로 변환.
      2. 그 값을 to진법 문자열로 다시 변환.
      3. 대문자로 바꾸고, 길이를 확인해
        • 7자 이하 → 7칸 오른쪽 정렬 출력
        • 7자 초과 → ERROR를 7칸 오른쪽 정렬로 출력
    • 자바 표준 라이브러리의 진법 변환 기능을 활용하면 구현이 매우 깔끔해진다.

    핵심: 코드가 하는 일

    • 입력 처리:
      • BufferedReader로 줄 단위 입력.
      • 각 줄에 대해:
        • trim()으로 양 끝 공백 제거.
        • 빈 줄이면 건너뜀.
        • StringTokenizer로 s, from, to를 순서대로 읽는다.
      • EOF까지 반복.
    • 진법 변환:
      • int value = Integer.parseInt(s, from);
        • s를 from진법으로 해석하여 10진수 정수로 변환.
      • String result = Integer.toString(value, to).toUpperCase();
        • 10진수 값을 to진법 문자열로 변환 후 대문자로 통일.
    • 출력 형식:
      • 만약 result.length() > 7이면:
        • String.format("%7s", "ERROR")를 사용해 7칸 오른쪽 정렬한 ERROR를 출력 버퍼에 추가.
      • 그 외에는:
        • String.format("%7s", result)로 변환 결과를 7칸 오른쪽 정렬해 추가.
      • 각 결과 뒤에 개행 문자 추가.
      • 모든 입력 처리가 끝나면 StringBuilder 내용을 한 번에 출력.
    • I/O 구현:
      • 여러 줄 입력 → 그때그때 변환 → 버퍼에 쌓아 마지막에 출력.
      • 여분 공백과 다양한 간격을 StringTokenizer로 자연스럽게 처리.

    시간 복잡도

    각 줄에 대해:

    • parseInt와 toString은 둘 다 수의 자릿수에 비례(O(L))
    전체 시간 복잡도는 입력 줄 수를 N이라 하면 O(N * L) 수준이며, 실제 입력 범위에선 매우 빠르게 동작한다.

    공간 복잡도

    한 줄씩 읽어 처리하고, 상수 개의 변수와 출력용 StringBuilder만 사용하므로 O(1) (출력 크기를 제외한 추가 메모리 기준).

    주의

    • 대문자 출력:
      • toString은 기본적으로 소문자(a-f)를 사용하므로 .toUpperCase()로 반드시 대문자로 변환해야 한다.
    • 7자리 초과 처리:
      • 자릿수만 본다. 값의 크기가 아니라 표현된 문자열 길이가 7자를 넘으면 무조건 ERROR.
    • 입력 형식:
      • 수와 진법 사이 공백이 여러 개일 수 있으므로, 공백 개수에 의존하지 말고 토크나이저 사용이 적절하다.
    • 정수 범위:
      • 이 코드는 int 범위 내에서 안전하게 동작한다.
      • 문제 테스트도 일반적으로 32비트 정수 범위를 넘기지 않으므로, 별도 오버플로 처리가 없어도 통과 가능하다.

    포인트 요약

    • Integer.parseInt(s, from)으로 입력 수를 10진수로 해석.
    • Integer.toString(value, to)로 목표 진법 문자열 생성 후 toUpperCase().
    • 길이 7 이하: String.format("%7s", result)로 오른쪽 정렬.
    • 길이 7 초과: String.format("%7s", "ERROR") 출력.
    • EOF까지 반복 처리, 현재 구현은 문제 요구사항과 형식에 잘 부합한다.
