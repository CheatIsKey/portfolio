    문제 4869 — Blackjack

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Blackjack (BOJ 4869)는 블랙잭 게임에서 처음 두 장이 나눠진 시점에,
    딜러의 오픈 카드 1장과 플레이어의 두 카드가 주어졌을 때 플레이어의 두 장 합이 딜러의 두 장 합보다 클 확률을 구하는 문제다.

    • 각 테스트 케이스:
      • n: 사용하는 덱의 개수 (1 ≤ n ≤ 10)
      • 다음 줄: dealerUp player1 player2
        • 각각 딜러의 공개 카드 1장, 플레이어의 카드 2장
        • 카드 표기: {A, K, Q, J, T, 9, ..., 2}
      • n = 0이면 입력 종료
    • 카드 값:
      • A: 1 또는 11 (21을 넘지 않도록 최대가 되게 선택)
      • K, Q, J, T: 10
      • 2~9: 숫자 그대로
      • 이 문제에서는 항상 두 장만 보므로, 합이 21을 넘는 경우는 없다 (버스트 고려 X).
    • 플레이어 승리 조건:
      • 플레이어의 두 카드 합 ≤ 21 (항상 성립)
      • 플레이어 합 > 딜러 합
      • (두 장만 보므로, 딜러 버스트도 사실상 의미 없음 → 단순히 player > dealer만 비교)
    • 목표:
      • 딜러의 숨겨진 카드(두 번째 카드)가 될 수 있는 모든 경우를 고려했을 때,
      • 플레이어가 이길 확률을 퍼센트(%)로, 소수 셋째 자리까지 반올림해서 출력.

    카드들은 덱에서 중복 없이 뽑힌다. n개의 덱을 섞은 상태에서, 이미 딜러 카드 1장, 플레이어 카드 2장이 빠진 뒤 남은 카드에서 딜러의 숨겨진 카드 1장을 랜덤하게 뽑는 상황을 모델링하면 된다.

    전략 개요

    • 덱 개수 n이 주어지면, 전체 카드 수는 52 × n장이다.
    • 각 랭크(A, K, Q, J, T, ..., 2)는 한 덱에 4장씩 있으므로, n덱이면 랭크별로 4 × n장 존재한다.
    • 딜러와 플레이어에게 이미 3장이 나간 상태이므로, 남은 카드들 중에서 딜러의 숨겨진 카드를 한 장 뽑는 모든 경우를 고려한다.
      • 각 랭크 r에 대해:
        • 남은 개수 cnt[r]를 세고,
        • 이 랭크가 딜러의 숨겨진 카드라 가정했을 때 딜러 점수 계산 → dealerScore
        • playerScore > dealerScore이면, 이 경우는 플레이어 승리 → 가능한 경우 수에 cnt[r]를 더한다.
    • 마지막으로,
      • winCount = 플레이어가 이기는 모든 카드 선택 수
      • totalRemainCards = 52 × n - 3
      • 승률(%) = winCount / totalRemainCards × 100

    즉, 남은 카드 하나가 어떤 랭크가 되느냐에 따라 승패가 갈리고, 각 랭크의 남아 있는 개수만큼 경우의 수를 곱해서 모두 더해주는 구조다.

    핵심: 코드가 하는 일

    • 랭크 배열 정의
      • static final char[] ranks = {'A','K','Q','J','T','9','8','7','6','5','4','3','2'};
      • 덱 배열의 인덱스와 랭크 문자를 매핑하기 위한 기준.
    • 입력 루프
      • 한 줄을 읽어 n을 파싱.
      • n == 0이면 종료.
      • 다음 줄에서 dealerCard, playerCard1, playerCard2를 입력받는다.
    • 덱 구성
      • int[] deck = new int[13];
      • 각 인덱스에 해당 랭크 카드 개수를 저장.
      • 초기 상태: 모든 랭크가 4 × n장씩 존재. 
        for (int i = 0; i < deck.length; i++) deck[i] = 4 * n;
      • 이미 뽑힌 3장의 카드를 덱에서 제거: 
        minusDeck(dealerCard, deck);
      minusDeck(playerCard1, deck);
      minusDeck(playerCard2, deck);
      • minusDeck()은 ranks에서 일치하는 문자를 찾아 해당 인덱스의 개수를 1 감소시킨다.
    • 플레이어 점수 계산
      • int playerScore = score(playerCard1, playerCard2);
      • score(card1, card2)는 두 카드의 점수를 합산하여 다음 규칙으로 계산:
        • int score = value(card1) + value(card2);
        • A의 개수를 세고 (aceCount),
        • A가 하나 이상 있고 score + 10 ≤ 21이면 score += 10 (A를 11로 한 번 승격).
      • value(card):
        • A → 1
        • K, Q, J, T → 10
        • 그 외 숫자 → card - '0' (문자 '2' ~ '9')
      • 두 장만 있기 때문에 A가 2장인 경우도:
        • 기본 합: 1 + 1 = 2
        • 2 + 10 = 12 ≤ 21이므로 결과는 12 (11 + 1)로 처리.
    • 딜러 숨겨진 카드에 대한 모든 경우 탐색
      • long winCount = 0;
      • int totalRemainCards = 52 * n - 3;
      • 모든 랭크 인덱스 i에 대해: 
        for (int i = 0; i < deck.length; i++) {
          int cnt = deck[i];
          if (cnt == 0) continue;
      
          int dealerScore = score(dealerCard, ranks[i]);
          if (playerScore > dealerScore) winCount += cnt;
      }
        • cnt는 해당 랭크가 남은 카드에서 몇 장 있는지.
        • 해당 랭크가 딜러의 숨겨진 카드가 될 수 있는 모든 경우의 수를 의미.
        • 딜러 점수 dealerScore를 계산하고, 플레이어 점수 playerScore가 더 크면 이 경우는 플레이어 승리 → 경우의 수 cnt를 winCount에 누적.
      • 즉, 남은 카드 한 장을 랭크별로 그룹화하여 확률을 합산하는 형태.
    • 확률 계산 및 출력
      • 확률(%) = winCount / totalRemainCards × 100.0
      • String.format("%.3f%%\n", prob)로 소수 셋째 자리까지 출력, 뒤에 % 포함.
      • StringBuilder에 누적 후 마지막에 한 번에 출력.

    시간 복잡도

    • 각 테스트 케이스마다:
      • 덱 초기화: O(13)
      • 카드 제거: O(13 × 3) (실질적으로 상수 시간)
      • 딜러 숨겨진 카드 후보 13개 순회: O(13)

    전체적으로 각 케이스는 O(1)에 가깝고,
    테스트케이스 수 T에 대해 O(T)로 충분히 빠르다.

    공간 복잡도

    deck[13]과 몇 개의 정수/실수 변수만 사용하므로 O(1) (상수 공간)이다.

    주의

    • 덱에서 카드 제거 순서/횟수
      • 딜러 오픈 카드, 플레이어 카드 2장 총 3장이 실제 덱에서 빠진 상태를 기준으로 딜러 숨겨진 카드를 뽑아야 한다.
    • A 처리 로직
      • 두 장만 있으므로 A를 11로 한 번만 승격하면 충분하다.
      • score + 10 ≤ 21 조건으로 처리.
    • 버스트(21 초과) 고려 X
      • 문제 조건상 두 장만 보기 때문에, 합이 22 이상이 될 수 없다.
      • 따라서 “딜러가 21을 넘는 경우”를 따로 처리하지 않고, playerScore > dealerScore 비교만 수행해도 된다.
    • 확률 출력 형식
      • %.3f%% 형식: 소수 셋째 자리까지, 뒤에 % 문자 포함.

    포인트 요약

    • 랭크별로 4 × n장씩 있는 덱을 만들고, 이미 사용된 3장의 카드를 빼서 남은 장 수를 관리한다.
    • 플레이어 점수는 score()로 한 번 계산하면 고정.
    • 딜러 숨겨진 카드 후보로 가능한 각 랭크 r에 대해:
      • 해당 랭크의 남은 카드 수 cnt[r]를 보면서,
      • 딜러 점수를 계산하고, playerScore > dealerScore이면 winCount += cnt[r].
    • 총 경우의 수는 남은 카드 수 totalRemainCards = 52 × n - 3.
    • 확률 = winCount / totalRemainCards × 100, 소수 셋째 자리까지 출력.
    • 덱 크기와 랭크 종류가 작아서 모든 경우를 단순 순회해도 성능은 충분히 여유롭다.

    문제 4877 — GYM

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    GYM (BOJ 4877)는 여러 개의 바구니와 그 사이를 이동하는 확률 과정(마코프 체인, Markov Chain)을 모델링하는 문제다.
    각 바구니에는 “다음에 이동할 바구니 이름이 적힌 카드들”이 들어 있고, 아이는 현재 위치 바구니에서 카드를 하나 무작위로 뽑아 그 목적지 바구니로 이동한다.

    • 총 N개의 바구니 (2 ≤ N ≤ 10)
    • 입력으로 각 바구니 i에서 j번 바구니로 이동시키는 카드 개수 count[i][j]가 주어진다.
    • 모든 아이는 처음에 0번 바구니에서 시작한다.
    • 한 스텝에서의 이동 규칙:
      • 바구니 i에 있을 확률이 P_s(i)라면,
      • 다음 스텝에서 바구니 j로 갈 확률은 count[i][j] / (i 바구니에 있는 카드의 총 개수)
      • 따라서, P_{s+1}(j) = Σ_i P_s(i) × (count[i][j] / rowSum[i])
    • 목표: 0번째 스텝부터 9번째 스텝까지 (총 10번) 각 바구니에 있을 확률 분포를 출력.

    예시에서 N=4인 경우, 입력은 각 행이 “현재 바구니”, 각 열이 “목적지 바구니”에 해당하는 카드 개수 행렬이다.
    실질적으로는 이 행렬을 전이 확률 행렬로 바꾸어, 확률 분포 벡터에 반복적으로 곱해 나가는 과정이다.

    전략 개요

    • 입력을 “카드 개수 행렬”로 읽는다.
      • count[i][j] = 바구니 i에서 바구니 j로 이동하는 카드 개수.
      • 각 행 i에 대해 rowSum[i] = Σ_j count[i][j]를 미리 계산해 둔다.
    • 확률 벡터를 한 스텝씩 갱신
      • 현재 스텝의 확률 분포를 curr[0..N-1]로 두고,
      • 다음 스텝 분포 next[0..N-1]를 next[j] = Σ_i curr[i] × (count[i][j] / rowSum[i])로 계산한다.
      • 처음에는 모든 아이가 0번 바구니에서 시작하므로 curr[0] = 1.0, curr[1..N-1] = 0.0.
    • 각 스텝마다 확률 출력 후, 분포 갱신
      • 0번째 스텝: curr = [1, 0, 0, ...] 그대로 출력.
      • 이후 9번 더:
        • next 계산 → curr = next로 교체 → 출력.
      • 총 10줄의 확률 분포가 출력된다.
    • 내부 계산은 double로 수행하고, 출력은 소수점 5자리까지 찍어준다.

    핵심: 코드가 하는 일

    • 입력 읽기 및 종료 조건
      • 한 줄을 읽고 앞뒤 공백을 제거: line = line.trim()
      • 라인이 "0"이거나 비어 있으면 입력 종료.
      • 그 외의 경우:
        • 공백 기준으로 split → 토큰 개수가 곧 N (바구니 개수)
    • 카드 개수 행렬 구성
      • int[][] count = new int[N][N];
      • 첫 줄에서 읽은 토큰들을 count[0][j]에 저장.
      • 이후 N-1줄을 더 읽어 count[1..N-1][j]를 채운다.
      • 이렇게 해서 “현재 바구니 i → 목적지 j” 카드 개수 행렬 완성.
    • 행별 카드 총합(rowSum) 계산
      • double[] rowSum = new double[N];
      • 각 행 i에 대해: 
        int sum = 0;
      for (int j = 0; j < N; j++) sum += count[i][j];
      rowSum[i] = sum;
      • 이 값으로 count[i][j] / rowSum[i] = “i에서 j로 갈 확률”을 계산한다.
    • 확률 분포 초기화
      • double[] curr = new double[N];
      • 모든 아이는 0번 바구니에서 시작: curr[0] = 1.0;, 나머지는 기본값 0.0.
    • 10 스텝 반복
      • for (int step = 0; step < 10; step++) { ... }
      • 각 스텝에서:
        1. 현재 분포 출력
          • 인덱스 0..N-1 순서대로, String.format(Locale.US, "%.5f", curr[j])으로 출력.
          • 바구니 사이에는 공백 하나씩, 마지막엔 개행 문자.
        2. 다음 분포 계산
          • double[] next = new double[N];
          • 모든 현재 위치 i에 대해: 
            for (int i = 0; i < N; i++) {
          if (curr[i] == 0.0) continue;        // 확률 0이면 건너뜀
          for (int j = 0; j < N; j++) {
              if (count[i][j] == 0) continue; // 카드 0장인 경우 건너뜀
              double probMove = count[i][j] / rowSum[i];
              next[j] += curr[i] * probMove;
          }
      }
          • 계산이 끝나면 curr = next;로 갱신.
    • 결과 출력
      • 모든 케이스를 처리한 후 System.out.print(sb);로 한 번에 출력.
      • StringBuilder를 사용해 출력 속도를 최적화.

    시간 복잡도

    한 케이스에서:

    • 입력 행렬 읽기: O(N^2)
    • rowSum 계산: O(N^2)
    • 각 스텝마다 next 계산:
      • 이중 for문 (i, j) → O(N^2)
      • 스텝 10번 → O(10 × N^2) = O(N^2)

    전체적으로 한 케이스당 O(N^2)이고, N ≤ 10 이므로 매우 빠르게 동작한다.

    공간 복잡도

    count[N][N], rowSum[N], curr[N], next[N] 정도만 필요하므로
    O(N^2) (N이 작아 메모리 사용량은 매우 적다).

    주의

    • 입력 형식
      • 문제 설명과 달리, 온라인 저지에서는 한 케이스가 끝나면 다음 케이스가 이어서 들어올 수 있고,
      • 단독으로 “0” 한 줄이 나오면 전체 입력이 끝난다고 가정하는 구현.
      • 코드에서는 line.equals("0") || line.length() == 0를 종료 조건으로 사용.
    • 부동소수점 정밀도
      • 내부 연산은 모두 double로 처리.
      • 출력은 소수점 5자리까지 String.format으로 포맷.
      • 문제 요구사항 “적어도 네 자리 정밀도”를 만족한다.
    • 0인 확률/카드 최적화
      • curr[i] == 0.0인 상태에서는 해당 i에서 어떤 j로도 이동할 확률이 0 → 통째로 건너뜀.
      • count[i][j] == 0인 경우도 건너뛰어 곱셈을 줄인다.
      • N이 작긴 하지만, 이런 최적화로 불필요한 연산을 피한다.

    포인트 요약

    • 바구니 간 이동을 “카드 개수 행렬 → 전이 확률 행렬”로 보고, 마코프 체인으로 모델링한다.
    • 모든 아이는 0번 바구니에서 시작하므로 초기 분포는 [1, 0, 0, ...]이다.
    • 각 스텝마다 next[j] = Σ_i curr[i] × (count[i][j] / rowSum[i])로 확률 분포를 갱신한다.
    • 0번째 스텝부터 9번째 스텝까지 총 10번의 분포를 출력해야 한다.
    • 시간 복잡도는 O(N^2), 공간은 O(N^2)로, N ≤ 10에 충분히 여유 있다.
    • 핵심은 “전이 확률 계산 + 확률 벡터를 10번 누적 곱하는 과정”을 정확히 구현하는 것이다.
