    문제 4884 — FIFA 월드컵

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    FIFA 월드컵 (BOJ 4884) 문제는
    조별 리그 + 토너먼트 형식의 대회가 주어졌을 때, 전체 경기 수와 토너먼트에 추가로 넣어야 하는 팀 수를 구하는 시뮬레이션/수학 문제다.

    • 입력: G T A D
      • G: 조(그룹)의 수
      • T: 각 조에 속한 팀 수
      • A: 각 조에서 토너먼트로 진출하는 팀 수
      • D: 조별 리그를 거치지 않고 바로 토너먼트로 진출하는 팀 수
      • 조건: 0 < A ≤ T, 네 수 모두 2^16 이하
      • 입력 마지막: -1 -1 -1 -1 → 종료
    • 토너먼트에 참가하는 팀 수는 항상 2의 거듭제곱이어야 한다.
      • 그렇지 않다면, 가장 가까운(크거나 같은) 2의 거듭제곱까지 팀을 더 추가해야 한다.
    • 출력 형식:
      • G*A/T+D=X+Y
      • 왼쪽은 그대로 문자열 포맷으로 출력하고,
      • X: 전체 경기 수 (조별 리그 + 토너먼트)
      • Y: 토너먼트에 추가로 넣어야 하는 팀 수

    실제 계산에서 토너먼트에 진출하는 기본 팀 수는 G * A + D로 처리한다 (조별 리그에서 각 조 A팀씩, 총 G * A팀 + 직행 D팀).

    전략 개요

    1. 토너먼트 기본 참가 팀 수 계산
      • base = G * A + D
    2. 가장 가까운 2의 거듭제곱 팀 수 K로 맞추기
      • K를 1에서 시작해, K < base인 동안 K *= 2 반복
      • 결과적으로 K는 base 이상인 최소 2의 거듭제곱
      • addTeams = K - base → 추가해야 하는 팀 수
    3. 조별 리그 경기 수
      • 각 조에는 팀이 T개 있고, 모든 팀이 서로 한 번씩 경기
      • 한 조의 경기 수: 조합 T C 2 = T * (T - 1) / 2
      • 조가 G개이므로 전체 조별 리그 경기 수: groupLeagues = G * T * (T - 1) / 2
    4. 토너먼트 경기 수
      • 단판 토너먼트에서 K팀이 참가하면, 경기 수는 항상 K - 1
      • 따라서 tournament = K - 1
    5. 총 경기 수
      • total = groupLeagues + tournament
    6. 문자열 포맷으로 출력
      • G*A/T+D=total+addTeams 형태로 그대로 붙여 출력한다.

    핵심: 코드가 하는 일

    • 입력 루프 및 종료 조건
      • 한 줄을 읽고 공백으로 나눈 뒤 G, T, A, D를 long으로 파싱.
      • G == -1 && T == -1 && A == -1 && D == -1이면 전체 종료.
    • 토너먼트 참가 팀 수와 2의 거듭제곱 K 계산
      • long base = G * A + D;
      • long K = 1;
      • while (K < base) K *= 2;
      • 루프 종료 후 K는 base 이상이면서 2의 거듭제곱인 최소값.
      • addTeams = K - base;
    • 조별 리그 경기 수 계산
      • groupLeagues = G * T * (T - 1) / 2;
      • T * (T - 1) / 2는 한 조의 경기 수, * G 해서 모든 조의 합.
    • 토너먼트 경기 수 계산
      • tournament = K - 1;
      • 단판 토너먼트에서 모든 경기는 한 팀이 탈락하므로, 우승팀 1팀만 남을 때까지 총 K - 1경기.
    • 총 경기 수 및 출력 구성
      • total = groupLeagues + tournament;
      • 출력: 
        sb.append(G).append("*").append(A).append("/").append(T).append("+").append(D)
        .append("=").append(total).append("+").append(addTeams).append('\n');
      • 원래 수식의 의미를 살려 왼쪽을 그대로 포맷팅하고, 오른쪽에 X + Y를 붙인다.

    시간 복잡도

    각 테스트 케이스마다 수행하는 연산은 모두 상수 시간이다.

    • 2의 거듭제곱 K 찾기: while (K < base) K *= 2;
      • base 최대는 대략 G * A + D ≤ (2^16)^2 + 2^16 수준이지만,
      • K는 2배씩 증가하므로 반복 횟수는 매우 작다 (로그 스케일).
    • 그 외 곱셈/덧셈/나눗셈 몇 번 → O(1)

    전체 시간 복잡도는 테스트 케이스 수를 T라 할 때 O(T)이다.

    공간 복잡도

    입력 파라미터와 몇 개의 long 변수만 사용하므로 O(1) (상수 공간)이다.

    주의

    • 자료형 오버플로:
      • G, T, A, D가 최대 2^16까지 올 수 있으므로, G * A, G * T * (T - 1) 같은 곱은 int 범위를 쉽게 넘는다.
      • 따라서 이 문제에서는 반드시 int가 아니라 long을 써야 한다.
    • 토너먼트 팀 수 계산:
      • 문제 출력 형식에는 G*A/T+D가 나오지만,
      • 실제 토너먼트 참가 팀 수는 코드처럼 G * A + D로 계산하고 있다.
      • T는 조별 리그 경기 수 계산(G * T * (T - 1) / 2)에만 사용된다.
    • 2의 거듭제곱 보정:
      • 이미 base가 2의 거듭제곱이면 K = base가 되고, addTeams = 0이 되어 추가 팀이 필요 없다.
      • 그 외에는 가장 가까운(크거나 같은) 2의 거듭제곱까지 확장한다.

    포인트 요약

    • 조별 리그 경기 수: 각 조에서 T C 2경기, 총 G * T * (T - 1) / 2경기.
    • 토너먼트 기본 참가 팀: base = G * A + D.
    • 토너먼트 팀 수는 가장 가까운 2의 거듭제곱 K로 맞추고, 추가 팀 수는 addTeams = K - base.
    • 토너먼트 경기 수: K - 1, 총 경기 수: groupLeagues + tournament.
    • 출력은 G*A/T+D=total+addTeams 형식을 그대로 맞춘다.
    • 곱셈이 크기 때문에 모든 계산은 반드시 long으로 처리해야 한다.

    문제 4955 — Pablo Squarson's Headache

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Pablo Squarson's Headache (BOJ 4955)는
    1×1 정사각형들을 하나씩 붙여 나가면서 만든 작품이 있을 때,
    이 작품 전체를 딱 덮을 수 있는 가장 작은 축 정렬 직사각형의 가로·세로 길이를 구하는 문제다.

    • 모든 정사각형의 한 변의 길이는 1이다.
    • 첫 번째 정사각형(번호 0)은 기준으로 아무 좌표나 두어도 되며, 코드에서는 (0, 0)에 둔다.
    • 이후 i(1 ≤ i ≤ N-1)번 정사각형은 ni di에 의해 이미 놓인 정사각형 ni의 왼쪽(0), 아래(1), 오른쪽(2), 위(3)에 인접하게 붙인다.
    • 정사각형들이 서로 겹치지 않는다는 것이 보장된다.
    • 모든 정사각형을 다 놓았을 때, 이들이 차지하는 좌표를 모두 포함하는 최소 직사각형의 width, height를 출력하면 된다.

    결국 해야 하는 일은: 각 정사각형의 (x, y) 위치를 하나씩 계산하고, 전체 x, y의 최소/최대를 구해서 범위를 계산하는 것이다.

    전략 개요

    • 각 정사각형은 좌표 평면 상의 정수 격자점에 중심이 있는 것처럼 생각할 수 있다.
      • 첫 번째 정사각형(0번)을 (0, 0)에 놓는다.
      • 이후, i번 정사각형의 위치는 “기준 정사각형 ni의 좌표 + 방향 벡터”로 결정된다.
    • 방향에 따른 좌표 변화:
      • 왼쪽(0): (x[ni] - 1, y[ni])
      • 아래(1): (x[ni], y[ni] - 1)
      • 오른쪽(2): (x[ni] + 1, y[ni])
      • 위(3): (x[ni], y[ni] + 1)
    • 정사각형들이 차지하는 x좌표의 최소/최대값, y좌표의 최소/최대값을 구하면:
      • width = maxX - minX + 1
      • height = maxY - minY + 1
      • +1을 하는 이유: 예를 들어 x가 0~2면, 실제 가로 길이는 3칸이기 때문.
    • 입력은 여러 데이터셋으로 주어지며, 첫 줄이 0이면 전체 입력이 끝난다.

    핵심: 코드가 하는 일

    • 입력 루프
      • 한 줄을 읽어서 정수 N으로 파싱.
      • N == 0이면 전체 입력 종료.
      • N < 200이므로, 배열을 N 크기로 잡아서 좌표를 저장한다.
    • 좌표 배열 초기화
      • int[] x = new int[N];
      • int[] y = new int[N];
      • 0번 정사각형은 기준: 
        x[0] = 0;
      y[0] = 0;
      • 동시에, 최소/최대값도 0으로 시작: 
        int minX = 0, maxX = 0;
      int minY = 0, maxY = 0;
    • 정사각형 배치 처리
      • 1번부터 (N-1)번까지 각 정사각형에 대해 한 줄씩 읽고:
        • square: 기준이 되는 정사각형 번호 ni
        • direct: 방향 di (0~3)
      • 방향에 따라 좌표를 계산: 
        if (direct == 0) {        // left
          x[i] = x[square] - 1;
          y[i] = y[square];
      } else if (direct == 1) { // down
          x[i] = x[square];
          y[i] = y[square] - 1;
      } else if (direct == 2) { // right
          x[i] = x[square] + 1;
          y[i] = y[square];
      } else {                   // up (3)
          x[i] = x[square];
          y[i] = y[square] + 1;
      }
      • 새 좌표를 반영하여 최소/최대값 갱신: 
        if (x[i] < minX) minX = x[i];
      if (x[i] > maxX) maxX = x[i];
      if (y[i] < minY) minY = y[i];
      if (y[i] > maxY) maxY = y[i];
    • 최종 가로/세로 길이 계산
      • 모든 정사각형의 좌표를 다 계산한 뒤, 
        int width  = maxX - minX + 1;
      int height = maxY - minY + 1;
      • 이 값을 width height 형식으로 한 줄에 출력 버퍼에 추가: 
        sb.append(width).append(' ').append(height).append('\n');
    • 출력
      • 모든 데이터셋을 처리한 뒤, System.out.print(sb);로 한 번에 출력.

    시간 복잡도

    • 각 데이터셋마다:
      • N개의 정사각형, 그 중 (N-1)개에 대해 한 번씩 좌표 계산 및 min/max 갱신
      • 각 연산은 상수 시간 → O(N)
    • N < 200 이므로 전체적으로 매우 빠르게 동작한다.

    전체 입력에 대해, 데이터셋 수를 D라 하면 복잡도는 O(Σ N) = O(total squares) 수준이다.

    공간 복잡도

    x[N], y[N] 두 개의 정수 배열과 몇 개의 변수만 사용하므로
    데이터셋당 공간 복잡도는 O(N)이다.

    주의

    • 좌표계 선택은 임의:
      • 첫 정사각형을 (0, 0)에 두는 것은 임의의 선택이지만, 모든 정사각형이 상대 위치만 유지하면 너비와 높이는 동일하게 나온다.
    • min/max 초기값:
      • 0번 정사각형이 (0, 0)에 있으므로, minX, maxX, minY, maxY를 모두 0으로 시작하면 자연스럽게 포함된다.
    • +1을 잊지 말 것:
      • 좌표 범위가 [minX, maxX]이면 실제 칸 수는 maxX - minX + 1이다.
      • 세로도 동일하게 +1을 해줘야 한다.
    • 입력 형식:
      • N이 1인 경우, 배치 명령이 없으므로 바로 (1, 1)을 출력하게 된다.
      • 이 경우에도 로직이 자연스럽게 동작한다 (for 루프가 한 번도 돌지 않음).

    포인트 요약

    • 0번 정사각형을 (0, 0)에 두고, 나머지 정사각형의 좌표를 “기준 + 방향”으로 차례대로 계산한다.
    • 계산 과정에서 x, y의 최소/최대값을 함께 갱신한다.
    • 최종적으로 width = maxX - minX + 1, height = maxY - minY + 1로 작품을 덮는 박스 크기를 구한다.
    • 정수 좌표와 단순 반복만 사용하므로 구현과 복잡도가 모두 간단하다.
