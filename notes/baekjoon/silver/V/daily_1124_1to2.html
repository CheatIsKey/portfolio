    문제 4962 — Next Mayor

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Next Mayor (BOJ 4962)는
    둥근 테이블에 앉은 후보들 사이에서 돌멩이(pebbles)를 주고받는 게임을 그대로 시뮬레이션해서
    마지막에 모든 돌을 가지게 되는 후보의 번호를 구하는 문제다.

    규칙을 정리하면:

    • 후보는 n명, 번호는 0 ~ n-1
    • 처음에 그릇(bowl)에 돌이 p개 들어 있고, 0번 후보에게 그릇이 주어진다.
    • 각 턴에서, 현재 그릇을 가진 후보 idx가:
      • 그릇에 돌이 있으면 돌 1개를 꺼내서 자신의 손에 보관하고, 그릇의 돌 개수는 1 감소.
      • 그릇에 돌이 없으면, 자신이 가지고 있던 모든 돌을 그릇에 다시 넣고, 자신의 손은 0개가 된다.
      • 그리고 그릇을 오른쪽(다음 후보, (idx+1) % n)에게 넘긴다.
    • 게임 종료 조건:
      • 어떤 후보가 그릇에서 마지막 돌을 가져간 직후,
      • 그 후보를 제외한 모든 후보의 손에 돌이 0개라면, 그 후보가 승자가 된다.

    문제에서 주어진 입력에 대해서는 항상 1,000,000번 이하의 스텝 안에 게임이 끝난다고 보장된다.
    따라서 그대로 시뮬레이션해도 충분하다.

    전략 개요

    • 후보별로 현재 손에 들고 있는 돌 개수를 저장하는 배열 player[0..n-1]를 둔다.
    • 그릇(bowl)에 남아 있는 돌 개수를 나타내는 정수 bowl을 둔다 (초기값 = p).
    • 현재 그릇을 가진 후보를 나타내는 인덱스 idx를 둔다 (초기값 = 0).
    • 무한 루프 안에서 규칙 그대로를 시뮬레이션:
      • 그릇에 돌이 있을 때와 없을 때를 나누어 처리.
      • 마지막 돌을 가져간 직후, 나머지 후보들이 모두 0개인지 검사해 승리 여부 판단.
      • 승리 시 바로 후보 번호를 출력 버퍼에 추가하고 루프 종료.
      • 그 외에는 idx = (idx + 1) % n으로 다음 후보에게 그릇을 넘겨 계속 반복.

    핵심: 코드가 하는 일

    • 입력 처리 및 종료 조건
      • 각 줄에서 n, p를 읽는다.
      • n == 0 && p == 0이면 입력 종료.
    • 상태 변수 초기화
      • int[] player = new int[n]; → 모든 후보의 손에 든 돌을 0으로 초기화.
      • int bowl = p; → 그릇에 초기 돌 개수 저장.
      • int idx = 0; → 0번 후보가 먼저 그릇을 받는다.
    • 게임 루프 
      while (true) {
          if (bowl > 0) {
              // 그릇에 돌이 있을 때
              player[idx]++;  // 자기 손에 1개 가져감
              bowl--;
      
              if (bowl == 0) {
                  // 방금 '마지막' 돌을 가져간 상태
                  boolean win = true;
                  for (int i = 0; i < n; i++) {
                      if (i == idx) continue;
                      if (player[i] != 0) {
                          win = false;
                          break;
                      }
                  }
                  if (win) {
                      sb.append(idx).append('\n');
                      break; // 게임 종료
                  }
              }
          } else {
              // 그릇이 비어 있을 때: 자기 손에 있는 돌을 전부 그릇에 넣는다.
              bowl = player[idx];
              player[idx] = 0;
          }
      
          // 그릇을 오른쪽(다음 후보)에게 넘김
          idx = (idx + 1) % n;
      }
    • 승리 판정 로직
      • 어떤 후보 idx가 그릇에서 돌 1개를 가져간 후, 그릇이 비어(bowl == 0)면 “마지막 돌”을 가져간 상황이다.
      • 이 시점에서 idx를 제외한 모든 후보의 player[i] == 0인지 검사한다.
      • 전부 0이면 게임 종료, idx가 승자.
      • 누군가 손에 돌을 더 가지고 있다면, 아직 게임은 끝나지 않은 상태이므로 계속 진행.
    • 다중 테스트케이스 처리
      • 각 테스트케이스마다 위 과정을 반복하고, 승자 번호를 StringBuilder에 한 줄씩 저장.
      • 모든 입력을 처리한 후 System.out.print(sb);로 한 번에 출력.

    시간 복잡도

    각 테스트케이스에 대해:

    • 게임 루프 안에서 하는 일은 대부분 상수 시간 연산.
    • 단, 그릇이 비었을 때 승리 조건을 검사하는 부분에서 O(n) 탐색을 수행.
    • 문제에서 각 게임은 최대 1,000,000 스텝 이내에 끝난다고 보장되며, n ≤ 50이므로
      최악의 경우에도 대략 1,000,000 × 50 정도의 연산 → 충분히 통과 가능한 수준.

    따라서 한 케이스의 시간 복잡도는 대략 O(steps × n)이며,
    실제 제한에서는 무리 없이 동작한다.

    공간 복잡도

    후보 수 n에 대해 player[n] 배열과 몇 개의 정수 변수만 사용하므로
    공간 복잡도는 O(n), 최대 n = 50에서 매우 작다.

    주의

    • 시뮬레이션 순서를 정확히 지켜야 한다:
      • 먼저 그릇에 돌이 있는지 확인 → 빼거나(보관) / 자기 돌을 모두 넣거나.
      • 마지막 돌을 가져간 직후에만 승리 여부를 검사해야 한다.
      • 그 후에 idx를 다음 후보로 넘긴다.
    • bowl이 0일 때 처리:
      • 무조건 자기 손에 있는 돌을 전부 그릇으로 옮기고 자신의 돌은 0으로 만들어야 한다.
      • 이 때는 승리 조건을 검사하지 않는다 (마지막 돌을 가져가는 상황이 아니기 때문).
    • 무한 루프 안전성:
      • 문제에서 “주어진 입력에 대해 1,000,000 스텝 이내에 항상 게임이 종료된다”고 보장하므로, 별도의 탈출 조건(카운터 등)을 두지 않아도 된다.

    포인트 요약

    • 각 후보가 가진 돌과 그릇의 돌을 배열/변수로 저장하고, 규칙 그대로 시뮬레이션한다.
    • 그릇에 돌이 있으면 1개 가져오고, 없으면 자기 돌을 전부 그릇에 넣는다.
    • 마지막 돌을 가져간 직후, 다른 모든 후보의 손에 돌이 없는지 확인해 승자를 결정한다.
    • 후보 번호는 idx를 (idx + 1) % n 방식으로 순환시켜 관리한다.
    • 문제에서 스텝 수 상한을 보장하므로, 단순 시뮬레이션으로 충분히 해결 가능하다.

    문제 5024 — Dirty Driving

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Dirty Driving (BOJ 5024)는
    현재 나와 앞차들 사이의 거리들이 주어졌을 때,
    앞차와의 최소 간격을 얼마나 둬야 모든 앞차와의 안전거리를 만족하는지를 구하는 문제다.

    조건은 다음과 같다.

    • 앞에 n대의 자동차가 있고, 각 차까지의 거리가 주어진다 (서로 다른 값).
    • 어떤 차 x와 나 사이에 n_between대의 차가 끼어 있다면, 그 차까지의 거리는 최소 p · (n_between + 1) 이상이어야 한다.
    • 우리가 속도를 줄여 앞차와의 간격을 늘리면, 나와 모든 앞차와의 거리가 똑같이 증가한다고 가정할 수 있다.

    목표: 앞차(가장 가까운 차)와의 최종 거리 D가 최소가 되도록 하면서,
    위 안전거리 조건을 모든 차에 대해 만족시키는 최소 D를 구한다.

    전략 개요

    • 현재 나와 각 차 사이의 거리를 오름차순으로 정렬하면:
      • distance[0]: 가장 가까운 차 (바로 앞차)
      • distance[1]: 그 다음 차
      • …
      • distance[i]: i번째로 가까운 차 → 나와 그 차 사이에는 i대가 끼어 있다.
    • 우리가 앞차와의 간격을 늘려서, 모든 거리들이 t만큼 증가한다고 생각하자.
      • 현재 거리: distance[i]
      • 조정 후 거리: distance[i] + t
      • 이 차와의 사이에 끼어 있는 차의 수는 i이므로, 필요한 최소 거리: p · (i + 1)
      • 따라서 조건: distance[i] + t ≥ p · (i + 1)
    • 모든 i에 대해 위를 만족하려면:
      • t ≥ p·(i+1) - distance[i] (각 i마다)
      • 결국 필요한 최소 t는 각 i에서의 부족분 중 최대값이 된다.
      • 부족분이 음수(이미 기준 이상 거리)인 경우는 0으로 취급해도 된다.
    • 따라서:
      • need = max(0, max_i (p·(i+1) - distance[i]))
      • 앞차(가장 가까운 차)와의 최종 거리: answer = distance[0] + need

    핵심: 코드가 하는 일

    • 입력
      • 첫 줄에서 n, p를 읽는다.
      • 다음 줄에서 n개의 거리 값을 읽어 distance[i]에 저장.
    • 거리 정렬 
      Arrays.sort(distance);
      • 오름차순 정렬로 가장 가까운 차가 distance[0]이 된다.
    • 필요한 추가 거리(need) 계산 
      int need = 0;
      for (int i = 0; i < n; i++) {
          int required = p * (i + 1);       // i번째 차에 필요한 최소 거리
          if (distance[i] < required) {
              int deficit = required - distance[i]; // 부족분
              if (deficit > need) need = deficit;   // 최대 부족분 갱신
          }
      }
      • required: i번째 차에 대해, 조건 p · (i + 1)
      • distance[i] < required이면, 해당 차와의 거리가 부족함.
      • 부족분들을 모아 그 중 최대값이 need가 된다.
    • 정답 계산 
      int answer = distance[0] + need;
      sb.append(answer).append('\n');
      • 가장 가까운 차와의 초기 거리 distance[0]에 추가로 확보해야 하는 거리 need를 더한다.
      • 이 값이 “앞차와 유지해야 할 최소 거리”가 된다.

    시간 복잡도

    • 정렬: Arrays.sort(distance) → O(n log n)
    • 한 번의 선형 스캔으로 부족분 계산: O(n)

    전체 시간 복잡도는 O(n log n) 이고, n ≤ 100000 조건에서 충분히 빠르다.

    공간 복잡도

    거리 배열 int[n]만 사용하므로 공간 복잡도는 O(n)이다.

    주의

    • 정렬 기준:
      • 반드시 오름차순으로 정렬해야, i번째 인덱스가 “사이에 i대가 있는 차”와 대응된다.
    • 추가 거리 t의 해석:
      • need는 우리가 현재 위치에서 뒤로 얼마나 더 물러나야 하는지를 나타낸다.
      • 그만큼 모든 거리들이 동시에 증가한다고 생각할 수 있다.
    • 거리 타입:
      • 거리 값은 최대 10^7이고, p ≤ 20, n ≤ 100000이므로 p · (i + 1)는 ≈ 2·10^6 수준 → int 안에 충분히 들어간다.

    포인트 요약

    • 현재 거리들을 오름차순 정렬해, i번째 차에 대해 필요한 최소 거리 p·(i+1)와 비교한다.
    • 부족한 만큼의 거리 required - distance[i] 중 최대값이 모두를 만족시키기 위해 추가로 필요한 거리 need이다.
    • 최종 답은 가장 가까운 차와의 거리 distance[0]에 need를 더한 값.
    • 정렬 + 한 번의 스캔으로 해결하는 깔끔한 그리디/시뮬레이션 문제다.
