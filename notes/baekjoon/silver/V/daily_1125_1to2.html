    문제 5078 — Shirts

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Shirts (BOJ 5078)는
    아내와 남편의 셔츠 목록을 하나로 합친 뒤,
    사이즈 → 색 순서대로 정렬해서 한 줄로 출력하는 문제다.

    정렬 기준은 다음과 같다.

    • 사이즈 우선:
      • S < M < L 순서로 정렬
    • 색상 차선:
      • 같은 사이즈끼리는 두 번째 문자(색을 나타내는 알파벳)를 알파벳 순으로 정렬한다.
      • 색 코드는 B, K, N, O, P, R, W이며,
        이 역시 알파벳 기준으로 비교하면 된다.

    입력은 여러 시나리오로 구성되며,
    각 시나리오마다 아내 옷장, 남편 옷장을 읽어서 합치고,
    정렬한 결과를 공백으로 구분해 한 줄에 출력하면 된다.

    전략 개요

    • 각 셔츠는 두 글자 문자열(예: "SB", "MP")로 표현된다.
      • 첫 글자: 사이즈(S, M, L)
      • 둘째 글자: 색상(B, K, N, O, P, R, W)
    • 한 시나리오에서:
      • 아내 셔츠 개수 W를 읽고, W개의 문자열을 리스트에 넣는다.
      • 남편 셔츠 개수 H를 읽고, H개의 문자열도 같은 리스트에 넣는다.
      • 이제 리스트에는 아내+남편 셔츠가 모두 들어 있다.
    • 리스트 전체를 사용자 정의 비교 함수로 정렬한다.
      • 사이즈는 S=0, M=1, L=2와 같이 가중치로 변환해 비교한다.
      • 사이즈가 같으면 두 번째 문자(색 코드)를 그대로 빼서 비교한다.
    • 정렬이 끝나면, 모든 문자열을 공백으로 구분해 출력 버퍼에 추가한다.

    입력 첫 줄의 W가 0이면 더 이상 시나리오가 없다는 뜻이므로
    전체 입력 처리를 종료한다.

    핵심: 코드가 하는 일

    • 시나리오 반복 처리
      • 입력을 한 줄씩 읽어서 W를 구한다. 
        String line;
while ((line = br.readLine()) != null) {
    line = line.trim();

    int W = Integer.parseInt(line);
    if (W == 0) break;
    ...
}
      • W == 0이면 입력 종료.
    • 아내 셔츠 입력
      • 아내 셔츠를 저장할 리스트를 만든다. 
        List<String> closet = new ArrayList<>();
      • W번 반복하면서, 각 줄을 그대로 리스트에 추가한다. 
        for (int i = 0; i < W; i++) closet.add(br.readLine());
    • 남편 셔츠 입력
      • 남편 셔츠 개수 H를 읽는다. 
        int H = Integer.parseInt(br.readLine());
      • H번 반복하면서, 각 줄을 동일한 리스트에 추가한다. 
        for (int i = 0; i < H; i++) closet.add(br.readLine());
      • 이 시점에서 closet에는 아내+남편 셔츠 전체가 들어 있다.
    • 정렬 기준 구현
      • Collections.sort와 람다 비교자를 사용한다. 
        Collections.sort(closet, (a, b) -> {
    char sizeA = a.charAt(0);
    char sizeB = b.charAt(0);
    char colorA = a.charAt(1);
    char colorB = b.charAt(1);

    int wa = (sizeA == 'S') ? 0 : (sizeA == 'M') ? 1 : 2;
    int wb = (sizeB == 'S') ? 0 : (sizeB == 'M') ? 1 : 2;

    if (wa != wb) {
        return wa - wb;   // 사이즈 우선 비교
    }
    return colorA - colorB; // 색상 알파벳 순 비교
});
      • wa, wb는 사이즈를 숫자로 치환한 값이다.
        • 'S' → 0
        • 'M' → 1
        • 'L' → 2
      • 사이즈 숫자가 다르면 그 차이를 반환해 S < M < L 순서가 되도록 한다.
      • 사이즈가 같으면 두 번째 문자(colorA, colorB)를 그대로 빼서 알파벳 순으로 정렬한다.
    • 출력 구성
      • 정렬이 끝난 리스트를 앞에서부터 순회하며, 각 셔츠 코드를 공백과 함께 출력 버퍼에 붙인다. 
        for (String s : closet) {
    sb.append(s).append(" ");
}
      • 모든 시나리오 처리가 끝나면 한 번에 출력한다. 
        System.out.print(sb);

    시간 복잡도

    한 시나리오에서 셔츠 총 개수를 N = W + H라고 하면:

    • 리스트에 넣는 데 걸리는 시간: O(N)
    • 정렬: Collections.sort → 평균적으로 O(N log N)

    문제에서 W, H ≤ 30이므로 N ≤ 60이고,
    실제 연산량은 매우 작다. 이론적으로는 O(N log N)이다.

    여러 시나리오가 주어져도 각 시나리오는 독립적으로 처리되므로,
    전체 시간도 시나리오 수에 비례하는 수준에서 충분히 빠르다.

    공간 복잡도

    한 시나리오에서 사용되는 주요 추가 메모리는:

    • 셔츠들을 저장하는 리스트 closet (최대 60개 문자열)
    • 출력 누적용 StringBuilder
    • 그 외 몇 개의 지역 변수

    따라서 한 시나리오당 공간 복잡도는 O(N)이다.
    여기서 N은 셔츠 개수(≤ 60)로, 실제로는 매우 작은 수준이다.

    주의

    • 사이즈 정렬 기준:
      • 사이즈는 알파벳 순이 아니라 S < M < L 순이다.
      • 따라서 가중치(0, 1, 2)로 변환한 뒤 그 값을 비교해야 한다.
    • 색상 정렬 기준:
      • 색상은 두 번째 문자 그대로 알파벳 순으로 비교하면 된다.
      • 색 코드 집합이 이미 서로 다른 알파벳이므로 추가 맵핑이 필요 없다.
    • 여러 시나리오 출력:
      • 각 시나리오의 결과는 한 줄에 출력해야 한다.
      • 현재 코드는 시나리오 사이에 줄바꿈을 넣지 않고 "셔츠 + 공백"만 이어 붙이고 있으므로,
        시나리오가 여러 개일 경우 각 시나리오 끝에서 sb.append('\n');를 추가해 주는 것이 안전하다.
    • 마지막 공백:
      • 각 시나리오 끝에 공백 하나가 더 붙지만, 대부분의 채점에서는 큰 문제 없이 처리된다.

    포인트 요약

    • 아내와 남편 셔츠를 하나의 리스트에 모은다.
    • 사이즈는 S < M < L 순으로, 색상은 두 번째 문자의 알파벳 순으로 정렬한다.
    • Collections.sort와 람다 비교자를 사용해 정렬 기준을 직접 구현한다.
    • 정렬된 결과를 공백으로 구분해 한 줄에 출력한다.
    • 입력 크기가 작아서 시간/공간 모두 매우 여유로운 문제다.

    문제 5091 — Hostel Nights

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Hostel Nights (BOJ 5091)는
    5일 동안 특정 조건에 의해 “조용했다고 판명되어 제거되는” 방들을 제외하고,
    각 방이 몇 밤이나 끝까지 의심 대상으로 남았는지를 세어
    가장 많이 살아남은 학생(방)의 이름을 구하는 시뮬레이션 문제다.

    한 주(week)는 다음과 같이 진행된다.

    • 방 번호는 101 ~ 120까지 총 20개이며, 각 방에 학생 이름이 하나씩 주어진다.
    • 5일 동안 매일 다음 조건들로 방들을 “제거”한다.
      • 짝수/홀수 방 전체 제거 (첫 글자 E 또는 O)
      • 어떤 수 n의 배수인 방 제거
      • 이름의 첫 글자가 특정 알파벳인 학생 제거
    • 위 조건 중 하나라도 만족하면 그 날은 “조용했다고 판명되어” 제거되고,
      어떤 조건도 만족하지 않으면 그 날은 끝까지 의심 대상으로 남는다.
    • 5일이 끝난 후, 가장 많은 밤 동안 제거되지 않고 남았던 학생들이 “가장 시끄러운 학생들”이 된다.

    출력은 주 번호(Week i)를 먼저 출력하고,
    그 주에서 최대 횟수만큼 살아남은 학생들의 이름을
    방 번호 오름차순으로 한 줄에 하나씩 출력한다.

    전략 개요

    • 각 방 번호(101 ~ 120)를 인덱스로 사용해:
      • name[room]: 해당 방의 학생 이름
      • survive[room]: 5일 동안 그 방이 제거되지 않고 남은 밤의 수
    • 각 밤마다 방 101 ~ 120을 모두 돌면서:
      • 주어진 조건 3개(짝/홀, 배수, 이름 첫 글자)를 순서대로 검사한다.
      • 어느 하나라도 만족하면 그 방은 그 날 제거된다.
      • 어떤 조건에도 걸리지 않으면 survive[room]++ 한다.
    • 5일이 끝난 뒤:
      • survive 배열에서 최댓값 max를 찾는다.
      • survive[room] == max인 방들의 이름을 방 번호 순으로 출력한다.
    • 이를 모든 주(week)에 대해 반복한다.

    핵심: 코드가 하는 일

    • 전체 주 수 읽기
      • 첫 줄에서 W를 읽어 총 주 수를 구한다. 
        int W = Integer.parseInt(line);

for (int i = 0; i < W; i++) {
    ...
}
    • 학생 정보 초기화
      • 방 번호를 그대로 인덱스로 쓰기 위해 크기 121 배열을 만든다. 
        String[] name = new String[121];
int[] survive = new int[121];

for (int j = 0; j < 20; j++) {
    StringTokenizer st = new StringTokenizer(br.readLine());
    int room = Integer.parseInt(st.nextToken());
    String student = st.nextToken();

    name[room] = student;
    survive[room] = 0;
}
      • 입력에서 방 번호는 101 ~ 120만 사용되므로,
        그 구간만 채워지고 나머지 인덱스는 사용하지 않는다.
    • 5일 동안 제거 여부 시뮬레이션
      • 각 밤마다 조건 3개를 읽는다. 
        for (int j = 0; j < 5; j++) {
    StringTokenizer st = new StringTokenizer(br.readLine());
    String isOdd = st.nextToken();   // "E" 또는 "O"
    int where = Integer.parseInt(st.nextToken()); // 배수 기준
    char letter = st.nextToken().charAt(0);       // 이름 첫 글자 기준
    ...
}
      • 모든 방(101 ~ 120)에 대해 제거 여부를 판단한다. 
        for (int room = 101; room <= 120; room++) {
    boolean removed = false;

    // 1) 짝/홀 기준 제거
    if (isOdd.equals("O") && room % 2 == 1) removed = true;
    else if (isOdd.equals("E") && room % 2 == 0) removed = true;

    // 2) where의 배수면 제거
    if (!removed && room % where == 0) removed = true;

    // 3) 이름 첫 글자가 letter면 제거
    if (!removed && name[room].charAt(0) == letter) removed = true;

    // 세 조건 모두 통과하면 그 날 끝까지 살아남은 것
    if (!removed) survive[room]++;
}
      • 위 로직에서:
        • removed가 한 번 true가 되면 더 이상 다른 조건은 보지 않는다.
        • 어느 조건에도 걸리지 않은 경우에만 survive[room]을 1 증가시킨다.
    • 최대 생존 횟수 찾기
      • 5일이 끝난 후, 가장 많이 살아남은 횟수 max를 구한다. 
        int max = 0;
for (int room = 101; room <= 120; room++) {
    if (survive[room] > max) max = survive[room];
}
    • 출력
      • 먼저 주 번호를 출력한다. 
        sb.append("Week ").append(i + 1).append('\n');
      • 그 다음, survive[room] == max인 방들을 방 번호 오름차순(101 → 120)으로 순회하면서 이름을 출력 버퍼에 추가한다. 
        for (int room = 101; room <= 120; room++) {
    if (survive[room] == max) {
        sb.append(name[room]).append('\n');
    }
}
      • 모든 주 처리가 끝나면 System.out.print(sb);로 한 번에 출력한다.

    시간 복잡도

    한 주(week)에 대해:

    • 학생 20명 정보 입력: O(20)
    • 5일 × 방 20개에 대해 조건 검사:
      • 각 밤마다 20개 방 검사 → O(20)
      • 5일이므로 총 O(5 × 20) = O(100)
    • 최대 생존 횟수 탐색: O(20)

    따라서 한 주당 전체 시간 복잡도는 상수에 가까운
    O(1) (정확히는 O(20 × 5) 정도)라고 볼 수 있다.
    주 수 W가 커져도 매우 빠르게 동작한다.

    공간 복잡도

    한 주에 사용되는 주요 추가 메모리는:

    • 이름 배열 name[121]
    • 생존 횟수 배열 survive[121]
    • 몇 개의 보조 변수들

    방 수(20개)에 비례하는 배열만 사용하므로
    공간 복잡도는 O(1) (상수)에 해당한다.

    주의

    • 조건은 OR 관계:
      • 짝/홀, 배수, 이름 첫 글자 조건 중 하나라도 만족하면 그 날 제거된다.
      • 코드에서는 removed 플래그를 사용해 이를 구현하고 있다.
    • 이름 없는 인덱스 접근 금지:
      • 실제 입력에서는 101 ~ 120만 유효하므로,
        반드시 그 구간만 반복해야 한다.
    • 주 번호 출력 형식:
      • 출력 형식은 Week 1, Week 2처럼 정확히 맞춰야 한다.
    • 최소 한 명은 존재:
      • 문제에서 “이 층은 악명 높기 때문에 최소 한 명은 남는다”고 보장하므로,
        max가 0인 경우라도 최소 한 명은 이름이 출력된다.

    포인트 요약

    • 방 번호(101 ~ 120)를 배열 인덱스로 사용해 이름과 생존 횟수를 관리한다.
    • 5일 동안 각 조건(짝/홀, 배수, 이름 첫 글자)에 따라 방을 제거하고,
      제거되지 않은 방만 밤마다 survive를 1씩 증가시킨다.
    • 5일 후 가장 많이 살아남은 횟수를 가진 방들을 찾고,
      그 방의 학생 이름을 방 번호 순으로 출력한다.
    • 입력 크기가 매우 작아서 단순 시뮬레이션만으로도 충분히 해결 가능하다.
