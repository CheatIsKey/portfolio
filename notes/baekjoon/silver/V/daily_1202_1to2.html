    문제 5349 — Duplicate SSN

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Duplicate SSN (BOJ 5349) 문제는
    여러 줄에 걸쳐 주어진 미국 사회보장번호(SSN) 목록에서
    두 번 이상 등장한 SSN만 오름차순으로 출력하는 문제다.

    • 각 줄에는 AAA-BB-CCCC 형식의 문자열이 주어진다.
    • 000-00-0000이 등장하면 입력이 종료되며,
      이 값은 절대 중복으로 취급하지 않는다.
    • 최종적으로, 입력에 두 번 이상 등장한 SSN을 모두 모아
      사전순(오름차순)으로 정렬하여 한 줄에 하나씩 출력한다.

    전략 개요

    • 입력을 한 줄씩 읽으면서, 각 SSN이 몇 번 등장했는지 카운트한다.
    • 카운트에는 해시맵을 사용한다:
      • Map<String, Integer> map : SSN → 등장 횟수
    • 입력 종료 후:
      • map을 순회하면서, 값이 2 이상인 키(SSN)만 골라 리스트에 담는다.
      • 리스트를 문자열 오름차순 정렬한다.
      • 정렬된 리스트를 한 줄씩 출력한다.
    • SSN 형식이 이미 정해져 있고 문자열 비교로 정렬해도
      문제에서 원하는 “오름차순”과 일치한다.

    핵심: 코드가 하는 일

    • 맵 준비 및 입력 처리 
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      StringBuilder sb = new StringBuilder();
      
      Map<String, Integer> map = new HashMap<>();
      
      String line;
      while ((line = br.readLine()) != null) {
          line = line.trim();
      
          if (line.equals("000-00-0000")) break;
      
          map.put(line, map.getOrDefault(line, 0) + 1);
      }
      • map은 각 SSN 문자열을 키로, 등장 횟수를 값으로 저장한다.
      • 한 줄씩 읽어 trim() 후,
        • 만약 "000-00-0000"이면 입력 종료.
        • 그렇지 않으면 map.getOrDefault(line, 0) + 1로 카운트를 1 증가.
      • 종료 코드 "000-00-0000"는 맵에 넣지 않으므로 중복 후보에서 제외된다.
    • 중복 SSN만 모으기 
      List<String> list = new ArrayList<>();
      for (Map.Entry<String, Integer> entry : map.entrySet()) {
          String key = entry.getKey();
          Integer value = entry.getValue();
      
          if (value > 1) {
              list.add(key);
          }
      }
      • map에 기록된 모든 SSN을 순회하면서 등장 횟수를 확인한다.
      • value > 1인 경우, 즉 두 번 이상 등장한 SSN만 리스트에 담는다.
      • 한 번만 등장한 SSN은 “고유 번호”이므로 출력 대상이 아니다.
    • 정렬 후 출력 
      list.sort(String::compareTo);
      
      for (String s : list) {
          sb.append(s).append('\n');
      }
      System.out.print(sb);
      • list.sort(String::compareTo)로 문자열 기준 오름차순 정렬한다.
      • SSN이 XXX-XX-XXXX 형식으로 고정이라, 이 문자열 정렬이 곧 숫자 기준 정렬과도 같다.
      • 정렬된 리스트를 순서대로 출력 버퍼에 쌓고, 마지막에 한 번에 출력한다.

    시간 복잡도

    • SSN 개수를 N이라고 하면:
      • 입력 읽으면서 HashMap에 카운팅: 평균적으로 O(N).
      • 중복 SSN 개수를 D라고 할 때, 리스트 정렬: O(D log D).

    전체 시간 복잡도는 대략 O(N + D log D)로, 실질적으로는 N에 비례하는 수준이다.

    공간 복잡도

    • map에 최대 N개의 서로 다른 SSN이 저장될 수 있다 → O(N).
    • 중복 SSN 리스트 list에는 최대 N개(실제는 그보다 적은 수) 저장 → O(N).

    따라서 전체 공간 복잡도는 O(N)이다.

    주의

    • 종료 조건:
      • "000-00-0000"를 읽으면 바로 break해야 하고, 이 값을 map에 넣으면 안 된다.
    • 공백 처리:
      • line.trim()으로 앞뒤 공백을 제거해 SSN 비교가 정확히 이루어지도록 한다.
    • 정렬 기준:
      • SSN 형식이 고정이므로, 단순 문자열 비교(String::compareTo)만으로도 올바른 오름차순 정렬이 된다.

    포인트 요약

    • 모든 SSN을 해시맵으로 카운팅한다.
    • 카운트가 2 이상인 키(SSN)들만 따로 리스트에 모은다.
    • 리스트를 문자열 기준 오름차순 정렬 후, 한 줄에 하나씩 출력한다.
    • 종료 코드 000-00-0000는 입력 종료용일 뿐, 중복 후보에 포함되지 않는다.

    문제 5426 — 비밀 편지

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    비밀 편지 (BOJ 5426) 문제는
    암호화된 문자열이 주어졌을 때, 다음 과정을 역으로 적용해서 원래 문자열을 복원하는 문제다.

    암호화 과정은 다음과 같다.

    1. 편지 내용을 공백 없이 이어붙인 문자열을 만든다.
    2. 길이가 항상 정사각수(L = k²)이므로, k × k 정사각형 격자에 행 기준(왼→오, 위→아래)으로 채운다.
    3. 이 정사각형을 시계 방향 90도 회전한다.
    4. 회전된 정사각형을 다시 행 기준으로 읽어서 암호 문자열을 만든다.

    우리는 이 과정을 반대로 적용해 원래 문자열을 구해야 한다.
    즉, 암호 문자열을 k × k 정사각형에 채운 뒤, 시계 반대 방향(또는 시계 방향 3번)으로 90도 회전해서
    원래 격자를 복원하고, 그걸 행 기준으로 읽어주면 된다.

    전략 개요

    • 테스트 케이스 개수 n을 읽는다.
    • 각 테스트 케이스마다:
      1. 암호 문자열 s를 읽는다. 길이를 len이라고 하자.
      2. len = k²이 되는 정수 k를 구한다 (코드에서는 단순히 j * j ≤ len인 최대 j를 찾는다).
      3. k × k 2차원 배열 src를 만들고, 암호 문자열을 행 기준으로 채운다.
      4. src를 반시계 방향 90도 회전한 결과를 dst에 만든다.
        • 수식: dst[r][c] = src[c][k - 1 - r]
        • 이 변환은 원래 “시계방향 90도 회전된 결과”를 다시 원상복구하는 회전이다.
      5. dst를 행 기준으로 읽어 StringBuilder에 붙여 출력 문자열을 만든다.

    핵심: 코드가 하는 일

    • 테스트 케이스 수 입력 
      String line;
      while ((line = br.readLine()) != null) {
          line = line.trim();
      
          int n = Integer.parseInt(line);
      
          for (int i = 0; i < n; i++) {
              line = br.readLine();
      • 첫 줄에서 테스트 케이스 수 n을 읽고,
      • 각 테스트 케이스마다 암호 문자열 한 줄을 읽는다.
    • 정사각형 한 변의 길이 k 구하기 
              int length = 0;
              for (int j = 1; j * j <= line.length(); j++) {
                  length = j;
              }
      • length가 곧 k이다.
      • 문제에서 길이는 항상 제곱수이므로, 마지막으로 갱신된 j가 정확히 √len이 된다.
    • 암호 문자열을 k × k 격자에 채우기 
              int idx = 0;
              char[][] src = new char[length][length];
              for (int r = 0; r < length; r++) {
                  for (int c = 0; c < length; c++) {
                      src[r][c] = line.charAt(idx++);
                  }
              }
      • 암호 문자열 line을 위에서부터 차례대로,
        • src[0][0], src[0][1], ..., src[0][k-1], src[1][0], ... 순으로 채운다.
      • src는 “시계 방향으로 90도 회전된 상태의 격자”라고 생각할 수 있다.
    • 반시계 방향 90도 회전해서 원래 격자 복원 
              char[][] dst = new char[length][length];
              for (int r = 0; r < length; r++) {
                  for (int c = 0; c < length; c++) {
                      dst[r][c] = src[c][length - 1 - r];
                  }
              }
      • 회전 수식: dst[r][c] = src[c][length - 1 - r]
      • 이는 반시계 방향 90도 회전 변환이다.
      • 암호화가 “시계 방향 90도 회전”이었으므로,
        이 연산을 통해 원래 편지가 담긴 정사각형을 되살릴 수 있다.
    • 복원된 글자를 행 기준으로 출력 
              for (int r = 0; r < length; r++) {
                  for (int c = 0; c < length; c++) {
                      sb.append(dst[r][c]);
                  }
              }
      
              sb.append('\n');
      • 복원된 격자 dst를 위에서부터, 왼쪽에서 오른쪽으로 읽어,
        원래 편지 문자열을 StringBuilder에 붙인다.
      • 각 테스트 케이스마다 줄바꿈을 하나 추가한다.
    • 최종 출력 
      System.out.println(sb.toString());
      • 누적된 모든 결과를 한 번에 출력한다.

    시간 복잡도

    • 각 테스트 케이스에서 문자열 길이를 L, 한 변의 길이를 k = √L라고 할 때:
      • k 계산: j * j ≤ L를 도는 루프 → O(√L)
      • 배열 채우기: k × k = L → O(L)
      • 회전 및 다시 출력: 각각 O(L)

    전체적으로 한 테스트 케이스당 O(L) 정도로 볼 수 있다 (제곱근 루프는 상대적으로 작다).
    편지 길이 최대 10,000, 테스트 케이스 최대 100개이므로 충분히 여유 있다.

    공간 복잡도

    • src[k][k], dst[k][k] 두 개의 2차원 배열만 사용한다.
    • 각 배열의 크기는 k² = L이므로, 추가 공간은 O(L)이다.

    주의

    • 회전 방향:
      • 암호화는 “시계 방향 90도 회전” → 복호화는 반시계 방향 90도 회전이어야 한다.
      • 수식 dst[r][c] = src[c][k - 1 - r]가 정확히 이를 구현한다.
    • k 계산:
      • 문자열 길이가 항상 제곱수이기 때문에, 단순히 j * j ≤ len인 최대 j를 찾는 방식으로도 충분하다.
      • 정확하게는 k = (int)Math.sqrt(len)처럼 구현해도 된다.
    • 인덱스 범위:
      • idx는 0에서 len - 1까지 정확히 한 번씩 증가하며, src를 채운 뒤에는 더 이상 사용하지 않으므로 인덱스 에러가 없다.
      • 회전과 출력에서도 r, c는 항상 0 ≤ r, c < k 범위 내에서만 사용된다.

    포인트 요약

    • 암호화는 “정사각형에 채우기 → 시계 방향 90도 회전 → 행 기준 읽기” 과정이다.
    • 복호화는 이를 역순으로 “정사각형에 채우기 → 반시계 방향 90도 회전 → 행 기준 읽기”로 구현한다.
    • 암호 문자열을 k × k에 채운 뒤, dst[r][c] = src[c][k - 1 - r]로 회전하여 원래 격자를 복원한다.
    • 복원된 격자를 행 기준으로 읽어 붙이면 원래 편지 내용이 나온다.
