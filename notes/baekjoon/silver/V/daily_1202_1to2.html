    문제 5462 — POI

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    POI (BOJ 5462) 문제는
    참가자들의 정답 여부가 주어졌을 때, 각 문제의 점수와 각 참가자의 총점을 계산하고,
    주어진 규칙에 따라 특정 참가자 P의 점수와 등수를 구하는 문제다.

    • 문제는 부분 점수 없이 0/1로만 채점된다.
    • 각 문제의 점수는 그 문제를 풀지 못한 참가자의 수이다.
    • 각 참가자의 점수는 자신이 푼 문제들의 점수 합이다.
    • 등수는 다음 네 항의 합 + 1로 정의된다.
      1. 자신보다 점수가 높은 사람의 수
      2. 점수는 같지만 더 많은 문제를 푼 사람의 수
      3. 점수도 같고 푼 문제 수까지 같지만, ID가 더 작은 사람의 수
      4. 마지막에 + 1 (자기 자신)

    전략 개요

    • 입력 행렬 저장
      • solved[i][j] : ID가 i+1인 참가자가 문제 j+1을 풀었는지(1) 여부.
    • 각 참가자가 푼 문제 수, 각 문제를 푼 참가자 수를 센다.
      • solvedCntByPerson[i] : 참가자 i+1이 푼 문제 수.
      • solvedCntByProblem[j] : 문제 j+1을 푼 참가자 수.
    • 각 문제 점수 계산
      • 문제 j의 점수 = N - solvedCntByProblem[j] (즉, 그 문제를 못 푼 사람 수).
    • 각 참가자 총점 계산
      • totalScore[i] = 참가자 i+1이 푼 문제들의 점수 합.
    • P번 참가자 점수·푼 문제 수 가져오기
      • pIdx = P - 1로 0-based 인덱스로 변환.
      • pScore = totalScore[pIdx]
      • pSolved = solvedCntByPerson[pIdx]
    • 등수 계산
      • 초기 등수 rank = 1로 시작.
      • 모든 참가자 i에 대해:
        • i가 P가 아니면서
          • totalScore[i] > pScore 이면 → rank++.
          • totalScore[i] == pScore 이고
            • solvedCntByPerson[i] > pSolved 이면 → rank++.
            • solvedCntByPerson[i] == pSolved 이고 (i+1) < P (ID가 더 작으면) → rank++.
      • 이 규칙이 문제에서 정의한 등수 규칙 그대로이다.
    • 마지막으로 P의 점수와 등수를 출력한다.

    핵심: 코드가 하는 일

    • 입력 파싱 및 기본 배열 생성 
      StringTokenizer st = new StringTokenizer(line);
      int N = Integer.parseInt(st.nextToken());
      int T = Integer.parseInt(st.nextToken());
      int P = Integer.parseInt(st.nextToken());
      
      int[][] solved = new int[N][T];
      int[] solvedCntByPerson = new int[N];
      int[] solvedCntByProblem = new int[T];
      • solved : N × T 정답 행렬.
      • solvedCntByPerson : 각 참가자가 푼 문제 수.
      • solvedCntByProblem : 각 문제를 푼 참가자 수.
    • 참가자별 정답 입력 및 카운트 
      for (int i = 0; i < N; i++) {
          st = new StringTokenizer(br.readLine());
          for (int j = 0; j < T; j++) {
              int chk = Integer.parseInt(st.nextToken());
              solved[i][j] = chk;
      
              if (chk == 1) {
                  solvedCntByPerson[i]++;
                  solvedCntByProblem[j]++;
              }
          }
      }
      • 각 줄은 ID가 i+1인 참가자의 정답 여부.
      • chk == 1이면,
        • 참가자 i가 푼 문제 수 증가,
        • 해당 문제를 푼 사람 수 증가.
    • 문제별 점수 계산 
      int[] problemScore = new int[T];
      for (int i = 0; i < T; i++) {
          problemScore[i] = N - solvedCntByProblem[i];
      }
      • 문제 i의 점수 = N - solvedCntByProblem[i], 즉 못 푼 사람 수.
    • 참가자별 총점 계산 
      int[] totalScore = new int[N];
      for (int i = 0; i < N; i++) {
          int score = 0;
      
          for (int j = 0; j < T; j++) {
              if (solved[i][j] == 1) {
                  score += problemScore[j];
              }
          }
      
          totalScore[i] = score;
      }
      • 참가자 i가 푼 문제들에 대해서만 문제 점수를 더해 총점을 만든다.
      • 이 값이 totalScore[i]가 된다.
    • 필립(P)의 점수·푼 문제 수 및 등수 계산 
      int rank = 1;
      int pIdx = P - 1;
      int pScore = totalScore[pIdx];
      int pSolved = solvedCntByPerson[pIdx];
      
      for (int i = 0; i < N; i++) {
          if (i == pIdx) continue;
      
          if (totalScore[i] > pScore) {
              rank++;
          } else if (totalScore[i] == pScore) {
              if (solvedCntByPerson[i] > pSolved) {
                  rank++;
              } else if (solvedCntByPerson[i] == pSolved && (i + 1) < P) {
                  rank++;
              }
          }
      }
      • pIdx는 P의 0-based 인덱스.
      • pScore, pSolved는 필립의 점수와 푼 문제 수.
      • 모든 다른 참가자 i에 대해:
        • 점수가 더 크면 → 필립보다 앞으로 감 → rank++.
        • 점수가 같으면:
          • 더 많은 문제를 풀었으면 → rank++.
          • 문제 수도 같고 ID(i+1) < P (ID가 더 작으면) → rank++.
      • 이 카운트에 +1을 한 값이 문제 정의 그대로의 등수가 된다.
        (코드에서는 rank를 1에서 시작해 비교할 때마다 증가시키는 방식으로 구현했다.)
    • 최종 출력 
      sb.append(pScore).append(" ").append(rank);
      System.out.println(sb.toString());
      • 필립의 점수와 등수를 공백으로 구분해 한 줄로 출력한다.

    시간 복잡도

    • 정답 행렬 입력 및 카운트:
      • 크기 N × T에 대해 한 번씩 읽고 처리 → O(N × T)
    • 문제 점수 계산:
      • T개에 대해 선형 → O(T)
    • 참가자 총점 계산:
      • 다시 N × T를 한 번 순회 → O(N × T)
    • 등수 계산:
      • 참가자 N명에 대해 한 번씩 비교 → O(N)

    전체 시간 복잡도는 O(N × T)이며,
    N, T ≤ 2,000이므로 최악의 경우에도 충분히 처리 가능하다.

    공간 복잡도

    • solved : N × T 크기의 2차원 배열.
    • solvedCntByPerson, solvedCntByProblem, problemScore, totalScore : 길이 N 또는 T의 1차원 배열들.

    전체 공간 복잡도는 O(N × T) 수준이다.

    주의

    • 문제 점수 정의:
      • “풀지 못한 사람 수”가 점수다. 실수로 푼 사람 수를 넣지 않도록 주의해야 한다.
    • 인덱스와 ID 차이:
      • ID는 1부터 시작, 배열 인덱스는 0부터 시작하므로,
        비교 시 (i + 1) < P처럼 맞춰줘야 한다.
    • 등수 조건 순서:
      • 먼저 점수 비교 → 점수가 같을 때만 푼 문제 수 비교 → 그마저 같을 때 ID 비교 순서를 반드시 지켜야 한다.
    • 테스트 케이스 루프:
      • 문제 자체는 한 번의 입력만 가정하지만, 코드에서는 while ((line = br.readLine()) != null)로 여러 번 입력을 받아도 처리할 수 있도록 구현되어 있다.

    포인트 요약

    • 각 참가자·문제에 대해 정답 여부를 저장하고, 행·열 방향으로 푼 문제 수를 집계한다.
    • 각 문제 점수는 “그 문제를 틀린 사람 수”로 계산한다.
    • 각 참가자의 총점은 자신이 맞힌 문제들의 점수를 합한 값이다.
    • P번 참가자의 점수와 푼 문제 수를 기준으로, 모든 참가자를 한 번씩 비교해 등수를 계산한다.
    • 점수 → 푼 문제 수 → ID 순으로 비교하는 것이 문제 정의와 정확히 일치한다.

    문제 5555 — 반지

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    반지 (BOJ 5555) 문제는
    찾고자 하는 문자열이 주어졌을 때, 각 반지에 새겨진 문자열을 원형(시작과 끝이 이어진 상태)으로 보면서
    그 문자열을 부분 문자열로 포함하는 반지의 개수를 세는 문제다.

    • 타겟 문자열 길이: 1 ~ 10
    • 각 반지 문자열 길이: 항상 10
    • 반지 문자열은 원형이므로, 끝과 처음이 이어져 있다고 생각해야 한다.
    • 거꾸로 읽는 경우는 고려하지 않아도 된다 (문제에서 걱정할 필요 없다고 명시).

    원형 문자열에서의 부분 문자열 검사는
    문자열을 한 번 더 이어 붙여 ring + ring 으로 만든 뒤,
    이 선형 문자열 안에 타겟 문자열이 포함되는지만 확인하면 간단하게 해결할 수 있다.

    전략 개요

    • 찾고자 하는 문자열 target을 입력으로 받는다.
    • 반지 개수 N을 입력으로 받는다.
    • 각 반지 문자열 ring에 대해:
      • doubled = ring + ring 형태로 문자열을 2번 이어 붙인다.
      • doubled 안에 target이 포함되는지 contains()로 검사한다.
      • 포함되면 해당 반지는 조건을 만족하므로 카운트를 1 증가시킨다.
    • 모든 반지를 검사한 뒤, 카운트 값을 출력한다.

    원형에서 길이 ≤ 10인 연속 부분 문자열은 모두
    ring + ring 이라는 선형 문자열 안의 연속 부분 문자열로 등장하므로,
    이렇게 검사해도 놓치는 경우가 없다.

    핵심: 코드가 하는 일

    • 입력 처리 
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      
      String target = br.readLine().trim();
      int N = Integer.parseInt(br.readLine());
      • 첫 줄: 찾고자 하는 문자열 target.
      • 둘째 줄: 반지 개수 N.
    • 각 반지에 대해 원형 포함 여부 검사 
      int cnt = 0;
      for (int i = 0; i < N; i++) {
          String ring = br.readLine().trim();
          String doubled = ring + ring;
      
          if (doubled.contains(target)) {
              cnt++;
          }
      }
      • ring은 길이 10의 대문자 문자열이다.
      • doubled = ring + ring:
        • 예: ring = "ABCDXXXXXX"이면
          doubled = "ABCDXXXXXXABCDXXXXXX"
        • 원형에서 시작점을 어디로 잡든, 길이 ≤ 10인 연속 부분 문자열은
          이 doubled 안의 연속 부분 문자열로 표현할 수 있다.
      • doubled.contains(target):
        • 자바 String의 내장 메서드를 이용해 부분 문자열 여부를 간단히 판단.
        • 포함되면 해당 반지는 조건을 만족 → cnt++.
    • 결과 출력 
      System.out.println(cnt);
      • 조건을 만족하는 반지의 총 개수를 한 줄에 출력한다.

    시간 복잡도

    • 반지 개수: N ≤ 100
    • 각 반지 문자열 길이: 10, doubled 길이: 20
    • contains()는 길이 20짜리 문자열에서 길이 ≤ 10짜리 문자열을 찾는 작업이므로 사실상 상수 시간.

    전체 시간 복잡도는 대략 O(N) 수준이다.
    상한이 매우 작으므로 성능 문제는 전혀 없다.

    공간 복잡도

    • 추가로 사용하는 것은
      • target, ring, doubled 문자열과
      • 정수 카운터 cnt뿐이다.

    입력 문자열들을 제외한 추가 공간은 O(1)이다.

    주의

    • 원형 처리:
      • 그냥 ring.contains(target)만 쓰면, 문자열이 끝을 넘어 이어지는 경우를 잡지 못한다.
      • 반드시 ring + ring을 만들어서 검사해야 한다.
    • 입력 공백 처리:
      • 문자열은 공백 없이 대문자만 주어지지만, trim()으로 양 끝 공백을 제거하는 습관이 안전하다.
    • 역순(거꾸로) 고려 불필요:
      • 문제에서 “거꾸로 읽는 걱정은 없다”고 했으므로, 역순으로 포함되는 경우는 신경 쓰지 않아도 된다.
      • 즉, 주어진 방향 그대로만 검사하면 된다.

    포인트 요약

    • 반지 문자열은 원형이므로, ring + ring으로 선형화해서 처리한다.
    • 각 반지에 대해 (ring + ring).contains(target)이면 카운트 증가.
    • 모든 반지를 검사한 뒤 카운트를 출력한다.
    • 원형 처리만 올바르게 하면 구현은 매우 간단한 문자열 문제다.
