    문제 5599 — カードの並び替え

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    カードの並び替え (BOJ 5599) 문제는
    처음에 위에서부터 1, 2, ..., 2n 순서로 쌓인 카드 더미에 대해,
    주어진 횟수만큼 두 가지 연산(카트/cut, 리플 셔플)을 순서대로 적용한 뒤,
    마지막 카드 배열 상태를 위에서부터 차례대로 출력하는 시뮬레이션 문제다.

    • 카드 수는 항상 2n장.
    • 연산은 총 m번 주어진다.
    • 연산 종류:
      • k = 0 → 리플 셔플 (Riffle Shuffle)
      • 1 ≤ k ≤ 2n - 1 → k로 컷 (Cut)

    문제 조건이 모두 작기 때문에, 각 연산을 그대로 배열에 시뮬레이션해도 충분히 빠르게 해결할 수 있다.

    전략 개요

    • 처음 카드 상태를 1 ~ 2n까지 순서대로 배열에 채운다.
    • 각 연산 k에 대해:
      • k = 0 (리플 셔플):
        • 위에서 n장 → 더미 A
        • 나머지 n장 → 더미 B
        • 새로운 카드 순서: A[0], B[0], A[1], B[1], ..., A[n-1], B[n-1]
      • k > 0 (cut):
        • 위에서 k장 → 더미 A (card[0..k-1])
        • 나머지 → 더미 B (card[k..2n-1])
        • 새로운 카드 순서: 먼저 B, 그 뒤 A
          즉, B[0..] + A[0..] = card[k..] + card[0..k-1]
    • 모든 연산을 순서대로 적용한 뒤, 최종 card 배열을 위에서부터 하나씩 출력한다.

    구현 자체는 단순한 배열 재배치이므로, 각 연산마다 새 배열을 만들어 복사해도 충분하다.

    핵심: 코드가 하는 일

    • 초기 카드 세팅 
      int n = Integer.parseInt(br.readLine());
      
      int[] card = new int[2 * n];
      for (int i = 0; i < card.length; i++) {
          card[i] = i + 1;
      }
      • 카드 수는 항상 2n장이므로 길이 2n짜리 배열을 만든다.
      • card[0] = 1, card[1] = 2, ..., card[2n-1] = 2n 으로 초기화.
    • 연산 개수 및 각 연산 처리 
      int m = Integer.parseInt(br.readLine());
      for (int i = 0; i < m; i++) {
          int k = Integer.parseInt(br.readLine());
      
          if (k == 0) {
              // 리플 셔플
              int[] A = new int[n];
              int[] B = new int[n];
      
              for (int j = 0; j < n; j++) {
                  A[j] = card[j];
                  B[j] = card[n + j];
              }
      
              int idx = 0;
              for (int j = 0; j < n; j++) {
                  card[idx++] = A[j];
                  card[idx++] = B[j];
              }
          } else {
              // k로 컷
              int[] A = Arrays.copyOfRange(card, 0, k);
              int[] B = Arrays.copyOfRange(card, k, card.length);
      
              int[] next = new int[card.length];
              int idx = 0;
      
              for (int j = k; j < card.length; j++) {
                  next[idx++] = card[j];
              }
      
              for (int j = 0; j < k; j++) {
                  next[idx++] = card[j];
              }
      
              card = next;
          }
      }
      • k = 0 (리플 셔플)
        • 위에서 n장과 나머지 n장을 A, B에 따로 복사한다.
        • 그 다음, card 배열을 다시 채우면서
          A[0], B[0], A[1], B[1], ..., A[n-1], B[n-1] 순으로 interleave 한다.
      • k > 0 (cut)
        • A = card[0..k-1], B = card[k..2n-1]로 나누는 의미의 코드가 들어있지만,
          실제로는 A, B 배열은 이후에 직접 사용하지 않고, card를 바로 재배열한다.
        • next 배열에
          • 먼저 card[k..] (더미 B 전체)를 채우고,
          • 그 다음 card[0..k-1] (더미 A 전체)를 이어 붙인다.
        • 마지막에 card = next;로 갱신해 다음 연산부터는 새 순서를 기준으로 진행한다.
        • 이 동작이 “A 위에 B를 올려서, 최종적으로 B가 위쪽에 오게 하는” 컷 연산과 동일하다.
    • 최종 출력 
      for (int i : card) {
          sb.append(i).append('\n');
      }
      
      System.out.println(sb.toString());
      • 마지막 상태의 card 배열을 위에서부터 순서대로 한 줄씩 출력한다.
      • 출력 형식: 1행에 맨 위 카드 번호, 2행에 2번째 카드 번호, ...

    시간 복잡도

    • 카드 수: 2n ≤ 200
    • 연산 수: m ≤ 1000
    • 각 연산마다 카드 배열 전체를 한 번 훑으면서 재배열 → O(2n)

    전체 시간 복잡도는 대략 O(m × n)이며,
    최악의 경우에도 1000 × 100 수준이라 충분히 빠르다.

    공간 복잡도

    • 기본 카드 배열 card[2n]
    • 리플 셔플 시 사용되는 A[n], B[n]
    • 컷 연산 시 임시 배열 next[2n]

    모두 합쳐도 카드 수에 비례하므로 공간 복잡도는 O(n)이다.

    주의

    • 카드 인덱스:
      • 카드는 문제에서 1 ~ 2n이지만, 배열 인덱스는 0 ~ 2n-1 이므로 항상 이 차이를 염두에 둔다.
    • 컷 연산 방향:
      • 문제 설명은 “위에서 k장을 A, 나머지를 B로 나누고, A의 위에 B를 올린다”고 되어 있지만,
        결과적으로 B가 더 위에 오게 된다.
      • 코드의 next[idx++] = card[j]; (j = k..) 후 j = 0..k-1 순으로 이어 붙이는 구현이 바로 이 의미다.
    • 리플 셔플 정의를 정확히 구현:
      • 반드시 A1, B1, A2, B2, ... 순으로 쌓아야 한다.
      • A만 먼저 몰아서 쌓거나, B 먼저 쌓는 실수를 하면 결과가 달라진다.
    • 여러 연산의 누적:
      • 각 연산은 현재 상태의 카드 순서를 기준으로 수행해야 한다.
      • 따라서 매번 card 배열을 업데이트한 뒤, 그 상태로 다음 연산을 진행해야 한다.

    포인트 요약

    • 초기 카드 상태는 1 ~ 2n 순서로 배열에 채운다.
    • k = 0: 위 n장, 아래 n장을 나누어 A, B로 두고 A1, B1, A2, B2, ... 순으로 interleave 한다.
    • k > 0: 위 k장과 나머지를 A, B로 나누고, card = B + A 형태로 재배열한다.
    • 모든 연산을 순서대로 적용한 뒤, 최종 카드 배열을 위에서부터 그대로 출력한다.
    • 단순한 배열 시뮬레이션만으로 해결 가능하며, 시간·공간 복잡도 모두 매우 작다.

    문제 5635 — 생일

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    생일 (BOJ 5635) 문제는
    반 학생들의 이름과 생년월일이 주어졌을 때,
    가장 나이가 적은 학생과 가장 나이가 많은 학생의 이름을 출력하는 문제다.

    • 입력 형식: 이름 dd mm yyyy
    • 날짜는 모두 유효하고, 이름도 생일도 서로 겹치지 않는다.
    • 출력:
      • 1행: 가장 나이가 적은(= 생일이 가장 늦은) 학생 이름
      • 2행: 가장 나이가 많은(= 생일이 가장 이른) 학생 이름

    생년월일 기준으로 정렬만 잘 하면 바로 해결되는 단순 정렬 문제다.

    전략 개요

    • 학생 정보를 담는 클래스(또는 구조체) Info를 만든다.
      • name : 이름
      • dd, mm, yyyy : 생일 (일, 월, 연도)
    • 모든 학생을 List<Info>에 저장한다.
    • 생년월일 기준으로 오름차순 정렬한다.
      • 연도(yyyy)가 작은 사람이 먼저
      • 연도가 같으면 월(mm)이 작은 사람이 먼저
      • 연, 월이 같으면 일(dd)이 작은 사람이 먼저
    • 이렇게 정렬하면:
      • students.get(0) : 생일이 가장 이른 사람 → 가장 나이가 많음
      • students.get(students.size() - 1) : 생일이 가장 늦은 사람 → 가장 나이가 적음
    • 문제 요구대로:
      • 1행: 가장 나이가 적은 사람 → 리스트의 마지막 원소 이름
      • 2행: 가장 나이가 많은 사람 → 리스트의 첫 번째 원소 이름

    핵심: 코드가 하는 일

    • 입력과 저장 
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      StringBuilder sb = new StringBuilder();
      List<Info> students = new ArrayList<>();
      
      int n = Integer.parseInt(br.readLine());
      for (int i = 0; i < n; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          String name = st.nextToken();
          int dd = Integer.parseInt(st.nextToken());
          int mm = Integer.parseInt(st.nextToken());
          int yyyy = Integer.parseInt(st.nextToken());
      
          students.add(new Info(name, dd, mm, yyyy));
      }
      • 학생 수 n을 읽고,
      • 각 줄에서 이름, 일, 월, 연도를 차례대로 파싱해서 Info 객체로 리스트에 추가한다.
    • 생년월일 기준 정렬 
      Collections.sort(students, (a, b) -> {
          if (a.yyyy == b.yyyy) {
              if (a.mm == b.mm) {
                  return Integer.compare(a.dd, b.dd);
              } else {
                  return Integer.compare(a.mm, b.mm);
              }
          } else {
              return Integer.compare(a.yyyy, b.yyyy);
          }
      });
      • yyyy → mm → dd 순으로 비교한다.
      • 모두 오름차순이므로:
        • 가장 먼저 오는 학생이 생일이 가장 빠른(→ 가장 나이 많은) 학생.
        • 가장 마지막 학생이 생일이 가장 늦은(→ 가장 나이 적은) 학생.
    • 정답 출력 
      sb.append(students.get(students.size() - 1).name).append('\n');
      sb.append(students.get(0).name);
      
      System.out.println(sb.toString());
      • students.size() - 1 : 가장 나이가 적은 사람 (생일이 가장 늦음).
      • 0 : 가장 나이가 많은 사람 (생일이 가장 이름).
      • 문제에서 요구한 순서대로 두 줄을 출력한다.
    • 도움 클래스 
      private static class Info {
          String name;
          int dd;
          int mm;
          int yyyy;
      
          public Info(String name, int dd, int mm, int yyyy) {
              this.name = name;
              this.dd = dd;
              this.mm = mm;
              this.yyyy = yyyy;
          }
      }
      • 학생 정보를 하나로 묶어서 다루기 위한 단순한 데이터 클래스.

    시간 복잡도

    • 학생 정보를 읽어 리스트에 넣는 데 O(n).
    • Collections.sort로 n명을 정렬: O(n log n).
    • 이후 인덱스로 접근하고 출력하는 부분은 O(n).

    전체 시간 복잡도는 O(n log n)이며,
    n ≤ 100이라 매우 여유롭다.

    공간 복잡도

    • List<Info>에 학생 n명의 정보를 저장 → O(n) 공간.
    • 추가로 사용하는 변수들은 모두 상수 크기.

    따라서 공간 복잡도는 O(n)이다.

    주의

    • 정렬 기준 방향:
      • 생일이 빠를수록(연도가 작을수록) 나이가 많다.
      • 그래서 “가장 나이 많은 사람”은 정렬 후 0번, “가장 나이 적은 사람”은 size-1 인덱스를 사용해야 한다.
    • 이름/생일 중복 없음:
      • 문제에서 이름과 생일 모두 중복되지 않는다고 했으므로, 같은 날짜/이름에 대한 추가 처리는 필요 없다.
    • 입력 형식:
      • 날짜에 0으로 시작하는 경우가 없으므로, 그대로 int로 파싱하면 된다.

    포인트 요약

    • 학생 정보를 Info 객체에 담아 리스트로 관리한다.
    • 연도 → 월 → 일 기준으로 오름차순 정렬한다.
    • 정렬 후:
      • 리스트 마지막 원소: 생일이 가장 늦음 → 가장 나이가 적은 학생.
      • 리스트 첫 번째 원소: 생일이 가장 이름 → 가장 나이가 많은 학생.
    • 요구 출력 순서에 맞게 “어린 → 많은” 순으로 이름을 출력하면 된다.
