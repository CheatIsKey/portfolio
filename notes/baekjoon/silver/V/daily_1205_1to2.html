    문제 5648 — 역원소 정렬

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    역원소 정렬 (BOJ 5648) 문제는
    양의 정수들로 이루어진 집합이 주어졌을 때, 각 원소의 자릿수를 거꾸로 뒤집은 값을 구하고,
    그 뒤집힌 값들을 오름차순 정렬해서 출력하는 문제다.

    • 각 원소는 양의 정수이고, 값은 1012 이하.
    • 뒤집었을 때 맨 앞에 오는 0은 모두 생략해야 한다.
      • 예: 1200 → 뒤집기: 0021 이지만, 수로는 21.
    • 입력은 첫 줄에 n이 나오고, 이후에는 여러 줄에 걸쳐 공백으로 구분된 정수들이 이어서 나온다.

    핵심은 각 숫자를 하나씩 읽어서 자릿수를 뒤집은 값을 만든 뒤 리스트에 넣고,
    마지막에 정렬 후 그대로 출력하면 된다.

    전략 개요

    • 입력 처리
      • 첫 줄에서 n을 읽어서, 총 몇 개의 정수를 읽어야 하는지 결정한다.
      • 이후 입력은 한 줄에 여러 개의 숫자가 섞여 있을 수 있으므로, StringTokenizer를 사용해 토큰 단위로 끊어서 읽는다.
      • 읽은 숫자 개수가 n개가 될 때까지 계속 줄을 읽고 토큰을 소비한다.
    • 자릿수 뒤집기
      • 각 숫자를 long으로 읽어 온 뒤, 나눗셈/나머지 연산으로 자릿수를 뒤집는다.
      • 예: num = 1601라면
        • res = 0에서 시작하여,
        • res = res * 10 + (num % 10)을 반복하면서 num /= 10 한다.
        • 그 결과 res = 1061이 된다.
      • 이 방식은 앞에 오는 0들을 자동으로 제거해 준다.
        • 1200 → 뒤집기 과정: 0 → 0 → 2 → 21 → 21.
    • 정렬 및 출력
      • 뒤집힌 숫자들을 List<Long>에 담는다.
      • 모두 수집한 뒤 오름차순 정렬을 수행한다.
      • 정렬된 리스트를 순서대로 출력한다.

    핵심: 코드가 하는 일

    • 첫 줄에서 n 읽기 
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      StringBuilder sb = new StringBuilder();
      
      StringTokenizer st = new StringTokenizer(br.readLine());
      long n = Long.parseLong(st.nextToken());
      
      List<Long> list = new ArrayList<>();
      int count = 0;
      • n : 앞으로 읽어야 할 전체 숫자 개수.
      • list : 뒤집힌 숫자들을 저장하는 리스트.
      • count : 지금까지 읽은 숫자 개수를 추적.
    • 여러 줄에 걸친 숫자 읽기 + 뒤집기 
      while (count < n) {
          if (!st.hasMoreTokens()) {
              st = new StringTokenizer(br.readLine());
              continue;
          }
      
          String s = st.nextToken();
          long num = Long.parseLong(s);
          long res = 0;
      
          while (num > 0) {
              res = res * 10 + (num % 10);
              num /= 10;
          }
      
          list.add(res);
          count++;
      }
      • while (count < n):
        • 총 n개의 숫자를 다 읽을 때까지 반복.
      • !st.hasMoreTokens():
        • 현재 줄의 토큰을 다 썼다면 다음 줄을 읽어 새로운 StringTokenizer로 바꾼다.
      • s = st.nextToken() → num = Long.parseLong(s):
        • 다음 정수를 문자열 토큰으로 받아서 long으로 변환.
      • res 계산:
        • num % 10으로 가장 마지막 자릿수를 뽑고,
        • res = res * 10 + (num % 10)으로 뒤집힌 수를 만들어 간다.
        • 매번 num /= 10으로 한 자리씩 줄이며, num == 0이 될 때까지 반복.
        • 이 과정에서 앞자리가 0이 되는 구조는 자연스럽게 사라진다.
      • list.add(res):
        • 뒤집힌 값을 리스트에 저장.
      • count++:
        • 처리한 숫자 개수를 1 증가시켜 전체 n개를 채웠는지 확인.
    • 정렬과 출력 
      list.sort(Long::compare);
      
      for (Long l : list) {
          sb.append(l).append('\n');
      }
      
      System.out.println(sb.toString());
      • list.sort(Long::compare):
        • Long 값 기준 오름차순 정렬을 수행.
      • 정렬이 끝난 뒤, 리스트의 각 원소를 한 줄씩 출력 버퍼에 쌓고 마지막에 한 번에 출력한다.

    시간 복잡도

    • 숫자 n개에 대해 자릿수를 뒤집는 과정:
      • 각 숫자의 자릿수는 최대 13자리 정도(1012)이므로, 뒤집기 연산은 O(자릿수) = 상수 수준.
      • 전체로 보면 O(n)에 가깝다.
    • 정렬:
      • n개의 Long 값을 정렬하므로 O(n log n).

    전체 시간 복잡도는 O(n log n)이며,
    n ≤ 106 조건에서 충분히 가능한 수준이다.

    공간 복잡도

    • List<Long>에 뒤집힌 숫자 n개를 저장 → O(n).
    • 그 외에는 입력 처리용 토크나이저와 몇 개의 보조 변수만 사용하므로 추가 공간은 상수.

    따라서 공간 복잡도는 O(n)이다.

    주의

    • 여러 줄 입력 처리:
      • 첫 줄에서 n만 읽은 뒤, 나머지 토큰들은 다시 사용하는 구조이므로
      • 반드시 while (count < n) 루프에서 st.hasMoreTokens()를 확인하고, 없으면 br.readLine()으로 새 줄을 가져와야 한다.
    • 앞자리 0 제거:
      • 문자열을 뒤집고 다시 Long.parseLong을 하는 방식 대신, 현재 코드처럼 num을 10으로 나누어가며 res를 만드는 방식이면 자연스럽게 앞자리 0이 제거된다.
    • 자료형 범위:
      • 입력 상한이 1012이므로 long으로 충분히 표현 가능하다.
      • 뒤집은 값 역시 최대 13자리 이내이므로 역시 long 범위 안에 있다.

    포인트 요약

    • 총 n개의 정수를 공백 기준으로 순서대로 읽어야 하므로 토크나이저를 적절히 재사용한다.
    • 각 수를 long으로 읽고, 나머지/나눗셈을 이용해 자릿수를 뒤집는다.
    • 뒤집힌 값들을 리스트에 모아 오름차순 정렬 후 그대로 출력한다.
    • 자릿수 뒤집기 과정에서 자동으로 앞자리 0이 제거되므로 별도 처리 필요 없다.

    문제 5671 — 호텔 방 번호

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    호텔 방 번호 (BOJ 5671) 문제는
    구간 [N, M] 안에 있는 정수들 중에서
    어떤 숫자도 두 번 이상 등장하지 않는 방 번호가 몇 개인지를 세는 문제다.

    • 예를 들어, 838, 1004처럼 같은 숫자가 두 번 이상 들어 있으면 불운한 번호로 간주하고 제외한다.
    • 123, 507처럼 모든 자릿수가 서로 다르면 허용되는 방 번호이다.
    • 입력은 여러 테스트 케이스가 한 줄씩 주어지고, 각 줄에 N, M이 공백으로 구분되어 주어진다.

    범위가 1 ≤ N ≤ M ≤ 5000으로 매우 작기 때문에,
    구간 내의 모든 정수에 대해 직접 자릿수를 검사하는 완전탐색으로 충분히 해결할 수 있다.

    전략 개요

    • 각 테스트 케이스에 대해 N부터 M까지 모든 수 i를 순회한다.
    • 각 수 i에 대해:
      • 길이 10짜리 boolean[] used 배열을 준비한다. (인덱스 0~9 → 각 자릿수 숫자 사용 여부)
      • 자릿수를 하나씩 떼어내면서:
        • 해당 숫자가 이미 used에 true라면 → 같은 숫자가 두 번 이상 등장 → 이 수는 불허.
        • 처음 보는 숫자라면 used[digit] = true로 체크하고 계속 진행.
      • 모든 자릿수를 검사했는데도 중복이 한 번도 없었다면 → 유효한 방 번호로 카운트.
    • 한 테스트 케이스에 대한 결과(유효한 번호 개수)를 출력 버퍼에 기록하고, 다음 줄(다음 테스트 케이스)을 처리한다.

    숫자 범위가 작기 때문에, 각 수의 자릿수는 최대 4자리(5000)이고,
    전체 연산량도 매우 적어 단순 구현으로 충분하다.

    핵심: 코드가 하는 일

    • 기본 입출력 세팅 
      BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
      StringBuilder sb = new StringBuilder();
      
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
          int N = Integer.parseInt(st.nextToken());
          int M = Integer.parseInt(st.nextToken());
      
          int cnt = 0;
      • EOF(입력 끝)까지 한 줄씩 읽으면서 각 줄을 하나의 테스트 케이스로 처리한다.
      • N, M을 파싱해서 현재 구간을 설정한다.
      • cnt는 이 구간에서 중복 없는 번호 개수를 세는 변수.
    • 각 숫자에 대한 중복 자릿수 검사 
          for (int i = N; i <= M; i++) {
              boolean[] used = new boolean[10];
              boolean ok = true;
      
              int num = i;
              while (num > 0) {
                  int tmp = num % 10;
      
                  if (used[tmp]) {
                      ok = false;
                      break;
                  }
      
                  used[tmp] = true;
                  num /= 10;
              }
      
              if (ok) cnt++;
          }
      
          sb.append(cnt).append('\n');
      }
      • for (int i = N; i <= M; i++):
        • 구간 [N, M]의 모든 정수에 대해 검사.
      • boolean[] used = new boolean[10];:
        • 0부터 9까지 각 자릿수가 이미 등장했는지를 기억하는 배열.
        • 매 숫자 i마다 새 배열을 만들어 초기화한다.
      • int num = i; / while (num > 0):
        • 현재 숫자 i의 자릿수들을 하나씩 떼어내는 과정.
        • tmp = num % 10 → 가장 오른쪽 자릿수.
        • num /= 10 → 자릿수를 하나 줄인다.
      • if (used[tmp]):
        • 이 자릿수 tmp가 이미 등장한 적이 있다면, 같은 숫자가 두 번 이상 나온 것이므로 ok = false로 표시 후 중단.
      • used[tmp] = true;:
        • 처음 등장한 숫자라면 등장 여부를 기록한다.
      • 루프 종료 후 ok가 true라면:
        • 중복된 자릿수가 없다는 뜻이므로 cnt++로 유효한 번호를 하나 더 센다.
      • 각 테스트 케이스가 끝나면 sb.append(cnt).append('\n');로 결과를 기록한다.
    • 최종 출력 
      System.out.println(sb.toString());
      • 모든 테스트 케이스에 대한 결과를 한 번에 출력한다.

    시간 복잡도

    • 각 테스트 케이스에서, 검사하는 숫자의 개수는 최대 M - N + 1 ≤ 5000.
    • 각 숫자의 자릿수는 최대 4자리(5000 기준)이므로, 자릿수 검사 비용은 O(1)에 가깝다.

    따라서 한 테스트 케이스에 대한 시간 복잡도는 O(M - N + 1),
    최악의 경우에도 충분히 빠르게 동작한다.

    공간 복잡도

    • 각 숫자마다 길이 10짜리 boolean[] used 배열을 사용 → O(10) = O(1).
    • 입력 전체를 저장하지 않고 한 줄씩 처리하므로, 추가 공간 사용은 상수 수준이다.

    전체 추가 공간 복잡도는 O(1)이다.

    주의

    • 여러 테스트 케이스:
      • 입력 끝(EOF)까지 while ((line = br.readLine()) != null)로 반복한다.
      • 각 줄마다 N, M 쌍을 하나의 테스트 케이스로 처리해야 한다.
    • 0 처리:
      • 문제 조건에서 N ≥ 1이므로 실제로 0을 검사할 일은 없다.
      • 참고로 0을 검사한다고 해도 while 루프를 안 타서 중복 없이 유효한 숫자로 취급된다.
    • used 배열 초기화:
      • 각 숫자마다 새로운 boolean[10]을 만들어야 한다.
      • 공유해서 쓰다가 초기화를 빼먹으면 이전 숫자의 기록이 남아 오답이 된다.

    포인트 요약

    • 각 테스트 케이스마다 구간 [N, M]의 모든 정수에 대해 자릿수를 검사한다.
    • boolean[10] 배열로 각 숫자(0~9)의 등장 여부를 체크해, 중복이 발생하면 바로 탈락시킨다.
    • 중복 없이 끝까지 통과한 숫자만 카운트해 결과로 출력한다.
    • 범위가 작아서 단순 완전탐색으로 충분히 해결 가능한 문제다.
