    문제 5800 — 성적 통계

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    성적 통계 (BOJ 5800) 문제는
    각 반의 수학 점수가 주어졌을 때, 반별로

    • 최고 점수(Max)
    • 최저 점수(Min)
    • 정렬했을 때 인접한 점수들 사이의 최대 차이(Largest gap)

    를 구해서 출력하는 문제다.

    각 반의 점수들을 내림차순으로 정렬한 뒤,
    인접한 두 점수의 차이들 중에서 가장 큰 값을 구하면 된다.

    문제 처리 순서

    1. 먼저 K (반의 개수)를 입력받는다.
    2. 각 반마다 다음을 반복한다.
      • 한 줄을 읽어 StringTokenizer로 나눈다.
      • 첫 토큰: 학생 수 N
      • 이후 N개의 토큰: 각 학생의 점수 → list에 추가
    3. 해당 반의 점수 리스트 list를 내림차순으로 정렬한다. 
      Collections.sort(list, (a, b) -> Integer.compare(b, a));
    4. 정렬된 상태에서
      • 최고 점수: 리스트의 첫 원소 list.get(0)
      • 최저 점수: 리스트의 마지막 원소 list.get(list.size() - 1)
      • Largest gap: 인접한 점수 차이의 최댓값 
        int max = 0;
      for (int j = 0; j < list.size() - 1; j++) {
          int minus = list.get(j) - list.get(j + 1);  // 내림차순이므로 항상 0 이상
          if (minus > max) max = minus;
      }
    5. 해당 반에 대해 
      Class X
      Max M, Min m, Largest gap G
      형식으로 출력한다.
    6. 다음 반을 위해 list.clear()로 리스트를 비우고 재활용한다.

    이렇게 하면 각 반마다 요구하는 세 값을 모두 얻을 수 있다.

    핵심: 코드가 하는 일

    • 입력 처리
      • K를 읽고, K번 반복
      • 각 반에 대해, 첫 토큰은 버리고 나머지 점수들만 list에 저장
    • 정렬 및 통계 계산
      • Collections.sort(list, (a, b) -> Integer.compare(b, a)); 로 내림차순 정렬
      • 최고 점수: list.get(0)
      • 최저 점수: list.get(list.size() - 1)
      • Largest gap: 인접한 두 점수의 차이 최댓값
    • 출력 형식 맞추기
      • Class i (1부터 시작)
      • Max ..., Min ..., Largest gap ... 한 줄에 출력
    • 리스트 재활용
      • 각 반 처리 후 list.clear()를 호출해 다음 반에서 다시 사용

    예시

    • 입력: 5 30 25 76 23 78

      • 점수들: [30, 25, 76, 23, 78]
      • 내림차순 정렬: [78, 76, 30, 25, 23]
      • Max = 78, Min = 23
      • 인접 차이:
        • 78 - 76 = 2
        • 76 - 30 = 46
        • 30 - 25 = 5
        • 25 - 23 = 2
      • Largest gap = 46
      출력: 
      Class 1
      Max 78, Min 23, Largest gap 46

    시간 복잡도

    • 각 반마다 학생 수를 N이라고 하면,
      • 정렬: O(N log N)
      • 인접 차이 순회: O(N)
    • 최대 N = 50, K = 100이므로 전체 연산량은 매우 작다.

    공간 복잡도

    • 한 반의 점수를 담는 리스트 list만 사용 → O(N)
    • 반마다 재사용하므로 전체 공간 사용량도 작다.

    포인트 요약

    • 각 반의 점수들을 내림차순 정렬한다.
    • 첫 원소 → 최고 점수, 마지막 원소 → 최저 점수.
    • 인접한 두 점수 차이의 최댓값이 Largest gap이다.
    • 반 번호를 1부터 증가시키며 지정된 출력 형식대로 출력한다.

    문제 5839 — Cow Race

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Cow Race (BOJ 5839) 문제는
    두 소 Bessie와 Elsie가 일정 시간 동안 달릴 때, 경기 중 선두가 몇 번 바뀌는지를 세는 문제다.

    각 소의 이동은 여러 구간(segment)으로 주어지고,
    각 구간에서는 일정한 속도로 일정 시간 동안 달린다.

    • 두 소의 전체 달리는 시간은 같다.
    • 어떤 순간에 Bessie가 더 앞서면 Bessie가 선두, Elsie가 더 앞서면 Elsie가 선두, 같으면 동점이다.
    • 선두가 바뀌는 순간을 세어야 한다.
      • Bessie가 앞서다가 Elsie가 앞서게 되면 1회.
      • Elsie가 앞서다가 Bessie가 앞서게 되어도 1회.
      • 선두 → 동점 → 다른 소가 선두가 되어도 이전 선두와 다르면 선두 교체 1회로 센다.

    전략 개요

    1. 각 소(Bessie, Elsie)에 대해
      • 구간 수와 각 구간의 (속도, 시간) 정보를 읽는다.
      • 시간 단위(1초)마다의 속도를 저장하는 배열(또는 리스트)을 만든다.
        • 예: 속도 5, 시간 3 → 5, 5, 5를 차례대로 push.
    2. 두 소의 전체 시간은 동일하므로, 같은 길이의 리스트 traceB, traceE가 만들어진다.
    3. 시간 0에서 시작해, 매 시간마다
      • Bessie의 누적 이동 거리 distB, Elsie의 누적 이동 거리 distE를 갱신한다.
      • 누적 거리를 비교해 선두 상태를 구한다.
        • Bessie가 앞서면 cur = 1
        • Elsie가 앞서면 cur = 2
        • 같으면 cur = 0 (동점)
      • 직전 선두를 저장해둔 변수(last)와 비교해서 선두가 다른 쪽으로 바뀌는 순간을 카운트한다.

    핵심: 코드가 하는 일

    • 입력 처리
      • 첫 줄에서 N, M을 읽는다.
        • N: Bessie의 구간 개수
        • M: Elsie의 구간 개수
      • 다음 N줄에서 Bessie 구간 정보를 읽고, 각 구간의 속도를 구간 시간만큼 traceB 리스트에 채운다. 
        for (int i = 0; i < N; i++) {
          int speed = ...;
          int hour  = ...;
          for (int j = 0; j < hour; j++) {
              traceB.add(speed);
          }
      }
      • 다음 M줄에서 Elsie 구간 정보도 동일하게 읽어 traceE에 채운다.
      • 두 리스트 길이 = 전체 경기 시간 time.
    • 시간별 누적 거리 및 선두 판별 
      int distB = 0;
      int distE = 0;
      int change = 0; // 선두 교체 횟수
      int last = 0;   // 마지막으로 선두였던 소 (0: 동점이나 아직 없음, 1: Bessie, 2: Elsie)
      
      for (int i = 0; i < time; i++) {
          distB += traceB.get(i);
          distE += traceE.get(i);
      
          int cur;
          if (distB > distE) cur = 1;
          else if (distB < distE) cur = 2;
          else cur = 0;
      
          if (cur != 0 && last != 0 && cur != last) {
              change++;        // 선두가 Bessie ↔ Elsie로 바뀐 순간
          }
          if (cur != 0) {
              last = cur;      // 동점이 아닐 때만 마지막 선두 갱신
          }
      }
      • 동점(cur == 0)인 구간에서는 last를 바꾸지 않아 “마지막 선두였던 소” 정보를 유지한다.
      • 이후 다른 소가 처음 선두로 올라가는 순간, cur != last가 되어 선두 교체 1회를 센다.
    • 정답 출력
      • 최종적으로 change를 그대로 출력한다.

    예시 흐름 (간단 요약)

    • 각 시간 t마다:
      • Bessie의 누적 거리 distB(t), Elsie의 누적 거리 distE(t) 계산
      • 비교해서 선두가 누구인지 결정
      • 이전 선두와 다르면 선두 교체 +1
      • 동점 구간은 “선두 없음” 상태이지만, last는 바꾸지 않고 유지 → 이후 다른 소가 앞으로 나가면 선두 교체로 카운트

    시간 복잡도

    • 각 구간을 시간 단위로 풀어서 리스트에 넣기: 전체 시간 T에 대해 O(T)
    • 시뮬레이션(누적 거리와 선두 비교): O(T)
    • 전체 시간은 최대 1000 × 1000 = 10^6 정도라 충분히 빠른 수준이다.

    공간 복잡도

    • Bessie와 Elsie의 속도를 시간 단위로 저장하는 리스트 2개: O(T)
    • 추가 변수는 상수 개뿐이므로 전체 공간 복잡도는 O(T)이다.

    포인트 요약

    • 각 소의 구간 정보를 시간 단위 속도 배열로 펼친다.
    • 매 시간마다 누적 거리를 갱신하며 누가 선두인지 계산한다.
    • 동점 구간을 건너뛰면서, 마지막으로 선두였던 소와 현재 선두를 비교해 선두 교체 횟수를 센다.
    • 구현은 단순한 시뮬레이션이지만, 동점 처리와 마지막 선두 관리가 핵심이다.
