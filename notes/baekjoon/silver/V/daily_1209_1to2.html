    문제 5840 — Breed Proximity

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Breed Proximity (BOJ 5840) 문제는
    한 줄에 서 있는 소들의 품종 ID가 주어졌을 때,
    서로의 위치 차이가 K 이하인 같은 품종 소 쌍들 중에서
    가장 큰 품종 ID를 구하는 문제다.

    정의를 다시 정리하면:

    • 소는 총 N마리, 줄 위치는 1번부터 N번까지.
    • 각 소는 정수 품종 ID(0 ~ 1,000,000)를 가진다.
    • 같은 품종 ID를 가진 두 소의 위치 차이가 ≤ K이면 그 둘은 crowded 상태.
    • crowded 쌍이 여러 개라면 그 중 품종 ID가 최대인 값을 출력, 없으면 -1 출력.

    전략 개요

    N이 최대 50,000이므로, 모든 쌍을 다 보는 O(N²) 방식은 사용할 수 없다.
    대신, 각 품종별로 마지막으로 등장한 위치만 기억하면서 한 번만 훑어가면 된다.

    • 입력
      • cows[1..N]에 각 위치의 품종 ID를 저장.
    • 직전 위치 기록 배열
      • lastPos[0..1_000_000]를 0으로 초기화.
      • lastPos[b] = 품종 b가 마지막으로 등장한 위치 인덱스 (없으면 0).
    • 한 번 순회하면서 처리
      • i를 1부터 N까지 증가시키며:
        • num = cows[i]
        • 이전에 같은 품종이 등장했다면(lastPos[num] != 0)
        • dist = i - lastPos[num]를 계산해서
          • dist ≤ K이면 이 두 소는 crowded → max 갱신
        • 항상 lastPos[num] = i 로 현재 위치로 갱신
    • 마지막에 max를 출력한다. 초기값은 -1로 두어, crowded 쌍이 하나도 없으면 그대로 -1이 출력된다.

    여기서 한 품종에 대해 가장 최근(오른쪽)의 위치만 기록해도 되는 이유:

    • 현재 위치 i에서, 같은 품종의 위치들 중 가장 가까운 것은 항상 가장 최근 위치이다.
    • 가장 최근 위치와의 거리도 K를 초과한다면, 그보다 더 왼쪽(옛날)의 위치와의 거리는 더 크므로 역시 K를 초과 → 어떤 쌍도 crowded가 될 수 없다.

    핵심: 코드가 하는 일 

    int[] cows = new int[N + 1];
      for (int i = 1; i <= N; i++) {
          cows[i] = Integer.parseInt(br.readLine());
      }
      
      int dist = 0, max = -1;
      int[] lastPos = new int[1000001];
      
      for (int i = 1; i <= N; i++) {
          int num = cows[i];
      
          if (lastPos[num] != 0) {        // 이 품종이 이전에 나온 적이 있다면
              dist = i - lastPos[num];    // 거리 계산
      
              if (dist <= K)              // K 이하면 crowded
                  max = Math.max(max, num);
          }
      
          lastPos[num] = i;               // 이 품종의 마지막 위치 갱신
      }
      
      System.out.println(max);
    • lastPos의 초기값을 모두 0으로 두고, 소의 위치 인덱스는 1부터 시작하므로, lastPos[num] == 0이면 아직 등장하지 않은 품종이라는 의미가 된다.
    • 품종 ID 0도 문제에서 허용되지만, 인덱스 0은 절대 실제 위치로 사용하지 않기 때문에 충돌이 없다.

    시간 복잡도

    • 소를 한 번씩만 보며, 각 위치에서 상수 시간 연산만 수행 → O(N)
    • N ≤ 50,000 이므로 충분히 빠르게 동작한다.

    공간 복잡도

    • cows[N+1] 배열과 lastPos[1_000_001] 배열만 사용 → O(N + B)
    • 여기서 B = 1,000,001 (품종 ID 범위). 메모리 제한 내에서 충분히 작다.

    주의

    • 배열 크기:
      • 품종 ID가 최대 1,000,000이므로 lastPos 배열 크기를 정확히 1000001로 잡아야 한다.
    • 인덱스 시작:
      • 소의 위치 인덱스를 1부터 시작해야 lastPos[num] == 0을 “아직 안 나온 상태”로 안전하게 사용할 수 있다.
    • 결과 초기값:
      • crowded 쌍이 하나도 없을 수도 있으므로 max를 -1로 초기화해 두는 것이 중요하다.

    문제 5866 — Meet and Greet

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Meet and Greet (BOJ 5866) 문제는
    베시(Bessie)와 엘시(Elsie)가 1차원 직선 위를 같은 속도로 걸어 다닐 때,
    서로 떨어져 있다가 다시 같은 위치에서 만나게 되는 순간의 개수(= "moo" 횟수)를 세는 문제다.

    조건을 정리하면:

    • 두 소는 모두 시작 위치 0에서 출발한다.
    • 각 소는 여러 구간으로 나뉘어 움직이며, 각 구간은
      • 거리(=시간)와 방향(L 또는 R)으로 주어진다.
      • 속도는 1이므로, "거리 3" → 3초 동안 한 칸씩 같은 방향으로 이동.
    • 베시와 엘시는 항상 같은 시간 축에서 동시에 움직이고, 각자 자신의 구간이 끝나면 그 이후로는 그 자리에서 멈춰 선다.
    • moo 발생 시점:
      • 이전 시각에는 두 소의 위치가 달랐는데,
      • 이번 시각에는 두 소의 위치가 같아진 경우.
      • 시작 시각(0초)에 둘 다 위치 0에 있는 것은 moo가 아니다.

    전략 개요

    각 구간의 거리(=시간)가 최대 1,000,000을 넘지 않으므로,
    전체 시간 구간을 1초 단위로 시뮬레이션하는 선형 시뮬레이션이 가능하다.

    1. 입력 파싱
      • B, E : 베시, 엘시의 구간 개수.
      • 베시의 움직임 B개를 읽어
        • bessieTime[i] : i번째 구간의 시간(거리)
        • bessie 리스트 : 각 구간의 방향('L' 또는 'R')
        • 총 이동 시간 timeBessie 누적
      • 엘시도 동일하게
        • elsieTime[i], elsie 리스트
        • 총 이동 시간 timeElsie 계산
      • 전체 시뮬레이션 시간:
        • time = max(timeBessie, timeElsie)
        • 더 오래 걷는 쪽 기준으로 끝까지 시뮬레이션하고,
          일찍 멈춘 쪽은 이후로는 그대로 정지한 위치를 유지하게 된다.
    2. 각 시각별 위치 배열 준비
      • bessiePos[0..time], elsiePos[0..time] 두 배열을 만든다.
      • bessiePos[0] = 0, elsiePos[0] = 0으로 시작.
    3. 베시 이동 시뮬레이션
      • 현재 위치 bPos, 현재 구간 번호 bSeg, 남은 시간 bRemain으로 관리한다.
      • 각 초 i = 1..time에 대해:
        • 아직 베시의 총 이동 시간 timeBessie 안이면(i <= timeBessie)
        • 현재 구간의 방향을 확인하고
          • 'L'이면 bPos--
          • 'R'이면 bPos++
        • bRemain-- 하고, 0이 되면 다음 구간으로 넘어가며 새 구간의 시간을 bRemain에 설정.
        • 그 시각의 위치를 bessiePos[i] = bPos에 기록.
      • i > timeBessie인 구간에서는 더 이상 움직이지 않으므로,
        이전 위치 bPos를 그대로 기록한다.
    4. 엘시 이동 시뮬레이션
      • 위와 같은 방식으로 ePos, eSeg, eRemain을 사용해 elsiePos[1..time]를 채운다.
    5. moo 횟수 세기
      • moo = 0으로 초기화.
      • 각 시각 i = 1..time에 대해:
        • 이번 시각에 두 소가 같은 위치인지 확인: bessiePos[i] == elsiePos[i]
        • 바로 직전 시각에는 서로 다른 위치였는지 확인: bessiePos[i-1] != elsiePos[i-1]
      • 두 조건을 모두 만족하면, 이번 시각에 떨어져 있다가 처음 만난 것이므로 moo++
      • 시작 시각(0초)을 moo로 세지 말아야 하므로, 루프는 1부터 시작한다.

    핵심: 코드가 하는 일 

    // 두 소의 총 이동 시간 계산
      int timeBessie = 0;
      int[] bessieTime = new int[B + 1];
      List<String> bessie = new ArrayList<>();
      ...
      timeBessie += dist;
      bessieTime[i] = dist;
      bessie.add(dir);
      
      int timeElsie = 0;
      int[] elsieTime = new int[E + 1];
      List<String> elsie = new ArrayList<>();
      ...
      timeElsie += dist;
      elsieTime[i] = dist;
      elsie.add(dire);
      
      // 전체 시뮬레이션 시간
      int time = Math.max(timeBessie, timeElsie);
      
      // 각 시간마다 위치 기록 배열
      int[] bessiePos = new int[time + 1];
      int[] elsiePos = new int[time + 1];
      
      bessiePos[0] = 0;
      elsiePos[0] = 0;
      
      // 베시 시뮬레이션
      int bPos = 0;
      int bSeg = 1;
      int bRemain = (B >= 1) ? bessieTime[1] : 0;
      
      for (int i = 1; i <= time; i++) {
          if (i <= timeBessie) {                // 아직 움직이는 시간
              char dir = bessie.get(bSeg - 1).charAt(0);
      
              if (dir == 'L') bPos--;
              else bPos++;
      
              bRemain--;
              if (bRemain == 0 && bSeg < B) {   // 다음 구간으로
                  bSeg++;
                  bRemain = bessieTime[bSeg];
              }
          }
          bessiePos[i] = bPos;
      }
      
      // 엘시 시뮬레이션
      int ePos = 0;
      int eSeg = 1;
      int eRemain = (E >= 1) ? elsieTime[1] : 0;  // (의도: 첫 구간 시간)
      
      for (int i = 1; i <= time; i++) {
          if (i <= timeElsie) {
              char dir = elsie.get(eSeg - 1).charAt(0);
      
              if (dir == 'L') ePos--;
              else ePos++;
      
              eRemain--;
              if (eRemain == 0 && eSeg < E) {
                  eSeg++;
                  eRemain = elsieTime[eSeg];
              }
          }
          elsiePos[i] = ePos;
      }
      
      // moo 카운트
      int moo = 0;
      for (int i = 1; i <= time; i++) {
          if (bessiePos[i] == elsiePos[i] && bessiePos[i - 1] != elsiePos[i - 1]) {
              moo++;
          }
      }
      
      System.out.println(moo);
    • bessiePos[i], elsiePos[i]에는 i초가 끝난 시점의 위치가 저장된다.
    • 따라서 i-1초에는 떨어져 있었고 i초에 처음 위치가 같아지는 시점마다 moo를 1번씩 센다.

    시간 복잡도

    • 각 소의 거리를 모두 합쳐도 최대 1,000,000 이내.
    • 1초 단위 시뮬레이션은 time ≤ 1,000,000 회 반복.
    • 따라서 전체 알고리즘은 O(time), 즉 대략 106 수준으로 충분히 빠르다.

    공간 복잡도

    • 각 시각별 위치 배열 2개: bessiePos[time+1], elsiePos[time+1]
    • 그 외 구간 정보 저장용 배열/리스트
    • ⇒ 전체 메모리는 대략 O(time + B + E), 문제 제한 내에서 충분히 작다.

    주의

    • 초 단위 시뮬레이션 기준:
      • 각 거리(시간)가 정수이므로, 1초씩 순회하면 모든 상태 변화를 정확히 추적할 수 있다.
    • 시작 시점 moo 제외:
      • i=1부터 검사하고, 항상 i-1과 비교하므로, 시작 위치(0초)에서의 동일 위치는 moo로 세지 않는다.
    • 정지 후 처리:
      • 한 소가 먼저 멈추면, 그 이후로는 계속 같은 위치에 머무르도록 구현해야 한다.
      • 코드에서 i > timeBessie 또는 i > timeElsie인 경우에는 위치만 복사하고 움직이지 않게 함으로써 이 조건을 만족한다.
