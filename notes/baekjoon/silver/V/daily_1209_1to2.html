    문제 5840 — Breed Proximity

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Breed Proximity (BOJ 5840) 문제는
    한 줄에 서 있는 소들의 품종 ID가 주어졌을 때,
    서로의 위치 차이가 K 이하인 같은 품종 소 쌍들 중에서
    가장 큰 품종 ID를 구하는 문제다.

    정의를 다시 정리하면:

    • 소는 총 N마리, 줄 위치는 1번부터 N번까지.
    • 각 소는 정수 품종 ID(0 ~ 1,000,000)를 가진다.
    • 같은 품종 ID를 가진 두 소의 위치 차이가 ≤ K이면 그 둘은 crowded 상태.
    • crowded 쌍이 여러 개라면 그 중 품종 ID가 최대인 값을 출력, 없으면 -1 출력.

    전략 개요

    N이 최대 50,000이므로, 모든 쌍을 다 보는 O(N²) 방식은 사용할 수 없다.
    대신, 각 품종별로 마지막으로 등장한 위치만 기억하면서 한 번만 훑어가면 된다.

    • 입력
      • cows[1..N]에 각 위치의 품종 ID를 저장.
    • 직전 위치 기록 배열
      • lastPos[0..1_000_000]를 0으로 초기화.
      • lastPos[b] = 품종 b가 마지막으로 등장한 위치 인덱스 (없으면 0).
    • 한 번 순회하면서 처리
      • i를 1부터 N까지 증가시키며:
        • num = cows[i]
        • 이전에 같은 품종이 등장했다면(lastPos[num] != 0)
        • dist = i - lastPos[num]를 계산해서
          • dist ≤ K이면 이 두 소는 crowded → max 갱신
        • 항상 lastPos[num] = i 로 현재 위치로 갱신
    • 마지막에 max를 출력한다. 초기값은 -1로 두어, crowded 쌍이 하나도 없으면 그대로 -1이 출력된다.

    여기서 한 품종에 대해 가장 최근(오른쪽)의 위치만 기록해도 되는 이유:

    • 현재 위치 i에서, 같은 품종의 위치들 중 가장 가까운 것은 항상 가장 최근 위치이다.
    • 가장 최근 위치와의 거리도 K를 초과한다면, 그보다 더 왼쪽(옛날)의 위치와의 거리는 더 크므로 역시 K를 초과 → 어떤 쌍도 crowded가 될 수 없다.

    핵심: 코드가 하는 일 

    int[] cows = new int[N + 1];
      for (int i = 1; i <= N; i++) {
          cows[i] = Integer.parseInt(br.readLine());
      }
      
      int dist = 0, max = -1;
      int[] lastPos = new int[1000001];
      
      for (int i = 1; i <= N; i++) {
          int num = cows[i];
      
          if (lastPos[num] != 0) {        // 이 품종이 이전에 나온 적이 있다면
              dist = i - lastPos[num];    // 거리 계산
      
              if (dist <= K)              // K 이하면 crowded
                  max = Math.max(max, num);
          }
      
          lastPos[num] = i;               // 이 품종의 마지막 위치 갱신
      }
      
      System.out.println(max);
    • lastPos의 초기값을 모두 0으로 두고, 소의 위치 인덱스는 1부터 시작하므로, lastPos[num] == 0이면 아직 등장하지 않은 품종이라는 의미가 된다.
    • 품종 ID 0도 문제에서 허용되지만, 인덱스 0은 절대 실제 위치로 사용하지 않기 때문에 충돌이 없다.

    시간 복잡도

    • 소를 한 번씩만 보며, 각 위치에서 상수 시간 연산만 수행 → O(N)
    • N ≤ 50,000 이므로 충분히 빠르게 동작한다.

    공간 복잡도

    • cows[N+1] 배열과 lastPos[1_000_001] 배열만 사용 → O(N + B)
    • 여기서 B = 1,000,001 (품종 ID 범위). 메모리 제한 내에서 충분히 작다.

    주의

    • 배열 크기:
      • 품종 ID가 최대 1,000,000이므로 lastPos 배열 크기를 정확히 1000001로 잡아야 한다.
    • 인덱스 시작:
      • 소의 위치 인덱스를 1부터 시작해야 lastPos[num] == 0을 “아직 안 나온 상태”로 안전하게 사용할 수 있다.
    • 결과 초기값:
      • crowded 쌍이 하나도 없을 수도 있으므로 max를 -1로 초기화해 두는 것이 중요하다.

    문제 5866 — Meet and Greet

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Cow Race (BOJ 5839) 문제는
    두 소 Bessie와 Elsie가 일정 시간 동안 달릴 때, 경기 중 선두가 몇 번 바뀌는지를 세는 문제다.

    각 소의 이동은 여러 구간(segment)으로 주어지고,
    각 구간에서는 일정한 속도로 일정 시간 동안 달린다.

    • 두 소의 전체 달리는 시간은 같다.
    • 어떤 순간에 Bessie가 더 앞서면 Bessie가 선두, Elsie가 더 앞서면 Elsie가 선두, 같으면 동점이다.
    • 선두가 바뀌는 순간을 세어야 한다.
      • Bessie가 앞서다가 Elsie가 앞서게 되면 1회.
      • Elsie가 앞서다가 Bessie가 앞서게 되어도 1회.
      • 선두 → 동점 → 다른 소가 선두가 되어도 이전 선두와 다르면 선두 교체 1회로 센다.

    전략 개요

    1. 각 소(Bessie, Elsie)에 대해
      • 구간 수와 각 구간의 (속도, 시간) 정보를 읽는다.
      • 시간 단위(1초)마다의 속도를 저장하는 배열(또는 리스트)을 만든다.
        • 예: 속도 5, 시간 3 → 5, 5, 5를 차례대로 push.
    2. 두 소의 전체 시간은 동일하므로, 같은 길이의 리스트 traceB, traceE가 만들어진다.
    3. 시간 0에서 시작해, 매 시간마다
      • Bessie의 누적 이동 거리 distB, Elsie의 누적 이동 거리 distE를 갱신한다.
      • 누적 거리를 비교해 선두 상태를 구한다.
        • Bessie가 앞서면 cur = 1
        • Elsie가 앞서면 cur = 2
        • 같으면 cur = 0 (동점)
      • 직전 선두를 저장해둔 변수(last)와 비교해서 선두가 다른 쪽으로 바뀌는 순간을 카운트한다.

    핵심: 코드가 하는 일

    • 입력 처리
      • 첫 줄에서 N, M을 읽는다.
        • N: Bessie의 구간 개수
        • M: Elsie의 구간 개수
      • 다음 N줄에서 Bessie 구간 정보를 읽고, 각 구간의 속도를 구간 시간만큼 traceB 리스트에 채운다. 
        for (int i = 0; i < N; i++) {
          int speed = ...;
          int hour  = ...;
          for (int j = 0; j < hour; j++) {
              traceB.add(speed);
          }
      }
      • 다음 M줄에서 Elsie 구간 정보도 동일하게 읽어 traceE에 채운다.
      • 두 리스트 길이 = 전체 경기 시간 time.
    • 시간별 누적 거리 및 선두 판별 
      int distB = 0;
      int distE = 0;
      int change = 0; // 선두 교체 횟수
      int last = 0;   // 마지막으로 선두였던 소 (0: 동점이나 아직 없음, 1: Bessie, 2: Elsie)
      
      for (int i = 0; i < time; i++) {
          distB += traceB.get(i);
          distE += traceE.get(i);
      
          int cur;
          if (distB > distE) cur = 1;
          else if (distB < distE) cur = 2;
          else cur = 0;
      
          if (cur != 0 && last != 0 && cur != last) {
              change++;        // 선두가 Bessie ↔ Elsie로 바뀐 순간
          }
          if (cur != 0) {
              last = cur;      // 동점이 아닐 때만 마지막 선두 갱신
          }
      }
      • 동점(cur == 0)인 구간에서는 last를 바꾸지 않아 “마지막 선두였던 소” 정보를 유지한다.
      • 이후 다른 소가 처음 선두로 올라가는 순간, cur != last가 되어 선두 교체 1회를 센다.
    • 정답 출력
      • 최종적으로 change를 그대로 출력한다.

    예시 흐름 (간단 요약)

    • 각 시간 t마다:
      • Bessie의 누적 거리 distB(t), Elsie의 누적 거리 distE(t) 계산
      • 비교해서 선두가 누구인지 결정
      • 이전 선두와 다르면 선두 교체 +1
      • 동점 구간은 “선두 없음” 상태이지만, last는 바꾸지 않고 유지 → 이후 다른 소가 앞으로 나가면 선두 교체로 카운트

    시간 복잡도

    • 각 구간을 시간 단위로 풀어서 리스트에 넣기: 전체 시간 T에 대해 O(T)
    • 시뮬레이션(누적 거리와 선두 비교): O(T)
    • 전체 시간은 최대 1000 × 1000 = 10^6 정도라 충분히 빠른 수준이다.

    공간 복잡도

    • Bessie와 Elsie의 속도를 시간 단위로 저장하는 리스트 2개: O(T)
    • 추가 변수는 상수 개뿐이므로 전체 공간 복잡도는 O(T)이다.

    포인트 요약

    • 각 소의 구간 정보를 시간 단위 속도 배열로 펼친다.
    • 매 시간마다 누적 거리를 갱신하며 누가 선두인지 계산한다.
    • 동점 구간을 건너뛰면서, 마지막으로 선두였던 소와 현재 선두를 비교해 선두 교체 횟수를 센다.
    • 구현은 단순한 시뮬레이션이지만, 동점 처리와 마지막 선두 관리가 핵심이다.
