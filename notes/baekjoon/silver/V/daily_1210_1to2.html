    문제 5929 — Awkward Digits

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Awkward Digits (BOJ 5929) 문제는
    어떤 정수 N의 2진수 표기와 3진수 표기가 각각 한 자리씩만 잘못 적힌 상태로 주어졌을 때,
    원래의 정수 N (10진수)을 찾아내는 문제다.

    • 2진수 문자열에는 정확히 한 자리가 잘못 적혀 있다. (0 ↔ 1)
    • 3진수 문자열에도 정확히 한 자리가 잘못 적혀 있다. (0, 1, 2 중 하나 → 다른 둘 중 하나)
    • 자릿수 개수는 변하지 않는다. (추가/삭제 없음, 앞에 0이 와도 허용)
    • 조건을 만족하는 N은 유일하게 존재한다.

    전략 개요

    아이디어는 단순하다:

    1. 2진수 문자열에서 한 자리씩 뒤집어서 가능한 모든 후보 숫자를 만들어 본다.
      • 각 자리 i에 대해:
        • 현재 값이 '0'이면 → '1'로 바꾸고
        • '1'이면 → '0'으로 바꾼 뒤,
        • 그 결과를 2진수로 해석해 정수로 변환하고 Set에 넣는다.
      • → 이렇게 하면 “2진수 표현에서 한 자리만 틀렸을 때 만들 수 있는 모든 원래 숫자 후보”가 된다.
    2. 3진수 문자열에서도 한 자리만 수정해 가능한 숫자들을 만들어 보면서,
      동시에 1번에서 만든 Set과 교집합을 찾는다.
      • 각 자리 i에 대해:
        • 해당 자리가 '0'이면 → '1', '2' 두 가지로 바꾸기
        • '1'이면 → '0', '2'
        • '2'이면 → '0', '1'
      • 각 변경 결과를 3진수로 해석해 정수로 바꾼 뒤,
      • Set 안에 있는지 확인 → 있다면, 그 값이 바로 정답 N.

    왜 이게 맞을까?

    • 실제 N은 “2진수에서 한 자리 수정” + “3진수에서 한 자리 수정”으로 동시에 얻을 수 있는 유일한 값이다.
    • 그래서 2진수 후보들의 집합과 3진수 후보들을 한 자리씩 바꿔 가며 비교했을 때
      처음 만나는 교집합 원소가 곧 답이다.

    코드가 하는 일

    • 입력 처리 
      String str1 = br.readLine();  // 잘못된 2진수
      String str2 = br.readLine();  // 잘못된 3진수
    • 1단계: 2진수 후보 생성 → Set에 저장 
      Set<Integer> set = new HashSet<>();
      
      char[] arr1 = str1.toCharArray();
      for (int i = 0; i < arr1.length; i++) {
          char[] tmp = arr1.clone();
          if (arr1[i] == '1') tmp[i] = '0';
          else tmp[i] = '1';
      
          String str = String.valueOf(tmp);
          int value = Integer.parseInt(str, 2);
          set.add(value);
      }
      • 각 인덱스 i 하나만 뒤집은 새로운 2진수 문자열을 만든 뒤 Integer.parseInt(str, 2)로 10진수로 변환.
      • 모든 가능한 후보를 set에 저장해 둔다.
    • 2단계: 3진수에서 한 자리씩 바꾸며 교집합 찾기 
      char[] arr2 = str2.toCharArray();
      for (int i = 0; i < arr2.length; i++) {
          char[] tmp1 = arr2.clone();
          char[] tmp2 = arr2.clone();
          if (arr2[i] == '0') {
              tmp1[i] = '1';
              tmp2[i] = '2';
          } else if (arr2[i] == '1') {
              tmp1[i] = '0';
              tmp2[i] = '2';
          } else {
              tmp1[i] = '0';
              tmp2[i] = '1';
          }
      
          String s1 = String.valueOf(tmp1);
          String s2 = String.valueOf(tmp2);
          int value1 = Integer.parseInt(s1, 3);
          int value2 = Integer.parseInt(s2, 3);
      
          if (set.contains(value1)) {
              System.out.println(value1);
              return;
          } else if (set.contains(value2)) {
              System.out.println(value2);
              return;
          }
      }
      • 각 자리마다 가능한 다른 두 숫자로 바꾼 버전 2개를 만든다.
      • 각 버전을 3진수로 해석해 10진수로 만들고,
      • 아까 만들어 둔 2진수 후보 set에 들어 있는지 확인.
      • 들어 있다면 그 값이 유일한 정답이므로 바로 출력하고 return.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 2진수 길이를 B, 3진수 길이를 T라고 하면:
        • 2진수 후보 생성: O(B)개 후보 × 각 변환 비용(길이만큼) → 대략 O(B²)에 가깝지만, 실제 입력 길이는 매우 짧다.
        • 3진수 쪽: 각 자리마다 2개 후보 → O(T)번 변환 및 Set 조회.
      • 문제 조건에서 N ≤ 1,000,000,000이기 때문에 자릿수는 최대 31비트(2진), 20자리(3진) 정도 → 실질적으로는 매우 빠르게 동작한다.
    • 공간 복잡도
      • 2진수에서 만든 후보들의 개수는 최대 B개.
      • 각 후보는 int로 저장되므로, 전체 공간은 O(B) 수준.
      • 부가적으로 사용하는 문자열/배열도 자릿수 길이만큼이라 전체적으로 상수에 가까운 공간 사용이다.

    문제 5939 — Race Results

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Cow Race (BOJ 5839) 문제는
    두 소 Bessie와 Elsie가 일정 시간 동안 달릴 때, 경기 중 선두가 몇 번 바뀌는지를 세는 문제다.

    각 소의 이동은 여러 구간(segment)으로 주어지고,
    각 구간에서는 일정한 속도로 일정 시간 동안 달린다.

    • 두 소의 전체 달리는 시간은 같다.
    • 어떤 순간에 Bessie가 더 앞서면 Bessie가 선두, Elsie가 더 앞서면 Elsie가 선두, 같으면 동점이다.
    • 선두가 바뀌는 순간을 세어야 한다.
      • Bessie가 앞서다가 Elsie가 앞서게 되면 1회.
      • Elsie가 앞서다가 Bessie가 앞서게 되어도 1회.
      • 선두 → 동점 → 다른 소가 선두가 되어도 이전 선두와 다르면 선두 교체 1회로 센다.

    전략 개요

    1. 각 소(Bessie, Elsie)에 대해
      • 구간 수와 각 구간의 (속도, 시간) 정보를 읽는다.
      • 시간 단위(1초)마다의 속도를 저장하는 배열(또는 리스트)을 만든다.
        • 예: 속도 5, 시간 3 → 5, 5, 5를 차례대로 push.
    2. 두 소의 전체 시간은 동일하므로, 같은 길이의 리스트 traceB, traceE가 만들어진다.
    3. 시간 0에서 시작해, 매 시간마다
      • Bessie의 누적 이동 거리 distB, Elsie의 누적 이동 거리 distE를 갱신한다.
      • 누적 거리를 비교해 선두 상태를 구한다.
        • Bessie가 앞서면 cur = 1
        • Elsie가 앞서면 cur = 2
        • 같으면 cur = 0 (동점)
      • 직전 선두를 저장해둔 변수(last)와 비교해서 선두가 다른 쪽으로 바뀌는 순간을 카운트한다.

    핵심: 코드가 하는 일

    • 입력 처리
      • 첫 줄에서 N, M을 읽는다.
        • N: Bessie의 구간 개수
        • M: Elsie의 구간 개수
      • 다음 N줄에서 Bessie 구간 정보를 읽고, 각 구간의 속도를 구간 시간만큼 traceB 리스트에 채운다. 
        for (int i = 0; i < N; i++) {
          int speed = ...;
          int hour  = ...;
          for (int j = 0; j < hour; j++) {
              traceB.add(speed);
          }
      }
      • 다음 M줄에서 Elsie 구간 정보도 동일하게 읽어 traceE에 채운다.
      • 두 리스트 길이 = 전체 경기 시간 time.
    • 시간별 누적 거리 및 선두 판별 
      int distB = 0;
      int distE = 0;
      int change = 0; // 선두 교체 횟수
      int last = 0;   // 마지막으로 선두였던 소 (0: 동점이나 아직 없음, 1: Bessie, 2: Elsie)
      
      for (int i = 0; i < time; i++) {
          distB += traceB.get(i);
          distE += traceE.get(i);
      
          int cur;
          if (distB > distE) cur = 1;
          else if (distB < distE) cur = 2;
          else cur = 0;
      
          if (cur != 0 && last != 0 && cur != last) {
              change++;        // 선두가 Bessie ↔ Elsie로 바뀐 순간
          }
          if (cur != 0) {
              last = cur;      // 동점이 아닐 때만 마지막 선두 갱신
          }
      }
      • 동점(cur == 0)인 구간에서는 last를 바꾸지 않아 “마지막 선두였던 소” 정보를 유지한다.
      • 이후 다른 소가 처음 선두로 올라가는 순간, cur != last가 되어 선두 교체 1회를 센다.
    • 정답 출력
      • 최종적으로 change를 그대로 출력한다.

    예시 흐름 (간단 요약)

    • 각 시간 t마다:
      • Bessie의 누적 거리 distB(t), Elsie의 누적 거리 distE(t) 계산
      • 비교해서 선두가 누구인지 결정
      • 이전 선두와 다르면 선두 교체 +1
      • 동점 구간은 “선두 없음” 상태이지만, last는 바꾸지 않고 유지 → 이후 다른 소가 앞으로 나가면 선두 교체로 카운트

    시간 복잡도

    • 각 구간을 시간 단위로 풀어서 리스트에 넣기: 전체 시간 T에 대해 O(T)
    • 시뮬레이션(누적 거리와 선두 비교): O(T)
    • 전체 시간은 최대 1000 × 1000 = 10^6 정도라 충분히 빠른 수준이다.

    공간 복잡도

    • Bessie와 Elsie의 속도를 시간 단위로 저장하는 리스트 2개: O(T)
    • 추가 변수는 상수 개뿐이므로 전체 공간 복잡도는 O(T)이다.

    포인트 요약

    • 각 소의 구간 정보를 시간 단위 속도 배열로 펼친다.
    • 매 시간마다 누적 거리를 갱신하며 누가 선두인지 계산한다.
    • 동점 구간을 건너뛰면서, 마지막으로 선두였던 소와 현재 선두를 비교해 선두 교체 횟수를 센다.
    • 구현은 단순한 시뮬레이션이지만, 동점 처리와 마지막 선두 관리가 핵심이다.
