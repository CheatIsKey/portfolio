    문제 5929 — Awkward Digits

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Awkward Digits (BOJ 5929) 문제는
    어떤 정수 N의 2진수 표기와 3진수 표기가 각각 한 자리씩만 잘못 적힌 상태로 주어졌을 때,
    원래의 정수 N (10진수)을 찾아내는 문제다.

    • 2진수 문자열에는 정확히 한 자리가 잘못 적혀 있다. (0 ↔ 1)
    • 3진수 문자열에도 정확히 한 자리가 잘못 적혀 있다. (0, 1, 2 중 하나 → 다른 둘 중 하나)
    • 자릿수 개수는 변하지 않는다. (추가/삭제 없음, 앞에 0이 와도 허용)
    • 조건을 만족하는 N은 유일하게 존재한다.

    전략 개요

    아이디어는 단순하다:

    1. 2진수 문자열에서 한 자리씩 뒤집어서 가능한 모든 후보 숫자를 만들어 본다.
      • 각 자리 i에 대해:
        • 현재 값이 '0'이면 → '1'로 바꾸고
        • '1'이면 → '0'으로 바꾼 뒤,
        • 그 결과를 2진수로 해석해 정수로 변환하고 Set에 넣는다.
      • → 이렇게 하면 “2진수 표현에서 한 자리만 틀렸을 때 만들 수 있는 모든 원래 숫자 후보”가 된다.
    2. 3진수 문자열에서도 한 자리만 수정해 가능한 숫자들을 만들어 보면서,
      동시에 1번에서 만든 Set과 교집합을 찾는다.
      • 각 자리 i에 대해:
        • 해당 자리가 '0'이면 → '1', '2' 두 가지로 바꾸기
        • '1'이면 → '0', '2'
        • '2'이면 → '0', '1'
      • 각 변경 결과를 3진수로 해석해 정수로 바꾼 뒤,
      • Set 안에 있는지 확인 → 있다면, 그 값이 바로 정답 N.

    왜 이게 맞을까?

    • 실제 N은 “2진수에서 한 자리 수정” + “3진수에서 한 자리 수정”으로 동시에 얻을 수 있는 유일한 값이다.
    • 그래서 2진수 후보들의 집합과 3진수 후보들을 한 자리씩 바꿔 가며 비교했을 때
      처음 만나는 교집합 원소가 곧 답이다.

    코드가 하는 일

    • 입력 처리 
      String str1 = br.readLine();  // 잘못된 2진수
      String str2 = br.readLine();  // 잘못된 3진수
    • 1단계: 2진수 후보 생성 → Set에 저장 
      Set<Integer> set = new HashSet<>();
      
      char[] arr1 = str1.toCharArray();
      for (int i = 0; i < arr1.length; i++) {
          char[] tmp = arr1.clone();
          if (arr1[i] == '1') tmp[i] = '0';
          else tmp[i] = '1';
      
          String str = String.valueOf(tmp);
          int value = Integer.parseInt(str, 2);
          set.add(value);
      }
      • 각 인덱스 i 하나만 뒤집은 새로운 2진수 문자열을 만든 뒤 Integer.parseInt(str, 2)로 10진수로 변환.
      • 모든 가능한 후보를 set에 저장해 둔다.
    • 2단계: 3진수에서 한 자리씩 바꾸며 교집합 찾기 
      char[] arr2 = str2.toCharArray();
      for (int i = 0; i < arr2.length; i++) {
          char[] tmp1 = arr2.clone();
          char[] tmp2 = arr2.clone();
          if (arr2[i] == '0') {
              tmp1[i] = '1';
              tmp2[i] = '2';
          } else if (arr2[i] == '1') {
              tmp1[i] = '0';
              tmp2[i] = '2';
          } else {
              tmp1[i] = '0';
              tmp2[i] = '1';
          }
      
          String s1 = String.valueOf(tmp1);
          String s2 = String.valueOf(tmp2);
          int value1 = Integer.parseInt(s1, 3);
          int value2 = Integer.parseInt(s2, 3);
      
          if (set.contains(value1)) {
              System.out.println(value1);
              return;
          } else if (set.contains(value2)) {
              System.out.println(value2);
              return;
          }
      }
      • 각 자리마다 가능한 다른 두 숫자로 바꾼 버전 2개를 만든다.
      • 각 버전을 3진수로 해석해 10진수로 만들고,
      • 아까 만들어 둔 2진수 후보 set에 들어 있는지 확인.
      • 들어 있다면 그 값이 유일한 정답이므로 바로 출력하고 return.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 2진수 길이를 B, 3진수 길이를 T라고 하면:
        • 2진수 후보 생성: O(B)개 후보 × 각 변환 비용(길이만큼) → 대략 O(B²)에 가깝지만, 실제 입력 길이는 매우 짧다.
        • 3진수 쪽: 각 자리마다 2개 후보 → O(T)번 변환 및 Set 조회.
      • 문제 조건에서 N ≤ 1,000,000,000이기 때문에 자릿수는 최대 31비트(2진), 20자리(3진) 정도 → 실질적으로는 매우 빠르게 동작한다.
    • 공간 복잡도
      • 2진수에서 만든 후보들의 개수는 최대 B개.
      • 각 후보는 int로 저장되므로, 전체 공간은 O(B) 수준.
      • 부가적으로 사용하는 문자열/배열도 자릿수 길이만큼이라 전체적으로 상수에 가까운 공간 사용이다.

    문제 5939 — Race Results

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Race Results (BOJ 5939)는
    여러 마리 소들의 마라톤 기록을 시:분:초 기준 오름차순 정렬해서 출력하는 아주 단순한 정렬 문제다.

    • 각 기록은 시(0~99), 분(0~59), 초(0~59) 세 정수로 주어진다.
    • 정렬 기준은
      1. 먼저 시가 작은 순
      2. 시가 같다면 분이 작은 순
      3. 시, 분이 모두 같다면 초가 작은 순
    • 정렬된 순서대로 다시 시 분 초 형태로 출력하면 된다.

    전략 개요

    • 입력
      • 첫 줄에서 소의 수 N을 읽는다.
      • 이후 N줄에 대해, 각 줄에서 시, 분, 초를 읽어 3칸짜리 배열로 저장한다.
    • 정렬
      • int[][] cows = new int[N][3]; 형태의 2차원 배열에 기록을 저장한다.
      • Arrays.sort에 커스텀 Comparator를 넘겨
        1. hour(a[0]) 먼저 비교,
        2. 같으면 minute(a[1]) 비교,
        3. 그래도 같으면 second(a[2]) 비교
        하는 방식으로 정렬한다.
    • 출력
      • 정렬된 cows 배열을 앞에서부터 순회하며 시 분 초를 공백으로 구분해 한 줄에 하나씩 출력한다.

    코드가 하는 일

    • 입력 및 저장 
      int N = Integer.parseInt(br.readLine());
      
      int[][] cows = new int[N][3];
      for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int hour = Integer.parseInt(st.nextToken());
          int minute = Integer.parseInt(st.nextToken());
          int second = Integer.parseInt(st.nextToken());
      
          cows[i][0] = hour;
          cows[i][1] = minute;
          cows[i][2] = second;
      }
      • 각 줄에서 시간 3개를 읽어 cows[i][0..2]에 저장한다.
    • 정렬 로직 
      Arrays.sort(cows, (a, b) -> {
          if (a[0] != b[0]) return Integer.compare(a[0], b[0]);
          if (a[1] != b[1]) return Integer.compare(a[1], b[1]);
          return Integer.compare(a[2], b[2]);
      });
      • a[0], b[0]: 시(hour) 비교
      • 시가 같으면 a[1], b[1]: 분(minute) 비교
      • 여기까지 같으면 a[2], b[2]: 초(second) 비교
      • 이 순서대로 오름차순 정렬이 이루어진다.
    • 출력 
      for (int[] cow : cows) {
          for (int i = 0; i < cow.length; i++) {
              sb.append(cow[i]).append(" ");
          }
          sb.append('\n');
      }
      
      System.out.print(sb.toString());
      • 정렬된 배열에서 각 레코드를 꺼내 시 분 초 형태로 붙이고 줄바꿈.
      • StringBuilder를 사용해 출력 속도를 높였다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬이 전체 시간의 대부분을 차지하며, N개의 원소 정렬이므로 O(N log N).
      • N ≤ 5,000이므로 충분히 빠르게 동작한다.
    • 공간 복잡도
      • int[ N ][ 3 ] 배열만 사용하므로 O(N).
