    문제 5957 — Cleaning the Dishes

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Cleaning the Dishes (BOJ 5957)는
    더러운 접시 더미에서 시작해 Bessie(세척)와 Canmuu(건조)가 번갈아 작업할 때, 최종적으로 깨끗한 접시 스택의 위에서부터 아래까지 번호를 출력하는 시뮬레이션 문제다.

    • 접시는 1번이 맨 위, N번이 맨 아래인 상태에서 시작한다.
    • 명령은 두 종류:
      • 1 D : Bessie가 더러운 접시 더미에서 D개를 순서대로 꺼내 씻고, 세척 전용 스택 위에 쌓는다.
      • 2 D : Canmuu가 세척된 스택에서 D개를 꺼내 말리고, 최종 깨끗한 스택 위에 쌓는다.
    • 두 사람 모두 “스택처럼” 위에서부터 하나씩 처리하고, 옮길 때마다 순서가 뒤집힌다.
    • 모든 명령이 끝난 뒤, 깨끗한 스택(dry)의 위에서부터 순서대로 접시 번호를 출력하면 된다.

    전략 개요

    • 핵심 아이디어
      • 스택 3개로 그대로 시뮬레이션한다.
        • dirty : 아직 안 씻은 접시들
        • wash : 씻었지만 아직 안 말린 접시들
        • dry : 씻고 말리기까지 끝난 최종 스택
      • 각 명령에 따라 dirty → wash → dry로 접시가 이동한다.
      • 한 번의 “세척 묶음”은 한 번 뒤집히고, 그 묶음이 통째로 “건조”될 때 다시 한 번 더 뒤집혀서 결국 원래 순서대로 dry에 쌓이게 된다.
    • 초기화
      • dirty 스택에는 N부터 1까지 push 한다.
        • 이렇게 하면 dirty.pop()을 했을 때 1, 2, 3, ... 순으로 꺼낼 수 있다.
    • 명령 처리
      • 1 D (세척):
        • D번 반복해서 dirty.pop()한 뒤 wash.push() 한다.
      • 2 D (건조):
        • D번 반복해서 wash.pop()한 뒤 dry.push() 한다.
    • 출력
      • 모든 명령 처리가 끝나면, dry의 top부터 bottom까지 순서대로 꺼내면서 출력한다.
      • 문제에서 요구하는 “위에서부터 i번째 접시” 순서와 정확히 일치한다.

    코드가 하는 일

    • 스택 초기화 
      int N = Integer.parseInt(br.readLine());
      
      Deque<Integer> dirty = new ArrayDeque<>();
      Deque<Integer> wash  = new ArrayDeque<>();
      Deque<Integer> dry   = new ArrayDeque<>();
      
      // dirty: top = 1, ..., bottom = N 이 되도록 push
      for (int i = N; i >= 1; i--) {
          dirty.push(i);
      }
      • dirty에 N, N-1, ..., 1을 차례대로 push 하여, pop() 시 1부터 꺼내지도록 만든다.
    • 명령 처리 루프 
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
      
          int C = Integer.parseInt(st.nextToken()); // 1: 세척, 2: 건조
          int D = Integer.parseInt(st.nextToken()); // 처리할 접시 개수
      
          if (C == 1) {
              // dirty -> wash
              for (int i = 0; i < D; i++) {
                  int toWash = dirty.pop();
                  wash.push(toWash);
              }
          } else {
              // wash -> dry
              for (int i = 0; i < D; i++) {
                  int toDry = wash.pop();
                  dry.push(toDry);
              }
          }
      }
      • C == 1이면 더러운 접시에서 D개를 씻어서 wash에 쌓는다.
      • C == 2이면 세척된 접시에서 D개를 말려서 dry에 쌓는다.
      • 두 경우 모두 스택을 사용하므로, 각 “묶음”은 이동할 때마다 순서가 뒤집힌다.
    • 최종 결과 출력 
      while (!dry.isEmpty()) {
          int removed = dry.pop();
          sb.append(removed).append('\n');
      }
      
      System.out.print(sb.toString());
      • dry 스택의 top부터 하나씩 꺼내서 출력한다.
      • 문제에서 요구하는 “위에서부터 i번째 접시”에 해당하므로 그대로 출력하면 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 접시는 세척(한 번 pop/push), 건조(한 번 pop/push)를 최대 한 번씩만 거친다.
      • 따라서 전체 연산은 O(N) 수준이다.
    • 공간 복잡도
      • 세 개의 스택에 접시 번호를 저장하므로 필요한 추가 공간은 O(N)이다.

    문제 5976 — A spiral walk

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    A spiral walk (BOJ 5976)는
    N×N 격자를 시계 방향 나선 형태로 채우는 고전적인 구현 문제다.

    • 시작 위치는 (0,0) (왼쪽 위)이고, 거기서부터 오른쪽 → 아래 → 왼쪽 → 위 순으로 나선을 그리며 채운다.
    • 채우는 값은 1부터 N²까지 증가한다.
    • 최종적으로 N행을 출력할 때, 각 행마다 N개의 정수를 공백으로 구분해서 출력하면 된다.

    전략 개요

    • 나선 채우기의 기본 아이디어
      • 현재 “아직 안 채운 영역”을 상단, 하단, 좌측, 우측 경계로 관리한다.
        • top : 아직 안 채운 영역의 가장 위 행 인덱스
        • bottom : 가장 아래 행 인덱스
        • left : 가장 왼쪽 열 인덱스
        • right : 가장 오른쪽 열 인덱스
      • 한 “레이어”를 채울 때마다 순서대로:
        1. 위쪽 행을 left → right 방향으로 채움, 그리고 top++
        2. 오른쪽 열을 top → bottom 방향으로 채움, 그리고 right--
        3. 아래쪽 행을 right → left 방향으로 채움, 그리고 bottom--
        4. 왼쪽 열을 bottom → top 방향으로 채움, 그리고 left++
      • 이 과정을 현재 숫자 cur가 N²에 도달할 때까지 반복하면 나선 모양으로 전체가 채워진다.
    • 배열 준비
      • int[][] field = new int[N][N]; 에 나선 순서대로 숫자를 기록한다.
      • 초기 값: 
        int top = 0, bottom = N - 1;
      int left = 0, right = N - 1;
      int cur = 1, end = N * N;
        • cur는 현재 채울 숫자 (1부터 시작해서 N²까지 증가).
        • end는 종료 기준값(=N²).

    코드가 하는 일

    • 나선 채우기 메인 루프 
      int[][] field = new int[N][N];
      int top = 0, bottom = N - 1;
      int left = 0, right = N - 1;
      int cur = 1, end = N * N;
      
      while (cur <= end) {
          // 1. 위쪽 행: 왼쪽 → 오른쪽
          for (int c = left; c <= right; c++) {
              field[top][c] = cur++;
          }
          top++;
      
          // 2. 오른쪽 열: 위 → 아래
          for (int r = top; r <= bottom; r++) {
              field[r][right] = cur++;
          }
          right--;
      
          // 3. 아래쪽 행: 오른쪽 → 왼쪽
          for (int c = right; c >= left; c--) {
              field[bottom][c] = cur++;
          }
          bottom--;
      
          // 4. 왼쪽 열: 아래 → 위
          for (int r = bottom; r >= top; r--) {
              field[r][left] = cur++;
          }
          left++;
      }
      • 위 행 채우기: (top, left) ~ (top, right)에 현재 숫자를 채우고 top을 한 칸 내려 새로운 “안쪽” 레이어로 들어간다.
      • 오른쪽 열: (top, right) ~ (bottom, right)를 채우고 right-- 해서 안쪽으로 이동.
      • 아래 행: (bottom, right) ~ (bottom, left)를 역방향으로 채우고 bottom--.
      • 왼쪽 열: (bottom, left) ~ (top, left)를 위로 채우고 left++.
      • 이 네 방향을 한 번 돌면 바깥 “테두리” 한 겹이 채워지고, 경계를 좁혀가며 계속 나선으로 들어간다.
      • cur <= end 조건만 보고 돌기 때문에, N이 홀수/짝수 상관없이 가운데까지 정확히 채워지며, 더 이상 채울 숫자가 없으면 반복을 종료한다.
    • 출력 부분 
      for (int i = 0; i < N; i++) {
          for (int j = 0; j < N; j++) {
              sb.append(field[i][j]).append(" ");
          }
          sb.append('\n');
      }
      
      System.out.println(sb.toString());
      • 완성된 field를 위에서부터 한 행씩 순회하면서, 각 행의 원소를 공백으로 구분해 출력한다.
      • StringBuilder를 사용해 입출력 속도도 최적화했다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 전체 격자 크기가 N×N이고, 각 칸은 정확히 한 번만 채운다.
      • 따라서 시간 복잡도는 O(N²)이다.
      • N ≤ 750이므로 최대 약 56만 칸으로 충분히 여유 있다.
    • 공간 복잡도
      • field 배열 하나만 사용하므로, 공간 복잡도는 O(N²).
