    문제 6021 — DNA Sequencing

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    DNA Sequencing (BOJ 6021)는
    각 황소(bull) i와 암소(cow) j의 조합마다, 부모가 될 수 있는 두 개체의 DNA를 기준으로 나머지 소들(황소/암소 포함) 중 자식이 될 수 있는 개체 수를 세는 문제다.

    • DNA 길이는 항상 25로 고정
    • 자식 조건:
      1. 자식은 부모 자신이 될 수 없음 (bull i, cow j 제외)
      2. 각 위치 l에서 자식 DNA의 문자가 bull[l] 또는 cow[l] 중 하나와 일치해야 함

    전략 개요

    • 관찰
      • M, F ≤ 20이라 전체 개체 수는 최대 40명이다.
      • 따라서 (bull, cow) 모든 조합 M*F에 대해, 나머지 개체 (M+F-2)를 전부 검사하는 완전탐색이 충분히 가능하다.
    • 구현 방향
      • 황소 DNA는 bulls[M][25], 암소 DNA는 cows[F][25]에 char로 저장
      • 각 (bull i, cow j)에 대해:
        1. 모든 후보 k(전체 M+F 중) 순회
        2. k가 부모(i 또는 j)면 건너뜀
        3. DNA 25칸을 전부 확인해서 조건 만족 시 카운트 증가

    코드가 하는 일

    • DNA 입력을 char 배열로 저장 
      char[][] bulls = new char[M][25];
      for (int i = 0; i < M; i++) {
          String s = br.readLine();
          for (int t = 0; t < 25; t++) bulls[i][t] = s.charAt(t);
      }
      
      char[][] cows = new char[F][25];
      for (int i = 0; i < F; i++) {
          String s = br.readLine();
          for (int t = 0; t < 25; t++) cows[i][t] = s.charAt(t);
      }
      • 문자열을 그대로 25칸 char 배열로 옮겨, 이후 비교를 빠르게 한다.
    • (bull i, cow j) 조합별로 자식 후보 세기 
      for (int i = 0; i < M; i++) {
          for (int j = 0; j < F; j++) {
              int cnt = 0;
      
              for (int k = 0; k < M + F; k++) {
                  if (k == i || k == M + j) continue; // 부모 자신 제외
      
                  char[] child = (k < M) ? bulls[k] : cows[k - M];
                  boolean ok = true;
      
                  for (int l = 0; l < 25; l++) {
                      if (child[l] != bulls[i][l] && child[l] != cows[j][l]) {
                          ok = false;
                          break;
                      }
                  }
      
                  if (ok) cnt++;
              }
      
              sb.append(cnt).append(" ");
          }
          sb.append('\n');
      }
      • k == i: i번째 황소가 자기 자신을 자식으로 두는 경우 방지
      • k == M + j: j번째 암소(전체 인덱스 기준)를 자식에서 제외
      • 각 위치 l에서 자식 문자가 부모 둘 중 하나와 일치하지 않으면 즉시 탈락
      • 끝까지 통과하면 해당 (i, j)의 자식 후보로 카운트
    • 출력 형식
      • 총 M줄 출력
      • 각 줄에 F개의 정수(암소 1..F에 대응)를 공백으로 출력

    시간 · 공간 복잡도

    • 시간 복잡도
      • (bull, cow) 조합: M * F
      • 각 조합에서 후보 검사: (M + F)
      • 각 후보 DNA 비교: 25
      • 따라서 전체는 O(M * F * (M + F) * 25)
      • 최대 20 * 20 * 40 * 25 = 400,000 수준이라 충분히 빠르다.
    • 공간 복잡도
      • DNA 저장: (M + F) * 25 → O(M + F) (상수 25)

    문제 6032 — Toy Shopping

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Toy Shopping (BOJ 6032)는
    N개의 장난감 중에서 서로 다른 3개를 골라 (Joy / Price) 값의 합이 최대가 되도록 만드는 문제다.

    • 장난감 i는 J_i(기쁨), P_i(가격)을 가진다.
    • 각 장난감의 “행복-절약 지표(HFM)”는 J_i / P_i.
    • 목표: HFM이 큰 장난감 3개를 선택(정답은 유일).

    핵심 아이디어

    • 합을 최대화하려면, 각 항(J/P)이 큰 것부터 3개를 고르는 것이 최적이다.
    • 즉, 모든 장난감을 J/P 내림차순으로 정렬한 뒤, 상위 3개를 선택하면 된다.
    • 문제는 실수 비교 오차를 피하기 위해 분수 비교를 곱셈으로 처리한다.

    분수 비교 (오차 없이)

    • a와 b의 HFM 비교는 다음과 같이 한다.
    a.J / a.P  vs  b.J / b.P
      
      실수로 나누지 않고,
      a.J * b.P  vs  b.J * a.P
      를 비교하면 된다.
    • 값 범위가 크므로 long으로 곱셈한다.

    전략 개요

    • 입력
      • N을 읽고, 각 장난감의 (index, J, P)를 리스트에 저장
    • 정렬
      • 비교 기준: J/P 내림차순
      • 구현: b.J * a.P 와 a.J * b.P 비교
      • 동률 처리: 문제는 정답 유일이지만, 안정적으로 idx로 한 번 더 정렬해둠
    • 선택 및 출력
      • 정렬 후 상위 3개 선택
      • 총 가격 = 세 장난감 가격 합
      • 출력은 “HFM 내림차순 순서대로” 3개 인덱스 출력

    코드가 하는 일

    • 정렬 로직 
      list.sort((a, b) -> {
          long left = (long) b.J * a.P;
          long right = (long) a.J * b.P;
      
          if (left != right) return Long.compare(left, right); // 내림차순
          return Integer.compare(a.idx, b.idx);
      });
      • left와 right는 분수 비교를 곱셈으로 바꾼 값
      • Long.compare(left, right)를 쓰면 HFM 큰 것이 앞으로 오게 된다
    • 상위 3개 선택 
      Toy toy1 = list.get(0);
      Toy toy2 = list.get(1);
      Toy toy3 = list.get(2);
      • 정렬이 끝났으니 앞의 3개가 최적 선택
    • 총 가격 출력 + 인덱스 출력 
      long price = (long) toy1.P + toy2.P + toy3.P;
      
      sb.append(price).append('\n');
      sb.append(toy1.idx).append('\n');
      sb.append(toy2.idx).append('\n');
      sb.append(toy3.idx).append('\n');
      • 총 가격은 최대 3 * 1e8 이라 int도 가능하지만, 안전하게 long
      • 인덱스는 이미 HFM 내림차순으로 정렬된 순서 그대로 출력

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬이 지배적: O(N log N)
      • N ≤ 25,000 이므로 충분히 빠름
    • 공간 복잡도
      • 장난감 리스트 저장: O(N)
