    문제 6052 — Cow Pals

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Cow Pals (BOJ 6052)는
    어떤 수 n에 대해 자기 자신을 제외한 약수의 합(proper divisor sum)을 sumDiv(n)이라 할 때, 서로가 서로의 cowpal인(= 우정수/amicable) 쌍을 찾는 문제다.

    • sumDiv(n) = n을 제외한 약수들의 합
    • superpal: 서로가 서로의 cowpal인 두 수 (a != b, sumDiv(a)=b 그리고 sumDiv(b)=a)
    • 자기 자신과의 superpal(완전수)은 제외하므로 sumDiv(i) == i는 건너뛴다.
    • 입력 S 이상인 수 중에서 가장 처음으로 superpal을 가지는 수 i를 찾아 i와 그 짝 x를 출력한다.

    전략 개요

    • 1) S부터 1씩 증가시키며 후보 i를 본다.
    • 2) x = sumDiv(i)를 계산한다.
    • 3) 완전수 제외: x == i면 continue
    • 4) 상호 검증: sumDiv(x) == i이면 i x 출력 후 종료

    코드가 하는 일

    • superpal 찾는 루프 
      for (int i = S;; i++) {
          int x = sumDiv(i);
      
          // superpal of itself(완전수)은 제외
          if (x == i) continue;
      
          // 상호 cowpal이면 정답
          if (x > 0 && sumDiv(x) == i) {
              System.out.println(i + " " + x);
              return;
          }
      }
      • i를 S부터 증가시키며 검사한다.
      • x == i면 자기 자신과의 쌍이므로 제외.
      • sumDiv(x) == i가 성립하면 i가 찾는 “첫 번째 superpal 보유 수”가 된다.
    • 약수 합(sumDiv) 계산 
      private static int sumDiv(int n) {
          if (n <= 1) return 0;
      
          int sum = 1; // 1은 항상 proper divisor
          for (int d = 2; d * d <= n; d++) {
              if (n % d == 0) {
                  sum += d;
      
                  int other = n / d;
                  if (other != d) sum += other; // 제곱수면 중복 방지
              }
          }
          return sum;
      }
      • n의 약수는 d와 n/d가 쌍으로 존재하므로 d를 sqrt(n)까지만 확인한다.
      • n 자신은 제외해야 하므로 초기값을 sum = 1로 두고 시작한다.
      • 제곱수인 경우(d*d == n)에는 같은 약수를 두 번 더하지 않도록 처리한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • sumDiv(n)는 O(sqrt(n)).
      • i를 S부터 정답까지 증가시키며(문제 조건상 해가 유일), 각 후보마다 sumDiv를 최대 2번 호출하므로 전체는 대략 O(K * sqrt(V)) 수준이다.
      • 여기서 V는 탐색 중 만나는 수의 크기(최대가 커도 S ≤ 18000이라 충분히 빠르게 통과).
    • 공간 복잡도
      • 추가 배열 없이 계산하므로 O(1).

    문제 6107 — Plumbing the Pond

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Plumbing the Pond (BOJ 6107)는
    레이더 값이 튀는 경우를 고려해서, 가장 깊은 값이라도 인접(8방향) 칸에 같은 값이 하나라도 있어야 믿는다는 규칙을 그대로 구현하면 되는 문제다.

    • 연못은 R x C 격자이며, 각 칸의 값은 깊이 D다.
    • 0은 연못이 아닌 칸이므로 무시한다.
    • 어떤 깊이 x가 인접한(상하좌우+대각선=8방향) 두 칸 이상에서 동시에 나타나면 “검증된 깊이”로 인정한다.
    • 그런 깊이들 중 최댓값을 출력한다.

    전략 개요

    • 모든 칸 (i, j)을 순회한다.
    • 현재 칸이 0이면 continue.
    • 8방향 이웃 (i + dr[k], j + dc[k])을 확인한다.
    • 범위 안이고, 이웃 값이 현재 값과 같으면 “검증 성공”이므로 max = max(max, 현재값) 갱신.

    코드가 하는 일

    • 8방향 이동 벡터 
      static final int[] row = {-1, -1, -1, 0, 0, 1, 1, 1};
      static final int[] col = {-1, 0, 1, -1, 1, -1, 0, 1};
      • 대각선을 포함한 8방향을 한 번에 탐색하기 위한 이동 배열이다.
    • 격자 입력 
      int[][] field = new int[R][C];
      for (int i = 0; i < R; i++) {
          st = new StringTokenizer(br.readLine());
          for (int j = 0; j < C; j++) {
              field[i][j] = Integer.parseInt(st.nextToken());
          }
      }
      • 깊이 정보를 2차원 배열에 저장한다.
    • 모든 칸에서 인접한 같은 값 찾기 
      int max = 0;
      for (int i = 0; i < R; i++) {
          for (int j = 0; j < C; j++) {
              if (field[i][j] == 0) continue;
      
              for (int k = 0; k < 8; k++) {
                  int nr = i + row[k];
                  int nc = j + col[k];
      
                  if (nr < 0 || nr >= R || nc < 0 || nc >= C) continue;
      
                  if (field[i][j] == field[nr][nc] && field[i][j] > max) {
                      max = field[i][j];
                  }
              }
          }
      }
      • 0은 연못이 아니므로 건너뛴다.
      • 8방향 이웃이 범위 밖이면 건너뛴다.
      • 이웃과 값이 같다면 “인접한 동일 깊이 2개 이상” 조건을 만족하므로 최댓값을 갱신한다.
    • 정답 출력 
      System.out.println(max);

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 칸마다 최대 8개의 이웃만 확인하므로 O(R*C).
      • R, C ≤ 50이라 매우 여유롭다.
    • 공간 복잡도
      • 격자 저장용 배열 하나 사용: O(R*C).
