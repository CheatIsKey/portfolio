    문제 6108 — The Perfect Cow

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    The Perfect Cow (BOJ 6108)는
    N×N 격자에 있는 소들의 아름다움 점수에서, 각 행의 중앙값(메디안)을 뽑고, 그 N개의 중앙값 중 다시 중앙값을 구하는 문제다.

    • N은 홀수이므로, 중앙값은 항상 N/2번째(0-index 기준)로 명확하다.
    • 각 행을 정렬해서 중앙값을 구하고, 그 결과 리스트를 다시 정렬해서 중앙값을 출력하면 끝.

    전략 개요

    • 1) 행 단위 중앙값 추출
      • 각 행의 N개 점수를 임시 리스트에 담는다.
      • 리스트를 오름차순 정렬한 뒤, N/2번째 값을 “그 행의 중앙값”으로 저장한다.
    • 2) 중앙값들의 중앙값 계산
      • 행 중앙값이 담긴 리스트(N개)를 정렬한다.
      • 그 리스트의 N/2번째 값을 출력한다.

    코드가 하는 일

    • 각 행을 정렬해서 중앙값을 뽑고 list2에 저장 
      for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          for (int j = 0; j < N; j++) {
              space[i][j] = Integer.parseInt(st.nextToken());
              list1.add(space[i][j]);
          }
      
          Collections.sort(list1);
          list2.add(list1.get(N / 2));
          list1.clear();
      }
      • list1은 “현재 행”의 값들을 담는 임시 버퍼.
      • 정렬 후 list1.get(N/2)가 그 행의 중앙값.
      • 중앙값은 list2에 누적하고, 다음 행을 위해 list1을 비운다.
    • 행 중앙값 리스트(list2)를 정렬한 뒤 중앙값 출력 
      Collections.sort(list2);
      System.out.println(list2.get(list2.size() / 2));
      • list2에는 총 N개의 행 중앙값이 들어있다.
      • 정렬 후 N/2번째가 “중앙값들의 중앙값” → 정답.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 행 정렬: N번 × O(N log N) → O(N^2 log N)
      • 중앙값 리스트 정렬: O(N log N) (위에 비해 매우 작음)
    • 공간 복잡도
      • 입력 저장 N×N + 리스트들 → 대략 O(N^2)

    문제 6110 — Watering - 2

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Plumbing the Pond (BOJ 6107)는
    레이더 값이 튀는 경우를 고려해서, 가장 깊은 값이라도 인접(8방향) 칸에 같은 값이 하나라도 있어야 믿는다는 규칙을 그대로 구현하면 되는 문제다.

    • 연못은 R x C 격자이며, 각 칸의 값은 깊이 D다.
    • 0은 연못이 아닌 칸이므로 무시한다.
    • 어떤 깊이 x가 인접한(상하좌우+대각선=8방향) 두 칸 이상에서 동시에 나타나면 “검증된 깊이”로 인정한다.
    • 그런 깊이들 중 최댓값을 출력한다.

    전략 개요

    • 모든 칸 (i, j)을 순회한다.
    • 현재 칸이 0이면 continue.
    • 8방향 이웃 (i + dr[k], j + dc[k])을 확인한다.
    • 범위 안이고, 이웃 값이 현재 값과 같으면 “검증 성공”이므로 max = max(max, 현재값) 갱신.

    코드가 하는 일

    • 8방향 이동 벡터 
      static final int[] row = {-1, -1, -1, 0, 0, 1, 1, 1};
      static final int[] col = {-1, 0, 1, -1, 1, -1, 0, 1};
      • 대각선을 포함한 8방향을 한 번에 탐색하기 위한 이동 배열이다.
    • 격자 입력 
      int[][] field = new int[R][C];
      for (int i = 0; i < R; i++) {
          st = new StringTokenizer(br.readLine());
          for (int j = 0; j < C; j++) {
              field[i][j] = Integer.parseInt(st.nextToken());
          }
      }
      • 깊이 정보를 2차원 배열에 저장한다.
    • 모든 칸에서 인접한 같은 값 찾기 
      int max = 0;
      for (int i = 0; i < R; i++) {
          for (int j = 0; j < C; j++) {
              if (field[i][j] == 0) continue;
      
              for (int k = 0; k < 8; k++) {
                  int nr = i + row[k];
                  int nc = j + col[k];
      
                  if (nr < 0 || nr >= R || nc < 0 || nc >= C) continue;
      
                  if (field[i][j] == field[nr][nc] && field[i][j] > max) {
                      max = field[i][j];
                  }
              }
          }
      }
      • 0은 연못이 아니므로 건너뛴다.
      • 8방향 이웃이 범위 밖이면 건너뛴다.
      • 이웃과 값이 같다면 “인접한 동일 깊이 2개 이상” 조건을 만족하므로 최댓값을 갱신한다.
    • 정답 출력 
      System.out.println(max);

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 칸마다 최대 8개의 이웃만 확인하므로 O(R*C).
      • R, C ≤ 50이라 매우 여유롭다.
    • 공간 복잡도
      • 격자 저장용 배열 하나 사용: O(R*C).
