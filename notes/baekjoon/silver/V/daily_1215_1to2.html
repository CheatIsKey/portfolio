    문제 6124 — Good Grass

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Good Grass (BOJ 6124)는
    NR×NC 격자에서 3×3 칸의 합이 가장 큰 구간을 찾고, 그 때의 최대 합과 좌상단 좌표(행, 열)를 출력하는 문제다.

    • 가능한 3×3의 좌상단은 (0..NR-3), (0..NC-3) 범위에서만 등장한다.
    • 각 좌상단마다 9칸을 더해 합을 구하고, 최대값을 갱신한다.
    • 동점일 때는 “행이 작은 것, 그 다음 열이 작은 것”을 선택해야 하는데, 행을 바깥 루프, 열을 안쪽 루프로 오름차순 순회하면서 sum > max일 때만 갱신하면 가장 먼저 나온 최대 구간이 자동으로 규칙을 만족한다.

    전략 개요

    • 입력
      • NR, NC를 읽고, field[NR][NC]에 값을 저장한다.
    • 모든 3×3 구간 탐색
      • 좌상단을 (r, c)라고 하면, r = 0..NR-3, c = 0..NC-3를 순회한다.
      • 각 위치에서 field[r+i][c+j] (i,j=0..2) 9칸을 더해 sum을 만든다.
      • sum이 현재 max보다 크면 max와 좌표를 갱신한다.
    • 출력
      • 최대 합을 출력하고, 좌표는 문제 요구가 1-based이므로 (r+1, c+1)로 출력한다.

    코드가 하는 일

    • 격자 입력 
      int[][] field = new int[NR][NC];
      for (int r = 0; r < NR; r++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          for (int c = 0; c < NC; c++) {
              field[r][c] = Integer.parseInt(st.nextToken());
          }
      }
      • NR줄을 읽어 NC개 정수를 field에 저장한다.
    • 3×3 합 계산 + 최대 갱신 
      int max = 0;
      int bestR = 0, bestC = 0;
      
      for (int r = 0; r <= NR - 3; r++) {
          for (int c = 0; c <= NC - 3; c++) {
              int sum = 0;
      
              for (int i = 0; i < 3; i++) {
                  for (int j = 0; j < 3; j++) {
                      sum += field[r + i][c + j];
                  }
              }
      
              if (sum > max) {
                  max = sum;
                  bestR = r;
                  bestC = c;
              }
          }
      }
      • 모든 좌상단 후보를 훑으며 9칸 합을 구한다.
      • sum > max일 때만 갱신하면, 같은 최대 합이 여러 개여도 가장 먼저 발견된(행/열이 가장 작은) 위치가 남아서 동점 규칙을 만족한다.
    • 출력(1-based 변환) 
      System.out.println(max);
      System.out.println((bestR + 1) + " " + (bestC + 1));
      • 문제는 행/열을 1부터 세므로 +1 해서 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 좌상단 후보가 약 (NR-2)(NC-2)개, 각 후보에서 9칸 합 → O(NR * NC).
      • NR, NC 최대가 100이라 충분히 빠르다.
    • 공간 복잡도
      • 격자 저장 field[NR][NC] → O(NR * NC).

    문제 6107 — Plumbing the Pond

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Plumbing the Pond (BOJ 6107)는
    레이더 값이 튀는 경우를 고려해서, 가장 깊은 값이라도 인접(8방향) 칸에 같은 값이 하나라도 있어야 믿는다는 규칙을 그대로 구현하면 되는 문제다.

    • 연못은 R x C 격자이며, 각 칸의 값은 깊이 D다.
    • 0은 연못이 아닌 칸이므로 무시한다.
    • 어떤 깊이 x가 인접한(상하좌우+대각선=8방향) 두 칸 이상에서 동시에 나타나면 “검증된 깊이”로 인정한다.
    • 그런 깊이들 중 최댓값을 출력한다.

    전략 개요

    • 모든 칸 (i, j)을 순회한다.
    • 현재 칸이 0이면 continue.
    • 8방향 이웃 (i + dr[k], j + dc[k])을 확인한다.
    • 범위 안이고, 이웃 값이 현재 값과 같으면 “검증 성공”이므로 max = max(max, 현재값) 갱신.

    코드가 하는 일

    • 8방향 이동 벡터 
      static final int[] row = {-1, -1, -1, 0, 0, 1, 1, 1};
      static final int[] col = {-1, 0, 1, -1, 1, -1, 0, 1};
      • 대각선을 포함한 8방향을 한 번에 탐색하기 위한 이동 배열이다.
    • 격자 입력 
      int[][] field = new int[R][C];
      for (int i = 0; i < R; i++) {
          st = new StringTokenizer(br.readLine());
          for (int j = 0; j < C; j++) {
              field[i][j] = Integer.parseInt(st.nextToken());
          }
      }
      • 깊이 정보를 2차원 배열에 저장한다.
    • 모든 칸에서 인접한 같은 값 찾기 
      int max = 0;
      for (int i = 0; i < R; i++) {
          for (int j = 0; j < C; j++) {
              if (field[i][j] == 0) continue;
      
              for (int k = 0; k < 8; k++) {
                  int nr = i + row[k];
                  int nc = j + col[k];
      
                  if (nr < 0 || nr >= R || nc < 0 || nc >= C) continue;
      
                  if (field[i][j] == field[nr][nc] && field[i][j] > max) {
                      max = field[i][j];
                  }
              }
          }
      }
      • 0은 연못이 아니므로 건너뛴다.
      • 8방향 이웃이 범위 밖이면 건너뛴다.
      • 이웃과 값이 같다면 “인접한 동일 깊이 2개 이상” 조건을 만족하므로 최댓값을 갱신한다.
    • 정답 출력 
      System.out.println(max);

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 칸마다 최대 8개의 이웃만 확인하므로 O(R*C).
      • R, C ≤ 50이라 매우 여유롭다.
    • 공간 복잡도
      • 격자 저장용 배열 하나 사용: O(R*C).
