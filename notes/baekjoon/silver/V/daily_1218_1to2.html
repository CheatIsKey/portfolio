    문제 6124 — Good Grass

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Good Grass (BOJ 6124)는
    NR×NC 격자에서 3×3 칸의 합이 가장 큰 구간을 찾고, 그 때의 최대 합과 좌상단 좌표(행, 열)를 출력하는 문제다.

    • 가능한 3×3의 좌상단은 (0..NR-3), (0..NC-3) 범위에서만 등장한다.
    • 각 좌상단마다 9칸을 더해 합을 구하고, 최대값을 갱신한다.
    • 동점일 때는 “행이 작은 것, 그 다음 열이 작은 것”을 선택해야 하는데, 행을 바깥 루프, 열을 안쪽 루프로 오름차순 순회하면서 sum > max일 때만 갱신하면 가장 먼저 나온 최대 구간이 자동으로 규칙을 만족한다.

    전략 개요

    • 입력
      • NR, NC를 읽고, field[NR][NC]에 값을 저장한다.
    • 모든 3×3 구간 탐색
      • 좌상단을 (r, c)라고 하면, r = 0..NR-3, c = 0..NC-3를 순회한다.
      • 각 위치에서 field[r+i][c+j] (i,j=0..2) 9칸을 더해 sum을 만든다.
      • sum이 현재 max보다 크면 max와 좌표를 갱신한다.
    • 출력
      • 최대 합을 출력하고, 좌표는 문제 요구가 1-based이므로 (r+1, c+1)로 출력한다.

    코드가 하는 일

    • 격자 입력 
      int[][] field = new int[NR][NC];
      for (int r = 0; r < NR; r++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          for (int c = 0; c < NC; c++) {
              field[r][c] = Integer.parseInt(st.nextToken());
          }
      }
      • NR줄을 읽어 NC개 정수를 field에 저장한다.
    • 3×3 합 계산 + 최대 갱신 
      int max = 0;
      int bestR = 0, bestC = 0;
      
      for (int r = 0; r <= NR - 3; r++) {
          for (int c = 0; c <= NC - 3; c++) {
              int sum = 0;
      
              for (int i = 0; i < 3; i++) {
                  for (int j = 0; j < 3; j++) {
                      sum += field[r + i][c + j];
                  }
              }
      
              if (sum > max) {
                  max = sum;
                  bestR = r;
                  bestC = c;
              }
          }
      }
      • 모든 좌상단 후보를 훑으며 9칸 합을 구한다.
      • sum > max일 때만 갱신하면, 같은 최대 합이 여러 개여도 가장 먼저 발견된(행/열이 가장 작은) 위치가 남아서 동점 규칙을 만족한다.
    • 출력(1-based 변환) 
      System.out.println(max);
      System.out.println((bestR + 1) + " " + (bestC + 1));
      • 문제는 행/열을 1부터 세므로 +1 해서 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 좌상단 후보가 약 (NR-2)(NC-2)개, 각 후보에서 9칸 합 → O(NR * NC).
      • NR, NC 최대가 100이라 충분히 빠르다.
    • 공간 복잡도
      • 격자 저장 field[NR][NC] → O(NR * NC).

    문제 6138 — Exploration

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Exploration (BOJ 6138)는
    소들이 있는 좌표들 중에서, Bessie가 항상 원점(0)과의 거리가 가장 가까운 미방문 랜드마크부터 방문한다고 할 때, 제한 시간 T 안에 최대로 몇 개를 방문할 수 있는지 구하는 문제다.

    • 방문 순서는 “원점과의 거리” 기준이므로, 결국 |x| 오름차순으로 정해진다. (문제에서 같은 거리는 없음)
    • 시작 위치는 pos = 0이고, 다음 랜드마크로 이동할 때 걸리는 시간은 거리만큼 누적된다.
    • 정해진 순서대로 하나씩 시뮬레이션하면서, 누적 시간이 T를 넘는 순간 멈추면 된다.

    전략 개요

    • 정렬
      • 모든 좌표를 배열에 담고, |x| 기준 오름차순 정렬한다.
    • 시뮬레이션
      • 현재 위치 pos, 누적 이동시간 dist를 두고 정렬된 순서대로 이동한다.
      • 각 랜드마크 next로 이동하는 비용은 1차원 직선 거리이므로 |next - pos|.
      • 누적 시간이 T를 초과하면 중단, 아니면 방문 수를 증가.
    • 출력
      • 방문에 성공한 개수만 출력한다.

    코드가 하는 일

    • |x| 기준 정렬 
      Arrays.sort(landmark, (a, b) -> Long.compare(Math.abs(a), Math.abs(b)));
      • 방문 규칙이 “원점에 가장 가까운 것부터”이므로, 방문 순서를 그대로 만들기 위해 |x|로 정렬한다.
    • 이동 시간 누적 + 방문 카운트 
      long pos = 0, dist = 0, cnt = 0;
      
      for (int i = 0; i < landmark.length; i++) {
          long next = landmark[i];
      
          // (코드는 부호 케이스로 나눠 계산하지만, 의미는 |next - pos| 누적이다)
          if ((next < 0 && pos > 0) || (next > 0 && pos < 0)) {
              dist += Math.abs(next) + Math.abs(pos);
          } else {
              long max = Math.max(Math.abs(next), Math.abs(pos));
              long min = Math.min(Math.abs(next), Math.abs(pos));
              dist += (max - min);
          }
      
          pos = next;
      
          if (dist > T) break;
          cnt++;
      }
      • 정렬된 순서대로 다음 랜드마크로 이동하면서 이동시간을 누적한다.
      • 누적 시간이 T를 넘는 순간 더 이상 방문 불가이므로 중단한다.
      • cnt가 정답(방문한 랜드마크 수).
    • 정답 출력 
      System.out.println(cnt);

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 시뮬레이션: O(N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 좌표 배열 저장: O(N)
