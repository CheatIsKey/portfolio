    문제 6147 — Bookshelf

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Bookshelf (BOJ 6147)는
    여러 마리 소의 키 H_i 중 일부를 골라 합이 책장 높이 B 이상이 되게 만들 때, 그때 필요한 최소 소 마릿수를 구하는 문제다.

    • 키는 전부 양수이므로, 마릿수를 최소로 하려면 한 마리당 키가 큰 소부터 쓰는 게 유리하다.
    • “k마리로 만들 수 있는 최대 높이”는 키가 큰 순서대로 k마리를 고른 합이다.
    • 따라서 큰 키부터 더해가며 합이 B 이상이 되는 순간의 마릿수가 정답이다.

    전략 개요

    • 입력
      • N, B를 읽고, N개의 키를 리스트에 저장한다.
    • 정렬
      • 키를 내림차순으로 정렬한다.
    • 누적
      • 가장 큰 키부터 차례대로 더하며(또는 B에서 빼며) B 이상이 되는 순간의 개수를 출력한다.

    코드가 하는 일

    • 키 내림차순 정렬 
      Collections.sort(list, (a, b) -> Integer.compare(b, a));
      • 큰 키부터 먼저 쓰기 위해 내림차순으로 정렬한다.
    • 큰 키부터 누적해서 최소 마릿수 찾기 
      int cnt = 0;
      
      for (int height : list) {
          B -= height;
          cnt++;
      
          if (B <= 0) break;
      }
      
      System.out.println(cnt);
      • 정렬된 리스트를 앞에서부터 보며 B에서 키를 뺀다.
      • B가 0 이하가 되는 순간, 그때까지 사용한 소의 수(cnt)가 최소다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 한 번 순회 누적: O(N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 키 리스트 저장: O(N)

    문제 6138 — Exploration

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Exploration (BOJ 6138)는
    소들이 있는 좌표들 중에서, Bessie가 항상 원점(0)과의 거리가 가장 가까운 미방문 랜드마크부터 방문한다고 할 때, 제한 시간 T 안에 최대로 몇 개를 방문할 수 있는지 구하는 문제다.

    • 방문 순서는 “원점과의 거리” 기준이므로, 결국 |x| 오름차순으로 정해진다. (문제에서 같은 거리는 없음)
    • 시작 위치는 pos = 0이고, 다음 랜드마크로 이동할 때 걸리는 시간은 거리만큼 누적된다.
    • 정해진 순서대로 하나씩 시뮬레이션하면서, 누적 시간이 T를 넘는 순간 멈추면 된다.

    전략 개요

    • 정렬
      • 모든 좌표를 배열에 담고, |x| 기준 오름차순 정렬한다.
    • 시뮬레이션
      • 현재 위치 pos, 누적 이동시간 dist를 두고 정렬된 순서대로 이동한다.
      • 각 랜드마크 next로 이동하는 비용은 1차원 직선 거리이므로 |next - pos|.
      • 누적 시간이 T를 초과하면 중단, 아니면 방문 수를 증가.
    • 출력
      • 방문에 성공한 개수만 출력한다.

    코드가 하는 일

    • |x| 기준 정렬 
      Arrays.sort(landmark, (a, b) -> Long.compare(Math.abs(a), Math.abs(b)));
      • 방문 규칙이 “원점에 가장 가까운 것부터”이므로, 방문 순서를 그대로 만들기 위해 |x|로 정렬한다.
    • 이동 시간 누적 + 방문 카운트 
      long pos = 0, dist = 0, cnt = 0;
      
      for (int i = 0; i < landmark.length; i++) {
          long next = landmark[i];
      
          // (코드는 부호 케이스로 나눠 계산하지만, 의미는 |next - pos| 누적이다)
          if ((next < 0 && pos > 0) || (next > 0 && pos < 0)) {
              dist += Math.abs(next) + Math.abs(pos);
          } else {
              long max = Math.max(Math.abs(next), Math.abs(pos));
              long min = Math.min(Math.abs(next), Math.abs(pos));
              dist += (max - min);
          }
      
          pos = next;
      
          if (dist > T) break;
          cnt++;
      }
      • 정렬된 순서대로 다음 랜드마크로 이동하면서 이동시간을 누적한다.
      • 누적 시간이 T를 넘는 순간 더 이상 방문 불가이므로 중단한다.
      • cnt가 정답(방문한 랜드마크 수).
    • 정답 출력 
      System.out.println(cnt);

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 시뮬레이션: O(N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 좌표 배열 저장: O(N)
