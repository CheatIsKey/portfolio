    문제 6147 — Bookshelf

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Bookshelf (BOJ 6147)는
    여러 마리 소의 키 H_i 중 일부를 골라 합이 책장 높이 B 이상이 되게 만들 때, 그때 필요한 최소 소 마릿수를 구하는 문제다.

    • 키는 전부 양수이므로, 마릿수를 최소로 하려면 한 마리당 키가 큰 소부터 쓰는 게 유리하다.
    • “k마리로 만들 수 있는 최대 높이”는 키가 큰 순서대로 k마리를 고른 합이다.
    • 따라서 큰 키부터 더해가며 합이 B 이상이 되는 순간의 마릿수가 정답이다.

    전략 개요

    • 입력
      • N, B를 읽고, N개의 키를 리스트에 저장한다.
    • 정렬
      • 키를 내림차순으로 정렬한다.
    • 누적
      • 가장 큰 키부터 차례대로 더하며(또는 B에서 빼며) B 이상이 되는 순간의 개수를 출력한다.

    코드가 하는 일

    • 키 내림차순 정렬 
      Collections.sort(list, (a, b) -> Integer.compare(b, a));
      • 큰 키부터 먼저 쓰기 위해 내림차순으로 정렬한다.
    • 큰 키부터 누적해서 최소 마릿수 찾기 
      int cnt = 0;
      
      for (int height : list) {
          B -= height;
          cnt++;
      
          if (B <= 0) break;
      }
      
      System.out.println(cnt);
      • 정렬된 리스트를 앞에서부터 보며 B에서 키를 뺀다.
      • B가 0 이하가 되는 순간, 그때까지 사용한 소의 수(cnt)가 최소다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 한 번 순회 누적: O(N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 키 리스트 저장: O(N)

    문제 6159 — Costume Party

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Costume Party (BOJ 6159)는
    소들의 길이 L_i가 주어질 때, 서로 다른 두 마리를 골라 합이 의상 길이 S 이하가 되는 쌍의 개수를 세는 문제다.

    • 쌍은 순서가 없으므로 (i, j)와 (j, i)는 같은 쌍이다.
    • N이 최대 20,000이라서, 모든 쌍을 전부 비교하는 방식은 비효율적이다.
    • 정렬 + 투 포인터로 쌍을 한 번의 스캔으로 계산할 수 있다.

    전략 개요

    • 1) 정렬
      • 소들의 길이를 오름차순 정렬한다.
    • 2) 투 포인터
      • 가장 짧은 쪽 l, 가장 긴 쪽 r를 잡는다.
      • 만약 cows[l] + cows[r] <= S라면:
        • 정렬되어 있으므로 l과 짝지을 수 있는 상대는 l+1..r 전부 가능하다.
        • 따라서 한 번에 (r - l)개의 쌍을 더하고 l++.
      • 만약 cows[l] + cows[r] > S라면:
        • 가장 큰 값 cows[r]가 너무 커서 어떤 l과도 맞추기 어렵다.
        • r--로 줄여서 합을 낮춘다.
    • 3) 정답
      • 누적한 쌍의 개수를 출력한다. (개수는 커질 수 있으니 long 권장)

    코드가 하는 일

    • 정렬 
      Arrays.sort(cows);
      • 오름차순 정렬로 투 포인터 판단이 가능해진다.
    • 투 포인터로 쌍 개수 누적 
      int l = 0, r = N - 1;
      long cnt = 0;
      
      while (l < r) {
          int sum = cows[l] + cows[r];
      
          if (sum <= S) {
              cnt += (r - l);
              l++;
          } else {
              r--;
          }
      }
      • sum <= S면 l은 l+1..r와 전부 짝 가능 → r-l만큼 한 번에 더한다.
      • sum > S면 큰 쪽을 줄이기 위해 r--.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 투 포인터 스캔: O(N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 배열 저장: O(N)
