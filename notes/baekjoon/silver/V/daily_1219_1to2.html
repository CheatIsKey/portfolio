    문제 6160 — Election Time

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Election Time (BOJ 6160)은 선거가 2단계로 진행되는 상황에서 최종 당선(2라운드 최다 득표) 후보의 인덱스를 구하는 문제다.

    • 1라운드: Ai 표가 많은 순으로 상위 K명이 2라운드 진출
    • 2라운드: 진출자 중 Bi 표가 가장 큰 후보가 당선
    • A 목록은 중복 없음, B 목록도 중복 없음 → 동점 처리를 고민할 필요가 없다

    전략 개요

    • 모든 후보를 (idx, A, B)로 저장한다.
    • A 내림차순 정렬을 하면, 앞에서부터 K명이 2라운드 진출자다.
    • 진출자 K명 중에서 B가 가장 큰 후보를 찾고 그 idx를 출력한다.

    코드가 하는 일

    • 1) 1라운드 기준(A)으로 정렬 
      Collections.sort(list, (a, b) -> Integer.compare(b.firstRound, a.firstRound));
      • firstRound(= A)를 큰 순서대로 정렬한다.
      • 정렬 후 list[0..K-1]가 2라운드 진출자다.
    • 2) 진출자 K명 중 2라운드 표(B)가 가장 큰 후보 선택 
      Cows best = list.get(0);
      for (int i = 1; i < K; i++) {
          if (list.get(i).finalRound > best.finalRound) {
              best = list.get(i);
          }
      }
      • 진출자들만 보면서 finalRound(= B) 최대값을 가진 후보를 찾는다.
      • B도 중복이 없으므로 최댓값 후보는 유일하다.
    • 3) 정답 출력 
      System.out.println(best.idx);
      • 당선 후보의 입력 순서(1-based 인덱스)를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정렬: O(N log N)
      • 진출자 K명 스캔: O(K)
      • 전체: O(N log N)
    • 공간 복잡도
      • 후보 리스트 저장: O(N)

    문제 6161 — iCow

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    iCow (BOJ 6161)은
    매번 현재 평점이 가장 높은 노래를 재생하고, 그 평점을 나머지 노래들에게 규칙대로 분배하는 과정을 T번 시뮬레이션해서 재생된 노래 번호를 출력하는 문제다.

    • 다음에 재생할 노래: 평점이 가장 큰 노래 (동점이면 인덱스가 가장 작은 노래)
    • 재생 후 처리:
      • 재생한 노래의 평점은 0이 된다.
      • 그 평점 x를 N-1개의 다른 노래에 균등 분배:
        • q = x / (N-1)는 모두에게 더해지고
        • r = x % (N-1)는 앞에서부터 1점씩(단, 방금 재생한 노래는 제외) 추가로 배분

    전략 개요

    • 자료구조
      • music[i] = i번 노래의 현재 평점 (1-based)
    • T번 반복 시뮬레이션
      1. music에서 최댓값(동점이면 작은 인덱스)인 cur를 찾는다.
      2. cur를 출력한다.
      3. 평점 x = music[cur]를 꺼내고, music[cur] = 0으로 만든다.
      4. q = x/(N-1), r = x%(N-1)를 구한다.
      5. cur를 제외한 모든 노래에 q를 더하고, 앞에서부터 r개 노래에 1씩 추가한다.
    • 주의
      • N-1로 나누는 구조라, N=1 입력이 가능한지는 문제 조건을 다시 확인하는 게 안전하다(가능하다면 예외 처리 필요). (확실하지 않음)
      • 이 풀이(배열 선형 탐색/선형 분배)는 N, T ≤ 1000 수준이면 충분히 통과한다.

    코드가 하는 일

    • 1) 다음 재생 노래 선택 (최대 평점, 동점이면 작은 인덱스) 
      int cur = 1;
      for (int j = 2; j <= N; j++) {
          if (music[j] > music[cur]) cur = j;
          else if (music[j] == music[cur] && j < cur) cur = j;
      }
      • 한 번의 재생마다 배열을 끝까지 훑어서 기준에 맞는 cur를 고른다.
    • 2) 평점 분배 준비 
      int x = music[cur];
      music[cur] = 0;
      
      int q = x / (N - 1);
      int r = x % (N - 1);
      • x를 꺼내고 cur은 0으로 만든 뒤, 몫/나머지를 구한다.
    • 3) 몫 q를 모든 다른 노래에 더하기 
      for (int j = 1; j <= N; j++) {
          if (j == cur) continue;
          music[j] += q;
      }
      • cur만 제외하고 전부 q를 더한다.
    • 4) 나머지 r을 앞에서부터 1씩 추가 (cur 제외) 
      for (int j = 1; j <= N && r > 0; j++) {
          if (j == cur) continue;
          music[j] += 1;
          r--;
      }
      • 인덱스가 작은 노래부터 1점씩 추가로 배분한다(단, 방금 재생한 노래는 제외).

    시간 · 공간 복잡도

    • 시간 복잡도
      • 재생 1회당
        • 최대값 탐색: O(N)
        • 몫 분배: O(N)
        • 나머지 분배: 최악 O(N)
      • 총 O(T · N) (상수 포함해도 N,T ≤ 1000이면 충분히 빠름)
    • 공간 복잡도
      • 평점 배열 music만 사용하므로 O(N)
