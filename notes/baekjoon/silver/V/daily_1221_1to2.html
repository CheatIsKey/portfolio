    문제 6169 — Long Distance Racing

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Long Distance Racing (BOJ 6169)는
    출발점(농장)에서 앞으로 T칸짜리 길을 달릴 때, 왕복 시간이 제한 시간 M을 넘지 않도록 하면서 가장 멀리(몇 칸)까지 갈 수 있는지 구하는 문제다.

    • 길은 한 칸씩 u(오르막), f(평지), d(내리막)로 주어진다.
    • 갈 때 시간:
      • u: 1칸에 U초
      • f: 1칸에 F초
      • d: 1칸에 D초
    • 올 때는 지형이 뒤집힌다:
      • 갈 때 u였던 칸은 돌아올 때 d가 되어 D초
      • 갈 때 d였던 칸은 돌아올 때 u가 되어 U초
      • f는 왕복 모두 F초
    • 따라서 한 칸을 “왕복”하는 비용은
      • u 또는 d: U + D
      • f: 2 * F

    전략 개요

    • 문제는 “연속된 앞부분을 몇 칸까지 밟을 수 있는가” 형태다.
    • 1칸부터 순서대로 보면서 해당 칸의 왕복 비용을 누적한다.
    • 누적 시간이 M을 초과하는 순간 더 이상 진행 불가 → 그 직전까지의 칸 수가 정답.

    코드가 하는 일

    • 1) 입력 및 기본 왕복 비용 계산 
      int M = ...;
      int T = ...;
      int U = ...;
      int F = ...;
      int D = ...;
      
      int hill = U + D;   // u/d 칸의 왕복 시간
      int flat = 2 * F;   // f 칸의 왕복 시간
      • u와 d는 왕복하면 항상 U + D로 동일하다.
      • f는 왕복이면 2F.
    • 2) 1칸씩 보면서 누적 시간 체크 
      int time = 0, cnt = 0;
      
      for (int i = 0; i < T; i++) {
          char s = br.readLine().charAt(0);
      
          int cost = (s == 'f') ? flat : hill;
      
          if (time + cost > M) break;
      
          time += cost;
          cnt++;
      }
      • 각 구간 문자를 읽고, 왕복 비용(cost)을 결정한다.
      • 누적 시간에 더했을 때 M을 넘으면 즉시 중단한다.
      • 가능한 구간 수를 cnt로 센다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 구간을 한 번만 순회하므로 O(T)
      • T ≤ 100,000이라 충분히 빠르다.
    • 공간 복잡도
      • 추가 배열 없이 누적 변수만 사용 → O(1)

    문제 6177 — Statistics

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Statistics (BOJ 6177)는
    주어진 정수 N개의 데이터로부터 평균(mean)과 중앙값(median)을 구해 출력하는 기본 통계 문제다.

    • 평균: 모든 수의 합을 N으로 나눈 값
    • 중앙값:
      • N이 홀수면, 정렬했을 때 가운데 값
      • N이 짝수면, 정렬했을 때 가운데 두 값의 평균
    • 출력은 정답과의 오차가 0.002 이내면 통과하므로, double로 계산 후 소수점 6자리 정도 출력하면 충분하다.

    전략 개요

    • 입력
      • 정수 N을 읽고, 길이 N 배열에 값을 저장한다.
      • 동시에 누적합 sum을 계산한다.
    • 평균
      • mean = (double) sum / N
    • 중앙값
      • 배열을 오름차순 정렬한다.
      • N이 홀수면 arr[N/2]
      • N이 짝수면 (arr[N/2 - 1] + arr[N/2]) / 2.0
    • 출력
      • printf("%.6f") 형태로 평균과 중앙값을 각각 한 줄씩 출력한다.

    코드가 하는 일

    • 입력 + 합 계산 
      int N = Integer.parseInt(br.readLine());
      int[] arr = new int[N];
      int sum = 0;
      
      for (int i = 0; i < N; i++) {
          int num = Integer.parseInt(br.readLine());
          arr[i] = num;
          sum += num;
      }
      • 배열에 값을 저장하면서 동시에 sum으로 누적합을 만든다.
    • 정렬 
      Arrays.sort(arr);
      • 중앙값을 구하려면 정렬이 필요하다.
    • 평균/중앙값 계산 
      double mean = (double) sum / N;
      
      int mid = N / 2;
      double median;
      
      if (N % 2 == 0) {
          median = (arr[mid - 1] + arr[mid]) / 2.0;
      } else {
          median = arr[mid];
      }
      • N이 짝수면 가운데 두 값의 평균을 내야 하므로 2.0으로 나눠 double로 만든다.
      • N이 홀수면 가운데 값이 그대로 중앙값이다.
    • 출력 
      System.out.printf("%.6f\n", mean);
      System.out.printf("%.6f\n", median);
      • 소수점 6자리로 출력하면 오차 조건(0.002)을 충분히 만족한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 입력/합: O(N)
      • 정렬: O(N log N)
      • 전체: O(N log N)
    • 공간 복잡도
      • 정수 배열 arr만 사용: O(N)
