    문제 6186 — Best Grass

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Best Grass (BOJ 6186)는
    R x C 격자에서 '#'로 표시된 풀을 보고, 서로 상하좌우로 붙어 있는 '#'들이 이루는 풀 뭉치(clump)의 개수를 세는 문제다.

    • 풀 뭉치는 최대 2칸으로만 이루어진다:
      • '#' 한 칸짜리
      • 또는 상하좌우로 붙은 '# #'(대각선은 아님) 두 칸짜리
    • 서로 다른 뭉치끼리는 인접하지 않는다(붙어 있지 않다).
    • 즉, 어떤 '#'(뭉치의 한 칸)를 발견하면 그 뭉치는 그 칸 + (있다면) 상하좌우 중 하나의 '#'로 끝난다.

    전략 개요

    • 격자를 위에서 아래로, 왼쪽에서 오른쪽으로 순회한다.
    • '#'를 발견하면 새 풀 뭉치 1개로 세고(cnt++)
    • 그 칸의 상하좌우를 확인해서 '#'가 있으면 '.'로 바꿔 같은 뭉치를 제거한다.
    • 마지막으로 현재 칸도 '.'로 바꿔 중복 카운트를 막는다.

    코드가 하는 일

    • 1) 방향 배열 준비 (상/좌/우/하) 
      static final int[] ROW = {-1, 0, 0, 1};
      static final int[] COL = {0, -1, 1, 0};
      • 현재 칸 기준으로 상하좌우를 쉽게 확인하기 위한 오프셋이다.
    • 2) 입력 받아 격자 구성 
      int R = Integer.parseInt(st.nextToken());
      int C = Integer.parseInt(st.nextToken());
      
      char[][] field = new char[R][C];
      for (int r = 0; r < R; r++) {
          field[r] = br.readLine().toCharArray();
      }
      • 각 줄을 그대로 char[]로 받아 field에 저장한다.
    • 3) 격자 순회하며 뭉치 발견 시 카운트 + 제거 
      int cnt = 0;
      for (int r = 0; r < R; r++) {
          for (int c = 0; c < C; c++) {
              if (field[r][c] == '#') {
                  cnt++;
      
                  for (int i = 0; i < 4; i++) {
                      int row = r + ROW[i];
                      int col = c + COL[i];
      
                      if (row < 0 || R <= row || col < 0 || C <= col) continue;
                      if (field[row][col] == '#') field[row][col] = '.';
                  }
      
                  field[r][c] = '.';
              }
          }
      }
      • '#'를 만나면 그 순간이 곧 새 뭉치의 시작이므로 cnt++.
      • 상하좌우를 확인해 범위를 벗어나면 건너뛰고, 인접 칸이 '#'면 '.'로 바꾼다.
      • 현재 칸도 마지막에 '.'로 바꿔 다음 순회에서 중복으로 세지 않게 한다.
      • 문제 조건상 뭉치 크기가 최대 2칸이고 뭉치끼리 인접하지 않아서, 현재 칸 주변의 '#'를 지우는 것만으로도 같은 뭉치를 충분히 제거할 수 있다.
    • 4) 정답 출력 
      System.out.println(cnt);
      • 세어 둔 풀 뭉치 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 격자 전체를 한 번 순회하며, 각 '#'에서 상하좌우 4칸만 확인한다.
      • 따라서 O(R * C).
    • 공간 복잡도
      • 입력 격자 field 외 추가 자료구조는 거의 없으므로 O(R * C).

    문제 6212 — Dream Counting

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Dream Counting (BOJ 6212)는
    두 정수 M부터 N까지의 모든 수를 순서대로 나열했을 때, 각 숫자 0..9가 몇 번 등장하는지를 세는 문제다.

    • 예를 들어 129..137이면 129, 130, 131, ... , 137을 모두 보면서 각 자리의 숫자를 카운트한다.
    • 출력은 0부터 9까지의 등장 횟수를 공백으로 구분해 한 줄에 출력한다.
    • 범위 조건:
      • 1 ≤ M ≤ N ≤ 2,000,000,000
      • N - M ≤ 500,000 (연속 구간 길이가 최대 500,001개)

    전략 개요

    • M부터 N까지 모든 수를 순회한다.
    • 각 수를 10으로 나누면서(% 10, /= 10) 자릿수를 하나씩 꺼내 cnt[digit]를 증가시킨다.
    • 순회가 끝나면 cnt[0..9]를 차례대로 출력한다.

    코드가 하는 일

    • 1) 입력 처리 및 카운트 배열 준비 
      long M = Integer.parseInt(st.nextToken());
      long N = Integer.parseInt(st.nextToken());
      
      long[] cnt = new long[10];
      • M, N을 입력받고, 각 숫자(0~9) 등장 횟수를 담을 배열 cnt를 만든다.
    • 2) M부터 N까지 순회하며 자릿수 카운트 
      for (long i = M; i <= N; i++) {
          long x = i;
      
          while (x > 0) {
              cnt[(int) (x % 10)]++;
              x /= 10;
          }
      }
      • 각 수 i를 x에 복사한 뒤, x가 0이 될 때까지 반복한다.
      • x % 10으로 마지막 자릿수를 꺼내 해당 인덱스의 카운트를 증가시킨다.
      • x /= 10으로 마지막 자릿수를 제거한다.
    • 3) 결과 출력 
      for (long i : cnt) {
          System.out.print(i + " ");
      }
      • cnt[0]부터 cnt[9]까지 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 수의 개수를 K = N - M + 1이라고 하면, 각 수는 자릿수만큼(최대 10자리) 나누기를 수행한다.
      • 따라서 대략 O(K * 자릿수) ≈ O(K) (자릿수는 상수 수준).
      • 단, 최악의 경우 K가 500,001이므로 반복문 기반 풀이도 충분히 동작한다.
    • 공간 복잡도
      • 카운트 배열(10칸)만 추가로 사용 → O(1).
