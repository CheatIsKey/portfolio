    문제 6296 — Crazy Search

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Crazy Search (BOJ 6296)는
    길이가 N인 부분 문자열(substring)을 텍스트에서 모두 만들어 보고, 그중 서로 다른 부분 문자열의 개수를 구하는 문제다.

    • 부분 문자열은 연속된 구간으로, 시작 위치 i에서 N글자를 뽑은 text[i..i+N-1] 형태다.
    • i는 0부터 text.length() - N까지 가능하다.
    • 입력에 NC(등장 가능한 문자 종류 수)가 주어지지만, 이 코드는 실제로는 슬라이딩 구간을 만들어 Set으로 중복 제거하는 방식으로 해결한다.

    전략 개요

    • HashSet을 준비한다. (중복을 자동으로 제거)
    • 텍스트를 왼쪽부터 훑으면서 길이 N짜리 부분 문자열을 하나씩 만든다.
    • 만든 부분 문자열을 Set에 넣는다.
    • 모든 구간을 처리한 뒤 Set의 크기(size())가 곧 서로 다른 부분 문자열의 개수다.

    코드가 하는 일

    • 1) 입력 처리 및 Set 준비 
      int N = Integer.parseInt(st.nextToken());
      int NC = Integer.parseInt(st.nextToken());
      
      Set<String> set = new HashSet<>();
      • N(부분 문자열 길이)과 NC를 입력받는다.
      • 중복 제거를 위해 HashSet을 사용한다.
      • 이 풀이에서는 NC는 직접 활용하지 않는다.
    • 2) 모든 길이 N 부분 문자열을 Set에 추가 
      String text = br.readLine();
      for (int i = 0; i <= text.length() - N; i++) {
          set.add(text.substring(i, i + N));
      }
      • i를 시작점으로 해서 substring(i, i + N)을 만든다.
      • 같은 문자열이 여러 번 등장해도 Set에는 한 번만 저장된다.
      • 따라서 마지막에는 “서로 다른 부분 문자열”만 남는다.
    • 3) Set 크기 출력 
      System.out.println(set.size());
      • 서로 다른 부분 문자열의 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 시작 위치가 총 L - N + 1개(L = text.length())이고, 각 구간에서 substring을 만들어 Set에 넣는다.
      • 따라서 전체는 대략 O((L - N + 1) * N) (부분 문자열 생성 비용 포함).
    • 공간 복잡도
      • 서로 다른 부분 문자열을 Set에 저장하므로, 서로 다른 개수를 K라 할 때 저장 공간은 대략 O(K * N) 수준이 된다.

    문제 6212 — Dream Counting

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Dream Counting (BOJ 6212)는
    두 정수 M부터 N까지의 모든 수를 순서대로 나열했을 때, 각 숫자 0..9가 몇 번 등장하는지를 세는 문제다.

    • 예를 들어 129..137이면 129, 130, 131, ... , 137을 모두 보면서 각 자리의 숫자를 카운트한다.
    • 출력은 0부터 9까지의 등장 횟수를 공백으로 구분해 한 줄에 출력한다.
    • 범위 조건:
      • 1 ≤ M ≤ N ≤ 2,000,000,000
      • N - M ≤ 500,000 (연속 구간 길이가 최대 500,001개)

    전략 개요

    • M부터 N까지 모든 수를 순회한다.
    • 각 수를 10으로 나누면서(% 10, /= 10) 자릿수를 하나씩 꺼내 cnt[digit]를 증가시킨다.
    • 순회가 끝나면 cnt[0..9]를 차례대로 출력한다.

    코드가 하는 일

    • 1) 입력 처리 및 카운트 배열 준비 
      long M = Integer.parseInt(st.nextToken());
      long N = Integer.parseInt(st.nextToken());
      
      long[] cnt = new long[10];
      • M, N을 입력받고, 각 숫자(0~9) 등장 횟수를 담을 배열 cnt를 만든다.
    • 2) M부터 N까지 순회하며 자릿수 카운트 
      for (long i = M; i <= N; i++) {
          long x = i;
      
          while (x > 0) {
              cnt[(int) (x % 10)]++;
              x /= 10;
          }
      }
      • 각 수 i를 x에 복사한 뒤, x가 0이 될 때까지 반복한다.
      • x % 10으로 마지막 자릿수를 꺼내 해당 인덱스의 카운트를 증가시킨다.
      • x /= 10으로 마지막 자릿수를 제거한다.
    • 3) 결과 출력 
      for (long i : cnt) {
          System.out.print(i + " ");
      }
      • cnt[0]부터 cnt[9]까지 공백으로 구분해 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 수의 개수를 K = N - M + 1이라고 하면, 각 수는 자릿수만큼(최대 10자리) 나누기를 수행한다.
      • 따라서 대략 O(K * 자릿수) ≈ O(K) (자릿수는 상수 수준).
      • 단, 최악의 경우 K가 500,001이므로 반복문 기반 풀이도 충분히 동작한다.
    • 공간 복잡도
      • 카운트 배열(10칸)만 추가로 사용 → O(1).
