    문제 6296 — Crazy Search

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Crazy Search (BOJ 6296)는
    길이가 N인 부분 문자열(substring)을 텍스트에서 모두 만들어 보고, 그중 서로 다른 부분 문자열의 개수를 구하는 문제다.

    • 부분 문자열은 연속된 구간으로, 시작 위치 i에서 N글자를 뽑은 text[i..i+N-1] 형태다.
    • i는 0부터 text.length() - N까지 가능하다.
    • 입력에 NC(등장 가능한 문자 종류 수)가 주어지지만, 이 코드는 실제로는 슬라이딩 구간을 만들어 Set으로 중복 제거하는 방식으로 해결한다.

    전략 개요

    • HashSet을 준비한다. (중복을 자동으로 제거)
    • 텍스트를 왼쪽부터 훑으면서 길이 N짜리 부분 문자열을 하나씩 만든다.
    • 만든 부분 문자열을 Set에 넣는다.
    • 모든 구간을 처리한 뒤 Set의 크기(size())가 곧 서로 다른 부분 문자열의 개수다.

    코드가 하는 일

    • 1) 입력 처리 및 Set 준비 
      int N = Integer.parseInt(st.nextToken());
      int NC = Integer.parseInt(st.nextToken());
      
      Set<String> set = new HashSet<>();
      • N(부분 문자열 길이)과 NC를 입력받는다.
      • 중복 제거를 위해 HashSet을 사용한다.
      • 이 풀이에서는 NC는 직접 활용하지 않는다.
    • 2) 모든 길이 N 부분 문자열을 Set에 추가 
      String text = br.readLine();
      for (int i = 0; i <= text.length() - N; i++) {
          set.add(text.substring(i, i + N));
      }
      • i를 시작점으로 해서 substring(i, i + N)을 만든다.
      • 같은 문자열이 여러 번 등장해도 Set에는 한 번만 저장된다.
      • 따라서 마지막에는 “서로 다른 부분 문자열”만 남는다.
    • 3) Set 크기 출력 
      System.out.println(set.size());
      • 서로 다른 부분 문자열의 개수를 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 시작 위치가 총 L - N + 1개(L = text.length())이고, 각 구간에서 substring을 만들어 Set에 넣는다.
      • 따라서 전체는 대략 O((L - N + 1) * N) (부분 문자열 생성 비용 포함).
    • 공간 복잡도
      • 서로 다른 부분 문자열을 Set에 저장하므로, 서로 다른 개수를 K라 할 때 저장 공간은 대략 O(K * N) 수준이 된다.

    문제 6308 — Blowing Fuses

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Blowing Fuses (BOJ 6308)는
    여러 전기기기들을 순서대로 ON/OFF 토글할 때, 어떤 순간이라도 켜진 기기들의 소비 전류 합이 퓨즈 용량 c를 초과하면 퓨즈가 터지는지 확인하는 문제다.

    • 각 테스트케이스는 기기 개수 n, 동작 횟수 m, 퓨즈 용량 c로 주어진다.
    • 각 기기에는 소비 전류 ci가 있고, 동작에서는 해당 기기의 상태가 반전(toggle)된다.
    • 처음에는 모든 기기가 꺼져 있음.
    • 퓨즈가 터지지 않았다면, 동작 과정에서의 최대 소비 전류(max)도 함께 출력해야 한다.

    전략 개요

    • 각 기기의 상태(켜짐/꺼짐)와 소비 전류를 저장해 둔다.
    • 동작을 하나씩 처리하면서
      • 켜져 있던 기기를 끄면 current -= consume
      • 꺼져 있던 기기를 켜면 current += consume
      처럼 현재 소비 전류 합(current)을 갱신한다.
    • 매 동작마다
      • max = Math.max(max, current)로 최대값 갱신
      • current > c면 퓨즈 터짐(blown = true) 표시
    • 테스트케이스 종료 후, 조건에 맞게 문구를 출력한다.

    코드가 하는 일

    • 1) 테스트케이스 루프 및 종료 조건 처리 
      int caseNum = 1;
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
      
          int n = Integer.parseInt(st.nextToken());
          int m = Integer.parseInt(st.nextToken());
          int c = Integer.parseInt(st.nextToken());
      
          if (n == 0 && m == 0 && c == 0) break;
      
          sb.append("Sequence " + caseNum).append('\n');
      • 여러 테스트케이스가 들어오므로 while로 반복 처리한다.
      • n = m = c = 0이면 입력 종료.
      • 각 케이스마다 Sequence k 헤더를 출력한다.
    • 2) 기기 정보 저장 (소비 전류 + 현재 상태) 
      Map<Integer, Device> map = new HashMap<>();
      
      for (int i = 1; i <= n; i++) {
          int consume = Integer.parseInt(br.readLine());
          map.put(i, new Device(consume, false));
      }
      • 기기 번호(1~n)를 키로 해서 소비 전류와 사용 여부를 저장한다.
      • 초기 상태는 모두 OFF이므로 false로 시작한다.
    • 3) 동작 처리: 토글하면서 current / max / blown 갱신 
      int current = 0, max = 0;
      boolean blown = false;
      
      for (int i = 0; i < m; i++) {
          int num = Integer.parseInt(br.readLine());
          int consume = map.get(num).consume;
          boolean isUse = map.get(num).isUse;
      
          if (isUse) current -= consume;
          else current += consume;
      
          map.get(num).isUse = !isUse;
      
          if (current > max) max = current;
          if (current > c) blown = true;
      }
      • 입력으로 들어온 번호의 기기를 토글한다.
      • 켜져 있던 상태면 끄는 것이므로 전류를 빼고, 꺼져 있던 상태면 켜는 것이므로 더한다.
      • 토글 후 상태를 반전시킨다.
      • 매번 최대 소비 전류 max를 갱신하고, 용량 초과면 blown을 표시한다.
    • 4) 케이스별 출력 구성 + 빈 줄 처리 
      if (blown) {
          sb.append("Fuse was blown.").append('\n');
      } else {
          sb.append("Fuse was not blown.").append('\n');
          sb.append("Maximal power consumption was ").append(max).append(" amperes.").append('\n');
      }
      
      sb.append('\n');
      caseNum++;
      • 퓨즈가 터졌다면 해당 문구만 출력한다.
      • 터지지 않았다면 최대 소비 전류를 함께 출력한다.
      • 문제 요구대로 테스트케이스마다 빈 줄을 한 줄 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 테스트케이스에서 기기 입력 O(n), 동작 처리 O(m).
      • 전체는 O(n + m).
    • 공간 복잡도
      • 기기 정보를 저장하는 자료구조가 n개이므로 O(n).
