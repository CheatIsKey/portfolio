    문제 6325 — Definite Values

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Definite Values (BOJ 6325)는
    여러 줄의 대입문 variable1 = variable2가 순서대로 실행될 때, 실행이 끝난 뒤 확정된 값(definite value)을 가지는 변수들을 찾는 문제다.

    • 실행 전에는 a만 확정된 값을 가지고, 나머지는 모두 정의되지 않음(unknown) 상태다.
    • 대입 x = y는
      • y가 확정이면 x도 확정
      • y가 미정이면 x도 미정(기존 값이 무엇이든 “확정”이라고 할 수 없음)
      으로 상태가 결정된다.
    • 모든 줄을 처리한 후, 확정 상태인 변수들을 알파벳 순으로 출력한다.
    • 확정인 변수가 하나도 없으면 none을 출력한다.

    전략 개요

    • 변수는 a~z로 한정이므로 크기 26짜리 boolean 배열 chk로 “확정 여부”를 관리한다.
    • 초기 상태는 chk['a'] = true, 나머지는 false.
    • 각 대입문 x = y에 대해
      • chk[y] == true면 chk[x] = true
      • chk[y] == false면 chk[x] = false
      로 갱신한다. (대입은 기존 상태를 덮어쓴다)
    • 마지막에 true인 변수들을 출력한다.

    코드가 하는 일

    • 1) 테스트케이스(프로그램 파트) 반복 및 초기화 
      int caseNum = 1;
      
      String line;
      while ((line = br.readLine()) != null) {
          int n = Integer.parseInt(line);
      
          if (n == 0) break;
      
          boolean[] chk = new boolean[26];
          chk[0] = true;
      • 각 프로그램 파트는 줄 수 n으로 시작한다.
      • n == 0이면 종료.
      • chk[i]는 변수('a'+i)가 “확정”이면 true.
      • 문제 조건에 따라 시작 시점에 a만 확정이므로 chk[0] = true.
    • 2) 대입문 처리: target이 확정이면 assign도 확정 
      for (int i = 0; i < n; i++) {
          String[] split = br.readLine().split("=");
          String assign = split[0].trim();
          String target = split[1].trim();
      
          int assignIdx = assign.charAt(0) - 'a';
          int targetIdx = target.charAt(0) - 'a';
          if (chk[targetIdx]) chk[assignIdx] = true;
          else chk[assignIdx] = false;
      }
      • 입력은 x = y 형태이므로 =로 분리한 뒤 trim()으로 공백을 제거한다.
      • y(target)가 확정이면 x(assign)도 확정이 된다.
      • y가 미정이면 대입 후 x도 미정이므로 false로 덮어쓴다.
      • 중요 포인트는 “확정 여부”만 추적하면 되고, 실제 값은 필요 없다는 점이다.
    • 3) 헤더 출력 + 확정 변수 출력(알파벳 순) 
      sb.append("Program #" + caseNum++).append('\n');
      
      int cnt = 0;
      for (int i = 0; i < chk.length; i++) {
          if (chk[i]) {
              sb.append((char) (i + 'a')).append(" ");
              cnt++;
          }
      }
      
      if (cnt == 0) sb.append("none");
      sb.append("\n\n");
      • 요구 형식대로 Program #k를 먼저 출력한다.
      • a부터 z까지 순서대로 보며 확정(true)인 변수를 출력하면 자연스럽게 정렬 출력이 된다.
      • 확정 변수가 하나도 없으면 none을 출력한다.
      • 각 테스트케이스마다 빈 줄을 한 줄 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 프로그램 파트에서 대입문 n개를 한 번 처리하므로 O(n).
      • 출력용으로 26개 변수를 한 번 확인하는 작업은 상수 → O(1).
    • 공간 복잡도
      • 확정 여부 배열 26칸만 사용 → O(1).

    문제 6344 — Prime Land

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Blowing Fuses (BOJ 6308)는
    여러 전기기기들을 순서대로 ON/OFF 토글할 때, 어떤 순간이라도 켜진 기기들의 소비 전류 합이 퓨즈 용량 c를 초과하면 퓨즈가 터지는지 확인하는 문제다.

    • 각 테스트케이스는 기기 개수 n, 동작 횟수 m, 퓨즈 용량 c로 주어진다.
    • 각 기기에는 소비 전류 ci가 있고, 동작에서는 해당 기기의 상태가 반전(toggle)된다.
    • 처음에는 모든 기기가 꺼져 있음.
    • 퓨즈가 터지지 않았다면, 동작 과정에서의 최대 소비 전류(max)도 함께 출력해야 한다.

    전략 개요

    • 각 기기의 상태(켜짐/꺼짐)와 소비 전류를 저장해 둔다.
    • 동작을 하나씩 처리하면서
      • 켜져 있던 기기를 끄면 current -= consume
      • 꺼져 있던 기기를 켜면 current += consume
      처럼 현재 소비 전류 합(current)을 갱신한다.
    • 매 동작마다
      • max = Math.max(max, current)로 최대값 갱신
      • current > c면 퓨즈 터짐(blown = true) 표시
    • 테스트케이스 종료 후, 조건에 맞게 문구를 출력한다.

    코드가 하는 일

    • 1) 테스트케이스 루프 및 종료 조건 처리 
      int caseNum = 1;
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
      
          int n = Integer.parseInt(st.nextToken());
          int m = Integer.parseInt(st.nextToken());
          int c = Integer.parseInt(st.nextToken());
      
          if (n == 0 && m == 0 && c == 0) break;
      
          sb.append("Sequence " + caseNum).append('\n');
      • 여러 테스트케이스가 들어오므로 while로 반복 처리한다.
      • n = m = c = 0이면 입력 종료.
      • 각 케이스마다 Sequence k 헤더를 출력한다.
    • 2) 기기 정보 저장 (소비 전류 + 현재 상태) 
      Map<Integer, Device> map = new HashMap<>();
      
      for (int i = 1; i <= n; i++) {
          int consume = Integer.parseInt(br.readLine());
          map.put(i, new Device(consume, false));
      }
      • 기기 번호(1~n)를 키로 해서 소비 전류와 사용 여부를 저장한다.
      • 초기 상태는 모두 OFF이므로 false로 시작한다.
    • 3) 동작 처리: 토글하면서 current / max / blown 갱신 
      int current = 0, max = 0;
      boolean blown = false;
      
      for (int i = 0; i < m; i++) {
          int num = Integer.parseInt(br.readLine());
          int consume = map.get(num).consume;
          boolean isUse = map.get(num).isUse;
      
          if (isUse) current -= consume;
          else current += consume;
      
          map.get(num).isUse = !isUse;
      
          if (current > max) max = current;
          if (current > c) blown = true;
      }
      • 입력으로 들어온 번호의 기기를 토글한다.
      • 켜져 있던 상태면 끄는 것이므로 전류를 빼고, 꺼져 있던 상태면 켜는 것이므로 더한다.
      • 토글 후 상태를 반전시킨다.
      • 매번 최대 소비 전류 max를 갱신하고, 용량 초과면 blown을 표시한다.
    • 4) 케이스별 출력 구성 + 빈 줄 처리 
      if (blown) {
          sb.append("Fuse was blown.").append('\n');
      } else {
          sb.append("Fuse was not blown.").append('\n');
          sb.append("Maximal power consumption was ").append(max).append(" amperes.").append('\n');
      }
      
      sb.append('\n');
      caseNum++;
      • 퓨즈가 터졌다면 해당 문구만 출력한다.
      • 터지지 않았다면 최대 소비 전류를 함께 출력한다.
      • 문제 요구대로 테스트케이스마다 빈 줄을 한 줄 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 테스트케이스에서 기기 입력 O(n), 동작 처리 O(m).
      • 전체는 O(n + m).
    • 공간 복잡도
      • 기기 정보를 저장하는 자료구조가 n개이므로 O(n).
