    문제 6325 — Definite Values

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Definite Values (BOJ 6325)는
    여러 줄의 대입문 variable1 = variable2가 순서대로 실행될 때, 실행이 끝난 뒤 확정된 값(definite value)을 가지는 변수들을 찾는 문제다.

    • 실행 전에는 a만 확정된 값을 가지고, 나머지는 모두 정의되지 않음(unknown) 상태다.
    • 대입 x = y는
      • y가 확정이면 x도 확정
      • y가 미정이면 x도 미정(기존 값이 무엇이든 “확정”이라고 할 수 없음)
      으로 상태가 결정된다.
    • 모든 줄을 처리한 후, 확정 상태인 변수들을 알파벳 순으로 출력한다.
    • 확정인 변수가 하나도 없으면 none을 출력한다.

    전략 개요

    • 변수는 a~z로 한정이므로 크기 26짜리 boolean 배열 chk로 “확정 여부”를 관리한다.
    • 초기 상태는 chk['a'] = true, 나머지는 false.
    • 각 대입문 x = y에 대해
      • chk[y] == true면 chk[x] = true
      • chk[y] == false면 chk[x] = false
      로 갱신한다. (대입은 기존 상태를 덮어쓴다)
    • 마지막에 true인 변수들을 출력한다.

    코드가 하는 일

    • 1) 테스트케이스(프로그램 파트) 반복 및 초기화 
      int caseNum = 1;
      
      String line;
      while ((line = br.readLine()) != null) {
          int n = Integer.parseInt(line);
      
          if (n == 0) break;
      
          boolean[] chk = new boolean[26];
          chk[0] = true;
      • 각 프로그램 파트는 줄 수 n으로 시작한다.
      • n == 0이면 종료.
      • chk[i]는 변수('a'+i)가 “확정”이면 true.
      • 문제 조건에 따라 시작 시점에 a만 확정이므로 chk[0] = true.
    • 2) 대입문 처리: target이 확정이면 assign도 확정 
      for (int i = 0; i < n; i++) {
          String[] split = br.readLine().split("=");
          String assign = split[0].trim();
          String target = split[1].trim();
      
          int assignIdx = assign.charAt(0) - 'a';
          int targetIdx = target.charAt(0) - 'a';
          if (chk[targetIdx]) chk[assignIdx] = true;
          else chk[assignIdx] = false;
      }
      • 입력은 x = y 형태이므로 =로 분리한 뒤 trim()으로 공백을 제거한다.
      • y(target)가 확정이면 x(assign)도 확정이 된다.
      • y가 미정이면 대입 후 x도 미정이므로 false로 덮어쓴다.
      • 중요 포인트는 “확정 여부”만 추적하면 되고, 실제 값은 필요 없다는 점이다.
    • 3) 헤더 출력 + 확정 변수 출력(알파벳 순) 
      sb.append("Program #" + caseNum++).append('\n');
      
      int cnt = 0;
      for (int i = 0; i < chk.length; i++) {
          if (chk[i]) {
              sb.append((char) (i + 'a')).append(" ");
              cnt++;
          }
      }
      
      if (cnt == 0) sb.append("none");
      sb.append("\n\n");
      • 요구 형식대로 Program #k를 먼저 출력한다.
      • a부터 z까지 순서대로 보며 확정(true)인 변수를 출력하면 자연스럽게 정렬 출력이 된다.
      • 확정 변수가 하나도 없으면 none을 출력한다.
      • 각 테스트케이스마다 빈 줄을 한 줄 추가한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 프로그램 파트에서 대입문 n개를 한 번 처리하므로 O(n).
      • 출력용으로 26개 변수를 한 번 확인하는 작업은 상수 → O(1).
    • 공간 복잡도
      • 확정 여부 배열 26칸만 사용 → O(1).

    문제 6344 — Prime Land

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Prime Land (BOJ 6344)는
    한 줄에 “소수 p와 지수 e” 쌍들이 주어질 때, 그것이 나타내는 수 x를 실제로 계산한 뒤 x - 1을 구하고, 다시 소인수분해 형태(내림차순)로 출력하는 문제다.

    • 입력 한 줄은 p1 e1 p2 e2 ... 형태이며, 각 쌍은 p^e를 의미한다.
    • 해당 줄이 표현하는 값은 x = ∏ p^e.
    • 출력은 x - 1을 소인수분해해서 소수 기준 내림차순으로 p e 쌍을 나열한다.
    • 입력의 마지막 줄 0은 종료 신호이며 처리하지 않는다.

    전략 개요

    • 한 줄을 읽고, (p, e) 쌍들을 이용해 x를 직접 계산한다.
    • y = x - 1을 만든다.
    • y를 2부터 sqrt(y)까지 나누어 보면서 소인수분해를 수행한다:
      • 나누어 떨어지면 해당 소수로 더 이상 안 나눠질 때까지 나누며 지수 카운트
      • (소수, 지수)를 리스트에 저장
    • 마지막에 y > 1이 남아있으면 그것도 소인수이므로 (y, 1) 추가
    • 리스트는 작은 소수부터 쌓이므로, 출력은 역순으로 돌려 내림차순으로 찍는다.

    코드가 하는 일

    • 1) 한 줄 입력 처리 + x 계산 
      String line;
      while ((line = br.readLine()) != null) {
          if (line.equals("0")) break;
          StringTokenizer st = new StringTokenizer(line);
      
          int x = 1;
          while (st.hasMoreTokens()) {
              int p = Integer.parseInt(st.nextToken());
              int e = Integer.parseInt(st.nextToken());
      
              for (int i = 0; i < e; i++) {
                  x *= p;
              }
          }
      • 각 줄은 하나의 수 x를 나타낸다.
      • (p, e)를 읽어 p를 e번 곱해 x에 누적한다.
      • 문제 조건상 x ≤ 32767이므로 int로 계산해도 범위를 넘지 않는다.
    • 2) y = x - 1 만들기 
      int y = x - 1;
      • 출력해야 하는 대상은 x가 아니라 x - 1이므로 y로 옮겨서 소인수분해한다.
    • 3) y 소인수분해 (작은 소수부터) 
      List<int[]> list = new ArrayList<>();
      for (int i = 2; i * i <= y; i++) {
          if (y % i == 0) {
              int cnt = 0;
      
              while (y % i == 0) {
                  y /= i;
                  cnt++;
              }
      
              list.add(new int[]{i, cnt});
          }
      }
      
      if (y > 1) list.add(new int[]{y, 1});
      • i를 2부터 증가시키며 나눠 떨어지는지 검사한다.
      • 나눠 떨어지면 while로 지수를 세면서 y를 계속 줄인다.
      • 반복이 끝난 뒤 y가 1보다 크면, 남은 값은 마지막 소인수이므로 추가한다.
    • 4) 내림차순으로 출력 
      for (int i = list.size() - 1; i >= 0; i--) {
          sb.append(list.get(i)[0]).append('\n').append(list.get(i)[1]);
      
          if (i > 0) sb.append(' ');
      }
      sb.append('\n');
      • 리스트는 작은 소수부터 저장되어 있으므로, 역순으로 출력하면 큰 소수부터 출력된다.
      • 각 (p, e)를 “p e” 형태로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • x 계산: 입력된 쌍들의 지수 합을 EO(E).
      • 소인수분해: y에 대해 2..sqrt(y)까지 시도하므로 O(√y).
      • 여기서 y ≤ 32766이라 충분히 빠르다.
    • 공간 복잡도
      • 소인수 목록을 저장하는 리스트만 사용 → O(소인수 개수) (최대 26 미만 수준).
