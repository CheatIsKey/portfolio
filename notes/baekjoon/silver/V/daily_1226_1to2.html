    문제 6361 — Number Base Conversion

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    Number Base Conversion (BOJ 6361)은
    입력 진법 inBase(2~62)로 표현된 수 num을 출력 진법 outBase(2~62)로 변환해 출력하는 문제다. 사용 가능한 숫자(자리)는 0-9, A-Z, a-z로 총 62개다.

    • 각 테스트마다 출력은 3줄 형식이다:
      • inBase와 입력 수를 그대로 출력
      • outBase와 변환된 수 출력
      • 빈 줄 1줄
    • 입력 수의 길이가 매우 길 수 있으므로 일반 정수형으로는 처리 불가 → BigInteger로 처리한다.
    • 핵심은
      • 입력 문자열을 “10진수(BigInteger)”로 해석(파싱)하고
      • 그 값을 다시 원하는 진법 문자열로 변환하는 것
      이다.

    전략 개요

    • num을 inBase 기준으로 해석해 BigInteger 값 val로 만든다.
    • val을 outBase 기준으로 나누기/나머지로 변환한다:
      • val을 outBase로 나눈 나머지가 마지막 자리
      • 몫으로 계속 갱신하며 반복
      • 나머지들을 뒤집으면 최종 진법 문자열
    • 문자 ↔ 값 매핑 규칙:
      • '0'~'9' → 0~9
      • 'A'~'Z' → 10~35
      • 'a'~'z' → 36~61

    코드가 하는 일

    • 1) 테스트케이스 입력 및 출력 형식 구성 
      int n = Integer.parseInt(br.readLine());
      for (int i = 0; i < n; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
      
          int inBase = Integer.parseInt(st.nextToken());
          int outBase = Integer.parseInt(st.nextToken());
          String num = st.nextToken();
      
          sb.append(inBase).append(' ').append(num).append('\n');
      
          BigInteger bigInteger = parseToBigInteger(num, inBase);
          String converted = toBaseString(bigInteger, outBase);
      
          sb.append(outBase).append(' ').append(converted).append('\n');
          sb.append('\n');
      }
      • 각 줄마다 inBase, outBase, num을 읽는다.
      • 첫 줄은 입력 그대로 출력한다.
      • 파싱 → 변환 후, 둘째 줄에 변환 결과를 출력한다.
      • 문제 요구대로 마지막에 빈 줄을 추가한다.
    • 2) inBase 문자열을 BigInteger로 파싱 
      private static BigInteger parseToBigInteger(String num, int inBase) {
          BigInteger b = BigInteger.valueOf(inBase);
          BigInteger val = BigInteger.ZERO;
      
          for (int i = 0; i < num.length(); i++) {
              int d = digitValue(num.charAt(i));
              val = val.multiply(b).add(BigInteger.valueOf(d));
          }
      
          return val;
      }
      • 진법 수를 10진수로 바꾸는 표준 방식이다.
      • 왼쪽부터 한 자리씩 보며 val = val * base + digit로 누적한다.
      • 길이가 길어도 BigInteger라 안전하다.
    • 3) 문자 → 숫자 값 변환 
      private static int digitValue(char c) {
          if ('0' <= c && c <= '9') return c - '0';
          if ('A' <= c && c <= 'Z') return 10 + (c - 'A');
          return 36 + (c - 'a');
      }
      • 0-9, A-Z, a-z를 각각 0~61 범위로 매핑한다.
    • 4) BigInteger를 outBase 문자열로 변환 
      private static String toBaseString(BigInteger bigInteger, int outBase) {
          if (bigInteger.equals(BigInteger.ZERO)) return "0";
      
          BigInteger b = BigInteger.valueOf(outBase);
          StringBuilder sb = new StringBuilder();
      
          while (!bigInteger.equals(BigInteger.ZERO)) {
              BigInteger[] dr = bigInteger.divideAndRemainder(b);
              int rem = dr[1].intValue();
              sb.append(digitChar(rem));
              bigInteger = dr[0];
          }
      
          return sb.reverse().toString();
      }
      • 반복해서 outBase로 나누며 나머지를 뽑는다.
      • 나머지는 “뒤에서부터” 자리이므로 append 후 마지막에 reverse 한다.
      • 값이 0이면 예외로 "0"을 반환한다.
    • 5) 숫자 값 → 문자 변환 
      private static char digitChar(int rem) {
          if (rem < 10) return (char)('0' + rem);
          if (rem < 36) return (char)('A' + (rem - 10));
          return (char)('a' + (rem - 36));
      }
      • 0~61 값을 다시 0-9, A-Z, a-z로 역매핑한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 파싱: 입력 길이를 L이라 하면 O(L)번의 BigInteger 연산(*, +)을 수행한다.
      • 변환: 출력 자리수를 K라 하면 O(K)번의 BigInteger 나눗셈을 수행한다.
      • 따라서 전체는 대략 O(L + K) (BigInteger 연산 비용은 자리수에 따라 증가).
    • 공간 복잡도
      • 변환 문자열과 BigInteger를 저장하므로 대략 O(L + K) 수준.

    문제 6463 — 팩트

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    팩트 (BOJ 6463)는
    주어진 N에 대해 N!의 값에서 0이 아닌 마지막 자리를 구하는 문제다. 입력은 여러 줄로 들어오며, 각 줄마다 결과를 형식에 맞춰 출력한다.

    • N! = 1 × 2 × ... × N
    • 팩토리얼은 매우 커지므로 실제 값을 그대로 계산하면 overflow가 난다.
    • 끝자리의 0은 2 × 5가 만들어내는 10의 개수 때문에 생긴다.
    • 따라서 곱셈을 진행하면서 생기는 0을 즉시 제거하고, 값의 크기도 적당히 잘라서 관리한다.

    전략 개요

    • mul에 1부터 N까지 차례대로 곱한다.
    • 곱할 때마다 mul의 끝자리가 0이면(= mul % 10 == 0) 10으로 나눠 0을 제거한다.
    • mul이 너무 커지는 것을 막기 위해 매번 mul %= 1_000_000_000로 일부 자리만 유지한다.
    • 마지막에 mul % 10이 곧 “0이 아닌 마지막 자리”가 된다.

    코드가 하는 일

    • 1) 여러 줄 입력 처리 
      String line;
      while ((line = br.readLine()) != null) {
          int N = Integer.parseInt(line);
      • 입력이 여러 줄로 주어지므로 EOF까지 반복한다.
      • 각 줄의 N에 대해 결과를 한 줄씩 출력한다.
    • 2) 1부터 N까지 곱하며 0 제거 + 크기 제한 
      long mul = 1;
      
      for (int i = 1; i <= N; i++) {
          mul *= i;
      
          while (mul % 10 == 0) {
              mul /= 10;
          }
      
          mul %= 1_000_000_000L;
      }
      • mul *= i로 팩토리얼을 순차적으로 만든다.
      • 곱셈 결과의 끝자리가 0이면 그 0은 최종 “0이 아닌 마지막 자리” 계산에 방해가 되므로 즉시 제거한다.
      • mul은 계속 커지므로, 뒤쪽 자리 계산에 충분한 범위만 남기고(여기서는 9자리) 잘라서 overflow를 방지한다.
    • 3) 마지막 자리 추출 및 출력 포맷 
      int rem = (int)(mul % 10);
      
      String str = String.format("%5d -> %d", N, rem);
      sb.append(str).append('\n');
      • 0을 제거해가며 유지한 값의 마지막 자리(mul % 10)가 정답이다.
      • 출력은 문제 요구대로 N을 5칸 오른쪽 정렬한 뒤 -> 와 정답을 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 입력에 대해 1..N을 한 번 순회하므로 O(N).
      • 중간의 while (mul % 10 == 0)는 10의 배수일 때만 짧게 수행되며, 전체적으로 큰 부담은 아니다.
    • 공간 복잡도
      • 추가 배열 없이 변수만 사용 → O(1).
