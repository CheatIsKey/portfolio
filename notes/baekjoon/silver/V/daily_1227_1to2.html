    문제 6504 — 킬로미터를 마일로

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    킬로미터를 마일로 (BOJ 6504)는
    정수 x를 피보나치 진법(서로 인접한 피보나치 수를 동시에 쓰지 않는 표현)으로 나타낸 뒤, 그 비트열을 오른쪽으로 1칸 시프트한 표현을 b(y)로 보고, 그 값 y를 출력하는 문제다.

    • 피보나치 수열은
      • F1 = 1, F2 = 2
      • Fn+1 = Fn + Fn-1
    • 피보나치 진법 표현은 “연속된 피보나치 수를 동시에 선택하지 않는(Zeckendorf)” 형태로 유일하게 만들 수 있다.
    • 오른쪽 시프트는 “각 자리의 피보나치 인덱스를 1 낮추는 것”과 같다:
      • Fi를 쓰는 비트가 1이면, 시프트 후에는 Fi-1을 쓰는 효과
      • 맨 마지막(F1) 비트는 버려진다

    전략 개요

    • x < 25000이므로 필요한 피보나치 수를 미리 전부 만들어 둔다.
    • x를 가장 큰 피보나치 수부터 그리디로 빼면(Zeckendorf) 연속 선택이 자동으로 방지된다.
    • 피보나치 진법에서 Fj를 선택했다는 것은, 시프트 후 결과 y에 F(j-1)가 더해진다는 뜻이다.
    • 따라서 비트열을 실제로 만들 필요 없이, “x에서 Fj를 선택할 때마다 y += F(j-1)” 로 바로 누적하면 된다.

    코드가 하는 일

    • 1) 25000 이하 피보나치 수열 미리 생성 
      List<Integer> list = new ArrayList<>();
      list.add(1);
      list.add(2);
      
      while (true) {
          int n = list.size();
          int next = list.get(n - 1) + list.get(n - 2);
      
          if (next > 25000) break;
          list.add(next);
      }
      • F1=1, F2=2부터 시작해 25000을 넘기 전까지 생성한다.
      • 이 리스트의 뒤쪽부터 보면 “가장 큰 피보나치 수부터” 사용할 수 있다.
    • 2) 각 테스트케이스 처리 
      int t = Integer.parseInt(br.readLine());
      for (int i = 0; i < t; i++) {
          int num = Integer.parseInt(br.readLine());
      
          int x = 0;
      
          for (int j = list.size() - 1; j >= 0; j--) {
              int y = list.get(j);
      
              if (y <= num) {
                  num -= y;
      
                  if (j - 1 >= 0) {
                      x += list.get(j - 1);
                  }
              }
          }
      
          sb.append(x).append('\n');
      }
      • num을 입력값(킬로미터)로 두고, 큰 피보나치 수부터 가능한 만큼 선택하며 빼 나간다.
      • y <= num이면 현재 피보나치 수 Fj를 사용한다는 의미로 num -= Fj.
      • 시프트 후 결과는 인덱스가 1 내려가므로, Fj를 골랐다면 F(j-1)을 결과(x)에 더한다.
      • j-1이 음수가 되는 경우(F1)는 “맨 마지막 비트 버림”에 해당하므로 더하지 않는다.
      • 이렇게 누적한 x가 곧 출력해야 할 마일 값이다.
    • 3) 결과 출력 
      System.out.println(sb.toString());
      • 각 테스트케이스 결과를 줄 단위로 출력한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 피보나치 수 개수는 25000까지 매우 적다(수십 개 수준).
      • 각 테스트케이스마다 리스트를 한 번 역순 순회하므로 O(F) (F = 피보나치 항 개수, 상수 수준).
      • 따라서 전체는 O(t * F)로 충분히 빠르다.
    • 공간 복잡도
      • 피보나치 리스트만 저장 → O(F) (상수 수준).

    문제 6550 — 부분 문자열

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    부분 문자열 (BOJ 6550)은
    두 문자열 s와 t가 주어졌을 때, s가 t의 부분 문자열(subsequence)인지 판별하는 문제다. 즉, t에서 몇 글자를 삭제하되 순서를 유지했을 때 s를 만들 수 있으면 Yes, 불가능하면 No를 출력한다.

    • 여기서 말하는 “부분 문자열”은 연속 substring이 아니라, 순서를 유지하는 subsequence이다.
    • 각 테스트케이스는 한 줄에 s와 t가 공백으로 구분되어 들어온다.
    • 문자열 길이는 최대 10만이므로, 매번 전체를 한 번만 훑는 방식이 필요하다.

    전략 개요

    • s의 현재 비교 위치를 나타내는 포인터 idx를 0부터 시작한다.
    • t를 왼쪽부터 끝까지 순회하며,
      • t[i]가 현재 원하는 문자 s[idx]와 같으면 idx++로 다음 문자를 찾는다.
    • 순회가 끝났을 때 idx == s.length()이면 s의 모든 문자를 순서대로 찾은 것이므로 Yes.
    • 끝까지 못 찾았으면 No.

    코드가 하는 일

    • 1) EOF까지 테스트케이스 입력 처리 
      String line;
      while ((line = br.readLine()) != null) {
          StringTokenizer st = new StringTokenizer(line);
          String s1 = st.nextToken();
          String s2 = st.nextToken();
      • 입력이 여러 줄로 주어지므로 EOF까지 반복한다.
      • 각 줄에서 s, t를 순서대로 읽는다. (코드에서는 s1, s2)
    • 2) 투 포인터 방식으로 subsequence 판별 
      int idx = 0;
      for (int i = 0; i < s2.length() && idx < s1.length(); i++) {
          if (s1.charAt(idx) == s2.charAt(i)) {
              idx++;
          }
      }
      • idx는 현재 s에서 찾고 싶은 문자 위치다.
      • t(s2)를 한 번 훑으면서, 같은 문자를 만나면 s의 다음 문자를 찾도록 idx를 증가시킨다.
      • idx가 s 길이에 도달하면 이미 다 찾은 것이므로 더 볼 필요 없다.
    • 3) 결과 출력 
      if (idx == s1.length()) sb.append("Yes").append('\n');
      else sb.append("No").append('\n');
      • s의 모든 문자를 순서대로 찾았으면 Yes, 아니면 No.

    시간 · 공간 복잡도

    • 시간 복잡도
      • t의 길이를 M, s의 길이를 N이라 할 때, 한 번 스캔으로 판별하므로 O(M).
      • 각 테스트케이스마다 선형 시간이라 길이가 10만이어도 충분히 빠르다.
    • 공간 복잡도
      • 포인터 변수만 사용 → O(1).
