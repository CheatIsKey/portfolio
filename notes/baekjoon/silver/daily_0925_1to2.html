    문제 1316 — 그룹 단어 체커

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 어떤 문자가 한 번 끊겼다가(직전 문자와 달라진 뒤) 다시 나타나면 그 단어는 그룹 단어가 아니다. 반대로, 각 문자는 오직 한 번의 연속 구간(덩어리)으로만 나와야 한다.

    상태 관리 — 현재 진행 중인 문자 하나(prev)와, 이미 한 번이라도 등장해 덩어리가 끝난 적 있는지를 기록하는 불리언 배열 visited[26]만 있으면 충분하다. 문자가 바뀌는 순간(prev != c) 그 새 문자 c가 visited에 이미 표시돼 있었다면 규칙 위반이다.

    알고리즘

    • 단어의 각 문자 c를 왼쪽→오른쪽으로 순회한다.
    • 첫 글자는 visited[c] 표시 후 prev = c로 시작한다.
    • 다음 글자에서
      • c == prev → 같은 덩어리이므로 계속 진행.
      • c != prev → 덩어리 전환. visited[c]가 이미 true면 그룹 단어 아님(false 반환). 아니라면 visited[c] = true로 표시하고 prev = c로 갱신.
    • 끝까지 위반이 없으면 그룹 단어(true).

    시간·공간 복잡도 — 각 단어 길이를 L이라 하면 한 번의 선형 순회로 판정(O(L)), 추가 공간은 visited[26]로 O(1)이다.

    정리

    • aaa처럼 같은 문자만 연속 → 그룹 단어(true).
    • aba처럼 중간에 다른 문자를 거쳐 다시 등장 → 그룹 단어 아님(false).
    • 길이 1 단어 → 항상 그룹 단어.
    • 입력 문자는 문제 조건상 소문자 알파벳이므로 c - 'a' 인덱싱이 안전하다.

    문제 1340 — 연도 진행바

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 올해 전체 시간을 분 단위로 환산하고, 1월 1일 00:00부터 현재 시각까지 지난 분을 계산한 뒤 (지난 분 ÷ 전체 분) × 100으로 백분율을 구한다. 윤년이면 전체 분(분모)이 하루(1440분) 더 많아진다는 점만 주의하면 된다.

    입력 파싱 — 입력 형식은 Month DD, YYYY HH:MM (영문 월 이름). StringTokenizer로 [Month][DD,][YYYY][HH:MM] 순서로 읽고, 날짜의 쉼표(,)를 제거한 뒤 정수로 변환한다. 월은 Map<String,Integer>로 영문 → 숫자 매핑한다(예: October → 10). 백준 Java 8 환경에서는 Map.of가 없으므로 HashMap 초기화 블록을 사용한다.

    윤년 판단 — 표준 규칙을 그대로 적용한다. y % 400 == 0 이면 윤년, 아니면 y % 100 != 0 이면서 y % 4 == 0이면 윤년. 윤년이면 연간 일수는 366, 아니면 365.

    달 직전까지 누적 일수 — 평년 기준 누적합 배열 cum = {0,31,59,90,120,151,181,212,243,273,304,334}를 사용해 d = cum[month-1]로 m월 시작 전까지의 일수를 얻는다. 윤년이고 m > 2(3월 이후)이면 2월이 하루 더 있으므로 d += 1로 보정한다.

    진행률 계산

    • 전체 분(분모) : total = daysInYear × 24 × 60
    • 지나간 일수 : daysPassed = daysBeforeMonth(year, month) + (day - 1)
    • 지난 분(분자) : passed = daysPassed × 24 × 60 + hour × 60 + minute
    • 백분율 : percent = passed / total × 100

    엣지 케이스

    • 자정(예: 3월 1일 00:00)은 해당 날짜 하루가 아직 시작 전이므로 day - 1 까지만 일수에 포함된다.
    • 윤년의 3월 이후는 누적 일수에 +1 보정이 반드시 필요하다.
    • 출력 형식은 스페셜 저지로 자유롭지만, 필요하면 System.out.printf("%.9f%n", percent);처럼 고정 소수점으로 출력 가능.

    정리

    • 연간 전체 시간과 현재까지의 누적 시간을 분 단위로 맞춘 다음 비율을 계산한다.
    • 윤년 여부(365/366)와 3월 이후의 +1 보정이 핵심.
    • 파싱은 Month DD, YYYY HH:MM 형식을 그대로 따라 처리한다.
