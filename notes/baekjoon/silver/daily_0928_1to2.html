    문제 1417 — 국회의원 선거

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 다솜(0번 후보)이 다른 모든 후보보다 표를 많이 얻도록 표를 옮기는 최소 횟수를 구하는 문제이다. 방법은 단순히 “현재 최다 득표자를 찾아서 표 1장을 빼고, 다솜에게 1장을 더한다”를 반복하면 된다. 다솜의 표가 모든 후보의 표보다 커지면 종료한다.

    입력 파싱 — 첫 줄은 후보 수 n, 이어서 각 후보의 득표수가 한 줄씩 주어진다. 배열에 저장하되, 다솜의 표는 변수 dasom으로 관리하고, 나머지 후보들은 배열에 그대로 둔다.

    알고리즘

    • dasom ← 첫 번째 후보의 득표수
    • 반복:
      • 다른 후보들 중 최댓값을 찾는다.
      • dasom이 그 값보다 크면 종료.
      • 최댓값을 가진 후보에서 표 1을 빼고, dasom에 1을 더한다.
      • 횟수를 1 증가시킨다.
    • 최종적으로 횟수를 출력한다.

    시간·공간 복잡도 — 후보 수는 최대 50명이므로, 각 반복에서 선형 탐색으로 최댓값을 찾는 O(n) 연산을 수행해도 충분하다. 최악의 경우 총 이동 횟수는 후보 표 수 합에 비례하므로, 전체 복잡도는 O(n × 이동횟수)이며 제한 내에서 문제없이 동작한다. 추가 메모리는 배열 O(n).

    정리

    • 다솜이 항상 1등이 될 때까지 최다 득표자에서 표를 뺏어온다.
    • 후보 수가 1이면 바로 0 출력.
    • 배열 선형 탐색으로 구현해도 충분히 빠르다.

    문제 1418 — K-세준수

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 1부터 N까지의 수 중에서 가장 큰 소인수(Largest Prime Factor, LPF)가 K 이하인 수만 세면 된다. 특별히 1은 소인수가 없으므로 항상 포함한다.

    입력 파싱 — 첫 줄에 N, 둘째 줄에 K가 주어진다. 구현에서는 readLine()으로 각 줄을 읽어 정수로 변환한다.

    알고리즘

    • count = 0
    • x = 1..N에 대해 반복:
      • x == 1이면 LPF를 1로 보고 카운트
      • 그 외에는 largestPrimeFactor(x)로 LPF 계산:
        • tmp = x, largest = 1
        • j = 2부터 j*j ≤ tmp 동안:
          • tmp % j == 0인 동안 tmp /= j, largest = j
        • 루프 종료 후 tmp > 1이면 남은 값이 소수이므로 largest = max(largest, tmp)
      • LPF ≤ K이면 count++
    • 최종적으로 count 출력

    시간·공간 복잡도 — 각 x마다 최대 √x까지만 나눠 보므로 최악의 경우 시간 복잡도는 대략 O(∑√x) ≈ O(N√N). 실제로는 인수로 나눌 때마다 tmp가 빠르게 줄어 더 빠르게 동작한다. 추가 공간은 상수로 O(1).

    정리

    • 핵심은 “가장 큰 소인수 ≤ K” 여부 판단
    • j*j ≤ tmp 조건으로 불필요한 나눗셈을 줄임
    • 마지막에 tmp > 1이면 그 값이 가장 큰 소인수
    • 1은 항상 포함
