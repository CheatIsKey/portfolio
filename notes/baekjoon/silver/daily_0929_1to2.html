    문제 1427 — 소트인사이드

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 입력받은 수의 각 자릿수를 분리해서 배열에 담고, 이를 내림차순으로 정렬하여 출력하는 문제이다. 구현은 단순히 (1) 입력을 정수로 변환한 뒤 10으로 나누어가며 각 자릿수를 배열에 저장하고, (2) 배열을 오름차순으로 정렬한 후, (3) 끝에서부터 출력하는 방식으로 내림차순을 구현한다.

    입력 파싱 — 입력은 문자열로 받은 뒤 Integer.parseInt()로 정수로 바꾸었다. 이후 while 루프에서 num % 10을 이용해 각 자릿수를 배열에 채워 넣는다. 마지막에는 Arrays.sort()로 정렬하고, 배열의 뒤쪽부터 출력하면 원하는 결과를 얻는다.

    알고리즘

    • 문자열 입력 → 정수 변환
    • while 루프로 각 자릿수를 num % 10으로 추출하여 배열에 저장
    • Arrays.sort()로 오름차순 정렬
    • 배열의 뒤쪽부터 출력하여 내림차순 결과 생성

    시간·공간 복잡도 — 배열 크기는 최대 자릿수(입력 길이)이고, 정렬은 O(d log d) (d = 자릿수)로 충분히 빠르다. 추가 메모리는 자릿수 배열 O(d).

    정리

    • 정수를 자릿수 단위로 쪼개 배열에 담고 정렬하는 방식으로 해결
    • 내림차순 정렬은 단순히 오름차순 정렬 뒤에 역순으로 출력
    • 자릿수를 추출하는 전통적인 % 10 방식으로 구현

    추가 학습 — 같은 문제를 Java Stream API로도 풀 수 있는데, input.chars()로 각 문자를 숫자로 변환한 뒤 .sorted(Comparator.reverseOrder())로 내림차순 정렬하면 반복문 없이 간결하게 작성할 수 있다. 이 방식은 가독성이 좋고 “문자열을 바로 처리한다”는 점에서 안전하다.

    문제 1436 — 영화감독 숌

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 문제에서 말하는 “종말의 숫자”는 정수 안에 연속된 ‘666’이 포함된 수이다. 따라서 666, 1666, 2666, …, 6660, 6661 같은 수들이 모두 해당된다. N번째 종말의 수를 구하려면 666부터 시작해 하나씩 검사하면서 ‘666’을 포함하는지 확인하고, 그때마다 카운트를 증가시켜 N번째가 되는 순간의 수를 출력하면 된다.

    입력 파싱 — 정수 N을 입력받아, 몇 번째 종말의 숫자를 찾을지를 정한다. 별도의 배열이나 자료구조는 필요하지 않고, 단일 변수로 카운트를 관리한다.

    알고리즘

    • num = 666부터 시작한다.
    • 반복:
      • String.valueOf(num).contains("666")로 종말의 숫자인지 확인한다.
      • 종말의 숫자면 count++를 증가시킨다.
      • count == N이 되면 반복을 종료한다.
      • 아니면 num++ 해서 다음 수를 검사한다.
    • 마지막으로 num을 출력한다.

    시간·공간 복잡도 — 각 숫자마다 문자열 변환 후 포함 여부를 확인하므로 검사 비용은 자릿수 O(d). 최대 N=10,000까지 반복해도 충분히 빠르며, 별도의 추가 메모리는 거의 사용하지 않는다.

    정리

    • 가장 작은 종말의 숫자인 666부터 검사한다.
    • ‘666’ 포함 여부는 contains() 메서드로 간단히 구현 가능하다.
    • 제한 범위가 작아 단순 브루트포스로 충분히 해결된다.

    추가 학습 — 문자열 변환 대신 숫자 연산으로도 같은 판별을 할 수 있다. 예를 들어 x % 10을 이용해 끝자리부터 차례로 확인하면서 연속된 6이 세 번 나오면 종말의 숫자로 판단하는 방식이다. 이 방법은 문자열 변환 없이 구현할 수 있어 조금 더 효율적이지만, 가독성 측면에서는 문자열 방식이 단순하고 직관적이다.
