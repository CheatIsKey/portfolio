    문제 1543 — 문서 검색

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 문자열 text 안에서 부분 문자열 str이 등장하는 횟수를 겹치지 않게 세는 문제다. 일반적인 indexOf 탐색을 이용하되, 한 번 발견된 위치 이후에는 str.length()만큼 인덱스를 건너뛰면 겹침 없이 세어낼 수 있다.

    입력 파싱 — 첫 번째 줄은 문서 전체 문자열(text), 두 번째 줄은 찾을 단어(str)이다. 공백, 특수문자 포함 문자열이므로 readLine()으로 그대로 받아야 하며, trim() 등으로 불필요한 공백을 제거하지 않는다.

    알고리즘

    • count = 0, idx = 0으로 초기화한다.
    • text.indexOf(str, idx)로 현재 인덱스 idx 이후 첫 등장 위치를 찾는다.
      • 결과가 -1이면 더 이상 등장하지 않으므로 루프 종료.
      • 발견되면 count++ 후 idx += str.length()로 건너뛴다(겹침 방지).
    • 루프가 끝나면 count를 출력한다.

    예시

    • text = "abababa", str = "aba" → 인덱스 0, 4에서 등장 → 정답 2 (겹치는 인덱스 2~4는 무시)
    • text = "aaaaa", str = "aa" → 인덱스 0, 2, 4에서 겹치지 않게 세면 정답 2

    시간·공간 복잡도 — indexOf는 내부적으로 O(N) 탐색을 수행하며, 각 호출 후 인덱스가 str.length()만큼 증가하므로 전체 복잡도는 O(N)에 가깝다. 문자열 탐색 외 추가 메모리는 정수 몇 개뿐이므로 O(1).

    정리

    • indexOf(찾을문자열, 시작인덱스)를 반복 호출해 간결하게 구현 가능.
    • 발견 후 인덱스를 + str.length()만큼 이동해 겹침을 피한다.
    • 시간·공간 효율 모두 우수하며, 문자열 길이가 수천~수만이어도 충분히 빠르다.

    문제 1652 — 누울 자리를 찾아라

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 방 구조를 나타내는 N×N 격자에서 연속된 빈칸('.')이 2칸 이상 이어지는 구간을 가로와 세로 각각 따로 세면 된다. 벽('X')을 기준으로 구간이 끊기며, 한 구간마다 1개의 누울 자리로 계산한다.

    입력 파싱 — 첫 줄에 방 크기 N이 주어지고, 이후 N줄에 걸쳐 '.'와 'X'로 이루어진 문자열이 주어진다. 각 줄을 문자 배열로 처리하여 2차원 배열에 저장한다.

    알고리즘

    • 가로 방향 탐색
      • 각 행(i)을 순회하며, sleep 변수를 통해 연속된 '.' 개수를 센다.
      • 'X'를 만나면 지금까지의 sleep 값이 2 이상인지 확인 후 row++ 증가, 이후 sleep=0으로 초기화한다.
      • 행 끝에 도달했을 때도 sleep>=2라면 마지막 구간을 반영해 row++ 한다.
    • 세로 방향 탐색
      • 각 열(j)을 순회하며, 행 단위로 sleep을 세는 동일한 방식으로 column++을 계산한다.
    • 최종적으로 System.out.println(row + " " + column)으로 출력한다.

    예시

          ....X
      ..XX.
      .....
      X....
      XX..X
    • 가로 기준: 1행(1), 2행(1), 3행(1), 4행(0), 5행(1) → 총 4
    • 세로 기준: 각 열 세면 총 3
    • 결과: 4 3

    시간·공간 복잡도 — 모든 칸을 한 번씩 순회하므로 O(N²), 배열 크기도 N×N으로 동일하게 O(N²) 공간을 사용한다.

    정리

    • 연속된 '.' 구간의 길이가 2 이상일 때만 카운트한다.
    • 가로/세로를 완전히 독립적으로 처리한다.
    • 벽을 만날 때마다 구간이 끊기므로, 루프 종료 후 잔여 구간 확인을 반드시 수행해야 한다.
    • 복잡도는 O(N²)로 충분히 빠르며, 단순 구현 문제의 전형적인 패턴이다.
