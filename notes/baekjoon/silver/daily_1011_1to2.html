    문제 2303 — 숫자 게임

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 이 문제는 각 참가자가 가진 다섯 장의 카드 중 세 장을 선택해 만든 합의 일의 자리가 최대가 되도록 하는 간단한 완전 탐색 문제이다. 모든 경우의 수(5C3=10가지)를 직접 계산해도 되지만, 이를 효율적으로 처리하기 위해 전체 합에서 두 장을 제외하는 방식으로 접근할 수 있다.

    아이디어 전개 — 한 사람이 가진 다섯 장의 합을 sum이라 할 때, 세 장을 선택하는 것은 곧 sum에서 두 장을 버리는 것과 같다. 즉, 가능한 모든 (i, j) 쌍(총 10가지)에 대해 three = sum - card[i] - card[j]를 계산하고, 그 결과의 일의 자리 (three % 10) 중 가장 큰 값을 해당 사람의 점수로 선택한다.

    이때 자바(Java)의 % 연산은 피연산자가 음수일 때 음수가 반환될 수 있으므로, 항상 0~9 범위를 유지하기 위해 ((three % 10) + 10) % 10 형태로 보정한다.

    입력 파싱 — 첫 줄에 사람의 수 N이 주어지고, 그다음 줄부터 각 사람의 카드 다섯 장이 주어진다. 각 줄을 빠르게 처리하기 위해 BufferedReader와 StringTokenizer를 사용한다. 입력은 다음과 같이 2차원 배열 people[N][5]에 저장된다.

    for (int i = 0; i < N; i++) {
          StringTokenizer st = new StringTokenizer(br.readLine());
          for (int j = 0; j < 5; j++) {
              people[i][j] = Integer.parseInt(st.nextToken());
          }
      }

    알고리즘

    • 1️⃣ 각 사람의 카드 합 sum을 미리 계산한다.
    • 2️⃣ 두 장을 버리는 모든 조합(5C2=10쌍)에 대해: 
      three = sum - people[i][x] - people[i][y];
      score = ((three % 10) + 10) % 10;
      best = Math.max(best, score);
    • 3️⃣ 각 사람의 최고 점수 best를 구한 뒤, 현재까지의 최고 점수 maxScore와 비교한다.
    • 4️⃣ 점수가 같을 경우 번호가 큰 사람이 승자이므로, 비교 조건은 if (best >= maxScore)로 설정한다. 
      if (best >= maxScore) {
          maxScore = best;
          win = i + 1;
      }

    예시

    입력:
      3
      1 2 3 4 5
      2 3 3 2 10
      1 1 1 1 1
      
      출력:
      2

    해설: 1번의 최대 일의 자리는 9, 2번은 (2+3+10=15 → 5)보다 큰 조합 (3+3+10=16 → 6), (3+2+10=15 → 5), (3+2+3=8 → 8) 중 최댓값 8, 3번은 항상 합이 3이므로 일의 자리 3. 결국 최고 점수 9를 가진 1번과 8을 가진 2번 중 더 큰 점수는 9지만, 만약 동점이었다면 번호가 큰 사람이 승자가 된다.

    시간·공간 복잡도 — 각 사람당 10가지 조합만 확인하므로 O(N×10) = O(N)이며, 추가 메모리는 O(1) 수준으로 매우 효율적이다.

    정리

    • 모든 조합(5C3=10)을 직접 탐색해도 충분히 빠르다.
    • 5C3 대신 5C2를 이용해 계산량을 단순화할 수 있다.
    • 일의 자리 계산 시 ((x % 10) + 10) % 10 보정이 필요하다.
    • 동점일 경우 번호가 큰 참가자가 승리한다.
    • 전형적인 완전 탐색(Brute Force) + 조건 비교 문제.

    문제 2508 — 사탕 박사 고창영

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 — 이 문제는 격자 형태의 상자 안에서 ‘사탕(o)’이 포장된 개수를 찾는 문제이다. 사탕은 막대기 모양의 포장으로 둘러싸여 있으며, 가로 방향으로는 > o <, 세로 방향으로는 v o ^ 형태로 주어진다. 즉, 'o'를 중심으로 양옆이 '>'와 '<'이거나, 위아래가 'v'와 '^'인 경우를 찾아 세면 된다.

    입력 파싱 — 첫 줄에 테스트 케이스 수 t가 주어지고, 각 테스트 케이스는 다음과 같은 형식으로 구성된다.

    t
      (빈 줄)
      r c
      (이후 r줄에 걸쳐 c개의 문자)
      (빈 줄)

    각 테스트 케이스 사이에는 공백 줄이 존재하므로 br.readLine()으로 한 줄을 건너뛰는 과정이 필요하다. 상자의 문자 배열은 char[][] box에 저장하며, 문자 그대로 비교하기 때문에 int[][]보다 char[][]가 적합하다.

    StringTokenizer st = new StringTokenizer(br.readLine());
      int r = Integer.parseInt(st.nextToken());
      int c = Integer.parseInt(st.nextToken());
      
      char[][] box = new char[r][c];
      for (int j = 0; j < r; j++) {
          box[j] = br.readLine().toCharArray();
      }

    알고리즘

    • 1️⃣ 모든 칸을 순회하며 'o'인 지점을 찾는다.
    • 2️⃣ 해당 위치를 중심으로
      • 세로 방향: 위가 'v'이고 아래가 '^'이면 사탕 1개.
      • 가로 방향: 왼쪽이 '>'이고 오른쪽이 '<'이면 사탕 1개.
    • 3️⃣ 인덱스 범위를 벗어나지 않도록 if 조건으로 상하좌우 경계를 검사한다.
    if (box[x][y] == 'o') {
          if (x > 0 && x < r - 1 && box[x - 1][y] == 'v' && box[x + 1][y] == '^') cnt++;
          if (y > 0 && y < c - 1 && box[x][y - 1] == '>' && box[x][y + 1] == '<') cnt++;
      }

    예시

    입력:
      1
      
      5 5
      .....
      .v.^.
      .>o<.
      .....
      .....
      
      출력:
      2

    해설: 위쪽에 세로로 v o ^ 한 개, 가운데 줄에 가로로 > o < 한 개가 존재하므로 총 2개의 사탕이 포장되어 있다.

    시간·공간 복잡도 — 격자의 모든 칸을 한 번씩만 확인하므로 시간 복잡도는 O(r×c), 공간 복잡도는 O(r×c)이다.

    정리

    • 사탕 패턴은 v o ^ 또는 > o < 두 가지뿐이다.
    • 문자 비교는 아스키 코드 대신 리터럴 'o', 'v', '^', '>', '<'를 그대로 사용한다.
    • 테스트 케이스 사이 공백 줄(br.readLine()) 처리에 유의한다.
    • 전형적인 2차원 완전 탐색 문제로, 각 셀의 주변 문자를 조건으로 검사하면 된다.
