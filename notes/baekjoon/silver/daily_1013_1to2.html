    문제 2622 — 삼각형만들기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    양의 정수 길이 a ≤ b ≤ c 세 변으로 삼각형을 만들 때, 이 문제가 요구하는 것은 세 변의 합이 정확히 N이면서 동시에 삼각형 부등식 a + b > c를 만족하는 조합의 개수다. 직각 여부와는 무관하므로 피타고라스 정리는 쓰지 않는다.

    제약 정리 —
    세 변의 합 제약 a + b + c = N을 이용해 c = N - a - b로 바꾸면, 세 변을 모두 돌 필요 없이 a, b만 순회해도 된다. 또한 정렬 가정 a ≤ b ≤ c를 넣어 중복을 제거한다.

    코드 핵심 (변수 매핑) —
    아래 구현에서 i, j, k는 각각 a, b, c에 대응한다.

    • k = N - i - j  →  합 제약 a + b + c = N 적용
    • if (k < j) break;  →  b ≤ c가 깨지면 이후 j 증가 시 k는 더 작아지므로 조기 종료
    • if (k ≤ 0) continue;  →  모든 변은 양수
    • if (i + j > k) cnt++;  →  삼각형 부등식 검증(a + b > c)

    동작 과정 —
    정렬 제약 a ≤ b ≤ c 하에서는 삼각형 부등식 세 개 중 a + b > c만 확인하면 충분하다. 또한 c를 명시적으로 돌지 않고 c = N - a - b로 두면, 탐색 공간이 O(N³)에서 O(N²)로 줄어든다. 중복(순열) 역시 a ≤ b ≤ c로 제거된다.

    엣지 케이스

    • N < 3이거나 k ≤ 0이면 삼각형 불가 → 자동으로 카운트되지 않음
    • k < j가 되는 순간 더 큰 j에서는 k가 더 작아지므로 내부 루프 break로 가지치기

    시간·공간 복잡도 —
    바깥 i와 안쪽 j의 이중 루프이므로 O(N²). 추가 공간은 상수 O(1).

    정리

    • 필요한 조건은 합 제약 a + b + c = N과 삼각형 부등식 a + b > c뿐
    • c를 식으로 대체하고 a ≤ b ≤ c를 활용해 중복 제거 + 조기 종료로 효율화
    • 복잡도는 O(N²)로 충분히 빠르며, 구현이 간단하고 검증이 쉽다

    문제 2628 — 종이자르기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    절단선들을 각각 가로 자름(0)과 세로 자름(1)으로 분리해 위치를 수집하고, 0과 끝점(H, W)을 함께 넣어 정렬한다. 인접한 절단선 사이의 최대 세로 간격(높이)과 최대 가로 간격(너비)을 구해 최대 넓이를 maxHeight × maxWidth로 계산한다.

    코드 핵심(변수 매핑) —
    입력 첫 줄 W H를 읽고, 이후 N개의 절단 정보를 처리한다.
    - type == 0: 가로로 자름 → 세로 길이 H를 분할하므로 row에 위치 추가
    - type == 1: 세로로 자름 → 가로 길이 W를 분할하므로 column에 위치 추가
    - 양 끝점을 꼭 포함: row = {0, H}, column = {0, W}
    - 정렬 후 인접 차의 최댓값이 각각 maxH, maxW

    • Collections.sort(row) / Collections.sort(column)
    • maxH = max(row[i] - row[i-1]) (1 ≤ i < row.size())
    • maxW = max(column[i] - column[i-1]) (1 ≤ i < column.size())
    • 정답: maxH * maxW

    동작 과정 —
    절단선들은 종이를 여러 구간으로 나누며, 각 방향에서 가장 넓은 인접 구간이 해당 방향의 최대 길이가 된다. 면적은 가로×세로이므로, 두 방향의 최대 간격 곱이 가장 큰 직사각형 조각의 넓이다. 끝점을 포함하지 않으면 가장자리 조각이 누락되므로 반드시 0과 W/H를 넣어야 한다.

    입력 형식

    W H
      N
      type₁ pos₁
      type₂ pos₂
      ...
      typeₙ posₙ

    type은 0(가로 절단), 1(세로 절단). pos는 절단선의 위치.

    엣지 케이스

    • 절단선이 하나도 없는 경우: maxH = H, maxW = W → 면적 W×H
    • 가장자리(0, W/H)와 동일한 위치로 자르는 입력은 주어지지 않지만, 중복이 들어와도 정렬+차이 계산으로 자연스럽게 0 폭/높이가 되어 무해하다.

    시간·공간 복잡도 —
    정렬이 지배적이므로 O(N log N), 간격 계산은 O(N). 추가 공간은 절단선 리스트 O(N).

    정리

    • 절단선 분리(0/1) → 끝점 포함 → 정렬 → 인접 차 최대 계산
    • 정답은 maxHeight × maxWidth
    • 입력 첫 줄은 반드시 StringTokenizer로 W, H를 같은 줄에서 파싱
