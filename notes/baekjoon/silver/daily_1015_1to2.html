    문제 2635 — 수 이어가기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    첫 번째 수 N과 두 번째 수 k를 정하면 이후 수열이 완전히 결정된다. 규칙은 aₙ₊₂ = aₙ - aₙ₊₁이며, 음수가 되는 순간 수열이 끝난다. 따라서 가능한 모든 k(1 ≤ k ≤ N)에 대해 시뮬레이션을 수행하고, 가장 긴 수열을 선택하면 된다.

    제약 정리 —
    모든 항은 음수가 되기 전까지 생성되고, 두 번째 수 k는 1부터 N까지의 정수이다. 각 시도에서 첫 번째 수 a와 두 번째 수 b로부터 다음 항 c = a - b를 구하며, c < 0이면 종료한다. 즉, 한 번의 k 시도는 완전히 독립적인 수열 생성 실험이다.

    코드 핵심 (변수 매핑) —
    a, b, c는 각각 현재 항, 바로 다음 항, 다음에 계산될 항을 의미한다.
    tmp 리스트는 현재 k로 생성한 수열을 담고, list는 그중 가장 긴 수열을 저장한다.

    • tmp.add(a), tmp.add(b) → 초기 두 항 저장
    • c = a - b → 다음 항 계산
    • if (c < 0) break; → 음수가 되면 수열 종료
    • a = b; b = c; → 다음 단계로 이동
    • if (tmp.size() > list.size()) list = tmp; → 최장 수열 갱신

    동작 과정 —
    각 k에 대해 수열을 만들어 보고, 그 길이가 가장 긴 경우를 list에 저장한다. 마지막에 수열의 길이와 항들을 공백으로 구분해 출력한다. StringBuilder를 사용해 문자열 연결 비용을 줄였다.

    엣지 케이스

    • N = 1일 때 두 번째 수는 항상 1이므로 수열은 [1, 1, 0]
    • c가 0이 되는 시점에서 다음 항은 음수가 되므로 자연스럽게 종료
    • N이 커도 각 수열 길이가 제한적이므로 메모리 초과 우려 없음

    시간·공간 복잡도 —
    두 번째 수 k를 1부터 N까지 완전 탐색하므로 O(N)번 반복. 각 시뮬레이션은 수열 길이만큼 O(L)이지만, L은 O(N)보다 작다. 따라서 전체 복잡도는 O(N²) 수준이며, 리스트 저장 공간은 상수(O(1))에 가깝다.

    정리

    • 두 번째 수 k를 1~N까지 전부 시도하여 최장 수열 탐색
    • 규칙 aₙ₊₂ = aₙ - aₙ₊₁으로 수열 생성
    • 가장 긴 수열의 길이와 항들을 출력 (StringBuilder 사용)
    • 시간 복잡도 O(N²), 공간 복잡도 O(1) (리스트 제외)

    문제 2628 — 종이자르기

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    좌표 평면을 1×1 격자로 보고, 입력으로 주어지는 네 개의 직사각형을 각 칸 단위로 칠해(mark) 합집합의 넓이를 카운트한다. 좌하단은 포함, 우상단은 미포함 범위 [x1, x2) × [y1, y2)로 칠하면 겹치는 부분을 중복 없이 정확히 처리할 수 있다.

    제약 정리 —
    좌표 범위가 0~100이므로 101×101 배열로 충분하다. 각 직사각형은 x1 < x2, y1 < y2를 만족하며, 칠할 때는 x = x1 .. x2-1, y = y1 .. y2-1만 포함한다 (우상단 경계 미포함).

    코드 핵심 (변수 매핑) —
    paper[x][y]는 (x, y) 칸이 직사각형들 중 하나에 포함되면 1로 표시한다.
    입력은 네 줄을 각각 StringTokenizer로 읽고, 각 줄마다 직사각형 좌표 (x1, y1, x2, y2)를 받아 칠한다.

    • for (int x = x1; x < x2; x++), for (int y = y1; y < y2; y++) → 우상단 경계 미포함
    • paper[x][y] = 1; → 포함 표시(겹침도 1로 그대로 유지)
    • 모든 입력 처리 후, 격자 전체를 순회하며 paper[x][y] == 1인 칸을 카운트

    동작 과정 —
    네 개의 직사각형을 순서대로 칠한다. 이미 칠해진 칸을 다시 칠해도 값은 1이므로 중복 면적이 추가로 더해지지 않는다. 마지막에 0~100 범위 전체를 훑으며 1로 표시된 칸 수를 합하면 합집합의 넓이가 된다.

    엣지 케이스

    • 직사각형들이 서로 겹치거나 맞닿아도 정확히 처리됨(중복 카운트 없음)
    • 좌표가 0 또는 100인 경계에 놓여도, 배열 크기를 101로 잡아 안전
    • 넓이가 0인 입력(x1 == x2 또는 y1 == y2)은 자연스럽게 아무 칸도 칠해지지 않음

    시간·공간 복잡도 —
    칠하는 루프는 각 직사각형 범위 내에서만 수행되고, 최종 카운트는 101×101 전수 검사이므로 전체는 상수에 가까운 O(10⁴). 공간은 paper 배열 O(10⁴).

    정리

    • 격자 마킹으로 합집합 면적을 직관적으로 계산
    • [x1, x2), [y1, y2)로 우상단 경계 미포함을 지켜 중복·경계 오류 방지
    • 마지막에 1로 칠해진 칸 수를 합산해 정답 출력
