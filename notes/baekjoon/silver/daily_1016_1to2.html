    문제 2822 — 점수 계산

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    총 8개의 점수 중 상위 5개를 선택해 합을 구하고, 선택된 문제의 원래 번호(1-based)를 오름차순으로 출력한다. 이를 위해 “점수와 원래 인덱스”를 함께 저장한 뒤, 점수 기준 내림차순 정렬 → 상위 5개 선택 → 인덱스만 따로 오름차순 정렬 → 두 줄로 출력한다.

    제약 정리 —
    입력은 줄마다 하나씩 총 8개의 정수 점수가 주어진다(각 줄 1개).
    출력은 (1) 첫 줄에 선택한 상위 5개 점수의 합, (2) 둘째 줄에 선택된 문제 번호 5개(1부터 시작)를 오름차순으로 공백 구분하여 출력한다.

    코드 핵심 (변수 매핑) —
    arr[i][0]: 원래 문제 번호(1-based)  |  arr[i][1]: 점수
    sum: 상위 5개 점수의 합  |  picked: 선택된 문제 번호 목록

    • int[][] arr = new int[8][2]; → (문제 번호, 점수) 쌍 저장
    • Arrays.sort(..., (a,b) -> Integer.compare(b[1], a[1])) → 점수 기준 내림차순 정렬
    • 상위 5개 순회하며 sum 누적, picked에 문제 번호 추가
    • Collections.sort(picked); → 선택된 번호 오름차순 정렬

    동작 과정 —
    입력 단계에서 각 점수에 원래 번호(1..8)를 붙여 저장한다. 이후 점수 기준으로 내림차순 정렬하고, 앞에서부터 5개를 뽑아 합을 더하고 번호만 모은다. 마지막으로 그 번호들을 오름차순 정렬해 형식에 맞춰 출력한다.

    // 1) (문제 번호, 점수) 쌍으로 입력 저장
      for (int i = 0; i < 8; i++) {
        arr[i][0] = i + 1;         // 원래 번호(1-based)
        arr[i][1] = score(i);      // i번째 줄 점수
      }
      
      // 2) 점수 기준 내림차순 정렬
      Arrays.sort(arr, (a, b) -> Integer.compare(b[1], a[1]));
      
      // 3) 상위 5개 선택: 합과 번호 수집
      int sum = 0;
      List<Integer> picked = new ArrayList<>();
      for (int i = 0; i < 5; i++) {
        sum += arr[i][1];
        picked.add(arr[i][0]);
      }
      
      // 4) 선택된 번호 오름차순 정렬 후 출력
      Collections.sort(picked);
      // 첫 줄: 합, 둘째 줄: 번호 5개 오름차순

    엣지 케이스

    • 동점(같은 점수)이 여러 개 있어도 상위 5개만 고르면 된다. (정렬 후 앞에서 5개 선택)
    • 출력 형식: 첫 줄 합, 둘째 줄 번호 5개를 공백으로 구분(번호는 오름차순)

    시간·공간 복잡도 —
    정렬이 지배적이며 원소가 8개이므로 시간 복잡도는 O(8 log 8) (일반화하면 O(n log n)).
    추가 공간은 배열 arr(상수 크기)와 선택 목록 picked(최대 5개)로 O(n) (n=8).

    정리

    • 점수와 원래 번호를 함께 보관 → 점수 내림차순 정렬 → 상위 5개 채택
    • 합은 즉시 누적, 번호는 따로 모아 오름차순 정렬 후 출력
    • 출력: 첫 줄 합, 둘째 줄 번호 5개(오름차순, 공백 구분)

    문제 2828 — 사과 담기 게임

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    길이 N인 1차원 칸 위에서, 길이 M인 바구니가 좌우로만 움직이며 낙하하는 사과를 받아야 한다. 사과의 위치가 현재 바구니 범위([left, right]) 안에 있으면 이동할 필요가 없고, 밖에 있다면 해당 방향으로 최소한만 이동해 사과를 포함시키면 된다. 모든 사과를 받는 데 필요한 바구니 이동 거리의 총합을 구한다.

    제약 정리 —

    • N: 칸의 개수 (1 ≤ N ≤ 10⁴)
    • M: 바구니의 길이 (1 ≤ M ≤ N)
    • J: 떨어지는 사과의 개수
    첫 줄에 N M, 둘째 줄에 J, 이후 J줄에 각 사과의 위치(1-based)가 주어진다.
    바구니의 초기 위치는 [1, M]이며, 한 번에 한 칸씩 좌우로 이동할 수 있다. 출력은 사과를 모두 받을 때까지 이동한 총 거리이다.

    코드 핵심 (변수 매핑) —
    left: 바구니의 왼쪽 끝 위치  |  right: 바구니의 오른쪽 끝 위치
    apple: 현재 떨어지는 사과의 위치  |  move: 누적 이동 거리

    • if (apple < left) → 사과가 바구니 왼쪽 바깥 → 왼쪽으로 이동
    • else if (apple > right) → 사과가 바구니 오른쪽 바깥 → 오른쪽으로 이동
    • 바구니가 사과를 포함할 때까지 이동 거리만큼 left·right를 함께 갱신
    • 사과가 범위 안이면 이동 없음

    동작 과정 —
    1️⃣ 바구니의 현재 위치 범위를 [left, right]로 관리한다.
    2️⃣ 각 사과의 위치 apple을 입력받아, 만약 사과가 현재 범위 왼쪽에 있으면 left - apple만큼 왼쪽으로 이동하고, 오른쪽에 있으면 apple - right만큼 오른쪽으로 이동한다.
    3️⃣ 이동 시마다 move에 더하고, 바구니 위치를 갱신한다.
    4️⃣ 모든 사과를 처리한 뒤, 이동 거리의 총합을 출력한다.

    // 초기 바구니 위치 [1, M]
      int left = 1, right = M, move = 0;
      
      for (int i = 0; i < J; i++) {
        int apple = Integer.parseInt(br.readLine());
      
        if (apple < left) {             // 왼쪽 바깥
          int tmp = left - apple;
          move += tmp;
          left -= tmp;
          right -= tmp;
        } else if (apple > right) {     // 오른쪽 바깥
          int tmp = apple - right;
          move += tmp;
          left += tmp;
          right += tmp;
        }
      }
      // 결과 출력: 총 이동 거리
      System.out.println(move);

    엣지 케이스

    • 모든 사과가 처음부터 바구니 범위 안에 있다면 이동 거리는 0.
    • 사과가 바구니의 양 끝 경계선에 정확히 떨어져도 이동 필요 없음.
    • 모든 입력은 1-based 좌표 기준으로 주어짐에 유의.

    시간·공간 복잡도 —
    모든 사과에 대해 한 번씩만 처리하므로 시간 복잡도는 O(J).
    추가 공간은 상수 크기의 변수 몇 개뿐이므로 O(1).

    정리

    • 현재 바구니 범위를 [left, right]로 관리
    • 사과가 왼쪽/오른쪽 바깥에 있을 때만 이동 거리 누적
    • 매 이동마다 move 갱신 및 위치 업데이트
    • 최종 출력: 사과를 모두 받을 때까지의 총 이동 거리
