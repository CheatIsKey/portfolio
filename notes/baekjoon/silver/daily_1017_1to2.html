    문제 2890 — 카약

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    각 레인(행)에서 카약 번호가 있는 열 위치를 찾아 “오른쪽으로 더 멀리 간 카약일수록 순위가 높다”는 규칙으로 판단한다. 즉, (팀 번호, 열 인덱스)을 모아 열 인덱스 기준 내림차순 정렬 → 같은 열이면 공동 순위 → 팀 번호 1부터 N까지의 순위를 차례대로 출력한다.

    제약 정리 —
    입력은 R C와 이어지는 R줄의 트랙 지도(R×C)이며, 각 행에는 정확히 하나의 카약 번호(‘1’~‘9’)가 존재한다고 가정한다(문제 조건).
    출력은 팀 번호 1부터 N(=행 수)까지의 최종 순위를 각 줄에 하나씩 출력한다.

    코드 핵심 (변수 매핑) —
    R, C: 행/열 크기  |  ground: 입력 보드(R×C)
    list: 각 행에서 찾은 (팀 번호, 열 인덱스) 쌍 목록
    rank[10]: 팀 번호(1~9)의 최종 순위를 저장하는 배열  |  std: 현재 부여할 순위(1부터 시작)

    • 한 행을 왼→오 순회하다가 숫자를 만나면 (팀 번호, 열)을 기록하고 그 행은 종료
    • list를 “열 인덱스 내림차순”으로 정렬(오른쪽일수록 앞섬)
    • 정렬 결과를 순서대로 보며, 바로 앞 원소와 열이 다를 때에만 순위 증가 → 공동 순위 처리
    • rank[팀]에 확정 순위를 기록 후, 팀 1..N 순서로 출력

    동작 과정 —
    각 행에서 가장 왼쪽에 등장하는 숫자가 그 팀의 현재 위치이므로, 그 열 인덱스를 비교 기준으로 사용한다. 열이 클수록 결승선에 가까우므로 내림차순 정렬 후 앞에서부터 순위를 매긴다. 같은 열 인덱스는 같은 순위를 부여한다(공동 순위).

    // 1) 입력 및 보드 저장
      int R, C;                // 행, 열
      char[][] ground;         // R x C 보드
      
      // 2) 각 행에서 (팀 번호, 열 인덱스) 추출
      List<int[]> list = new ArrayList<>();
      for (int i = 0; i < R; i++) {
          for (int j = 0; j < C; j++) {
              if (Character.isDigit(ground[i][j])) {
                  int team = ground[i][j] - '0';
                  list.add(new int[]{ team, j });
                  break; // 해당 행은 완료
              }
          }
      }
      
      // 3) 열 인덱스 기준 내림차순 정렬(오른쪽일수록 앞섬)
      list.sort((a, b) -> Integer.compare(b[1], a[1]));
      
      // 4) 공동 순위 처리하며 최종 순위 매기기
      int[] rank = new int[10];  // 팀 번호: 1~9
      int std = 1;               // 현재 순위
      for (int i = 0; i < list.size(); i++) {
          if (i > 0 && list.get(i)[1] != list.get(i - 1)[1]) {
              std++; // 열이 다르면 순위 증가
          }
          rank[list.get(i)[0]] = std;
      }
      
      // 5) 팀 번호 오름차순(1..N)으로 출력
      for (int team = 1; team <= list.size(); team++) {
          System.out.println(rank[team]);
      }

    엣지 케이스

    • 공동 순위: 같은 열 인덱스(같은 위치)면 같은 순위를 부여한다(다음 다른 열에서 순위 증가).
    • 노이즈 문자: 숫자가 아닌 문자는 모두 무시하고, 각 행에서 최초로 만난 숫자만 사용한다.

    시간·공간 복잡도 —
    행 탐색: O(R·C), 정렬: O(R log R) → 전체 O(R·C + R log R).
    추가 공간: 리스트 O(R)와 보드 O(R·C).

    정리

    • 각 레인에서 팀 번호와 열 위치를 추출
    • 열 인덱스 내림차순 정렬로 “오른쪽이 더 앞선다”를 구현
    • 같은 열은 공동 순위, 다르면 순위를 1 증가시켜 기록
    • 팀 번호 1..N 순서로 최종 순위를 출력

    문제 2941 — 크로아티아 알파벳

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    크로아티아 알파벳은 일반 알파벳 조합을 하나의 문자로 취급한다. 즉, c=, c-, dz=, d-, lj, nj, s=, z=의 8개 패턴을 각각 1글자로 본다. 문자열을 왼쪽부터 순회하면서, 현재 문자가 이들 조합의 시작이라면 다음 문자(또는 다음 두 문자)를 확인하고, 매칭될 경우 인덱스를 건너뛴다. 그렇지 않으면 일반 문자로 취급해 1 증가시킨다.

    제약 정리 —
    입력: 하나의 문자열 s (최대 길이 100, 소문자와 특수문자 =, - 포함).
    출력: 크로아티아 알파벳 규칙에 따라 계산된 글자 수(정수).

    코드 핵심 (변수 매핑) —
    s: 입력 문자열  |  cnt: 크로아티아 알파벳 기준으로 센 글자 수
    i: 현재 인덱스  |  c: 현재 문자

    • 문자 c가 ‘c’, ‘d’, ‘l’, ‘n’, ‘s’, ‘z’ 중 하나인 경우 → 다음 문자 확인
    • 특수 조합이 발견되면 i를 1 또는 2 증가시켜 중복 카운트 방지
    • 모든 경우에서 cnt는 +1 (각 조합이 1개의 문자로 간주됨)

    동작 과정 —
    문자열을 처음부터 끝까지 순회하면서, 조합에 해당하는 부분 문자열을 발견하면 인덱스를 건너뛰며 카운트를 증가시킨다. 예를 들어 dz=ak → dz=는 1글자, a, k 각각 1글자 → 총 3글자.

    // 1) 입력 문자열 읽기
      String s = br.readLine();
      int cnt = 0;
      
      // 2) 왼쪽부터 순회하며 패턴 판별
      for (int i = 0; i < s.length(); i++) {
          char c = s.charAt(i);
      
          if (c == 'c') {
              if (i + 1 < s.length() && (s.charAt(i + 1) == '=' || s.charAt(i + 1) == '-'))
                  i++;
          } else if (c == 'd') {
              if (i + 1 < s.length() && s.charAt(i + 1) == '-')
                  i++;
              else if (i + 2 < s.length() && s.charAt(i + 1) == 'z' && s.charAt(i + 2) == '=')
                  i += 2;
          } else if (c == 'l' || c == 'n') {
              if (i + 1 < s.length() && s.charAt(i + 1) == 'j')
                  i++;
          } else if (c == 's' || c == 'z') {
              if (i + 1 < s.length() && s.charAt(i + 1) == '=')
                  i++;
          }
      
          cnt++; // 조합이든 단일 문자든 1글자 처리
      }
      
      // 3) 결과 출력
      System.out.println(cnt);

    엣지 케이스

    • dz=는 d + z=로 중복 처리되지 않도록 i += 2로 건너뜀
    • 문자열 끝부분(i+1, i+2 접근 시) 범위 체크 필수
    • 단일 문자(a~z)는 항상 +1 처리

    시간·공간 복잡도 —
    문자열을 한 번만 순회하므로 시간 복잡도는 O(n).
    공간 복잡도는 상수 크기 변수만 사용하므로 O(1).

    정리

    • 문자열을 한 글자씩 탐색하며 크로아티아 알파벳 조합 판별
    • 조합 발견 시 인덱스를 건너뛰고, cnt는 1만 증가
    • 최종적으로 cnt를 출력해 전체 글자 수를 구함
