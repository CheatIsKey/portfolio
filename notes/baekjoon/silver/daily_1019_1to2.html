    문제 3054 — 피터팬 프레임

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    문자열의 각 문자 주위를 5×(4n+1) 격자에 다이아몬드 모양 프레임으로 그린다. 기본 프레임 문자는 #이고, 매 3번째 문자(1-기준)는 * 프레임을 사용한다. 프레임이 겹치면 *가 #를 우선해서 덮어쓴다. 각 문자의 중심 좌표를 (r, c) = (2, 4i+2)(0-기준 i)로 고정해 상하좌우 두 칸과 대각선 한 칸을 채워 규칙을 간단히 구현한다.

    제약 정리 —
    입력: 공백 없는 단일 문자열(문자 종류 무관).
    출력: 문제의 ‘피터팬 프레임’ 규칙에 맞춘 5행 문자 아트.

    코드 핵심 (변수 매핑) —
    array: 입력 문자 배열  |  n: 길이
    row = 5, column = 4n+1: 전체 캔버스 크기
    chars: 2차원 캔버스(초기값 '.')
    i: 현재 문자 인덱스(0-기준) → 중심 (r, c) = (2, 4i+2)
    chr: 프레임 문자(기본 #, (i+1) % 3 == 0이면 *)
    place(): 경계 체크 + 겹침 시 * 우선 규칙 적용

    • 프레임 좌표 패턴: (r±2, c), (r, c±2), (r±1, c±1) 채움 후, 중심 (r, c)에 실제 문자 배치
    • 겹침 처리: 이미 값이 있어도 *는 무조건 기록, #는 해당 칸이 *가 아닐 때만 기록
    • 캔버스 폭을 4n+1로 두어 양끝 문자 프레임도 항상 범위 내에 위치

    동작 과정 —
    1) 5×(4n+1) 캔버스를 '.'로 초기화 → 2) 각 문자에 대해 중심 좌표 계산 → 3) #/* 결정 → 4) 패턴 좌표에 프레임 찍기(place()) → 5) 중심에 문자 기록 → 6) 5행을 이어서 출력.

    엣지 케이스

    • 프레임 겹침: 인접 문자의 프레임이 교차하는 칸은 *가 우선 적용되어 규칙을 만족.
    • n=1: 단일 문자도 동일 패턴으로 안전하게 처리.
    • 양끝 문자: 첫/마지막 문자의 프레임도 4n+1 폭으로 경계 이탈 없음.

    시간·공간 복잡도 —
    문자 1개당 상수 개의 좌표만 갱신하므로 전체 시간 O(n).
    캔버스 크기 5 × (4n+1)에 비례하여 공간 O(n).

    정리

    • 문자 i의 중심을 (2, 4i+2)로 정해 다이아몬드 프레임을 찍는다.
    • 매 3번째 문자는 * 프레임, 겹칠 때는 *가 #보다 우선.
    • 단순 규칙의 반복으로 선형 시간·공간에 구현 가능.

    문제 3060 — 욕심쟁이 돼지

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    여섯 마리의 초기 섭취량 합을 S라 하면, 하루가 지날 때마다 총 섭취량이 정확히 4배가 된다. 따라서 임계치 N에 대해 S, 4S, 16S, ...를 순차적으로 비교하여 S · 4d-1 > N을 처음 만족하는 최소 일수 d를 구하면 된다 (코드는 반복문으로 구현).

    제약 정리 —
    입력: 테스트케이스 수 T.
    각 케이스: 정수 N (임계치), 이어서 6개의 초기 섭취량(정수).
    출력: 각 케이스마다 임계치를 처음 초과하는 최소 날짜 d 한 줄.

    코드 핵심 (변수 매핑) —
    T: 테스트케이스 수  |  N: 임계치
    sum: 6마리 초기 섭취량의 합 S
    day: 현재 날짜(초기값 1) — sum이 N을 넘을 때의 날짜가 정답
    반복: while (N >= sum) → sum *= 4, day++

    • 초기 합 sum = S를 만든 뒤, N 이상인 동안 4배씩 키우며 날짜를 증가
    • 루프 종료 시점의 day가 최소 일수 d
    • 수식 관점: d = 1 + ⌊log4(N/S)⌋ (단, S > 0)

    동작 과정 —
    1) sum에 6개 초기값을 더해 S 계산 → 2) day=1로 시작 → 3) sum이 N을 초과할 때까지 sum *= 4와 day++ 반복 → 4) day 출력.

    엣지 케이스

    • 즉시 초과: S > N이면 정답은 1.
    • 경계 동일: S == N이면 다음 날에야 초과하므로 2.
    • 오버플로(주의): 반복 중 sum이 커질 수 있음. 제약에 따라 long 사용 권장 (원문 한도 미확인 시 — 확실하지 않음).

    시간·공간 복잡도 —
    각 케이스에 대해 반복 횟수는 O(log(N/S)) (밑은 4). 덧셈 6회는 상수. 따라서 시간은 O(T + Σ log(N/S)), 추가 공간은 O(1).

    정리

    • 총합이 매일 4배가 되는 점을 이용해 기하급수 수열로 모델링한다.
    • sum이 N을 처음 초과하는 시점의 day가 정답.
    • 간단한 반복문으로 구현 가능하며, 로그 시간에 수렴.
