    문제 3553 — Dragon’s Question

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    길이가 정확히 n자리인 정수들 중에서 D로 나누어떨어지는 가장 작은 수를 찾는 문제다. n = 1이면 최솟값은 1, 그 외에는 최솟값 min = 10^(n-1), 상한(배제) max = 10^n으로 두고, min 이상에서의 D의 최초 배수 M을 계산한다. 이는 M = ceil(min / D) * D 와 동일하며, M < max면 답이고, 아니면 No solution이다. 큰 자릿수를 다루기 위해 Java의 BigInteger를 사용한다.

    제약 정리 —
    입력: 정수 n(자릿수), 정수 D.
    출력: D로 나누어떨어지는 가장 작은 n자리 수가 존재하면 그 수, 없으면 No solution.
    정확한 범위(예: n, D의 상한)는 원문을 따름.

    코드 핵심 (변수/구조 매핑) —

    • BigInteger min: n==1이면 1, 아니면 10^(n-1)
    • BigInteger max: 10^n (상한, 배제)
    • BigInteger d: D의 BigInteger 표현
    • tmp = min.mod(d), M = (tmp==0) ? min : min + (d - tmp): min 이상 최초의 D 배수
    • 판정: M.compareTo(max) < 0 이면 M 출력, 아니면 No solution

    동작 과정 —
    1) min과 max를 설정한다(자릿수 경계값 계산).
    2) min을 D로 나눈 나머지 tmp를 구한다.
    3) tmp==0이면 min 자체가 답 후보, 아니면 d - tmp만큼 올려 최초 배수 M을 만든다.
    4) M < max면 M은 n자리이므로 출력, 아니면 존재하지 않으므로 No solution.

    정확성 근거 —
    정수의 배수 집합은 등차수열이므로, min 이상에서의 최초 배수는 나머지 보정 (d - (min mod d))를 더해 얻는 값과 일치한다. 또한 n자리 수의 범위는 [10^(n-1), 10^n)이므로, M이 이 구간에 속하는지(M < 10^n)만 확인하면 충분하다.

    엣지 케이스

    • n = 1: min = 1로 시작.
    • D = 1: 항상 해 존재, 답은 min.
    • M = max가 되는 경우: max는 n+1자리의 경계값이므로 n자리가 아니어서 No solution이 맞다.
    • 아주 큰 n: 10^n 구성 및 출력 비용이 커질 수 있음.

    시간·공간 복잡도 —
    자릿수 길이를 L ≈ n이라 보면, 10^(n-1), 10^n 생성과 나머지/덧셈/비교 같은 BigInteger 연산은 자릿수에 선형~준선형으로 비례한다. 따라서 전체는 BigInteger 산술 기준으로 O(L)~O(L log L) 수준이며, 출력도 O(L)가 된다 (상수·구현 세부는 JVM/라이브러리 의존).

    정리

    • n자리 구간을 [10^(n-1), 10^n)로 잡고, 그 안에서의 D의 첫 배수 M을 계산.
    • M < 10^n이면 답, 아니면 존재하지 않으므로 No solution.
    • 큰 자릿수를 다루기 위해 BigInteger 연산으로 안전하게 처리.

    문제 3711 — 학번

    문제 링크

    백준 문제 보기

    코드 (IDE 스크린샷)

    해설

    핵심 아이디어 —
    각 학생의 학번 배열이 주어졌을 때, 어떤 양의 정수 M에 대해 학번 % M 값들이 모두 서로 달라지도록 만드는 가장 작은 M을 찾는 문제다.
    M을 1부터 증가시키며, 모든 학번의 나머지를 구해 정렬한 뒤 인접 중복이 있는지 검사한다. 중복이 없을 때의 M이 정답이다.

    제약 정리 —
    입력: 테스트케이스 수 N. 각 테스트케이스마다 학생 수 G, 이어서 G개의 학번.
    출력: 각 테스트케이스마다 조건을 만족하는 가장 작은 M.
    정확한 수치 범위(N, G, 학번의 최댓값)는 원문을 따름.

    코드 핵심 (변수/구조 매핑) —

    • int N: 테스트케이스 수.
    • int G: 학생 수.
    • int[] students: 학번 목록.
    • M: 1부터 증가시키며 검사하는 모듈러 값.
    • int[] r: students[j] % M 나머지들을 담는 배열.
    • 중복 판정: Arrays.sort(r) 후 인접한 두 값이 같은지 확인.

    동작 과정 —
    1) M = 1로 시작한다.
    2) 모든 학번에 대해 r[j] = students[j] % M 계산.
    3) r을 정렬하고 인접 중복이 있으면 실패(충돌) → M 증가 후 반복.
    4) 중복이 없으면 현재 M을 출력하고 다음 테스트케이스로 진행.

    정확성 근거 —
    주어진 M에 대해 모든 나머지가 서로 다르면, 서로 다른 두 학번 a ≠ b에 대해 a % M ≠ b % M이므로 충돌 없이 G개의 상이한 클래스로 분류된다.
    가장 작은 M을 1부터 선형 탐색하므로, 처음으로 중복이 사라지는 순간의 M이 문제에서 요구하는 최소값과 일치한다.

    엣지 케이스

    • G = 1: 어떤 M ≥ 1에서도 항상 유일하므로 정답은 1.
    • 아주 큰 학번: 나머지 연산은 int 범위 내에서 안전(학번 입력 범위는 원문 참고).

    시간·공간 복잡도 —
    한 M에 대한 검사는 나머지 계산 O(G) + 정렬 O(G log G) + 스캔 O(G)로 O(G log G).
    정답 M*까지 반복하므로 전체는 대략 O(M* · G log G). (최악의 M*은 입력에 의존)
    공간은 나머지 배열 O(G).

    정리

    • M을 1부터 증가시키며 학번 % M의 충돌 여부를 검사.
    • 나머지 배열을 정렬해 인접 중복이 없으면 그 M이 최소 정답.
    • 간단하고 구현이 직관적이며, 공간은 O(G)로 작다.
