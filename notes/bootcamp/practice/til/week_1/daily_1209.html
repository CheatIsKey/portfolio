    내일배움캠프 Spring 백엔드 2주차 TIL — SQL 기초 & Java·Spring 이해

    작성일 2025-12-09 · 분류: 부트캠프, TIL, SQL, Java, Spring

    2주차 개요 — SQL 기본 문법과 개념 정리

    정리

    2주차 첫날은 SQL 기본 문법 복습과 Java·Spring 개념 정리에 집중했다. 제공된 퀘스트용 임시 데이터를 활용해 SELECT·WHERE·BETWEEN·LIKE·SUM 같은 기초 문법을 실제로 다시 써 보았고, 별도의 아티클을 통해 Java와 Spring의 역할과 관계를 백엔드 로드맵 관점에서 정리했다.

    • 직원(camp_employees), 상품(camp_products), 주문(camp_orders) 테이블을 이용해 SQL 퀘스트 12문제를 풀이했다.
    • 조건 조회, 중복 제거, 집계, 정렬 등 자주 쓰이는 패턴을 다시 한 번 손으로 써 보며 익혔다.
    • “Java는 언어, Spring은 프레임워크”라는 기본 관점을 정리하고, 내가 만들고 싶은 서비스에 어떻게 적용할지 생각해 보았다.

    SQL 기초 퀘스트 — 직원 테이블(camp_employees)

    예시 데이터

    직원 정보를 담은 camp_employees 테이블 임시 데이터는 다음과 같다.

    id name position salary hire_date
    1 홍길동 주니어 개발자 32000 2022-04-15
    2 김개발 PM 42000 2021-09-10
    3 박매니저 파트장 36000 2023-06-10
    4 이팀장 팀장 52000 2021-07-01

    풀이한 쿼리

    1. camp_employees 테이블에서 모든 직원의 이름(name)과 직급(position)을 선택하는 쿼리

    SELECT name AS "이름", position AS "직급"
FROM camp_employees;

    2. camp_employees 테이블에서 중복 없이 모든 직급(position)을 선택하는 쿼리

    SELECT DISTINCT position
FROM camp_employees;

    3. camp_employees 테이블에서 연봉(salary)이 40000과 60000 사이인 직원들을 선택하는 쿼리

    SELECT *
FROM camp_employees
WHERE salary BETWEEN 40000 AND 60000;

    4. camp_employees 테이블에서 입사일(hire_date)이 2023년 1월 1일 이전인 모든 직원들을 선택하는 쿼리

    SELECT *
FROM camp_employees
WHERE hire_date < '2023-01-01';

    SQL 기초 퀘스트 — 상품 테이블(camp_products)

    예시 데이터

    상품 정보를 담은 camp_products 테이블 임시 데이터는 다음과 같다.

    id product_name price category
    1 울트라북 프로 1200 컴퓨터
    2 무선 청소기 300 생활가전
    3 갤럭시 탭 600 컴퓨터
    4 에스프레소 머신 200 주방가전

    풀이한 쿼리

    5. camp_products 테이블에서 제품 이름(product_name)과 가격(price)만을 선택하는 쿼리

    SELECT product_name AS "제품 이름", price AS "가격"
FROM camp_products;

    6. camp_products 테이블에서 제품 이름에 '프로'가 포함된 모든 제품을 선택하는 쿼리

    SELECT *
FROM camp_products
WHERE product_name LIKE '%프로%';

    7. camp_products 테이블에서 제품 이름이 '갤'로 시작하는 모든 제품을 선택하는 쿼리

    SELECT *
FROM camp_products
WHERE product_name LIKE '갤%';

    8. camp_products 테이블에서 모든 제품을 구매하기 위해 필요한 총 금액을 계산하는 쿼리

    SELECT SUM(price) AS "총액"
FROM camp_products;

    SQL 기초 퀘스트 — 주문 테이블(camp_orders)

    예시 데이터

    주문 정보를 담은 camp_orders 테이블 임시 데이터는 다음과 같다.

    id customer_id product_id amount shipping_fee order_date
    1 101 1 3 48000 2023-11-01
    2 202 2 1 9000 2023-11-02
    3 303 4 1 21000 2023-11-05
    4 404 3 2 26000 2023-11-05
    5 505 1 1 16000 2023-11-09

    풀이한 쿼리

    9. camp_orders 테이블에서 주문 수량(amount)이 2개 이상인 주문을 진행한 소비자의 ID(customer_id)만 선택하는 쿼리

    SELECT customer_id AS "소비자 ID"
FROM camp_orders
WHERE amount >= 2;

    10. camp_orders 테이블에서 2023년 11월 2일 이후에 주문된, 주문 수량(amount)이 2개 이상인 주문을 선택하는 쿼리

    SELECT *
FROM camp_orders
WHERE order_date > '2023-11-02'
  AND amount >= 2;

    11. camp_orders 테이블에서 주문 수량이 3개 미만이면서 배송비(shipping_fee)가 15000원보다 비싼 주문을 선택하는 쿼리

    SELECT *
FROM camp_orders
WHERE amount < 3
  AND shipping_fee > 15000;

    12. camp_orders 테이블에서 배송비가 높은 금액 순으로 정렬하는 쿼리

    SELECT id, customer_id, product_id, amount, shipping_fee, order_date
FROM camp_orders
ORDER BY shipping_fee DESC;

    Java와 Spring 이해하기 — 언어와 프레임워크의 역할

    오늘 정리한 개념

    강의를 통해 “Java와 Spring의 차이와 관계”를 다시 정리했다. 단순 암기가 아니라, 내가 만들고 싶은 서비스를 예로 들면서 두 기술이 각각 어디에 쓰이는지 연결해 보는 데 집중했다.

    • Java
      객체 지향 프로그래밍 언어로, JVM 위에서 동작해 운영체제에 상관없이 실행될 수 있다.
      타입이 명확하고 안정성이 높아, 금융·공공·대규모 웹서비스 등에서 많이 사용된다.
      내가 작성하는 도메인 로직, 계산, 검증, 데이터 변환을 담당한다.
    • Spring
      Java로 서버 개발을 할 때 반복되는 작업을 줄여주는 프레임워크이다.
      IoC/DI를 통해 객체 생성과 의존성 관리를 대신해 주고, MVC 구조로 웹 요청 흐름을 정리해 준다.
      트랜잭션, 예외 처리, 인증/인가, 설정 관리 등 인프라성 기능을 제공해 개발자는 비즈니스 로직에 집중할 수 있게 해준다.
    • Spring Boot
      복잡한 설정을 자동화해 “바로 실행 가능한” Spring 애플리케이션을 만들기 쉽게 도와준다.
      내 쇼핑몰 프로젝트에서도 스타터 의존성과 자동 설정 덕분에 서버 구동에 드는 시간을 줄일 수 있었다.

    나의 서비스에 대입해 보기

    예시로, 내가 이미 진행 중인 쇼핑몰 + 고객센터 백엔드 서비스를 기준으로 Java와 Spring의 역할을 다시 정리했다.

    • Java가 맡는 역할
      • 상품, 주문, 장바구니, 회원, 문의글 같은 도메인 모델과 비즈니스 로직 구현
      • 가격 계산, 재고 검증, 주문 상태 변경, 유효성 검사 같은 순수 로직 처리
      • 테스트 코드에서 동작을 검증할 수 있는 핵심 메서드 작성
    • Spring이 맡는 역할
      • HTTP 요청을 컨트롤러에 매핑하고, 서비스·리포지토리로 흐름을 전달
      • 스프링 컨테이너를 통한 의존성 주입(DI)으로 객체 생명주기 관리
      • JPA와 연동해 트랜잭션을 관리하고, 예외를 공통 방식으로 처리
      • 로그인, 권한 체크, 세션/토큰 기반 인증 구조 제공

    오늘의 회고와 내일 계획

    오늘 느낀 점

    SQL 퀘스트는 이미 아는 문법이 많았지만, 실제 테이블 구조를 놓고 다시 쿼리를 쓰면서 WHERE 조건과 집계, 정렬을 자연스럽게 복습할 수 있었다. 특히 임시 데이터라도 직접 표로 정리해 보니, 나중에 포트폴리오에서 “어떤 데이터 모델을 상정하고 퀘스트를 풀었는지” 보여주기 좋은 자료가 될 것 같았다.

    Java와 Spring에 대해서는 “Java는 언어, Spring은 구조(프레임워크)”라는 큰 그림을 다시 확인했다. 앞으로는 단순히 기능을 붙이는 데서 끝나는 것이 아니라, 비즈니스 로직은 Java 코드로, 인프라와 흐름 관리는 Spring으로 명확히 나눠서 생각하는 습관을 더 굳혀야겠다고 느꼈다.

    내일 계획

    • SQL에서 JOIN과 GROUP BY, HAVING 같은 구문을 다시 정리하고 간단한 예제를 스스로 만들어 보기.
    • Java 기본 문법과 컬렉션, 예외 처리 파트를 복습하고, 이해가 애매한 부분은 TIL에 따로 정리하기.
    • Spring 핵심 개념(IoC/DI, 빈 스코프, MVC 흐름)에 대한 아티클을 읽고, 쇼핑몰 프로젝트 구조와 비교해 보기.
