    오늘의 개요 — JD 분석과 핵심 역량 점검

    정리

    오늘은 단순히 강의를 듣는 것보다, “내가 실제로 가고 싶은 포지션에서는 무엇을 요구하는지”를 직접 확인하는 데 초점을 맞췄다. 토스증권, 한울, 카카오뱅크의 서버 개발자 JD를 비교하면서 Java/Spring 백엔드 개발자로서 앞으로 어떤 역량을 더 쌓아야 할지 정리했고, 오후에는 SQL 기초 문제를 변형한 예제 테이블을 만들어 다시 한 번 쿼리를 손으로 써 보며 감각을 확인했다.

    • 관심 분야: 커머스 백엔드, 공공데이터 기반 스마트 모빌리티, 데이터 기반 업무 자동화·예측 서비스.
    • 토스증권, 한울, 카카오뱅크 서버 개발자 JD를 리서치하고 공통 역량과 차이점을 정리했다.
    • JD 분석을 바탕으로 나의 커리어 방향과 지금 부족한 역량을 구체적으로 적어 보았다.
    • 연습용 학생/프로젝트/게임 유저 테이블을 만들어 SQL 쿼리를 복습했다.

    2️⃣ JD 공통점과 차이점 정리

    공통적으로 요구되는 핵심 역량

    세 공고를 함께 놓고 보면, 먼저 Java 계열과 Spring 기반 서버 개발 역량을 모두 핵심으로 본다는 점이 뚜렷하다. 토스증권은 Java/Kotlin + Spring Framework, 카카오뱅크는 Kotlin 또는 Java + Spring Boot + JPA, 한울은 Java + Spring Framework를 주요 기술 스택으로 제시한다. 결국 “실무에서 통하는 백엔드 개발자”의 기본 축은 Java와 Spring 라인이라는 메시지가 분명하다.

    두 번째 공통점은 코드만 짜는 개발자가 아니라 서비스 전체를 보는 개발자를 원한다는 점이다. 토스증권과 카카오뱅크는 아키텍처 개선, 기술부채 해결, 대용량 트래픽에서의 트러블슈팅과 지표 개선 경험을 요구하고, 한울 역시 현장 시스템 구축·운영까지 포함하는 역할을 맡긴다. 즉, “배포 이후를 책임지는 서버 개발자”를 상정하고 있다.

    마지막으로, 협업과 커뮤니케이션이 빠지지 않는다. 코드리뷰, 스터디, 피드백 문화, 형상관리 경험 등을 JD에 명시하며 “팀으로 일하는 개발자”를 강조한다. 정리하면, 세 JD의 공통 핵심 역량은 Java/Spring 기반 서버 개발 실력 + 운영·아키텍처 관점 + 협업 능력이다.

    JD마다 드러나는 차이점

    • 토스증권: 금융 원장과 트레이딩 시스템, 실시간 데이터 처리, Kubernetes·ELK 등 분산 인프라 환경이 강조된다. 기술적으로 도전적인 환경에서 성장하고 싶은 서버 개발자를 강하게 끌어당기는 JD라는 느낌이었다.
    • 한울: 출입통제·RFID·안전관리·IoT, 전자정부 프레임워크 등 공공·SI + IoT 도메인 색이 강하다. 다양한 기술 스택을 폭넓게 다루는 실무형 개발자 포지션으로, 전통적인 자격요건과 현장 경험을 중시한다.
    • 카카오뱅크: 금융 뱅킹 도메인과 대용량 트래픽, 장기간 운영·아키텍처 개선 경험을 가진 시니어급 백엔드 개발자를 상정한다. 동시에 복지·문화까지 상세히 보여주며 “어떤 환경에서 일하게 되는지”를 투명하게 전달한다.

    3️⃣ 나의 커리어 방향과 부족한 역량

    Spring 백엔드에서 관심 있는 영역

    나는 문제를 구조화하고, 루틴으로 기본기를 증명하는 방식으로 성장해 왔다. 엑셀 매크로로 고객 정보 입력을 자동화할 때도 단순히 반복 작업을 줄이는 데서 끝나지 않고, “이 과정을 하나의 시스템으로 만들면 얼마나 안정적으로 돌릴 수 있을까”를 고민했었다. 음악 데이터 처리 파이프라인에서는 파싱 오류가 발생해도 시스템이 멈추지 않도록 자동 복구·재시도 구조와 로그 기반 검증 절차를 설계하면서, 예외 처리와 안정적인 운영 구조를 설계하는 일에 재미를 느꼈다.

    현재 진행 중인 쇼핑몰 프로젝트에서도 단순히 기능을 구현하는 수준을 넘어서, 양방향 연관관계 조정, Fetch Join을 통한 N+1 문제 해결 등 도메인 모델링과 성능을 함께 고민하는 영역을 즐기고 있다. 이런 경험을 합쳐 보면, 나는 Spring 백엔드에서 비즈니스 로직과 데이터 흐름을 설계하고, 예외와 성능까지 고려해 구조를 다듬는 영역에 가장 큰 흥미를 느끼고 있다.

    흥미가 생긴 도메인

    도메인 관점에서 보면, 개인 쇼핑몰 프로젝트는 커머스 도메인에 속한다. 회원·상품·주문·결제 흐름을 설계하면서 데이터 정합성과 사용 경험이 직접적으로 연결되는 구조를 다루다 보니, “한 줄의 로직이 서비스 경험과 매출에 바로 영향을 준다”는 점이 매우 흥미로웠다.

    영주시 데이터 분석·활용 공모전에서는 국토교통부 공공데이터를 기반으로 자전거 대여소 위치를 선정하고, 지도 API와 QR 기반 대여/반납, 실시간 현황 조회 기능을 넣어 무인 자전거 시스템 최적화를 목표로 앱을 개발했다. 이 경험 덕분에 공공데이터 기반 스마트 모빌리티·생활 편의 서비스 도메인에도 관심이 생겼다. 여기에 CPI·환율 등 거시 경제 데이터를 활용한 물가 예측 프로젝트와 IoT 센서 연동 앱 경험까지 더하면, 결국 내가 끌리는 분야는 “데이터가 많이 흐르고, 그 데이터를 바탕으로 사람들의 불편을 줄이는 서비스”라고 정리할 수 있다.

    아직 부족하다고 느낀 역량

    • 1. 대규모 분산 인프라 운영 경험
      파이프라인 자동화, 로그 기반 검증, 예외 처리 구조 설계는 경험했지만, Redis·Kafka·Kubernetes·ELK 같은 도구를 실제 서비스 규모에서 운용해 본 경험은 아직 없다. 세 JD 모두 대용량 트래픽과 실시간 처리, 트러블슈팅을 강조하기 때문에, 분산 환경에서의 운영 경험은 앞으로 반드시 채워야 할 과제다.
    • 2. 특정 생태계에 특화된 기술
      한울 JD에서 언급된 .NET, iOS 네이티브, 전자정부 프레임워크 등은 아직 경험이 없는 영역이다. 현재는 Java/Spring·Android 중심이지만, 공공·SI 영역까지 커리어를 넓힌다면 언젠가는 마주칠 스택이기 때문에 “지금은 부족하지만 향후 학습 후보군”으로 인식하게 되었다.
    • 3. 도메인에 깊게 뿌리내린 장기간 운영 경험
      지금까지는 근로·공모전·연구활동·개인 프로젝트 중심이라, 수년 단위로 하나의 서비스를 운영하며 아키텍처를 개선해 온 경험은 없다. 이 부분은 첫 커리어를 선택할 때 “어떤 도메인에서 이런 경험을 쌓을지”를 고민하게 만드는 지점이다.

    이 사전캠프에서 꼭 성장시키고 싶은 능력

    • 탄탄한 Spring 백엔드 기본기
      도메인 설계, JPA 활용, 예외 처리, 테스트 코드까지 포함한 “하나의 일관된 개발 철학”을 세우는 것이 목표다.
    • 운영을 염두에 둔 서버 개발
      장애 상황을 가정한 설계, 로그·모니터링 구조, 성능 측정과 개선, 배포·롤백 전략 등을 미니 프로젝트에서라도 경험해 보고 싶다.
    • 팀 기반 협업 역량
      Git 브랜치 전략, 코드리뷰, 이슈 트래킹, 간단한 설계 문서 공유 등을 통해 “함께 일하기 편한 백엔드 개발자”에 가까워지는 것이 목표다.

    4️⃣ SQL 복습 — 연습용 테이블과 쿼리

    1. 학습용 학생 테이블 (bootcamp_students)

    오늘은 스파르타 예제 문제를 그대로 쓰지 않고, 테이블 이름과 컬럼명을 살짝 바꿔서 연습용 데이터를 만들었다.

    id name course grade start_year
    1 르탄 Node.js A 2023
    2 배캠 Spring B 2022
    3 구구 Unity C 2021
    4 이션 Node.js B 2022

    1) 모든 학생의 이름과 트랙을 선택하는 쿼리

    SELECT name AS "이름", course AS "트랙"
FROM bootcamp_students;

    2) Unity 트랙이 아닌 학생들을 선택하는 쿼리

    SELECT *
FROM bootcamp_students
WHERE course <> 'Unity';

    3) 시작 연도가 2021년 또는 2023년인 학생을 선택하는 쿼리

    SELECT *
FROM bootcamp_students
WHERE start_year IN (2021, 2023);

    4) Node.js 트랙이면서 학점이 A인 학생의 시작 연도를 선택하는 쿼리

    SELECT name AS "이름", start_year AS "수강 시작년도"
FROM bootcamp_students
WHERE course = 'Node.js'
  AND grade = 'A';

    2. 팀 프로젝트 테이블 (bootcamp_projects)

    id project_name started_at finished_at cloud_cost
    1 일조 2023-01-01 2023-01-07 30000
    2 꿈꾸는이조 2023-03-15 2023-03-22 50000
    3 보람삼조 2023-11-20 2023-11-30 80000
    4 사조참치 2022-07-01 2022-07-30 75000

    1) 클라우드 비용이 40000 이상인 프로젝트의 이름을 선택하는 쿼리

    SELECT project_name AS "프로젝트 명"
FROM bootcamp_projects
WHERE cloud_cost >= 40000;

    2) 2022년에 시작된 프로젝트를 선택하는 쿼리 (start_date < '2023-01-01' 사용 X)

    SELECT *
FROM bootcamp_projects
WHERE YEAR(started_at) = 2022;

    3) 현재 진행 중인 프로젝트를 선택하는 쿼리 (오늘 날짜 하드코딩 X)

    SELECT *
FROM bootcamp_projects
WHERE CURDATE() BETWEEN started_at AND finished_at;

    4) 각 프로젝트의 지속 기간(일 수)을 계산하는 쿼리

    SELECT project_name,
       DATEDIFF(finished_at, started_at) AS duration_days
FROM bootcamp_projects;

    3. 게임 유저 테이블 (summoner_users)

    id name region rank_score joined_at
    1 르탄이 한국 1300 2019-06-15
    2 배캠이 미국 1500 2020-09-01
    3 구구이 한국 1400 2021-01-07
    4 이션이 미국 1350 2019-11-15

    1) 전체 유저의 레이팅 순위를 계산하는 쿼리 (점수가 높을수록 순위↑)

    SELECT *,
       RANK() OVER (ORDER BY rank_score DESC) AS "순위"
FROM summoner_users;

    2) 가장 늦게 게임을 시작한 유저의 이름을 선택하는 쿼리

    SELECT name
FROM summoner_users
ORDER BY joined_at DESC
LIMIT 1;

    3) 지역별로 레이팅이 높은 순으로 유저를 정렬하는 쿼리

    SELECT *
FROM summoner_users
ORDER BY region, rank_score DESC;

    4) 지역별 평균 레이팅을 계산하는 쿼리

    SELECT region,
       AVG(rank_score) AS "평균 점수"
FROM summoner_users
GROUP BY region;
