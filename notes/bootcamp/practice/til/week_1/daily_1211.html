    내일배움캠프 Spring 백엔드 4일차 TIL — SQL 그룹 함수 & Hello World

    작성일 2025-12-11 · 분류: 부트캠프, TIL, SQL, Article

    4일차 개요 — 집계·날짜 함수 복습과 Hello World 정리

    정리

    4일차에는 SQL의 집계 함수와 날짜 함수를 다시 손으로 익히고, 개발자의 상징 같은 문장인 Hello World의 기원과 의미를 정리했다. 단순 문법 암기가 아니라, 실제로 자주 등장할 만한 테이블 구조를 상상해서 예시 데이터를 만들고 그 위에 쿼리를 올려 보는 흐름으로 공부했다.

    • 피드백·의사·환자 테이블을 예시로, GROUP BY, COUNT, AVG, DATEDIFF, DATE_SUB 등을 복습했다.
    • 윈도 함수 대신, 오늘은 그룹 단위 집계와 날짜 조건 중심으로 패턴을 다시 정리했다.
    • Hello World의 기원과 사용 방식, 개발자 문화에서 가지는 의미를 짧게 정리하며 “첫 출력문”의 느낌을 되짚었다.

    SQL 기초 퀘스트 — 피드백 테이블(game_feedbacks)

    예시 데이터

    피드백 정보를 담은 game_feedbacks 테이블 임시 데이터는 다음과 같다.

    id summoner_name score created_at
    1 르탄이 5 2023-03-01
    2 배캠이 4 2023-03-02
    3 구구이 3 2023-03-01
    4 이션이 5 2023-03-03
    5 구구이 4 2023-03-04

    풀이한 쿼리

    1. game_feedbacks 테이블에서 만족도 점수(score)에 따라 피드백을 내림차순으로 정렬하는 쿼리

    SELECT *
FROM game_feedbacks
ORDER BY score DESC;

    2. game_feedbacks 테이블에서 각 유저별 최신 피드백을 찾는 쿼리

    SELECT summoner_name,
       MAX(created_at) AS latest_feedback_at
FROM game_feedbacks
GROUP BY summoner_name;

    3. game_feedbacks 테이블에서 만족도 점수가 5점인 피드백의 수를 계산하는 쿼리

    SELECT COUNT(*) AS "5점 피드백 수"
FROM game_feedbacks
WHERE score = 5;

    4. game_feedbacks 테이블에서 가장 많은 피드백을 남긴 상위 3명의 유저를 찾는 쿼리

    SELECT summoner_name,
       COUNT(*) AS feedback_count
FROM game_feedbacks
GROUP BY summoner_name
ORDER BY feedback_count DESC
LIMIT 3;

    5. game_feedbacks 테이블에서 평균 만족도 점수가 가장 높은 날짜를 찾는 쿼리

    SELECT created_at AS feedback_date
FROM game_feedbacks
GROUP BY created_at
ORDER BY AVG(score) DESC
LIMIT 1;

    SQL 기초 퀘스트 — 의사 테이블(hospital_doctors)

    예시 데이터

    의사 정보를 담은 hospital_doctors 테이블 임시 데이터는 다음과 같다.

    id doctor_name department hired_at
    1 르탄이 피부과 2018-05-10
    2 배캠이 성형외과 2019-06-15
    3 구구이 안과 2020-07-20

    풀이한 쿼리

    1. hospital_doctors 테이블에서 전공(department)이 성형외과인 의사의 이름을 알아내는 쿼리

    SELECT doctor_name
FROM hospital_doctors
WHERE department = '성형외과';

    2. hospital_doctors 테이블에서 각 전공별 의사 수를 계산하는 쿼리

    SELECT department,
       COUNT(*) AS doctor_count
FROM hospital_doctors
GROUP BY department;

    3. hospital_doctors 테이블에서 현재 날짜 기준으로 5년 이상 근무한 의사 수를 계산하는 쿼리

    SELECT COUNT(*) AS num_of_doctors
FROM hospital_doctors
WHERE hired_at <= DATE_SUB(CURDATE(), INTERVAL 5 YEAR);

    4. hospital_doctors 테이블에서 각 의사의 근무 기간을 계산하는 쿼리

    SELECT doctor_name,
       DATEDIFF(CURDATE(), hired_at) AS worked_days
FROM hospital_doctors;

    SQL 기초 퀘스트 — 환자 테이블(clinic_patients)

    예시 데이터

    환자 정보를 담은 clinic_patients 테이블 임시 데이터는 다음과 같다.

    id patient_name birth_date gender last_visit_at
    1 르탄이 1985-04-12 남자 2023-03-15
    2 배캠이 1990-08-05 여자 2023-03-20
    3 구구이 1982-12-02 여자 2023-02-18
    4 이션이 1999-03-02 남자 2023-03-17

    풀이한 쿼리

    1. clinic_patients 테이블에서 각 성별(gender)에 따른 환자 수를 계산하는 쿼리

    SELECT gender,
       COUNT(*) AS patient_count
FROM clinic_patients
GROUP BY gender;

    2. clinic_patients 테이블에서 현재 나이가 40세 이상인 환자들의 수를 계산하는 쿼리

    SELECT COUNT(*) AS over_40_count
FROM clinic_patients
WHERE birth_date <= DATE_SUB(CURDATE(), INTERVAL 40 YEAR);

    3. clinic_patients 테이블에서 마지막 방문 날짜(last_visit_at)가 1년 이상 된 환자들을 선택하는 쿼리

    SELECT *
FROM clinic_patients
WHERE last_visit_at <= DATE_SUB(CURDATE(), INTERVAL 1 YEAR);

    4. clinic_patients 테이블에서 생년월일이 1980년대인 환자들의 수를 계산하는 쿼리

    SELECT COUNT(*) AS born_in_80s
FROM clinic_patients
WHERE YEAR(birth_date) BETWEEN 1980 AND 1989;

    Hello World — 개발자의 첫 인사

    기원과 의미

    프로그래밍을 처음 배울 때 가장 먼저 출력해 보는 문장이 바로 Hello, World!다. 단순한 한 줄이지만, 대부분의 개발자에게는 “처음으로 컴퓨터와 대화가 통했던 순간”을 떠올리게 하는 문장이다.

    정확한 기원은 명확하지 않지만, 오늘날 가장 유력하게 언급되는 출발점은 데니스 리치와 브라이언 커니핸이 집필한 C 언어 서적 『The C Programming Language』이다. 이 책에서 함수와 프로그램 구조를 설명하기 위한 예제 코드로 hello, world를 사용했고, 이후 여러 개정판과 다른 언어로 번역되면서 다양한 형태의 Hello World 예제가 퍼져 나갔다.

    이 설이 절대적인 “정답”은 아닐 수 있지만, 오늘날 Hello World를 전 세계 개발자 사이에서 유명하게 만든 계기가 이 책이라는 점은 거의 합의된 사실에 가깝다.

    어떻게 쓰이고 있을까

    • 새로운 언어를 배울 때 가장 먼저 작성해 보는 출력 예제로 사용된다.
    • 개발 환경을 구축한 뒤, 컴파일·실행이 제대로 되는지 확인하는 기본 점검 코드로 쓰인다.
    • 펌웨어를 수정하거나 새로운 기기를 해킹했을 때 “정상적으로 내 코드가 올라갔다”는 신호로 출력하기도 한다.
    • 개발을 넘어 게임·영상·노래 제목 등 대중문화 속에서도 종종 인용되는 표현이 되었다.

    C 언어를 만든 사람 중 한 명인 데니스 리치가 세상을 떠났을 때, 많은 개발자들이 “Goodbye, world!”라는 메시지로 추모를 남겼다는 이야기도 인상적이다. 한 줄짜리 예제가 세대와 언어를 넘어 하나의 상징처럼 자리 잡았다는 느낌이 든다.

    개인적으로 느낀 점

    Hello World의 기원을 알고 나니, 앞으로 새로운 언어나 프레임워크를 시작할 때 단순한 예제 한 줄에서 끝내기보다는 “이 코드로 무엇을 해 보고 싶은지”를 같이 떠올려 보는 게 좋겠다는 생각이 들었다. 단순 출력문이지만, 나한테는 새로운 기술과 세상에 인사하는 첫 버튼 같은 역할을 하니까.

    오늘의 회고와 내일 계획

    오늘 느낀 점

    오늘 SQL 퀘스트는 GROUP BY와 날짜 함수를 한 번에 복습하기 좋은 세트였다. 특히 DATE_SUB와 DATEDIFF처럼 날짜를 기준으로 “몇 년 이상 근무”, “나이 계산” 같은 로직을 쿼리 레벨에서 처리하는 패턴을 다시 써 보면서, 나중에 실제 서비스에서 자주 쓰게 될 장면들이 떠올랐다.

    Hello World 아티클을 정리하면서는, 그냥 당연하게 치던 문장이 사실은 개발자 문화와 역사 속에서 꽤 상징적인 역할을 하고 있다는 걸 다시 느꼈다. 새 프로젝트를 시작할 때마다 찍어 보는 이 한 줄이, 앞으로는 “새로운 스택에 도전하는 나 자신”을 리셋하는 신호처럼 느껴지면 좋겠다.

    내일 계획

    • 오늘 다룬 집계·날짜 함수에 이어, JOIN + GROUP BY가 섞인 문제를 직접 만들어 보고 풀어 보기.
    • 쇼핑몰 프로젝트에서 “최근 3개월 주문 수” 같은 통계를 어떤 쿼리로 뽑을지 스케치해 보기.
    • Hello World처럼 자주 쓰는 예제 코드를, 내 프로젝트 기준으로 어떻게 바꿀지 작은 아이디어를 메모해 두기.
    • 내일도 TIL을 이어 가면서, 하루에 적어도 하나는 “나중에 다시 써먹을 수 있는 쿼리 패턴”을 남기는 것을 목표로 하기.
