    내일배움캠프 Spring 백엔드 5일차 TIL — SQL JOIN & 집계 심화

    작성일 2025-12-17 · 분류: 부트캠프, TIL, SQL

    5일차 개요 — JOIN과 GROUP BY 패턴 정리

    정리

    5일차에는 두 테이블을 묶는 JOIN과 집계 함수 + GROUP BY 패턴을 중심으로 SQL을 복습했다. 직원/부서, 상품/주문 테이블을 예시로 만들어 두고, 부서별 인원 수, 가장 많이 팔린 상품, 판매되지 않은 상품처럼 실제 서비스에서 자주 등장할 법한 쿼리들을 직접 작성해 보았다.

    • 직원·부서 테이블로 INNER JOIN, LEFT JOIN, GROUP BY, COUNT를 다시 한 번 손으로 쳐 보며 감을 잡았다.
    • 상품·주문 테이블에서는 매출 합계, 평균 수량, 판매되지 않은 상품 찾기 등 리포트성 쿼리를 연습했다.
    • 단순 SELECT를 넘어, “어떤 질문을 하고 싶은지”를 먼저 정하고 그에 맞는 쿼리를 설계하는 연습이 됐다.

    SQL 기초 퀘스트 — 직원·부서 테이블(corp_employees, corp_departments)

    예시 데이터

    직원과 부서 정보를 담은 corp_employees, corp_departments 테이블 임시 데이터는 다음과 같다.

    corp_employees

    emp_id dept_id emp_name
    1 101 르탄이
    2 102 배캠이
    3 103 구구이
    4 101 이션이

    corp_departments

    dept_id dept_name
    101 인사팀
    102 마케팅팀
    103 기술팀

    풀이한 쿼리

    1. 현재 존재하고 있는 총 부서의 수를 구하는 쿼리

    SELECT COUNT(*) AS dept_count
FROM corp_departments;

    2. 모든 직원과 그들이 속한 부서의 이름을 나열하는 쿼리

    SELECT e.emp_name  AS "직원 이름",
       d.dept_name AS "부서 이름"
FROM corp_employees   e
JOIN corp_departments d ON e.dept_id = d.dept_id;

    3. '기술팀' 부서에 속한 직원들의 이름을 나열하는 쿼리

    SELECT e.emp_name AS "직원 이름"
FROM corp_employees   e
JOIN corp_departments d ON e.dept_id = d.dept_id
WHERE d.dept_name = '기술팀';

    4. 부서별로 직원 수를 계산하는 쿼리

    SELECT d.dept_name              AS "부서 이름",
       COUNT(e.emp_id)          AS employee_count
FROM corp_departments d
LEFT JOIN corp_employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name;

    5. 직원이 없는 부서의 이름을 찾는 쿼리

    SELECT d.dept_name
FROM corp_departments d
LEFT JOIN corp_employees e ON d.dept_id = e.dept_id
WHERE e.emp_id IS NULL;

    6. '마케팅팀' 부서에만 속한 직원들의 이름을 나열하는 쿼리

    SELECT e.emp_name AS "직원 이름"
FROM corp_employees   e
JOIN corp_departments d ON e.dept_id = d.dept_id
WHERE d.dept_name = '마케팅팀';

    SQL 기초 퀘스트 — 상품·주문 테이블(store_products, store_orders)

    예시 데이터

    상품과 주문 정보를 담은 store_products, store_orders 테이블 임시 데이터는 다음과 같다.

    store_products

    product_id product_name unit_price
    1 랩톱 1200
    2 핸드폰 800
    3 타블렛 400

    store_orders

    order_id product_id quantity ordered_at
    101 1 2 2023-03-01
    102 2 1 2023-03-02
    103 3 5 2023-03-04

    풀이한 쿼리

    1. 모든 주문의 주문 ID와 주문된 상품의 이름을 나열하는 쿼리

    SELECT o.order_id,
       p.product_name
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id;

    2. 총 매출(unit_price * quantity 합)이 가장 높은 상품의 ID와 해당 상품의 총 매출을 가져오는 쿼리

    SELECT p.product_id,
       SUM(p.unit_price * o.quantity) AS total_revenue
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id
GROUP BY p.product_id
ORDER BY total_revenue DESC
LIMIT 1;

    3. 각 상품 ID별로 판매된 총 수량(quantity)을 계산하는 쿼리

    SELECT p.product_id,
       SUM(o.quantity) AS total_quantity
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id
GROUP BY p.product_id;

    4. 2023년 3월 3일 이후에 주문된 모든 상품의 이름을 나열하는 쿼리

    SELECT DISTINCT p.product_name
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id
WHERE o.ordered_at > '2023-03-03';

    5. 가장 많이 판매된 상품의 이름을 찾는 쿼리

    SELECT p.product_name,
       SUM(o.quantity) AS total_sold
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id
GROUP BY p.product_id, p.product_name
ORDER BY total_sold DESC
LIMIT 1;

    6. 각 상품 ID별로 평균 주문 수량을 계산하는 쿼리

    SELECT p.product_id,
       AVG(o.quantity) AS avg_quantity
FROM store_orders   o
JOIN store_products p ON o.product_id = p.product_id
GROUP BY p.product_id;

    7. 판매되지 않은 상품의 ID와 이름을 찾는 쿼리

    SELECT p.product_id,
       p.product_name
FROM store_products p
LEFT JOIN store_orders o ON p.product_id = o.product_id
WHERE o.order_id IS NULL;

    오늘의 회고와 내일 계획

    오늘 느낀 점

    오늘 문제들은 JOIN이 왜 필요한지, 그리고 GROUP BY와 집계 함수로 어떤 질문에 답할 수 있는지를 다시 떠올리게 해 줬다. 특히 “직원이 없는 부서”, “판매되지 않은 상품”처럼 LEFT JOIN + NULL 체크 패턴은 실제 서비스에서도 자주 쓰일 것 같아서 기억해 둘 만했다.

    단순히 정답만 맞추는 것보다, 각 쿼리가 “어떤 비즈니스 질문에 대한 답인지”를 같이 붙여 생각하니 쇼핑몰 프로젝트나 이후 포트폴리오에서 어떤 통계를 뽑을 수 있을지 아이디어가 조금씩 정리되는 느낌이었다.
