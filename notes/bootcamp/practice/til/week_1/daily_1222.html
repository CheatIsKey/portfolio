    8일차 개요 — “입력 검증 + 예외 처리”가 프로젝트 완성도를 좌우한다

    정리

    오늘은 Java로 구현한 콘솔 계산기 프로젝트 코드를 기준으로 정리했다. 단순히 연산만 되는 프로그램이 아니라, 사용자가 어떤 값을 입력하더라도 프로그램이 멈추지 않도록 입력 검증과 예외 처리를 넣었고, 결과를 List에 저장해 이력까지 확인할 수 있도록 구성했다.

    • 메뉴 선택(계산/이력/삭제/종료)을 반복하는 콘솔 프로그램 구조
    • InputMismatchException을 이용한 숫자 입력 검증
    • sqrt 단항 연산 처리 및 음수 입력 차단
    • /, %에서 분모 0 입력을 재입력 루프로 방지
    • List<Double>로 결과 이력 저장 및 출력

    Java 계산기 프로젝트 구조 — 메뉴 루프와 역할 분리

    코드 흐름

    전체 구조는 무한 루프(while) 안에서 메뉴를 출력하고, 사용자 선택에 따라 switch로 분기하는 방식이다. 입력과 계산 로직은 아래처럼 메서드로 분리되어 있어서, 예외 대응과 유지보수가 쉬워진다.

    구성 요소 역할 핵심 포인트
    inputNumber() 정수 입력 받기 숫자 외 입력을 예외로 잡고 재입력
    inputOperator() 연산자 입력 받기 지원하는 연산자만 통과(+,-,*,/,% ,^,sqrt)
    calculate() 연산 결과 반환 연산자별 분기, 잘못된 연산자면 예외 발생
    resultList 결과 이력 저장 계산 결과를 누적, 이력 조회/삭제 가능

    메인 루프 핵심 부분 

    while (true) {
    System.out.println("=== 계산기 메뉴 ===");
    System.out.println("1. 계산하기");
    System.out.println("2. 계산 이력 보기");
    System.out.println("3. 이력 지우기");
    System.out.println("0. 종료");

    int select = inputNumber(scanner, "선택: ");

    switch (select) {
        case 1:
            // 계산하기
            break;
        case 2:
            // 이력 보기
            break;
        case 3:
            // 이력 지우기
            break;
        case 0:
            // 종료
            return;
        default:
            System.out.println("지원하지 않는 메뉴입니다.");
    }
}

    입력 검증과 예외 처리 — 프로그램이 멈추지 않게 만드는 장치

    숫자 입력 검증(InputMismatchException)

    Scanner.nextInt()는 숫자가 아닌 값이 들어오면 InputMismatchException이 발생한다. 이 프로젝트에서는 예외를 잡고, 잘못 입력된 토큰을 scanner.next()로 버린 뒤 다시 입력받도록 했다. 

    private static int inputNumber(Scanner scanner, String msg) {
    while (true) {
        System.out.print(msg);

        try {
            return scanner.nextInt();
        } catch (InputMismatchException e) {
            System.out.println("숫자를 입력해주세요.");
            scanner.next(); // 잘못 들어온 토큰 소비
        }
    }
}
    • 핵심은 “예외가 나도 프로그램이 종료되지 않게” 루프를 유지하는 것이다.
    • scanner.next()를 호출하지 않으면 같은 잘못된 입력이 계속 남아서 무한히 예외가 난다.

    분모 0 입력 방지(/, %)

    /와 %는 분모가 0이면 계산 자체가 불가능하므로, 0을 입력하면 다시 입력받도록 재입력 루프를 구성했다. 

    if ((operator.equals("/") || operator.equals("%")) && num2 == 0) {
    System.out.println("0은 분모로 입력할 수 없습니다. 다시 입력해주세요.");

    while (true) {
        num2 = inputNumber(scanner, "두 번째 숫자: ");
        if (num2 != 0) break;
        System.out.println("0은 분모로 입력할 수 없습니다. 다시 입력해주세요.");
    }
}
    • 예외를 던져서 종료하는 대신, 사용자 입력을 유도해 정상 흐름으로 복귀시키는 방식이다.
    • 이런 검증은 “예외 처리”라기보다 사전 검증에 가깝다.

    sqrt 처리(단항 연산)와 음수 방지

    sqrt는 두 번째 숫자가 필요 없는 단항 연산이라서, 연산자가 sqrt인 경우는 별도로 처리했다. 또한 음수의 제곱근은 실수 범위에서 계산할 수 없다고 안내하고, 메뉴로 되돌아가도록 했다. 

    if (operator.equals("sqrt")) {
    if (num1 < 0) {
        System.out.println("음수의 제곱근은 실수 범위에서 계산할 수 없습니다.");
        continue;
    }

    System.out.printf("결과: %d의 제곱근 = %.2f %n", num1, Math.sqrt(num1));
    System.out.println();
    resultList.add(Math.sqrt(num1));
    continue;
}

    연산 처리(calculate) — switch로 연산자별 결과 반환

    calculate 메서드 역할

    실제 계산은 calculate(num1, operator, num2)에서 처리한다. 여기서 sqrt는 메인 로직에서 이미 분기 처리되기 때문에, calculate는 이항 연산 중심으로 동작한다. 

    private static double calculate(int num1, String operator, int num2) {
    switch (operator) {
        case "+":
            return num1 + num2;
        case "-":
            return num1 - num2;
        case "*":
            return num1 * num2;
        case "/":
            return (double) num1 / num2;
        case "%":
            return num1 % num2;
        case "^":
            return Math.pow(num1, num2);
        default:
            throw new IllegalArgumentException("계산 과정에서 문제가 발생했습니다.");
    }
}
    • /는 정수 나눗셈이 아니라 실수 결과를 위해 (double) 캐스팅을 사용했다.
    • ^는 Math.pow를 사용해 제곱 연산을 처리했다.
    • 기본값에서 IllegalArgumentException을 던져, 예상치 못한 연산자 유입을 방지한다.

    결과 이력 관리(List) — 저장, 조회, 삭제

    List를 쓴 이유

    이력은 “계산이 발생한 순서대로 쌓이고, 그 순서대로 출력”하는 구조다. 그래서 순서를 보장하는 List가 자연스럽고, 결과 타입을 통일하기 위해 List<Double>로 관리했다. 

    List<Double> resultList = new ArrayList<>();

// 계산 결과 저장
resultList.add(result);

// 이력 출력
for (int i = 0; i < resultList.size(); i++) {
    System.out.printf("%d. %.2f %n", i + 1, resultList.get(i));
}

// 이력 삭제
resultList.clear();
    • 이력이 비어있을 때는 “계산 이력이 없습니다.”를 출력한다.
    • 이력 출력은 인덱스를 이용해 번호를 붙여 보여준다.

    오늘 체크한 개선 포인트 — 다음 리팩토링 방향

    정리

    • case 3(이력 지우기)에서 사용자 안내 메시지를 추가하면 UX가 좋아진다.
    • 연산자 처리 방식을 enum 또는 Map으로 바꾸면 조건 분기와 중복이 줄어든다.
    • 입력 타입 확장(정수 → 실수)을 고려하면 입력 메서드가 double 기반으로 바뀌어야 한다.
    • 결과 이력에 “식”도 함께 저장하려면 List<Double> 대신 DTO(예: ExpressionResult)를 쓰는 방식이 더 깔끔하다.
