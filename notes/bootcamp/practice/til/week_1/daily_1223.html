    8일차 개요 — “안 멈추는 프로그램”을 만드는 핵심 개념들

    정리

    오늘은 Java로 구현한 콘솔 계산기 프로젝트 코드를 기반으로, 프로그램이 예외 상황에서도 멈추지 않게 만드는 방법을 정리했다. 추가로 예외/예외 처리, Optional, 컬렉션, 제네릭 개념을 함께 묶어서, 나중에 다시 봤을 때 바로 떠올릴 수 있게 핵심만 정리했다.

    • 계산기: 입력 검증(숫자/연산자), 분모 0 방지, sqrt 처리, 결과 이력 저장
    • 예외: 검사/비검사 예외 차이, try-catch, throws, finally
    • Optional: null 안전 처리 기본 패턴
    • 컬렉션: List/Set/Map 특징과 사용 상황
    • 제네릭: 타입 안정성, 컴파일 타임 오류 방지

    Java 계산기 프로젝트 — 입력 검증, 예외 처리, 이력 관리

    프로젝트 핵심 기능

    기능 구현 방식 포인트
    메뉴 반복 while + switch 종료(0) 전까지 계속 실행
    숫자 입력 검증 try-catch(InputMismatchException) 숫자 외 입력 시 재입력
    연산자 입력 검증 허용 연산자 체크 +, -, *, /, %, ^, sqrt만 허용
    분모 0 방지 사전 검증 + 재입력 루프 /, %에서 0 입력 금지
    sqrt 처리 단항 연산 분기 음수는 거부, 결과 저장
    이력 관리 List<Double> + add/get/clear 조회/삭제 메뉴 제공

    입력 검증 메서드(숫자) 

    private static int inputNumber(Scanner scanner, String msg) {
    while (true) {
        System.out.print(msg);
        try {
            return scanner.nextInt();
        } catch (InputMismatchException e) {
            System.out.println("숫자를 입력해주세요.");
            scanner.next(); // 잘못 입력된 토큰 제거
        }
    }
}

    입력 검증 메서드(연산자) 

    private static String inputOperator(Scanner scanner, String msg) {
    while (true) {
        System.out.print(msg);
        String operator = scanner.next();

        if (operator.equals("+") || operator.equals("-") || operator.equals("*") || operator.equals("/")
                || operator.equals("%") || operator.equals("^") || operator.equals("sqrt")) {
            return operator;
        }
        System.out.println("지원하지 않는 연산자입니다.");
    }
}

    계산 메서드(calculate) — 예상 밖 입력은 예외로 차단 

    private static double calculate(int num1, String operator, int num2) {
    switch (operator) {
        case "+": return num1 + num2;
        case "-": return num1 - num2;
        case "*": return num1 * num2;
        case "/": return (double) num1 / num2;
        case "%": return num1 % num2;
        case "^": return Math.pow(num1, num2);
        default:
            throw new IllegalArgumentException("계산 과정에서 문제가 발생했습니다.");
    }
}

    예외와 예외 처리 — 검사 예외 vs 비검사 예외

    개념 정리

    구분 대표 예시 특징 대응 방법
    검사(Checked) 예외 IOException, SQLException 컴파일 단계에서 처리 강제 try-catch 또는 throws
    비검사(Unchecked) 예외 NullPointerException, IllegalArgumentException 런타임에 발생, 컴파일러가 강제하지 않음 사전 검증 + 필요한 경우 catch

    try-catch 기본 패턴 

    try {
    int n = scanner.nextInt();
} catch (InputMismatchException e) {
    System.out.println("숫자를 입력해주세요.");
    scanner.next();
}

    검사 예외(Checked) 예시 — throws / try-catch 

    // 컴파일러가 예외 처리를 강제하는 형태(개념 예시)
void readFile() throws java.io.IOException {
    // 파일 읽기 로직
}

    체크 예외는 “반드시 처리해야 컴파일이 되는 예외”라서, 호출하는 쪽에서 try-catch로 잡거나, 메서드 시그니처에 throws로 위임해야 한다.

    비검사 예외(Unchecked) 예시 — 사전 검증이 우선 

    // 잘못된 인자 유입을 방지(사전 검증)
if (num2 == 0) {
    throw new IllegalArgumentException("분모는 0이 될 수 없습니다.");
}

    비검사 예외는 “프로그램 논리/입력 자체가 잘못된 경우”가 많아서, try-catch로 무조건 덮기보다 원인을 제거하는 방식(검증/설계)이 우선이다.

    Optional — null 안전 처리를 위한 기본 도구

    왜 쓰는가

    • null 반환/NullPointerException을 줄이기 위한 의도 표현
    • “값이 없을 수도 있다”를 타입 수준에서 드러내기
    • 있으면 처리/없으면 대체값/없으면 예외 같은 패턴을 깔끔하게 작성

    자주 쓰는 패턴 예시 

    import java.util.Optional;

Optional<String> nick = Optional.ofNullable(getNickname());

// 값이 없으면 기본값
String value1 = nick.orElse("guest");

// 값이 없으면 지연 계산(람다)
String value2 = nick.orElseGet(() -> "guest");

// 값이 없으면 예외
String value3 = nick.orElseThrow(() -> new IllegalStateException("닉네임이 없습니다."));

// 값이 있으면 실행
nick.ifPresent(v -> System.out.println(v));

    Optional은 만능이 아니라, “없을 수 있는 값”을 명확하게 다루기 위한 도구로 쓰는 게 핵심이다.

    컬렉션 — List/Set/Map 핵심 정리

    컬렉션 선택 기준

    종류 특징 언제 쓰는가
    List 순서 O, 중복 O 이력/목록처럼 “순서가 중요”할 때
    Set 순서 X(구현체에 따라 다름), 중복 X 중복 제거, 유일 값 관리
    Map Key-Value 빠른 조회(키 기반), 카운팅/인덱싱

    계산기 코드에 적용된 컬렉션(List) 

    List<Double> resultList = new ArrayList<>();

resultList.add(result);

for (int i = 0; i < resultList.size(); i++) {
    System.out.printf("%d. %.2f %n", i + 1, resultList.get(i));
}

resultList.clear();

    계산 결과는 “계산이 수행된 순서대로” 쌓이고 출력되기 때문에 List 선택이 자연스럽다.

    제네릭(Generics) — 타입 안정성을 확보하는 문법

    제네릭을 쓰는 이유

    • 컬렉션에 들어가는 타입을 고정해서 컴파일 타임에 오류를 잡는다
    • 캐스팅을 줄이고 코드 가독성과 안정성을 높인다
    • 재사용 가능한 클래스/메서드를 만들 수 있다

    컬렉션 제네릭 예시 

    List<Double> list = new ArrayList<>();
list.add(3.14);

// 컴파일 단계에서 타입 오류를 막아줌
// list.add("문자열"); // 불가능

    제네릭 메서드 예시(개념) 

    public static <T> T first(List<T> list) {
    return list.get(0);
}

    제네릭은 “아무 타입이나 받되, 그 타입을 유지한 채로 안전하게 처리”하는 게 핵심이다.

    오늘 느낀 점

    계산기 프로젝트는 기능 자체는 단순하지만, 입력 검증/예외 처리/컬렉션 저장을 붙이는 순간 “서비스가 멈추지 않게 만드는 기본기”가 그대로 드러난다. 특히 예외를 무조건 잡는 것보다, 사전 검증으로 예외를 발생시키지 않는 구조가 더 중요하다는 걸 다시 확인했다.
