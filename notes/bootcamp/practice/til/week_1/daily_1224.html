    내일배움캠프 Spring 백엔드 9일차 TIL — SQL 퀘스트(Lv1~Lv4)

    작성일 2025-12-23 · 분류: 부트캠프, TIL, SQL, JOIN, GROUP BY

    9일차 개요 — 조건 조회부터 JOIN/집계 심화까지

    정리

    오늘은 SQL 퀘스트를 중심으로 LIKE 조건, 날짜 단위 집계, LEFT JOIN + NULL 처리, 고객/부서/부서별 최고값 같은 그룹 집계 문제를 풀었다. “결과를 내는 쿼리”에서 끝내지 않고, NULL 처리와 정렬 기준이 결과에 미치는 영향까지 함께 정리했다.

    • Lv1: 성씨(김)로 시작하는 유저 수 집계
    • Lv2: 날짜별 평균 포인트 집계(반올림)
    • Lv3: users는 있지만 point_users가 없는 유저를 0점으로 처리
    • Lv4: 고객별 주문 건수/총합, 국가별 최고 고객
    • Lv4: 부서별 최고 연봉자, 평균 연봉이 가장 높은 부서

    SQL 퀘스트 Lv1 — 데이터 속 김서방 찾기(users)

    데이터 설명

    컬럼 설명 타입
    user_id 익명화된 유저 아이디 varchar(255)
    created_at 아이디 생성 날짜 timestamp
    updated_at 정보 업데이트 날짜 timestamp
    name 익명화된 이름 varchar(255)
    email 이메일 varchar(255)

    풀이한 쿼리

    Lv1. “김”씨 성을 가지고 있는 교육생의 수(name_cnt)

    SELECT COUNT(*) AS name_cnt
FROM users
WHERE name LIKE '김%';

    SQL 퀘스트 Lv2 — 날짜별 획득 포인트 조회하기(point_users)

    데이터 설명

    컬럼 설명 타입
    point_user_id point_users 행을 구별하는 키 varchar(255)
    created_at 기록 생성 날짜 timestamp
    updated_at 정보 업데이트 날짜 timestamp
    user_id 익명화된 유저 아이디 varchar(255)
    point 보유 포인트 int

    풀이한 쿼리

    Lv2. 날짜별 평균 포인트(average_points)를 반올림하여 조회

    SELECT DATE(created_at) AS created_at,
       ROUND(AVG(point)) AS average_points
FROM point_users
GROUP BY DATE(created_at)
ORDER BY DATE(created_at);

    SQL 퀘스트 Lv3 — 이용자의 포인트 조회하기(users + point_users)

    요구사항 정리

    • users에는 있지만 point_users에는 없는 유저가 존재한다.
    • 포인트가 없는 유저는 0점으로 처리한다.
    • 포인트 기준 내림차순 정렬한다.

    풀이한 쿼리 

    SELECT u.user_id AS user_id,
       u.email AS email,
       COALESCE(pu.point, 0) AS point
FROM users u
LEFT JOIN point_users pu ON u.user_id = pu.user_id
ORDER BY COALESCE(pu.point, 0) DESC;

    SQL 퀘스트 Lv4 — 단골 고객님 찾기(Customers + Orders)

    예시 데이터

    주문 정보를 담은 Orders 테이블 임시 데이터는 다음과 같다.

    OrderID CustomerID OrderDate TotalAmount
    10112024-01-01150
    10222024-01-03200
    10312024-01-04300
    10432024-01-0450
    10522024-01-0580
    10642024-01-06400

    고객 정보를 담은 Customers 테이블 임시 데이터는 다음과 같다.

    CustomerID CustomerName Country
    1AliceUSA
    2BobUK
    3CharlieUSA
    4DavidCanada

    풀이한 쿼리

    Lv4-1. 고객별 주문 건수와 총 주문 금액(주문 없는 고객 포함)

    SELECT c.CustomerName AS CustomerName,
       COUNT(o.OrderID) AS OrderCount,
       COALESCE(SUM(o.TotalAmount), 0) AS TotalSpent
FROM Customers c
LEFT JOIN Orders o ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.CustomerName
ORDER BY c.CustomerName;

    Lv4-2. 나라별 총 주문 금액이 가장 높은 고객과 총 주문 금액

    SELECT t.Country AS Country,
       t.CustomerName AS Top_Customer,
       t.total_spent AS Top_Spent
FROM (
    SELECT c.CustomerID,
           c.CustomerName,
           c.Country,
           COALESCE(SUM(o.TotalAmount), 0) AS total_spent
    FROM Customers c
    LEFT JOIN Orders o ON c.CustomerID = o.CustomerID
    GROUP BY c.CustomerID, c.CustomerName, c.Country
) t
JOIN (
    SELECT x.Country,
           MAX(x.total_spent) AS top_spent
    FROM (
        SELECT c2.CustomerID,
               c2.Country,
               COALESCE(SUM(o2.TotalAmount), 0) AS total_spent
        FROM Customers c2
        LEFT JOIN Orders o2 ON c2.CustomerID = o2.CustomerID
        GROUP BY c2.CustomerID, c2.Country
    ) x
    GROUP BY x.Country
) m
ON m.Country = t.Country AND m.top_spent = t.total_spent
ORDER BY t.Country;

    SQL 퀘스트 Lv4 — 가장 높은 월급을 받는 직원은?(Employees)

    예시 데이터

    직원 정보를 담은 Employees 테이블 임시 데이터는 다음과 같다.

    EmployeeID Name Department Salary ManagerID
    1AliceHR70000NULL
    2BobIT900001
    3CharlieIT800002
    4DavidIT850002
    5EveHR750001
    6FrankFinance95000NULL
    7GraceFinance800006
    8HeidiIT950002

    풀이한 쿼리

    Lv4-3. 각 직원 + 부서 내 최고 연봉자의 이름/연봉 조회

    SELECT e1.Name,
       e1.Department,
       e1.Salary,
       e2.Name AS Top_Earner,
       e2.Salary AS Top_Salary
FROM Employees e1
JOIN Employees e2 ON e1.Department = e2.Department
WHERE e2.Salary = (
    SELECT MAX(Salary)
    FROM Employees e3
    WHERE e3.Department = e1.Department
);

    Lv4-4. 부서별 평균 월급이 가장 높은 부서와 평균 월급

    SELECT Department,
       AVG(Salary) AS Avg_Salary
FROM Employees
GROUP BY Department
HAVING AVG(Salary) = (
    SELECT MAX(Avg_Salary)
    FROM (
        SELECT AVG(Salary) AS Avg_Salary
        FROM Employees
        GROUP BY Department
    ) AS subquery
);

    오늘의 회고와 다음 학습 포인트

    오늘 느낀 점

    오늘 퀘스트는 “JOIN을 할 줄 안다” 수준을 넘어서, LEFT JOIN + NULL 처리(COALESCE), 그리고 집계 결과를 기준으로 다시 필터링하는 패턴이 핵심이었다. 특히 “정렬 기준이 NULL을 포함할 때 결과가 어떻게 달라지는지”를 직접 확인하면서, 결과 요구사항을 정확히 읽는 습관이 중요하다는 걸 느꼈다.
