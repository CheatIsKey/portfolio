    cv와 iv 정리

    정의

    구분 의미 접근 방식 저장/공유 감각
    cv(Class Variable) static 필드로 선언되는 클래스 변수 클래스명으로 접근 가능 클래스 단위로 공유
    iv(Instance Variable) 인스턴스가 가지는 멤버 변수 인스턴스 생성 후 접근 인스턴스마다 개별 값

    예시 코드 

    class Counter {
  static int cv = 0;  // 클래스 변수
  int iv = 0;         // 인스턴스 변수

  void increase() {
    cv++;
    iv++;
  }
}

public class Main {
  public static void main(String[] args) {
    Counter a = new Counter();
    Counter b = new Counter();

    a.increase();
    a.increase();
    b.increase();

    System.out.println(Counter.cv); // 3 (공유)
    System.out.println(a.iv);       // 2 (개별)
    System.out.println(b.iv);       // 1 (개별)
  }
}

    멤버 변수 초기화 순서

    큰 흐름

    초기화는 크게 클래스 초기화가 먼저, 그 다음 인스턴스 초기화가 진행된다. 그리고 각 단계 안에서 기본값 → 명시적 초기화 → 블록/생성자 순으로 흐름이 이어진다.

    단계 순서 무슨 의미인가
    클래스 초기화(cv) 기본값 → 명시적 초기화 → 클래스 초기화 블록(static) 클래스가 로딩될 때 1회 진행
    인스턴스 초기화(iv) 기본값 → 명시적 초기화 → 인스턴스 초기화 블록 → 생성자 인스턴스 생성할 때마다 진행

    예시 코드로 확인 

    class InitOrder {
  static int cv = 1;

  int iv = 10;

  static {
    cv = 2;
    System.out.println("static block: cv=" + cv);
  }

  {
    iv = 20;
    System.out.println("instance block: iv=" + iv);
  }

  InitOrder() {
    iv = 30;
    System.out.println("constructor: iv=" + iv);
  }
}

public class Main {
  public static void main(String[] args) {
    System.out.println("start main");
    new InitOrder();
    new InitOrder();
  }
}
    • static 블록은 클래스 로딩 시점에 1회 실행된다
    • 인스턴스 블록과 생성자는 인스턴스 생성할 때마다 실행된다

    클래스 관계: 상속과 포함

    핵심 기준

    관계 설명 판단 문장 예시
    상속 부모의 기능을 재사용하고 확장 is-a Dog is an Animal
    포함 클래스 내부에 참조형 변수를 둬서 조립 has-a Car has an Engine

    예시 코드 

    // 상속(is-a)
class Animal {
  void move() {}
}
class Dog extends Animal {
  void bark() {}
}

// 포함(has-a)
class Engine {
  void start() {}
}
class Car {
  private final Engine engine = new Engine();
  void run() {
    engine.start();
  }
}

    super()가 생성자 첫 줄에 오는 이유

    핵심 포인트

    • 자식 객체 안에는 부모 영역이 포함되기 때문에, 부모 초기화가 먼저 끝나야 자식 초기화가 안전하다
    • 그래서 생성자에서 super() 호출은 반드시 첫 줄이어야 한다
    • 명시하지 않으면 컴파일러가 기본 super() 호출을 자동으로 넣는다

    Singleton과 객체 반환 메서드

    Singleton 기본 형태 

    class Singleton {
  private static final Singleton INSTANCE = new Singleton();

  private Singleton() {}

  public static Singleton getInstance() {
    return INSTANCE;
  }
}
    • private 생성자로 외부에서 new를 막는다
    • 클래스 내부에서 인스턴스를 1개 만들어 두고, 같은 인스턴스만 반환한다
    • 이런 형태의 getInstance는 실무에서 정적 팩터리 메서드로 자주 부른다

    다형성 정리

    정의

    다형성은 참조변수 하나로 여러 종류의 객체를 다루는 것이다. 즉, 조상 타입 참조변수로 자손 객체를 참조할 수 있다. 

    class Animal { void sound() {} }
class Dog extends Animal { void sound() { System.out.println("bow"); } }
class Cat extends Animal { void sound() { System.out.println("meow"); } }

public class Main {
  public static void main(String[] args) {
    Animal a1 = new Dog();
    Animal a2 = new Cat();
    a1.sound(); // Dog 동작
    a2.sound(); // Cat 동작
  }
}

    열거형(Enum) 정리

    Enum 특징

    • 열거형은 상수들의 모음을 타입으로 묶어 관리한다
    • 필드/생성자/메서드를 함께 둘 수 있어, 의미 있는 값을 가진 상수 집합을 만들기 좋다
    • Enum 생성자는 외부에서 호출할 수 없고, 암묵적으로 private다

    예시 코드 

    import java.util.Arrays;

public enum InputMenu {
  CALCULATE(1),
  PRINT_RESULT(2),
  REMOVE_FIRST_RESULT(3),
  EXIT(0);

  private final int menuNumber;

  InputMenu(int menuNumber) {
    this.menuNumber = menuNumber;
  }

  public static InputMenu findMenuByNumber(int menuNumber) {
    return Arrays.stream(InputMenu.values())
      .filter(x -> x.menuNumber == menuNumber)
      .findFirst()
      .orElseThrow(() -> new IllegalArgumentException("메뉴 번호가 올바르지 않습니다."));
  }
}
