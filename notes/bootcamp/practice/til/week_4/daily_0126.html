    내일배움캠프 Spring 백엔드 29일차 TIL — 논리적 사고와 플로우차트 설계

    작성일 2026-01-22 · 분류: 부트캠프, TIL, Flowchart, Logic

    29일차 개요

    오늘 목표

    • 코드를 작성하기 전에 로직의 흐름을 먼저 설계하는 습관을 기른다
    • 순서도(Flowchart)를 통해 조건과 반복의 분기점을 시각화한다
    • 명세서(입출력, 조건, 흐름)를 기반으로 논리적 오류를 사전에 차단한다

    1. 1부터 10까지 출력

    입력/출력 정리

    • 입력: 없음
    • 출력: 1부터 10까지 순서대로 출력
    • 상태: 카운터 변수(cnt)

    조건 정리

    • 종료 조건: cnt가 10을 초과하는 순간 종료
    • 분기 조건: cnt >= 11 조건을 달성하는 경우
    • 예외/에러: 없음 (정상 범위 내 동작)

    흐름 설계

    1. 초기화: 시작할 때 cnt를 1로 설정한다.
    2. 반복 구간:
      • cnt가 10을 초과하는지 검사한다.
      • Yes (종료): 프로그램을 종료한다.
      • No (진행): 현재 cnt 값을 출력한다.
      • cnt를 1 증가시키고 다시 검사 단계로 돌아간다.
    3. 종료: 1부터 10까지 출력이 완료된다.

    2. 구구단 출력 (특정 단)

    입력/출력 정리

    • 입력: 정수 input
    • 출력: 입력 받은 input에 해당하는 구구단
    • 상태: 없음 (임시 변수만 사용)

    조건 정리

    • 종료 조건: 0을 입력받는 순간
    • 분기 조건:
      • 입력한 값이 정수가 아닌 경우
      • 입력한 값이 0인 경우
      • 반복문에서 num이 9가 되는 경우
    • 예외 처리: 정수가 아닐 때 "정수를 입력해주세요" 출력

    흐름 설계

    1. 입력 단계: 사용자로부터 값을 입력받는다.
    2. 검증 단계:
      • 값이 정수인지 확인한다. (아니면 에러 메시지 후 재입력)
      • 값이 0인지 확인한다. (맞으면 프로그램 종료)
    3. 구구단 출력 반복:
      • num을 1로 초기화한다.
      • 구구단을 출력하고 num이 9인지 검사한다.
      • 9가 아니라면 num을 1 증가시키고 반복한다.
      • 9라면 출력을 마치고 다시 입력 단계(1번)로 돌아간다.

    3. 팩토리얼 계산

    입력/출력 정리

    • 입력: 양의 정수 n
    • 출력: 팩토리얼 n의 결과값
    • 상태: 누적곱 변수 multi

    조건 정리

    • 종료 조건: (n - i) 값이 1이 되는 순간
    • 분기 조건: 정수 여부, 0 여부, 음수 여부, 계산 완료 여부
    • 예외 처리:
      • 정수가 아닐 때: "정수를 입력해주세요."
      • 음수일 때: "음수는 입력할 수 없습니다."

    흐름 설계

    1. 초기화: multi 변수를 1로 설정한다.
    2. 검증 단계:
      • n을 입력받고 정수인지, 음수인지 검사한다. (실패 시 에러 출력 후 재입력)
      • n이 0이면 1을 출력하고 종료한다.
    3. 계산 반복:
      • i = 0으로 초기화한다.
      • multi에 (n - i)를 곱하여 누적한다.
      • (n - i)가 1이 되면 반복을 멈추고 결과를 출력한다.
      • 아니라면 i를 1 증가시키고 계속 곱한다.

    4. 성적 등급 변환기

    입력/출력 정리

    • 입력: 성적 등급 문자열 grade (A, B, C, D, F)
    • 출력: 해당 등급의 정수 점수 구간
    • 상태: 없음

    조건 정리

    • 종료 조건: 점수 구간을 정상적으로 출력하거나, 잘못된 값을 입력받은 경우
    • 분기 조건: grade가 A, B, C, D, F 중 어디에 해당하는가

    흐름 설계

    1. 입력: grade를 입력받고 대문자로 변환한다.
    2. 조건 분기 (Switch/If 구조):
      • A: "90-100" 출력
      • B: "80-89" 출력
      • C: "70-79" 출력
      • D: "60-69" 출력
      • F: "0-59" 출력
      • 그 외: "성적 등급에 없는 값이 입력되었습니다." 출력
    3. 종료: 결과를 출력한 후 프로그램을 즉시 종료한다.

    오늘의 회고

    오늘 느낀 점

    • 코딩보다 설계가 먼저라는 말을 체감했다. 흐름을 그림으로 그리니 예외 상황(음수, 0, 문자 입력 등)이 한눈에 보였다.
    • 조건문이 복잡해질 때, switch 문이나 다중 if 문 중 무엇을 쓸지 결정하는 기준이 명확해졌다.
    • 앞으로 복잡한 비즈니스 로직을 짤 때도 오늘처럼 흐름도를 먼저 그려보는 습관을 들여야겠다.
