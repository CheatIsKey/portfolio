    자료구조의 기초

    자료구조란?

    데이터를 담는 정리함과 같다.
    물건을 아무렇게나 쌓아두면 찾기 힘들듯, 데이터를 일정한 규칙에 따라 체계적으로 저장하고 관리하는 방식을 말한다.
    목적에 맞는 자료구조를 사용하면 검색 속도가 빨라지고 메모리를 효율적으로 사용할 수 있다.

    주요 자료구조 특징

    구분 종류 비유 (직관적 이해) 특징
    기본형 배열 책장에 번호 순서대로 꽂힌 책들 데이터를 순서대로 저장하는 방식이다. 인덱스로 빠르게 접근할 수 있다.
    연결 리스트 보물찾기 쪽지 (다음 장소만 알려줌) 데이터가 노드로 연결된 구조이다. 중간 위치에 데이터를 추가하거나 삭제하기 쉽다.
    스택 (Stack) 프링글스 통 (맨 위 과자부터 먹음) 나중에 넣은 것이 먼저 나오는 후입선출(LIFO) 방식이다.
    큐 (Queue) 버스 정류장 줄서기 먼저 넣은 것이 먼저 나오는 선입선출(FIFO) 방식이다.
    응용형 트리 회사 조직도, 컴퓨터 폴더 구조 데이터를 계층적 관계로 저장하며, 부모-자식 관계를 표현한다.
    그래프 지하철 노선도 데이터 간의 연결 관계를 표현하는 구조이다.
    해시 테이블 사물함 (번호키로 내용물 찾기) 키-값 쌍으로 데이터를 저장하여 검색 속도가 매우 빠르다.

    효율적인 탐색 기술

    단순히 처음부터 끝까지 찾는 것보다 훨씬 효율적인 알고리즘들을 정리했다.

    1. 이진 탐색 (Binary Search)

    • 개념: 정렬된 배열에서 탐색 범위를 절반씩 줄여가며 원하는 값을 찾는 알고리즘이다.
    • 비유: 업-다운(Up-Down) 숫자 맞추기 게임에서 중간값을 부르며 범위를 좁히는 것과 같다.
    • 특징: 속도가 O(log N)으로 매우 빠르지만, 데이터가 반드시 미리 정렬되어 있어야 한다는 전제 조건이 있다.

    2. 투 포인터 (Two Pointers)

    • 개념: 배열의 시작과 끝, 혹은 특정 위치에 두 개의 점(포인터)을 두고 조작하며 탐색하는 방법이다.
    • 비유: 애벌레가 몸의 앞부분과 뒷부분을 번갈아 움직이며 앞으로 나아가는 모습과 비슷하다.
    • 특징: 연속된 구간의 합을 구하거나 특정 조건을 만족하는 부분 수열을 찾을 때 주로 사용한다.

    정렬 알고리즘 비교

    버블 정렬 (Bubble Sort)

    • 개념: 인접한 두 요소를 반복적으로 비교하여 순서가 잘못되었다면 서로 자리를 바꾸는 방식이다.
    • 비유: 물속에서 무거운 돌은 가라앉고 가벼운 거품(Bubble)이 위로 올라오는 과정과 유사하다.
    • 특징: 시간 복잡도가 O(N²)이다. 구현은 간단하지만 데이터가 많아질수록 성능이 급격히 떨어진다.

    계수 정렬 (Counting Sort)

    • 개념: 데이터의 값을 직접 비교하지 않고, 각 데이터가 몇 번 등장하는지 세어서 정렬하는 알고리즘이다.
    • 비유: 투표 용지를 후보자별 바구니에 분류해 넣고, 나중에 바구니 순서대로 표를 세는 것과 같다.
    • 특징: 시간 복잡도가 O(N + K)로 매우 빠르다. 하지만 숫자의 범위(K)만큼 추가 메모리가 필요하므로, 데이터 값의 범위가 너무 크면 비효율적이다.

    안정 계수 정렬 (Stable Counting Sort)

    • 개념: 값이 같은 요소들이 있을 때, 정렬 전의 원래 순서가 정렬 후에도 유지되도록 하는 방식이다.
    • 비유: 시험 점수로 학생을 정렬할 때, 점수가 같다면 먼저 시험지를 제출한 순서(번호순)를 그대로 지켜주는 것이다.
    • 특징: 누적합 배열을 사용하여 요소의 최종 위치를 결정한다. 다중 조건으로 정렬해야 할 때 필수적으로 사용된다.

    2차원 배열과 행렬 변환

    알고리즘 문제에서 2차원 배열을 다룰 때 가장 많이 사용하는 두 가지 패턴을 정리했다. 글로만 보면 헷갈리기 쉬워 코드와 함께 익히는 것이 좋다.

    1. 델타 탐색 (Delta Search)

    현재 위치 (r, c)를 기준으로 상하좌우나 대각선 등 인접한 칸을 탐색하는 기법이다. 방향 배열(dr, dc)을 미리 정의해두고 반복문을 돌리면 코드가 훨씬 간결해진다. 

    // 상, 하, 좌, 우 방향 정의
int[] dr = {-1, 1, 0, 0};
int[] dc = {0, 0, -1, 1};

for (int i = 0; i < 4; i++) {
    int nr = r + dr[i];
    int nc = c + dc[i];
    
    // 배열 범위를 벗어나지 않는지 체크 필수
    if (nr >= 0 && nc >= 0 && nr < N && nc < M) {
        // 이동한 위치(nr, nc) 로직 수행
    }
}

    2. 행렬 회전 (Matrix Rotation)

    2차원 배열을 90도, 180도, 270도로 회전시키는 변환 작업이다. 인덱스가 어떻게 변하는지 규칙을 찾는 것이 핵심이다.

    • 90도 회전 (시계 방향): 행(Row) 정보가 열(Col)로 이동하고, 열 정보는 반전되어 행으로 이동한다.
      공식: (i, j) → (j, N-1-i)
    • 180도 회전: 배열이 완전히 뒤집힌 형태이다.
      공식: (i, j) → (N-1-i, M-1-j)
    • 270도 회전: 90도 회전을 반시계 방향으로 한 것과 같다.
      공식: (i, j) → (M-1-j, i) 
    // 90도 회전 예시 코드
int[][] rotate90(int[][] arr) {
    int n = arr.length;
    int m = arr[0].length;
    int[][] newArr = new int[m][n]; // 행과 열의 크기가 바뀜

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            // 공식 적용: (i, j) -> (j, n-1-i)
            newArr[j][n - 1 - i] = arr[i][j];
        }
    }
    return newArr;
}

    오늘의 회고

    오늘 느낀 점

    • 자료구조는 단순히 데이터를 저장하는 것이 아니라, '나중에 어떻게 꺼낼 것인가'를 고려하여 선택해야 한다는 점을 배웠다.
    • 계수 정렬이 빠르긴 하지만, 메모리 사용량과 데이터의 범위에 따라 사용이 불가능할 수도 있다는 한계를 명확히 알게 되었다.
    • 2차원 배열 회전은 머리로만 생각하면 인덱스가 꼬이기 쉬워서, 직접 (0,0) 위치가 어디로 이동하는지 그려보며 공식을 적용하는 것이 가장 정확했다.
