    자료구조의 기초

    자료구조란?

    데이터를 담는 정리함
    물건을 아무렇게나 쌓아두면 찾기 힘들듯, 데이터를 일정한 규칙에 따라 체계적으로 저장하고 관리하는 방식이다.
    목적에 맞는 자료구조를 쓰면 검색 속도가 빨라지고 메모리를 아낄 수 있다.

    주요 자료구조 비교

    구분 종류 비유 (직관적 이해) 특징
    기본형 배열 책장에 번호 순서대로 꽂힌 책들 순서대로 저장됨. 인덱스로 빠르게 접근 가능
    연결 리스트 보물찾기 쪽지 (다음 장소만 알려줌) 데이터가 노드로 연결됨. 중간 삽입/삭제가 쉬움
    스택 (Stack) 프링글스 통 (맨 위 과자부터 먹음) 후입선출(LIFO). 나중에 넣은 게 먼저 나옴
    큐 (Queue) 버스 정류장 줄서기 선입선출(FIFO). 먼저 온 사람이 먼저 나감
    응용형 트리 회사 조직도, 컴퓨터 폴더 구조 부모-자식의 계층적 관계 표현
    그래프 지하철 노선도 데이터(역) 간의 연결 관계 표현
    해시 테이블 사물함 (번호키로 내용물 찾기) Key-Value 쌍으로 저장하여 매우 빠른 검색

    효율적인 탐색 기술

    1. 이진 탐색 (Binary Search)

    • 개념: 정렬된 배열에서 탐색 범위를 절반씩 줄여가며 값을 찾는 알고리즘.
    • 비유: 업-다운(Up-Down) 숫자 맞추기 게임. (50을 불렀는데 "Down"이면 1~49만 보면 됨)
    • 특징: 속도가 매우 빠르다($O(\log N)$). 단, 데이터가 반드시 미리 정렬되어 있어야 한다.

    2. 투 포인터 (Two Pointers)

    • 개념: 배열의 양 끝이나 특정 위치에 2개의 점(포인터)을 두고 조작하며 탐색하는 방법.
    • 비유: 애벌레가 몸을 늘렸다 줄였다 하면서 앞으로 나아가는 모습.
    • 특징: 연속된 구간의 합을 구하거나 특정 조건을 만족하는 부분 수열을 찾을 때 효율적이다.

    정렬 알고리즘의 세계

    데이터를 순서대로 나열하는 방법은 여러 가지가 있고, 상황에 따라 효율이 극명하게 갈린다.

    버블 정렬 (Bubble Sort)

    • 개념: 인접한 두 요소를 비교해서 순서가 잘못됐으면 서로 자리를 바꾼다.
    • 비유: 무거운 돌은 가라앉고 가벼운 거품(Bubble)은 위로 올라오는 과정.
    • 특징: 구현이 쉽지만 느리다. 시간복잡도가 $O(N^2)$이라 데이터가 2배 늘면 시간은 4배 늘어난다.

    계수 정렬 (Counting Sort)

    • 개념: 값을 비교하지 않고, 각 숫자가 몇 번 나왔는지 세어서(Counting) 정렬한다.
    • 비유: 선거 개표할 때 후보자 이름 바구니에 표를 하나씩 던져놓고, 나중에 바구니별로 표 개수만 세는 것.
    • 특징: 비교 연산이 없어 매우 빠르다($O(N+K)$). 하지만 숫자의 범위만큼 추가 메모리(배열)가 필요해서, 숫자가 1억처럼 매우 크면 사용할 수 없다.

    안정 계수 정렬 (Stable Counting Sort)

    • 개념: 값이 같을 때, 원래 앞에 있던 데이터가 정렬 후에도 앞에 오도록 순서를 유지하는 방식.
    • 비유: 점수가 같은 학생들을 정렬할 때, 먼저 시험지를 낸 순서(번호순)를 그대로 유지해 주는 것.
    • 특징: 누적합 배열을 사용하여 위치를 계산한다(누적합[num] - 1). 엑셀에서 '이름순 정렬' 후 '점수순 정렬'을 할 때 이름 순서가 섞이지 않게 하려면 필수적이다.

    2차원 배열과 델타 탐색

    델타 탐색

    • 개념: 현재 위치 $(r, c)$를 기준으로 상하좌우나 대각선에 있는 인접한 칸을 탐색하는 기법.
    • 구현: dr = [-1, 1, 0, 0], dc = [0, 0, -1, 1] 같은 방향 배열을 만들어 반복문으로 처리한다.

    행렬 회전 (Matrix Rotation)

    알고리즘 문제에서 자주 나오지만 헷갈리기 쉬운 부분이다.
    기준: N x N (또는 N x M) 행렬, 좌표는 (i, j)

    회전 각도 공식 변환 이해하기 (규칙)
    90도 (시계) (i, j) → (j, N-1-i) 행(row)이 열(col)로 가고, 열은 뒤집혀서 행이 된다.
    180도 (i, j) → (N-1-i, M-1-j) 완전히 뒤집힘. 끝에서부터 역순으로 배치된다.
    270도 (i, j) → (M-1-j, i) 90도 회전을 반시계로 한 것과 같다. 열이 뒤집혀서 행이 되고, 행은 그대로 열이 된다.

    오늘의 회고

    오늘 느낀 점

    • 자료구조는 단순히 저장하는 게 아니라, '어떻게 꺼낼 것인가'를 고민하는 것이 핵심이었다
    • 계수 정렬(Counting Sort)이 빠르긴 하지만, 메모리 낭비가 심할 수 있다는 점(음수 처리 등)에서 만능은 아님을 알았다
    • 2차원 배열 회전 공식은 외우기보다 (0,0)이 어디로 가는지 직접 그려보면서 규칙을 찾는 게 더 빨랐다
