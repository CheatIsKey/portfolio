    자료구조의 기초

    자료구조란?

    데이터를 효율적으로 관리하는 규칙이다.
    데이터를 단순히 저장하는 것을 넘어, 빠르게 저장하고 검색하고 삭제하기 위해 만들어진 구조이다.
    상황에 맞는 자료구조를 선택하면 프로그램의 성능(속도, 메모리)을 획기적으로 개선할 수 있다.

    주요 자료구조 특징

    구분 종류 특징 및 용도
    기본형 배열 데이터를 순서대로 저장한다. 인덱스(Index)로 즉시 접근이 가능하다.
    연결 리스트 노드(Node)들이 줄줄이 연결된 구조이다. 데이터의 중간 삽입/삭제가 빈번할 때 유리하다.
    스택 (Stack) LIFO(Last In First Out). 뒤로가기 버튼이나 실행 취소(Undo) 기능에 쓰인다.
    큐 (Queue) FIFO(First In First Out). 프린터 대기열이나 선착순 처리에 사용된다.
    응용형 트리 계층적 구조를 표현한다. 파일 시스템이나 조직도 관리에 적합하다.
    그래프 데이터 간의 복잡한 연결 관계를 표현한다. 지도 경로 탐색이나 SNS 친구 관계에 쓰인다.
    해시 테이블 Key-Value 쌍으로 저장한다. 검색 속도가 O(1)에 가까워 매우 빠르다.

    효율적인 탐색 기술

    단순 반복문보다 훨씬 빠르게 데이터를 찾는 두 가지 핵심 기법을 코드로 정리했다.

    1. 이진 탐색 (Binary Search)

    • 개념: 정렬된 데이터에서 범위를 절반씩 좁혀가며 값을 찾는다.
    • 핵심 코드:
    while (start <= end) {
    int mid = (start + end) / 2; // 중간 위치 선정
    if (arr[mid] == target) return mid; // 찾음
    else if (arr[mid] < target) start = mid + 1; // 오른쪽 탐색
    else end = mid - 1; // 왼쪽 탐색
}
    • 효과: 10억 개의 데이터도 약 30번의 비교만으로 찾을 수 있을 만큼 빠르다(O(log N)).

    2. 투 포인터 (Two Pointers)

    • 개념: 두 개의 인덱스(포인터)를 사용하여 배열을 한 번만 순회하며 원하는 결과를 얻는다.
    • 핵심 코드 (구간 합 예시):
    int start = 0, end = 0, sum = 0;
while (start < N) {
    if (sum > target || end == N) sum -= arr[start++]; // 범위 축소
    else sum += arr[end++]; // 범위 확장
    if (sum == target) count++;
}
    • 효과: 이중 반복문(O(N²))을 써야 할 문제를 한 번의 반복(O(N))으로 해결할 수 있다.

    정렬 알고리즘 비교

    버블 정렬 (Bubble Sort)

    • 개념: 인접한 두 수를 비교해서 큰 수를 뒤로 보낸다.
    • 구현 로직: 
      for (int i = 0; i < N - 1; i++) {
    for (int j = 0; j < N - 1 - i; j++) {
        if (arr[j] > arr[j + 1]) swap(arr, j, j + 1);
    }
}
    • 특징: 구현이 매우 쉽지만, 시간 복잡도가 O(N²)이라 데이터가 많으면 사용할 수 없다.

    계수 정렬 (Counting Sort)

    • 개념: 값을 비교하지 않고, 각 숫자가 몇 번 등장했는지 개수를 세어 정렬한다.
    • 구현 로직: 
      int[] count = new int[MAX_VALUE + 1];
for (int num : arr) count[num]++; // 개수 세기

for (int i = 0; i < count.length; i++) {
    while (count[i]-- > 0) System.out.print(i + " "); // 출력
}
    • 특징: O(N)으로 매우 빠르지만, 숫자의 범위(MAX_VALUE)만큼 메모리를 추가로 써야 하는 단점이 있다.

    안정 계수 정렬 (Stable Counting Sort)

    • 개념: 값이 같을 때, 기존의 순서가 뒤섞이지 않도록 보장하는 계수 정렬이다.
    • 구현 로직: 단순 개수가 아니라 누적합(Prefix Sum) 배열을 만들어, 뒤에서부터 채워 넣는 방식을 사용한다.
    • 특징: 엑셀 정렬처럼 여러 기준(예: 점수 후 이름순)으로 정렬해야 할 때 반드시 필요하다.

    2차원 배열과 행렬 변환

    알고리즘 문제, 특히 구현 문제에서 필수적인 2차원 배열 제어 기술이다.

    1. 델타 탐색 (Delta Search)

    현재 위치 (r, c)를 기준으로 상하좌우를 탐색할 때, if문을 4번 쓰는 대신 방향 배열을 쓴다. 

    // 상, 하, 좌, 우
int[] dr = {-1, 1, 0, 0};
int[] dc = {0, 0, -1, 1};

for (int i = 0; i < 4; i++) {
    int nr = r + dr[i];
    int nc = c + dc[i];
    
    // 배열 범위를 벗어나지 않는지 확인 (필수)
    if (nr >= 0 && nc >= 0 && nr < N && nc < M) {
        // 이동한 곳의 로직 수행
    }
}

    2. 행렬 회전 (Matrix Rotation)

    2차원 배열을 회전시킬 때는 (0, 0) 좌표가 어디로 이동하는지 그려보면 공식을 유도하기 쉽다.

    • 90도 회전 (시계 방향)
      • 공식: newArr[j][N-1-i] = arr[i][j]
      • 설명: 행(Row) 정보가 열(Col)이 되고, 열 정보는 반전되어 행이 된다.
    • 180도 회전
      • 공식: newArr[N-1-i][M-1-j] = arr[i][j]
      • 설명: 배열을 180도 뒤집은 모양이다.
    // 90도 회전 전체 코드
public int[][] rotate(int[][] arr) {
    int n = arr.length;
    int m = arr[0].length;
    int[][] result = new int[m][n]; // 행열 크기 반전

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            result[j][n - 1 - i] = arr[i][j];
        }
    }
    return result;
}

    오늘의 회고

    오늘 느낀 점

    • 자료구조는 단순히 '저장'이 아니라 '나중에 어떻게 효율적으로 꺼낼지'를 고민하는 과정임을 깨달았다.
    • 말로만 듣던 정렬 알고리즘을 코드로 직접 짜보니, 시간 복잡도가 왜 O(N²)이고 O(N)인지 확실히 체감되었다.
    • 2차원 배열 회전은 머리로만 하려다 인덱스 지옥에 빠지기 쉬우니, 꼭 그림을 그려서 공식을 확인하는 습관을 들여야겠다.
