    내일배움캠프 Spring 백엔드 33일차 TIL — 알고리즘 실전 풀이 (완전탐색 & 2차원 배열)

    작성일 2026-01-28 · 분류: 부트캠프, TIL, Algorithm, Java

    33일차 개요

    오늘 목표

    • 반복되는 패턴을 처리하기 위한 나머지 연산(Modulo) 활용법을 익힌다.
    • 어제 학습한 델타 탐색을 2차원 배열 문제에 직접 적용해본다.

    1. 음악 테스트 응시자의 패턴

    서로 다른 길이의 패턴을 가진 3명의 응시자 중, 가장 높은 점수를 받은 사람을 찾는 문제이다. 문제의 핵심은 정답 배열의 길이가 응시자의 패턴 길이보다 훨씬 길 때 어떻게 처리하느냐에 있다.

    핵심 로직: 나머지 연산 (%)

    • 문제 상황: 정답(answers)의 길이는 5,000까지 가능하지만, 1번 응시자의 패턴은 4개, 2번은 6개로 짧다.
    • 해결 방법: 인덱스 i를 패턴의 길이로 나눈 나머지(i % length)를 사용하면, 배열을 무한히 반복하는 효과를 낼 수 있다.
    • 코드 예시: candidateOne[i % candidateOneLength]

    풀이 코드 (Java) 

    import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();

        // 1. 각 응시자의 찍기 패턴 정의
        String[] candidateOne = new String[]{"도", "레", "미", "파"};
        String[] candidateTwo = new String[]{"레", "레", "파", "파", "도", "도"};
        String[] candidateThree = new String[]{"미", "파", "미", "도", "레", "도"};

        int[] scores = new int[3]; // 점수 저장용 배열
        int N = Integer.parseInt(br.readLine());

        // 반복 계산을 줄이기 위해 길이 미리 저장
        int candidateOneLength = candidateOne.length;
        int candidateTwoLength = candidateTwo.length;
        int candidateThreeLength = candidateThree.length;

        StringTokenizer st = new StringTokenizer(br.readLine());

        // 2. 정답 채점 (나머지 연산으로 패턴 반복 처리)
        for (int i = 0; i < N; i++) {
            String s = st.nextToken();

            if (candidateOne[i % candidateOneLength].equals(s)) {
                scores[0]++;
            }
            if (candidateTwo[i % candidateTwoLength].equals(s)) {
                scores[1]++;
            }
            if (candidateThree[i % candidateThreeLength].equals(s)) {
                scores[2]++;
            }
        }

        // 3. 최대로 많이 맞춘 개수(max) 찾기
        int max = 0;
        for (int score : scores) {
            if (score > max) max = score;
        }

        // 4. 최댓값과 같은 점수를 가진 사람 모두 출력 (오름차순)
        for (int i = 0; i < scores.length; i++) {
            if (scores[i] == max) sb.append(i + 1).append(" ");
        }

        System.out.println(sb);
    }
}

    2. 실험실 산성 반응 테스트

    2차원 배열에서 특정 지점을 선택했을 때, 상하좌우로 퍼지는 값들의 합이 최대가 되는 경우를 찾는 문제이다. 어제 정리했던 델타 탐색(Delta Search)을 그대로 활용할 수 있는 정석적인 문제였다.

    핵심 로직: 방향 배열과 범위 체크

    • 방향 배열 정의: 상(-1,0), 하(1,0), 좌(0,-1), 우(0,1)의 변화량을 배열로 미리 선언한다.
    • 완전 탐색: 2중 for문으로 모든 비커의 위치 (r, c)를 순회하며 기준점으로 잡는다.
    • 범위 검증: 인덱스가 배열 범위를 벗어나지 않는지(row < 0 등) 반드시 확인해야 IndexOutOfBounds 예외를 막을 수 있다.

    풀이 코드 (Java) 

    import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    // 상하좌우 이동을 위한 델타 배열
    static final int[] ROW = {-1, 0, 0, 1};
    static final int[] COL = {0, -1, 1, 0};

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();

        int T = Integer.parseInt(br.readLine());

        for (int tc = 1; tc <= T; tc++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int P = Integer.parseInt(st.nextToken());
            int Q = Integer.parseInt(st.nextToken());

            int[][] testBoard = new int[P][Q];

            // 1. 2차원 배열 입력 받기
            for (int r = 0; r < P; r++) {
                st = new StringTokenizer(br.readLine());
                for (int c = 0; c < Q; c++) {
                    testBoard[r][c] = Integer.parseInt(st.nextToken());
                }
            }

            int maxSum = 0;

            // 2. 모든 좌표를 완전 탐색
            for (int r = 0; r < P; r++) {
                for (int c = 0; c < Q; c++) {
                    int sum = testBoard[r][c]; // 기준 위치의 값 포함

                    // 3. 4방향(상하좌우) 탐색
                    for (int k = 0; k < 4; k++) {
                        int row = r + ROW[k];
                        int col = c + COL[k];

                        // 배열 범위를 벗어나면 스킵 (필수 예외 처리)
                        if (row < 0 || row >= P || col < 0 || col >= Q) continue;

                        sum += testBoard[row][col];
                    }

                    // 최댓값 갱신
                    if (maxSum < sum) maxSum = sum;
                }
            }

            sb.append("#" + tc + ' ').append(maxSum).append("\n");
        }

        System.out.println(sb);
    }
}

    오늘의 회고

    문제 풀이 후기

    • 패턴 문제: 배열의 길이를 넘어설 때 % 연산자를 쓰면 인덱스 관리가 훨씬 쉬워진다는 점을 체감했다.
    • 2차원 배열: 어제 이론으로만 배운 델타 탐색을 코드로 옮기니, if문을 4번 쓰는 것보다 for문 하나로 처리하는 게 얼마나 깔끔한지 알 수 있었다.
    • 알고리즘은 결국 배운 자료구조(배열)와 테크닉(나머지 연산, 델타 탐색)을 적재적소에 꺼내 쓰는 싸움인 것 같다.
