    오늘 목표

    java.time 패키지의 등장

    왜 새로 만들었을까?

    과거의 실수: Date와 Calendar
    예전 자바의 날짜 클래스들은 불변(Immutable)하지 않아서, 누군가 날짜를 수정하면 연결된 모든 곳에서 날짜가 바뀌는 대참사가 일어났다. 또한 월(Month)을 0부터 시작(1월이 0)하는 등 직관적이지 않았다.
    JDK 8부터는 이를 완전히 뜯어고친 java.time 패키지를 도입했다.

    핵심 클래스 3대장

    새로운 시간 패키지는 날짜와 시간을 분리해서 다룬다. 필요하면 합칠 수도 있다.

    1. LocalDate (날짜)

    • 비유: 벽에 걸린 달력. 시간은 없고 '2026년 2월 4일'이라는 정보만 있다.
    • 용도: 생일, 기념일, 휴가 시작일 등.
    LocalDate date = LocalDate.of(2026, 2, 4); // 2026-02-04
LocalDate today = LocalDate.now(); // 오늘 날짜

    2. LocalTime (시간)

    • 비유: 손목 시계. 날짜는 모르고 '14시 30분'이라는 시간 정보만 있다.
    • 용도: 알람 시간, 수업 시작 시간 등.
    LocalTime time = LocalTime.of(14, 30); // 14:30
LocalTime now = LocalTime.now(); // 현재 시간

    3. LocalDateTime (날짜 + 시간)

    • 비유: 스마트폰의 잠금 화면. 날짜와 시간이 같이 표시된다.
    • 공식: LocalDate + LocalTime = LocalDateTime
    • 용도: 약속 시간, 게시글 작성 일시, 로그 기록 등.
    LocalDateTime dt = LocalDateTime.of(date, time); 
// 결과: 2026-02-04T14:30

    시간대(Time Zone)와 ZonedDateTime

    지구 반대편의 시간

    LocalDateTime은 "서울의 2시"인지 "뉴욕의 2시"인지 모른다. 여기에 위치 정보(ZoneId)를 더해주면 비로소 전 세계 어디서든 통용되는 시간이 된다.

    • 공식: LocalDateTime + ZoneId = ZonedDateTime
    • 예시: 한국 시간(KST)을 표현할 때 사용한다.
    // 서울 시간대 설정
ZonedDateTime seoulTime = ZonedDateTime.now(ZoneId.of("Asia/Seoul"));
System.out.println(seoulTime); 
// 2026-02-04T14:30:00+09:00[Asia/Seoul] -> '+09:00'이 시차 정보

    시간의 간격: Period vs Duration

    "얼마나 걸렸어?"를 물어볼 때, 날짜 단위인지 시간 단위인지에 따라 사용하는 도구가 다르다.

    1. Period (날짜 차이)

    • 비유: 달력을 넘기며 세는 것. "생일까지 몇 달, 며칠 남았지?"
    • 단위: 년(Year), 월(Month), 일(Day)
    • 공식: 날짜 - 날짜 = Period
    LocalDate start = LocalDate.of(2026, 1, 1);
LocalDate end = LocalDate.of(2026, 2, 4);

Period diff = Period.between(start, end);
System.out.println(diff.getMonths() + "개월 " + diff.getDays() + "일 지남");
// 1개월 3일 지남

    2. Duration (시간 차이)

    • 비유: 스톱워치로 재는 것. "영화 상영 시간이 몇 분 몇 초지?"
    • 단위: 시(Hour), 분(Minute), 초(Second), 나노초(Nano)
    • 공식: 시간 - 시간 = Duration
    LocalTime start = LocalTime.of(9, 0, 0);
LocalTime end = LocalTime.of(10, 30, 0);

Duration diff = Duration.between(start, end);
System.out.println(diff.toMinutes() + "분 걸림");
// 90분 걸림

    그 외 유용한 기능들

    실무에서 정말 자주 쓰이는 "날짜 조작"과 "형식 변환" 기능이다.

    TemporalAdjusters (날짜 조정기)

    "다음 주 금요일이 며칠이지?"
    단순히 +1일 하는 게 아니라, 복잡한 날짜 계산을 대신 해주는 똑똑한 비서다. 
    LocalDateTime now = LocalDateTime.now();

// 다음 주 금요일 찾기
LocalDateTime nextFriday = now.with(TemporalAdjusters.next(DayOfWeek.FRIDAY));

// 이번 달의 마지막 날 찾기 (월말 정산 때 유용)
LocalDateTime lastDay = now.with(TemporalAdjusters.lastDayOfMonth());

    DateTimeFormatter (출력 형식)

    자바의 날짜 객체를 우리가 보기 편한 문자열(String)로 바꾸거나, 반대로 문자열을 날짜 객체로 바꿀 때 쓴다. 

    LocalDateTime now = LocalDateTime.now();

// 1. 날짜 -> 문자열 (포맷팅)
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy년 MM월 dd일 HH시 mm분");
String text = now.format(formatter); 
// 결과: "2026년 02월 04일 14시 30분"

// 2. 문자열 -> 날짜 (파싱)
LocalDateTime parsedDate = LocalDateTime.parse("2026-02-04 14:30", formatter);

    오늘의 회고

    오늘 느낀 점

    • Date와 Calendar를 쓸 때는 월(Month) 계산이 항상 헷갈렸는데, java.time은 직관적이라 훨씬 편했다.
    • Period(달력 차이)와 Duration(물리적 시간 차이)을 구분해서 써야 한다는 점을 배웠다. D-Day 계산엔 Period가 맞고, 실행 시간 측정엔 Duration이 맞다.
    • TemporalAdjusters는 실무에서 정산일이나 배송 예정일을 계산할 때 정말 유용하게 쓰일 것 같다.
