    1. S3와 RDS를 이용한 상태 분리

    EC2만으로 모든 것을 처리하면 파일은 EC2 로컬에 저장되고 DB도 EC2 내부에서 실행된다. EC2가 재시작되거나 교체되면 데이터가 유실되고, 트래픽 분산을 위해 EC2를 여러 대로 늘리면 파일과 DB가 각 서버에 흩어져 데이터 일관성을 보장할 수 없다. 이를 해결하기 위해 파일은 S3, DB는 RDS로 분리한다.

    서비스 역할 해결하는 문제
    S3 (Simple Storage Service) 외부 파일 저장소 EC2 교체/재시작 시 파일 유실 방지, 무제한 확장
    RDS (Relational Database Service) 외부 DB 서비스 DB 관리 부담 제거 (백업, 패치, 장애 조치 자동화)

    2. S3 구조

    개념 설명
    버킷 (Bucket) 파일을 담는 저장소 컨테이너다. 전 세계에서 고유(unique)한 이름을 가져야 한다.
    오브젝트 (Object) 버킷 안에 저장된 파일 자체다.
    키 (Key) 오브젝트의 고유 경로/이름이다. uploads/profile/user1.jpg 전체가 하나의 키다.
    S3에는 실제 폴더가 없다 uploads/profile/user1.jpg처럼 디렉토리 구조처럼 보이지만, 이것은 하나의 키 문자열 전체다. S3는 플랫(flat)한 key-value 저장소이며, AWS 콘솔이 /를 기준으로 폴더처럼 시각화해줄 뿐이다.

    버킷 이름 규칙 및 보안

    버킷 이름은 <프로젝트>-<환경>-<용도> 패턴을 권장한다. 예: myapp-prod-uploads

    보안 정책 설명
    퍼블릭 액세스 차단 기본적으로 모든 퍼블릭 접근을 차단한다. 의도하지 않은 공개를 방지한다.
    최소 권한 원칙 IAM Policy로 필요한 권한만 부여한다. (예: 업로드는 가능하지만 삭제는 불가)

    3. BOM (Bill Of Materials)

    관련 라이브러리들의 버전을 한 곳에서 통합 관리하는 버전 목록표다. BOM을 사용하면 각 라이브러리의 버전을 일일이 지정하지 않아도 호환되는 버전이 자동으로 결정된다. 버전 충돌 문제를 사전에 방지할 수 있다. 

    // BOM으로 Spring Cloud AWS 라이브러리 버전 통합 관리
implementation platform('io.awspring.cloud:spring-cloud-aws-dependencies:4.0.0-RC1')

// 버전 명시 없이 추가 — BOM이 호환 버전을 자동 결정
implementation 'io.awspring.cloud:spring-cloud-aws-starter-parameter-store'
implementation 'io.awspring.cloud:spring-cloud-aws-starter-s3'

    4. S3Template을 활용한 파일 업로드 구현

    Presigned URL

    프라이빗 버킷의 파일은 일반 URL로 직접 접근할 수 없다. Presigned URL은 임시로 접근을 허용하는 서명된 URL이다. 유효 시간을 설정할 수 있으며, 유효 시간 내에는 URL을 가진 누구나 해당 파일에 접근 가능하다. 클라이언트가 파일을 직접 다운로드할 수 있도록 이 URL을 발급해주는 방식으로 구현한다.

    Presigned URL 동작 방식 1. 클라이언트가 서버에 다운로드 URL 요청
    2. 서버가 S3에 서명된 임시 URL 생성 요청
    3. 서버가 클라이언트에 Presigned URL 반환
    4. 클라이언트가 Presigned URL로 S3에 직접 파일 접근 (서버를 거치지 않음)

    서버가 파일을 중계하지 않으므로 서버 대역폭 부담이 없다.

    응답 DTO 

    // 파일 업로드 응답 — 저장된 키를 반환
@Getter
public class FileUploadResponse {
    private final String key;

    public FileUploadResponse(String key) {
        this.key = key;
    }
}

// 다운로드 URL 응답 — Presigned URL을 반환
@Getter
public class FileDownloadUrlResponse {
    private final String url;

    public FileDownloadUrlResponse(String url) {
        this.url = url;
    }
}

    S3Service 

    @Service
@RequiredArgsConstructor
public class S3Service {

    // Presigned URL 유효 시간: 10분
    private static final Duration PRESIGNED_URL_EXPIRATION = Duration.ofMinutes(10);

    private final S3Template s3Template; // Spring Cloud AWS가 Bean으로 등록해준다

    @Value("${spring.cloud.aws.s3.bucket}")
    private String bucket;

    public String upload(MultipartFile file) {
        try {
            // UUID로 파일명 중복 방지
            String key = "uploads/" + UUID.randomUUID() + "_" + file.getOriginalFilename();
            s3Template.upload(bucket, key, file.getInputStream());
            return key;
        } catch (IOException e) {
            // 적절한 커스텀 예외로 교체하고 GlobalExceptionHandler로 핸들링 필요
            throw new RuntimeException("파일 업로드 실패", e);
        }
    }

    public URL getDownloadUrl(String key) {
        // 서명된 임시 URL 생성 — 10분간 유효
        return s3Template.createSignedGetURL(bucket, key, PRESIGNED_URL_EXPIRATION);
    }
}

    FileController 

    @RestController
@RequiredArgsConstructor
public class FileController {

    private final S3Service s3Service;

    // 파일 업로드 — multipart/form-data로 받아 S3에 저장 후 키 반환
    @PostMapping("/files/upload")
    public ResponseEntity<FileUploadResponse> upload(@RequestParam("file") MultipartFile file) {
        String key = s3Service.upload(file);
        return ResponseEntity.ok(new FileUploadResponse(key));
    }

    // 다운로드 URL 발급 — 키를 받아 Presigned URL 반환
    @GetMapping("/files/download-url")
    public ResponseEntity<FileDownloadUrlResponse> getDownloadUrl(@RequestParam String key) {
        URL url = s3Service.getDownloadUrl(key);
        return ResponseEntity.ok(new FileDownloadUrlResponse(url.toString()));
    }
}

    로컬 환경 설정 

    # application-local.properties — 로컬 개발용 AWS 자격증명
spring.cloud.aws.credentials.access-key=AKIA...
spring.cloud.aws.credentials.secret-key=+JIr...
    ⚠️ Access Key 보안 주의 application-local.properties의 Access Key는 절대 Git에 커밋하면 안 된다. .gitignore에 반드시 추가해야 한다. 운영 환경에서는 Access Key 대신 EC2 IAM Role을 사용하는 것이 올바른 방법이다.

    5. RDS (Relational Database Service)

    AWS가 관리해주는 관계형 DB 서비스다. 직접 EC2에 MySQL을 설치해 운영하면 백업, 패치, 장애 복구를 모두 직접 처리해야 하지만, RDS를 사용하면 AWS가 다음 기능을 자동으로 처리한다.

    기능 AWS가 대신 해주는 것
    백업 자동으로 데이터를 주기적으로 백업한다.
    패치 유지 관리 기간에 DB 엔진 패치를 자동 적용한다.
    장애 조치 Multi-AZ 구성 시, 한 AZ의 DB가 장애가 나면 다른 AZ의 복제본으로 자동 전환한다.
    확장 콘솔에서 클릭 몇 번으로 스펙을 변경하거나 자동 확장할 수 있다.

    마무리

    오늘의 핵심은 EC2에서 상태(State)를 분리하는 것이다. 파일을 EC2 로컬에 저장하면 서버가 교체될 때마다 데이터가 사라진다. S3는 EC2와 독립된 외부 저장소이므로 EC2를 몇 대로 늘리든, 재시작하든 데이터가 안전하게 유지된다. S3의 키가 실제 폴더가 아닌 단순한 문자열 경로라는 점, 버킷 이름이 전 세계에서 고유해야 한다는 점도 중요한 특성이다.

    Presigned URL은 프라이빗 버킷의 파일을 외부에 안전하게 노출하는 핵심 메커니즘이다. 서버가 파일을 직접 전달하지 않고 임시 서명 URL을 발급해 클라이언트가 S3에 직접 접근하도록 하기 때문에, 서버의 대역폭 부담이 없고 유효 시간이 지나면 URL이 자동으로 만료된다. S3Template.createSignedGetURL() 한 줄로 구현할 수 있어 실용적이다.

    BOM은 여러 라이브러리의 버전을 한 곳에서 통합 관리해 호환성 문제를 방지한다. implementation platform(...)으로 BOM을 선언하면 하위 라이브러리는 버전을 명시하지 않아도 된다. RDS는 DB 자체의 운영 부담(백업, 패치, 장애 복구, 확장)을 AWS에 위임해 개발팀이 비즈니스 로직에만 집중할 수 있게 해준다. 운영 환경의 EC2에서는 Access Key 대신 IAM Role로 S3와 RDS에 접근 권한을 부여하는 것이 보안 원칙에 맞는 방법이다.
