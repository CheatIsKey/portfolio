    CSS 실습: 회원가입 폼 스타일링

    작성일 2025-10-19 · 분류: CSS

    개요

    이 실습은 앞서 만든 회원등록 HTML에 style.css를 연결해 실제로 보이는 모양을 다듬는다. 이미 HTML에 id와 class 선택자를 부여했고, <head>에서 외부 스타일시트를 연결했다.

    <link rel="stylesheet" href="style.css">
    • id (예: #wrapper): 문서 내 하나만 존재해야 하는 고유 요소에 사용.
    • class (예: .inputGroup): 여러 요소에 반복 적용할 스타일을 위해 사용.

    전역 설정: 변수와 박스 모델

    전역에서 사용할 색상은 :root에 CSS 변수로 선언해 재사용한다. 박스 계산은 border-box로 두어 테두리/패딩이 포함된 폭을 기준으로 생각하면 편하다.

    :root{
  --primaryColor: green;
  --errorColor: red;
  --backgroundColor: rgb(67, 144, 233);
}
html{ box-sizing: border-box; }
    box-sizing 차이 정확히 이해하기
    • content-box (기본): width는 content만 의미. 여기에 padding과 border가 더해져 실제 박스 폭이 커진다.
    • border-box: width가 content + padding + border를 통틀어 의미. 수치 관리가 단순해짐.

    참고: <head>는 문서의 정보를 담는 영역이므로 style.css에서 제외하는 게 자연스럽다(스타일 대상은 화면에 렌더되는 <body> 이하).

    레이아웃: 화면 중앙 정렬

    body를 플렉스 컨테이너로 만들어, 하위에 있는 폼 래퍼(#wrapper)를 가로·세로 중앙에 둔다. min-height: 100vh;는 브라우저 뷰포트 높이에 맞춰 빈 화면 없이 꽉 차게 해 준다.

    body{
  display: flex;
  align-items: center;     /* 수직 중앙 정렬 */
  justify-content: center; /* 수평 중앙 정렬 */
  min-height: 100vh;       /* viewport 높이 기준 */
  background-color: white;
}
    • align-items: center → 교차축(세로) 기준 중앙.
    • justify-content: center → 주축(가로) 기준 중앙.

    카드 래퍼: 그림자와 제목

    #wrapper는 카드 형태를 위한 컨테이너다. box-shadow로 살짝 떠 보이게 만든다.

    #wrapper{
  width: 420px;
  box-shadow: 0px 4px 14px rgba(0, 0, 0, 0.25);
}
h2{
  text-align: center;
  margin-top: 20px;
}

    box-shadow 파라미터는 순서대로 수평 오프셋, 수직 오프셋, 흐림 반경, 색상이다. 예) 0 4px 14px rgba(0,0,0,.25) → 아래로 4px 떨어진 부드러운 그림자.

    입력 그룹: 블록 배치와 패딩

    각 필드는 .inputGroup으로 묶고, label과 input을 block으로 잡아 한 줄에 하나씩 보이게 한다. 패딩은 상하 8px, 좌우 12px로 통일했다.

    .inputGroup{
  padding: 8px 12px;
}
.inputGroup label{
  display: block;
  padding: 12px 0;
}
.inputGroup input{
  display: block;
  width: 93%;
  padding: 12px;
  border: 2px solid #aaa9a9;
  font-size: 16px;
}
    • block vs inline: block은 한 줄을 통째로 차지해 폭/높이 지정이 가능하고, inline은 줄 안에서 흐르며 폭/높이 지정이 제한된다.
    • width: 93%: 좌우 패딩/테두리를 고려해 약간 줄여 입력창이 래퍼 폭을 넘지 않도록 한 값.

    입력 포커스: 시각적 피드백

    입력창에 포커스가 들어오면 테두리 색을 바꿔 현재 위치가 어디인지 바로 보이게 한다.

    .inputGroup input:focus{
  outline: none;
  border-color: #777;
}

    검증 상태: valid / invalid

    자바스크립트로 각 .inputGroup에 .valid 또는 .invalid 클래스를 토글해 시각 상태를 바꾼다. 올바른 값이면 --primaryColor, 틀린 값이면 --errorColor를 사용한다.

    .inputGroup.valid  input{ border-color: var(--primaryColor); }
.inputGroup.invalid input{ border-color: var(--errorColor); }

    헬퍼 텍스트(.helperText)는 기본적으로 보이게 두고, valid일 때만 숨김, invalid일 때 빨간색으로 강조한다.

    .inputGroup .helperText{
  display: block;
  padding-top: 8px;
}
.inputGroup.valid  .helperText{ visibility: hidden; }
.inputGroup.invalid .helperText{ color: var(--errorColor); }

    visibility: hidden은 요소를 ‘숨기지만’ 자리(레이아웃)는 유지한다. 즉, 메시지가 사라져도 주변 레이아웃이 휘청거리지 않게 하려는 의도에 맞다. (이번 실습에서 사용한 그대로의 의미)

    제출 버튼: 테마 색상 적용

    버튼은 래퍼 중앙에 두고, 배경/테두리에 --backgroundColor를 써서 테마와 맞춘다.

    #wrapper button{
  display: block;
  width: 50%;
  margin: auto;
  height: 50px;
  margin-bottom: 25px;
  font-size: 18px;
  background-color: var(--backgroundColor);
  border: 2px solid var(--backgroundColor);
  color: white;
  cursor: pointer;
}

    결과 스크린샷

    주의/체크리스트

    • 선택자 타이핑: .inputGroup .helperText처럼 띄어쓰기(부모-자식)와 붙여쓰기(두 클래스 동시) 차이를 헷갈리지 말 것.
    • 변수 사용: var(--primaryColor) 등 변수명을 오탈자 없이 정확히. (키: --primaryColor, 값: green)
    • visibility와 display 차이: 숨길 때 자리 유지(visibility) vs 자리까지 제거(display).
    • 레이아웃 기준: 중앙 정렬은 body가 플렉스 부모이기 때문에 가능(자식: #wrapper).
