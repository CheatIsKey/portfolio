    회원 등록 폼 만들기

    작성일 2025-10-18 · 분류: HTML

    개요

    이 장에서는 회원 등록 폼을 직접 만들어보며 <div>, <form>, <input>, <label>, <button>의 기초를 익힌다. HTML만으로도 많은 기본 검증(이메일 형식, 필수 입력 등)을 지원하며, CSS/JavaScript 없이도 어느 정도 UX를 확보할 수 있다.

    div와 span

    • <div>: 블록 레벨 요소. 기본적으로 가로 전체를 차지하고, 내용이 없으면 높이가 0에 가깝지만(패딩/보더가 없다면), “자리” 자체는 차지한다. 레이아웃 그룹핑에 사용.
    • <span>: 인라인 요소. 문장 흐름 안에서 일부 텍스트나 아이콘을 감싸는 데 사용.

    form의 핵심 속성

    • action: 데이터를 보낼 URL.
    • method: HTTP 메서드(보통 GET 또는 POST).
    • enctype: 전송 인코딩(파일 업로드 시 multipart/form-data 등).

    참고: 전통적인 HTML 폼은 주로 application/x-www-form-urlencoded로 전송된다. JSON/XML 기반 REST API 전송은 보통 fetch() 등 JS를 써서 구현한다(이 장에서는 폼 기본에 집중).

    회원 등록 폼

    아래 예시는 접근성과 자동완성을 고려해 id/for/name/autocomplete를 명시하고, HTML5 기본 검증을 사용한다.

    <form action="/signup" method="post">
  <h2>회원등록</h2>

  <div>
    <label for="nickname">사용자 이름</label>
    <input id="nickname" name="nickname" type="text" required
           minlength="2" maxlength="20" autocomplete="nickname">
    <span aria-live="polite">알림</span>
  </div>

  <div>
    <label for="userEmail">이메일 주소</label>
    <input id="userEmail" name="email" type="email" required
           inputmode="email" autocomplete="email">
    <span aria-live="polite">알림</span>
  </div>

  <div>
    <label for="userPassword">비밀번호</label>
    <input id="userPassword" name="password" type="password" required
           minlength="8" autocomplete="new-password">
    <span aria-live="polite">알림</span>
  </div>

  <div>
    <label for="confirmPassword">비밀번호 확인</label>
    <input id="confirmPassword" name="confirmPassword" type="password" required
           minlength="8" autocomplete="new-password">
    <span aria-live="polite">알림</span>
  </div>

  <div>
    <label for="phone">휴대폰번호</label>
    <input id="phone" name="phone" type="tel"
           inputmode="tel" autocomplete="tel-national"
           placeholder="010-1234-5678"
           pattern="^01[016789]-\\d{3,4}-\\d{4}$">
    <span>예시: 010-1234-5678</span>
  </div>

  <button type="submit">등록</button>
</form>

    포인트

    • label ↔ input 연결: for와 id가 일치해야 포커스 이동/스크린리더에 유리.
    • name: 서버가 받는 파라미터 이름. name이 없으면 데이터가 전송되지 않는다.
    • required/minlength/pattern: HTML5 기본 검증. JS 없이도 브라우저가 형식을 체크한다.
    • autocomplete: 브라우저 자동완성 힌트 제공(UX 향상).
    • type=email: @ 미포함 등 형식 오류 시 브라우저가 기본 메시지를 보여준다.
    • button type="submit": 폼 내부 제출 버튼은 반드시 type="submit"으로.

    여러 필드 묶기: 레이아웃 힌트

    지금은 스타일을 적용하지 않았지만, 다음 CSS 챕터에서 <div>들을 그리드/플렉스로 정렬하고 간격, 오류 스타일(.is-invalid) 등을 추가할 예정이다.

    정리

    • <form>은 서버 전송의 기본 단위다. action은 URL, method는 전송 방식.
    • HTML5 검증 속성으로 기본 UX를 확보하고, 상세 검증은 JS/서버에서 보완한다.
    • 접근성을 위해 label–input 연결, name 지정, autocomplete 제공을 습관화한다.

    다음: CSS 챕터 1에서 레이아웃과 폼 스타일링을 적용한다.
