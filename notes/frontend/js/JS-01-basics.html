    JS 기초: 변수와 타입

    작성일 2025-10-20 · 분류: JavaScript

    개요

    이 문서는 자바스크립트 변수 선언(var, let, const)과 데이터 타입, 템플릿 리터럴, null / undefined, 타입 변환의 기초를 정리한다. 모든 예제는 콘솔 기준이다.

    변수 선언: var / let / const

    자바스크립트는 동적 타이핑 언어다. 즉, 변수의 타입은 대입된 값에 의해 결정되며 var, let, const 어느 키워드로든 어떤 타입의 값을 담을 수 있다.

    // let: 블록 스코프 변수
let age = 20;         // number
age = "스무살";       // string (동적 타이핑)

// var: 함수 스코프 변수 (같은 스코프 내 재선언 가능)
var nickname = "jay";
var nickname = "kim"; // 가능

// const: 바인딩(식별자→값) 불변 — 반드시 초기화 필요, 재할당 불가
const API_KEY = "abc123";
// API_KEY = "def";   // ✗ 재할당 불가

    메모: const는 “바인딩”이 불변일 뿐, 참조한 객체/배열의 내부 값은 바뀔 수 있다.

    const user = { name: "Kim" };
user.name = "Lee"; // ✓ 내부 변경은 가능
// user = {}        // ✗ 재할당은 불가
    • 스코프: var는 함수 스코프, let/const는 블록 스코프.
    • 재선언: 같은 스코프에서 var는 재선언 가능, let/const는 불가.
    • 호이스팅 & TDZ: 모두 호이스팅되지만 let/const는 TDZ 때문에 선언 전 접근 시 에러.

    데이터 타입 개요

    주요 타입을 실무 친화적으로 분류하면 다음과 같다.

    • 원시 타입(primitive): number, string, boolean, undefined, null (그리고 확장으로 symbol, bigint)
    • 객체 타입(object): 객체 리터럴, 배열(배열은 객체의 한 종류), 함수, 날짜 등
    let n = 42;                   // number
let s = "hello";              // string
let ok = true;                // boolean
let empty = null;             // 의도적으로 "없음"을 표현하는 값
let notSet;                   // undefined (선언했지만 초기화X)

let user = { name: "Kim" };   // object
let list = [1, "a", false];   // array (object의 한 종류)

    템플릿 리터럴(백틱)과 보간

    백틱(` `)으로 문자열을 감싸고 ${식} 형태로 값을 끼워 넣는다.

    const name = "Jae";
const score = 95;
const msg = `이름: ${name}, 점수: ${score}`;
console.log(msg); // "이름: Jae, 점수: 95"

    객체와 배열

    객체는 {키:값} 쌍의 집합이며, 배열은 []로 초기화하는 순서 있는 컬렉션이다. 배열에는 숫자/문자열/불린/객체/배열 등 서로 다른 타입을 혼합해서 담을 수 있다.

    // 객체
const person = { name: "Lee", age: 28 };
// 배열
const mixed = [123, "abc", true, { x: 1 }, [1, 2, 3]];

console.log(person.name); // "Lee"
console.log(mixed[3].x);  // 1

    null vs undefined

    • null: 값 그 자체로 비어있음을 표현하는 의도적 값.
    • undefined: 변수를 선언만 하고 초기화하지 않았거나, 존재하지 않는 프로퍼티 접근 결과.
    let a = null;   // 비어있음을 명시
let b;          // 선언만 함 → undefined
console.log(a, b); // null undefined

    타입 변환: 암시적 vs 명시적

    자바스크립트의 형 변환은 크고/작은 타입 개념이 아니라 문맥에 의해 결정된다.

    • 암시적 변환: 연산/비교 등 문맥에서 엔진이 자동 변환.
    • 명시적 변환: Number(...), String(...), Boolean(...) 같은 함수 호출로 직접 변환.
    // 암시적 변환 예시
1 + "2"      // "12"  (문자열 결합 방향)
"5" * 2      // 10    (숫자 산술 방향)
"10" - 3     // 7
"10px" - 3   // NaN   (숫자로 해석 실패)

// 명시적 변환 예시
let str = "123";
let num = 123;

Number(str)  // 123 (number)
String(num)  // "123" (string)
Boolean("")  // false
Boolean("a") // true

    주의: 서로 다른 타입을 연산하면 자동 변환이 일어나지만, 항상 “큰 타입으로 승격”되는 규칙은 없다. 덧셈은 문자열이 끼면 보통 문자열 결합, 산술 연산은 보통 숫자 변환이 일어난다.

    핵심 개념 요약

    • 동적 타이핑: 변수 타입은 대입값으로 결정된다.
    • var/let/const: 스코프/재선언/TDZ 차이를 이해하고, const는 초기화 필수·재할당 불가(단, 참조 객체의 내부 변경은 가능).
    • 데이터 타입: 원시 타입(숫자/문자열/불린/undefined/null (+ symbol/bigint))과 객체 타입. 배열은 객체의 한 종류.
    • 템플릿 리터럴: 백틱과 ${}로 값 보간.
    • 타입 변환: 암시적(문맥 기반) / 명시적(함수 호출 기반). “큰/작은 타입” 개념은 사용하지 않는다.
