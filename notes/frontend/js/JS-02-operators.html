    JS 기초: 연산자·제어문·함수·객체

    작성일 2025-10-21 · 분류: JavaScript

    동등/일치 연산자: == vs ===

    자바스크립트의 ==는 타입을 암시적으로 변환한 뒤 값만 비교한다. ===는 타입과 값을 모두 비교한다.

    5 == "5"   // true  (문자열 "5" → 숫자 5 로 변환 후 비교)
5 === "5"  // false (타입이 다르므로 불일치)
0 == false // true  (암시적 변환 때문에 헷갈릴 수 있음)
0 === false // false

    권장: 예측 가능성을 위해 ===와 !== 사용을 기본으로 하자.

    조건문: if / else if / else

    기본 형태는 자바와 유사하다. 조건은 truthy/falsy 규칙을 따른다(예: 0, "", null, undefined, NaN은 falsy).

    const score = 81;

if (score >= 90) {
  console.log("A");
} else if (score >= 80) {
  console.log("B"); // 81 → "B"
} else {
  console.log("C");
}

    switch 문

    switch는 엄격 비교(===)를 사용한다. 즉, 타입이 다르면 매칭되지 않는다.

    const v = "2";

switch (v) {
  case 2:           // 매칭 안 됨 (number 2)
    console.log("num 2");
    break;
  case "2":         // 매칭됨 (string "2")
    console.log("str 2");
    break;
  default:
    console.log("기타");
}

    반복문: for (전통)

    전통적인 for문은 인덱스로 순회한다.

    const arr = ["a", "b", "c"];
for (let i = 0; i < arr.length; i++) {
  console.log(i, arr[i]); // 0 "a" / 1 "b" / 2 "c"
}

    반복문: for...of (값 순회)

    for...of는 배열 등 이터러블의 값을 순회한다. 일반 객체에는 바로 사용하지 않는다 (객체는 이터러블이 아님).

    const arr = [10, 20, 30];
for (const v of arr) {
  console.log(v); // 10 / 20 / 30
}

// 객체에 값을 돌고 싶다면:
const user = { name: "Kim", age: 30 };
// for (const v of user) { ... } // ✗ 에러
for (const v of Object.values(user)) {
  console.log(v); // "Kim", 30
}

    반복문: for...in (키 순회)

    for...in은 객체의 열거 가능한 키를 순회한다. 자신의 키만 보려면 Object.hasOwn() 체크를 해 주는 습관을 들이자.

    const user = { name: "Lee", age: 28 };
for (const key in user) {
  if (Object.hasOwn(user, key)) {
    console.log(key, user[key]); // name "Lee" / age 28
  }
}

    반복문: while / do...while

    while은 먼저 조건을 평가하고, do...while은 한 번은 무조건 실행한 뒤 조건을 평가한다.

    // while
let n = 3;
while (n > 0) {
  console.log(n); // 3, 2, 1
  n--;
}

// do...while
let m = 0;
do {
  console.log("최소 1번은 실행"); // 이 줄은 항상 한 번 실행됨
  m++;
} while (m < 0);

    함수: 선언문 vs 표현식

    함수는 정의하고 호출한다. 자바스크립트는 동적 타이핑이므로 매개변수에 타입을 적지 않는다.

    // 함수 선언문 (Declaration) — 전체가 호이스팅되어 어디서든 호출 가능
console.log(add(1, 2)); // 3
function add(a, b) {
  return a + b;
}

// 함수 표현식 (Expression) — 값으로 변수에 담는다
// 권장: 재할당 방지를 위해 const 사용
const mul = function (a, b) {
  return a * b;
};
console.log(mul(2, 3)); // 6

// 주의: var로 함수 표현식을 선언하면 호이스팅/재선언 이슈가 있다 (지양).
// let도 가능하지만 보통 const를 써서 의도치 않은 재할당을 막는다.

    호이스팅: 함수 선언문은 정의 자체가 호이스팅되어 선언 위치보다 위에서 호출 가능하다. 함수 표현식은 변수에 할당되기 전에는 사용할 수 없다.

    객체와 this

    객체는 {키:값} 형태다. 값에는 숫자/문자열/불린/함수(메서드) 등이 올 수 있다. this는 호출 방식에 따라 달라진다.

    const counter = {
  value: 0,
  inc: function () {
    this.value++; // 메서드로 호출되면 this → counter
  }
};

counter.inc();
console.log(counter.value); // 1

function showThis() {
  "use strict";
  console.log(this);
}
showThis(); // 엄격 모드: undefined (비엄격 모드 브라우저에선 window)

    생성자 함수는 new와 함께 호출해 인스턴스를 만든다. 내부에서 this로 프로퍼티를 설정한다.

    function User(name, age) {
  this.name = name;
  this.age = age;
}
const u = new User("Kim", 30);
console.log(u.name, u.age); // "Kim" 30

    프로토타입과 프로토타입 체인

    생성자 함수로 만든 인스턴스는 생성자함수.prototype에 연결되어 있다. 여기에 메서드를 정의하면 모든 인스턴스가 공유해서 사용한다.

    function User(name) { this.name = name; }
User.prototype.sayHi = function () {
  console.log(`Hi, ${this.name}`);
};

const a = new User("Lee");
const b = new User("Park");
a.sayHi(); // Hi, Lee
b.sayHi(); // Hi, Park

    프로토타입 체인: 객체에 없는 프로퍼티/메서드를 찾을 때 자바스크립트는 그 객체의 프로토타입, 그 위의 프로토타입… 순서로 탐색한다. Object.prototype을 임의로 수정하는 것은 지양하자.

    요약

    • 연산자: ==는 암시적 변환, ===는 타입+값 비교(권장).
    • 조건/분기: if는 truthy/falsy, switch는 === 기반.
    • 반복: 전통 for(인덱스), for...of(값·이터러블), for...in(키·객체).
    • 함수: 선언문은 호이스팅, 표현식은 할당 후 사용. 표현식엔 const 권장.
    • 객체/this: 호출 방식에 따라 this 달라짐. 생성자+prototype으로 공통 메서드 공유.
