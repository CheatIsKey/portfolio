    프론트엔드와 백엔드 구조

    웹 애플리케이션은 기본적으로 프론트엔드와 백엔드 두 서버로 구성된다.

    • 프론트엔드: HTML, CSS, JavaScript를 사용하는 클라이언트(화면) 쪽 서버.
    • 백엔드: Java, Spring Boot, 내장 Tomcat 등을 사용하는 서버(로직/DB) 쪽.

    두 서버 간에 데이터를 주고받을 때 사용하는 대표적인 함수가 fetch()이다.

    fetch() — 프론트엔드 ↔ 백엔드 통신

    fetch()는 HTTP 요청을 보내고, Promise 객체를 반환한다. 기본 구조는 다음과 같다.

    fetch(url, options)
  .then(response => response.json())
  .then(data => console.log("성공:", data))
  .catch(error => console.error("에러:", error));

    options에는 HTTP 메서드와 요청 헤더, 본문(body) 등을 지정할 수 있다.

    fetch("http://localhost:8080/api/register", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "Jin", age: 25 })
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error(error));

    참고: fetch()는 비동기로 동작하며, 항상 Promise를 반환한다.

    HTTP 메서드 요약

    • GET — 서버의 데이터를 조회할 때 사용. URL 쿼리(?key=value) 형태로 데이터가 보인다.
    • POST — 데이터를 서버로 보낼 때 사용(회원가입, 로그인 등). body에 포함되므로 URL에 노출되지 않는다.
    • PUT — 기존 데이터를 통째로 수정(덮어쓰기)할 때 사용.
    • DELETE — 데이터를 삭제할 때 사용. 일반적으로 URL에 대상 식별자를 포함한다.
    // GET 예시
fetch("/api/users?id=3");

// POST 예시
fetch("/api/users", {
  method: "POST",
  body: JSON.stringify({ name: "Lee" }),
  headers: { "Content-Type": "application/json" }
});

    웹 스토리지(Web Storage)

    웹 스토리지는 브라우저(프론트엔드)에서 데이터를 저장하는 기능이다. 서버 세션(Session)과는 완전히 다르며, 사용자의 브라우저 내부에 데이터가 저장된다.

    1) localStorage

    브라우저나 컴퓨터를 꺼도 데이터가 유지된다. 각 도메인(URL)마다 별도의 저장소를 가진다.

    // 저장
localStorage.setItem("user", "Kim");

// 읽기
const name = localStorage.getItem("user");
console.log(name); // "Kim"

// 수정 (같은 key를 덮어쓰기)
localStorage.setItem("user", "Lee");

// 삭제
localStorage.removeItem("user");

// 전체 삭제
localStorage.clear();

    2) sessionStorage

    브라우저 탭이 닫히면 데이터가 사라진다. 도메인별로 별도의 저장소를 갖는다.

    sessionStorage.setItem("temp", "123");
console.log(sessionStorage.getItem("temp")); // "123"
sessionStorage.clear(); // 탭 닫거나 수동 삭제 시 초기화

    정리: localStorage — 브라우저를 꺼도 유지
    sessionStorage — 탭을 닫으면 삭제

    var / let / const 비교

    자바스크립트에서 변수 선언 방식은 세 가지다.

    // var — 재선언 가능 (지양)
var a = 1;
var a = 2; // 에러 없음, 기존 값 덮어씀

// let — 재선언 불가, 재할당 가능
let b = 1;
// let b = 2; // ✗ 재선언 불가
b = 3;       // ✓ 재할당 가능

// const — 재선언/재할당 모두 불가
const c = 1;
// c = 2; // ✗ 불가능

    요약: var은 함수 스코프, let과 const는 블록 스코프를 가진다. 실무에서는 주로 const (불변) 또는 let (변경 필요 시)만 사용한다.

    스코프(Scope)

    스코프는 변수에 접근할 수 있는 유효 범위를 뜻한다.

    • 전역 스코프(Global): 어디서든 접근 가능.
    • 함수 스코프(Function): function 내부에서만 유효.
    • 블록 스코프(Block): { } 내부에서만 유효 (let, const에만 적용).
    let x = 10; // 전역

function foo() {
  let y = 20; // 함수 스코프
  if (true) {
    let z = 30; // 블록 스코프
    console.log(x, y, z); // 가능
  }
  // console.log(z); // ✗ 접근 불가
}
foo();

    호이스팅(Hoisting)

    자바스크립트는 변수나 함수의 선언부를 스코프의 최상단으로 끌어올린다. 이를 호이스팅이라 한다.

    1) 함수 선언문 — 호이스팅 적용

    sayHi(); // 호출 가능

function sayHi(){
  console.log("안녕!");
}

    2) 함수 표현식 — 호이스팅 미적용

    // greet(); // ✗ ReferenceError

const greet = function(){
  console.log("Hello!");
};
greet(); // 실행 가능

    요약

    • fetch() — HTTP 통신 함수 (Promise 기반).
    • HTTP 메서드 — GET(조회), POST(등록), PUT(수정), DELETE(삭제).
    • localStorage / sessionStorage — 브라우저 내 데이터 저장소.
    • var / let / const — 스코프 및 재선언 여부 차이.
    • 스코프 — 변수의 유효 범위 (전역 / 함수 / 블록).
    • 호이스팅 — 선언이 최상단으로 끌어올려지는 JS의 동작 방식.
