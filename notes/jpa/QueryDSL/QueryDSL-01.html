    QueryDSL 기본 사용법

    1. QueryDSL이란 & JPQL과의 비교

    QueryDSL은 Java 코드로 타입 세이프(Type-safe)하게 JPA 쿼리를 작성할 수 있도록 도와주는 프레임워크이다. 문자열 기반의 JPQL과 달리 컴파일 시점에 문법 오류를 감지할 수 있다는 점이 가장 큰 장점이다.

    비교 항목 JPQL QueryDSL
    쿼리 작성 방식 문자열(String)로 작성한다. Java 메서드 체이닝으로 작성한다.
    오류 감지 시점 런타임(Runtime)에 오류가 발생한다. (서버 기동 후에야 알 수 있다.) 컴파일(Compile) 시점에 오류가 발생한다. (코딩 중에 바로 알 수 있다.)
    파라미터 바인딩 :param 형태로 직접 지정해야 한다. PreparedStatement로 자동 처리된다. SQL Injection을 방지하고 DB 쿼리 캐싱 성능에서도 유리하다.
    동적 쿼리 문자열 조합으로 만들어야 해서 복잡하고 실수가 잦다. BooleanBuilder나 where() 다중 파라미터로 깔끔하게 처리한다.

    2. 의존성 설정 (build.gradle)

    QueryDSL을 사용하기 위해서는 build.gradle에 다음 의존성을 추가해야 한다. Jakarta 네임스페이스 기반(Spring Boot 3.x)을 기준으로 한다. 

    // build.gradle

dependencies {
    // QueryDSL JPA 라이브러리
    implementation 'com.querydsl:querydsl-jpa:5.0.0:jakarta'

    // Q타입 자동 생성을 위한 어노테이션 프로세서
    annotationProcessor "com.querydsl:querydsl-apt:${dependencyManagement.importedProperties['querydsl.version']}:jakarta"
    annotationProcessor "jakarta.annotation:jakarta.annotation-api"
    annotationProcessor "jakarta.persistence:jakarta.persistence-api"
}
    Q타입 생성 방법 의존성을 추가한 후에는 QueryDSL이 사용하는 검증용 Q 타입 클래스를 직접 생성해야 한다. Q타입은 각 엔티티에 대응하는 메타 클래스로, QueryDSL이 타입 세이프하게 쿼리를 빌드하는 데 사용된다.

    아래 중 하나의 방법으로 생성할 수 있다.
    방법 1. IntelliJ Gradle 패널 → Tasks → other → compileJava 실행
    방법 2. 터미널에서 ./gradlew clean compileJava 실행

    초기화 방법: Gradle → Tasks → build → clean 을 실행하면 생성된 Q타입이 제거된다.
    ⚠️ Q타입은 Git에 포함하지 않는다 Q타입은 컴파일 시점에 자동 생성되는 파일이다. 소스 코드가 아니므로 .gitignore에 추가하여 버전 관리(Git)에 포함시키지 않아야 한다. 팀원이 각자 컴파일하면 자동으로 생성된다.

    3. JPAQueryFactory 설정 및 Q클래스 인스턴스

    JPAQueryFactory Bean 등록

    QueryDSL을 사용하려면 JPAQueryFactory를 스프링 Bean으로 등록해야 한다. EntityManager를 주입받아 생성하며, 실제 쿼리 실행은 이 객체가 담당한다. 

    @Configuration
public class QueryDslConfig {

    @PersistenceContext
    private EntityManager em;

    @Bean
    public JPAQueryFactory jpaQueryFactory() {
        return new JPAQueryFactory(em);
    }
}
    // Repository에서 사용 예시
@Repository
public class MemberQueryRepository {

    private final JPAQueryFactory queryFactory;

    // 생성자 주입
    public MemberQueryRepository(JPAQueryFactory queryFactory) {
        this.queryFactory = queryFactory;
    }

    public List<Member> findAll() {
        return queryFactory
            .selectFrom(member)   // QMember.member (static import)
            .fetch();
    }
}

    Q클래스 인스턴스 사용법 2가지 

    // 방법 1: 별칭(alias) 직접 지정
QMember qMember = new QMember("m");

// 방법 2: 기본 인스턴스 사용 (권장)
QMember qMember = QMember.member;
    언제 어떤 방법을 쓸까? 별칭 직접 지정은 같은 테이블을 두 번 이상 조인해야 하는 셀프 조인(Self Join) 상황에서만 사용한다. 이 경우 두 Q타입이 같은 별칭을 가지면 쿼리가 충돌하므로, 서로 다른 별칭을 부여해야 한다.

    그 외 모든 일반적인 경우에는 기본 인스턴스를 Static import해서 코드를 더 간결하게 유지한다. 
    // Static import 사용 시 (권장)
import static com.example.entity.QMember.member;

queryFactory.selectFrom(member).fetch(); // 깔끔하다

    4. 기본 검색 조건 (where 절)

    QueryDSL은 Q타입 필드의 메서드를 이용하여 WHERE 절의 조건을 표현한다. 아래는 실무에서 자주 쓰이는 조건 메서드 전체 목록이다. 

    // ===== 동등 / 비교 =====
member.username.eq("member1")       // username = 'member1'
member.username.ne("member1")       // username != 'member1'
member.username.eq("member1").not() // username != 'member1' (not()으로도 표현 가능)

// ===== NULL 체크 =====
member.username.isNotNull()         // username IS NOT NULL

// ===== 범위 =====
member.age.in(10, 20)               // age IN (10, 20)
member.age.notIn(10, 20)            // age NOT IN (10, 20)
member.age.between(10, 30)          // age BETWEEN 10 AND 30
member.age.goe(30)                  // age >= 30  (Greater Or Equal)
member.age.gt(30)                   // age > 30   (Greater Than)
member.age.loe(30)                  // age <= 30  (Less Or Equal)
member.age.lt(30)                   // age < 30   (Less Than)

// ===== 문자열 검색 =====
member.username.like("member%")     // LIKE 'member%'
member.username.contains("member")  // LIKE '%member%'
member.username.startsWith("member")// LIKE 'member%'
    where() 에 조건을 여러 개 넣으면? where()에 조건을 쉼표로 여러 개 넣으면 AND 조건으로 자동 결합된다. 조건 중 null이 들어오면 해당 조건은 무시된다. 이 특성을 이용하면 동적 쿼리를 매우 깔끔하게 작성할 수 있다. 
    // null이면 해당 조건은 자동으로 무시된다
queryFactory
    .selectFrom(member)
    .where(
        usernameEq(username),  // null이면 이 조건은 빠짐
        ageGoe(ageMin)         // null이면 이 조건은 빠짐
    )
    .fetch();

private BooleanExpression usernameEq(String username) {
    return username != null ? member.username.eq(username) : null;
}

    5. 결과 조회 메서드

    메서드 설명 주의사항
    fetch() 리스트로 반환한다. 결과가 없으면 빈 리스트([])를 반환한다. 가장 많이 사용한다.
    fetchOne() 단 건을 반환한다. 결과가 없으면 null을 반환한다. 결과가 2건 이상이면 NonUniqueResultException이 발생한다.
    fetchFirst() limit(1).fetchOne()과 동일하다. 첫 번째 결과 1건만 반환한다. -
    fetchResults() 페이징 정보(total count)를 포함한 QueryResults 객체를 반환한다. ⚠️ QueryDSL 5.0 이후 Deprecated. 복잡한 쿼리에서 count 쿼리가 비효율적으로 실행될 수 있다. 아래 설명 참조.
    fetchCount() count 쿼리를 별도로 실행하여 전체 건수를 long으로 반환한다. ⚠️ QueryDSL 5.0 이후 Deprecated. fetch().size() 또는 별도 count 쿼리 작성을 권장한다. 
    // 가장 기본적인 조회 패턴

// 1. 리스트 조회
List<Member> members = queryFactory
    .selectFrom(member)
    .fetch();

// 2. 단 건 조회 (2건 이상이면 예외 발생)
Member findMember = queryFactory
    .selectFrom(member)
    .where(member.username.eq("member1"))
    .fetchOne();

// 3. 첫 번째 1건만 조회
Member firstMember = queryFactory
    .selectFrom(member)
    .fetchFirst();
    ⚠️ fetchResults() / fetchCount() Deprecated 주의 fetchResults()는 내부적으로 count 쿼리를 자동으로 생성하는데, 조인이 포함된 복잡한 쿼리에서는 원본 쿼리의 모든 조인을 그대로 수행하기 때문에 성능이 떨어진다. 성능이 중요한 페이징 상황에서는 데이터 조회 쿼리와 count 쿼리를 별도로 2번 작성하는 것이 올바른 방법이다. 
    // 권장 패턴: 데이터 쿼리와 count 쿼리 분리
List<Member> content = queryFactory
    .selectFrom(member)
    .offset(pageable.getOffset())
    .limit(pageable.getPageSize())
    .fetch();

long total = queryFactory
    .select(member.count())
    .from(member)
    .fetchOne();

return new PageImpl<>(content, pageable, total);

    6. 정렬 (orderBy)

    // 정렬 예시
// 조건: 1. 나이 내림차순(desc), 2. 이름 오름차순(asc), 3. 이름이 null이면 마지막 출력
List<Member> result = queryFactory
    .selectFrom(member)
    .orderBy(
        member.age.desc(),
        member.username.asc().nullsLast()
    )
    .fetch();
    메서드 설명
    desc() 내림차순 정렬이다. (큰 값 → 작은 값)
    asc() 오름차순 정렬이다. (작은 값 → 큰 값)
    nullsLast() NULL 값을 가진 데이터를 가장 마지막에 출력한다.
    nullsFirst() NULL 값을 가진 데이터를 가장 먼저 출력한다.

    7. 페이징 (Paging)

    // 기본 페이징 - 1번째(0 index)부터 건너뛰고, 최대 2건 조회
List<Member> result = queryFactory
    .selectFrom(member)
    .orderBy(member.username.desc())
    .offset(1)   // 0부터 시작 (zero index). 1이면 첫 번째 결과를 건너뛴다.
    .limit(2)    // 최대 2건 조회
    .fetch();

assertThat(result.size()).isEqualTo(2);
    offset과 limit의 의미 offset(n): 결과 목록의 앞에서 n개를 건너뛴 뒤부터 가져온다. 0이면 처음부터 가져온다.
    limit(m): 최대 m개만 가져온다.

    예) 전체가 10건일 때 offset(2).limit(3)이면 3번째 ~ 5번째 결과를 가져온다.

    8. 집합 (Aggregation)

    select()에 집계 함수를 사용하면 반환 타입이 단일 엔티티가 아니므로 Tuple 타입으로 받아야 한다. 실무에서는 Tuple보다 DTO 프로젝션을 사용하는 경우가 많다. 

    // 팀별 평균 나이 조회 예시
List<Tuple> result = queryFactory
    .select(team.name, member.age.avg())
    .from(member)
    .join(member.team, team)
    .groupBy(team.name)
    // .having(team.name.eq("teamA"))  // HAVING 절 추가 예시
    .fetch();

Tuple teamA = result.get(0);
String teamName = teamA.get(team.name);        // "teamA"
Double avgAge  = teamA.get(member.age.avg());  // 15.0
    집계 함수 설명
    .count() 전체 건수를 반환한다.
    .sum() 합계를 반환한다.
    .avg() 평균을 반환한다.
    .max() 최댓값을 반환한다.
    .min() 최솟값을 반환한다. 
    // HAVING 절 예시: 가격이 1000 초과인 그룹만 필터링
queryFactory
    .select(item.price, item.count())
    .from(item)
    .groupBy(item.price)
    .having(item.price.gt(1000))
    .fetch();

    9. 조인 (Join)

    기본 조인 (Inner / Left / Right)

    조인의 기본 문법은 첫 번째 파라미터에 조인 대상, 두 번째 파라미터에 별칭으로 사용할 Q타입을 지정한다. 

    // teamA 소속인 모든 회원 조회 (Inner Join)
List<Member> result = queryFactory
    .selectFrom(member)
    .join(member.team, team)          // INNER JOIN member.team
    .where(team.name.eq("teamA"))
    .fetch();

// Left Join 예시
List<Tuple> leftResult = queryFactory
    .select(member, team)
    .from(member)
    .leftJoin(member.team, team)      // LEFT JOIN member.team
    .fetch();

    세타 조인 (Theta Join) — 연관관계 없는 조인

    FK 연관관계가 없는 필드를 기준으로 조인한다. from()에 조인할 엔티티를 나열하는 방식으로 작성한다. 단, 이 방법은 외부 조인(Outer Join)이 불가능하다는 단점이 있다. 외부 조인이 필요하면 on 절을 사용해야 한다. 

    // 이름(username)과 팀 이름(team.name)이 같은 회원 조회 (세타 조인)
List<Member> result = queryFactory
    .select(member)
    .from(member, team)               // 카테시안 곱 후 where로 필터
    .where(member.username.eq(team.name))
    .fetch();

    ON 절을 이용한 조인

    on()은 조인 시 추가 필터 조건을 걸거나, 연관관계 없는 엔티티를 외부 조인할 때 사용한다. 

    // 예시 1: 조인 대상 필터링 (teamA인 팀만 조인, 회원은 모두 조회)
// SQL: SELECT m.*, t.* FROM member m LEFT JOIN team t ON m.team_id = t.id AND t.name = 'teamA'
List<Tuple> result = queryFactory
    .select(member, team)
    .from(member)
    .leftJoin(member.team, team).on(team.name.eq("teamA"))
    .fetch();

// 예시 2: 연관관계 없는 엔티티 외부 조인
// SQL: SELECT m.*, t.* FROM member m LEFT JOIN team t ON m.username = t.name
List<Tuple> result2 = queryFactory
    .select(member, team)
    .from(member)
    .leftJoin(team).on(member.username.eq(team.name))  // 주의: team Q타입만 넣는다
    .fetch();
    일반 조인 vs ON 조인 문법 차이 연관관계가 있는 일반 조인에서는 .leftJoin(member.team, team)처럼 연관관계 경로와 Q타입을 함께 넣는다.
    연관관계 없는 엔티티를 ON으로 조인할 때는 .leftJoin(team).on(...)처럼 Q타입만 단독으로 넣는다. 이 차이를 혼동하면 안 된다.
    내부 조인(Inner Join)에서 ON 절 사용 시 주의 ON 절을 활용한 조인 대상 필터링 시, 내부 조인(INNER JOIN)이라면 ON 절 대신 where 절을 사용하는 것이 더 직관적이다. ON 절의 효과가 where 절과 동일하기 때문이다. ON 절은 외부 조인이 꼭 필요한 경우에만 활용하도록 한다.

    페치 조인 (Fetch Join)

    페치 조인은 SQL 기능이 아닌 JPA 전용 최적화 기능이다. 지연 로딩(LAZY)으로 설정된 연관 엔티티를 SQL 한 번에 함께 조회하여 N+1 문제를 해결한다. join() 뒤에 .fetchJoin()만 추가하면 된다. 

    // member를 조회할 때 연관된 team도 한 번에 가져온다 (N+1 방지)
Member findMember = queryFactory
    .selectFrom(member)
    .join(member.team, team).fetchJoin()
    .where(member.username.eq("member1"))
    .fetchOne();

    10. 서브쿼리 (SubQuery)

    서브쿼리는 JPAExpressions를 사용하여 작성한다. 이때 서브쿼리의 Q타입 별칭은 외부 쿼리의 Q타입과 달라야 하므로, 새로운 Q타입 인스턴스를 선언해야 한다. 

    import com.querydsl.jpa.JPAExpressions;

// 나이가 가장 많은 회원을 조회하는 서브쿼리 예시
QMember memberSub = new QMember("memberSub"); // 별칭이 겹치지 않도록 새로 선언

List<Member> result = queryFactory
    .selectFrom(member)
    .where(member.age.eq(
        JPAExpressions
            .select(memberSub.age.max())
            .from(memberSub)
    ))
    .fetch();

// IN 절 서브쿼리 예시: 평균 나이 이상인 회원 조회
List<Member> result2 = queryFactory
    .selectFrom(member)
    .where(member.age.goe(
        JPAExpressions
            .select(memberSub.age.avg())
            .from(memberSub)
    ))
    .fetch();
    ⚠️ FROM 절 서브쿼리(인라인 뷰)는 지원하지 않는다 JPA/JPQL의 한계로 FROM 절의 서브쿼리(인라인 뷰)는 지원하지 않는다. QueryDSL도 동일하게 지원하지 않는다. (SELECT 절과 WHERE 절의 서브쿼리는 지원한다.)

    FROM 절 서브쿼리 해결 방법 3가지
    1. 서브쿼리를 JOIN으로 변경한다. (가능한 상황도 있고, 불가능한 상황도 있다.)
    2. 쿼리를 2번 분리해서 실행한다. (애플리케이션 레이어에서 합친다.)
    3. nativeQuery를 사용한다. (최후의 수단)

    11. Case 문

    SELECT, WHERE, ORDER BY 절에서 사용 가능하다. 단순 조건은 .when().then()으로, 복잡한 조건은 CaseBuilder를 사용한다. 

    // 단순 Case 예시
List<String> result = queryFactory
    .select(member.age
        .when(10).then("열살")
        .when(20).then("스무살")
        .otherwise("기타"))
    .from(member)
    .fetch();

// 범위 조건이 필요한 복잡한 Case — CaseBuilder 사용
List<String> result2 = queryFactory
    .select(new CaseBuilder()
        .when(member.age.between(0, 20)).then("0~20살")
        .when(member.age.between(21, 30)).then("21~30살")
        .otherwise("기타"))
    .from(member)
    .fetch();

    Case 결과를 ORDER BY에 활용하기

    Case 문의 결과를 변수에 담아 orderBy()에 직접 활용할 수 있다. 복잡한 정렬 우선순위를 표현할 때 강력하다. 

    // 정렬 우선순위:
// 1순위: 0~20살 (rankPath = 2이지만 desc 정렬이므로 나중에 나옴)
// 2순위: 21~30살
// 3순위: 그 외 (rankPath = 3이 desc 정렬에서 가장 먼저 나옴)

NumberExpression<Integer> rankPath = new CaseBuilder()
    .when(member.age.between(0, 20)).then(2)
    .when(member.age.between(21, 30)).then(1)
    .otherwise(3);

List<Tuple> result = queryFactory
    .select(member.username, member.age, rankPath)
    .from(member)
    .orderBy(rankPath.desc())
    .fetch();

    12. 상수 · 문자 더하기

    상수 추가 — Expressions.constant()

    쿼리 결과에 DB에 없는 고정 상수 값을 함께 반환하고 싶을 때 사용한다. SQL에는 상수가 나가지 않고, QueryDSL이 결과 조합 시 추가해준다. 

    // select 에 고정 문자 "A"를 함께 반환
// 실제 SQL: SELECT member.username FROM member (상수 "A"는 SQL에 포함되지 않는다)
Tuple result = queryFactory
    .select(member.username, Expressions.constant("A"))
    .from(member)
    .fetchFirst();

// 결과: [member1, A]

    문자 더하기 — concat() · stringValue()

    문자열 타입 필드끼리는 concat()으로 이을 수 있다. 문자열이 아닌 타입(예: Integer인 age)은 반드시 stringValue()로 변환한 후 연결해야 한다. ENUM 타입을 문자열로 변환할 때도 stringValue()를 자주 사용한다. 

    // username + "_" + age 형태로 합쳐서 반환하는 예시
// age는 Integer 타입이므로 .stringValue()로 반드시 변환해야 한다
String result = queryFactory
    .select(member.username.concat("_").concat(member.age.stringValue()))
    .from(member)
    .where(member.username.eq("member1"))
    .fetchOne();

// 결과: "member1_10"

    13. 전체 문법 요약 치트시트

    기능 주요 메서드 / 사용법
    전체 조회 queryFactory.selectFrom(member).fetch()
