    문제 1 — 주사위 게임 3

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    int 배열과 Map을 활용했다. Map은 만약 같은 주사위 눈이 나온다면 그 횟수를 저장해서 switch문으로 분기하도록 했으며, 각 경우의 수를 메서드로 추출해서 주사위 눈을 구하도록 했다. 추가로 다른 사람 코드를 확인해보니, 초기에 정렬을 한 상태로 로직을 구현했으면 훨씬 간결하고 쉽게 해결할 수 있었다는 점을 알게되었다.

    문제 2 — 글자 이어 붙여 문자열 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    StringBuffer를 선언하고 my_string의 문자열을 문자 단위로 받도록 char 배열을 선언한다. 그리고 각 인덱스에 해당하는 글자들을 연결해서 반환한다.

    문제 3 — 9로 나눈 나머지

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문제에 핵심 내용이 적혀있다. "9로 나눈 나머지는 그 정수의 각 자리 숫자의 합을 9로 나눈 나머지와 같다" 즉, 자리수 합의 mod 9와 원래 수의 mod 9가 같다. 그리고 각 문자는 유니코드로 값을 가지기 때문에 숫자의 기준(0)을 뺀 수가 저장되도록 구현하면 됐다. 해당 문제는 20자리 수가 주어지기 때문에 int와 long으로 단순히 받는 건 불가능하다.

    문제 4 — 문자열 여러 번 뒤집기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    StringBuffer를 초기화 선언하고 인덱스들을 시작, 종료값으로 설정한 후 해당 범위에 속한 문자열들을 짤라낸다. 그리고 그 문자열들을 거꾸로 전환하고 StringBuffer에 교체한다.

    문제 5 — 배열 만들기 5

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    int[] 배열은 빈 공간을 0으로 채우기 때문에 짤라내야 하는 과정이 필요하다. 여기서는 List로 선언하고 반복문을 통해 문제의 조건을 반영했으며, 마지막에 stream과 메소드 참조를 통해 반환했다. 추가로 성능을 더 향상시키려면 반복문 내 substring 대신 직접 파싱하는 방법이 있었다. 그리고 박싱/생성/스트림을 최소화하면 더 빨라질 수 있을 것이다.
