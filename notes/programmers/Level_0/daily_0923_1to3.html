    문제 1 — 문자열 뒤집기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열에서 인덱스 s부터 e까지의 부분 문자열을 잘라낸 뒤, 이를 뒤집어서 다시 원래 문자열에 붙여 넣는 방식으로 문제를 해결했다.

    가장 간단한 방법은 세 부분(substring(0, s), substring(s, e+1), substring(e+1))으로 나누고, 중간 부분만 뒤집은 뒤 이어 붙이는 방식이다.

    추가적으로, toCharArray()를 이용해 문자 배열로 변환한 뒤, 양 끝에서 서로 교환하는 투 포인터(two-pointer) 방식으로도 구현할 수 있다. 이 방법은 별도의 문자열 객체를 새로 만들지 않아 효율적이다.

    또는 StringBuilder 하나만 사용하여 setCharAt()으로 직접 교환하는 방법도 가능하다. 상황에 따라 가장 읽기 쉽고 유지보수하기 좋은 방식을 선택하면 된다.

    문제 2 — 세로 읽기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열을 m개씩 끊어 가로로 채운다고 생각하면, c번째 열에 해당하는 문자들만 세로로 읽어 오면 된다.

    굳이 2차원 배열을 만들 필요 없이, 인덱스 규칙을 이용할 수 있다. 세로로 읽히는 위치는 c - 1, c - 1 + m, c - 1 + 2m, … 처럼 m씩 증가한다.

    따라서 반복문을 for (int i = c - 1; i < my_string.length(); i += m) 형태로 돌리면서 my_string.charAt(i)를 StringBuilder에 붙여주면 된다.

    이 방법은 문자열을 여러 번 잘라 만들 필요가 없어서 간단하고 효율적이다. 추가적으로, 전체 문자를 순회하면서 i % m == c - 1 조건을 만족하는 문자만 골라 붙이는 방식으로도 구현할 수 있다.

    문제 3 — qr code

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 전체를 순회하면서 i % q == r인 인덱스의 문자만 StringBuilder에 붙이는 방식으로 문제를 해결했다. 구현이 단순하고 직관적이라는 장점이 있다.

    추가적으로, 반복문의 시작점을 r로 두고 i += q씩 증가시키며 필요한 문자만 직접 가져오는 방법도 가능하다. 이 방식은 조건 검사가 필요 없으므로 약간 더 효율적이다.

    두 방법 모두 결과는 동일하며, 상황에 따라 가독성과 효율성 중 원하는 쪽을 선택하면 된다.
