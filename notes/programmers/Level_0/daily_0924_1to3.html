    문제 1 — 문자 개수 세기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열을 순회하면서 각 문자가 대문자인지 소문자인지 판별한 뒤, 문제에서 요구하는 52칸짜리 배열의 해당 인덱스를 증가시키는 방식으로 해결했다.

    구체적으로, 대문자는 'A'를 기준으로 빼서 0~25 인덱스에 매핑하고, 소문자는 'a'를 기준으로 빼서 26~51 인덱스에 매핑하면 된다. 이렇게 하면 A~Z, a~z의 빈도를 각각 배열에 저장할 수 있다.

    추가적으로, Map<Character, Integer>을 활용하는 방법도 있다. 각 문자의 빈도를 getOrDefault()로 누적한 뒤, 최종적으로 for문을 돌면서 A~Z, a~z 순서에 맞게 배열로 변환하면 문제 조건을 만족시킬 수 있다. 이 방식은 유연하지만, 변환 과정이 필요하므로 단순히 배열을 직접 채우는 방식이 더 직관적이다.

    문제 2 — 배열 만들기 1

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    1부터 n까지의 정수 중에서 k의 배수만 뽑아 배열로 반환하는 문제다. 배수의 개수는 정확히 n / k개이므로, 먼저 그 크기만큼 배열을 잡아두고(int[] answer = new int[n / k]), 인덱스 변수 idx를 이용해 채워 넣는다.

    구현은 1부터 n까지 순회하며(for (int i = 1; i ≤ n; i++)) i % k == 0인 값들만 answer[idx++] = i;로 담으면 된다. 이렇게 하면 조건을 만족하는 값들만 순서대로 배열에 저장된다.

    시간 복잡도는 전체를 한 번 도는 O(n), 추가 공간은 결과 배열 O(n/k)이다.

    문제 3 — 글자 지우기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 my_string에서 indices 배열에 해당하는 위치의 문자들을 삭제한 결과를 반환하는 문제다.

    처음에는 StringBuilder를 이용해 deleteCharAt()으로 지우는 방식을 생각했지만, 앞에서 문자가 삭제되면 뒤쪽 인덱스가 당겨지는 문제가 있었다. 이를 해결하려면 인덱스를 정렬한 뒤 뒤에서부터 차례대로 삭제해야 한다.

    보다 직관적인 방법으로는, 삭제해야 할 인덱스를 Set에 담아두고 문자열 전체를 순회하면서 해당 인덱스가 Set에 없는 경우에만 StringBuilder에 추가하는 방식이다. 이 접근은 인덱스 꼬임 문제를 피할 수 있고 코드 가독성도 좋다.
