    문제 1 — 카운트 다운

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 구간 [start_num, end_num]에서 start_num부터 1씩 감소시키며 배열에 담아 반환하는 문제다.

    두 수의 차이를 이용해 결과 배열의 크기를 먼저 계산할 수 있다. 구체적으로 start_num - end_num + 1개의 원소가 필요하므로, 이 길이를 가진 배열을 미리 생성한다.

    이후 for문을 돌면서 start_num에서 시작해 매 반복마다 1씩 줄여가며 배열에 채워 넣으면 된다. 인덱스 i번째 원소에는 start_num - i를 저장하는 식으로 구현하면 깔끔하다.

    시간 복잡도는 배열을 한 번 채우는 O(n), 추가 공간은 결과 배열 O(n)으로 충분하다.

    문제 2 — 가까운 1 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 arr과 시작 위치 idx가 있을 때, idx 이후에서 가장 가까운 값이 1인 원소의 인덱스를 반환하는 문제다.

    풀이 방법은 간단하다. 먼저 idx부터 배열 끝까지 순회하면서 원소가 1인 지점을 찾는다. 가장 먼저 발견되는 순간 그 인덱스를 반환하면 된다.

    만약 끝까지 순회했는데 1을 발견하지 못했다면, 조건에 맞는 값이 없으므로 -1을 반환한다.

    시간 복잡도는 최악의 경우 배열 전체를 한 번 도는 O(n), 추가 공간은 사용하지 않으므로 O(1)이다.

    문제 3 — 리스트 자르기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 num_list에서 슬라이싱 정보를 담은 slicer = {a, b, c}와 모드 n에 따라 부분 배열을 잘라 반환하는 문제다. 파이썬의 슬라이싱 문법과 달리 자바에서는 Arrays.copyOfRange()와 직접 순회를 조합해 구현한다.

    규칙은 다음과 같다.

    • n == 1 → num_list[0..b]
    • n == 2 → num_list[a..끝]
    • n == 3 → num_list[a..b]
    • n == 4 → num_list[a..b] 구간에서 간격 c로 추출

    구현 포인트는 두 가지다. 첫째, Arrays.copyOfRange(arr, from, to)는 from ≤ i < to 범위를 반환하므로, 상한 b를 포함하려면 to = b + 1로 지정해야 한다. 둘째, 간격 c가 필요한 n == 4의 경우에는 범용 슬라이서가 없으므로 for (int i = a; i ≤ b; i += c)로 직접 인덱스를 건너뛰며 선택하고, 임시 리스트에 담았다가 int[]로 변환한다.

    시간 복잡도는 잘라내는 구간 길이를 m이라 할 때 각 경우 모두 O(m), 추가 공간은 반환 배열에 해당하는 O(m)이다.
