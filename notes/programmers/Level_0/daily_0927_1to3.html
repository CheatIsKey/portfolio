    문제 1 — 배열 조각하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 arr와 query가 주어졌을 때, query의 각 인덱스를 순서대로 보면서 구간을 잘라내는 문제다. 규칙은 다음과 같다.

    • 짝수 번째 인덱스라면 해당 위치 이후의 원소들을 버린다.
    • 홀수 번째 인덱스라면 해당 위치 이전의 원소들을 버린다.

    이를 효율적으로 구현하기 위해 배열 전체를 매번 잘라내지 않고, 단순히 남겨야 하는 구간의 시작점(left)과 끝점(right)만 추적한다. 반복이 끝난 뒤에 최종적으로 남은 구간 [left, right]만 Arrays.copyOfRange(arr, left, right + 1)으로 잘라내면 된다.

    이렇게 하면 매번 새로운 배열을 생성하지 않고도 답을 구할 수 있어 시간과 메모리 모두 효율적이다.

    • right = left + q : 짝수 인덱스에서 뒤쪽을 자르는 경우
    • left = left + q : 홀수 인덱스에서 앞쪽을 자르는 경우

    시간 복잡도는 query 길이에 비례하는 O(m), 배열 복사는 마지막 한 번만 수행되므로 공간 사용도 최소화된다.

    추가적으로, 학습 과정에서 실제로 배열을 잘라내는 방식도 시도해 보았다. 예를 들어 ArrayList의 subList()를 사용하면 매 단계에서 눈으로 잘려나가는 과정을 확인할 수 있어 직관적으로 이해하기 쉽다. 다만 이 경우 매번 새로운 리스트와 배열 변환이 필요해 효율성은 떨어진다. 따라서 실전에서는 경계값만 관리하는 방식이 더 적합하다는 점을 배웠다.

    문제 2 — n 번째 원소부터

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 num_list에서 n번째 원소부터 끝까지 잘라 반환하는 문제다.

    가장 간단한 방법은 자바 표준 라이브러리의 Arrays.copyOfRange()를 활용하는 것이다. 시작 인덱스를 n - 1로 주고 끝 인덱스를 num_list.length로 주면, 원하는 구간을 한 줄로 추출할 수 있다. 

    return Arrays.copyOfRange(num_list, n - 1, num_list.length);

    직접 구현하는 방법도 가능하다. 우선 결과 배열을 원본 배열과 같은 크기로 준비한 뒤, 인덱스 n - 1부터 끝까지 순회하며 차례대로 결과 배열에 담는다. 사용된 원소 개수(cnt)만큼 채운 후, 마지막에 Arrays.copyOfRange(answer, 0, cnt)로 실제 길이에 맞게 잘라 반환한다.

    • 활용 풀이: 코드가 간결하고 배열 복사도 한 번만 발생한다.
    • 직접 구현 풀이: 라이브러리를 쓰지 않고도 결과를 만들 수 있다는 점에서 기초 훈련에 적합하다.

    두 방법 모두 정답을 구할 수 있으며, 시간 복잡도는 O(k) (여기서 k = num_list.length - (n - 1))이다.

    문제 3 — 순서 바꾸기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 num_list에서 앞쪽 n개의 원소를 잘라 뒤에 이어붙이는 문제다. 구현 방법은 다음과 같다.

    • Arrays.copyOfRange(num_list, n, num_list.length)으로 뒷부분을 추출한다.
    • Arrays.copyOfRange(num_list, 0, n)으로 앞부분을 추출한다.
    • 새 배열 result를 준비하고, System.arraycopy()로 두 구간을 순서대로 복사해 붙인다.

    이렇게 하면 두 구간을 정확히 이어붙인 새로운 배열을 얻을 수 있다. copyOfRange의 두 번째 인자가 상한(미포함)이라는 점을 주의해야 한다.

    시간 복잡도는 배열 복사에 필요한 O(n), 추가 공간은 결과 배열 하나가 필요하므로 O(n)이다.

    이 외에도 ArrayList를 활용해 뒷부분과 앞부분을 순서대로 추가한 뒤, 마지막에 stream().mapToInt()으로 int[]로 변환하는 방법도 있다. 구현은 조금 더 간단해 보이지만, 내부적으로 박싱/언박싱이 일어나기 때문에 성능 면에서는 배열을 직접 다루는 방식이 더 효율적이다.
