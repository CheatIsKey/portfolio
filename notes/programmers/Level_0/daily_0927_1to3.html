    문제 1 — 첫 번째로 나오는 음수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 num_list를 왼쪽부터 차례대로 순회하면서 처음으로 0보다 작은 값(num < 0)이 등장하는 위치의 인덱스를 즉시 반환하는 방식이다. 구체적으로 인덱스 i = 0부터 num_list.length - 1까지 반복하며 num_list[i] < 0이면 곧바로 i를 반환하고, 끝까지 찾지 못하면 -1을 반환한다.

    이 방법은 인덱스가 증가하는 순서대로 검사하므로, 반환되는 값이 “처음(가장 왼쪽)으로 등장하는 음수의 인덱스”임이 보장된다. 또한 조건을 만족하는 순간 탐색을 멈추기 때문에 불필요한 연산을 줄일 수 있다.

    • 배열이 비어 있으면 반복이 한 번도 실행되지 않아 -1을 반환한다.
    • 첫 원소가 음수면 즉시 0을 반환한다.
    • 반복 범위는 i < num_list.length로 설정해 인덱스 범위를 벗어나지 않도록 한다.

    문제 2 — 배열 만들기 3

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 arr에서 두 구간 [a, b], [c, d]를 잘라낸 뒤 이어붙여 반환하는 문제다. 풀이 접근은 비교적 단순하다. 우선 intervals 배열을 순회하면서 각 구간의 시작과 끝 인덱스를 꺼내온다. 그리고 해당 범위를 for문으로 돌면서 원소를 하나씩 리스트에 추가한다. 마지막에 리스트를 int[]로 변환하면 정답 배열을 얻을 수 있다.

    이 방식은 자바에서 파이썬처럼 [start:end] 슬라이싱 문법이 없다는 점을 고려해, 직접 반복문으로 잘라내는 아이디어다. 다만 이때 주의할 점은 인덱스 자체(i)를 리스트에 넣는 것이 아니라 반드시 배열 값(arr[i])을 넣어야 한다는 것이다. 그렇지 않으면 원하는 구간의 값이 아니라 단순히 숫자 인덱스 나열이 되기 때문이다.

    시간 복잡도는 자르는 구간의 길이를 m이라고 할 때 O(m), 추가 공간은 반환 배열에 해당하는 O(m)이 필요하다.

    추가적으로, 자바 표준 라이브러리에는 Arrays.copyOfRange()라는 슬라이싱 메서드가 제공되므로 이를 활용해 코드를 더 간결하게 만들 수 있다. 두 구간을 각각 copyOfRange()로 잘라낸 뒤 System.arraycopy()나 IntStream.concat()으로 합치는 방법도 있다. 이번 문제를 풀면서 이런 방식도 있다는 것을 함께 배웠다.

    문제 3 — 2의 영역

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 arr에서 값이 2인 원소가 등장하는 구간을 잘라내어 반환하는 문제다. 조건은 다음과 같다.

    • 만약 2가 한 번도 없다면 [-1]을 반환한다.
    • 2가 한 번 이상 존재한다면, 가장 처음 나타난 2의 위치부터 마지막 2의 위치까지 잘라낸 부분 배열을 반환한다.

    풀이 방식은 단순하다. 먼저 start와 end를 -1로 초기화하여 아직 2를 발견하지 못한 상태를 표시한다. 배열을 왼쪽부터 순회하면서 값이 2인 원소를 만날 때마다, start가 -1인 경우라면 첫 번째 등장이므로 start를 기록하고, 이후로는 계속 end를 갱신하여 마지막 2의 위치를 추적한다.

    순회를 마치고 start == -1이라면 배열에 2가 한 번도 없다는 뜻이므로 [-1]을 반환한다. 그렇지 않다면 Arrays.copyOfRange(arr, start, end + 1)을 이용해 start부터 end까지 구간을 잘라 반환한다. 이때 copyOfRange는 상한이 미포함이므로 반드시 end + 1을 지정해야 한다.

    시간 복잡도는 배열을 한 번 도는 O(n), 추가 공간은 결과 배열 크기 O(k) (여기서 k = end - start + 1)이다.
