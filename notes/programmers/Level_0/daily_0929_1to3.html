    문제 1 — 홀수 vs 짝수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    해설

    주어진 배열 num_list에서 인덱스가 짝수인 원소들의 합과 홀수인 원소들의 합을 구한 뒤, 두 값 중 더 큰 값을 반환하는 문제다.

    • 인덱스가 짝수(i % 2 == 0)일 때는 odd에 누적한다.
    • 인덱스가 홀수(i % 2 == 1)일 때는 even에 누적한다.
    • 마지막에 두 합 중 더 큰 값을 반환한다. (Math.max(odd, even))

    구현은 일반적인 for 루프를 돌며 조건문으로 짝수/홀수를 구분해 합을 누적하는 방식이다. 이 경우 배열을 한 번만 순회하면 되므로 시간 복잡도는 O(n)이고, 추가적인 자료구조 없이 정수 변수 두 개만 사용하므로 공간 복잡도는 O(1)이다.

    참고로, 같은 로직을 향상된 for문으로도 구현할 수 있다. 이때는 별도의 인덱스 변수를 두고 idx % 2로 짝수/홀수를 판별한다는 점만 다르다. 

    int odd = 0, even = 0;
int idx = 0;
for (int num : num_list) {
    if (idx % 2 == 0) odd += num;
    else even += num;
    idx++;
}
return Math.max(odd, even);

    두 방식 모두 직관적이며, 배열의 길이에 비례해 선형적으로 동작한다.

    문제 2 — 5명씩

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    학생들의 이름이 담긴 배열 names에서 앞에서부터 5명마다 한 명씩 뽑아 배열로 반환하는 문제다. 예를 들어 ["a","b","c","d","e","f","g"]라면, 0번째("a")와 5번째("f")가 선택된다.

    • 인덱스를 0부터 시작해, 5의 배수 위치(i % 5 == 0)에 있는 원소를 선택한다.
    • 선택된 원소들은 리스트에 담아두고, 마지막에 배열로 변환한다.

    구현은 간단히 for 루프에서 5칸씩 건너뛰며 해당 원소를 리스트에 추가하는 방식으로 할 수 있다. 이렇게 하면 불필요한 조건문 없이 바로 원하는 원소만 선택할 수 있다. 

    class Solution {
    public String[] solution(String[] names) {
        List<String> list = new ArrayList<>();
        for (int i = 0; i < names.length; i += 5) {
            list.add(names[i]);
        }
        return list.toArray(new String[0]);
    }
}

    시간 복잡도는 선택된 원소 개수에 비례하는 O(n/5), 즉 전체 길이에 대해 선형적으로 동작한다. 공간 복잡도는 결과 배열 크기 O(m)만큼만 필요하다.

    문제 3 — 할 일 목록

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 todo_list와 동일 길이의 불리언 배열 finished가 주어졌을 때, 아직 완료되지 않은(false) 할 일만 순서대로 골라 배열로 반환하는 문제다.

    • finished[i] == false이면 todo_list[i]를 결과에 포함한다.
    • 입력 순서를 보존해야 하므로 앞에서부터 순차적으로 확인한다.

    구현은 단순히 for 루프로 인덱스를 순회하면서, finished[i]가 false인 경우만 리스트에 추가한 뒤 마지막에 배열로 변환해 반환하면 된다. 추가 자료구조가 필요 없고 가장 직관적이다. 

    import java.util.*;

class Solution {
    public String[] solution(String[] todo_list, boolean[] finished) {
        List<String> list = new ArrayList<>();

        for (int i = 0; i < todo_list.length; i++) {
            if (!finished[i]) list.add(todo_list[i]);
        }

        return list.toArray(new String[0]);
    }
}

    시간 복잡도는 배열을 한 번 순회하므로 O(n), 공간 복잡도는 미완료 항목 개수를 m이라 할 때 결과 배열 크기만큼 O(m)이 필요하다.
