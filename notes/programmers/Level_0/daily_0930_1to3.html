    문제 1 — n보다 커질 때까지 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 numbers의 원소들을 앞에서부터 차례대로 더하다가, 누적합이 n보다 커지는 순간의 합을 반환하는 문제다.

    • 배열을 순서대로 순회하면서 sum에 값을 더한다.
    • sum이 n보다 커지는 즉시 반복문을 종료한다.
    • 종료 시점의 sum을 반환한다.

    구현은 간단히 for-each 문을 이용해 배열을 순회하면서 누적합을 구하고, 조건을 만족하면 break로 탈출하는 방식이다. 배열을 한 번만 확인하므로 시간 복잡도는 O(n), 추가 메모리를 쓰지 않으므로 공간 복잡도는 O(1)이다.

    직관적이고 가장 단순한 접근 방식이며, 불필요한 조건문이나 자료구조 없이 효율적으로 해결할 수 있다.

    문제 2 — 수열과 구간 쿼리 1

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 arr에 대해, 각 쿼리 [start, end] 범위 안의 원소들을 모두 1씩 증가시키는 문제다. 모든 쿼리를 순서대로 처리한 후 최종 배열을 반환한다.

    • 먼저 입력 배열을 List<Integer>로 변환해 원소를 갱신하기 쉽게 만든다.
    • 각 쿼리를 읽어 start부터 end까지 순회하며 list.set(i, list.get(i) + 1)로 값을 1씩 증가시킨다.
    • 모든 쿼리를 처리한 뒤 최종 리스트를 다시 int[]로 변환해 반환한다.

    이 방식은 배열을 직접 다루기보다는 리스트의 set 메서드를 활용하는 점이 특징이다. 직관적이고 구현이 명확하지만, 변환 과정이 있어 배열을 직접 수정하는 것보다 살짝 비효율적일 수 있다.

    시간 복잡도는 쿼리 개수를 m, 배열 길이를 n이라 할 때 최악의 경우 O(m·n)이 될 수 있으며, 공간 복잡도는 O(n)이다.

    추가적으로

    배열을 List로 변환하지 않고, arr를 직접 수정하는 방식도 가능하다. 이 경우 변환 과정이 사라져 코드가 더 간단하고 효율적이다. 다만 문제 해결 아이디어 자체는 동일하다.

    문제 3 — 조건에 맞게 수열 변환하기 1

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 arr의 각 원소에 대해 조건에 따라 값을 변환하는 문제다.

    • 원소가 50 이상이고 짝수이면 2로 나눈다.
    • 원소가 50 미만이고 홀수이면 2를 곱한다.
    • 그 외의 경우는 값이 변하지 않는다.

    구현은 단순히 for 루프를 돌면서 각 원소를 조건문으로 판별해 바로 갱신하는 방식이다. 따라서 배열을 한 번만 순회하므로 시간 복잡도는 O(n), 추가 메모리를 쓰지 않으므로 공간 복잡도는 O(1)이다.

    조건 분기만 정확히 구현하면 되며, 별도의 자료구조나 추가적인 최적화는 필요 없는 직관적인 문제다.
