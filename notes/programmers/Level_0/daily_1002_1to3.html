    문제 1 — 조건에 맞게 수열 변환하기 2

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 arr의 각 원소를 조건에 따라 반복적으로 변환하면서, 더 이상 변화가 없을 때까지의 반복 횟수를 구하는 문제다.

    • 반복문 안에서 배열의 각 원소를 새 배열 newArr로 변환한다.
    • 변환 규칙은 다음과 같다:
      • 값 ≥ 50이고 짝수라면 2로 나눈다.
      • 값 < 50이고 홀수라면 값 × 2 + 1로 바꾼다.
      • 그 외의 경우는 그대로 둔다.
    • 변환 결과가 이전 배열과 완전히 동일하다면 반복을 종료한다.
    • 그 외의 경우, arr = newArr로 갱신하고 반복 횟수(count)를 1 증가시킨다.

    즉, 배열의 상태가 안정화될 때까지 변환 과정을 시뮬레이션하는 형태다. Arrays.equals()를 이용해 이전 상태와 현재 상태를 비교하여 변화가 멈추는 시점을 정확히 판별한다.

    시간 복잡도는 각 단계마다 배열을 한 번 순회하므로 O(k·n)이다. 여기서 n은 배열 길이, k는 안정화까지의 반복 횟수다. 추가 배열 하나(newArr)만 사용하므로 공간 복잡도는 O(n)이다.

    문제의 핵심은 “배열이 변하지 않을 때까지 규칙적으로 갱신”하는 시뮬레이션 로직이며, 주어진 조건문을 정확히 구현하면 자연스럽게 해결된다.

    문제 2 — 1로 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 num_list의 각 원소에 대해, 1이 될 때까지 특정 규칙에 따라 값을 나누는 과정을 반복하며 총 연산 횟수를 구하는 문제다.

    • 배열의 각 원소를 순회하면서, 해당 값이 1이 될 때까지 반복한다.
    • 현재 값이 짝수라면 num_list[i] /= 2로 나눈다.
    • 홀수라면 (num_list[i] - 1) / 2로 계산한다.
    • 각 나눗셈 연산마다 count를 1씩 증가시킨다.

    예를 들어, 원소가 10이라면 다음과 같이 변한다: 10 → 5 → 2 → 1 (총 3회 연산). 모든 원소에 대해 이 과정을 수행한 뒤 누적 연산 횟수를 반환한다.

    반복문은 각 원소가 1이 될 때까지 진행되므로, 연산 횟수는 각 숫자의 크기에 비례해 로그 형태로 감소한다. 따라서 전체 시간 복잡도는 O(n·log m)이며, m은 배열 내 최댓값이다. 추가적인 자료구조를 사용하지 않으므로 공간 복잡도는 O(1)이다.

    문제의 핵심은 “짝수면 나누기 2, 홀수면 (n−1)/2” 규칙을 정확히 적용하며 반복 횟수를 세는 시뮬레이션 구현이다. 단순하지만 루프의 종료 조건(num_list[i] != 1)을 명확히 처리해야 한다.

    문제 3 — 길이에 따른 연산

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 num_list의 길이에 따라 두 가지 연산 방식 중 하나를 선택하여 결과를 반환하는 문제다.

    • 배열의 길이가 11 이상이면 모든 원소의 합을 구한다.
    • 그보다 짧으면 모든 원소의 곱을 구한다.

    이를 위해 if 조건문으로 배열 길이를 판별하고, 해당 조건에 맞는 반복문을 수행한다.

    • sum은 원소 합계를 저장하며,
    • mul은 원소 곱을 저장한다.
    • 조건에 따라 하나만 유효하게 계산되므로, 반환 시 sum == 0 ? mul : sum 형태로 구분한다.

    예를 들어, num_list = [2, 3, 4]이면 길이가 3으로 11 미만이므로 2 × 3 × 4 = 24를 반환한다. 반면 num_list의 길이가 11 이상이면 모든 원소의 합을 계산해 반환한다.

    배열을 한 번만 순회하므로 시간 복잡도는 O(n), 추가 메모리를 사용하지 않아 공간 복잡도는 O(1)이다.

    핵심은 배열의 길이에 따라 “합 또는 곱” 중 적절한 연산을 선택하는 분기 처리이며, 문제의 요구 조건을 그대로 코드에 옮긴 직관적인 구현이다.
