    문제 1 — 원하는 문자열 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 myString이 특정 문자열 pat을 (대소문자를 구분하지 않고) 포함하고 있는지를 판별하는 문제다.

    • 두 문자열 모두 소문자로 변환해 비교하면, 대소문자 구분 없이 동일한 조건으로 확인할 수 있다.
    • String.toLowerCase() 메서드로 변환한 뒤, contains() 메서드를 사용해 부분 문자열 포함 여부를 검사한다.
    • 포함되어 있다면 1, 아니라면 0을 반환한다.

    예를 들어, myString = "AbCdEfG", pat = "aBc"인 경우 모두 소문자로 변환하면 "abcdefg"과 "abc"가 되어 contains() 결과가 true이므로 1을 반환한다.

    문자열 내 포함 여부만 확인하므로 시간 복잡도는 O(n)이며, 이는 문자열 길이에 비례한다. 추가 메모리를 거의 사용하지 않으므로 공간 복잡도는 O(1)이다.

    핵심은 “대소문자를 무시하고 부분 문자열 포함 여부를 판별”하는 점이며, Java의 기본 문자열 메서드만으로 간단하게 해결할 수 있는 직관적인 문제다.

    문제 2 — 대문자로 바꾸기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 myString의 모든 알파벳을 대문자로 변환해 반환하는 문제다.

    • Java의 문자열 메서드 toUpperCase()를 사용하면 문자열 내 알파벳을 손쉽게 모두 대문자로 변환할 수 있다.
    • 영문 이외의 문자(숫자, 기호, 공백 등)는 영향을 받지 않고 그대로 유지된다.
    • 변환된 결과 문자열을 그대로 반환하면 된다.

    예를 들어, myString = "Hello World"라면 toUpperCase() 호출 결과는 "HELLO WORLD"가 된다.

    내부적으로는 각 문자를 순회하면서 알파벳의 유니코드 값을 이용해 변환하므로 시간 복잡도는 문자열 길이에 비례하는 O(n)이다. 새로운 문자열을 생성하기 때문에 공간 복잡도는 O(n)이다.

    문제의 핵심은 문자열 내 문자의 대소문자 변환이며, Java 표준 메서드를 그대로 활용하면 가장 간결하고 효율적으로 해결할 수 있다.

    문제 3 — 소문자로 바꾸기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 myString의 모든 알파벳을 소문자로 변환해 반환하는 문제다.

    • Java의 문자열 메서드 toLowerCase()를 사용하면 문자열 내 알파벳을 손쉽게 모두 소문자로 변환할 수 있다.
    • 영문자가 아닌 문자(숫자, 기호, 공백 등)는 영향을 받지 않고 그대로 유지된다.
    • 변환된 결과 문자열을 그대로 반환하면 된다.

    예를 들어, myString = "Hello World!"라면 toLowerCase()의 호출 결과는 "hello world!"가 된다.

    내부적으로는 각 문자를 순회하면서 알파벳 문자를 소문자로 변환하므로 시간 복잡도는 문자열 길이에 비례하는 O(n)이고, 변환된 새 문자열을 생성하므로 공간 복잡도는 O(n)이다.

    이 문제의 핵심은 문자열 내 알파벳을 모두 소문자로 변환하는 것이며, Java의 표준 문자열 메서드만으로 간단하고 직관적으로 해결할 수 있다.
