    문제 1 — 배열에서 문자열 대소문자 변환하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 배열 strArr의 각 원소에 대해, 인덱스가 짝수이면 소문자로, 홀수이면 대문자로 변환한 후 다시 배열을 반환하는 문제다.

    • 반복문을 통해 배열을 순회하면서 인덱스 i를 확인한다.
    • i % 2 == 0이면 strArr[i].toLowerCase()로 소문자로 변환한다.
    • 그 외의 경우(즉 i % 2 == 1)에는 strArr[i].toUpperCase()로 대문자로 변환한다.
    • 변환된 문자열을 다시 배열에 저장하고, 반복이 끝나면 최종 배열을 반환한다.

    예를 들어, strArr = ["aBc", "Def", "GhI"] 라면 인덱스 0 → 소문자, 인덱스 1 → 대문자, 인덱스 2 → 소문자로 바꿔 결과는 ["abc", "DEF", "ghi"]가 된다.

    배열을 한 번만 순회하므로 시간 복잡도는 O(n)이며, 변환된 문자열을 배열에 바로 덮어쓰므로 공간 복잡도는 추가 구조 없이 O(1)이다 (입력 배열 내에서 변경).

    이 문제의 핵심은 인덱스의 짝수/홀수 여부에 따라 서로 다른 문자열 메서드를 적용하는 조건 분기이며, Java의 기본 문자열 메서드만으로 간단하고 명료하게 해결할 수 있다.

    문제 2 — A 강조하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 myString이 주어질 때, 조건에 따라 각 문자를 변환한 새 문자열을 반환하는 문제다. 구체적으로는:

    • 문자 'a'는 모두 대문자 'A'로 변환한다.
    • 이미 'A'인 문자는 그대로 유지한다.
    • 그 밖의 모든 대문자 알파벳은 소문자로 변환한다.
    • 소문자 알파벳 (단, 'a'는 위 규칙) 은 그대로 유지한다.

    주어진 코드에서는 다음 방식으로 구현했다:

    • 문자열을 순회하며 각 문자를 char c = myString.charAt(i)로 가져온다.
    • 만약 c == 'a'이면, 결과로 'A'를 더한다.
    • 그 외에 c가 대문자 범위('A' ≤ c ≤ 'Z')이고 c != 'A'이면, Character.toLowerCase(c)를 사용해 소문자로 변환한다.
    • 그 외의 경우(소문자이거나 대문자가 아닌 문자는) 그대로 c를 더한다.
    • 최종적으로 StringBuilder에 쌓은 문자열을 반환한다.

    예를 들어, myString = "AbstraCt aAlphA" 같은 경우를 보면: - 소문자 'a'는 'A'로 바뀌고, - 기존의 'A'는 유지되며, - 그 외 대문자(예: 'C', 'T', 'A' 아닌 대문자)는 소문자가 된다. 결과로 “AbstrAct AAlphA” 와 같은 패턴이 나올 수 있다.

    이 문제는 각 문자에 대한 조건 판단 + 문자 단위 변환을 정확히 구현하는 것이 핵심이다. 문자열 길이를 한 번 순회하므로 시간 복잡도는 O(n)이며, 변환된 새 문자열을 생성하므로 공간 복잡도는 O(n)이다.

    문제 3 — 특정한 문자를 대문자로 바꾸기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string 안에서 특정 문자 alp를 찾아 해당 문자를 모두 대문자로 바꾼 뒤 반환하는 문제다.

    • 먼저 alp.charAt(0)으로 대문자로 바꿀 대상 문자를 target에 저장한다.
    • for문으로 문자열을 순회하면서 각 문자를 char c = my_string.charAt(i)로 읽는다.
    • 현재 문자가 target과 같다면 Character.toUpperCase(c)로 변환해 추가한다.
    • 그 외의 경우는 그대로 StringBuilder에 추가한다.
    • 모든 문자를 처리한 후 sb.toString()을 반환하면 완성된다.

    예를 들어, my_string = "programmers", alp = "p"일 때 첫 번째 문자 ‘p’가 대문자로 바뀌어 "Programmers"가 된다. 같은 문자가 여러 개 있다면 모두 변환된다.

    문자열을 한 번만 순회하므로 시간 복잡도는 O(n), 새 문자열을 생성하므로 공간 복잡도는 O(n)이다.

    핵심은 대상 문자를 정확히 식별하고, Character.toUpperCase()를 사용해 효율적으로 대문자로 변환하는 부분이다.
