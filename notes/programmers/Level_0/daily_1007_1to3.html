    문제 1 — 공백으로 구분하기 1

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 my_string을 공백(" ") 기준으로 분리해 각 단어를 배열 형태로 반환하는 문제다.

    • String.split(" ") 메서드를 사용하면 지정한 구분자(여기서는 공백)를 기준으로 문자열을 나눌 수 있다.
    • 분리된 문자열들은 자동으로 String[] 배열로 반환된다.
    • 추가적인 후처리(공백 제거 등)는 요구되지 않으므로 split(" ") 결과를 그대로 반환하면 된다.

    예를 들어, my_string = "i love java"라면 split(" ")의 결과는 ["i", "love", "java"]가 된다.

    문자열을 한 번만 순회하므로 시간 복잡도는 O(n), 단어 개수만큼 새로운 문자열 배열이 생성되므로 공간 복잡도는 O(n)이다.

    핵심은 split() 메서드를 이용해 문자열을 공백 기준으로 손쉽게 분리하는 점이다. Java의 기본 문자열 처리 메서드만으로 간결하게 해결할 수 있는 문제다.

    문제 2 — 공백으로 구분하기 2

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string을 공백을 기준으로 분리하되, 연속된 공백이 여러 개여도 하나의 구분자로 처리하여 단어들을 배열 형태로 반환하는 문제다.

    • trim() 메서드로 문자열의 앞뒤 공백을 제거한다.
    • 그다음 split("\\s+")을 사용해 하나 이상의 공백(탭, 줄바꿈 포함) 기준으로 문자열을 분리한다.
    • 결과적으로 단어 사이에 공백이 여러 개 있더라도 중복 공백을 무시하고 정확히 단어만 추출할 수 있다.

    예를 들어, my_string = " Java is fun "일 때 trim()으로 양쪽 공백을 제거하면 "Java is fun"이 되고, split("\\s+") 결과는 ["Java", "is", "fun"]이 된다.

    문자열을 한 번만 순회하여 분리하므로 시간 복잡도는 O(n), 분리된 단어 개수만큼 배열이 생성되어 공간 복잡도는 O(n)이다.

    이 문제의 핵심은 split()에 전달되는 정규식 "\\s+"를 활용해 “연속된 공백도 하나의 구분자로 처리”하는 점이다. 단순한 split(" ")과 달리, 불필요한 빈 문자열이 결과에 포함되지 않는다.

    문제 3 — x 사이의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 myString을 문자 'x'를 기준으로 분리하고, 각 구간(분리된 문자열)의 길이를 배열로 반환하는 문제다.

    • split("x", -1)을 사용해 문자열을 'x' 기준으로 나눈다.
    • 두 번째 인자로 -1을 지정하면, 문자열 끝에 'x'가 포함되어 있을 때 생기는 빈 문자열("")도 결과에 포함시킬 수 있다.
    • 분리된 문자열 배열을 순회하며 각 요소의 길이를 구해 새로운 정수 배열에 저장한다.

    예를 들어, myString = "oxooxoxx"인 경우 split("x", -1) 결과는 ["o", "oo", "o", "", ""]이며, 각 문자열의 길이를 구하면 [1, 2, 1, 0, 0]이 된다.

    이처럼 문자열 내 'x'의 개수와 구간 사이의 문자 수를 그대로 정수 배열 형태로 표현할 수 있다.

    문자열을 한 번만 순회하므로 시간 복잡도는 O(n), 분리된 문자열 개수만큼 배열이 생성되므로 공간 복잡도는 O(n)이다.

    핵심은 split()의 두 번째 인자 -1을 이용해 “문자열 끝의 공백 구간까지 유지”하는 점이며, 이를 통해 마지막에 'x'로 끝나는 경우도 정확히 처리할 수 있다.
