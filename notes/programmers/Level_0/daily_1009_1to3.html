    문제 1 — 문자열 잘라서 정렬하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 myString을 문자 'x' 기준으로 나누고, 빈 문자열을 제외한 나머지 부분 문자열들을 사전순으로 정렬해 배열 형태로 반환하는 문제다.

    • split("x")으로 문자열을 'x' 기준으로 분리한다.
    • 이때 "xx"처럼 'x'가 연속되면 빈 문자열("")이 생기므로 이를 제거해야 한다.
    • Arrays.stream(split)을 통해 스트림으로 변환한 뒤, filter(s -> !s.isEmpty())로 빈 문자열을 걸러낸다.
    • 그다음 sorted()를 호출해 문자열을 사전순으로 정렬하고, toArray(String[]::new)으로 다시 배열로 변환한다.

    예를 들어, myString = "axbxcxdx"일 경우 split("x") 결과는 ["a", "b", "c", "d", ""]이며, 빈 문자열을 제거하고 정렬하면 ["a", "b", "c", "d"]가 된다.

    문자열을 한 번 분리하고 정렬하기 때문에 전체 시간 복잡도는 O(n log n)이며, 분리된 문자열 배열을 저장하므로 공간 복잡도는 O(n)이다.

    핵심은 split()으로 나눈 결과에서 빈 문자열을 제거하고, 스트림 API의 filter()와 sorted()를 적절히 조합해 간결하게 구현하는 점이다.

    문제 2 — 간단한 식 계산하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 형태의 간단한 수식 binomial이 주어질 때, 덧셈(+), 뺄셈(-), 곱셈(*) 중 해당 연산을 수행한 결과를 반환하는 문제다.

    • split(" ")을 사용해 공백을 기준으로 수식을 세 부분으로 나눈다.
    • 왼쪽 피연산자 a, 연산자 op, 오른쪽 피연산자 b를 각각 추출한다.
    • Integer.parseInt()로 문자열을 정수로 변환한 뒤, 연산자에 따라 해당 산술 연산을 수행한다.
    • Java 14 이상에서 지원하는 switch 식을 사용해 코드의 가독성을 높였다.

    예를 들어, binomial = "3 * 4"인 경우 parts = ["3", "*", "4"]로 분리되고, a = 3, b = 4, op = "*"가 되어 연산 결과는 12가 된다.

    문자열 분리와 간단한 산술 연산만 수행하므로 시간 복잡도는 O(1), 공간 복잡도도 O(1)이다.

    핵심은 문자열에서 피연산자와 연산자를 정확히 분리하고, switch 문으로 연산을 명확하게 분기 처리하는 점이다. 단, 문제에서 주어지지 않은 연산자가 등장할 경우 예외를 던지도록 구현하면 안전하다.

    문제 3 — 문자열 바꿔서 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 myString의 모든 문자를 변환한 뒤, 변환된 문자열 안에 부분 문자열 pat이 포함되어 있는지를 확인하는 문제다.

    • myString의 각 문자를 순회하며 다음 규칙으로 변환한다:
      • 'A' → 'B'
      • 'B' → 'A'
    • 변환 결과를 StringBuilder에 차례대로 추가한다.
    • 모든 문자를 처리한 뒤, 완성된 문자열에 대해 contains(pat) 메서드를 사용해 pat이 포함되어 있는지 검사한다.
    • 포함되어 있으면 1, 아니면 0을 반환한다.

    예를 들어, myString = "ABBA", pat = "AA"일 때 문자 변환 결과는 "BAAB"이 되며, "BAAB" 안에는 "AA"가 포함되므로 결과는 1이다.

    문자열을 한 번 순회하면서 새 문자열을 생성하므로 시간 복잡도는 O(n), 변환된 문자열을 저장하기 위한 StringBuilder가 필요하므로 공간 복잡도도 O(n)이다.

    핵심은 문자 변환 규칙을 정확히 적용한 뒤, 변환된 결과 문자열에서 부분 문자열 탐색을 수행하는 것이다. StringBuilder를 사용하면 매번 문자열을 새로 만드는 비효율을 줄일 수 있다.
