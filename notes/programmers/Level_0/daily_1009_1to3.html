    문제 1 — rny_string

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 rny_string 안에서 모든 'm' 문자를 문자열 "rn"으로 바꾼 결과를 반환하는 문제다.

    • String.replaceAll() 메서드를 사용해 대상 문자와 대체 문자열을 지정하면 된다.
    • 첫 번째 인자에는 바꿀 문자("m"), 두 번째 인자에는 대체 문자열("rn")을 전달한다.
    • 모든 'm'이 "rn"으로 바뀐 새 문자열이 반환된다.

    예를 들어, rny_string = "programmer"라면 "programmer" → "prorngramrner"로 변환되어 반환된다.

    문자열을 한 번만 순회하며 변환하므로 시간 복잡도는 O(n), 새로운 문자열을 생성하므로 공간 복잡도는 O(n)이다.

    핵심은 replaceAll()을 이용해 간단히 모든 'm'을 "rn"으로 치환하는 것이며, 정규식 기능을 활용하지만 이 경우 단순 문자열 치환으로 충분하다.

    문제 2 — 세 개의 구분자

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 myStr에서 구분자 'a', 'b', 'c'를 기준으로 나누고, 비어 있지 않은 부분 문자열만 추출해 배열로 반환하는 문제다. 만약 모든 부분이 빈 문자열이라면 ["EMPTY"]를 반환한다.

    • split("a|b|c")를 사용하면 정규식 OR 패턴으로 'a' 또는 'b' 또는 'c'를 기준으로 문자열이 분리된다.
    • Arrays.stream()을 통해 분리된 문자열 배열을 스트림으로 변환한 뒤, filter(s -> !s.isEmpty())로 빈 문자열을 제거한다.
    • 그 결과를 toArray(String[]::new)로 다시 문자열 배열로 변환한다.
    • 만약 유효한 문자열이 하나도 없다면, 조건 연산자를 이용해 new String[]{"EMPTY"}를 반환하도록 처리한다.

    예를 들어, myStr = "baconlettucetomato"일 경우 "a", "b", "c"를 기준으로 분리하면 ["", "", "onlettu", "etom", "to"]가 되고, 빈 문자열을 제거하면 ["onlettu", "etom", "to"]가 반환된다.

    문자열을 한 번만 순회하므로 시간 복잡도는 O(n), 분리된 결과를 저장하기 위한 배열과 스트림이 필요하므로 공간 복잡도는 O(n)이다.

    핵심은 정규식 분리(split("a|b|c"))와 스트림 필터링(filter)을 조합해 간결하게 불필요한 공백 문자열을 제거하고 결과를 정리하는 것이다.

    문제 3 — 배열의 원소만큼 추가하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 arr의 각 원소 i를, 그 값만큼 반복하여 새로운 배열로 만드는 문제다.

    • List<Integer>를 사용해 임시로 값을 누적한다.
    • 바깥 for문에서 배열의 각 원소 i를 순회하고, 내부 for문을 통해 i를 i번 반복하여 list에 추가한다.
    • 모든 반복이 끝나면, 리스트의 크기만큼 정수 배열 answer를 생성한 뒤 for문으로 리스트의 값을 배열로 복사한다.
    • 마지막에 완성된 배열을 반환한다.
    예를 들어, arr = [5, 1, 4]인 경우
      5 → [5, 5, 5, 5, 5]
      1 → [1]
      4 → [4, 4, 4, 4]
      이므로 최종 결과는 [5, 5, 5, 5, 5, 1, 4, 4, 4, 4]가 된다.

    중첩 반복문을 사용하므로 시간 복잡도는 O(n * m) 정도로 볼 수 있으며(여기서 m은 원소의 평균 크기), 공간 복잡도는 최종 생성되는 배열의 크기에 비례하는 O(k)이다.

    핵심은 각 원소의 값을 그대로 그 횟수만큼 반복 추가하는 단순한 구현이며, 리스트를 사용해 동적으로 크기를 확장한 뒤 마지막에 배열로 변환하는 접근이 깔끔하다.
