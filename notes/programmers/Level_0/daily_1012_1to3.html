    문제 1 — 배열의 길이를 2의 거듭제곱으로 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 arr의 길이를 가장 가까운 2의 거듭제곱 크기로 맞추는 문제다. 만약 배열의 길이가 이미 2의 거듭제곱이면 그대로 반환하고, 아니라면 2의 거듭제곱이 될 때까지 0을 채워 배열의 길이를 확장한다.

    • 먼저 arr.length를 size 변수에 저장하고, i를 1로 초기화한다.
    • while (size > i) 반복문을 통해 i를 2배씩 증가시키면서 size 이상이 되는 최소의 2의 거듭제곱 값을 찾는다.
    • 이후 새 배열 ints를 크기 i로 생성한다. 자바의 int 배열은 기본값이 0으로 초기화되므로 Arrays.fill(ints, 0)은 생략해도 무방하지만, 명시적으로 0을 채워 의도를 분명히 했다.
    • 기존 배열 arr의 원소를 새 배열로 복사하기 위해 System.arraycopy(arr, 0, ints, 0, arr.length)를 사용한다. 이 메서드는 원본 배열의 내용을 대상 배열로 빠르게 복사한다.
    • 마지막으로 완성된 배열 ints를 반환한다.

    예를 들어, arr = [1, 2, 3]인 경우 길이는 3이므로 다음 2의 거듭제곱은 4이다. 따라서 새 배열은 길이 4로 확장되고 기존 값 3개 뒤에는 0이 채워진다. 

    
      arr = [1, 2, 3]
      i = 1 → 2 → 4 (3보다 크거나 같을 때까지)
      ints = [0, 0, 0, 0]
      복사 후 → [1, 2, 3, 0]

    배열 복사는 O(n) 시간에 수행되며, 메모리 역시 확장된 배열 크기에 비례한다. 전체 알고리즘의 시간 복잡도는 O(n), 공간 복잡도는 O(n)이다.

    핵심은 다음 2의 거듭제곱 크기를 구하고, 기존 배열을 그 크기의 새 배열로 옮기는 과정이다. 자바의 System.arraycopy()와 Arrays.fill()을 적절히 활용해 간결하고 효율적인 방식으로 문제를 해결할 수 있다.

    문제 2 — 배열 비교하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 정수 배열 arr1과 arr2를 비교하여 배열의 길이와 합에 따라 결과를 반환하는 문제다. 우선 두 배열의 길이가 다르면, 더 긴 배열이 큰 것으로 간주해 1 또는 -1을 반환하고, 길이가 같다면 각 배열의 합을 비교하여 결과를 결정한다.

    • 먼저 두 배열의 길이를 비교하여 arr1.length != arr2.length인 경우, arr1이 더 길면 1, 짧으면 -1을 반환한다.
    • 길이가 같을 경우, 각 배열의 모든 원소를 합산하기 위해 두 개의 변수 sum1과 sum2를 0으로 초기화한다.
    • 향상된 for문(for (int i : arr))을 사용하여 각 배열의 모든 원소를 순회하면서 누적 합을 계산한다.
    • 합산이 끝난 뒤, 두 합이 같으면 0을 반환하고, sum1이 더 크면 1, 작으면 -1을 반환한다.

    예를 들어, arr1 = [1, 2, 3], arr2 = [1, 3, 2]라면 두 배열의 길이는 같고, 각각의 합도 모두 6이므로 0을 반환한다. 반면 arr1 = [5, 2], arr2 = [3, 4, 1]이라면 arr2의 길이가 더 길기 때문에 -1이 반환된다. 

    
      arr1 = [1, 2, 3]
      arr2 = [1, 3, 2]
      → 길이 동일 → 합 비교 (6 == 6)
      → 결과: 0
      
      arr1 = [5, 2]
      arr2 = [3, 4, 1]
      → 길이 다름 (2 vs 3)
      → 결과: -1

    각 배열을 한 번씩만 순회하므로 전체 시간 복잡도는 O(n)이며, 추가 메모리 사용은 상수 수준으로 O(1)이다.

    핵심은 길이 → 합 순서로 비교하는 단순한 규칙을 조건문과 반복문으로 명확하게 구현한 점이다. 특히 for-each 문법을 통해 인덱스 관리 없이 간결하게 합계를 구할 수 있다는 점이 장점이다.

    문제 3 — 문자열 묶기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 strArr에서 각 문자열의 길이를 기준으로 개수를 세어, 가장 많이 등장한 문자열 길이의 빈도를 구하는 문제다. 즉, 길이가 같은 문자열이 몇 개씩 있는지 계산하고, 그중 최대 개수를 반환한다.

    • 먼저 문자열 길이에 따라 개수를 저장하기 위해 크기가 31인 정수 배열 cnt를 선언한다. 문제의 제한사항에서 문자열 길이가 1 이상 30 이하이므로, 인덱스 1부터 30까지 사용하기 위해 배열 크기를 31로 지정한다.
    • for (String s : strArr) 반복문을 이용해 각 문자열 s의 길이를 구하고, 해당 길이 인덱스의 카운트를 1 증가시킨다 (cnt[s.length()]++).
    • 이후 두 번째 반복문에서 cnt 배열을 순회하며 가장 큰 값(즉, 가장 많이 등장한 문자열 길이의 개수)을 찾는다.
    • max 변수는 최대 빈도를 저장하며, if (cnt[i] > max) max = cnt[i]; 형태로 최댓값을 갱신한다.

    예를 들어, strArr = ["a", "bc", "de", "fg", "hij", "kl", "mn"]일 경우를 살펴보면 다음과 같다. 

    
      길이 1: 1개 ("a")
      길이 2: 4개 ("bc", "de", "fg", "kl", "mn")
      길이 3: 1개 ("hij")
      따라서 가장 많이 등장한 길이의 문자열 개수는 4.

    배열의 크기는 31로 고정되어 있고, 입력 배열을 한 번만 순회하므로 시간 복잡도는 O(n), 추가 공간 복잡도는 O(1)이다.

    핵심은 문자열 길이별 등장 횟수를 인덱스로 카운팅하는 것이다. 문제의 제약을 활용해 int[] 배열로 직접 카운팅하면 해시맵보다 훨씬 빠르고 간결하게 해결할 수 있다.
