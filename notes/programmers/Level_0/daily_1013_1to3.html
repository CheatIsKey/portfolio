    문제 1 — 전국 대회 선발 고사

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    참석한 학생(attendance[i] == true) 중에서 등수(rank[i])가 가장 낮은 값 3명 (즉, 숫자가 작은 3명)의 학생 번호(인덱스)를 찾아 10000 * a + 100 * b + c로 합성하는 문제다. 등수로 정렬하면서도 학생 번호를 잃지 않도록 [rank, index] 쌍을 리스트에 담아 정렬한다.

    • for 루프로 참석자만 걸러 list.add(new int[]{rank[i], i}) 형태로 저장 (0번은 등수, 1번은 학생 번호).
    • list.sort((a, b) -> a[0] - b[0])로 등수 기준 오름차순 정렬 → 앞에서부터 rank가 작은(성적이 좋은) 순서가 된다.
    • 상위 3명의 학생 번호를 a = list.get(0)[1], b = list.get(1)[1], c = list.get(2)[1]로 추출한다.
    • 요구식 10000 * a + 100 * b + c를 계산해 반환한다. 
    // 예시
      rank       = [3, 1, 2, 4]
      attendance = [T,  T, F,  T]
      
      참석자만 [rank, index] → [[3,0], [1,1], [4,3]]
      등수 오름차순 정렬     → [[1,1], [3,0], [4,3]]
      a=1(학생번호), b=0, c=3 → 결과 = 10000*1 + 100*0 + 3 = 10003

    시간 복잡도는 정렬이 지배하므로 O(n log n), 추가 공간은 리스트에 참석자 수만큼 담아 O(n)이다.

    주의

    • 오름차순 정렬 후 뒤에서 3개를 뽑으면 “등수가 큰 값(성적이 낮은 학생)”이 되므로 앞에서 3개를 선택해야 한다.
    • 등수만 저장하면 학생 번호를 잃으므로 [rank, index] 쌍을 반드시 함께 보관한다.
    • 문제 조건상 참석자가 최소 3명이라는 전제가 있어야 한다(문제 원문 기준). 그 외 상황이면 범위 체크가 필요하다.
    대안 아이디어(선택)
    • PriorityQueue<int[]>로 참석자 중 상위 3명만 유지(최대 힙/크기 3)하면 평균적으로 메모리를 덜 쓸 수 있다.
    • 스트림 사용: 참석자 인덱스 스트림 → 등수 기준 정렬 → 상위 3명 추출 후 포맷팅.

    문제 2 — 정수 부분

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    실수형(double) 값 flo가 주어졌을 때, 소수점 이하를 버리고 내림(floor)한 정수 부분만 반환하는 문제다. 자바의 Math.floor() 메서드를 이용하면 손쉽게 해결할 수 있다.

    • Math.floor(flo)는 주어진 실수보다 작거나 같은 가장 큰 정수를 double 형태로 반환한다. 예를 들어, 3.7 → 3.0, -2.3 → -3.0 이다.
    • 문제의 반환형은 int이므로, (int)로 명시적 형 변환을 수행해 double → int로 캐스팅한다.
    • 최종적으로 (int) Math.floor(flo)는 소수점 이하를 내림한 정수 부분을 반환하게 된다. 
    // 예시
      flo = 3.7  → Math.floor(3.7) = 3.0 → (int) 3.0 = 3
      flo = 7.0  → Math.floor(7.0) = 7.0 → (int) 7.0 = 7
      flo = -1.2 → Math.floor(-1.2) = -2.0 → (int) -2.0 = -2

    시간 복잡도는 O(1) (단일 산술 연산), 공간 복잡도도 O(1)이다. 즉, 매우 간단하고 효율적인 문제다.

    단순히 (int) flo로 캐스팅해도 양수일 때는 동일한 결과를 얻을 수 있지만, 음수의 경우 절삭(truncation)이 일어나 -1.2 → -1이 되어 버린다. 따라서 이 문제에서는 **정확한 내림**을 보장하는 Math.floor() 사용이 안전하다.

    문제 3 — 문자열 정수의 합

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자로만 이루어진 문자열 num_str이 주어질 때, 각 자리의 숫자를 모두 더한 합을 구하는 문제다. 문자열을 한 글자씩 순회하며 각 문자를 숫자로 변환한 뒤 누적 합계를 구하면 된다.

    • 먼저 toCharArray()를 사용해 문자열을 문자 배열로 변환한다. 이렇게 하면 각 자리의 숫자를 개별 문자로 쉽게 접근할 수 있다.
    • 향상된 for 문을 이용해 문자 배열을 순회하면서 각 문자 c에서 문자 '0'을 빼면 실제 정수값으로 변환된다. 예를 들어, '3' - '0' == 3, '9' - '0' == 9.
    • 변환된 정수값을 sum 변수에 누적하여 더한다.
    • 모든 문자를 처리한 후, 누적된 합계 sum을 반환한다. 
    // 예시
      num_str = "1234"
      → 문자 배열 ['1', '2', '3', '4']
      → '1'-'0' = 1, '2'-'0' = 2, '3'-'0' = 3, '4'-'0' = 4
      → 합계 = 1 + 2 + 3 + 4 = 10

    시간 복잡도는 문자열 길이에 비례하는 O(n), 공간 복잡도는 문자 배열을 사용하는 O(n)이다. 입력 문자열의 길이가 작기 때문에 매우 효율적으로 동작한다.

    이 방법은 자바의 기본적인 문자-숫자 변환 규칙('0'~'9'의 연속된 ASCII 코드값)을 이용하므로 불필요한 파싱 없이 빠르게 동작한다. 또한 Character.getNumericValue()나 스트림 API를 사용하는 다른 방법보다 명확하고 직관적인 풀이 방식이다.
