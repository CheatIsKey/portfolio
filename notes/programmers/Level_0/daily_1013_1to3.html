    문제 1 — 배열의 길이에 따라 다른 연산하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 arr과 정수 n이 주어질 때, 배열 길이의 짝/홀 여부에 따라 특정 인덱스들에만 n을 더하는 문제다. 길이가 짝수면 (0-기준) 홀수 인덱스들(1, 3, 5, …)에만, 길이가 홀수면 짝수 인덱스들(0, 2, 4, …)에만 값을 더한다.

    • 배열 길이의 짝/홀을 arr.length % 2로 판별한다.
    • 시작 인덱스를 길이에 따라 결정한다: start = (arr.length % 2 == 0) ? 1 : 0.
    • for (int i = start; i < arr.length; i += 2)로 두 칸씩 건너뛰며 해당 위치의 원소에 n을 더한다. (원본 배열을 제자리(in-place)로 수정)
    • 수정이 끝난 arr를 반환한다.

    예시를 통해 확인해 보자. 

    
      // 길이 짝수 → 홀수 인덱스에만 더함
      arr = [1, 2, 3, 4], n = 3
      인덱스 1, 3에 +3 → [1, 5, 3, 7]
      
      // 길이 홀수 → 짝수 인덱스에만 더함
      arr = [1, 2, 3], n = 10
      인덱스 0, 2에 +10 → [11, 2, 13]

    모든 원소를 방문하는 것이 아니라 절반 정도만 순회하므로 시간 복잡도는 O(n) (상수배 절감), 추가 공간은 O(1)이다. n이 음수여도 동일한 로직으로 감소 연산이 적용된다.

    핵심은 짝/홀 길이 구분에 따라 인덱스 시작점을 다르게 하여 반복문을 구성하는 것이다. 조건문 분기만으로 명확하게 구현할 수 있어 가독성과 효율성이 모두 좋은 풀이 방식이다.

    문제 2 — 뒤에서 5등까지

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 num_list에서 가장 작은 값부터 오름차순으로 정렬한 뒤, 앞쪽 5개 원소(즉, 가장 작은 5개)를 잘라내는 문제다. 자바의 Arrays 유틸리티 메서드들을 활용해 간결하게 처리할 수 있다.

    • 먼저 Arrays.sort(num_list)를 호출해 배열을 오름차순으로 정렬한다. (기본적으로 자바의 sort()는 작은 수 → 큰 수 순서)
    • 정렬된 배열에서 앞의 5개 원소만 추출하기 위해 Arrays.copyOfRange(num_list, 0, 5)를 사용한다.
    • 이 메서드는 원본 배열의 0번 인덱스부터 5번째 원소 전까지(0 ≤ i < 5)의 구간을 복사하여 새 배열로 반환한다.
    • 최종적으로 새 배열을 반환하면, 작은 값 5개만 담긴 결과가 완성된다.

    예를 들어, num_list = [12, 4, 15, 46, 38, 1, 14]인 경우, 

    
      정렬 후: [1, 4, 12, 14, 15, 38, 46]
      앞 5개 추출: [1, 4, 12, 14, 15]

    Arrays.sort()는 O(n log n) 시간 복잡도이며, copyOfRange()는 O(k) (여기서는 k=5) 복사 연산을 수행한다. 전체적으로 효율적인 풀이이며 추가 메모리 사용은 최소화된다.

    핵심은 정렬 후 부분 배열을 복사하는 두 단계이며, 자바의 표준 메서드를 이용해 별도의 루프 없이도 한 줄씩 해결할 수 있다는 점이다.

    문제 3 — 뒤에서 5등 위로

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 num_list가 주어질 때, 가장 작은 값 5개를 제외한 나머지 원소들만 남겨 오름차순으로 반환하는 문제다. Arrays.sort()와 Arrays.copyOfRange()를 이용하면 매우 간단하게 해결할 수 있다.

    • 먼저 Arrays.sort(num_list)로 배열을 오름차순 정렬한다. (즉, 가장 작은 값이 앞쪽에 오게 된다)
    • 그다음 Arrays.copyOfRange(num_list, 5, num_list.length)를 호출해 인덱스 5부터 마지막까지의 구간을 복사한다. 이렇게 하면 앞쪽 5개의 원소(가장 작은 값들)은 자동으로 제외된다.
    • 복사된 배열을 반환하면, ‘뒤에서 5등까지’를 제외한 나머지 값들이 담긴 새 배열이 완성된다.

    예를 들어, num_list = [12, 4, 15, 46, 38, 1, 14]인 경우를 살펴보자. 

    
      정렬 후: [1, 4, 12, 14, 15, 38, 46]
      앞의 5개 제외: [38, 46]
      결과 → [38, 46]

    Arrays.sort()의 시간 복잡도는 O(n log n), copyOfRange()는 복사 구간 길이에 비례하는 O(k) 연산이다. 입력 배열의 크기가 작기 때문에 전체적으로 효율적이다.

    핵심은 정렬 후 필요 없는 앞쪽 부분을 슬라이싱하는 아이디어다. 별도의 반복문 없이, Arrays 클래스의 내장 메서드 두 개로 문제를 깔끔하게 해결할 수 있다.
