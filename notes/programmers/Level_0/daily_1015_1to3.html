    문제 1 — 문자열로 변환

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 n을 문자열(String)로 변환하는 문제다. 자바에서는 내장 메서드 String.valueOf()를 사용하면 정수를 손쉽게 문자열로 변환할 수 있다.

    • String.valueOf(n)은 기본형 int 값을 해당 값을 표현하는 문자열로 변환해 반환한다.
    • 예를 들어, 1234 → "1234", 0 → "0", -7 → "-7" 이 된다.
    • 기본형을 문자열로 바꾸는 동작이므로 예외가 발생하지 않는다 (숫자 → 문자열 방향이기 때문에 NumberFormatException과 무관). 
    // 예시
      int n = 10;      // 10  →  String.valueOf(10)   = "10"
      int m = 00123;   // 123 →  String.valueOf(123)  = "123"  (정수 리터럴의 선행 0은 의미 없음)
      int k = -987654; // -987654 → String.valueOf(-987654) = "-987654"

    시간 복잡도는 결과 문자열의 길이(자릿수)에 비례하므로 O(L), 공간 복잡도는 새 문자열 1개만 생성하므로 O(L)이다.

    주의

    • int 범위를 벗어나는 값은 애초에 변수에 담을 수 없으므로 입력 제약만 만족하면 문제 없음.
    • 선행 0이 필요한 포맷(예: 자리수 고정)이라면 별도 포맷팅이 필요하다(String.format 등 참조).
    • n이 기본형이므로 null 처리는 고려 대상이 아니다.
    대안 아이디어
    • Integer.toString(n)도 동일한 결과를 반환한다(성능/의미상 동등).
    • "" + n과 같은 문자열 연결도 가능하지만, 내부적으로 불필요한 StringBuilder가 생겨 권장되지 않는다.
    • String.format("%d", n)은 가독성은 좋지만 포맷터 오버헤드로 가장 비효율적이다.

    문제 2 — 배열의 원소 삭제하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 배열 arr에서 delete_list에 포함된 원소를 모두 제거한 뒤, 남은 값들을 순서대로 반환하는 문제다. 즉, arr의 각 원소가 delete_list에 존재하지 않는 경우만 결과 배열에 포함된다.

    • 외부 컬렉션인 List<Integer>를 이용해 조건에 맞는 원소를 임시로 저장한 후, 마지막에 stream()을 통해 int[]로 변환한다.
    • 중첩 반복문을 사용하여 arr의 각 원소가 delete_list에 존재하는지 검사한다. 존재하면 건너뛰고, 존재하지 않으면 리스트에 추가한다.
    • 최종적으로 list.stream().mapToInt(Integer::intValue).toArray()를 호출하여 List<Integer>를 기본형 배열로 변환한다. 
    // 예시
      arr = [1, 2, 3, 4, 5]
      delete_list = [2, 4]
      → 삭제 대상은 2와 4
      → 결과: [1, 3, 5]
      
      arr = [10, 20, 30]
      delete_list = [40]
      → 삭제 대상 없음
      → 결과: [10, 20, 30]
      
      arr = [5, 6, 7]
      delete_list = [5, 6, 7]
      → 모두 삭제됨
      → 결과: []

    시간 복잡도는 이중 반복문 구조로 인해 O(N × M) (N = arr 길이, M = delete_list 길이)이며, 공간 복잡도는 결과를 담는 리스트의 크기에 비례하므로 O(N)이다.

    주의

    • 현재 구현은 delete_list의 길이가 길 경우 비효율적일 수 있다(이중 루프 때문).
    • 성능을 개선하려면 delete_list를 HashSet으로 변환해 포함 여부를 O(1)에 검사할 수 있다.
    • 원본 배열 순서를 유지해야 하므로 정렬 기반 접근은 부적절하다.
    대안 아이디어
    • Set<Integer> delSet = new HashSet<>(Arrays.asList(delete_list)); 로 변환 후 필터링 시 delSet.contains()를 사용하면 효율적이다.
    • Arrays.stream(arr).filter(x -> !delSet.contains(x)).toArray(); 형태로도 간결하게 표현 가능하다.
    • 스트림 기반 접근은 코드 가독성이 높지만, 작은 입력에서는 성능 차이가 미미하다.

    문제 3 — 부분 문자열인지 확인하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string 안에 부분 문자열 target이 포함되어 있는지 확인해, 포함되어 있으면 1, 아니면 0을 반환하는 문제다. 자바에서는 String.contains()를 사용해 부분 문자열 포함 여부를 간단히 판단할 수 있다.

    • my_string.contains(target)는 target이 my_string에 존재하면 true, 아니면 false를 반환한다.
    • 불리언 결과를 삼항 연산자(?:)로 1/0으로 변환해 요구 형식에 맞춘다.
    • contains는 대소문자를 구분한다. (예: "Abc".contains("abc")는 false)
    // 예시
      my_string = "programmers", target = "gram"   →  contains == true  →  결과 1
      my_string = "hello",       target = "ell"    →  contains == true  →  결과 1
      my_string = "hello",       target = "world"  →  contains == false →  결과 0
      my_string = "abc",         target = ""       →  contains == true  →  결과 1  // 빈 문자열은 항상 포함으로 간주

    시간 복잡도는 내부 검색 알고리즘에 따라 달라질 수 있다. 일반적으로 입력 길이에 비례하는 선형 시간에 가깝게 동작하지만, 최악의 경우 O(N × M)일 수 있다(N=my_string 길이, M=target 길이). 공간 복잡도는 추가 자료구조를 거의 쓰지 않으므로 O(1)이다.

    주의

    • target이 null이면 NullPointerException이 발생한다.
    • 대소문자를 무시해야 하는 요구사항이라면 toLowerCase()/toUpperCase()로 정규화 후 비교가 필요하다.
    • 문자열의 유니코드 정규화(NFC/NFD)가 서로 다르면 동일하게 보이는 문자가 불일치할 수 있다.
    대안 아이디어
    • my_string.indexOf(target) >= 0도 동일한 판별이 가능하다.
    • 정규식을 써야 한다면 Pattern.compile(Pattern.quote(target))로 이스케이프 후 matcher.find()를 사용할 수 있다.
    • 여러 키워드 중 하나라도 포함되는지 확인하려면 루프나 스트림(anyMatch)로 contains를 결합한다.
