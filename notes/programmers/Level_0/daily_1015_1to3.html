    문제 1 — 문자열을 정수로 변환하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열로 주어진 정수값 n_str을 실제 정수형(int)으로 변환하는 문제다. 자바에서는 내장 메서드 Integer.parseInt()를 이용하면 문자열을 손쉽게 정수로 변환할 수 있다.

    • Integer.parseInt(n_str)은 문자열이 정수 형태로만 구성되어 있다면 해당 값을 int 타입으로 변환하여 반환한다.
    • 예를 들어, "1234" → 1234, "007" → 7 이 된다.
    • 만약 숫자가 아닌 문자가 포함된 경우("12a4" 등)에는 NumberFormatException 예외가 발생한다.
    • 이 문제의 입력은 항상 숫자 문자열임이 보장되므로, 예외 처리를 따로 할 필요는 없다. 
    // 예시
      n_str = "10"  →  Integer.parseInt("10") = 10
      n_str = "00123" → Integer.parseInt("00123") = 123
      n_str = "987654" → Integer.parseInt("987654") = 987654

    시간 복잡도는 문자열 길이에 비례하므로 O(n), 공간 복잡도는 상수 공간만 사용하므로 O(1)이다.

    주의

    • Integer.parseInt()는 입력이 32비트 정수 범위를 넘어가면 예외를 던진다.
    • 문자열이 음수일 경우("-5" 등)도 올바르게 파싱되어 -5를 반환한다.
    • 단순히 숫자로 구성된 문자열만 다루므로, 별도의 정규식 검증은 불필요하다.
    대안 아이디어
    • Integer.valueOf(n_str)를 사용하면 int 대신 Integer 객체를 반환한다.
    • 스트림이나 람다를 쓸 이유는 없으며, 단일 호출이 가장 간단하고 효율적이다.

    문제 2 — 0 떼기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자 문자열 n_str의 앞쪽에 붙은 선행 0(leading zero)을 모두 제거하고, 남은 숫자 부분만 반환하는 문제다. 문자열 전체가 '0'으로만 구성되어 있다면 결과는 "0"이 되어야 한다.

    • 먼저 문자열을 처음부터 순회하면서, '0'이 아닌 첫 번째 문자의 인덱스(idx)를 찾는다. 이 위치 이전의 모든 문자는 제거 대상이다.
    • 인덱스를 찾은 뒤에는 해당 위치부터 문자열 끝까지 문자들을 StringBuilder에 순서대로 추가한다.
    • 마지막으로 sb.toString()을 반환하여 선행 0이 제거된 문자열을 얻는다.
    • 만약 문자열 전체가 '0'으로만 이루어져 있다면, for 루프에서 조건을 만족하지 않으므로 결과적으로 sb가 비게 되고, 반환값은 빈 문자열이 아니라 "0"이 되어야 한다. (이 부분은 추가 조건문으로 보완 가능하다.) 
    // 예시
      n_str = "00123" → idx = 2 → sb = "123" → 결과: "123"
      n_str = "0000"  → 모든 문자가 '0' → 결과: "0"
      n_str = "100"   → idx = 0 → sb = "100" → 결과: "100"

    시간 복잡도는 문자열 길이에 비례하는 O(n), 공간 복잡도는 StringBuilder를 사용하므로 O(n)이다.

    주의

    • 문자열의 모든 문자가 '0'인 경우를 별도로 처리해야 한다.
    • StringBuilder 대신 substring()을 활용하면 더 간단히 구현할 수 있다.
    • 불필요한 lastIndexOf() 호출은 실제 로직에는 영향을 주지 않으므로 제거 가능하다.

    문제 3 — 두 수의 합

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    매우 큰 숫자 문자열 a, b가 주어졌을 때, 두 수의 합을 문자열로 반환하는 문제다. 내장 정수형(int, long)으로는 범위를 초과할 수 있으므로, 문자열을 자릿수 단위로 더하면서 올림(carry)을 처리하는 방식으로 해결한다.

    • 포인터 i, j를 각각 a, b의 마지막 인덱스(일의 자리)로 두고, carry를 0으로 초기화한다.
    • 반복문에서 현재 자리의 숫자 x, y를 추출한다. 인덱스가 범위를 벗어나면 해당 자릿수는 0으로 간주한다.
    • sum = x + y + carry로 더하고, 현재 자릿수는 sum % 10을 StringBuilder에 추가한다. 다음 자릿수로 넘길 올림은 carry = sum / 10으로 갱신한다.
    • 계산은 일의 자리부터 진행되므로 결과가 역순으로 쌓인다. 따라서 마지막에 sb.reverse()로 뒤집어 최종 문자열을 만든다. 
    // 예시 1
      a = "12345", b = "678"
      반복 진행(뒤에서부터):
        5+8=13 → 자리수 3, carry 1
        4+7+1=12 → 자리수 2, carry 1
        3+6+1=10 → 자리수 0, carry 1
        2+0+1=3  → 자리수 3, carry 0
        1+0+0=1  → 자리수 1, carry 0
      역순으로 쌓인 "32031"을 뒤집으면 결과 "13023"
      
    // 예시 2 (긴 올림 체인)
      a = "99999999999999999999"
      b = "1"
      매 자리마다 9+0+carry = 10 → 자리수 0, carry 1 반복
      마지막에 carry 1을 추가 → 결과 "100000000000000000000"

    시간 복잡도는 두 문자열 길이 합에 비례하는 O(n), 공간 복잡도는 결과 길이에 비례하는 O(n)이다.

    주의

    • 덧셈은 반드시 오른쪽(일의 자리)부터 수행해야 올림을 정확히 전달할 수 있다.
    • 두 문자열 길이가 달라도 인덱스가 음수인 자리는 0으로 처리해야 한다.
    • 마지막 반복 후 carry > 0이면 한 자리 더 추가해야 한다(루프 조건에서 이미 처리).
    • 계산을 역순으로 추가하므로, 최종 반환 전에 reverse()를 호출해야 올바른 자릿수가 된다.
