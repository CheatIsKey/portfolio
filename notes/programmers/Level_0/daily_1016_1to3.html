    문제 1 — 부분 문자열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 str1이 문자열 str2 안에 포함되어 있는지를 확인하는 문제다. 자바에서는 String 클래스의 내장 메서드 contains()를 이용해 간단하게 부분 문자열 포함 여부를 판단할 수 있다.

    • str2.contains(str1)은 str1이 str2의 부분 문자열이면 true를, 아니면 false를 반환한다.
    • 반환값을 조건식으로 바로 사용할 수 있으므로, 삼항 연산자 ? :를 통해 문제에서 요구하는 결과값(1 또는 0)을 간결하게 반환할 수 있다. 
    // 예시
      String str1 = "abc";
      String str2 = "aabbcc";
      System.out.println(str2.contains(str1)); // false
      
      str1 = "bb";
      System.out.println(str2.contains(str1)); // true

    시간 복잡도는 O(N × M)으로, N은 str2의 길이, M은 str1의 길이이다. (내부적으로 부분 문자열 탐색을 수행하기 때문) 공간 복잡도는 별도의 추가 메모리를 사용하지 않으므로 O(1)이다.

    주의

    • null이 전달될 경우 NullPointerException이 발생할 수 있으므로, 입력값이 항상 유효한 문자열임이 보장되어야 한다.
    • 대소문자를 구분하므로, 대소문자 무시 비교가 필요하다면 toLowerCase()나 equalsIgnoreCase() 등을 활용한다.
    • 공백도 문자로 취급되므로 "abc "와 "abc"는 다르다.
    대안 아이디어
    • str2.indexOf(str1) >= 0으로도 동일한 결과를 얻을 수 있다.
    • Pattern.matches()를 사용하면 정규식을 활용한 복잡한 포함 조건도 처리할 수 있다.
    • 반복문으로 직접 부분 문자열을 탐색할 수도 있으나, 내장 메서드 사용이 가장 효율적이다.

    문제 2 — 꼬리 문자열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 str_list에서 특정 문자열 ex가 포함된 원소를 제외하고, 남은 문자열들을 순서대로 이어붙인 결과를 반환하는 문제다. 자바에서는 문자열 조합 시 StringBuilder를 사용하면 효율적으로 문자열을 처리할 수 있다.

    • s.contains(ex)는 각 문자열 s가 ex를 포함하는지를 확인한다. 포함되어 있다면 continue로 해당 문자열을 건너뛴다.
    • 포함되어 있지 않은 문자열만 StringBuilder에 append()하여 누적한다.
    • 반복이 끝나면 sb.toString()을 호출해 누적된 결과 문자열을 반환한다. 
    // 예시
      String[] str_list = {"abc", "def", "cdx"};
      String ex = "c";
      → "def"   // "abc"와 "cdx"는 "c"를 포함하므로 제외, 남은 "def"만 연결

    시간 복잡도는 O(N × M)으로, N은 문자열 배열의 길이, M은 각 문자열의 평균 길이이다. (각 문자열마다 contains() 탐색이 수행되기 때문) 공간 복잡도는 결과 문자열의 길이에 비례하므로 O(L)이다.

    주의

    • StringBuilder는 문자열 덧셈(+)보다 훨씬 효율적이다. String은 불변(immutable) 객체이므로 반복 덧셈은 비효율적이다.
    • 대소문자를 구분하여 비교하므로, 대소문자 무시가 필요할 경우 toLowerCase()나 toUpperCase()를 함께 사용한다.
    • 모든 문자열이 제외되어 비어 있는 경우, 빈 문자열 ""이 반환된다.
    대안 아이디어
    • String.join("", Arrays.stream(str_list).filter(s -> !s.contains(ex)).toArray(String[]::new)) 형태로 스트림(Stream)을 이용한 풀이도 가능하다.
    • StringBuffer를 사용할 수도 있지만, 단일 스레드 환경에서는 StringBuilder가 더 빠르다.
    • 만약 구분자(예: 공백, 쉼표 등)를 포함하여 연결하려면 String.join(" ", ...)이나 Collectors.joining()을 사용할 수 있다.

    문제 3 — 정수 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 num_list에 특정 정수 n이 존재하는지를 판별하는 문제다. 배열을 순회하면서 n과 동일한 값이 있는지 비교하고, 존재하면 1을, 없으면 0을 반환한다.

    • 향상된 for문(for (int i : num_list))을 사용하여 배열의 모든 원소를 순회한다.
    • 각 원소 i가 n과 같을 경우 즉시 1을 반환하여 탐색을 종료한다.
    • 끝까지 찾지 못하면 0을 반환한다. 
    // 예시
      int[] num_list = {1, 2, 3, 4, 5};
      int n = 3;
      → 1   // 3이 배열에 존재
      
      n = 7;
      → 0   // 7은 배열에 없음

    시간 복잡도는 최악의 경우 배열 전체를 탐색하므로 O(N), 공간 복잡도는 추가 메모리를 사용하지 않으므로 O(1)이다.

    주의

    • 입력 배열이 null일 경우 NullPointerException이 발생하므로, 문제 조건에서 유효한 배열이 주어지는 것이 전제다.
    • 배열의 크기가 크더라도 단순 선형 탐색만 수행하므로 구현은 간단하지만, 성능이 중요한 경우 HashSet 등 자료구조 활용을 고려할 수 있다.
    • 정수 비교는 기본형이므로 ==로 충분하다(객체 비교 아님).
    대안 아이디어
    • Arrays.stream(num_list).anyMatch(i -> i == n)으로 스트림 API를 사용해 한 줄로 표현할 수 있다.
    • 정렬된 배열이라면 Arrays.binarySearch(num_list, n)를 사용해 O(log N) 시간에 탐색할 수 있다.
    • 반복문이 단순하므로, 가독성 면에서는 현재 코드가 가장 직관적이다.
