    문제 1 — 주사위 게임 1

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 정수 a와 b가 주어졌을 때, 각각의 홀짝 여부에 따라 서로 다른 계산을 수행하는 문제다. 조건문(if-else)을 이용해 경우를 명확히 구분할 수 있다.

    • 두 수가 모두 홀수이면, a * a + b * b — 즉 두 수의 제곱의 합을 반환한다.
    • 두 수가 모두 짝수이면, Math.abs(a - b) — 두 수의 차의 절댓값을 반환한다.
    • 하나는 홀수이고 하나는 짝수라면, 2 * (a + b) — 두 수의 합에 2를 곱한 값을 반환한다. 
    // 예시
      int a = 3, b = 5; // 둘 다 홀수
      System.out.println(solution(a, b)); // 3*3 + 5*5 = 34
      
      a = 4; b = 8; // 둘 다 짝수
      System.out.println(solution(a, b)); // |4 - 8| = 4
      
      a = 3; b = 4; // 하나는 홀수, 하나는 짝수
      System.out.println(solution(a, b)); // 2 * (3 + 4) = 14

    시간 복잡도는 조건문 1회 판단만 수행하므로 O(1)이고, 공간 복잡도 역시 추가 메모리를 사용하지 않아 O(1)이다.

    주의

    • a와 b가 음수여도 문제없이 계산된다.
    • 정수형 범위를 초과할 수 있는 큰 입력값이라면 long 자료형으로 변경하는 것이 안전하다.
    • Math.abs()는 절댓값 계산에 유용하며, 음수 결과를 방지한다.
    대안 아이디어
    • 삼항 연산자를 활용해 한 줄로 표현할 수도 있다.
    • 홀짝 여부를 비트 연산((a & 1))으로 판별하면 미세하게 빠를 수 있다.
    • 입력 조건이 확실하다면, if-else 대신 switch문으로도 구성 가능하다.

    문제 2 — 날짜 비교하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 날짜 date1, date2가 주어졌을 때, date1이 date2보다 이전이면 1, 아니면 0을 반환하는 문제다. 자바의 LocalDate를 사용하면 날짜 생성과 비교를 간단히 처리할 수 있다.

    • LocalDate.of(year, month, day)로 각각의 날짜 객체를 만든다.
    • d1.isBefore(d2)가 true면 d1이 d2보다 이전 날짜임을 의미하므로 1을, 그렇지 않으면 0을 반환한다. 
    // 예시
      int[] date1 = {2023, 5, 10};
      int[] date2 = {2023, 5, 11};
      System.out.println(new Solution().solution(date1, date2)); // 1 (date1 < date2)
      
      date1 = new int[]{2024, 1, 1};
      date2 = new int[]{2023, 12, 31};
      System.out.println(new Solution().solution(date1, date2)); // 0 (date1 > date2)
      
      date1 = new int[]{2022, 3, 15};
      date2 = new int[]{2022, 3, 15};
      System.out.println(new Solution().solution(date1, date2)); // 0 (같은 날짜)

    시간 복잡도는 날짜 비교 1회만 수행하므로 O(1), 공간 복잡도도 추가 메모리가 거의 없어 O(1)이다.

    주의

    • date1, date2는 각각 길이 3의 배열(연, 월, 일)이어야 한다.
    • 존재하지 않는 날짜(예: 2월 30일)를 전달하면 DateTimeException이 발생한다.
    • LocalDate는 시간대(Time Zone)를 고려하지 않는 순수한 날짜 타입이므로, 시차에 영향을 받지 않는다.
    대안 아이디어
    • API 없이 사전식(lexicographical) 비교: (year1 < year2) || (year1 == year2 && month1 < month2) || (year1 == year2 && month1 == month2 && day1 < day2)
    • LocalDate.compareTo(d2) < 0를 사용해도 동일한 판정을 얻을 수 있다.
    • 성능상 큰 차이는 없지만, 표준 라이브러리(LocalDate) 사용이 가장 가독성이 좋다.

    문제 3 — 커피 심부름

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 order에 담긴 각 주문 항목을 순회하며, 항목 이름에 따라 가격을 누적해 총합을 반환하는 문제다. 문자열 포함 여부를 contains()로 판별해 아메리카노 계열과 카페라테 계열을 구분한다.

    • 항목에 "americano"가 포함되어 있으면 4,500원을 더한다. (예: "americano", "iceamericano", "hotamericano")
    • 항목에 "cafelatte"가 포함되어 있으면 5,000원을 더한다. (예: "cafelatte", "icecafelatte", "hotcafelatte")
    • 둘 다 아니면 기본적으로 4,500원을 더한다. (문제의 "anything" 등 기타 항목) 
    // 예시
      String[] order = {"americano", "cafelatte", "iceamericano", "anything"};
      System.out.println(new Solution().solution(order)); 
      // 4500 + 5000 + 4500 + 4500 = 18,000
      
      order = new String[]{"hotcafelatte", "anything", "iceamericano"};
      System.out.println(new Solution().solution(order)); 
      // 5000 + 4500 + 4500 = 14,000
      
      order = new String[]{"anything", "anything"};
      System.out.println(new Solution().solution(order)); 
      // 4500 + 4500 = 9,000

    시간 복잡도는 주문 개수를 N이라 할 때 각 원소를 한 번씩만 확인하므로 O(N), 공간 복잡도는 추가 자료구조를 사용하지 않아 O(1)이다.

    주의

    • 대소문자를 구분하므로 입력이 소문자로 주어진다는 전제가 필요하다. 그렇지 않다면 s.toLowerCase().contains(...)로 보정한다.
    • contains()는 부분 문자열을 찾으므로, 가격 규칙에 해당하는 접두사/접미사가 붙어도 정확히 처리된다.
    • 명세에 없는 문자열이 와도 기본값(4,500원)으로 처리되도록 else 분기를 둔다.
    대안 아이디어
    • switch 대신 Map<String, Integer>로 정확한 키 매칭을 쓰고, 접두사/접미사가 붙는 경우만 별도 처리(startsWith() 등)할 수도 있다.
    • 가격 상수를 static final로 분리하면 가독성과 유지보수성이 올라간다.
    • 입력이 모두 정확한 키로만 주어진다면 contains() 대신 정확 비교(equals())가 더 안전하다.
