    문제 1 — 그림 확대

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열로 구성된 그림 picture가 주어졌을 때, 각 문자를 가로와 세로로 k배 확대한 그림을 반환하는 문제다. 문자열 조작과 반복문을 활용하여 간단히 해결할 수 있다.

    • 각 문자열(행)에 대해, 문자 하나하나를 String.valueOf(c).repeat(k)으로 가로로 k배 확장한다.
    • 이렇게 완성된 문자열을 세로로 k번 반복하여 동일한 행을 여러 번 추가한다.
    • 모든 행에 대해 위 과정을 수행한 뒤, 결과를 List<String>에 저장하고 마지막에 배열로 변환해 반환한다. 
    // 예시
      String[] picture = {"x.x", ".x."};
      int k = 3;
      System.out.println(Arrays.toString(new Solution().solution(picture, k)));
      
      // 출력 결과:
      // [xxx...xxx, xxx...xxx, xxx...xxx,
      //  ...xxx...,
      //  ...xxx...,
      //  ...xxx...]

    시간 복잡도는 각 문자마다 k번 반복하며, 각 행을 다시 k번 추가하므로 O(n * m * k²) (n=행 수, m=열 수)이다. 공간 복잡도는 생성되는 결과 문자열 크기에 비례해 O(n * m * k²)이다.

    주의

    • StringBuilder를 사용하면 문자열 덧셈보다 효율적으로 처리할 수 있다.
    • String.repeat(k) 메서드는 Java 11 이상에서만 사용 가능하다.
    • 출력 순서를 보장하기 위해 입력 순서를 그대로 유지해야 한다.
    대안 아이디어
    • 2차원 문자 배열로 변환 후 직접 인덱싱하며 복제할 수도 있다.
    • Stream API를 사용하면 한 줄로도 표현 가능하지만 가독성이 떨어질 수 있다.
    • 메모리 사용을 줄이려면 중간 문자열을 즉시 출력하거나 버퍼링을 최소화할 수 있다.

    문제 2 — 조건에 맞게 수열 변환하기 3

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 arr과 정수 k가 주어졌을 때, k의 홀짝 여부에 따라 배열의 각 원소를 다르게 변환하는 문제다. 단순한 반복문과 조건문으로 쉽게 해결할 수 있다.

    • k가 홀수이면, 각 원소에 arr[i] *= k — 즉 k를 곱한다.
    • k가 짝수이면, 각 원소에 arr[i] += k — 즉 k를 더한다.
    • 모든 요소에 대해 이 연산을 수행한 뒤, 변환된 배열을 그대로 반환한다. 
    // 예시
      int[] arr = {1, 2, 3};
      int k = 3; // 홀수
      System.out.println(Arrays.toString(new Solution().solution(arr, k)));
      // [3, 6, 9]  (각 원소에 3을 곱함)
      
      arr = new int[]{1, 2, 3};
      k = 4; // 짝수
      System.out.println(Arrays.toString(new Solution().solution(arr, k)));
      // [5, 6, 7]  (각 원소에 4를 더함)

    시간 복잡도는 배열의 길이를 n이라 할 때 모든 원소를 한 번씩만 순회하므로 O(n)이다. 공간 복잡도는 입력 배열을 직접 수정하므로 추가 메모리를 사용하지 않아 O(1)이다.

    주의

    • 입력 배열 arr의 값이 크고 k도 큰 경우, 곱셈 시 오버플로우에 주의해야 한다.
    • 입력 배열을 그대로 수정하므로, 원본 배열이 필요하다면 복사본을 만들어야 한다.
    • 홀짝 판별은 k % 2 == 0으로 간단히 확인할 수 있다.
    대안 아이디어
    • 스트림(IntStream)을 사용하면 함수형 방식으로 한 줄로 구현할 수도 있다.
    • 삼항 연산자를 활용하면 arr[i] = (k % 2 == 0 ? arr[i] + k : arr[i] * k) 형태로 더 간결하게 작성할 수 있다.
    • 배열이 아닌 리스트로 처리할 경우, set() 메서드로 값을 갱신할 수 있다.

    문제 3 — l로 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    해설

    주어진 문자열 myString에서 알파벳 순서상 'l'보다 앞에 있는 모든 문자를 'l'로 바꾼 뒤, 변환된 문자열을 반환하는 문제다. 문자를 직접 비교하며 수정하는 간단한 문자열 처리 문제다.

    • 문자열을 문자 배열로 변환한다. (char[] array = myString.toCharArray();)
    • 각 문자를 순회하면서, 'l'보다 앞에 있는 문자(즉, array[i] < 'l')일 경우 해당 문자를 'l'로 바꾼다.
    • 모든 문자를 검사한 뒤, 수정된 배열을 문자열로 다시 변환해 반환한다. 
    // 예시
      String myString = "abcdevwxyz";
      System.out.println(new Solution().solution(myString));
      // "lllllvwxyz"  ('l'보다 앞인 a~e → 'l'로 변경)
      
      myString = "hello";
      System.out.println(new Solution().solution(myString));
      // "llllo"

    시간 복잡도는 문자열 길이를 n이라 할 때 각 문자를 한 번씩만 확인하므로 O(n)이다. 공간 복잡도는 문자 배열로 한 번 변환하므로 O(n)이다.

    주의

    • 입력 문자열이 이미 모두 'l' 이상인 경우, 변화 없이 그대로 반환된다.
    • 영문 소문자 외의 문자가 포함될 경우, 비교 결과가 의도와 다를 수 있다.
    • 대문자는 'A' < 'l'로 처리되므로 모두 'l'로 바뀔 수 있다 — 문제에서는 모든 입력이 소문자였다.
    대안 아이디어
    • StringBuilder를 사용해 직접 문자를 덧붙이는 방식으로도 구현할 수 있다.
    • Stream API와 map을 이용해 함수형으로 처리할 수도 있다.
    • 정규식을 활용하여 [a-k] 패턴을 'l'로 치환하는 방법도 있다. (myString.replaceAll("[a-k]", "l"))
