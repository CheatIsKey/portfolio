    문제 1 — 정사각형으로 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    2차원 정수 배열 arr이 주어졌을 때, 가장 긴 행의 길이와 행의 개수 중 더 큰 값을 한 변의 길이로 하여 정사각형 크기의 새 배열을 만들고, 기존 원소를 같은 위치에 복사해 반환하는 문제다. 부족한 칸은 기본값 0으로 채워진다(자바의 배열 초기화 특성).

    • row = arr.length로 행의 개수를 구한다.
    • 모든 행을 돌며 column = 각 행 길이의 최댓값을 계산한다(자바의 2차원 배열은 가변 길이일 수 있음).
    • size = max(row, column)로 목표 정사각형 한 변의 길이를 정한다.
    • size × size 새 배열을 생성한다(새 배열의 나머지 칸은 자동으로 0).
    • 원본 배열의 유효 범위(i < row, j < arr[i].length)만 순회하며 값을 그대로 복사한다.

    시간 복잡도는 한 번의 최대 길이 탐색과 한 번의 복사로, O(R + S) (여기서 R은 행 수, S는 모든 원소 수의 합)이며, 보통 O(R · Cmax)로 볼 수 있다. 공간 복잡도는 결과 배열 크기에 비례해 O(size²)이다.

    주의

    • 자바의 2차원 배열은 jagged 구조일 수 있으므로, 각 행마다 arr[i].length를 사용해 인덱스 범위를 확인한다.
    • 복사 시 인덱스 조건을 명확히 해 ArrayIndexOutOfBoundsException을 방지한다.
    • 빈 배열(arr.length == 0)이나 빈 행(arr[i].length == 0)이 있어도 로직이 자연스럽게 처리되도록 한다.
    대안 아이디어
    • 각 행을 Arrays.copyOf로 size 길이까지 확장한 뒤, 부족한 행 수만큼 길이 size의 0-배열을 추가해도 된다.
    • 행 복사에는 System.arraycopy를 써서 미세한 성능 향상을 노릴 수 있다(유효 범위만 복사).
    • 불변성을 강조하려면, 원본을 건드리지 않고 항상 새 배열을 만들어 반환하는 방식을 유지한다.

    문제 2 — 이차원 배열 대각선 순회하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    2차원 정수 배열 board와 정수 k가 주어졌을 때, 행 인덱스와 열 인덱스의 합이 k 이하인 위치의 원소들만 골라 그 값을 모두 더한 결과를 반환하는 문제다.

    • 이중 반복문을 이용해 배열의 모든 좌표 (i, j)를 순회한다.
    • 각 좌표마다 i + j ≤ k이면 해당 원소를 합산한다.
    • 모든 원소를 확인한 후 누적된 합을 반환한다.

    시간 복잡도는 배열의 모든 칸을 한 번씩 확인하므로 O(n × m)이고, 공간 복잡도는 추가 변수 sum만 사용하므로 O(1)이다.

    주의

    • 조건은 i + j ≤ k이므로, 대각선 형태로 제한된 영역만 합산된다.
    • k가 충분히 크면 전체 배열의 합과 동일해질 수 있다.
    • 빈 배열이나 열의 길이가 불규칙한 경우(jagged array)에도 인덱스 접근 시 예외가 발생하지 않도록 유효 범위를 확인한다.
    대안 아이디어
    • 이중 반복문 대신 IntStream을 이용해 조건 필터링 및 합산을 간결하게 표현할 수 있다. 
      // 예시
      int sum = IntStream.range(0, board.length)
          .map(i -> IntStream.range(0, board[i].length)
              .filter(j -> i + j <= k)
              .map(j -> board[i][j])
              .sum())
          .sum();
    • 단, 반복문 방식이 더 명확하고 성능상 유리하다.

    문제 3 — 옹알이 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 babbling이 주어졌을 때, 각 문자열이 "aya", "ye", "woo", "ma" 네 가지 발음만으로 구성되어 있다면 그 문자열을 발음할 수 있다고 판단하여, 발음 가능한 문자열의 개수를 세어 반환하는 문제다.

    • 가능한 발음 목록을 {"aya", "ye", "woo", "ma"}로 정의한다.
    • 각 문자열 s에 대해 모든 발음을 순회하며 s.replace(word, " ")로 해당 발음을 제거한다.
    • 모든 발음을 제거한 후 남은 문자열의 공백을 제거한다(s.replace(" ", "")).
    • 결과 문자열이 빈 문자열("")이라면, 해당 단어는 네 가지 발음만으로 이루어진 것이므로 카운트를 증가시킨다.

    시간 복잡도는 각 단어마다 4개의 발음을 검사하므로 대략 O(N × M) (여기서 N은 단어 수, M은 평균 문자열 길이)이다. 공간 복잡도는 문자열 복사에 비례하지만, 보조 변수 외엔 추가 공간이 없어 O(1)로 볼 수 있다.

    주의

    • 이 방식은 연속해서 같은 발음이 반복된 경우(예: "ayaaya")도 유효하게 처리한다.
    • 문제의 확장 버전에서는 같은 발음을 연속해서 사용할 수 없다는 제약이 있으므로, 그 경우엔 별도의 검증 로직이 필요하다.
    • 문자열 치환 시 공백을 다른 문자로 바꾸더라도 무방하지만, 이후 replace(" ", "")로 일관되게 제거해야 한다.
    대안 아이디어
    • String.replaceAll()과 정규식을 사용해 한 번에 처리할 수도 있다. 
      // 예시
      s = s.replaceAll("aya|ye|woo|ma", "");
      if (s.isEmpty()) cnt++;
    • 정규식 사용 시 코드가 간결해지지만, 반복 발음 검증을 세밀하게 제어하기는 어렵다.
