    문제 1 — 문자열 밀기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 문자열 A와 B가 주어졌을 때, 문자열 A를 오른쪽으로 몇 번 회전하면 B가 되는지를 구하는 문제다.

    • B를 두 번 이어붙여 B + B를 만든다.
    • 이 문자열 안에 A가 포함되어 있다면, 그 시작 인덱스가 바로 회전 횟수다.
    • 포함되어 있지 않다면, -1을 반환한다.

    예를 들어, A = "hello", B = "ohell"일 때, B + B = "ohellohell" 안에서 A는 인덱스 1에 위치하므로 정답은 1이다.

    시간 복잡도는 문자열 탐색에 걸리는 시간으로 O(N)이다 (N은 문자열의 길이). 공간 복잡도는 B + B를 저장하므로 O(N)이다.

    주의

    • indexOf()는 찾는 문자열이 없을 경우 -1을 반환한다.
    • 문자열 길이가 다르면 어떤 회전으로도 같아질 수 없으므로, 이 경우에도 -1을 반환해야 한다.
    • 문자열 비교는 대소문자를 구분하므로, 입력이 동일한 형식으로 주어진다고 가정한다.
    대안 아이디어
    • 직접 회전을 반복하며 비교하는 방법도 가능하지만, 문자열 연결(B + B) 방식이 훨씬 효율적이다.
    • Java의 contains()를 이용해 포함 여부만 판별할 수도 있다.
    • 회전 횟수를 찾지 않고 단순히 가능 여부만 판단하는 문제라면 B.concat(B).contains(A) 한 줄로 해결 가능하다.

    문제 2 — 잘라서 배열로 저장하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_str을 길이 n으로 잘라서 순서대로 문자열 배열로 반환하는 문제다.

    • 문자열을 앞에서부터 n개씩 잘라 List에 저장한다.
    • 마지막 부분은 문자열의 길이가 n으로 나누어떨어지지 않더라도 남은 부분만큼 잘라서 포함시킨다.
    • 모든 조각을 저장한 후, List를 String[] 배열로 변환하여 반환한다.

    예를 들어, my_str = "abc1Addfggg4556b", n = 6이라면 문자열은 아래와 같이 잘린다. 

          ["abc1Ad", "dfggg4", "556b"]

    시간 복잡도는 문자열을 한 번 순회하므로 O(N)이며, 공간 복잡도는 잘린 문자열을 저장하는 리스트에 비례하여 O(N)이다.

    주의

    • substring(i, end)에서 end 인덱스는 포함되지 않으므로 자르기 구간을 정확히 지정해야 한다.
    • 문자열 길이가 n보다 작을 경우, 전체 문자열이 한 조각으로 반환된다.
    • 빈 문자열 입력은 주어지지 않는다고 가정한다.
    대안 아이디어
    • for문 대신 Stream API를 사용해 함수형 스타일로 구현할 수도 있다.
    • 직접 배열 크기를 계산 후, String[] result = new String[(my_str.length() + n - 1) / n] 형태로 배열에 바로 저장하는 방법도 있다.
    • 성능 차이는 미미하므로, 가독성이 높은 현재 방식이 일반적으로 더 선호된다.

    문제 3 — 7의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 array가 주어졌을 때, 배열의 모든 원소를 확인하여 숫자 7이 등장하는 횟수를 세어 반환하는 문제다.

    • 각 원소를 하나씩 확인하면서, 숫자에 7이 포함되어 있으면 개수를 증가시킨다.
    • 한 자리 수가 7인 경우는 바로 카운트한다.
    • 그 외의 경우에는 각 자리수를 나누어(i % 10) 일의 자리부터 순차적으로 검사한다.
    • i /= 10으로 자릿수를 줄이며 모든 자리를 확인한다.

    예를 들어, array = [7, 77, 17]일 경우 7은 1회, 77은 2회, 17은 1회 등장하므로 총 4를 반환한다.

    시간 복잡도는 배열의 길이를 N, 각 숫자의 자릿수를 최대 K라 할 때 O(N × K)이다. 공간 복잡도는 추가적인 자료구조를 사용하지 않으므로 O(1)이다.

    주의

    • 각 원소가 0인 경우는 자릿수 검사 루프(while (i > 0))에 들어가지 않으므로 별도로 고려할 필요가 없다.
    • 음수 입력은 문제 조건상 주어지지 않는다.
    • 숫자를 문자열로 변환하지 않고, 나눗셈과 나머지 연산으로 자릿수를 직접 확인하기 때문에 불필요한 객체 생성이 없다.
    대안 아이디어
    • 각 숫자를 문자열로 변환해 String.valueOf(i).chars()로 검사할 수도 있다.
    • 하지만 현재 방식은 문자열 변환 없이 연산하므로 성능 면에서 더 효율적이다.
    • 간단한 문제에서는 Arrays.stream(array)와 filter를 조합한 함수형 접근도 가능하다.
