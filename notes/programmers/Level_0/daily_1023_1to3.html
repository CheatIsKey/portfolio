    문제 1 — 문자열 정렬하기 (2)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string이 주어졌을 때, 모든 문자를 소문자로 바꾼 뒤 사전 순으로 정렬하여 반환하는 문제다.

    • 문자열을 모두 소문자로 변환한다. (toLowerCase())
    • 문자 배열로 변환하여 (toCharArray()) 각 문자를 정렬한다. (Arrays.sort())
    • 정렬된 문자 배열을 다시 문자열로 만든다. (new String(array))

    예를 들어, "Bcad" → 소문자로 바꾸면 "bcad", 정렬하면 "abcd"가 된다.

    시간 복잡도는 정렬 과정에서 O(N log N)이고 (N은 문자열의 길이), 공간 복잡도는 문자 배열을 저장하므로 O(N)이다.

    주의

    • 대문자와 소문자가 섞여 있어도 반드시 소문자로 통일해야 한다.
    • Arrays.sort()는 기본적으로 유니코드 순서에 따라 정렬한다.
    • 문자열 길이가 0이거나 공백 문자만 있는 경우에도 정상적으로 동작한다.
    대안 아이디어
    • Java 8 이상에서는 Stream을 활용해 my_string.toLowerCase().chars().sorted()로도 구현 가능하다.
    • 하지만 배열로 변환 후 정렬하는 방식이 가장 단순하고 직관적이다.

    문제 2 — 세균 증식

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    어떤 수 n이 1시간에 2배씩 증가한다고 할 때, t시간 후의 세균 수를 구하는 문제다.

    • 세균 수는 매 시간마다 2배가 되므로, t시간 후에는 n × 2t가 된다.
    • Java에서는 거듭제곱 계산을 위해 Math.pow(2, t)를 사용한다.
    • 결과를 정수형으로 반환하기 위해 (int)로 형 변환한다.

    예를 들어, n = 2, t = 10일 때 2 × 210 = 2048이므로 결과는 2048이다.

    시간 복잡도는 단순 산술 연산이므로 O(1)이며, 공간 복잡도 또한 상수 크기이므로 O(1)이다.

    주의

    • Math.pow()는 double형을 반환하므로 정수형으로 명시적 변환이 필요하다.
    • 입력값이 매우 커질 경우, 오버플로(overflow)가 발생할 수 있다.
    • 지수 t가 0이면 결과는 항상 n이 된다.
    대안 아이디어
    • 비트 연산을 이용하면 더 효율적으로 계산할 수 있다. 예: n << t (왼쪽 시프트 연산은 2t을 곱하는 것과 동일)
    • 단, 비트 연산은 정수 범위를 벗어나는 경우에 주의해야 한다.

    문제 3 — 제곱수 판별하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    어떤 자연수 n이 주어졌을 때, n이 제곱수인지 판별하는 문제다. 제곱수라면 1을, 아니면 2를 반환한다.

    • Math.sqrt(n)을 사용해 n의 제곱근을 구한다.
    • 제곱근이 정수라면 n은 어떤 수의 제곱이므로 1을 반환한다.
    • 정수가 아니라면 제곱수가 아니므로 2를 반환한다.

    예를 들어, n = 144일 때 √144 = 12.0이므로 정수 → 1을 반환하고, n = 976일 때는 √976 ≈ 31.24로 정수가 아니므로 2를 반환한다.

    시간 복잡도는 제곱근 계산이 한 번만 이루어지므로 O(1), 공간 복잡도도 상수 크기이므로 O(1)이다.

    주의

    • Math.sqrt()는 double형을 반환하므로 정수 비교를 위해 (int)로 변환해야 한다.
    • 부동소수점 연산 특성상 매우 큰 수에서는 오차가 발생할 수 있다.
    • 문제의 입력 범위 내에서는 오차 문제 없이 안전하게 동작한다.
    대안 아이디어
    • 제곱근의 정수 여부를 확인하는 대신, int x = (int) Math.sqrt(n); 후 x * x == n으로 판단할 수도 있다.
    • 이 방식은 부동소수점 비교 오차를 피할 수 있다는 장점이 있다.
