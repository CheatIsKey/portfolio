    문제 1 — 문자열안에 문자열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 str1과 str2가 주어졌을 때, str2가 str1 안에 포함되어 있는지를 판별하는 문제다.

    • String.contains() 메서드를 사용하여 str1에 str2가 포함되어 있는지 확인한다.
    • 포함되어 있다면 1, 포함되어 있지 않다면 2를 반환한다.

    예를 들어, str1 = "ab6CDE443fgh22iJKlmn1o", str2 = "6CD"라면 str1.contains(str2)의 결과는 true이므로 1을 반환한다.

    시간 복잡도는 String.contains()가 내부적으로 문자열 탐색을 수행하므로 O(N × M)이다. (N은 str1의 길이, M은 str2의 길이)

    공간 복잡도는 추가적인 메모리를 거의 사용하지 않으므로 O(1)이다.

    주의

    • contains()는 대소문자를 구분한다. 필요 시 toLowerCase()나 toUpperCase()로 통일해야 한다.
    • str2가 빈 문자열("")인 경우, contains()는 항상 true를 반환한다.
    • 공백이나 특수문자도 문자열의 일부로 인식하므로 그대로 비교된다.
    대안 아이디어
    • indexOf()를 사용하여 str1.indexOf(str2) != -1로 판별할 수도 있다.
    • 두 방법 모두 동일한 결과를 주지만, contains()가 더 직관적이고 가독성이 높다.

    문제 2 — OX퀴즈

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 quiz가 주어졌을 때, 각 문자열이 수식 형태("X [+, -] Y = Z")로 주어지며, 해당 수식이 참이면 "O", 거짓이면 "X"를 반환하는 문제다.

    • 결과를 저장할 문자열 배열 arr을 생성한다.
    • 각 수식 문자열을 공백(" ") 기준으로 나눈다. (split(" "))
    • 연산자(split[1])에 따라 덧셈 또는 뺄셈을 수행한다.
    • 계산 결과가 우변(split[4])과 같다면 "O", 다르면 "X"를 저장한다.

    예를 들어, "3 - 4 = -1"의 경우 3 - 4 == -1이므로 "O"를, "5 + 6 = 10"의 경우 5 + 6 ≠ 10이므로 "X"를 반환한다.

    시간 복잡도는 각 문자열을 한 번씩 처리하므로 O(N)이다. (N은 quiz 배열의 길이)

    공간 복잡도는 결과 배열 arr을 저장하므로 O(N)이다.

    주의

    • 각 수식은 정확히 공백으로 구분되어야 한다. (예: "3-4=-1" 형태는 오류 발생)
    • 덧셈(+)과 뺄셈(-)만 처리하며, 다른 연산자가 등장할 경우 예외를 발생시킨다.
    • 정수 범위를 벗어나는 연산은 문제 조건상 주어지지 않는다.
    대안 아이디어
    • eval()과 같은 식 평가 함수를 직접 구현하여 문자열을 파싱하고 계산하는 방식으로도 가능하다.
    • 하지만 본 풀이처럼 split()과 Integer.parseInt()를 이용하는 방식이 가장 단순하고 명확하다.

    문제 3 — 자릿수 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 n이 주어졌을 때, 각 자리 숫자의 합을 구하여 반환하는 문제다.

    • sum 변수를 0으로 초기화한다.
    • while 문을 사용해 n이 0보다 큰 동안 반복한다.
    • 매 반복마다 n % 10으로 마지막 자릿수를 구해 sum에 더한다.
    • n /= 10을 통해 마지막 자릿수를 제거한다.
    • 모든 자릿수를 더한 후 sum을 반환한다.

    예를 들어, n = 1234라면 1 + 2 + 3 + 4 = 10이므로 결과는 10이 된다.

    시간 복잡도는 자릿수의 개수에 비례하므로 O(log₁₀N)이다. (N은 입력된 정수 n)

    공간 복잡도는 추가 변수 sum만 사용하므로 O(1)이다.

    주의

    • n이 0일 경우, 반복문을 실행하지 않고 sum = 0을 반환한다.
    • 음수가 입력되는 경우는 문제 조건상 주어지지 않는다.
    • 문자열 변환 없이 정수 연산만으로 처리하므로 효율적이다.
    대안 아이디어
    • 문자열로 변환한 뒤 각 문자를 더하는 방법도 가능하다. (String.valueOf(n).chars().map(c - '0').sum())
    • 하지만 본 풀이처럼 나머지 연산(%)을 이용한 방식이 더 빠르고 메모리 사용이 적다.
