    문제 1 — 문자열 계산하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    공백(" ")으로 구분된 간단한 사칙연산 수식 문자열 my_string을 왼쪽부터 순차적으로 평가하여 정수 결과를 반환하는 문제다. 이 풀이에서는 +, -, *, / 네 가지 연산자를 인식하고, "="가 등장하면 계산을 즉시 종료한다.

    • split(" ")으로 수식을 토큰 단위(피연산자와 연산자)로 분리한다.
    • 현재 연산자를 저장하는 변수 operator를 "+"로 초기화한다.
    • 토큰을 앞에서부터 순회하며
      • 정수로 파싱 가능한 경우(피연산자): 현재 operator에 따라 누적값 sum에 적용한다.
      • 정수로 파싱 불가한 경우(연산자/기호): "="이면 반복을 종료하고, "+", "-", "*", "/" 중 하나면 operator를 갱신한다.
    • 모든 처리가 끝나면 누적값 sum을 반환한다.

    예를 들어, my_string = "3 + 4 * 2 ="라면 토큰을 왼쪽부터 순차 적용하여 (((0 + 3) + 4) * 2) = 14가 결과로 반환된다. (이 풀이는 연산자 우선순위를 적용하지 않고 좌에서 우로만 계산한다.)

    시간 복잡도는 토큰의 개수를 T라 할 때 각 토큰을 한 번씩만 처리하므로 O(T)다.

    공간 복잡도는 입력 분할을 위한 배열과 상수 크기의 변수를 사용하므로 O(T) (토큰 배열) / 추가 작업 변수는 O(1)이다.

    주의

    • "/" 연산에서 피연산자가 0이면 ArithmeticException이 발생한다.
    • 이 로직은 연산자 우선순위(예: 곱셈·나눗셈 우선)를 고려하지 않고, 입력 순서대로 계산한다.
    • int 범위를 벗어나는 큰 수는 오버플로가 발생할 수 있다.
    • 정수가 아닌 토큰은 NumberFormatException으로 분기 처리한다. (알 수 없는 문자는 경고 출력)
    • "="이 없는 경우 끝까지 계산한 누적값을 반환한다.
    • "/"는 자바의 int 나눗셈이므로 버림이 적용된다.
    대안 아이디어
    • 문제 조건이 +, -만이라면: *, /, "=" 처리를 제거하고, 토큰을 num, op, num, op, ... 형태로 가정해 더 간결하게 구현할 수 있다.
    • 스트림 사용: my_string.split(" ")로 토큰화한 뒤, 인덱스를 기준으로 홀/짝 토큰을 분리해 누적하는 방식으로도 구현 가능하다. (가독성은 취향 차이)
    • 우선순위를 반영하려면 스택 기반 파서(또는 Shunting Yard)로 확장한다.

    문제 2 — 가장 큰 수 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 array가 주어졌을 때, 배열에서 가장 큰 값(최댓값)과 그 값의 인덱스를 함께 반환하는 문제다. 반환 형태는 [최댓값, 인덱스]이다.

    • 초기값으로 max = array[0], idx = 0을 설정한다.
    • for문으로 배열의 두 번째 원소(i = 1)부터 끝까지 순회한다.
    • 현재 원소가 max보다 크면, max와 idx를 각각 현재 값과 인덱스로 갱신한다.
    • 모든 탐색이 끝나면 new int[]{max, idx} 배열을 반환한다.

    예를 들어, array = [1, 8, 3]일 때 순회 과정은 다음과 같다.

    • 초기값: max = 1, idx = 0
    • i = 1 → array[1] = 8 > 1 → max = 8, idx = 1
    • i = 2 → array[2] = 3 < 8 → 변화 없음
    • 결과: [8, 1]

    시간 복잡도는 배열을 한 번만 순회하므로 O(N)이다. (N은 배열의 길이)

    공간 복잡도는 상수 크기의 변수(max, idx)와 반환 배열만 사용하므로 O(1)이다.

    주의

    • 배열의 길이가 1인 경우, 결과는 [array[0], 0]이다.
    • 동일한 최댓값이 여러 개 존재할 경우, 가장 먼저 등장한 인덱스를 반환한다.
    • 빈 배열이 주어지면 ArrayIndexOutOfBoundsException이 발생하므로 문제 조건상 입력 배열은 비어 있지 않아야 한다.
    대안 아이디어
    • 스트림 사용: IntStream.range(0, array.length)를 이용하여 max()와 indexOf()를 조합할 수도 있다.
    • Collections 사용: Arrays.asList()로 리스트 변환 후 Collections.max()로 최댓값을 찾을 수도 있으나, 기본형 int[]은 자동 박싱이 필요해 비효율적이다.
    • 이 문제의 핵심은 단일 순회로 최댓값과 인덱스를 함께 추적하는 기본 로직이다.

    문제 3 — 편지

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 message가 주어졌을 때, 각 문자당 2cm의 폭을 차지한다고 가정하면 전체 문장의 총 길이를 구하는 문제다. 즉, 문자 개수 × 2의 값을 반환한다.

    • 변수 result를 0으로 초기화한다.
    • for문을 이용해 문자열의 모든 문자(i = 0부터 message.length() - 1까지)를 순회한다.
    • 각 문자마다 2cm를 더하므로, 루프마다 result += 2;를 수행한다.
    • 모든 문자를 처리한 후 result를 반환한다.

    예를 들어, message = "happy birthday!"일 때 총 문자의 수는 15개이므로 15 × 2 = 30을 반환한다.

    시간 복잡도는 문자열의 길이를 N이라 할 때 각 문자를 한 번씩 순회하므로 O(N)이다.

    공간 복잡도는 상수 크기의 변수 result만 사용하므로 O(1)이다.

    주의

    • message가 빈 문자열이면 result는 0이 된다.
    • 공백 문자(' ')나 특수문자도 1글자로 계산되어 동일하게 2cm로 더해진다.
    • 문자열 길이를 직접 계산할 수 있으므로 반복문 대신 message.length() * 2로 단순화 가능하다.
    대안 아이디어
    • 간결한 표현: return message.length() * 2; 로 한 줄로 구현 가능하다.
    • 문제의 의도는 반복문을 통해 문자열의 길이를 직접 순회하며 기본적인 문자열 처리 구조를 익히는 것이다.
