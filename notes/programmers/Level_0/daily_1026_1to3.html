    문제 1 — n의 배수 고르기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 n과 정수 배열 numlist가 주어졌을 때, numlist 중에서 n의 배수인 원소만 추출하여 새로운 배열로 반환하는 문제다.

    • for-each 문을 이용해 배열의 각 원소 i를 순회한다.
    • i % n == 0 조건을 만족하면 리스트에 추가한다.
    • 모든 검사가 끝난 뒤, List<Integer>를 int[]로 변환해 반환한다.

    예를 들어, n = 3, numlist = [4, 5, 6, 7, 9, 12]라면 3의 배수인 [6, 9, 12]가 결과로 반환된다.

    시간 복잡도는 numlist의 모든 원소를 한 번씩 검사하므로 O(N)이다. (N은 배열의 길이)

    공간 복잡도는 배수인 원소를 저장하기 위한 리스트 공간이 필요하므로 최악의 경우 O(N)이다.

    주의

    • n이 0인 경우, i % n 연산은 오류(ArithmeticException)를 일으킨다.
    • 입력 배열이 비어 있으면 결과도 빈 배열이 된다.
    • 결과 배열의 순서는 입력 배열의 순서를 그대로 유지한다.
    대안 아이디어
    • Java 8 이상의 버전에서는 Arrays.stream(numlist)를 사용하여 filter(i -> i % n == 0)로 더 간결하게 표현할 수 있다.
    • 하지만 문제 풀이의 의도는 반복문과 조건문을 이용한 기본적인 필터링 로직 이해에 있다.

    문제 2 — 숫자 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 num과 한 자리 정수 k가 주어졌을 때, num의 자릿수 중 k가 처음으로 등장하는 위치(1-based index)를 찾아 반환하는 문제다. 만약 k가 포함되어 있지 않다면 -1을 반환한다.

    • String.valueOf(num)으로 num을 문자열로 변환한다.
    • contains()로 k가 문자열 안에 존재하는지 확인한다.
    • 포함되어 있다면 indexOf()로 처음 등장하는 인덱스를 구한 뒤 0-based이므로 +1을 더해 반환한다.
    • 포함되어 있지 않다면 -1을 반환한다.

    예를 들어, num = 29183, k = 1이라면 "29183".contains("1")의 결과는 true이고, indexOf("1")의 결과는 2이므로 최종적으로 2 + 1 = 3을 반환한다.

    시간 복잡도는 contains()와 indexOf()가 내부적으로 문자열을 선형 탐색하므로 O(N)이다. (N은 num의 자릿수)

    공간 복잡도는 정수를 문자열로 변환하는 데 필요한 공간만 사용하므로 O(N)이다.

    주의

    • k는 반드시 한 자리 정수여야 한다.
    • num이 0으로 시작하는 경우는 입력으로 주어지지 않는다.
    • 결과 인덱스는 1부터 시작한다는 점에 유의한다.
    대안 아이디어
    • indexOf()만 사용하여 int idx = s.indexOf(String.valueOf(k)); 후 return idx == -1 ? -1 : idx + 1;로 구현할 수도 있다.
    • 하지만 contains()를 함께 사용하면 논리 흐름이 더 명확하고 가독성이 높다.

    문제 3 — 배열의 유사도

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 s1과 s2가 주어졌을 때, 두 배열에서 공통으로 포함된 문자열의 개수를 구하는 문제다.

    • Arrays.asList(s2)를 이용해 s2를 리스트로 변환한다.
    • 이 리스트를 기반으로 HashSet을 생성하여 O(1) 평균 시간에 포함 여부를 확인할 수 있도록 한다.
    • s1의 각 원소를 순회하며, set.contains(s)가 true일 경우 공통 원소로 간주하고 카운트를 1 증가시킨다.
    • 모든 순회를 마친 뒤, 최종적으로 공통 문자열의 개수를 반환한다.

    예를 들어, s1 = ["a", "b", "c"], s2 = ["com", "b", "d", "p", "c"]라면 공통된 원소는 "b"와 "c"로 총 2개이므로 결과는 2가 된다.

    시간 복잡도는 s2를 HashSet으로 만드는 데 O(M), s1을 순회하며 포함 여부를 검사하는 데 O(N)이 걸리므로 전체는 O(N + M)이다. (N은 s1의 길이, M은 s2의 길이)

    공간 복잡도는 s2의 원소를 저장하는 HashSet에 의해 O(M)이다.

    주의

    • 배열의 원소는 중복되지 않는다는 보장이 없으므로, 중복 문자열이 있을 경우 set으로 중복이 자동 제거된다.
    • 대소문자를 구분하므로, "A"와 "a"는 다른 문자열로 취급된다.
    • 배열이 비어 있는 경우 결과는 0이다.
    대안 아이디어
    • List.contains()를 이용해 set 대신 리스트를 순회하며 검사할 수도 있으나, 시간 복잡도가 O(N × M)으로 증가한다.
    • 따라서 HashSet을 사용하는 방법이 훨씬 효율적이다.
