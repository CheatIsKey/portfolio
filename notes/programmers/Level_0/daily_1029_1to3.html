    문제 1 — 369게임

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    양의 정수 order의 각 자릿수를 검사해 3, 6, 9가 등장한 횟수를 세어 반환하는 문제다. 이 풀이에서는 나눗셈과 나머지 연산을 이용해 자릿수를 뒤에서부터 하나씩 분리하며, 분리된 자릿수가 3 또는 6 또는 9인지 확인해 카운트를 증가시킨다.

    • while (order > 0) 루프를 돌며 각 반복에서 order % 10으로 마지막 자릿수를 구한다.
    • 자릿수가 3·6·9 중 하나면 cnt++로 증가시킨다.
    • order /= 10으로 마지막 자릿수를 버리고 다음 자릿수로 진행한다.
    • 모든 자릿수를 확인한 뒤 누적된 cnt를 반환한다.

    예시

    • order = 3 ⟶ 자릿수: 3 ⟶ 카운트: 1 ⟶ 반환: 1
    • order = 29423 ⟶ 자릿수: 3, 2, 4, 9, 2 ⟶ 3과 9 2개 ⟶ 반환: 2
    • order = 999 ⟶ 자릿수: 9, 9, 9 ⟶ 카운트: 3 ⟶ 반환: 3
    • order = 0 ⟶ 루프 미진입 ⟶ 반환: 0 (0에는 3,6,9가 없음)

    시간 복잡도

    자릿수 수를 D라 할 때 각 자릿수를 한 번씩 확인하므로 O(D)다. (동치로 O(log10 order).)

    공간 복잡도

    상수 크기의 변수(cnt 등)만 사용하므로 추가 공간은 O(1)이다.

    주의

    • 입력 범위: 문제 조건상 정수가 주어진다. order = 0인 경우 루프에 들어가지 않으나, 결과는 올바르게 0이다.
    • 조건식 평가: (digit == 3 || digit == 6 || digit == 9)는 상수 비교 3회로 충분히 가볍다.
    대안 아이디어
    • 문자열 변환: String s = String.valueOf(order); 후 for (char c : s.toCharArray())에서 c == '3' || c == '6' || c == '9'인지 검사. 구현이 직관적이며 복잡도는 동일(O(D)).
    • 스트림 사용(Java 8+): String.valueOf(order).chars().filter(ch -> ch=='3'||ch=='6'||ch=='9').count()로 구현. 코드가 짧아지나 가독성/선호는 팀 컨벤션에 따름.

    문제 2 — 가까운 수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 array에서 주어진 수 n과의 차이가 가장 작은 원소를 찾는 문제다. 만약 차이가 같은 원소가 여러 개라면, 더 작은 수를 반환해야 한다. 이 풀이에서는 모든 원소를 순회하며 n과의 절댓값 차이를 계산하고, 최소 차이와 그때의 값을 갱신하는 방식으로 해결한다.

    • min에는 현재까지의 최소 차이를 저장하며 초기값은 Integer.MAX_VALUE로 설정한다.
    • result에는 최소 차이를 가진 원소를 저장하며 초기값은 Integer.MAX_VALUE로 설정한다.
    • for (int i : array) 루프로 각 원소를 순회하며 tmp = Math.abs(i - n)로 차이를 계산한다.
    • 만약 tmp < min이면 새로운 최소 차이를 발견한 것이므로 min과 result를 갱신한다.
    • 또는 tmp == min인데 i < result라면, 같은 거리의 더 작은 수이므로 result를 갱신한다.
    • 모든 원소를 검사한 뒤 최종 result를 반환한다.

    예시

    • array = [3, 10, 28], n = 20 ⟶ 차이: [17, 10, 8] ⟶ 최소 차이 8인 28 반환
    • array = [10, 11, 12], n = 13 ⟶ 차이: [3, 2, 1] ⟶ 최소 차이 1인 12 반환
    • array = [7, 12, 15, 17], n = 14 ⟶ 차이: [7, 2, 1, 3] ⟶ 최소 차이 1인 15 반환
    • array = [4, 9, 8], n = 7 ⟶ 차이: [3, 2, 1] ⟶ 최소 차이 1인 8 반환

    시간 복잡도

    배열의 길이를 N이라 할 때, 모든 원소를 한 번씩 순회하므로 O(N)이다.

    공간 복잡도

    고정 크기의 변수(min, result, tmp)만 사용하므로 O(1)이다.

    주의

    • 동일 거리 처리: 같은 거리의 수가 둘 이상이면 더 작은 값을 선택해야 한다. (tmp == min && i < result) 조건으로 처리.
    • 정렬 불필요: 단순 선형 탐색으로 충분하므로 배열을 정렬할 필요가 없다.
    대안 아이디어
    • 정렬 기반 접근: 배열을 정렬한 뒤 이진 탐색(Arrays.binarySearch)으로 n의 삽입 위치를 찾고, 양옆 원소 중 더 가까운 값을 선택하는 방법. 복잡도는 O(N log N)으로 증가하므로 단순 순회보다 비효율적이다.
    • 스트림 사용(Java 8+): Arrays.stream(array).boxed().min(Comparator.comparingInt(i -> Math.abs(i - n)).thenComparingInt(i -> i)).get() 와 같이 표현할 수 있다. 동작은 동일하나 가독성은 팀 선호에 따라 다르다.

    문제 3 — 삼각형의 완성조건 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    세 변의 길이가 주어졌을 때 삼각형이 성립하는지 판별하는 문제다. 삼각형의 성립 조건은 ‘가장 긴 변의 길이가 나머지 두 변의 합보다 작을 것’이다. 이 풀이에서는 세 변을 정렬하여 가장 긴 변을 쉽게 찾고, 조건식으로 판별한다.

    • Arrays.sort(sides)로 세 변을 오름차순 정렬한다. (sides[2]가 가장 긴 변이 됨)
    • 삼각형 조건 sides[2] < sides[0] + sides[1]을 검사한다.
    • 조건이 참이면 1 (삼각형 가능), 거짓이면 2 (불가능)을 반환한다.

    예시

    • sides = [1, 2, 3] ⟶ 정렬 후 [1, 2, 3] ⟶ 3 < 1 + 2 → 거짓 → 반환: 2
    • sides = [3, 6, 2] ⟶ 정렬 후 [2, 3, 6] ⟶ 6 < 2 + 3 → 거짓 → 반환: 2
    • sides = [3, 4, 5] ⟶ 정렬 후 [3, 4, 5] ⟶ 5 < 3 + 4 → 참 → 반환: 1

    시간 복잡도

    Arrays.sort로 인해 O(N log N)이지만, N=3이므로 상수 시간(O(1))로 간주할 수 있다.

    공간 복잡도

    정렬 과정에서 사용하는 추가 메모리는 상수 크기이므로 O(1)이다.

    주의

    • 정렬 필요성: 세 변 중 어떤 것이 가장 긴 변인지 확실히 하기 위해 정렬을 수행한다.
    • 경계 조건: 가장 긴 변의 길이가 정확히 나머지 두 변의 합과 같을 경우(a + b == c) 삼각형은 성립하지 않는다.
    대안 아이디어
    • 수학적 비교 직접 수행: 정렬 없이 max와 나머지 두 변의 합을 구해 비교할 수도 있다. 예: int max = Math.max(sides[0], Math.max(sides[1], sides[2])); 후 나머지 두 변의 합과 비교.
    • 스트림 사용(Java 8+): int max = Arrays.stream(sides).max().getAsInt();로 최댓값을 구하고, 합과 비교하는 방식으로도 가능하다.
