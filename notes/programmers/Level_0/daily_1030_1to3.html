    문제 1 — 중복된 문자 제거

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string에서 중복 문자를 제거하되, 등장 순서를 보존하여 한 번씩만 남기는 문제다. 제시한 풀이에서는 Java 스트림을 사용해 문자를 순서대로 흘려보내며 distinct()로 최초 등장만 남기고, 다시 이어 붙여 결과 문자열을 만든다.

    • my_string.chars()로 문자열을 IntStream으로 변환한다(코드 유닛 순서 유지).
    • .mapToObj(c -> (char) c)로 각 정수 값을 Character 객체로 박싱한다.
    • .distinct()가 ordered stream의 만남 순서(encounter order)를 보존하며 최초로 등장한 문자만 통과시킨다.
    • .map(String::valueOf)로 각 문자를 문자열로 바꾸고 .collect(Collectors.joining())으로 이어 붙여 최종 결과를 만든다.

    예시

    • my_string = "people" ⟶ 순서: p, e, o, p, l, e ⟶ 남는 문자: p, e, o, l ⟶ 반환: "peol"
    • my_string = "banana" ⟶ 순서: b, a, n, a, n, a ⟶ 남는 문자: b, a, n ⟶ 반환: "ban"
    • my_string = "abc" ⟶ 중복 없음 ⟶ 반환: "abc"
    • my_string = "" ⟶ 빈 문자열 ⟶ 반환: ""

    시간 복잡도

    길이를 N이라 하면 각 문자를 한 번씩 지나며 distinct() 내부에서 해시 기반의 방문 체크가 이뤄지므로 평균 O(N)이다.

    공간 복잡도

    중복 제거를 위해 서로 다른 문자 수를 K라 할 때 방문 집합에 O(K)가 추가로 필요하다. 결과 문자열 보관에도 O(K)가 든다.

    주의

    • 순서 보존: distinct()는 ordered stream에서 최초 등장을 남긴다. 본 코드는 chars()가 입력 순서를 유지하므로 요구사항(등장 순서 보존)에 부합한다.
    대안 아이디어
    • LinkedHashSet 사용(순서 보존):
      var set = new LinkedHashSet<Character>();
      for (char ch : my_string.toCharArray()) set.add(ch);
      return set.stream().map(String::valueOf).collect(Collectors.joining());
      명시적으로 순서 보존 집합을 사용한다. 복잡도는 동일(O(N)).
    • codePoints()로 이모지 안전 처리 (제약 없을 때):
      return my_string.codePoints()
        .distinct()
        .mapToObj(cp -> new String(Character.toChars(cp)))
        .collect(Collectors.joining());
    • 배열 방문 체크(입력이 소문자 a–z로 제한일 때):
      boolean[] seen = new boolean[26];
      var sb = new StringBuilder();
      for (char c : my_string.toCharArray()) {
        int i = c - 'a';
        if (!seen[i]) { seen[i] = true; sb.append(c); }
      }
      return sb.toString();

    문제 2 — k의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 구간 [i, j] 내의 모든 숫자를 확인하며, 각 숫자에서 자릿수 k가 등장하는 횟수를 세어 합산하는 문제다. 이 풀이에서는 각 수를 10으로 나누어 자릿수를 하나씩 분리하고, 각 자릿수가 k와 일치하면 카운트를 증가시킨다.

    • for (int s = i; s <= j; s++)로 구간의 모든 수를 순회한다.
    • 각 수 s를 tmp에 복사한 뒤 while (tmp > 0) 루프로 자릿수를 분리한다.
    • tmp % 10으로 마지막 자릿수를 구하고, 이 값이 k와 같으면 cnt++로 카운트를 증가시킨다.
    • tmp /= 10으로 마지막 자릿수를 버리고 다음 자릿수로 넘어간다.
    • 모든 수를 검사한 뒤 누적된 cnt를 반환한다.

    예시

    • i = 1, j = 13, k = 1 ⟶ 검사 대상: 1~13
      1, 10, 11, 12, 13에서 총 6번 등장 ⟶ 반환: 6
    • i = 10, j = 50, k = 5 ⟶ 15, 25, 35, 45, 50 ⟶ 총 5번 등장 ⟶ 반환: 5
    • i = 3, j = 3, k = 3 ⟶ 한 수만 검사 ⟶ 자릿수 3 한 번 ⟶ 반환: 1
    • i = 20, j = 22, k = 1 ⟶ 모든 자릿수에 1 없음 ⟶ 반환: 0

    시간 복잡도

    구간 크기를 N = j - i + 1, 각 수의 자릿수를 D라 하면, 모든 수를 순회하며 각 자릿수를 확인하므로 전체 복잡도는 O(N × D)다. (10진수 기준 O(N × log₁₀(max(j))).)

    공간 복잡도

    상수 크기의 변수(cnt, tmp 등)만 사용하므로 추가 공간은 O(1)이다.

    주의

    • 0 포함 처리: tmp > 0 조건 때문에 s == 0인 경우 루프를 돌지 않는다. 단, 문제에서 i와 j는 양의 정수이므로 실제 입력에는 영향이 없다.
    • 한 자릿수 비교: tmp % 10 == k 비교 시 k는 0~9 범위의 정수이므로 형 변환이 필요 없다.
    대안 아이디어
    • 문자열 변환: for (int s = i; s <= j; s++) {
        String str = String.valueOf(s);
        for (char c : str.toCharArray())
          if (c == ('0' + k)) cnt++;
      }
      직관적이며 동일한 결과를 낸다. 복잡도도 동일 O(N × D).
    • 스트림 활용(Java 8+): return (int) IntStream.rangeClosed(i, j)
        .mapToObj(String::valueOf)
        .flatMapToInt(String::chars)
        .filter(ch -> ch == ('0' + k))
        .count();
      간결하지만, 내부적으로 문자열 변환 오버헤드가 있다.

    문제 3 — A로 B 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 문자열 before와 after가 같은 문자들로 이루어졌는지(순서 무관, 개수 동일)를 판별하는 문제다. 제시한 풀이는 각 문자열을 문자 배열로 변환해 Arrays.sort로 정렬한 뒤, 정렬 결과가 동일한지 Arrays.equals로 비교한다. 정렬하면 동일한 멀티셋을 가진 문자열은 같은 배열이 된다.

    • before.toCharArray(), after.toCharArray()로 문자 배열을 만든다.
    • Arrays.sort(arr1), Arrays.sort(arr2)로 두 배열을 사전순 정렬한다.
    • Arrays.equals(arr1, arr2)가 참이면 같은 멀티셋이므로 1, 아니면 0을 반환한다.

    예시

    • before = "olleh", after = "hello" ⟶ 정렬: ehllo, ehllo ⟶ 반환: 1
    • before = "allpe", after = "apple" ⟶ 정렬: aellp, aelpp ⟶ 반환: 0
    • before = "abc", after = "abc" ⟶ 이미 동일 ⟶ 정렬 동일 ⟶ 반환: 1
    • before = "", after = "" ⟶ 둘 다 빈 문자열 ⟶ 정렬 결과 동일 ⟶ 반환: 1

    시간 복잡도

    길이를 N이라 하면 정렬이 지배적이므로 O(N log N)이다.

    공간 복잡도

    toCharArray()로 만든 두 배열에 O(N) 추가 공간이 든다. Arrays.sort는 원소 타입에 따라 내부 보조 공간을 사용할 수 있으나 전체적으로 상수~선형 수준이며, 본 문제 관점에서는 추가 공간을 O(N)로 본다.

    주의

    • 대소문자/공백/특수문자: 정렬 비교는 문자 값을 그대로 비교하므로 입력에 대소문자나 공백, 숫자, 특수문자가 포함되면 모두 문자 그대로 개수까지 동일해야 한다.
    대안 아이디어
    • 빈도 배열(영문 소문자 가정):
      int[] cnt = new int[26];
      for (char c : before.toCharArray()) cnt[c - 'a']++;
      for (char c : after.toCharArray()) cnt[c - 'a']--;
      for (int v : cnt) if (v != 0) return 0;
      return 1;
      선형 시간 O(N)에 가능하나, 문자 집합 제약이 필요하다.
    • Map<Character,Integer> 사용 (문자 집합 제약 없음):
      해시맵으로 빈도를 세고 상쇄시키는 방식. 복잡도 O(N) 평균, 구현은 다소 장황.
