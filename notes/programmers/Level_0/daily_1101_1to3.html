    문제 1 — 등수 매기기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    각 학생의 두 과목 점수의 평균(또는 합)을 기준으로 내림차순 정렬했을 때의 공동 순위를 구하는 문제다. 제시한 코드는 학생 i의 평균 avg[i]를 계산한 뒤, avg[j] > avg[i]인 학생 수를 세어 rank[i] = (자기보다 평균이 높은 학생 수) + 1로 순위를 정한다. 이는 동점자가 있을 때 같은 등수를 부여하는 요구와 정확히 일치한다.

    • 평균 계산: avg[i] = (score[i][0] + score[i][1]) / 2.0
    • 순위 산출: 모든 쌍 (i, j)에 대해 avg[j] > avg[i]면 카운트하고, 그 개수에 1을 더해 rank[i]에 기록
    • 반환: 각 학생의 순위를 담은 정수 배열

    예시

    • score = [[80, 70], [90, 50], [40, 70], [50, 80]]
      평균(또는 합): [75, 70], [80, 70], [55, 70], [65, 70] → 순위: [2, 1, 4, 3]
    • score = [[100, 100], [100, 100], [90, 90]]
      평균(또는 합): [100, 100, 90] → 동점 처리 → 순위: [1, 1, 3]

    시간 복잡도

    평균 계산은 O(N), 순위 산출은 모든 쌍을 비교하므로 O(N^2). 전체적으로 O(N^2).

    공간 복잡도

    평균과 순위를 담는 배열 두 개만 추가로 사용하므로 O(N).

    주의

    • 부동소수 연산: 평균을 double로 계산해도 본 문제 범위에선 안전하나, 순위 기준은 평균과 합이 완전히 동일한 순서를 만든다. 따라서 정수 합(score[i][0] + score[i][1])으로 비교하면 부동소수 오차 우려를 없앨 수 있다.
    • 동일 점수: 비교 연산을 >로 둬야 동점자가 같은 등수를 받는다(>=가 되면 한쪽이 밀려 잘못된 순위가 된다).
    대안 아이디어
    • 정수 합으로 비교(부동소수 제거)
      
      import java.util.*;
      
      class Solution {
          public int[] solution(int[][] score) {
              int n = score.length;
              int[] sum = new int[n];
              int[] rank = new int[n];
              for (int i = 0; i < n; i++) sum[i] = score[i][0] + score[i][1];
              for (int i = 0; i < n; i++) {
                  int r = 1;
                  for (int j = 0; j < n; j++) if (sum[j] > sum[i]) r++;
                  rank[i] = r;
              }
              return rank;
          }
      }

      시간/공간 복잡도는 기존과 동일하다 (O(N^2), O(N)).

    • 정렬로 O(N log N)에 순위 매기기 — 큰 입력에서 유리
      
      import java.util.*;
      
      class Solution {
          public int[] solution(int[][] score) {
              int n = score.length;
              int[] ans = new int[n];
              int[][] a = new int[n][3]; // {sum, index, dummy}
              for (int i = 0; i < n; i++) a[i] = new int[]{score[i][0] + score[i][1], i, 0};
              Arrays.sort(a, (x, y) -> Integer.compare(y[0], x[0])); // 합 내림차순
              int rank = 1;
              for (int i = 0; i < n; i++) {
                  if (i > 0 && a[i][0] != a[i - 1][0]) rank = i + 1; // 동점 처리
                  ans[a[i][1]] = rank;
              }
              return ans;
          }
      }

      정렬로 한 번에 순위를 배정하므로 O(N log N).

    • 빈도 맵으로 순위 매기기 — 값 범위가 좁을 때 유용
      합 점수의 출현 빈도를 세어(예: Map<Integer, Integer>), 합 점수를 높은 순으로 누적해 시작 순위를 계산한 뒤 원래 인덱스에 해당 순위를 할당한다. 구현 난이도는 약간 높지만 O(N log K)(고유 합 개수 K)에 가능.

    문제 2 — 특이한 정렬

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    기준값 n에 대해 각 원소 x의 거리(|x - n|)를 계산해 거리 오름차순으로 정렬하고, 거리가 같으면 값이 큰 수가 먼저 오도록 값 내림차순으로 타이브레이크하는 문제다. 풀이 요지는 각 원소를 {값, 거리}로 확장해 비교 규칙을 적용해 정렬한 뒤, 정렬된 원소의 값만 순서대로 반환하는 것이다.

    • 기본 키: |x - n| 오름차순
    • 보조 키(동일 거리): x 내림차순
    • 정렬 후: 확장 배열에서 원본 값만 추려 반환

    예시

    • numlist = [1, 2, 3, 4, 5, 6], n = 4
      거리: [3,2,1,0,1,2] → 결과: [4, 5, 3, 6, 2, 1] (거리 1에서 5와 3 중 값 큰 5가 먼저)
    • numlist = [10, 12, 9, 8, 11], n = 10
      거리: [0,2,1,2,1] → 결과: [10, 11, 9, 12, 8] (거리 1에서 11 > 9)

    시간 복잡도

    정렬이 지배적이므로 O(N log N).

    공간 복잡도

    {값, 거리} 확장 배열과 결과 배열 사용으로 O(N).

    주의

    • 타이브레이크 방향: “거리 같으면 값 큰 순”을 정확히 구현해야 한다(값 내림차순).
    • 뺄셈 비교 오버플로: b - a 같은 방식은 극단값에서 오버플로 가능. Integer.compare / Long.compare 사용이 안전하다.
    • 절대값 계산: |x - n| 계산 시 경계값을 고려하면 (long)x로 승격해 절대값을 취하는 방식이 더 안전하다.
    대안 아이디어
    • 박싱 후 단일 Comparator: Integer[]로 변환해 dx = |x - n|, dy = |y - n| 비교 → 거리 오름차순, 동률 시 값 내림차순.
    • 안정 정렬 2-pass: ① 값 내림차순 정렬 → ② 안정 정렬로 거리 오름차순. 두 번째 정렬에서 동거리 그룹의 기존(값 내림차순) 순서가 유지된다.
    • 버킷 아이디어 (값/거리 범위가 좁을 때): 거리에 대해 버킷을 만들고 각 버킷 내부를 값 내림차순으로 합치는 방식(개념적으로 O(N + K)).

    문제 3 — 유한소수 판별하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    분수 a/b가 유한소수가 되려면, 기약분수로 만들었을 때의 분모가 2 또는 5의 거듭제곱으로만 구성되어야 한다. 즉, 기약분수의 분모에서 2와 5를 모두 제거했을 때 1이 되면 유한소수, 그렇지 않으면 무한소수이다.

    • ① 기약분수화: gcd(a, b)로 최대공약수를 구해 분모를 b /= gcd로 나눈다.
    • ② 2와 5 제거: while (b % 2 == 0) b /= 2;, while (b % 5 == 0) b /= 5;
    • ③ 판정: b == 1이면 유한소수(1), 아니면 무한소수(2).

    예시

    • a = 7, b = 20 → 기약분수 7/20 → 분모 = 2²×5 → 유한소수 → 1
    • a = 12, b = 21 → 기약분수 4/7 → 분모에 7 포함 → 무한소수 → 2
    • a = 11, b = 22 → 기약분수 1/2 → 분모 = 2 → 유한소수 → 1

    시간 복잡도

    최대공약수 계산은 유클리드 알고리즘으로 O(log min(a, b)), 이후 소인수 제거 반복은 O(log b) 수준으로 전체 O(log b).

    공간 복잡도

    변수 몇 개와 재귀 호출 깊이만 사용하므로 O(1).

    주의

    • 기약분수화 누락: gcd로 약분하지 않으면 잘못된 판정이 나온다.
    • 소인수 제거 순서는 중요하지 않으나, while로 반복적으로 나눠야 한다.
    • 2와 5 외의 소인수가 남는 순간 결과는 무한소수(2).
    대안 아이디어
    • BigInteger 활용 — 큰 수 처리 가능
      
      import java.math.BigInteger;
      
      class Solution {
          public int solution(int a, int b) {
              BigInteger A = BigInteger.valueOf(a);
              BigInteger B = BigInteger.valueOf(b);
              BigInteger gcd = A.gcd(B);
              B = B.divide(gcd);
              while (B.mod(BigInteger.TWO).equals(BigInteger.ZERO)) B = B.divide(BigInteger.TWO);
              while (B.mod(BigInteger.valueOf(5)).equals(BigInteger.ZERO)) B = B.divide(BigInteger.valueOf(5));
              return B.equals(BigInteger.ONE) ? 1 : 2;
          }
      }

      정수 오버플로 위험이 없으며, 임의 정밀도에서도 안전하게 판정 가능.

    • 소인수 분해 방식 — 이론적 확장
      
      class Solution {
          public int solution(int a, int b) {
              b /= gcd(a, b);
              for (int p = 2; p * p <= b; p++) {
                  if (b % p == 0 && p != 2 && p != 5) return 2; // 다른 소인수 존재
                  while (b % p == 0) b /= p;
              }
              return (b == 1 || b == 2 || b == 5) ? 1 : 2;
          }
      
          private int gcd(int a, int b) {
              return b == 0 ? a : gcd(b, a % b);
          }
      }

      소인수를 직접 검사하여 동일한 판정을 수행. (O(√b))
