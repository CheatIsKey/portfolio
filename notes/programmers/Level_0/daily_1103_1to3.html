    프로그래머스 Level 0 Daily 문제 묶음

    작성일 2025-11-03 · 분류: 코딩 테스트(프로그래머스), Level 0

    문제 1 — 외계어 사전

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자 배열 spell의 문자들을 모두 한 번씩만 사용해 사전 dic의 어떤 단어를 만들 수 있는지 판별하는 문제다. 제시한 풀이는 문자 멀티셋 동치를 정렬로 판정한다. 즉, spell의 문자들을 정렬한 문자열과, 길이가 같은 각 후보 단어의 문자들을 정렬한 문자열이 equals이면 성립한다.

    • 전처리: spell의 원소는 한 글자 문자열이므로 charAt(0)으로 문자 배열로 모은 후 정렬 → 기준 문자열 s 생성
    • 필터링: 후보 단어 word의 길이가 spell.length와 다르면 건너뜀 (문자 개수 불일치)
    • 동치 판정: word를 문자 배열로 바꿔 정렬하고 s와 비교 → 같으면 1 반환
    • 없으면: 모든 단어가 불일치 → 2 반환

    핵심 아이디어

    • 정렬 = 멀티셋 비교 — 같은 문자들을 같은 개수만큼 가지면 정렬 결과가 동일해진다.
    • 길이 체크 — 개수 조건을 빠르게 배제하여 불필요한 정렬을 줄인다.

    예시

    • spell = ["p","o","s"], dic = ["sod","eocd","qixm","adio","soo"]
      "sod" 정렬 ⇒ "dos", spell 정렬 ⇒ "ops" (불일치)… 모든 단어 불일치 → 결과: 2
    • spell = ["z","d","x"], dic = ["def","dww","dzx","loveaw"]
      "dzx" 정렬 ⇒ "dxz", spell 정렬 ⇒ "dxz" (일치) → 결과: 1

    시간 복잡도

    spell 정렬: O(k log k) (k = spell.length) — 한 번.
    각 단어에 대해 길이 체크 후 정렬 비교: 최악 O(n · k log k) (n = dic.length).
    일반적인 입력 크기에서 충분히 빠르다.

    공간 복잡도

    spell 문자 배열과 각 단어 정렬용 임시 배열로 O(k) (단어 처리 시 O(k) 추가).

    주의

    • 중복 문자: 집합(Set) 비교는 개수를 무시하므로 부적절. 정렬 또는 빈도 비교가 필요하다.
    • 입력 가정: 문제에선 spell의 각 원소가 한 글자 문자열이다. 제시 코드의 charAt(0)은 이 가정을 전제로 한다.
    • 길이 불일치 조기 스킵: 불필요한 정렬을 줄여 성능을 개선한다.
    대안 아이디어
    • 빈도 배열(알파벳 26가정) 비교 — 정렬 없이 O(k + |word|)
      
      import java.util.*;
      
      class Solution {
          public int solution(String[] spell, String[] dic) {
              int[] target = new int[26];
              for (String s : spell) target[s.charAt(0) - 'a']++;
      
              for (String w : dic) {
                  if (w.length() != spell.length) continue;
                  if (matches(target, w)) return 1;
              }
              return 2;
          }
      
          private boolean matches(int[] target, String w) {
              int[] cnt = new int[26];
              for (int i = 0; i < w.length(); i++) {
                  int idx = w.charAt(i) - 'a';
                  if (++cnt[idx] > target[idx]) return false; // 조기 배제
              }
              return Arrays.equals(cnt, target);
          }
      }

      정렬 비용 없이 문자 개수만 맞추므로 상수 계수가 작다.

    • 정렬 비교 (제시 풀이 요약) — 구현 간단, 직관적
      
      import java.util.*;
      
      class Solution {
          public int solution(String[] spell, String[] dic) {
              char[] chars = new char[spell.length];
              for (int i = 0; i < spell.length; i++) chars[i] = spell[i].charAt(0);
              Arrays.sort(chars);
              String s = new String(chars);
      
              for (String word : dic) {
                  if (word.length() != spell.length) continue;
                  char[] arr = word.toCharArray();
                  Arrays.sort(arr);
                  if (s.equals(new String(arr))) return 1;
              }
              return 2;
          }
      }

      멀티셋 일치 여부를 정렬 결과 비교로 판정한다.

    문제 2 — 안전지대

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 두 변 sides로 삼각형을 만들 수 있을 때, 나머지 한 변이 될 수 있는 정수의 개수를 구하는 문제다. 조건은 삼각형의 성립 조건인 두 변의 합 > 나머지 한 변을 활용한다. 제시한 풀이는 sides의 두 변을 정렬해 작은 변을 a, 큰 변을 b로 두고, 가능한 세 번째 변의 개수를 수학적으로 단순화해 2 * a - 1로 계산한다.

    • 정렬: sides를 오름차순 정렬 → a = sides[0], b = sides[1]
    • 조건 분리:
      • 세 번째 변 x가 가장 긴 변일 경우: x < a + b → 가능한 x는 b보다 크고 a + b 미만
      • b가 가장 긴 변일 경우: b < a + x → 가능한 x는 a + b보다 크거나 같고 b 이하
    • 개수 합산: 두 경우의 가능한 정수 x의 개수 합 = (a + b - 1 - b) + (b - (b - a + 1) + 1) = 2a - 1

    예시

    • sides = [1, 2]
      가능한 세 번째 변 x: 2가 가장 긴 변 → x > 1 ⇒ x = 2 가능
      x가 가장 긴 변 → x < 3, x > 2 불가능
      결과: 1개 (수식으로도 2*1 - 1 = 1)
    • sides = [3, 6]
      가능한 x: 4, 5, 6, 7, 8 → 5개
      수식으로도 2*3 - 1 = 5

    시간 복잡도

    정렬: O(1) (배열 길이 2 고정). 수식 계산만 수행하므로 전체 O(1).

    공간 복잡도

    추가 변수 a, b 두 개만 사용하므로 O(1).

    주의

    • 세 번째 변은 항상 정수 — 문제 조건상 가능한 변의 정수 개수만 센다.
    • 음수/0 처리 — 입력으로 주어지는 두 변은 항상 양수이므로 추가 검증 불필요.
    • 정렬 필수 — sides[0], sides[1] 순서가 보장되지 않기 때문에 반드시 정렬해야 한다.
    대안 아이디어
    • 직접 탐색형 구현 — 수식을 도출하지 않고 범위를 직접 세는 방식
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] sides) {
              Arrays.sort(sides);
              int a = sides[0], b = sides[1];
              int count = 0;
              for (int x = b - a + 1; x < a + b; x++) count++;
              return count;
          }
      }

      직관적이지만, 수식 2*a - 1로 단순화 가능해 효율적이지는 않다.

    • 수식형 (제시 풀이 요약) — 정렬 후 즉시 계산
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] sides) {
              Arrays.sort(sides);
              int a = sides[0];
              int b = sides[1];
              return 2 * a - 1;
          }
      }

      삼각형 부등식을 단순화해 상수 시간에 답을 구한다.

    문제 3 — 안전지대

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    지뢰(값 1)가 설치된 칸과 그 인접 8방향(대각 포함) 칸들을 모두 위험 지역으로 표시하고, 위험하지 않은(false) 칸의 개수를 구하는 문제다. 제시한 풀이는 9개 방향(자기 자신 포함)의 델타 배열을 사용해 지뢰가 있는 위치에서 주변을 순회하며 danger 보조 배열을 true로 마킹한 뒤, false의 개수를 세어 반환한다.

    • 델타 준비: row, column에 9개 방향(상·하·좌·우·대각선 + 자기 자신)을 정의
    • 마킹: 각 지뢰 칸 (i, j)에 대해 9방향 (i + dr[k], j + dc[k])가 격자 밖인지 검사 후 danger를 true로 설정
    • 카운트: 모든 칸을 순회하여 !danger[i][j]인 칸의 개수 합산

    예시

    • board = [[0,0,0],[0,1,0],[0,0,0]]
      중앙이 지뢰 → 3×3 전부 위험(true) → 안전 칸 개수: 0
    • board = [[1,0,0],[0,0,0],[0,0,0]]
      좌상단 지뢰 → 영향을 받는 칸: (0,0),(0,1),(1,0),(1,1) → 안전 칸 개수: 9 - 4 = 5

    시간 복잡도

    격자 크기를 H × W, 지뢰 수를 M라 할 때, 각 지뢰마다 최대 9칸을 마킹하므로 O(M)(상수 9 배). 이후 전체 순회로 O(HW). 전체는 O(HW).

    공간 복잡도

    보조 배열 danger를 사용하므로 O(HW).

    주의

    • 경계 검사: 인접 칸이 격자 밖인지 반드시 체크(0 ≤ r < H, 0 ≤ c < W).
    • 중복 마킹 허용: 같은 칸을 여러 번 true로 설정해도 부작용 없음(불리언 OR 효과).
    • 자기 자신 포함: 델타 배열에 (0,0)이 포함되어 있어 지뢰 칸도 위험 처리된다. 바로 아래에서 한 번 더 danger[i][j] = true를 수행하는 부분은 중복이지만 결과엔 영향이 없다(미세 최적화 여지).
    대안 아이디어
    • 8방향만 사용(자기 자신 제외) — 중복 할당 제거, 의도 명확화
      
      import java.util.*;
      
      class Solution {
          private static final int[] DR = {-1,-1,-1, 0, 0, 1, 1, 1};
          private static final int[] DC = {-1, 0, 1,-1, 1,-1, 0, 1};
      
          public int solution(int[][] board) {
              int H = board.length, W = board[0].length;
              boolean[][] danger = new boolean[H][W];
      
              for (int i = 0; i < H; i++) {
                  for (int j = 0; j < W; j++) {
                      if (board[i][j] == 1) {
                          danger[i][j] = true; // 자기 자신
                          for (int k = 0; k < 8; k++) {
                              int nr = i + DR[k], nc = j + DC[k];
                              if (0 <= nr && nr < H && 0 <= nc && nc < W) {
                                  danger[nr][nc] = true;
                              }
                          }
                      }
                  }
              }
      
              int cnt = 0;
              for (int r = 0; r < H; r++) {
                  for (int c = 0; c < W; c++) {
                      if (!danger[r][c]) cnt++;
                  }
              }
              return cnt;
          }
      }

      의미가 분리되어 가독성이 좋고, 불필요한 중복 연산이 없다.

    • 인접 지뢰 수 누적형 — 위험도(개수)가 필요할 때 확장 가능
      지뢰 발견 시 인접 8칸의 카운트를 증가시키고, > 0인 칸을 위험으로 간주하는 방식. 본 문제는 불리언만 필요하므로 현재 풀이가 더 단순하다.
