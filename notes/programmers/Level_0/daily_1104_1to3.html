    문제 1 — 숨어있는 숫자의 덧셈 (2)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string 속에 섞여 있는 숫자들의 연속된 자리수를 하나의 수로 읽어 합산하는 문제다. 제시한 풀이는 한 글자씩 순회하며 숫자('0'~'9')면 누적 값 cur에 자릿수를 곱해 붙이고, 숫자가 아닌 문자를 만나면 지금까지 만든 수(cur)를 sum에 더한 뒤 cur을 0으로 초기화한다. 순회가 끝난 뒤 남아 있을 수 있는 마지막 수를 한 번 더 더해 결과를 반환한다.

    • 누적 구성: 숫자 c를 만나면 cur = cur * 10 + (c - '0')
    • 경계 처리: 숫자가 아닌 문자를 만나면 sum += cur 후 cur = 0
    • 마무리: 루프 종료 후 sum += cur로 끝에 숫자로 끝나는 경우 반영

    예시

    • my_string = "aAb1B2cC34oOp"
      읽은 수: 1, 2, 34 → 합: 37
    • my_string = "1a2b3c"
      읽은 수: 1, 2, 3 → 합: 6
    • my_string = "abc007def90"
      읽은 수: 7, 90 (선행 0 무시) → 합: 97

    시간 복잡도

    문자열을 한 번 순회하므로 O(n) (n = my_string.length()).

    공간 복잡도

    상수 크기 보조 변수(sum, cur)만 사용하므로 O(1).

    주의

    • 마지막 숫자 처리: 루프 종료 후 sum += cur을 잊지 말아야 끝이 숫자인 경우를 반영한다.
    • 숫자 범위: 현재 코드는 '0'~'9'만 숫자로 본다. 문제 조건과 일치한다.
    대안 아이디어
    • 정규식으로 숫자 블록 추출 — 구현 간결
      
      import java.util.regex.*;
      import java.util.*;
      
      class Solution {
          public int solution(String s) {
              Matcher m = Pattern.compile("\\\\d+").matcher(s);
              int sum = 0;
              while (m.find()) {
                  sum += Integer.parseInt(m.group());
              }
              return sum;
          }
      }

      숫자 덩어리(\\d+)만 골라 합산. 가독성이 좋다.

    • 스트림 + 정규식 — 짧게 표현
      
      import java.util.Arrays;
      
      class Solution {
          public int solution(String s) {
              return Arrays.stream(s.split("[^0-9]+"))
                           .filter(t -> !t.isEmpty())
                           .mapToInt(Integer::parseInt)
                           .sum();
          }
      }

      비숫자 기준으로 분할 후 숫자 토큰만 합산.

    • 원본 방식(제시 풀이 요약) — 상태 변수로 직접 파싱
      
      import java.util.*;
      
      class Solution {
          public int solution(String my_string) {
              int sum = 0, cur = 0;
              for (int i = 0; i < my_string.length(); i++) {
                  char c = my_string.charAt(i);
                  if ('0' <= c && c <= '9') cur = cur * 10 + (c - '0');
                  else { sum += cur; cur = 0; }
              }
              return sum + cur;
          }
      }

      추가 라이브러리 없이 빠르고 메모리 사용이 적다.

    문제 2 — 다항식 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 형태의 다항식 my_string이 주어졌을 때, 동류항을 계산하여 계수를 정리한 다항식의 형태로 반환하는 문제다. 제시한 코드는 다항식의 항별 계수 합산을 수행하지 않고, 주어진 문자열에서 숫자를 직접 추출해 합산하는 방식으로 구현되어 있다. 즉, 이 코드는 문제의 본래 요구사항(동류항 정리)에는 맞지 않으며, 단순히 문자열 속 숫자들의 합을 구하는 형태로 작성되어 있다.

    • 숫자 추출: 각 문자 c가 '0'~'9'이면 cur = cur * 10 + (c - '0')으로 숫자를 누적
    • 경계 처리: 숫자가 아닌 문자를 만나면 지금까지 누적한 cur을 sum에 더하고 초기화
    • 마무리: 반복이 끝난 후 마지막 숫자 cur을 sum에 더해 반환

    예시 (현재 코드 기준)

    • my_string = "3x + 7 + x"
      코드 동작 결과 → 숫자 3, 7 합산 → 10 반환
      ※ 실제 정답(다항식 정리)은 "4x + 7"이어야 함
    • my_string = "x + x + 5"
      코드 결과 → 5 반환
      실제 정답은 "2x + 5"

    시간 복잡도

    문자열을 한 번 순회하므로 O(n) (n = my_string.length()).

    공간 복잡도

    상수 크기의 변수 sum, cur만 사용하므로 O(1).

    주의

    • 문제 요구사항 불일치: 실제 문제는 다항식의 x항과 상수항을 분리해 정리해야 한다.
    • 현재 코드는 단순히 숫자만 추출하여 더하는 로직으로, 문제의 의도에 맞지 않는다.
    • 정확한 풀이를 위해선 "x"가 포함된 항과 숫자만 있는 항을 구분해 계수를 누적해야 한다.
    대안 아이디어 (정답 풀이)
    • 동류항 정리 방식 — x항과 상수항을 구분해 누적
      
      import java.util.*;
      
      class Solution {
          public String solution(String polynomial) {
              String[] tokens = polynomial.split(" \\+ ");
              int xSum = 0, numSum = 0;
      
              for (String token : tokens) {
                  if (token.contains("x")) {
                      String coef = token.replace("x", "");
                      xSum += coef.equals("") ? 1 : Integer.parseInt(coef);
                  } else {
                      numSum += Integer.parseInt(token);
                  }
              }
      
              if (xSum == 0) return String.valueOf(numSum);
              if (numSum == 0) return xSum == 1 ? "x" : xSum + "x";
              return (xSum == 1 ? "x" : xSum + "x") + " + " + numSum;
          }
      }

      문자열을 “+” 기준으로 나누고, x항과 상수항을 따로 합산해 포맷팅한다.

    • 정규식 활용 — 항 분리 간결화 가능
      
      import java.util.regex.*;
      class Solution {
          public String solution(String s) {
              Matcher m = Pattern.compile("([0-9]*x|[0-9]+)").matcher(s);
              int xSum = 0, numSum = 0;
              while (m.find()) {
                  String t = m.group();
                  if (t.contains("x"))
                      xSum += t.equals("x") ? 1 : Integer.parseInt(t.replace("x",""));
                  else
                      numSum += Integer.parseInt(t);
              }
              if (xSum == 0) return String.valueOf(numSum);
              if (numSum == 0) return xSum == 1 ? "x" : xSum + "x";
              return (xSum == 1 ? "x" : xSum + "x") + " + " + numSum;
          }
      }

      정규식을 이용해 “x항”과 “숫자항”을 직접 추출해도 깔끔하게 구현할 수 있다.

    문제 3 — 최댓값 만들기 (2)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 numbers가 주어졌을 때, 임의의 두 수를 곱했을 때의 최댓값을 구하는 문제다. 음수 곱의 경우도 고려해야 하므로, 단순히 큰 두 수의 곱만이 아니라 가장 작은 두 수의 곱도 후보에 포함해야 한다. 제시한 풀이는 배열을 정렬한 뒤, 양 끝단 두 가지 경우를 비교하여 정답을 도출한다.

    • 정렬: Arrays.sort(numbers)로 오름차순 정렬
    • 후보 계산:
      • max1 = numbers[0] * numbers[1] — 가장 작은 두 수(음수일 수 있음)
      • max2 = numbers[n-1] * numbers[n-2] — 가장 큰 두 수
    • 비교 반환: Math.max(max1, max2)

    예시

    • numbers = [1, 2, -3, 4, -5]
      정렬 → [-5, -3, 1, 2, 4]
      max1 = (-5)*(-3) = 15, max2 = 4*2 = 8 → 결과: 15
    • numbers = [0, -31, 24, 10, 1, 9]
      정렬 → [-31, 0, 1, 9, 10, 24]
      max1 = (-31)*0 = 0, max2 = 24*10 = 240 → 결과: 240
    • numbers = [-1, -2, -3, -4]
      정렬 → [-4, -3, -2, -1]
      max1 = (-4)*(-3) = 12, max2 = (-1)*(-2) = 2 → 결과: 12

    시간 복잡도

    정렬에 O(n log n).
    이후 상수 연산만 수행하므로 전체 시간 복잡도는 O(n log n).

    공간 복잡도

    입력 배열 외 추가 메모리 사용 없음. O(1).

    주의

    • 음수 고려 필수: 두 음수의 곱이 양수가 되어 최댓값이 될 수 있다.
    • 정렬 후 인덱스 접근: 배열 길이가 2 이상임이 보장되므로 인덱스 오류 없음.
    • 부동소수점 없음: 입력과 결과 모두 정수형 연산으로 충분하다.
    대안 아이디어
    • 정렬 없이 한 번의 순회 — O(n)로 개선
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] numbers) {
              int max1 = Integer.MIN_VALUE, max2 = Integer.MIN_VALUE;
              int min1 = Integer.MAX_VALUE, min2 = Integer.MAX_VALUE;
      
              for (int n : numbers) {
                  if (n > max1) { max2 = max1; max1 = n; }
                  else if (n > max2) { max2 = n; }
      
                  if (n < min1) { min2 = min1; min1 = n; }
                  else if (n < min2) { min2 = n; }
              }
      
              return Math.max(max1 * max2, min1 * min2);
          }
      }

      정렬 없이 최대/최소 두 쌍만 찾는 방식으로 O(n) 시간에 해결할 수 있다.

    • 정렬 방식 (제시 풀이 요약) — 구현 단순
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] numbers) {
              Arrays.sort(numbers);
              int n = numbers.length;
              return Math.max(numbers[0] * numbers[1],
                              numbers[n - 1] * numbers[n - 2]);
          }
      }

      간결하며, 음수·양수 조합 모두 자동 처리된다.
