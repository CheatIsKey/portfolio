    문제 1 — 겹치는 선분의 길이

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    세 개의 선분이 1차원 좌표축 위에 주어질 때, 둘 이상의 선분이 겹쳐 덮는 전체 길이를 구하는 문제다. 제시한 코드는 좌표 범위가 [-100, 100]인 점을 이용해 크기 201의 배열을 준비하고, 각 선분의 구간 [s, e)에 대해 단위 구간(정수 좌표 사이 칸)에 덮임 횟수를 누적한다. 마지막에 누적값이 ≥ 2인 칸의 개수를 합하면 답(겹치는 총 길이)이 된다.

    • 정규화: 각 선분 line = [a, b]에 대해 s = min(a, b), e = max(a, b)
    • 오프셋: 음수 좌표 처리를 위해 offset = 100을 더해 인덱싱
    • 누적: 모든 정수 i에 대해 i = s부터 i < e까지 arr[i + offset]++
      ※ [s, e)의 반열림 구간으로 처리해야 끝점 중복 없이 길이를 정확히 센다.
    • 집계: 배열에서 값이 ≥ 2인 칸의 수를 세어 반환

    예시

    • lines = [[0, 2], [1, 5], [4, 6]]
      덮임(단위 칸 기준): [1,2]에서 2겹, [4,5]에서 2겹 → 결과: 2
    • lines = [[-1, 1], [1, 3], [2, 4]]
      [2,3]에서만 2겹 → 결과: 1

    시간 복잡도

    각 선분 길이(e - s)만큼 배열을 증가시키므로 총합은 좌표 범위 내에서 O(범위). 본 문제의 범위([-100, 100])에선 상수에 가깝다.

    공간 복잡도

    고정 크기 배열 201개 원소만 사용하므로 O(범위) (= 상수).

    주의

    • 경계 처리: 반드시 i < e로 반복해 [s, e)를 사용해야 한다. i <= e로 두면 길이가 1 더해져 오답.
    • 오프셋 가정: offset = 100은 좌표가 [-100,100]임을 전제로 한다. 입력 범위가 바뀌면 배열 크기/오프셋을 조정하거나 다른 기법을 써야 한다.
    대안 아이디어
    • 차분 배열(누적합)로 동일 로직 정리 — 구현을 더 깔끔하게
      
      import java.util.*;
      
      class Solution {
          public int solution(int[][] lines) {
              int[] diff = new int[202]; // 인접 칸 수(201)보다 1 크게
              int offset = 100;
      
              for (int[] line : lines) {
                  int s = Math.min(line[0], line[1]) + offset;
                  int e = Math.max(line[0], line[1]) + offset;
                  // [s, e) 구간에 +1
                  diff[s]++; 
                  diff[e]--;
              }
      
              int ans = 0, cur = 0;
              for (int i = 0; i < 201; i++) { // 각 단위 칸에 대한 누적 덮임 수
                  cur += diff[i];
                  if (cur >= 2) ans++;
              }
              return ans;
          }
      }

      동일한 아이디어를 누적합으로 표현. 가독성이 좋아지고, 경계([s,e)) 표현이 명확해진다. 시간/공간 복잡도 동일.

    • 맵 기반 스위핑 — 좌표 범위가 넓을 때 유용
      오프셋/고정 배열 대신 TreeMap<Integer,Integer>로 이벤트(s:+1, e:-1)를 기록하고 키 오름차순으로 누적값을 갱신하며 겹침 ≥ 2인 구간 길이를 합산한다. 범위가 매우 클 때도 메모리 효율적이다.

    문제 2 — 평행

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    네 점이 주어질 때, 두 선분이 서로 평행한 경우가 있는지를 판별하는 문제다. 제시한 코드는 가능한 세 가지 짝짓기 (0–1, 2–3), (0–2, 1–3), (0–3, 1–2)에 대해 두 선분의 방향벡터를 구하고, 두 벡터의 외적(2D 크로스 곱) x1*y2 - y1*x2가 0인지로 평행 여부를 판단한다. 외적이 0이면 두 벡터가 같은 방향(또는 정반대 방향)이라 평행이다.

    • 벡터 구성: (x1, y1) = dot1 - dot, (x2, y2) = dot3 - dot2
    • 평행 판정: x1 * y2 - y1 * x2 == 0이면 평행
    • 정수 오버플로 방지: 곱 연산을 위해 long으로 형변환해 안전하게 계산
    • 세 경우 중 하나라도 평행이면 1 반환, 아니면 0

    예시

    • dots = [[0,0],[3,3],[1,0],[4,3]]
      선분 (0,0)-(3,3)와 (1,0)-(4,3)의 방향벡터가 모두 (1,1) 계열 → 평행 → 결과: 1
    • dots = [[0,0],[1,2],[2,1],[3,3]]
      가능한 어떤 짝에서도 외적이 0이 아님 → 평행 없음 → 결과: 0

    시간 복잡도

    고정된 세 경우만 검사하므로 O(1).

    공간 복잡도

    추가 메모리 거의 없음 → O(1).

    주의

    • 기울기 비교 대신 외적: 기울기(y/x)로 비교하면 x=0(수직선) 처리와 0으로 나누는 문제가 생긴다. 외적 0 판정은 이를 깔끔히 해결한다.
    • 오버플로: 좌표 범위가 크면 x1*y2 등이 int에서 넘칠 수 있다. 제시 코드처럼 long 캐스팅이 안전하다.
    • 퇴화 선분(길이 0): 한 선분의 양 끝점이 같으면 외적이 0이 되어 형식상 평행으로 판정될 수 있다. 문제에서 이런 입력이 있는지는 확실하지 않음. 필요하다면 벡터가 (0,0)인지 먼저 확인해 예외 처리하라.
    대안 아이디어
    • 조합 루프로 일반화 — 하드코딩 대신 모든 짝 자동 탐색
      
      import java.util.*;
      
      class Solution {
          public int solution(int[][] dots) {
              int[][] pair = {{0,1,2,3}, {0,2,1,3}, {0,3,1,2}};
              for (int[] p : pair) {
                  if (parallel(dots[p[0]], dots[p[1]], dots[p[2]], dots[p[3]])) return 1;
              }
              return 0;
          }
      
          private boolean parallel(int[] a, int[] b, int[] c, int[] d) {
              long x1 = b[0] - a[0], y1 = b[1] - a[1];
              long x2 = d[0] - c[0], y2 = d[1] - c[1];
              return x1 * y2 - y1 * x2 == 0;
          }
      }

      논리는 동일하되, 조합을 표로 모아 가독성을 높인다.

    • 기울기를 기약분수로 비교 — 수학적 명료성(단, 구현은 더 김)
      
      import java.util.*;
      
      class Solution {
          public int solution(int[][] dots) {
              int[][] pair = {{0,1,2,3}, {0,2,1,3}, {0,3,1,2}};
              for (int[] p : pair) {
                  if (sameSlope(dots[p[0]], dots[p[1]], dots[p[2]], dots[p[3]])) return 1;
              }
              return 0;
          }
      
          private boolean sameSlope(int[] a, int[] b, int[] c, int[] d) {
              int dx1 = b[0] - a[0], dy1 = b[1] - a[1];
              int dx2 = d[0] - c[0], dy2 = d[1] - c[1];
              if (dx1 == 0 && dx2 == 0) return true;            // 둘 다 수직선
              if (dx1 == 0 || dx2 == 0) return false;           // 하나만 수직선
              int g1 = gcd(Math.abs(dx1), Math.abs(dy1));
              int g2 = gcd(Math.abs(dx2), Math.abs(dy2));
              dx1 /= g1; dy1 /= g1; dx2 /= g2; dy2 /= g2;
              // 부호 정규화
              if (dx1 < 0) { dx1 = -dx1; dy1 = -dy1; }
              if (dx2 < 0) { dx2 = -dx2; dy2 = -dy2; }
              return dx1 == dx2 && dy1 == dy2;
          }
      
          private int gcd(int a, int b) { return b == 0 ? a : gcd(b, a % b); }
      }

      외적법과 결과는 같지만, 기울기를 기약분수로 맞춰 비교한다. 연산량/코드는 더 복잡하다.

    문제 3 — 저주의 숫자 3

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    3의 배수이거나 숫자에 3이 포함된 수를 모두 건너뛰며 1부터 차례로 세었을 때, n번째로 해당하는 수를 구하는 문제다. 제시한 코드는 1씩 증가시키면서 건너뛸 조건을 while문으로 처리한다.

    • 초기화: answer = 0
    • 반복: i = 0부터 n - 1까지
      • answer++로 다음 수 후보를 만든다.
      • while (answer % 3 == 0 || String.valueOf(answer).contains("3")) → 3의 배수거나 숫자 3을 포함하면 계속 answer++로 건너뜀
    • 모든 반복 후 answer 반환 → 이것이 ‘3을 건너뛰며 센’ n번째 수

    예시

    • n = 1 → 1 (건너뛸 수 없음)
    • n = 5 → 순서대로 1, 2, 4, 5, 7 → 결과: 7
    • n = 10 → 1, 2, 4, 5, 7, 8, 10, 11, 14, 16 → 결과: 16

    시간 복잡도

    각 수를 검사하며, 건너뛰는 횟수가 많을수록 반복이 늘어나지만 일반적으로 O(n)에 근사한다. (조건문 내부 String.contains 변환 포함)

    공간 복잡도

    단일 정수 변수만 사용하므로 O(1).

    주의

    • 3의 배수와 ‘3’ 포함 수는 모두 건너뛴다. 두 조건을 ||로 연결해야 하며, &&로 바꾸면 오답.
    • 문자열 변환 비용: String.valueOf(answer)는 매 반복마다 문자열 생성. 입력 n이 매우 크면 약간의 성능 손실이 있을 수 있으나, 본 문제 범위에서는 무시 가능.
    • while 조건 순서: 3의 배수 또는 ‘3’ 포함이 한 번이라도 참이면 다시 검사해야 하므로 while로 처리하는 것이 정확하다 (if로는 연속 건너뛰기 불가능).
    대안 아이디어
    • 미리 ‘3 없는 수열’ 생성 — 반복 조건 제거
      
      import java.util.*;
      
      class Solution {
          public int solution(int n) {
              int num = 0, count = 0;
              while (count < n) {
                  num++;
                  if (num % 3 != 0 && !String.valueOf(num).contains("3")) {
                      count++;
                  }
              }
              return num;
          }
      }

      조건을 만족하는 수만 셀 때마다 카운트를 올리는 방식. 논리는 동일하지만 가독성이 조금 더 높다.

    • 문자열 검사 없이 3진법 변환 이용 — 확장 아이디어
      ‘3 없는 수’를 3진법으로 변환해 3이 등장하지 않도록 만드는 방식(예: ‘369게임’의 역수열). 하지만 이 문제에서는 단순 시뮬레이션이 더 효율적이므로 굳이 필요하지 않다.
