    문제 1 — 캐릭터의 좌표

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    방향 키 입력(keyinput)에 따라 캐릭터의 좌표를 이동시키되, 보드(board = [width, height]) 경계를 넘지 않도록 제한하는 문제다. 폭/높이의 절반을 정수 나눗셈으로 구해 허용 좌표 범위를 x: [-maxW, maxW], y: [-maxH, maxH]로 두고, 시작 좌표 (0, 0)에서 키를 하나씩 처리하며 범위 안에서만 이동한다.

    • 경계 설정: maxW = board[0] / 2, maxH = board[1] / 2 → 허용 범위는 [-maxW, maxW], [-maxH, maxH]
    • 이동 처리: "up", "down", "left", "right"에 대해 다음 좌표가 범위를 넘지 않을 때만 반영
    • 초기값/반환: 시작은 (0, 0), 모든 입력을 처리한 뒤 최종 좌표 반환

    예시

    • keyinput = ["left","left","left","right"], board = [3,3]
      허용 범위 x: [-1,1] → 최종 (-1, 0)
    • keyinput = ["up","up","up"], board = [5,5]
      허용 범위 y: [-2,2] → 최종 (0, 2)
    • keyinput = ["up","left","down","right"], board = [7,3]
      왕복 이동 → 최종 (0, 0)

    시간 복잡도

    입력 배열을 한 번 순회하므로 O(n) (n = keyinput.length).

    공간 복잡도

    상수 크기 보조 변수만 사용하므로 O(1).

    주의

    • 경계 포함: 좌표는 -maxW와 maxW (또는 -maxH, maxH)를 포함한다.
    • 정수 나눗셈: board[i] / 2는 내림으로 동작한다. 보드 크기가 짝수여도 이 방식은 정상적으로 경계를 형성한다.
    • 예외 키: 지정된 네 키 외 문자열은 무시된다(현재 구현 기준).
    • 문제 제약: 보드 길이가 항상 홀수인지 여부는 확실하지 않음. (풀이 자체에는 영향 없음)
    대안 아이디어
    • 클램프(clamp) 방식 — 보정으로 경계 유지
      
      import java.util.*;
      
      class Solution {
          public int[] solution(String[] keyinput, int[] board) {
              int maxW = board[0] / 2, maxH = board[1] / 2;
              int x = 0, y = 0;
              for (String k : keyinput) {
                  int dx = 0, dy = 0;
                  switch (k) {
                      case "up":    dy = 1;  break;
                      case "down":  dy = -1; break;
                      case "left":  dx = -1; break;
                      case "right": dx = 1;  break;
                      default: /* ignore */ 
                  }
                  x = Math.max(-maxW, Math.min(maxW, x + dx));
                  y = Math.max(-maxH, Math.min(maxH, y + dy));
              }
              return new int[]{x, y};
          }
      }

      다음 좌표를 먼저 이동시킨 뒤 Math.max/Math.min으로 경계 안으로 끌어당긴다.

    • 맵으로 방향 벡터 관리 — 키-방향 분리로 가독성 향상
      
      import java.util.*;
      
      class Solution {
          public int[] solution(String[] keyinput, int[] board) {
              int maxW = board[0] / 2, maxH = board[1] / 2;
              Map<String, int[]> dir = Map.of(
                  "up",    new int[]{0, 1},
                  "down",  new int[]{0, -1},
                  "left",  new int[]{-1, 0},
                  "right", new int[]{1, 0}
              );
              int x = 0, y = 0;
              for (String k : keyinput) {
                  int[] d = dir.get(k);
                  if (d == null) continue; // 알 수 없는 키 무시
                  int nx = x + d[0], ny = y + d[1];
                  if (Math.abs(nx) <= maxW) x = nx;
                  if (Math.abs(ny) <= maxH) y = ny;
              }
              return new int[]{x, y};
          }
      }

      방향 정의를 자료구조로 분리해 스위치 분기를 줄인다.

    문제 2 — 직사각형 넓이 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    네 개의 점(dots)이 주어졌을 때 이 점들이 이루는 직사각형의 가로 길이와 세로 길이를 계산해 넓이를 구하는 문제다. 먼저 모든 점의 x좌표 가운데 최소값 minX와 최대값 maxX를 구하고, y좌표 가운데 최소값 minY와 최대값 maxY를 구한다. 그 뒤 가로 길이 (maxX − minX)와 세로 길이 (maxY − minY)를 곱해 넓이를 반환한다.

    • 최소/최대 찾기: 네 점을 순회하며 minX, maxX, minY, maxY를 갱신한다.
    • 길이 계산: x축 길이는 maxX − minX, y축 길이는 maxY − minY
    • 넓이 반환: 위 두 길이의 곱을 반환한다.

    예시

    • dots = [[1,1],[2,1],[2,2],[1,2]]
      minX = 1, maxX = 2, minY = 1, maxY = 2 → 가로 1, 세로 1 → 넓이 1
    • dots = [[-1,-1],[1,1],[1,-1],[-1,1]]
      minX = -1, maxX = 1, minY = -1, maxY = 1 → 가로 2, 세로 2 → 넓이 4

    시간 복잡도

    네 점(고정된 개수)을 순회하므로 O(1)로 볼 수 있으나, 일반적으로 점 개수가 n이라면 O(n)다.

    공간 복잡도

    보조 변수 minX, maxX, minY, maxY만 사용하므로 O(1)이다.

    주의

    • 점 개수: 문제는 네 개의 점만 주어진다. 일반화된 경우가 아니라는 점을 확인해야 한다.
    • 좌표 정렬 없음: 입력이 정렬되어 있거나 직사각형의 꼭짓점 순서로 주어진다는 보장이 없으므로 모든 점을 검사해야 한다.
    • 음수 좌표: x나 y 좌표가 음수일 수 있으며, 최소값/최대값 계산 방식에 영향을 받지 않는다.
    대안 아이디어
    • 단일 반복 방식 — 구조만 약간 변화:
      
      import java.util.*;
      
      class Solution {
          public int solution(int[][] dots) {
              int minX = Integer.MAX_VALUE, maxX = Integer.MIN_VALUE;
              int minY = Integer.MAX_VALUE, maxY = Integer.MIN_VALUE;
      
              for (int[] dot : dots) {
                  int x = dot[0], y = dot[1];
                  if (x < minX) minX = x;
                  if (x > maxX) maxX = x;
                  if (y < minY) minY = y;
                  if (y > maxY) maxY = y;
              }
      
              return (maxX - minX) * (maxY - minY);
          }
      }

      조건문을 직접 사용해 최소/최대값을 갱신한다.

    • 배열 정렬 활용 방식 — 가독성 위주:
      
      import java.util.*;
      
      class Solution {
          public int solution(int[][] dots) {
              int[] xs = new int[4], ys = new int[4];
              for (int i = 0; i < dots.length; i++) {
                  xs[i] = dots[i][0];
                  ys[i] = dots[i][1];
              }
              Arrays.sort(xs);
              Arrays.sort(ys);
              return (xs[3] - xs[0]) * (ys[3] - ys[0]);
          }
      }

      x좌표 배열과 y좌표 배열을 정렬한 뒤 가장 작은 값과 가장 큰 값의 차이를 사용한다.

    • 람다 및 스트림 방식 (Java 8+) — 최신 문법 활용:
      
      import java.util.*;
      import java.util.stream.*;
      
      class Solution {
          public int solution(int[][] dots) {
              int minX = Arrays.stream(dots).mapToInt(d -> d[0]).min().orElse(0);
              int maxX = Arrays.stream(dots).mapToInt(d -> d[0]).max().orElse(0);
              int minY = Arrays.stream(dots).mapToInt(d -> d[1]).min().orElse(0);
              int maxY = Arrays.stream(dots).mapToInt(d -> d[1]).max().orElse(0);
              return (maxX - minX) * (maxY - minY);
          }
      }

      람다식으로 최소/최대값을 구해 코드 길이를 줄인다.

    문제 3 — 배열 원소의 길이

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 배열 strlist의 각 원소 길이를 구해 정수 배열로 반환하는 문제다. 각 문자열의 length() 값을 구해 동일 인덱스의 정수 배열에 저장하면 된다. 반복문으로 문자열을 하나씩 순회하며 길이를 채워 넣는다.

    • 배열 생성: 결과용 정수 배열 ints를 strlist.length 크기로 생성
    • 반복 처리: 각 문자열 s에 대해 ints[i++] = s.length()
    • 결과 반환: 완성된 ints 배열 반환

    예시

    • strlist = ["We", "are", "the", "world!"]
      길이 배열: [2, 3, 3, 6]
    • strlist = ["I", "Love", "Java"]
      길이 배열: [1, 4, 4]
    • strlist = ["a", "ab", "abc", "abcd"]
      길이 배열: [1, 2, 3, 4]

    시간 복잡도

    문자열 배열을 한 번 순회하므로 O(n) (n = strlist.length).

    공간 복잡도

    결과 배열과 상수 크기의 보조 변수(i)만 사용하므로 O(n).

    주의

    • 문자열 길이: String.length()는 유니코드 문자의 개수를 기준으로 한다.
    • 빈 문자열: 길이가 0인 문자열도 정상적으로 0으로 처리된다.
    • 입력 크기: 배열 크기가 0일 경우 빈 배열을 반환한다.
    대안 아이디어
    • 일반 for문 사용 — 명시적 인덱스 활용
      
      import java.util.*;
      
      class Solution {
          public int[] solution(String[] strlist) {
              int[] res = new int[strlist.length];
              for (int i = 0; i < strlist.length; i++) {
                  res[i] = strlist[i].length();
              }
              return res;
          }
      }

      전통적인 for 문으로 인덱스 제어가 명확하다.

    • 스트림 방식 (Java 8+) — 간결한 표현
      
      import java.util.*;
      import java.util.stream.*;
      
      class Solution {
          public int[] solution(String[] strlist) {
              return Arrays.stream(strlist)
                           .mapToInt(String::length)
                           .toArray();
          }
      }

      함수형 스타일로 짧고 가독성이 높지만, 내부적으로 스트림 생성 비용이 있다.
