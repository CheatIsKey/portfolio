    문제 1 — 문자열 정렬하기 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 my_string에서 숫자 문자만 골라 정수 배열로 변환한 뒤, 오름차순으로 정렬해 반환하는 문제다.
    제시한 풀이처럼 문자 배열을 한 번 순회하며 숫자('0' ~ '9')만 골라 list에 담고, Collections.sort로 정렬한 뒤 int[]로 변환하면 된다.

    • 문자 순회: toCharArray()로 변환 후 각 문자를 확인한다.
    • 숫자 판별: '0' <= ch && ch <= '9' 범위 검사로 숫자만 추출한다.
    • 정수 변환: list.add(ch - '0')로 문자 숫자를 정수로 변환한다.
    • 정렬 & 반환: Collections.sort(list) 수행 후 stream()으로 int[] 생성.

    예시

    • my_string = "hi12392"
      추출: 1,2,3,9,2 → 정렬 ⇒ [1, 2, 2, 3, 9]
    • my_string = "p2o4i8gj2"
      추출: 2,4,8,2 → 정렬 ⇒ [2, 2, 4, 8]
    • my_string = "abcde0"
      추출: 0 → 정렬 ⇒ [0]

    시간 복잡도

    순회는 O(n), 정렬은 추출된 숫자 개수를 k라 할 때 O(k log k). 전체는 O(n + k log k) (최대 k ≈ n이면 O(n log n)).

    공간 복잡도

    숫자만 저장하는 리스트 크기 O(k), 반환 배열도 O(k).

    주의

    • 문자 → 정수: ch - '0'는 ASCII 코드 차이를 이용한 안전한 변환(전제: '0'~'9' 범위 내부일 때).
    • 유니코드 숫자: 문제 조건은 영문/숫자이므로 다른 숫자 범주(예: 전각 숫자)는 고려하지 않아도 된다.
    • 스트림 변환: list.stream().mapToInt(Integer::intValue).toArray()로 박싱 해제 후 배열화.
    대안 아이디어
    • 스트림 기반 간결 풀이 — 필터/매핑/정렬을 한 번에 처리 
      
      import java.util.*;
      import java.util.stream.*;
      
      class Solution {
          public int[] solution(String my_string) {
              return my_string.chars()                      // IntStream of code points
                      .filter(ch -> '0' <= ch && ch <= '9') // 숫자만 통과
                      .map(ch -> ch - '0')                 // 정수로 변환
                      .sorted()                            // 오름차순
                      .toArray();
          }
      }

      가독성이 좋고 중간 저장 없이 바로 정렬 배열을 만든다.

    • 정규식으로 숫자만 잔존 — 숫자 외 문자 제거 후 처리 
      
      import java.util.*;
      
      class Solution {
          public int[] solution(String my_string) {
              String digits = my_string.replaceAll("[^0-9]", "");
              int[] arr = new int[digits.length()];
              for (int i = 0; i < digits.length(); i++) {
                  arr[i] = digits.charAt(i) - '0';
              }
              Arrays.sort(arr);
              return arr;
          }
      }

      전처리로 숫자만 남긴 뒤 배열을 만들어 정렬한다.

    문제 2 — 모음 제거

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 문자열 my_string에서 모음(a, e, i, o, u)을 모두 제거한 뒤 남은 문자들로 새로운 문자열을 만드는 문제다.
    문자 하나씩 순회하면서 모음에 포함되지 않은 경우만 StringBuilder에 추가하면 된다.

    • 모음 정의: 문자열 "aeiou"에 모음 집합을 정의한다.
    • 문자 순회: toCharArray()로 각 문자를 순회한다.
    • 포함 검사: vowels.indexOf(c)가 -1이면 모음이 아님을 의미한다.
    • 문자 추가: 모음이 아닌 문자는 StringBuilder에 누적한다.
    • 결과 반환: sb.toString()으로 문자열을 생성해 반환한다.

    예시

    • my_string = "bus"
      제거: ‘u’ → 결과 "bs"
    • my_string = "nice to meet you"
      제거: i, e, o, e, e, o, u → 결과 "nc t mt y"
    • my_string = "aeiou"
      모든 문자가 모음 → 결과 "" (빈 문자열)

    시간 복잡도

    문자열 길이를 n이라 하면, 각 문자를 한 번씩 확인하므로 O(n).
    indexOf 연산은 상수 시간(최대 5회 비교)이므로 전체적으로 O(n).

    공간 복잡도

    결과 문자열을 저장하는 StringBuilder만 사용하므로 O(n).

    주의

    • 대문자 처리: 문제 조건은 소문자만 포함하지만, 대문자를 고려하려면 Character.toLowerCase(c)로 변환 후 비교 가능.
    • 문자열 누적: String 대신 StringBuilder 사용 시 불필요한 문자열 객체 생성 방지.
    • 특수문자·공백: 모음이 아니므로 그대로 유지된다.
    대안 아이디어
    • 정규식 활용 방식 — 모음 패턴을 한 번에 제거 
      
      class Solution {
          public String solution(String my_string) {
              return my_string.replaceAll("[aeiou]", "");
          }
      }

      정규식 [aeiou]으로 모든 모음을 찾아 공백 문자열로 대체한다. 코드가 매우 간결하다.

    • 스트림 기반 처리 — 함수형 접근 
      
      import java.util.stream.*;
      
      class Solution {
          public String solution(String my_string) {
              return my_string.chars()
                      .filter(c -> "aeiou".indexOf(c) == -1)
                      .mapToObj(c -> String.valueOf((char)c))
                      .collect(Collectors.joining());
          }
      }

      문자 스트림을 필터링해 모음을 제외하고, 다시 문자열로 결합한다.

    문제 3 — 팩토리얼

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 n 이하인 가장 큰 팩토리얼(k!)을 만족하는 정수 k를 구하는 문제다.
    1부터 시작해 k! 값을 순차적으로 계산하면서 n을 초과하기 전까지 곱을 누적하고, 초과 시 직전의 k - 1을 반환한다.

    • 초기값 설정: fact = 1, k = 1로 시작한다.
    • 팩토리얼 계산: fact *= k로 1씩 증가시키며 누적 곱을 구한다.
    • 범위 초과 검사: fact > n이 되는 시점에 루프를 종료한다.
    • 정답 반환: 루프 종료 후 k - 1이 n 이하의 최대 팩토리얼을 만드는 정수다.

    예시

    • n = 3628800
      1! = 1 → 2! = 2 → 3! = 6 → … → 10! = 3628800 → 11!은 초과 ⇒ 결과 10
    • n = 7
      1! = 1 → 2! = 2 → 3! = 6 → 4! = 24(초과) ⇒ 결과 3
    • n = 1
      1! = 1 → 2! = 2(초과) ⇒ 결과 1

    시간 복잡도

    k!이 n을 초과할 때까지 반복하므로 반복 횟수는 k (팩토리얼 성장은 매우 빠르다). 따라서 시간 복잡도는 O(k) (사실상 매우 작다, 예를 들어 n ≤ 10⁶이면 k ≤ 9 정도).

    공간 복잡도

    fact와 k 두 개의 변수만 사용하므로 O(1).

    주의

    • 정수 오버플로: int로 계산하므로 팩토리얼이 매우 커지면 int 범위를 초과할 수 있다. (문제 제한에서 안전함)
    • 반복 종료 시점: fact > n 조건으로 루프를 탈출하므로 k - 1 반환이 필수.
    • 시작 값 주의: fact = 1, k = 1로 초기화해야 1! 계산이 올바르게 수행된다.
    대안 아이디어
    • for 루프 방식 — 종료 조건을 명시적으로 표현 
      
      class Solution {
          public int solution(int n) {
              int fact = 1;
              for (int k = 1; ; k++) {
                  fact *= k;
                  if (fact > n) return k - 1;
              }
          }
      }

      무한 루프 대신 for를 사용해 가독성을 높인 형태.

    • 사전 계산 방식 — 미리 팩토리얼 배열 저장 후 탐색 
      
      class Solution {
          public int solution(int n) {
              int[] factorials = {1, 2, 6, 24, 120, 720, 5040, 40320, 362880, 3628800};
              int k = 0;
              for (int f : factorials) {
                  if (f > n) break;
                  k++;
              }
              return k;
          }
      }

      문제의 입력 범위가 작을 경우 사전 계산으로 더 빠르게 해결 가능하다.
