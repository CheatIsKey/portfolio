    문제 1 — 최댓값 만들기(1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 numbers에서 원소 두 개를 골라 만들 수 있는 곱 중 최댓값을 구하는 문제다.
    이 문제(최댓값 만들기 (1))에서는 배열의 원소들이 양의 정수이므로, 가장 큰 두 수를 곱하는 것이 항상 최댓값이 된다.

    제시한 풀이처럼 Arrays.sort(numbers)로 배열을 오름차순 정렬한 뒤, numbers[numbers.length - 1]과 numbers[numbers.length - 2]를 곱해 반환하면 된다.

    • 정렬: Arrays.sort(numbers)로 오름차순 정렬한다.
    • 최댓값 선택: 정렬 후 가장 큰 값 두 개는 numbers[length - 1], numbers[length - 2]에 위치한다.
    • 곱 반환: 두 값을 곱한 결과를 정답으로 반환한다.

    예시

    • numbers = [1, 2, 3, 4, 5]
      정렬 결과: [1, 2, 3, 4, 5]
      가장 큰 두 수: 5, 4 → 곱: 20
    • numbers = [0, 31, 24, 10, 1, 9]
      정렬 결과: [0, 1, 9, 10, 24, 31]
      가장 큰 두 수: 31, 24 → 곱: 744
    • numbers = [1, 2]
      가장 큰 두 수: 2, 1 → 곱: 2

    시간 복잡도

    Arrays.sort()는 O(n log n) 시간 복잡도를 가진다.
    정렬 이후 최댓값 두 개를 선택하는 연산은 O(1)이므로, 전체 시간 복잡도는 O(n log n)이다.

    공간 복잡도

    주어진 배열을 제자리(in-place)에서 정렬하므로, 입력 배열 외에 추가로 사용하는 공간은 상수 수준이다.
    전체 공간 복잡도는 O(1) (입력 배열 제외 기준)이다.

    주의

    • 문제 조건 확인: 이 문제 버전은 양의 정수 위주 입력을 가정하므로, 정렬 후 가장 큰 두 수의 곱이 최댓값이라는 전략이 항상 성립한다.
    • 다른 버전과 혼동 주의: 음수가 포함되는 확장 버전(예: 최댓값 만들기 (2))에서는 가장 큰 두 수의 곱뿐 아니라 가장 작은 두 수(큰 음수)의 곱도 함께 비교해야 한다. (이 문제에는 해당되지 않음.)
    • 배열 길이: 최소 길이가 2 이상이므로 인덱스 범위 예외는 발생하지 않는다.
    대안 아이디어
    • 한 번의 순회로 최댓값 두 개 찾기 — 정렬 없이 O(n)에 해결 
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] numbers) {
              int max1 = 0;
              int max2 = 0;
      
              for (int n : numbers) {
                  if (n > max1) {
                      max2 = max1;
                      max1 = n;
                  } else if (n > max2) {
                      max2 = n;
                  }
              }
      
              return max1 * max2;
          }
      }

      정렬 비용 없이 상수 공간과 한 번의 순회로 가장 큰 두 수를 추적해 곱을 계산하는 방법이다. (이 문제 조건에서도 올바르게 동작한다.)

    문제 2 — 합성수 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    1 이상 n 이하의 자연수 중에서 합성수(약수가 3개 이상인 수)의 개수를 구하는 문제다.
    즉, 소수(약수 2개)나 1(약수 1개)을 제외한 모든 수를 세면 된다.

    제시한 풀이에서는 각 수 i에 대해 1부터 i까지 순회하며 약수 개수를 세고, 그 개수가 3 이상이면 합성수로 판단하여 카운트한다.

    • 외부 반복: 1부터 n까지 순회한다.
    • 내부 반복: 각 i에 대해 1부터 i까지 나누어떨어지는 수를 세며 약수 개수를 구한다.
    • 합성수 판별: 약수 개수가 3개 이상이면 cnt++ 증가.
    • 결과 반환: 전체 합성수 개수를 반환한다.

    예시

    • n = 10
      합성수: 4(약수 1,2,4), 6(1,2,3,6), 8(1,2,4,8), 9(1,3,9), 10(1,2,5,10)
      ⇒ 총 5개
    • n = 15
      합성수: 4,6,8,9,10,12,14,15 ⇒ 8개
    • n = 3
      합성수 없음 ⇒ 0

    시간 복잡도

    각 수 i에 대해 1부터 i까지 나누어보므로, 전체 반복 횟수는
    1 + 2 + 3 + ... + n = O(n²).
    따라서 전체 시간 복잡도는 O(n²)이다.

    공간 복잡도

    별도의 배열이나 리스트를 사용하지 않고, 변수 cnt와 tmp만 사용하므로 O(1)이다.

    주의

    • 1은 합성수가 아님: 약수가 1개뿐이므로 제외해야 한다.
    • 소수(약수 2개)는 포함하지 않는다. (예: 2, 3, 5, 7, 11 등)
    • 시간 효율: n이 클 경우 이중 반복문은 비효율적이므로 개선이 가능하다.
    대안 아이디어
    • 소수 판별 최적화 — 제곱근까지만 검사하여 효율 개선 
      
      import java.util.*;
      
      class Solution {
          public int solution(int n) {
              int cnt = 0;
              for (int i = 1; i <= n; i++) {
                  if (!isPrime(i) && i != 1) cnt++;
              }
              return cnt;
          }
      
          private boolean isPrime(int num) {
              if (num <= 1) return false;
              for (int i = 2; i * i <= num; i++) {
                  if (num % i == 0) return false;
              }
              return true;
          }
      }

      O(√n)으로 각 수의 소수 여부를 판별해 합성수를 찾는다. 전체 복잡도는 O(n√n)로 개선된다.

    • 배수 마킹 방식 — 에라토스테네스의 체 응용 
      
      import java.util.*;
      
      class Solution {
          public int solution(int n) {
              boolean[] composite = new boolean[n + 1];
              for (int i = 2; i * 2 <= n; i++) {
                  for (int j = i * 2; j <= n; j += i) {
                      composite[j] = true;
                  }
              }
      
              int cnt = 0;
              for (int i = 4; i <= n; i++) if (composite[i]) cnt++;
              return cnt;
          }
      }

      소수의 배수를 미리 표시해 합성수를 한 번에 구할 수 있다. 시간 복잡도는 O(n log log n) 수준으로 효율적이다.

    문제 3 — 주사위의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 상자의 크기 box와 정육면체 주사위 한 변의 길이 n이 주어질 때, 상자 안에 들어갈 수 있는 주사위의 최대 개수를 구하는 문제다.
    상자 안에 주사위를 빈틈없이 채워 넣되, 상자 경계를 넘지 않아야 한다.

    각 변의 길이를 n으로 나눈 몫이 해당 축 방향에 들어갈 수 있는 주사위 개수이므로, x = box[0] / n, y = box[1] / n, z = box[2] / n로 계산한 뒤
    전체 개수는 x * y * z로 구할 수 있다.

    • 입력: box = [가로, 세로, 높이], n = 주사위 한 변의 길이
    • 각 축 방향의 개수: box[i] / n (정수 나눗셈, 몫만 취함)
    • 전체 주사위 개수: 세 방향의 곱 x * y * z

    예시

    • box = [10, 8, 6], n = 3
      각 축 방향 개수: 10/3 = 3, 8/3 = 2, 6/3 = 2
      전체 개수: 3 × 2 × 2 = 12
    • box = [1, 1, 1], n = 1
      각 축 방향 개수: 1/1 = 1
      전체 개수: 1 × 1 × 1 = 1
    • box = [10, 10, 10], n = 5
      각 축 방향 개수: 10/5 = 2
      전체 개수: 2 × 2 × 2 = 8

    시간 복잡도

    단순한 산술 연산 세 번과 곱셈 한 번만 수행하므로,
    전체 시간 복잡도는 O(1)이다.

    공간 복잡도

    상수 개수의 변수(x, y, z)만 사용하므로,
    전체 공간 복잡도는 O(1)이다.

    주의

    • 정수 나눗셈: box[i] / n은 소수점 이하를 버리는 정수 나눗셈이다. (즉, 실제로 들어갈 수 있는 개수만 계산)
    • 0이 될 수 있음: n이 box의 어떤 변보다 크면 그 방향으로는 주사위가 들어가지 않아 결과가 0이 된다.
    • 음수 없음: 입력 조건상 모든 값은 양의 정수이다.
    대안 아이디어
    • 스트림 기반 간결 풀이 — 반복문 없이 곱셈 수행 
      
      import java.util.*;
      
      class Solution {
          public int solution(int[] box, int n) {
              return Arrays.stream(box)
                           .map(v -> v / n)
                           .reduce(1, (a, b) -> a * b);
          }
      }

      각 변을 n으로 나눈 몫을 스트림으로 처리해 간결하게 결과를 얻는 방식이다.
