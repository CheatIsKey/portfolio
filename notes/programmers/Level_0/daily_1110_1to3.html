    문제 1 — 배열 회전시키기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 정수 배열 numbers를 오른쪽 또는 왼쪽으로 한 칸 회전하는 문제다.
    방향은 문자열 direction으로 주어지며, "right"이면 오른쪽으로, "left"이면 왼쪽으로 회전한다.

    오른쪽 회전의 경우 마지막 원소를 맨 앞으로 옮기고 나머지를 한 칸씩 뒤로 밀어내며,
    왼쪽 회전의 경우 첫 번째 원소를 맨 뒤로 보내고 나머지를 한 칸씩 앞으로 당기면 된다.

    • 오른쪽 회전: result[0] = numbers[numbers.length - 1]로 마지막 원소를 맨 앞으로 이동
    • 왼쪽 회전: result[numbers.length - 1] = numbers[0]로 첫 번째 원소를 맨 뒤로 이동
    • 나머지 이동: for문을 통해 나머지 원소들을 한 칸씩 이동

    예시

    • numbers = [1, 2, 3], direction = "right"
      마지막 원소 3이 앞으로 이동 → [3, 1, 2]
    • numbers = [1, 2, 3], direction = "left"
      첫 번째 원소 1이 뒤로 이동 → [2, 3, 1]
    • numbers = [4, 455, 6, 4, -1, 45, 6], direction = "left"
      결과: [455, 6, 4, -1, 45, 6, 4]

    시간 복잡도

    배열을 한 번 순회하며 원소를 이동시키므로 시간 복잡도는 O(n)이다.

    공간 복잡도

    입력 배열과 동일한 크기의 result 배열을 새로 생성하므로 공간 복잡도는 O(n)이다.

    주의

    • 방향 문자열 확인: "right" 또는 "left" 외의 입력이 들어올 가능성은 없으나, 조건 분기에서 문자열 비교에 주의해야 한다.
    • 새 배열 반환: 기존 배열을 직접 수정하지 않고 새로운 배열을 생성하여 반환한다.
    • 인덱스 범위: 반복문에서 인덱스가 numbers.length - 1을 넘지 않도록 주의한다.
    대안 아이디어
    • Deque(덱)를 이용한 간결한 회전 — LinkedList로 구현 가능 
      
      import java.util.*;
      
      class Solution {
          public int[] solution(int[] numbers, String direction) {
              Deque<Integer> deque = new LinkedList<>();
              for (int n : numbers) deque.add(n);
      
              if (direction.equals("right")) {
                  deque.addFirst(deque.removeLast());
              } else {
                  deque.addLast(deque.removeFirst());
              }
      
              int[] result = new int[numbers.length];
              int i = 0;
              for (int n : deque) result[i++] = n;
      
              return result;
          }
      }

      Deque을 사용하면 직접 인덱스를 조정하지 않아도 addFirst / removeLast로 간단히 회전할 수 있다.

    문제 2 — 공 던지기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    원형으로 앉아 있는 사람들에게 공을 던질 때, 두 칸씩 건너뛰며 던졌을 때 k번째로 공을 받는 사람의 번호를 구하는 문제다.
    사람들의 번호는 배열 numbers로 주어지고, numbers의 길이를 넘어가면 다시 처음으로 돌아간다(즉, 원형 구조).

    한 번 던질 때마다 인덱스가 2씩 증가하므로, (k - 1)번 이동한 후의 인덱스를 계산하면 된다.
    이동 시 배열 길이를 넘어가면 mod 연산을 이용해 순환시킬 수 있다.

    • 초기화: 시작 인덱스 num = 0
    • 반복 이동: (num + 2) % numbers.length로 두 칸씩 이동
    • 정답 반환: 최종 인덱스는 0-based이므로, 실제 번호는 num + 1

    예시

    • numbers = [1, 2, 3, 4], k = 2
      첫 번째 공: 1번 → 두 번째 공: 3번 → 정답: 3
    • numbers = [1, 2, 3, 4, 5, 6], k = 5
      이동: 1 → 3 → 5 → 1 → 3 → 정답: 3
    • numbers = [1, 2, 3], k = 3
      이동: 1 → 3 → 2 → 정답: 2

    시간 복잡도

    for문이 k - 1번 반복되므로 시간 복잡도는 O(k)이다.
    단, 각 연산이 상수 시간이므로 입력 크기에 비해 매우 효율적이다.

    공간 복잡도

    입력 배열 외에 별도의 추가 자료구조를 사용하지 않으므로 공간 복잡도는 O(1)이다.

    주의

    • 0-based 인덱스: 계산 과정은 인덱스를 기준으로 하므로 마지막에 +1을 더해 사람 번호로 변환해야 한다.
    • 원형 구조: 배열을 벗어날 때 % numbers.length로 인덱스를 순환시켜야 한다.
    • 이동 규칙: 한 번 던질 때마다 “두 칸씩” 이동하므로 +2로 계산해야 한다. +1로 잘못 구현하면 오답이 된다.
    대안 아이디어
    • 수식으로 바로 계산 — 반복문 없이 한 줄로 해결 
      
      class Solution {
          public int solution(int[] numbers, int k) {
              return ((k - 1) * 2 % numbers.length) + 1;
          }
      }

      이동 횟수 (k - 1)과 한 번당 이동 거리 2를 곱하고, 배열 길이로 나눈 나머지를 이용해 직접 위치를 계산할 수 있다.
      반복문을 생략하므로 O(1) 시간에 해결 가능하다.

    문제 3 — 2차원으로 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 1차원 정수 배열 num_list를 n개의 원소씩 묶어 2차원 배열로 변환하는 문제다.
    즉, num_list를 행(row) 단위로 나누어 [[a₁, a₂, ..., aₙ], [aₙ₊₁, ..., a₂ₙ], ...] 형태로 재배치해야 한다.

    입력 배열의 길이를 n으로 나누어 행(row)의 개수를 구하고,
    이중 반복문을 사용해 순서대로 값을 채워 넣으면 된다.

    • 행의 개수: row = num_list.length / n
    • 열의 개수: column = n
    • 값 채우기: result[r][c] = num_list[idx++]로 순서대로 채운다.

    예시

    • num_list = [1, 2, 3, 4, 5, 6], n = 2
      행 개수: 6 / 2 = 3 → 결과: [[1, 2], [3, 4], [5, 6]]
    • num_list = [100, 95, 2, 4, 5, 6, 18, 33, 948], n = 3
      행 개수: 9 / 3 = 3 → 결과: [[100, 95, 2], [4, 5, 6], [18, 33, 948]]
    • num_list = [1, 2, 3, 4], n = 4
      행 개수: 4 / 4 = 1 → 결과: [[1, 2, 3, 4]]

    시간 복잡도

    배열의 모든 원소를 한 번씩 순회하며 채우므로, 시간 복잡도는 O(n) (정확히는 O(num_list.length))이다.

    공간 복잡도

    입력 배열과 별도로 크기 row × column의 2차원 배열을 생성하므로, 공간 복잡도는 O(n)이다.

    주의

    • 배열 길이와 n의 관계: num_list.length는 항상 n의 배수로 주어지므로 나머지는 고려하지 않아도 된다.
    • 인덱스 증가: idx++를 통해 1차원 배열의 순서를 유지하며 2차원 배열을 채워야 한다.
    • 이중 반복문의 순서: 행 → 열 순서로 채워야 올바른 2차원 형태가 만들어진다.
    대안 아이디어
    • Stream을 활용한 간결한 구현 — 반복문 없이 배열 분할 
      
      import java.util.*;
      import java.util.stream.*;
      
      class Solution {
          public int[][] solution(int[] num_list, int n) {
              return IntStream.range(0, num_list.length / n)
                              .mapToObj(i -> Arrays.copyOfRange(num_list, i * n, (i + 1) * n))
                              .toArray(int[][]::new);
          }
      }

      Java Stream API의 IntStream.range와 Arrays.copyOfRange를 이용하면 코드가 간결해지며, 동일한 시간 복잡도로 동작한다.
