    문제 1 — 모스부호 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    공백(" ")으로 구분된 모스 부호들을 영문 소문자 문자열로 변환하는 문제다.
    입력된 모스 문자열을 키로, 알파벳 문자를 값으로 하는 해시맵(모스 → 문자)을 구성해 각 토큰을 매핑한다.

    입력 문자열을 공백 기준으로 나누어 각 모스 부호를 변환 후 StringBuilder에 순서대로 이어 붙인다.
    모든 토큰을 처리한 뒤 누적된 문자열을 반환하면 된다.

    • 핵심 아이디어: 해시맵으로 빠른(O(1)) 모스 부호 → 문자 변환 수행
    • 자료구조: HashMap<String, Character> 이용, 26개 고정 매핑
    • 변환 과정: split(" ") → 각 토큰 get() → append()

    예시

    • letter = ".... . .-.. .-.. ---"
      → "hello"
    • letter = ".--- .- ...- .-"
      → "java"

    시간 복잡도

    모스 부호의 개수를 n이라 하면, 각 변환은 해시맵 조회로 O(1)이므로 전체는 O(n).

    공간 복잡도

    고정 크기 매핑(26개) 외에는 입력 및 출력 문자열만 사용하므로 O(1) (출력 제외).

    주의

    • 입력 전처리: trim() 후 split(" +")로 처리하면 불필요한 공백에도 안전하다.
    • 매핑 보장: 문제 조건상 모든 모스 부호는 유효하므로 NullPointerException 위험은 없다.
    • 초기화 방식: 더블 브레이스 초기화는 편리하지만 익명 클래스 생성 비용이 있으므로 정적 초기화로 대체 가능.
    대안 아이디어
    • 정적 초기화 블록으로 매핑 구성
      더블 브레이스 초기화 대신 클래스 로딩 시 1회만 실행되어 불필요한 익명 클래스 생성을 피할 수 있다. 
      import java.util.*;
      
      class Solution {
          private static final Map<String, Character> MORSE = new HashMap<>(32);
          static {
              MORSE.put(".-", 'a'); MORSE.put("-...", 'b'); MORSE.put("-.-.", 'c');
              MORSE.put("-..", 'd'); MORSE.put(".", 'e'); MORSE.put("..-.", 'f');
              MORSE.put("--.", 'g'); MORSE.put("....", 'h'); MORSE.put("..", 'i');
              MORSE.put(".---", 'j'); MORSE.put("-.-", 'k'); MORSE.put(".-..", 'l');
              MORSE.put("--", 'm'); MORSE.put("-.", 'n'); MORSE.put("---", 'o');
              MORSE.put(".--.", 'p'); MORSE.put("--.-", 'q'); MORSE.put(".-.", 'r');
              MORSE.put("...", 's'); MORSE.put("-", 't'); MORSE.put("..-", 'u');
              MORSE.put("...-", 'v'); MORSE.put(".--", 'w'); MORSE.put("-..-", 'x');
              MORSE.put("-.--", 'y'); MORSE.put("--..", 'z');
          }
      
          public String solution(String letter) {
              StringBuilder sb = new StringBuilder();
              for (String token : letter.trim().split(" ")) {
                  sb.append(MORSE.get(token));
              }
              return sb.toString();
          }
      }

      기능은 동일하지만 더블 브레이스 초기화보다 메모리 및 성능 효율이 좋다.

    문제 2 — 개미 군단

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    개미 병사들이 각각 5, 3, 1의 공격력을 가지고 있을 때,
    주어진 체력 hp를 정확히 0으로 만들기 위해 필요한 최소 병사 수를 구하는 문제다.

    공격력이 큰 개미부터 차례로 최대한 많이 사용하면, 항상 최소 개수가 보장된다.
    이는 그리디(Greedy) 접근으로 해결할 수 있다.

    • 공격력 순서: 5 → 3 → 1
    • 계산 순서:
      1. hp를 5로 나눈 몫을 더하고 나머지를 계산
      2. 그 나머지를 3으로 나눈 몫을 더하고 나머지를 계산
      3. 마지막으로 1로 나눈 몫을 더하면 총 병사 수 완성
    • 공격력 단위가 서로 배수 관계이므로 Greedy로도 항상 최적해를 얻을 수 있다.

    예시

    • hp = 23
      → 5로 4번(20), 나머지 3은 3으로 1번 → 총 5마리
    • hp = 24
      → 5로 4번(20), 나머지 4는 3으로 1번 + 1로 1번 → 총 6마리
    • hp = 999
      → 5로 199번(995), 나머지 4는 3으로 1번 + 1로 1번 → 총 201마리

    시간 복잡도

    단순한 산술 연산 3회만 수행하므로 시간 복잡도는 O(1).

    공간 복잡도

    추가적인 자료구조를 사용하지 않으므로 공간 복잡도는 O(1).

    주의

    • 순서 중요: 반드시 공격력이 큰 병사부터 나누어야 최소 개수가 된다.
    • 입력 범위: hp는 항상 양의 정수로 주어진다.
    • 효율성: 단순 나눗셈 기반이므로 추가 최적화가 필요 없다.
    대안 아이디어
    • 일반화된 배열 기반 접근
      공격력 종류가 여러 개로 바뀌더라도 동일한 원리로 처리할 수 있다. 
      import java.util.*;
      
      class Solution {
          public int solution(int hp) {
              int[] power = {5, 3, 1};
              int count = 0;
              for (int p : power) {
                  count += hp / p;
                  hp %= p;
              }
              return count;
          }
      }

      하드코딩 대신 배열을 사용하면 공격력 종류가 바뀌어도 유연하게 대응할 수 있다.

    문제 3 — 순서쌍의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 자연수 n의 약수 개수를 구하는 문제다.
    1부터 √n까지만 반복하면서, i가 n을 나누는지 확인하면 된다.

    약수는 짝을 이루므로(i와 n / i), 하나를 찾을 때마다 2개를 세고,
    단, i * i == n인 경우(즉, 제곱수)는 중복으로 세지 않도록 1만 더한다.

    • 반복 범위: 1 ≤ i ≤ √n
    • 판별 조건: n % i == 0일 때 약수 발견
    • 중복 처리: 제곱수 여부를 조건문으로 구분

    예시

    • n = 20
      약수: 1, 2, 4, 5, 10, 20 → 총 6개
    • n = 25
      약수: 1, 5, 25 → 총 3개 (5는 중복 제외)
    • n = 1
      약수: 1 → 총 1개

    시간 복잡도

    1부터 √n까지만 탐색하므로 시간 복잡도는 O(√n).

    공간 복잡도

    단일 변수만 사용하므로 공간 복잡도는 O(1).

    주의

    • 중복 약수 처리: 제곱수의 경우 1번만 카운트해야 한다.
    • 자료형: i * i 계산 시 int 범위 내에서 안전하지만, n이 매우 크면 long으로 캐스팅 고려.
    • 효율성: 단순 반복보다 √n까지만 순회하므로 충분히 빠르다.
    대안 아이디어
    • 소인수분해 기반 약수 개수 계산
      약수의 개수는 각 소인수의 지수에 1을 더해 곱한 값으로도 구할 수 있다. 
      import java.util.*;
      
      class Solution {
          public int solution(int n) {
              int count = 1;
              for (int i = 2; i * i <= n; i++) {
                  int exponent = 0;
                  while (n % i == 0) {
                      n /= i;
                      exponent++;
                  }
                  if (exponent > 0) count *= (exponent + 1);
              }
              if (n > 1) count *= 2; // 마지막 남은 소인수
              return count;
          }
      }

      수학적 접근으로, 약수의 개수를 효율적으로 구할 수 있다.
      단, 구현 복잡도가 다소 높으므로 기본 문제에서는 단순 √n 탐색이 더 직관적이다.
