    문제 1 — 점의 위치 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    서로 다른 balls개의 공 중에서 share개의 공을 뽑는 경우의 수를 구하는 문제다.
    이는 수학적 조합(Combination)으로 표현되며, C(n, r) = n! / (r! × (n - r)!) 공식을 사용한다.

    단, 팩토리얼을 직접 계산하면 수가 매우 커질 수 있으므로,
    이 문제에서는 분자와 분모를 단계별로 나누어 계산해 효율적으로 조합 값을 구한다.

    • r = min(share, balls - share) 로 불필요한 반복을 줄인다.
    • 반복문을 통해 result = result × (balls - i + 1) / i 형태로 누적 계산한다.
    • 계산 도중 overflow를 방지하기 위해 long 타입을 사용한다.

    예시

    • balls = 3, share = 2
      C(3, 2) = 3 → 3
    • balls = 5, share = 3
      C(5, 3) = 10 → 10
    • balls = 30, share = 15
      큰 수이지만 반복 계산을 통해 정확히 구할 수 있다.

    시간 복잡도

    반복문이 r = min(share, balls - share) 번 수행되므로 시간 복잡도는 O(r)이다.

    공간 복잡도

    별도의 배열이나 추가 자료구조를 사용하지 않으므로 공간 복잡도는 O(1)이다.

    주의

    • 특수 케이스: share == 0 또는 balls == share인 경우 경우의 수는 항상 1이다.
    • 정수 나눗셈 주의: 곱셈 후 나눗셈 순서를 지켜야 손실 없이 정확한 결과를 얻을 수 있다.
    • 오버플로 방지: 계산 과정에서 수가 커질 수 있으므로 int 대신 long을 사용한다.
    대안 아이디어
    • 팩토리얼 직접 계산 방식 
      class Solution {
          public long solution(int balls, int share) {
              return factorial(balls) / (factorial(share) * factorial(balls - share));
          }
      
          private long factorial(int n) {
              long result = 1;
              for (int i = 1; i <= n; i++) result *= i;
              return result;
          }
      }

      구현은 간단하지만, 입력값이 커질수록 오버플로가 발생할 위험이 있다.
      따라서 실무나 대형 입력에서는 누적 계산 방식이 더 안전하다.

    문제 2 — 구슬을 나누는 경우의 수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    서로 다른 balls개의 공 중에서 share개를 뽑는 경우의 수를 구하는 문제다.
    이는 수학에서 말하는 조합(Combination)으로, C(n, r) = n! / (r! × (n - r)!) 공식을 사용해 계산할 수 있다.

    단, 팩토리얼을 그대로 계산하면 값이 매우 커질 수 있기 때문에,
    이 문제에서는 불필요한 곱셈을 줄이고, 중간 값의 크기를 최소화하는 방식으로 조합을 계산하는 것이 핵심이다.

    • C(n, r)에서 r과 n - r 중 더 작은 값을 선택하면 계산 횟수를 줄일 수 있다.
    • 분자는 n × (n - 1) × ... 형태로 감소시키며 곱하고,
    • 분모는 그때그때 나누어 주어 result = result × (n - i + 1) / i 형태로 안전하게 계산한다.

    예시

    • balls = 3, share = 2
      C(3, 2) = 3 → 3
    • balls = 5, share = 3
      C(5, 3) = 10 → 10
    • balls = 5, share = 0 또는 balls = 5, share = 5
      모든 공을 선택하거나, 아무 것도 선택하지 않는 경우 → 경우의 수는 항상 1

    시간 복잡도

    r = min(share, balls - share)에 대해 한 번의 반복만 수행하므로 시간 복잡도는 O(r)이다.

    공간 복잡도

    추가적인 배열이나 컬렉션을 사용하지 않고 누적 변수만 사용하므로 공간 복잡도는 O(1)이다.

    주의

    • 경계 값 처리: share == 0 또는 balls == share인 경우 결과는 항상 1이다.
    • 정수 나눗셈: 매 단계에서 곱셈 후 나눗셈 순서를 지켜야 올바른 조합 값이 나온다.
    • 오버플로: 중간 연산 값이 커질 수 있으므로 int가 아닌 long을 사용하는 것이 안전하다.
    대안 아이디어
    • 팩토리얼 직접 계산 방식 
      class Solution {
          public long solution(int balls, int share) {
              return factorial(balls) / (factorial(share) * factorial(balls - share));
          }
      
          private long factorial(int n) {
              long result = 1;
              for (int i = 1; i <= n; i++) result *= i;
              return result;
          }
      }

      구현은 직관적이지만, 값이 매우 커지는 입력에서 오버플로 위험이 있어
      실제 테스트 환경에서는 앞서 설명한 조합 누적 계산 방식을 사용하는 것이 더 안전하다.

    문제 3 — 가위 바위 보

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 형태로 주어진 가위바위보 결과 rsp에 대해, 이기는 경우의 손동작을 반환하는 문제다.
    각 문자는 다음과 같이 대응된다.

    • '2' → 가위
    • '0' → 바위
    • '5' → 보

    규칙은 가위(2)는 보(5)를 이기고, 바위(0)는 가위(2)를 이기며, 보(5)는 바위(0)를 이긴다.
    즉, 이기는 손동작으로 치환하는 매핑 관계는 다음과 같다.

    • '2' → '0'
    • '0' → '5'
    • '5' → '2'

    이를 문자열을 문자 배열로 변환한 뒤, 각 문자를 switch문으로 순회하며 변환하면 된다.

    예시

    • rsp = "2"
      가위(2)를 이기려면 바위(0) → "0"
    • rsp = "205"
      '2' → '0', '0' → '5', '5' → '2' → 결과: "052"
    • rsp = "220"
      '2' → '0', '2' → '0', '0' → '5' → 결과: "005"

    시간 복잡도

    문자열의 길이를 n이라 하면, 각 문자를 한 번씩만 검사하므로 시간 복잡도는 O(n)이다.

    공간 복잡도

    입력 문자열을 문자 배열로 변환한 뒤 결과 문자열을 새로 생성하므로 공간 복잡도는 O(n)이다.

    주의

    • 입력 검증: 문제 조건상 '2', '0', '5' 이외의 문자가 등장하지 않는다.
    • 문자 인덱스 관리: 반복문 내에서 인덱스를 직접 증가시키므로, i++의 위치에 유의한다.
    • 문자 vs 숫자: 입력은 문자열이므로 문자 단위로 처리해야 하며, 정수형 연산을 수행하지 않는다.
    대안 아이디어
    • 문자열 치환을 활용한 간단한 구현 
      class Solution {
          public String solution(String rsp) {
              return rsp.replace('2', 'a')
                        .replace('0', '5')
                        .replace('5', '2')
                        .replace('a', '0');
          }
      }

      임시 문자('a')를 활용해 중복 치환 문제를 방지하는 방식으로,
      간결하지만 효율성은 다소 떨어질 수 있다.
