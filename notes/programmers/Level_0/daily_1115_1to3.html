    문제 1 — 진료순서 정하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    응급도 배열 emergency가 주어졌을 때, 각 요소가 전체에서 어느 정도로 높은 우선순위를 가지는지를 “등수(rank)”로 매기는 문제다.
    값이 클수록 더 높은 등수(1등)에 해당한다.

    각 원소 emergency[i]에 대해, 다른 모든 원소들과 비교하며 자신보다 더 큰 값이 몇 개 있는지를 센 뒤,
    기본 등수 1에서 그 개수만큼 더해 등수를 결정한다.

    • 핵심 아이디어: 모든 요소를 서로 비교해 상대적 우선순위 계산
    • 비교 방식: emergency[i] < emergency[j]면 등수 +1
    • 자료 구조: 별도 정렬 없이 단순 이중 반복문만 사용

    예시

    • emergency = [3, 76, 24]
      비교 결과 → [3등, 1등, 2등] → [3, 1, 2]
    • emergency = [30, 10, 23, 6, 100]
      비교 결과 → [2, 5, 3, 4, 1]

    시간 복잡도

    배열 길이를 n이라 하면 모든 원소를 서로 비교하므로
    전체 시간 복잡도는 O(n²).

    공간 복잡도

    출력 배열 rank만 사용하므로 추가 공간은 O(n).

    주의

    • 중복 값: 문제 조건상 중복 응급도는 등장하지 않으므로 순위 충돌은 발생하지 않는다.
    • 정렬 기반 대안 가능: 정렬로도 풀 수 있지만, 현재 구현은 단순 비교 기반.
    • 입력 크기: n이 작기 때문에 O(n²)도 무리 없이 동작.
    대안 아이디어
    • 정렬을 이용해 등수 매기기
      값과 인덱스를 함께 정렬한 뒤 순위를 기록하는 방식으로도 가능하며, 시간 복잡도는 O(n log n)으로 개선된다. 
      import java.util.*;
      
      class Solution {
          public int[] solution(int[] emergency) {
              int n = emergency.length;
              int[] answer = new int[n];
      
              // 값과 인덱스를 쌍으로 관리
              int[][] arr = new int[n][2];
              for (int i = 0; i < n; i++) {
                  arr[i][0] = emergency[i]; // 값
                  arr[i][1] = i;            // 원래 인덱스
              }
      
              // 값 기준 내림차순 정렬
              Arrays.sort(arr, (a, b) -> b[0] - a[0]);
      
              // 정렬 순서대로 등수 매기기
              for (int rank = 1; rank <= n; rank++) {
                  int originalIndex = arr[rank - 1][1];
                  answer[originalIndex] = rank;
              }
      
              return answer;
          }
      }

      이중 반복문보다 효율적이며, 대규모 입력에도 적합한 방식이다.

    문제 2 — 외계행성의 나이

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    주어진 나이 age를 문제에서 정의한 방식대로 알파벳 소문자로 치환된 문자열로 변환하는 문제다.
    각 자리의 숫자 0~9를 각각 a~j로 대응시키는 단순 매핑이다.

    입력된 정수를 10으로 나누어가며 일의 자리 숫자를 추출하고, 그 숫자에 문자 'a'를 더해 알파벳으로 변환한다.
    뒤에서부터 처리되므로 StringBuilder에 추가한 뒤 마지막에 reverse()로 순서를 바로잡아 문자열을 완성한다.

    • 핵심 아이디어: 각 자릿수를 0→a, 1→b … 9→j 로 매핑
    • 문자 변환 방식: (char)(digit + 'a')
    • 문자열 조립: 뒤집혀 누적되므로 reverse()로 정방향 변환

    예시

    • age = 23
      → 2 → 'c', 3 → 'd'
      → "cd"
    • age = 907
      → 9→'j', 0→'a', 7→'h'
      → "jah"

    시간 복잡도

    나이의 자릿수를 k라 하면 각 자릿수 변환을 한 번씩 수행하므로 전체는 O(k).

    공간 복잡도

    StringBuilder 한 개만 사용하므로 O(k).

    주의

    • 0 처리: 입력이 0일 경우 문자열은 "a"가 되어야 하므로 별도 처리 필요(해당 문제는 age ≥ 1 조건 존재).
    • 문자 변환: 'a' + digit은 문자 코드 기반 덧셈으로, 문자열 결합과 다름에 주의.
    • 순서 반전: 뒤에서부터 자릿수를 처리하므로 reverse()는 필수.
    대안 아이디어
    • 문자열 변환 후 배열 기반 매핑
      숫자를 문자열로 바꾸고, 각 문자(숫자)를 직접 변환하는 방식도 가능하다. 
      import java.util.*;
      
      class Solution {
          public String solution(int age) {
              char[] map = "abcdefghij".toCharArray();
              String s = String.valueOf(age);
      
              StringBuilder sb = new StringBuilder();
              for (char c : s.toCharArray()) {
                  sb.append(map[c - '0']);
              }
              return sb.toString();
          }
      }

      자릿수 분해 없이 문자 인덱싱만 사용하므로 구현 난이도가 낮고 가독성이 좋다.

    문제 3 — 배열 자르기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 numbers에서 인덱스 num1부터 num2까지의 구간을 잘라 새로운 배열로 반환하는 문제다.
    자바 스트림의 IntStream.rangeClosed()를 사용하여 범위에 포함되는 인덱스만 순회하며 원하는 요소를 추출한다.

    rangeClosed(num1, num2)는 양 끝단을 모두 포함하는 정수 스트림을 생성한다.
    이후 map(i → numbers[i])로 해당 인덱스의 값을 매핑하고 toArray()로 배열을 만들어 반환한다.

    • 핵심 아이디어: 인덱스 범위를 그대로 스트림으로 생성해 간결하게 배열 조각 추출
    • 스트림 변환: rangeClosed() → map() → toArray()
    • 자료 구조: 별도 임시 배열 없이 즉시 최종 배열 생성

    예시

    • numbers = [1, 2, 3, 4, 5], num1 = 1, num2 = 3
      → 추출 구간: 인덱스 1~3 → [2, 3, 4]
    • numbers = [10, 20, 30], num1 = 0, num2 = 1
      → [10, 20]

    시간 복잡도

    추출 범위의 길이를 k = num2 - num1 + 1이라 하면 순차적으로 k개 요소를 처리하므로
    전체 시간 복잡도는 O(k).

    공간 복잡도

    출력 배열 k 길이만큼만 사용되므로 O(k).

    주의

    • 인덱스 포함 범위: rangeClosed()는 끝값 포함이므로 문제 조건과 정확히 일치한다.
    • 예외 상황: 입력이 항상 정상 범위 내에서 주어지므로 인덱스 예외는 발생하지 않는다.
    • stream 오버헤드: 배열 슬라이싱만 필요할 경우 단순 반복문이 더 빠르지만, 가독성 면에서 스트림도 충분히 적합하다.
    대안 아이디어
    • 배열 복사 메서드 Arrays.copyOfRange() 사용
      Java에서 제공하는 가장 간단한 배열 슬라이싱 방식으로, 내부적으로 효율적이다. 
      import java.util.*;
      
      class Solution {
          public int[] solution(int[] numbers, int num1, int num2) {
              // 끝 인덱스는 exclusive 이므로 num2 + 1 필요
              return Arrays.copyOfRange(numbers, num1, num2 + 1);
          }
      }

      스트림보다 오버헤드가 적고 배열 슬라이싱에 가장 직관적이어서 실무에서도 자주 사용된다.
