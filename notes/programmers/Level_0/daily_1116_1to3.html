    문제 1 — 짝수의 합

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 n이 주어졌을 때, 0부터 n까지 존재하는 모든 짝수의 합을 구하는 문제다.
    반복문을 2씩 증가시키며 짝수만 누적하는 단순한 방식으로 해결할 수 있다.

    for (int i = 0; i <= n; i += 2) 와 같이 초기값을 0으로, 증가폭을 2로 설정하면
    조건 검사 없이 모든 짝수를 순회할 수 있다. 각 반복에서 누적 변수 sum에 더한 뒤 마지막에 반환한다.

    • 핵심 아이디어: 짝수만 순회하도록 증가폭을 2로 설정
    • 누적 방식: 반복문에서 sum += i 적용
    • 불필요한 연산 제거: i % 2 == 0 검사 없이 효율적 처리

    예시

    • n = 10
      짝수: 0, 2, 4, 6, 8, 10 → 합: 30
    • n = 5
      짝수: 0, 2, 4 → 합: 6

    시간 복잡도

    짝수 개수를 k ≈ n/2라 하면 반복은 k번 수행되므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    누적 변수 하나만 사용하므로 O(1).

    주의

    • 초기값 포함: 0 또한 짝수이므로 합에 포함된다.
    • 음수 입력 없음: 문제 조건상 n ≥ 0이므로 음수 처리 고려 불필요.
    • 성능 최적: 명시적인 짝수 검사(i % 2)를 쓰지 않고 가장 효율적인 순회 방식.
    대안 아이디어
    • 수학 공식으로 계산
      0부터 n까지의 짝수 합은 2 × (1 + 2 + ... + (n/2))와 동일하므로 등차수열 합 공식으로 계산할 수 있다. 
      class Solution {
          public int solution(int n) {
              int k = n / 2;               // 짝수 개수
              return k * (k + 1);          // 2 * (1 + 2 + ... + k) = k(k+1)
          }
      }

      반복문 없이 O(1) 시간에 바로 계산 가능해 가장 효율적이다.

    문제 2 — 양꼬치

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    양꼬치 n인분과 음료 k개를 주문했을 때의 총 비용을 계산하는 문제다.
    양꼬치 10인분마다 음료 1개가 서비스로 무료 제공되므로, 주문한 음료 개수에서 무료 개수를 차감해야 한다.

    무료 음료 개수는 n / 10이며, 실제 결제해야 하는 음료 수는 k - (n / 10)이다.
    이후 양꼬치 가격(12,000원 × n)과 유료 음료 가격(2,000원 × 남은 k)을 더하면 최종 금액이 된다.

    • 핵심 아이디어: 10인분당 음료 1개 무료 → k -= n / 10
    • 비용 계산: 양꼬치 12,000원, 음료 2,000원
    • 직접 계산 방식: 별도 반복 없이 단순 산술 연산만 수행

    예시

    • n = 10, k = 3
      무료 음료: 10 / 10 = 1개
      결제 음료: 3 - 1 = 2개
      총 비용: 12000×10 + 2000×2 = 124,000원
    • n = 25, k = 5
      무료 음료: 25 / 10 = 2개
      결제 음료: 5 - 2 = 3개
      총 비용: 12000×25 + 2000×3 = 306,000원

    시간 복잡도

    간단한 산술 계산만 수행하므로 전체 시간 복잡도는 O(1).

    공간 복잡도

    추가 자료구조를 사용하지 않으므로 O(1).

    주의

    • 음료 최소 결제 개수: 무료 개수가 음료 수보다 많아도 음수로 내려가선 안 되지만 문제 입력 조건상 그런 경우는 없다.
    • 정수 나눗셈: n / 10은 소수점 버림 정수 나눗셈이다.
    • 단위 계산: 문제에서 가격이 원 단위이므로 별도 변환 필요 없다.
    대안 아이디어
    • 무료 음료 계산을 명확하게 분리하는 방식
      가독성을 중시하여 변수명을 분리하는 방식이다. 
      class Solution {
          public int solution(int n, int k) {
              int free = n / 10;        // 서비스 음료 개수
              int paid = k - free;      // 실제 결제 음료 개수
      
              return (n * 12000) + (paid * 2000);
          }
      }

      의미를 명확히 분리하여 유지보수가 조금 더 쉬워지는 형태다.

    문제 3 — 각도기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    입력된 각도 angle이 어떤 유형의 각인지 판별하여
    문제에서 지정한 번호로 반환하는 조건 분기 문제다.
    각도의 범위에 따른 분류는 다음과 같다:

    • 예각 (0° < angle < 90°) → 1
    • 직각 (angle = 90°) → 2
    • 둔각 (90° < angle < 180°) → 3
    • 평각 (angle = 180°) → 4

    단순 조건문(if-else if-else)으로 각 범위를 검사해 해당하는 번호를 바로 반환한다.

    • 핵심 아이디어: 명확한 범위 조건을 순서대로 검사
    • 분기 구조: 작은 범위부터 큰 범위로 조건을 배치해 충돌 방지
    • 문제 조건: 항상 0 < angle ≤ 180 이므로 예외 입력 없음

    예시

    • angle = 70 → 예각 → 1
    • angle = 90 → 직각 → 2
    • angle = 120 → 둔각 → 3
    • angle = 180 → 평각 → 4

    시간 복잡도

    조건문만 검사하므로 전체 시간 복잡도는 O(1).

    공간 복잡도

    추가 변수 사용이 없으므로 O(1).

    주의

    • 각도 범위 확인: 문제 조건상 0은 등장하지 않으며, 180도는 평각으로 처리된다.
    • 조건 순서: 90 전후의 경계값을 명확히 구분해야 한다.
    • 중복 조건 없음: 각 범위가 서로 겹치지 않으므로 단순 분기만으로 충분하다.
    대안 아이디어
    • 삼항 연산자 체인으로 축약
      간단한 비교만 반복되므로 삼항 연산자로 줄일 수도 있다. 
      class Solution {
          public int solution(int angle) {
              return angle < 90 ? 1 :
                     angle == 90 ? 2 :
                     angle < 180 ? 3 : 4;
          }
      }

      코드 길이는 짧아지지만 가독성이 떨어질 수 있어 취향에 따라 선택하면 된다.
