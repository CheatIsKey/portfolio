    문제 1 — 직각삼각형 출력하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    입력된 정수 n을 기준으로, 1행부터 n행까지 별(*)을 계단식으로 출력하는 문제다.
    각 행 번호 i에 대해 i개의 별을 출력하고 줄바꿈을 반복하는 단순한 중첩 반복 구조로 구현한다.

    바깥 반복문은 1행부터 n행까지의 줄 수를 처리하고,
    안쪽 반복문은 해당 행 번호만큼 '*' 문자를 출력한다.
    각 줄마다 System.out.println()으로 개행을 수행해 다음 줄을 이어서 출력한다.

    • 핵심 아이디어: 행 번호 = 출력해야 할 별 개수
    • 구조: 바깥 루프(행), 안쪽 루프(별 출력)
    • 입력 처리: Scanner로 정수 하나 입력

    예시

    • n = 3
            *
      **
      ***
    • n = 5
            *
      **
      ***
      ****
      *****

    시간 복잡도

    첫 행에 1개, 둘째 행에 2개 … n행에 n개 별을 찍으므로 총 출력 수는 1 + 2 + ... + n = n(n+1)/2이다.
    따라서 시간 복잡도는 O(n²).

    공간 복잡도

    출력만 수행하며 추가 자료구조가 없으므로 O(1).

    주의

    • 개행 위치: 각 행 출력 후 반드시 println()을 호출해야 올바른 형태가 된다.
    • Scanner 입력: 백준/프로그래머스 콘솔 환경에서 표준 입력 처리 방식임.
    • 출력 형식 유지: 공백 없이 별만 출력해야 한다.
    대안 아이디어
    • 문자열 반복 repeat() 사용
      Java 11 이상에서는 반복 루프 없이 한 줄을 더 간단하게 출력할 수 있다. 
      import java.util.*;
      
      public class Solution {
          public static void main(String[] args) {
              Scanner sc = new Scanner(System.in);
              int n = sc.nextInt();
      
              for (int i = 1; i <= n; i++) {
                  System.out.println("*".repeat(i));
              }
          }
      }

      내부적으로 동일한 시간 복잡도지만 코드가 훨씬 간결해진다.

    문제 2 — 뒤집힌 문자열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string을 역순(reverse)으로 뒤집어 반환하는 문제다.
    Java의 StringBuilder는 문자열을 뒤집는 reverse() 메서드를 제공하므로 이를 활용하여 간단하게 구현할 수 있다.

    입력 문자열을 기반으로 StringBuilder 객체를 생성한 뒤,
    내부 버퍼를 reverse()로 뒤집고 toString()으로 문자열 형태로 변환해 반환한다.

    • 핵심 아이디어: StringBuilder.reverse() 사용
    • 자료구조: 변경 가능한 문자열 구조인 StringBuilder
    • 간결한 구현: 한 줄로도 표현 가능 (return new StringBuilder(my_string).reverse().toString();)

    예시

    • my_string = "jaron"
      → "noraj"
    • my_string = "bread"
      → "daerb"

    시간 복잡도

    문자열 길이를 n이라 하면 reverse()는 내부 문자를 교환하며 한 번 순회하므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    StringBuilder 내부 버퍼가 필요하므로 공간 복잡도는 O(n).

    주의

    • 문자열 불변성: String은 불변(immutable)이므로 reverse()를 직접 사용할 수 없다.
    • 공백/특수문자 유지: 역순으로만 재배열하며 문자 내용은 변경되지 않는다.
    • 대소문자 구분 그대로 유지: 변환 과정에서 문자의 속성은 그대로 유지된다.
    대안 아이디어
    • 수동으로 문자 교환(reverse) 구현
      투 포인터 방식으로 직접 문자열을 뒤집을 수도 있다. 
      class Solution {
          public String solution(String my_string) {
              char[] arr = my_string.toCharArray();
              int left = 0, right = arr.length - 1;
      
              while (left < right) {
                  char temp = arr[left];
                  arr[left] = arr[right];
                  arr[right] = temp;
                  left++;
                  right--;
              }
      
              return new String(arr);
          }
      }

      기능은 동일하지만, StringBuilder를 쓰는 것이 훨씬 간결하고 안전하다.

    문제 3 — 배열 뒤집기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 num_list를 역순(reverse)으로 뒤집어 반환하는 문제다.
    투 포인터(two-pointer) 방식을 사용하여 배열을 제자리에서(in-place) 뒤집는다.

    시작 인덱스 idx는 0에서 시작하고, 끝 인덱스 last는 배열의 마지막 인덱스에서 시작한다.
    두 인덱스가 만나기 전까지 서로의 값을 교환(swap)하며, 각 단계마다 idx++, last--를 수행한다.
    모든 스왑이 끝나면 역순으로 뒤집힌 배열을 반환한다.

    • 핵심 아이디어: 투 포인터로 배열 앞뒤를 교환해 역순으로 뒤집기
    • 제자리 뒤집기: 추가 배열 없이 기존 배열을 그대로 수정
    • 스왑 방식: tmp 변수를 이용해 값 교환

    예시

    • num_list = [1, 2, 3, 4, 5]
      → [5, 4, 3, 2, 1]
    • num_list = [9, 8, 7]
      → [7, 8, 9]
    • num_list = [10]
      → [10] (단일 원소는 변경 없음)

    시간 복잡도

    배열 길이를 n이라 하면, 양 끝에서 중앙으로 이동하며 약 n/2번 스왑을 수행하므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    별도의 배열을 생성하지 않으므로 O(1) (상수 공간).

    주의

    • 스왑 후 인덱스 이동: 교환 후 반드시 idx++, last-- 해야 한다.
    • 배열 수정 방식: 원본 배열이 변경되는 방식이므로 복사본이 필요하다면 별도 처리 필요.
    • 홀수·짝수 길이: 길이가 홀수일 때 중앙 원소는 그대로 유지된다.
    대안 아이디어
    • 배열을 새로 만들어 역순 복사
      제자리(reverse) 방식 대신 새 배열을 만들어 채우는 방식. 
      class Solution {
          public int[] solution(int[] num_list) {
              int n = num_list.length;
              int[] result = new int[n];
      
              for (int i = 0; i < n; i++) {
                  result[i] = num_list[n - 1 - i];
              }
      
              return result;
          }
      }

      원본 배열을 보존하면서 새로 뒤집은 배열을 얻고 싶을 때 적합한 방식이다.
