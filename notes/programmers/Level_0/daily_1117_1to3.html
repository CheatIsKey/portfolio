    문제 1 — 특정 문자 제거하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string에서 특정 문자 letter를 모두 제거한 새로운 문자열을 반환하는 문제다.
    letter는 길이 1인 문자열이므로 char로 변환하여 비교한다.

    전체 문자열을 toCharArray()로 순회하면서, 제거 대상 문자가 아닌 경우에만
    StringBuilder에 추가하여 결과 문자열을 구성한다.

    • 핵심 아이디어: 단일 문자 비교로 필요 없는 문자만 제외
    • 문자열 조립: StringBuilder로 효율적인 누적
    • 단순 반복 처리: 모든 문자를 한 번씩만 검사

    예시

    • my_string = "abcdef", letter = "f"
      → "abcde"
    • my_string = "hello", letter = "l"
      → "heo"
    • my_string = "aaaaa", letter = "a"
      → "" (빈 문자열)

    시간 복잡도

    문자열 길이를 n이라 할 때 한 번만 순회하므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    결과 문자열을 저장하기 위한 StringBuilder가 필요하므로 O(n).

    주의

    • 문자 비교: letter.charAt(0)로 확실히 단일 문자 비교를 수행해야 한다.
    • 문자열 비교 금지: String.equals()로 비교하면 오히려 비효율적이며 필요도 없다.
    • 중복 제거: 중복 문자가 여러 번 등장해도 전부 제거된다.
    대안 아이디어
    • replace() 사용하기
      Java 문자열 치환 기능을 사용하면 훨씬 간결하게 처리할 수 있다. 
      class Solution {
          public String solution(String my_string, String letter) {
              return my_string.replace(letter, "");
          }
      }

      내부적으로 전체 문자열을 스캔하지만 구현이 매우 간단하며 직관적이다.

    문제 2 — 문자 반복 출력하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 my_string의 각 문자를 n번씩 반복하여 새 문자열을 만드는 문제다.
    예를 들어 "abc"와 n = 3이면 "aaabbbccc" 형태로 변환한다.

    전체 결과 길이는 my_string.length() * n이므로, 이를 미리 계산해
    char[] 배열을 생성하고 직접 문자를 채워 넣는 방식으로 구현한다.
    이중 반복문을 사용하되, 내부 반복에서 동일 문자를 연속 n번 삽입한다.

    • 핵심 아이디어: 각 문자를 n번 반복해 순서대로 배열에 삽입
    • 문자 처리 방식: toCharArray()로 순회 후 반복 삽입
    • 메모리 최적화: 결과 크기를 미리 계산해 char[]에 직접 채움

    예시

    • my_string = "hello", n = 3
      → "hhheeellllllooo"
    • my_string = "abc", n = 2
      → "aabbcc"
    • my_string = "z", n = 5
      → "zzzzz"

    시간 복잡도

    문자열 길이를 m이라 하면 각 문자를 n번 반복하므로 전체 시간 복잡도는 O(m × n).

    공간 복잡도

    결과 배열 크기 m × n만큼 필요하므로 O(m × n).

    주의

    • StringBuilder 미사용 이유: 반복 횟수가 크면 char[]가 더 효율적
    • 문자 하나씩 삽입: 내부 반복에서 idx++ 증가 순서에 주의
    • 결과 변환: 최종 문자열 생성 시 new String(result) 사용
    대안 아이디어
    • StringBuilder 활용
      결과 크기가 매우 크지 않다면 StringBuilder로 직관적으로 누적할 수 있다. 
      class Solution {
          public String solution(String my_string, int n) {
              StringBuilder sb = new StringBuilder(my_string.length() * n);
      
              for (char c : my_string.toCharArray()) {
                  sb.append(String.valueOf(c).repeat(n));
              }
      
              return sb.toString();
          }
      }

      Java 11 이상에서는 repeat()을 이용해 더 깔끔하고 가독성 높은 코드 작성이 가능하다.

    문제 3 — 짝수 홀수 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 num_list가 주어졌을 때, 배열 안에 포함된 짝수의 개수와 홀수의 개수를 계산해
    [짝수 개수, 홀수 개수] 형태의 배열로 반환하는 문제다.

    배열을 순회하면서 각 값이 짝수인지(i % 2 == 0) 홀수인지 판단해 각각의 카운터를 증가시킨다.
    모든 요소의 검사 후 짝수 개수와 홀수 개수를 순서대로 새 배열에 담아 반환하면 된다.

    • 핵심 아이디어: 나머지 연산(%)으로 짝수/홀수 판별
    • 카운팅 방식: even, odd 두 변수에 누적
    • 출력 형식: new int[]{even, odd} 형태로 반환

    예시

    • num_list = [1, 2, 3, 4, 5]
      짝수: 2, 4 → 2개
      홀수: 1, 3, 5 → 3개
      → [2, 3]
    • num_list = [0, 7, 14]
      짝수: 0, 14 → 2개
      홀수: 7 → 1개
      → [2, 1]

    시간 복잡도

    배열 길이를 n이라 하면 한 번만 순회하므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    결과 배열 크기 2만 추가로 필요하므로 O(1) (상수 공간).

    주의

    • 0은 짝수이므로 짝수 카운터에 포함된다.
    • 배열 순서: 반드시 [짝수, 홀수] 순서로 반환해야 한다.
    • 음수도 동일: 음수도 나머지 연산으로 동일하게 짝/홀 판정 가능.
    대안 아이디어
    • 스트림(Stream) 사용
      자바 스트림을 사용해 더 선언적으로 표현할 수 있다. 
      import java.util.*;
      
      class Solution {
          public int[] solution(int[] num_list) {
              int even = (int) Arrays.stream(num_list)
                                     .filter(i -> i % 2 == 0)
                                     .count();
      
              int odd = num_list.length - even;
              return new int[]{even, odd};
          }
      }

      가독성은 좋아지지만 기본 반복문보다 약간의 오버헤드가 있다.
