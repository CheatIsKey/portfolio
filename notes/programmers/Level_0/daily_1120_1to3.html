    문제 1 — 나이 출력

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    나이를 기준으로 출생 연도를 계산하는 문제다.
    문제에서는 2022년 기준이며, 한국식 나이 계산(태어났을 때 1살) 규칙을 따른다.

    한국식 나이를 이용하면 다음 공식으로 출생 연도를 계산할 수 있다:
    출생연도 = 기준연도 - (나이 - 1)
    이유는 한국식 나이는 나이 = 기준연도 - 출생연도 + 1이므로 이를 변형한 것이다.

    • 핵심 아이디어: 한국식 나이 → 출생연도 = 2022 - (age - 1)
    • 직접 계산: 별도 조건문 없이 한 줄 수식으로 처리
    • 기준 연도: 문제에서 고정된 2022년 사용

    예시

    • age = 40
      → 2022 - 39 = 1983
    • age = 1
      → 2022 - 0 = 2022
    • age = 10
      → 2022 - 9 = 2013

    시간 복잡도

    단순 산술 연산만 수행하므로 O(1).

    공간 복잡도

    추가 자료구조가 없으므로 O(1).

    주의

    • 한국식 나이 공식 사용: 단순히 2022 - age가 아니며 - (age - 1)을 해야 한다.
    • 기준 연도 고정: 문제 기준이 2022년이므로 달라지면 공식도 변해야 한다.
    • age ≥ 1: 입력은 항상 자연수이므로 예외 케이스 없음.
    대안 아이디어
    • 공식을 직관적으로 풀어 쓰기
      동일한 계산을 좀 더 직관적으로 표현할 수 있다. 
      class Solution {
          public int solution(int age) {
              int birthYear = 2022 - age + 1;  // 한국식 나이 변환
              return birthYear;
          }
      }

      동일한 결과를 주지만 개념적으로 더 이해하기 쉬운 형태다.

    문제 2 — 아이스 아메리카노

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    가진 돈 money로 아메리카노(1잔 5,500원)를 최대 몇 잔 살 수 있는지, 그리고 구매 후 남는 잔돈이 얼마인지 계산하여 [구매 가능 잔 수, 잔돈] 형태로 반환하는 문제다.

    구매 가능한 잔 수는 money / 5500으로 계산하며, 이는 정수 나눗셈이므로 자동으로 몫만 남는다.
    남은 금액은 총 금액에서 사용한 금액을 뺀 값으로, money - (5500 * cnt)로 계산한다.

    • 핵심 아이디어: 정수 나눗셈으로 구매 가능 잔 수 계산
    • 잔돈 계산: money - 5500 × cnt
    • 반환 형식: new int[]{잔수, 잔돈}

    예시

    • money = 15000
      구매 가능 잔 수: 15000 / 5500 = 2
      잔돈: 15000 - 11000 = 4000
      → [2, 4000]
    • money = 5500
      구매 가능 잔 수: 1
      잔돈: 0
      → [1, 0]
    • money = 3000
      구매 가능 잔 수: 0
      잔돈: 3000
      → [0, 3000]

    시간 복잡도

    단순 산술 연산이므로 전체 시간 복잡도는 O(1).

    공간 복잡도

    반환 배열 크기가 고정이므로 O(1).

    주의

    • 정수 나눗셈: money / 5500은 자동으로 몫만 남는다.
    • 가격 고정: 아메리카노 가격은 문제에서 항상 5,500원으로 고정.
    • 음수 입력 없음: 문제 조건상 money ≥ 0이므로 예외 상황 없음.
    대안 아이디어
    • 나머지 연산 활용
      잔돈은 직접 계산 대신 money % 5500으로 구할 수도 있다. 
      class Solution {
          public int[] solution(int money) {
              int cnt = money / 5500;
              int remain = money % 5500;   // 남은 잔돈 계산
      
              return new int[]{cnt, remain};
          }
      }

      나머지(%) 연산을 사용하면 잔돈 계산이 더 직관적이다.

    문제 3 — 옷가게 할인 받기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    구매 금액 price에 따라 할인율을 적용해 최종 결제 금액을 계산하는 문제다.
    조건에 따라 다음 할인 규칙을 가진다:

    • 50만 원 이상: 20% 할인 → price × 0.8
    • 30만 원 이상: 10% 할인 → price × 0.9
    • 10만 원 이상: 5% 할인 → price × 0.95
    • 그 외: 할인 없음

    삼항 연산자(?:)를 중첩해 조건을 순서대로 검사하고, 해당하는 할인율을 적용한 뒤 (int)로 정수형으로 변환해 반환한다.

    • 핵심 아이디어: 조건 범위가 큰 금액부터 내려오며 할인율 결정
    • 정수 반환 처리: 할인 후 금액은 실수이므로 (int)로 변환
    • 순차적 조건: 500k → 300k → 100k 순으로 검사

    예시

    • price = 580000
      → 20% 할인 → 580000 × 0.8 = 464000
    • price = 320000
      → 10% 할인 → 320000 × 0.9 = 288000
    • price = 120000
      → 5% 할인 → 120000 × 0.95 = 114000
    • price = 90000
      → 할인 없음 → 90000

    시간 복잡도

    조건문 검사와 산술 연산만 수행하므로 시간 복잡도는 O(1).

    공간 복잡도

    별도의 자료구조가 필요 없으므로 공간 복잡도는 O(1).

    주의

    • 강제 형변환: 할인 후 값은 double이므로 반드시 (int)로 캐스팅해야 한다.
    • 내림 처리: 형변환은 자동으로 소수점 아래를 버리는 내림 방식이다.
    • 조건 순서 중요: 큰 금액부터 검사해야 정확한 할인율 적용이 가능하다.
    대안 아이디어
    • if–else 조건문으로 가독성 향상
      삼항 연산자 대신 일반 조건문을 사용하면 읽기 쉬운 구조가 된다. 
      class Solution {
          public int solution(int price) {
              if (price >= 500000) return (int) (price * 0.8);
              else if (price >= 300000) return (int) (price * 0.9);
              else if (price >= 100000) return (int) (price * 0.95);
              else return price;
          }
      }

      논리가 동일하지만 유지보수 측면에서는 if-else 구조가 더 직관적이다.
