    문제 1 — 배열의 평균값

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    정수 배열 numbers의 모든 원소의 평균값을 계산하는 문제다.
    전체 합을 구한 뒤, 배열의 길이로 나누어 평균을 반환한다.

    반복문을 통해 각 요소를 sum에 누적하고,
    마지막에 sum / numbers.length를 수행하여 평균을 구한다.
    평균은 실수(double)로 반환해야 하므로 sum은 double로 선언한다.

    • 핵심 아이디어: 전체 합 → 길이로 나누기
    • 데이터 타입: 정수 배열 평균은 실수이므로 double 계산 필요
    • 안정성: 배열 길이는 항상 1 이상이므로 0으로 나누는 상황 없음

    예시

    • numbers = [1, 2, 3, 4]
      합: 10 → 평균: 10 / 4 = 2.5
    • numbers = [5, 5, 5]
      합: 15 → 평균: 15 / 3 = 5.0
    • numbers = [10, 0]
      합: 10 → 평균: 10 / 2 = 5.0

    시간 복잡도

    배열 길이를 n이라 하면 모든 요소를 한 번씩 순회하므로 전체 시간 복잡도는 O(n).

    공간 복잡도

    추가 공간이 거의 없으므로 O(1) (상수 공간).

    주의

    • 정수 나눗셈 방지: sum이 double이므로 자동으로 실수 나눗셈이 수행된다.
    • 배열 길이 ≥ 1: 문제 조건상 항상 최소 1개 이상의 원소가 주어진다.
    • 자료형 변환 주의: sum을 int로 선언하면 정수 연산이 되어 잘못된 결과가 나온다.
    대안 아이디어
    • Java Stream API 사용
      import java.util.*;
      
      class Solution {
          public double solution(int[] numbers) {
              return Arrays.stream(numbers)
                           .average()
                           .orElse(0);
          }
      }

      내장된 average() 메서드를 활용하면 더 간결하고 안전한 평균 계산이 가능하다.

    문제 2 — 피자 나눠 먹기 (3)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    피자 한 판을 slice조각으로 나눌 수 있을 때, 총 n명이 모두 1조각 이상 먹기 위해 필요한 피자 판 수를 구하는 문제다.

    기본적으로 한 판에서 slice명이 먹을 수 있으므로, n / slice로 필요한 기본 피자 수를 구하고,
    나누어 떨어지지 않을 경우(n % slice != 0) 한 판을 더 주문해야 한다.

    • 핵심 아이디어: 필요한 조각 수 → 조각 수로 나누고 올림 처리
    • 기본 계산: pizza = n / slice
    • 올림 처리: 나머지가 있으면 pizza++

    예시

    • slice = 7, n = 10
      → 10 / 7 = 1, 나머지 3 → 한 판 더 필요 → 2
    • slice = 4, n = 8
      → 8 / 4 = 2, 나머지 0 → 추가 필요 없음 → 2
    • slice = 5, n = 16
      → 16 / 5 = 3, 나머지 1 → 한 판 더 → 4

    시간 복잡도

    단순한 산술 연산과 조건문만 있으므로 전체 시간 복잡도는 O(1).

    공간 복잡도

    추가 자료구조 없음 → O(1).

    주의

    • 정확한 올림 처리: 단순 정수 나눗셈으로는 부족하며 나머지 처리 필수.
    • slice ≥ 1, n ≥ 1: 문제 조건상 0이 입력되지 않아 예외 없음.
    • 조각 개수 그대로 사용: 다른 할인/누락 조건 없음.
    대안 아이디어
    • Math.ceil()을 이용한 올림 처리
      실수 계산을 활용하면 더욱 직관적으로 표현할 수 있다. 
      class Solution {
          public int solution(int slice, int n) {
              return (int) Math.ceil(n / (double) slice);
          }
      }

      나눗셈을 실수로 수행한 뒤 올림하면 추가 조건문 없이 결과를 얻을 수 있다.

    문제 3 — 피자 나눠 먹기 (2)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    각 피자는 6조각이며, n명이 모두 똑같은 조각 수를 먹으려면 최소 몇 판의 피자가 필요한지를 구하는 문제다.
    핵심은 6조각이 n명에게 정확히 나누어떨어지도록 피자 판 수를 결정하는 것이다.

    식으로 표현하면, 피자 판 수를 i라고 할 때 (n × i) % 6 == 0을 만족하는 최소의 i를 찾으면 된다. 즉, n과 6의 최소공배수(LCM)를 이용한 문제이며, 코드에서는 반복문으로 가장 작은 i를 찾는다.

    • 핵심 아이디어: (n × i)가 6의 배수가 되는 최소 i 찾기
    • 최종 피자 판 수: pizza = (n × i) / 6
    • LCM 기반: 사실상 LCM(n, 6) / 6

    예시

    • n = 3
      (3 × 2) % 6 == 0 → 최소 i=2 → pizza = (3×2)/6 = 1 → 피자 1판 필요
    • n = 4
      (4 × 3) % 6 == 0 → i=3 → (4×3)/6 = 2 → 피자 2판 필요
    • n = 6
      (6×1) % 6 == 0 → 바로 가능 → 6/6 = 1 → 피자 1판 필요

    시간 복잡도

    최악의 경우 증가하는 i를 순차적으로 검사하므로 O(k) (k는 조건을 만족하는 최초 i). n은 1 ≤ n ≤ 100이므로 충분히 작아 실질적으로 매우 빠르게 종료된다.

    공간 복잡도

    변수 몇 개만 사용하므로 O(1).

    주의

    • 정확히 나누어떨어져야 함: 조각이 남거나 모자라면 안 된다.
    • i 증가 방식: 1부터 증가하면서 가장 작은 i를 찾아야 한다.
    • 무한 루프 문제 없음: 6과 n의 최소공배수는 반드시 존재하므로 종료된다.
    대안 아이디어
    • 최소공배수(LCM) 공식을 직접 이용
      반복문 없이 더 효율적으로 계산할 수 있다. 
      class Solution {
          public int solution(int n) {
              int gcd = gcd(n, 6);
              int lcm = n * 6 / gcd;
              return lcm / 6;
          }
      
          private int gcd(int a, int b) {
              while (b != 0) {
                  int temp = a % b;
                  a = b;
                  b = temp;
              }
              return a;
          }
      }

      LCM(n, 6) / 6이 필요한 피자 판 수와 동일하며, 공식 기반의 정석 풀이다.
