    문제 1 — 피자 나눠 먹기 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    피자 나눠 먹기 (1) (프로그래머스 120814) 문제는
    사람 수 n명이 있을 때, 한 판에 7조각이 나오는 피자를
    모두가 최소 1조각씩 먹도록 하려면 피자가 최소 몇 판 필요한지 계산하는 단순 수학 문제다.

    핵심은 7조각짜리 피자를 몇 판 주문해야 사람 n명이 모두 1조각 이상 먹을 수 있는지 구하는 것이다. 즉, 필요한 최소 피자 판 수는 아래와 같다.

    • n명을 기준으로 1판 = 7조각 → 필요 판 수 = n ÷ 7
    • 7로 정확히 나누어떨어지지 않으면 한 판을 더 추가해야 한다.

    코드에서는 정수 나눗셈을 이용하여 n / 7로 기본 판 수를 계산하고,
    나머지가 있으면 +1을 해주는 방식으로 해결한다.

    • 핵심 아이디어: 사람 수 n을 7로 나눈 뒤, 나머지가 있으면 한 판 추가
    • 연산 방식: pizza = n / 7 → n % 7 != 0이면 pizza++
    • 수학적 의미: ceil(n / 7) 과 동일

    예시

    • n = 7 → 7 / 7 = 1 → 1판
    • n = 10 → 10 / 7 = 1, 나머지 존재 → 2판
    • n = 1 → 1 / 7 = 0, 나머지 존재 → 1판

    시간 복잡도

    단순한 사칙연산만 수행하므로 O(1).

    공간 복잡도

    추가 자료구조 없이 상수 변수만 사용 → O(1).

    주의

    • 정수 나눗셈이므로 n / 7은 소수점을 버린다.
    • ceil(n / 7)을 직접 구현하면 불필요하므로, 나머지 체크 후 +1 방식이 가장 단순하고 빠르다.

    문제 2 — 짝수는 싫어요

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    짝수는 싫어요 (프로그래머스 120813) 문제는
    자연수 n이 주어졌을 때, 1부터 n까지의 모든 홀수를
    오름차순 배열로 반환하는 간단한 구현 문제다.

    핵심은 1부터 n까지 중에서 홀수만 추출하는 것이며,
    홀수의 개수는 n이 홀수인지 짝수인지에 따라 달라진다.

    • n이 홀수 → 홀수 개수 = n/2 + 1
    • n이 짝수 → 홀수 개수 = n/2

    즉, 미리 배열 크기를 정해놓고 1부터 n까지
    2씩 증가시키면서 홀수만 채워 넣으면 된다.

    • 핵심 아이디어: 홀수는 1, 3, 5, ...처럼 2씩 증가하는 규칙성 활용
    • 자료 구조: 결과 크기를 먼저 계산해 int[] 생성
    • 반복 구조: for (int i = 1; i <= n; i += 2)

    예시

    • n = 10 → {1, 3, 5, 7, 9}
    • n = 5 → {1, 3, 5}
    • n = 1 → {1}

    시간 복잡도

    1부터 n까지 홀수만 순회하므로 반복 횟수는 약 n/2 → O(n).

    공간 복잡도

    결과 배열 크기는 약 n/2이므로 O(n) (출력 배열 크기만큼).

    주의

    • 홀수 개수 계산: n이 홀수인지 짝수인지에 따라 배열 크기가 다르다.
    • 반복 조건 설정: i += 2로 홀수만 순회하면 조건문 없이 빠르게 처리 가능.
    • 정렬 필요 없음: 1 → 3 → 5 순서로 채워지므로 이미 오름차순이 된다.

    문제 3 — 최빈값 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    최빈값 구하기 (프로그래머스 120812) 문제는
    정수 배열에서 가장 자주 등장하는 값(최빈값)을 찾되,
    최빈값이 둘 이상이면 -1을 반환하는 구현 문제다.

    주어진 배열의 원소 범위가 0 ~ 1000으로 제한되어 있기 때문에
    등장 횟수를 저장할 카운트 배열을 활용하는 것이 가장 효율적이다.

    핵심 절차는 다음과 같다.

    • 1단계: 각 숫자의 등장 횟수 저장
    • 2단계: 가장 많이 등장한 값을 찾아 mode에 기록
    • 3단계: 동일한 최빈도 값이 또 나오면 dup = true로 표시
    • 4단계: 중복 최빈값이 있으면 -1, 아니면 mode 반환
    • 핵심 아이디어: 데이터 범위가 좁으므로 Counting Array(cnt[1001]) 사용
    • 중복 판정: 최빈값 빈도가 이미 기록된 값과 같으면 dup = true
    • 조건 처리: 최빈값이 한 개일 때만 그 값을 반환

    예시

    • array = {1, 2, 2, 3, 3}
      → 2와 3이 모두 빈도 2로 중복 → -1
    • array = {1, 1, 2, 3}
      → 최빈값은 1 → 1
    • array = {0}
      → 최빈값은 0 → 0

    시간 복잡도

    입력 배열을 한 번 순회하고(O(n)),
    카운트 배열(1001 길이)을 한 번 순회하므로 O(n + 1000) → O(n).

    공간 복잡도

    카운트 배열 int[1001]만 추가로 사용 → O(1) (상수 공간).

    주의

    • 최빈값 중복 여부 판단이 중요
      • 최대 빈도와 같은 빈도를 가진 값이 또 나오면 dup = true
      • 마지막에 dup이 true이면 -1 반환
    • maxFreq 초기값 0 → 배열에 등장하는 값 기준으로 갱신됨
    • mode 초기값 -1은 문제 조건에 맞는 안전한 초기값
