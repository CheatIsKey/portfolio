    문제 1 — 중앙값 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    중앙값 구하기 (프로그래머스 120811) 문제는
    주어진 정수 배열 array에서 중앙값(가운데 값)을 찾는 아주 기본적인 구현 문제다.
    문제 조건상 배열 길이는 항상 홀수이다.

    중앙값을 구하는 가장 직관적인 방법은:

    1. 배열을 오름차순 정렬한다.
    2. 가운데 인덱스 array.length / 2 위치의 값을 반환한다.

    길이가 홀수인 배열에서 인덱스는 0부터 시작하므로,
    예를 들어 길이가 5라면 인덱스는 0~4이고, 중앙값은 인덱스 2(5 / 2)가 된다.

    • 핵심 아이디어: 정렬 후 length / 2 번째 원소가 중앙값
    • 정렬 메서드: Arrays.sort(array)로 제자리 정렬
    • 배열 길이: 항상 홀수라 별도의 예외 처리 필요 없음

    예시

    • array = [1, 2, 7, 10, 11]
      정렬 후: [1, 2, 7, 10, 11] (이미 정렬)
      길이 5 → 인덱스 5 / 2 = 2 → 값은 7 → 중앙값 7
    • array = [9, -1, 0]
      정렬 후: [-1, 0, 9]
      길이 3 → 인덱스 3 / 2 = 1 → 값은 0 → 중앙값 0

    시간 복잡도

    Arrays.sort()는 일반적으로 O(n log n) 시간 복잡도를 가진다.
    (n은 배열의 길이)

    공간 복잡도

    배열을 제자리 정렬하므로 추가로 큰 메모리를 사용하지 않는다.
    구현 관점에서는 O(1) (입력 배열 외에 추가 배열을 만들지 않음)으로 볼 수 있다.

    주의

    • 문제에서 배열 길이가 항상 홀수라고 보장되므로, 중앙 두 값의 평균을 구하는 등의 처리가 필요 없다.
    • 정렬 후 반드시 인덱스 length / 2를 그대로 반환하면 된다.
    • 배열을 직접 정렬하므로, 원본 배열이 필요하다면 별도 복사 후 정렬해야 하지만, 이 문제에서는 원본 유지가 요구되지 않는다.

    문제 2 — 나머지 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    나머지 구하기 (프로그래머스 120810) 문제는
    두 정수 num1, num2가 주어졌을 때 num1 % num2 — 즉, num1을 num2로 나눈 나머지를 반환하는 가장 기본적인 산술 연산 문제다.

    자바에서는 나머지 연산자가 %이며,
    정수 간 연산에서 몫을 제외한 나머지를 직접 반환한다.

    • 핵심 아이디어: 나머지 연산 a % b 그대로 사용
    • 자료형: 두 입력 모두 int, 반환 타입 또한 int
    • 예외 처리: 문제 조건상 num2 != 0이므로 0 나눗셈 걱정 없음

    예시

    • num1 = 10, num2 = 3
      10 % 3 = 1 → 결과 1
    • num1 = 7, num2 = 2
      7 % 2 = 1 → 결과 1
    • num1 = 15, num2 = 5
      15 % 5 = 0 → 결과 0

    시간 복잡도

    나머지 연산 한 번만 실행 → O(1).

    공간 복잡도

    추가 메모리 사용 없음 → O(1).

    주의

    • num2가 0이 되는 경우는 문제에서 주어지지 않지만, 일반적으로 나머지 연산에서 0으로 나눌 수 없다는 점은 알고 있어야 한다.
    • 음수에 대한 나머지는 자바에서 몫의 방향에 따라 결정되지만, 이 문제에서는 양수만 들어오므로 고려할 필요 없다.

    문제 3 — 배열 두 배 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    배열 두 배 만들기 (프로그래머스 120809) 문제는
    정수 배열 numbers의 원소 각각을 2배로 만들어 새로운 배열로 반환하는 간단한 반복·수정 문제다.

    입력 배열의 각 요소에 직접 접근하여 numbers[i] *= 2 형태로 값을 2배로 변경한 뒤,
    변경된 배열을 그대로 반환하면 된다.

    • 핵심 아이디어: 배열을 순회하면서 각 요소에 2를 곱해 직접 갱신
    • 자료구조: 입력 배열을 그대로 사용 (in-place 변경)
    • 반복 방식: 인덱스 기반 for 루프로 모든 요소 변경

    예시

    • [1, 2, 3, 4] → [2, 4, 6, 8]
    • [5, 10] → [10, 20]
    • [0, 7] → [0, 14]

    시간 복잡도

    길이가 n인 배열을 한 번 순회하므로 O(n).

    공간 복잡도

    입력 배열을 그대로 수정하므로 추가 메모리 사용 없음 → O(1).

    주의

    • 원본 배열을 그대로 수정(in-place)한다는 점을 기억해야 한다.
    • 원래 배열을 보존하고 싶다면 new int[n]으로 복사 후 변경하는 방식도 가능하다.
