    문제 1 — 분수의 덧셈

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    분수의 덧셈 (프로그래머스 120808) 문제는
    두 분수 numer1/denom1, numer2/denom2를 더한 뒤,
    기약분수 형태로 나타내는 문제다.

    분수 덧셈의 기본 공식은 다음과 같다.

    a/b + c/d = (a·(공통분모/b) + c·(공통분모/d)) / 공통분모
    여기에서 공통분모는 일반적으로 두 분모의 최소공배수(LCM)를 사용하면 된다.

    이 풀이에서는:

    1. 두 분모의 최대공약수(GCD)를 구해 LCM 계산
    2. 그 LCM을 공통분모로 맞춰 두 분수를 더함
    3. 결과 분수의 분자/분모를 다시 GCD로 나눠 기약분수로 단순화
    • 공통분모: lcm = denom1 * denom2 / gcd(denom1, denom2)
    • 결과 분자: resultNumer = numer1 * (lcm / denom1) + numer2 * (lcm / denom2)
    • 결과 분모: resultDenom = lcm
    • 기약분수화:
      • g = gcd(resultNumer, resultDenom)
      • resultNumer / g, resultDenom / g로 나누어 반환

    최대공약수 gcd는 유클리드 호제법 재귀 구현으로 구한다.

    예시

    • numer1 = 1, denom1 = 2
      numer2 = 3, denom2 = 4

      1. gcd(2, 4) = 2
      2. lcm = 2 * 4 / 2 = 4
      3. resultNumer = 1 * (4 / 2) + 3 * (4 / 4)
        = 1 * 2 + 3 * 1 = 2 + 3 = 5
      4. resultDenom = 4
      5. g = gcd(5, 4) = 1 → 이미 기약분수
      6. 최종 결과: [5, 4] (= 5/4)
    • numer1 = 1, denom1 = 3
      numer2 = 1, denom2 = 6

      • gcd(3, 6) = 3, lcm = 3 * 6 / 3 = 6
      • resultNumer = 1 * (6/3) + 1 * (6/6) = 2 + 1 = 3
      • resultDenom = 6
      • g = gcd(3, 6) = 3 → 3/6 → 1/2
      • 최종 결과: [1, 2]

    시간 복잡도

    분자/분모 연산은 모두 상수 개의 정수 연산이다.
    유클리드 호제법 gcd(a, b)는 O(log min(a, b))에 종료하므로,
    전체 알고리즘은 사실상 O(log M) (M: 분모/분자 크기) 수준이다.

    공간 복잡도

    고정된 개수의 지역 변수와 결과 배열 int[2]만 사용하므로
    공간 복잡도는 O(1)이다.

    주의

    • 유클리드 호제법 구현:
      • gcd(a, b)는 b == 0이면 a를 반환하고, 아니면 gcd(b, a % b)로 재귀 호출한다.
    • 기약분수화:
      • 공통분모로 합친 뒤 바로 반환하면 안 되고, 반드시 분자/분모의 GCD로 나눠 기약분수로 만들고 반환해야 한다.
    • 곱셈 범위:
      • denom1 * denom2, numer × (lcm/denom) 연산이 있으므로 실제 테스트 조건에서 오버플로가 발생하지 않는지 확인해 두는 것이 좋다.

    문제 2 — 숫자 비교하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자 비교하기 (프로그래머스 120807) 문제는
    두 정수 num1과 num2가 같은지 비교하여
    같으면 1, 다르면 -1을 반환하는 매우 단순한 조건 분기 문제다.

    자바에서는 삼항 연산자(?:)를 이용해 조건에 따라 값을 바로 반환할 수 있으므로,
    num1 == num2 ? 1 : -1 형태로 간결하게 작성할 수 있다.

    • 핵심 아이디어: 두 값의 동등 비교(==) 결과로 반환 값을 선택
    • 사용 문법: 조건 연산자(삼항 연산자) condition ? A : B
    • 로직:
      • 두 값이 동일하면 1
      • 그렇지 않으면 -1

    예시

    • num1 = 10, num2 = 10 → 1
    • num1 = 3, num2 = 7 → -1
    • num1 = 0, num2 = 1 → -1
    • num1 = -5, num2 = -5 → 1

    시간 복잡도

    단순 비교 연산 1회이므로 O(1).

    공간 복잡도

    추가 메모리를 사용하지 않으므로 O(1).

    주의

    • 두 값이 정확히 같은지 비교하는 것이므로, 정수 비교에서는 ==를 그대로 사용해도 안전하다.
    • long 등 다른 정수형도 동일한 로직으로 처리된다.
    • 문제에서는 정확히 “같으면 1, 다르면 -1”을 반환해야 하므로, 0 등을 반환하면 오답이 된다.

    문제 3 — 두 수의 나눗셈

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 수의 나눗셈 (프로그래머스 120806) 문제는
    num1 / num2를 계산한 뒤, 그 결과에 1000을 곱한 값의 정수 부분을 반환하는 문제다.

    중요한 점은 num1 / num2를 정수 나눗셈이 아닌 실수 나눗셈으로 계산해야 한다는 것이다.
    따라서 double 타입으로 변환하여 나누기를 수행한 뒤, 최종적으로 (int)로 소수점을 버리고 정수 부분만 남긴다.

    • 핵심 아이디어: 정수 나눗셈이 아닌 부동소수점 나눗셈 수행 후 ×1000
    • 주의: (int)(num1 / num2)처럼 정수끼리 나누면 결과가 0 또는 다른 잘못된 값이 나온다.
    • 변환 방식: (double)num1, (double)num2 또는 변수에 바로 double 할당

    예시

    • num1 = 3, num2 = 2
      3 / 2 = 1.5 → 1.5 × 1000 = 1500 → 정수 부분 1500
    • num1 = 7, num2 = 3
      7 / 3 ≈ 2.333… → 2333.333… → 정수 부분 2333
    • num1 = 1, num2 = 16
      1 / 16 = 0.0625 → 62.5 → 정수 부분 62

    시간 복잡도

    나눗셈과 곱셈은 모두 상수 시간 연산이므로 O(1)이다.

    공간 복잡도

    단순한 변수만 사용하므로 공간 복잡도는 O(1)이다.

    주의

    • 정수 나눗셈 금지:
      • num1 / num2를 그대로 계산하면 정수 나눗셈이 되어 0 또는 잘못된 값이 된다.
    • 캐스팅 위치:
      • (int)(n1 / n2 * 1000)처럼 전체 결과를 변환해야 한다.
      • (int)(n1 / n2)처럼 먼저 버림이 일어나면 정확한 결과가 나오지 않는다.
    • 부동소수점 연산:
      • 문제 요구가 "소수점 버림"이므로 (int) 캐스팅이 정확히 맞다.
      • 반올림을 요구하는 문제가 아니다.
