    문제 1 — 몫 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    몫 구하기 (프로그래머스 120805) 문제는
    두 정수 num1, num2가 주어졌을 때,
    num1을 num2로 나눈 정수 나눗셈의 몫을 반환하는 문제다.

    자바에서 정수 타입 int끼리의 나눗셈 a / b는
    소수점 이하는 모두 버리고 몫만 남기는 나눗셈이다.

    따라서 이 문제는 별도의 복잡한 연산 없이
    한 줄로 return num1 / num2;를 수행하면 된다.

    이 풀이에서는:

    1. num1과 num2를 매개변수로 입력받는다.
    2. num1 / num2를 계산해 나온 몫을 그대로 반환한다.
    • 입력: 정수 num1, num2
    • 연산: num1 / num2 (정수 나눗셈)
    • 반환값: 나눗셈의 결과인 정수 몫

    자바의 정수 나눗셈은 0 방향으로 버림을 수행하므로,
    음수가 섞여 있어도 일관된 규칙에 따라 몫이 잘린다는 점만 기억하면 된다.

    예시

    • num1 = 10, num2 = 3

      1. 10 / 3 = 3.333... 이지만 정수 나눗셈이므로 소수점 이하는 버린다.
      2. 따라서 반환값은 3이다.
    • num1 = 7, num2 = 2

      • 7 / 2 = 3.5 → 정수 나눗셈 결과는 3
      • 나머지 1은 이 문제에서는 사용하지 않는다.

    시간 복잡도

    이 풀이는 한 번의 정수 나눗셈 연산만 수행한다.
    정수 나눗셈은 상수 시간 연산이므로,
    전체 알고리즘의 시간 복잡도는 O(1)이다.

    공간 복잡도

    반환값 외에 추가로 사용하는 메모리가 없으므로
    공간 복잡도 역시 O(1)이다.

    주의

    • 0으로 나누기:
      • 일반적으로 b == 0인 상태에서 a / b를 수행하면 런타임 에러(ArithmeticException)가 발생한다.
    • 정수 나눗셈의 성질:
      • 정수끼리 나누면 항상 소수점 이하는 버려진다. 필요하다면 (double) num1 / num2처럼 형 변환을 통해 실수 나눗셈을 사용해야 한다.
    • 실제 서비스 코드:
      • 문제에서는 보통 나누는 수가 0이 되지 않도록 제한하지만, 실무 코드에서는 나누기 전에 num2 == 0 체크를 해 두는 것이 안전하다.

    문제 2 — 두 수의 곱 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 수의 곱 (프로그래머스 120804) 문제는
    두 정수 num1, num2가 주어졌을 때,
    num1과 num2를 곱한 정수 곱을 반환하는 문제다.

    자바에서 정수 타입 int끼리의 곱셈 a * b는
    두 값을 그대로 곱한 뒤, 결과를 int 범위에서 표현한다.

    이 문제는 별도의 복잡한 연산 없이
    한 줄로 return num1 * num2;를 수행하면 해결된다.

    이 풀이에서는:

    1. num1과 num2를 매개변수로 입력받는다.
    2. num1 * num2를 계산해 나온 곱을 그대로 반환한다.
    • 입력: 정수 num1, num2
    • 연산: num1 * num2 (정수 곱셈)
    • 반환값: 곱셈의 결과인 정수 값

    정수 곱셈이기 때문에, 실수와 달리 소수 부분은 애초에 존재하지 않으며,
    단순히 두 정수의 곱을 구해 반환하면 된다.

    예시

    • num1 = 3, num2 = 4

      1. 3 * 4 = 12
      2. 따라서 반환값은 12이다.
    • num1 = -5, num2 = 6

      • -5 * 6 = -30
      • 음수와 양수를 곱하면 결과는 음수가 되며, 그대로 -30이 반환된다.

    시간 복잡도

    이 풀이는 한 번의 정수 곱셈 연산만 수행한다.
    정수 곱셈은 상수 시간 연산이므로,
    전체 알고리즘의 시간 복잡도는 O(1)이다.

    공간 복잡도

    반환값 외에 추가로 사용하는 메모리가 없으므로
    공간 복잡도 역시 O(1)이다.

    주의

    • 오버플로 가능성:
      • 정수 곱셈 결과가 int 범위를 넘어가면 오버플로가 발생해
        의도와 다른 값이 저장될 수 있다.
    • 부호 처리:
      • 양수 × 양수 = 양수, 음수 × 음수 = 양수, 양수 × 음수 = 음수처럼
        곱셈 결과의 부호 규칙을 기본적으로 알고 있으면 테스트 케이스를 검증하기 쉽다.
    • 실제 서비스 코드:
      • 문제에서는 입력 범위를 제한해 두는 경우가 많지만,
        실무에서는 필요에 따라 long 등의 더 넓은 정수형을 고려해야 한다.

    문제 3 — 두 수의 차 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 수의 차 (프로그래머스 120803) 문제는
    두 정수 num1, num2가 주어졌을 때,
    num1에서 num2를 뺀 정수 차이를 반환하는 문제다.

    자바에서 정수 타입 int끼리의 뺄셈 a - b는
    그대로 두 값을 빼서 int 범위의 결과를 만든다.

    이 문제는 별도의 복잡한 연산 없이
    한 줄로 return num1 - num2;를 수행하면 해결된다.

    이 풀이에서는:

    1. num1과 num2를 매개변수로 입력받는다.
    2. num1 - num2를 계산해 나온 차이를 그대로 반환한다.
    • 입력: 정수 num1, num2
    • 연산: num1 - num2 (정수 뺄셈)
    • 반환값: 뺄셈의 결과인 정수 값

    정수 뺄셈이기 때문에, 실수 연산이나 소수점 처리가 필요 없고,
    단순히 두 정수의 차를 구해 반환하면 된다.

    예시

    • num1 = 10, num2 = 3

      1. 10 - 3 = 7
      2. 따라서 반환값은 7이다.
    • num1 = 3, num2 = 10

      • 3 - 10 = -7
      • 작은 수에서 큰 수를 빼면 결과는 음수가 되며, 그대로 -7이 반환된다.

    시간 복잡도

    이 풀이는 한 번의 정수 뺄셈 연산만 수행한다.
    정수 뺄셈은 상수 시간 연산이므로,
    전체 알고리즘의 시간 복잡도는 O(1)이다.

    공간 복잡도

    반환값 외에 추가로 사용하는 메모리가 없으므로
    공간 복잡도 역시 O(1)이다.

    주의

    • 오버플로 가능성:
      • 정수 뺄셈 결과가 int 범위를 넘어가면 오버플로가 발생해
        의도와 다른 값이 저장될 수 있다.
    • 부호 처리:
      • 뺄셈 결과가 양수인지, 0인지, 음수인지에 따라 의미가 달라질 수 있으므로
        테스트 케이스에서 부호를 함께 확인해 두는 것이 좋다.
    • 실제 서비스 코드:
      • 입력 범위가 커질 수 있는 환경에서는 long 등의 더 넓은 정수형을
        상황에 따라 고려해야 한다.
