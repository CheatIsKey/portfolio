    문제 1 — 두 수의 합 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 수의 합 (프로그래머스 120802) 문제는
    두 정수 num1, num2가 주어졌을 때,
    num1과 num2를 더한 정수 합을 그대로 반환하는 문제다.

    자바에서 정수 타입 int끼리의 덧셈 a + b는
    두 값을 더한 결과를 int 범위에서 표현한다.

    이 문제는 별도의 복잡한 연산 없이
    한 줄로 return num1 + num2;를 수행하면 해결된다.

    이 풀이에서는:

    1. num1과 num2를 매개변수로 입력받는다.
    2. num1 + num2를 계산해 나온 합을 그대로 반환한다.
    • 입력: 정수 num1, num2
    • 연산: num1 + num2 (정수 덧셈)
    • 반환값: 덧셈의 결과인 정수 값

    정수 덧셈이기 때문에, 실수 연산이나 소수점 처리가 필요 없고,
    단순히 두 정수의 합을 구해 반환하면 된다.

    예시

    • num1 = 10, num2 = 3

      1. 10 + 3 = 13
      2. 따라서 반환값은 13이다.
    • num1 = -5, num2 = 6

      • -5 + 6 = 1
      • 음수와 양수를 더하면, 그 차이만큼 더 큰 쪽의 부호를 따라 결과가 결정된다.

    시간 복잡도

    이 풀이는 한 번의 정수 덧셈 연산만 수행한다.
    정수 덧셈은 상수 시간 연산이므로,
    전체 알고리즘의 시간 복잡도는 O(1)이다.

    공간 복잡도

    반환값 외에 추가로 사용하는 메모리가 없으므로
    공간 복잡도 역시 O(1)이다.

    주의

    • 오버플로 가능성:
      • 정수 덧셈 결과가 int 범위를 넘어가면 오버플로가 발생해
        의도와 다른 값이 저장될 수 있다.
    • 부호 처리:
      • 양수/음수가 섞인 경우, 두 수의 크기 차이에 따라 결과가 양수/0/음수가 될 수 있으므로
        테스트 케이스에서 부호를 함께 확인해 보는 것이 좋다.
    • 실제 서비스 코드:
      • 입력 범위가 매우 크거나 누적합을 오래 유지해야 하는 경우에는
        long 같은 더 넓은 정수형 사용을 고려해야 한다.

    문제 2 — 머쓱이보다 키 큰 사람

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    머쓱이보다 키 큰 사람 (프로그래머스 120585) 문제는
    정수 배열 array에 여러 사람의 키가 주어졌을 때,
    기준 키 height보다 엄Strict하게 큰 키를 가진 사람이 몇 명인지 세어 반환하는 문제다.

    조건은 매우 단순하다.

    • array의 각 원소는 한 사람의 키를 나타낸다.
    • height보다 큰 값만 카운트해야 하며,
      height와 같으면 세지 않는다.

    따라서, 배열을 한 번 돌면서
    i > height인 원소의 개수만 세어 주면 된다.

    이 풀이에서는:

    1. 카운트를 저장할 변수 cnt를 0으로 초기화한다.
    2. for (int i : array) 향상된 for 문으로 배열을 순회한다.
    3. 각 원소 i에 대해 i > height이면 cnt++로 1 증가시킨다.
    4. 반복이 끝나면 cnt를 반환한다.
    • 입력: 정수 배열 array, 기준 키 height
    • 연산: 각 원소에 대해 조건 i > height를 검사
    • 반환값: 기준 키보다 큰 사람의 수 cnt

    예시

    • array = [149, 180, 192, 170], height = 167

      1. 149 > 167 → 거짓, 세지 않음
      2. 180 > 167 → 참, cnt = 1
      3. 192 > 167 → 참, cnt = 2
      4. 170 > 167 → 참, cnt = 3
      5. 최종 반환값: 3
    • array = [155, 160, 165], height = 170

      • 모든 원소가 170 이하 → 조건을 만족하는 사람이 없다.
      • 최종 반환값: 0

    시간 복잡도

    이 풀이는 배열 array를 한 번 끝까지 순회한다.
    배열의 길이를 n이라고 할 때, 전체 알고리즘의 시간 복잡도는 O(n)이다.

    공간 복잡도

    추가로 사용하는 변수는 정수형 cnt 하나뿐이다.
    따라서 공간 복잡도는 O(1)이다.

    주의

    • 비교 연산자:
      • 문제에서 요구하는 것은 height보다 큰 사람의 수이므로,
        반드시 >를 사용해야 한다.
      • >=를 사용하면 키가 같은 사람도 포함되어 오답이 된다.
    • 향상된 for 문 사용:
      • for (int i : array)는 인덱스를 직접 다루지 않고
        원소를 바로 순회할 수 있어 코드가 더 간결해진다.
    • 배열이 비어 있는 경우:
      • array 길이가 0이면 반복문이 실행되지 않고,
        초기값 cnt = 0이 그대로 반환된다.

    문제 3 — 중복된 숫자 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    중복된 숫자 개수 (프로그래머스 120583) 문제는
    정수 배열 array 안에 정수 n이 몇 번 등장하는지를 세어
    그 개수를 반환하는 문제다.

    즉, 배열에서 == n인 원소의 개수를 세는
    단순한 빈도(count) 계산 문제다.

    여기서는 자바의 Stream API를 활용해서
    코드를 한 줄로 간결하게 작성하고 있다.

    이 풀이에서는:

    1. Arrays.stream(array)로 int[]를 IntStream으로 변환한다.
    2. filter(i -> i == n)으로 값이 n인 원소만 남긴다.
    3. count()로 필터를 통과한 원소의 개수를 long 타입으로 얻는다.
    4. 문제 요구가 int이므로 (int)로 형 변환하여 반환한다.
    • 입력: 정수 배열 array, 정수 n
    • 연산: array의 원소 중 i == n인 것의 개수 계산
    • 반환값: n이 등장하는 횟수

    예시

    • array = [1, 1, 2, 3, 4, 1], n = 1

      1. 배열에서 1인 원소는 index 0, 1, 5 → 총 3개
      2. 따라서 반환값은 3
    • array = [0, 2, 3, 4], n = 1

      • 1은 한 번도 등장하지 않는다.
      • 따라서 반환값은 0

    시간 복잡도

    스트림으로 변환한 뒤, filter와 count는 내부적으로
    배열을 처음부터 끝까지 한 번 순회한다.
    배열 길이를 n이라고 하면 전체 알고리즘의 시간 복잡도는
    O(n)이다.

    공간 복잡도

    Stream 연산 자체는 입력 배열 외에
    개수를 세기 위한 고정 크기의 추가 메모리만 사용한다.
    따라서 공간 복잡도는 O(1)로 본다.

    주의

    • count()의 반환 타입:
      • count()는 long을 반환하므로,
        문제에서 요구하는 int로 사용하기 위해 (int) 캐스팅이 필요하다.
      • 일반적인 코딩 테스트 문제에서 배열 길이는 int 범위를 넘지 않도록 주어지므로,
        이 캐스팅은 안전하게 사용 가능하다.
    • 동등 비교:
      • 문제는 “같은 숫자”의 개수를 묻는 것이므로 == 비교가 맞다.
      • 크다/작다 비교(>, <)와 혼동하지 않도록 주의한다.
    • Stream vs 반복문:
      • 기본적인 풀이로는 단순 for문으로 카운트해도 되지만,
        여기서는 Stream을 이용해 표현을 간결하게 만든 형태다.
