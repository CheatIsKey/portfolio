    문제 1 — 달리기 경주

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    달리기 경주 (프로그래머스 178871) 문제는
    선수들의 초기 순서가 담긴 배열 players와,
    경기 도중 불리며 한 칸씩 추월하는 선수 이름들이 담긴 배열 callings가 주어졌을 때,
    모든 호출이 끝난 뒤의 최종 등수 순서를 구하는 문제다.

    규칙은 간단하다.

    • players는 현재 등수 순서(0번 인덱스가 1등)를 나타낸다.
    • callings의 각 이름 s에 대해:
      • 선수 s는 자신의 바로 앞 선수와 자리를 맞바꾼다.
      • 즉, 한 번 불릴 때마다 한 칸씩 앞당겨진다.

    단순하게 매 호출마다 players 배열에서 이름을 찾으면
    O(N)이 걸리므로 전체는 O(N × M)이 되어 느리다.
    이 풀이에서는 이름 → 인덱스를 빠르게 찾기 위해 HashMap을 함께 사용해
    각 호출을 O(1)에 처리한다.

    이 풀이에서는:

    1. players 배열: 현재 등수 순서를 그대로 저장.
    2. rank 맵: 선수 이름을 키, 현재 인덱스를 값으로 저장.
    3. callings를 순회하면서:
      • 해당 선수의 현재 인덱스를 맵에서 찾고,
      • 앞에 있는 선수와 배열에서 자리를 바꾼 뒤,
      • 두 선수의 인덱스를 rank 맵에서 동시에 갱신.
    4. 모든 호출이 끝난 뒤 players 배열을 그대로 반환.
    • 입력: String[] players, String[] callings
    • 출력: 모든 추월이 반영된 최종 등수 배열

    핵심 로직

    • 초기 맵 구성
      • 각 선수 이름과 인덱스를 매핑한다. 
        Map<String, Integer> rank = new HashMap<>();
      for (int i = 0; i < players.length; i++) rank.put(players[i], i);
      • 이제 rank.get("선수")로 현재 등수를 O(1)에 조회할 수 있다.
    • 호출 처리 및 자리 교환
      • callings의 각 이름 s에 대해: 
        for (String s : callings) {
          int idx = rank.get(s);
      
          if (idx > 0) {
              int prev = idx - 1;
              String prevPlayer = players[prev];
              players[prev] = players[idx];
              players[idx] = prevPlayer;
      
              rank.put(s, prev);
              rank.put(prevPlayer, idx);
          }
      }
      • idx: 선수 s의 현재 인덱스(등수).
      • prev = idx - 1: 바로 앞 선수의 인덱스.
      • players[prev]와 players[idx]를 서로 교환 → 실제 순서 변경.
      • 그 후:
        • s의 인덱스를 prev로 갱신.
        • 추월당한 선수(prevPlayer)의 인덱스를 idx로 갱신.
      • idx > 0 조건은 맨 앞(1등)인 경우를 보호하기 위한 안전 장치다.

    모든 호출을 처리한 뒤에는 players 배열이 최종 순서를 나타내므로
    그대로 반환하면 된다.

    예시 (개념)

    • 초기:
      • players = ["mumu", "soe", "poe", "kai", "mine"]
      • 등수: mumu(0), soe(1), poe(2), kai(3), mine(4)
    • callings = ["kai", "kai", "mine", ...]라고 해 보자.

      1. 첫 번째 "kai"
        • rank.get("kai") = 3, prev = 2
        • players[2] = "poe", players[3] = "kai" → 둘이 자리 교환
        • 결과: ["mumu", "soe", "kai", "poe", "mine"]
      2. 두 번째 "kai"
        • 이제 rank.get("kai") = 2, prev = 1
        • players[1] = "soe", players[2] = "kai" → 둘이 자리 교환
        • 결과: ["mumu", "kai", "soe", "poe", "mine"]
      3. "mine"
        • rank.get("mine") = 4, prev = 3
        • players[3] = "poe", players[4] = "mine" → 둘이 자리 교환
        • 결과: ["mumu", "kai", "soe", "mine", "poe"]

    시간 복잡도

    초기 맵 구성:

    • players 길이를 N이라고 하면,
      한 번의 루프에서 rank.put을 N번 호출 → O(N).

    호출 처리:

    • callings 길이를 M이라고 하면,
      각 호출마다 rank.get, 배열 스왑, rank.put 2회가 모두 상수 시간 →
      전체가 O(M).

    따라서 전체 알고리즘의 시간 복잡도는 O(N + M)이다.
    호출 수가 매우 많아도 효율적으로 처리할 수 있다.

    공간 복잡도

    추가로 사용하는 주요 메모리는:

    • 이름 → 인덱스를 저장하는 rank 맵: 선수 수 N만큼 저장
    • 몇 개의 보조 변수(idx, prev, prevPlayer 등)

    따라서 공간 복잡도는 O(N)이다.

    주의

    • 선수 이름의 유일성:
      • 맵에서 이름을 키로 사용하므로, 선수 이름은 서로 다르다고 가정한다.
    • 인덱스 갱신:
      • 배열에서 자리를 바꾸고 나면, rank 맵에 저장된 인덱스도
        반드시 함께 갱신해야 한다.
      • 둘 중 하나라도 갱신이 누락되면 이후 호출에서 잘못된 위치로 참조하게 된다.
    • 경계 조건:
      • idx > 0 체크 없이 0번 인덱스에서 앞 사람과 교환을 시도하면
        인덱스가 음수가 되어 예외가 발생한다.
      • 현재 구현처럼 idx > 0일 때만 스왑하도록 하는 것이 안전하다.

    문제 2 — 추억 점수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    추억 점수 (프로그래머스 176963) 문제는
    사람 이름 배열 name과 그 사람에 대한 그리움 점수 yearning,
    그리고 여러 장의 사진에 찍힌 사람들 목록 photo가 주어졌을 때,
    각 사진별 추억 점수의 합을 구해 배열로 반환하는 문제다.

    규칙은 다음과 같다.

    • name[i] 사람의 그리움 점수는 yearning[i]이다.
    • photo[j]는 j번째 사진에 등장한 사람 이름 배열이다.
    • 사진 속 이름 중 name에 없는 이름은 점수 0점으로 본다.
    • 각 사진에 등장한 사람들의 그리움 점수를 모두 더한 값이 그 사진의 추억 점수다.

    이 풀이에서는 이름으로 점수를 빠르게 찾기 위해
    HashMap을 사용한다.

    전체 흐름은 다음과 같다.

    1. name과 yearning를 이용해
      이름 → 그리움 점수를 담은 맵을 만든다.
    2. photo의 각 사진에 대해:
      • 사진 속 이름들을 돌면서, 맵에서 점수를 꺼내 합산한다.
      • 맵에 없는 이름이면 0점으로 더한다.
      • 계산한 합을 결과 배열에 저장한다.
    3. 모든 사진에 대해 계산이 끝나면 결과 배열을 반환한다.
    • 입력: String[] name, int[] yearning, String[][] photo
    • 출력: 각 사진의 추억 점수를 담은 int[]

    핵심 로직

    • 이름 → 점수 맵 만들기
      • 동일한 인덱스를 사용하는 두 배열을 한 번에 순회하면서 맵에 넣는다. 
        Map<String, Integer> map = new HashMap<>();
      for (int i = 0; i < name.length; i++) map.put(name[i], yearning[i]);
      • 이제 map.get("이름")으로 점수를 빠르게 찾을 수 있다.
    • 각 사진의 점수 계산
      • 결과를 담을 배열 points를 사진 개수만큼 만든다. 
        int[] points = new int[photo.length];
      int idx = 0;
      • 각 사진(문자열 배열)을 순회하면서 합을 구한다. 
        for (String[] str : photo) {
          int sum = 0;
      
          for (int i = 0; i < str.length; i++) {
              sum += map.getOrDefault(str[i], 0);
          }
          points[idx++] = sum;
      }
      • getOrDefault(str[i], 0)를 사용해,
        맵에 없는 이름이면 자동으로 0점을 더하도록 처리한다.
      • 사진 하나에 대한 합 sum을 points의 다음 위치에 저장한다.
    • 결과 반환
      • 모든 사진을 처리한 후 points 배열을 반환한다. 
        return points;

    예시 (개념)

    • name = ["may", "kein", "kain", "radi"]
      yearning = [5, 10, 1, 3]
      photo[0] = ["may", "kein", "kain"] 이라고 하면,

      1. "may" → 5점
      2. "kein" → 10점
      3. "kain" → 1점
      4. 합계: 5 + 10 + 1 = 16
      따라서 points[0] = 16이 된다.
    • 만약 사진에 "may", "abc"처럼
      "abc"가 name에 없다면,
      • "may" → 5점
      • "abc" → 맵에 없음 → 0점
      • 합계: 5

    시간 복잡도

    맵 구성:

    • 이름 개수를 N이라고 하면, 한 번의 루프 → O(N).

    사진 점수 계산:

    • 사진 개수를 P, 각 사진에 평균 K명이 찍혀 있다고 하면,
      내부 이중 반복의 총 연산 수는 P × K 정도다.
    • 각 이름에 대해 map.getOrDefault 호출은 O(1)이므로,
      전체는 O(P × K).

    전체 시간 복잡도는 O(N + P × K)로 볼 수 있고,
    입력 크기 제약 안에서는 충분히 빠르게 동작한다.

    공간 복잡도

    추가로 사용하는 주요 메모리는:

    • 이름 → 점수를 저장하는 맵 map (크기 N)
    • 사진별 점수를 저장하는 결과 배열 points (크기 P)

    따라서 공간 복잡도는 O(N + P)이다.

    주의

    • getOrDefault 활용:
      • map.getOrDefault(str[i], 0)를 사용하면,
        이름이 맵에 없을 때 null 체크 없이 바로 0점을 처리할 수 있다.
    • 동일 인덱스 매핑:
      • name[i]와 yearning[i]가 짝을 이루므로,
        반드시 같은 인덱스를 기준으로 맵에 넣어야 한다.
    • 사진이 비어 있는 경우:
      • 어떤 사진 배열이 길이 0이라면 내부 반복이 실행되지 않아
        해당 사진의 점수는 자연스럽게 0으로 남는다.
