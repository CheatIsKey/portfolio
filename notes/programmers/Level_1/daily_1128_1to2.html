    문제 1 — 덧칠하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    덧칠하기 (프로그래머스 161989) 문제는
    1번부터 n번까지 연속된 구역이 있을 때,
    롤러의 길이 m과 반드시 다시 칠해야 하는 구역들의 번호 배열 section이 주어졌을 때,
    롤러질을 최소 몇 번 해야 모든 필요한 구역을 다시 칠할 수 있는지를 구하는 그리디 문제다.

    전제는 다음과 같다.

    • 한 번 롤러질을 하면, 연속된 m개의 구역을 한 번에 칠할 수 있다.
      • 예: m = 4일 때, i번에서 시작하면 i, i+1, i+2, i+3까지 칠해진다.
    • section 배열에 있는 인덱스들은 반드시 다시 칠해야 하는 구역 번호다.
    • 이미 칠해져 있는 구역을 다시 칠해도 상관없지만, 최소 횟수를 원한다.

    핵심 아이디어는:

    • 왼쪽에서 오른쪽으로 다시 칠해야 하는 구역을 순서대로 보며,
    • 아직 칠해지지 않은 가장 왼쪽 구역에서 롤러를 시작해, 그 롤러로 갈 수 있는 가장 오른쪽까지 한 번에 칠해버리는 방식으로 진행하는 것.

    전략 개요

    • cnt: 롤러질을 몇 번 했는지 세는 변수
    • painted: 지금까지 롤러질을 해서 칠해진 가장 오른쪽 구역 번호

    section 배열을 순회하면서:

    • 현재 구역 번호를 idx라고 할 때,
      • 만약 idx <= painted라면 → 이미 이전 롤러질로 칠해진 범위에 포함되므로 건너뛴다.
      • idx > painted라면 → 아직 칠해지지 않은 구역이므로,
        • 여기서 새로 롤러질을 1번 하고,
        • 이 롤러로 idx부터 idx + m - 1까지 칠해지므로
          painted = idx + m - 1로 갱신한다.

    이렇게 하면 항상 “아직 칠하지 않은 가장 왼쪽 구역”에서 롤러질을 시작하므로,
    불필요한 중복 롤러질을 최소화할 수 있다.

    핵심 로직

    • 변수 초기화 
      public int solution(int n, int m, int[] section) {
          int cnt = 0, painted = 0;
      • cnt = 0 → 아직 롤러질을 한 번도 하지 않았다.
      • painted = 0 → 처음에는 칠해진 구역이 없다고 보고, 1번 구역도 칠해지지 않은 상태다.
    • 다시 칠해야 하는 구역 순회 
          for (int idx : section) {
              if (idx <= painted) continue;
      
              cnt++;
              painted = idx + m - 1;
          }
      • for (int idx : section):
        • section에 들어 있는 구역 번호들을 하나씩 꺼낸다.
      • if (idx <= painted) continue;
        • 현재 idx가 이미 painted 이하라면,
          이전에 했던 롤러질의 범위 안에 포함되어 있으므로 그대로 넘어간다.
      • 아직 칠해지지 않은 구역인 경우:
        • cnt++; → 롤러질 한 번 추가.
        • painted = idx + m - 1; → 이번에 시작한 롤러질로 칠해지는 가장 오른쪽 구역 번호.
    • 결과 반환 
          return cnt;
      }
      • cnt에는 모든 section을 커버하기 위해 필요한 최소 롤러질 횟수가 담겨 있다.

    시간 복잡도

    section 배열의 길이를 k라고 하면:

    • 배열을 한 번 순회하면서 각 원소에 대해 상수 시간 연산만 수행하므로
      전체 시간 복잡도는 O(k)이다.

    n 전체 구역을 직접 순회하지 않고,
    실제로 다시 칠해야 하는 구역만 보기 때문에 효율적이다.

    공간 복잡도

    입력으로 주어진 section 배열 외에는

    • 정수 변수 cnt, painted, idx 정도만 사용하므로

    추가 공간 복잡도는 O(1)이다.

    주의

    • section이 정렬되어 있다고 가정
      • 문제에서 section이 오름차순으로 주어진다고 보장되기 때문에,
        왼쪽에서 오른쪽으로 차례대로 처리하는 그리디가 유효하다.
      • 만약 정렬이 안 되어 있다면, 먼저 정렬을 해줘야 이 로직이 올바르게 동작한다.
    • painted의 의미
      • painted는 지금까지의 롤러질로 칠해진 “가장 오른쪽 구역 번호”를 의미한다.
      • 그래서 idx <= painted인 구역은 이미 칠해진 영역 안에 들어간다.
    • 인덱스 기준
      • 문제에서 구역 번호는 1부터 시작하고, 코드에서도 그대로 1 기반 번호를 사용한다.
      • 따라서 idx + m - 1로 오른쪽 끝을 계산할 때 인덱스 혼동이 없다.

    포인트 요약

    • 다시 칠해야 하는 구역들을 왼쪽에서 오른쪽 순서대로 보면서 처리한다.
    • painted로 지금까지 칠해진 가장 오른쪽 구역 번호를 관리한다.
    • 이미 칠해진 범위 안에 있는 구역은 무시하고, 범위 밖의 첫 구역에서만 새로 롤러질을 한다.
    • 새 롤러질이 시작되면, 해당 위치부터 m칸 만큼 범위를 한 번에 커버하도록 painted를 갱신한다.
    • 이 과정을 반복하면, 필요한 롤러질 횟수의 최소값을 O(k)에 구할 수 있다.

    문제 2 — 대충 만든 자판

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    대충 만든 자판 (프로그래머스 160586) 문제는
    여러 개의 자판 문자열 keymap이 주어졌을 때,
    각 알파벳을 누르기 위해 필요한 최소 키 입력 횟수를 미리 계산해 두고,
    주어진 문자열들 targets를 입력하는 데 드는 총 키 입력 횟수를 구하는 문제다.

    규칙은 다음과 같다.

    • keymap의 각 문자열은 하나의 자판 줄(row)을 의미한다.
    • 문자열에서 인덱스 i에 있는 문자를 입력하려면, 그 키를 i + 1번 눌러야 한다.
    • 같은 알파벳이 여러 자판 줄에 여러 위치에 있을 수 있고,
      이때는 그 알파벳을 누르기 위한 최소 키 입력 횟수를 사용한다.
    • 어떤 알파벳이 어떤 자판에도 존재하지 않으면, 그 알파벳이 들어간 문자열은 입력 불가이므로 결과는 -1이다.

    따라서 전체 로직은:

    1. 먼저 A~Z까지 각 알파벳을 누르기 위한 최소 횟수를 전처리로 계산.
    2. 그 다음, 각 target 문자열의 문자들을 돌며 위에서 계산한 최소 횟수를 더한다.
    3. 중간에 하나라도 입력 불가능한 알파벳이 나오면 해당 문자열의 결과는 -1로 처리.

    전략 개요

    • 전처리 단계 – 알파벳별 최소 입력 횟수 계산
      • 크기 26인 배열 keySet을 만든다. (A~Z)
      • 초기값은 모두 Integer.MAX_VALUE로 설정해서 “아직 입력 불가능”을 의미하게 한다.
      • 각 keymap 문자열을 돌면서, 해당 위치의 문자에 대해
        최소 키 입력 횟수를 갱신한다.
    • 질의 단계 – 각 target 문자열 처리
      • 각 target마다 문자들을 순회하면서 keySet에서 해당 알파벳의 키 입력 횟수를 가져와 모두 더한다.
      • 만약 어떤 문자에 대해 keySet[idx] == Integer.MAX_VALUE라면,
        그 문자는 어느 자판에서도 입력할 수 없다는 뜻이므로 결과는 -1로 처리하고 중단한다.

    핵심 로직

    • 알파벳별 최소 입력 횟수 전처리 
      int[] keySet = new int[26];
      Arrays.fill(keySet, Integer.MAX_VALUE);
      
      for (String key : keymap) {
          for (int i = 0; i < key.length(); i++) {
              char c = key.charAt(i);
              int idx = c - 'A';
              int press = i + 1;
      
              keySet[idx] = Math.min(keySet[idx], press);
          }
      }
      • keySet[0] → 'A', keySet[1] → 'B', ... , keySet[25] → 'Z'.
      • 각 자판 줄에서:
        • 문자 위치 i에서 그 문자를 누르려면 i + 1번 누른다.
        • 같은 문자에 대해 여러 번 등장할 수 있으므로 Math.min으로 최소값만 유지한다.
      • 전처리가 끝나면, keySet[ch - 'A']는 문자 ch를 입력하기 위한 최소 키 입력 횟수가 된다.
        • 만약 여전히 Integer.MAX_VALUE라면, 그 문자는 어떤 자판에도 없다.
    • 각 target 문자열의 입력 횟수 계산 
      int[] sum = new int[targets.length];
      
      for (int i = 0; i < targets.length; i++) {
          String target = targets[i];
          int tmp = 0;
          boolean ok = true;
      
          for (int j = 0; j < target.length(); j++) {
              char c = target.charAt(j);
              int idx = c - 'A';
      
              if (keySet[idx] == Integer.MAX_VALUE) {
                  ok = false;
                  break;
              }
      
              tmp += keySet[idx];
          }
      
          sum[i] = ok ? tmp : -1;
      }
      • tmp: 현재 target 문자열을 입력하는 데 필요한 총 키 입력 횟수.
      • ok: 이 문자열을 끝까지 입력할 수 있는지 여부.
        • 중간에 하나라도 입력 불가능한 문자가 나오면 ok = false로 두고 루프 탈출.
      • 모든 문자를 정상적으로 처리했다면 sum[i] = tmp,
        아니면 sum[i] = -1로 저장한다.
    • 최종 반환 
      return sum;
      • sum 배열에는 각 targets[i]를 입력하는 데 필요한 키 입력 횟수 또는 -1가 담겨 있다.

    시간 복잡도

    다음과 같이 볼 수 있다.

    • 전처리:
      • keymap의 모든 문자열 길이의 합을 L이라고 하면,
        전처리는 각 문자를 한 번씩만 보므로 O(L).
    • 각 target 처리:
      • targets의 모든 문자열 길이 합을 T라고 하면,
        각 문자를 한 번씩만 보므로 O(T).

    전체 시간 복잡도는 O(L + T)이며,
    단순한 배열 접근만 사용하므로 충분히 빠르게 동작한다.

    공간 복잡도

    • keySet[26], sum[targets.length] 정도만 추가로 사용한다.
    • 알파벳 배열은 상수 크기이므로, 전체 추가 공간은 O(1)에 가깝고,
      sum 배열까지 포함해도 O(#targets) 정도다.

    주의

    • 문자 인덱싱:
      • 문자들은 모두 대문자라고 가정하고 c - 'A'로 인덱스를 만든다.
      • 만약 소문자가 섞일 수 있는 상황이라면, Character.toUpperCase 같은 처리가 필요했겠지만,
        이 문제에서는 대문자만 등장한다.
    • 입력 불가능 문자 처리:
      • keySet[idx] == Integer.MAX_VALUE인 문자를 만나면,
        그 문자열 전체를 -1로 처리해야 한다.
      • 중간까지 더한 tmp는 버리고, 즉시 루프를 빠져나오는 것이 핵심.
    • 중복 문자에 대한 최소값 유지:
      • 같은 알파벳이 여러 자판 줄에 있을 수 있으므로,
        항상 Math.min(keySet[idx], press)로 최소값만 유지해야 한다.

    포인트 요약

    • 먼저 A~Z에 대해 각 알파벳을 누르는 최소 횟수를 keySet에 전처리로 저장한다.
    • 각 target은 문자별로 keySet을 참고해서 횟수를 더해가며, 하나라도 입력 불가면 결과를 -1로 한다.
    • 전처리 + 한 번의 순회로 해결 가능한 전형적인 전처리 + 질의 형태의 문제다.
    • 배열 인덱싱과 초기값(Integer.MAX_VALUE) 처리만 주의하면 구현이 간단하다.
