    문제 1 — 공원 산책

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    공원 산책 (프로그래머스 172928) 문제는
    문자열 배열 park로 주어진 공원 지도를 기준으로,
    시작 위치 'S'에서 여러 이동 명령 routes를 순서대로 수행한 뒤의
    최종 좌표를 구하는 시뮬레이션 문제다.

    공원 지도는 다음과 같이 구성된다.

    • 'S': 시작 위치 (단 하나)
    • 'O': 빈 칸 (이동 가능)
    • 'X': 장애물 (통과 불가)

    이동 명령 routes는 "방향 거리" 형태의 문자열이다.

    • 방향: "E"(동), "W"(서), "S"(남), "N"(북)
    • 거리: 해당 방향으로 이동하려는 칸 수

    한 명령을 처리할 때는 해당 거리만큼의 경로 전체가 유효해야만 실제로 이동한다.

    • 이동 도중에 공원 밖으로 나가거나, 'X'를 만나면 그 명령 전체를 무시하고 제자리에 머문다.
    • 도중에 문제 없으면, 해당 거리만큼 이동한 위치로 현재 좌표를 갱신한다.

    전략 개요

    • 공원 크기 파악
      • R = park.length → 행(세로) 크기
      • C = park[0].length() → 열(가로) 크기
    • 시작 위치 찾기
      • 이중 반복으로 'S'가 있는 위치를 찾아 (sr, sc)에 저장한다.
      • 초기 현재 위치 (curR, curC)를 (sr, sc)로 설정한다.
    • 각 이동 명령 처리
      • "방향 거리" 문자열을 공백 기준으로 분리한다.
        • parts[0] → 방향 문자열 dir
        • parts[1] → 정수 거리 dist
      • 방향에 따라 행/열 증가량 (dr, dc)를 정한다.
        • "E" → (dr, dc) = (0, 1)
        • "W" → (0, -1)
        • "S" → (1, 0)
        • "N" → (-1, 0)
      • 이동을 시도하기 전에, 현재 위치에서 한 칸씩 이동해 보며 경로가 유효한지 검사한다.

    핵심 로직

    • 시작 위치 탐색 
      int R = park.length;
      int C = park[0].length();
      int sr = 0, sc = 0;
      
      for (int r = 0; r < R; r++) {
          for (int c = 0; c < C; c++) {
              if (park[r].charAt(c) == 'S') {
                  sr = r;
                  sc = c;
              }
          }
      }
      
      int curR = sr;
      int curC = sc;
    • 명령별 이동 검증 
      for (String route : routes) {
          String[] parts = route.split(" ");
          String dir = parts[0];
          int dist = Integer.parseInt(parts[1]);
      
          int dr = 0, dc = 0;
          switch (dir) {
              case "E": dc = 1; break;
              case "W": dc = -1; break;
              case "S": dr = 1; break;
              case "N": dr = -1; break;
          }
      
          boolean ok = true;
          int nr = curR;
          int nc = curC;
      
          // dist 칸 이동이 가능한지 한 칸씩 미리 확인
          for (int i = 0; i < dist; i++) {
              nr += dr;
              nc += dc;
      
              // 1) 공원 경계 체크
              if (nr < 0 || nr >= R || nc < 0 || nc >= C) {
                  ok = false;
                  break;
              }
      
              // 2) 장애물 체크
              if (park[nr].charAt(nc) == 'X') {
                  ok = false;
                  break;
              }
          }
      
          // 경로 전체가 유효한 경우에만 실제 위치 갱신
          if (ok) {
              curR = nr;
              curC = nc;
          }
      }
    • 최종 위치 반환 
      return new int[]{curR, curC};

    이때 curR, curC는 0부터 시작하는 행/열 인덱스다.
    문제에서도 0 기반 좌표로 답을 요구하므로 그대로 반환하면 된다.

    예시 (개념)

    • 예를 들어, 
      park = ["SOO",
              "OXX",
              "OOO"]
      시작 위치는 (0, 0)이다.
    • routes = ["E 2", "S 2", "W 1"]라고 하면:
      1. "E 2"
        • (0, 0) → (0, 1) → (0, 2) 이동 경로에 공원 밖이나 'X' 없음
        • 명령 전체 유효 → 최종 위치 (0, 2)
      2. "S 2"
        • (0, 2) → (1, 2) → (2, 2) 순서로 가야 하는데,
          (1, 2)가 'X'라면, 이 명령은 전체 무효
        • 위치 그대로 (0, 2)에 남음
      3. "W 1"
        • (0, 2) → (0, 1) 이동 가능
        • 최종 위치는 (0, 1)

    시간 복잡도

    공원 크기를 R × C, 이동 명령 개수를 K,
    한 명령에서 이동하는 최대 거리를 D라고 하면:

    • 시작 위치 탐색: 공원 전체 한 번 순회 → O(R × C)
    • 명령 처리: 각 명령마다 최대 D칸 미리 검사 → O(K × D)

    전체 시간 복잡도는 대략
    O(R × C + K × D) 수준이다.
    문제 조건에서 R, C, K, D 모두 크지 않으므로 충분히 빠르게 동작한다.

    공간 복잡도

    입력으로 주어진 park, routes 외에
    현재 위치 좌표, 방향 벡터, 임시 변수들만 사용하므로
    추가 공간 사용은 O(1)이다.

    주의

    • 명령 단위 이동:
      • 한 칸씩 이동하면서 검사하지만,
        중간에 한 번이라도 범위를 벗어나거나 'X'를 만나면
        그 명령 전체를 무시하고 원래 위치를 유지해야 한다.
    • 시작 위치 여러 번 갱신 방지:
      • 문제에서 'S'는 한 번만 등장이라고 가정하지만,
        코드처럼 전체를 순회해도 문제는 없다.
    • 좌표 체계:
      • 행 인덱스는 위에서 아래로 증가, 열 인덱스는 왼쪽에서 오른쪽으로 증가한다.
      • "N"은 행 감소(dr = -1), "S"는 행 증가(dr = 1)라는 점을 헷갈리지 않아야 한다.
    • 경계 조건:
      • nr < 0, nr >= R, nc < 0, nc >= C 중 하나라도 만족하면 공원 밖이다.

    문제 2 — 바탕화면 정리

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    바탕화면 정리 (프로그래머스 161990) 문제는
    문자열 배열 wallpaper 상에서 파일이 놓인 칸('#')들을 모두 포함하는
    가장 작은 직사각형을 찾아, 그 직사각형을 나타내는 좌표 4개를 반환하는 문제다.

    규칙은 다음과 같다.

    • wallpaper[r].charAt(c)가 '#'이면 해당 위치에 파일이 있다.
    • 드래그 범위는 위쪽, 왼쪽은 포함하고 아래쪽, 오른쪽은 바로 한 칸 뒤까지를 의미한다.
      • 즉, 좌표는 [lux, luy, rdx, rdy] 형태로
        • (lux, luy): 가장 위, 가장 왼쪽 파일의 위치를 포함하는 좌상단 모서리
        • (rdx, rdy): 가장 아래, 가장 오른쪽 파일의 다음 칸을 나타내는 우하단 모서리

    따라서 모든 '#'의 좌표를 훑으면서 행/열의 최소값과 최대값을 잘 관리해주면 된다.

    전략 개요

    • 전체 격자를 한 번 순회하면서:
      • '#'를 만나면,
        • 행 인덱스 r, 열 인덱스 c를 기준으로
        • 현재까지의 최소 행(lux), 최소 열(luy)을 갱신
        • 최대 행(rdx), 최대 열(rdy)도 갱신하되, 아래/오른쪽 경계를 위해 항상 +1을 더해 저장
    • 최종적으로 [lux, luy, rdx, rdy]를 그대로 반환하면,
      모든 파일을 덮는 가장 작은 직사각형 영역이 된다.

    핵심 로직

    • 초기값 설정
      • 최솟값은 가능한 한 큰 값에서 시작, 최댓값은 가능한 한 작은 값에서 시작한다. 
        int lux = Integer.MAX_VALUE, luy = Integer.MAX_VALUE;
      int rdx = Integer.MIN_VALUE, rdy = Integer.MIN_VALUE;
      • 이렇게 두면 첫 번째 '#'를 만났을 때 자연스럽게 갱신된다.
    • 전체 격자 순회
      • 행 인덱스 r, 열 인덱스 c를 이중 반복으로 순회한다. 
        for (int r = 0; r < wallpaper.length; r++) {
          for (int c = 0; c < wallpaper[r].length(); c++) {
              if (wallpaper[r].charAt(c) == '#') {
                  lux = Math.min(lux, r);
                  luy = Math.min(luy, c);
                  rdx = Math.max(rdx, r + 1);
                  rdy = Math.max(rdy, c + 1);
              }
          }
      }
      • lux, luy는 각 축에 대해 가장 작은 인덱스를 유지한다.
        • 위쪽으로 가장 위 파일 행
        • 왼쪽으로 가장 왼쪽 파일 열
      • rdx, rdy는 가장 큰 인덱스 + 1을 유지한다.
        • 파일이 있는 최대 행 인덱스가 r라면, 아래쪽 경계는 r + 1이 되어야 한다.
        • 열도 마찬가지로 c + 1까지 포함해야 파일 위치를 덮을 수 있다.
    • 결과 반환
      • 구해진 네 값을 그대로 배열로 묶어 반환한다. 
        return new int[]{lux, luy, rdx, rdy};
      • 문제에서 요구하는 형식과 정확히 일치한다.

    예시 (개념)

    • 예를 들어, 
      wallpaper = [
        "..#.",
        ".##.",
        ".#.."
      ]
      라고 하면,
      • '#' 위치: (0,2), (1,1), (1,2), (2,1)
      • 행 기준:
        • 최소 행 lux = 0
        • 최대 행 인덱스는 2 → rdx = 2 + 1 = 3
      • 열 기준:
        • 최소 열 luy = 1
        • 최대 열 인덱스는 2 → rdy = 2 + 1 = 3
      • 결과는 [0, 1, 3, 3]이 되고,
        이는 행 0~2, 열 1~2를 모두 포함하는 직사각형을 나타낸다.

    시간 복잡도

    wallpaper의 행 수를 R, 열 수를 C라고 하면:

    • 이중 반복으로 전체 칸을 한 번씩만 확인 → O(R × C)

    각 칸에서 하는 작업은 단순한 비교와 대입뿐이라
    전체적으로 매우 빠르게 동작한다.

    공간 복잡도

    추가로 사용하는 공간은 네 개의 정수 변수(lux, luy, rdx, rdy)뿐이다.

    • 입력 배열 wallpaper 외에 별도의 2차원 배열이나 리스트를 만들지 않는다.
    • 따라서 추가 공간 복잡도는 O(1)이다.

    주의

    • rdx, rdy에 +1을 하는 이유:
      • 문제에서 영역을 [시작, 끝) 형태(끝은 한 칸 뒤)로 표현하기 때문에,
        마지막 파일이 있는 위치의 인덱스에 +1을 해줘야 한다.
    • 파일이 반드시 하나 이상 있다고 가정:
      • 문제 조건상 최소 하나의 '#'가 있다고 보면,
        lux와 rdx가 갱신되지 않는 경우는 고려하지 않아도 된다.
    • 행/열 순서:
      • 반환 형식은 [위쪽 행, 왼쪽 열, 아래쪽 경계 행, 오른쪽 경계 열] 순서다.
      • 행과 열 순서를 바꾸지 않도록 주의해야 한다.
