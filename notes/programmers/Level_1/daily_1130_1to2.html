    문제 1 — 크기가 작은 부분 문자열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    크기가 작은 부분문자열 (프로그래머스 147355) 문제는
    숫자로 이루어진 문자열 t와, 비교 대상 숫자 문자열 p가 주어졌을 때,
    t에서 길이가 p와 같은 모든 연속 부분 문자열을 숫자로 해석했을 때,
    그 값이 p보다 작거나 같은 경우의 개수를 세는 문제다.

    구체적으로:

    • p의 길이를 len = p.length()라고 하면,
    • t에서 인덱스 0부터 t.length() - len까지 순회하면서,
      • 각 위치 i에서 부분 문자열 t.substring(i, i + len)을 뽑고,
      • 그 부분 문자열을 숫자로 바꾼 뒤 p를 숫자로 바꾼 값과 비교해서
        부분문자열 ≤ p이면 카운트를 1 증가시키면 된다.

    전략 개요

    • 먼저 p의 길이 length를 구하고, p를 long으로 변환해 num1에 저장한다.
    • t에서 길이 length짜리 모든 연속 부분 문자열을 순회하면서:
      • 각 부분 문자열을 long으로 변환해 num2로 두고,
      • num2 ≤ num1이면 카운트를 증가시킨다.
    • 마지막에 카운트 값을 반환하면 된다.

    핵심 로직

    • 초기 설정 
      public int solution(String t, String p) {
          int length = p.length();
          long num1 = Long.parseLong(p);
          int cnt = 0;
      • length: 비교에 사용할 부분 문자열의 길이 (항상 p와 동일).
      • num1: 문자열 p를 숫자로 바꾼 값.
      • cnt: 조건을 만족하는 부분 문자열의 개수를 세는 카운터.
    • 슬라이딩하며 부분 문자열 추출 및 비교 
          for (int i = 0; i <= t.length() - length; i++) {
              long num2 = Long.parseLong(t.substring(i, i + length));
      
              if (num1 >= num2) {
                  cnt++;
              }
          }
      • i는 부분 문자열의 시작 인덱스:
        • 끝 인덱스는 항상 i + length 이므로,
          시작 인덱스의 최댓값은 t.length() - length가 된다.
      • t.substring(i, i + length):
        • t에서 길이 length만큼 잘라낸 연속 부분 문자열이다.
      • Long.parseLong(...):
        • 해당 부분 문자열을 10진수 정수로 해석해 num2에 저장한다.
      • if (num1 >= num2):
        • p의 값이 부분 문자열 값보다 크거나 같다면 조건 만족 → cnt++.
    • 결과 반환 
          return cnt;
      }
      • 전체를 다 순회한 뒤, 조건을 만족한 부분 문자열의 개수 cnt를 그대로 반환한다.

    시간 복잡도

    t의 길이를 N, p의 길이를 L이라고 하면:

    • 부분 문자열의 시작 위치는 총 N - L + 1개이다.
    • 각 위치에서 substring과 Long.parseLong을 수행하는데,
      • 부분 문자열 길이가 항상 L이므로, 각 연산은 대략 O(L)이다.

    따라서 전체 시간 복잡도는 대략 O((N - L + 1) × L)이고,
    N과 L의 범위에서 충분히 감당 가능한 수준이다.

    공간 복잡도

    • 추가로 사용하는 변수는 long 두 개와 int 몇 개뿐이다.
    • 매 루프마다 새로 만드는 부분 문자열은 substring 결과이지만,
      입력 길이에 비해 작은 크기이므로, 알고리즘 자체의 추가 공간은 O(1) 수준으로 볼 수 있다.

    주의

    • 길이 조건:
      • for (int i = 0; i <= t.length() - length; i++)처럼 등호(<=)가 포함되어야
        마지막 시작 위치까지 포함해 부분 문자열을 모두 검사할 수 있다.
    • 숫자 크기:
      • 문제 조건 상 p와 부분 문자열 값이 long 범위를 벗어나지 않도록 설정되어 있다.
      • 그래서 Long.parseLong으로 안전하게 비교할 수 있다.
    • 문자열 비교 vs 숫자 비교:
      • 현재 코드는 숫자로 변환해서 비교하는 방식이다.
      • p와 부분 문자열의 길이가 항상 같기 때문에, 문자열 사전순 비교로 푸는 방식도 가능하지만,
        여기서는 문제 조건을 믿고 단순하게 long 변환 후 비교하는 구현을 사용했다.

    포인트 요약

    • p의 길이 length를 기준으로, t에서 같은 길이의 모든 연속 부분 문자열을 만든다.
    • p와 각 부분 문자열을 long으로 바꿔서 부분문자열 ≤ p인지 비교한다.
    • 조건을 만족하면 카운트를 증가시키고, 마지막에 카운트 값을 반환한다.
    • 슬라이딩 윈도우처럼 인덱스를 1씩 증가시키며 전체를 훑는 방식이라 구현이 단순하고 직관적이다.

    문제 2 — 가장 가까운 같은 글자

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    가장 가까운 같은 글자 (프로그래머스 142086) 문제는
    문자열 s가 주어졌을 때, 각 위치의 문자 기준으로
    왼쪽에 있는 같은 문자들 중 가장 가까운 문자까지의 거리를 구하는 문제다.

    인덱스 i에서의 답은 다음과 같다.

    • 왼쪽(0 ~ i-1) 중에 s[i]와 같은 문자가 없다면 → -1
    • 있다면 → 가장 가까운 같은 문자와의 거리 (즉, i - j 값, 여기서는 카운트를 앞으로 세며 구함)

    예를 들어, s = "banana"라면:

    • i = 0, 'b' → 왼쪽에 아무 글자 없음 → -1
    • i = 1, 'a' → 왼쪽에 'a' 없음 → -1
    • i = 2, 'n' → 왼쪽에 'n' 없음 → -1
    • i = 3, 'a' → 왼쪽에서 가장 가까운 'a'는 인덱스 1 → 거리 2
    • i = 4, 'n' → 왼쪽에서 가장 가까운 'n'은 인덱스 2 → 거리 2
    • i = 5, 'a' → 왼쪽에서 가장 가까운 'a'는 인덱스 3 → 거리 2

    이런 식으로 각 위치마다 왼쪽을 뒤로 거슬러 올라가면서 같은 문자를 찾으면 된다.

    전략 개요

    • 답을 저장할 배열 answer를 문자열 길이만큼 준비한다.
    • 문자열의 각 인덱스 i에 대해:
      • 현재 문자 s.charAt(i)를 기준으로,
      • 왼쪽 인덱스 j = i - 1부터 0까지 역순으로 탐색한다.
      • 처음으로 s[j] == s[i]인 위치를 찾으면, 그때의 거리(바로 앞에서부터 1, 2, 3… 세어간 값)를 기록한다.
      • 끝까지 못 찾으면 -1을 기록한다.
    • 이 과정을 모든 인덱스에 대해 반복하여 최종 answer 배열을 반환한다.

    핵심: 코드가 하는 일

    • 결과 배열 초기화 
      public int[] solution(String s) {
          int[] answer = new int[s.length()];
      • answer[i]에는 인덱스 i에서의 정답(거리 또는 -1)이 들어간다.
    • 각 위치 i에 대해 왼쪽으로 탐색 
          for (int i = 0; i < s.length(); i++) {
              boolean ok = false;
              int cnt = 1;
              if (i == 0) answer[i] = -1;
      
              for (int j = i - 1; j >= 0; j--) {
                  if (s.charAt(i) == s.charAt(j)) {
                      ok = true;
                      answer[i] = cnt;
                      break;
                  };
      
                  cnt++;
              }
      
              if (!ok) answer[i] = -1;
          }
      • for (int i = 0; i < s.length(); i++):
        • 문자열의 모든 인덱스를 순서대로 순회한다.
      • boolean ok = false;:
        • 현재 인덱스 i에서 왼쪽에 같은 문자를 발견했는지 여부를 표시한다.
      • int cnt = 1;:
        • 왼쪽으로 한 칸 이동할 때마다 1, 2, 3… 증가시키며 거리를 세는 카운터.
      • if (i == 0) answer[i] = -1;:
        • 문자열의 첫 글자(인덱스 0)는 왼쪽에 아무 글자도 없으므로, 바로 -1로 세팅한다.
        • 아래 루프에서도 ok가 false로 남기 때문에, 결국 -1이 유지된다.
      • for (int j = i - 1; j >= 0; j--):
        • 현재 인덱스 i에서 출발해, 왼쪽으로 한 칸씩 이동하면서 같은 문자를 찾는다.
      • if (s.charAt(i) == s.charAt(j)):
        • 왼쪽에서 같은 문자를 찾았다면:
          • ok = true;로 표시하고,
          • answer[i] = cnt;로 현재까지 센 거리를 저장한다.
          • 가장 가까운 것만 필요하므로 break;로 바로 루프 종료.
      • 같은 문자를 찾지 못한 경우:
        • 안쪽 루프를 끝까지 돌았는데도 ok가 false라면,
          왼쪽 어디에도 같은 문자가 없다는 뜻이다.
        • if (!ok) answer[i] = -1;로 처리해 -1을 저장한다.
    • 결과 반환 
          return answer;
      }
      • 각 인덱스별로 계산된 거리 값을 담은 answer 배열을 그대로 반환한다.

    시간 복잡도

    문자열 길이를 n이라 하면:

    • 바깥 루프: i = 0 ~ n-1 → 최대 n번
    • 각 i마다 안쪽 루프에서 j = i-1 ~ 0까지 최대 i번 검사

    최악의 경우 전체 비교 횟수는
    1 + 2 + ... + (n-1) = O(n^2)이 된다.
    따라서 시간 복잡도는 O(n^2)이다.

    공간 복잡도

    • 결과를 담는 배열 answer만 추가로 사용한다.
    • 크기는 문자열 길이 n에 비례하므로, 공간 복잡도는 O(n)이다.
    • 그 외의 변수는 모두 상수 개의 지역 변수뿐이다.

    주의

    • 첫 문자 처리:
      • 인덱스 0은 왼쪽이 없으므로 무조건 -1이어야 한다.
      • 코드에서는 if (i == 0) answer[i] = -1;로 먼저 처리하고,
        안쪽 루프가 i = 0일 때는 애초에 돌지 않기 때문에 -1이 유지된다.
    • 안쪽 루프 종료 조건:
      • 왼쪽에서 같은 문자를 발견하면 바로 break로 탈출해야
        가장 가까운 문자만 사용하게 된다.
      • 계속 뒤로 더 가면 더 멀리 있는 같은 문자까지 보게 되어, 원하는 거리보다 더 큰 값이 될 수 있다.
    • 거리 계산 방식:
      • 거리 자체는 i - j이지만, 코드에서는 cnt를 1부터 시작해
        왼쪽으로 한 칸 움직일 때마다 1씩 증가시키는 방식으로 구현했다.
      • 처음 만난 같은 문자에서 cnt 값이 곧 i - j와 동일하다.

    포인트 요약

    • 각 인덱스 i에서 왼쪽으로 역방향 탐색을 하며 같은 문자를 찾는다.
    • 처음 만나는 같은 문자가 가장 가까운 문자이므로, 그때까지의 거리만 기록하고 바로 종료한다.
    • 왼쪽에 같은 문자가 없으면 -1을 기록한다.
    • 단순한 이중 루프로 구현 가능하며, 시간 복잡도는 O(n^2), 공간 복잡도는 O(n)이다.
