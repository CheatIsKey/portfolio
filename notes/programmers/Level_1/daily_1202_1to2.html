    문제 1 — 기사단원의 무기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    기사단원의 무기 (프로그래머스 136798) 문제는
    1번부터 number번 기사까지 각각의 기사 번호에 대해 약수의 개수를 구한 뒤,
    그 개수가 limit을 넘으면 지정된 공격력 power로 대체하고,
    최종적으로 필요한 철의 무게(= 공격력 총합)를 구하는 문제다.

    • 각 기사 번호 i마다 약수 개수 cnt를 센다.
    • 만약 cnt > limit이면 → 그 기사의 공격력을 power로 바꾼다.
    • 그렇지 않으면 → 공격력은 약수 개수 cnt 그대로 사용한다.
    • 모든 기사에 대해 공격력을 합산한 값을 반환한다.

    핵심은 각 수의 약수 개수를 효율적으로 세는 것이다.
    이 코드는 약수를 1부터 √i까지 확인해서 약수 개수를 빠르게 구한다.

    전략 개요

    • 1번부터 number번 기사까지 반복하면서, 각 번호 i에 대해:
      • 약수 개수 cnt를 0으로 초기화.
      • num을 1부터 num * num ≤ i까지 올리면서, 약수인지 확인한다.
      • num이 i의 약수라면:
        • num * num == i (완전제곱수)일 때 → cnt에 1만 더한다.
        • 그 외에는 → 서로 다른 두 약수 num, i / num이 생기므로 cnt += 2.
      • 약수 개수 cnt가 limit을 넘으면 → cnt = power로 대체.
      • 이 값을 sum에 누적한다.
    • 마지막에 sum을 반환하면, 모든 기사 무기의 공격력 합이 된다.

    핵심: 코드가 하는 일

    • 초기 합계 준비 
      public int solution(int number, int limit, int power) {
          int sum = 0;
      • sum은 모든 기사 무기의 공격력을 합산할 변수다.
    • 기사 번호별로 약수 개수 구하기 
          for (int i = 1; i <= number; i++) {
              int cnt = 0;
      
              for (int num = 1; num * num <= i; num++) {
                  if (i % num == 0) {
                      if (num * num == i) cnt++;
                      else cnt += 2;
                  }
              }
      • for (int i = 1; i <= number; i++):
        • 각 기사 번호 i에 대해 반복.
      • cnt: 번호 i의 약수 개수를 저장.
      • for (int num = 1; num * num <= i; num++):
        • 약수는 쌍으로 존재하므로, 1부터 √i까지만 확인해도 충분하다.
      • if (i % num == 0):
        • num이 i의 약수인 경우만 처리.
      • if (num * num == i) cnt++;
        • 완전제곱수인 경우, 약수 쌍이 (num, num)으로 하나뿐이다.
        • 예: i = 16이고 num = 4일 때, 쌍은 (4, 4) 한 개 → 약수 1개만 추가.
      • else cnt += 2;
        • 그 외에는 약수 쌍 (num, i / num) 두 개를 한 번에 센다.
        • 예: i = 12, num = 3이면 약수 3, 4 두 개를 한 번에 더한다.
    • limit 초과 시 power로 대체 
              if (cnt > limit) cnt = power;
      
              sum += cnt;
          }
      • cnt > limit이면, 이 수의 공격력은 power로 제한된다.
      • 그 밖의 경우에는 실제 약수 개수 cnt가 공격력이 된다.
      • 최종적으로 sum에 누적한다.
    • 결과 반환 
          return sum;
      }
      • 모든 기사 번호에 대해 처리 후, 공격력 총합 sum을 반환한다.

    시간 복잡도

    • i를 1부터 number까지 순회: O(number)
    • 각 i마다 num을 1부터 √i까지 검사:
      • 각 수당 최대 O(√i) 연산.

    전체적으로 대략 O(number × √number) 정도의 시간 복잡도를 가진다.
    약수를 1부터 i까지 모두 확인하는 O(number²) 방식보다 훨씬 효율적이다.

    공간 복잡도

    • 추가로 사용하는 변수는 sum, cnt, num 정도의 상수 개수뿐이다.

    따라서 공간 복잡도는 O(1)이다.

    주의

    • 약수 카운트 방식:
      • 중복 카운트를 막기 위해 num * num == i일 때는 cnt++만 해야 한다.
      • 그 외에는 항상 두 개의 약수를 한 번에 세기 때문에 cnt += 2 처리.
    • limit 비교 시점:
      • 모든 약수 개수를 다 센 뒤, cnt > limit이면 바로 cnt = power로 교체해야 한다.
      • 이전 기사들의 값에도 영향을 주면 안 되므로, 각 i마다 독립적으로 처리한다.
    • int 범위:
      • 약수 개수는 i의 약수 개수이므로, 범위가 작아서 int로 충분하다.
      • sum도 문제 조건 안에서는 int로 오버플로가 나지 않도록 설계되어 있다.

    포인트 요약

    • 각 기사 번호 i의 약수 개수를 1 ~ √i 범위만 검사해서 효율적으로 구한다.
    • 약수 쌍을 이용해, 일반적인 경우 cnt += 2, 완전제곱수일 때만 cnt++를 사용한다.
    • 약수 개수가 limit을 넘으면 power로 대체하여 공격력을 제한한다.
    • 모든 기사에 대해 이 공격력을 합산해 최종 답을 구한다.
    • 시간 복잡도는 대략 O(number × √number), 공간 복잡도는 O(1)이다.

    문제 2 — 과일 장수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    과일 장수 (프로그래머스 135808) 문제는
    과일들의 점수 배열 score가 주어졌을 때,
    점수가 높은 과일부터 m개씩 상자를 만들어 팔 때 얻는
    최대 이익을 구하는 문제다.

    • 한 상자는 항상 정확히 m개의 과일로 구성한다.
    • 각 상자의 가격은 그 상자 안에서 가장 낮은 점수 × m이다.
    • 점수 k는 과일 점수의 최대값(1~k 범위) 제한일 뿐, 로직에는 직접 쓰이지 않는다.
    • 남는 과일(마지막에 m개를 채우지 못한 과일)은 버리게 되며, 상자를 만들지 않는다.

    이익을 최대로 하려면, 점수가 높은 과일들끼리 묶어서 상자를 만드는 것이 유리하다.
    따라서 점수를 정렬한 뒤, 뒤에서부터 m개씩 끊어 상자 가격을 계산하면 된다.

    전략 개요

    • score 배열을 오름차순으로 정렬한다.
    • 가장 점수가 높은 과일들부터 m개씩 그룹을 만들어 상자를 구성한다.
      • 정렬 후 배열의 뒤쪽(큰 값들)에서부터 m개씩 묶는다.
      • 예: 점수가 [1, 2, 3, 4, 5, 6], m = 3이면
        • 상자1: [4, 5, 6] → 최소값 4 → 가격 = 4 × 3
        • 상자2: [1, 2, 3] → 최소값 1 → 가격 = 1 × 3
    • 각 상자마다:
      • 그 상자의 최소 점수 = 현재 묶는 구간의 맨 앞 원소 (오름차순 정렬 기준).
      • 즉, 인덱스 범위 [idx - m, idx - 1]에 대해 최소값은 score[idx - m].
      • 상자 가격 = score[idx - m] * m이므로 이를 합계에 더한다.
    • 이 과정을 뒤에서부터 인덱스 m씩 감소시키며 반복한다.

    핵심: 코드가 하는 일

    • 정렬 및 합계 변수 초기화 
      public int solution(int k, int m, int[] score) {
          Arrays.sort(score);
      
          int price = 0;
      • Arrays.sort(score)로 점수 배열을 오름차순 정렬한다.
      • price는 모든 상자의 가격 합(최종 이익)을 저장하는 변수다.
      • k는 점수 상한이지만, 실제 계산에는 필요 없으므로 사용하지 않는다.
    • 뒤에서부터 m개씩 상자 구성 
          for (int idx = score.length; idx >= m; idx -= m) {
              int min = score[idx - m];
              price += min * m;
          }
      • idx를 배열의 길이에서 시작해서 m씩 감소시키며 반복한다.
        • 한 번의 반복마다 구간 [idx - m, idx - 1]이 하나의 상자에 해당.
        • idx >= m 조건을 통해, 상자 하나를 꾸릴 수 있는 과일 수가 남아 있을 때만 상자를 만든다.
      • int min = score[idx - m];
        • 배열이 오름차순 정렬이므로, 구간 [idx - m, idx - 1]에서 최소값은 맨 앞 원소인 score[idx - m]이다.
      • price += min * m;
        • 해당 상자의 가격 = 최소 점수 × 상자 크기 m.
        • 이를 총합 price에 누적한다.
      • 이렇게 하면 자연스럽게 가장 높은 점수들끼리 먼저 묶인 상자들부터 가격을 계산하게 된다.
    • 결과 반환 
          return price;
      }
      • 모든 가능한 상자를 만든 뒤, 상자 가격의 총합 price를 반환한다.

    시간 복잡도

    • Arrays.sort(score): 길이를 n이라고 할 때, O(n log n)
    • for (idx = n; idx >= m; idx -= m) 루프:
      • 한 번 돌 때마다 m개씩 건너뛰므로 반복 횟수는 대략 n / m회.
      • 루프 내부는 상수 시간 → O(n) 이하.

    전체 시간 복잡도는 O(n log n)이 되고,
    정렬이 지배적인 연산이다.

    공간 복잡도

    • 추가로 사용하는 변수는 price, idx, min 정도뿐이다.
    • 정렬은 Arrays.sort로 제자리(in-place)에서 수행되므로, 배열 외 추가 큰 공간을 사용하지 않는다.

    따라서 공간 복잡도는 O(1) (입력 배열 제외)이다.

    주의

    • 정렬 방향:
      • 오름차순 정렬 후, 뒤에서부터 m개씩 묶어야 “가장 높은 점수들끼리” 상자가 만들어진다.
    • min 인덱스:
      • 상자 구간이 [idx - m, idx - 1]일 때, 최소값은 score[idx - m]임에 주의.
      • 실수로 score[idx - 1]를 쓰면 최고값이 되어 잘못된 결과가 나온다.
    • 남는 과일 처리:
      • idx >= m 조건으로 인해, m개를 채우지 못하는 과일들은 상자를 만들지 않고 버린다.
    • k 사용 여부:
      • k는 점수의 최대값을 의미하는 파라미터지만,
        이 풀이에서는 별도의 로직에 필요하지 않으므로 사용하지 않아도 된다.

    포인트 요약

    • score를 오름차순 정렬한다.
    • 배열의 뒤에서부터 m개씩 묶어 상자를 만든다.
    • 각 상자의 가격은 해당 구간의 최소값(score[idx - m]) × m이다.
    • 모든 상자의 가격을 더한 값이 정답이며, 남는 과일은 사용하지 않는다.
    • 시간 복잡도는 O(n log n), 공간 복잡도는 O(1)이다.
