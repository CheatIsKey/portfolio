    문제 1 — 문자열 나누기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 나누기 (프로그래머스 140108) 문제는
    문자열 s를 왼쪽부터 차례대로 읽으면서, 다음 규칙에 따라 최대한 많은 조각으로 나누고
    그 조각의 개수를 구하는 문제다.

    • 각 조각은 “기준 문자”와 “다른 문자”의 개수를 세다가,
      두 개수가 처음으로 같아지는 지점에서 끊어 하나의 묶음으로 만든다.
    • 만약 문자열 끝까지 가도 기준 문자와 다른 문자의 개수가 같아지지 않으면,
      남은 부분 전체를 마지막 조각으로 친다.

    이 풀이에서는 문자열을 한 번만 훑으면서,
    현재 조각의 시작 위치를 가리키는 인덱스와, 그 조각 안의 기준 문자/다른 문자 개수만 관리해 답을 구한다.

    전략 개요

    • 전체 문자열을 한 번 훑되, 조각 단위로 나눠서 처리한다.
    • 바깥 쪽에서:
      • idx를 “아직 처리하지 않은 첫 인덱스”로 두고,
      • i = idx를 현재 조각의 시작 위치로 사용한다.
    • 각 조각마다:
      • 시작 문자를 기준 문자 cur로 선택하고,
      • same: 기준 문자와 같은 글자 개수
      • diff: 기준 문자와 다른 글자 개수
      • 를 세면서 문자열을 오른쪽으로 진행한다.
      • 조건:
        • same == diff가 되는 순간 → 한 조각 완성 → cnt++ 후 다음 조각으로.
        • 끝까지(idx == s.length()) 갔는데도 같아지지 않으면 → 남은 부분 전체를 한 조각으로 보고 cnt++.
    • 이 과정을 반복하면, 규칙에 따라 만든 조각의 총 개수가 cnt가 된다.

    핵심: 코드가 하는 일

    • 초기 설정 
      public int solution(String s) {
          int cnt = 0, idx = 0;
      • cnt: 최종적으로 조각의 개수를 셀 변수.
      • idx: 현재 조각을 어디까지 읽었는지 나타내는 인덱스.
        다음 조각의 시작 위치이기도 하다.
    • 조각 단위로 문자열 순회 
          for (int i = 0; i < s.length(); i = idx) {
              char cur = s.charAt(i);
              int diff = 0, same = 0;
      • i = idx:
        • 바깥 for문에서 i를 직접 증가시키지 않고,
          항상 “아직 처리하지 않은 첫 문자 위치”인 idx로 갱신한다.
        • 이 덕분에 각 조각은 서로 겹치지 않고, 문자열 전체가 조각들로 정확히 이어진다.
      • cur = s.charAt(i):
        • 이번 조각의 기준 문자.
      • same, diff:
        • 각 조각마다 기준 문자와 같/다른 문자의 개수를 새로 세기 위해 0으로 초기화한다.
    • 현재 조각에서 기준 문자/다른 문자 개수 세기 
              while (true) {
                  char nextChar = s.charAt(idx++);
      
                  if (cur == nextChar) same++;
                  else diff++;
      
                  if (same == diff) {
                      cnt++;
                      break;
                  } else if (idx == s.length()) {
                      cnt++;
                      break;
                  }
              }
          }
      • char nextChar = s.charAt(idx++);
        • idx 위치의 문자를 읽고 난 후, idx를 1 증가시킨다.
        • 이렇게 해서 이번 조각이 어디까지 읽혔는지를 idx가 항상 가리키게 된다.
      • if (cur == nextChar) same++; else diff++;
        • 기준 문자와 같으면 same 증가, 다르면 diff 증가.
      • if (same == diff):
        • 기준 문자와 다른 문자의 개수가 같아지는 순간,
          • 조각 하나가 완성된 것이므로 cnt++;
          • break;로 while을 빠져나와 다음 조각으로 넘어간다.
      • else if (idx == s.length()):
        • 아직 same == diff는 아니지만 문자열 끝까지 왔을 때,
          • 남은 모든 문자들을 하나의 조각으로 취급해야 하므로 cnt++;
          • 마찬가지로 break;로 루프를 종료한다.
      • 이 while (true)는 위 두 상황 중 하나에서 반드시 break되므로 무한 루프가 되지 않는다.
      • while이 끝난 시점의 idx는 “이번 조각이 끝난 다음 위치”가 되어,
        바깥 for문의 다음 반복에서 i = idx로 설정되어 새 조각을 시작한다.
    • 결과 반환 
          return cnt;
      }
      • 모든 조각을 세고 난 뒤, 최종적으로 조각의 개수 cnt를 반환한다.

    시간 복잡도

    • 중첩된 for와 while이 있지만,
      • idx가 문자열 전체를 통틀어 각 문자 위치를 한 번씩만 지나간다.
      • 즉, 문자 하나가 여러 조각에서 중복해서 읽히지 않는다.

    따라서 문자열 길이를 n이라고 하면,
    전체 알고리즘의 시간 복잡도는 O(n)이다.

    공간 복잡도

    • 추가로 사용하는 것은 정수 변수 몇 개(cnt, idx, same, diff)와 문자 하나(cur, nextChar)뿐이다.

    따라서 공간 복잡도는 O(1)이다.

    주의

    • idx와 i의 관계:
      • 반드시 for (int i = 0; i < s.length(); i = idx)처럼,
        i를 idx로 갱신해야 한다.
      • 만약 i++로만 증가시키면, 한 조각 안에서 이미 소비한 문자들을 다시 처리하게 되어
        로직이 깨지거나 무한 루프 위험이 생긴다.
    • while(true)와 인덱스 범위:
      • char nextChar = s.charAt(idx++);에서 이미 문자를 읽고 나서 idx를 증가시키므로,
        그 후에 idx == s.length()인지 확인해서 마지막 문자까지 안전하게 처리한다.
      • 이 조건에서 break가 보장되기 때문에 인덱스 범위를 벗어나지 않는다.
    • same == diff가 먼저인지, 끝에 도달했는지가 먼저인지:
      • 문제 요구사항대로 “처음으로 두 개수가 같아지는 지점”에서 우선적으로 끊어야 하므로,
        코드에서도 same == diff를 먼저 검사하고, 그 다음에 idx == s.length()를 검사한다.

    포인트 요약

    • 각 조각은 “기준 문자”와 “다른 문자” 개수를 세다가 처음으로 같아지는 지점에서 끊는다.
    • idx를 이용해 “아직 처리하지 않은 첫 문자”에서만 새 조각을 시작한다.
    • 기준 문자/다른 문자 개수는 조각마다 same, diff로 새로 세고,
      same == diff이거나 문자열 끝에 도달하면 조각 개수 cnt를 1 늘린다.
    • 문자열을 한 번만 훑는 구조라 시간 복잡도는 O(n), 추가 공간은 O(1)이다.

    문제 2 — 명예의 전당 (1)

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    명예의 전당 (프로그래머스 138477) 문제는
    명예의 전당에 최대 k명까지 점수를 올려둘 수 있을 때,
    매일 새로운 점수가 들어올 때마다 그날 기준 명예의 전당 최하위 점수를 기록하는 문제다.

    규칙을 정리하면:

    • 매일 하나씩 점수 score[i]가 들어온다.
    • 명예의 전당에는 최대 k개의 점수만 유지한다.
    • 지금까지 들어온 점수들 중에서 상위 k개를 명예의 전당에 올려둔다.
    • 그날 명예의 전당에 올라간 점수들 중 가장 낮은 점수를 결과 배열에 기록한다.

    그래서 각 날짜 i마다 해야 할 일은 간단히 말해:

    • 지금까지의 점수 중 상위 k개를 골라서,
    • 그 중 최소값을 answer[i]에 넣으면 된다.

    전략 개요

    • 명예의 전당을 표현할 리스트 list를 하나 만든다.
    • 매일 점수 score[i]가 들어올 때:
      • 아직 명예의 전당에 k개보다 적게 들어있으면 → 그냥 넣고 정렬.
      • 이미 k개가 꽉 찼으면:
        • 현재 명예의 전당 최하위 점수(list.get(0))와 새 점수를 비교.
        • 새 점수가 더 크면 → 최하위 점수를 새 점수로 교체하고 다시 정렬.
        • 새 점수가 더 작거나 같으면 → 명예의 전당 변화 없음.
    • 이때 리스트를 항상 오름차순으로 유지하므로,
      • list.get(0)는 항상 “현재 명예의 전당 최하위 점수”가 된다.
      • 그 값을 매일 answer[i]에 그대로 기록하면 된다.

    핵심: 코드가 하는 일

    • 자료 구조 준비 
      public int[] solution(int k, int[] score) {
          List<Integer> list = new ArrayList<>();
          int[] answer = new int[score.length];
      • list: 명예의 전당에 올라가 있는 점수들을 저장하는 리스트.
      • answer: 각 날짜마다 명예의 전당 최하위 점수를 저장할 결과 배열.
    • 매일 점수 갱신 
          for (int i = 0; i < score.length; i++) {
              if (i < k) {
                  list.add(score[i]);
                  list.sort(Integer::compare);
              } else {
                  if (list.get(0) < score[i]) {
                      list.set(0, score[i]);
                      list.sort((a, b) -> a - b);
                  }
              }
      
              answer[i] = list.get(0);
          }
      • i < k인 경우 (초반 k일):
        • 명예의 전당 자리가 아직 남아 있으므로 새 점수를 그냥 추가한다.
        • list.sort(...)로 오름차순 정렬 → 최솟값이 항상 list.get(0).
      • i ≥ k인 경우 (명예의 전당이 이미 꽉 찬 이후):
        • list.get(0)는 현재 명예의 전당 최하위 점수.
        • 새 점수 score[i]가 그 값보다 크면:
          • list.set(0, score[i])로 최하위 점수를 새 점수로 교체.
          • 다시 정렬해서 list를 오름차순으로 맞춰준다.
        • 새 점수가 더 작거나 같으면 → 명예의 전당 구성은 그대로 두고 아무 것도 하지 않는다.
      • 그날의 답 기록:
        • answer[i] = list.get(0);
        • 항상 “현재 명예의 전당에서 가장 낮은 점수”를 저장하게 된다.
    • 결과 반환 
          return answer;
      }
      • 각 날짜별로 구한 최하위 점수 배열 answer를 그대로 반환한다.

    시간 복잡도

    • 날짜 수를 n = score.length, 명예의 전당 크기를 k라고 하면:
      • 매일 최대 한 번 list.sort(...)를 호출한다.
      • list의 크기는 최대 k이므로, 정렬 비용은 O(k log k).

    따라서 전체 시간 복잡도는 대략
    O(n · k log k) 이고,
    문제 조건에서 k가 크게 주어지지 않기 때문에 충분히 통과 가능한 수준이다.

    공간 복잡도

    • 명예의 전당 리스트 list: 최대 k개의 정수 저장 → O(k)
    • 결과 배열 answer: 길이 n → O(n)

    전체 공간 복잡도는 O(n + k)이며, 입력 크기에 비례하는 수준이다.

    주의

    • 정렬 방향:
      • 항상 오름차순으로 정렬해야 list.get(0)가 “최하위 점수”가 된다.
    • 초기 k일 처리:
      • 처음 k일은 명예의 전당이 꽉 차기 전이므로, 단순히 쌓고 정렬만 해도 된다.
      • 이때도 list.get(0)는 “현재까지 들어온 점수들 중 최솟값”이 되어 문제에서 요구하는 값과 일치한다.
    • 최적화 여지:
      • 이 코드는 매번 전체 리스트를 정렬하는 방식이라 구현은 직관적이지만,
        더 효율적으로 하려면 우선순위 큐(최소 힙)를 사용하는 방법도 있다.
      • 다만 이 구현도 문제 조건 안에서는 충분히 빠르게 동작한다.

    포인트 요약

    • 명예의 전당을 리스트로 두고, 항상 오름차순 정렬 상태를 유지한다.
    • 처음 k일은 점수를 추가만 하고 정렬한다.
    • 그 이후는 “현재 최하위 점수보다 클 때만 교체 + 정렬”을 한다.
    • 매일 list.get(0)를 결과 배열에 넣으면, 문제에서 원하는 “그날의 명예의 전당 최하위 점수”를 얻을 수 있다.
