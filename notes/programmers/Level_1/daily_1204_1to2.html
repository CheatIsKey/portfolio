    문제 1 — 푸드 파이트 대회

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    푸드 파이트 대회 (프로그래머스 134240) 문제는
    각 음식 종류별 개수가 들어 있는 배열 food가 주어졌을 때,
    양쪽 플레이어에게 완전히 대칭이 되도록 음식을 배치한 문자열을 만드는 문제다.

    • food[i] : 번호 i 음식의 개수 (0번 인덱스는 물).
    • 음식은 왼쪽 플레이어와 오른쪽 플레이어가 똑같이 가져야 한다.
    • 중앙에는 항상 0 (물 한 잔)이 위치한다.
    • 양쪽의 음식 배치는 중앙을 기준으로 좌우 대칭이 되어야 한다.

    따라서 각 음식 번호 i에 대해, 실제로는 한 사람당 food[i] / 2개만 쓸 수 있고,
    왼쪽에 이 순서대로 쌓은 뒤, 중앙에 0을 두고, 그 왼쪽 문자열을 뒤집어서 오른쪽에 붙이면
    조건을 만족하는 문자열을 쉽게 만들 수 있다.

    전략 개요

    • 음식 번호 1부터 마지막까지 반복하면서:
      • 한 사람에게 줄 수 있는 개수 = food[i] / 2.
      • 이 개수만큼 왼쪽에 번호 i를 문자열로 이어붙인다.
    • 이렇게 만든 왼쪽 문자열을 left라고 하면,
      • 정답 문자열 = left + "0" + reverse(left).
      • 이렇게 하면 중앙 0을 기준으로 양쪽이 완전 대칭이 된다.
    • 구현에서는 StringBuilder를 사용해
      • 왼쪽 문자열을 만든 뒤,
      • 그 내용을 그대로 쓰고, 0을 붙이고, reverse()한 내용을 덧붙인다.

    핵심: 코드가 하는 일

    • 왼쪽 구간(한 사람 분) 만들기 
      public String solution(int[] food) {
          StringBuilder sb = new StringBuilder();
      
          for (int i = 1; i < food.length; i++) {
              int placed = food[i] / 2;
      
              for (int j = 0; j < placed; j++) {
                  sb.append(i);
              }
          }
      • i = 1부터 시작:
        • food[0]는 물이므로, 양 옆에 배치하지 않아도 되고, 문제에서도 0번은 따로 취급한다.
      • placed = food[i] / 2:
        • 양쪽에 똑같이 나눠야 하므로, 한 사람에게 줄 수 있는 개수는 전체의 절반이다.
        • 나머지가 생기는 부분은 버려진다고 보면 된다.
      • 안쪽 반복문: 
        for (int j = 0; j < placed; j++) {
          sb.append(i);
      }
        • 해당 음식 번호 i를 placed번 왼쪽 문자열에 이어붙인다.
        • 최종적으로 sb에는 왼쪽 플레이어의 음식 배치가 순서대로 들어가게 된다.
    • 정답 문자열 완성 
          StringBuilder sb2 = new StringBuilder();
          sb2.append(sb).append(0).append(sb.reverse());
      
          return sb2.toString();
      }
      • sb : 현재 상태에서는 왼쪽에 놓일 음식 문자열.
      • sb2.append(sb):
        • 왼쪽 문자열을 먼저 복사한다.
      • .append(0):
        • 중앙에 물(0)을 배치한다.
      • .append(sb.reverse()):
        • 왼쪽 문자열을 뒤집어서 오른쪽에 붙인다.
        • 이 때, StringBuilder.reverse()는 원본 sb도 같이 뒤집힌다는 점에 주의.
        • 하지만 이미 sb2.append(sb)로 왼쪽 부분을 붙여 놓은 뒤이므로,
          뒤집힌 내용은 오른쪽 부분으로 쓰이게 되어 의도한 대칭 구조가 완성된다.
      • 마지막으로 sb2.toString()을 반환하면,
        • 왼쪽 음식들 + 0 + 오른쪽 음식들이 대칭을 이루는 최종 문자열이 된다.

    시간 복잡도

    • 각 음식 번호 i에 대해 food[i] / 2번씩 append를 수행한다.
    • 전체 문자열 길이를 L라고 하면,
      • 왼쪽 문자열 생성: O(L)
      • sb.reverse(): 길이 L 문자열에 대해 O(L)
      • 전체적으로 O(L)에 비례.

    입력 크기 제한을 고려하면 충분히 빠르게 동작한다.

    공간 복잡도

    • 왼쪽 문자열을 담는 sb와 최종 문자열을 담는 sb2 두 개의 StringBuilder를 사용한다.
    • 문자열 길이에 비례하는 공간만 사용하므로 O(L)이다.

    주의

    • 0번 인덱스는 건너뛴다:
      • i = 1부터 시작해야 한다. food[0]은 중앙의 물을 의미하며 양쪽에 배치하지 않는다.
    • 절반만 사용:
      • placed = food[i] / 2로 한 사람당 사용할 개수만 고려해야 양쪽이 똑같아진다.
      • 홀수 개가 있어도 나머지 1개는 버려진다고 생각하면 된다.
    • reverse()의 부작용:
      • sb.reverse()는 sb 자체를 뒤집는다.
      • 왼쪽 문자열을 먼저 sb2.append(sb)로 복사해 둔 뒤에 뒤집어야 한다.
    • 문자 vs 숫자:
      • sb.append(i)는 정수 i를 문자열로 변환하여 붙이므로 문제에서 요구하는 “숫자 나열” 형식과 일치한다.

    포인트 요약

    • 각 음식 번호 i에 대해 한 사람당 쓸 수 있는 개수는 food[i] / 2개다.
    • 왼쪽 문자열을 1번 인덱스부터 순서대로 쌓는다.
    • 정답은 왼쪽 문자열 + "0" + 왼쪽 문자열의 역순으로 만든다.
    • StringBuilder와 reverse()를 이용해 대칭 구조를 효율적으로 구현한다.
    • 시간 복잡도는 전체 문자열 길이에 비례하는 O(L), 공간 복잡도도 O(L)이다.

    문제 2 — 햄버거 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    햄버거 만들기 (프로그래머스 133502) 문제는
    재료들이 순서대로 담긴 배열 ingredient가 주어졌을 때,
    연속된 4개의 재료가 1-2-3-1 순서가 될 때마다 햄버거 1개를 만들고 해당 네 개를 제거한다고 했을 때,
    최종적으로 만들 수 있는 햄버거의 개수를 구하는 문제다.

    • 1 : 빵
    • 2 : 야채
    • 3 : 고기
    • 패턴 [1, 2, 3, 1]을 찾으면 햄버거 1개 완성, 바로 그 네 칸이 사라진다.
    • 제거 후 뒤에 있는 재료들이 앞으로 당겨지므로, 새로운 패턴이 생길 수 있다.

    이 동작은 전형적인 스택 시뮬레이션으로 처리할 수 있다.
    재료를 앞에서부터 하나씩 스택에 쌓으면서, 스택의 상단 4개가 1-2-3-1인지 계속 검사하면 된다.

    전략 개요

    • 배열 ingredient를 처음부터 끝까지 순회한다.
    • 정수 배열 stack과 포인터 top을 이용해 스택처럼 사용한다.
      • stack[top++] = i; 로 재료를 하나씩 쌓는다.
    • 매번 재료를 하나 쌓을 때마다:
      • 현재 쌓인 개수 top이 4 이상이라면,
      • 맨 위 4개가 [1, 2, 3, 1]인지 검사한다.
    • 패턴이 맞으면:
      • top -= 4; 로 스택에서 네 개를 한꺼번에 제거.
      • 햄버거 개수 cnt++.
    • 모든 재료를 처리한 뒤, cnt를 반환한다.

    이렇게 하면 “중간에서 재료가 사라지며 뒤의 재료가 당겨지는” 효과를
    스택의 push/pop 동작으로 자연스럽게 시뮬레이션할 수 있다.

    핵심: 코드가 하는 일

    • 초기화 
      public int solution(int[] ingredient) {
          int cnt = 0;
      
          int[] stack = new int[ingredient.length];
          int top = 0;
      • cnt : 완성된 햄버거 개수.
      • stack : 재료를 쌓아둘 배열(스택 역할).
      • top : 스택의 현재 크기이자, 다음에 쌓일 위치를 가리키는 인덱스.
    • 재료를 스택에 쌓으면서 패턴 검사 
          for (int i : ingredient) {
              stack[top++] = i;
      
              if (top >= 4 &&
                      stack[top - 4] == 1 &&
                      stack[top - 3] == 2 &&
                      stack[top - 2] == 3 &&
                      stack[top - 1] == 1) {
                  top -= 4;
                  cnt++;
              }
          }
      • for (int i : ingredient):
        • 재료 배열을 앞에서부터 하나씩 꺼낸다.
      • stack[top++] = i;:
        • 현재 재료 i를 스택의 맨 위에 쌓고, top을 1 증가시킨다.
      • if (top >= 4 && ...):
        • 스택에 최소 4개 이상 쌓였을 때만 패턴을 검사한다.
        • stack[top - 4] ~ stack[top - 1]는 스택 상단의 4개 원소이다.
        • 이 네 값이 차례대로 1, 2, 3, 1인지 확인한다.
      • 패턴이 완성된 경우:
        • top -= 4;:
          • 스택에서 4개를 한 번에 제거한 것과 같은 효과.
          • 이로 인해 그 아래에 있던 재료들이 “앞으로 당겨진 것처럼” 된다.
        • cnt++;:
          • 햄버거 하나 완성.
    • 결과 반환 
          return cnt;
      }
      • 모든 재료를 처리한 뒤, 완성된 햄버거 개수 cnt를 반환한다.

    시간 복잡도

    • 배열 ingredient를 한 번 순회하면서,
      • 각 원소마다 push(1회)와 최대 4칸 비교만 수행한다.

    따라서 전체 시간 복잡도는 O(n) (n: 재료 개수)이다.

    공간 복잡도

    • 스택 역할을 하는 int[] stack이 길이 n만큼 필요하다.
    • 추가로 사용하는 변수는 cnt, top 정도의 상수 개수.

    따라서 공간 복잡도는 O(n)이다.

    주의

    • 인덱스 범위:
      • top - 4부터 접근하므로, 반드시 top >= 4 조건을 먼저 확인해야 한다.
    • 패턴 위치:
      • 항상 “맨 위 4개”만 검사해도 되는 이유:
        • 새 재료는 항상 스택의 끝에 추가되기 때문에,
        • 새로운 1-2-3-1 패턴이 생기는 위치는 항상 스택 상단 뿐이다.
    • 배열을 스택처럼 사용:
      • Stack<Integer> 같은 자료구조 대신 int[]와 top으로 직접 구현하면
        오버헤드 없이 효율적으로 동작한다.

    포인트 요약

    • 재료들을 앞에서부터 스택에 쌓으면서, 스택 상단 4개가 1-2-3-1인지 계속 확인한다.
    • 패턴이 나오면 top -= 4로 네 개를 제거하고, 햄버거 개수를 1 증가시킨다.
    • 스택 방식으로 구현하면 “중간에서 재료가 사라지고 뒤가 당겨지는” 효과를 자연스럽게 표현할 수 있다.
    • 시간 복잡도는 O(n), 공간 복잡도는 O(n)이다.
