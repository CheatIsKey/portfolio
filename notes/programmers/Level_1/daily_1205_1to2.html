    문제 1 — 삼총사

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    옹알이 (2) (프로그래머스 133499) 문제는
    문자열 배열 babbling이 주어졌을 때,
    각 문자열이 허용된 옹알이 패턴만으로 이루어져 있고, 동시에 같은 패턴이 연속으로 반복되지 않으면
    그 문자열을 유효한 옹알이로 인정해 총 몇 개가 유효한지를 구하는 문제다.

    • 사용할 수 있는 패턴(옹알이 단어):
      • "aya", "ye", "woo", "ma"
    • 조건:
      • 문자열 전체가 위 네 단어만 이어 붙인 형태여야 한다.
      • 같은 단어를 연속해서 두 번 사용할 수 없다 (예: "ayaaya"는 불가).

    각 문자열에 대해 앞에서부터 가능한 패턴을 하나씩 매칭해 나가면서,
    끝까지 규칙을 지키며 매칭이 가능하면 유효한 문자열로 센다.

    전략 개요

    • 허용된 패턴 배열을 준비한다:
      • String[] words = {"aya", "ye", "woo", "ma"};
    • 각 문자열 s에 대해:
      • pos : 현재 읽고 있는 위치(인덱스)
      • last : 바로 직전에 매칭한 단어 (연속 반복 방지용)
      • ok : 이 문자열이 지금까지 유효한지 여부
    • 반복:
      • pos < s.length() 인 동안:
        • words 배열을 돌면서, 현재 위치 pos에서 시작하는 단어가 있는지 확인한다.
        • s.startsWith(word, pos) 이고, word가 last와 다를 때만 사용 가능.
        • 해당 단어를 선택하면:
          • pos를 단어 길이만큼 증가
          • last = word로 갱신
        • 네 단어 중 어느 것도 매칭되지 않으면 → 이 문자열은 규칙 위반 → ok = false로 표시 후 중단.
    • 루프를 끝까지 돌 때까지 위반이 없었다면(ok == true) → 유효한 문자열 카운트 증가.

    이렇게 하면 각 문자열에 대해 왼쪽에서 오른쪽으로 탐욕적으로 패턴을 붙여 보면서,
    규칙을 만족하는지 쉽게 검사할 수 있다.

    핵심: 코드가 하는 일

    • 기본 설정 
      public int solution(String[] babbling) {
          String[] words = {"aya", "ye", "woo", "ma"};
          int cnt = 0;
      • words : 사용할 수 있는 옹알이 단어 목록.
      • cnt : 유효한 문자열 개수 누적 결과.
    • 각 문자열 검사 
          for (String s : babbling) {
              int pos = 0;
              String last = "";
              boolean ok = true;
      
              while (pos < s.length()) {
                  boolean matched = false;
      
                  for (String word : words) {
                      if (s.startsWith(word, pos) && !word.equals(last)) {
                          pos += word.length();
                          last = word;
                          matched = true;
                          break;
                      }
                  }
      
                  if (!matched) {
                      ok = false;
                      break;
                  }
              }
      
              if (ok) cnt++;
          }
      
          return cnt;
      }
      • for (String s : babbling)
        • 모든 문자열을 하나씩 검사.
      • pos:
        • 현재 문자열 s에서 어디까지 해석했는지 나타내는 인덱스.
      • last:
        • 직전에 사용된 패턴을 저장해, 같은 단어가 연속으로 쓰이는 것을 막는다.
      • ok:
        • 이 문자열이 끝까지 규칙을 지키는지 여부.
      • while (pos < s.length()):
        • 문자열 끝까지 처리할 때까지 반복.
        • 반복 안에서 매 단계마다 네 단어 중 하나를 매칭하려고 시도.
      • matched:
        • 현재 pos에서 시작하는 위치에 허용된 단어를 찾았는지 여부.
      • for (String word : words) 내부:
        • s.startsWith(word, pos)로 현재 위치에서 해당 패턴이 시작되는지 확인.
        • !word.equals(last)로 바로 직전과 같은 단어인지 체크해서, 연속 반복을 방지.
        • 조건을 만족하면:
          • pos += word.length(); → 해당 길이만큼 앞으로 이동.
          • last = word; → 마지막 사용 단어 갱신.
          • matched = true; break; → 이번 단계는 성공, 다음 단계로.
      • if (!matched):
        • 네 단어 중 아무 것도 매칭되지 않으면, 이 문자열은 규칙 위반.
        • ok = false; break;로 처리 중단.
      • while문 종료 후:
        • ok == true이면 문자열 전체가 규칙에 맞게 분해되었으므로 cnt++.
      • 모든 문자열 처리 후 cnt를 반환.

    시간 복잡도

    • 문자열 개수를 N, 각 문자열 길이의 총합을 L이라고 하면,
    • 각 위치에서 최대 4개의 단어(words 원소 수)를 확인한다.

    따라서 전체 시간 복잡도는 O(L) (상수 4를 무시하면, 전체 길이에 선형) 정도다.

    공간 복잡도

    • 고정된 크기의 words[4] 배열만 추가로 사용한다.
    • 루프 내 변수(pos, last, ok, matched)는 모두 상수 개수.

    따라서 추가 공간 복잡도는 O(1)이다.

    주의

    • 연속 반복 금지:
      • 단순히 네 단어로만 구성되었는지만 체크하는 것이 아니라,
      • 바로 이전에 쓴 단어와 같은 단어를 다시 연속으로 쓰면 안 된다.
      • 이 부분이 !word.equals(last) 조건으로 구현되어 있다.
    • 부분 매칭 주의:
      • startsWith(word, pos)로 현재 위치에서 정확히 그 단어가 시작되는지만 본다.
      • 예를 들어 "ayaye"를 검사하면:
        • 처음에는 "aya"로 매칭 → pos = 3
        • 그 다음 "ye"로 매칭 → pos = 5 == length → 유효.
    • 탐욕적 매칭의 안전성:
      • 단어 후보가 4개밖에 없고, 서로 충분히 구분되는 문자열이기 때문에
        앞에서부터 가능한 단어를 바로 선택하는 탐욕적 방식이 문제 없다.

    포인트 요약

    • 허용된 옹알이 단어는 {"aya", "ye", "woo", "ma"} 네 가지.
    • 각 문자열에 대해 왼쪽에서부터 가능한 단어들을 차례대로 붙여가며 분해한다.
    • 직전 단어와 같은 단어를 연속해서 사용할 수 없도록 last를 기억한다.
    • 끝까지 규칙을 지키며 분해에 성공하면 유효한 문자열로 세고, 그렇지 않으면 버린다.
    • 전체 시간은 문자열 길이 총합에 비례하는 O(L), 추가 공간은 상수 O(1)이다.

    문제 2 — 콜라 문제

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    콜라 문제 (프로그래머스 132267)는
    빈 병 a개를 가져가면 새 콜라 b개로 바꿔주는 이벤트에서,
    처음 빈 병이 n개 있을 때 최종적으로 마실 수 있는 콜라의 총 개수를 구하는 문제다.

    • a : 필요한 빈 병 수
    • b : 교환으로 얻는 새 콜라 수
    • n : 초기에 가지고 있는 빈 병 수
    • 새 콜라를 마시면 다시 그만큼 빈 병이 생기므로, 이 빈 병으로 다시 교환을 반복할 수 있다.

    핵심은 현재 빈 병 수 n으로 최대 몇 번 교환할 수 있는지를 계속 반복해서 계산하고,
    그때마다 얻은 콜라 수를 모두 합산하는 단순 시뮬레이션이다.

    전략 개요

    • 현재 가지고 있는 빈 병 수를 n으로 유지하면서, 더 이상 교환이 불가능할 때까지 반복한다.
    • 한 번 반복에서:
      • n / a : 이번에 교환할 수 있는 횟수 (한 번에 a병씩 묶어서 교환).
      • 이번에 얻는 새 콜라 수: colaCount = n / a, changed = colaCount * b
      • 얻은 콜라 개수 changed를 정답 누적 변수 cnt에 더한다.
      • 새로운 빈 병 수:
        • 교환 후 남은 빈 병: n % a
        • 새 콜라를 마셔서 생긴 빈 병: changed
        • 총합: n = n % a + changed
    • n / a == 0이 되는 순간, 더 이상 교환이 불가능하므로 반복을 종료한다.
    • 누적된 cnt가 최종 마신 콜라의 개수다.

    교환 비율이 고정되어 있고, 매 반복마다 빈 병이 줄어들기 때문에
    단순 while 반복만으로 충분히 해결된다.

    핵심: 코드가 하는 일

    • 초기 설정과 빠른 종료 
      public int solution(int a, int b, int n) {
          int cnt = 0;
      
          if (n < a) return 0;
      • cnt : 지금까지 마신 콜라의 총 개수.
      • 처음부터 n < a이면 한 번도 교환할 수 없으므로 0을 바로 반환한다.
    • 교환 반복 루프 
          while (n / a != 0) {
              int colaCount = n / a;
              int changed = colaCount * b;
      
              cnt += changed;
              n = n % a + changed;
          }
      • while (n / a != 0):
        • n개를 a개씩 나눴을 때 몫이 0이 아닌 동안 → 아직 교환 가능.
        • n / a == 0이면 n < a이므로 더 이상 교환 불가.
      • colaCount = n / a:
        • 이번 회차에서 몇 번 교환할 수 있는지(= 교환 횟수).
      • changed = colaCount * b:
        • 이번 회차 교환으로 얻는 새 콜라 수.
      • cnt += changed;:
        • 얻은 콜라를 모두 마신다고 가정하므로, 정답에 누적한다.
      • n = n % a + changed;:
        • 교환에 사용되고 남은 빈 병: n % a
        • 새 콜라를 마시고 생긴 빈 병: changed
        • 두 개를 더한 값이 다음 반복에서 사용할 새 n이 된다.
    • 최종 결과 반환 
          return cnt;
      }
      • 교환이 더 이상 불가능해졌을 때까지 누적된 콜라 수를 반환한다.

    시간 복잡도

    각 루프마다 n이 대략 a / b 배 비율로 줄어들거나 정해진 비율로 감소하므로,
    반복 횟수는 대략 로그 수준 또는 상수에 가까운 수준에 머문다.

    따라서 전체 시간 복잡도는 테스트 케이스 제약 내에서는 사실상 O(1)로 볼 수 있다.

    공간 복잡도

    추가로 사용하는 변수는 cnt, colaCount, changed 정도의 상수 개 뿐이므로
    공간 복잡도는 O(1)이다.

    주의

    • 초기 교환 불가 조건:
      • n < a인 경우 즉시 0을 반환하는 최적화가 있다.
      • 이 부분이 없더라도 while 조건이 바로 false가 되긴 하지만, 코드 가독성이 좋아진다.
    • while 조건:
      • while (n / a != 0)는 while (n >= a)와 동일한 의미다.
      • 나눗셈을 한 번 더 수행한다는 차이만 있고, 의미 상 문제는 없다.
    • 정수 범위:
      • 모든 변수를 int로 두었고, 문제에서 주어지는 입력 범위 내에서는 합산 결과가 int 범위를 크게 벗어나지 않도록 설정되어 있다.

    포인트 요약

    • 현재 빈 병 수 n으로 교환 가능한 최대 횟수 n / a를 계속 계산한다.
    • 한 번의 반복에서:
      • 얻는 콜라 수: changed = (n / a) * b
      • 누적 콜라 수: cnt += changed
      • 새로운 빈 병 수: n = n % a + changed
    • n / a == 0이 되는 순간, 더 이상 교환할 수 없으므로 반복을 종료한다.
    • 마신 콜라의 총 개수 cnt를 반환하면 정답이다.
