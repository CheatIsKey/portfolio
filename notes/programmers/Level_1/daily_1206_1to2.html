    문제 1 — 삼총사

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    삼총사 (프로그래머스 131705) 문제는
    정수 배열 number에서 서로 다른 인덱스의 원소 3개를 골라
    세 수의 합이 0이 되는 조합의 개수를 세는 문제다.

    • 서로 다른 인덱스 i, j, k (i < j < k)를 선택한다.
    • number[i] + number[j] + number[k] == 0이면 유효한 "삼총사" 한 개로 센다.
    • 모든 가능한 조합을 검사하고, 조건을 만족하는 경우의 수를 반환한다.

    이 풀이에서는 브루트 포스(완전 탐색)로
    가능한 3중 조합을 전부 확인한다.

    전략 개요

    • 배열 길이를 n이라 하면, 서로 다른 3개 인덱스를 고르는 모든 경우는 nC3개다.
    • 이를 직접 3중 for문으로 순회한다:
      • 바깥 루프: 첫 번째 인덱스 i를 0 ~ n-3까지.
      • 중간 루프: 두 번째 인덱스 j를 i+1 ~ n-2까지.
      • 안쪽 루프: 세 번째 인덱스 k를 j+1 ~ n-1까지.
    • 각 조합에 대해 number[i] + number[j] + number[k]를 계산하고, 합이 0이면 카운트 증가.
    • 모든 루프가 끝났을 때 누적된 카운트 cnt를 반환한다.

    핵심: 코드가 하는 일

    • 카운트 변수 초기화 
      public int solution(int[] number) {
          int cnt = 0;
      • cnt : 합이 0이 되는 "삼총사" 조합 개수를 세는 변수.
    • 3중 for문으로 모든 3원소 조합 탐색 
          for (int i = 0; i < number.length - 2; i++) {
              for (int j = i + 1; j < number.length - 1; j++) {
                  for (int k = j + 1; k < number.length; k++) {
                      if (number[i] + number[j] + number[k] == 0) {
                          cnt++;
                      }
                  }
              }
          }
      • i < number.length - 2:
        • 뒤에 두 개 인덱스 j, k가 더 필요하므로, 최소 2칸은 남기고 선택.
      • j = i + 1부터 시작:
        • 항상 i < j를 유지해 중복 조합(순서만 다른 같은 조합)을 제거.
      • k = j + 1부터 시작:
        • 항상 i < j < k를 만족시키며 세 번째 원소를 선택.
      • if (number[i] + number[j] + number[k] == 0):
        • 세 수의 합이 0이면 조건을 만족하는 "삼총사"이므로 cnt++.
    • 결과 반환 
          return cnt;
      }
      • 모든 가능한 3개 조합을 검사한 뒤, 누적된 개수 cnt를 반환한다.

    시간 복잡도

    배열 길이를 n이라고 하면,

    • 가능한 3원소 조합의 개수는 nC3 = n(n-1)(n-2)/6 개.
    • 코드는 모든 조합을 한 번씩 보므로, 시간 복잡도는 O(n^3)이다.

    실제 문제에서 주어지는 배열 길이가 크지 않기 때문에,
    이 방식으로도 충분히 통과할 수 있도록 제한이 설정되어 있다.

    공간 복잡도

    • 입력 배열 number 외에, cnt와 인덱스 변수 i, j, k만 사용한다.
    • 추가적인 자료구조를 사용하지 않으므로 O(1) 추가 공간.

    주의

    • 인덱스 범위:
      • 바깥 루프가 i < length - 2, 중간 루프가 j < length - 1, 안쪽 루프가 k < length이므로 ArrayIndexOutOfBoundsException을 피할 수 있다.
    • 중복 조합 방지:
      • i < j < k 순서로만 선택하므로, 같은 세 수를 다른 순서로 세는 일이 없다.
    • 합이 0인지만 확인:
      • 문제 조건 상, 단순히 합이 0인지 여부만 중요하고, 값의 위치나 크기 순서에는 제약이 없다.

    포인트 요약

    • 배열에서 서로 다른 3개의 인덱스를 고르는 모든 조합을 3중 for문으로 순회한다.
    • 각 조합에 대해 세 수의 합이 0이면 cnt를 1 증가시킨다.
    • i < j < k 형태로만 조합을 생성해 중복 카운트를 방지한다.
    • 추가 메모리 없이 단순 브루트 포스로 해결하는 구조이며, 시간 복잡도는 O(n^3), 공간 복잡도는 O(1)이다.

    문제 2 — 숫자 짝꿍

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자 짝꿍 (프로그래머스 131128) 문제는
    두 문자열 X, Y에 등장하는 숫자들을 가지고
    공통으로 가지고 있는 숫자들만 골라 만들 수 있는 가장 큰 수를 문자열로 반환하는 문제다.

    • X, Y는 숫자(문자)로만 이루어진 문자열이다.
    • 각 문자열에서 가져올 수 있는 숫자는 해당 문자열에 등장한 횟수만큼만 사용 가능하다.
    • 두 문자열에서 공통으로 사용할 수 있는 숫자의 개수는 각 숫자별 등장 횟수의 최솟값이다.
    • 이렇게 모은 숫자들로 만들 수 있는 수 중에서 가장 큰 수를 만들어 반환해야 한다.

    그리고 결과에 대한 특수 규칙이 있다.

    • 공통 숫자가 전혀 없으면 "-1"을 반환.
    • 공통 숫자가 0만 여러 개라면 "0"을 반환 (예: 공통이 000 → "0").

    이 풀이에서는 숫자(0~9)에 대해 각 문자열의 등장 횟수를 카운트한 뒤,
    9부터 0까지 내려가며 최소 등장 횟수만큼 붙여서 가장 큰 수를 만든다.

    전략 개요

    • 각 문자열의 숫자 빈도 세기
      • 0부터 9까지 각 숫자별로 등장 횟수를 저장할 배열 useX[10], useY[10]을 만든다.
      • X를 한 글자씩 순회하면서 useX[digit]++.
      • Y도 동일하게 useY[digit]++.
    • 공통 숫자로 만들 수 있는 가장 큰 수 생성
      • 9부터 0까지 내려가면서, 각 숫자 i에 대해 min(useX[i], useY[i])만큼 결과 문자열에 붙인다.
      • 내림차순(9 → 0)으로 붙이기 때문에 결과는 자연스럽게 가장 큰 수가 된다.
    • 예외 처리
      • 결과 문자열 길이가 0이면 → 공통 숫자 없음 → "-1".
      • 결과 문자열의 첫 글자가 '0'이면 → 공통 숫자가 전부 0 → "0".
      • 그 외에는 그대로 문자열을 반환.

    핵심: 코드가 하는 일

    • 각 문자열의 숫자 빈도 배열 구축 
      int[] useX = new int[10];
      int[] useY = new int[10];
      
      for (char c : X.toCharArray()) {
          useX[c - '0']++;
      }
      
      for (char c : Y.toCharArray()) {
          useY[c - '0']++;
      }
      • c - '0' : 문자 숫자를 실제 숫자(0~9) 인덱스로 변환.
      • useX[d], useY[d]에 각각의 등장 횟수를 누적.
    • 9부터 0까지 내려가며 공통 숫자 붙이기 
      StringBuilder sb = new StringBuilder();
      for (int i = 9; i >= 0; i--) {
          int min = Math.min(useX[i], useY[i]);
      
          if (min == 0) continue;
      
          for (int j = 0; j < min; j++) {
              sb.append(i);
          }
      }
      • 숫자 i가 X, Y에 각각 몇 번씩 등장하는지 확인하고, 그 중 최솟값만큼만 쓸 수 있다.
      • i를 9에서 0으로 내려가며 붙이므로, 가장 큰 숫자부터 채워진 결과를 얻는다.
    • 예외 케이스 처리 
      if (sb.length() == 0) {
          return "-1";
      }
      
      if (sb.charAt(0) == '0') {
          return "0";
      }
      
      return sb.toString();
      • sb.length() == 0:
        • 공통으로 쓸 수 있는 숫자가 하나도 없는 경우 → "-1".
      • sb.charAt(0) == '0':
        • 가장 앞자리(가장 큰 자리)가 0이라는 것은, 0만 여러 개 있는 경우 뿐이다.
        • 예: 000 → 실제로는 수 0 하나와 같으므로 "0"만 반환.
      • 그 외에는 sb에 쌓인 문자열이 바로 답이다.

    시간 복잡도

    • X 순회: 길이를 |X|라고 할 때 O(|X|).
    • Y 순회: 길이를 |Y|라고 할 때 O(|Y|).
    • 9부터 0까지 도는 루프: 항상 길이 10 → O(1).

    전체 시간 복잡도는 O(|X| + |Y|)로, 문자열 길이에 선형적으로 비례한다.

    공간 복잡도

    • 숫자 빈도 배열 useX[10], useY[10] → O(1).
    • 결과를 담는 StringBuilder는 최악의 경우 min(|X|, |Y|) 길이.

    따라서 전체 공간 복잡도는 O(|X| + |Y|) (입력 크기와 같은 정도)이다.

    주의

    • 숫자 빈도는 문자 기준
      • c - '0'로 인덱스를 계산할 때, 입력이 모두 숫자 문자라는 전제가 필요하다.
    • 결과가 0만으로 이루어진 경우
      • 예: "0000" → 실제로는 값 0 하나와 다르지 않기 때문에, 문제 조건에 따라 "0"만 반환해야 한다.
      • 이 코드는 sb.charAt(0) == '0'로 이를 처리한다.
    • 공통 숫자 없음 처리
      • 한 번도 sb.append(i)가 호출되지 않았다면 → sb.length() == 0 → "-1".

    포인트 요약

    • X, Y 각각에 대해 0~9 숫자의 등장 횟수를 카운트한다.
    • 각 숫자별로 min(사용 가능 개수)만큼 사용 가능하다.
    • 9부터 0까지 내려가며 최소 개수만큼 붙여 가장 큰 수를 만든다.
    • 공통 숫자가 없으면 "-1", 공통 숫자가 전부 0이면 "0"을 반환한다.
