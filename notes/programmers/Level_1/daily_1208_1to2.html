    문제 1 — 직사각형 별찍기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    직사각형 별찍기 (프로그래머스 12969) 문제는
    입력으로 주어지는 두 정수 a, b를 이용해
    가로로 a개의 '*' 문자, 세로로 b줄을 출력하는 간단한 별 출력 문제다.

    즉, a × b 크기의 직사각형 모양으로 별을 찍어야 한다.

    • 각 줄마다 a개의 *를 연속해서 출력하고,
    • 이 줄을 총 b번 반복해서 출력하면 된다.

    주어진 코드는 이를 중첩 반복문으로 구현하고 있다.

    • Scanner로 두 정수 a, b를 입력받는다.
    • 바깥 for 루프: 줄 수를 담당, 총 b번 반복.
    • 안쪽 for 루프: 한 줄에 a개의 별을 찍는 역할.
    • 한 줄의 별 출력이 끝나면 System.out.println()으로 줄바꿈.

    예시

    • 입력: 
      5 3
      동작:
      • 가로 a = 5 → 한 줄에 별 5개
      • 세로 b = 3 → 이런 줄을 3번 출력
      출력: 
      *****
      *****
      *****
    • 입력: 
      2 4
      출력: 
      **
      **
      **
      **

    시간 복잡도

    바깥 반복문이 b번, 안쪽 반복문이 각 줄마다 a번 실행되므로
    전체 출력 연산 수는 a × b에 비례한다.

    따라서 시간 복잡도는 O(a × b)이다.

    공간 복잡도

    입력으로 받은 두 정수와 반복문에서 사용하는 몇 개의 지역 변수만 사용한다.
    별 출력은 모두 System.out.print로 바로바로 찍기 때문에,
    별들을 따로 배열 등에 저장하지 않는다.

    따라서 추가 공간 복잡도는 O(1)이다.

    주의

    • 바깥 루프 변수 i는 행 수(줄 수)를 의미한다: 0 ~ b-1.
    • 안쪽 루프 변수 j는 각 줄에서 열 수(별 개수)를 의미한다: 0 ~ a-1.
    • 각 줄 끝마다 System.out.println();으로 줄바꿈을 반드시 해줘야
      직사각형 모양으로 제대로 출력된다.
    • 프로그래머스에서는 보통 Solution 클래스를 제출하고,
      main이 아니라 제공되는 입력/출력 환경을 사용한다는 점도 함께 기억해 두면 좋다.

    포인트 요약

    • a, b를 입력받아 a × b 크기의 별 직사각형을 출력하는 문제.
    • 이중 반복문으로 바깥에서 줄 수 b, 안에서 별 개수 a를 제어한다.
    • 각 줄마다 별을 a개 출력하고, 줄바꿈을 한 번 해주면 된다.
    • 시간 복잡도는 O(a × b), 공간 복잡도는 O(1)이다.

    문제 2 — x만큼 간격이 있는 n개의 숫자

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    x만큼 간격이 있는 n개의 숫자 (프로그래머스 12954) 문제는
    정수 x와 n이 주어졌을 때,
    x, 2x, 3x, ..., n×x 형태의 수열을 길이 n짜리 배열로 만들어 반환하는 문제다.

    즉, 결과 배열 answer의 i번째 원소(0-based)는 다음과 같다.

    answer[i] = x × (i + 1) (i = 0 ~ n-1)

    주어진 코드는 이것을 누적합 방식으로 구현하고 있다.

    • sum에 x를 계속 더해 가면서, 각 단계마다 현재 sum 값을 배열에 채우는 방식.

    코드 동작 흐름

    • 변수 초기화
      • int idx = 0; → 현재 채울 배열 인덱스
      • long sum = 0; → 현재까지의 누적합 (long으로 선언)
      • long[] answer = new long[n]; → 결과 배열
    • 반복문으로 수열 채우기 
      while (idx < n) {
          sum += x;
          answer[idx++] = sum;
      }
      • 매 반복마다 sum에 x를 더해준다.
      • 그 직후의 sum 값을 answer[idx]에 저장하고, idx를 1 증가.
      • 결과적으로:
        • 첫 번째 반복 후: sum = x → answer[0] = x
        • 두 번째 반복 후: sum = 2x → answer[1] = 2x
        • ...
        • n번째 반복 후: sum = n×x → answer[n-1] = n×x
    • 결과 반환
      • 반복이 끝나면 answer 배열을 그대로 반환.

    예시

    • x = 2, n = 5일 때
      • 1번째: sum = 2 → answer[0] = 2
      • 2번째: sum = 4 → answer[1] = 4
      • 3번째: sum = 6 → answer[2] = 6
      • 4번째: sum = 8 → answer[3] = 8
      • 5번째: sum = 10 → answer[4] = 10
      최종 결과: 
      [2, 4, 6, 8, 10]
    • x = -4, n = 3일 때
      • 1번째: sum = -4 → answer[0] = -4
      • 2번째: sum = -8 → answer[1] = -8
      • 3번째: sum = -12 → answer[2] = -12
      최종 결과: 
      [-4, -8, -12]

    시간 복잡도

    • 반복문이 idx = 0부터 n-1까지 한 번씩 돌기 때문에,
      전체 연산 수는 O(n)이다.

    공간 복잡도

    • 길이가 n인 long[] answer 배열 하나만 추가로 사용하므로
      공간 복잡도는 O(n)이다.
    • sum, idx 등은 상수 개수의 변수이므로 무시 가능.

    주의

    • 자료형:
      • x는 int이지만, sum과 answer는 long을 사용한다.
      • 이는 x의 크기와 n 값이 커지는 경우를 대비해, 곱셈이나 누적합에서의 오버플로를 방지하기 위함이다.
    • 반복 조건:
      • while (idx < n)으로 작성했기 때문에
        정확히 n개의 원소가 채워진다.

    포인트 요약

    • 정수 x와 n이 주어지면 x의 배수 수열 x, 2x, ..., n×x를 만든다.
    • sum에 x를 계속 더해 나가며, 각 단계의 sum을 배열에 저장하는 방식으로 구현.
    • 시간 복잡도는 O(n), 공간 복잡도는 O(n).
    • 값 범위를 고려해 결과 타입과 누적합은 long으로 처리한다.
