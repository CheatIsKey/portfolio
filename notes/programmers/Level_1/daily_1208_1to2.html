    문제 1 — 행렬의 덧셈

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    행렬의 덧셈 (프로그래머스 12950) 문제는
    같은 크기의 2차원 정수 배열 arr1, arr2가 주어졌을 때,
    각 위치의 원소를 더한 결과 행렬을 구하는 문제다.

    즉, 두 행렬이 모두 n × m 크기라면,
    결과 행렬 answer의 각 원소는 다음과 같이 정의된다.

    answer[i][j] = arr1[i][j] + arr2[i][j]

    주어진 코드는 새 배열을 만들지 않고,
    arr1을 곧바로 결과 행렬로 덮어 쓰는 방식으로 구현되어 있다.

    풀이 과정

    • 행의 개수: arr1.length
    • 열의 개수: arr1[i].length (각 행의 길이)

    이중 반복문을 사용하여 모든 인덱스 (i, j)에 대해 다음 연산을 수행한다.

    • 바깥 반복문: 각 행을 순회 
      for (int i = 0; i < arr1.length; i++) { ... }
    • 안쪽 반복문: 해당 행의 각 열을 순회 
      for (int j = 0; j < arr1[i].length; j++) {
          arr1[i][j] += arr2[i][j];
      }
      • arr1[i][j] += arr2[i][j]; 는 arr1[i][j] = arr1[i][j] + arr2[i][j]; 와 같다.
      • 즉, 각 위치의 두 원소를 더한 값을 arr1에 다시 저장한다.

    모든 반복이 끝난 후, 이미 합으로 덮어 써진 arr1을 곧바로 반환한다. 

    return arr1;

    문제에서 요구하는 것은 단순히 “두 행렬의 합”이므로,
    새 배열을 만들어도 되고, 기존 배열 중 하나를 재사용해도 상관없다.
    이 코드는 arr1을 재사용해 추가 메모리 사용을 줄이는 방식을 택한 것이다.

    예시

    • arr1 = [[1, 2], [3, 4]]
      arr2 = [[5, 6], [7, 8]]
      1. (0,0) 위치: 1 + 5 = 6
      2. (0,1) 위치: 2 + 6 = 8
      3. (1,0) 위치: 3 + 7 = 10
      4. (1,1) 위치: 4 + 8 = 12

      최종 결과 행렬은 [[6, 8], [10, 12]]가 되고,
      이 값들이 arr1에 덮어 써진 뒤 arr1이 반환된다.

    시간 복잡도

    행렬 크기가 n × m일 때,

    • 각 원소마다 덧셈을 한 번 수행하므로 전체 연산 수는 n × m개

    따라서 전체 시간 복잡도는 O(n × m)이다.

    공간 복잡도

    추가로 사용하는 것은 반복문 인덱스 변수뿐이며,
    결과를 기존 배열 arr1에 덮어 쓰기 때문에
    추가 공간은 O(1)이다.

    주의

    • 두 행렬의 크기가 같다는 전제가 있기 때문에, 인덱스 범위 체크는 따로 하지 않는다.
    • arr1을 그대로 수정하므로,
      원본 arr1이 필요하면 먼저 복사한 뒤 연산해야 한다.

    포인트 요약

    • 이중 for문으로 모든 (i, j) 위치를 순회한다.
    • arr1[i][j] += arr2[i][j]; 로 각 원소를 더해 arr1에 덮어 쓴다.
    • 계산이 끝난 arr1을 결과 행렬로 반환한다.
    • 시간 복잡도: O(n × m), 공간 복잡도: O(1) (추가 메모리 거의 없음).

    문제 2 — 핸드폰 번호 가리기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    핸드폰 번호 가리기 (프로그래머스 12948) 문제는
    문자열 phone_number에서 마지막 4자리만 그대로 두고,
    나머지 앞자리를 모두 * 문자로 가리는 문제다.

    예를 들어,

    • "01033334444" → "*******4444"
    • "027778888" → "*****8888"

    즉, 길이가 L인 문자열에 대해 인덱스 0 ~ L-5까지는 *로, L-4 ~ L-1은 원래 숫자 그대로 남겨두면 된다.

    풀이 과정

    • 결과를 만들 StringBuilder sb를 하나 만든다.
    • 인덱스 i = 0부터 phone_number.length() - 1까지 한 글자씩 순회한다.
    for (int i = 0; i < phone_number.length(); i++) {
          if (i < phone_number.length() - 4) {
              sb.append("*");
          } else {
              sb.append(phone_number.charAt(i));
          }
      }
    • 앞부분 처리:
      • i < phone_number.length() - 4 인 구간은 마지막 4자리를 제외한 모든 위치이다.
      • 이 구간에서는 숫자를 보지 않고 그냥 "*"를 붙인다.
    • 마지막 4자리 처리:
      • i ≥ phone_number.length() - 4인 경우, 원래 번호의 해당 문자 phone_number.charAt(i)를 그대로 붙인다.

    이렇게 한 바퀴를 돌면 sb 안에는
    “앞은 전부 *, 뒤 4글자는 원래 번호” 형태의 문자열이 완성된다. 

    return sb.toString();

    StringBuilder를 사용했기 때문에
    문자열을 매번 새로 만드는 것보다 효율적으로 결과 문자열을 생성할 수 있다.

    예시

    • phone_number = "01033334444"

      길이 L = 11이므로, L - 4 = 7
      • i = 0 ~ 6 → * 7개 추가 → "*******"
      • i = 7, 8, 9, 10 → '4','4','4','4' 그대로 추가
      최종 결과: "*******4444"

    시간 복잡도

    • 문자열 길이를 L이라고 하면,
      • for문에서 각 인덱스를 한 번씩만 방문 → L번 반복

    따라서 시간 복잡도는 O(L)이다.

    공간 복잡도

    • 추가로 사용하는 공간은 결과 문자열을 담는 StringBuilder 한 개뿐이며,
      길이는 원본과 동일하게 L이다.

    따라서 공간 복잡도는 O(L)이다.

    주의

    • 문제에서 항상 길이가 4 이상인 핸드폰 번호가 주어진다고 보장되므로,
      따로 “길이가 4 미만”인 경우를 처리하지 않아도 된다.
    • length() - 4 기준으로 앞/뒤를 잘 나누는지만 조심하면 된다.

    포인트 요약

    • 핵심은 “뒤 4글자만 그대로, 앞은 모두 *로 치환”이다.
    • 인덱스 0 ~ (L-5) → *, (L-4 ~ L-1) → 원래 문자.
    • StringBuilder로 한 번에 문자열을 만들어 반환한다.
