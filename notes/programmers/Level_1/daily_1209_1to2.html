    문제 1 — 하샤드 수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    하샤드 수 (프로그래머스 12947) 문제는
    어떤 정수 x가 자릿수의 합으로 나누어떨어지는지를 판별하는 문제다.
    이런 수를 하샤드 수(또는 니븐 수)라고 부른다.

    정의를 정리하면:

    • x의 각 자릿수를 모두 더해서 sum을 만든다.
    • x % sum == 0이면 하샤드 수 → true 반환
    • 나누어떨어지지 않으면 하샤드 수가 아님 → false 반환

    자릿수의 합은 나머지와 나눗셈 연산으로 간단히 구할 수 있다.

    • num = x로 복사해 두고,
    • 반복하면서:
      • sum += num % 10; → 마지막 자릿수 더하기
      • num /= 10; → 마지막 자릿수 제거
      • num이 0이 될 때까지 반복

    최종적으로 sum이 완성되면,
    x % sum == 0인지 검사해서 그대로 반환하면 된다.

    예시

    • x = 10
      • 자릿수 합: 1 + 0 = 1
      • 10 % 1 = 0 → 나누어떨어짐 → 하샤드 수 → true
    • x = 12
      • 자릿수 합: 1 + 2 = 3
      • 12 % 3 = 0 → 나누어떨어짐 → 하샤드 수 → true
    • x = 11
      • 자릿수 합: 1 + 1 = 2
      • 11 % 2 = 1 → 나머지 존재 → 하샤드 수 아님 → false

    시간 복잡도

    자릿수 개수를 d라고 하면,

    • while (num > 0) 루프에서 각 자릿수를 한 번씩만 보므로 O(d)
    • 이 문제에서 x의 자리 수는 매우 작기 때문에 사실상 상수 시간에 가깝다.

    공간 복잡도

    추가로 사용하는 변수는 sum, num 뿐이므로
    공간 복잡도는 O(1)이다.

    주의

    • num을 줄여가며 자릿수 합을 계산하므로, 원래 값 x를 바로 나누지 않고 복사본을 쓰는 것이 중요하다.
    • 문제 조건상 x는 양의 정수이므로, sum은 항상 0보다 크고 나눗셈에서 0으로 나누는 일은 발생하지 않는다.

    문제 2 — 평균 구하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    평균 구하기 (프로그래머스 12944) 문제는
    정수 배열 arr가 주어졌을 때, 모든 원소의 평균값을 double 형으로 반환하는 문제다.

    요구사항을 정리하면:

    • 배열의 모든 원소를 더해서 sum을 구한다.
    • sum / arr.length를 계산해 평균을 구한다.
    • 이때 실수 평균을 구해야 하므로, 정수 나눗셈을 피하기 위해 형 변환이 필요하다.

    코드에서는 다음과 같은 순서로 처리한다.

    1. sum을 0으로 초기화한다.
    2. for (int i : arr) 향상된 for문으로 배열을 순회하면서 sum += i;로 합을 구한다.
    3. 마지막에 (double) sum / arr.length; 를 반환한다.
      • (double) sum으로 먼저 실수 형변환을 한 뒤,
      • 정수 arr.length로 나누면 실수 나눗셈이 수행된다.

    만약 (double) 캐스팅 없이 sum / arr.length만 사용하면,
    둘 다 int이기 때문에 정수 나눗셈이 되어 소수점 이하가 잘려 버린다.

    예시

    • arr = [1, 2, 3, 4]
      • sum = 1 + 2 + 3 + 4 = 10
      • arr.length = 4
      • (double) sum / arr.length = 10.0 / 4 = 2.5
    • arr = [5, 5]
      • sum = 10
      • length = 2
      • 10.0 / 2 = 5.0

    시간 복잡도

    배열 길이를 n이라고 하면,

    • 합을 구하는 for 루프는 모든 원소를 한 번씩만 순회 → O(n)
    • 나눗셈 연산은 한 번만 수행 → O(1)

    전체 시간 복잡도는 O(n)이다.

    공간 복잡도

    입력 배열 arr 외에 사용하는 변수는 sum과 반환값뿐이라
    추가 공간은 O(1)이다.

    주의

    • 형 변환 위치:
      • (double) (sum / arr.length) 처럼 뒤에 캐스팅하면 이미 정수 나눗셈이 끝난 뒤라
        소수점 이하가 잘려버린 값을 실수로 바꾸는 것뿐이다.
      • 반드시 (double) sum / arr.length 처럼 피제수 쪽을 먼저 실수로 변환해야 한다.
