    문제 1 — 콜라츠 추측

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    이 문제는 주어진 정수 num에 대해, 콜라츠 추측 규칙을 따라가면서
    1이 될 때까지 몇 번 연산을 수행하는지를 구하는 문제다.
    단, 연산 횟수가 500번을 넘으면 -1을 반환해야 한다.

    • 콜라츠 규칙
      • 현재 수 n이 짝수라면: n = n / 2
      • 현재 수 n이 홀수라면: n = n * 3 + 1
      • 이 과정을 반복하다가 n == 1이 되면 멈춘다.
    • 최대 500번까지만 연산해 보고,
      • 그 안에 1이 되면 걸린 횟수를 반환,
      • 끝까지 1이 안 되면 -1 반환.

    코드 흐름 요약

    • 초기 설정
      • long n = num;으로 받는 이유:
        • n = n * 3 + 1 과정에서 int 범위를 넘을 수 있으므로 long으로 안전하게 계산.
      • 처음부터 num == 1이면 연산을 안 해도 되므로 0을 바로 반환.
    • 최대 500번까지 반복 
      for (int i = 0; i < 500; i++) {
          if (n % 2 == 0) {
              n /= 2;
          } else {
              n = n * 3 + 1;
          }
      
          if (n == 1) {
              return i + 1;
          }
      }
      • 매 반복마다 현재 수 n이 짝수인지 홀수인지에 따라 콜라츠 규칙 적용.
      • 연산을 한 번 수행한 후, n == 1이면
        • 이번에 수행한 횟수 = i + 1 이므로 그 값을 반환.
      • i는 0부터 시작하므로, 실제 연산 횟수는 항상 i + 1이 된다.
    • 500번 안에 1이 안 됐을 때 
      return -1;
      • for 루프를 다 돌고도 n == 1이 한 번도 안 나왔다면
        500회를 초과해도 1이 되지 않는다고 보고 -1을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도:
      • 최대 500번만 반복하므로 O(1) (상수 시간).
    • 공간 복잡도:
      • 추가로 사용하는 변수는 long n과 반복용 변수 정도이므로 O(1).

    문제 2 — 최대공약수와 최소공배수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    최대공약수와 최소공배수 (프로그래머스 12940) 문제는
    두 자연수 n, m이 주어졌을 때, 최대공약수(GCD)와 최소공배수(LCM)를 구해
    [최대공약수, 최소공배수] 형태의 배열로 반환하는 문제다.

    • 최대공약수(GCD):
      • 두 수가 동시에 나누어떨어지는 양의 정수 중 가장 큰 값.
      • 코드에서는 유클리드 호제법으로 계산한다.
    • 최소공배수(LCM):
      • 두 수의 공통 배수 중 가장 작은 양의 정수.
      • 공식: lcm(n, m) = n * m / gcd(n, m)

    코드 핵심 로직

    • solution 메서드
      • int gcd = gcd(n, m);
        → 아래에 정의된 재귀 함수 gcd로 최대공약수를 구한다.
      • int lcm = n * m / gcd;
        → 곱 / GCD 공식을 사용해 최소공배수를 구한다.
      • return new int[]{gcd, lcm};
        → 결과를 [최대공약수, 최소공배수] 배열로 반환.
    • gcd (유클리드 호제법) 
      private int gcd(int n, int m) {
          if (m == 0) return n;
          return gcd(m, n % m);
      }
      • 아이디어:
        • gcd(a, b)에서 b == 0 이면, 최대공약수는 a.
        • 그렇지 않으면 gcd(b, a % b)로 문제 크기를 줄여서 재귀 호출.
      • 예: gcd(12, 8)
        1. gcd(12, 8) → gcd(8, 12 % 8 = 4)
        2. gcd(8, 4) → gcd(4, 8 % 4 = 0)
        3. gcd(4, 0) → 4 반환

    예시

    • n = 12, m = 18일 때
      1. gcd(12, 18) = 6
      2. lcm = 12 * 18 / 6 = 216 / 6 = 36
      3. 반환 값: [6, 36]

    시간 · 공간 복잡도

    • 시간 복잡도
      • 유클리드 호제법은 O(log(min(n, m))) 시간에 동작.
      • 나머지 연산은 상수 시간이므로 전체도 O(log(min(n, m)) 수준.
    • 공간 복잡도
      • 재귀 깊이가 O(log(min(n, m))이지만, 별도 큰 자료구조는 없으므로
        공간 복잡도는 사실상 O(1)로 볼 수 있다.
