    문제 1 — 짝수와 홀수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    이 문제는 주어진 정수 num이 짝수인지 홀수인지 판별해서
    짝수면 "Even", 홀수면 "Odd" 문자열을 반환하는 매우 단순한 문제다.

    • 짝수 조건: num % 2 == 0
    • 홀수 조건: 위 조건이 아니라면 모두 홀수로 간주
    • 조건에 따라 삼항 연산자로 "Even" 또는 "Odd"를 돌려준다.

    코드가 하는 일 

    class Solution {
          public String solution(int num) {
              return num % 2 == 0 ? "Even" : "Odd";
          }
      }
    • num % 2 == 0
      • num을 2로 나눈 나머지가 0이면 짝수이므로 "Even"을 반환한다.
    • ?: 삼항 연산자
      • 조건이 참이면 왼쪽 값, 거짓이면 오른쪽 값을 선택한다.
      • 여기서는
        조건 ? "Even" : "Odd" 형태로 사용해서
        한 줄에 깔끔하게 짝/홀 판별 로직을 표현하고 있다.

    최종적으로 입력 받은 num이 짝수면 "Even", 아니면 "Odd"를 그대로 반환하면 된다.

    문제 2 — 제일 작은 수 제거하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    이 문제는 정수 배열 arr에서 가장 작은 값들을 모두 제거한 배열을 반환하는 문제다.
    단, 제거하고 나서 배열이 비게 되면 [-1] 하나만 담긴 배열을 반환해야 한다.

    • 원소가 1개뿐인 경우 → 무조건 그 값이 최소이므로 제거하면 빈 배열 → {-1} 반환.
    • 그 외의 경우
      • 배열에서 최소값을 한 번 찾고,
      • 그 최소값과 다른 값들만 모아서 새 배열을 만든다.
      • 이때 최소값이 여러 개면 모두 제거된다.

    코드가 하는 일 

    import java.util.*;
      
      class Solution {
          public int[] solution(int[] arr) {
              // 1. 길이가 1이면 제거 후 비게 되므로 [-1] 반환
              if (arr.length == 1) return new int[]{-1};
      
              // 2. 최소값(min) 찾기
              int min = arr[0];
              for (int i : arr) {
                  if (i < min) min = i;
              }
      
              // 3. 최소값이 아닌 원소 개수(cnt) 세기
              int cnt = 0;
              for (int i : arr) {
                  if (i != min) cnt++;
              }
      
              // 4. 모두 최소값이었다면 (cnt == 0) → 제거 후 비어버리므로 [-1] 반환
              if (cnt == 0) return new int[]{-1};
      
              // 5. 최소값이 아닌 원소들만 담을 배열 생성
              int[] answer = new int[cnt];
              int idx = 0;
              for (int i : arr) {
                  if (i != min) {
                      answer[idx++] = i;
                  }
              }
      
              return answer;
          }
      }
    • 최소값 탐색
      • min을 arr[0]으로 두고, 전체를 돌며 가장 작은 값을 갱신한다.
    • 새 배열 크기 결정
      • 최소값이 아닌 원소만 남겨야 하므로, 먼저 그런 원소의 개수를 cnt로 센다.
      • cnt == 0이면 전부 최소값 → 제거하면 빈 배열 → {-1} 반환.
    • 결과 배열 채우기
      • 길이가 cnt인 answer 배열을 만들고,
      • 다시 한 번 순회하면서 최소값이 아닌 값만 앞에서부터 차례로 채워 넣는다.
      • 원래의 순서가 유지된 상태로 최소값만 제거된다.

    시간 복잡도는 배열을 여러 번 순회하지만 모두 O(n) 수준이라, 전체적으로 O(n)에 해결된다.
