    문제 1 — 이상한 문자 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    이상한 문자 만들기 (Programmers 12930)는
    문자열을 단어 단위로 보면서, 각 단어의 0번째, 2번째, 4번째… 문자는 대문자,
    1번째, 3번째, 5번째… 문자는 소문자로 바꿔서 결과를 만드는 문제다.

    • 핵심은 공백을 만나면 “단어 인덱스”를 0으로 초기화하는 것.
    • 공백은 그대로 유지해야 하므로, 입력 문자열을 왼쪽부터 끝까지 한 글자씩 처리하는 방식이 안전하다.

    전략 개요

    • StringBuilder로 결과 문자열을 만든다.
    • idx를 “현재 단어에서 몇 번째 글자인지”를 나타내는 카운터로 둔다.
    • 문자를 하나씩 확인하면서
      • 공백이면: 그대로 추가하고 idx = 0
      • 문자면: idx가 짝수면 대문자, 홀수면 소문자로 바꾸고 idx++

    코드가 하는 일

    • 단어 인덱스 관리(공백 처리) 
      if (c == ' ') {
          sb.append(' ');
          idx = 0;
      }
      • 공백은 그대로 유지해야 하고, 공백 이후에는 새 단어가 시작되므로 인덱스를 0으로 초기화한다.
    • 짝/홀 인덱스에 따라 대소문자 변환 
      else {
          if (idx % 2 == 0) sb.append(Character.toUpperCase(c));
          else sb.append(Character.toLowerCase(c));
          idx++;
      }
      • idx가 짝수(0,2,4,...)면 대문자
      • idx가 홀수(1,3,5,...)면 소문자
      • 문자를 처리했으니 단어 인덱스를 1 증가

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열을 한 번만 순회하므로 O(L) (L = 문자열 길이)
    • 공간 복잡도
      • 결과를 담는 StringBuilder만 사용하므로 O(L)

    문제 2 — 약수의 합

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    약수의 합 (Programmers 12928)는
    정수 n의 모든 약수의 합을 구하는 문제다.

    • 약수는 항상 짝(pair)으로 등장한다: i가 약수면 n / i도 약수.
    • 그래서 1부터 √n까지만 검사해도 전체 약수를 모두 더할 수 있다.
    • 단, n이 완전제곱수면 i == n/i가 되는 지점(√n)은 한 번만 더해야 한다.

    전략 개요

    • i를 1부터 증가시키면서 i*i <= n까지만 확인한다.
    • n % i == 0이면
      • sum += i
      • i*i == n이 아니라면 짝 약수인 n/i도 더한다.

    코드가 하는 일

    • √n까지만 약수 탐색 
      for (int i = 1; i * i <= n; i++) {
          if (n % i == 0) {
              sum += i;
      
              if (i * i == n) continue;
              else sum += n / i;
          }
      }
      • i가 약수면 i를 더하고, 짝 약수 n/i도 더한다.
      • 단, i*i == n이면 i와 n/i가 같아서 중복이므로 한 번만 더한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • O(√n) (1부터 √n까지만 검사)
    • 공간 복잡도
      • 추가 배열 없이 합계 변수만 쓰므로 O(1)
