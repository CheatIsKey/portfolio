    문제 1 — 시저 암호

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    카이사르 암호 (Programmers 12926)는
    문자열에서 알파벳을 n칸만큼 밀어서 새로운 문자로 바꾸는 문제다. 공백은 그대로 두고, 알파벳은 대문자/소문자 범위를 각각 유지하면서 순환(=z 다음은 a)하도록 처리하면 된다.

    • n은 26을 주기로 반복되므로 n %= 26으로 줄여도 된다.
    • 문자가
      • 공백이면 그대로 추가
      • 소문자면 'a' 기준으로 밀고 26으로 나머지 처리
      • 대문자면 'A' 기준으로 밀고 26으로 나머지 처리

    전략 개요

    • 1) n 정규화
      • n %= 26으로 불필요하게 큰 이동을 줄인다.
    • 2) 문자열 순회하며 변환
      • 각 문자에 대해 공백은 그대로 append.
      • 소문자는 ('a' + (c - 'a' + n) % 26)로 변환.
      • 대문자는 ('A' + (c - 'A' + n) % 26)로 변환.
    • 3) StringBuilder 결과 반환

    코드가 하는 일

    • 이동 값 정리 
      n %= 26;
      • 26칸 이동은 원래 문자로 돌아오므로, n을 0~25로 줄여 처리한다.
    • 문자별 처리 (공백 / 소문자 / 대문자) 
      for (char c : s.toCharArray()) {
          if (c == ' ') {
              sb.append(' ');
              continue;
          }
      
          if ('a' <= c && c <= 'z') {
              sb.append((char) ('a' + (c - 'a' + n) % 26));
          } else if ('A' <= c && c <= 'Z'){
              sb.append((char) ('A' + (c - 'A' + n) % 26));
          } else {
              sb.append(c);
          }
      }
      • 공백은 암호화 대상이 아니므로 그대로 유지한다.
      • 소문자는 c - 'a'로 0~25로 바꾼 뒤 n을 더하고 26으로 순환한다.
      • 대문자도 동일하게 'A' 기준으로 처리한다.
      • 문제 조건상 알파벳/공백만 오지만, 안전하게 그 외 문자는 그대로 붙이도록 처리했다.
    • 결과 반환 
      return sb.toString();
      • 누적한 결과 문자열을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 L이라 하면 한 번 순회하므로 O(L)
    • 공간 복잡도
      • 결과를 담는 StringBuilder 때문에 O(L)

    문제 2 — 문자열을 정수로 바꾸기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열을 정수로 바꾸기 (Programmers 12925)는
    "+123", "-123" 처럼 부호가 포함될 수도 있는 문자열을 정수로 변환하는 문제다.

    • 자바에서는 Integer.parseInt(String)가 부호까지 포함해서 정수 변환을 처리해준다.
    • 따라서 별도 로직 없이 그대로 파싱해서 반환하면 끝이다.

    전략 개요

    • 1) 문자열 입력: 매개변수 s를 받는다.
    • 2) 정수 변환: Integer.parseInt(s)로 변환한다.
    • 3) 반환: 변환된 정수를 그대로 반환한다.

    코드가 하는 일

    • 문자열 → int 변환 
      return Integer.parseInt(s);
      • s가 "123", "+123", "-123" 형태여도 정상적으로 정수로 변환된다.
      • 문제 조건에서 변환 가능한 값만 들어오므로 예외 처리는 필요 없다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 L이라 하면 파싱은 한 번 훑으므로 O(L)
    • 공간 복잡도
      • 추가 자료구조 없이 변환만 하므로 O(1)
