    문제 1 — 수박수박수박수박수박수?

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    수박수박수박수박수박수? (프로그래머스 12922)는
    길이 n의 문자열을 만들되, 인덱스가 짝수면 "수", 홀수면 "박"을 붙여서 "수박수박..." 형태로 완성하는 구현 문제다.

    • n번 반복하면서 문자열을 이어 붙이면 된다.
    • 매번 문자열을 더하면 비효율적일 수 있으니 StringBuilder를 사용한다.

    전략 개요

    • 반복
      • i = 0부터 n-1까지 반복한다.
      • i % 2 == 0이면 "수", 아니면 "박"을 붙인다.
    • 출력 문자열 생성
      • StringBuilder에 누적 후 toString()으로 반환한다.

    코드가 하는 일

    • 문자열 누적 
      StringBuilder sb = new StringBuilder();
      
      for (int i = 0; i < n; i++) {
          if (i % 2 == 0) {
              sb.append("수");
          } else {
              sb.append("박");
          }
      }
      
      return sb.toString();
      • i가 짝수면 "수", 홀수면 "박"을 차례대로 붙여 길이 n의 패턴 문자열을 만든다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • n번 반복하면서 append → O(n)
    • 공간 복잡도
      • 길이 n 문자열을 저장 → O(n)

    문제 2 — 소수 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    소수 찾기 (프로그래머스 12921)는
    1부터 n까지의 수 중에서 소수(prime)의 개수를 구하는 문제다.

    • 소수 판별을 매번 O(√n)으로 하면 전체가 느려질 수 있다.
    • 여기서는 에라토스테네스의 체로 합성수를 미리 표시해서 빠르게 개수를 센다.

    전략 개요

    • 합성수 표시 배열
      • composite[i] = true면 i는 합성수(소수 아님)로 처리한다.
      • 초기에는 전부 false이며, 표시되지 않은 수는 소수로 본다.
    • 체(Sieve) 수행
      • i를 2부터 √n까지 돌면서,
      • i가 아직 합성수가 아니면(composite[i] == false)
      • i*i부터 시작해 i의 배수를 합성수로 표시한다.
    • 소수 개수 세기
      • 2부터 n까지 중 composite[i] == false인 수를 카운트한다.

    코드가 하는 일

    • 체 배열 생성 
      boolean[] composite = new boolean[n + 1];
      • 0..n까지 합성수 여부를 저장할 배열을 만든다.
    • 합성수 마킹 (에라토스테네스의 체) 
      for (int i = 2; (long)i * i <= n; i++) {
          if (!composite[i]) {
              for (int j = i * i; j <= n; j += i) {
                  composite[j] = true;
              }
          }
      }
      • i가 소수로 판정되면(composite[i] == false)
      • i*i부터 시작해 배수들을 합성수로 표시한다.
      • i*i 이전의 배수는 더 작은 소수에서 이미 걸러지므로 시작점을 i*i로 잡는다.
      • 조건에서 (long)i * i로 캐스팅해 오버플로우를 방지한다.
    • 소수 개수 카운트 
      int count = 0;
      for (int i = 2; i <= n; i++) {
          if (!composite[i]) count++;
      }
      return count;
      • 2..n 범위에서 합성수 표시가 안 된 값만 세면 소수의 개수가 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 에라토스테네스의 체: O(n log log n) 수준
      • 소수 개수 세기: O(n)
    • 공간 복잡도
      • boolean[n+1] 사용 → O(n)
