    문제 1 — 서울에서 김서방 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    서울에서 김서방 찾기 (프로그래머스 12919)는
    문자열 배열 seoul 안에서 문자열 "Kim"이 등장하는 위치(인덱스)를 찾아, "김서방은 X에 있다" 형식으로 반환하는 문제다.

    • seoul 배열은 사람 이름들로 구성되어 있고, 그 중 하나는 반드시 "Kim"이다.
    • 정답은 "Kim"이 있는 인덱스를 찾아 그대로 문자열에 넣어주면 된다.

    전략 개요

    • 순회
      • 0부터 끝까지 배열을 돌면서 seoul[i].equals("Kim")인지 확인한다.
      • 찾는 순간 그 인덱스를 저장하고 반복을 종료한다.
    • 문자열 생성
      • "김서방은 " + pos + "에 있다" 형태로 만들어 반환한다.

    코드가 하는 일

    • Kim 위치 찾기 
      for (int i = 0; i < seoul.length; i++) {
          if (seoul[i].equals("Kim")) {
              pos = i;
              break;
          }
      }
      • equals로 문자열을 비교해서 정확히 "Kim"인지 확인한다.
      • 찾았으면 pos에 인덱스를 저장하고 더 볼 필요가 없으니 break.
    • 정답 문자열 반환 
      return "김서방은 " + pos + "에 있다";
      • 문제에서 요구하는 포맷 그대로 인덱스를 끼워 넣어 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열을 한 번 순회하므로 O(N)
    • 공간 복잡도
      • 추가로 사용하는 변수는 상수 개라서 O(1)

    문제 2 — 문자열 다루기 기본

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 다루기 기본 (프로그래머스 12918)은
    주어진 문자열 s가 길이가 4 또는 6이고, 동시에 모든 문자가 숫자(0~9)로만 이루어져 있는지 판별하는 문제다.

    • 길이가 4나 6이 아니면 무조건 false.
    • 길이가 조건을 만족하더라도, 문자 하나라도 숫자가 아니면 false.
    • 둘 다 만족하면 true.

    전략 개요

    • 1) 길이 체크
      • s.length()가 4 또는 6인지 먼저 확인한다.
      • 아니면 바로 false 반환.
    • 2) 숫자만 있는지 검사
      • 문자 배열로 순회하면서 각 문자가 '0' ~ '9' 범위인지 확인한다.
      • 범위를 벗어나면 바로 false.
    • 3) 모두 통과하면 true

    코드가 하는 일

    • 길이 조건 확인 
      int length = s.length();
      if (length != 4 && length != 6) return false;
      • 길이가 4도 아니고 6도 아니면 조건 불만족이므로 즉시 false.
    • 문자 하나씩 숫자인지 검사 
      for (char c : s.toCharArray()) {
          if (c < '0' || '9' < c) return false;
      }
      • 각 문자가 ASCII 기준으로 '0'보다 작거나 '9'보다 크면 숫자가 아니다.
      • 하나라도 걸리면 즉시 false.
    • 모두 통과하면 true 
      return true;
      • 길이 조건도 만족했고, 모든 문자가 숫자였으면 최종적으로 true.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열을 한 번 순회하므로 O(N) (N은 문자열 길이)
    • 공간 복잡도
      • 추가 변수만 사용하므로 O(1)
