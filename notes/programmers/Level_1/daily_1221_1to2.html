    문제 1 — 문자열 내림차순으로 배치하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 내림차순으로 배치하기 (프로그래머스 12917)는
    문자열 s에 들어있는 문자들을 내림차순(큰 문자 → 작은 문자)으로 정렬해서 하나의 문자열로 반환하는 문제다.

    • 대문자와 소문자는 ASCII 코드값이 다르므로, 단순 내림차순 정렬이면 소문자가 대문자보다 뒤(=더 큼)에 위치한다.
    • 예: 'z' > 'a' > 'Z' > 'A' (ASCII 기준)

    전략 개요

    • 카운팅 배열을 이용해 정렬을 구현한다.
      • cnt[문자코드]에 해당 문자가 몇 번 등장했는지 누적
      • 그 뒤 큰 문자코드부터 역순으로 내려오며 등장 횟수만큼 붙여서 결과 생성
    • 문자 종류가 ASCII 범위(여기서는 0~127)라고 가정하면, 정렬 대신 카운팅으로 빠르게 처리 가능하다.

    코드가 하는 일

    • 1) 문자 등장 횟수 세기 
      int[] cnt = new int[128];
      
      for (char c : s.toCharArray()) {
          cnt[c]++;
      }
      • 각 문자를 순회하면서 문자 코드값(c)을 인덱스로 사용해 개수를 누적한다.
      • 예를 들어 'a'가 3번 나오면 cnt['a'] = 3.
    • 2) 큰 문자부터 역순으로 문자열 만들기 
      StringBuilder sb = new StringBuilder();
      
      for (int i = cnt.length - 1; i >= 0; i--) {
          while (cnt[i]-- > 0) sb.append((char) i);
      }
      • i를 127부터 0까지 내려오며, 해당 문자가 존재하면(cnt[i] > 0) 그 횟수만큼 (char)i를 붙인다.
      • 결과적으로 ASCII 코드 기준 내림차순 정렬 효과가 난다.
    • 3) 완성된 문자열 반환 
      return sb.toString();

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자 카운팅: O(N)
      • 역순 생성: O(128 + N) → 상수 128은 무시 가능
      • 총합: O(N)
    • 공간 복잡도
      • 카운팅 배열 cnt[128] + 결과 문자열: O(N)

    문제 2 — 문자열 내 p와 y의 개수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 내 p와 y의 개수 (프로그래머스 12916)는
    문자열 s에서 p(또는 P)의 개수와 y(또는 Y)의 개수를 비교해서, 두 개수가 같으면 true, 다르면 false를 반환하는 문제다.

    • 대소문자를 구분하지 않으므로 p/P, y/Y를 모두 같은 문자로 취급한다.
    • p와 y가 둘 다 0개인 경우도 “같다”이므로 true가 된다.

    전략 개요

    • 문자열을 한 번 순회하면서
      • p/P면 pCnt 증가
      • y/Y면 yCnt 증가
    • 순회가 끝나면 pCnt == yCnt인지 비교해서 결과를 반환한다.

    코드가 하는 일

    • 1) 카운터 준비 
      int pCnt = 0;
      int yCnt = 0;
      • p 개수와 y 개수를 각각 누적할 변수를 만든다.
    • 2) 문자열 순회하며 개수 세기 
      for (char c : s.toCharArray()) {
          if (c == 'p' || c == 'P') {
              pCnt++;
          } else if (c == 'y' || c == 'Y') {
              yCnt++;
          }
      }
      • 각 문자를 확인해서 p/P면 pCnt++, y/Y면 yCnt++.
      • 그 외 문자는 무시한다.
    • 3) 최종 비교 후 반환 
      return pCnt == yCnt;
      • p와 y의 개수가 같으면 true, 다르면 false.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 N이라 할 때, 한 번 순회하므로 O(N).
    • 공간 복잡도
      • 카운터 변수만 사용하므로 O(1).
