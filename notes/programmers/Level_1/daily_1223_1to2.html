    문제 1 — 문자열 내 마음대로 정렬하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    문자열 내 마음대로 정렬하기 (프로그래머스 12915)는
    문자열 배열을 n번째 문자 기준 오름차순으로 정렬하되, n번째 문자가 같다면 사전순(전체 문자열)으로 정렬하는 문제다.

    • 정렬 우선순위
      1. strings[i].charAt(n) (n번째 문자) 오름차순
      2. n번째 문자가 같으면 strings[i] (전체 문자열) 오름차순
    • 자바에서는 Arrays.sort에 Comparator(람다)를 넘기면 조건 정렬을 바로 구현할 수 있다.

    전략 개요

    • 정렬 기준을 Comparator로 정의
      • 먼저 두 문자열의 n번째 문자를 비교한다.
      • 값이 다르면 그 결과로 정렬을 결정한다.
      • 값이 같으면 a.compareTo(b)로 사전순 정렬한다.
    • 정렬이 끝난 배열을 그대로 반환한다.

    코드가 하는 일

    • 핵심 정렬 로직 
      Arrays.sort(strings, (a, b) -> {
          int compare = Character.compare(a.charAt(n), b.charAt(n));
          if (compare != 0) return compare;
          return a.compareTo(b);
      });
      • Character.compare(a.charAt(n), b.charAt(n))
        • 두 문자열의 n번째 문자를 비교해 오름차순 정렬 기준을 만든다.
      • n번째 문자가 다르면(compare != 0) 그 결과로 바로 정렬이 끝난다.
      • n번째 문자가 같다면 a.compareTo(b)로 전체 문자열 사전순 정렬을 적용한다.
    • 반환 
      return strings;
      • Arrays.sort는 원본 배열을 정렬하므로, 정렬된 strings를 그대로 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 N개 정렬이므로 O(N log N).
      • 비교 함수에서 charAt은 O(1), compareTo는 최악의 경우 문자열 길이만큼 비교한다.
    • 공간 복잡도
      • 정렬 과정에서의 내부 버퍼 등을 제외하면 추가 저장은 거의 없으므로 O(1) 수준(입력 배열 재사용).

    문제 2 — 두 정수 사이의 합

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 정수 사이의 합 (프로그래머스 12912)는
    두 정수 a, b가 주어졌을 때, a와 b 사이(포함)의 모든 정수를 더한 값을 반환하는 문제다.

    • a가 b보다 클 수도, 작을 수도 있으므로 범위를 먼저 정리해야 한다.
    • a == b인 경우는 그 수 자체가 답이다.
    • 합이 커질 수 있으므로 누적 합은 long으로 처리하는 게 안전하다.

    전략 개요

    • a == b면 바로 a를 반환한다.
    • Math.max, Math.min으로 큰 값/작은 값을 구해 범위를 고정한다.
    • 작은 값부터 큰 값까지 1씩 증가시키며 누적 합을 더한다.

    코드가 하는 일

    • 1) 같은 수면 바로 반환 
      if (a == b) return a;
      • 사이의 정수가 하나뿐이므로 더할 것도 없이 a(= b)가 정답이다.
    • 2) 범위 정리 (작은 값 ~ 큰 값) 
      int num1 = Math.max(a, b);
      int num2 = Math.min(a, b);
      • a, b의 대소 관계와 상관없이 항상 num2부터 num1까지로 순회할 수 있게 만든다.
    • 3) 누적 합 계산 
      long sum = 0;
      for (int i = num2; i <= num1; i++) {
          sum += i;
      }
      • num2부터 num1까지 포함해서 모두 더한다.
      • 누적 변수는 long으로 두어 합이 커지는 케이스를 대비한다.
    • 4) 결과 반환 
      return sum;
      • 구한 누적 합을 그대로 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 두 수의 차이를 D = |a - b|라고 하면, 구간을 한 번 순회하므로 O(D).
    • 공간 복잡도
      • 누적 변수 등 상수 크기만 사용하므로 O(1).
