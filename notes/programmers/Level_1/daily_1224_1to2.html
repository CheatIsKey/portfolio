    문제 1 — 나누어 떨어지는 숫자 배열

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    나누어 떨어지는 숫자 배열 (프로그래머스 12910)은
    정수 배열 arr에서 divisor로 나누어 떨어지는 값만 골라 오름차순 정렬해서 반환하는 문제다.

    • arr의 원소 중 divisor로 나누어 떨어지는 수만 결과에 포함한다.
    • 그런 수가 하나도 없으면 [-1]을 반환한다.
    • 결과 배열은 오름차순 정렬되어야 한다.

    전략 개요

    • 먼저 한 번 순회하며 divisor로 나누어 떨어지는 원소의 개수(cnt)를 센다.
    • cnt == 0이면 [-1]을 즉시 반환한다.
    • 크기가 cnt인 배열을 만들고 다시 순회하며 조건을 만족하는 원소만 채운다.
    • 채운 배열을 Arrays.sort로 정렬 후 반환한다.

    코드가 하는 일

    • 1) 조건 만족 개수 세기 
      int cnt = 0;
      
      for (int i : arr) {
          if (i % divisor == 0) {
              cnt++;
          }
      }
      • 배열을 한 번 훑으며 i % divisor == 0인 원소 개수를 cnt에 누적한다.
    • 2) 예외 처리: 하나도 없으면 [-1] 반환 
      if (cnt == 0) return new int[]{-1};
      • 조건을 만족하는 원소가 없으면 문제 요구대로 [-1]을 반환한다.
    • 3) 결과 배열 생성 후 값 채우기 
      int[] answer = new int[cnt];
      int idx = 0;
      for (int i : arr) {
          if (i % divisor == 0) answer[idx++] = i;
      }
      • 정확한 크기의 배열을 만든 뒤, 다시 순회하면서 조건을 만족하는 값만 차례대로 넣는다.
      • idx는 현재 채울 위치를 가리킨다.
    • 4) 오름차순 정렬 후 반환 
      Arrays.sort(answer);
      
      return answer;
      • 문제 조건에 맞게 answer를 오름차순으로 정렬하고 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열을 두 번 순회: O(N)
      • 결과 배열 정렬: 결과 크기를 K라 하면 O(K log K)
      • 전체는 O(N + K log K) (최악의 경우 K = N).
    • 공간 복잡도
      • 결과 배열 크기 K만큼 추가 사용 → O(K)

    문제 2 — 가운데 글자 가져오기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    가운데 글자 가져오기 (프로그래머스 12903)은
    문자열 s의 길이에 따라 가운데 글자(들)을 반환하는 문제다.

    • 문자열 길이가 홀수면 가운데 글자 1개를 반환한다.
    • 문자열 길이가 짝수면 가운데 글자 2개를 반환한다.
    • 인덱스는 0부터 시작하므로, 가운데 위치를 계산할 때 이를 고려해야 한다.

    전략 개요

    • 문자열 길이 length를 구한다.
    • mid = length / 2로 가운데 기준 인덱스를 잡는다.
    • length가
      • 홀수면 s.charAt(mid) 1개
      • 짝수면 s.charAt(mid - 1), s.charAt(mid) 2개
      를 이어서 반환한다.

    코드가 하는 일

    • 1) 길이와 가운데 인덱스 계산 
      StringBuilder sb = new StringBuilder();
      int length = s.length();
      
      int mid = length / 2;
      • length는 문자열 길이, mid는 가운데 기준 인덱스다.
      • 짝수 길이일 때는 가운데가 2칸이므로 mid를 기준으로 좌/우를 선택한다.
    • 2) 짝수/홀수에 따라 가운데 글자 추가 
      if (length % 2 == 0) {
          sb.append(s.charAt(mid - 1));
          sb.append(s.charAt(mid));
      } else {
          sb.append(s.charAt(mid));
      }
      • 짝수면 가운데 두 글자는 mid - 1, mid 위치다.
      • 홀수면 가운데 한 글자는 mid 위치다.
    • 3) 문자열로 변환해 반환 
      return sb.toString();
      • StringBuilder에 모아둔 결과를 문자열로 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 인덱스 계산과 문자 접근만 수행하므로 O(1).
    • 공간 복잡도
      • 결과로 최대 2글자만 저장하므로 O(1).
