    작성일 2025-12-24 · 분류: 코딩 테스트(프로그래머스), Level 1

    문제 1 — 2016년

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    2016년 (프로그래머스 12901)은
    2016년의 a월 b일이 무슨 요일인지를 구해서, 요일을 문자열("MON" 등)로 반환하는 문제다.

    • 2016년 1월 1일은 금요일(FRI)로 고정되어 있다.
    • 2016년은 윤년이므로 2월은 29일이다.
    • 입력된 날짜까지의 “일 수 오프셋”을 구한 뒤, 7로 나눈 나머지로 요일을 찾으면 된다.

    전략 개요

    • 각 달의 일 수를 배열로 준비한다. (2016년이므로 2월은 29)
    • 1월 1일이 금요일이므로 요일 배열을 {"FRI","SAT","SUN","MON","TUE","WED","THU"}로 준비한다.
    • a월 b일까지의 누적 일 수를 구한다:
      • 1월부터 (a-1)월까지의 일 수 합
      • + (b-1) (해당 달에서 몇 일 지났는지)
    • 구한 오프셋을 offset % 7로 요일 배열에서 찾아 반환한다.

    코드가 하는 일

    • 1) 달별 일 수/요일 배열 준비 
      int[] days = {31, 29, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
      String[] str = {"FRI", "SAT", "SUN", "MON", "TUE", "WED", "THU"};
      • days: 2016년 각 달의 일 수(윤년이라 2월=29).
      • str: 2016-01-01이 금요일이므로 인덱스 0을 FRI로 둔다.
    • 2) a월 이전까지의 누적 일 수 계산 
      int offset = 0;
      for (int i = 0; i < a - 1; i++) {
          offset += days[i];
      }
      • a월의 직전 달(a-1월)까지 총 며칠이 있었는지 누적한다.
      • 예: 5월이면 1~4월 일 수를 더한다.
    • 3) 해당 달에서 (b-1)일 더하기 
      offset += (b - 1);
      • 1월 1일을 기준(오프셋 0)으로 잡았기 때문에, 같은 달의 날짜는 b-1만큼 이동한 값이 된다.
      • 예: 1월 1일이면 b-1 = 0이라 그대로 금요일.
    • 4) 요일 배열에서 찾아 반환 
      return str[offset % 7];
      • 7일마다 요일이 반복되므로 offset % 7 인덱스로 요일을 선택한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최대 12개월을 더하는 반복문만 수행하므로 O(1).
    • 공간 복잡도
      • 고정 크기 배열과 변수만 사용 → O(1).

    문제 2 — 택배 상자 꺼내기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    (프로그래머스 389478)은
    1부터 n까지 번호가 붙은 상자를 가로 w칸짜리 선반에 지그재그(한 줄은 왼→오, 다음 줄은 오→왼)로 채워 놓았을 때, 특정 상자 번호 num을 꺼내기 위해 같은 열(column) 위에 쌓인 상자를 포함해서 총 몇 개를 꺼내야 하는지 구하는 문제다.

    • 선반은 한 줄에 w개씩 들어가며, 행(row)은 ceil(n / w)개가 된다.
    • 배치는 지그재그로 진행된다:
      • 0번째 행: 왼 → 오른쪽으로 채움
      • 1번째 행: 오른쪽 → 왼으로 채움
      • 2번째 행: 다시 왼 → 오른쪽 ...
    • num이 놓인 칸을 찾은 뒤, 같은 열에서 아래로 내려가며 실제로 존재하는 상자(0이 아닌 값)의 개수를 세면 된다.

    전략 개요

    • rows = (n + w - 1) / w로 필요한 행 개수를 구한다.
    • rows x w 크기의 2차원 배열 box를 만들고, 1..n을 지그재그로 채운다.
    • 배열을 순회하면서 num의 위치(r, c)를 찾는다.
    • 찾은 위치 기준으로 같은 열 c에서 아래 행으로 내려가며 값이 0이 아닌 칸을 만날 때마다 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 행 개수 계산 
      int rows = (n + w - 1) / w;
      • 전체 n개를 한 줄에 w개씩 채울 때 필요한 행 수를 올림으로 계산한다.
    • 2) 지그재그로 상자 번호 채우기 
      int idx = 1;
      int[][] box = new int[rows][w];
      for (int r = 0; r < box.length; r++) {
          if (r % 2 == 0) {
              for (int c = 0; c < box[r].length; c++) {
                  if (idx > n) break;
                  box[r][c] = idx++;
              }
          } else {
              for (int c = box[r].length - 1; c >= 0; c--) {
                  if (idx > n) break;
                  box[r][c] = idx++;
              }
          }
      }
      • 짝수 행은 왼→오로 채우고, 홀수 행은 오→왼으로 채운다.
      • idx > n이면 더 채울 상자가 없으므로 해당 행 채우기를 중단한다.
      • 채워지지 않은 칸은 기본값 0으로 남는다.
    • 3) num 위치 찾고 같은 열에서 아래로 카운트 
      int cnt = 1;
      for (int r = 0; r < box.length; r++) {
          for (int c = 0; c < box[r].length; c++) {
              if (box[r][c] == num) {
                  int curR = r + 1;
      
                  while (curR < box.length) {
                      if (box[curR][c] != 0) {
                          cnt++;
                      } else break;
      
                      curR++;
                  }
              }
          }
      }
      • num 자체는 반드시 꺼내야 하므로 cnt를 1에서 시작한다.
      • num을 발견하면 그 칸의 열 c를 고정하고, 아래 행(r+1)부터 내려간다.
      • 아래 칸이 0이면 실제 상자가 없는 자리이므로 그 이후로는 더 내려가도 의미가 없어 중단한다.
      • 0이 아니면 그 칸의 상자도 같이 꺼내야 하므로 카운트를 증가시킨다.
    • 4) 결과 반환 
      return cnt;
      • num을 꺼내기 위해 같이 꺼내야 하는 상자 총 개수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 채우기: O(rows * w) (최대 n칸까지 채움)
      • num 찾기: O(rows * w)
      • 아래로 카운트: 최악에 rows만큼 내려가므로 O(rows)
      • 전체는 대략 O(rows * w).
    • 공간 복잡도
      • rows x w 2차원 배열을 사용하므로 O(rows * w).
