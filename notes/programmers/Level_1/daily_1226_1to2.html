    프로그래머스 Level 1 Daily 문제 묶음

    작성일 2025-12-25 · 분류: 코딩 테스트(프로그래머스), Level 1

    문제 1 — 유연근무제

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    (프로그래머스 388351)은
    직원별로 정해진 출근 예정 시간 schedules[i]가 있을 때, 실제 출근 기록 timelogs[i][j]를 보고 평일(월~금)에 한 번도 지각하지 않은 직원 수를 구하는 문제다.

    • 각 직원은 예정 시간보다 10분 이내로 도착하면 통과다. (예정시간 + 10분이 “마감 시간”)
    • 주말(토/일)은 평가에서 제외한다.
    • startday는 timelogs[i][0]이 무슨 요일인지 나타내며, 요일은 1~7로 주어진다. (6, 7이 주말)

    전략 개요

    • 직원별로 “마감 시간” limit = schedules[i] + 10을 만든다.
    • 시간이 HHMM 형태이므로, +10 했을 때 분이 60 이상이 되면 보정한다:
      • limit % 100 >= 60이면 limit += 40 (분 60 → 시 1 증가 효과)
    • 7일을 돌면서, 해당 요일이 주말(6,7)이면 건너뛰고 평일만 검사한다.
    • 평일 중 한 번이라도 timelogs[i][j] > limit이면 지각이므로 탈락.
    • 끝까지 통과하면 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 직원 수 및 정답 카운터 준비 
      int employees = schedules.length;
      int cnt = 0;
      • 직원 수만큼 반복하며 “통과한 직원”을 센다.
    • 2) 직원별 마감 시간(limit) 계산 (10분 보정 포함) 
      int limit = schedules[i] + 10;
      if (limit % 100 >= 60) limit += 40;
      • 예정 시간에 10분을 더해 마감 시간을 만든다.
      • HHMM 형태에서 분이 60 이상이면 시로 넘겨야 하므로 +40으로 보정한다.
      • 예: 0855 + 10 = 0865 → 분이 65이므로 +40 해서 0905로 만든다.
    • 3) 7일 순회하며 평일만 검사 
      boolean ok = true;
      
      for (int j = 0; j < 7; j++) {
          int day = ((startday - 1 + j) % 7) + 1;
          if (day == 6 || day == 7) continue;
      
          if (timelogs[i][j] > limit) {
              ok = false;
              break;
          }
      }
      • day 계산은 startday를 기준으로 7일을 순환시키기 위한 식이다.
      • day == 6 또는 7이면 주말이므로 평가 제외 → continue.
      • 평일에 한 번이라도 마감 시간보다 늦으면(> limit) 즉시 탈락 처리한다.
      • 반대로 더 일찍 온 경우(<= limit)는 정상 출근으로 그대로 통과한다.
    • 4) 통과한 직원 카운트 
      if (ok) cnt++;
      • 평일 모두 지각이 없었던 직원만 정답에 포함한다.
    • 5) 최종 반환 
      return cnt;
      • 조건을 만족한 직원 수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 직원 수를 E라 하면, 각 직원에 대해 최대 7일을 확인하므로 O(E * 7) → O(E).
    • 공간 복잡도
      • 추가로 사용하는 변수는 상수 크기 → O(1).

    문제 2 — 동영상 재생기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    (프로그래머스 340213)은
    영상 길이 video_len과 현재 위치 pos가 주어지고, 오프닝 구간(op_start ~ op_end)을 자동으로 건너뛰면서 commands에 따라 10초 전/후 이동을 반복한 뒤 최종 위치를 "mm:ss" 형식으로 반환하는 문제다.

    • 명령은 "prev"(10초 뒤로), "next"(10초 앞으로) 형태로 처리한다. (코드에서는 prev가 아니면 전부 next로 처리)
    • 이동 후 위치는 범위를 벗어나지 않게 보정한다:
      • 최소 0초
      • 최대 video_len초
    • 현재 위치가 오프닝 구간에 들어가면 자동으로 op_end로 점프한다.
    • 시간 문자열은 비교/계산이 불편하므로 초 단위(int)로 변환해서 처리한다.

    전략 개요

    • "mm:ss" 문자열을 모두 초 단위로 변환한다:
      • len = 영상 길이
      • cur = 현재 위치
      • opS, opE = 오프닝 시작/끝
    • 시작 위치가 오프닝 구간이면 먼저 opE로 이동시킨다.
    • 명령을 순서대로 처리하면서
      • prev면 cur - 10, next면 cur + 10
      • 0..len 범위를 벗어나지 않게 clamp
      • 이동 후 오프닝 구간에 들어가면 다시 opE로 스킵
    • 최종 초 값을 다시 "mm:ss"로 바꿔 반환한다.

    코드가 하는 일

    • 1) 시간 문자열을 초로 변환 
      int len = toSec(video_len);
      int cur = toSec(pos);
      int opS = toSec(op_start);
      int opE = toSec(op_end);
      • 모든 비교/이동을 초 단위로 처리하기 위해 toSec로 변환한다.
    • 2) 시작 위치에서 오프닝 스킵 적용 
      cur = skipOpening(cur, opS, opE);
      • 현재 위치가 오프닝 구간(opS..opE)에 있으면 바로 opE로 이동시킨다.
    • 3) 명령 처리: 10초 이동 + 범위 보정 + 오프닝 스킵 
      for (String cmd : commands) {
          if (cmd.equals("prev")) {
              cur = Math.max(0, cur - 10);
          } else {
              cur = Math.min(len, cur + 10);
          }
      
          cur = skipOpening(cur, opS, opE);
      }
      • prev면 10초 뒤로 이동하되 0 아래로 내려가지 않게 한다.
      • 그 외 명령은 10초 앞으로 이동하되 len을 넘지 않게 한다.
      • 이동한 위치가 오프닝 구간에 걸리면 다시 skipOpening으로 opE로 점프한다.
    • 4) 최종 위치를 "mm:ss"로 변환해 반환 
      return toStr(cur);
      • 초 단위 결과를 toStr로 포맷팅해서 반환한다.
    • 5) 보조 메서드들 
      private int toSec(String s) {
          String[] split = s.split(":");
          int mm = Integer.parseInt(split[0]);
          int sec = Integer.parseInt(split[1]);
      
          return mm * 60 + sec;
      }
      • "mm:ss"를 mm * 60 + ss로 바꾼다.
      private int skipOpening(int cur, int opS, int opE) {
          if (opS <= cur && cur <= opE) return opE;
      
          return cur;
      }
      • 현재 위치가 오프닝 구간에 포함되면 opE로 스킵한다.
      • 포함되지 않으면 그대로 유지한다.
      private String toStr(int t) {
          int mm = t / 60;
          int ss = t % 60;
      
          return String.format("%02d:%02d", mm, ss);
      }
      • 초를 분/초로 나누고, 두 자리(%02d)로 맞춰 "mm:ss" 형식으로 만든다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 명령 개수를 K라 하면, 각 명령을 한 번씩 처리하므로 O(K).
      • 시간 변환/스킵은 모두 상수 시간.
    • 공간 복잡도
      • 추가로 쓰는 변수는 상수 크기 → O(1).
