    문제 1 — 지폐 접기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    (프로그래머스 340199)은
    지갑 크기 wallet에 지폐 크기 bill을 넣을 수 있을 때까지, 지폐의 긴 변을 반으로 접는 작업을 반복했을 때 접는 횟수를 구하는 문제다.

    • 지갑은 회전 가능하고, 지폐도 회전 가능하다고 보고 각각의 (짧은 변, 긴 변)을 비교해서 들어가는지 판단한다.
    • 지폐가 안 들어가면 지폐의 두 변 중 더 긴 변을 반으로 줄인다.
    • 들어갈 때까지 반복하며 줄인 횟수를 센다.

    전략 개요

    • 지갑과 지폐의 가로/세로를 각각 변수로 꺼낸다.
    • 매 반복마다
      • 지갑의 (minimum, maximum) = (짧은 변, 긴 변)
      • 지폐의 (min, max) = (짧은 변, 긴 변)
      을 만든다.
    • minimum >= min 그리고 maximum >= max면 회전 포함 조건으로 “들어감”이므로 종료한다.
    • 안 들어가면 지폐의 긴 변을 반으로 줄이고(/= 2) 카운트를 증가시킨다.

    코드가 하는 일

    • 1) 지갑/지폐 크기 변수로 분리 
      int widthSize = wallet[0];
      int lengthSize = wallet[1];
      int width = bill[0];
      int length = bill[1];
      int cnt = 0;
      • 지갑 크기와 지폐 크기를 각각 (가로/세로)로 꺼내 둔다.
      • cnt는 지폐를 접은 횟수다.
    • 2) 들어갈 때까지 반복 
      while (true) {
          int minimum = Math.min(widthSize, lengthSize);
          int maximum = Math.max(widthSize, lengthSize);
          int min = Math.min(width, length);
          int max = Math.max(width, length);
          if (minimum >= min && maximum >= max) break;
      
          if (width > length) {
              width /= 2;
              cnt++;
          } else {
              length /= 2;
              cnt++;
          }
      }
      • 지갑과 지폐 각각의 짧은 변/긴 변을 만들어 회전 가능한 상태로 비교한다.
      • 지갑의 짧은 변이 지폐의 짧은 변 이상이고, 지갑의 긴 변이 지폐의 긴 변 이상이면 들어간다.
      • 안 들어가면 지폐의 두 변 중 긴 쪽을 반으로 줄인다.
      • 줄일 때마다 cnt를 1 증가시킨다.
    • 3) 접은 횟수 반환 
      return cnt;
      • 지폐가 들어갈 때까지 접은 총 횟수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 반복할 때마다 지폐의 한 변이 1/2로 줄어들므로 반복 횟수를 KO(K).
      • 실제로는 지폐 크기가 계속 절반으로 감소하므로 K는 매우 작다(로그 수준).
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).

    문제 2 — 공원

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    (프로그래머스 340198)은
    공원 격자 park에서 "-1"로 표시된 칸들만 이용해 정사각형 매트를 깔 수 있을 때, 주어진 mats 중 가장 큰(한 변 길이) 매트를 깔 수 있는 값을 구하는 문제다.

    • 매트는 한 변 길이가 k인 정사각형이며, k x k 영역이 전부 "-1"이어야 한다.
    • 여러 크기의 매트 후보(mats)가 주어지며, 가능한 것 중 최댓값을 반환한다.
    • 어떤 매트도 못 깔면 -1을 반환한다.

    전략 개요

    • field[r][c]를 (r, c)를 오른쪽 아래 꼭짓점으로 하는 “"-1"로만 이루어진 최대 정사각형의 한 변 길이(DP)”로 정의한다.
    • DP 점화식:
      • park[r][c] != "-1"이면 field[r][c] = 0
      • 첫 행/첫 열에서 "-1"이면 field[r][c] = 1
      • 그 외 "-1"이면 field[r][c] = min(field[r-1][c-1], field[r-1][c], field[r][c-1]) + 1
    • DP를 채우면서 가능한 정사각형의 최대 변 길이 maxSize를 구한다.
    • mats를 정렬한 뒤, 큰 것부터 보며 mats[i] <= maxSize인 첫 값을 반환한다.

    코드가 하는 일

    • 1) DP 배열 준비 + 최대값 변수 
      int[][] field = new int[park.length][park[0].length];
      int maxSize = 0;
      • field[r][c]에 각 칸을 오른쪽 아래로 하는 최대 정사각형 변 길이를 저장한다.
      • maxSize는 전체에서 만들 수 있는 정사각형의 최대 변 길이.
    • 2) 격자 순회하며 DP 채우기 
      for (int r = 0; r < park.length; r++) {
          for (int c = 0; c < park[r].length; c++) {
              if (!park[r][c].equals("-1")) field[r][c] = 0;
              else if (r == 0 || c == 0) {
                  field[r][c] = 1;
              } else {
                  int curDp = Math.min(Math.min(field[r - 1][c - 1], field[r - 1][c]), field[r][c - 1]) + 1;
                  field[r][c] = curDp;
              }
      
              if (field[r][c] > maxSize) maxSize = field[r][c];
          }
      }
      • "-1"이 아닌 칸은 매트를 놓을 수 없으므로 0.
      • 첫 행/첫 열에서는 주변 3칸을 참고할 수 없으니, "-1"이면 최소 정사각형(1x1)만 가능 → 1.
      • 나머지 칸에서 "-1"이면, 위(r-1,c), 왼쪽(r,c-1), 좌상단(r-1,c-1) 중 가장 작은 정사각형 크기에 1을 더해 확장한다.
      • DP를 채우는 동시에 maxSize를 갱신한다.
    • 3) mats에서 가능한 최대 매트 크기 찾기 
      Arrays.sort(mats);
      
      for (int i = mats.length - 1; i >= 0 ; i--) {
          if (mats[i] <= maxSize) return mats[i];
      }
      
      return -1;
      • mats를 오름차순 정렬 후, 큰 값부터 확인한다.
      • 전체 공원에서 만들 수 있는 최대 정사각형 변 길이가 maxSize이므로, mats[i] <= maxSize인 가장 큰 매트가 정답이다.
      • 끝까지 없으면 어떤 매트도 못 깔므로 -1.

    시간 · 공간 복잡도

    • 시간 복잡도
      • DP 채우기: 격자 크기를 R x C라 하면 O(R * C)
      • 매트 정렬: 매트 개수를 K라 하면 O(K log K)
      • 전체: O(R * C + K log K)
    • 공간 복잡도
      • DP 배열 field 사용 → O(R * C)
