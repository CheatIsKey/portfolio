    문제 1 — 붕대 감기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    붕대 감기 (프로그래머스 250137)은
    캐릭터가 시간 단위로 자동 회복(붕대)을 하다가, 특정 시간에 공격을 맞으면 체력이 감소하는 상황에서 마지막 공격 이후 남는 체력을 구하는 시뮬레이션 문제다.

    • bandage = [t, x, y]
      • t: 연속으로 붕대를 유지해야 하는 시간(초)
      • x: 1초마다 회복되는 체력
      • y: t초 연속 성공 시 추가 회복량
    • attacks[i] = [time, damage]는 해당 시간에 공격을 받아 체력이 감소한다.
    • 공격을 받으면 연속 성공(콤보)이 끊기고, 그 초에는 회복이 일어나지 않는다.
    • 회복은 최대 체력(health 초기값) 이상으로 올라가지 않는다.
    • 체력이 0 이하가 되는 순간 즉시 -1을 반환한다.

    전략 개요

    • 마지막 공격 시간까지(t) 1초씩 시뮬레이션한다.
    • 현재 초가 공격 시간이라면
      • 체력을 감소시키고
      • 연속 붕대 시간(persist)을 0으로 초기화
      • 체력이 0 이하이면 -1 반환
    • 공격이 없다면 붕대 회복:
      • 연속 시간 persist 증가
      • persist == bandage[0]이면 기본 회복 + 추가 회복 후 persist를 0으로 초기화
      • 그 외에는 기본 회복만
    • 회복 후 체력은 항상 maxHealth를 넘지 않게 clamp 한다.

    코드가 하는 일

    • 1) 기본 변수 세팅 
      int t = attacks[attacks.length - 1][0];
      int persist = 0;
      int time = bandage[0];
      int heal = bandage[1];
      int addHeal = bandage[2];
      int maxHealth = health;
      int idx = 0;
      • t: 마지막 공격 시간이 곧 시뮬레이션의 종료 시점이다.
      • persist: 연속으로 붕대를 유지한 시간(초)을 누적한다.
      • time/heal/addHeal: 붕대 스킬 정보.
      • maxHealth: 최대 체력(초기 체력값) 저장.
      • idx: 현재 처리할 공격 인덱스.
    • 2) 1초부터 마지막 공격 시간까지 시뮬레이션 
      for (int i = 1; i <= t; i++) {
      • 각 초마다 “공격인지 / 회복인지”를 판단한다.
    • 3) 공격 처리: 체력 감소 + 연속 초기화 
      if (idx < attacks.length && i == attacks[idx][0]) {
          health -= attacks[idx][1];
          if (health <= 0) return -1;
          persist = 0;
          idx++;
          continue;
      }
      • 현재 시간이 다음 공격 시간과 같으면 데미지를 적용한다.
      • 체력이 0 이하가 되면 즉시 죽으므로 -1을 반환한다.
      • 공격을 받으면 연속 붕대가 끊기므로 persist = 0.
      • 공격이 있는 초에는 회복을 하지 않으므로 continue로 다음 초로 넘어간다.
    • 4) 회복 처리: 기본 회복 / (t초 연속 성공 시) 추가 회복 
      persist++;
      
      if (time == persist) {
          if (maxHealth <= health + heal + addHeal) health = maxHealth;
          else health += heal + addHeal;
          persist = 0;
      } else {
          if (maxHealth <= health + heal) health = maxHealth;
          else health += heal;
      }
      • 공격이 없으면 연속 시간이 1 증가한다.
      • persist == time이면 “연속 성공”이므로 heal + addHeal만큼 회복하고, 콤보를 다시 0으로 만든다.
      • 그 외에는 1초 회복(heal)만 적용한다.
      • 회복은 항상 최대 체력(maxHealth)을 넘지 않도록 처리한다.
    • 5) 최종 체력 반환 
      return health;
      • 마지막 공격 시간까지 모두 처리한 뒤 남은 체력을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 마지막 공격 시간(초)을 T라 하면, 1초씩 한 번 순회하므로 O(T).
      • 공격은 인덱스로 한 번씩만 처리하므로 추가 비용은 없다.
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).

    문제 2 — 이웃한 칸

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    이웃한 칸 (프로그래머스 250125)은
    2차원 격자 board에서 좌표 (h, w)를 기준으로 상/하/좌/우 인접 4칸 중 board[h][w]와 같은 값(색)을 가진 칸이 몇 개인지 세는 문제다.

    • 인접은 대각선이 아니라 상/하/좌/우만 포함한다.
    • 격자 밖으로 나가는 방향은 무시한다.
    • 조건을 만족하는 인접 칸의 개수를 반환한다.

    전략 개요

    • 상/하/좌/우 이동을 위한 방향 배열 ROW, COL을 만든다.
    • 4방향을 하나씩 이동해 인접 칸 좌표 (posR, posC)를 계산한다.
    • 좌표가 보드 범위를 벗어나면 건너뛴다.
    • 범위 안이면 board[h][w]와 board[posR][posC]가 같은지 비교해 같으면 카운트한다.

    코드가 하는 일

    • 1) 4방향 이동 배열 준비 
      static final int[] ROW = {-1, 0, 0, 1};
      static final int[] COL = {0, -1, 1, 0};
      • 같은 인덱스끼리 묶어서 보면
        • (-1, 0): 위
        • (0, -1): 왼쪽
        • (0, 1): 오른쪽
        • (1, 0): 아래
        방향을 의미한다.
    • 2) 4방향 순회하며 범위 체크 + 같은 값 세기 
      int cnt = 0;
      
      for (int i = 0; i < 4; i++) {
          int posR = h + ROW[i];
          int posC = w + COL[i];
      
          if ((posR < 0 || board.length <= posR) || (posC < 0 || board[posR].length <= posC)) {
              continue;
          }
      
          if (board[h][w].equals(board[posR][posC])) {
              cnt++;
          }
      }
      • 인접 칸 좌표를 계산한 뒤, 보드 범위를 벗어나면 continue로 무시한다.
      • 범위 안에 있으면 기준 칸과 값이 같은지 비교하고, 같으면 cnt를 증가시킨다.
    • 3) 결과 반환 
      return cnt;
      • 조건을 만족한 인접 칸의 개수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 상/하/좌/우 4칸만 확인하므로 O(1).
    • 공간 복잡도
      • 추가 자료구조 없이 상수 변수만 사용 → O(1).
