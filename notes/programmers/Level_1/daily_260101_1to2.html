    문제 1 — 개인정보 수집 유효기간

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    개인정보 수집 유효기간 (프로그래머스 150370)은
    오늘 날짜 today와 약관별 유효기간 terms, 그리고 개인정보 수집 날짜/약관 종류 privacies가 주어졌을 때, 파기해야 하는 개인정보의 번호를 오름차순으로 구하는 문제다.

    • 모든 달은 28일로 고정이다.
    • 개인정보는 “수집일 + 약관 개월 수”까지 유효하며, 그 다음 날부터 파기 대상이 된다.
    • 즉, 만료일은
      • expire = (수집일 + months * 28) - 1
      로 계산할 수 있다.
    • 만료일이 오늘보다 이전이면 파기 대상이다.

    전략 개요

    • 날짜를 비교하기 쉽게, 모든 날짜를 “총 일수”로 변환한다.
    • 28일/달이므로
      • total = (YYYY * 12 + MM) * 28 + DD
      형태로 단일 정수로 바꿀 수 있다.
    • terms는 약관 종류 → 개월 수를 Map에 저장한다.
    • 각 개인정보에 대해 만료일을 계산하고, 만료일이 오늘보다 작으면(이미 만료) 인덱스를 결과에 추가한다.

    코드가 하는 일

    • 1) today를 총 일수로 변환 
      String[] split = today.split("\\.");
      int YYYY = Integer.parseInt(split[0]);
      int MM = Integer.parseInt(split[1]);
      int DD = Integer.parseInt(split[2]);
      
      int totalToday = (YYYY * 12 + MM) * 28 + DD;
      • 문자열 날짜를 파싱해서 정수로 바꾼다.
      • 모든 달이 28일이므로 “YYYY/MM/DD”를 단일 값으로 변환해 비교한다.
    • 2) 약관 종류별 개월 수를 Map에 저장 
      Map<String, Integer> map = new HashMap<>();
      for (String term : terms) {
          String[] s = term.split(" ");
          map.put(s[0], Integer.parseInt(s[1]));
      }
      • 예: "A 6"이면 map.get("A") == 6으로 저장된다.
    • 3) 각 개인정보의 만료일 계산 후 파기 대상 체크 
      List<Integer> list = new ArrayList<>();
      for (int i = 0; i < privacies.length; i++) {
          String[] s = privacies[i].split(" ");
          String[] date = s[0].split("\\.");
      
          YYYY = Integer.parseInt(date[0]);
          MM = Integer.parseInt(date[1]);
          DD = Integer.parseInt(date[2]);
      
          int totalExpire = ((YYYY * 12 + MM) * 28 + DD);
          int months = map.get(s[1]);
      
          int expire = totalExpire + months * 28 - 1;
          if (expire < totalToday) {
              list.add(i + 1);
          }
      }
      • totalExpire는 수집일을 총 일수로 바꾼 값이다.
      • 만료일은 “수집일 + 유효기간 - 1일”이므로 -1을 해준다.
      • expire < totalToday이면 오늘 기준 이미 만료 → 파기 대상이다.
      • 문제에서 번호는 1부터 시작하므로 i + 1을 넣는다.
    • 4) 리스트를 배열로 변환해 반환 
      int idx = 0;
      int[] answer = new int[list.size()];
      for (int i : list) {
          answer[idx++] = i;
      }
      
      return answer;
      • 파기 대상 번호를 담은 리스트를 정답 배열로 옮겨 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 약관 파싱: terms 길이를 T라 하면 O(T)
      • 개인정보 검사: privacies 길이를 P라 하면 O(P)
      • 전체: O(T + P)
    • 공간 복잡도
      • 약관 Map과 결과 리스트를 저장 → O(T + P) (결과는 최대 P개)

    문제 2 — 성격 유형 검사하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    성격 유형 검사하기 (프로그래머스 118666)는
    설문 문항 survey와 선택지 choices가 주어졌을 때, 각 성격 지표(4쌍)에서 점수를 계산해 최종 성격 유형 문자열을 만드는 문제다.

    • 각 문항은 두 성격 유형 문자로 구성된다. 예: "RT"
    • 선택지 choices는 1~7이며, 4는 중립이다.
    • 1~3은 첫 번째 문자(c1)에 점수, 5~7은 두 번째 문자(c2)에 점수를 준다.
    • 점수 크기는 중립(4)과의 거리로 결정된다:
      • 1,7 → 3점
      • 2,6 → 2점
      • 3,5 → 1점
      • 4 → 0점
    • 각 지표 쌍은 RT, CF, JM, AN 순서로 결정한다.
    • 동점이면 사전순(알파벳이 더 빠른 문자)을 선택한다.

    전략 개요

    • choices[i]를 choice = choices[i] - 4로 변환하면,
      • choice < 0: 첫 번째 유형에 점수(-choice)
      • choice > 0: 두 번째 유형에 점수(choice)
      • choice == 0: 점수 없음
      처럼 한 번에 처리할 수 있다.
    • 문자별 누적 점수는 알파벳 인덱스를 이용해 total[26]에 저장한다.
    • 각 지표 쌍에 대해 누적 점수를 비교해 더 큰 쪽을 선택하고, 같으면 사전순으로 선택한다.

    코드가 하는 일

    • 1) 문자별 점수 누적 배열 준비 
      int[] total = new int[26];
      • total['A' - 'A']부터 total['Z' - 'A']까지, 각 문자 점수를 저장한다.
    • 2) 설문 문항을 돌며 점수 누적 
      for (int i = 0; i < survey.length; i++) {
          char c1 = survey[i].charAt(0);
          char c2 = survey[i].charAt(1);
          int choice = choices[i] - 4;
      
          if (choice < 0) {
              total[c1 - 'A'] += -choice;
          } else if (choice > 0) {
              total[c2 - 'A'] += choice;
          }
      }
      • choices[i] - 4로 중립 기준 거리(점수)를 바로 계산한다.
      • 음수면 첫 번째 유형(c1)에 -choice만큼 누적한다.
      • 양수면 두 번째 유형(c2)에 choice만큼 누적한다.
      • 0이면 중립이므로 아무 것도 하지 않는다.
    • 3) 4개 지표쌍을 순서대로 선택해 문자열 만들기 
      StringBuilder sb = new StringBuilder();
      sb.append(selectChar(total, 'R', 'T'));
      sb.append(selectChar(total, 'C', 'F'));
      sb.append(selectChar(total, 'J', 'M'));
      sb.append(selectChar(total, 'A', 'N'));
      
      return sb.toString();
      • RT, CF, JM, AN 순서로 한 글자씩 결정한다.
      • 결정된 4글자를 이어 최종 성격 유형을 반환한다.
    • 4) selectChar(): 두 문자 점수 비교 후 선택 
      private char selectChar(int[] total, char c1, char c2) {
          if (total[c1 - 'A'] > total[c2 - 'A']) {
              return c1;
          } else if (total[c1 - 'A'] < total[c2 - 'A']) {
              return c2;
          } else {
              return c1 < c2 ? c1 : c2;
          }
      }
      • 누적 점수가 큰 문자를 선택한다.
      • 동점이면 사전순으로 더 빠른 문자를 선택한다. (예: R vs T면 R)

    시간 · 공간 복잡도

    • 시간 복잡도
      • 설문 문항 수를 N이라 하면 한 번 순회하므로 O(N).
      • 최종 선택은 4쌍 비교로 상수 시간이다.
    • 공간 복잡도
      • 알파벳 점수 배열 total[26]만 사용 → O(1).
