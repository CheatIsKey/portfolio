    문제 1 — 개인정보 수집 유효기간

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    개인정보 수집 유효기간 (프로그래머스 150370)은
    오늘 날짜 today와 약관별 유효기간 terms, 그리고 개인정보 수집 날짜/약관 종류 privacies가 주어졌을 때, 파기해야 하는 개인정보의 번호를 오름차순으로 구하는 문제다.

    • 모든 달은 28일로 고정이다.
    • 개인정보는 “수집일 + 약관 개월 수”까지 유효하며, 그 다음 날부터 파기 대상이 된다.
    • 즉, 만료일은
      • expire = (수집일 + months * 28) - 1
      로 계산할 수 있다.
    • 만료일이 오늘보다 이전이면 파기 대상이다.

    전략 개요

    • 날짜를 비교하기 쉽게, 모든 날짜를 “총 일수”로 변환한다.
    • 28일/달이므로
      • total = (YYYY * 12 + MM) * 28 + DD
      형태로 단일 정수로 바꿀 수 있다.
    • terms는 약관 종류 → 개월 수를 Map에 저장한다.
    • 각 개인정보에 대해 만료일을 계산하고, 만료일이 오늘보다 작으면(이미 만료) 인덱스를 결과에 추가한다.

    코드가 하는 일

    • 1) today를 총 일수로 변환 
      String[] split = today.split("\\.");
      int YYYY = Integer.parseInt(split[0]);
      int MM = Integer.parseInt(split[1]);
      int DD = Integer.parseInt(split[2]);
      
      int totalToday = (YYYY * 12 + MM) * 28 + DD;
      • 문자열 날짜를 파싱해서 정수로 바꾼다.
      • 모든 달이 28일이므로 “YYYY/MM/DD”를 단일 값으로 변환해 비교한다.
    • 2) 약관 종류별 개월 수를 Map에 저장 
      Map<String, Integer> map = new HashMap<>();
      for (String term : terms) {
          String[] s = term.split(" ");
          map.put(s[0], Integer.parseInt(s[1]));
      }
      • 예: "A 6"이면 map.get("A") == 6으로 저장된다.
    • 3) 각 개인정보의 만료일 계산 후 파기 대상 체크 
      List<Integer> list = new ArrayList<>();
      for (int i = 0; i < privacies.length; i++) {
          String[] s = privacies[i].split(" ");
          String[] date = s[0].split("\\.");
      
          YYYY = Integer.parseInt(date[0]);
          MM = Integer.parseInt(date[1]);
          DD = Integer.parseInt(date[2]);
      
          int totalExpire = ((YYYY * 12 + MM) * 28 + DD);
          int months = map.get(s[1]);
      
          int expire = totalExpire + months * 28 - 1;
          if (expire < totalToday) {
              list.add(i + 1);
          }
      }
      • totalExpire는 수집일을 총 일수로 바꾼 값이다.
      • 만료일은 “수집일 + 유효기간 - 1일”이므로 -1을 해준다.
      • expire < totalToday이면 오늘 기준 이미 만료 → 파기 대상이다.
      • 문제에서 번호는 1부터 시작하므로 i + 1을 넣는다.
    • 4) 리스트를 배열로 변환해 반환 
      int idx = 0;
      int[] answer = new int[list.size()];
      for (int i : list) {
          answer[idx++] = i;
      }
      
      return answer;
      • 파기 대상 번호를 담은 리스트를 정답 배열로 옮겨 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 약관 파싱: terms 길이를 T라 하면 O(T)
      • 개인정보 검사: privacies 길이를 P라 하면 O(P)
      • 전체: O(T + P)
    • 공간 복잡도
      • 약관 Map과 결과 리스트를 저장 → O(T + P) (결과는 최대 P개)

    문제 2 — 성격 유형 검사하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    데이터 분석 (프로그래머스 250121)은
    2차원 배열 data에서 특정 컬럼(ext)의 값이 val_ext보다 작은 행만 골라낸 뒤, 다른 컬럼(sort_by) 기준으로 오름차순 정렬하여 반환하는 문제다.

    • data의 각 행은 항상 길이 4이고, 컬럼 의미는 다음과 같다:
      • code
      • date
      • maximum
      • remain
    • ext로 주어진 컬럼에서 < val_ext 조건을 만족하는 행만 추출한다.
    • 추출한 결과를 sort_by 컬럼 기준으로 오름차순 정렬한다.
    • 정렬된 결과를 2차원 배열 형태로 반환한다.

    전략 개요

    • 문자열로 주어진 컬럼명(ext, sort_by)을 실제 인덱스로 바꾼다.
    • data를 순회하며 datum[extIdx] < val_ext인 행만 리스트에 담는다.
    • 리스트를 sortIdx 기준으로 오름차순 정렬한다.
    • 리스트를 2차원 배열로 변환해 반환한다.

    코드가 하는 일

    • 1) 컬럼명 → 인덱스 변환 
      int extIdx = col(ext);
      int sortIdx = col(sort_by);
      • ext, sort_by는 문자열이므로 실제 배열 인덱스로 변환해 사용한다.
    • 2) 조건에 맞는 행만 필터링 
      List<int[]> list = new ArrayList<>();
      for (int[] datum : data) {
          if (datum[extIdx] < val_ext) list.add(datum);
      }
      • ext 컬럼 값이 val_ext보다 작은 행만 list에 담는다.
    • 3) sort_by 기준 오름차순 정렬 
      list.sort(Comparator.comparingInt(a -> a[sortIdx]));
      • 정렬 기준 컬럼의 값을 비교하도록 Comparator를 만들어 정렬한다.
    • 4) 리스트 → 2차원 배열 변환 
      int[][] ans = new int[list.size()][4];
      for (int i = 0; i < list.size(); i++) {
          ans[i] = list.get(i);
      }
      
      return ans;
      • 필터링된 행 개수만큼 배열을 만들고, 리스트 원소를 그대로 옮긴다.
      • 정렬된 결과를 반환한다.
    • 5) col(): 컬럼 문자열을 인덱스로 매핑 
      private int col(String str) {
          switch (str) {
              case "code": return 0;
              case "date": return 1;
              case "maximum": return 2;
              case "remain": return 3;
          }
      
          return -1;
      }
      • 문제에서 정한 컬럼 순서를 그대로 인덱스로 매핑한다.
      • 유효하지 않은 값은 입력으로 주어지지 않지만, 방어적으로 -1을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 필터링: 데이터 개수를 N이라 하면 O(N)
      • 정렬: 필터링 결과를 K라 하면 O(K log K)
      • 전체: O(N + K log K)
    • 공간 복잡도
      • 필터링 결과 리스트/배열 저장 → O(K)
