    문제 1 — 가장 많이 받은 선물

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    가장 많이 받은 선물 (프로그래머스 258712)은
    친구들 간의 선물 기록 gifts가 주어졌을 때, 다음 달에 각 친구가 받을 선물 개수를 규칙대로 계산하고 그 중 최댓값을 구하는 문제다.

    • 두 친구 A, B를 비교한다.
      • 이번 달에 A → B가 더 많으면 다음 달에는 A가 선물 1개 받는다.
      • 이번 달에 B → A가 더 많으면 다음 달에는 B가 선물 1개 받는다.
      • 서로 주고받은 수가 같으면 “선물 지수”로 비교한다.
    • 선물 지수 = (내가 준 선물 수) - (내가 받은 선물 수)
    • 지수도 같으면 다음 달에는 서로 선물을 주고받지 않는다.

    전략 개요

    • 이름을 인덱스로 바꾸기 위해 Map<String, Integer>로 매핑한다.
    • give[a][b]에 “a가 b에게 준 선물 횟수”를 누적한다.
    • 각 사람의 총 준 개수(out) / 받은 개수(in)를 누적하고, score[i] = out[i] - in[i]로 선물 지수를 만든다.
    • 모든 친구 쌍 (i, j)에 대해 규칙대로 “다음 달에 받을 선물” 카운트(recv)를 증가시킨다.
    • recv 중 최대값을 반환한다.

    코드가 하는 일

    • 1) 친구 이름 → 인덱스 매핑 
      int n = friends.length;
      Map<String, Integer> map = new HashMap<>();
      for (int i = 0; i < n; i++) map.put(friends[i], i);
      • 이름을 배열 인덱스로 바꿔서 2차원 카운팅을 쉽게 만든다.
    • 2) 이번 달 선물 기록 누적 (give/out/in) 
      int[][] give = new int[n][n];
      int[] out = new int[n];
      int[] in = new int[n];
      
      for (String gift : gifts) {
          String[] split = gift.split(" ");
          int a = map.get(split[0]);
          int b = map.get(split[1]);
          give[a][b]++;
          out[a]++;
          in[b]++;
      }
      • give[a][b]: a가 b에게 준 횟수
      • out[a]: a가 준 총합
      • in[b]: b가 받은 총합
    • 3) 선물 지수(score) 계산 
      int[] score = new int[n];
      for (int i = 0; i < n; i++) score[i] = out[i] - in[i];
      • 문제에서 정의한 선물 지수: (준 선물) - (받은 선물)
    • 4) 모든 친구 쌍 비교해서 다음 달 받을 선물 계산 
      int[] recv = new int[n];
      
      for (int i = 0; i < n; i++) {
          for (int j = i + 1; j < n; j++) {
              if (give[i][j] > give[j][i]) recv[i]++;
              else if (give[i][j] < give[j][i]) recv[j]++;
              else {
                  if (score[i] > score[j]) recv[i]++;
                  else if (score[i] < score[j]) recv[j]++;
              }
          }
      }
      • 쌍을 중복 없이 보기 위해 j = i + 1부터 순회한다.
      • 이번 달에 더 많이 준 사람이 다음 달에 1개를 받는다.
      • 주고받은 개수가 같으면 선물 지수로 비교한다.
      • 지수까지 같으면 아무도 받지 않으므로 카운트 변화 없음.
    • 5) recv 배열의 최댓값 반환 
      int max = 0;
      for (int i : recv) if (i > max) max = i;
      
      return max;
      • 다음 달에 가장 많이 받는 선물 개수를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 선물 기록 누적: gifts 길이를 G라 하면 O(G)
      • 친구 쌍 비교: 친구 수를 N이라 하면 O(N^2)
      • 전체: O(G + N^2)
    • 공간 복잡도
      • give가 N x N이므로 O(N^2)
      • 나머지 배열은 O(N)

    문제 2 — 데이터 분석

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    데이터 분석 (프로그래머스 250121)은
    2차원 배열 data에서 특정 컬럼(ext)의 값이 val_ext보다 작은 행만 골라낸 뒤, 다른 컬럼(sort_by) 기준으로 오름차순 정렬하여 반환하는 문제다.

    • data의 각 행은 항상 길이 4이고, 컬럼 의미는 다음과 같다:
      • code
      • date
      • maximum
      • remain
    • ext로 주어진 컬럼에서 < val_ext 조건을 만족하는 행만 추출한다.
    • 추출한 결과를 sort_by 컬럼 기준으로 오름차순 정렬한다.
    • 정렬된 결과를 2차원 배열 형태로 반환한다.

    전략 개요

    • 문자열로 주어진 컬럼명(ext, sort_by)을 실제 인덱스로 바꾼다.
    • data를 순회하며 datum[extIdx] < val_ext인 행만 리스트에 담는다.
    • 리스트를 sortIdx 기준으로 오름차순 정렬한다.
    • 리스트를 2차원 배열로 변환해 반환한다.

    코드가 하는 일

    • 1) 컬럼명 → 인덱스 변환 
      int extIdx = col(ext);
      int sortIdx = col(sort_by);
      • ext, sort_by는 문자열이므로 실제 배열 인덱스로 변환해 사용한다.
    • 2) 조건에 맞는 행만 필터링 
      List<int[]> list = new ArrayList<>();
      for (int[] datum : data) {
          if (datum[extIdx] < val_ext) list.add(datum);
      }
      • ext 컬럼 값이 val_ext보다 작은 행만 list에 담는다.
    • 3) sort_by 기준 오름차순 정렬 
      list.sort(Comparator.comparingInt(a -> a[sortIdx]));
      • 정렬 기준 컬럼의 값을 비교하도록 Comparator를 만들어 정렬한다.
    • 4) 리스트 → 2차원 배열 변환 
      int[][] ans = new int[list.size()][4];
      for (int i = 0; i < list.size(); i++) {
          ans[i] = list.get(i);
      }
      
      return ans;
      • 필터링된 행 개수만큼 배열을 만들고, 리스트 원소를 그대로 옮긴다.
      • 정렬된 결과를 반환한다.
    • 5) col(): 컬럼 문자열을 인덱스로 매핑 
      private int col(String str) {
          switch (str) {
              case "code": return 0;
              case "date": return 1;
              case "maximum": return 2;
              case "remain": return 3;
          }
      
          return -1;
      }
      • 문제에서 정한 컬럼 순서를 그대로 인덱스로 매핑한다.
      • 유효하지 않은 값은 입력으로 주어지지 않지만, 방어적으로 -1을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 필터링: 데이터 개수를 N이라 하면 O(N)
      • 정렬: 필터링 결과를 K라 하면 O(K log K)
      • 전체: O(N + K log K)
    • 공간 복잡도
      • 필터링 결과 리스트/배열 저장 → O(K)
