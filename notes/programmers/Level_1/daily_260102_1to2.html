    문제 1 — 신고 결과 받기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    신고 결과 받기 (프로그래머스 92334)는
    유저 목록 id_list와 신고 기록 report, 정지 기준 k가 주어졌을 때, 각 유저가 받게 될 메일(정지 결과 알림) 횟수를 구하는 문제다.

    • 신고 기록은 "신고자 피신고자" 형태다.
    • 같은 신고자가 같은 대상을 여러 번 신고해도 1번으로만 처리한다.
    • 어떤 유저가 k번 이상 신고되면 정지 대상이다.
    • 정지된 유저를 신고한 사람들에게 메일이 발송된다.
    • 정답은 id_list 순서대로 메일 횟수를 담은 배열이다.

    전략 개요

    • 중복 신고를 제거하기 위해 “신고자 → (신고한 대상 Set)” 구조를 만든다.
    • 신고 데이터를 순회하며
      • 신고자의 Set에 대상이 새로 추가되는 경우에만
      • 피신고자의 신고 횟수를 1 증가시킨다.
    • 신고 횟수가 k 이상인 유저들을 banned Set에 모은다.
    • 각 유저가 신고한 대상 중 banned에 포함된 대상 수를 세어 결과 배열에 넣는다.

    코드가 하는 일

    • 1) 신고자 → 신고 대상 Set 초기화 
      Map<String, Set<String>> map = new HashMap<>();
      
      for (String s : id_list) {
          map.put(s, new HashSet<>());
      }
      • 각 유저마다 “내가 신고한 대상들”을 저장할 Set을 준비한다.
      • Set을 쓰면 동일 대상 중복 신고가 자동으로 걸러진다.
    • 2) report 순회하며 중복 제거 + 피신고 횟수 누적 
      Map<String, Integer> cnt = new HashMap<>();
      
      for (String s : report) {
          String[] split = s.split(" ");
          String reporter = split[0];
          String target = split[1];
      
          Set<String> set = map.get(reporter);
      
          if (set.add(target)) {
              cnt.put(target, cnt.getOrDefault(target, 0) + 1);
          }
      }
      • set.add(target)가 true일 때만 “처음 신고”이므로 피신고자 카운트를 올린다.
      • 이미 신고했던 대상이면 Set에 추가되지 않으므로 카운트도 증가하지 않는다.
    • 3) k 이상 신고된 유저를 banned로 수집 
      Set<String> banned = new HashSet<>();
      for (Map.Entry<String, Integer> entry : cnt.entrySet()) {
          if (entry.getValue() >= k) banned.add(entry.getKey());
      }
      • 피신고 횟수가 k 이상인 유저는 정지 대상이므로 banned에 넣는다.
    • 4) 각 유저가 받을 메일 횟수 계산 
      int[] result = new int[id_list.length];
      for (int i = 0; i < id_list.length; i++) {
          Set<String> value = map.get(id_list[i]);
          int mail = 0;
      
          for (String s : value) {
              if (banned.contains(s)) mail++;
          }
      
          result[i] = mail;
      }
      • id_list 순서대로 해당 유저가 신고한 대상 Set을 가져온다.
      • 그 중 banned에 포함된 대상의 개수를 세면, 그 유저가 받을 메일 수가 된다.
    • 5) 결과 반환 
      return result;
      • 유저 순서대로 메일 횟수를 담은 배열을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • report 길이를 R, 유저 수를 N이라 하면
        • 신고 처리: O(R) (Set/Map 평균 O(1))
        • 메일 계산: 각 유저의 신고 목록을 합쳐도 최대 R개 수준 → 평균적으로 O(R)
      • 전체적으로 O(N + R)로 충분히 빠르다.
    • 공간 복잡도
      • 신고 관계 저장(Set) + 카운트(Map) → 최악의 경우 신고 수만큼 저장하므로 O(R).

    문제 2 — 나머지가 1이 되는 수 찾기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    나머지가 1이 되는 수 찾기 (프로그래머스 87389)는
    자연수 n이 주어졌을 때, n을 어떤 자연수 x로 나눈 나머지가 1이 되도록 하는 가장 작은 x를 찾는 문제다.

    • 조건: n % x == 1
    • 정답은 조건을 만족하는 x 중 최솟값이다.
    • 가장 작은 값을 찾는 문제이므로 x를 1부터 순서대로 검사하면 된다.

    전략 개요

    • x를 1부터 n-1까지 증가시키며 확인한다.
    • n % x == 1을 만족하는 순간의 x가 최솟값이므로 즉시 반환한다.

    코드가 하는 일

    • 1) 1부터 n-1까지 순회하며 조건 확인 
      for (int i = 1; i < n; i++) {
          if (n % i == 1) return i;
      }
      • i를 후보 x로 두고 n % i를 확인한다.
      • 나머지가 1인 가장 첫 번째 i를 만나면 바로 반환한다.
    • 2) (도달하지 않는) 기본 반환 
      return -1;
      • 문제 조건상 항상 정답이 존재하므로 실제로는 실행되지 않는 안전장치다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 최악의 경우 1부터 n-1까지 확인하므로 O(n).
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).
