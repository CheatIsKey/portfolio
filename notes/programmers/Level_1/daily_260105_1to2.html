    문제 1 — 최소직사각형

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    최소직사각형 (프로그래머스 86491)은
    명함들의 가로/세로 크기 sizes가 주어졌을 때, 모든 명함을 담을 수 있는 가장 작은 지갑의 넓이를 구하는 문제다.

    • 각 명함은 회전(가로/세로를 바꾸는 것)이 가능하다.
    • 지갑의 가로/세로는 한 번 정하면 모든 명함을 그 방향으로 넣어야 한다.
    • 정답은 (지갑 가로 최대) × (지갑 세로 최대)가 된다.

    전략 개요

    • 각 명함은 회전 가능하므로, 항상
      • w = max(가로, 세로)
      • h = min(가로, 세로)
      로 정리한다. (큰 변을 지갑의 “가로”, 작은 변을 “세로”로 통일)
    • 모든 명함에 대해
      • 가로 후보 중 최대값 = maxW
      • 세로 후보 중 최대값 = maxH
      를 구한다.
    • 그럼 어떤 명함이든 maxW × maxH 지갑 안에 들어가므로 정답은 maxW * maxH.

    코드가 하는 일

    • 1) 지갑 가로/세로의 최대 후보 변수 준비 
      int maxW = 0;
      int maxH = 0;
      • 명함을 담기 위한 최종 지갑 크기의 기준값을 저장한다.
    • 2) 각 명함을 회전 고려해서 (큰 변, 작은 변)으로 정규화 
      for (int[] size : sizes) {
          int w = Math.max(size[0], size[1]);
          int h = Math.min(size[0], size[1]);
      • 회전이 가능하므로 큰 변을 “가로”로, 작은 변을 “세로”로 통일한다.
      • 이렇게 해야 모든 명함을 같은 기준으로 비교할 수 있다.
    • 3) 가로 최대/세로 최대 갱신 
          maxW = Math.max(maxW, w);
          maxH = Math.max(maxH, h);
      }
      • 모든 명함이 들어가려면 지갑 가로는 모든 w를 커버해야 하므로 최대값을 취한다.
      • 지갑 세로도 마찬가지로 모든 h를 커버해야 하므로 최대값을 취한다.
    • 4) 넓이 반환 
      return maxW * maxH;
      • 최종 지갑 크기는 maxW × maxH, 정답은 그 넓이이다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 명함 개수를 N이라 하면 한 번 순회하며 최대값만 갱신 → O(N).
    • 공간 복잡도
      • 추가 배열 없이 상수 변수만 사용 → O(1).

    문제 2 — 없는 숫자 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    없는 숫자 더하기 (프로그래머스 86051)는
    0~9 중에서 numbers에 없는 숫자들의 합을 구하는 문제다.

    • numbers에는 0~9 사이의 일부 숫자만 들어있다.
    • 0~9 전체를 기준으로 “등장하지 않은 숫자”를 찾아 합산하면 된다.

    전략 개요

    • 0~9의 등장 여부를 체크할 boolean[10] 배열을 만든다.
    • numbers를 순회하며 등장한 숫자만 true로 표시한다.
    • 0부터 9까지 다시 순회하면서 false인 숫자만 합산한다.

    코드가 하는 일

    • 1) 0~9 체크 배열 준비 
      boolean[] chk = new boolean[10];
      • 인덱스가 숫자 자체(0~9)가 되도록 만든다.
    • 2) numbers에 등장한 숫자 표시 
      for (int number : numbers) {
          chk[number] = true;
      }
      • chk[x] = true면 숫자 x가 존재한다는 뜻이다.
    • 3) 등장하지 않은 숫자만 합산 
      int sum = 0;
      
      for (int i = 0; i < chk.length; i++) {
          if (chk[i]) continue;
          sum += i;
      }
      • false인 인덱스(숫자)만 더해준다.
    • 4) 결과 반환 
      return sum;

    시간 · 공간 복잡도

    • 시간 복잡도
      • numbers 길이를 N이라 하면, 표시 O(N) + 0~9 순회 O(10) → O(N).
    • 공간 복잡도
      • boolean[10]만 사용 → O(1).
