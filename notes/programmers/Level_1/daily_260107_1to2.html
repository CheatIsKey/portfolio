    문제 1 — 약수의 개수와 덧셈

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    약수의 개수와 덧셈 (프로그래머스 77884)는
    구간 [left, right]의 각 정수 i에 대해 약수의 개수가 짝수면 더하고, 홀수면 빼서 최종 합을 구하는 문제다.

    • 약수 개수 짝수 → sum += i
    • 약수 개수 홀수 → sum -= i
    • 핵심은 각 i의 약수 개수를 빠르게 세는 것이다.

    전략 개요

    • 각 수 i에 대해 1부터 sqrt(i)까지만 검사한다.
    • j가 i의 약수라면, 보통은 j와 i/j가 한 쌍으로 생긴다.
    • 따라서
      • j * j == i(제곱수)면 약수 1개만 추가
      • 그 외에는 약수 2개 추가
      로 약수 개수를 셀 수 있다.
    • 약수 개수의 홀짝에 따라 더/빼기를 결정한다.

    코드가 하는 일

    • 1) left부터 right까지 순회 
      for (int i = left; i <= right; i++) {
          int cnt = 0;
          ...
      }
      • 각 수 i의 약수 개수(cnt)를 구해서 홀짝을 판단한다.
    • 2) 1부터 sqrt(i)까지 약수 검사 
      for (int j = 1; j * j <= i; j++) {
          if (i % j == 0) {
              if (j * j == i) cnt += 1;
              else cnt += 2;
          }
      }
      • j가 약수면, 보통 j와 i/j 두 개가 생긴다 → cnt += 2
      • 단, j * j == i인 경우는 같은 약수(제곱근)라서 하나만 추가 → cnt += 1
      • 이렇게 하면 전체 약수를 전부 찾지 않고도 개수만 정확히 셀 수 있다.
    • 3) 약수 개수의 홀짝에 따라 더/빼기 
      if (cnt % 2 == 0) sum += i;
      else sum -= i;
      • 짝수면 더하고, 홀수면 뺀다.
    • 4) 최종 합 반환 
      return sum;

    시간 · 공간 복잡도

    • 시간 복잡도
      • 각 i마다 1부터 sqrt(i)까지 확인한다.
      • 구간 길이를 R = right - left + 1이라 하면 대략 O(R * sqrt(right)).
      • 문제 범위가 크지 않아 충분히 통과 가능한 방식이다.
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).

    문제 2 — 로또의 최고 순위와 최저 순위

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    로또의 최고 순위와 최저 순위 (프로그래머스 77484)는
    로또 번호 배열 lottos와 당첨 번호 배열 win_nums가 주어졌을 때, 0(알 수 없는 번호)을 어떻게 채우느냐에 따라 가능한 최고 순위와 최저 순위를 구하는 문제다.

    • lottos의 0은 가려진 번호(미확인)다.
    • 최고 순위: 0이 전부 당첨 번호로 채워진다고 가정
    • 최저 순위: 0이 전부 꽝(당첨 번호가 아님)이라고 가정
    • 순위는 “맞춘 개수”로 결정된다. (6개면 1등, …, 0~1개면 6등)

    전략 개요

    • 당첨 번호를 빠르게 확인하기 위해 win_nums를 boolean 배열에 표시한다.
    • lottos를 순회하며
      • 0의 개수(zero)를 센다.
      • 0이 아닌 숫자는 당첨 여부를 확인해 실제로 맞춘 개수(cnt)를 센다.
    • 가능한 맞춘 개수 범위는
      • 최고: cnt + zero
      • 최저: cnt
      이므로, 각각을 순위로 변환한다.

    코드가 하는 일

    • 1) 당첨 번호를 boolean 배열에 표시 
      boolean[] lotto = new boolean[46];
      
      for (int winNum : win_nums) {
          lotto[winNum] = true;
      }
      • 로또 번호 범위가 1~45이므로 크기 46 배열을 사용한다.
      • lotto[x] == true면 x는 당첨 번호다.
    • 2) 실제 일치 개수(cnt)와 0 개수(zero) 계산 
      int cnt = 0;
      int zero = 0;
      
      for (int i : lottos) {
          if (i == 0) {
              zero++;
          } else if (lotto[i]) {
              cnt++;
          }
      }
      • 0이면 미확인 번호이므로 zero 증가.
      • 0이 아니면 당첨 여부를 확인해 맞으면 cnt 증가.
    • 3) 최고/최저 순위 계산 
      int max = lottoCase(cnt + zero);
      int min = lottoCase(cnt);
      
      return new int[]{max, min};
      • 최고는 0이 전부 맞는 경우이므로 cnt + zero.
      • 최저는 0이 전부 틀리는 경우이므로 cnt.
      • 각각 맞춘 개수를 순위로 변환해 반환한다.
    • 4) lottoCase(): 맞춘 개수 → 순위 변환 
      private int lottoCase(int i) {
          return switch (i) {
              case 6 -> 1;
              case 5 -> 2;
              case 4 -> 3;
              case 3 -> 4;
              case 2 -> 5;
              default -> 6;
          };
      }
      • 맞춘 개수에 따라 등수를 고정 매핑한다.
      • 0~1개는 모두 6등이므로 default로 처리한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • win_nums 표시: O(6) → 사실상 상수
      • lottos 순회: O(6) → 사실상 상수
      • 전체: O(1)
    • 공간 복잡도
      • boolean 배열 lotto[46] 사용 → O(1)
