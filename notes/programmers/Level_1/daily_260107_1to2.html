    문제 1 — 부족한 금액 계산하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    부족한 금액 계산하기 (프로그래머스 82612)는
    놀이기구 이용료가 price이고, i번째 이용 시 비용이 price * i로 증가할 때, 총 count번 타기 위해 필요한 금액에서 현재 가진 돈 money를 뺀 부족한 금액을 구하는 문제다.

    • 총 비용 = price * (1 + 2 + ... + count)
    • 부족한 금액 = 총 비용 - money
    • 만약 돈이 충분하면 정답은 0이다.

    전략 개요

    • 1부터 count까지의 합은 등차수열 합 공식으로 구한다.
    • 1 + 2 + ... + n = n(n+1)/2 이므로,
      • total = price * count * (count + 1) / 2
      로 총 비용을 한 번에 계산할 수 있다.
    • 정답은 max(0, total - money).
    • 곱셈 과정에서 int 범위를 넘을 수 있으므로 long으로 계산한다.

    코드가 하는 일

    • 1) count를 long으로 변환 
      long n = count;
      • 뒤에서 곱셈을 할 때 오버플로우를 피하기 위해 long으로 승격한다.
    • 2) 총 비용 계산 (등차수열 합) 
      long result = (long) price * n * (n + 1) / 2;
      • i번째 비용이 price * i이므로 총합은 price * (1+...+n).
      • 합 공식 n(n+1)/2를 사용해 한 줄로 계산한다.
      • (long) price로 먼저 캐스팅해 연산 전체가 long으로 진행되게 한다.
    • 3) 부족한 금액 반환 (충분하면 0) 
      return Math.max(0L, result - money);
      • result - money가 음수(돈이 충분)면 0을 반환한다.
      • 부족하면 그 차액을 그대로 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 공식으로 한 번에 계산하므로 O(1).
    • 공간 복잡도
      • 상수 변수만 사용 → O(1).

    문제 2 — 숫자 문자열과 영단어

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    숫자 문자열과 영단어 (프로그래머스 81301)은
    숫자(0~9)와 영단어(zero~nine)가 섞여 있는 문자열 s가 주어졌을 때, 이를 전부 숫자로 바꿔서 최종 정수로 반환하는 문제다.

    • 예: "one4seveneight" → "1478"
    • 문자열에서 영단어를 해당 숫자로 치환한 뒤 정수로 변환하면 된다.

    전략 개요

    • zero부터 nine까지의 매핑을 Map으로 만든다.
    • 매핑을 순회하며 s에서 해당 영단어를 숫자 문자열로 replace한다.
    • 모든 치환이 끝난 문자열은 숫자만 남으므로 Integer.parseInt로 변환한다.

    코드가 하는 일

    • 1) 영단어 → 숫자 매핑 준비 
      Map<String, Integer> map = new HashMap<>();
      map.put("zero", 0);
      map.put("one", 1);
      map.put("two", 2);
      map.put("three", 3);
      map.put("four", 4);
      map.put("five", 5);
      map.put("six", 6);
      map.put("seven", 7);
      map.put("eight", 8);
      map.put("nine", 9);
      • 문제에서 주어진 영단어 10개를 숫자 값으로 연결해 둔다.
    • 2) map을 순회하며 문자열 치환 
      for (Map.Entry<String, Integer> entry : map.entrySet()) {
          String key = entry.getKey();
          int value = entry.getValue();
      
          s = s.replace(key, String.valueOf(value));
      }
      • replace(key, value)는 문자열 내의 모든 key를 전부 치환한다.
      • 예: "oneone"도 한 번에 "11"로 바뀐다.
      • 치환 결과는 새 문자열이므로 다시 s에 재할당한다.
    • 3) 숫자 문자열을 정수로 변환 
      return Integer.parseInt(s);
      • 모든 영단어가 숫자로 바뀐 뒤에는 숫자만 남으므로 정수 변환이 가능하다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 영단어 10개에 대해 replace를 수행한다.
      • 문자열 길이를 L이라 하면, 각 치환이 문자열을 훑으므로 대략 O(10 * L) → O(L).
    • 공간 복잡도
      • 치환 과정에서 새 문자열이 만들어지며, map도 상수 크기(10개) → O(L) (실질적으로는 문자열 저장 비용).
