    문제 1 — 음양 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    음양 더하기 (프로그래머스 76501)은
    절댓값 배열 absolutes와 각 수의 부호를 나타내는 불리언 배열 signs가 주어졌을 때, 각 원소에 부호를 적용해 전체 합을 구하는 문제다.

    • signs[i] == true면 +absolutes[i]
    • signs[i] == false면 -absolutes[i]
    • 모든 원소를 한 번씩만 처리하면 된다.

    전략 개요

    • 인덱스 i를 0부터 끝까지 순회한다.
    • signs[i] 값에 따라
      • true면 더하고
      • false면 뺀다
    • 누적합 sum을 반환한다.

    코드가 하는 일

    • 1) 누적합 변수 준비 
      int sum = 0;
      • 최종 결과를 누적할 변수다.
    • 2) 배열을 끝까지 순회하며 부호 적용 
      for (int i = 0; i < absolutes.length; i++) {
          if (!signs[i]) {
              sum -= absolutes[i];
          } else sum += absolutes[i];
      }
      • signs[i]가 false면 음수이므로 sum -= absolutes[i].
      • true면 양수이므로 sum += absolutes[i].
    • 3) 누적합 반환 
      return sum;
      • 모든 원소를 처리한 결과를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 길이를 N이라 하면 한 번 순회 → O(N).
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).

    문제 2 — 신규 아이디 추천

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    신규 아이디 추천 (프로그래머스 72410)은
    문자열 new_id가 주어졌을 때, 문제에서 제시한 7단계 규칙을 순서대로 적용해 추천 아이디를 만들어 반환하는 문제다.

    • 대문자를 소문자로 바꾸고
    • 허용된 문자만 남기며
    • 연속된 점(..)을 하나로 줄이고
    • 앞/뒤의 점을 제거하고
    • 비어있으면 "a"를 넣고
    • 길이를 3~15로 맞추되, 15 초과면 자르고 끝 점을 제거하며
    • 길이가 2 이하라면 마지막 문자를 반복해서 3으로 맞춘다

    전략 개요

    • 문자열을 한 글자씩 보면서 소문자 변환 + 허용 문자 필터링 + 연속 점 제거를 한 번에 처리한다.
    • 이후 규칙(앞/뒤 점 제거, 빈 문자열 처리, 길이 제한/보정)을 순서대로 적용한다.
    • StringBuilder를 사용해 불필요한 문자열 생성(복사)을 줄인다.

    코드가 하는 일

    • 1) 소문자 변환 + 허용 문자만 추가 + 연속 점 제거 
      StringBuilder sb = new StringBuilder();
      String s = new_id;
      char prev = 0;
      
      for (char c : s.toLowerCase().toCharArray()) {
          if (!(('a' <= c && c <= 'z') || ('0' <= c && c <= '9') || c == '-' || c == '_' || c == '.')) {
              continue;
          } else if (prev == '.' && c == '.') {
              continue;
          } else {
              sb.append(c);
          }
      
          prev = c;
      }
      • toLowerCase()로 1단계(소문자 치환)를 처리한다.
      • 허용 문자가 아니면 건너뛰어 2단계(필터링)를 처리한다.
      • 이전 문자가 점이고 현재도 점이면 스킵해서 3단계(연속 점 제거)를 처리한다.
    • 2) 앞/뒤 점 제거 
      if (sb.length() > 0 && sb.charAt(0) == '.') {
          sb.deleteCharAt(0);
      }
      if (sb.length() > 0 && sb.charAt(sb.length() - 1) == '.') {
          sb.deleteCharAt(sb.length() - 1);
      }
      • 문자열이 비어있지 않을 때만 검사한다.
      • 4단계(처음/끝의 점 제거)를 그대로 구현한 부분이다.
    • 3) 빈 문자열이면 "a"로 채우기 
      if (sb.length() == 0) sb.append("a");
      • 5단계 규칙을 처리한다.
    • 4) 길이 16 이상이면 15로 자르고, 끝 점이면 제거 
      if (sb.length() >= 16) {
          sb = new StringBuilder(sb.substring(0, 15));
      
          if (sb.charAt(sb.length() - 1) == '.') {
              sb.replace(sb.length() - 1, sb.length(), "");
          }
      }
      • 6단계에서 15자까지만 남긴다.
      • 자른 뒤 마지막 문자가 점이면 제거한다. (앞에서 한 번 더 수행하는 형태)
    • 5) 길이가 2 이하이면 마지막 문자로 3이 될 때까지 채우기 
      if (sb.length() <= 2) {
          for (int i = sb.length(); i < 3; i++) {
              sb.append(sb.charAt(sb.length() - 1));
          }
      }
      • 7단계 규칙 처리다.
      • 현재 길이부터 3이 될 때까지 마지막 문자를 반복해서 붙인다.
    • 6) 결과 반환 
      return sb.toString();

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이를 N이라 하면, 한 번 순회 + 일부 보정 처리 → O(N).
    • 공간 복잡도
      • StringBuilder에 결과를 저장하므로 최악 O(N).
