    문제 1 — 내적

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    내적 (프로그래머스 70128)은
    길이가 같은 두 배열 a, b가 주어졌을 때, 같은 인덱스끼리 곱한 값을 모두 더해 내적을 구하는 문제다.

    • 각 인덱스 i마다 a[i] * b[i]를 계산한다.
    • 계산한 곱들을 전부 더한 값을 반환한다.
    • 배열을 한 번만 순회하면 된다.

    전략 개요

    • 배열 길이 length를 구한다.
    • 누적합 변수 sum을 0으로 시작한다.
    • i = 0부터 length - 1까지 순회하며 a[i] * b[i]를 누적한다.
    • 최종 sum을 반환한다.

    코드가 하는 일

    • 1) 길이와 누적합 변수 준비 
      int length = a.length;
int sum = 0;
      • length는 반복 범위를 정하기 위한 배열 길이다.
      • sum은 내적 결과를 누적할 변수다.
    • 2) 같은 인덱스끼리 곱해서 누적 
      for (int i = 0; i < length; i++) {
    sum += a[i] * b[i];
}
      • a[i]와 b[i]를 곱한 값을 sum에 더한다.
      • 모든 인덱스를 한 번씩만 처리한다.
    • 3) 누적합 반환 
      return sum;
      • 모든 곱셈 결과를 더한 값이 곧 내적이므로 그대로 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 길이를 N이라 하면 한 번 순회 → O(N).
    • 공간 복잡도
      • 추가 자료구조 없이 변수만 사용 → O(1).

    문제 2 — 3진법 뒤집기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    3진법 뒤집기 (프로그래머스 68935)은
    자연수 n을 3진법으로 바꾼 뒤, 그 숫자를 뒤집어서 다시 10진법으로 변환해 반환하는 문제다.

    • n을 3진법 문자열로 변환한다.
    • 문자열을 뒤집는다.
    • 뒤집힌 3진법 문자열을 다시 10진법 정수로 변환한다.

    전략 개요

    • Integer.toString(n, 3)로 n을 3진법 문자열로 만든다.
    • StringBuilder로 문자열을 뒤집는다.
    • Integer.parseInt(뒤집힌문자열, 3)로 3진법을 10진법으로 해석해 반환한다.

    코드가 하는 일

    • 1) 3진법 문자열로 변환 
      String s = Integer.toString(n, 3);
      • n을 3진법 형태의 문자열로 만든다.
    • 2) StringBuilder로 뒤집기 
      StringBuilder sb = new StringBuilder(s);
sb.reverse();
      • StringBuilder는 문자열 뒤집기 같은 조작에 편하다.
      • reverse()로 3진법 문자열을 뒤집는다.
    • 3) 뒤집힌 3진법을 10진법으로 변환해 반환 
      return Integer.parseInt(sb.toString(), 3);
      • parseInt(문자열, 3)은 해당 문자열을 3진법 숫자로 해석해 10진법 정수로 바꾼다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • n의 3진법 자릿수를 L이라 하면 변환/뒤집기/파싱이 모두 자릿수만큼 처리 → O(L).
    • 공간 복잡도
      • 3진법 문자열과 빌더를 저장 → O(L).
