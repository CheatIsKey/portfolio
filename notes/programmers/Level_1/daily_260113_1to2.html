    문제 1 — 두 개 뽑아서 더하기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    두 개 뽑아서 더하기 (프로그래머스 68644)는
    정수 배열 numbers에서 서로 다른 두 원소를 뽑아 더한 값을 만들고, 그 결과를 중복 없이 모아서 오름차순 배열로 반환하는 문제다.

    • 모든 쌍 (i, j) (i < j)에 대해 numbers[i] + numbers[j]를 만든다.
    • 같은 합이 여러 번 나올 수 있으므로 중복 제거가 필요하다.
    • 중복 제거 후 결과를 오름차순으로 정렬해 배열로 반환한다.

    전략 개요

    • HashSet을 사용해 합을 저장하면서 중복을 제거한다.
    • 이중 반복문으로 모든 두 원소 조합 (i, j)를 만든다.
    • set에 저장된 결과를 스트림으로 꺼내 정렬 후 int[]로 변환해 반환한다.

    코드가 하는 일

    • 1) 정렬과 Set 준비 
      Arrays.sort(numbers);

Set<Integer> set = new HashSet<>();
      • HashSet은 같은 값을 여러 번 넣어도 한 번만 저장되므로 중복 제거에 적합하다.
      • 정렬은 필수는 아니지만, 이후 흐름을 보기 쉽게 만들고(또는 디버깅 시) 안정적으로 접근할 수 있다.
    • 2) 모든 두 원소 조합을 만들어 합을 저장 
      for (int i = 0; i < numbers.length-1; i++) {
    for (int j = i + 1; j < numbers.length; j++) {
        set.add(numbers[i] + numbers[j]);
    }
}
      • i < j로 잡아 같은 조합을 중복해서 계산하지 않는다.
      • 각 쌍의 합을 set에 넣으며 중복은 자동으로 제거된다.
    • 3) 오름차순 정렬 후 int 배열로 변환해 반환 
      return set.stream()
        .sorted()
        .mapToInt(Integer::intValue)
        .toArray();
      • set.stream() : Set의 원소들을 스트림으로 변환한다.
      • sorted() : 오름차순 정렬한다.
      • mapToInt(Integer::intValue) : Stream<Integer>를 IntStream으로 바꾼다.
      • toArray() : IntStream을 int[]로 변환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 모든 쌍을 만드는 데 O(N^2).
      • 결과 개수를 K라 하면 정렬에 O(K log K).
      • 전체는 O(N^2 + K log K)로 볼 수 있다.
    • 공간 복잡도
      • 중복 제거를 위한 Set에 결과를 저장하므로 O(K).

    문제 2 — 키패드 누르기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    키패드 누르기 (프로그래머스 67256)는
    왼손과 오른손이 각각 시작 위치(*, #)에서 출발해, 주어진 숫자들을 순서대로 누를 때 어떤 손으로 눌렀는지(L/R)를 문자열로 반환하는 문제다.

    • 1, 4, 7은 항상 왼손.
    • 3, 6, 9는 항상 오른손.
    • 2, 5, 8, 0은 두 손 중 더 가까운 손을 사용한다.
    • 거리가 같으면 사용자의 주손(hand)을 따른다.

    전략 개요

    • 키패드를 좌표로 바꾸지 않고, 번호를 이용해 행/열을 계산해 거리(맨해튼 거리)를 구한다.
    • 왼손/오른손의 현재 위치를 각각 정수로 관리한다.
      • *는 10, 0은 11, #는 12로 취급한다.
    • 각 숫자를 누를 때
      • 왼쪽 열(1,4,7)이면 무조건 L
      • 오른쪽 열(3,6,9)이면 무조건 R
      • 가운데 열(2,5,8,0)이면 거리 비교 후 결정
    • 결정한 손의 위치를 갱신하고 결과에 L/R을 붙인다.

    코드가 하는 일

    • 1) 시작 위치와 주손 설정 
      StringBuilder sb = new StringBuilder();

int left = 10;
int right = 12;
boolean leftHand = hand.equals("left");
      • 왼손 시작은 *이므로 10, 오른손 시작은 #이므로 12로 둔다.
      • 거리가 같을 때 주손을 따르기 위해 leftHand로 저장한다.
    • 2) 0은 11로 치환해서 동일한 규칙으로 처리 
      int target = (number == 0) ? 11 : number;
      • 키패드를 1~12로 보면 가운데 열 계산이 편해진다.
      • 0을 11로 바꿔 *=10, 0=11, #=12 형태로 통일한다.
    • 3) 왼쪽 열 / 오른쪽 열은 즉시 결정 
      if (target == 1 || target == 4 || target == 7) {
    sb.append('L');
    left = target;
    continue;
}

if (target == 3 || target == 6 || target == 9) {
    sb.append('R');
    right = target;
    continue;
}
      • 1,4,7은 항상 왼손이므로 결과에 L을 붙이고 left를 갱신한다.
      • 3,6,9는 항상 오른손이므로 결과에 R을 붙이고 right를 갱신한다.
      • 이미 결정됐으니 가운데 열 계산은 하지 않기 위해 continue로 넘어간다.
    • 4) 가운데 열은 거리(맨해튼 거리)로 비교 
      int leftDist = Math.abs(((left - 1) / 3) - ((target - 1) / 3)) +
        Math.abs(((left - 1) % 3) - ((target - 1) % 3));

int rightDist = Math.abs(((right - 1) / 3) - ((target - 1) / 3)) +
        Math.abs(((right - 1) % 3) - ((target - 1) % 3));
      • 각 번호를 0-index 좌표처럼 계산한다.
      • 행(row): (num - 1) / 3, 열(col): (num - 1) % 3.
      • 거리 = |row1-row2| + |col1-col2| (상하좌우 이동 횟수)로 계산한다.
      예)
1 2 3
4 5 6
7 8 9
* 0 #

여기서 * = 10, 0 = 11, # = 12 로 보면
( num - 1 ) / 3 , ( num - 1 ) % 3 로 행/열이 딱 떨어진다.
    • 5) 더 가까운 손 선택, 같으면 주손 
      if (leftDist < rightDist || (leftDist == rightDist && leftHand)) {
    sb.append('L');
    left = target;
} else {
    sb.append('R');
    right = target;
}
      • 왼손이 더 가까우면 L, 오른손이 더 가까우면 R.
      • 거리가 같으면 hand가 left면 왼손, right면 오른손을 사용한다.
      • 선택된 손의 위치를 target으로 갱신한다.
    • 6) 결과 문자열 반환 
      return sb.toString();
      • 각 숫자에 대해 누른 손을 누적한 문자열을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 입력 길이를 N이라 하면 각 숫자마다 O(1) 계산 → O(N).
    • 공간 복잡도
      • 결과 문자열을 담는 StringBuilder 외 추가 자료구조 없음 → O(1) (출력 제외).
