    문제 1 — 크레인 인형뽑기 게임

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    크레인 인형뽑기 게임 (프로그래머스 64061)은
    2차원 배열 board에 쌓여 있는 인형들을 크레인으로 뽑아 바구니에 넣을 때, 바구니에서 같은 인형이 연속으로 쌓이면 둘이 터져서 사라지는 규칙에 따라 최종적으로 터진 인형의 개수(사라진 인형 수)를 구하는 문제다.

    • moves는 크레인이 내려갈 열(column)을 1부터 시작하는 인덱스로 준다.
    • 해당 열에서 가장 위에 있는(행이 가장 작은) 인형 1개를 뽑는다.
    • 뽑은 인형을 바구니에 넣는데,
      • 바구니 맨 위 인형과 같으면 둘이 터진다(삭제) → 결과 +2
      • 다르면 그대로 쌓는다

    전략 개요

    • 바구니는 “마지막에 넣은 인형”과 비교해야 하므로 Stack으로 관리한다.
    • moves를 순서대로 처리하면서
      • 해당 열에서 위에서부터 내려오며 0이 아닌 첫 인형을 찾는다.
      • 찾은 인형을 뽑고(보드 값 0으로 변경) 스택과 비교한다.
      • 스택 top과 같으면 pop하고 터진 개수 +2, 다르면 push한다.
    • 모든 이동이 끝난 후 터진 인형 개수를 반환한다.

    코드가 하는 일

    • 1) 스택과 카운트 변수 준비 
      int n = board.length;

Stack<Integer> stack = new Stack<>();

int popCnt = 0;
      • n은 보드의 크기다.
      • stack은 바구니 역할을 한다. (맨 위 인형과 비교하기 쉽다)
      • popCnt는 터져서 사라진 인형의 총 개수(2개씩 증가)를 누적한다.
    • 2) 각 move마다 해당 열에서 인형을 하나 뽑는다 
      for (int move : moves) {
    for (int r = 0; r < n; r++) {
        int c = move - 1;

        if (board[r][c] == 0) continue;

        int doll = board[r][c];
        board[r][c] = 0;

        ...
        break;
    }
}
      • moves의 값은 1부터 시작하는 열 번호이므로 c = move - 1로 0-index로 바꾼다.
      • 해당 열을 위에서부터(r = 0) 내려오며 0이 아닌 첫 값을 찾는다.
      • 인형을 뽑으면 그 칸은 빈칸이 되므로 board[r][c] = 0으로 만든다.
      • 한 번 뽑았으면 그 move는 끝이므로 break로 내부 반복문을 종료한다.
    • 3) 바구니(스택) top과 비교해서 터뜨리거나 쌓기 
      if (!stack.isEmpty() && stack.peek() == doll) {
    stack.pop();
    popCnt += 2;
} else {
    stack.push(doll);
}
      • 스택이 비어있지 않고, 맨 위 인형이 지금 뽑은 인형과 같으면 둘이 터진다.
      • 그래서 pop()으로 제거하고, 사라진 인형 개수 +2를 더한다.
      • 다르면 인형을 스택에 그대로 쌓는다(push).
    • 4) 최종 터진 인형 개수 반환 
      return popCnt;
      • 문제에서 요구하는 값은 터진 인형의 총 개수이므로 누적한 popCnt를 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • moves 길이를 M, 보드 크기를 N이라 하면, 각 move마다 최악의 경우 위에서 아래로 한 열을 전부 확인 → O(MN).
    • 공간 복잡도
      • 바구니 스택에 최대 인형이 쌓이는 수는 M 이하 → O(M).

    문제 2 — 실패율

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    실패율 (프로그래머스 42889)은
    전체 스테이지가 1부터 N까지 있고, 각 사용자가 현재 도전 중인 스테이지 번호 stages가 주어질 때, 각 스테이지의 실패율을 구해 실패율이 높은 스테이지부터 정렬해 반환하는 문제다.

    • 실패율 = (해당 스테이지에 머물러 있는 사람 수) / (해당 스테이지에 도달한 사람 수)
    • 어떤 스테이지에 도달한 사람이 0명이면 실패율은 0으로 본다.
    • 실패율이 같으면 스테이지 번호가 작은 것이 먼저 온다.

    전략 개요

    • difficulty[x]를 “현재 x 스테이지에 있는 사람 수”로 카운트한다.
    • 1번 스테이지부터 N번 스테이지까지 순회하면서
      • 현재 도달한 사람 수(people)를 관리한다.
      • 실패율을 fail / people로 계산하고, 다음 스테이지로 넘어갈 때 people -= fail로 갱신한다.
    • 스테이지 번호 배열(1..N)을 만들어 실패율 내림차순, 번호 오름차순으로 정렬한다.
    • 정렬된 스테이지 번호를 int[]로 변환해 반환한다.

    코드가 하는 일

    • 1) 각 스테이지에 머물러 있는 사람 수 카운트 
      int[] difficulty = new int[N + 2];

for (int stage : stages) {
    difficulty[stage]++;
}
      • difficulty[s]는 현재 s 스테이지에 있는 사람 수다.
      • N+1은 “모든 스테이지를 클리어한 사람”이므로 배열을 N+2로 잡아 안전하게 카운트한다.
    • 2) 실패율 계산을 위한 변수 준비 
      double[] failRate = new double[N + 1];
int people = stages.length;
      • failRate[i]에 i번 스테이지의 실패율을 저장한다.
      • people은 “현재 스테이지에 도달한 사람 수”로 시작은 전체 인원이다.
    • 3) 1번부터 N번까지 실패율 계산 및 people 갱신 
      for (int i = 1; i <= N; i++) {
    int fail = difficulty[i];
    failRate[i] = (people == 0) ? 0.0 : (double) fail / people;
    people -= fail;
}
      • fail은 현재 스테이지에서 막힌 사람 수다.
      • people이 0이면 나눌 수 없으므로 실패율은 0으로 처리한다.
      • 다음 스테이지로 넘어갈 때는, 현재 스테이지에 머문 사람(fail)을 제외한 인원만 도달하므로 people -= fail로 갱신한다.
      예)
people = 전체 인원
i=1 실패율 = 1번에 머문 사람 / 1번에 도달한 사람(전체)
그 다음 people에서 1번에 머문 사람을 빼고 2번 계산...
    • 4) 정렬을 위해 스테이지 번호 배열 준비 
      Integer[] idx = new Integer[N];
for (int i = 0; i < N; i++) idx[i] = i + 1;
      • 정렬 결과로 “스테이지 번호”가 필요하므로 1..N을 담은 배열을 만든다.
      • Arrays.sort에 커스텀 비교를 넣기 위해 Integer[]를 사용한다.
    • 5) 실패율 내림차순, 번호 오름차순으로 정렬 
      Arrays.sort(idx, (a, b) -> {
    int cmp = Double.compare(failRate[b], failRate[a]);
    if (cmp != 0) return cmp;
    return Integer.compare(a, b);
});
      • 먼저 실패율을 비교해서 큰 실패율이 앞에 오도록 한다. (내림차순)
      • 실패율이 같으면 스테이지 번호가 작은 것이 먼저 오도록 한다. (오름차순)
    • 6) 정렬된 결과를 int 배열로 변환해 반환 
      int[] answer = new int[N];
for (int i = 0; i < N; i++) answer[i] = idx[i];

return answer;
      • Integer[]로 정렬한 결과를 최종 요구 형태인 int[]로 옮겨 담아 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 카운팅: stages 길이를 M이라 하면 O(M).
      • 실패율 계산: O(N).
      • 정렬: N개 정렬 → O(N log N).
      • 전체는 O(M + N log N).
    • 공간 복잡도
      • 카운팅 배열/실패율 배열/인덱스 배열 사용 → O(N).
