    문제 1 — 체육복

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    체육복 (프로그래머스 42862)는
    학생 1..n 중 체육복을 도난당한 학생(lost)과 여벌 체육복이 있는 학생(reserve)이 주어질 때, 인접한 학생에게만 체육복을 빌릴 수 있다는 규칙으로 체육수업을 들을 수 있는 학생 수의 최댓값을 구하는 문제다.

    • 각 학생은 기본적으로 체육복 1벌을 가지고 시작한다.
    • lost에 있으면 1벌을 잃어서 0벌이 된다.
    • reserve에 있으면 여벌 1벌이 추가되어 2벌이 된다.
    • 빌려줄 수 있는 대상은 바로 앞/바로 뒤 학생뿐이다.

    전략 개요

    • students[i]를 i번째 학생의 체육복 개수로 관리한다. (0, 1, 2)
    • 초기값은 모두 1로 채운 뒤,
      • lost는 1 감소
      • reserve는 1 증가
    • 왼쪽부터 순회하면서 체육복이 0인 학생을 발견하면
      • 먼저 앞 학생이 2벌이면 빌린다.
      • 그게 안 되면 뒤 학생이 2벌이면 빌린다.
      • 둘 다 안 되면 수업을 못 들으므로 카운트를 1 감소시킨다.
    • 이 과정을 마치면 남은 카운트가 최댓값이 된다.

    코드가 하는 일

    • 1) 학생 체육복 배열 초기화 
      int[] students = new int[n];
Arrays.fill(students, 1);
      • 모든 학생은 기본 1벌로 시작한다.
    • 2) 도난/여벌 정보 반영 
      for (int idx : lost) students[idx - 1]--;
for (int idx : reserve) students[idx - 1]++;
      • lost에 있으면 1벌을 잃으므로 -1.
      • reserve에 있으면 여벌이 있으므로 +1.
      • 같은 학생이 lost와 reserve에 모두 있으면 결과적으로 1이 되어 “빌려줄 여벌도 없고, 잃지도 않은 상태”처럼 처리된다.
    • 3) 수업 가능 인원 카운트 준비 
      int cnt = n;
      • 일단 모두 수업 가능하다고 가정하고 시작한 뒤, 못 듣는 경우에만 줄인다.
    • 4) 왼쪽부터 순회하며 0인 학생 처리(그리디) 
      for (int i = 0; i < n; i++) {
    if (students[i] == 0) {
        int prev = i - 1;
        int next = i + 1;

        if (0 <= prev && students[prev] == 2) {
            students[prev] = 1;
            students[i] = 1;
        } else if (next < n && students[next] == 2) {
            students[next] = 1;
            students[i] = 1;
        } else {
            cnt--;
        }
    }
}
      • 체육복이 없는 학생(0)이면 주변에서 빌릴 수 있는지 확인한다.
      • 먼저 prev(앞 학생)를 우선으로 보는 이유는
        • 왼쪽부터 순회 중이므로, 앞에서 해결 가능한 건 앞에서 확정해두는 게 깔끔하고
        • 뒤 학생의 여벌은 다음 인덱스에서도 필요할 수 있어 “뒤의 선택지”를 남겨두는 효과가 있다.
      • 앞/뒤 모두 2벌이 아니면 빌릴 수 없으므로 수업 불가 → cnt--.
    • 5) 최종 수업 가능 인원 반환 
      return cnt;
      • 그리디로 최대한 빌려준 뒤 남은 수업 가능 인원을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • lost, reserve 반영 + 학생 순회 → O(n + |lost| + |reserve|).
    • 공간 복잡도
      • 학생 배열 students 사용 → O(n).

    문제 2 — 모의고사

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    모의고사 (프로그래머스 42840)는
    문제의 정답 배열 answers가 주어질 때, 각 수포자가 정해진 찍기 패턴대로 답을 찍었을 때의 점수를 계산하고, 가장 많이 맞힌 사람(들)의 번호를 오름차순으로 반환하는 문제다.

    • 각 수포자는 고정된 패턴을 반복한다.
    • answers[i]와 각 수포자의 i번째 찍는 답을 비교해 맞힌 개수를 센다.
    • 최대 점수를 가진 사람 번호를 모두 반환한다.

    전략 개요

    • 각 수포자의 찍기 패턴을 배열로 미리 저장한다.
    • 문제를 0번부터 끝까지 돌면서,
      • 각 수포자의 패턴 인덱스를 i % 패턴길이로 구한다.
      • 정답과 같으면 해당 수포자 점수를 +1 한다.
    • 점수 배열에서 최댓값을 찾고, 최댓값과 같은 사람 번호를 리스트에 모은다.
    • 리스트를 배열로 바꿔 반환한다.

    코드가 하는 일

    • 1) 수포자 패턴과 점수 배열 준비 
      int[][] students = {
        {1,2,3,4,5},
        {2,1,2,3,2,4,2,5},
        {3,3,1,1,2,2,4,4,5,5}
};

int[] score = new int[3];
      • students[k]는 k번째 수포자의 반복 패턴이다.
      • score[k]는 k번째 수포자가 맞힌 개수(점수)다.
    • 2) answers를 순회하며 각 수포자의 답과 비교 
      for (int i = 0; i < answers.length; i++) {
    int answer = answers[i];

    int firstStudent = i % students[0].length;
    int secondStudent = i % students[1].length;
    int thirdStudent = i % students[2].length;

    if (students[0][firstStudent] == answer) score[0]++;
    if (students[1][secondStudent] == answer) score[1]++;
    if (students[2][thirdStudent] == answer) score[2]++;
}
      • i % 패턴길이를 쓰면 패턴이 끝나도 다시 처음으로 돌아가며 반복된다.
      • 각 수포자의 i번째 찍는 답을 구해서 정답과 같으면 점수를 올린다.
      예)
1번 패턴 길이 5라면,
i=0..4는 그대로,
i=5부터는 5%5=0으로 다시 첫 패턴으로 돌아간다.
    • 3) 최고 점수 찾고 해당 수포자 번호 모으기 
      List<Integer> list = new ArrayList<>();
int max = -1;

for (int i = 0; i < score.length; i++) {
    if (max < score[i]) {
        max = score[i];
        list.clear();
        list.add(i + 1);
    } else if (max == score[i]) {
        list.add(i + 1);
    }
}
      • max보다 큰 점수를 발견하면 최댓값을 갱신하고 리스트를 초기화한다.
      • 점수가 최댓값과 같으면 공동 1등이므로 리스트에 추가한다.
      • 수포자 번호는 1번부터이므로 i + 1로 넣는다.
    • 4) 리스트를 int 배열로 변환해 반환 
      int[] answer = new int[list.size()];
int idx = 0;

for (int i : list) {
    answer[idx++] = i;
}

return answer;
      • 문제 요구사항이 int[] 반환이므로 리스트를 배열로 옮겨 담는다.
      • 리스트에는 이미 오름차순(1,2,3 순회하면서 추가)으로 들어가므로 그대로 반환해도 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 정답 배열 길이를 N이라 하면 한 번 순회하며 비교 → O(N).
      • 최댓값 찾기는 수포자 3명이라 상수 시간.
    • 공간 복잡도
      • 점수 배열과 결과 리스트만 사용 → O(1) (패턴 배열은 고정 상수).
