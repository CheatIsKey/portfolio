    문제 1 — K번째수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    K번째수 (프로그래머스 42748)는
    배열 array와 여러 개의 명령 commands가 주어졌을 때, 각 명령 [i, j, k]마다 array의 i번째부터 j번째까지 자른 뒤 정렬하고, 그 중 k번째 원소를 찾아 결과 배열로 반환하는 문제다.

    • 각 명령은 1-based 인덱스(i, j, k)로 주어진다.
    • 부분 배열을 만든 뒤 오름차순 정렬한다.
    • 정렬된 배열에서 k번째 값을 뽑아 정답 배열에 담는다.

    전략 개요

    • 명령 개수만큼 결과 배열 answer를 만든다.
    • 각 명령마다
      • 구간 길이만큼 새 배열을 만든다.
      • array에서 해당 구간을 복사한다.
      • 정렬 후 k-1 인덱스 값을 결과에 저장한다.
    • 모든 명령을 처리한 answer를 반환한다.

    코드가 하는 일

    • 1) 정답 배열 준비 
      int[] answer = new int[commands.length];
      • 명령 개수만큼 결과를 담을 배열을 만든다.
    • 2) 각 명령을 순회하며 i~j 구간 추출 
      for (int i = 0; i < commands.length; i++) {
    int s = commands[i][0];
    int e = commands[i][1];
    int k = commands[i][2];

    int size = e - s + 1;
    int[] arr = new int[size];
    int pos = 0;

    for (int j = s - 1; j < e; j++) {
        arr[pos++] = array[j];
    }

    Arrays.sort(arr);

    answer[i] = arr[k - 1];
}
      • s, e, k는 1-based이므로 실제 배열 인덱스는 s-1부터 e-1까지다.
      • size = e - s + 1만큼 새 배열을 만들고 해당 구간을 복사한다.
      • 부분 배열을 정렬한 뒤 k-1번째 값을 결과로 저장한다.
      예)
array = [1,5,2,6,3,7,4]
command = [2,5,3]
→ 2~5 구간: [5,2,6,3]
→ 정렬: [2,3,5,6]
→ 3번째: 5
    • 3) 최종 결과 반환 
      return answer;
      • 모든 명령에 대한 k번째 수를 담은 배열을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 명령 개수를 C, 각 구간 길이를 평균 L
      • 구간 복사 O(L) + 정렬 O(L log L)를 명령마다 수행 → O(C · L log L).
    • 공간 복잡도
      • 명령마다 부분 배열 arr를 생성 → O(L) (최대 구간 길이 기준).

    문제 2 — 완주하지 못한 선수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    완주하지 못한 선수 (프로그래머스 42576)는
    마라톤 참가자 명단 participant와 완주자 명단 completion이 주어질 때, 완주하지 못한 선수 1명의 이름을 찾는 문제다. 동명이인이 있을 수 있으므로 단순 비교가 아니라 개수(카운트)까지 고려해야 한다.

    • completion은 participant보다 길이가 1 작다.
    • 동명이인이 있을 수 있어 이름만으로는 구분이 안 된다.
    • 따라서 이름별 등장 횟수를 세고, 완주자를 하나씩 빼는 방식이 안전하다.

    전략 개요

    • HashMap<이름, 개수>로 참가자 이름을 카운트한다.
    • 완주자 배열을 순회하며 해당 이름의 카운트를 1씩 감소시킨다.
      • 감소 후 0이 되면 Map에서 제거한다.
    • 마지막에 Map에 남아 있는 키 1개가 완주하지 못한 선수다.

    코드가 하는 일

    • 1) 참가자 이름 카운팅 
      Map<String, Integer> map = new HashMap<>();

for (String runner : participant) {
    map.put(runner, map.getOrDefault(runner, 0) + 1);
}
      • getOrDefault로 기존 값이 없으면 0부터 시작해 +1 한다.
      • 동명이인도 이름별로 개수를 올려서 정확히 관리한다.
    • 2) 완주자 처리: 카운트 감소(0이면 제거) 
      for (String runner : completion) {
    int num = map.get(runner);

    if (num == 1) {
        map.remove(runner);
    } else {
        map.put(runner, num - 1);
    }
}
      • 완주자는 참가자에서 “한 명씩 빠지는 것”과 동일하다.
      • 현재 카운트가 1이면 감소 후 0이 되므로 Map에서 제거한다.
      • 카운트가 2 이상이면 1 감소시켜 다시 저장한다.
    • 3) Map에 남은 키가 정답 
      return map.keySet().iterator().next();
      • 완주자 처리 후에는 완주하지 못한 선수 이름만 Map에 남는다.
      • 문제 조건상 남는 선수는 1명이므로 첫 번째 키를 꺼내 반환하면 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 참가자 카운팅 O(N) + 완주자 처리 O(N) → 평균적으로 O(N).
      • (HashMap 평균 접근/삽입/삭제가 O(1) 기준)
    • 공간 복잡도
      • 이름별 카운트를 저장하는 Map 사용 → O(N) (최악: 전부 다른 이름).
