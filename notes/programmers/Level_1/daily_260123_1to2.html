    문제 1 — [1차] 다트 게임

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    [1차] 다트 게임 (프로그래머스 17682)은
    다트 게임의 점수 계산 로직을 구현하는 문자열 처리 문제다.
    총 3번의 기회가 주어지며, 각 기회는 점수(0~10), 보너스(S, D, T), 옵션(*, #)으로 구성된 문자열로 들어온다.

    • 점수: 0에서 10 사이의 정수다. (10은 두 글자임에 유의해야 한다.)
    • 보너스: S(1제곱), D(2제곱), T(3제곱) 중 하나가 점수마다 반드시 존재한다.
    • 옵션: *(스타상: 해당 점수와 바로 전 점수 2배), #(아차상: 해당 점수 마이너스)이 있을 수도 있고 없을 수도 있다.

    전략 개요

    • 문자열 파싱: 문자열을 한 글자씩 순회하며 숫자, 영문자(보너스), 특수문자(옵션)를 구분해 처리한다.
    • 점수 저장 배열: * 옵션이 "직전 점수"에도 영향을 주기 때문에, 각 기회의 점수를 저장해둘 배열(크기 3)이 필요하다.
    • Enum 활용: 보너스 계산 로직과 옵션 처리 로직을 각각 Enum으로 분리하여, 메인 로직의 복잡도를 낮추고 가독성을 높인다.

    코드가 하는 일

    • 1) 숫자 파싱과 인덱스 관리 
      if (Character.isDigit(array[i])) {
    idx++; // 새 점수가 나오면 인덱스 증가

    if (array[i] == '1' && i + 1 < array.length && array[i + 1] == '0') {
        score[idx] = 10;
        i++; // '0' 건너뛰기
    } else {
        score[idx] = array[i] - '0';
    }
}
      • 숫자가 등장하면 새로운 기회가 시작된 것이므로 idx를 증가시킨다.
      • 숫자 1 뒤에 0이 있으면 10점으로 처리하고 인덱스를 하나 더 건너뛴다.
    • 2) 보너스 처리 (Enum: BonusType) 
      else if (Character.isAlphabetic(array[i])) {
    score[idx] = BonusType.valueOf(String.valueOf(array[i]))
                          .calculate(score[idx]);
}
      • 문자(S, D, T)가 나오면 BonusType Enum을 통해 해당 문자에 맞는 제곱 연산(Math.pow)을 수행한다.
      • Enum 내부의 calculate 메서드가 계산을 담당한다.
    • 3) 옵션 처리 (Enum: OptionType) 
      else {
    OptionType.of(array[i]).apply(score, idx);
}
      • 특수문자(*, #)가 나오면 OptionType Enum을 찾아서 로직을 적용한다.
      • OptionType은 BiConsumer<int[], Integer>를 사용하여 배열 상태를 직접 수정한다.
      // OptionType의 STAR('*') 구현 로직
score[idx] *= 2;
if (idx > 0) score[idx - 1] *= 2; // 이전 점수가 있으면 함께 2배
    • 4) 최종 점수 합산 
      return Arrays.stream(score).sum();
      • 계산이 끝난 3개의 점수를 모두 더해 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 입력 문자열의 길이를 N이라 할 때, 한 번 순회하며 처리하므로 O(N)이다.
      • 문제 조건상 문자열 길이는 매우 짧으므로 사실상 상수 시간 복잡도에 가깝다.
    • 공간 복잡도
      • 점수를 저장하는 크기 3의 배열과 Enum 객체 등을 사용하므로 O(1)이다.

    문제 2 — [1차] 비밀지도

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    [1차] 비밀지도 (프로그래머스 17681)는
    두 개의 정사각형 배열(지도)을 겹쳐서 얻어지는 최종 비밀지도를 해독하는 문제다.
    각 칸은 공백(" ") 또는 벽("#")으로 이루어져 있으며, 두 지도 중 하나라도 벽이면 최종 지도에서도 벽이 된다.

    • 지도 1과 지도 2는 각각 정수 배열로 주어진다.
    • 각 정수를 이진수로 변환했을 때, 1은 벽(#), 0은 공백( )을 의미한다.
    • 두 지도를 겹치는 규칙은 비트 연산의 OR(|) 연산과 동일하다.

    전략 개요

    • 비트 연산 (OR): 두 정수 배열의 같은 인덱스 값을 OR(|) 연산하면, 둘 중 하나라도 1인 비트는 1이 되고 둘 다 0이어야 0이 된다. 이는 문제의 "벽+공백=벽" 규칙과 정확히 일치한다.
    • 이진수 변환 및 패딩: 연산 결과를 이진수 문자열로 바꾸되, 지도 한 변의 길이 n에 맞춰 앞부분을 0으로 채워야(padding) 한다.
    • 문자열 치환: 완성된 이진수 문자열에서 1을 #으로, 0을 공백으로 변환하여 반환한다.

    코드가 하는 일

    • 1) 비트 OR 연산 및 이진수 포맷팅 
      int filterWall = arr1[i] | arr2[i];

String binary = String.format("%" + n + "s", Integer.toBinaryString(filterWall))
                      .replace(' ', '0');
      • arr1[i] | arr2[i]를 통해 두 지도를 겹친 값을 구한다.
      • Integer.toBinaryString으로 이진수 문자열로 변환한다.
      • String.format("%" + n + "s", ...)를 사용해 길이 n보다 짧은 경우 앞에 공백을 채운다.
      • 그 후 .replace(' ', '0')를 통해 앞의 공백을 0으로 바꿔, "00101" 같은 형태를 완성한다.
    • 2) 출력 형식 변환 (Stream) 
      return Arrays.stream(field)
        .map(s -> s.replace('1', '#')
        .replace('0', ' '))
        .toArray(String[]::new);
      • 완성된 이진수 문자열 배열을 스트림으로 순회한다.
      • 1은 #으로, 0은 (공백)으로 치환하여 최종 정답 배열을 생성한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 지도의 한 변의 길이를 N이라 할 때, 총 N개의 행을 처리한다.
      • 각 행마다 문자열 변환 및 치환(길이 N)이 일어나므로 전체 연산은 O(N^2)이다. (N ≤ 16이므로 매우 빠르다.)
    • 공간 복잡도
      • 결과를 저장할 문자열 배열이 필요하므로 O(N^2) (글자 수 기준)이다.
