    문제 1 — 같은 숫자는 싫어

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    같은 숫자는 싫어 (프로그래머스 12906)는
    배열 arr의 각 원소는 0부터 9까지의 숫자로 이루어져 있는데, 이 중에서 연속적으로 나타나는 숫자는 하나만 남기고 전부 제거하여 남은 수들을 반환하는 문제다.

    • 제거된 후 남은 수들을 반환할 때는 원 배열의 순서를 그대로 유지해야 한다.
    • 단순히 중복을 제거하는 것이 아니라, 연속된 중복만 제거한다는 점이 핵심이다. (예: [1, 3, 0, 1]은 1이 떨어져 있으므로 그대로 유지)

    전략 개요

    • 순차 탐색: 배열을 처음부터 끝까지 순회하면서 "이전 숫자"와 "현재 숫자"를 비교한다.
    • 필터링: 이전 숫자와 다른 경우에만 결과 배열에 추가한다.
    • 배열 크기 조절: 결과 배열의 크기를 미리 알 수 없으므로, 원본 배열 크기만큼 잡아둔 뒤 마지막에 유효한 데이터 개수(idx)만큼만 잘라내서(Copy) 반환한다.

    코드가 하는 일

    • 1) 결과 배열 초기화 및 첫 원소 등록 
      int[] answer = new int[arr.length];

int idx = 0;
answer[idx++] = arr[0];
      • 최악의 경우(중복이 하나도 없는 경우) 결과 배열 크기는 arr.length와 같으므로 미리 넉넉하게 생성한다.
      • 첫 번째 숫자는 비교 대상이 없으므로 무조건 정답 배열에 넣고 시작한다.
    • 2) 연속 중복 체크 및 저장 
      for (int i = 1; i < arr.length; i++) {
    if (arr[i - 1] != arr[i]) {
        answer[idx++] = arr[i];
    }
}
      • 1번 인덱스부터 시작하여 바로 앞의 숫자(arr[i-1])와 현재 숫자(arr[i])를 비교한다.
      • 두 숫자가 다를 때만 answer 배열에 값을 넣고 인덱스(idx)를 증가시킨다.
      • 같으면 연속된 중복이므로 무시하고 넘어간다.
    • 3) 배열 크기 조정 후 반환 
      return Arrays.copyOf(answer, idx);
      • answer 배열은 처음에 크게 잡았으므로 뒤쪽에 0이 채워져 있다.
      • 실제로 값이 들어간 개수 idx만큼만 잘라서 정확한 크기의 배열을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열을 한 번만 순회(for 루프)하므로 O(N)이다. (N은 배열의 길이)
    • 공간 복잡도
      • 결과를 저장할 배열을 입력 크기만큼 할당하므로 O(N)이다.

    문제 2 — 폰켓몬

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    폰켓몬 (프로그래머스 1845)은
    총 N마리의 폰켓몬 중에서 N/2마리를 가져갈 수 있을 때, 선택할 수 있는 폰켓몬의 종류(Type) 개수의 최댓값을 구하는 문제다.

    • 같은 종류의 폰켓몬이 여러 마리 있을 수 있다.
    • 가장 많은 종류를 가져가려 해도, 최대로 가져갈 수 있는 마릿수는 N/2로 정해져 있다.
    • 즉, 전체 종류의 수와 N/2 중 더 작은 값이 가져갈 수 있는 최대 종류 수다.

    전략 개요

    • 해시셋(HashSet) 활용: 중복을 허용하지 않는 Set 자료구조에 모든 폰켓몬을 넣으면, 폰켓몬의 고유한 종류 개수를 쉽게 구할 수 있다.
    • 비교 및 선택:
      • 우리가 최대로 가져갈 수 있는 마릿수는 nums.length / 2이다.
      • 만약 종류 개수가 이보다 많다면, 다 가져갈 수 없으므로 nums.length / 2가 최대 종류 수가 된다.
      • 만약 종류 개수가 이보다 적다면, 아무리 많이 가져가도 존재하는 종류만큼만 가져갈 수 있으므로 set.size()가 답이 된다.
      • 결국 Math.min()으로 두 값 중 작은 것을 선택한다.

    코드가 하는 일

    • 1) 종류 파악 및 선택 한도 계산 
      Set<Integer> set = new HashSet<>();
int available = nums.length / 2;

for (int num : nums) {
    set.add(num);
}
      • available은 가져갈 수 있는 최대 마릿수(N/2)다.
      • 배열을 순회하며 set에 넣음으로써 중복된 번호를 제거하고 고유한 종류만 남긴다.
    • 2) 최댓값 결정 
      return Math.min(available, set.size());
      • '가져갈 수 있는 수'와 '실제 존재하는 종류 수' 중 더 작은 값이 정답이 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 nums를 한 번 순회하며 Set에 추가하므로 O(N)이다.
    • 공간 복잡도
      • 최악의 경우 모든 폰켓몬 종류가 다를 수 있으므로, Set에 저장하기 위해 O(N)이 필요하다.
