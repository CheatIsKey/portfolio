    문제 1 — 예산

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    예산 (프로그래머스 12982)은
    각 부서가 신청한 금액 d와 전체 예산 budget이 주어질 때, 최대 몇 개의 부서에 물품을 지원할 수 있는지 구하는 문제다.

    • 물품을 지원할 때는 신청한 금액 전액을 지원해야 한다. (일부 지원 불가)
    • 최대한 많은 부서를 지원하려면, 신청 금액이 적은 부서부터 지원하는 것이 유리하다.
    • 즉, 적은 금액 순으로 누적 합을 구했을 때 예산을 넘지 않는 최대 개수를 찾으면 된다.

    전략 개요

    • 정렬(Sorting): 신청 금액이 작은 순서대로 처리하기 위해 배열을 오름차순 정렬한다.
    • DP (혹은 누적 합): dp[i]를 "i개의 부서를 지원할 때 드는 최소 비용"으로 정의한다. 정렬된 상태이므로 단순히 앞에서부터 i개를 더한 값이 최소 비용이 된다.
    • 역순 탐색: 가장 많은 부서를 지원하는 경우(배열 끝)부터 확인하여, 예산 내에 들어오는 즉시 그 개수를 반환한다.

    코드가 하는 일

    • 1) 정렬 및 DP 배열 초기화 
      Arrays.sort(d);
int[] dp = new int[d.length + 1];
Arrays.fill(dp, 1_000_000_000);
dp[0] = 0;
      • 금액이 적은 부서부터 선택하기 위해 정렬한다.
      • dp 배열을 큰 값으로 초기화하고, 0개 부서 지원 비용인 dp[0]은 0으로 설정한다.
    • 2) 최소 비용 계산 (누적 합) 
      for (int i = 1; i <= d.length; i++) {
    dp[i] = Math.min(dp[i], dp[i - 1] + d[i - 1]);
}
      • dp[i-1](i-1개 지원 비용)에 현재 가장 작은 남은 요청 d[i-1]을 더해 dp[i]를 갱신한다.
      • 사실상 정렬된 배열의 누적 합(Prefix Sum)을 구하는 과정과 동일하다.
    • 3) 예산 내 최대 부서 개수 찾기 
      for (int i = d.length; i >= 0; i--) {
    if (dp[i] <= budget) {
        return i;
    }
}
      • 최대 개수를 구해야 하므로, 가장 큰 개수(d.length)부터 역순으로 탐색한다.
      • 누적 비용 dp[i]가 budget 이하가 되는 순간의 i가 정답이다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열 정렬에 O(N log N)이 소요된다.
      • DP 테이블 채우기와 탐색은 O(N)이므로 전체는 O(N log N)이다.
    • 공간 복잡도
      • 별도의 dp 배열을 사용하므로 O(N)이다.

    문제 2 — 소수 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    소수 만들기 (프로그래머스 12977)는
    주어진 숫자 배열 nums에 있는 수들 중, 서로 다른 3개를 골라 더했을 때 그 합이 소수(Prime Number)가 되는 경우의 수를 구하는 문제다.

    • 배열에서 3개의 숫자를 선택하는 모든 조합을 고려해야 한다.
    • 선택된 3개 숫자의 합이 1과 자기 자신으로만 나누어떨어지는지(소수인지) 판별해야 한다.
    • 중복된 숫자는 없으며, 3개의 합이 소수가 되는 횟수를 카운트한다.

    전략 개요

    • 완전 탐색 (Brute Force): 입력 배열의 크기(N)가 최대 50으로 매우 작다. 따라서 3중 반복문을 이용해 3개의 숫자를 고르는 모든 경우($_NC_3$)를 직접 확인해도 충분하다.
    • 소수 판별: 합계(sum)에 대해 2부터 √sum까지 나누어 떨어지는 수가 있는지 확인하는 효율적인 판별법을 사용한다.

    코드가 하는 일

    • 1) 3중 반복문을 통한 숫자 3개 선택 
      for (int i = 0; i < nums.length - 2; i++) {
    for (int j = i + 1; j < nums.length - 1; j++) {
        for (int k = j + 1; k < nums.length; k++) {
            int sum = nums[i] + nums[j] + nums[k];
            if (isPrime(sum)) cnt++;
        }
    }
}
      • i < j < k 순서를 유지하며 서로 다른 3개의 인덱스를 선택한다.
      • 선택된 세 숫자의 합 sum을 구하고, 소수인지 확인하여 카운트를 증가시킨다.
    • 2) 소수 판별 함수 (제곱근 최적화) 
      private boolean isPrime(int sum) {
    for (int l = 2; l * l <= sum; l++) {
        if (sum % l == 0) {
            return false;
        }
    }
    return true;
}
      • 어떤 수 N이 소수가 아니라면, N의 약수는 반드시 √N 이하의 범위에 존재한다.
      • 따라서 2부터 √sum까지만 반복하여 나누어 떨어지는지 검사하면 효율적이다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • N개 중 3개를 뽑는 경우의 수 $_NC_3$은 대략 O(N^3)이다. (N ≤ 50)
      • 각 합(최대 3000)에 대해 소수 판별을 수행하며, 이는 O(√M) (M은 합의 최댓값)이 걸린다.
      • 전체 복잡도는 O(N^3 √M)이지만 입력 크기가 작아 매우 빠르게 수행된다.
    • 공간 복잡도
      • 별도의 추가 배열 없이 변수만 사용하므로 O(1)이다.
