    문제 1 — 완전범죄

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    완전범죄 (프로그래머스 389480)는
    여러 개의 물건을 훔칠 때 A가 훔치면 A의 흔적이, B가 훔치면 B의 흔적이 남는 상황에서,
    B의 누적 흔적을 m 미만으로 유지하면서 A의 누적 흔적을 최소화하는 문제다.

    • 각 물건마다 반드시 A 또는 B 중 한 명이 훔쳐야 한다.
    • 최종적으로 구한 A의 최소 흔적이 n 미만이어야 하며, 불가능하거나 n 이상이면 -1을 반환한다.
    • 제약 조건이 있는 최소 비용 문제이므로 DP(다이나믹 프로그래밍)가 적합하다.

    전략 개요

    • DP 상태 정의: dp[i][j]는 i번째 물건까지 고려했을 때, B의 누적 흔적이 j일 때 A의 최소 흔적을 의미한다.
    • 점화식 (Transition):
      • A가 훔치는 경우: B의 흔적 j는 변하지 않고, A의 흔적만 thiefA만큼 증가한다.
      • B가 훔치는 경우: B의 흔적이 j + thiefB로 증가하고, A의 흔적은 그대로다. 단, 증가한 B의 흔적이 m 미만일 때만 가능하다.
    • 초기값을 무한대(INF)로 설정하고, 0번째 단계에서 시작해 바텀업(Bottom-up)으로 테이블을 채운다.

    코드가 하는 일

    • 1) DP 테이블 초기화 
      int[][] dp = new int[info.length + 1][m + 1];
for (int[] ints : dp) Arrays.fill(ints, Integer.MAX_VALUE);
dp[0][0] = 0;
      • 최솟값을 구해야 하므로 Integer.MAX_VALUE로 초기화한다.
      • 시작점(물건 0개, B 흔적 0)의 A 흔적은 0이다.
    • 2) A가 훔치는 경우와 B가 훔치는 경우 갱신 
      // A가 훔치는 경우 (B 흔적 j 유지)
dp[i][j] = Math.min(dp[i][j], dp[i - 1][j] + thiefA);

// B가 훔치는 경우 (B 흔적 j + thiefB 로 증가)
if (j + thiefB < m) {
    dp[i][j + thiefB] = Math.min(dp[i][j + thiefB], dp[i - 1][j]);
}
      • 이전 단계(i-1)에서 도달 가능한 상태(MAX_VALUE 아님)에 대해 두 가지 분기를 모두 고려한다.
      • A 선택: 현재 상태 j 그대로, 값에 thiefA를 더해 최소를 갱신한다.
      • B 선택: 새로운 상태 j + thiefB가 한계 m보다 작을 때만, A 값 변동 없이 갱신한다.
    • 3) 결과 도출 및 조건 확인 
      int answer = Integer.MAX_VALUE;
for (int j = 0; j <= m; j++) {
    answer = Math.min(answer, dp[info.length][j]);
}

return answer < n ? answer : -1;
      • 모든 물건을 고려한 후(info.length), B의 흔적이 가능한 모든 범위(0~m) 중에서 A의 최소 흔적을 찾는다.
      • 그 값이 한계 n보다 작으면 반환하고, 아니면(불가능하거나 한도 초과) -1을 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 물건의 개수를 L, B의 흔적 한도를 M이라 할 때, 2중 반복문을 수행하므로 O(L × M)이다.
    • 공간 복잡도
      • DP 테이블 크기만큼 필요하므로 O(L × M)이다.

    문제 2 — 최댓값과 최솟값

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    최댓값과 최솟값 (프로그래머스 12939)은
    공백으로 구분된 숫자들로 이루어진 문자열 s가 주어질 때, 그중 최솟값과 최댓값을 찾아 "(최솟값) (최댓값)" 형태의 문자열로 반환하는 문제다.

    • 입력 문자열에는 둘 이상의 정수가 공백으로 구분되어 있다.
    • 모든 수를 비교하여 가장 작은 수와 가장 큰 수를 찾아야 한다.
    • 최종 반환 시 숫자 사이에 공백을 한 칸 둔 문자열 포맷을 맞춰야 한다.

    전략 개요

    • 문자열 파싱: StringTokenizer를 사용하여 공백을 기준으로 문자열을 숫자 토큰으로 분리한다. (split보다 메모리 효율이 좋은 경우가 많다.)
    • 최대/최소 갱신:
      • max는 가장 작은 정수(Integer.MIN_VALUE)로, min은 가장 큰 정수(Integer.MAX_VALUE)로 초기화한다.
      • 토큰을 하나씩 읽어 정수로 변환한 뒤, 현재의 max, min과 비교하여 값을 갱신한다.

    코드가 하는 일

    • 1) 토크나이저 및 변수 초기화 
      StringTokenizer st = new StringTokenizer(s);

int max = Integer.MIN_VALUE;
int min = Integer.MAX_VALUE;
      • 문자열 s를 공백 기준으로 자를 준비를 한다.
      • 비교를 위해 max는 가능한 가장 작은 값, min은 가장 큰 값으로 설정해 둔다.
    • 2) 순회 및 대소 비교 
      while (st.hasMoreTokens()) {
    int num = Integer.parseInt(st.nextToken());

    if (num > max) {
        max = num;
    }

    if (num < min) {
        min = num;
    }
}
      • 남은 토큰이 없을 때까지 반복한다.
      • 문자열 토큰을 int로 변환한 뒤, 각각 최댓값과 최솟값인지 확인하고 갱신한다.
    • 3) 결과 문자열 생성 
      return min + " " + max;
      • 구해진 최솟값과 최댓값을 공백으로 이어 붙여 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열에 포함된 숫자의 개수를 N이라 할 때, 모든 토큰을 한 번씩 순회하므로 O(N)이다.
    • 공간 복잡도
      • StringTokenizer가 문자열을 관리하고 토큰을 생성하는 데 입력 문자열 크기에 비례하는 공간을 사용하므로 O(N)이다. (문자열 길이 기준)
