    문제 1 — 서버 증설 횟수

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    서버 증설 횟수 (프로그래머스 389479)는
    시간대별 이용자 수(players)가 주어질 때, 어느 시간대에 이용자 수가 m명 늘어날 때마다 서버를 1대씩 증설해야 하며, 증설된 서버는 k시간 동안 유지된 후 반납된다고 할 때 총 증설 횟수를 구하는 문제다.

    • 이용자 수가 m명 미만이면 증설이 필요 없다.
    • m명 이상 2m명 미만이면 1대, n×m명 이상이면 n대의 서버가 필요하다.
    • 증설된 서버는 현재 시간 i부터 i + k - 1까지 유효하며, i + k 시간에 만료된다.

    전략 개요

    • 시뮬레이션 & 카운팅: 0시부터 마지막 시간까지 순차적으로 진행하며 현재 필요한 서버 수와 활성화된 서버 수를 비교한다.
    • 만료 시간 관리: 증설된 서버가 언제 사라지는지 기록하기 위해 expire 배열을 사용한다.
    • 현재 상태 유지: active 변수에 현재 운영 중인 서버 개수를 유지하며, 시간이 지남에 따라 만료된 서버는 빼고 새로 증설된 서버는 더한다.

    코드가 하는 일

    • 1) 만료 서버 처리 및 필요 서버 계산 
      active -= expire[i];
int need = players[i] / m;
      • expire[i]에는 시간 i에 만료되는 서버의 수가 저장되어 있으므로, 이를 active에서 뺀다.
      • players[i] / m을 통해 현재 접속자 수 기준으로 필요한 총 증설 서버 수(need)를 구한다.
    • 2) 서버 증설 및 만료 예약 
      if (active < need) {
    int cnt = need - active;

    active += cnt;
    answer += cnt;

    expire[i + k] += cnt;
}
      • 현재 활성화된 서버(active)가 필요한 서버(need)보다 적다면, 부족한 만큼(cnt) 추가한다.
      • active와 총 횟수(answer)를 갱신한다.
      • 지금 증설한 서버 cnt대는 k시간 뒤인 i + k에 만료되므로 expire 배열에 기록해 둔다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • `players` 배열의 길이만큼 한 번 순회하므로 O(N)이다. (N은 시간대 길이)
    • 공간 복잡도
      • 만료 시간을 기록하는 고정 크기(50)의 배열을 사용하므로 O(1)이다. (단, N과 k가 커질 경우 O(N+K)가 필요하다.)

    문제 2 — JadenCase 문자열 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    JadenCase 문자열 만들기 (프로그래머스 12951)는
    문자열의 모든 단어에서 첫 문자는 대문자로, 그 외의 문자는 소문자로 변환하여 반환하는 문제다.

    • 첫 문자가 알파벳이 아닌 경우(숫자 등)는 그대로 둔다.
    • 문자열 중간에 공백이 연속해서 나올 수 있음을 고려해야 한다.
    • 단어의 시작인지 아닌지 판단하는 기준은 "직전 문자가 공백인지" 여부다.

    전략 개요

    • 순차 탐색: 문자열을 앞에서부터 한 글자씩 확인하며 대소문자 변환을 결정한다.
    • 상태 추적: 이전 문자(prev)가 공백(' ')이었다면, 현재 문자는 단어의 시작이므로 대문자로 변환한다. 그 외의 경우는 단어의 중간이므로 소문자로 변환한다.
    • 첫 글자 처리: 반복문 시작 전, 문자열의 맨 첫 글자는 무조건 단어의 시작으로 간주하여 대문자로 처리한다.

    코드가 하는 일

    • 1) 첫 글자 처리 및 초기화 
      char[] arr = s.toCharArray();
char prev = arr[0];

sb.append(Character.toUpperCase(arr[0]));
      • 문자열을 다루기 쉽게 문자 배열로 변환한다.
      • 첫 번째 문자는 항상 단어의 시작이므로 toUpperCase를 적용해 StringBuilder에 담는다.
      • prev 변수에 첫 글자를 저장하여 루프에서 참조할 수 있게 한다.
    • 2) 나머지 문자 순회 및 변환 
      for (int i = 1; i < arr.length; i++) {
    char c = arr[i];

    if (prev == ' ') {
        sb.append(Character.toUpperCase(c));
    } else {
        sb.append(Character.toLowerCase(c));
    }

    prev = c;
}
      • 두 번째 문자(인덱스 1)부터 끝까지 순회한다.
      • prev == ' '이면 현재 문자 c는 새 단어의 시작이므로 대문자로 변환한다.
      • 그렇지 않다면(이전 문자가 공백이 아님) 단어 내부이므로 소문자로 변환한다.
      • 처리가 끝난 현재 문자 c를 prev로 업데이트하여 다음 반복에서 활용한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이만큼 한 번 순회하므로 O(N)이다.
    • 공간 복잡도
      • 결과 문자열을 생성하기 위한 StringBuilder와 char 배열을 사용하므로 O(N)이다.
