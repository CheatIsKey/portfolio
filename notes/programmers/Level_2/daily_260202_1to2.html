    문제 1 — 지게차와 크레인

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    지게차와 크레인 (프로그래머스 388353)은
    창고에 쌓인 2차원 격자 형태의 컨테이너들을 정해진 규칙에 따라 출고할 때,
    최종적으로 남은 컨테이너의 개수를 구하는 시뮬레이션 문제다.

    • 크레인 (길이 2, 예: "AA"): 해당 알파벳을 가진 모든 컨테이너를 꺼낸다. 위치와 상관없이 전부 제거된다.
    • 지게차 (길이 1, 예: "A"): 해당 알파벳을 가진 컨테이너 중 외부와 연결된 것만 꺼낼 수 있다. 내부에 갇힌 건 꺼낼 수 없다.
    • 외부와 연결되었다는 것은 상하좌우 인접한 칸을 통해 창고 밖으로 나갈 수 있음을 의미한다.

    전략 개요

    • 패딩 (Padding): 2차원 배열의 테두리를 빈 공간('0')으로 감싸 확장한다. 이렇게 하면 (0, 0)에서 시작하는 탐색으로 "외부 공간"을 명확히 정의할 수 있다.
    • BFS (외부 공기 판별): 지게차 요청이 올 때마다 BFS를 수행하여, 현재 시점에서 외부에서 접근 가능한 빈 공간을 visited 배열에 표시한다.
    • 조건별 삭제 구현:
      • 크레인 요청은 전체 순회하며 무조건 삭제한다.
      • 지게차 요청은 컨테이너의 상하좌우 중 하나라도 visited(외부 공간)라면 삭제한다.

    코드가 하는 일

    • 1) 배열 확장 및 초기화 
      char[][] boxes = new char[row + 2][col + 2];
for (char[] box : boxes) Arrays.fill(box, '0');

for (int r = 0; r < row; r++) {
    for (int c = 0; c < col; c++) {
        boxes[r + 1][c + 1] = storage[r].charAt(c);
    }
}
      • 원본 크기보다 상하좌우 1칸씩 큰 배열을 만들고 '0'(빈 공간)으로 채운다.
      • 중앙에 원본 데이터를 복사하여, 테두리가 자연스럽게 외부 공간 역할을 하도록 만든다.
    • 2) 요청 처리: 크레인 (길이 2) 
      if (request.length() == 2) {
    for (int r = 0; r < boxes.length; r++) {
        for (int c = 0; c < boxes[r].length; c++) {
            if (boxes[r][c] == remove) {
                boxes[r][c] = '0';
            }
        }
    }
}
      • 요청 문자열 길이가 2인 경우, 위치나 외부 접촉 여부를 따지지 않고 해당 문자와 일치하는 모든 컨테이너를 '0'으로 바꾼다.
    • 3) 요청 처리: 지게차 (길이 1) 
      else {
    boolean[][] visited = new boolean[row + 2][col + 2];
    checkOutside(boxes, visited, row + 2, col + 2); // 외부 공간 탐색

    for (int r = 1; r <= row; r++) {
        for (int c = 1; c <= col; c++) {
            if (boxes[r][c] == remove) {
                // 상하좌우 중 하나라도 외부(visited)와 닿아있으면 제거
                for (int k = 0; k < 4; k++) {
                    if (visited[r + ROW[k]][c + COL[k]]) {
                        boxes[r][c] = '0';
                        break;
                    }
                }
            }
        }
    }
}
      • checkOutside 메서드로 (0, 0)부터 시작하는 BFS를 돌려 외부와 연결된 빈 공간을 visited에 마킹한다.
      • 그 후, 해당 문자인 컨테이너를 찾고, 4방향 중 한 곳이라도 visited(외부)라면 제거('0')한다.
    • 4) 남은 컨테이너 개수 세기 
      int cnt = 0;
for (int r = 1; r <= row; r++) {
    for (int c = 1; c <= col; c++) {
        if (boxes[r][c] != '0') cnt++;
    }
}
      • 모든 요청 처리 후 '0'이 아닌 칸의 개수를 세어 반환한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 요청의 개수를 Q, 맵의 크기를 R × C라 할 때, 각 요청마다 BFS 또는 전체 순회를 수행한다.
      • BFS와 순회 모두 O(R × C)이므로 전체 시간 복잡도는 O(Q × R × C)이다.
      • R, C가 최대 50 정도로 작기 때문에 충분히 빠르다.
    • 공간 복잡도
      • 확장된 맵과 방문 배열을 위해 O(R × C) 공간을 사용한다.

    문제 2 — 올바른 괄호

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    올바른 괄호 (프로그래머스 12909)는
    괄호 문자열 s가 올바르게 짝지어진 문자열(VPS)인지 확인하여 참(true) 또는 거짓(false)을 반환하는 문제다.

    • 모든 여는 괄호 (는 반드시 대응되는 닫는 괄호 )와 짝을 이뤄야 한다.
    • 닫는 괄호가 나올 때 짝이 맞는 여는 괄호가 없다면 올바르지 않다.
    • 모든 처리가 끝난 후 스택에 괄호가 남아있다면 올바르지 않다.

    전략 개요

    • 스택(Stack) 활용: 짝 맞추기 문제의 전형적인 해법으로 스택을 사용한다. 여기서는 자바의 ArrayDeque를 사용하여 스택처럼 동작하게 한다.
    • 순차 탐색:
      • (가 나오면 스택에 넣는다(push).
      • )가 나오면 스택에서 뺀다(pop). 이때 스택이 비어있으면 짝이 안 맞는 것이므로 즉시 false를 반환한다.
    • 최종 검증: 문자열을 모두 확인한 뒤 스택이 텅 비어있어야(true) 올바른 괄호다.

    코드가 하는 일

    • 1) 큐(Deque) 초기화 및 여는 괄호 처리 
      ArrayDeque<Character> deque = new ArrayDeque<>();

for (char c : s.toCharArray()) {
    if (c == '(') {
        deque.push(c);
    }
      • ArrayDeque를 생성하여 스택 용도로 사용한다.
      • 문자열을 순회하며 (를 만나면 push로 저장한다.
    • 2) 닫는 괄호 처리 및 예외 확인 
          else {
        if (deque.isEmpty()) {
            return false;
        }
        deque.pop();
    }
}
      • )를 만났을 때, 스택이 비어있다면 짝이 맞는 (가 없다는 뜻이므로 false를 반환한다.
      • 비어있지 않다면 pop()을 수행하여 짝을 맞춘 (를 제거한다.
    • 3) 최종 결과 반환 
      return deque.isEmpty();
      • 모든 과정을 마친 뒤 스택에 잔여물이 없어야 올바른 괄호 문자열이다.
      • 남아있다면 여는 괄호가 더 많은 경우이므로 false가 된다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 문자열 길이 N만큼 한 번 순회하므로 O(N)이다.
    • 공간 복잡도
      • 최악의 경우(모두 여는 괄호일 때) 스택에 N개의 문자가 쌓이므로 O(N)이다.
