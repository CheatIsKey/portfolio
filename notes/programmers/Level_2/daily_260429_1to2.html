    문제 1 — 비밀 코드 해독

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    비밀 코드의 해독 (프로그래머스 388352)은
    1부터 n까지의 정수 중 5개의 서로 다른 정수로 구성된 '비밀 코드'를 찾기 위해,
    여러 번의 시도(q)와 그에 따른 일치 개수(ans) 정보를 바탕으로 가능한 비밀 코드의 경우의 수를 구하는 문제다.

    • 비밀 코드는 5개의 정수로 이루어져 있으며, 중복되지 않는다.
    • 주어진 힌트(시도한 조합과 맞춘 개수)를 모두 만족하는 조합만 유효한 비밀 코드다.
    • n의 범위가 크지 않으므로(최대 40 정도 예상), 가능한 모든 5개 조합을 만들어 검사하는 완전 탐색이 가능하다.

    전략 개요

    • 조합 (Combination): 1부터 n까지의 숫자 중에서 순서 없이 5개를 뽑는 모든 경우의 수를 구한다. (백트래킹 활용)
    • 검증 (Validation): 생성된 5개의 조합이 주어진 q(시도)와 ans(결과) 배열의 조건과 일치하는지 확인한다.
    • 조건 일치 확인: 생성된 조합과 각 시도(q[i])의 교집합 개수를 세고, 이것이 ans[i]와 같은지 비교한다. 하나라도 다르면 탈락이다.

    코드가 하는 일

    • 1) 조합 생성 (재귀 호출) 
      private void combination(int cnt, int start, int[] numbers) {
    if (cnt == 5) {
        if (check(numbers)) {
            answer++;
        }
        return;
    }

    for (int i = start; i <= N; i++) {
        numbers[cnt] = i;
        combination(cnt + 1, i + 1, numbers);
    }
}
      • cnt는 현재까지 뽑은 숫자의 개수, start는 탐색 시작 번호다.
      • 5개를 모두 뽑으면(cnt == 5) check() 메서드를 호출해 조건을 만족하는지 확인한다.
      • 중복을 방지하기 위해 start부터 탐색하며, 다음 재귀 호출 시 i + 1을 넘긴다.
    • 2) 조건 검증 (Check 함수) 
      private boolean check(int[] numbers) {
    for (int i = 0; i < Q.length; i++) {
        int matchCnt = 0;

        for (int num1 : numbers) {
            for (int num2 : Q[i]) {
                if (num1 == num2) matchCnt++;
            }
        }

        if (matchCnt != Ans[i]) {
            return false;
        }
    }
    return true;
}
      • 현재 만들어진 조합(numbers)과 주어진 힌트(Q, Ans)를 비교한다.
      • 각 시도(Q[i])마다 일치하는 숫자의 개수(matchCnt)를 센다.
      • 계산된 개수가 실제 정답 개수(Ans[i])와 다르면, 이 조합은 비밀 코드가 될 수 없으므로 false를 반환한다.
    • 3) 정답 카운팅 
      if (check(numbers)) {
    answer++;
}
      • check() 함수가 true를 반환하면, 모든 힌트를 만족하는 경우이므로 answer를 1 증가시킨다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 1부터 N까지 중 5개를 뽑는 조합의 수는 $_NC_5$이다.
      • 각 조합마다 M개의 쿼리를 확인하며, 각 쿼리당 5×5=25번 비교하므로 검증은 O(M)이다.
      • 전체 복잡도는 O(_NC_5 × M)이다. (N이 40 이하일 때 약 65만×M 정도로, 제한 시간 내 충분하다.)
    • 공간 복잡도
      • 재귀 호출의 깊이는 5이므로 스택 메모리는 O(1)에 가깝다.
      • 입력 데이터를 저장하는 배열 외 추가 메모리는 미미하다.

    문제 2 — 최솟값 만들기

    문제 링크

    프로그래머스 문제 보기

    코드 (IDE 스크린샷)

    해설

    최솟값 만들기 (프로그래머스 12941)은
    길이가 같은 두 배열 A, B에서 각각 숫자를 하나씩 뽑아 곱한 뒤 더했을 때, 그 누적 합이 최소가 되도록 만드는 문제다.

    • 각 배열의 원소는 한 번씩만 사용해야 한다.
    • 곱의 합을 최소로 만들기 위해서는 가장 작은 수와 가장 큰 수를 짝지어 곱해야 한다.
    • 큰 수는 큰 수끼리 곱하면 값이 급격히 커지므로, 최대한 작은 수와 곱해서 상쇄시켜야 한다.

    전략 개요

    • 정렬(Sorting): 두 배열을 모두 오름차순으로 정렬한다.
    • 그리디(Greedy):
      • 배열 A는 앞에서부터(작은 값 순서), 배열 B는 뒤에서부터(큰 값 순서) 접근한다.
      • 즉, A의 최솟값 × B의 최댓값 형태로 짝을 지어 더하면 전체 합이 최소가 된다.

    코드가 하는 일

    • 1) 배열 정렬 
      Arrays.sort(A);
Arrays.sort(B);
      • 두 배열을 오름차순으로 정렬하여 크기 순서대로 배치한다.
    • 2) 교차 곱셈 및 누적 합 계산 
      int sum = 0;

for (int i = 0; i < A.length; i++) {
    sum += A[i] * B[B.length - i - 1];
}
      • 배열 A는 0번 인덱스(가장 작은 값)부터, 배열 B는 length - 1번 인덱스(가장 큰 값)부터 접근한다.
      • 인덱스 i가 증가할수록 A는 커지고 B는 작아지므로, 항상 (최소 × 최대) 조합이 유지된다.
      • 곱한 값을 sum에 누적하여 최종 정답을 구한다.

    시간 · 공간 복잡도

    • 시간 복잡도
      • 배열의 길이를 N이라 할 때, 정렬에 O(N log N)이 소요된다.
      • 반복문은 O(N)이므로 전체 시간 복잡도는 O(N log N)이다.
    • 공간 복잡도
      • 별도의 추가 배열 없이 변수 하나만 사용하므로 O(1)이다. (정렬 내부 공간 제외)
