    템플릿 — Interceptor (커스텀 인터셉터)

    분류: 템플릿, Spring Boot, Interceptor, 로깅, 인가

    이 템플릿의 구성 커스텀 어노테이션(@AdminLogging) + 인터셉터(CustomInterceptor) + WebMvc 등록(PersistenceConfig) 3개 파일로 구성된다.
    특정 경로에 대해 관리자 권한 검증 + 접근 로그 기록을 수행하는 구조다.

    1. 커스텀 어노테이션

    인터셉터를 메서드 단위로 선택적으로 적용하고 싶을 때 사용한다. 이 템플릿에서는 @AdminLogging을 마커(marker) 용도로 활용한다. 단순히 인터셉터 경로 패턴(addPathPatterns)만으로 제어한다면 이 어노테이션 없이도 동작한다.

    ⚠️ 수정 필요 어노테이션 이름(AdminLogging)을 프로젝트 목적에 맞게 변경한다. 메서드 레벨이 아니라 클래스 레벨에도 적용하려면 @Target에 ElementType.TYPE을 추가한다. 
    @Target(ElementType.METHOD)          // 메서드에만 적용 가능
@Retention(RetentionPolicy.RUNTIME)  // 런타임까지 어노테이션 정보 유지 (인터셉터가 런타임에 읽음)
public @interface AdminLogging {
    // 필요 시 속성 추가 가능 — 예: String value() default "";
}

    2. 커스텀 인터셉터

    HandlerInterceptor를 구현하고 preHandle()에서 요청을 가로챈다. preHandle()이 false를 반환하면 이후 컨트롤러 로직이 실행되지 않는다.

    ⚠️ 수정 필요 excludeRequest 조건의 HTTP 메서드와 경로 패턴을 프로젝트 요구사항에 맞게 교체한다.
    request.getAttribute("userId")와 request.getAttribute("userRole")은 JWT 필터 등 앞단에서 request.setAttribute()로 값을 넣어두어야 정상 동작한다.
    UserRole.of()는 프로젝트의 실제 역할 변환 메서드로 대체한다. 
    @Slf4j
@Component
public class CustomInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler)
            throws Exception {

        String method = request.getMethod();      // 요청 HTTP 메서드 (GET, POST, DELETE 등)
        String path   = request.getRequestURI();  // 요청 경로 (/admin/comments/1 등)

        // 수정 포인트: 로깅/검증을 적용할 경로 조건 — 해당하지 않으면 바로 통과
        boolean excludeRequest =
                (method.equals("DELETE") && path.matches("/admin/comments/\\d+")) ||
                (method.equals("PATCH")  && path.matches("/admin/users/\\d+"));

        if (!excludeRequest) {
            return true;  // 대상 경로가 아니면 다음 단계로 통과
        }

        // 수정 포인트: JWT 필터 등 앞단에서 setAttribute()로 넣어둔 값을 꺼낸다.
        Long     userId = (Long)   request.getAttribute("userId");
        UserRole role   = UserRole.of((String) request.getAttribute("userRole"));

        // 관리자 권한 검증 — 권한이 없으면 403 응답 후 요청 중단
        if (role == null || !UserRole.ADMIN.equals(role)) {
            response.sendError(HttpServletResponse.SC_FORBIDDEN, "접근 권한이 없습니다.");
            return false;
        }

        // 관리자 접근 로그 기록
        log.info("[관리자 기능 접근] userId = {}, requestTime = {}, url = {}",
                userId, LocalDateTime.now(), path);

        return true;  // 검증 통과 → 컨트롤러 실행 허용
    }
}

    실행 결과 (권한 있는 관리자가 DELETE /admin/comments/1 요청 시)

    // console [관리자 기능 접근] userId = 1, requestTime = 2026-03-05T10:42:40.690139500, url = /admin/comments/1

    실행 결과 (권한 없는 일반 사용자가 요청 시)

    // HTTP Response HTTP 403 Forbidden
    { "error": "접근 권한이 없습니다." }

    실행 결과 (excludeRequest 조건에 해당하지 않는 경로 — 예: GET /admin/comments/1)

    // console — 아무것도 출력되지 않고 바로 통과 (로그 없음 — 컨트롤러 정상 실행)

    3. WebMvc 등록 설정

    인터셉터는 만들기만 해서는 동작하지 않는다. WebMvcConfigurer를 구현한 설정 클래스에서 addInterceptors()로 등록해야 한다. addPathPatterns()로 인터셉터가 적용될 경로를, excludePathPatterns()로 제외할 경로를 지정한다.

    ⚠️ 수정 필요 addPathPatterns()의 경로를 프로젝트 실제 경로로 교체한다.
    AuthUserArgumentResolver나 @EnableJpaAuditing이 없는 프로젝트라면 해당 줄을 제거한다.
    설정 클래스가 이미 있다면 addInterceptors() 메서드만 별도로 오버라이드해서 추가한다. 
    @Configuration
@RequiredArgsConstructor
@EnableJpaAuditing  // JPA Auditing 미사용 시 제거
public class PersistenceConfig implements WebMvcConfigurer {

    // 프로젝트에 ArgumentResolver가 없으면 아래 두 줄 제거
    private final AuthUserArgumentResolver authUserArgumentResolver;
    private final CustomInterceptor customInterceptor;

    @Override
    public void addArgumentResolvers(List<HandlerMethodArgumentResolver> resolvers) {
        resolvers.add(authUserArgumentResolver);  // 불필요 시 메서드 전체 제거
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(customInterceptor)
                .addPathPatterns("/admin/comments/*")  // 인터셉터 적용 경로
                .addPathPatterns("/admin/users/*");    // 인터셉터 적용 경로 추가
                // .excludePathPatterns("/admin/health") // 제외할 경로가 있으면 추가
    }
}

    파일 역할 수정 포인트
    AdminLogging.java 마커 어노테이션 어노테이션 이름, @Target 범위
    CustomInterceptor.java 권한 검증 + 로그 기록 경로 조건, getAttribute 키 이름, 역할 비교 로직
    PersistenceConfig.java 인터셉터 등록 addPathPatterns 경로, 불필요한 Bean 제거
