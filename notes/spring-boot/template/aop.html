    템플릿 — AOP 로깅 (전체 컨트롤러 마스킹 / 어노테이션 기반 / JSON 직렬화)

    분류: 템플릿, Spring Boot, AOP, 로깅, 마스킹, JSON

    이 템플릿의 구성 세 가지 AOP 로깅 케이스를 독립적으로 제공한다. 케이스마다 Pointcut 방식과 파라미터 출력 형태가 다르며, 서로 혼용하지 않는다.

    케이스 1 — 패키지 경로 기반 Pointcut + 민감 파라미터 마스킹 유틸(LogMaskingUtils) 포함
    케이스 2 — 커스텀 어노테이션 기반 Pointcut + userId/url 포함 관리자 로그 (문자열 출력)
    케이스 3 — 커스텀 어노테이션 기반 Pointcut + 요청/응답 데이터를 JSON으로 직렬화해서 출력

    케이스 1. 전체 컨트롤러 로깅 + 민감 정보 마스킹

    특정 패키지 하위의 모든 컨트롤러 메서드 호출을 자동으로 기록한다. password, token 등 민감한 파라미터 이름이 포함된 경우 값을 ****로 치환해 로그에 노출되지 않도록 처리한다.

    LogMaskingUtils

    파라미터 이름이 민감 키워드를 포함하는지 검사하고, 해당되면 값을 ****로 대체한다. 민감 키워드 목록(SENSITIVE_KEYS)은 프로젝트 요구사항에 따라 자유롭게 추가한다.

    수정 포인트 SENSITIVE_KEYS에 프로젝트에서 마스킹이 필요한 파라미터 이름을 추가한다.
    현재는 파라미터 이름 기준으로 검사한다. DTO 내부 필드까지 마스킹하려면 리플렉션 또는 직렬화 후 치환 방식으로 확장해야 한다. 
    public class LogMaskingUtils {

    // 수정 포인트: 마스킹할 민감 파라미터 이름 목록 — 소문자 포함 여부로 검사
    private static final Set<String> SENSITIVE_KEYS = Set.of(
            "password", "passwd", "pw",
            "token", "accessToken", "refreshToken",
            "secret", "secretKey",
            "creditCard", "cardNumber", "cvv"
    );

    /**
     * 파라미터 이름 배열과 값 배열을 받아 [이름=값] 형태의 문자열로 변환한다.
     * 민감 키워드를 포함하는 이름은 값을 ****로 마스킹한다.
     */
    public static String buildMaskedParamsString(String[] parameterNames, Object[] args) {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < parameterNames.length; i++) {
            if (i > 0) sb.append(", ");
            String name  = parameterNames[i];
            Object value = args[i];
            // 민감 키워드 포함 여부 검사 (대소문자 무시)
            if (isSensitive(name)) {
                sb.append(name).append("=****");
            } else {
                sb.append(name).append("=").append(value);
            }
        }
        return sb.toString();
    }

    // 파라미터 이름에 민감 키워드가 포함되어 있는지 확인
    private static boolean isSensitive(String name) {
        String lower = name.toLowerCase();
        return SENSITIVE_KEYS.stream()
                .anyMatch(key -> lower.contains(key.toLowerCase()));
    }
}

    ApiLoggingAspect

    수정 포인트 execution() 안의 패키지 경로를 프로젝트 실제 컨트롤러 패키지로 교체한다.
    로그 메시지 포맷([API 요청], [API 응답])을 프로젝트 로그 컨벤션에 맞게 변경한다. 
    @Slf4j
@Aspect     // AOP 모듈 선언
@Component  // 스프링 빈으로 등록
public class ApiLoggingAspect {

    // 수정 포인트: 적용할 컨트롤러 패키지 경로
    @Around("execution(* com.example.instagramclone.controller.rest.*.*(..))")
    public Object logApi(ProceedingJoinPoint joinPoint) throws Throwable {

        // 호출된 클래스명과 메서드명 추출
        String className  = joinPoint.getTarget().getClass().getSimpleName();
        String methodName = joinPoint.getSignature().getName();

        // 파라미터 이름 배열과 값 배열 추출
        MethodSignature signature      = (MethodSignature) joinPoint.getSignature();
        String[]        parameterNames = signature.getParameterNames();
        Object[]        args           = joinPoint.getArgs();

        // 민감 파라미터를 마스킹한 파라미터 문자열 생성
        String paramsString = LogMaskingUtils.buildMaskedParamsString(parameterNames, args);

        log.info("[API 요청] {} / {} | 파라미터: [{}]", className, methodName, paramsString);

        long startTime = System.currentTimeMillis();

        Object result = joinPoint.proceed();  // 원래 컨트롤러 메서드 실행

        long executionTime = System.currentTimeMillis() - startTime;

        log.info("[API 응답] {} / {} (수행시간: {}ms)", className, methodName, executionTime);

        return result;
    }
}

    케이스 2. 어노테이션 기반 관리자 로깅 (문자열 출력)

    @AdminLogging 어노테이션이 붙은 메서드에만 선택적으로 AOP를 적용한다. 요청한 관리자의 userId와 접근 URL을 함께 기록한다. 파라미터는 [이름=값] 형태의 문자열로 출력한다. 앞단(JWT 필터 등)에서 request.setAttribute("userId", ...)로 값을 넣어두어야 정상 동작한다.

    수정 포인트 @annotation() 안의 패키지 경로를 실제 @AdminLogging 어노테이션의 전체 경로로 교체한다.
    request.getAttribute("userId") 키 이름은 JWT 필터에서 setAttribute()로 넣은 키와 반드시 일치해야 한다.
    RequestContextHolder.getRequestAttributes()가 null을 반환하는 비-HTTP 환경(스케줄러 등)에서는 NPE가 발생한다. 해당 케이스가 있다면 null 체크를 추가한다. 
    @Aspect
@Component
@Slf4j
public class Logger {

    // 수정 포인트: @AdminLogging 어노테이션의 실제 전체 패키지 경로
    @Around("@annotation(org.example.expert.config.AdminLogging)")
    public Object executionLogger(ProceedingJoinPoint joinPoint) throws Throwable {

        // 호출된 클래스명과 메서드명 추출
        String className  = joinPoint.getTarget().getClass().getSimpleName();
        String methodName = joinPoint.getSignature().getName();

        // 파라미터 이름 배열과 값 배열 추출
        MethodSignature signature      = (MethodSignature) joinPoint.getSignature();
        String[]        parameterNames = signature.getParameterNames();
        Object[]        args           = joinPoint.getArgs();

        String paramsString = buildParamsString(parameterNames, args);

        // 수정 포인트: JWT 필터에서 setAttribute()로 넣은 키 이름과 일치해야 한다.
        HttpServletRequest request =
                ((ServletRequestAttributes) RequestContextHolder.getRequestAttributes()).getRequest();
        Long   userId = (Long)   request.getAttribute("userId");
        String url    = request.getRequestURI();

        log.info("[API 요청] userId = {}, url = {}, {}.{} | 파라미터: [{}]",
                userId, url, className, methodName, paramsString);

        long start = System.currentTimeMillis();

        Object proceed = joinPoint.proceed();  // 원래 메서드 실행

        long end = System.currentTimeMillis();

        log.info("[API 응답] userId = {}, {}.{} (수행시간: {}ms)",
                userId, className, methodName, (end - start));

        return proceed;
    }

    // 파라미터를 [이름=값] 형태로 이어 붙이는 유틸 메서드 (마스킹 없음)
    private String buildParamsString(String[] parameterNames, Object[] args) {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < parameterNames.length; i++) {
            if (i > 0) sb.append(", ");
            sb.append(parameterNames[i]).append("=").append(args[i]);
        }
        return sb.toString();
    }
}

    케이스 3. 어노테이션 기반 관리자 로깅 (JSON 직렬화 출력)

    케이스 2와 동일하게 @AdminLogging 어노테이션 기반으로 동작하지만, 파라미터를 문자열로 이어 붙이는 대신 ObjectMapper로 JSON 직렬화해서 출력한다. 응답 객체(proceed)도 JSON으로 직렬화하여 응답 로그에 포함시킨다. 파라미터 구조가 복잡한 DTO일 때 케이스 2보다 훨씬 가독성 좋은 로그를 남길 수 있다.

    수정 포인트 @annotation() 안의 패키지 경로를 실제 @AdminLogging 어노테이션의 전체 경로로 교체한다.
    ObjectMapper를 필드로 직접 생성(new ObjectMapper())하지 않고, 스프링 컨텍스트에 등록된 @Autowired ObjectMapper를 주입받는 방식을 권장한다. 프로젝트의 Jackson 설정(날짜 포맷, 직렬화 옵션 등)이 자동으로 반영되기 때문이다.
    응답이 void인 메서드는 proceed가 null을 반환하므로 응답 로그에 null이 찍힌다. 정상 동작이다. 
    @Aspect
@Component
@Slf4j
public class Logger {

    // 수정 포인트: new ObjectMapper() 대신 @Autowired로 주입받는 방식을 권장한다.
    // 프로젝트의 Jackson 설정(날짜 포맷 등)을 그대로 반영할 수 있다.
    private final ObjectMapper mapper = new ObjectMapper();

    // 수정 포인트: @AdminLogging 어노테이션의 실제 전체 패키지 경로
    @Around("@annotation(org.example.expert.config.AdminLogging)")
    public Object executionLogger(ProceedingJoinPoint joinPoint) throws Throwable {

        // 호출된 클래스명과 메서드명 추출
        String className  = joinPoint.getTarget().getClass().getSimpleName();
        String methodName = joinPoint.getSignature().getName();

        // 파라미터 이름 배열과 값 배열 추출
        MethodSignature signature      = (MethodSignature) joinPoint.getSignature();
        String[]        parameterNames = signature.getParameterNames();
        Object[]        args           = joinPoint.getArgs();

        // 파라미터를 LinkedHashMap에 담아 순서를 유지한 채 JSON 직렬화
        Map<String, Object> paramsMap = new LinkedHashMap<>();
        for (int i = 0; i < parameterNames.length; i++) {
            paramsMap.put(parameterNames[i], args[i]);
        }

        // 수정 포인트: JWT 필터에서 setAttribute()로 넣은 키 이름과 일치해야 한다.
        HttpServletRequest request =
                ((ServletRequestAttributes) RequestContextHolder.getRequestAttributes()).getRequest();
        Long   userId = (Long)   request.getAttribute("userId");
        String url    = request.getRequestURI();

        // 파라미터 Map을 JSON 문자열로 직렬화
        String paramsJson = mapper.writeValueAsString(paramsMap);

        log.info("[API 요청] userId = {}, url = {}, {}.{} | 요청: [{}]",
                userId, url, className, methodName, paramsJson);

        long start = System.currentTimeMillis();

        Object proceed = joinPoint.proceed();  // 원래 메서드 실행

        // 응답 객체를 JSON 직렬화 (void 메서드는 proceed = null → "null" 출력)
        String responseJson = mapper.writeValueAsString(proceed);

        long end = System.currentTimeMillis();

        log.info("[API 응답] userId = {}, {}.{} | 응답: {} (수행시간: {}ms)",
                userId, className, methodName, responseJson, (end - start));

        return proceed;
    }
}

    파일 구성 요약

    케이스 파일 역할 수정 포인트
    1 LogMaskingUtils.java 민감 파라미터 마스킹 유틸 SENSITIVE_KEYS 목록 추가/제거
    1 ApiLoggingAspect.java 패키지 경로 기반 전체 컨트롤러 로깅 execution() 패키지 경로 교체
    2 Logger.java (문자열) 어노테이션 기반 관리자 로깅 (문자열) @annotation() 경로, getAttribute 키 이름
    3 Logger.java (JSON) 어노테이션 기반 관리자 로깅 (JSON) @annotation() 경로, ObjectMapper 주입 방식
    케이스별 핵심 차이 Pointcut 방식: 케이스 1은 패키지 경로 기반(execution())으로 해당 패키지의 모든 메서드에 자동 적용된다. 케이스 2, 3은 어노테이션 기반(@annotation())으로 @AdminLogging이 붙은 메서드에만 선택적으로 적용된다.

    파라미터 출력 방식: 케이스 1, 2는 [이름=값] 문자열로 출력한다. 케이스 3은 ObjectMapper로 JSON 직렬화하여 출력한다. DTO 내부 구조가 복잡할수록 케이스 3이 가독성이 높다.

    응답 로그: 케이스 1, 2는 응답 객체를 로그에 포함하지 않는다. 케이스 3은 응답 객체도 JSON으로 직렬화해서 기록한다. 응답이 void이면 null이 출력되는데, 이는 정상 동작이다.
