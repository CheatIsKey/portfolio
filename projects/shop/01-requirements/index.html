핵심 기능 요구

  • 회원가입/로그인/로그아웃, 마이페이지(주문 내역). 세션 고정 방지 및 내비게이션 분기.
  • 카테고리/키워드 기반 검색, 목록/상세, 이미지 업로드, 관리자 상품/카테고리 관리.
  • 장바구니 담기/변경/삭제/비우기(재고 초과 자동 보정, Flash 메시지).
  • 주문 생성/목록/상세/취소(배송 완료 전 취소 허용), 본인 주문만 조회.
  • 고객 문의: 작성/조회/답변/종결, 관리자 조회.

비즈니스 규칙(발췌)

  • 장바구니: 담긴 수량 + 추가 수량 ≤ 재고, 0 이하면 품목 삭제, 초과 시 자동 보정 + 메시지.
  • 주문: 배송 상태가 COMP면 취소 불가. 총액은 도메인 메서드로 계산.
  • POST 성공 후 반드시 Redirect. 뷰에서 지연 컬렉션 직접 순회 금지.
  • 조회: 목록/상세에 맞춘 fetch 전략(요약: 배송만, 상세: 배송+아이템+상품).

품질/운영 요구

  • JPA N+1 방지(EntityGraph/Fetch Join), Repository 메서드는 파생 쿼리 한계 고려.
  • 권한 분리(ADMIN/USER), 인터셉터로 접근 제어, 세션에는 DTO만 저장.
  • Flyway 마이그레이션으로 스키마 정합성 유지, 레거시 컬럼 정리.
  • Cloud Build/Run 배포 파이프라인, Cloud SQL, GCS(이미지) 연동.
